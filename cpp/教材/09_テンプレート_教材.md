# 第9章：テンプレート

> **この章のゴール**: C++のテンプレート（template）の仕組みを理解し、関数テンプレート・クラステンプレート・テンプレートの特殊化・可変引数テンプレート・`if constexpr` を使いこなせるようになることを目指します。

---

## 9.1 テンプレートとは何か

テンプレート（template）とは、**型をパラメータとして受け取る仕組み**です。同じロジックを異なる型に対して使い回すことができ、コードの重複を大幅に削減できます。

```
┌─────────────────────────────────────────────┐
│            テンプレートの考え方              │
├─────────────────────────────────────────────┤
│                                             │
│   テンプレート（型の「型紙」）              │
│   ┌───────────────┐                         │
│   │ template<T>   │                         │
│   │ T add(T a, b) │                         │
│   └───────┬───────┘                         │
│           │                                 │
│     ┌─────┼─────┐                           │
│     ▼     ▼     ▼                           │
│   int   double  string   ← 型を当てはめる  │
│                                             │
│   コンパイラが型ごとに関数を自動生成        │
└─────────────────────────────────────────────┘
```

テンプレートを使わない場合、型ごとに関数を書く必要があります。

```cpp
// テンプレートなし：型ごとに別の関数が必要
int add_int(int a, int b) { return a + b; }
double add_double(double a, double b) { return a + b; }
```

テンプレートを使えば、1つの定義で済みます。

---

## 9.2 関数テンプレート（Function Template）

### 基本構文

`template<typename T>` を関数の前に書くことで、型 `T` をパラメータ化できます。

```cpp
// 09_function_template_basic.cpp
// コンパイル: g++ -std=c++17 -o func_tmpl 09_function_template_basic.cpp
#include <iostream>
#include <string>

// 関数テンプレートの定義
template<typename T>
T add(T a, T b) {
    return a + b;
}

int main() {
    // int 型で使用
    std::cout << "int:    " << add(3, 5) << std::endl;

    // double 型で使用
    std::cout << "double: " << add(1.5, 2.3) << std::endl;

    // std::string 型で使用
    std::cout << "string: " << add(std::string("Hello, "), std::string("World!")) << std::endl;

    return 0;
}
```

**出力例:**

```
int:    8
double: 3.8
string: Hello, World!
```

### テンプレートの型推論（Template Argument Deduction）

コンパイラは引数の型から `T` を自動的に推論（deduction）します。明示的に型を指定することもできます。

```cpp
// 型推論に任せる（推奨）
add(3, 5);           // T = int と推論

// 明示的に型を指定する
add<double>(3, 5);   // T = double を指定（3と5がdoubleに変換される）
```

### 複数の型パラメータ

```cpp
// 09_multi_param_template.cpp
#include <iostream>

template<typename T, typename U>
auto multiply(T a, U b) {
    return a * b;   // 戻り値の型はautoで推論（C++14以降）
}

int main() {
    // int × double → double
    std::cout << multiply(3, 2.5) << std::endl;   // 7.5

    // double × int → double
    std::cout << multiply(1.5, 4) << std::endl;   // 6

    return 0;
}
```

### ポイントまとめ

| 項目 | 説明 |
|---|---|
| `template<typename T>` | 型パラメータ `T` を宣言 |
| 型推論 | 引数から `T` を自動決定 |
| `typename` と `class` | テンプレートパラメータではどちらも同じ意味 |
| `auto` 戻り値 | C++14以降、戻り値の型を自動推論 |

## 9.3 クラステンプレート（Class Template）

クラスにもテンプレートを適用できます。STL（Standard Template Library）の `std::vector<T>` や `std::map<K, V>` もクラステンプレートです。

### 基本的なクラステンプレート

```cpp
// 09_class_template.cpp
// コンパイル: g++ -std=c++17 -o class_tmpl 09_class_template.cpp
#include <iostream>
#include <stdexcept>

// 固定サイズのスタック（stack）をテンプレートで実装
template<typename T, int MaxSize = 10>
class Stack {
private:
    T data_[MaxSize];
    int top_;

public:
    Stack() : top_(0) {}

    void push(const T& value) {
        if (top_ >= MaxSize) {
            throw std::overflow_error("スタックが満杯です");
        }
        data_[top_++] = value;
    }

    T pop() {
        if (top_ <= 0) {
            throw std::underflow_error("スタックが空です");
        }
        return data_[--top_];
    }

    T peek() const {
        if (top_ <= 0) {
            throw std::underflow_error("スタックが空です");
        }
        return data_[top_ - 1];
    }

    bool empty() const { return top_ == 0; }
    int size() const { return top_; }
};

int main() {
    // int型のスタック（最大5要素）
    Stack<int, 5> intStack;
    intStack.push(10);
    intStack.push(20);
    intStack.push(30);

    std::cout << "=== int スタック ===" << std::endl;
    while (!intStack.empty()) {
        std::cout << intStack.pop() << std::endl;
    }

    // string型のスタック（デフォルトサイズ10）
    Stack<std::string> strStack;
    strStack.push("Apple");
    strStack.push("Banana");
    strStack.push("Cherry");

    std::cout << "\n=== string スタック ===" << std::endl;
    while (!strStack.empty()) {
        std::cout << strStack.pop() << std::endl;
    }

    return 0;
}
```

**出力例:**

```
=== int スタック ===
30
20
10

=== string スタック ===
Cherry
Banana
Apple
```

```
┌───────────────────────────────────────┐
│    クラステンプレートの実体化         │
├───────────────────────────────────────┤
│                                       │
│  Stack<int, 5>    → コンパイラが      │
│                     int版を生成       │
│                                       │
│  Stack<string>    → コンパイラが      │
│                     string版を生成    │
│                                       │
│  Stack<double, 3> → コンパイラが      │
│                     double版を生成    │
│                                       │
│  ※ 使われた型の組み合わせだけ生成    │
│    （使わない型の分は生成されない）   │
└───────────────────────────────────────┘
```

### 非型テンプレートパラメータ（Non-Type Template Parameter）

上の `MaxSize` のように、型ではなく**値**をテンプレートパラメータにできます。

```cpp
// コンパイル時に配列サイズが決まる
template<typename T, int N>
struct Array {
    T data[N];
    constexpr int size() const { return N; }
};

Array<double, 3> point;  // double型 3要素の配列
```

### ポイントまとめ

| 項目 | 説明 |
|---|---|
| `template<typename T>` | クラスの型をパラメータ化 |
| デフォルトテンプレート引数 | `template<typename T, int N = 10>` |
| 非型パラメータ | `int`, `bool` などの値を渡せる |
| 実体化（instantiation） | 使用時にコンパイラが具体的なクラスを生成 |

## 9.4 テンプレートの特殊化（Template Specialization）

特定の型に対して、テンプレートの振る舞いを変えたい場合があります。これを**特殊化**（specialization）と呼びます。

### 完全特殊化（Full Specialization）

```cpp
// 09_specialization.cpp
// コンパイル: g++ -std=c++17 -o special 09_specialization.cpp
#include <iostream>
#include <string>
#include <cstring>

// 汎用テンプレート：2つの値のうち大きい方を返す
template<typename T>
T my_max(T a, T b) {
    return (a > b) ? a : b;
}

// const char* に対する完全特殊化
// （ポインタの比較ではなく、文字列の中身を比較したい）
template<>
const char* my_max<const char*>(const char* a, const char* b) {
    return (std::strcmp(a, b) > 0) ? a : b;
}

int main() {
    std::cout << my_max(10, 20) << std::endl;           // 20
    std::cout << my_max(3.14, 2.72) << std::endl;       // 3.14
    std::cout << my_max("apple", "banana") << std::endl; // banana（文字列比較）

    return 0;
}
```

**出力例:**

```
20
3.14
banana
```

### 部分特殊化（Partial Specialization）

クラステンプレートでは、型パラメータの一部だけを特殊化できます（関数テンプレートでは不可）。

```cpp
// 09_partial_specialization.cpp
// コンパイル: g++ -std=c++17 -o partial 09_partial_specialization.cpp
#include <iostream>

// 汎用テンプレート
template<typename T, typename U>
struct Pair {
    T first;
    U second;
    void print() const {
        std::cout << "汎用: (" << first << ", " << second << ")" << std::endl;
    }
};

// U が int の場合の部分特殊化
template<typename T>
struct Pair<T, int> {
    T first;
    int second;
    void print() const {
        std::cout << "部分特殊化(U=int): (" << first << ", " << second << ")" << std::endl;
    }
};

// 両方 int の場合の完全特殊化
template<>
struct Pair<int, int> {
    int first;
    int second;
    void print() const {
        std::cout << "完全特殊化(int,int): (" << first << ", " << second << ")" << std::endl;
    }
};

int main() {
    Pair<std::string, double> p1{"Hello", 3.14};
    p1.print();  // 汎用

    Pair<double, int> p2{2.5, 42};
    p2.print();  // 部分特殊化

    Pair<int, int> p3{10, 20};
    p3.print();  // 完全特殊化

    return 0;
}
```

**出力例:**

```
汎用: (Hello, 3.14)
部分特殊化(U=int): (2.5, 42)
完全特殊化(int,int): (10, 20)
```

```
┌────────────────────────────────────────────────┐
│        特殊化の優先順位（マッチング）          │
├────────────────────────────────────────────────┤
│                                                │
│  Pair<string, double>                          │
│    → 汎用 Pair<T, U> にマッチ                 │
│                                                │
│  Pair<double, int>                             │
│    → 部分特殊化 Pair<T, int> にマッチ         │
│                                                │
│  Pair<int, int>                                │
│    → 完全特殊化 Pair<int, int> にマッチ       │
│                                                │
│  ルール: より具体的な特殊化が優先される       │
│  完全特殊化 > 部分特殊化 > 汎用テンプレート   │
└────────────────────────────────────────────────┘
```

### ポイントまとめ

| 種類 | 構文 | 対象 |
|---|---|---|
| 完全特殊化 | `template<>` | 関数・クラス |
| 部分特殊化 | `template<typename T>` で一部を固定 | クラスのみ |

## 9.5 可変引数テンプレート（Variadic Templates）

C++11で導入された可変引数テンプレート（variadic templates）を使うと、**任意の個数・任意の型**の引数を受け取る関数やクラスを定義できます。`std::tuple` や `std::make_unique` の内部でも使われている重要な機能です。

### パラメータパック（Parameter Pack）

```
┌─────────────────────────────────────────────┐
│       パラメータパックの展開イメージ        │
├─────────────────────────────────────────────┤
│                                             │
│  print(1, "hello", 3.14)                    │
│                                             │
│  呼び出し1: head=1,     rest=("hello",3.14) │
│  呼び出し2: head="hello", rest=(3.14)       │
│  呼び出し3: head=3.14,  rest=()             │
│  呼び出し4: 引数なし → ベースケース        │
│                                             │
│  再帰的に1つずつ引数を処理していく         │
└─────────────────────────────────────────────┘
```

### 再帰による展開

```cpp
// 09_variadic_template.cpp
// コンパイル: g++ -std=c++17 -o variadic 09_variadic_template.cpp
#include <iostream>
#include <string>

// ベースケース（引数なし）
void print() {
    std::cout << std::endl;
}

// 再帰テンプレート：先頭の引数を処理し、残りを再帰呼び出し
template<typename T, typename... Args>
void print(const T& first, const Args&... rest) {
    std::cout << first;
    if constexpr (sizeof...(rest) > 0) {
        std::cout << ", ";
    }
    print(rest...);
}

int main() {
    print(1, "hello", 3.14, true);
    print("名前", "太郎", "年齢", 25);
    print(42);

    return 0;
}
```

**出力例:**

```
1, hello, 3.14, 1
名前, 太郎, 年齢, 25
42
```

### 折りたたみ式（Fold Expression）— C++17

C++17では**折りたたみ式**（fold expression）により、再帰なしでパラメータパックを展開できます。

```cpp
// 09_fold_expression.cpp
// コンパイル: g++ -std=c++17 -o fold 09_fold_expression.cpp
#include <iostream>
#include <string>

// 全引数の合計を返す（左折りたたみ）
template<typename... Args>
auto sum(Args... args) {
    return (args + ...);  // 左折りたたみ: ((a1 + a2) + a3) + ...
}

// 全引数を出力する（カンマ演算子の折りたたみ）
template<typename... Args>
void print_all(const Args&... args) {
    ((std::cout << args << " "), ...);
    std::cout << std::endl;
}

// 全引数が正であるかチェック
template<typename... Args>
bool all_positive(Args... args) {
    return (... && (args > 0));  // 左折りたたみ: (a1 > 0) && (a2 > 0) && ...
}

int main() {
    std::cout << "sum(1,2,3,4,5) = " << sum(1, 2, 3, 4, 5) << std::endl;
    std::cout << "sum(1.1, 2.2)  = " << sum(1.1, 2.2) << std::endl;

    print_all("C++", 17, "is", "great!");

    std::cout << "all_positive(1,2,3)   = " << std::boolalpha
              << all_positive(1, 2, 3) << std::endl;
    std::cout << "all_positive(1,-2,3)  = " << std::boolalpha
              << all_positive(1, -2, 3) << std::endl;

    return 0;
}
```

**出力例:**

```
sum(1,2,3,4,5) = 15
sum(1.1, 2.2)  = 3.3
C++ 17 is great!
all_positive(1,2,3)   = true
all_positive(1,-2,3)  = false
```

### 折りたたみ式の4パターン

| パターン | 構文 | 展開結果 |
|---|---|---|
| 単項左折りたたみ | `(... op pack)` | `((a1 op a2) op a3)` |
| 単項右折りたたみ | `(pack op ...)` | `(a1 op (a2 op a3))` |
| 二項左折りたたみ | `(init op ... op pack)` | `((init op a1) op a2)` |
| 二項右折りたたみ | `(pack op ... op init)` | `(a1 op (a2 op init))` |

### ポイントまとめ

| 項目 | 説明 |
|---|---|
| `typename... Args` | テンプレートパラメータパック |
| `Args... args` | 関数パラメータパック |
| `sizeof...(Args)` | パック内の要素数を取得 |
| `args...` | パックの展開 |
| 折りたたみ式 | C++17で追加、再帰不要でパック展開 |

## 9.6 if constexpr — コンパイル時の条件分岐

C++17で導入された `if constexpr` は、**コンパイル時に条件を評価**して、不要な分岐を完全に除去します。テンプレートプログラミングで非常に強力な機能です。

### 型に応じた処理の切り替え

```cpp
// 09_if_constexpr.cpp
// コンパイル: g++ -std=c++17 -o constexpr_if 09_if_constexpr.cpp
#include <iostream>
#include <string>
#include <type_traits>
#include <vector>
#include <numeric>

// 型に応じて異なる処理を行う汎用的な変換関数
template<typename T>
std::string to_description(const T& value) {
    if constexpr (std::is_integral_v<T>) {
        // 整数型の場合
        return "整数: " + std::to_string(value);
    } else if constexpr (std::is_floating_point_v<T>) {
        // 浮動小数点型の場合
        return "小数: " + std::to_string(value);
    } else if constexpr (std::is_same_v<T, std::string>) {
        // std::string型の場合
        return "文字列: " + value;
    } else {
        // それ以外
        return "不明な型";
    }
}

// コンテナかどうかで処理を分ける
template<typename T>
void process(const T& value) {
    if constexpr (std::is_arithmetic_v<T>) {
        std::cout << "数値を2倍: " << value * 2 << std::endl;
    } else {
        std::cout << "そのまま出力: " << value << std::endl;
    }
}

int main() {
    std::cout << to_description(42) << std::endl;
    std::cout << to_description(3.14) << std::endl;
    std::cout << to_description(std::string("テンプレート")) << std::endl;

    std::cout << std::endl;

    process(21);
    process(1.5);
    process(std::string("Hello"));

    return 0;
}
```

**出力例:**

```
整数: 42
小数: 3.140000
文字列: テンプレート

数値を2倍: 42
数値を2倍: 3
そのまま出力: Hello
```

### if constexpr と通常の if の違い

```
┌────────────────────────────────────────────────────┐
│         if constexpr  vs  通常の if                │
├────────────────────────────────────────────────────┤
│                                                    │
│  通常の if:                                        │
│    ・実行時に条件を評価                            │
│    ・両方の分岐がコンパイルされる                  │
│    ・型が合わないコードがあるとエラー              │
│                                                    │
│  if constexpr:                                     │
│    ・コンパイル時に条件を評価                      │
│    ・条件が false の分岐は完全に除去される         │
│    ・型が合わないコードでもエラーにならない        │
│                                                    │
│  例: T が string のとき value * 2 は不正だが、     │
│      if constexpr なら除去されてエラーにならない   │
└────────────────────────────────────────────────────┘
```

### 可変引数テンプレートとの組み合わせ

```cpp
// 09_constexpr_variadic.cpp
// コンパイル: g++ -std=c++17 -o constexpr_var 09_constexpr_variadic.cpp
#include <iostream>
#include <type_traits>

// if constexpr を使って再帰の終了条件を記述
template<typename T, typename... Rest>
void type_info(const T& first, const Rest&... rest) {
    if constexpr (std::is_integral_v<T>) {
        std::cout << first << " は整数型です" << std::endl;
    } else if constexpr (std::is_floating_point_v<T>) {
        std::cout << first << " は浮動小数点型です" << std::endl;
    } else {
        std::cout << "\"" << first << "\" はその他の型です" << std::endl;
    }

    // 残りの引数があれば再帰
    if constexpr (sizeof...(rest) > 0) {
        type_info(rest...);
    }
}

int main() {
    type_info(42, 3.14, "hello", 100L, 2.0f);
    return 0;
}
```

**出力例:**

```
42 は整数型です
3.14 は浮動小数点型です
"hello" はその他の型です
100 は整数型です
2 は浮動小数点型です
```

### ポイントまとめ

| 項目 | 説明 |
|---|---|
| `if constexpr` | コンパイル時に条件を評価し分岐を除去 |
| `std::is_integral_v<T>` | `T` が整数型かどうか |
| `std::is_floating_point_v<T>` | `T` が浮動小数点型かどうか |
| `std::is_same_v<T, U>` | `T` と `U` が同じ型かどうか |
| `std::is_arithmetic_v<T>` | `T` が算術型（整数または浮動小数点）かどうか |

## 9.7 実践例：型安全なフォーマッタ

これまで学んだテンプレートの知識を組み合わせた実践的な例を紹介します。

```cpp
// 09_practical_formatter.cpp
// コンパイル: g++ -std=c++17 -o formatter 09_practical_formatter.cpp
#include <iostream>
#include <string>
#include <sstream>
#include <type_traits>
#include <iomanip>

// 型に応じたフォーマット処理
template<typename T>
std::string format_value(const T& value) {
    std::ostringstream oss;
    if constexpr (std::is_floating_point_v<T>) {
        oss << std::fixed << std::setprecision(2) << value;
    } else if constexpr (std::is_same_v<T, bool>) {
        oss << (value ? "true" : "false");
    } else {
        oss << value;
    }
    return oss.str();
}

// 可変引数テンプレートで複数の値をフォーマット
template<typename... Args>
std::string format_line(const std::string& separator, const Args&... args) {
    std::string result;
    bool first = true;
    // 折りたたみ式で各引数を処理
    ((result += (first ? (first = false, "") : separator) + format_value(args)), ...);
    return result;
}

int main() {
    // CSVライクな出力
    std::cout << "=== CSVフォーマット ===" << std::endl;
    std::cout << format_line(", ", "太郎", 25, 175.5, true) << std::endl;
    std::cout << format_line(", ", "花子", 30, 162.3, false) << std::endl;

    // タブ区切り出力
    std::cout << "\n=== タブ区切り ===" << std::endl;
    std::cout << format_line("\t", "商品A", 1980, 1.08) << std::endl;
    std::cout << format_line("\t", "商品B", 3500, 1.10) << std::endl;

    return 0;
}
```

**出力例:**

```
=== CSVフォーマット ===
太郎, 25, 175.50, true
花子, 30, 162.30, false

=== タブ区切り ===
```

## 9.8 よくある間違い

### 間違い1: テンプレートの定義をソースファイルに分離する

```cpp
// NG: ヘッダとソースを分離するとリンクエラーになる
// mytemplate.h
template<typename T>
T add(T a, T b);

// mytemplate.cpp
template<typename T>
T add(T a, T b) { return a + b; }  // 他のファイルから見えない！

// OK: テンプレートの定義はヘッダファイルに書く
// mytemplate.h
template<typename T>
T add(T a, T b) {
    return a + b;  // 宣言と定義を同じヘッダに
}
```

テンプレートはコンパイル時に型ごとのコードを生成するため、**定義がヘッダファイルにある必要があります**。

### 間違い2: 型推論の曖昧さ

```cpp
template<typename T>
T add(T a, T b) { return a + b; }

// NG: int と double で T の推論が曖昧
// add(1, 2.0);  // コンパイルエラー！ T=int? T=double?

// OK: 明示的に型を指定する
add<double>(1, 2.0);  // T=double と明示
```

### 間違い3: if constexpr を使うべき場面で通常の if を使う

```cpp
template<typename T>
void process(T value) {
    // NG: 通常の if だと、T=string のとき value * 2 がコンパイルエラー
    // if (std::is_arithmetic_v<T>) {
    //     std::cout << value * 2;
    // }

    // OK: if constexpr なら T=string のとき分岐が除去される
    if constexpr (std::is_arithmetic_v<T>) {
        std::cout << value * 2;
    }
}
```

### 間違い4: パラメータパックの展開忘れ

```cpp
template<typename... Args>
void foo(Args... args) {
    // NG: ... を忘れている
    // bar(args);

    // OK: パックを展開する
    bar(args...);
}
```
