# 実践課題03：パーミッション設定演習 ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第2章（ファイルとディレクトリ）、第3章（パーミッションとユーザー管理）
> **課題の種類**: ミニプロジェクト
> **学習目標**: ファイルとディレクトリのパーミッションを正しく理解し、適切に設定できるようになる

---

## 完成イメージ

Webサーバーのような環境を想定し、適切なパーミッションを設定したディレクトリ構造を作ります。

```
webapp/
├── public/          drwxr-xr-x  (755) ← 誰でも読めるが、書き込みは所有者のみ
│   ├── index.html   -rw-r--r--  (644) ← 誰でも読めるが、編集は所有者のみ
│   └── style.css    -rw-r--r--  (644)
├── scripts/         drwxr-xr-x  (755)
│   └── deploy.sh    -rwxr-xr--  (754) ← 所有者は実行可、グループも実行可
├── config/          drwxr-x---  (750) ← グループまで読めるが、他者はアクセス不可
│   └── database.conf -rw-r-----  (640) ← グループまで読めるが、他者は読めない
├── secrets/         drwx------  (700) ← 所有者しかアクセスできない
│   └── api_key.txt  -rw-------  (600) ← 所有者しか読み書きできない
└── logs/            drwxrwxr-x  (775) ← グループメンバーも書き込み可
    └── access.log   -rw-rw-r--  (664)
```

---

## 課題の要件

1. 上記のディレクトリ構造を作成する
2. 各ファイル・ディレクトリに指定のパーミッションを設定する
3. `ls -la` で設定結果を確認する
4. パーミッションの意味を理解し、なぜその設定が適切かを説明できる

---

## ステップガイド

<details>
<summary>ステップ1：パーミッションの読み方を確認する</summary>

パーミッション（permission）は3桁の数値で表現されます。

```
  所有者(u)  グループ(g)  その他(o)
  ┌──┴──┐   ┌──┴──┐    ┌──┴──┐
    rwx        r-x         r--
    421        4_1         4__
    = 7        = 5         = 4

  → 754
```

| 数値 | 権限 | 意味 |
|:---:|:---:|------|
| 4 | r (read) | 読み取り |
| 2 | w (write) | 書き込み |
| 1 | x (execute) | 実行（ディレクトリの場合はアクセス） |

よく使われるパターン：
- `755`: 所有者はフル権限、他は読み取りと実行のみ
- `644`: 所有者は読み書き、他は読み取りのみ
- `700`: 所有者のみフル権限
- `600`: 所有者のみ読み書き

</details>

<details>
<summary>ステップ2：ディレクトリとファイルを作成する</summary>

```bash
# ディレクトリ作成
mkdir -p webapp/{public,scripts,config,secrets,logs}

# ファイル作成
echo '<html><body>Hello</body></html>' > webapp/public/index.html

echo "[$(date)] Server started" > webapp/logs/access.log
```

<details>
<summary>ステップ3：パーミッションを設定する</summary>

`chmod` コマンドで数値指定（8進数表記）を使います。

```bash
# ディレクトリのパーミッション
chmod 755 webapp/public
chmod 755 webapp/scripts
chmod 750 webapp/config
chmod 700 webapp/secrets
chmod 775 webapp/logs

# ファイルのパーミッション
chmod 644 webapp/public/index.html
chmod 644 webapp/public/style.css
chmod 754 webapp/scripts/deploy.sh
chmod 640 webapp/config/database.conf
chmod 600 webapp/secrets/api_key.txt
chmod 664 webapp/logs/access.log
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

```bash
#!/bin/bash
# パーミッション設定演習
# 学べる内容：chmod、パーミッションの数値表現、セキュリティの基本
# 実行方法：bash setup_permissions.sh

# --- ディレクトリ作成 ---
mkdir -p webapp/public
mkdir -p webapp/scripts
mkdir -p webapp/config
mkdir -p webapp/secrets
mkdir -p webapp/logs

# --- ファイル作成 ---
echo '<html><body><h1>Welcome</h1></body></html>' > webapp/public/index.html
echo 'body { font-family: sans-serif; color: #333; }' > webapp/public/style.css

cat > webapp/scripts/deploy.sh << 'EOF'
#!/bin/bash
echo "=== デプロイ開始 ==="
echo "ファイルをコピー中..."
echo "=== デプロイ完了 ==="
EOF

cat > webapp/config/database.conf << 'EOF'
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_USER=appuser
EOF

echo "API_KEY=sk-secret-12345-abcde" > webapp/secrets/api_key.txt

echo "[$(date '+%Y-%m-%d %H:%M:%S')] Server started" > webapp/logs/access.log

# --- パーミッション設定 ---
# 公開ディレクトリ：誰でも読める
chmod 755 webapp/public
chmod 644 webapp/public/index.html
chmod 644 webapp/public/style.css

# スクリプト：所有者とグループが実行可能
chmod 755 webapp/scripts
chmod 754 webapp/scripts/deploy.sh

# 設定ファイル：グループまで読める、他者は不可
chmod 750 webapp/config
chmod 640 webapp/config/database.conf

# 秘密情報：所有者のみ
chmod 700 webapp/secrets
chmod 600 webapp/secrets/api_key.txt

# ログ：グループも書き込み可
chmod 775 webapp/logs
chmod 664 webapp/logs/access.log

# --- 確認 ---
echo "=== パーミッション確認 ==="
echo ""
echo "--- webapp/ ---"
ls -la webapp/
echo ""
echo "--- webapp/public/ ---"
ls -la webapp/public/
echo ""
echo "--- webapp/scripts/ ---"
ls -la webapp/scripts/
echo ""
echo "--- webapp/config/ ---"
ls -la webapp/config/
echo ""
echo "--- webapp/secrets/ ---"
ls -la webapp/secrets/
echo ""
echo "--- webapp/logs/ ---"
ls -la webapp/logs/
```

<details>
<summary>解答例（改良版 ─ シンボリック表記とumask活用）</summary>

```bash
#!/bin/bash
# パーミッション設定演習（改良版）
# 学べる内容：シンボリック表記のchmod、umask、stat
# 実行方法：bash setup_permissions_v2.sh

set -e

WEBAPP="webapp"

# --- 作成前にクリーンアップ ---
if [ -d "$WEBAPP" ]; then
    echo "既存の $WEBAPP を削除します..."
    rm -rf "$WEBAPP"
fi

# --- ディレクトリ・ファイル一括作成 ---
mkdir -p "$WEBAPP"/{public,scripts,config,secrets,logs}

cat > "$WEBAPP/public/index.html" << 'EOF'
<!DOCTYPE html>
<html><head><link rel="stylesheet" href="style.css"></head>
<body><h1>Welcome</h1></body></html>
EOF

echo 'body { font-family: sans-serif; color: #333; }' > "$WEBAPP/public/style.css"

cat > "$WEBAPP/scripts/deploy.sh" << 'SCRIPT'
#!/bin/bash
set -e
echo "=== デプロイ開始: $(date) ==="
echo "ファイルをコピー中..."
echo "=== デプロイ完了 ==="
SCRIPT

cat > "$WEBAPP/config/database.conf" << 'EOF'
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_USER=appuser
EOF

echo "API_KEY=sk-secret-12345-abcde" > "$WEBAPP/secrets/api_key.txt"
echo "[$(date '+%Y-%m-%d %H:%M:%S')] Server started" > "$WEBAPP/logs/access.log"

# --- パーミッション設定（シンボリック表記） ---
# シンボリック表記は「誰に(u/g/o/a)」「何を(+/-/=)」「どの権限(r/w/x)」で指定

# 公開ディレクトリ
chmod u=rwx,go=rx "$WEBAPP/public"       # 755 と同じ
chmod u=rw,go=r "$WEBAPP/public/index.html"  # 644 と同じ
chmod u=rw,go=r "$WEBAPP/public/style.css"

# スクリプト
chmod u=rwx,go=rx "$WEBAPP/scripts"
chmod u=rwx,g=rx,o=r "$WEBAPP/scripts/deploy.sh"  # 754

# 設定ファイル（他者のアクセスを削除）
chmod u=rwx,g=rx,o= "$WEBAPP/config"     # 750
chmod u=rw,g=r,o= "$WEBAPP/config/database.conf"  # 640

# 秘密情報（所有者のみ）
chmod u=rwx,go= "$WEBAPP/secrets"        # 700
chmod u=rw,go= "$WEBAPP/secrets/api_key.txt"  # 600

# ログ（グループ書き込み可）
chmod u=rwx,g=rwx,o=rx "$WEBAPP/logs"    # 775
chmod u=rw,g=rw,o=r "$WEBAPP/logs/access.log"  # 664

# --- 確認（stat コマンドで詳細表示） ---
echo "=== パーミッション一覧 ==="
echo ""
printf "%-40s %-12s %s\n" "パス" "パーミッション" "8進数"
echo "---------------------------------------------------------------"

find "$WEBAPP" | sort | while read -r path; do
    perms=$(stat -c "%A" "$path" 2>/dev/null || stat -f "%Sp" "$path" 2>/dev/null)
    octal=$(stat -c "%a" "$path" 2>/dev/null || stat -f "%OLp" "$path" 2>/dev/null)
    printf "%-40s %-12s %s\n" "$path" "$perms" "$octal"
done

echo ""
echo "=== セキュリティチェック ==="
# 秘密情報が他者に見えないか確認
secret_perms=$(stat -c "%a" "$WEBAPP/secrets/api_key.txt" 2>/dev/null || stat -f "%OLp" "$WEBAPP/secrets/api_key.txt" 2>/dev/null)
if [ "$secret_perms" = "600" ]; then
    echo "OK: api_key.txt は所有者のみアクセス可能（$secret_perms）"
else
    echo "WARNING: api_key.txt のパーミッションが不適切です（$secret_perms）"
fi
```

**初心者向けとの違い:**
- シンボリック表記（`u=rwx,go=rx`）を使用 → 数値より意図が明確
- `stat` コマンドで8進数とシンボリック表記を両方表示
- セキュリティチェック機能を追加 → 重要ファイルのパーミッションを自動検証
- `set -e` でエラー時に即停止

</details>
