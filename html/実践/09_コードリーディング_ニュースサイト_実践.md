## 設問

以下の10問に答えてください。HTMLの読解力が試されます。

### 設問1（理解度：基本）
`<meta property="og:type" content="website">` の役割は何ですか？ `og:type` が `"article"` の場合と何が違いますか？

### 設問2（理解度：基本）
`<link rel="icon" href="favicon.ico">` は何のためのタグですか？ブラウザのどこに反映されますか？

### 設問3（セマンティクス理解）
ヘッダー内の `<form>` に `role="search"` が付いています。これはどのような効果がありますか？なぜ `role` 属性を使うのですか？

### 設問4（セマンティクス理解）
`aria-labelledby="top-news-heading"` と `id="top-news-heading"` の組み合わせは何をしていますか？`aria-label` を使う場合と何が違いますか？

### 設問5（処理追跡）
トップニュースの画像に `loading="eager"` が指定されていますが、最新ニュースの記事には画像がありません。なぜトップニュースだけ `eager` を指定していると思いますか？デフォルトの `loading` 値は何ですか？

### 設問6（構造分析）
このページの見出し階層（h1〜h3）を書き出してください。アウトライン構造は適切ですか？

### 設問7（設計理解）
`<nav>` タグが4箇所で使われています。それぞれの `aria-label` を書き出し、なぜ複数の `<nav>` に異なる `aria-label` を付けるのか説明してください。

### 設問8（アクセシビリティ分析）
検索フォームの `<label>` に `class="visually-hidden"` が付いています。このクラスの目的は何ですか？なぜ `display: none` ではなくCSSで視覚的に隠す必要があるのですか？

### 設問9（改善提案）
サイドバーの「アーカイブ」セクションで `<details>` / `<summary>` が使われています。このHTML要素の特徴は何ですか？JavaScriptなしでどのように動作しますか？

### 設問10（拡張設計）
このニュースサイトに「パンくずリスト（breadcrumb）」を追加するとしたら、どの位置に配置し、どのようなHTMLで記述しますか？WAI-ARIA（Web Accessibility Initiative - Accessible Rich Internet Applications）を使った構造を提案してください。

## 解答例

<details>
<summary>設問1〜5の解答</summary>

### 設問1
OGP（Open Graph Protocol）のメタタグで、このページの種類をSNS等に伝えます。

- `"website"` → Webサイトのトップページとして扱われる
- `"article"` → 記事・ブログ投稿として扱われ、`article:published_time` 等の追加メタタグが有効になる

トップページなので `"website"` が適切です。個別記事ページなら `"article"` を使います。

### 設問2
**ファビコン（favicon）** を指定するタグです。ブラウザの**タブ**、ブックマーク一覧、履歴に小さなアイコンとして表示されます。サイトの視認性を高め、複数タブを開いたときにサイトを区別しやすくなります。

### 設問3
`role="search"` は、このフォームが**検索機能**であることをスクリーンリーダーに伝えるランドマーク（landmark）ロールです。

効果:
- スクリーンリーダーのユーザーが「検索」ランドマークに直接ジャンプできる
- ページの構造を把握しやすくなる

`<form>` だけでは汎用的なフォームとしか認識されないため、`role` で用途を明示します。HTML5では `<search>` 要素も使えますが、まだ対応ブラウザが限られるため `role` を使うのが確実です。

### 設問4
`aria-labelledby` は、ページ上の既存の要素（ここでは `<h2 id="top-news-heading">トップニュース</h2>`）をラベルとして参照します。

```
section の名前 = "トップニュース"（h2の内容をそのまま使う）
```

`aria-label="トップニュース"` と書く場合との違い:
- `aria-labelledby` → 画面に表示されるテキストを再利用。変更時に1箇所だけ修正すれば良い
- `aria-label` → 別テキストを直接指定。表示テキストと異なる名前を付けたいときに使う

同じテキストを使うなら `aria-labelledby` がDRY（Don't Repeat Yourself）です。

### 設問5
トップニュースの画像は**ファーストビュー（最初に画面に表示される領域）** にあるため、`loading="eager"` で即座に読み込みます。

- `loading="eager"` → すぐに読み込み開始（デフォルト）
- `loading="lazy"` → スクロールして画面内に入りそうになったら読み込み開始

デフォルト値は `"eager"` なので、実は明示しなくても同じ動作です。しかし「意図的にeagerにしている」ことをコードで明示する目的で書いています。画面外の画像には `loading="lazy"` を指定してページの読み込み速度を改善するのがベストプラクティスです。

</details>

### 設問7

4つの `<nav>` とそのラベル:

1. `aria-label="メインナビゲーション"` → ヘッダーのカテゴリリンク
2. `aria-label="カテゴリナビゲーション"` → サイドバーのカテゴリ一覧
3. `aria-label="フッターナビゲーション"` → フッターのリンク
4. ラベルなし（ヘッダーの検索フォームは `<nav>` ではない）

実際にラベルが付いている `<nav>` は3つです。

複数の `<nav>` に異なる `aria-label` を付ける理由:
- スクリーンリーダーが「ナビゲーション」と読み上げるとき、どのナビゲーションか区別できない
- `aria-label` でそれぞれに名前を付けると「メインナビゲーション」「フッターナビゲーション」と区別できる

### 設問8
`visually-hidden` は、**視覚的には非表示だがスクリーンリーダーには読み上げられる** CSSクラスです。

典型的な実装:
```css
.visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

`display: none` や `visibility: hidden` を使うと、スクリーンリーダーもそのテキストを無視してしまいます。`visually-hidden` なら:
- 目の見えるユーザー → `placeholder` で入力のヒントが分かる
- スクリーンリーダーのユーザー → `<label>` の「記事を検索」が読み上げられる

### 設問9
`<details>` / `<summary>` は、**JavaScriptなしで折りたたみ/展開ができるHTML標準要素** です。

特徴:
- `<summary>` の部分がクリック可能なトグルボタンになる
- クリックすると `<details>` の中身が展開/折りたたまれる
- ブラウザが三角マークの表示とクリック処理を自動的に行う
- `open` 属性を付けると初期状態で展開される（例: `<details open>`）
- キーボード操作（Enter / Space）にも標準で対応

このニュースサイトでは、過去のアーカイブを年ごとに折りたたんで表示領域を節約しています。

### 設問10
`<main>` の直下、最初の `<section>` の前に配置するのが一般的です。

```html
<main>
  <!-- パンくずリスト -->
  <nav aria-label="パンくずリスト">
    <ol>
      <li><a href="/">ホーム</a></li>
      <li><a href="/ai">AI</a></li>
      <li aria-current="page">Google、次世代AIチップ「Tensor G5」を発表</li>
    </ol>
  </nav>

  <!-- 記事本文... -->
</main>
```

ポイント:
- `<nav>` で囲み、`aria-label="パンくずリスト"` で用途を明示
- `<ol>` で階層の順序を表現（`<ul>` ではなく `<ol>` が適切）
- 最後の項目に `aria-current="page"` を付けて現在のページを明示
- トップページにはパンくずリストは不要（表示するのは個別記事ページ）
