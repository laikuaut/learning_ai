## 基本問題

### 問題1：スラッシュコマンドの理解

以下のスラッシュコマンドと、その機能を正しく組み合わせてください。

| コマンド | 機能 |
|---|---|
| 1. /clear | A. セッションのコストを表示 |
| 2. /compact | B. 会話履歴を完全にクリア |
| 3. /cost | C. 使用するモデルを変更 |
| 4. /model | D. 会話を要約して圧縮 |

**期待される解答：**
```
1-?, 2-?, 3-?, 4-?
```

<details>
<summary>ヒント</summary>

`/clear`は「クリア（消去）」、`/compact`は「コンパクト（圧縮）」という英語の意味を考えてみましょう。

</details>

<details>
<summary>解答例</summary>

```
1-B, 2-D, 3-A, 4-C
```

- `/clear`: 会話履歴を完全にクリアし、新しいセッションとして始めます
- `/compact`: 会話を要約して圧縮し、コンテキストウィンドウの空きを確保します
- `/cost`: 現在のセッションで使用したトークン数と概算コストを表示します
- `/model`: 使用するAIモデル（Opus、Sonnetなど）を切り替えます

</details>

### 問題2：起動オプションの穴埋め

以下の空欄を埋めてください。

1. ワンショットモードで実行するフラグ: `claude （ ① ） "READMEを要約して"`
2. 前回のセッションを再開するフラグ: `claude （ ② ）`
3. モデルを指定して起動するフラグ: `claude （ ③ ） claude-sonnet-4-6`
4. 出力をJSON形式にするフラグ: `claude -p "質問" （ ④ ） json`

**期待される解答：**
```
① ___
② ___
③ ___
④ ___
```

<details>
<summary>ヒント</summary>

- ①はpromptの頭文字です
- ②はresume（再開）という英語です
- ③はmodelの指定です
- ④はoutput-format（出力形式）です

</details>

<details>
<summary>解答例</summary>

```
① -p
② --resume
③ --model
④ --output-format
```

- `-p`はプロンプトを直接指定してワンショット実行するフラグです
- `--resume`は前回のセッションの会話を引き継いで再開します
- `--model`は使用するAIモデルを指定します
- `--output-format`は出力形式（text, json等）を指定します

</details>

### 問題3：権限モードの理解

Claude Codeでツール実行の確認が表示されたとき、以下の選択肢は何を意味しますか？

```
Claude Code が Bash(npm test) を実行しようとしています。
[Y]es / [N]o / [A]lways allow
```

各選択肢の意味を説明してください。

**期待される解答：**
```
Y: ___
N: ___
A: ___
```

<details>
<summary>ヒント</summary>

Yは一時的、Aは永続的な許可の違いに注目してください。

</details>

<details>
<summary>解答例</summary>

```
Y (Yes): 今回のこの操作だけを許可します。次回同じ操作が必要な場合は再度確認されます。
N (No): 今回の操作を拒否します。Claude Codeは別のアプローチを検討します。
A (Always allow): このツール（この場合はBash）の実行を以降自動的に許可します。
  セッション中は同じツールの確認がスキップされます。
```

</details>

### 問題5：キーボードショートカット

以下の操作に使うキーボードショートカットを答えてください。

1. 複数行のプロンプトを入力する際に改行を挿入する
2. 現在の操作をキャンセルする
3. 実行中のタスクを中断する
4. セッションを終了する
5. メッセージを送信する

**期待される解答：**
```
1. ___
2. ___
3. ___
4. ___
5. ___
```

<details>
<summary>ヒント</summary>

通常のターミナル操作のショートカットに近いものが多いです。送信と改行の違いに注意しましょう。

</details>

<details>
<summary>解答例</summary>

```
1. Shift + Enter  （改行を挿入して入力を続ける）
2. Escape         （現在の操作をキャンセル）
3. Ctrl + C       （実行中のタスクを強制中断）
4. Ctrl + D       （セッションを終了して閉じる）
5. Enter          （メッセージを送信）
```

</details>

### 問題6：コンテキスト管理

以下の状況で、どのスラッシュコマンドまたは操作を使うべきか答えてください。

1. 長い会話の後、コンテキストウィンドウの残りが少なくなった。会話の文脈は維持したい。
2. バグ修正が完了し、次にまったく別の機能開発を始めたい。
3. 前日に途中まで作業していたタスクを今日続けたい。
4. 現在のセッションでいくらコストがかかっているか知りたい。

**期待される解答：**
```
1. ___
2. ___
3. ___
4. ___
```

<details>
<summary>ヒント</summary>

- 文脈を維持したい場合と、リセットしたい場合で異なるコマンドを使います
- セッションの再開には起動オプションを使います

</details>

<details>
<summary>解答例</summary>

```
1. /compact
   会話を要約して圧縮し、コンテキストの空きを確保します。
   文脈は要約として保持されるため、話の流れは維持されます。

2. /clear
   会話履歴を完全にクリアします。前のタスクの情報が不要な場合、
   新しいタスクはクリーンな状態で始めるのが効率的です。

3. claude --resume
   前回のセッションを再開します。起動時に --resume フラグを付けることで、
   前回の会話の続きから作業を再開できます。

4. /cost
   現在のセッションで使用したトークン数と概算コストを表示します。
```

</details>

## チャレンジ問題

### 問題7：allowedTools設定の作成

以下の要件を満たす`.claude/settings.json`の権限設定を作成してください。

**要件：**
- ファイルの読み込みと検索は自動許可
- `npm test`と`npm run lint`の実行は自動許可
- ファイルの編集と作成は都度確認
- `rm`コマンドの実行は明示的に拒否

**期待される出力：**
```json
{
  "permissions": {
    "allow": [...],
    "deny": [...]
  }
}
```

<details>
<summary>ヒント</summary>

- 読み込み系ツール: Read, Glob, Grep
- Bashコマンドの許可は `Bash(コマンド名)` 形式で指定します
- `deny`リストで明示的に禁止できます

</details>

<details>
<summary>解答例</summary>

```json
{
  "permissions": {
    "allow": [
      "Read",
      "Glob",
      "Grep",
      "Bash(npm test)",
      "Bash(npm run lint)"
    ],
    "deny": [
      "Bash(rm *)",
      "Bash(rm -rf *)"
    ]
  }
}
```

- `Read`, `Glob`, `Grep`はファイルの読み込み・検索系ツールです
- `Bash(npm test)`のように、特定のコマンドだけを許可できます
- `Edit`と`Write`はallowリストに含めないことで、都度確認が必要になります
- `deny`リストに`rm`関連コマンドを入れることで、明示的に拒否されます

</details>

### 問題8：ワークフロー設計

あなたは新しいWebアプリケーションのバグ修正を担当することになりました。以下の情報を基に、Claude Codeを使った効率的なワークフローを設計してください。

**状況：**
- バグ報告：「ユーザー一覧ページで、名前に日本語が含まれるユーザーが表示されない」
- 技術スタック：React + Express + PostgreSQL
- テストフレームワーク：Jest（フロントエンド）、Mocha（バックエンド）

Claude Codeに対してどのような指示を、どの順番で出すかを5ステップで記述してください。

<details>
<summary>ヒント</summary>

1. まずバグの原因を調査する
2. フロントエンドとバックエンドのどちらが原因か切り分ける
3. 修正を実施する
4. テストで確認する
5. コミットする

各ステップでClaude Codeに出す具体的なプロンプトを考えましょう。

</details>

<details>
<summary>解答例</summary>

```
Step 1: 原因調査
> ユーザー一覧ページで、名前に日本語が含まれるユーザーが
> 表示されないバグがあります。関連するコードを調査してください。
> フロントエンド（React）とバックエンド（Express）の両方を
> 確認してください。

Step 2: 原因の特定
> （調査結果を確認後）
> APIのレスポンスで日本語が正しく返っているか、
> バックエンドのユーザー一覧取得APIを確認してください。
> PostgreSQLのクエリでエンコーディングの問題がないかも
> チェックしてください。

Step 3: 修正の実施
> （原因特定後、例えばAPIのフィルタリングが原因だった場合）
> src/api/users.ts の getUserList 関数で、
> 名前のフィルタリングが ASCII 文字のみを想定しています。
> Unicode文字（日本語含む）に対応するように修正してください。

Step 4: テストの確認
> 修正に対するテストを追加してください。
> 日本語名のユーザーが正しく取得・表示されることを
> バックエンド（Mocha）とフロントエンド（Jest）の
> 両方でテストしてください。
> その後、全テストを実行して既存機能への影響がないか
> 確認してください。

Step 5: コミット
> /commit
```

ポイント：
- 各ステップでClaude Codeの回答を確認してから次に進む
- 原因の切り分けを先に行い、的確な修正を行う
- テストは修正箇所だけでなく、既存テストも実行して回帰確認する

</details>
