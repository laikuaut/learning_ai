# 第3章：公開鍵認証の設定

## この章のゴール
- 公開鍵認証の仕組みを理解する
- ssh-keygenで鍵ペアを生成できるようになる
- 公開鍵をサーバーに登録して鍵認証でログインできるようになる
- 鍵の種類と推奨設定を理解する

---

## 3.1 公開鍵認証とは

公開鍵認証は、**秘密鍵**と**公開鍵**のペアを使った認証方式です。パスワード認証よりも安全で、現場では標準的に使われています。

### パスワード認証 vs 公開鍵認証

```
【パスワード認証】
クライアント → パスワード送信 → サーバーが検証
問題: ブルートフォース攻撃に弱い、パスワード管理が煩雑

【公開鍵認証】
1. 事前準備: 公開鍵をサーバーに登録
2. 接続時:
   クライアント                        サーバー
      |   「ログインしたい」              |
      |--------------------------------->|
      |                                  |
      |   チャレンジ（ランダムデータ）     |
      |<---------------------------------|
      |                                  |
      |   秘密鍵で署名して返す            |
      |--------------------------------->|
      |                                  |
      |   公開鍵で署名を検証 → OK!       |
      |<---------------------------------|

※ 秘密鍵そのものはネットワーク上を流れない！
```

### 公開鍵認証のメリット

| メリット | 説明 |
|---------|------|
| 高セキュリティ | 秘密鍵がなければログインできない |
| ブルートフォース耐性 | パスワード推測攻撃が無効 |
| パスワード不要 | パスフレーズを設定しなければ自動ログイン可能 |
| 鍵ごとの管理 | 接続先ごとに異なる鍵を使い分けられる |
| 自動化に適する | スクリプトやCI/CDでの自動接続に最適 |

## 3.2 鍵の種類

### 主要な鍵アルゴリズム

| アルゴリズム | 鍵長 | 推奨度 | 説明 |
|------------|------|--------|------|
| Ed25519 | 256bit（固定） | **★★★ 推奨** | 高速・安全・鍵が短い |
| RSA | 2048〜4096bit | ★★ 互換性重視 | 広く対応、3072bit以上推奨 |
| ECDSA | 256/384/521bit | ★ 条件付き | 楕円曲線暗号、一部懸念あり |
| DSA | 1024bit（固定） | **× 非推奨** | 脆弱、OpenSSH 7.0以降で無効 |

### Ed25519 を推奨する理由

```
Ed25519の優位性:
1. 速度: RSAより署名生成・検証が高速
2. 安全性: 定数時間で動作し、タイミング攻撃に強い
3. 鍵サイズ: 公開鍵が小さい（RSA 3072bitの1/10以下）
4. シンプル: 鍵長の選択が不要（常に256bit）

Ed25519公開鍵の例（1行で収まる短さ）:
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIBxxx... user@host

RSA 4096bit公開鍵の例（非常に長い）:
ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAACAQ... (数百文字続く)
```

## 3.3 鍵ペアの生成（ssh-keygen）

### Ed25519 鍵の生成（推奨）

```bash
$ ssh-keygen -t ed25519 -C "your_email@example.com"
Generating public/private ed25519 key pair.
Enter file in which to save the key (/home/user/.ssh/id_ed25519):  # Enterで既定値
Enter passphrase (empty for no passphrase):   # パスフレーズを入力（推奨）
Enter same passphrase again:                  # 再入力
Your identification has been saved in /home/user/.ssh/id_ed25519
Your public key has been saved in /home/user/.ssh/id_ed25519.pub
The key fingerprint is:
SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx your_email@example.com
The key's randomart image is:
+--[ED25519 256]--+
|     .o+*=.      |
|      .o=+.o     |
|       .+.+.o    |
|      . .=.+ .   |
|     o .S.= .    |
|    . o.++ +     |
|     ..o..+ .    |
|    .E .oo .     |
|    ..  .o.      |
+----[SHA256]-----+
```

### RSA 鍵の生成（互換性が必要な場合）

```bash
# RSA 4096bit で生成
$ ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

### 生成オプション

| オプション | 説明 | 例 |
|-----------|------|-----|
| `-t` | 鍵の種類 | `-t ed25519` |
| `-b` | ビット長（RSAで使用） | `-b 4096` |
| `-C` | コメント（識別用） | `-C "user@host"` |
| `-f` | 出力ファイルパス | `-f ~/.ssh/mykey` |
| `-N` | パスフレーズ（非対話的） | `-N "passphrase"` |
| `-a` | KDFラウンド数（パスフレーズ保護強度） | `-a 100` |

### 用途別に鍵を分ける（推奨）

```bash
# GitHub用
$ ssh-keygen -t ed25519 -C "github" -f ~/.ssh/id_ed25519_github

# 仕事のサーバー用
$ ssh-keygen -t ed25519 -C "work-server" -f ~/.ssh/id_ed25519_work

# 個人のVPS用
$ ssh-keygen -t ed25519 -C "personal-vps" -f ~/.ssh/id_ed25519_vps
```

```
生成されるファイル:
~/.ssh/
├── id_ed25519_github       ← 秘密鍵（絶対に他人に渡さない）
├── id_ed25519_github.pub   ← 公開鍵（サーバーに登録する）
├── id_ed25519_work
├── id_ed25519_work.pub
├── id_ed25519_vps
└── id_ed25519_vps.pub
```

## 3.4 パスフレーズ

パスフレーズは秘密鍵を保護する「鍵の鍵」です。

### パスフレーズの必要性

```
秘密鍵にパスフレーズを設定すると:

【パスフレーズなし】
秘密鍵が盗まれる → 即座に不正アクセス可能

【パスフレーズあり】
秘密鍵が盗まれる → パスフレーズがないと使えない → 時間稼ぎができる
                   （その間に鍵を無効化できる）
```

### パスフレーズの変更

```bash
# 既存の鍵のパスフレーズを変更
$ ssh-keygen -p -f ~/.ssh/id_ed25519
Enter old passphrase:         # 現在のパスフレーズ
Enter new passphrase:         # 新しいパスフレーズ
Enter same passphrase again:  # 再入力
Your identification has been saved with the new passphrase.
```

> **ポイント**: パスフレーズの入力が煩わしい場合は、ssh-agent（第7章で解説）を使えば、セッション中に一度だけ入力すれば済みます。

## 3.5 鍵ファイルのパーミッション

SSH鍵のパーミッションは厳格に管理する必要があります。パーミッションが緩いと、SSHは鍵の使用を拒否します。

```bash
# 正しいパーミッション設定
$ chmod 700 ~/.ssh               # ディレクトリ: 所有者のみ
$ chmod 600 ~/.ssh/id_ed25519    # 秘密鍵: 所有者のみ読み書き
$ chmod 644 ~/.ssh/id_ed25519.pub # 公開鍵: 所有者は読み書き、他は読み取り
$ chmod 600 ~/.ssh/config        # 設定ファイル: 所有者のみ
$ chmod 600 ~/.ssh/authorized_keys # 認証鍵リスト: 所有者のみ

# 確認
$ ls -la ~/.ssh/
drwx------  2 user user 4096 Jan  1 00:00 .
-rw-------  1 user user  464 Jan  1 00:00 id_ed25519
-rw-r--r--  1 user user  104 Jan  1 00:00 id_ed25519.pub
-rw-------  1 user user  200 Jan  1 00:00 config
-rw-------  1 user user  104 Jan  1 00:00 authorized_keys
```

### パーミッションが原因のエラー

```bash
# パーミッションが緩すぎるとこのエラーが出る
$ ssh user@example.com
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@         WARNING: UNPROTECTED PRIVATE KEY FILE!          @
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
Permissions 0644 for '/home/user/.ssh/id_ed25519' are too open.
It is required that your private key files are NOT accessible by others.
This private key will be ignored.

# 修正
$ chmod 600 ~/.ssh/id_ed25519
```

## 3.6 公開鍵のサーバー登録

### ssh-copy-id を使う方法（推奨）

```bash
# 最も簡単な方法
$ ssh-copy-id user@example.com
/usr/bin/ssh-copy-id: INFO: Source of key(s) to be installed: "/home/user/.ssh/id_ed25519.pub"
/usr/bin/ssh-copy-id: INFO: attempting to log in with the new key(s), to filter out any that are already installed
user@example.com's password:    # パスワードを入力

Number of key(s) added: 1

# 特定の鍵を指定
$ ssh-copy-id -i ~/.ssh/id_ed25519_work.pub user@example.com

# ポート番号を指定
$ ssh-copy-id -p 2222 user@example.com
```

### 手動で登録する方法

```bash
# 1. 公開鍵の内容を確認
$ cat ~/.ssh/id_ed25519.pub
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIBxxx... your_email@example.com

# 2. サーバーにログイン
$ ssh user@example.com

# 3. .ssh ディレクトリとauthorized_keysファイルを作成
user@server:~$ mkdir -p ~/.ssh
user@server:~$ chmod 700 ~/.ssh
user@server:~$ touch ~/.ssh/authorized_keys
user@server:~$ chmod 600 ~/.ssh/authorized_keys

# 4. 公開鍵を追記
user@server:~$ echo "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIBxxx... your_email@example.com" >> ~/.ssh/authorized_keys
```

### ワンライナーでの登録

```bash
# パイプで公開鍵を送り込む
$ cat ~/.ssh/id_ed25519.pub | ssh user@example.com "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

## 3.7 鍵認証でのログイン確認

```bash
# 公開鍵認証で接続（パスワードを聞かれなければ成功）
$ ssh user@example.com
Welcome to Ubuntu 22.04.3 LTS
user@server:~$

# 詳細モードで認証方式を確認
$ ssh -v user@example.com 2>&1 | grep "Authentication succeeded"
debug1: Authentication succeeded (publickey).
```

### トラブルシューティング

```bash
# 公開鍵認証が失敗する場合のチェックリスト

# 1. ローカル側のパーミッション確認
$ ls -la ~/.ssh/
# id_ed25519 が 600 であること

# 2. リモート側のパーミッション確認
$ ssh user@example.com "ls -la ~/.ssh/"
# .ssh が 700、authorized_keys が 600 であること

# 3. リモート側のホームディレクトリのパーミッション確認
$ ssh user@example.com "ls -ld ~"
# 他者に書き込み権限がないこと（755 or 700）

# 4. サーバーの認証ログを確認（root権限が必要）
$ ssh root@example.com "tail /var/log/auth.log"

# 5. sshd設定で公開鍵認証が有効か確認
$ ssh root@example.com "grep PubkeyAuthentication /etc/ssh/sshd_config"
PubkeyAuthentication yes
```

## 3.8 鍵のフィンガープリント

フィンガープリント（指紋）は、鍵を識別するための短い文字列です。

```bash
# 公開鍵のフィンガープリントを表示
$ ssh-keygen -l -f ~/.ssh/id_ed25519.pub
256 SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx your_email@example.com (ED25519)

# MD5形式で表示（古い形式）
$ ssh-keygen -l -f ~/.ssh/id_ed25519.pub -E md5
256 MD5:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx your_email@example.com (ED25519)

# ランダムアートを表示
$ ssh-keygen -lv -f ~/.ssh/id_ed25519.pub
256 SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx your_email@example.com (ED25519)
+--[ED25519 256]--+
|     .o+*=.      |
|      .o=+.o     |
...
+----[SHA256]-----+
```

## 3.9 GitHubへの公開鍵登録

GitHub でSSH接続を設定する手順です。

```bash
# 1. 鍵を生成
$ ssh-keygen -t ed25519 -C "your_email@example.com" -f ~/.ssh/id_ed25519_github

# 2. 公開鍵の内容をコピー
$ cat ~/.ssh/id_ed25519_github.pub
# 表示された内容をクリップボードにコピー

# macOS の場合
$ pbcopy < ~/.ssh/id_ed25519_github.pub

# Linux の場合
$ xclip -selection clipboard < ~/.ssh/id_ed25519_github.pub

# Windows（Git Bash）の場合
$ clip < ~/.ssh/id_ed25519_github.pub

# 3. GitHubの Settings → SSH and GPG keys → New SSH key で公開鍵を登録

# 4. 接続テスト
$ ssh -T git@github.com -i ~/.ssh/id_ed25519_github
Hi username! You've successfully authenticated, but GitHub does not provide shell access.

# 5. ~/.ssh/config に設定を追加（第4章で詳説）
```

## ポイントまとめ

```
1. 公開鍵認証は秘密鍵をネットワーク上に送信しない安全な認証方式
2. Ed25519 が現在の推奨アルゴリズム
3. ssh-keygen -t ed25519 で鍵ペアを生成
4. パスフレーズの設定を推奨（ssh-agentと組み合わせる）
5. パーミッションは厳密に: 秘密鍵600、公開鍵644、.sshディレクトリ700
6. ssh-copy-id で公開鍵を簡単にサーバーに登録できる
7. 用途別に鍵を分けるのがベストプラクティス
```
