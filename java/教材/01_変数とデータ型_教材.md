## 1.2 Javaプログラムの構造

### 最初のプログラム

```java
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
    }
}
```

Javaのプログラムは必ず **クラス（class）** の中に書きます。プログラムの実行は **mainメソッド（main method）** から始まります。

### 構造の解説

```
public class Hello {                    ← クラスの宣言（ファイル名と一致させる）
    public static void main(String[] args) {  ← mainメソッド（実行の入口）
        System.out.println("Hello, Java!");    ← 文（statement）
    }                                          ← mainメソッドの終わり
}                                              ← クラスの終わり
```

各要素の意味は以下の通りです。

| キーワード | 意味 |
|---|---|
| `public` | どこからでもアクセスできる |
| `class` | クラスの定義 |
| `static` | インスタンスを作らずに呼べる |
| `void` | 戻り値がない |
| `main` | プログラムの開始地点 |
| `String[] args` | コマンドライン引数 |

> **よくある間違い：** ファイル名とクラス名が一致していないとコンパイルエラーになります。`Hello` クラスは `Hello.java` というファイル名で保存してください。

### ポイントまとめ

- Javaのコードは必ずクラスの中に書く
- 実行は `public static void main(String[] args)` から始まる
- ファイル名はクラス名と同じにする（Hello.java）
- 文の末尾にはセミコロン（`;`）を付ける

## 1.3 コンパイルと実行

Javaは **コンパイル言語（compiled language）** です。ソースコードを直接実行するのではなく、一度 **バイトコード（bytecode）** に変換してから実行します。

### コンパイルから実行までの流れ

```
+-------------+     javac      +--------------+     java      +----------+
| Hello.java  | ─────────────> | Hello.class  | ────────────> |  出力    |
| (ソースコード) |  (コンパイル)   | (バイトコード)  |  (実行)       | Hello,   |
|             |               |              |              | Java!    |
+-------------+               +--------------+              +----------+
   テキスト                     バイナリ                      画面に表示
   エディタで編集可               JVMが読む形式
```

### コマンドでの実行方法

```bash
# 1. ソースファイルを作成（Hello.java）
# 2. コンパイル
javac Hello.java

# 3. 実行（.classは付けない！）
java Hello
```

実行結果：
```
Hello, Java!
```

> **よくある間違い：** 実行時に `java Hello.class` と書いてしまう人が多いですが、正しくは `java Hello` です。`.class` は付けません。

### JVM（Java Virtual Machine）とは

JVM（Java仮想マシン）は、バイトコードを各OS向けの命令に変換して実行するソフトウェアです。これにより、同じ `.class` ファイルがどのOSでも動作します。

```
                      +------ Windows用JVM ─── Windows上で実行
                      |
Hello.class ─────────+─────── Mac用JVM ────── Mac上で実行
(共通のバイトコード)    |
                      +------ Linux用JVM ───── Linux上で実行
```

### ポイントまとめ

- `javac` でコンパイル、`java` で実行
- ソースコード（.java）→ バイトコード（.class）→ 実行
- JVMのおかげでどのOSでも同じコードが動く
- 実行時に `.class` は付けない

## 1.4 変数の宣言と初期化

**変数（variable）** は、データを入れておく「名前の付いた箱」です。

### 変数の宣言

```java
public class VariableDemo {
    public static void main(String[] args) {
        // 宣言（箱を用意する）
        int age;

        // 代入（箱にデータを入れる）
        age = 25;

        // 宣言と初期化を同時に行う（推奨）
        int height = 170;
        String name = "田中太郎";

        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.println("身長: " + height + "cm");
    }
}
```

実行方法：
```bash
javac VariableDemo.java && java VariableDemo
```

実行結果：
```
名前: 田中太郎
年齢: 25歳
身長: 170cm
```

### 変数の命名規則

| ルール | 例 | 備考 |
|---|---|---|
| 先頭は英字・`_`・`$` | `name`, `_count`, `$value` | 数字で始めるのはNG |
| 予約語は使えない | `class`, `int` などはNG | |
| キャメルケースが慣例 | `firstName`, `totalScore` | 小文字で始める |
| 定数は全大文字 | `MAX_VALUE`, `PI` | アンダースコアで区切る |

> **よくある間違い：** 変数を宣言しただけで初期化していない場合、ローカル変数はコンパイルエラーになります。必ず値を代入してから使いましょう。

### ポイントまとめ

- 変数は「型 変数名 = 値;」で宣言・初期化する
- 変数名はキャメルケース（camelCase）が慣例
- ローカル変数は必ず初期化してから使う

## 1.5 プリミティブ型（基本データ型）

Javaには **8つのプリミティブ型（primitive type）** があります。これらはJavaに最初から組み込まれている基本的なデータ型です。

### 8つのプリミティブ型一覧

| 型名 | サイズ | 範囲 | 用途 | デフォルト値 |
|---|---|---|---|---|
| `byte` | 1バイト | -128 〜 127 | 小さな整数 | 0 |
| `short` | 2バイト | -32,768 〜 32,767 | やや小さな整数 | 0 |
| `int` | 4バイト | 約-21億 〜 約21億 | **整数（最もよく使う）** | 0 |
| `long` | 8バイト | 約-922京 〜 約922京 | 大きな整数 | 0L |
| `float` | 4バイト | 約±3.4×10³⁸ | 小数（精度低） | 0.0f |
| `double` | 8バイト | 約±1.8×10³⁰⁸ | **小数（最もよく使う）** | 0.0 |
| `char` | 2バイト | Unicode 0 〜 65,535 | 1文字 | '\u0000' |
| `boolean` | 1ビット | true / false | 真偽値 | false |

### 使用例

```java
public class PrimitiveTypes {
    public static void main(String[] args) {
        // 整数型
        byte smallNumber = 100;
        short mediumNumber = 30000;
        int number = 2000000000;
        long bigNumber = 9000000000L;  // Lを付ける

        // 小数型
        float pi1 = 3.14f;            // fを付ける
        double pi2 = 3.141592653589793;

        // 文字型
        char grade = 'A';             // シングルクォーテーション
        char emoji = '☆';

        // 真偽値型
        boolean isStudent = true;
        boolean isAdult = false;

        System.out.println("byte: " + smallNumber);
        System.out.println("short: " + mediumNumber);
        System.out.println("int: " + number);
        System.out.println("long: " + bigNumber);
        System.out.println("float: " + pi1);
        System.out.println("double: " + pi2);
        System.out.println("char: " + grade);
        System.out.println("boolean: " + isStudent);
    }
}
```

実行方法：
```bash
javac PrimitiveTypes.java && java PrimitiveTypes
```

実行結果：
```
byte: 100
short: 30000
int: 2000000000
long: 9000000000
float: 3.14
double: 3.141592653589793
char: A
boolean: true
```

> **よくある間違い：** `long` 型のリテラルには末尾に `L` を、`float` 型のリテラルには末尾に `f` を付ける必要があります。付け忘れるとコンパイルエラーや意図しない動作になります。

### ポイントまとめ

- 整数は基本的に `int` を使う（大きな値は `long`）
- 小数は基本的に `double` を使う
- `long` には `L`、`float` には `f` をリテラルに付ける
- `char` はシングルクォーテーション、`String` はダブルクォーテーション

## 1.6 参照型（String）

**参照型（reference type）** は、プリミティブ型以外のすべてのデータ型です。最もよく使う参照型は **String（文字列）** です。

### Stringの基本

```java
public class StringDemo {
    public static void main(String[] args) {
        // 文字列の作成
        String greeting = "こんにちは";
        String name = "太郎";

        // 文字列の結合（concatenation）
        String message = greeting + "、" + name + "さん！";
        System.out.println(message);

        // 文字列の長さ
        System.out.println("文字数: " + message.length());

        // 文字列の一部を取得
        System.out.println("最初の3文字: " + message.substring(0, 3));

        // 大文字・小文字
        String text = "Hello Java";
        System.out.println("大文字: " + text.toUpperCase());
        System.out.println("小文字: " + text.toLowerCase());

        // 文字列の検索
        System.out.println("Javaを含む: " + text.contains("Java"));

        // 文字列の置換
        System.out.println("置換: " + text.replace("Java", "World"));
    }
}
```

実行方法：
```bash
javac StringDemo.java && java StringDemo
```

実行結果：
```
こんにちは、太郎さん！
文字数: 10
最初の3文字: こんに
大文字: HELLO JAVA
小文字: hello java
Javaを含む: true
置換: Hello World
```

### プリミティブ型と参照型の違い

```
プリミティブ型:                    参照型:
+--------+                      +--------+     +----------------+
| age    |                      | name   |---->| "太郎"          |
|   25   |                      | (参照)  |     | (ヒープメモリ上) |
+--------+                      +--------+     +----------------+
 変数に値が直接入る                 変数にはオブジェクトへの参照が入る
```

### ポイントまとめ

- Stringはダブルクォーテーションで囲む
- `+` で文字列を結合できる
- `length()`, `substring()`, `contains()` など多数のメソッドがある
- プリミティブ型は値を直接保持、参照型はオブジェクトへの参照を保持

## 1.7 型変換（キャスト）

### 自動型変換（暗黙的変換・拡大変換）

小さい型から大きい型への変換は自動的に行われます。

```
byte → short → int → long → float → double
                char ↗
```

```java
public class TypeCasting {
    public static void main(String[] args) {
        // === 自動型変換（widening） ===
        int intValue = 100;
        long longValue = intValue;    // int → long（自動）
        double doubleValue = intValue; // int → double（自動）

        System.out.println("int: " + intValue);
        System.out.println("long: " + longValue);
        System.out.println("double: " + doubleValue);

        System.out.println();

        // === 明示的型変換（narrowing / キャスト） ===
        double price = 1980.75;
        int rounded = (int) price;  // 小数部が切り捨てられる

        System.out.println("double: " + price);
        System.out.println("int: " + rounded);

        System.out.println();

        // === 文字列への変換 ===
        int age = 25;
        String ageStr = String.valueOf(age);
        System.out.println("文字列: " + ageStr);

        // === 文字列から数値への変換 ===
        String numStr = "42";
        int num = Integer.parseInt(numStr);
        double numD = Double.parseDouble("3.14");

        System.out.println("整数: " + num);
        System.out.println("小数: " + numD);
    }
}
```

実行方法：
```bash
javac TypeCasting.java && java TypeCasting
```

実行結果：
```
int: 100
long: 100
double: 100.0

double: 1980.75
int: 1980

文字列: 25
整数: 42
小数: 3.14
```

> **よくある間違い：** `(int)` でキャストすると小数部分は **四捨五入ではなく切り捨て** されます。四捨五入したい場合は `Math.round()` を使いましょう。

### ポイントまとめ

- 小さい型→大きい型は自動変換される
- 大きい型→小さい型は `(型名)` で明示的にキャストが必要
- `String.valueOf()` で数値を文字列に変換
- `Integer.parseInt()` / `Double.parseDouble()` で文字列を数値に変換

## 1.8 算術演算子

**演算子（operator）** は、値に対して計算や処理を行う記号です。

### 算術演算子一覧

| 演算子 | 意味 | 例 | 結果 |
|---|---|---|---|
| `+` | 加算 | `10 + 3` | `13` |
| `-` | 減算 | `10 - 3` | `7` |
| `*` | 乗算 | `10 * 3` | `30` |
| `/` | 除算 | `10 / 3` | `3`（整数同士） |
| `%` | 剰余 | `10 % 3` | `1` |

```java
public class Operators {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;

        System.out.println("a + b = " + (a + b));  // 13
        System.out.println("a - b = " + (a - b));  // 7
        System.out.println("a * b = " + (a * b));  // 30
        System.out.println("a / b = " + (a / b));  // 3（整数除算！）
        System.out.println("a % b = " + (a % b));  // 1

        // 小数の除算
        double c = 10.0;
        double d = 3.0;
        System.out.println("c / d = " + (c / d));  // 3.3333...

        System.out.println();

        // インクリメント・デクリメント
        int count = 5;
        count++;  // count = count + 1
        System.out.println("count++: " + count);  // 6
        count--;  // count = count - 1
        System.out.println("count--: " + count);  // 5

        // 複合代入演算子
        int total = 100;
        total += 50;   // total = total + 50
        System.out.println("total += 50: " + total);   // 150
        total -= 30;   // total = total - 30
        System.out.println("total -= 30: " + total);   // 120
        total *= 2;    // total = total * 2
        System.out.println("total *= 2: " + total);    // 240
    }
}
```

実行方法：
```bash
javac Operators.java && java Operators
```

実行結果：
```
a + b = 13
a - b = 7
a * b = 30
a / b = 3
a % b = 1
c / d = 3.3333333333333335

count++: 6
count--: 5
total += 50: 150
total -= 30: 120
total *= 2: 240
```

> **よくある間違い：** 整数同士の除算 `10 / 3` は `3` になります（小数部分は切り捨て）。小数の結果を得るには、少なくとも片方を `double` にする必要があります（例：`10.0 / 3`）。

### ポイントまとめ

- 整数同士の `/` は小数部分が切り捨てられる
- `%` は余りを求める演算子
- `++` / `--` で1ずつ増減
- `+=`, `-=`, `*=` などの複合代入演算子が便利

## 1.9 入出力

### 出力：System.out.println / printf

```java
public class OutputDemo {
    public static void main(String[] args) {
        String name = "太郎";
        int age = 20;
        double height = 170.5;

        // println：改行あり
        System.out.println("名前: " + name);

        // print：改行なし
        System.out.print("年齢: ");
        System.out.println(age + "歳");

        // printf：フォーマット指定（C言語風）
        System.out.printf("名前: %s, 年齢: %d歳, 身長: %.1fcm%n", name, age, height);

        // printfのフォーマット指定子
        System.out.printf("整数: %d%n", 42);
        System.out.printf("小数: %.2f%n", 3.14159);
        System.out.printf("文字列: %s%n", "Hello");
        System.out.printf("ゼロ埋め: %05d%n", 42);
        System.out.printf("右寄せ: %10d%n", 42);
        System.out.printf("左寄せ: %-10d|%n", 42);
    }
}
```

実行方法：
```bash
javac OutputDemo.java && java OutputDemo
```

実行結果：
```
名前: 太郎
年齢: 20歳
名前: 太郎, 年齢: 20歳, 身長: 170.5cm
整数: 42
小数: 3.14
文字列: Hello
ゼロ埋め: 00042
右寄せ:         42
左寄せ: 42        |
```

### 入力：Scanner

```java
import java.util.Scanner;

public class InputDemo {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("名前を入力してください: ");
        String name = scanner.nextLine();

        System.out.print("年齢を入力してください: ");
        int age = scanner.nextInt();

        System.out.print("身長(cm)を入力してください: ");
        double height = scanner.nextDouble();

        System.out.println();
        System.out.println("=== 入力結果 ===");
        System.out.println("名前: " + name);
        System.out.println("年齢: " + age + "歳");
        System.out.printf("身長: %.1fcm%n", height);

        scanner.close();
    }
}
```

実行方法：
```bash
javac InputDemo.java && java InputDemo
```

実行例：
```
名前を入力してください: 太郎
年齢を入力してください: 20
身長(cm)を入力してください: 170.5

=== 入力結果 ===
名前: 太郎
年齢: 20歳
身長: 170.5cm
```

> **よくある間違い：** `nextInt()` や `nextDouble()` の後に `nextLine()` を使うと、改行文字が残っていて空文字列が返されます。数値入力の後に `scanner.nextLine()` を一度空読みして改行を消費しましょう。

### ポイントまとめ

- `System.out.println()` で改行あり出力
- `System.out.printf()` でフォーマット指定出力
- `Scanner` クラスでキーボード入力を受け取る
- 使い終わったら `scanner.close()` で閉じる

## 1.10 var キーワード（Java 10+）

Java 10以降では、**ローカル変数型推論（local variable type inference）** として `var` キーワードが使えます。

```java
public class VarDemo {
    public static void main(String[] args) {
        // 従来の書き方
        String message1 = "Hello";
        int number1 = 42;

        // varを使った書き方（右辺から型が推論される）
        var message2 = "Hello";   // String と推論される
        var number2 = 42;         // int と推論される
        var pi = 3.14;            // double と推論される
        var flag = true;          // boolean と推論される

        System.out.println(message2 + " (型: String)");
        System.out.println(number2 + " (型: int)");
        System.out.println(pi + " (型: double)");
        System.out.println(flag + " (型: boolean)");
    }
}
```

実行方法：
```bash
javac VarDemo.java && java VarDemo
```

実行結果：
```
Hello (型: String)
42 (型: int)
3.14 (型: double)
true (型: boolean)
```

### varの注意点

| 使える場面 | 使えない場面 |
|---|---|
| ローカル変数 | フィールド（クラスの変数） |
| forループの変数 | メソッドの引数 |
| try-with-resources | 戻り値の型 |

> **よくある間違い：** `var` は動的型付けではありません。コンパイル時に型が決まる **静的型推論** です。一度決まった型は変更できません。

### ポイントまとめ

- `var` は右辺の値から型を自動推論する
- ローカル変数でのみ使用可能
- 型が明確な場合にコードを短くするために使う
- 読みやすさが下がる場合は明示的に型を書く方がよい

10. **var**：型推論によるローカル変数宣言

## 次章への橋渡し

ここまでで、変数にデータを格納し、計算や入出力ができるようになりました。しかし、現実のプログラムでは「条件によって処理を変える」ことが不可欠です。

次の第2章では、**条件分岐（if文・switch文）** を学び、プログラムに「判断力」を持たせる方法を習得します。
