## 課題の説明

以下のコードは「リングバッファ（ring buffer）」と呼ばれるデータ構造の実装です。
リングバッファは、固定サイズの配列を循環的に使うことで、FIFO（First In, First Out）のキューを効率的に実現するデータ構造です。

**コードを読んで、後に続く10個の設問に答えてください。**
実行して動作を確認してもかまいませんが、まずはコードを読んで理解することを目標にしてください。

```
リングバッファのイメージ:

  書き込み位置(head)が先に進む
        ↓
  ┌───┬───┬───┬───┬───┐
  │ 3 │ 4 │   │ 1 │ 2 │
  └───┴───┴───┴───┴───┘
              ↑
        読み出し位置(tail)が追いかける

  配列の末尾に達したら先頭に戻る（循環）
```

## 読解対象コード

```c
/*
 * ring_buffer.c - リングバッファの実装
 * 組み込みシステムやログ管理でよく使われるデータ構造
 */
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

/* ==================== データ構造 ==================== */

#define RINGBUF_DEFAULT_CAPACITY 8

typedef struct {
    int *data;
    int capacity;
    int head;    /* 次に書き込む位置 */
    int tail;    /* 次に読み出す位置 */
    int count;   /* 現在の要素数 */
} RingBuffer;

/* ==================== 初期化と破棄 ==================== */

RingBuffer *ringbuf_create(int capacity) {
    RingBuffer *rb = (RingBuffer *)malloc(sizeof(RingBuffer));
    if (rb == NULL) return NULL;

    rb->data = (int *)calloc(capacity, sizeof(int));
    if (rb->data == NULL) {
        free(rb);
        return NULL;
    }

    rb->capacity = capacity;
    rb->head = 0;
    rb->tail = 0;
    rb->count = 0;
    return rb;
}

void ringbuf_destroy(RingBuffer *rb) {
    if (rb != NULL) {
        free(rb->data);
        free(rb);
    }
}

/* ==================== 状態確認 ==================== */

int ringbuf_is_empty(const RingBuffer *rb) {
    return rb->count == 0;
}

int ringbuf_is_full(const RingBuffer *rb) {
    return rb->count == rb->capacity;
}

int ringbuf_size(const RingBuffer *rb) {
    return rb->count;
}

/* ==================== データ操作 ==================== */

int ringbuf_enqueue(RingBuffer *rb, int value) {
    if (ringbuf_is_full(rb)) {
        return 0;  /* 失敗 */
    }
    rb->data[rb->head] = value;
    rb->head = (rb->head + 1) % rb->capacity;
    rb->count++;
    return 1;  /* 成功 */
}

int ringbuf_dequeue(RingBuffer *rb, int *out_value) {
    if (ringbuf_is_empty(rb)) {
        return 0;  /* 失敗 */
    }
    *out_value = rb->data[rb->tail];
    rb->tail = (rb->tail + 1) % rb->capacity;
    rb->count--;
    return 1;  /* 成功 */
}

int ringbuf_peek(const RingBuffer *rb, int *out_value) {
    if (ringbuf_is_empty(rb)) {
        return 0;
    }
    *out_value = rb->data[rb->tail];
    return 1;
}

/* ==================== ユーティリティ ==================== */

void ringbuf_clear(RingBuffer *rb) {
    rb->head = 0;
    rb->tail = 0;
    rb->count = 0;
}

void ringbuf_print(const RingBuffer *rb) {
    int i;
    printf("[");
    for (i = 0; i < rb->count; i++) {
        int index = (rb->tail + i) % rb->capacity;
        if (i > 0) printf(", ");
        printf("%d", rb->data[index]);
    }
    printf("] (size=%d, capacity=%d)\n", rb->count, rb->capacity);
}

void ringbuf_debug(const RingBuffer *rb) {
    int i;
    printf("内部配列: [");
    for (i = 0; i < rb->capacity; i++) {
        if (i > 0) printf(", ");
        printf("%d", rb->data[i]);
    }
    printf("]\n");
    printf("head=%d, tail=%d, count=%d\n", rb->head, rb->tail, rb->count);
}

/* ==================== テスト ==================== */

void test_basic(void) {
    printf("=== 基本テスト ===\n");
    RingBuffer *rb = ringbuf_create(5);

    ringbuf_enqueue(rb, 10);
    ringbuf_enqueue(rb, 20);
    ringbuf_enqueue(rb, 30);
    printf("3件追加後: ");
    ringbuf_print(rb);

    int val;
    ringbuf_dequeue(rb, &val);
    printf("取り出し: %d\n", val);
    printf("取り出し後: ");
    ringbuf_print(rb);

    ringbuf_enqueue(rb, 40);
    ringbuf_enqueue(rb, 50);
    ringbuf_enqueue(rb, 60);
    printf("3件追加後: ");
    ringbuf_print(rb);
    printf("バッファ満杯: %s\n", ringbuf_is_full(rb) ? "Yes" : "No");

    int result = ringbuf_enqueue(rb, 70);
    printf("満杯時の追加: %s\n", result ? "成功" : "失敗");

    ringbuf_debug(rb);
    ringbuf_destroy(rb);
}

void test_wraparound(void) {
    printf("\n=== 循環テスト ===\n");
    RingBuffer *rb = ringbuf_create(4);
    int val;

    ringbuf_enqueue(rb, 1);
    ringbuf_enqueue(rb, 2);
    ringbuf_enqueue(rb, 3);
    ringbuf_dequeue(rb, &val);
    ringbuf_dequeue(rb, &val);
    printf("2件取り出し後: ");
    ringbuf_print(rb);

    ringbuf_enqueue(rb, 4);
    ringbuf_enqueue(rb, 5);
    ringbuf_enqueue(rb, 6);
    printf("3件追加後: ");
    ringbuf_print(rb);

    printf("内部状態:\n");
    ringbuf_debug(rb);

    ringbuf_destroy(rb);
}

int main(void) {
    test_basic();
    test_wraparound();
    return 0;
}
```

## 解答例

<details>
<summary>設問1〜5の解答</summary>

### 設問1

初期状態は以下のとおりです。

- `capacity` = 5（最大5要素格納可能）
- `head` = 0（次に書き込む位置）
- `tail` = 0（次に読み出す位置）
- `count` = 0（格納されている要素数）
- `data` = `{0, 0, 0, 0, 0}`（`calloc` でゼロ初期化される）

### 設問2

`head` は追加のたびに1つ進みます。ただし `% rb->capacity` で余りを取ることで、配列の末尾に達したら先頭に戻ります。

```
capacity=5 の場合:
head: 0 → 1 → 2 → 3 → 4 → 0 → 1 → ...
```

これが「リング（環）」の由来です。

### 設問3

C言語には例外機構がないため、関数の成功/失敗を戻り値（`int`: 1=成功, 0=失敗）で返し、実際のデータはポインタ引数で受け渡しています。これにより、呼び出し側は以下のように安全に使えます。

```c
int val;
if (ringbuf_dequeue(rb, &val)) {
    printf("取得: %d\n", val);
} else {
    printf("バッファが空です\n");
}
```

### 設問4

`test_wraparound` 実行後の内部配列と状態は以下のとおりです。

```
内部配列: [5, 6, 3, 4]
head=2, tail=2, count=4
```

手順を追うと以下の通りです。
1. enqueue(1): data=[1,0,0,0], head=1, tail=0, count=1
2. enqueue(2): data=[1,2,0,0], head=2, tail=0, count=2
3. enqueue(3): data=[1,2,3,0], head=3, tail=0, count=3
4. dequeue→1: head=3, tail=1, count=2
5. dequeue→2: head=3, tail=2, count=1
6. enqueue(4): data=[1,2,3,4], head=0, tail=2, count=2
7. enqueue(5): data=[5,2,3,4], head=1, tail=2, count=3
8. enqueue(6): data=[5,6,3,4], head=2, tail=2, count=4

### 設問5

`const` は「この関数はバッファの内容を変更しない」ことを宣言しています。

- `is_empty`, `peek`, `print` → 読み取りのみなので `const` をつけて安全性を保証
- `enqueue`, `dequeue` → バッファの内容を変更するので `const` はつけられない

`const` をつけることで、コンパイラが誤った変更を検出でき、コードの意図も明確になります。

<summary>設問6〜10の解答</summary>

### 設問6

`malloc` が成功して `calloc` が失敗した場合、先に確保した `rb` のメモリを `free(rb)` で解放してから `NULL` を返しています。

```c
rb->data = (int *)calloc(capacity, sizeof(int));
if (rb->data == NULL) {
    free(rb);      /* 先に確保したメモリを解放 */
    return NULL;
}
```

これがないとメモリリーク（memory leak）が発生します。確保したメモリを解放しないまま関数を抜けると、そのメモリは永久に回収されません。

### 設問7

リングバッファでは `tail` が必ずしも 0 から始まるわけではありません。循環によって `tail` が配列の中間にある場合、論理的な先頭（最も古い要素）は `data[tail]` にあります。

`(rb->tail + i) % rb->capacity` は「tail から i 個先の要素」を循環を考慮して計算しています。これにより、物理的な配列の位置にかかわらず、論理的な順序で要素を表示できます。

### 設問8

改善案の例です。

1. **容量の自動拡張**: バッファが満杯になったら `realloc` で容量を2倍に拡張する機能を追加する
2. **型の汎用化**: `int` 固定ではなく、`void *` とサイズ情報を持たせて任意の型に対応させる
3. **スレッドセーフ対応**: mutex やアトミック操作を使って、複数スレッドから安全にアクセスできるようにする
4. **イテレータの追加**: バッファ内の要素を順番に走査する関数を追加する

### 設問9

リングバッファの使用場面は以下のとおりです。

- **ログシステム**: 最新N件のログを保持する（古いログを自動上書き）
- **音声・映像のバッファリング**: 生産者と消費者の速度差を吸収する
- **組み込みシステム**: メモリが限られた環境で固定サイズのキューが必要な場合
- **ネットワーク通信**: パケットの受信バッファ

通常の配列と比べたメリットは以下のとおりです。
- dequeue時にデータをずらす必要がない（O(1)で追加・取り出し）
- メモリの動的確保が不要（固定サイズで動作）
- リンクリストと比べてメモリ効率が良い（ポインタ分のオーバーヘッドがない）

### 設問10

上書きモードの `ringbuf_enqueue` は以下のとおりです。

```c
/* 満杯時に古いデータを上書きするバージョン */
int ringbuf_enqueue_overwrite(RingBuffer *rb, int value) {
    if (ringbuf_is_full(rb)) {
        /* 最も古い要素を上書き: tail を1つ進める */
        rb->tail = (rb->tail + 1) % rb->capacity;
        rb->count--;
    }
    rb->data[rb->head] = value;
    rb->head = (rb->head + 1) % rb->capacity;
    rb->count++;
    return 1;  /* 常に成功 */
}
```

満杯のときは `tail` を1つ進めて最も古い要素を「捨て」、空いた場所に新しいデータを書き込みます。
