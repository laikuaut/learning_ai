### パート2：EVM分析

```
プロジェクト4週目終了時点の状況：

  BAC（総予算）:    500万円
  PV（計画価値）:   250万円（4週目までに完了予定だった作業の予算）
  EV（出来高）:     200万円（4週目までに実際に完了した作業の予算）
  AC（実コスト）:   280万円（4週目までに実際にかかった費用）

  SV = EV - PV = 200 - 250 = -50万円（スケジュール遅延）
  CV = EV - AC = 200 - 280 = -80万円（コスト超過）
  SPI = EV / PV = 200 / 250 = 0.80（計画の80%しか進んでいない）
  CPI = EV / AC = 200 / 280 = 0.71（1円あたり0.71円分の成果）
  EAC = BAC / CPI = 500 / 0.71 = 704万円（完了時の予測総コスト）
  ETC = EAC - AC = 704 - 280 = 424万円（残りに必要な予測コスト）

  診断: スケジュール遅延かつコスト超過。是正措置が必要。
```

## 課題の要件

### パート1：三点見積もりの作成

以下のプロジェクト（社内FAQ検索システム）の各タスクについて、三点見積もりを行い、PERT期待値と標準偏差を計算してください。

| タスク | 楽観値(O) | 最可能値(M) | 悲観値(P) |
|--------|:---------:|:-----------:|:---------:|
| 要件定義 | 5人日 | 8人日 | 14人日 |
| UI設計 | 3人日 | 5人日 | 10人日 |
| DB設計 | 2人日 | 3人日 | 7人日 |
| 検索エンジン構築 | 10人日 | 15人日 | 25人日 |
| 管理画面開発 | 5人日 | 8人日 | 15人日 |
| テスト | 4人日 | 6人日 | 12人日 |

**計算する項目：**
1. 各タスクのPERT期待値: `(O + 4M + P) / 6`
2. 各タスクの標準偏差: `(P - O) / 6`
3. プロジェクト全体の期待値（合計）
4. プロジェクト全体の標準偏差: `√(各タスクの分散の合計)`

### パート2：EVM分析

以下のプロジェクト状況をもとに、EVM指標を計算し、プロジェクトの健全性を診断してください。

**プロジェクト概要：**
- 総予算（BAC: Budget at Completion）: 800万円
- 計画期間: 8ヶ月
- 現在: 4ヶ月目終了時点

**4ヶ月目終了時点の状況：**
- 計画では全体の50%の作業が完了しているはず
- 実際に完了した作業は全体の40%相当
- 実際にかかった費用は350万円

## ステップガイド

<details>
<summary>ステップ1：PERT期待値と標準偏差の計算</summary>

PERT（Program Evaluation and Review Technique）の公式を使います。

```
期待値（E）= (O + 4M + P) / 6
標準偏差（σ）= (P - O) / 6
分散（σ²）= 標準偏差の2乗
```

**例：要件定義の場合**
```
E = (5 + 4×8 + 14) / 6 = (5 + 32 + 14) / 6 = 51 / 6 = 8.5人日
σ = (14 - 5) / 6 = 9 / 6 = 1.50人日
σ² = 1.50² = 2.25
```

</details>

<details>
<summary>ステップ2：プロジェクト全体の統計量を計算する</summary>

```
プロジェクト全体の期待値 = 各タスクの期待値の合計
プロジェクト全体の分散  = 各タスクの分散の合計
プロジェクト全体の標準偏差 = √(全体の分散)
```

**なぜ標準偏差は単純合計しないのか？**
分散は足し算できますが、標準偏差は直接足せません。
分散を合計してから平方根を取ります。

</details>

<details>
<summary>ステップ3：EVM指標を計算する</summary>

まず基本の3つの値を整理します。

```
BAC = 800万円
PV（Planned Value）= BAC × 計画進捗率 = 800 × 50% = 400万円
EV（Earned Value）= BAC × 実際進捗率 = 800 × 40% = 320万円
AC（Actual Cost）= 350万円（実績）
```

次に差異と指標を計算します。

```
SV（Schedule Variance）= EV - PV   ← マイナスなら遅延
CV（Cost Variance）    = EV - AC   ← マイナスなら超過
SPI（Schedule Performance Index）= EV / PV  ← 1未満なら遅延
CPI（Cost Performance Index）    = EV / AC  ← 1未満なら超過
```

</details>

<details>
<summary>ステップ4：予測値を計算し診断する</summary>

```
EAC（Estimate at Completion）= BAC / CPI
  → 今のペースが続いた場合の完了時総コスト

ETC（Estimate to Complete）= EAC - AC
  → 残りの作業に必要な予測コスト

TCPI（To-Complete Performance Index）= (BAC - EV) / (BAC - AC)
  → 予算内で完了するために必要な残りのCPI
```

**診断の目安：**
```
SPI > 1.0 かつ CPI > 1.0 → 順調
SPI < 1.0 かつ CPI > 1.0 → 遅延しているがコストは管理内
SPI > 1.0 かつ CPI < 1.0 → 進んでいるがコスト超過
SPI < 1.0 かつ CPI < 1.0 → 遅延かつコスト超過（要対策）
```

</details>

<summary>解答例（初心者向け）</summary>

### パート1：三点見積もり

```
タスク別PERT計算：

タスク           O    M    P    期待値E    標準偏差σ   分散σ²
要件定義         5    8   14    8.5        1.50       2.25
UI設計           3    5   10    5.5        1.17       1.36
DB設計           2    3    7    3.5        0.83       0.69
検索エンジン    10   15   25   15.8        2.50       6.25
管理画面開発     5    8   15    8.7        1.67       2.78
テスト           4    6   12    6.7        1.33       1.78
──────────────────────────────────────────────────────
合計                           48.7                  15.11

計算過程（例：検索エンジン構築）：
  E = (10 + 4×15 + 25) / 6 = (10 + 60 + 25) / 6 = 95/6 ≒ 15.8人日
  σ = (25 - 10) / 6 = 15/6 = 2.50人日

プロジェクト全体の標準偏差:
  σ全体 = √15.11 ≒ 3.89人日

信頼区間（68%信頼度）: 48.7 ± 3.89 = 44.8〜52.6人日
信頼区間（95%信頼度）: 48.7 ± 7.78 = 40.9〜56.5人日
```

### パート2：EVM分析

```
基本値の算出：
  BAC = 800万円
  PV  = 800 × 0.50 = 400万円
  EV  = 800 × 0.40 = 320万円
  AC  = 350万円

差異指標：
  SV = EV - PV = 320 - 400 = -80万円  → スケジュール遅延
  CV = EV - AC = 320 - 350 = -30万円  → コスト超過

効率指標：
  SPI = EV / PV = 320 / 400 = 0.80  → 計画の80%の進捗
  CPI = EV / AC = 320 / 350 = 0.91  → 1円あたり0.91円の成果

予測指標：
  EAC = BAC / CPI = 800 / 0.91 ≒ 879万円
  ETC = EAC - AC = 879 - 350 = 529万円
  TCPI = (BAC - EV) / (BAC - AC) = (800-320) / (800-350) = 480/450 ≒ 1.07

診断：
  SPI < 1.0 かつ CPI < 1.0 → スケジュール遅延かつコスト超過

  ・計画の80%しか進捗していない（10%の遅延）
  ・コスト効率も91%で、予算超過の傾向がある
  ・このままのペースでは完了時に約879万円（79万円超過）かかる見込み
  ・予算内で完了するには、残り期間のCPIを1.07に改善する必要がある
```

</details>

### パート1：改良版（感度分析付き）

```
タスク別PERT計算（同上）：
  合計期待値: 48.7人日
  全体標準偏差: 3.89人日

リスク感度分析（分散が大きいタスクに注目）：

  タスク          分散   全体に占める割合
  検索エンジン    6.25   41.4%  ← 最もリスクが高い
  管理画面開発    2.78   18.4%
  要件定義        2.25   14.9%
  テスト          1.78   11.8%
  UI設計          1.36    9.0%
  DB設計          0.69    4.6%

  → 検索エンジン構築が全体の不確実性の41%を占める
  → このタスクの見積もり精度向上がプロジェクト成功の鍵

対策案：
  1. 検索エンジンの技術調査を先行して行い、見積もり精度を上げる
  2. 既存のOSS（Elasticsearchなど）の採用を検討し、悲観値を下げる
  3. 検索エンジンのMVP（最小機能）を先に作り、早期にリスクを検証する
```

### パート2：改良版（是正措置と月次推移）

```
EVM指標サマリ：

是正措置の提案（優先度順）：

   - 影響: EACを800万円以内に収められる可能性が高い

2. リソースの追加投入（即効性: 中）
   - 一時的に1名追加して遅延を回復

   - タスクの並行度を上げる（依存関係の見直し）
   - レビューサイクルを短縮する
   - 自動テストの導入で手動テスト工数を削減

4. ステークホルダーへの報告
   - 現状の数値（SPI, CPI）を透明に報告
   - 3つの選択肢を提示:
     (a) スコープを削減して予算・期間内に収める
     (b) 期間を延長する（追加コストを最小化）
     (c) 予算を追加して当初スコープを維持する
```

**初心者向けとの違い：**
