# 第2章：条件分岐

## この章のゴール

- 比較演算子と論理演算子を使えるようになる
- if / else if / else で条件に応じた処理を書ける
- 三項演算子を適切に使える
- switch文（従来型・拡張型）を使い分けられる

## 2.1 比較演算子

**比較演算子（comparison operator）** は、2つの値を比較して `true` または `false` を返す演算子です。

| 演算子 | 意味 | 例 | 結果 |
|---|---|---|---|
| `==` | 等しい | `5 == 5` | `true` |
| `!=` | 等しくない | `5 != 3` | `true` |
| `<` | より小さい | `3 < 5` | `true` |
| `>` | より大きい | `5 > 3` | `true` |
| `<=` | 以下 | `5 <= 5` | `true` |
| `>=` | 以上 | `5 >= 3` | `true` |

```java
public class ComparisonDemo {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;

        System.out.println("a == b: " + (a == b));  // false
        System.out.println("a != b: " + (a != b));  // true
        System.out.println("a < b: " + (a < b));    // true
        System.out.println("a > b: " + (a > b));    // false
        System.out.println("a <= 10: " + (a <= 10)); // true
        System.out.println("a >= 10: " + (a >= 10)); // true
    }
}
```

実行方法：
```bash
javac ComparisonDemo.java && java ComparisonDemo
```

実行結果：
```
a == b: false
a != b: true
a < b: true
a > b: false
a <= 10: true
a >= 10: true
```

### ポイントまとめ

- 比較演算子は `boolean` 型の値（true / false）を返す
- `=` は代入、`==` は比較（間違えやすい！）
- 比較式は条件分岐で使う

## 2.2 論理演算子

**論理演算子（logical operator）** は、複数の条件を組み合わせるために使います。

| 演算子 | 意味 | 例 | 結果 |
|---|---|---|---|
| `&&` | AND（かつ） | `true && false` | `false` |
| `\|\|` | OR（または） | `true \|\| false` | `true` |
| `!` | NOT（否定） | `!true` | `false` |

### 真理値表

```
AND (&&):              OR (||):               NOT (!):
A     B    A&&B        A     B    A||B        A     !A
true  true  true       true  true  true       true  false
true  false false      true  false true       false true
false true  false      false true  true
false false false      false false false
```

```java
public class LogicalDemo {
    public static void main(String[] args) {
        int age = 20;
        boolean hasLicense = true;

        // AND: 両方の条件を満たす
        boolean canDrive = (age >= 18) && hasLicense;
        System.out.println("運転できる: " + canDrive);  // true

        // OR: どちらかの条件を満たす
        boolean isWeekend = false;
        boolean isHoliday = true;
        boolean isDayOff = isWeekend || isHoliday;
        System.out.println("休日: " + isDayOff);  // true

        // NOT: 条件を反転
        boolean isRaining = false;
        System.out.println("雨ではない: " + !isRaining);  // true

        // 組み合わせ
        int score = 75;
        boolean isPass = (score >= 60) && (score <= 100);
        System.out.println("合格: " + isPass);  // true
    }
}
```

実行方法：
```bash
javac LogicalDemo.java && java LogicalDemo
```

実行結果：
```
運転できる: true
休日: true
雨ではない: true
合格: true
```

> **よくある間違い：** `&` と `&&` は異なります。`&&` は **短絡評価（short-circuit evaluation）** を行い、左辺が `false` なら右辺を評価しません。通常は `&&` と `||` を使いましょう。

### ポイントまとめ

- `&&`（AND）：両方 true のとき true
- `||`（OR）：どちらかが true のとき true
- `!`（NOT）：true と false を反転
- `&&` と `||` は短絡評価を行う

## 2.3 if文

**if文（if statement）** は、条件に応じて処理を分岐させる最も基本的な制御構文です。

### 基本構文

```
if (条件式) {
    // 条件が true のとき実行される処理
}
```

### if-else構文

```java
public class IfElseDemo {
    public static void main(String[] args) {
        int age = 20;

        // 基本のif文
        if (age >= 18) {
            System.out.println("成人です。");
        }

        // if-else
        int temperature = 35;
        if (temperature >= 30) {
            System.out.println("暑い日です。水分をこまめに取りましょう。");
        } else {
            System.out.println("過ごしやすい日です。");
        }

        // if - else if - else
        int score = 85;
        if (score >= 90) {
            System.out.println("評価: A（優秀）");
        } else if (score >= 80) {
            System.out.println("評価: B（良好）");
        } else if (score >= 70) {
            System.out.println("評価: C（普通）");
        } else if (score >= 60) {
            System.out.println("評価: D（可）");
        } else {
            System.out.println("評価: F（不可）");
        }
    }
}
```

実行方法：
```bash
javac IfElseDemo.java && java IfElseDemo
```

実行結果：
```
成人です。
暑い日です。水分をこまめに取りましょう。
評価: B（良好）
```

### if文のフローチャート

```
          +----------+
          | score=85 |
          +----+-----+
               |
        +------+------+
        | score >= 90? |
        +------+------+
         Yes /   \ No
            /     \
   +--------+  +---+----------+
   | 評価: A |  | score >= 80? |
   +--------+  +------+-------+
                 Yes /   \ No
                    /     \
           +--------+  +---+----------+
           | 評価: B |  | score >= 70? |
           +--------+  +------+-------+
                               |  ...
```

### ネストされたif文

```java
public class NestedIfDemo {
    public static void main(String[] args) {
        int age = 25;
        boolean isMember = true;

        if (age >= 18) {
            if (isMember) {
                System.out.println("会員価格: 1,500円");
            } else {
                System.out.println("一般価格: 2,000円");
            }
        } else {
            System.out.println("子供価格: 800円");
        }
    }
}
```

実行方法：
```bash
javac NestedIfDemo.java && java NestedIfDemo
```

実行結果：
```
会員価格: 1,500円
```

> **よくある間違い：** if文のネストが深くなりすぎると、コードが読みにくくなります。3段階以上のネストは、条件を整理するか、メソッドに分割することを検討しましょう。

### ポイントまとめ

- `if` は条件が true のとき処理を実行
- `else if` で複数の条件を順番にチェック
- `else` はどの条件にも当てはまらない場合の処理
- ネストは浅く保つのがベストプラクティス

## 2.4 三項演算子

**三項演算子（ternary operator）** は、if-else を1行で書ける簡潔な記法です。

### 構文

```
変数 = (条件式) ? 真の場合の値 : 偽の場合の値;
```

```java
public class TernaryDemo {
    public static void main(String[] args) {
        int age = 20;

        // if-elseで書く場合
        String status1;
        if (age >= 18) {
            status1 = "成人";
        } else {
            status1 = "未成年";
        }

        // 三項演算子で書く場合（同じ結果）
        String status2 = (age >= 18) ? "成人" : "未成年";

        System.out.println("if-else: " + status1);
        System.out.println("三項演算子: " + status2);

        // 実用例：絶対値の計算
        int number = -5;
        int abs = (number >= 0) ? number : -number;
        System.out.println(number + "の絶対値: " + abs);

        // 実用例：最大値
        int a = 30, b = 50;
        int max = (a > b) ? a : b;
        System.out.println("最大値: " + max);

        // 実用例：偶数・奇数判定
        int num = 7;
        String result = (num % 2 == 0) ? "偶数" : "奇数";
        System.out.println(num + "は" + result + "です。");
    }
}
```

実行方法：
```bash
javac TernaryDemo.java && java TernaryDemo
```

実行結果：
```
if-else: 成人
三項演算子: 成人
-5の絶対値: 5
最大値: 50
7は奇数です。
```

> **よくある間違い：** 三項演算子をネストすると非常に読みにくくなります。`(a > b) ? a : (b > c) ? b : c` のような書き方は避け、if-elseを使いましょう。

### ポイントまとめ

- `条件 ? 真の値 : 偽の値` で簡潔に分岐を書ける
- 単純な代入の分岐に向いている
- 複雑な条件にはif-elseを使う方が読みやすい

## 2.5 switch文

**switch文（switch statement）** は、一つの値に対して複数の分岐を行うときに使います。

### 従来のswitch文

```java
public class SwitchDemo {
    public static void main(String[] args) {
        int month = 8;

        switch (month) {
            case 3:
            case 4:
            case 5:
                System.out.println("春です。");
                break;
            case 6:
            case 7:
            case 8:
                System.out.println("夏です。");
                break;
            case 9:
            case 10:
            case 11:
                System.out.println("秋です。");
                break;
            case 12:
            case 1:
            case 2:
                System.out.println("冬です。");
                break;
            default:
                System.out.println("無効な月です。");
                break;
        }
    }
}
```

実行方法：
```bash
javac SwitchDemo.java && java SwitchDemo
```

実行結果：
```
夏です。
```

### 拡張switch文（Java 14以降）

Java 14以降では、**アロー構文（arrow syntax）** を使った拡張switch文が使えます。`break` が不要で、より簡潔に書けます。

```java
public class EnhancedSwitchDemo {
    public static void main(String[] args) {
        int dayOfWeek = 3;

        // 拡張switch文（アロー構文）
        String dayName = switch (dayOfWeek) {
            case 1 -> "月曜日";
            case 2 -> "火曜日";
            case 3 -> "水曜日";
            case 4 -> "木曜日";
            case 5 -> "金曜日";
            case 6 -> "土曜日";
            case 7 -> "日曜日";
            default -> "無効な値";
        };

        System.out.println("今日は" + dayName + "です。");

        // 複数の値をまとめる
        String dayType = switch (dayOfWeek) {
            case 1, 2, 3, 4, 5 -> "平日";
            case 6, 7 -> "休日";
            default -> "無効";
        };

        System.out.println(dayType + "です。");

        // Stringでのswitch
        String command = "start";
        switch (command) {
            case "start" -> System.out.println("処理を開始します。");
            case "stop" -> System.out.println("処理を停止します。");
            case "restart" -> System.out.println("処理を再起動します。");
            default -> System.out.println("不明なコマンドです。");
        }
    }
}
```

実行方法：
```bash
javac EnhancedSwitchDemo.java && java EnhancedSwitchDemo
```

実行結果：
```
今日は水曜日です。
平日です。
処理を開始します。
```

### 従来のswitch vs 拡張switch

| 特徴 | 従来のswitch | 拡張switch（Java 14+） |
|---|---|---|
| 構文 | `case 値:` | `case 値 ->` |
| breakの必要性 | 必要（忘れるとフォールスルー） | 不要 |
| 値を返す | 不可 | 可能（switch式） |
| 複数値 | 個別のcaseを並べる | `case 1, 2, 3 ->` |

> **よくある間違い：** 従来のswitch文で `break` を忘れると、次の `case` の処理も実行されてしまいます（フォールスルー）。拡張switch文ではこの問題が起きません。

### ポイントまとめ

- switch文は一つの値に対する多分岐に適している
- 従来のswitch文では `break` を忘れないこと
- 拡張switch文（Java 14+）はアロー構文で簡潔に書ける
- switch文は `int`, `String`, `enum` などに使える

## 2.6 文字列の比較：== vs .equals()

Javaで文字列を比較する際、最も重要な注意点があります。

```java
public class StringCompare {
    public static void main(String[] args) {
        // リテラルの場合（同じプールを参照するのでtrueになることがある）
        String s1 = "Hello";
        String s2 = "Hello";
        System.out.println("s1 == s2: " + (s1 == s2));         // true（ただし偶然）
        System.out.println("s1.equals(s2): " + s1.equals(s2)); // true

        // newで生成した場合
        String s3 = new String("Hello");
        String s4 = new String("Hello");
        System.out.println("s3 == s4: " + (s3 == s4));         // false！
        System.out.println("s3.equals(s4): " + s3.equals(s4)); // true

        System.out.println();
        System.out.println("--- 比較のまとめ ---");
        System.out.println("== は参照（アドレス）の比較");
        System.out.println(".equals() は値（中身）の比較");
    }
}
```

実行方法：
```bash
javac StringCompare.java && java StringCompare
```

実行結果：
```
s1 == s2: true
s1.equals(s2): true
s3 == s4: false
s3.equals(s4): true

--- 比較のまとめ ---
== は参照（アドレス）の比較
.equals() は値（中身）の比較
```

### メモリ上のイメージ

```
== の動作（参照の比較）:

s1 ──────────┐
             ▼
        +---------+
        | "Hello" |   ← 文字列プール上の同じオブジェクト
        +---------+
             ▲
s2 ──────────┘

s3 ──────> +---------+
           | "Hello" |   ← 別のオブジェクト（中身は同じ）
           +---------+

s4 ──────> +---------+
           | "Hello" |   ← さらに別のオブジェクト（中身は同じ）
           +---------+
```

> **よくある間違い：** 文字列の比較には **必ず `.equals()` を使ってください**。`==` は「同じオブジェクトかどうか」を比較するため、同じ文字列でも `false` になることがあります。これはJava初心者が最もハマりやすいバグの一つです。

### ポイントまとめ

- 文字列の比較には必ず `.equals()` を使う
- `==` は参照（メモリ上のアドレス）の比較
- `.equals()` は値（文字列の中身）の比較
- 大文字小文字を無視する比較は `.equalsIgnoreCase()` を使う

## 2.7 実践例：成績判定プログラム

学んだことを組み合わせた実践的なプログラムです。

```java
import java.util.Scanner;

public class GradeCalculator {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.println("=== 成績判定プログラム ===");
        System.out.print("点数を入力してください（0〜100）: ");
        int score = scanner.nextInt();

        // 入力値の検証
        if (score < 0 || score > 100) {
            System.out.println("エラー: 0〜100の範囲で入力してください。");
            scanner.close();
            return;
        }

        // 成績の判定
        String grade;
        String comment;

        if (score >= 90) {
            grade = "A";
            comment = "素晴らしい成績です！";
        } else if (score >= 80) {
            grade = "B";
            comment = "よく頑張りました！";
        } else if (score >= 70) {
            grade = "C";
            comment = "もう少し頑張りましょう。";
        } else if (score >= 60) {
            grade = "D";
            comment = "復習が必要です。";
        } else {
            grade = "F";
            comment = "再試験が必要です。";
        }

        // 合否判定
        String passOrFail = (score >= 60) ? "合格" : "不合格";

        // 結果の表示
        System.out.println();
        System.out.println("=== 判定結果 ===");
        System.out.printf("点数: %d点%n", score);
        System.out.printf("評価: %s%n", grade);
        System.out.printf("合否: %s%n", passOrFail);
        System.out.printf("コメント: %s%n", comment);

        scanner.close();
    }
}
```

実行方法：
```bash
javac GradeCalculator.java && java GradeCalculator
```

実行例：
```
=== 成績判定プログラム ===
点数を入力してください（0〜100）: 85

=== 判定結果 ===
点数: 85点
評価: B
合否: 合格
コメント: よく頑張りました！
```

## 2.8 実践例：BMI計算プログラム

```java
import java.util.Scanner;

public class BmiCalculator {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.println("=== BMI計算プログラム ===");
        System.out.print("身長(cm)を入力: ");
        double heightCm = scanner.nextDouble();

        System.out.print("体重(kg)を入力: ");
        double weight = scanner.nextDouble();

        // BMI = 体重(kg) / 身長(m)の2乗
        double heightM = heightCm / 100.0;
        double bmi = weight / (heightM * heightM);

        // 判定
        String category;
        if (bmi < 18.5) {
            category = "低体重（やせ）";
        } else if (bmi < 25.0) {
            category = "普通体重";
        } else if (bmi < 30.0) {
            category = "肥満（1度）";
        } else if (bmi < 35.0) {
            category = "肥満（2度）";
        } else {
            category = "肥満（3度以上）";
        }

        System.out.println();
        System.out.println("=== 結果 ===");
        System.out.printf("BMI: %.1f%n", bmi);
        System.out.printf("判定: %s%n", category);

        scanner.close();
    }
}
```

実行方法：
```bash
javac BmiCalculator.java && java BmiCalculator
```

実行例：
```
=== BMI計算プログラム ===
身長(cm)を入力: 170
体重(kg)を入力: 65

=== 結果 ===
BMI: 22.5
判定: 普通体重
```

### ポイントまとめ

- 実践的なプログラムでは入力検証が重要
- 複数の条件分岐を組み合わせて判定ロジックを作る
- 結果をフォーマットして見やすく出力する

## まとめ

この章では、プログラムに「判断力」を持たせる条件分岐を学びました。

1. **比較演算子**：`==`, `!=`, `<`, `>`, `<=`, `>=`
2. **論理演算子**：`&&`（AND）, `||`（OR）, `!`（NOT）
3. **if文**：最も基本的な条件分岐
4. **三項演算子**：簡潔な条件式
5. **switch文**：一つの値に対する多分岐
6. **文字列比較**：必ず `.equals()` を使う（`==` は使わない）

特に「文字列の比較には `.equals()` を使う」ことは、Java開発で最も大切なルールの一つです。必ず覚えてください。

## 次章への橋渡し

条件分岐ができるようになったことで、プログラムに「判断力」が付きました。次は「繰り返し」です。

第3章では、**ループ（for文・while文・do-while文）** を学び、同じ処理を効率的に繰り返す方法を習得します。条件分岐とループを組み合わせることで、より複雑で実用的なプログラムが書けるようになります。
