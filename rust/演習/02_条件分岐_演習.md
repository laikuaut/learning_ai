この演習では、Rustにおける条件分岐の基本から、Rust特有の強力な `match` 式やパターンマッチングまでを実践的に学びます。

## 基本問題

### 問題1：if/else による数値の分類

整数の変数 `temperature` を受け取り、以下のルールで気温の状態を出力するプログラムを作成してください。

- 35以上：「猛暑です！熱中症に注意してください」
- 25以上35未満：「暑いですね。水分補給を忘れずに」
- 15以上25未満：「過ごしやすい気温です」
- 5以上15未満：「肌寒いです。上着を持っていきましょう」
- 5未満：「寒いです！暖かくしてお出かけください」

3つの気温（`38`, `22`, `-3`）で動作を確認してください。

**期待される出力例：**
```
気温: 38°C → 猛暑です！熱中症に注意してください
気温: 22°C → 過ごしやすい気温です
気温: -3°C → 寒いです！暖かくしてお出かけください
```

<details><summary>ヒント</summary>

- `if ... else if ... else` の構文を使います
- 条件は上から順に評価されるので、大きい値から判定すると整理しやすいです
- 繰り返し処理を関数にまとめると、3つの気温をテストしやすくなります

</details>

<details><summary>解答例</summary>

```rust
fn describe_temperature(temp: i32) -> &'static str {
    if temp >= 35 {
        "猛暑です！熱中症に注意してください"
    } else if temp >= 25 {
        "暑いですね。水分補給を忘れずに"
    } else if temp >= 15 {
        "過ごしやすい気温です"
    } else if temp >= 5 {
        "肌寒いです。上着を持っていきましょう"
    } else {
        "寒いです！暖かくしてお出かけください"
    }
}

fn main() {
    let temperatures = [38, 22, -3];

    for &temp in &temperatures {
        let description = describe_temperature(temp);
        println!("気温: {}°C → {}", temp, description);
    }
}
```

**補足：** Rustでは `if` は式（expression）です。つまり値を返すことができます。この解答例では、関数の最後の式が戻り値となるRustの仕組みを利用して、`if/else if/else` の各分岐から直接文字列を返しています。セミコロン（`;`）を付けないことで、その式の値が関数の戻り値になります。

`&'static str` は「プログラムの実行期間全体にわたって有効な文字列スライス」を意味します。文字列リテラル（`"..."` で書いた文字列）は全て `'static` ライフタイムを持ちます。

</details>

### 問題2：比較演算子と論理演算子

ある年が以下の条件に該当するかを判定するプログラムを作成してください。

- **うるう年の判定ルール：**
  1. 4で割り切れる年はうるう年
  2. ただし、100で割り切れる年はうるう年ではない
  3. ただし、400で割り切れる年はうるう年

以下の年でテストしてください：`2024`, `1900`, `2000`, `2025`

**期待される出力例：**
```
2024年: うるう年です（366日）
1900年: うるう年ではありません（365日）
2000年: うるう年です（366日）
2025年: うるう年ではありません（365日）
```

<details><summary>ヒント</summary>

- `%` 演算子で余り（剰余）を計算できます
- 論理演算子 `&&`（かつ）、`||`（または）、`!`（否定）を組み合わせます
- 条件を整理すると「400で割り切れる OR（4で割り切れる AND 100で割り切れない）」になります

</details>

<details><summary>解答例</summary>

```rust
fn is_leap_year(year: u32) -> bool {
    // 400で割り切れるか、4で割り切れて100で割り切れない場合はうるう年
    year % 400 == 0 || (year % 4 == 0 && year % 100 != 0)
}

fn main() {
    let years = [2024, 1900, 2000, 2025];

    for &year in &years {
        if is_leap_year(year) {
            println!("{}年: うるう年です（366日）", year);
        } else {
            println!("{}年: うるう年ではありません（365日）", year);
        }
    }
}
```

**補足：** 論理演算子の優先順位は `!` > `&&` > `||` です。この解答では括弧を使って意図を明確にしています。Rustでは条件式に括弧は必須ではありませんが、複雑な条件では可読性のために括弧を付けることが推奨されます。

うるう年判定のロジックを図で整理すると：

```
400で割り切れる？ → Yes → うるう年
       ↓ No
100で割り切れる？ → Yes → うるう年ではない
       ↓ No
  4で割り切れる？ → Yes → うるう年
       ↓ No
       うるう年ではない
```

</details>

### 問題3：if式を値として使う

以下の要件を満たすプログラムを作成してください。

1. 変数 `score` に `75` を代入
2. `if` 式を使って、60点以上なら `"合格"` 、未満なら `"不合格"` を変数 `result` に代入
3. `if` 式を使って、スコアに応じた評価（A:90以上、B:80以上、C:70以上、D:60以上、F:60未満）を変数 `grade` に代入
4. 結果を出力

**期待される出力例：**
```
スコア: 75点
結果: 合格
評価: C
```

<details><summary>ヒント</summary>

- Rustでは `let 変数 = if 条件 { 値1 } else { 値2 };` のように `if` を式として使えます
- 各分岐が返す値の型は同じでなければなりません
- ネストした `if/else if/else` も式として使えます
- 各分岐の最後の式にセミコロンを付けないことで値を返します

</details>

<details><summary>解答例</summary>

```rust
fn main() {
    let score = 75;

    // if式を使って合否を判定
    let result = if score >= 60 { "合格" } else { "不合格" };

    // if式を使って評価を決定
    let grade = if score >= 90 {
        "A"
    } else if score >= 80 {
        "B"
    } else if score >= 70 {
        "C"
    } else if score >= 60 {
        "D"
    } else {
        "F"
    };

    println!("スコア: {}点", score);
    println!("結果: {}", result);
    println!("評価: {}", grade);
}
```

**補足：** Rustでは `if` は文（statement）ではなく式（expression）です。これは他の多くの言語（C、Java、JavaScriptなど）とは異なる重要な特徴です。

```
// 他の言語の三項演算子に相当
// JavaScript: const result = score >= 60 ? "合格" : "不合格";
// Rust:       let result = if score >= 60 { "合格" } else { "不合格" };
```

Rustには三項演算子（`? :`）がありません。代わりに `if` 式を使います。`if` 式を使うことで、変数の宣言と値の決定を同時に行えるため、`mut` を使わずに済むケースが増えます。不変変数を増やすことでコードの安全性が高まります。

</details>

### 問題4：match式の基本

`match` 式を使って、HTTP ステータスコードの説明を返すプログラムを作成してください。

以下のステータスコードに対応してください：
- 200: "OK - リクエスト成功"
- 201: "Created - リソース作成成功"
- 301: "Moved Permanently - 恒久的リダイレクト"
- 400: "Bad Request - リクエスト不正"
- 401: "Unauthorized - 認証が必要"
- 403: "Forbidden - アクセス禁止"
- 404: "Not Found - リソースが見つかりません"
- 500: "Internal Server Error - サーバー内部エラー"
- その他: "Unknown - 不明なステータスコード"

テストするコード：`200`, `404`, `500`, `302`, `418`

**期待される出力例：**
```
200: OK - リクエスト成功
404: Not Found - リソースが見つかりません
500: Internal Server Error - サーバー内部エラー
302: Unknown - 不明なステータスコード
418: Unknown - 不明なステータスコード
```

<details><summary>ヒント</summary>

- `match` 式は `match 値 { パターン => 結果, ... }` の形式で書きます
- 具体的な値にマッチさせるには `200 => "..."` のように書きます
- 全パターンを網羅する必要があり、残りは `_ => "..."` でキャッチします（ワイルドカードパターン）
- `match` も式なので、値を返すことができます

</details>

<details><summary>解答例</summary>

```rust
fn describe_status(code: u16) -> &'static str {
    // match式で各ステータスコードに対応する説明を返す
    match code {
        200 => "OK - リクエスト成功",
        201 => "Created - リソース作成成功",
        301 => "Moved Permanently - 恒久的リダイレクト",
        400 => "Bad Request - リクエスト不正",
        401 => "Unauthorized - 認証が必要",
        403 => "Forbidden - アクセス禁止",
        404 => "Not Found - リソースが見つかりません",
        500 => "Internal Server Error - サーバー内部エラー",
        // _ はワイルドカード：上のどのパターンにもマッチしなかった場合
        _ => "Unknown - 不明なステータスコード",
    }
}

fn main() {
    let codes = [200, 404, 500, 302, 418];

    for &code in &codes {
        println!("{}: {}", code, describe_status(code));
    }
}
```

**補足：** `match` 式はRustで最もよく使われる制御フロー構文の一つです。`if/else` チェーンよりも以下の点で優れています。

1. **網羅性チェック** : コンパイラが全パターンを網羅しているか確認してくれます
2. **可読性** : 多数の分岐がある場合、`match` の方が見やすいです
3. **パターンマッチ** : 値の構造を分解しながらマッチングできます

`match` は全てのケースを網羅しなければコンパイルエラーになります。`_`（ワイルドカード）を使うことで「その他全て」をキャッチできます。

</details>

### 問題5：match式の範囲パターンとガード

`match` 式の範囲パターン（range pattern）とガード（guard）を使って、BMI（ボディマス指数）を判定するプログラムを作成してください。

要件：
1. 身長（cm）と体重（kg）からBMIを計算する関数を作成
2. BMIの値に応じて以下の分類を返す：
   - 18.5未満：「低体重（やせ型）」
   - 18.5以上25.0未満：「普通体重」
   - 25.0以上30.0未満：「肥満（1度）」
   - 30.0以上35.0未満：「肥満（2度）」
   - 35.0以上：「肥満（3度以上）」
3. 複数のテストケースで動作を確認

**期待される出力例：**
```
身長: 170.0cm, 体重: 55.0kg
  BMI: 19.03 → 普通体重

身長: 160.0cm, 体重: 45.0kg
  BMI: 17.58 → 低体重（やせ型）

身長: 175.0cm, 体重: 90.0kg
  BMI: 29.39 → 肥満（1度）

身長: 165.0cm, 体重: 100.0kg
  BMI: 36.73 → 肥満（3度以上）
```

<details><summary>ヒント</summary>

- BMI = 体重(kg) / (身長(m))^2 で計算します
- 身長はcmで入力するので、100で割ってメートルに変換してください
- `match` で浮動小数点数を直接扱うのは難しいため、`if/else if` で判定するか、整数に変換してから `match` を使う方法があります
- もう一つの方法として、`match` の中でガード条件（`if` 条件）を使えます：`x if x < 18.5 => ...`

</details>

<details><summary>解答例</summary>

```rust
// BMIを計算する関数
fn calculate_bmi(height_cm: f64, weight_kg: f64) -> f64 {
    let height_m = height_cm / 100.0;
    weight_kg / (height_m * height_m)
}

// BMIの分類を返す関数
fn classify_bmi(bmi: f64) -> &'static str {
    // match でガード条件を使って浮動小数点を分類
    match bmi {
        x if x < 18.5 => "低体重（やせ型）",
        x if x < 25.0 => "普通体重",
        x if x < 30.0 => "肥満（1度）",
        x if x < 35.0 => "肥満（2度）",
        _ => "肥満（3度以上）",
    }
}

fn main() {
    // テストデータ: (身長cm, 体重kg)
    let test_cases = [
        (170.0, 55.0),
        (160.0, 45.0),
        (175.0, 90.0),
        (165.0, 100.0),
    ];

    for &(height, weight) in &test_cases {
        let bmi = calculate_bmi(height, weight);
        let category = classify_bmi(bmi);

        println!("身長: {:.1}cm, 体重: {:.1}kg", height, weight);
        println!("  BMI: {:.2} → {}", bmi, category);
        println!();
    }
}
```

**補足：** `match` のガード条件は `パターン if 条件 => 結果` の形式で書きます。`x` にマッチした値が束縛され、`if` の条件でさらにフィルタリングできます。

```
match bmi {
    x if x < 18.5 => ...,    // x に値が束縛され、x < 18.5 を追加チェック
    x if x < 25.0 => ...,    // 上にマッチしなかった場合、ここで x < 25.0 をチェック
    _ => ...,                  // それ以外全て
}
```

ガード条件を使うと、単純なパターンマッチだけでは表現できない複雑な条件を記述できます。ただし、ガード条件を使う場合はコンパイラが網羅性を完全にチェックできないため、`_`（ワイルドカード）で残りのケースをカバーすることが推奨されます。

</details>

### 問題6：Option型とmatch

`match` を使って `Option<T>` 型を扱うプログラムを作成してください。

要件：
1. 整数のベクタから指定した値を検索し、見つかったインデックスを返す関数 `find_index` を作成（戻り値は `Option<usize>`）
2. 検索結果に応じて適切なメッセージを表示
3. 複数の検索テストを実行

**期待される出力例：**
```
データ: [10, 20, 30, 40, 50]

30 を検索: インデックス 2 で見つかりました
60 を検索: 見つかりませんでした
10 を検索: インデックス 0 で見つかりました
50 を検索: インデックス 4 で見つかりました
99 を検索: 見つかりませんでした
```

<details><summary>ヒント</summary>

- `Option<T>` は `Some(値)` または `None` のいずれかです
- `match` で `Some(value)` と `None` を網羅的に処理できます
- ベクタの要素を検索するには `iter().position(|&x| x == target)` が使えます
- 自分で実装する場合は、`for` ループで回して見つけたら `Some(index)` を返します

</details>

<details><summary>解答例</summary>

```rust
// ベクタから値を検索してインデックスを返す関数
fn find_index(data: &[i32], target: i32) -> Option<usize> {
    // enumerate() でインデックスと値のペアを取得
    for (index, &value) in data.iter().enumerate() {
        if value == target {
            return Some(index); // 見つかったらSome(インデックス)を返す
        }
    }
    None // 見つからなかったらNoneを返す
}

fn main() {
    let data = vec![10, 20, 30, 40, 50];
    println!("データ: {:?}", data);
    println!();

    let search_values = [30, 60, 10, 50, 99];

    for &target in &search_values {
        // match でOption型を処理
        match find_index(&data, target) {
            Some(index) => {
                println!("{} を検索: インデックス {} で見つかりました", target, index);
            }
            None => {
                println!("{} を検索: 見つかりませんでした", target);
            }
        }
    }
}
```

**補足：** `Option<T>` はRustで「値があるかもしれないし、ないかもしれない」ことを表す列挙型（enum）です。他の言語の `null` や `nil` に相当しますが、型システムで安全に扱える点が異なります。

```
Option<T> の構造:

  Option<usize>
    ├── Some(usize)  ← 値がある（例: Some(2)）
    └── None         ← 値がない
```

`match` で `Option` を処理すると、`Some` と `None` の両方を必ず処理することがコンパイラによって強制されます。これにより、「null参照によるクラッシュ」（他の言語でよくあるバグ）がRustでは原理的に発生しません。

なお、`find_index` と同等の機能は標準ライブラリの `iter().position()` で実現できます：

```rust
let result = data.iter().position(|&x| x == target);
```

</details>

### 問題7：matchとパターンマッチの組み合わせ - コマンドパーサー

簡易的なコマンドパーサーを作成してください。文字列コマンドを解析して、適切な処理を行うプログラムです。

要件：
1. 列挙型 `Command` を定義：
   - `Greet(String)` - 挨拶コマンド（名前を持つ）
   - `Add(i32, i32)` - 加算コマンド（2つの数値を持つ）
   - `Repeat(String, u32)` - 繰り返しコマンド（文字列と回数を持つ）
   - `Quit` - 終了コマンド
2. コマンドを解析する関数 `parse_command` を作成（文字列からCommandを返す）
3. コマンドを実行する関数 `execute_command` を作成（matchで各コマンドを処理）
4. 複数のコマンドを順に処理する

**期待される出力例：**
```
コマンド実行開始
---
> greet 太郎
こんにちは、太郎さん！

> add 15 27
15 + 27 = 42

> repeat hello 3
hello hello hello

> greet Rust
こんにちは、Rustさん！

> add 100 200
100 + 200 = 300

> unknown
不明なコマンドです: unknown

> quit
プログラムを終了します
---
全 7 コマンドを処理しました
```

<details><summary>ヒント</summary>

- 列挙型は `enum Command { Greet(String), Add(i32, i32), ... }` で定義します
- 文字列の分割には `.split_whitespace()` を使います
- `.collect::<Vec<&str>>()` でベクタに変換できます
- `match` でスライスパターンを使うと便利です：`match &parts[..] { ["greet", name] => ..., ... }`
- 文字列から整数への変換は `.parse::<i32>().ok()` で `Option<i32>` を返します

</details>

<details><summary>解答例</summary>

```rust
// コマンドを表す列挙型
enum Command {
    Greet(String),         // 挨拶（名前）
    Add(i32, i32),         // 加算（数値1, 数値2）
    Repeat(String, u32),   // 繰り返し（文字列, 回数）
    Quit,                  // 終了
    Unknown(String),       // 不明なコマンド
}

// 文字列をパースしてCommand列挙型を返す関数
fn parse_command(input: &str) -> Command {
    // 空白で分割してベクタに変換
    let parts: Vec<&str> = input.split_whitespace().collect();

    // スライスパターンでマッチング
    match &parts[..] {
        // "greet 名前" の形式
        ["greet", name] => Command::Greet(name.to_string()),

        // "add 数値1 数値2" の形式
        ["add", a, b] => {
            // 数値のパースを試みる
            match (a.parse::<i32>(), b.parse::<i32>()) {
                (Ok(x), Ok(y)) => Command::Add(x, y),
                _ => Command::Unknown(input.to_string()),
            }
        }

        // "repeat 文字列 回数" の形式
        ["repeat", text, count] => {
            match count.parse::<u32>() {
                Ok(n) => Command::Repeat(text.to_string(), n),
                Err(_) => Command::Unknown(input.to_string()),
            }
        }

        // "quit" コマンド
        ["quit"] => Command::Quit,

        // その他は不明なコマンド
        _ => Command::Unknown(input.to_string()),
    }
}

// コマンドを実行する関数（matchで各バリアントを処理）
fn execute_command(command: &Command) {
    match command {
        // Greet: 名前を使って挨拶
        Command::Greet(name) => {
            println!("こんにちは、{}さん！", name);
        }

        // Add: 2つの数値を加算
        Command::Add(a, b) => {
            println!("{} + {} = {}", a, b, a + b);
        }

        // Repeat: 文字列を指定回数繰り返す
        Command::Repeat(text, count) => {
            let repeated: Vec<&str> = (0..*count).map(|_| text.as_str()).collect();
            println!("{}", repeated.join(" "));
        }

        // Quit: 終了メッセージ
        Command::Quit => {
            println!("プログラムを終了します");
        }

        // Unknown: エラーメッセージ
        Command::Unknown(input) => {
            println!("不明なコマンドです: {}", input);
        }
    }
}

fn main() {
    // テストするコマンド一覧
    let commands = [
        "greet 太郎",
        "add 15 27",
        "repeat hello 3",
        "greet Rust",
        "add 100 200",
        "unknown",
        "quit",
    ];

    println!("コマンド実行開始");
    println!("---");

    for &input in &commands {
        println!("> {}", input);
        let command = parse_command(input);
        execute_command(&command);
        println!();
    }

    println!("---");
    println!("全 {} コマンドを処理しました", commands.len());
}
```

**補足：** この問題はRustの `match` の真価を示す例です。ここでは以下のパターンマッチング機能を活用しています。

1. **スライスパターン** : `["greet", name]` で配列の構造にマッチし、要素を変数に束縛
2. **列挙型のパターンマッチ** : `Command::Greet(name)` でバリアントとその中の値を同時にマッチ
3. **タプルパターン** : `(Ok(x), Ok(y))` で2つの `Result` を同時にマッチ
4. **ワイルドカード** : `_` と `Err(_)` で残りのケースをキャッチ

```
パターンマッチングの流れ:

入力: "add 15 27"
  ↓ split_whitespace()
  ["add", "15", "27"]
  ↓ match &parts[..]
  ["add", a, b] にマッチ → a="15", b="27"
  ↓ parse()
  (Ok(15), Ok(27)) にマッチ → Command::Add(15, 27)
  ↓ execute_command
  Command::Add(a, b) にマッチ → "15 + 27 = 42" を出力
```

実務では、このようなパターンを発展させてCLIツールやプロトコルパーサーを構築します。`match` による網羅性チェックのおかげで、新しいコマンドを追加した際に処理漏れがあるとコンパイルエラーで教えてくれます。

</details>

### 問題8：Result型とエラーハンドリング

`Result<T, E>` 型と `match` を使った堅牢なエラーハンドリングを実装してください。

要件：
1. 文字列から数値を読み取り、各種計算を行う関数群を作成
2. 独自のエラー型 `CalcError` を列挙型で定義：
   - `ParseError(String)` - 数値のパースに失敗
   - `DivisionByZero` - ゼロ除算
   - `Overflow` - オーバーフロー（結果が10000を超える）
3. 安全な除算関数 `safe_divide` を作成
4. 文字列ペアを受け取り、パース→計算→結果返却を行う関数 `calculate` を作成
5. 各種エラーケースを含むテストを実行

**期待される出力例：**
```
=== 計算テスト ===
"100" ÷ "4" = 25.00
"hello" ÷ "5" → エラー: "hello" を数値に変換できません
"42" ÷ "0" → エラー: ゼロで割ることはできません
"50000" ÷ "3" → エラー: 計算結果が大きすぎます（16666.67）
"999" ÷ "10" = 99.90
"-100" ÷ "3" = -33.33

=== エラー集計 ===
成功: 3件
パースエラー: 1件
ゼロ除算: 1件
オーバーフロー: 1件
```

<details><summary>ヒント</summary>

- 独自エラー型は `enum CalcError { ParseError(String), DivisionByZero, Overflow }` で定義します
- `Result<f64, CalcError>` を返す関数を作成します
- `str::parse::<f64>()` の結果を `match` で処理し、エラーなら `CalcError::ParseError` に変換します
- `?` 演算子を使うと `Result` のエラーを自動的に返却できますが、この問題では `match` で明示的に処理してみましょう

</details>

<details><summary>解答例</summary>

```rust
use std::fmt;

// 独自のエラー型を定義
enum CalcError {
    ParseError(String),  // パース失敗（元の文字列を保持）
    DivisionByZero,      // ゼロ除算
    Overflow,            // オーバーフロー
}

// エラー型の表示を実装
impl fmt::Display for CalcError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            CalcError::ParseError(s) => write!(f, "\"{}\" を数値に変換できません", s),
            CalcError::DivisionByZero => write!(f, "ゼロで割ることはできません"),
            CalcError::Overflow => write!(f, "計算結果が大きすぎます"),
        }
    }
}

// 文字列を数値にパースする関数
fn parse_number(s: &str) -> Result<f64, CalcError> {
    match s.parse::<f64>() {
        Ok(n) => Ok(n),
        Err(_) => Err(CalcError::ParseError(s.to_string())),
    }
}

// 安全な除算関数
fn safe_divide(a: f64, b: f64) -> Result<f64, CalcError> {
    // ゼロ除算チェック
    if b == 0.0 {
        return Err(CalcError::DivisionByZero);
    }

    let result = a / b;

    // オーバーフローチェック（結果の絶対値が10000を超える場合）
    if result.abs() > 10000.0 {
        return Err(CalcError::Overflow);
    }

    Ok(result)
}

// 文字列ペアを受け取って計算する関数
fn calculate(a_str: &str, b_str: &str) -> Result<f64, CalcError> {
    // まず両方の文字列をパース
    let a = match parse_number(a_str) {
        Ok(n) => n,
        Err(e) => return Err(e),
    };

    let b = match parse_number(b_str) {
        Ok(n) => n,
        Err(e) => return Err(e),
    };

    // 除算を実行
    safe_divide(a, b)
}

fn main() {
    // テストケース: (被除数, 除数)
    let test_cases = [
        ("100", "4"),
        ("hello", "5"),
        ("42", "0"),
        ("50000", "3"),
        ("999", "10"),
        ("-100", "3"),
    ];

    // エラー集計用カウンタ
    let mut success_count = 0;
    let mut parse_error_count = 0;
    let mut div_zero_count = 0;
    let mut overflow_count = 0;

    println!("=== 計算テスト ===");

    for &(a, b) in &test_cases {
        match calculate(a, b) {
            Ok(result) => {
                println!("\"{}\" ÷ \"{}\" = {:.2}", a, b, result);
                success_count += 1;
            }
            Err(ref e) => {
                // エラーの種類に応じてカウントと表示
                match e {
                    CalcError::ParseError(_) => {
                        println!("\"{}\" ÷ \"{}\" → エラー: {}", a, b, e);
                        parse_error_count += 1;
                    }
                    CalcError::DivisionByZero => {
                        println!("\"{}\" ÷ \"{}\" → エラー: {}", a, b, e);
                        div_zero_count += 1;
                    }
                    CalcError::Overflow => {
                        // オーバーフローの場合は計算結果も表示
                        let a_num: f64 = a.parse().unwrap();
                        let b_num: f64 = b.parse().unwrap();
                        println!(
                            "\"{}\" ÷ \"{}\" → エラー: 計算結果が大きすぎます（{:.2}）",
                            a, b, a_num / b_num
                        );
                        overflow_count += 1;
                    }
                }
            }
        }
    }

    println!();
    println!("=== エラー集計 ===");
    println!("成功: {}件", success_count);
    println!("パースエラー: {}件", parse_error_count);
    println!("ゼロ除算: {}件", div_zero_count);
    println!("オーバーフロー: {}件", overflow_count);
}
```

**補足：** `Result<T, E>` はRustにおけるエラーハンドリングの中核です。`Option<T>` が「値があるかないか」を表すのに対し、`Result<T, E>` は「成功か失敗か」を表し、失敗時にはエラー情報を持ちます。

```
Result<T, E> の構造:

  Result<f64, CalcError>
    ├── Ok(f64)              ← 成功（例: Ok(25.0)）
    └── Err(CalcError)       ← 失敗
          ├── ParseError(String)
          ├── DivisionByZero
          └── Overflow
```

実務では `?` 演算子を使うことでエラー処理を簡潔に書けます。上記の `calculate` 関数は以下のように書き直せます：

```rust
fn calculate(a_str: &str, b_str: &str) -> Result<f64, CalcError> {
    let a = parse_number(a_str)?;  // エラーなら即座にErrを返す
    let b = parse_number(b_str)?;  // エラーなら即座にErrを返す
    safe_divide(a, b)
}
```

`?` 演算子は `match` による明示的なエラー処理のシンタックスシュガーです。最初は `match` で書いてエラーの流れを理解し、慣れてきたら `?` 演算子を活用するのが学習のよい進め方です。

</details>
