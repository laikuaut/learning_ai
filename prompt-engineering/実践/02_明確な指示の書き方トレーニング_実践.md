# 実践課題02：明確な指示の書き方トレーニング ★1

> **難易度**: ★☆☆☆☆（入門）
> **前提知識**: 第2章（基本テクニック）
> **課題の種類**: 改善演習
> **学習目標**: 曖昧なプロンプトを具体的で明確な指示に改善するスキルを身につける

## 課題の要件

以下の8つの「曖昧なプロンプト」を、明確で効果的なプロンプトに改善してください。

### 改善対象の曖昧なプロンプト

| 番号 | 曖昧なプロンプト | 想定タスク |
|:----:|-----------------|-----------|
| 1 | 「AIについて教えて」 | AIの概要説明 |
| 2 | 「いい感じのメール書いて」 | ビジネスメール作成 |
| 3 | 「このコードを直して」 | コードの改善 |
| 4 | 「面白い話を作って」 | ストーリー作成 |
| 5 | 「英語に直して」 | 翻訳 |
| 6 | 「レポートまとめて」 | レポート作成 |
| 7 | 「アイデアちょうだい」 | ブレインストーミング |
| 8 | 「もっと良くして」 | 文章の改善 |

### 各問題で実施すること

1. 曖昧なプロンプトの**問題点を3つ以上**指摘する
2. 明確化の5つのルール（具体的な動詞・数量指定・対象限定・条件明示・肯定表現）のうち、どれを適用するか決める
3. 改善後のプロンプトを作成する
4. 改善前と改善後の両方をLLMに入力し、出力の違いを比較する

## ステップガイド

<details>
<summary>ステップ1：問題点の分析方法を学ぶ</summary>

曖昧なプロンプトの問題点を見つけるには、以下のチェックリストを使いましょう。

**曖昧さチェックリスト**:
- [ ] 何についての話か不明確ではないか？（対象の曖昧さ）
- [ ] 何をしてほしいのか不明確ではないか？（動作の曖昧さ）
- [ ] どの程度の量・深さが不明確ではないか？（範囲の曖昧さ）
- [ ] 誰向けの出力か不明確ではないか？（対象読者の曖昧さ）
- [ ] どのような形式で出力すべきか不明確ではないか？（形式の曖昧さ）
- [ ] 主観的な表現（「いい感じ」「面白い」等）が含まれていないか？

**例：「AIについて教えて」の問題点分析**:
1. 「AI」の範囲が広すぎる（機械学習？生成AI？歴史？）
2. 「教えて」の深さが不明（概要？詳細？）
3. 対象読者が不明（専門家向け？初心者向け？）
4. 出力形式が指定されていない
5. 知りたい観点が不明（技術的仕組み？ビジネス活用？倫理問題？）

</details>

<details>
<summary>ステップ2：5つのルールを適用する</summary>

第2章で学んだ「明確な指示を書くための5つのルール」を適用しましょう。

| ルール | 内容 | 適用例 |
|--------|------|--------|
| 具体的な動詞 | 「教えて」→「3つの観点で比較して」 | 問題1、4、7 |
| 数量の指定 | 「いくつか」→「5つ」 | 問題7 |
| 対象の限定 | 「AI」→「生成AIのビジネス活用」 | 問題1、6 |
| 条件の明示 | 「いい感じ」→「丁寧語で300文字以内」 | 問題2、5 |
| 肯定表現 | 「難しくなく」→「中学生でも分かるように」 | 問題4、8 |

各問題に最低2つのルールを適用してください。

</details>

<details>
<summary>ステップ3：改善プロンプトを作成する</summary>

問題1「AIについて教えて」を例に、段階的に改善してみましょう。

**レベル1（最低限の改善）**:
```
生成AIの基本的な仕組みを、初心者向けに300文字以内で説明してください。
```
適用ルール：対象の限定、条件の明示

**レベル2（さらに明確に）**:
```
生成AI（Generative AI）について、以下の3つの観点で初心者向けに説明してください。
各観点は200文字以内でまとめてください。

1. 生成AIとは何か（定義と従来のAIとの違い）
2. 生成AIの主な用途（テキスト生成、画像生成、コード生成）
3. 生成AIを使う際の注意点（ハルシネーション、著作権）
```
適用ルール：具体的な動詞、数量の指定、対象の限定、条件の明示

残りの問題2〜8も同様に改善してください。

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

### 問題1：「AIについて教えて」

**問題点**:
1. 「AI」の範囲が広すぎる（どの分野のAI？）
2. 「教えて」が曖昧（概要？詳細？歴史？）
3. 対象読者の想定がない
4. 出力の分量が不明

**適用ルール**: 対象の限定、数量の指定、条件の明示

**改善後のプロンプト**:
```
生成AI（Generative AI）の基本について、以下の3つの観点で説明してください。
対象読者はITに詳しくない社会人です。各観点は150文字以内でまとめてください。

1. 生成AIとは何か
2. 日常生活での活用例を3つ
3. 使う際に知っておくべき注意点
```

---

### 問題2：「いい感じのメール書いて」

**問題点**:
1. 「いい感じ」が主観的で曖昧
2. メールの種類（ビジネス？私信？）が不明
3. 宛先・目的が不明
4. トーンや形式の指定がない

**適用ルール**: 具体的な動詞、対象の限定、条件の明示

**改善後のプロンプト**:
```
取引先の担当者（山田様）に、打ち合わせの日程調整をお願いするビジネスメールを
作成してください。

条件：
- 丁寧語（です・ます調）を使用
- 候補日を3つ提示する形式
- メールの長さは200文字以内
- 件名も含めて出力
```

---

### 問題7：「アイデアちょうだい」

**問題点**:
1. 何のアイデアか不明
2. 求める数が不明
3. 制約条件がない
4. アイデアの粒度（大まかな方向性？具体的な施策？）が不明

**適用ルール**: 具体的な動詞、数量の指定、対象の限定

**改善後のプロンプト**:
```
社内の新入社員歓迎会の企画アイデアを7つ提案してください。

条件：
- 参加人数：20名程度
- 予算：1人あたり3,000円以内
- 場所：オフィスの会議室またはオンライン
- 各アイデアには「概要（1文）」「メリット」「必要な準備」を記載
```

---

### 自分なりのチェックリスト

1. 「誰に向けた出力か」を明記しているか
2. 出力の分量（文字数・項目数）を指定しているか
3. 主観的な表現（「いい感じ」「面白い」）を具体的な条件に置き換えているか
4. タスクの背景・目的を含めているか
5. 出力形式（箇条書き、表、段落など）を指定しているか

<summary>解答例（改良版）</summary>

### 問題1：「AIについて教えて」（改良版）

**問題点の詳細分析**:
1. **対象の曖昧さ**: 「AI」は機械学習、深層学習、生成AI、ロボティクスなど広範な分野を含む
2. **動作の曖昧さ**: 「教えて」は説明・要約・比較・推薦など複数の解釈が可能
3. **範囲の曖昧さ**: 技術的仕組み、ビジネス活用、倫理問題、歴史など方向性が不明
4. **読者レベルの欠如**: 専門家と初心者では必要な情報の粒度が異なる
5. **出力制約の欠如**: 長さ、形式、スタイルが未定義

**改善後のプロンプト**:
```
あなたはIT教育の専門家です。

生成AI（Generative AI）について、IT業界への転職を考えている非エンジニアの
社会人向けに、入門ガイドを作成してください。

以下の4つのセクションで構成してください：

## 1. 生成AIとは（100文字以内）
従来のAIとの違いを1つの比喩を使って説明してください。

## 2. 主要サービス（表形式）
| サービス名 | 得意なこと | 無料で試せるか |
の形式で、代表的な3サービスを紹介してください。

## 3. ビジネスでの活用シーン（箇条書き5つ）
非エンジニアでもすぐに実践できるものを優先してください。

## 4. 注意すべきポイント（箇条書き3つ）
特にハルシネーション（hallucination）について必ず触れてください。
```

**改善のポイント**:
- 役割設定を追加し、教育的な視点での出力を誘導
- 対象読者を「IT業界への転職を考えている非エンジニアの社会人」に具体化
- セクション構成を明示し、出力の構造を完全に制御
- 各セクションに個別の条件（比喩を使う、表形式、非エンジニア向けなど）を設定
- 必須トピック（ハルシネーション）を指定し、重要な内容の欠落を防止

---

### 問題8：「もっと良くして」（改良版）

**問題点の詳細分析**:
1. 「もっと良く」が極めて主観的（読みやすさ？正確性？説得力？）
2. 「して」が曖昧（書き直す？追記する？削除する？）
3. 改善対象のテキストが提示されていない（前提条件の欠如）
4. 改善の基準・ゴールが不明
5. 現状の何が問題なのかが示されていない

**改善後のプロンプト**:
```
以下の文章を、3つの観点で改善してください。

【改善対象の文章】
---
弊社の新サービスはすごく良いです。いろいろな機能があって便利です。
ぜひ使ってみてください。
---

【改善の観点】
1. **具体性**: 「すごく良い」「いろいろな」を具体的な事実・数値に置き換える
2. **説得力**: 読者がサービスを試したくなる根拠を追加する
3. **構成**: 「課題提起 → 解決策 → 行動喚起」の流れに再構成する

【制約条件】
- 200文字以内
- ビジネス文書として適切な敬体
- 改善前の内容と大きく乖離しないこと

【出力形式】
1. 改善後の文章
2. 改善のポイント一覧（各観点で何を変えたか1文ずつ）
```

---

### 全問題の改善ルール整理

| 問題 | 最も効果的だったルール | 改善効果 |
|:----:|----------------------|---------|
| 1 | 対象の限定 + セクション構成指定 | 焦点が明確になり、網羅性が向上 |
| 2 | 条件の明示 + 出力形式指定 | ビジネスメールとして過不足ない内容に |
| 3 | 具体的な動詞 + 条件の明示 | 改善の方向性が明確に |
| 4 | 対象の限定 + 数量の指定 | 創作の方向性と規模が制御可能に |
| 5 | 条件の明示 + 文脈の追加 | 翻訳の精度とニュアンスが向上 |
| 6 | 対象の限定 + 出力形式指定 | レポートの構成と深さが適切に |
| 7 | 数量の指定 + 条件の明示 | 実行可能な具体的アイデアが得られた |
| 8 | 具体的な動詞 + 評価基準の提示 | 改善の方向性が一貫し、比較可能に |

**初心者向けとの違い**:
- 問題点の分析をより体系的に（5つの曖昧さカテゴリで分類）
- 改善後のプロンプトに「構成指定」と「個別条件」を追加
- 「なぜその改善が効果的か」の理由を明文化
- 全問題を横断した改善ルールの整理表を作成

</details>
