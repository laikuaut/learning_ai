# 第5章：シェルスクリプト基礎

## この章のゴール

- シェルスクリプトの基本的な書き方を理解し、自分で書けるようになること
- 変数、条件分岐、繰り返し、関数といったプログラミングの基本構文を習得すること
- パイプとリダイレクトを使いこなし、コマンドを効率的に組み合わせられるようになること
- 実務でよく使う自動化スクリプトを作成できるようになること

---

## 5.1 シェルスクリプトとは

シェルスクリプトとは、シェル（bashなど）で実行するコマンドをファイルにまとめたものです。複数のコマンドを順番に実行したり、条件に応じて処理を分岐したりできます。サーバー管理の自動化には欠かせない技術です。

### 最初のシェルスクリプト

```bash
#!/bin/bash
# hello.sh - 最初のシェルスクリプト

echo "Hello, World!"
echo "現在の日時: $(date)"
echo "ホスト名: $(hostname)"
echo "カレントディレクトリ: $(pwd)"
```

### シェルスクリプトの作成と実行

```bash
# 1. ファイルを作成
$ vim hello.sh

# 2. 実行権限を付与
$ chmod +x hello.sh

# 3. 実行
$ ./hello.sh
Hello, World!
現在の日時: Thu Mar 20 15:30:00 JST 2024
ホスト名: myserver
カレントディレクトリ: /home/tanaka

# bash コマンドで直接実行することもできる（実行権限不要）
$ bash hello.sh
Hello, World!
...
```

### シバン（Shebang）

スクリプトの1行目の `#!/bin/bash` は**シバン（shebang）**と呼ばれ、このスクリプトをどのインタプリタで実行するかを指定します。

```bash
#!/bin/bash       # bashで実行
#!/bin/sh         # shで実行（POSIX互換）
#!/usr/bin/env bash  # 環境に依存しない書き方（推奨）
#!/usr/bin/python3   # Pythonスクリプトの場合
```

> **推奨**: `#!/usr/bin/env bash` と書くと、環境変数PATHからbashを自動的に探してくれるため、bashのパスが環境によって異なる場合にも対応できます。

## 5.2 変数

### 変数の基本

```bash
#!/bin/bash

# 変数の代入（= の前後にスペースを入れてはならない！）
name="Tanaka"
age=30
server_name="web-server-01"

# 変数の参照（$を付ける）
echo "名前: $name"
echo "年齢: $age"
echo "サーバー名: $server_name"
```

実行結果：
```
名前: Tanaka
年齢: 30
サーバー名: web-server-01
```

### 変数代入の注意点

```bash
# 【正しい】= の前後にスペースなし
name="Tanaka"

# 【誤り】スペースがあるとコマンドと解釈される
name = "Tanaka"    # エラー: name: command not found

# 文字列にスペースが含まれる場合はクォートが必要
greeting="Hello World"
path="/home/tanaka/my documents"    # スペースを含むパスもクォート
```

### 変数の展開とクォーティング

```bash
#!/bin/bash
name="Tanaka"

# ダブルクォート: 変数が展開される
echo "Hello, $name"          # Hello, Tanaka

# シングルクォート: 変数が展開されない（リテラル）
echo 'Hello, $name'          # Hello, $name

# 波括弧で変数名を明確にする
file="report"
echo "${file}_2024.txt"      # report_2024.txt
echo "$file_2024.txt"        # 空（$file_2024 という変数を探してしまう）

# コマンドの実行結果を変数に代入
today=$(date +%Y-%m-%d)
echo "今日の日付: $today"    # 今日の日付: 2024-03-20

# バッククォートでも同じ（古い書き方）
today=`date +%Y-%m-%d`
```

### 特殊な変数

| 変数 | 意味 |
|------|------|
| `$0` | スクリプト名 |
| `$1`, `$2`, ... | 引数（位置パラメータ） |
| `$#` | 引数の個数 |
| `$@` | すべての引数（個別に展開） |
| `$*` | すべての引数（一つの文字列として） |
| `$?` | 直前のコマンドの終了ステータス（0=成功） |
| `$$` | 現在のシェルのPID |
| `$!` | 最後にバックグラウンドで実行したプロセスのPID |

```bash
#!/bin/bash
# special_vars.sh

echo "スクリプト名: $0"
echo "第1引数: $1"
echo "第2引数: $2"
echo "引数の個数: $#"
echo "すべての引数: $@"
echo "終了ステータス: $?"
echo "現在のPID: $$"
```

実行結果：
```bash
$ ./special_vars.sh hello world extra
スクリプト名: ./special_vars.sh
第1引数: hello
第2引数: world
引数の個数: 3
すべての引数: hello world extra
終了ステータス: 0
現在のPID: 12345
```

### 環境変数と局所変数

```bash
# 環境変数（子プロセスに引き継がれる）
export DB_HOST="localhost"
export DB_PORT="3306"

# 局所変数（現在のシェルのみ）
local_var="これは局所変数です"

# 主な環境変数
echo $HOME       # ホームディレクトリ
echo $USER       # ユーザー名
echo $PATH       # コマンドの検索パス
echo $SHELL      # 使用中のシェル
echo $PWD        # カレントディレクトリ
echo $LANG       # 言語設定

# 環境変数の一覧
$ env
# または
$ printenv
```

### readonly（読み取り専用変数）

```bash
readonly MAX_RETRY=3
MAX_RETRY=5    # エラー: readonly variable
```

## 5.3 条件分岐（if文）

### 基本構文

```bash
if [ 条件 ]; then
    # 条件が真の場合の処理
elif [ 条件2 ]; then
    # 条件2が真の場合の処理
else
    # どの条件も偽の場合の処理
fi
```

### 文字列の比較

| 演算子 | 意味 | 例 |
|--------|------|-----|
| `=` | 等しい | `[ "$a" = "$b" ]` |
| `!=` | 等しくない | `[ "$a" != "$b" ]` |
| `-z` | 空文字列 | `[ -z "$a" ]` |
| `-n` | 空でない | `[ -n "$a" ]` |

### 数値の比較

| 演算子 | 意味 | 例 |
|--------|------|-----|
| `-eq` | 等しい（equal） | `[ "$a" -eq "$b" ]` |
| `-ne` | 等しくない（not equal） | `[ "$a" -ne "$b" ]` |
| `-gt` | より大きい（greater than） | `[ "$a" -gt "$b" ]` |
| `-ge` | 以上（greater or equal） | `[ "$a" -ge "$b" ]` |
| `-lt` | より小さい（less than） | `[ "$a" -lt "$b" ]` |
| `-le` | 以下（less or equal） | `[ "$a" -le "$b" ]` |

### ファイルテスト

| 演算子 | 意味 |
|--------|------|
| `-e FILE` | ファイルが存在する |
| `-f FILE` | 通常ファイルである |
| `-d FILE` | ディレクトリである |
| `-r FILE` | 読み取り可能 |
| `-w FILE` | 書き込み可能 |
| `-x FILE` | 実行可能 |
| `-s FILE` | サイズが0でない |
| `-L FILE` | シンボリックリンクである |

### if文の実用例

```bash
#!/bin/bash
# check_file.sh - ファイルの存在確認

FILE="/etc/nginx/nginx.conf"

if [ -f "$FILE" ]; then
    echo "$FILE は存在します"
    echo "サイズ: $(stat -c %s "$FILE") バイト"
else
    echo "$FILE は存在しません"
    echo "nginxがインストールされていない可能性があります"
fi
```

実行結果：
```
/etc/nginx/nginx.conf は存在します
サイズ: 1482 バイト
```

```bash
#!/bin/bash
# check_disk.sh - ディスク使用率チェック

THRESHOLD=80
USAGE=$(df / | tail -1 | awk '{print $5}' | tr -d '%')

echo "ディスク使用率: ${USAGE}%"

if [ "$USAGE" -ge "$THRESHOLD" ]; then
    echo "【警告】ディスク使用率が${THRESHOLD}%を超えています！"
    echo "不要なファイルを削除してください"
elif [ "$USAGE" -ge 60 ]; then
    echo "【注意】ディスク使用率が60%を超えています"
else
    echo "ディスク使用率は正常です"
fi
```

実行結果：
```
ディスク使用率: 72%
【注意】ディスク使用率が60%を超えています
```

### [[ ]] の使い方（bash拡張）

`[[ ]]` は bash の拡張構文で、`[ ]` よりも高機能です。

```bash
#!/bin/bash

# 正規表現マッチング（[[ ]] でのみ使用可能）
filename="backup_2024-03-20.tar.gz"
if [[ "$filename" =~ ^backup_[0-9]{4} ]]; then
    echo "バックアップファイルです"
fi

# AND/OR 条件
age=25
if [[ "$age" -ge 18 && "$age" -lt 65 ]]; then
    echo "労働年齢です"
fi

# [ ] の場合は -a / -o を使う
if [ "$age" -ge 18 -a "$age" -lt 65 ]; then
    echo "労働年齢です"
fi

# 変数の引用符を省略しても安全（[[ ]] の利点）
name=""
if [[ -z $name ]]; then    # [ ] だとエラーになる場合がある
    echo "名前が空です"
fi
```

### 論理演算子

```bash
# AND 条件
if [ -f "$FILE" ] && [ -r "$FILE" ]; then
    echo "ファイルが存在し、読み取り可能です"
fi

# OR 条件
if [ -f "$FILE" ] || [ -d "$FILE" ]; then
    echo "ファイルまたはディレクトリが存在します"
fi

# NOT 条件
if [ ! -f "$FILE" ]; then
    echo "ファイルが存在しません"
fi
```

## 5.4 case文

複数の条件を判定する場合、if-elif を繰り返すよりも `case` 文が読みやすくなります。

```bash
#!/bin/bash
# service_control.sh - サービス制御スクリプト

SERVICE="nginx"

case "$1" in
    start)
        echo "${SERVICE} を起動します..."
        sudo systemctl start "$SERVICE"
        ;;
    stop)
        echo "${SERVICE} を停止します..."
        sudo systemctl stop "$SERVICE"
        ;;
    restart)
        echo "${SERVICE} を再起動します..."
        sudo systemctl restart "$SERVICE"
        ;;
    status)
        systemctl status "$SERVICE"
        ;;
    *)
        echo "使い方: $0 {start|stop|restart|status}"
        exit 1
        ;;
esac
```

実行結果：
```bash
$ ./service_control.sh start
nginx を起動します...

$ ./service_control.sh help
使い方: ./service_control.sh {start|stop|restart|status}
```

### パターンマッチの活用

```bash
#!/bin/bash
# file_type.sh - ファイル種類の判定

case "$1" in
    *.tar.gz|*.tgz)
        echo "gzip圧縮されたtarアーカイブです"
        echo "展開: tar xzf $1"
        ;;
    *.tar.bz2)
        echo "bzip2圧縮されたtarアーカイブです"
        echo "展開: tar xjf $1"
        ;;
    *.zip)
        echo "ZIPアーカイブです"
        echo "展開: unzip $1"
        ;;
    *.log)
        echo "ログファイルです"
        echo "末尾を表示: tail -f $1"
        ;;
    *.sh)
        echo "シェルスクリプトです"
        ;;
    *)
        echo "不明なファイル形式です: $1"
        ;;
esac
```

## 5.5 繰り返し（for文）

### 基本構文

```bash
for 変数 in リスト; do
    # 処理

```

### for文の実用例

```bash
#!/bin/bash

# リストを指定して繰り返し
for fruit in apple banana cherry; do
    echo "フルーツ: $fruit"
done
```

実行結果：
```
フルーツ: apple
フルーツ: banana
フルーツ: cherry
```

```bash
#!/bin/bash
# 数値の範囲で繰り返し
for i in {1..5}; do
    echo "カウント: $i"
done
```

実行結果：
```
カウント: 1
カウント: 2
カウント: 3
カウント: 4
カウント: 5
```

```bash
#!/bin/bash
# ステップ付き（2刻み）
for i in {0..10..2}; do
    echo "$i"
done
```

実行結果：
```
0
2
4
6
8
10
```

```bash
#!/bin/bash
# C言語スタイルのfor文
for ((i=1; i<=5; i++)); do
    echo "番号: $i"
done

# ファイルの一覧を処理
for file in /var/log/*.log; do
    echo "ログファイル: $file ($(wc -l < "$file") 行)"
done

# コマンドの出力を処理
for user in $(cat /etc/passwd | cut -d: -f1); do
    echo "ユーザー: $user"
done
```

### 実務での使用例

```bash
#!/bin/bash
# multi_server_check.sh - 複数サーバーの死活監視

SERVERS="web01 web02 web03 db01 db02"

for server in $SERVERS; do
    if ping -c 1 -W 2 "$server" > /dev/null 2>&1; then
        echo "[OK] $server は応答しています"
    else
        echo "[NG] $server が応答しません！"
    fi
done
```

実行結果：
```
[OK] web01 は応答しています
[OK] web02 は応答しています
[NG] web03 が応答しません！
[OK] db01 は応答しています
[OK] db02 は応答しています
```

## 5.6 繰り返し（while文）

### 基本構文

```bash
while [ 条件 ]; do
    # 処理
done
```

### while文の実用例

```bash
#!/bin/bash
# カウントダウン
count=5
while [ "$count" -gt 0 ]; do
    echo "$count..."
    count=$((count - 1))
    sleep 1
done
echo "発射！"
```

実行結果：
```
5...
4...
3...
2...
1...
発射！
```

```bash
#!/bin/bash
# ファイルを1行ずつ読み込む（最も重要なパターン）
while IFS= read -r line; do
    echo "行: $line"
done < /etc/hosts
```

実行結果：
```
行: 127.0.0.1 localhost
行: 127.0.1.1 myserver
行: ::1 ip6-localhost ip6-loopback
```

```bash
#!/bin/bash
# サービスが起動するまで待機
MAX_RETRY=30
count=0

while [ "$count" -lt "$MAX_RETRY" ]; do
    if curl -s http://localhost:8080/health > /dev/null 2>&1; then
        echo "サービスが起動しました（${count}秒後）"
        exit 0
    fi
    echo "待機中... (${count}/${MAX_RETRY})"
    count=$((count + 1))
    sleep 1
done

echo "タイムアウト：サービスが起動しませんでした"
exit 1
```

### 無限ループ

```bash
#!/bin/bash
# 監視スクリプト（Ctrl+Cで停止）
while true; do
    echo "$(date): CPU使用率 $(top -bn1 | grep "Cpu(s)" | awk '{print $2}')%"
    sleep 10
done
```

## 5.7 関数

### 基本構文

```bash
# 定義方法1
function_name() {
    # 処理
}

# 定義方法2
function function_name {
    # 処理
}
```

### 関数の実用例

```bash
#!/bin/bash

# ログ出力関数
log_info() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [INFO] $1"
}

log_error() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [ERROR] $1" >&2
}

log_info "スクリプトを開始します"
log_info "処理を実行中..."
log_error "ファイルが見つかりません"
log_info "スクリプトを終了します"
```

実行結果：
```
[2024-03-20 15:30:00] [INFO] スクリプトを開始します
[2024-03-20 15:30:00] [INFO] 処理を実行中...
[2024-03-20 15:30:00] [ERROR] ファイルが見つかりません
[2024-03-20 15:30:00] [INFO] スクリプトを終了します
```

### 引数と戻り値

```bash
#!/bin/bash

# 引数を受け取る関数
greet() {
    local name="$1"    # local で局所変数を定義
    local time="$2"
    echo "こんにちは、${name}さん。今は${time}です。"
}

greet "田中" "15:30"
greet "鈴木" "16:00"

# 戻り値を返す関数（return は 0-255 の終了ステータスのみ）
is_file_exists() {
    local file="$1"
    if [ -f "$file" ]; then
        return 0    # 成功（真）
    else
        return 1    # 失敗（偽）
    fi
}

if is_file_exists "/etc/hosts"; then
    echo "/etc/hosts は存在します"
fi

# 文字列を返したい場合はechoで出力して受け取る
get_hostname() {
    hostname
}

MY_HOST=$(get_hostname)
echo "ホスト名: $MY_HOST"
```

実行結果：
```
こんにちは、田中さん。今は15:30です。
こんにちは、鈴木さん。今は16:00です。
/etc/hosts は存在します
ホスト名: myserver
```

## 5.8 パイプとリダイレクト

### パイプ（|）

パイプは、あるコマンドの標準出力を次のコマンドの標準入力に渡す仕組みです。Linuxの最も強力な機能の一つです。

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ コマンド1 │───→│ コマンド2 │───→│ コマンド3 │───→ 結果
│  (出力)   │ │  │  (入出力) │ │  │  (入力)   │
└──────────┘ │  └──────────┘ │  └──────────┘
             パイプ          パイプ
```

```bash
# 基本的なパイプの使い方
$ cat /etc/passwd | wc -l              # ユーザー数を数える
35

$ ps aux | grep nginx                  # nginxプロセスを検索

$ ls -la | sort -k5 -n                 # ファイルサイズ順にソート

$ cat access.log | sort | uniq -c | sort -rn | head -10
# アクセスログで最も多いIPアドレスのトップ10

# 複数のパイプを連結
$ cat /etc/passwd | cut -d: -f1 | sort | head -5
daemon
games
gnats
list
lp
```

### リダイレクト

リダイレクトはコマンドの入出力先を変更する仕組みです。

```
┌───────────────────────────────────────────────────┐
│               ファイルディスクリプタ                 │
│                                                   │
│  0: 標準入力（stdin）  ← キーボードから入力        │
│  1: 標準出力（stdout） → 画面に出力               │
│  2: 標準エラー出力（stderr） → 画面にエラーを出力  │
└───────────────────────────────────────────────────┘
```

| リダイレクト | 意味 | 説明 |
|-------------|------|------|
| `>` | 出力リダイレクト | ファイルに書き込む（上書き） |
| `>>` | 追記リダイレクト | ファイルに追記する |
| `<` | 入力リダイレクト | ファイルから読み込む |
| `2>` | エラー出力リダイレクト | エラーをファイルに書き込む |
| `2>&1` | エラーを標準出力にマージ | エラーも含めてリダイレクト |
| `&>` | 両方をリダイレクト | 標準出力とエラーを同時に |
| `/dev/null` | 出力を破棄 | ブラックホール |

### リダイレクトの使用例

```bash
# 標準出力をファイルに書き込む（上書き）
$ ls -la > file_list.txt
$ cat file_list.txt
total 32
drwxr-xr-x 2 tanaka tanaka 4096 ...

# ファイルに追記する
$ echo "追加の行" >> file_list.txt

# 標準エラーをファイルに書き込む
$ ls /nonexistent 2> error.log
$ cat error.log
ls: cannot access '/nonexistent': No such file or directory

# 標準出力とエラー出力を別々のファイルに
$ ./script.sh > output.log 2> error.log

# 標準出力とエラー出力を同じファイルに
$ ./script.sh > all.log 2>&1
# または（簡潔な書き方）
$ ./script.sh &> all.log

# 出力を破棄（画面に何も表示しない）
$ ping -c 1 example.com > /dev/null 2>&1

# ファイルから入力を読み込む
$ wc -l < /etc/passwd
35

# ヒアドキュメント（複数行の入力）
$ cat << EOF > config.txt
server_name=web01
port=8080
environment=production
EOF

$ cat config.txt
server_name=web01
port=8080
environment=production
```

### teeコマンド（画面とファイルに同時出力）

```bash
# 画面にも表示しつつファイルにも保存する
$ ls -la | tee file_list.txt
total 32
drwxr-xr-x 2 tanaka tanaka 4096 ...
# ↑ 画面に表示されると同時に file_list.txt にも保存される

# 追記モード
$ echo "追加" | tee -a file_list.txt
```

## 5.9 コマンドの実行制御

### 終了ステータス

すべてのコマンドは終了時に**終了ステータス**を返します。`0` は成功、`0以外` は失敗です。

```bash
$ ls /etc/hosts
/etc/hosts
$ echo $?
0              # 成功

$ ls /nonexistent
ls: cannot access '/nonexistent': No such file or directory
$ echo $?
2              # 失敗
```

### コマンドの連結

```bash
# && : 左のコマンドが成功した場合のみ右を実行
$ mkdir newdir && cd newdir
# mkdirが成功した場合のみ cd する

# || : 左のコマンドが失敗した場合のみ右を実行
$ cd /opt/app || echo "ディレクトリに移動できません"

# ; : 成功/失敗に関係なく順番に実行
$ echo "処理1"; echo "処理2"; echo "処理3"

# 組み合わせ例
$ ping -c 1 server01 > /dev/null 2>&1 && echo "OK" || echo "NG"
OK
```

### set コマンドによるエラー制御

```bash
#!/bin/bash

# エラーが発生したら即座にスクリプトを終了する（推奨）
set -e

# 未定義の変数を使うとエラーにする
set -u

# パイプの途中でエラーが発生しても検出する
set -o pipefail

# デバッグモード（実行するコマンドを表示する）
set -x

# まとめて設定（本番スクリプトの冒頭に書く推奨パターン）
set -euo pipefail
```

## 5.10 配列

```bash
#!/bin/bash

# 配列の定義
servers=("web01" "web02" "web03" "db01" "db02")

# 要素の参照
echo "最初のサーバー: ${servers[0]}"    # web01
echo "2番目: ${servers[1]}"             # web02

echo "全サーバー: ${servers[@]}"        # web01 web02 web03 db01 db02

# 要素数
echo "サーバー数: ${#servers[@]}"       # 5

# 配列をループ
for server in "${servers[@]}"; do
    echo "サーバー: $server"
done

# 要素の追加
servers+=("cache01")
echo "追加後: ${servers[@]}"            # web01 web02 web03 db01 db02 cache01
```

## 5.11 実用的なスクリプト例

### バックアップスクリプト

```bash
#!/bin/bash
# backup.sh - ディレクトリのバックアップ
set -euo pipefail

# 設定
BACKUP_SRC="/var/www/html"
BACKUP_DST="/backup"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DST}/backup_${DATE}.tar.gz"
MAX_BACKUPS=7

# ログ関数
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1"
}

# バックアップ先ディレクトリの確認
if [ ! -d "$BACKUP_DST" ]; then
    mkdir -p "$BACKUP_DST"
    log "バックアップディレクトリを作成: $BACKUP_DST"
fi

# バックアップの実行
log "バックアップ開始: $BACKUP_SRC → $BACKUP_FILE"
tar czf "$BACKUP_FILE" -C "$(dirname "$BACKUP_SRC")" "$(basename "$BACKUP_SRC")"

# 結果の確認
if [ -f "$BACKUP_FILE" ]; then
    SIZE=$(du -h "$BACKUP_FILE" | cut -f1)
    log "バックアップ完了: $BACKUP_FILE ($SIZE)"
else
    log "バックアップ失敗！"
    exit 1
fi

# 古いバックアップの削除
BACKUP_COUNT=$(ls -1 "${BACKUP_DST}"/backup_*.tar.gz 2>/dev/null | wc -l)
if [ "$BACKUP_COUNT" -gt "$MAX_BACKUPS" ]; then
    DELETE_COUNT=$((BACKUP_COUNT - MAX_BACKUPS))
    ls -1t "${BACKUP_DST}"/backup_*.tar.gz | tail -n "$DELETE_COUNT" | while read -r old_file; do
        rm "$old_file"
        log "古いバックアップを削除: $old_file"
    done
fi

log "バックアップ処理完了"
```

## 5.12 よくある間違い

### 間違い1: 変数代入時にスペースを入れる

```bash
# 【誤り】
name = "Tanaka"    # エラー

# 【正解】
name="Tanaka"
```

### 間違い2: 変数をクォートしない

```bash
# 【誤り】ファイル名にスペースがあると壊れる
FILE=my file.txt
if [ -f $FILE ]; then    # エラー: too many arguments

# 【正解】常にダブルクォートで囲む
FILE="my file.txt"
if [ -f "$FILE" ]; then
```

### 間違い3: [ ] の前後にスペースがない

```bash
# 【誤り】
if [$a -eq $b]; then    # エラー

# 【正解】[ ] の前後にスペースが必要
if [ "$a" -eq "$b" ]; then
```

### 間違い4: = と == の混同

```bash
# [ ] 内では = を使う（POSIX互換）
if [ "$a" = "$b" ]; then

# [[ ]] 内では == も使える
if [[ "$a" == "$b" ]]; then
```

## 5.13 現場での注意点

1. **スクリプトの冒頭に `set -euo pipefail` を書く**: エラーの早期検出に必須
2. **変数は常にダブルクォートで囲む**: 空白やグロブの誤展開を防ぐ
3. **コマンドはフルパスで書く**: cron等から実行する場合、PATHが異なる場合がある
4. **ログを必ず残す**: 何が起きたか後から追跡できるようにする
5. **テスト環境で十分にテストしてから本番に反映する**
