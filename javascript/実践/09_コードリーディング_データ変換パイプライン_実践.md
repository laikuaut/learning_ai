## 読み解くコード

```javascript
// ===== データ変換パイプライン =====
// 売上CSVデータを分析するユーティリティ

const csvData = `date,product,category,price,quantity
2024-01-15,ノートPC,電子機器,89000,3
2024-01-15,マウス,電子機器,2500,10
2024-01-20,デスク,家具,35000,2
2024-01-22,チェア,家具,28000,4
2024-02-01,キーボード,電子機器,8000,7
2024-02-05,本棚,家具,22000,1
2024-02-10,モニター,電子機器,45000,5
2024-02-15,ランプ,家具,6000,8
2024-03-01,タブレット,電子機器,55000,4
2024-03-10,ソファ,家具,120000,1`;

// --- ステップ1: CSV解析 ---
function parseCSV(csv) {
  const lines = csv.trim().split("\n");
  const headers = lines[0].split(",");

  return lines.slice(1).map(line => {
    const values = line.split(",");
    const obj = {};
    headers.forEach((header, i) => {
      obj[header] = values[i];
    });
    return obj;
  });
}

// --- ステップ2: 型変換 ---
function convertTypes(records) {
  return records.map(record => ({
    ...record,
    price: Number(record.price),
    quantity: Number(record.quantity),
    get total() { return this.price * this.quantity; }
  }));
}

// --- ステップ3: フィルタリング ---
const filterByCategory = (records, category) =>
  records.filter(r => r.category === category);

const filterByDateRange = (records, start, end) =>
  records.filter(r => r.date >= start && r.date <= end);

const filterByMinTotal = (records, min) =>
  records.filter(r => r.total >= min);

// --- ステップ4: 集計 ---
function summarize(records) {
  const totalRevenue = records.reduce((sum, r) => sum + r.total, 0);
  const avgPrice = records.length > 0
    ? records.reduce((sum, r) => sum + r.price, 0) / records.length
    : 0;
  const totalQuantity = records.reduce((sum, r) => sum + r.quantity, 0);

  return { totalRevenue, avgPrice, totalQuantity, count: records.length };
}

// --- ステップ5: グループ化 ---
function groupBy(records, key) {
  return records.reduce((groups, record) => {
    const groupKey = record[key];
    if (!groups[groupKey]) {
      groups[groupKey] = [];
    }
    groups[groupKey].push(record);
    return groups;
  }, {});
}

// --- ステップ6: ソートと上位抽出 ---
const sortByTotal = (records, desc = true) =>
  [...records].sort((a, b) => desc ? b.total - a.total : a.total - b.total);

const topN = (records, n) => sortByTotal(records).slice(0, n);

// --- ステップ7: レポート生成 ---
function generateReport(records) {
  const grouped = groupBy(records, "category");

  return Object.entries(grouped).map(([category, items]) => {
    const stats = summarize(items);
    const topItem = topN(items, 1)[0];

    return {
      category,
      ...stats,
      topProduct: topItem.product,
      topProductRevenue: topItem.total,
    };
  });
}

// --- パイプライン実行 ---
const raw = parseCSV(csvData);
const typed = convertTypes(raw);

console.log("=== 全データ ===");
console.log(typed);

console.log("\n=== 電子機器のみ ===");
const electronics = filterByCategory(typed, "電子機器");
console.log(electronics.map(r => `${r.product}: ${r.total.toLocaleString()}円`));

console.log("\n=== 2024年2月のデータ ===");
const feb = filterByDateRange(typed, "2024-02-01", "2024-02-28");
console.log(summarize(feb));

console.log("\n=== 売上TOP3 ===");
const top3 = topN(typed, 3);
console.log(top3.map(r => `${r.product}: ${r.total.toLocaleString()}円`));

console.log("\n=== カテゴリ別レポート ===");
const report = generateReport(typed);
console.log(report);
```

## 設問

### 設問1（基礎）：`parseCSV` の動作

<details>
<summary>問題</summary>

`parseCSV(csvData)` の戻り値の最初の要素はどのようなオブジェクトになりますか？
プロパティ名と値を具体的に書いてください。

</details>

<details>
<summary>解答</summary>

```javascript
{
  date: "2024-01-15",
  product: "ノートPC",
  category: "電子機器",
  price: "89000",   // ← この時点では全て文字列
  quantity: "3"      // ← 文字列のまま
}
```

**ポイント:**
- `split(",")` で分割した値は全て**文字列**です
- 数値変換は `convertTypes` で後から行われます
- ヘッダー行（`lines[0]`）はスキップされ、`lines.slice(1)` でデータ行だけ処理されます

</details>

### 設問2（中級）：スプレッド構文と `get` アクセサ

<details>
<summary>問題</summary>

`convertTypes` 関数内の以下のコードについて：

```javascript
return records.map(record => ({
  ...record,
  price: Number(record.price),
  quantity: Number(record.quantity),
  get total() { return this.price * this.quantity; }
}));
```

1. `...record` は何をしていますか？
2. `price: Number(record.price)` が `...record` の後にある理由は？
3. `get total()` はプロパティとメソッドのどちらですか？通常のメソッドとの違いは？

</details>

<details>
<summary>解答</summary>

1. **`...record`** はスプレッド構文で、`record` オブジェクトの全プロパティを新しいオブジェクトにコピーしています。
   `{ date: "2024-01-15", product: "ノートPC", category: "電子機器", price: "89000", quantity: "3" }`
   のプロパティが全て展開されます。

2. **`price: Number(record.price)` が後にある理由:** スプレッド構文で展開した `price: "89000"`（文字列）を、`price: 89000`（数値）で**上書き**するためです。オブジェクトリテラルでは、同じキーが複数あると後に書いたものが有効になります。

3. **`get total()`** は「ゲッター」と呼ばれる特殊なプロパティです。

   ```javascript
   // ゲッター: プロパティのようにアクセス（()不要）
   record.total     // → 89000 * 3 → 267000

   // 通常のメソッド: ()をつけて呼ぶ
   record.total()   // ← ゲッターの場合これはエラー
   ```

   ゲッターはアクセスされるたびに関数が実行され、計算結果を返します。`price` や `quantity` が変わると `total` の値も自動的に変わります。

</details>

### 設問3（中級）：`reduce` による集計

<details>
<summary>問題</summary>

`summarize(electronics)` の実行結果を予測してください。
（`electronics` は `filterByCategory(typed, "電子機器")` の結果）

ヒント: 電子機器の商品は「ノートPC(89000x3)、マウス(2500x10)、キーボード(8000x7)、モニター(45000x5)、タブレット(55000x4)」の5件です。

</details>

<details>
<summary>解答</summary>

```javascript
{
  totalRevenue: 89000*3 + 2500*10 + 8000*7 + 45000*5 + 55000*4,
  // = 267000 + 25000 + 56000 + 225000 + 220000 = 793000
  totalRevenue: 793000,

  avgPrice: (89000 + 2500 + 8000 + 45000 + 55000) / 5,
  // = 199500 / 5 = 39900
  avgPrice: 39900,

  totalQuantity: 3 + 10 + 7 + 5 + 4,
  // = 29
  totalQuantity: 29,

  count: 5
}
```

**`reduce` の動作:**
```javascript
// reduce((accumulator, currentValue) => 新しいaccumulator, 初期値)
records.reduce((sum, r) => sum + r.total, 0)
// 0 + 267000 → 267000 + 25000 → 292000 + 56000 → ... → 793000
```

`reduce` は配列の全要素を1つの値に畳み込む（reduce = 減らす）メソッドです。第2引数の `0` が初期値です。

</details>

### 設問4（上級）：`groupBy` の仕組み

<details>
<summary>問題</summary>

`groupBy(typed, "category")` は内部でどのように動作しますか？
`reduce` の各ステップで `groups` オブジェクトがどう変化するか、最初の3レコード分を追跡してください。

</details>

<details>
<summary>解答</summary>

```javascript
// 初期値: {}

// 1回目（ノートPC）:
// groupKey = "電子機器"
// groups["電子機器"] は undefined → 新規作成
// groups = { "電子機器": [{ product: "ノートPC", ... }] }

// 2回目（マウス）:
// groupKey = "電子機器"
// groups["電子機器"] は既に存在 → push
// groups = { "電子機器": [{ product: "ノートPC", ... }, { product: "マウス", ... }] }

// 3回目（デスク）:
// groupKey = "家具"
// groups["家具"] は undefined → 新規作成
// groups = {
//   "電子機器": [{ ... }, { ... }],
//   "家具": [{ product: "デスク", ... }]
// }
```

**ポイント:**
- `reduce` の初期値 `{}` が空のグループオブジェクト
- `record[key]` は `record["category"]` と同じ（ブラケット記法）
- `!groups[groupKey]` で「そのカテゴリが初めて出現したか」を判定
- 毎回 `groups` を `return` して次のステップに渡す

</details>

### 設問5（上級）：`sortByTotal` のスプレッド構文

<details>
<summary>問題</summary>

```javascript
const sortByTotal = (records, desc = true) =>
  [...records].sort((a, b) => desc ? b.total - a.total : a.total - b.total);
```

1. `[...records]` ではなく `records.sort(...)` と書いた場合、どのような問題が起きますか？
2. `desc = true` は何という機能ですか？
3. `topN(typed, 3)` の結果（商品名と売上）を予測してください。

</details>

<details>
<summary>解答</summary>

1. **`records.sort(...)` の問題:** `sort()` は**破壊的メソッド**で、元の配列を直接変更します。`[...records]` でコピーを作ることで、元の `records` 配列の順序を壊しません。データパイプラインでは元データを保持することが重要です。

2. **`desc = true`** はデフォルト引数（default parameter）です。引数が省略された場合に `true` が使われます。ES6で導入された機能です。

3. **`topN(typed, 3)` の結果:**

   全10商品の売上（price x quantity）:
   - ノートPC: 267,000円
   - マウス: 25,000円
   - デスク: 70,000円
   - チェア: 112,000円
   - キーボード: 56,000円
   - 本棚: 22,000円
   - モニター: 225,000円
   - ランプ: 48,000円
   - タブレット: 220,000円
   - ソファ: 120,000円

   TOP3:
   ```
   ノートPC: 267,000円
   モニター: 225,000円
   タブレット: 220,000円
   ```

</details>

### 設問6（上級）：`generateReport` の `Object.entries` と分割代入

<details>
<summary>問題</summary>

```javascript
Object.entries(grouped).map(([category, items]) => {
```

1. `Object.entries(grouped)` はどのような形式のデータを返しますか？
2. `([category, items])` の `[]` は何という構文ですか？なぜ必要ですか？
3. `generateReport(typed)` の戻り値の構造を予測してください。

</details>

<details>
<summary>解答</summary>

1. **`Object.entries()`** はオブジェクトの `[キー, 値]` のペアを配列の配列で返します。

   ```javascript
   Object.entries({ a: 1, b: 2 })
   // → [["a", 1], ["b", 2]]

   Object.entries(grouped)
   // → [["電子機器", [...5件の商品...]], ["家具", [...5件の商品...]]]
   ```

2. **`([category, items])`** は**分割代入（destructuring assignment）**です。`map` のコールバックが受け取る各要素（`["電子機器", [...]]`）を、配列の位置に基づいて `category` と `items` に分解して代入しています。

   分割代入を使わない場合:
   ```javascript
   .map(entry => {
     const category = entry[0];
     const items = entry[1];
     // ...
   })
   ```

3. **戻り値の構造:**
   ```javascript
   [
     {
       category: "電子機器",
       totalRevenue: 793000,
       avgPrice: 39900,
       totalQuantity: 29,
       count: 5,
       topProduct: "ノートPC",
       topProductRevenue: 267000
     },
     {
       category: "家具",
       totalRevenue: 372000,
       avgPrice: 42200,
       totalQuantity: 16,
       count: 5,
       topProduct: "ソファ",
       topProductRevenue: 120000
     }
   ]
   ```

</details>

## まとめ：このコードで学べるES6+のテクニック

| テクニック | 使用箇所 | 説明 |
|---|---|---|
| テンプレートリテラル | `${r.product}: ...` | 文字列埋め込み |
| アロー関数 | `records.map(r => ...)` | 関数の簡潔な記法 |
| スプレッド構文 | `{...record}`, `[...records]` | オブジェクト/配列のコピー |
| 分割代入 | `([category, items])` | 配列/オブジェクトの分解 |
| デフォルト引数 | `desc = true` | 引数の初期値 |
| ゲッター | `get total()` | 計算プロパティ |
| `reduce` | `groupBy`, `summarize` | 配列の畳み込み |
| `Object.entries` | `generateReport` | オブジェクトのイテレーション |
| メソッドチェーン | `.filter().map().sort()` | 処理の連結 |
