# 第4章：後方参照と先読み・後読み

## この章で学ぶこと

- 後方参照（Back Reference）`\1` `\2` の使い方
- 名前付きグループ（Named Group）`(?P<name>...)` と `(?P=name)`
- 肯定先読み（Positive Lookahead）`(?=...)` と否定先読み（Negative Lookahead）`(?!...)`
- 肯定後読み（Positive Lookbehind）`(?<=...)` と否定後読み（Negative Lookbehind）`(?<!...)`
- 実用例：パスワード強度チェック、重複単語の検出

---

## 4.1 後方参照（Back Reference）

### 後方参照とは

後方参照とは、**前のキャプチャグループでマッチした文字列と同じ文字列**にマッチさせる機能です。「さっきマッチしたものと同じもの」を指定できます。

`\1` は1番目のグループ、`\2` は2番目のグループでマッチした文字列を参照します。

```python
import re

# 同じ文字が2回繰り返されるパターン
text = "aabbcc abab hello llooll"

# (.)\1 → 任意の1文字(.)をキャプチャし、同じ文字(\1)が続く
results = re.findall(r"(.)\1", text)
print(results)  # ['a', 'b', 'c', 'l', 'o', 'l']
# ← aa, bb, cc, ll, oo, ll の最初の文字がキャプチャされている
```

```
【後方参照のイメージ】

パターン: (.)\1

テキスト: a a b b c c
          ↑ ↑
          │ └── \1 = グループ1と同じ文字（a）にマッチ
          └──── (.) = グループ1で 'a' をキャプチャ

"aa" → マッチ ✓（. が a をキャプチャ、\1 が a にマッチ）
"ab" → マッチ ✗（. が a をキャプチャ、\1 は a を期待するが b）
"bb" → マッチ ✓（. が b をキャプチャ、\1 が b にマッチ）
```

### 重複単語の検出

後方参照の実用的な使い方として、**うっかり同じ単語を連続して書いてしまった箇所**を検出できます。

```python
import re

text = "This is is a test test of the the system"

# (\b\w+)\s+\1\b → 単語をキャプチャし、空白の後に同じ単語が続く
results = re.findall(r"(\b\w+)\s+\1\b", text)
print(results)  # ['is', 'test', 'the']
```

```
【重複単語の検出パターン】

パターン: (\b\w+)\s+\1\b

(\b\w+)  → 単語境界から始まる1文字以上の単語文字をキャプチャ
\s+      → 1つ以上の空白
\1       → グループ1と同じ文字列
\b       → 単語境界

テキスト: "is is" → (\b\w+) が "is" をキャプチャ、\1 が "is" にマッチ → ✓
テキスト: "is at" → (\b\w+) が "is" をキャプチャ、\1 が "is" を期待するが "at" → ✗
```

### 重複単語を置換で修正

```python
import re

text = "This is is a test test of the the system"

# 重複単語を1つに置換
fixed = re.sub(r"(\b\w+)\s+\1\b", r"\1", text)
print(fixed)  # This is a test of the system
```

### HTMLの開始タグと閉じタグの対応

```python
import re

html = "<h1>見出し</h1> <p>本文</p> <h1>間違い</h2>"

# <(タグ名)>中身</同じタグ名> のパターン
results = re.findall(r"<(\w+)>(.+?)</\1>", html)
print(results)  # [('h1', '見出し'), ('p', '本文')]
# ← <h1>間違い</h2> は h1 ≠ h2 なのでマッチしない
```

> **よくある間違い**: `re.findall()` で `\1` を使う場合、`\1` はパターン内の後方参照であり、置換文字列ではありません。置換で使う `\1` は `re.sub()` の第2引数で使います。

## 4.2 名前付きグループ（Named Group）

### 名前付きグループとは

番号（`\1`, `\2`）の代わりに、グループに**名前**を付けて参照できます。パターンが複雑になったとき、番号より名前の方が分かりやすくなります。

- **定義**: `(?P<name>...)` で名前付きグループを作成
- **パターン内参照**: `(?P=name)` で同じパターン内から参照
- **マッチオブジェクト**: `.group('name')` で取得

```python
import re

text = "2026-04-05"

# 名前付きグループで年・月・日をキャプチャ
result = re.search(r"(?P<year>\d{4})-(?P<month>\d{2})-(?P<day>\d{2})", text)

if result:
    print(result.group("year"))   # 2026
    print(result.group("month"))  # 04
    print(result.group("day"))    # 05

    # groupdict() で辞書として取得
    print(result.groupdict())
    # {'year': '2026', 'month': '04', 'day': '05'}
```

### 名前付きグループでの後方参照

```python
import re

# 名前付きグループで重複単語を検出
text = "the the quick brown fox fox"

results = re.findall(r"(?P<word>\b\w+)\s+(?P=word)\b", text)
print(results)  # ['the', 'fox']
```

```
【名前付きグループの構文】

定義:           (?P<name>パターン)
パターン内参照:  (?P=name)
マッチ取得:      result.group('name')
辞書取得:        result.groupdict()
置換での参照:    \g<name>
```

### 名前付きグループでの置換

```python
import re

# 日付形式の変換: YYYY-MM-DD → DD/MM/YYYY
text = "日付: 2026-04-05"

result = re.sub(
    r"(?P<year>\d{4})-(?P<month>\d{2})-(?P<day>\d{2})",
    r"\g<day>/\g<month>/\g<year>",
    text
)
print(result)  # 日付: 05/04/2026
```

## 4.3 先読み（Lookahead）

### 先読みとは

先読み（Lookahead）とは、**現在位置の後ろに特定のパターンがある（またはない）ことを確認**するゼロ幅アサーションです。文字を消費しません（マッチ結果に含まれません）。

### 肯定先読み `(?=...)`

「この後に `...` が続く位置」にマッチします。

```python
import re

text = "100円 200ドル 300円 400ユーロ"

# \d+ の後に「円」が続くもの（「円」自体はマッチに含まない）
results = re.findall(r"\d+(?=円)", text)
print(results)  # ['100', '300']
```

```
【肯定先読みのイメージ】

テキスト: 100円 200ドル 300円
パターン: \d+(?=円)

100円   → 100 の後に「円」がある → マッチ ✓（"100" を返す）
200ドル → 200 の後に「ドル」がある → マッチ ✗
300円   → 300 の後に「円」がある → マッチ ✓（"300" を返す）

※ 「円」自体はマッチ結果に含まれない（ゼロ幅）
```

### 否定先読み `(?!...)`

「この後に `...` が続かない位置」にマッチします。

```python
import re

text = "100円 200ドル 300円 400ユーロ"

# \d+ の後に「円」が続かないもの
results = re.findall(r"\d+(?!円)", text)
print(results)  # ['10', '200', '30', '400']
# ← 注意：100 から '10' がマッチ、0円 の手前で止まる
```

```python
import re

text = "100円 200ドル 300円 400ユーロ"

# より正確に：数字の後に「円」以外の非数字文字が続く
results = re.findall(r"\d+(?![\d円])", text)
print(results)  # ['200', '400']
```

> **よくある間違い**: 否定先読みは「次の1文字が条件に合わない」だけでマッチが成功します。`\d+(?!円)` では、`100円` の `10` の後の `0` が「円ではない」のでマッチしてしまいます。意図通りに動かすには、パターンを工夫する必要があります。

## 4.4 後読み（Lookbehind）

### 後読みとは

後読み（Lookbehind）とは、**現在位置の前に特定のパターンがある（またはない）ことを確認**するゼロ幅アサーションです。

### 肯定後読み `(?<=...)`

「この前に `...` がある位置」にマッチします。

```python
import re

text = "$100 €200 $300 ¥400"

# $ の後に続く数字（$記号自体はマッチに含まない）
results = re.findall(r"(?<=\$)\d+", text)
print(results)  # ['100', '300']
```

```
【肯定後読みのイメージ】

テキスト: $100 €200 $300
パターン: (?<=\$)\d+

$100  → 数字の前に "$" がある → マッチ ✓（"100" を返す）
€200  → 数字の前に "€" がある → マッチ ✗
$300  → 数字の前に "$" がある → マッチ ✓（"300" を返す）

※ "$" 自体はマッチ結果に含まれない（ゼロ幅）
```

### 否定後読み `(?<!...)`

「この前に `...` がない位置」にマッチします。

```python
import re

text = "$100 €200 $300 ¥400"

# $ が前にない数字
results = re.findall(r"(?<!\$)\d+", text)
print(results)  # ['00', '200', '00', '400']
# ← $100 の '1' の前に '$' があるため '1' は除外、残りの '00' がマッチ
```

```python
import re

text = "$100 €200 $300 ¥400"

# 通貨記号の後の数字全体を取得し、$ 以外の通貨のものだけ
# 方法：スペースか先頭の後に $ 以外の1文字 + 数字
results = re.findall(r"(?<!\$)(?<=.)\d+", text)
# より簡単なアプローチ
results = re.findall(r"[€¥]\d+", text)
print(results)  # ['€200', '¥400']
```

### 後読みの制限

Pythonの `re` モジュールでは、後読みのパターンは**固定長**でなければなりません。

```python
import re

# ✅ 固定長の後読み
result = re.findall(r"(?<=\$)\d+", "$100")     # OK: \$ は1文字

# ✅ 固定長の後読み
result = re.findall(r"(?<=USD)\d+", "USD100")  # OK: USD は3文字

# ❌ 可変長の後読み → エラー
# result = re.findall(r"(?<=\$+)\d+", "$100")  # NG: \$+ は可変長
```

> **実務でのポイント**: 可変長の後読みが必要な場合は、`regex` モジュール（サードパーティ）を使うか、パターンを工夫して先読みやグループ化で代替します。

## 4.5 先読み・後読みのまとめ

```
┌─────────────────┬───────────────────────┬──────────────────────┐
│ 種類             │ 構文                   │ 意味                  │
├─────────────────┼───────────────────────┼──────────────────────┤
│ 肯定先読み       │ (?=...)               │ 後ろに...がある        │
│ 否定先読み       │ (?!...)               │ 後ろに...がない        │
│ 肯定後読み       │ (?<=...)              │ 前に...がある          │
│ 否定後読み       │ (?<!...)              │ 前に...がない          │
└─────────────────┴───────────────────────┴──────────────────────┘

共通の特徴：
  - ゼロ幅（文字を消費しない）
  - マッチ結果に含まれない
  - 「条件チェック」として使う
```

```
【先読み・後読みの方向】

          後読み          先読み
          (?<=...)        (?=...)
          (?<!...)        (?!...)
              ↓              ↓
テキスト: .....|.....
              ↑
          現在位置

後読み = 現在位置の「左側（前）」をチェック
先読み = 現在位置の「右側（後）」をチェック
```

## 4.6 実用例

### パスワード強度チェック

先読みを組み合わせることで、複数の条件を同時にチェックできます。

```python
import re

def check_password(password):
    """
    パスワード強度チェック:
    - 8文字以上
    - 英大文字を含む
    - 英小文字を含む
    - 数字を含む
    - 記号を含む
    """
    pattern = r"^(?=.*[A-Z])(?=.*[a-z])(?=.*\d)(?=.*[!@#$%^&*]).{8,}$"
    return re.match(pattern, password) is not None

passwords = [
    "Passw0rd!",    # ✓ 全条件を満たす
    "password123",  # ✗ 大文字なし、記号なし
    "SHORT1!",      # ✗ 小文字なし（7文字）
    "MyP@ss1",      # ✗ 7文字（8文字未満）
    "MyP@ssw0rd",   # ✓ 全条件を満たす
]

for pw in passwords:
    result = "✓ 強い" if check_password(pw) else "✗ 弱い"
    print(f"{pw:15s} → {result}")
```

```
【パスワードチェックパターンの分解】

^(?=.*[A-Z])(?=.*[a-z])(?=.*\d)(?=.*[!@#$%^&*]).{8,}$

^              → 文字列の先頭
(?=.*[A-Z])    → 先読み：どこかに大文字がある
(?=.*[a-z])    → 先読み：どこかに小文字がある
(?=.*\d)       → 先読み：どこかに数字がある
(?=.*[!@#$%^&*]) → 先読み：どこかに記号がある
.{8,}          → 任意の文字が8文字以上
$              → 文字列の末尾

※ 先読みはゼロ幅なので、4つの先読みは全て
  文字列の先頭位置から「条件だけ」をチェックする
```

> **実務でのポイント**: 先読みを使った複数条件の同時チェックは、パスワード検証だけでなく、入力値のバリデーション全般で活用できる強力なテクニックです。

### 数値の3桁区切りカンマ挿入

先読みと後読みを組み合わせた実用例です。

```python
import re

def add_commas(number_str):
    """数値文字列に3桁ごとのカンマを挿入"""
    # 後読み：前に数字がある位置
    # 先読み：後ろに3桁ずつの数字のかたまりがあり、その後は数字でない位置
    return re.sub(r"(?<=\d)(?=(\d{3})+(?!\d))", ",", number_str)

print(add_commas("1234567890"))  # 1,234,567,890
print(add_commas("100"))         # 100
print(add_commas("1000"))        # 1,000
print(add_commas("12345"))       # 12,345
```

```
【3桁区切りパターンの分解】

(?<=\d)(?=(\d{3})+(?!\d))

(?<=\d)         → 後読み：前が数字である位置
(?=(\d{3})+     → 先読み：後ろに数字3桁のかたまりが1回以上
    (?!\d))     →   かつ、そのさらに後ろが数字でない

テキスト: 1 2 3 4 5 6 7 8 9 0
位置:    ↑ ↑ ↑ ↑ ↑ ↑ ↑ ↑ ↑ ↑

位置1と2の間: 後ろに "234567890" = 9桁 = 3桁×3 → カンマ挿入 ✓
位置2と3の間: 後ろに "34567890"  = 8桁 ≠ 3の倍数 → ✗
位置3と4の間: 後ろに "4567890"   = 7桁 ≠ 3の倍数 → ✗
位置4と5の間: 後ろに "567890"    = 6桁 = 3桁×2 → カンマ挿入 ✓
位置7と8の間: 後ろに "890"       = 3桁 = 3桁×1 → カンマ挿入 ✓

結果: 1,234,567,890
```

### 重複単語の検出と修正

```python
import re

text = """
このレポートは は 重要です。
テスト テスト のために作成しました。
問題は ありません ありません。
"""

# 日本語の重複（スペースあり）を検出
results = re.findall(r"(\S+)\s+\1", text)
print("重複:", results)  # ['は', 'テスト', 'ありません']

# 重複を1つに修正
fixed = re.sub(r"(\S+)\s+\1", r"\1", text)
print(fixed)
```

### 特定の文字列を含まない行の抽出

```python
import re

log = """
2026-04-05 INFO  アプリケーション開始
2026-04-05 ERROR データベース接続失敗
2026-04-05 INFO  リクエスト処理完了
2026-04-05 WARN  メモリ使用率80%超過
2026-04-05 ERROR ファイル書き込み失敗
"""

# ERROR を含まない行を抽出（否定先読み）
results = re.findall(r"^(?!.*ERROR).+$", log, re.MULTILINE)
for line in results:
    print(line)
# 2026-04-05 INFO  アプリケーション開始
# 2026-04-05 INFO  リクエスト処理完了
# 2026-04-05 WARN  メモリ使用率80%超過
```
