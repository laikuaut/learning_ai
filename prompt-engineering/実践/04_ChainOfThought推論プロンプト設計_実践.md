## 課題の要件

以下の6つの問題について、「通常プロンプト」と「CoTプロンプト」の2パターンを作成し、推論精度の違いを比較してください。

### 問題一覧

| 番号 | 問題タイプ | 内容 |
|:----:|-----------|------|
| 1 | 算数の推論 | 買い物の計算問題（割引・税込み） |
| 2 | 論理的推論 | 条件から正しい結論を導く |
| 3 | 比較分析 | 2つの選択肢のメリット・デメリット比較 |
| 4 | 因果関係の分析 | ある現象の原因を推論する |
| 5 | スケジュール調整 | 複数の制約条件から最適な日程を見つける |
| 6 | 意思決定 | 複数の評価基準に基づいて最適案を選ぶ |

### 各問題で実施すること

1. **通常プロンプト**: 問題を提示し、直接回答を求める
2. **CoTプロンプト**: 「ステップバイステップで考えてください」等のCoTトリガー（trigger）を含める
3. 両方の出力を比較し、推論過程と最終回答の正確性を評価する

問題文：
> 太郎さんはスーパーで、りんご（1個150円）を3個、牛乳（1本250円）を2本、

> 合計金額（税込み、消費税10%）はいくらですか？

**通常プロンプト**:
```

太郎さんはスーパーで、りんご（1個150円）を3個、牛乳（1本250円）を2本、
パン（1つ200円）を1つ買いました。りんごは本日20%割引です。
合計金額（税込み、消費税10%）はいくらですか？
```

**CoTプロンプト**:
```
以下の問題をステップバイステップで解いてください。
各ステップで計算過程を明示してから、最終回答を出してください。

太郎さんはスーパーで、りんご（1個150円）を3個、牛乳（1本250円）を2本、
パン（1つ200円）を1つ買いました。りんごは本日20%割引です。
合計金額（税込み、消費税10%）はいくらですか？
```

CoTプロンプトでは「ステップバイステップで」「計算過程を明示して」という指示が重要です。

| CoTトリガー | 効果 | 適したタスク |
|------------|------|-------------|
| 「ステップバイステップで考えてください」 | 汎用的な推論促進 | 全般 |

| 「Let's think step by step.」 | 英語での定番CoTトリガー | 全般（英語） |

### 問題1：算数の推論

#### 通常プロンプトの出力例

```
合計金額は1,155円です。
```
（計算過程なし。正答の場合もあるが、間違える場合もある）

#### CoTプロンプトの出力例

```
ステップ1：各商品の小計を計算します。
- りんご：150円 × 3個 = 450円
- 牛乳：250円 × 2本 = 500円
- パン：200円 × 1つ = 200円

ステップ2：りんごの割引を適用します。
- りんご割引後：450円 × (1 - 0.20) = 450円 × 0.80 = 360円

ステップ3：割引後の合計を計算します。
- 合計：360円 + 500円 + 200円 = 1,060円

ステップ4：消費税（10%）を加算します。
- 税込合計：1,060円 × 1.10 = 1,166円

最終回答：税込合計は1,166円です。
```

**比較**: CoTプロンプトでは計算過程が明示されるため、どこで間違えたかを検証できます。通常プロンプトでは回答だけが出力され、過程が見えないため正誤の確認が困難です。

### 問題5：スケジュール調整

#### 通常プロンプト

```
以下の条件で、全員が参加できる会議日時を教えてください。
（条件省略）
```

**出力例**: 「水曜日の午後が良いでしょう」（根拠不明、Dさんの制約を見落とすことがある）

#### CoTプロンプト

```
以下の条件で、全員が参加できる2時間の会議日時を見つけてください。
ステップバイステップで、各メンバーの空き時間を整理してから判断してください。

（条件省略）
```

**出力例**:

```
ステップ1：各メンバーの可能な時間帯を整理します。

| メンバー | 月 | 火 | 水 | 木 | 金 |
|---------|----|----|----|----|-----|
| Aさん | 午前 | × | 午前 | × | 午前 |
| Bさん | × | 終日 | 終日 | 終日 | × |
| Cさん | 午後 | 午後 | 午後 | 午後 | × |
| Dさん | 10時〜 | 10時〜 | 10時〜 | × | × |
| Eさん | × | × | 午後 | 午後 | 午後 |

ステップ2：全員が○の日を探します。
- 月曜：Bさん×、Eさん× → 不可
- 火曜：Aさん×、Eさん× → 不可
- 水曜：全員に可能な時間帯あり → 候補
- 木曜：Aさん×、Dさん× → 不可
- 金曜：Bさん×、Cさん×、Dさん× → 不可

ステップ3：水曜日の共通時間帯を特定します。
- Aさん：午前のみ
- Cさん：午後のみ
- → AさんとCさんが重なる時間がないため、水曜も不可

ステップ4：再検討...全員参加は不可能です。
最大参加人数の組み合わせを提案します。
```

**比較**: CoTにより「全員参加は不可能」という正しい結論に到達できました。通常プロンプトでは制約の見落としが発生しやすいです。

### 問題6：意思決定（改良版）

改良版では、CoTに加えて**構造化された推論フレームワーク**を提供します。

#### 構造化CoTプロンプト

```
あなたはオフィス移転の専門コンサルタントです。

以下の3つの物件候補から最適な物件を選定してください。
意思決定マトリクス（Decision Matrix）を使って、体系的に評価してください。

【物件情報】
- 物件A：駅徒歩3分、家賃月50万円、面積80㎡、築5年
- 物件B：駅徒歩10分、家賃月35万円、面積120㎡、築15年
- 物件C：駅徒歩7分、家賃月42万円、面積100㎡、築8年

【条件】
- 従業員は現在15名、来年20名に増員予定
- 通勤の利便性とコストを最も重視
- オフィスの拡張性も考慮したい

【以下の手順で分析してください】

ステップ1：評価基準を定義し、重み（ウェイト）を設定する
（例：利便性 30%、コスト 30%、面積 20%、築年数 10%、拡張性 10%）

ステップ2：各物件を各基準で5段階評価する

ステップ3：重み付きスコアを計算する
（各基準の評価 × 重みの合計）

ステップ4：スコアに基づいて順位を付ける

ステップ5：数値だけでは判断できないリスクや定性的な要素を考慮する

ステップ6：最終推奨とその根拠を述べる
```

**出力例**:

```
ステップ1：評価基準と重み付け

| 基準 | 重み | 理由 |
|------|:----:|------|
| 通勤利便性 | 30% | クライアントの重視事項 |
| コスト | 30% | クライアントの重視事項 |
| 面積（現在＋将来） | 20% | 増員予定を考慮 |
| 築年数 | 10% | 設備の新しさに関連 |
| 拡張性 | 10% | 将来の成長に対応 |

ステップ2：各物件の評価（5段階）

| 基準 | 物件A | 物件B | 物件C |
|------|:-----:|:-----:|:-----:|
| 通勤利便性 | 5 | 2 | 3 |
| コスト | 2 | 5 | 3 |
| 面積 | 2 | 5 | 4 |
| 築年数 | 5 | 2 | 4 |
| 拡張性 | 1 | 5 | 3 |

ステップ3：重み付きスコア

物件A：5×0.3 + 2×0.3 + 2×0.2 + 5×0.1 + 1×0.1 = 3.0
物件B：2×0.3 + 5×0.3 + 5×0.2 + 2×0.1 + 5×0.1 = 3.8
物件C：3×0.3 + 3×0.3 + 4×0.2 + 4×0.1 + 3×0.1 = 3.3

ステップ4：順位
1位：物件B（3.8点）
2位：物件C（3.3点）
3位：物件A（3.0点）

ステップ5：定性的な考慮事項
- 物件Bは築15年で設備の老朽化リスクあり（修繕費用を確認すべき）
- 物件Aは増員後に手狭になるリスクが高い（80㎡ ÷ 20人 = 4㎡/人）
- 物件Cはバランスが良いが、突出した強みがない

ステップ6：最終推奨
物件Bを推奨します。コストと面積の優位性が高く、増員計画にも対応できます。
ただし、築15年のため内覧時に設備状態を確認し、必要なら修繕費用を見積もることを
推奨します。次点は物件Cで、物件Bの設備状態に問題がある場合の代替案です。
```

### CoTの3つのレベル

```
┌─────────────────────────────────────────────────────────┐
│                 CoTの3つのレベル                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  レベル1：ベーシックCoT                                  │
│  → 「ステップバイステップで考えてください」を追加するだけ │
│  → 簡単な推論タスクに有効                                │
│                                                         │
│  レベル2：構造化CoT                                      │
│  → ステップの内容を具体的に指示する                      │
│  → 「まず○○を整理し、次に○○を比較し...」              │
│  → 中程度の複雑さのタスクに有効                          │
│                                                         │
│  レベル3：フレームワークCoT                              │
│  → 意思決定マトリクス等の既存フレームワークを活用        │
│  → 各ステップの出力形式まで指定する                      │
│  → 複雑な分析・意思決定タスクに有効                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**初心者向けとの違い**:
- CoTを3つのレベルに分類し、タスクの複雑さに応じた使い分けを提示
- 意思決定マトリクスという具体的なフレームワークを導入
- 定量評価に加えて定性的な考慮事項の分析を追加
- ステップの出力形式（表形式、計算式）まで制御する手法を提示

</details>
