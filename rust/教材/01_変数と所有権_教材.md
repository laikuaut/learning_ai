## 1.1 Rustとは何か

### Rustの概要

Rust（ラスト）は、Mozilla が開発を主導し、現在は Rust Foundation が管理するシステムプログラミング言語（systems programming language）です。2015年に安定版 1.0 がリリースされ、Stack Overflow の開発者調査で「最も愛されている言語」に何年も連続で選ばれています。

### Rustの3つの柱

```
┌─────────────────────────────────────────────────┐
│                  Rust の 3 つ の 柱              │
├─────────────────┬───────────────┬───────────────┤
│    安全性        │    速度       │    並行性      │
│   (Safety)      │  (Speed)     │ (Concurrency) │
├─────────────────┼───────────────┼───────────────┤
│ メモリ安全を     │ C/C++に匹敵   │ データ競合を    │
│ コンパイル時に   │ するゼロコスト │ コンパイル時に  │
│ 保証            │ 抽象化        │ 防止           │
└─────────────────┴───────────────┴───────────────┘
```

Rustが他の言語と大きく異なるのは、**ガベージコレクタ（GC）なしでメモリ安全を実現している**点です。C/C++のような手動メモリ管理の危険性も、GC言語（Java, Go, Python等）のランタイムオーバーヘッドもありません。これを可能にしているのが「所有権システム」です。

### Rustが使われている場面

| 分野 | 具体例 |
|------|--------|
| Webブラウザ | Firefox（Servo エンジン） |
| OS・システム | Linux カーネルモジュール、Android |
| Web開発 | Actix Web, Axum（バックエンド） |
| CLI ツール | ripgrep, fd, bat, exa |
| WebAssembly | wasm-bindgen, Yew |
| クラウド | AWS Firecracker, CloudFlare Workers |
| ブロックチェーン | Solana, Polkadot |

### ポイントまとめ

- Rustは安全性・速度・並行性を兼ね備えたシステムプログラミング言語です
- GCなしでメモリ安全を実現する「所有権システム」が最大の特徴です
- Webからシステム開発まで幅広い分野で採用が進んでいます

## 1.2 環境構築

### rustup のインストール

Rustの公式ツールチェーン管理ツールが `rustup`（ラストアップ）です。これ1つで Rust コンパイラ（rustc）、パッケージマネージャ（cargo）、標準ライブラリなどが全てインストールされます。

**インストール方法（Linux / macOS / WSL）：**

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

**Windowsの場合：**

[https://rustup.rs](https://rustup.rs) から `rustup-init.exe` をダウンロードして実行します。

インストール後、ターミナルを再起動して確認します。

```bash
rustc --version
# 例: rustc 1.82.0 (f6e511eec 2024-10-15)

cargo --version
# 例: cargo 1.82.0 (8f40fc59f 2024-10-11)
```

### Cargo — Rustのパッケージマネージャ兼ビルドツール

`cargo`（カーゴ）は Rust のビルドシステムとパッケージマネージャを兼ねたツールです。プロジェクトの作成からビルド、テスト、依存関係の管理まで全てを担います。

```
┌──────────────────────────────────────────────┐

├──────────────┬───────────────────────────────┤
│ cargo new    │ 新しいプロジェクトを作成        │
│ cargo build  │ プロジェクトをビルド            │

└──────────────┴───────────────────────────────┘
```

```bash
# 新しいプロジェクトを作成

# ディレクトリ構造

### ポイントまとめ

- `rustup` でツールチェーン一式をインストールします
- `cargo` がビルド・実行・テスト・依存管理を全て担当します
- `cargo new` でプロジェクトの雛形が自動生成されます

## 1.3 Hello World

### 最初のプログラム

```rust
fn main() {
    println!("Hello, World!");
}
```

**実行方法：**

```bash
# cargo プロジェクト内で実行する場合
cargo run

# 出力:
# Hello, World!
```

### コードの解説

```rust
fn main() {          // fn: 関数（function）を定義するキーワード
                     // main: プログラムのエントリーポイント（最初に実行される関数）
    println!("Hello, World!");
    // println!: 標準出力に文字列を表示するマクロ（macro）
    // 末尾の ! はマクロ呼び出しの印（関数呼び出しではない）
    // 文字列は "" ダブルクォートで囲む
    // 文末に ; セミコロンが必要
}
```

> **補足：マクロ（macro）とは？**
> `println!` の `!` に注目してください。Rustでは `!` が付いているものはマクロです。マクロは通常の関数より柔軟で、可変長引数を受け取れます。今は「`println!` は画面に文字を表示するもの」と覚えておけば十分です。

### フォーマット出力

`println!` は Python の f-string や JavaScript のテンプレートリテラルのような書式指定ができます。

```rust
fn main() {
    let name = "Rust";
    let version = 2024;

    // 基本的な埋め込み
    println!("Hello, {}!", name);
    // 出力: Hello, Rust!

    // 複数の値
    println!("{} was first released in {}", name, version);
    // 出力: Rust was first released in 2024

    // デバッグ表示（{:?}）
    println!("Debug: {:?}", (name, version));
    // 出力: Debug: ("Rust", 2024)

    // 桁揃え
    println!("{:>10}", "right");   // 右寄せ10桁
    println!("{:<10}", "left");    // 左寄せ10桁
    println!("{:^10}", "center");  // 中央寄せ10桁

    // 小数点以下の桁数
    println!("{:.2}", 3.14159);    // 出力: 3.14
}
```

### ポイントまとめ

- `fn main()` がプログラムの開始点です
- `println!` はマクロで、`{}` をプレースホルダとして値を埋め込めます
- Rustでは文の末尾にセミコロン `;` が必要です

## 1.4 変数宣言

### let による変数束縛

Rust では `let` キーワードで変数を宣言します。これを変数束縛（variable binding）と呼びます。

```rust
fn main() {
    let x = 5;
    println!("x = {}", x);
    // 出力: x = 5
}
```

### Rustの変数はデフォルトで不変（immutable）

**これがRustの最大の特徴の1つです。** 他の多くの言語では変数は自由に変更できますが、Rustでは `let` で宣言した変数は**デフォルトで変更できません**。

```rust
fn main() {
    let x = 5;
    // x = 10;  // コンパイルエラー！ cannot assign twice to immutable variable
    println!("x = {}", x);
}
```

```
┌──────────────────────────────────────────────────────┐
│  他の言語との比較                                      │
│                                                       │
│  JavaScript:  let x = 5;   → 変更可能                │
│               const x = 5; → 変更不可                 │
│                                                       │
│  Rust:        let x = 5;     → 変更不可（デフォルト）  │
│               let mut x = 5; → 変更可能               │
│                                                       │
│  → Rustは「安全側」をデフォルトにしている               │
└──────────────────────────────────────────────────────┘
```

### mut で可変変数にする

変数を変更可能にするには、`mut`（mutable の略）キーワードを付けます。

```rust
fn main() {
    let mut x = 5;
    println!("x = {}", x);
    // 出力: x = 5

    x = 10;  // OK! mut が付いているので変更可能
    println!("x = {}", x);
    // 出力: x = 10
}
```

### 型注釈（type annotation）

Rust は強い静的型付け言語ですが、多くの場合**型推論（type inference）**が働くため、型を明示的に書く必要はありません。

```rust
fn main() {
    // 型推論に任せる（推奨）
    let x = 5;          // i32 と推論される
    let y = 3.14;       // f64 と推論される
    let flag = true;    // bool と推論される
    let ch = 'A';       // char と推論される

    // 型を明示的に指定する
    let x: i32 = 5;
    let y: f64 = 3.14;
    let flag: bool = true;
    let ch: char = 'A';

    println!("x={}, y={}, flag={}, ch={}", x, y, flag, ch);
    // 出力: x=5, y=3.14, flag=true, ch=A
}
```

> **現場のベストプラクティス：** 型が文脈から明らかな場合は型推論に任せ、曖昧な場合のみ型注釈を書きます。コードレビューで「型が分かりにくい」と指摘された箇所に注釈を追加するのが実務での一般的な流れです。

### シャドーイング（shadowing）

Rust では同じ名前の変数を `let` で再度宣言できます。これをシャドーイング（shadowing）と呼びます。

```rust
fn main() {
    let x = 5;
    println!("x = {}", x);
    // 出力: x = 5

    let x = x + 1;  // 新しい x で前の x を隠す（シャドーイング）
    println!("x = {}", x);
    // 出力: x = 6

    let x = x * 2;
    println!("x = {}", x);
    // 出力: x = 12

    // 型を変えることもできる！
    let x = "hello";
    println!("x = {}", x);
    // 出力: x = hello
}
```

シャドーイングと `mut` の違いに注意してください。

```
┌───────────────────────────────────────────────────────────┐
│  シャドーイング vs mut                                     │
│                                                            │
│  シャドーイング（let x = ...）:                            │
│    - 新しい変数を作る（前の変数は消える）                    │
│    - 型を変えられる                                        │
│    - 新しい変数はデフォルトで不変                            │
│                                                            │
│  mut（let mut x = ...）:                                   │
│    - 同じ変数の値を書き換える                               │
│    - 型は変えられない                                      │
│    - 変更可能な状態が続く                                   │
└───────────────────────────────────────────────────────────┘
```

### 定数（constant）

変更されない値には `const` も使えます。`let` との違いを押さえましょう。

```rust
// const: コンパイル時に値が確定している必要がある
const MAX_POINTS: u32 = 100_000;
// ・型注釈が必須
// ・命名は SCREAMING_SNAKE_CASE（大文字 + アンダースコア）
// ・グローバルスコープに定義できる

fn main() {
    println!("最大ポイント: {}", MAX_POINTS);
    // 出力: 最大ポイント: 100000
}
```

### ポイントまとめ

- Rustの変数はデフォルトで不変（immutable）です
- `mut` を付けると可変（mutable）になります
- 型推論が強力なので、多くの場合型注釈は不要です
- シャドーイングで同名の変数を再定義でき、型の変更も可能です
- 定数は `const` で宣言し、型注釈が必須です

## 1.5 基本データ型

### スカラー型（scalar types）

Rust のスカラー型は**整数・浮動小数点数・ブーリアン・文字**の4種類です。

#### 整数型（integer types）

```
┌─────────┬──────────┬────────────────────────────┐
│  符号付き │ 符号なし  │ 範囲                        │
├─────────┼──────────┼────────────────────────────┤
│  i8     │  u8      │ i8: -128 〜 127             │
│  i16    │  u16     │ u8: 0 〜 255                │
│  i32    │  u32     │ i32: 約 ±21億               │
│  i64    │  u64     │ u64: 0 〜 約1845京          │
│  i128   │  u128    │ i128: 非常に大きな範囲       │
│  isize  │  usize   │ アーキテクチャ依存（32/64bit）│
└─────────┴──────────┴────────────────────────────┘

※ デフォルトの整数型は i32（最も一般的）
※ usize はインデックスやコレクションのサイズに使われる
```

```rust
fn main() {
    let a: i32 = 42;           // 符号付き32ビット整数
    let b: u8 = 255;           // 符号なし8ビット整数（0〜255）
    let c: usize = 100;        // ポインタサイズの符号なし整数

    // 数値リテラルの書き方
    let decimal = 98_222;      // 10進数（_ で桁区切り可能）
    let hex = 0xff;            // 16進数
    let octal = 0o77;          // 8進数
    let binary = 0b1111_0000;  // 2進数
    let byte = b'A';           // バイトリテラル（u8のみ）

    println!("a={}, b={}, c={}", a, b, c);
    println!("decimal={}, hex={}, octal={}, binary={}, byte={}",
             decimal, hex, octal, binary, byte);
    // 出力:
    // a=42, b=255, c=100
    // decimal=98222, hex=255, octal=63, binary=240, byte=65
}
```

#### 浮動小数点型（floating-point types）

```rust
fn main() {
    let x = 2.0;        // f64（デフォルト、倍精度）
    let y: f32 = 3.0;   // f32（単精度）

    // 四則演算
    let sum = 5.0 + 10.0;
    let difference = 95.5 - 4.3;
    let product = 4.0 * 30.0;
    let quotient = 56.7 / 32.2;
    let remainder = 43.0 % 5.0;

    println!("sum={}, diff={}, prod={}, quot={:.4}, rem={}",
             sum, difference, product, quotient, remainder);
    // 出力: sum=15, diff=91.2, prod=120, quot=1.7609, rem=3
}
```

> **よくある間違い：** 整数と浮動小数点数を直接演算しようとするとエラーになります。
> ```rust
> let x: i32 = 5;
> let y: f64 = 3.0;
> // let z = x + y;  // エラー！型が違う
> let z = x as f64 + y;  // OK: as で型変換（キャスト）
> ```
> Rustは暗黙の型変換を行いません。これは安全性のための設計です。

#### ブーリアン型（bool）

```rust
fn main() {
    let t = true;
    let f: bool = false;

    println!("t={}, f={}", t, f);
    // 出力: t=true, f=false

    // 論理演算
    println!("AND: {}", t && f);   // false
    println!("OR: {}", t || f);    // true
    println!("NOT: {}", !t);       // false
}
```

#### 文字型（char）

```rust
fn main() {
    let c = 'z';
    let heart = '❤';
    let kanji = '漢';
    let emoji = '😀';

    println!("{} {} {} {}", c, heart, kanji, emoji);
    // 出力: z ❤ 漢 😀

    // char は4バイト（Unicode スカラー値）
    println!("'z' のサイズ: {} bytes", std::mem::size_of_val(&c));
    // 出力: 'z' のサイズ: 4 bytes
}
```

> **注意：** Rust の `char` は1バイトではなく**4バイト**です。Unicode のスカラー値を表現するため、日本語や絵文字も1つの `char` に格納できます。

### タプル型（tuple）

複数の異なる型の値をひとまとめにできます。

```rust
fn main() {
    // タプルの作成
    let tup: (i32, f64, bool) = (500, 6.4, true);

    // 分割代入（destructuring）
    let (x, y, z) = tup;
    println!("x={}, y={}, z={}", x, y, z);
    // 出力: x=500, y=6.4, z=true

    // インデックスアクセス（0始まり）
    println!("0番目: {}", tup.0);  // 500
    println!("1番目: {}", tup.1);  // 6.4
    println!("2番目: {}", tup.2);  // true
}
```

### 配列型（array）

同じ型の要素を固定長で格納します。

```rust
fn main() {
    // 配列の宣言
    let a = [1, 2, 3, 4, 5];

    // 型とサイズを明示
    let b: [i32; 5] = [1, 2, 3, 4, 5];

    // 同じ値で初期化
    let c = [0; 5];  // [0, 0, 0, 0, 0]

    // アクセス
    println!("a[0] = {}", a[0]);  // 1
    println!("a[4] = {}", a[4]);  // 5

    // 長さ
    println!("長さ: {}", a.len());  // 5

    println!("b={:?}, c={:?}", b, c);
    // 出力: b=[1, 2, 3, 4, 5], c=[0, 0, 0, 0, 0]
}
```

> **よくある間違い：** 配列の範囲外アクセスはコンパイルは通りますが、実行時にパニック（panic）します。
> ```rust
> let a = [1, 2, 3];
> // println!("{}", a[5]);  // 実行時エラー: index out of bounds
> ```
> 安全にアクセスするには `a.get(5)` を使います（後の章で解説）。

### ポイントまとめ

- 整数のデフォルトは `i32`、浮動小数点のデフォルトは `f64` です
- Rustは暗黙の型変換を行わず、明示的な `as` キャストが必要です
- `char` は4バイトで Unicode スカラー値を表します
- タプルは異なる型を、配列は同じ型の固定長データを格納します

## 1.6 所有権（ownership）

### Rustの所有権とは

所有権（ownership）は Rust の最も重要な概念です。これにより、ガベージコレクタなしでメモリ安全を実現しています。

### 所有権の3つのルール

```
┌────────────────────────────────────────────────────┐
│            所有権（ownership）の 3 つのルール        │
│                                                     │
│  1. 各値は「所有者」（owner）という変数を持つ         │
│  2. 所有者は常に1つだけ                              │
│  3. 所有者がスコープを抜けると値は破棄される           │
└────────────────────────────────────────────────────┘
```

### スコープとメモリ

```rust
fn main() {
    {
        let s = String::from("hello");  // s がスコープに入る（メモリ確保）
        println!("{}", s);              // s は有効
    }                                   // s がスコープを抜ける → メモリ自動解放

    // println!("{}", s);  // エラー！ s はもう存在しない
}
```

```
  メモリの様子:

  スコープ開始       スコープ中        スコープ終了
  ┌──────────┐    ┌──────────┐    ┌──────────┐
  │ s ─────┐ │    │ s ─────┐ │    │ (解放)   │
  │        ▼ │    │        ▼ │    │          │
  │  [hello] │    │  [hello] │    │  [-----] │
  └──────────┘    └──────────┘    └──────────┘
    メモリ確保       使用中          自動解放(drop)
```

### スタック（stack）とヒープ（heap）

所有権を理解するには、メモリの2つの領域を知る必要があります。

```
┌─────────────────────────────────────────────────────┐
│                  メモリレイアウト                      │
│                                                      │
│  スタック（Stack）          ヒープ（Heap）            │
│  ┌──────────────┐         ┌──────────────────┐      │
│  │ 固定サイズ    │         │ 可変サイズ        │      │
│  │ 高速アクセス  │         │ 低速アクセス      │      │
│  │              │         │                   │      │
│  │ i32, f64     │    ┌──→ │ String のデータ   │      │
│  │ bool, char   │    │    │ Vec のデータ      │      │
│  │ ポインタ ────┼────┘    │ Box のデータ      │      │
│  └──────────────┘         └──────────────────┘      │
│                                                      │
│  整数や固定サイズの型は      String やVecなど可変長の   │
│  スタック上に直接格納される   データはヒープに格納される  │
└─────────────────────────────────────────────────────┘
```

- **スタック上のデータ**（i32, f64, bool, char など）：固定サイズ → **コピー**される
- **ヒープ上のデータ**（String, Vec など）：可変サイズ → **ムーブ**される

### ポイントまとめ

- 所有権は Rust の中核概念で、メモリ安全を保証します
- 各値には唯一の所有者がいて、所有者がスコープを抜けると値は自動解放されます
- スタック上の固定サイズデータはコピー、ヒープ上のデータはムーブが基本です

---

## 1.7 ムーブ（move）とコピー（copy）

### ムーブセマンティクス

ヒープ上のデータ（`String` など）を別の変数に代入すると、**所有権が移動（move）**します。

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1 の所有権が s2 にムーブされる

    // println!("{}", s1);  // コンパイルエラー！ s1 はもう使えない
    println!("{}", s2);     // OK: s2 が所有者
    // 出力: hello
}
```

```
  ムーブの様子:

  代入前:                    代入後:
  s1 ──→ "hello"(ヒープ)    s1  (無効)
                             s2 ──→ "hello"(ヒープ)

  ※ データのコピーは発生しない（ポインタが移るだけ）
  ※ s1 は無効になり、使用するとコンパイルエラー
```

**なぜムーブなのか？** もし `s1` と `s2` の両方が同じヒープメモリを指していたら、スコープを抜けるときに二重解放（double free）が発生してしまいます。Rustはこれをムーブで防いでいます。

### コピーセマンティクス

スタック上に格納される型（整数、浮動小数点数、bool、char など）は `Copy` トレイト（trait）を実装しているため、代入時に**値がコピー**されます。

```rust
fn main() {
    let x = 5;
    let y = x;  // 値がコピーされる（x も y も有効）

    println!("x={}, y={}", x, y);
    // 出力: x=5, y=5

    // 整数はスタック上にあり、コピーが安い → ムーブではなくコピー
}
```

```
  コピーの様子:

  代入前:          代入後:
  x = [5]          x = [5]   ← そのまま有効
                   y = [5]   ← 新しいコピー
```

### Copy が実装されている型の一覧

```
┌──────────────────────────────────────────┐
│  Copy トレイトが実装されている型           │
├──────────────────────────────────────────┤
│  整数型: i8, i16, i32, i64, i128, isize │
│         u8, u16, u32, u64, u128, usize  │
│  浮動小数点: f32, f64                    │
│  ブーリアン: bool                        │
│  文字: char                              │
│  タプル（要素が全てCopyの場合）           │
│    例: (i32, bool) は Copy              │
│    例: (i32, String) は Copy ではない    │
└──────────────────────────────────────────┘
```

### 関数の引数とムーブ

関数に値を渡すときも、ムーブまたはコピーが発生します。

```rust
fn main() {
    let s = String::from("hello");
    takes_ownership(s);       // s の所有権が関数にムーブされる
    // println!("{}", s);     // エラー！ s はもう使えない

    let x = 5;
    makes_copy(x);            // x の値がコピーされる
    println!("x = {}", x);   // OK: x はまだ使える
    // 出力: x = 5
}

fn takes_ownership(some_string: String) {
    println!("{}", some_string);
    // 出力: hello
}   // some_string がスコープを抜け、メモリが解放される

fn makes_copy(some_integer: i32) {
    println!("{}", some_integer);
    // 出力: 5
}   // some_integer がスコープを抜けるが、特別なことは起きない
```

### 戻り値と所有権

関数の戻り値でも所有権が移動します。

```rust
fn main() {
    let s1 = gives_ownership();         // 戻り値の所有権を受け取る
    println!("s1 = {}", s1);
    // 出力: s1 = hello

    let s2 = String::from("world");
    let s3 = takes_and_gives_back(s2);  // s2 をムーブして戻り値を受け取る
    // println!("{}", s2);              // エラー！ s2 はムーブ済み
    println!("s3 = {}", s3);
    // 出力: s3 = world
}

fn gives_ownership() -> String {
    let s = String::from("hello");
    s  // 所有権を呼び出し元に返す（セミコロンなし = 戻り値）
}

fn takes_and_gives_back(a_string: String) -> String {
    a_string  // 受け取った所有権をそのまま返す
}
```

### clone で明示的にコピーする

ヒープデータを本当にコピーしたい場合は `clone()` を使います。

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();  // ヒープデータの深いコピー（deep copy）

    println!("s1={}, s2={}", s1, s2);
    // 出力: s1=hello, s2=hello
    // 両方有効！（別々のヒープメモリを指している）
}
```

> **現場のベストプラクティス：** `clone()` は便利ですが、ヒープデータの完全なコピーが発生するため、大きなデータでは性能に影響します。後述する「参照と借用」を使う方が効率的です。

### ポイントまとめ

- ヒープデータの代入はムーブ（所有権の移動）が起きます
- スタックデータ（`Copy` トレイト実装型）は値がコピーされます
- 関数への引数渡しや戻り値でも所有権の移動が発生します
- `clone()` でヒープデータの明示的なコピーが可能ですが、性能に注意が必要です

## 1.8 参照と借用（references & borrowing）

### 所有権を渡さずにデータを使いたい

毎回所有権をムーブして返してもらうのは面倒です。そこで登場するのが**参照（reference）**と**借用（borrowing）**です。

```rust
fn main() {
    let s = String::from("hello");

    // & を付けて参照を渡す（所有権はムーブしない）
    let len = calculate_length(&s);

    println!("'{}' の長さは {} です", s, len);
    // 出力: 'hello' の長さは 5 です
    // s はまだ使える！
}

fn calculate_length(s: &String) -> usize {
    s.len()
}   // s はスコープを抜けるが、所有権を持っていないので何も起きない
```

```
  参照の様子:

  main のスコープ:         calculate_length のスコープ:
  s ──→ "hello"(ヒープ)    s(参照) ──→ s(main の s を指す)
                                         │
                                         ▼
                                       "hello"(ヒープ)

  ※ 参照は「データへのポインタ」であり、所有権は移動しない
  ※ 参照を作ることを「借用（borrowing）」と呼ぶ
```

### 不変参照（immutable reference）

`&T` は不変参照（immutable reference）で、データを**読むことはできるが変更できません**。

```rust
fn main() {
    let s = String::from("hello");

    let r1 = &s;  // OK: 不変参照
    let r2 = &s;  // OK: 不変参照は複数作れる

    println!("{}, {}", r1, r2);
    // 出力: hello, hello
}
```

### 可変参照（mutable reference）

`&mut T` は可変参照（mutable reference）で、データを**変更できます**。

```rust
fn main() {
    let mut s = String::from("hello");

    change(&mut s);

    println!("{}", s);
    // 出力: hello, world
}

fn change(s: &mut String) {
    s.push_str(", world");
}
```

### 借用のルール

Rustには**データ競合（data race）を防ぐ**ための厳格なルールがあります。

```
┌────────────────────────────────────────────────────────┐
│              借用（borrowing）のルール                    │
│                                                         │
│  1. 不変参照（&T）は同時に何個でも作れる                  │
│  2. 可変参照（&mut T）は同時に1つしか作れない              │
│  3. 不変参照と可変参照は同時に存在できない                 │
│                                                         │
│  → まとめ：「読み取りは複数OK、書き込みは排他的」          │
└────────────────────────────────────────────────────────┘
```

```rust
fn main() {
    let mut s = String::from("hello");

    // OK: 不変参照は複数作れる
    let r1 = &s;
    let r2 = &s;
    println!("{}, {}", r1, r2);
    // 出力: hello, hello
    // r1 と r2 はここ以降使われない（ライフタイム終了）

    // OK: 不変参照のライフタイムが終わった後なので可変参照を作れる
    let r3 = &mut s;
    r3.push_str(", world");
    println!("{}", r3);
    // 出力: hello, world
}
```

> **よくある間違い：** 不変参照と可変参照を同時に使おうとするとエラーになります。
> ```rust
> let mut s = String::from("hello");
> let r1 = &s;          // 不変参照
> let r2 = &mut s;      // エラー！ 不変参照がある間は可変参照を作れない
> println!("{}", r1);    // r1 がまだ使われている
> ```

### 実践例：関数で参照を使うパターン

```rust
fn main() {
    let mut name = String::from("田中");

    // 読み取りだけの関数には不変参照を渡す
    greet(&name);
    // 出力: こんにちは、田中さん！

    // 変更が必要な関数には可変参照を渡す
    add_title(&mut name);
    println!("変更後: {}", name);
    // 出力: 変更後: 田中様
}

fn greet(name: &String) {
    println!("こんにちは、{}さん！", name);
}

fn add_title(name: &mut String) {
    name.clear();
    name.push_str("田中様");
}
```

> **現場のベストプラクティス：** 関数のパラメータは、変更が不要なら常に不変参照（`&T`）を使います。可変参照（`&mut T`）は本当に変更が必要な場合にだけ使いましょう。これにより、コードの意図が明確になり、バグの混入を防げます。

### ポイントまとめ

- `&T` で不変参照（読み取り専用の借用）を作れます
- `&mut T` で可変参照（変更可能な借用）を作れます
- 借用は所有権を移動しないので、元の変数はそのまま使えます
- 不変参照は複数作れますが、可変参照は同時に1つだけです
- 不変参照と可変参照は同時に存在できません

## 1.9 スコープとライフタイム入門

### ライフタイム（lifetime）とは

ライフタイム（lifetime）は「参照が有効な期間」を表す概念です。Rustコンパイラはライフタイムを自動的に推論して、**ダングリング参照（dangling reference）**（解放済みメモリへの参照）を防ぎます。

```rust
fn main() {
    let r;                      // 参照を格納する変数を宣言

    {
        let x = 5;
        r = &x;                 // x への参照を作成
    }                           // x がスコープを抜け、破棄される

    // println!("{}", r);       // エラー！ x はもう存在しない
    // → ダングリング参照を防いでくれる
}
```

```
  ライフタイムの図解:

  fn main() {
      let r;                    // ------+-- 'a（r のライフタイム）
                                //       |
      {                         //       |
          let x = 5;            // -+--  |  'b（x のライフタイム）
          r = &x;               //  |    |
      }                         // -+    |  ← x が破棄される
                                //       |
      // println!("{}", r);     //       |  ← r が x を参照しようとする
  }                             // ------+     → 'b < 'a なのでエラー！
```

### 正しいライフタイムの例

```rust
fn main() {
    let x = 5;            // ------+-- 'a（x のライフタイム）
    let r = &x;           // --+-- |  'b（r のライフタイム）
                           //   |  |
    println!("{}", r);     //   |  |  OK: x はまだ有効
    // 出力: 5             // --+  |
}                          // ------+
```

### ライフタイム注釈（lifetime annotation）の入門

通常、コンパイラがライフタイムを自動推論してくれますが、関数の引数と戻り値に参照がある場合、明示的な注釈が必要になることがあります。

```rust
// 'a はライフタイムパラメータ（ジェネリックの一種）
// 2つの参照のうち、短い方のライフタイムが戻り値に適用される
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

fn main() {
    let string1 = String::from("long string");
    let result;

    {
        let string2 = String::from("xyz");
        result = longest(string1.as_str(), string2.as_str());
        println!("longest: {}", result);
        // 出力: longest: long string
    }
}
```

> **初心者へのアドバイス：** ライフタイムは Rust で最も難しい概念の1つです。最初は「コンパイラが安全でないコードを教えてくれるんだな」程度の理解で大丈夫です。実際にコードを書く中で、コンパイラのエラーメッセージを読みながら徐々に理解を深めていきましょう。

### ポイントまとめ

- ライフタイムは参照が有効な期間を表します
- コンパイラがライフタイムを自動チェックし、ダングリング参照を防ぎます
- 多くの場合は自動推論されますが、複雑なケースでは注釈が必要です
- 最初から完璧に理解する必要はなく、コンパイラのメッセージに従って学べます

## 1.10 まとめと実践例

### 第1章の総合サンプル

ここまで学んだ内容を使った実践的なプログラムを見てみましょう。

```rust
fn main() {
    // === 変数と型 ===
    let user_name = String::from("田中太郎");
    let age: u32 = 25;
    let height: f64 = 170.5;
    let is_student = true;

    // 不変参照を渡して表示
    print_profile(&user_name, age, height, is_student);

    // === 所有権とムーブ ===
    let greeting = create_greeting(&user_name);
    println!("{}", greeting);
    // user_name はムーブされていないので、まだ使える
    println!("名前はまだ使えます: {}", user_name);

    // === 可変参照 ===
    let mut scores: Vec<u32> = vec![85, 92, 78, 95, 88];
    println!("\n成績処理:");
    println!("元の成績: {:?}", scores);

    add_bonus(&mut scores, 5);
    println!("ボーナス後: {:?}", scores);

    let average = calculate_average(&scores);
    println!("平均点: {:.1}", average);
}

fn print_profile(name: &String, age: u32, height: f64, is_student: bool) {
    println!("=== プロフィール ===");
    println!("名前: {}", name);
    println!("年齢: {}歳", age);
    println!("身長: {}cm", height);
    println!("学生: {}", if is_student { "はい" } else { "いいえ" });
    println!("====================");
}

fn create_greeting(name: &String) -> String {
    format!("ようこそ、{}さん！", name)
}

fn add_bonus(scores: &mut Vec<u32>, bonus: u32) {
    for score in scores.iter_mut() {
        *score += bonus;
    }
}

fn calculate_average(scores: &Vec<u32>) -> f64 {
    let sum: u32 = scores.iter().sum();
    sum as f64 / scores.len() as f64
}
```

学生: はい

名前はまだ使えます: 田中太郎

成績処理:
元の成績: [85, 92, 78, 95, 88]
ボーナス後: [90, 97, 83, 100, 93]
平均点: 92.6
```
