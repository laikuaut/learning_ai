<details>
<summary>ステップ1：要件を整理する</summary>

機能要件と非機能要件を分けて整理しましょう。

**機能要件の例**:
- FAQに基づく質問回答ができる
- 注文番号で注文状況を検索できる
- 返品手続きの案内と申請ができる

**非機能要件の例**:
- 応答時間: 5秒以内
- 可用性: 99.5%以上
- 同時対応数: 100セッション

<details>
<summary>ステップ2：アーキテクチャを設計する</summary>

エージェントの内部構造を設計します。

主要コンポーネント:
1. **意図分類（Intent Classification）**: ユーザーの質問を分類する
2. **ルーティング**: 分類結果に応じて適切なハンドラーに振り分ける
3. **各ドメインハンドラー**: FAQ検索、注文対応、技術サポートなど
4. **RAG検索**: FAQやマニュアルからの情報検索
5. **外部API連携**: 注文管理システム、CRMなど

**よくある間違い**: すべてを1つのプロンプトで処理しようとすること。意図分類 → 専門ハンドラーの2段階にすることで、各ハンドラーのプロンプトをシンプルに保てます。

</details>

### 1. 意図分類の実装設計

```python
# 擬似コード: 2段階意図分類

class IntentClassifier:
    def classify(self, user_message, conversation_history):
        """ユーザーの意図を分類する"""

        # 1段階目: 高速なルールベース分類
        rule_result = self.rule_based_classify(user_message)
        if rule_result.confidence > 0.95:
            return rule_result

        # 2段階目: LLMベース分類（ルールで判定できなかった場合）
        llm_result = self.llm_classify(user_message, conversation_history)
        return llm_result

    def rule_based_classify(self, message):
        """キーワードベースの高速分類"""
        rules = {
            "order_status": ["注文", "届く", "配送", "追跡"],
            "return_exchange": ["返品", "交換", "返金", "初期不良"],
            "escalation": ["オペレーター", "人間", "上司", "クレーム"],
        }

        for intent, keywords in rules.items():
            matches = sum(1 for kw in keywords if kw in message)
            if matches >= 2:
                return IntentResult(intent=intent, confidence=0.96)

        return IntentResult(intent="unknown", confidence=0.0)

    def llm_classify(self, message, history):
        """LLMを使った高精度分類"""
        prompt = f"""
        以下のユーザーメッセージを分類してください。

        カテゴリ:
        - faq_general: 一般的な質問（送料、営業時間、支払い方法など）
        - order_status: 注文状況の確認
        - order_change: 注文の変更・キャンセル
        - tech_support: 技術的な問題
        - return_exchange: 返品・交換
        - escalation: 人間のオペレーターへの引き継ぎ要求
        - other: その他

        会話履歴: {format_history(history)}
        ユーザーメッセージ: {message}

        JSON形式で回答: {{"intent": "カテゴリ名", "confidence": 0.0-1.0}}
        """
        return llm.generate(prompt)
```

```python
# 擬似コード: 感情分析とエスカレーション判定

class SentimentAnalyzer:
    def analyze(self, message, conversation_history):
        """ユーザーの感情を分析する"""
        prompt = f"""
        ユーザーメッセージの感情を分析してください。

        メッセージ: {message}
        直近の会話: {format_recent(conversation_history, n=3)}

        JSON形式:
        {{
          "sentiment": "positive|neutral|negative|angry",
          "frustration_level": 1-5,
          "escalation_recommended": true/false,
          "reason": "判断理由"
        }}
        """
        return llm.generate(prompt)

class EscalationManager:
    def should_escalate(self, sentiment, session_state):
        """エスカレーション判定"""

        # 即時エスカレーション条件
        if sentiment.sentiment == "angry" and sentiment.frustration_level >= 4:
            return True, "ユーザーの不満が高いレベルに達しています"

        if session_state.failed_resolution_count >= 3:
            return True, "3回以上解決に至っていません"

        if session_state.order_value >= 100000:
            return True, "高額案件のため人間対応が必要です"

        # エスカレーション提案条件（ユーザーに確認）
        if sentiment.frustration_level >= 3:
            return "suggest", "ご不便をおかけしているようです。人間のオペレーターにお繋ぎしましょうか？"

        return False, None
```

### 6. 改善サイクル

```python
# 擬似コード: 月次改善サイクル

def monthly_improvement_cycle():
    """月次の改善サイクル"""

    # 1. 未解決の質問を収集
    unresolved = query_logs(
        filter={"escalated": True, "auto_resolved": False},
        period="last_30_days"
    )

    # 2. 未解決質問をクラスタリング
    clusters = cluster_questions(unresolved)

    # 3. 頻出パターンを特定
    for cluster in sorted(clusters, key=lambda c: c.count, reverse=True):
        print(f"パターン: {cluster.label}")
        print(f"  出現回数: {cluster.count}")
        print(f"  対応策:")

        if cluster.type == "missing_faq":
            print(f"  → FAQに回答を追加")
            faq_db.add_entry(cluster.suggested_faq)

        elif cluster.type == "ambiguous_intent":
            print(f"  → 意図分類の学習データを追加")
            intent_classifier.add_training_data(cluster.examples)

        elif cluster.type == "tool_limitation":
            print(f"  → 新しいツール/APIの追加を検討")
            create_feature_request(cluster.description)

    # 4. 改善後の評価
    run_regression_tests()
```

**プロの知見**: カスタマーサポートエージェントで最も重要なのは「いつ人間に引き継ぐか」の判断です。エージェントが「できない」ことを素直に認めてスムーズにエスカレーションできるシステムは、無理に回答しようとして顧客を苛立たせるシステムよりも、ユーザー満足度が高くなります。完璧な自動応答を目指すのではなく、「自動化すべきところは自動化し、人間が対応すべきところはスムーズに引き継ぐ」という設計思想が成功の鍵です。
