# 第2章：基本操作とプロンプト入力

> **この章のゴール**: Claude Codeの基本的な操作方法を習得し、効果的なプロンプト入力で開発作業を効率化できるようになること

---

## 2.1 起動オプション

Claude Codeにはさまざまな起動オプション（startup options）があります。

### 主要な起動オプション

```bash
# 通常の対話モードで起動
claude

# ワンショットモード（1回だけ実行して終了）
claude -p "READMEを要約して"

# モデルを指定して起動
claude --model claude-sonnet-4-6

# 出力形式を指定（JSON形式）
claude -p "関数一覧を出して" --output-format json

# 会話を再開（前回のセッションを継続）
claude --resume

# 最大ターン数を指定
claude --max-turns 10

# システムプロンプトを追加
claude --append-system-prompt "常にTypeScriptで回答して"

# ヘルプを表示
claude --help
```

### よく使う組み合わせ

```bash
# パイプで入力を渡してワンショット実行
cat error.log | claude -p "このエラーの原因は？"

# ファイルの内容を渡して分析
claude -p "このコードをレビューして" < app.py

# 結果をファイルに保存
claude -p "テストケースを生成して" > tests.py
```

### ポイントまとめ
- `-p`フラグでワンショットモード（対話なしの1回実行）
- `--model`でAIモデルを選択可能
- `--resume`で前回の会話を再開できる
- パイプやリダイレクトと組み合わせて自動化が可能

## 2.2 スラッシュコマンド

対話モード中に使える**スラッシュコマンド**（slash commands）は、Claude Codeの操作を効率化する重要な機能です。

### コマンド一覧

```
┌──────────────────┬───────────────────────────────────────┐
│ コマンド          │ 説明                                  │
├──────────────────┼───────────────────────────────────────┤
│ /help            │ ヘルプを表示                           │
│ /clear           │ 会話履歴をクリア                       │
│ /compact          │ 会話を要約して圧縮                    │
│ /cost            │ 現在のセッションのコストを表示          │
│ /model           │ 使用するモデルを変更                   │
│ /status          │ 現在の設定状態を表示                   │
│ /fast            │ 高速モード（同じモデルで高速出力）     │
│ /commit          │ 変更をGitコミット                      │
│ /review-pr       │ PRをレビュー                           │
│ /help            │ 使い方のヘルプを表示                   │
└──────────────────┴───────────────────────────────────────┘
```

### 各コマンドの使い方

#### /clear - 会話のリセット

```
> /clear
```

新しいタスクに取り掛かるときに使います。会話履歴がクリアされ、新しいセッションが始まります。前のタスクのコンテキスト（context：文脈情報）が不要な場合に有効です。

#### /compact - 会話の圧縮

```
> /compact
```

会話が長くなるとコンテキストウィンドウ（context window：AIが一度に処理できるテキスト量）の上限に近づきます。`/compact`を使うと、これまでの会話内容を要約して圧縮し、コンテキストの空きを確保できます。

#### /cost - コスト確認

```
> /cost

Session cost: $0.42
Tokens used: 125,000 input / 8,500 output
```

#### /model - モデル変更

```
> /model
# → モデル選択画面が表示されます
# claude-opus-4-6 (高性能)
# claude-sonnet-4-6 (バランス型)
```

### よくある間違い
> **/clearと/compactの混同**: `/clear`は会話を完全にリセットします。直前のやり取りを覚えておいてほしい場合は`/compact`を使ってください。`/compact`は会話を要約して保持するので、文脈を維持したまま容量を節約できます。

### ポイントまとめ
- スラッシュコマンドは`/`で始まる特別なコマンド
- `/compact`は会話を圧縮してコンテキストを節約
- `/cost`でセッションのコストを随時確認
- `/model`でモデルを切り替え可能

## 2.3 キーボードショートカット

Claude Codeの対話モードでは、効率的な操作のためのキーボードショートカット（keyboard shortcuts）が用意されています。

### 主要なショートカット

```
┌─────────────────────┬──────────────────────────────────┐
│ ショートカット       │ 機能                             │
├─────────────────────┼──────────────────────────────────┤
│ Enter               │ メッセージを送信                 │
│ Shift + Enter       │ 改行を挿入（複数行入力）         │
│ Escape              │ 現在の操作をキャンセル           │
│ Ctrl + C            │ 実行中のタスクを中断             │
│ Ctrl + D            │ セッションを終了                 │
│ ↑ / ↓               │ 入力履歴を移動                   │
│ Tab                 │ 補完候補を表示                   │
└─────────────────────┴──────────────────────────────────┘
```

### 複数行入力

長い指示を入力する場合は、`Shift + Enter`で改行できます：

```
> 以下の要件でユーザー登録機能を作成してください：  [Shift+Enter]
  - メールアドレスのバリデーション                   [Shift+Enter]
  - パスワードは8文字以上                            [Shift+Enter]
  - 確認用パスワードの一致チェック                   [Enter で送信]
```

### ポイントまとめ
- `Shift + Enter`で改行、`Enter`で送信
- `Escape`で操作キャンセル、`Ctrl + C`でタスク中断
- 複数行で入力すると構造化された指示を出しやすい

## 2.4 権限モード（Permission Modes）

Claude Codeは安全のため、ツール実行時にユーザーの許可を求めます。この挙動は権限モード（permission modes）で制御できます。

### 承認の選択肢

ツール実行の確認時に、以下の選択肢が表示されます：

```
Claude Code が Edit(src/app.ts) を実行しようとしています。
[Y]es / [N]o / [A]lways allow
```

| 選択肢 | 説明 |
|---|---|
| **Y（Yes）** | 今回だけ許可します |
| **N（No）** | 今回は拒否します |
| **A（Always allow）** | このツールを以降自動的に許可します |

### allowedTools設定

特定のツールを事前に許可しておくことで、毎回の確認を省略できます。設定は`.claude/settings.json`で行います：

```json
{
  "permissions": {
    "allow": [
      "Read",
      "Glob",
      "Grep"
    ]
  }
}
```

この設定により、ファイルの読み込みと検索は確認なしで実行され、書き込みや実行は引き続き確認が求められます。

### 推奨設定

```
┌─────────────────────────────────────────────────────────┐
│              推奨権限設定                                 │
├─────────────────┬───────────────────────────────────────┤
│ 常に許可        │ Read, Glob, Grep（読み取り系）        │
│ 都度確認        │ Edit, Write, Bash（書き込み・実行系） │
│ 慎重に確認      │ git push, rm などの破壊的操作         │
└─────────────────┴───────────────────────────────────────┘
```

### よくある間違い
> **すべて自動承認にする危険性**: 初心者のうちは全ツールを自動承認にしないでください。Claude Codeがどのような操作を行うのか確認しながら学ぶことで、ツールの動作を理解できます。慣れてきたら読み取り系のツールから自動承認を設定していくのがおすすめです。

### ポイントまとめ
- ツール実行時に Yes / No / Always の選択肢が表示される
- `.claude/settings.json`で事前に許可するツールを設定できる
- 読み取り系は自動許可、書き込み系は都度確認が安全

## 2.5 効果的なプロンプトの書き方

Claude Codeを使いこなすには、**適切なプロンプト**（prompt：指示文）を書くことが重要です。

### 基本原則

#### 1. 具体的に指示する

```
❌ 悪い例：
> コードを直して

✅ 良い例：
> src/utils/validate.ts の validateEmail 関数で、
> ドメイン部分に数字が含まれるメールアドレスが
> 不正と判定されるバグを修正してください
```

#### 2. コンテキストを提供する

```
❌ 悪い例：
> テストを書いて

✅ 良い例：
> src/services/userService.ts の createUser 関数に対する
> ユニットテストを Jest で書いてください。
> 正常系と異常系（バリデーションエラー、重複エラー）を
> カバーしてください
```

#### 3. 段階的に進める

```
# 大きなタスクは分割して指示
> まず、このプロジェクトの構成を説明してください

（回答を確認後）

> 次に、src/api/ ディレクトリのエンドポイント一覧を教えてください

（回答を確認後）

> では、POST /api/users エンドポイントにバリデーションを追加してください
```

### プロンプトのパターン集

```
┌─────────────────┬───────────────────────────────────────┐
│ 目的             │ プロンプト例                           │
├─────────────────┼───────────────────────────────────────┤
│ コード理解       │ 「この関数の処理の流れを説明して」     │
│ バグ修正         │ 「○○のエラーの原因を調べて修正して」  │
│ 機能追加         │ 「△△の機能を追加してください」        │
│ リファクタリング │ 「この関数を小さな関数に分割して」     │
│ テスト作成       │ 「□□のユニットテストを書いて」        │
│ コードレビュー   │ 「このファイルの問題点を指摘して」     │
│ ドキュメント     │ 「この関数にJSDocコメントを追加して」  │
│ デバッグ         │ 「このエラーログの原因を分析して」     │
└─────────────────┴───────────────────────────────────────┘
```

### よくある間違い
> **曖昧な指示を出す**: 「いい感じにして」「もっときれいに」などの曖昧な指示は、期待と異なる結果になりやすいです。何をどう変更したいのか具体的に伝えましょう。

> **一度に大量の変更を依頼する**: 「プロジェクト全体をリファクタリングして」のような大きすぎるタスクは、品質が下がりやすいです。ファイル単位や機能単位で分割して依頼しましょう。

### ポイントまとめ
- 具体的な指示を出すことで精度が上がる
- ファイル名・関数名・エラーメッセージなど具体的な情報を含める
- 大きなタスクは段階的に分割して進める

## 2.6 セッション管理

### セッションの継続と再開

Claude Codeの会話はセッション（session：一連のやり取り）として管理されます。

```bash
# 前回のセッションを再開
claude --resume

# セッション一覧を表示して選択
claude --resume
```

### セッション中の状態確認

```
# 現在の状態を確認
> /status

# 出力例:
# Model: claude-opus-4-6
# Context: 45,000 / 200,000 tokens
# Session: 15 messages
# Working directory: /home/user/my-project
```

### コンテキストウィンドウの管理

AIモデルが一度に処理できるテキスト量には上限があります。効率的に使うためのコツを紹介します：

```
┌──────────────────────────────────────────────────────┐
│        コンテキスト管理のベストプラクティス            │
├──────────────────────────────────────────────────────┤
│                                                      │
│  1. 不要になった話題は /clear でリセット              │
│  2. 長い会話は /compact で圧縮                       │
│  3. 新しいタスクは新しいセッションで始める            │
│  4. 大きなファイルは必要な部分だけ参照させる          │
│  5. 自動圧縮機能が働くので過度に心配しなくてOK       │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### よくある間違い
> **1つのセッションで何でもやろうとする**: タスクが変わったら`/clear`で新しく始めるか、新しいターミナルで`claude`を起動しましょう。無関係な会話履歴があると、AIの回答精度が下がることがあります。
