# 第1章：ネットワークの基礎 演習問題

## 基本問題

### 問題1-1：ネットワークの分類
以下のネットワークの分類について、正しい組み合わせを選んでください。

| 分類 | 範囲 |
|------|------|
| (a) PAN | (1) 都市規模 |
| (b) LAN | (2) 数メートル |
| (c) MAN | (3) 国・大陸規模 |
| (d) WAN | (4) 建物内〜敷地内 |

<details>
<summary>解答</summary>

- (a) PAN → (2) 数メートル（例：Bluetooth接続）
- (b) LAN → (4) 建物内〜敷地内（例：社内ネットワーク）
- (c) MAN → (1) 都市規模（例：市内光ファイバ網）
- (d) WAN → (3) 国・大陸規模（例：インターネット）

</details>

### 問題1-2：LAN構成要素
以下のネットワーク機器とその役割の正しい組み合わせを答えてください。

1. NIC
2. L2スイッチ
3. ルータ

(A) 異なるネットワーク間のパケット転送
(B) PCをネットワークに接続するインターフェース
(C) 同一LAN内のフレーム転送

<details>
<summary>解答</summary>

1. NIC → (B) PCをネットワークに接続するインターフェース
2. L2スイッチ → (C) 同一LAN内のフレーム転送（MACアドレスを基に転送）
3. ルータ → (A) 異なるネットワーク間のパケット転送（IPアドレスを基にルーティング）

</details>

### 問題1-3：OSI参照モデル
OSI参照モデルの7層を下位層から順番に並べ、各層の主なプロトコルまたは機器を1つずつ挙げてください。

<details>
<summary>解答</summary>

| 層 | 名称 | プロトコル/機器の例 |
|----|------|-------------------|
| 第1層 | 物理層 | リピータ、ハブ、ケーブル |
| 第2層 | データリンク層 | Ethernet、L2スイッチ |
| 第3層 | ネットワーク層 | IP、ICMP、ルータ |
| 第4層 | トランスポート層 | TCP、UDP |
| 第5層 | セッション層 | NetBIOS、RPC |
| 第6層 | プレゼンテーション層 | SSL/TLS、JPEG |
| 第7層 | アプリケーション層 | HTTP、DNS、SMTP |

覚え方：「ブ（物理）デ（データリンク）ネ（ネットワーク）ト（トランスポート）セ（セッション）プ（プレゼンテーション）ア（アプリケーション）」

</details>

### 問題1-4：PDUの名称
各層で扱うPDU（Protocol Data Unit）の名称を答えてください。

1. トランスポート層（TCP使用時）
2. ネットワーク層
3. データリンク層
4. 物理層

<details>
<summary>解答</summary>

1. トランスポート層（TCP）→ **セグメント**（UDPの場合は「データグラム」）
2. ネットワーク層 → **パケット**
3. データリンク層 → **フレーム**
4. 物理層 → **ビット**

</details>

## 応用問題

### 問題1-5：カプセル化
WebブラウザでHTTPリクエストを送信する際のカプセル化の流れを、各層で付加されるヘッダとともに説明してください。

<details>
<summary>解答</summary>

1. **アプリケーション層**：HTTPリクエストメッセージ（データ）を作成
2. **トランスポート層**：TCPヘッダ（送信元ポート、宛先ポート80、シーケンス番号等）を付加 → **セグメント**
3. **ネットワーク層**：IPヘッダ（送信元IP、宛先IP、TTL等）を付加 → **パケット**
4. **データリンク層**：MACヘッダ（送信元MAC、宛先MAC）とFCS（誤り検出）を付加 → **フレーム**
5. **物理層**：フレームを電気信号/光信号のビット列に変換して送出

受信側では逆の順序（脱カプセル化）で各ヘッダを除去し、最終的にHTTPデータを取り出します。

</details>

### 問題1-6：TCP/IPモデル
OSI参照モデルとTCP/IPモデルの対応関係について、以下の問いに答えてください。

(1) TCP/IPモデルのアプリケーション層は、OSI参照モデルのどの層に対応しますか。
(2) TCP/IPモデルのネットワークインターフェース層は、OSI参照モデルのどの層に対応しますか。
(3) TCP/IPモデルが4層なのに対してOSI参照モデルが7層である理由を簡単に述べてください。

<details>
<summary>解答</summary>

(1) OSI参照モデルの第5層（セッション層）、第6層（プレゼンテーション層）、第7層（アプリケーション層）に対応します。

(2) OSI参照モデルの第1層（物理層）と第2層（データリンク層）に対応します。

(3) OSI参照モデルは通信機能を理論的に整理した「参照モデル」であり、各層の役割を明確に分離しています。一方TCP/IPモデルは実際のインターネットで使われるプロトコル群に基づいた実装寄りのモデルであり、実運用では分ける必要性が低い層（セッション層・プレゼンテーション層等）をまとめた結果、4層構成となっています。

</details>

### 問題1-7：ネットワークトポロジ
以下のトポロジの特徴を簡潔に述べてください。

1. スター型
2. メッシュ型（フルメッシュ）
3. バス型

<details>
<summary>解答</summary>

1. **スター型**：中央のハブやスイッチに各端末が接続される形態。1台の端末の障害が他に影響しないが、中央機器が故障すると全体が停止する。現在のLANで最も一般的。

2. **メッシュ型（フルメッシュ）**：すべての機器同士が相互に接続される形態。冗長性が最も高く1つのリンクが切れても通信を継続できるが、接続数が多くコストが高い。n台の場合 n(n-1)/2 本の接続が必要。

3. **バス型**：1本の共有ケーブル（バス）に全端末が接続される形態。構造がシンプルで低コストだが、バスが切断されると全体が影響を受ける。現在はほとんど使われていない。

</details>

## チャレンジ問題

### 問題1-8：総合問題
ある企業が新オフィスにネットワークを構築する際、以下の要件を満たす構成を考えてください。

- 社内PCが50台
- 社内にWebサーバとファイルサーバを設置
- インターネット接続が必要
- 1台のPC障害が他のPCに影響しないこと

(1) 適切なトポロジを選び、理由を述べてください。
(2) 必要なネットワーク機器を列挙してください。
(3) 簡単なネットワーク構成図を描いてください（テキストで可）。

<details>
<summary>解答例</summary>

**(1) トポロジの選択**
**スター型（スター拡張型）** が適切です。理由：
- 1台のPCの障害が他のPCに影響しない
- 管理が容易（端末の追加・撤去が簡単）
- 現在のイーサネットLANの標準的な構成

**(2) 必要なネットワーク機器**
- **ルータ** × 1台：インターネット接続用
- **L3スイッチ**（またはルータ）× 1台：VLAN間ルーティング（必要に応じて）
- **L2スイッチ** × 2〜3台：PC50台を収容（24ポートまたは48ポートスイッチ）
- **NIC**：各PCに内蔵
- **UTPケーブル（Cat6以上）**：各PC接続用
- **ファイアウォール**：インターネットとの境界

**(3) ネットワーク構成図**
```
インターネット
     │
 [ルータ/FW]
     │
 [L3スイッチ]
  /       \
[L2スイッチ1]  [L2スイッチ2]
 / | | \      / | | \
PC PC PC PC  PC PC PC PC ... (合計50台)

[Webサーバ]  [ファイルサーバ]
（L2スイッチに接続、またはDMZに配置）
```

</details>

### 問題1-9：WAN接続方式の選択
以下のシナリオに最適なWAN接続方式を選び、理由を述べてください。

(1) 大企業の本社と支社間で、高いセキュリティと帯域保証が必要
(2) 中小企業の拠点間接続で、コストを抑えたい
(3) 銀行の基幹系システムで、最高レベルの信頼性が必要

<details>
<summary>解答例</summary>

(1) **IP-VPN** が適切です。通信事業者のMPLS網を利用した閉域網で、セキュリティが高く、帯域保証（SLA）のあるサービスを選択できます。専用線ほど高コストではなく、企業拠点間接続に広く使われています。

(2) **インターネットVPN** が適切です。インターネット上にIPsecやSSL-VPNでトンネルを構築する方式で、低コストで拠点間接続を実現できます。ベストエフォートですが、中小企業の用途には十分です。

(3) **専用線** が適切です。帯域保証があり、他の通信の影響を受けない完全な占有回線です。最もコストが高いですが、銀行の基幹系のような高信頼性が求められるシステムには最適です。

</details>
