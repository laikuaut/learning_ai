# 実践課題02：プロンプト設計ワークショップ ★1

> **難易度**: ★☆☆☆☆（入門）
> **前提知識**: 第2章（テキスト生成AIの活用）
> **課題の種類**: ミニプロジェクト
> **学習目標**: 良いプロンプトと悪いプロンプトの違いを理解し、テキスト生成AIから期待通りの出力を得るための基本的なプロンプト設計力を身につける

## 課題の要件

1. 以下の5つのテーマそれぞれについて、「悪いプロンプト」と「良いプロンプト」のペアを作成する
   - テーマ1：文章の要約
   - テーマ2：メールの作成
   - テーマ3：アイデア出し（ブレインストーミング（brainstorming））
   - テーマ4：翻訳
   - テーマ5：説明文の作成
2. 各テーマで「悪いプロンプト」を実際にAIに入力し、出力結果を記録する
3. 「良いプロンプト」に改善し、同様に出力結果を記録する
4. 各テーマごとに「なぜ良いプロンプトの方が優れた結果になるのか」を解説する
5. 最後に「良いプロンプトの共通ルール」を5つ以上抽出してまとめる

---

## ステップガイド

<details>
<summary>ステップ1：悪いプロンプトのパターンを理解する</summary>

悪いプロンプトには以下の共通パターンがあります。

1. **曖昧すぎる**: 「いい感じにして」「なんかまとめて」
2. **コンテキスト不足**: 前提条件や背景情報がない
3. **出力形式が不明**: どんな形式で返してほしいか指定がない
4. **対象読者が不明**: 誰向けの出力かわからない
5. **一度に多くを求めすぎる**: 1つのプロンプトに複数の要求を詰め込む

まず各テーマで「ありがちな悪いプロンプト」を書いてみましょう。

</details>

<details>
<summary>ステップ2：良いプロンプトの原則を適用する</summary>

以下の原則を使って、悪いプロンプトを改善しましょう。

- **役割（Role）を指定する**: 「あなたは〇〇の専門家です」
- **具体的な指示を出す**: 何を、どのように、どれくらいの量で
- **出力形式を指定する**: 箇条書き、表形式、〇文字以内 など
- **コンテキストを提供する**: 背景情報や前提条件を伝える
- **制約条件（constraint）を明示する**: やってほしくないことも伝える

</details>

<details>
<summary>ステップ3：実際にAIに入力して比較する</summary>

各テーマで以下のフォーマットで記録しましょう。

```
【テーマ名】
■ 悪いプロンプト：
（プロンプト文）

■ 悪いプロンプトの出力結果：
（AIの出力を貼り付け）

■ 良いプロンプト：
（プロンプト文）

■ 良いプロンプトの出力結果：
（AIの出力を貼り付け）

■ 改善ポイント：
（なぜ良くなったかの解説）
```

**よくある間違い**: 出力結果を記録せずに「良くなった」とだけ書くこと。具体的な出力を比較しないと、改善ポイントが曖昧になります。

</details>

<details>
<summary>ステップ4：共通ルールを抽出する</summary>

5テーマの改善パターンを振り返り、共通する「良いプロンプトのルール」を抽出しましょう。

例：
- ルール1：出力形式を明示する
- ルール2：具体的な数値や条件を含める
- ...

5つ以上を目標にまとめてください。

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

### プロンプト設計ワークショップ 成果物

---

#### テーマ1：文章の要約

**悪いプロンプト:**
```
要約して
```

**悪いプロンプトの出力結果:**
何を要約すべきか不明なため、AIが推測で対応。長い文章や的外れな内容になりがちです。

**良いプロンプト:**
```
以下のニュース記事を、小学6年生でも理解できるように3行で要約してください。
専門用語は使わず、平易な日本語でお願いします。

（記事本文をここに貼り付け）
```

**良いプロンプトの出力結果:**
指定通り3行で、分かりやすい表現で要約が得られます。

**改善ポイント:**
- 対象読者（小学6年生）を指定した
- 出力量（3行）を明示した
- 文体の制約（専門用語を使わない）を指定した

---

#### テーマ2：メールの作成

**悪いプロンプト:**
```
お礼のメールを書いて
```

**良いプロンプト:**
```
以下の状況でビジネスメールを作成してください。

- 宛先：取引先の田中部長
- 目的：先日の会食のお礼
- トーン：丁寧かつフォーマル
- 長さ：200文字程度
- 含めたい内容：料理が美味しかったこと、次回の打ち合わせの提案
```

**改善ポイント:**
- 宛先・目的・トーンを具体的に指定した
- 文字数の目安を示した
- 含めるべき内容を箇条書きで整理した

---

#### テーマ3：アイデア出し

**悪いプロンプト:**
```
新商品のアイデアを出して
```

**良いプロンプト:**
```
あなたは食品メーカーの商品企画担当です。
以下の条件で新商品のアイデアを5つ提案してください。

- ターゲット：20代の一人暮らし社会人
- カテゴリ：冷凍食品
- 価格帯：300〜500円
- 差別化ポイント：健康志向 + 時短
- 各アイデアに「商品名」「概要（50文字以内）」「想定される訴求ポイント」を含めてください
```

**改善ポイント:**
- 役割を指定した（商品企画担当）
- アイデアの数を指定した（5つ）
- 制約条件を具体的に設定した（ターゲット、価格帯等）
- 出力フォーマットを指定した

---

#### テーマ4：翻訳

**悪いプロンプト:**
```
英語に訳して：この商品は高品質です
```

**良いプロンプト:**
```
以下の日本語を英語に翻訳してください。

対象文：「この商品は高品質で、10年以上の耐久性があります」

条件：
- ビジネス向けの製品カタログで使用する文章です
- フォーマルかつ説得力のあるトーンでお願いします
- 直訳ではなく、英語として自然な表現にしてください
- 翻訳文の後に、選んだ表現の理由を簡単に補足してください
```

**改善ポイント:**
- 使用場面（製品カタログ）を明示した
- トーンを指定した（フォーマル・説得力）
- 直訳でなく自然な訳を求めた
- 翻訳の根拠説明を求めた

---

#### テーマ5：説明文の作成

**悪いプロンプト:**
```
AIについて説明して
```

**良いプロンプト:**
```
IT企業の新入社員研修資料に掲載する「生成AIとは何か」の説明文を作成してください。

条件：
- 読者：ITの基礎知識はあるが、AIの専門知識はない新入社員
- 長さ：300〜400文字
- 構成：①定義 → ②身近な例 → ③業務での活用場面
- 専門用語を使う場合は必ず簡単な説明を付けてください
```

**改善ポイント:**
- 読者のレベルを明示した
- 用途（研修資料）を指定した
- 文字数と構成を指定した
- 専門用語の扱い方を指示した

---

### 良いプロンプトの共通ルール

1. **出力形式を指定する**: 箇条書き、文字数、行数など具体的に
2. **対象読者・利用場面を明示する**: 誰が読むか、どこで使うか
3. **具体的な条件・制約を設定する**: 曖昧さを排除する
4. **役割を与える**: 「あなたは〇〇です」で専門性を引き出す
5. **含めるべき要素を列挙する**: 何を含めてほしいか箇条書きで整理

<summary>解答例（改良版）</summary>

### プロンプト設計ワークショップ 成果物（改良版）

改良版では以下を追加しています。

- **プロンプトの改善を段階的に見せる**（悪い → まあまあ → 良い の3段階）
- **各テーマにスコアリング基準を設定**
- **プロンプトテンプレートを提供**

---

#### テーマ1：文章の要約（3段階改善）

**レベル1（悪い）:**
```
要約して
```
→ 問題：何を、どの程度、誰向けに要約するか不明

**レベル2（まあまあ）:**
```
以下の文章を3行で要約してください。
（文章）
```
→ 改善：出力量を指定。まだ対象読者や目的が不明

**レベル3（良い）:**
```
あなたはビジネスライターです。
以下のプレスリリースを、社内の週次報告メールに掲載するため、
3つの箇条書き（各30文字以内）で要約してください。

要約に含めるべきポイント：
- 何が発表されたか
- いつから開始か
- ビジネスへの影響

（プレスリリース本文）
```

**スコアリング:**
| 評価軸 | レベル1 | レベル2 | レベル3 |
|--------|:------:|:------:|:------:|
| 明確さ | 1/5 | 3/5 | 5/5 |
| 再現性 | 1/5 | 3/5 | 5/5 |
| 出力品質 | 1/5 | 3/5 | 4/5 |

---

#### 汎用プロンプトテンプレート

以下のテンプレートを使えば、あらゆるタスクで安定した結果を得られます。

```
【役割】あなたは{専門分野}の専門家です。

【タスク】以下の{入力内容}を{目的}のために{アクション}してください。

【対象読者】{誰が読むか}

【出力形式】
- 形式：{箇条書き/表/段落}
- 長さ：{文字数や行数}
- トーン：{フォーマル/カジュアル/技術的}

【含めるべき要素】
1. {要素1}
2. {要素2}
3. {要素3}

【制約条件】
- {やってほしくないこと}
- {避けるべき表現}

【入力】
{ここに処理対象のテキストを貼り付け}
```

---

### 良いプロンプトの共通ルール（拡張版：8つ）

1. **出力形式を指定する**: 箇条書き、表、段落、文字数を明示
2. **対象読者を明示する**: 読者の知識レベルや立場を伝える
3. **具体的な条件・制約を設定する**: 数値、範囲、除外条件を含める
4. **役割を与える**: 専門的な視点からの回答を引き出す
5. **含めるべき要素を列挙する**: 出力に必ず含めてほしい情報を指定
6. **目的・用途を伝える**: 何のために使うかを説明する
7. **例を示す（Few-shot（フューショット））**: 期待する出力の例を1〜2個提示する
8. **段階的に指示する（Chain of Thought（思考の連鎖））**: 複雑なタスクは手順を分けて指示する

**初心者向けとの違い**:
- 2段階（悪い→良い）ではなく3段階（悪い→まあまあ→良い）で改善プロセスを可視化
- スコアリング基準を導入し、プロンプトの品質を定量的に評価
- 汎用テンプレートを提供し、どんなタスクにも応用できるようにした
- ルールを5つから8つに拡張し、Few-shotやChain of Thoughtなど上級テクニックも紹介

</details>
