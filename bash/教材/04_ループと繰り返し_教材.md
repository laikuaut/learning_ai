# 第4章 ループと繰り返し

## この章のゴール

- `for` ループの各種書き方（リスト形式、C言語スタイル、ブレース展開）を使い分けられる
- `while` ループでファイルの行読み取りなど実践的な処理が書ける
- `until` ループの使いどころを理解する
- `break` と `continue` でループ制御ができる
- `select` 文で対話的メニューを作成できる
- パイプやプロセス置換とループを組み合わせた高度な処理が書ける

## 4.1 forループ

### リスト形式の for ループ

最も基本的な `for` ループは、リスト（一覧）の各要素に対して処理を繰り返します。

```bash
# 基本構文
for 変数 in リスト; do
    処理
done
```

```bash
#!/usr/bin/env bash
# 基本的な for ループ

# 固定リストを使う
for fruit in apple banana cherry; do
    echo "果物: $fruit"
done

# 出力:
# 果物: apple
# 果物: banana
# 果物: cherry
```

```bash
# 変数やコマンド置換を使う
files=$(ls *.txt 2>/dev/null)
for file in $files; do
    echo "テキストファイル: $file"
done

# グロブパターンを直接使う（推奨）
for file in *.txt; do
    # ファイルが存在しない場合のガード
    [[ -f "$file" ]] || continue
    echo "テキストファイル: $file"
done
```

> **現場のベストプラクティス**: `for file in $(ls *.txt)` ではなく `for file in *.txt` を使いましょう。`ls` の出力をパースすると、スペースを含むファイル名で問題が起きます。

### ブレース展開を使った for ループ

```bash
# 数値の範囲
for i in {1..5}; do
    echo "番号: $i"
done
# 出力: 1, 2, 3, 4, 5

# ステップ（増分）を指定
for i in {0..20..5}; do
    echo "値: $i"
done
# 出力: 0, 5, 10, 15, 20

# ゼロ埋め
for i in {01..10}; do
    echo "ファイル_$i"
done
# 出力: ファイル_01, ファイル_02, ... ファイル_10

# 文字の範囲
for c in {a..f}; do
    echo "文字: $c"
done
# 出力: a, b, c, d, e, f

# 複数のブレース展開を組み合わせる
for item in {A,B}{1,2}; do
    echo "$item"
done
# 出力: A1, A2, B1, B2
```

### C言語スタイルの for ループ

```bash
# C言語スタイル（算術 for ループ）
for ((i = 0; i < 5; i++)); do
    echo "インデックス: $i"
done
# 出力: 0, 1, 2, 3, 4

# ステップを指定
for ((i = 0; i <= 100; i += 10)); do
    echo "値: $i"
done

# 複数の変数を使う
for ((i = 0, j = 10; i < j; i++, j--)); do
    echo "i=$i, j=$j"
done
# 出力:
# i=0, j=10
# i=1, j=9
# i=2, j=8
# i=3, j=7
# i=4, j=6

# 変数を使った範囲指定
max=5
for ((i = 1; i <= max; i++)); do
    echo "$i"
done
```

> **注意**: ブレース展開 `{1..5}` では変数が使えません（`{1..$max}` はエラー）。変数で範囲を指定する場合は、C言語スタイルか `seq` コマンドを使います。

```bash
# 変数を使う場合の代替手段
max=5

# 方法1: C言語スタイル（推奨）
for ((i = 1; i <= max; i++)); do echo $i; done

# 方法2: seq コマンド
for i in $(seq 1 $max); do echo $i; done
```

### 配列を使った for ループ

```bash
# 配列の全要素をループ
fruits=("apple" "banana" "cherry" "dragon fruit")

for fruit in "${fruits[@]}"; do
    echo "果物: $fruit"
done

# インデックス付きでループ
for i in "${!fruits[@]}"; do
    echo "[$i] ${fruits[$i]}"
done
# 出力:
# [0] apple
# [1] banana
# [2] cherry
# [3] dragon fruit
```

## 4.2 whileループ

### 基本構文

```bash
# 基本形
while 条件式; do
    処理
done
```

```bash
#!/usr/bin/env bash
# 基本的な while ループ

count=1
while [[ $count -le 5 ]]; do
    echo "カウント: $count"
    ((count++))
done

# 出力:
# カウント: 1
# カウント: 2
# カウント: 3
# カウント: 4
# カウント: 5
```

### 無限ループ

```bash
# 方法1: true を使う
while true; do
    echo "実行中..."
    sleep 1
    # break で抜ける条件を入れる
done

# 方法2: : (コロン) を使う（true と同じ）
while :; do
    echo "実行中..."
    sleep 1
done

# 方法3: 条件式で常に真
while [[ 1 ]]; do
    echo "実行中..."
    sleep 1
done
```

### ファイルの行読み取り（while read）

`while read` はファイルを1行ずつ読み込む、非常によく使われるパターンです。

```bash
#!/usr/bin/env bash
# ファイルを1行ずつ読み取る

# 方法1: リダイレクトを使う（推奨）
while IFS= read -r line; do
    echo "行: $line"
done < /etc/hostname

# 方法2: パイプを使う（サブシェルで実行されるため注意）
cat /etc/hostname | while IFS= read -r line; do
    echo "行: $line"
done
```

```
IFS= read -r line の各部分の意味:

  IFS=      → 先頭と末尾の空白を削除しない
  read -r   → バックスラッシュをエスケープとして解釈しない
  line      → 読み取った行を格納する変数名

  ※ この3つのオプションを全て付けることで、
     元のデータを正確に読み取ることができます
```

```bash
#!/usr/bin/env bash
# CSVファイルの読み取り

# サンプルCSVデータ
csv_data="太郎,25,東京
花子,30,大阪
次郎,22,名古屋"

# IFS でデリミタ（区切り文字）を指定して読み取る
while IFS=',' read -r name age city; do
    echo "$name さんは $age 歳で $city に住んでいます"
done <<< "$csv_data"

# 出力:
# 太郎 さんは 25 歳で 東京 に住んでいます
# 花子 さんは 30 歳で 大阪 に住んでいます
# 次郎 さんは 22 歳で 名古屋 に住んでいます
```

### パイプとサブシェルの問題

```bash
# パイプの後の while はサブシェルで実行される
count=0
cat /etc/hostname | while IFS= read -r line; do
    ((count++))
done
echo "行数: $count"    # 0！サブシェル内の変更は反映されない

# 解決方法1: リダイレクトを使う（推奨）
count=0
while IFS= read -r line; do
    ((count++))
done < /etc/hostname
echo "行数: $count"    # 正しい行数が表示される

# 解決方法2: プロセス置換を使う
count=0
while IFS= read -r line; do
    ((count++))
done < <(cat /etc/hostname)
echo "行数: $count"    # 正しい行数が表示される
```

> **よくある間違い**: `cat file | while read` はサブシェルで実行されるため、ループ内で設定した変数はループ外からアクセスできません。`while read ... done < file` を使いましょう。

### ユーザー入力を受け取る while ループ

```bash
#!/usr/bin/env bash
# ユーザーが "quit" と入力するまで繰り返す

while true; do
    read -p "コマンドを入力 (quit で終了): " cmd
    case "$cmd" in
        quit | exit | q)
            echo "終了します"
            break
            ;;
        hello)
            echo "こんにちは！"
            ;;
        date)
            date
            ;;
        *)
            echo "不明なコマンド: $cmd"
            ;;
    esac
done
```

## 4.3 untilループ

### 基本構文

`until` ループは `while` の反対で、**条件が偽の間**繰り返します。条件が真になるとループを終了します。

```bash
# 基本形
until 条件式; do
    処理
done
```

```bash
#!/usr/bin/env bash
# until ループの基本例

count=1
until [[ $count -gt 5 ]]; do
    echo "カウント: $count"
    ((count++))
done

# 出力（while [[ $count -le 5 ]] と同じ結果）:
# カウント: 1
# カウント: 2
# カウント: 3
# カウント: 4
# カウント: 5
```

### 実用例：サービスの起動待ち

```bash
#!/usr/bin/env bash
# サービスが起動するまで待つ

echo "サービスの起動を待っています..."

until curl -s http://localhost:8080/health > /dev/null 2>&1; do
    echo "  まだ起動していません。5秒後にリトライ..."
    sleep 5
done

echo "サービスが起動しました！"
```

```bash
#!/usr/bin/env bash
# ファイルが作成されるまで待つ

target_file="/tmp/signal.txt"
echo "ファイルの作成を待っています: $target_file"

until [[ -f "$target_file" ]]; do
    sleep 1
done

echo "ファイルが見つかりました！"
cat "$target_file"
```

## 4.4 break と continue

### break：ループを抜ける

```bash
#!/usr/bin/env bash
# break でループを抜ける

for i in {1..100}; do
    if [[ $i -gt 5 ]]; then
        echo "5を超えたのでループを終了します"
        break
    fi
    echo "値: $i"
done

# 出力:
# 値: 1
# 値: 2
# 値: 3
# 値: 4
# 値: 5
# 5を超えたのでループを終了します
```

### ネストしたループでの break

```bash
#!/usr/bin/env bash
# break N で N 階層のループを抜ける

for i in {1..3}; do
    for j in {1..3}; do
        if [[ $i -eq 2 && $j -eq 2 ]]; then
            echo "i=2, j=2 で外側のループも終了"
            break 2    # 2階層のループを抜ける
        fi
        echo "i=$i, j=$j"
    done
done

# 出力:
# i=1, j=1
# i=1, j=2
# i=1, j=3
# i=2, j=1
# i=2, j=2 で外側のループも終了
```

### continue：現在のイテレーションをスキップ

```bash
#!/usr/bin/env bash
# continue で偶数をスキップ

for i in {1..10}; do
    if (( i % 2 == 0 )); then
        continue    # 偶数はスキップ
    fi
    echo "奇数: $i"
done

# 出力: 1, 3, 5, 7, 9
```

```bash
#!/usr/bin/env bash
# 実務例: 隠しファイルをスキップして処理

for file in *; do
    # ディレクトリはスキップ
    [[ -d "$file" ]] && continue

    # 隠しファイルはスキップ
    [[ "$file" == .* ]] && continue

    echo "処理中: $file"
done
```

## 4.5 select文

### 基本構文

`select` 文（select statement）は、対話的なメニューを簡単に作成できる構文です。

```bash
select 変数 in 選択肢リスト; do
    処理
done
```

```bash
#!/usr/bin/env bash
# select を使ったメニュー

echo "好きな果物を選んでください:"
select fruit in "りんご" "バナナ" "みかん" "終了"; do
    case $fruit in
        "りんご")
            echo "りんごを選びました！"
            ;;
        "バナナ")
            echo "バナナを選びました！"
            ;;
        "みかん")
            echo "みかんを選びました！"
            ;;
        "終了")
            echo "プログラムを終了します"
            break
            ;;
        *)
            echo "無効な選択です。もう一度選んでください。"
            ;;
    esac
done

# 実行時の表示:
# 好きな果物を選んでください:
# 1) りんご
# 2) バナナ
# 3) みかん
# 4) 終了
# #? 1
# りんごを選びました！
# #? 4
# プログラムを終了します
```

### PS3 でプロンプトをカスタマイズ

```bash
#!/usr/bin/env bash
# PS3 で select のプロンプトを変更

PS3="番号を入力してください: "    # デフォルトは "#? "

select option in "表示" "追加" "削除" "終了"; do
    case $REPLY in    # $REPLY には入力された番号が入る
        1) echo "表示を実行します" ;;
        2) echo "追加を実行します" ;;
        3) echo "削除を実行します" ;;
        4) echo "終了します"; break ;;
        *) echo "1〜4の番号を入力してください" ;;
    esac
done
```

### 配列を使った select

```bash
#!/usr/bin/env bash
# 配列の内容をメニューにする

services=("nginx" "mysql" "redis" "postgres")

PS3="操作するサービスを選んでください: "
select service in "${services[@]}" "全て" "終了"; do
    if [[ "$service" == "終了" ]]; then
        break
    elif [[ "$service" == "全て" ]]; then
        echo "全サービスの状態を確認します"
        for s in "${services[@]}"; do
            echo "  $s: 確認中..."
        done
    elif [[ -n "$service" ]]; then
        echo "$service の状態を確認します"
    else
        echo "無効な選択です"
    fi
done
```

## 4.6 ループとパイプ

### パイプラインでのループ

```bash
#!/usr/bin/env bash
# コマンドの出力をループで処理する

# プロセス一覧からメモリ使用量が多い順に表示
ps aux --sort=-%mem | head -6 | while IFS= read -r line; do
    echo "$line"
done

# find の結果をループで処理
find /tmp -name "*.log" -mtime +7 2>/dev/null | while IFS= read -r file; do
    echo "古いログファイル: $file"
done
```

### プロセス置換（Process Substitution）

プロセス置換（process substitution）を使うと、コマンドの出力をファイルのように扱えます。

```bash
# <(command) : コマンドの出力をファイルとして読み取る
# >(command) : ファイルへの書き込みをコマンドの入力にする

# 2つのディレクトリの内容を比較
diff <(ls /tmp) <(ls /var/tmp)

# サブシェル問題を回避してファイルを読む
count=0
while IFS= read -r line; do
    ((count++))
done < <(find /etc -name "*.conf" 2>/dev/null)
echo "設定ファイル数: $count"
```

### ヒアストリング（Here String）でのループ

```bash
# <<< でヒアストリングを入力として使う
while IFS=',' read -r name age; do
    echo "$name ($age歳)"
done <<< "太郎,25"

# 複数行にはヒアドキュメントを使う
while IFS=',' read -r name age city; do
    echo "$name ($age歳) - $city"
done << 'EOF'
太郎,25,東京
花子,30,大阪
次郎,22,名古屋
EOF
```

### xargs との組み合わせ

```bash
# find + xargs でファイルを一括処理（for ループの代替）
find . -name "*.tmp" -print0 | xargs -0 rm -f

# -I{} でプレースホルダーを使う
find . -name "*.txt" -print0 | xargs -0 -I{} echo "テキストファイル: {}"

# 並列実行（-P オプション）
find . -name "*.jpg" -print0 | xargs -0 -P 4 -I{} convert {} -resize 50% "thumb_{}"
```

## 4.7 実践例

### ファイル一括処理

```bash
#!/usr/bin/env bash
# ファイル拡張子の一括変更

target_dir="${1:-.}"    # 引数がなければカレントディレクトリ

echo "=== .jpeg → .jpg 拡張子変更 ==="

count=0
for file in "$target_dir"/*.jpeg; do
    # ファイルが存在しない場合のガード
    [[ -f "$file" ]] || continue

    new_name="${file%.jpeg}.jpg"
    echo "  $file → $new_name"
    mv "$file" "$new_name"
    ((count++))
done

echo "完了: $count ファイルを変更しました"
```

### リトライ処理

```bash
#!/usr/bin/env bash
# コマンドのリトライ処理

retry() {
    local max_attempts=${1:-3}
    local delay=${2:-5}
    shift 2
    local cmd=("$@")

    local attempt=1
    until "${cmd[@]}"; do
        if (( attempt >= max_attempts )); then
            echo "エラー: $max_attempts 回試行しましたが失敗しました" >&2
            return 1
        fi
        echo "失敗 (試行 $attempt/$max_attempts)。${delay}秒後にリトライ..."
        sleep "$delay"
        ((attempt++))
    done
    echo "成功しました (試行 $attempt/$max_attempts)"
}

# 使用例: 最大5回、3秒間隔でリトライ
retry 5 3 curl -sf http://example.com
```

### 進捗表示

```bash
#!/usr/bin/env bash
# プログレスバーの表示

progress_bar() {
    local current=$1
    local total=$2
    local width=50
    local percentage=$((current * 100 / total))
    local filled=$((current * width / total))
    local empty=$((width - filled))

    # プログレスバーの描画
    printf "\r["
    printf "%${filled}s" | tr ' ' '#'
    printf "%${empty}s" | tr ' ' '-'
    printf "] %3d%% (%d/%d)" "$percentage" "$current" "$total"
}

# 使用例
total=50
for ((i = 1; i <= total; i++)); do
    progress_bar $i $total
    sleep 0.1   # 実際の処理に置き換える
done
echo ""   # 改行
echo "完了！"

# 出力例:
# [########################--------------------------]  48% (24/50)
```

### ログファイルの監視

```bash
#!/usr/bin/env bash
# ログファイルをリアルタイムで監視し、エラーを検出する

log_file="${1:-/var/log/syslog}"

if [[ ! -f "$log_file" ]]; then
    echo "エラー: ファイルが見つかりません: $log_file" >&2
    exit 1
fi

echo "ログ監視開始: $log_file"
echo "Ctrl+C で終了"

tail -f "$log_file" | while IFS= read -r line; do
    if [[ "$line" == *"ERROR"* || "$line" == *"error"* ]]; then
        echo "[警告] $line"
    elif [[ "$line" == *"WARN"* || "$line" == *"warn"* ]]; then
        echo "[注意] $line"
    fi
done
```

### ディレクトリの再帰的処理

```bash
#!/usr/bin/env bash
# ディレクトリを再帰的に走査し、ファイル統計を表示

target_dir="${1:-.}"

file_count=0
dir_count=0
total_size=0

while IFS= read -r -d '' entry; do
    if [[ -f "$entry" ]]; then
        ((file_count++))
        size=$(stat -c %s "$entry" 2>/dev/null || echo 0)
        ((total_size += size))
    elif [[ -d "$entry" ]]; then
        ((dir_count++))
    fi
done < <(find "$target_dir" -print0 2>/dev/null)

echo "=== ディレクトリ統計: $target_dir ==="
echo "ファイル数: $file_count"
echo "ディレクトリ数: $dir_count"
echo "合計サイズ: $((total_size / 1024)) KB"
```
