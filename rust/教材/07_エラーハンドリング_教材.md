## 7.1 Rust のエラー処理の考え方

### 7.1.1 回復可能なエラーと回復不可能なエラー

Rustはエラーを2つのカテゴリに分類します。

```
┌──────────────────────────────────────────────────┐
│  Rustのエラー処理                                  │
│                                                  │
│  ┌──────────────────────────────────────────┐    │
│  │  回復不可能なエラー（unrecoverable）        │    │
│  │  → panic! マクロ                          │    │
│  │  例: 配列の範囲外アクセス、論理的なバグ      │    │
│  │  → プログラムを即座に停止させる              │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
│  ┌──────────────────────────────────────────┐    │
│  │  回復可能なエラー（recoverable）            │    │
│  │  → Result<T, E> 型                        │    │
│  │  例: ファイルが見つからない、入力の解析失敗   │    │
│  │  → 呼び出し元が適切に対処する                │    │
│  └──────────────────────────────────────────┘    │
│                                                  │
│  ※ Rustには例外（try/catch）機構がありません      │
│    すべてのエラーは型システムで明示的に扱います     │
└──────────────────────────────────────────────────┘
```

他の多くの言語では例外（exception）を使いますが、Rustでは型システムを活用してコンパイル時にエラー処理の漏れを検出します。

## 7.2 panic! による異常終了

### 7.2.1 panic! マクロの基本

`panic!` はプログラムを即座に停止させるマクロです。回復の見込みがないバグや、プログラムの前提条件が崩れた場合に使います。

```rust
fn main() {
    println!("panic!の前");
    // panic!("致命的なエラーが発生しました！");  // ← コメントを外すとここで停止
    println!("panic!の後（ここには到達する）");

    // 配列の範囲外アクセスもpanicを引き起こす
    let v = vec![1, 2, 3];
    // let x = v[10];  // ← コメントを外すとpanic!

    // 安全な方法: getを使う
    match v.get(10) {
        Some(val) => println!("値: {}", val),
        None => println!("インデックス10は範囲外です"),
    }
}
```

**実行結果：**
```
panic!の前
panic!の後（ここには到達する）
インデックス10は範囲外です
```

### 7.2.2 panicが起きる典型的なケース

```rust
fn main() {
    // 以下はすべてpanicを引き起こす操作です（コメントのまま残します）

    // 1. 配列の範囲外アクセス
    // let v = vec![1, 2, 3];
    // let _ = v[5];

    // 2. unwrapでNoneを開く
    // let x: Option<i32> = None;
    // let _ = x.unwrap();

    // 3. unwrapでErrを開く
    // let result: Result<i32, &str> = Err("error");
    // let _ = result.unwrap();

    // 4. 整数のゼロ除算
    // let _ = 10 / 0;

    // 安全な代替方法をデモ
    let v = vec![1, 2, 3];
    println!("安全なアクセス: {:?}", v.get(5));  // None

    let x: Option<i32> = None;
    println!("安全なunwrap: {}", x.unwrap_or(0));  // 0

    let result: Result<i32, &str> = Err("error");
    println!("安全なunwrap: {}", result.unwrap_or(-1));  // -1

    // ゼロ除算の回避
    let divisor = 0;
    if divisor != 0 {
        println!("結果: {}", 10 / divisor);
    } else {
        println!("ゼロ除算を回避しました");
    }
}
```

**実行結果：**
```
安全なアクセス: None
安全なunwrap: 0
安全なunwrap: -1
ゼロ除算を回避しました
```

### 7.2.3 バックトレースの読み方

panicが発生した場合、環境変数 `RUST_BACKTRACE=1` を設定するとバックトレースが表示されます。

```
# 実行方法（ターミナルで）
RUST_BACKTRACE=1 cargo run

# Windowsの場合
set RUST_BACKTRACE=1 && cargo run
```

> **現場のプロの知見：** 開発中はバックトレースを有効にしておくと、panicの原因を素早く特定できます。本番環境ではpanicを起こさない設計を心がけましょう。

### ポイントまとめ

- `panic!` は回復不可能なエラーでプログラムを停止させる
- 配列の範囲外アクセス、`unwrap` の失敗などもpanicを引き起こす
- `RUST_BACKTRACE=1` でバックトレースを確認できる
- **panicは「バグ」を示す最終手段。通常のエラー処理には `Result` を使う**

## 7.3 Result<T, E> 型

### 7.3.1 Result の定義

`Result<T, E>` は、成功（`Ok(T)`）または失敗（`Err(E)`）を表す列挙型です。

```
┌──────────────────────────────────────┐
│  enum Result<T, E> {                 │
│      Ok(T),    // 成功、値Tを持つ      │
│      Err(E),   // 失敗、エラーEを持つ   │
│  }                                   │
│                                      │
│  Result<i32, String>の例:            │
│    Ok(42)           → 成功、値は42    │
│    Err("失敗".into()) → 失敗          │
└──────────────────────────────────────┘
```

```rust
fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err(String::from("ゼロ除算エラー"))
    } else {
        Ok(a / b)
    }
}

fn main() {
    // matchで処理
    match divide(10.0, 3.0) {
        Ok(result) => println!("10 / 3 = {:.4}", result),
        Err(e) => println!("エラー: {}", e),
    }

    match divide(10.0, 0.0) {
        Ok(result) => println!("10 / 0 = {:.4}", result),
        Err(e) => println!("エラー: {}", e),
    }
}
```

**実行結果：**
```
10 / 3 = 3.3333
エラー: ゼロ除算エラー
```

### 7.3.2 標準ライブラリのResult

多くの標準ライブラリ関数が `Result` を返します。

```rust
fn main() {
    // 文字列を数値にパース
    let valid = "42".parse::<i32>();
    let invalid = "abc".parse::<i32>();

    println!("\"42\"のパース: {:?}", valid);     // Ok(42)
    println!("\"abc\"のパース: {:?}", invalid);  // Err(...)

    // matchで安全に処理
    let input = "123";
    match input.parse::<i32>() {
        Ok(n) => println!("パース成功: {}", n),
        Err(e) => println!("パースエラー: {}", e),
    }

    // 複数の入力を処理
    let inputs = ["42", "hello", "100", "3.14", "-7"];
    for input in inputs {
        match input.parse::<i32>() {
            Ok(n) => println!("  \"{}\" → {}", input, n),
            Err(_) => println!("  \"{}\" → 整数ではありません", input),
        }
    }
}
```

**実行結果：**
```
"42"のパース: Ok(42)
"abc"のパース: Err(invalid digit found in string)
パース成功: 123
  "42" → 42
  "hello" → 整数ではありません
  "100" → 100
  "3.14" → 整数ではありません
  "-7" → -7
```

### 7.3.3 ファイル操作のResult

```rust
use std::fs;

fn main() {
    // ファイルの読み取り（存在しないファイル）
    match fs::read_to_string("nonexistent.txt") {
        Ok(contents) => println!("内容: {}", contents),
        Err(e) => println!("ファイル読み取りエラー: {}", e),
    }

    // ファイルの書き込みと読み取り
    let filename = "test_output.txt";

    // 書き込み
    match fs::write(filename, "Hello, Rust!") {
        Ok(()) => println!("ファイルに書き込みました: {}", filename),
        Err(e) => println!("書き込みエラー: {}", e),
    }

    // 読み取り
    match fs::read_to_string(filename) {
        Ok(contents) => println!("読み取った内容: {}", contents),
        Err(e) => println!("読み取りエラー: {}", e),
    }

    // 後片付け
    let _ = fs::remove_file(filename);
}
```

**実行結果：**
```
ファイル読み取りエラー: No such file or directory (os error 2)
ファイルに書き込みました: test_output.txt
読み取った内容: Hello, Rust!
```

### ポイントまとめ

- `Result<T, E>` は成功（`Ok(T)`）または失敗（`Err(E)`）を表す
- 標準ライブラリの多くの関数が `Result` を返す
- `match` で `Ok` と `Err` の両方を処理する
- エラーの型 `E` にはエラーの詳細情報が含まれる

## 7.4 unwrap と expect

### 7.4.1 unwrap：成功時の値を取り出す

`unwrap()` は `Ok` の中身を取り出します。`Err` の場合はpanicします。

```rust
fn main() {
    // 成功が確実な場合のunwrap
    let n: i32 = "42".parse().unwrap();
    println!("n = {}", n);

    // Optionにも使える
    let v = vec![10, 20, 30];
    let first = v.first().unwrap();
    println!("最初の要素: {}", first);
}
```

**実行結果：**
```
n = 42
最初の要素: 10
```

### 7.4.2 expect：エラーメッセージ付きunwrap

`expect()` は `unwrap()` と同じですが、panicメッセージをカスタマイズできます。

```rust
fn main() {
    // expectでメッセージを付ける
    let port: u16 = "8080".parse()
        .expect("ポート番号のパースに失敗");
    println!("ポート: {}", port);

    let config_value = std::env::var("MY_APP_CONFIG")
        .unwrap_or_else(|_| String::from("default_value"));
    println!("設定値: {}", config_value);
}
```

**実行結果：**
```
ポート: 8080
設定値: default_value
```

### 7.4.3 unwrap系メソッドの安全な代替

```rust
fn main() {
    let result: Result<i32, &str> = Err("エラー");

    // unwrap_or: エラー時のデフォルト値
    let val = result.unwrap_or(0);
    println!("unwrap_or: {}", val);

    // unwrap_or_else: エラー時にクロージャで値を生成
    let val = result.unwrap_or_else(|e| {
        println!("エラーが発生: {}", e);
        -1
    });
    println!("unwrap_or_else: {}", val);

    // unwrap_or_default: Default トレイトのデフォルト値を使用
    let val: i32 = result.unwrap_or_default();  // i32のデフォルトは0
    println!("unwrap_or_default: {}", val);

    // ok(): Result → Option に変換（Errを捨てる）
    let opt: Option<i32> = result.ok();
    println!("ok(): {:?}", opt);

    // 成功の場合
    let success: Result<i32, &str> = Ok(42);
    println!("成功 unwrap_or: {}", success.unwrap_or(0));
    println!("成功 ok(): {:?}", success.ok());
}
```

**実行結果：**
```
unwrap_or: 0
エラーが発生: エラー
unwrap_or_else: -1
unwrap_or_default: 0
ok(): None
成功 unwrap_or: 42
成功 ok(): Some(42)
```

> **よくある間違い：** 本番コードで `unwrap()` を多用するのは危険です。入力データやI/O操作の結果には `unwrap()` を使わず、適切なエラー処理を行いましょう。

### 7.4.4 unwrapが許される場面

```rust
fn main() {
    // 1. リテラルのパース（絶対に成功する）
    let pi: f64 = "3.14159".parse().unwrap();
    println!("π ≈ {}", pi);

    // 2. テストコード
    // #[test]
    // fn test_something() {
    //     let result = do_something().unwrap(); // テストならOK
    //     assert_eq!(result, expected);
    // }

    // 3. プロトタイプ・ラフスケッチ（後で適切なエラー処理に置き換え）
    // let data = fs::read_to_string("config.txt").unwrap(); // TODO: エラー処理

    // 4. expectでコンテキストを残す方が良い
    let port: u16 = "8080".parse()
        .expect("環境変数PORTに不正な値が設定されている");
    println!("ポート: {}", port);
}
```

**実行結果：**
```
π ≈ 3.14159
ポート: 8080
```

### ポイントまとめ

| メソッド | 動作 | 用途 |
|---|---|---|
| `unwrap()` | Ok→値、Err→panic | テスト、確実に成功する場合 |
| `expect("msg")` | Ok→値、Err→メッセージ付きpanic | unwrapの改良版 |
| `unwrap_or(default)` | Ok→値、Err→デフォルト | 安全なデフォルト値がある場合 |
| `unwrap_or_else(f)` | Ok→値、Err→クロージャ | デフォルト値の生成にコストがある場合 |
| `unwrap_or_default()` | Ok→値、Err→Default | Default実装がある型 |

## 7.5 ?演算子（エラー伝播）

### 7.5.1 ?演算子の基本

`?` 演算子は、`Result` が `Err` の場合に現在の関数から即座にそのエラーを返すシンタックスシュガーです。

```
┌──────────────────────────────────────────────┐
│  ?演算子の動作                                 │
│                                              │
│  let value = some_function()?;               │
│                                              │
│  ↓ 以下と同等                                │
│                                              │
│  let value = match some_function() {         │
│      Ok(v)  => v,                            │
│      Err(e) => return Err(e.into()),         │
│  };                                          │
│                                              │
│  ※ 自動で From::from() による型変換が行われる  │
└──────────────────────────────────────────────┘
```

```rust
use std::fs;

// ?演算子を使ったエラー伝播
fn read_number_from_file(path: &str) -> Result<i32, String> {
    let content = fs::read_to_string(path)
        .map_err(|e| format!("ファイル読み取りエラー: {}", e))?;

    let number = content.trim().parse::<i32>()
        .map_err(|e| format!("数値パースエラー: {}", e))?;

    Ok(number)
}

fn main() {
    // テスト用ファイルを作成
    let _ = fs::write("test_number.txt", "42");

    match read_number_from_file("test_number.txt") {
        Ok(n) => println!("読み取った数値: {}", n),
        Err(e) => println!("エラー: {}", e),
    }

    match read_number_from_file("nonexistent.txt") {
        Ok(n) => println!("読み取った数値: {}", n),
        Err(e) => println!("エラー: {}", e),
    }

    // 後片付け
    let _ = fs::remove_file("test_number.txt");
}
```

**実行結果：**
```
読み取った数値: 42
エラー: ファイル読み取りエラー: No such file or directory (os error 2)
```

### 7.5.2 ?演算子のチェーン

`?` 演算子を連続して使うことで、複数のエラーが発生しうる処理を簡潔に書けます。

```rust
fn parse_config(input: &str) -> Result<(String, u16, String), String> {
    // "host:port:protocol" 形式の文字列をパース
    let parts: Vec<&str> = input.split(':').collect();

    if parts.len() != 3 {
        return Err(format!("無効な形式: '{}' (host:port:protocolが必要)", input));
    }

    let host = parts[0].to_string();

    let port: u16 = parts[1].parse()
        .map_err(|e| format!("ポート番号が無効: '{}' ({})", parts[1], e))?;

    let protocol = parts[2].to_string();
    if protocol != "http" && protocol != "https" {
        return Err(format!("未対応のプロトコル: '{}'", protocol));
    }

    Ok((host, port, protocol))
}

fn main() {
    let test_cases = [
        "localhost:8080:http",
        "example.com:443:https",
        "bad:port:http",
        "missing_parts",
        "host:80:ftp",
    ];

    for input in test_cases {
        match parse_config(input) {
            Ok((host, port, proto)) => {
                println!("OK: {}://{}:{}", proto, host, port);
            }
            Err(e) => {
                println!("NG: {}", e);
            }
        }
    }
}
```

**実行結果：**
```
OK: http://localhost:8080
OK: https://example.com:443
NG: ポート番号が無効: 'port' (invalid digit found in string)
NG: 無効な形式: 'missing_parts' (host:port:protocolが必要)
NG: 未対応のプロトコル: 'ftp'
```

### 7.5.3 Option に対する?演算子

`?` 演算子は `Option` にも使えます。`None` の場合に即座に `None` を返します。

```rust
fn get_middle_name(full_name: &str) -> Option<&str> {
    let parts: Vec<&str> = full_name.split_whitespace().collect();
    // 3つの部分がある場合のみミドルネームを返す
    if parts.len() == 3 {
        Some(parts[1])
    } else {
        None
    }
}

fn get_middle_initial(full_name: &str) -> Option<char> {
    let middle = get_middle_name(full_name)?;  // Noneなら即return None
    middle.chars().next()  // 最初の文字を返す（Option<char>）
}

fn main() {
    let names = [
        "John William Smith",
        "田中太郎",
        "Mary Jane Watson",
    ];

    for name in names {
        match get_middle_initial(name) {
            Some(initial) => println!("{} → ミドルイニシャル: {}", name, initial),
            None => println!("{} → ミドルネームなし", name),
        }
    }
}
```

**実行結果：**
```
John William Smith → ミドルイニシャル: W
田中太郎 → ミドルネームなし
Mary Jane Watson → ミドルイニシャル: J
```

### ポイントまとめ

- `?` 演算子は `Err`（または `None`）を即座に返すシンタックスシュガー
- `match` によるエラー処理を大幅に簡潔化できる
- 戻り値が `Result` または `Option` の関数内でのみ使用可能
- `map_err` と組み合わせてエラーメッセージをカスタマイズ

## 7.6 カスタムエラー型

### 7.6.1 enum でエラー型を定義

アプリケーション固有のエラーを enum で定義するのが一般的です。

```rust
use std::fmt;
use std::num::ParseIntError;

// カスタムエラー型
#[derive(Debug)]
enum AppError {
    NotFound(String),
    InvalidInput(String),
    ParseError(ParseIntError),
    OutOfRange { value: i32, min: i32, max: i32 },
}

// Displayトレイトの実装（エラーメッセージ）
impl fmt::Display for AppError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            AppError::NotFound(name) =>
                write!(f, "見つかりません: {}", name),
            AppError::InvalidInput(msg) =>
                write!(f, "無効な入力: {}", msg),
            AppError::ParseError(e) =>
                write!(f, "数値変換エラー: {}", e),
            AppError::OutOfRange { value, min, max } =>
                write!(f, "範囲外: {} (有効範囲: {}~{})", value, min, max),
        }
    }
}

// 使用例
fn find_user(id: &str) -> Result<String, AppError> {
    // IDを数値にパース
    let id_num: i32 = id.parse()
        .map_err(AppError::ParseError)?;

    // 範囲チェック
    if id_num < 1 || id_num > 1000 {
        return Err(AppError::OutOfRange {
            value: id_num,
            min: 1,
            max: 1000,
        });
    }

    // ユーザー検索
    match id_num {
        1 => Ok(String::from("田中太郎")),
        2 => Ok(String::from("佐藤花子")),
        _ => Err(AppError::NotFound(format!("ID={}", id_num))),
    }
}

fn main() {
    let test_ids = ["1", "2", "abc", "5", "9999"];

    for id in test_ids {
        match find_user(id) {
            Ok(name) => println!("ID '{}' → {}", id, name),
            Err(e) => println!("ID '{}' → エラー: {}", id, e),
        }
    }
}
```

**実行結果：**
```
ID '1' → 田中太郎
ID '2' → 佐藤花子
ID 'abc' → エラー: 数値変換エラー: invalid digit found in string
ID '5' → エラー: 見つかりません: ID=5
ID '9999' → エラー: 範囲外: 9999 (有効範囲: 1~1000)
```

### 6.6.2 From トレイトによる自動変換

`From` トレイトを実装すると、`?` 演算子で自動的にエラー型が変換されます。

```rust
use std::fmt;
use std::num::ParseIntError;
use std::io;

#[derive(Debug)]
enum ConfigError {
    Io(io::Error),
    Parse(ParseIntError),
    Validation(String),
}

impl fmt::Display for ConfigError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            ConfigError::Io(e) => write!(f, "I/Oエラー: {}", e),
            ConfigError::Parse(e) => write!(f, "パースエラー: {}", e),
            ConfigError::Validation(msg) => write!(f, "検証エラー: {}", msg),
        }
    }
}

// From<io::Error> を実装 → io::Errorを?で自動変換
impl From<io::Error> for ConfigError {
    fn from(e: io::Error) -> ConfigError {
        ConfigError::Io(e)
    }
}

// From<ParseIntError> を実装 → ParseIntErrorを?で自動変換
impl From<ParseIntError> for ConfigError {
    fn from(e: ParseIntError) -> ConfigError {
        ConfigError::Parse(e)
    }
}

fn load_port_from_file(path: &str) -> Result<u16, ConfigError> {
    // io::Errorが?演算子でConfigError::Ioに自動変換される
    let content = std::fs::read_to_string(path)?;

    // ParseIntErrorが?演算子でConfigError::Parseに自動変換される
    let port: u16 = content.trim().parse()?;

    // バリデーション
    if port < 1024 {
        return Err(ConfigError::Validation(
            format!("ポート{}は予約済み（1024以上を指定）", port)
        ));
    }

    Ok(port)
}

fn main() {
    // テスト用ファイルを作成
    std::fs::write("port.txt", "8080").ok();

    match load_port_from_file("port.txt") {
        Ok(port) => println!("ポート: {}", port),
        Err(e) => println!("エラー: {}", e),
    }

    match load_port_from_file("nonexistent.txt") {
        Ok(port) => println!("ポート: {}", port),
        Err(e) => println!("エラー: {}", e),
    }

    // 不正な内容のファイル
    std::fs::write("port_bad.txt", "abc").ok();
    match load_port_from_file("port_bad.txt") {
        Ok(port) => println!("ポート: {}", port),
        Err(e) => println!("エラー: {}", e),
    }

    // 予約ポート
    std::fs::write("port_reserved.txt", "80").ok();
    match load_port_from_file("port_reserved.txt") {
        Ok(port) => println!("ポート: {}", port),
        Err(e) => println!("エラー: {}", e),
    }

    // 後片付け
    let _ = std::fs::remove_file("port.txt");
    let _ = std::fs::remove_file("port_bad.txt");
    let _ = std::fs::remove_file("port_reserved.txt");
}
```

**実行結果：**
```

エラー: I/Oエラー: No such file or directory (os error 2)
エラー: パースエラー: invalid digit found in string
エラー: 検証エラー: ポート80は予約済み（1024以上を指定）
```

- `enum` でアプリケーション固有のエラー型を定義

## 7.7 実践的なエラーハンドリングパターン

### 7.7.1 map と and_then による Result のチェーン

```rust
fn main() {
    // map: Ok の中身を変換
    let result: Result<i32, String> = Ok(5);
    let doubled = result.map(|n| n * 2);
    println!("map: {:?}", doubled);  // Ok(10)

    // and_then: Ok の中身で別の Result を返す
    let result = parse_and_validate("42");
    println!("42: {:?}", result);

    let result = parse_and_validate("abc");
    println!("abc: {:?}", result);

    let result = parse_and_validate("-5");
    println!("-5: {:?}", result);

    // チェーンの実例
    println!("\n--- 計算チェーン ---");
    let inputs = ["10", "abc", "0", "5"];
    for input in inputs {
        let result = "100".parse::<f64>()
            .and_then(|a| input.parse::<f64>().map(|b| (a, b)))
            .map(|(a, b)| if b != 0.0 { Ok(a / b) } else { Err("ゼロ除算") });

        match result {
            Ok(Ok(v)) => println!("  100 / {} = {:.2}", input, v),
            Ok(Err(e)) => println!("  100 / {} → {}", input, e),
            Err(e) => println!("  100 / {} → パースエラー: {}", input, e),
        }
    }
}

fn parse_and_validate(input: &str) -> Result<i32, String> {
    input.parse::<i32>()
        .map_err(|e| format!("パースエラー: {}", e))
        .and_then(|n| {
            if n >= 0 {
                Ok(n)
            } else {
                Err(format!("負の値は無効: {}", n))
            }
        })
}
```

**実行結果：**
```
map: Ok(10)
42: Ok(42)
abc: Err("パースエラー: invalid digit found in string")
-5: Err("負の値は無効: -5")

--- 計算チェーン ---
  100 / 10 = 10.00
  100 / abc → パースエラー: invalid digit found in string
  100 / 0 → ゼロ除算
  100 / 5 = 20.00
```

### 7.7.2 collect で Result のベクタを処理

```rust
fn main() {
    // 全部成功する場合
    let inputs = vec!["1", "2", "3", "4", "5"];
    let result: Result<Vec<i32>, _> = inputs.iter()
        .map(|s| s.parse::<i32>())
        .collect();
    println!("全成功: {:?}", result);

    // 途中で失敗する場合（最初のエラーで停止）
    let inputs = vec!["1", "2", "abc", "4", "5"];
    let result: Result<Vec<i32>, _> = inputs.iter()
        .map(|s| s.parse::<i32>())
        .collect();
    println!("途中失敗: {:?}", result);

    // エラーを無視して成功したものだけ集める
    let inputs = vec!["1", "abc", "3", "def", "5"];
    let successes: Vec<i32> = inputs.iter()
        .filter_map(|s| s.parse::<i32>().ok())
        .collect();
    println!("成功のみ: {:?}", successes);

    // 成功とエラーを分離する
    let inputs = vec!["1", "abc", "3", "def", "5"];
    let (successes, errors): (Vec<_>, Vec<_>) = inputs.iter()
        .map(|s| s.parse::<i32>())
        .partition(|r| r.is_ok());
    let successes: Vec<i32> = successes.into_iter().map(|r| r.unwrap()).collect();
    let errors: Vec<String> = errors.into_iter()
        .map(|r| r.unwrap_err().to_string())
        .collect();
    println!("成功: {:?}", successes);
    println!("エラー: {:?}", errors);
}
```

**実行結果：**
```
全成功: Ok([1, 2, 3, 4, 5])
途中失敗: Err(invalid digit found in string)
成功のみ: [1, 3, 5]
成功: [1, 3, 5]
エラー: ["invalid digit found in string", "invalid digit found in string"]
```

### 7.7.3 エラーのコンテキスト追加

```rust
use std::fmt;
use std::num::ParseIntError;

#[derive(Debug)]
struct ContextError {
    context: String,
    source: String,
}

impl fmt::Display for ContextError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "{}: {}", self.context, self.source)
    }
}

// エラーにコンテキストを追加するヘルパー
trait WithContext<T> {
    fn with_context(self, ctx: &str) -> Result<T, ContextError>;
}

impl<T, E: fmt::Display> WithContext<T> for Result<T, E> {
    fn with_context(self, ctx: &str) -> Result<T, ContextError> {
        self.map_err(|e| ContextError {
            context: ctx.to_string(),
            source: e.to_string(),
        })
    }
}

fn parse_config_line(line: &str) -> Result<(String, i32), ContextError> {
    let parts: Vec<&str> = line.split('=').collect();
    if parts.len() != 2 {
        return Err(ContextError {
            context: format!("行 '{}' のパース", line),
            source: "key=value 形式ではありません".to_string(),
        });
    }

    let key = parts[0].trim().to_string();
    let value: i32 = parts[1].trim().parse()
        .with_context(&format!("キー '{}' の値パース", key))?;

    Ok((key, value))
}

fn main() {
    let config_lines = [
        "port = 8080",
        "max_connections = 100",
        "timeout = abc",
        "invalid_line",
    ];

    for line in config_lines {
        match parse_config_line(line) {
            Ok((key, value)) => println!("OK: {} = {}", key, value),
            Err(e) => println!("NG: {}", e),
        }
    }
}
```

**実行結果：**
```
OK: port = 8080
OK: max_connections = 100
NG: キー 'timeout' の値パース: invalid digit found in string
NG: 行 'invalid_line' のパース: key=value 形式ではありません
```

### ポイントまとめ

- `map` と `and_then` で Result をチェーンできる
- `collect()` で `Vec<Result>` を `Result<Vec>` に変換できる
- `filter_map(|x| x.ok())` でエラーを無視して成功だけ集められる
- エラーにコンテキスト情報を追加すると、問題の特定が容易になる

## 7.8 panic vs Result の使い分け

### 7.8.1 判断基準

```
┌──────────────────────────────────────────────────────┐
│  panic を使う場面                                      │
│  ┌────────────────────────────────────────────────┐  │
│  │ ・プログラムの論理的なバグ（到達不能なはずの場所）   │  │
│  │ ・不変条件（invariant）が破られた場合               │  │
│  │ ・テストコードでのアサーション                      │  │
│  │ ・プロトタイプ（後でResultに置き換える）             │  │
│  │ ・mainで最終的にエラーを処理する場所               │  │
│  └────────────────────────────────────────────────┘  │
│                                                      │
│  Result を使う場面                                     │
│  ┌────────────────────────────────────────────────┐  │
│  │ ・ファイルI/O、ネットワーク通信                     │  │
│  │ ・ユーザー入力のパース・バリデーション               │  │
│  │ ・外部システムとの連携                              │  │
│  │ ・ライブラリのAPI（呼び出し元に判断を委ねる）        │  │
│  │ ・回復可能なあらゆるエラー                          │  │
│  └────────────────────────────────────────────────┘  │
│                                                      │
│  迷ったら Result を使う！                               │
└──────────────────────────────────────────────────────┘
```

### 7.8.2 具体例で比較

```rust
// panicが適切な例：内部不変条件の検証
fn get_grade(score: u32) -> &'static str {
    // この関数の前提条件: scoreは0~100
    // 呼び出し元が正しい値を渡す責任がある
    match score {
        90..=100 => "A",
        80..=89 => "B",
        70..=79 => "C",
        60..=69 => "D",
        0..=59 => "F",
        _ => panic!("スコアは0~100の範囲で指定してください: {}", score),
    }
}

// Resultが適切な例：外部入力の処理
fn parse_score(input: &str) -> Result<u32, String> {
    let score: u32 = input.parse()
        .map_err(|_| format!("'{}' は数値ではありません", input))?;

    if score > 100 {
        return Err(format!("スコア {} は範囲外です (0~100)", score));
    }

    Ok(score)
}

fn main() {
    // panicの例：プログラマが正しい値を渡す
    println!("85点: {}", get_grade(85));
    println!("92点: {}", get_grade(92));

    // Resultの例：ユーザー入力を処理
    let inputs = ["85", "abc", "150", "72"];
    for input in inputs {
        match parse_score(input) {
            Ok(score) => println!("入力 '{}' → {}点 ({})", input, score, get_grade(score)),
            Err(e) => println!("入力 '{}' → エラー: {}", input, e),
        }
    }
}
```

**実行結果：**
```
85点: B
92点: A
入力 '85' → 85点 (B)
入力 'abc' → エラー: 'abc' は数値ではありません
入力 '150' → エラー: スコア 150 は範囲外です (0~100)
入力 '72' → 72点 (C)
```

### 7.8.3 main関数でのエラー処理

`main` 関数も `Result` を返せます。

```rust
use std::num::ParseIntError;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let input = "42";
    let number: i32 = input.parse()?;
    println!("パース成功: {}", number);

    // Box<dyn std::error::Error> は任意のエラー型を受け入れる
    let sum: i32 = ["1", "2", "3"].iter()
        .map(|s| s.parse::<i32>())
        .collect::<Result<Vec<_>, _>>()?
        .iter()
        .sum();
    println!("合計: {}", sum);

    Ok(())
}
```

**実行結果：**
```
パース成功: 42
合計: 6
```

> **現場のプロの知見：** `Box<dyn std::error::Error>` は「任意のエラー型」を受け入れる便利な型です。小規模なツールやmain関数では十分ですが、ライブラリではカスタムエラー型を定義する方が利用者にとって親切です。

### ポイントまとめ

- **迷ったら `Result` を使う**（最も安全な選択）
- `panic!` はバグ検出用。正常な実行パスではなく「ここに来たらバグ」という場所で使う
- ライブラリは `Result` で、アプリケーションの最上位で最終判断する
- `main` 関数は `Result<(), Box<dyn std::error::Error>>` を返せる

## 7.9 実践例：CSVパーサー

学んだ知識を組み合わせた実践例です。

```rust
use std::fmt;
use std::num::{ParseFloatError, ParseIntError};

// カスタムエラー型
#[derive(Debug)]
enum CsvError {
    InvalidRowFormat { line: usize, expected: usize, got: usize },
    ParseInt { line: usize, column: &'static str, source: ParseIntError },
    ParseFloat { line: usize, column: &'static str, source: ParseFloatError },
    EmptyInput,
}

impl fmt::Display for CsvError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            CsvError::InvalidRowFormat { line, expected, got } =>
                write!(f, "{}行目: {}列期待ですが{}列です", line, expected, got),
            CsvError::ParseInt { line, column, source } =>
                write!(f, "{}行目の'{}'列: 整数変換エラー ({})", line, column, source),
            CsvError::ParseFloat { line, column, source } =>
                write!(f, "{}行目の'{}'列: 小数変換エラー ({})", line, column, source),
            CsvError::EmptyInput =>
                write!(f, "入力が空です"),
        }
    }
}

// 社員データ
#[derive(Debug)]
struct Employee {
    id: u32,
    name: String,
    department: String,
    salary: f64,
}

impl fmt::Display for Employee {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "ID:{:>4} | {:8} | {:10} | ¥{:>12.0}",
            self.id, self.name, self.department, self.salary)
    }
}

fn parse_employee(line: &str, line_num: usize) -> Result<Employee, CsvError> {
    let fields: Vec<&str> = line.split(',').map(|s| s.trim()).collect();

    if fields.len() != 4 {
        return Err(CsvError::InvalidRowFormat {
            line: line_num,
            expected: 4,
            got: fields.len(),
        });
    }

    let id: u32 = fields[0].parse().map_err(|e| CsvError::ParseInt {
        line: line_num,
        column: "id",
        source: e,
    })?;

    let name = fields[1].to_string();
    let department = fields[2].to_string();

    let salary: f64 = fields[3].parse().map_err(|e| CsvError::ParseFloat {
        line: line_num,
        column: "salary",
        source: e,
    })?;

    Ok(Employee { id, name, department, salary })
}

fn parse_csv(input: &str) -> Result<Vec<Employee>, Vec<CsvError>> {
    if input.trim().is_empty() {
        return Err(vec![CsvError::EmptyInput]);
    }

    let mut employees = Vec::new();
    let mut errors = Vec::new();

    for (i, line) in input.lines().enumerate() {
        let line = line.trim();
        if line.is_empty() || line.starts_with('#') {
            continue;  // 空行とコメント行をスキップ
        }

        match parse_employee(line, i + 1) {
            Ok(emp) => employees.push(emp),
            Err(e) => errors.push(e),
        }
    }

    if errors.is_empty() {
        Ok(employees)
    } else {
        // エラーがあっても成功分は表示したいので、先に表示
        if !employees.is_empty() {
            println!("--- 正常に読み込めたデータ ---");
            for emp in &employees {
                println!("  {}", emp);
            }
            println!();
        }
        Err(errors)
    }
}

fn main() {
    let csv_data = "\
# 社員データ
1, 田中太郎, 開発部, 5000000
2, 佐藤花子, 営業部, 4500000
3, 鈴木一郎, 人事部, abc
4, 高橋, 総務部
5, 山本次郎, 開発部, 5500000";

    println!("=== CSV パース結果 ===\n");

    match parse_csv(csv_data) {
        Ok(employees) => {
            println!("全{}件の読み込みに成功しました\n", employees.len());
            for emp in &employees {
                println!("  {}", emp);
            }

            // 統計情報
            let total_salary: f64 = employees.iter().map(|e| e.salary).sum();
            let avg_salary = total_salary / employees.len() as f64;
            println!("\n  平均年収: ¥{:.0}", avg_salary);
        }
        Err(errors) => {
            println!("--- エラー一覧 ({}) ---", errors.len());
            for err in &errors {
                println!("  [ERROR] {}", err);
            }
        }
    }
}
```

**実行結果：**
```
=== CSV パース結果 ===

--- 正常に読み込めたデータ ---
  ID:   1 | 田中太郎   | 開発部       | ¥     5000000
  ID:   2 | 佐藤花子   | 営業部       | ¥     4500000
  ID:   5 | 山本次郎   | 開発部       | ¥     5500000

--- エラー一覧 (2) ---
  [ERROR] 4行目の'salary'列: 小数変換エラー (invalid float literal)
  [ERROR] 5行目: 4列期待ですが2列です
```
