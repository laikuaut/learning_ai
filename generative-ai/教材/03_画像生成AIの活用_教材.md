## 3.1 画像生成AIの基本

### 拡散モデル（Diffusion Model）とは

現在主流の画像生成AIは、**拡散モデル（Diffusion Model）** という技術に基づいています。
この仕組みを直感的に理解しましょう。

```
【拡散モデルの基本的な仕組み】

＜学習フェーズ（Forward Process）＞
  きれいな画像 → ノイズを少し追加 → さらに追加 → ... → 完全なノイズ
  [犬の写真]   → [少しぼやけた犬] → [もっとぼやけた] → [砂嵐のような画像]

＜生成フェーズ（Reverse Process）＞
  完全なノイズ → ノイズを少し除去 → さらに除去 → ... → きれいな画像
  [砂嵐]       → [何かの輪郭]     → [犬っぽい形] → [犬の写真]
```

学習時には画像にノイズを段階的に加え、生成時にはその逆のプロセス（ノイズ除去）を行います。テキストプロンプトは、このノイズ除去の「方向性」をガイドする役割を果たします。

### テキストから画像（Text-to-Image）の流れ

テキストから画像を生成する際、内部では以下のプロセスが進行します。

```
【Text-to-Image の処理フロー】

  テキスト入力              エンコード             ノイズ除去            画像出力
  ┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────┐
  │ "夕焼けの │───>│テキスト      │───>│ 拡散モデル   │───>│ 生成     │
  │  海辺"    │    │エンコーダー  │    │（U-Net等）   │    │ 画像     │
  └──────────┘    └──────────────┘    └──────────────┘    └──────────┘
                         │                    ↑
                         │    ┌──────────┐    │
                         └───>│条件付け  │────┘
                              │(Conditioning)│
                              └──────────┘
```

1. **テキストエンコーダー** がプロンプトを数値ベクトルに変換します
2. **拡散モデル** がランダムノイズから段階的にノイズを除去します
3. テキストの情報が「条件」として与えられ、プロンプトに沿った画像が生成されます

### 画像生成AIでできること

| 機能 | 説明 | 例 |
|------|------|-----|
| Text-to-Image | テキストから画像を生成 | 「夕焼けの海辺」→ 画像 |
| Image-to-Image | 既存画像を別の画像に変換 | 写真 → イラスト風に変換 |
| Inpainting | 画像の一部を修正・置換 | 背景だけを変更 |
| Outpainting | 画像の範囲を拡張 | 写真の外側を生成で補完 |
| Style Transfer | 画風を変換 | 写真 → 水彩画風 |
| Upscaling | 低解像度画像を高解像度に | 512px → 2048px |

> **ポイントまとめ**
> - 拡散モデルは「ノイズ除去」のプロセスで画像を生成する
> - テキストプロンプトがノイズ除去の方向をガイドする
> - Text-to-Image以外にも多様な機能がある

## 3.2 主要サービスの比較

### 主要な画像生成AIサービス

2026年時点で広く使われている主要サービスを比較します。

| サービス | 提供元 | 特徴 | 料金体系 | 商用利用 |
|---------|--------|------|---------|---------|
| **Stable Diffusion** | Stability AI | オープンソース、ローカル実行可 | 無料（自前実行）/ API有料 | モデルによる |
| **DALL-E 3** | OpenAI | ChatGPT統合、自然言語理解が強い | ChatGPT Plus に含まれる | 生成画像は利用者に帰属 |
| **Midjourney** | Midjourney | アート表現が高品質、Discord連携 | 月額$10〜 | 有料プランで可 |
| **Adobe Firefly** | Adobe | Creative Cloud統合、商用安全 | Adobe契約に含まれる | 商用利用を前提に設計 |
| **Imagen** | Google | 高品質、Gemini統合 | Google One AI Premium等 | プランによる |

### 各サービスの得意分野

```
【サービス別の得意分野マップ】

                芸術的
                  ↑
                  │
    Midjourney ●  │
                  │    ● Stable Diffusion
  写実的 ←────────┼────────→ イラスト的
                  │
       DALL-E 3 ● │  ● Adobe Firefly
                  │
       Imagen ●   │
                  ↓
                実用的
```

### 選択の指針

- **個人の創作・アート表現** → Midjourney（高品質なアート表現）
- **ビジネス資料・商用利用** → Adobe Firefly（著作権リスクが低い）
- **チャットでの手軽な生成** → DALL-E 3（ChatGPTから直接利用）
- **細かいカスタマイズ** → Stable Diffusion（パラメータを自由に調整）
- **Google環境との統合** → Imagen（Gemini・Workspace連携）

> **ポイントまとめ**
> - 用途に応じてサービスを使い分けることが重要
> - 商用利用する場合は、各サービスの利用規約を必ず確認する
> - Stable Diffusionはオープンソースのため、最もカスタマイズ性が高い

## 3.3 効果的なプロンプトの書き方

### 画像生成プロンプトの基本構造

画像生成AIのプロンプトは、テキスト生成AIとは異なる書き方が求められます。**キーワードの組み合わせ** が基本です。

```
【画像生成プロンプトの構造】

  [主題] + [スタイル] + [構図] + [照明] + [品質指定] + [追加詳細]

  例：
  "A Japanese garden in autumn, watercolor painting style,
   wide angle shot, golden hour lighting, highly detailed,
   koi pond, red maple leaves"
```

### プロンプトの各要素

| 要素 | 説明 | 例 |
|------|------|-----|
| **主題（Subject）** | 何を描くか | a cat, a cityscape, a portrait |
| **スタイル（Style）** | 画風・表現方法 | oil painting, anime style, photorealistic |
| **構図（Composition）** | カメラアングル・配置 | close-up, bird's eye view, centered |
| **照明（Lighting）** | 光の当て方 | golden hour, studio lighting, neon lights |
| **品質（Quality）** | 画像品質の指定 | highly detailed, 4K, masterpiece |
| **雰囲気（Mood）** | 作品の雰囲気 | peaceful, dramatic, mysterious |

### スタイル指定の具体例

```
【よく使われるスタイル指定】

写実系：
  photorealistic, hyperrealistic, DSLR photo, cinematic

イラスト系：
  digital art, anime style, cartoon, cel shading

絵画系：
  oil painting, watercolor, impressionist, ukiyo-e

デザイン系：
  flat design, minimalist, isometric, vector art

特殊効果：
  cyberpunk, steampunk, vaporwave, double exposure
```

### ネガティブプロンプト（Negative Prompt）

生成画像に **含めたくない要素** を指定するのがネガティブプロンプトです。主にStable Diffusionで使用されます。

```
【ネガティブプロンプトの例】

プロンプト：
  "a portrait of a young woman, natural lighting, soft focus"

ネガティブプロンプト：
  "blurry, low quality, distorted face, extra fingers,
   deformed hands, watermark, text, logo"
```

よく使われるネガティブプロンプト：

| カテゴリ | キーワード |
|---------|-----------|
| 品質低下防止 | blurry, low quality, pixelated, jpeg artifacts |
| 人物の崩れ防止 | deformed, extra fingers, mutated hands, bad anatomy |
| 不要要素の排除 | watermark, text, logo, signature, frame |
| スタイル制御 | cartoon（写実的にしたい場合）, photorealistic（イラストにしたい場合） |

### よくある間違い

- **プロンプトが長すぎる** → 重要なキーワードが埋もれます。核となる要素を前方に配置しましょう
- **曖昧な形容詞の多用** → 「beautiful」「nice」などは効果が薄いです。具体的なスタイルや技法名を使いましょう
- **矛盾する指示** → 「photorealistic anime style」のような矛盾は意図しない結果になります
- **日本語プロンプトへの過信** → 多くのモデルは英語で学習されています。英語プロンプトの方が精度が高い傾向があります

> **ポイントまとめ**
> - 主題・スタイル・構図・照明・品質をバランスよく指定する
> - 重要なキーワードはプロンプトの前方に置く
> - ネガティブプロンプトで不要要素を排除する
> - 英語プロンプトの方が多くのモデルで精度が高い

## 3.4 実践的な活用シーン

### プレゼン資料での活用

ビジネスプレゼンテーションでの画像素材作成に活用できます。

```
【プレゼン用画像のプロンプト例】

スライド背景：
  "Abstract blue gradient background with subtle geometric
   patterns, corporate style, clean, minimal, 16:9 ratio"

コンセプトイメージ：
  "Business team collaborating around a holographic display,
   futuristic office, soft blue lighting, professional,
   corporate photography style"

アイコン・図解素材：
  "Flat design icon set for business presentation,
   growth chart, target, teamwork, innovation,
   consistent blue color scheme, white background"
```

### SNS用画像の作成

ソーシャルメディア投稿用のビジュアル作成も得意分野です。

| プラットフォーム | 推奨サイズ | 活用例 |
|----------------|-----------|--------|
| Instagram投稿 | 1080x1080 | 商品イメージ、ブランドビジュアル |
| X（Twitter）ヘッダー | 1500x500 | プロフィール用バナー |
| YouTubeサムネイル | 1280x720 | 動画サムネイル素材 |
| ブログアイキャッチ | 1200x630 | 記事トップ画像 |

### プロトタイプ・モックアップ

Webサイトやアプリのデザイン初期段階で、イメージ共有に活用できます。

```
【プロトタイプ活用の流れ】

  1. コンセプトキーワードの整理
     └─ 「ナチュラル」「清潔感」「20代女性向け」

  2. ムードボード用画像の生成
     └─ 複数のスタイルバリエーションを素早く生成

  3. クライアントとの方向性確認
     └─ 生成画像でイメージを共有、認識のずれを早期に解消

  4. 最終デザインの制作
     └─ 方向性が決まったらプロのデザイナーが仕上げる
```

### ロゴデザインの初期案作成

ロゴのアイデア出しにも活用できますが、注意点があります。

```
プロンプト例：
  "Minimalist logo design for a tech startup called 'AuraFlow',
   clean lines, gradient from blue to purple, modern,
   scalable vector style, white background"
```

> **注意：** 画像生成AIで作成したロゴをそのまま最終版として使用することは推奨しません。あくまでアイデアの出発点として活用し、プロのデザイナーがベクターデータとして仕上げるのがベストプラクティスです。

> **ポイントまとめ**
> - プレゼン資料では背景やコンセプトイメージの作成に活用できる
> - SNS用画像はプラットフォームに合わせたサイズ指定が重要
> - プロトタイプ段階でのイメージ共有ツールとして有効
> - ロゴ等のブランド素材は最終版をそのまま使わず、プロが仕上げる

## 3.5 画像編集・加工への活用

### インペインティング（Inpainting）

画像の**一部分だけを修正・置換**する技術です。マスク（修正範囲）を指定し、その部分だけを再生成します。

```
【インペインティングの仕組み】

  ┌────────────────┐     ┌────────────────┐     ┌────────────────┐
  │                │     │                │     │                │
  │   元の画像     │     │  マスク指定    │     │   修正後画像   │
  │                │     │                │     │                │
  │  [人物と背景]  │ ──> │  [背景部分を   │ ──> │  [人物はそのまま│
  │                │     │   黒く塗る]    │     │   背景が変化]  │
  │                │     │                │     │                │
  └────────────────┘     └────────────────┘     └────────────────┘
```

活用例：
- 写真の背景を別のシーンに差し替える
- 不要なオブジェクト（電線、ゴミ箱など）を自然に除去する
- 服装や小物の色・デザインを変更する

### アウトペインティング（Outpainting）

画像の**外側を拡張**する技術です。元の画像の境界を超えた部分をAIが補完生成します。

```
【アウトペインティングの例】

  元の画像（正方形）        拡張後（横長）
  ┌──────────┐        ┌──────────────────────┐
  │          │        │///│          │///│
  │  人物の  │   ──>  │生成│  人物の  │生成│
  │  ポートレ│        │部分│  ポートレ│部分│
  │  ート    │        │///│  ート    │///│
  └──────────┘        └──────────────────────┘
```

活用例：
- 正方形の写真を16:9のバナーサイズに拡張する
- 被写体の周囲の風景を追加する
- トリミングで失われた部分を復元する

### スタイル変換（Style Transfer）

写真やイラストの**画風を別のスタイルに変換**する技術です。

| 変換元 | 変換先 | 用途例 |
|--------|--------|--------|
| 写真 | 水彩画風 | アート作品制作 |
| 写真 | アニメ風 | SNSアイコン作成 |
| ラフスケッチ | 完成イラスト | デザイン作業の効率化 |
| 写真 | 線画 | 塗り絵素材の作成 |
| 昼の写真 | 夜景風 | 時間帯の変更シミュレーション |

### よくある間違い

- **インペインティングのマスク範囲が狭すぎる** → 修正対象の周辺も少し含めると自然な結果になります
- **アウトペインティングで大幅に拡張しすぎる** → 段階的に少しずつ拡張する方が品質が安定します
- **元画像と生成部分のスタイル不一致** → プロンプトで元画像のスタイルを明確に指定しましょう

> **ポイントまとめ**
> - インペインティングは部分修正、アウトペインティングは範囲拡張
> - マスク指定は修正対象より少し広めに取るのがコツ
> - スタイル変換で写真を様々な画風に変換できる
> - いずれも元画像の品質が結果に大きく影響する

## 3.6 著作権と利用規約

### 生成画像の著作権

画像生成AIで作成した画像の著作権は、国や地域、状況によって解釈が異なります。

```
【生成画像の著作権に関する主な論点】

  ┌─────────────────────────────────────────────┐
  │          生成画像の著作権は誰のもの？         │
  ├─────────────────────────────────────────────┤
  │                                             │
  │  ● AI自体は著作権の主体になれない           │
  │    （多くの国で共通の見解）                  │
  │                                             │
  │  ● 人間の創作的関与が十分にあれば            │
  │    著作権が認められる可能性がある            │
  │                                             │
  │  ● 単純なプロンプト入力だけでは              │
  │    著作権が認められない場合がある            │
  │                                             │
  │  ● 法整備は各国で進行中（2026年時点）       │
  │                                             │
  └─────────────────────────────────────────────┘
```

### 各サービスの商用利用ポリシー

| サービス | 商用利用 | 主な条件 |
|---------|---------|---------|
| Stable Diffusion | モデルライセンスに依存 | オープンモデルは比較的自由 |
| DALL-E 3 | 可（有料プラン） | OpenAIの利用規約に準拠 |
| Midjourney | 可（有料プラン） | 年間収益$1M超の企業はProプラン必須 |
| Adobe Firefly | 可 | 著作権補償（IP Indemnity）あり |
| Imagen | プランによる | Googleの利用規約に準拠 |

### 学習データの問題

画像生成AIの学習データに関しては、以下の議論が続いています。

- **アーティストの権利**：学習データに含まれる既存作品の著作者への補償
- **オプトアウト（Opt-out）**：自分の作品を学習データから除外する権利
- **スタイルの模倣**：特定アーティストのスタイルを再現することの是非
- **データの透明性**：どの作品が学習に使われたかの開示

### 安全に活用するためのガイドライン

```
【商用利用時のチェックリスト】

  □ 利用するサービスの利用規約を確認したか
  □ 商用利用が許可されたプランを使用しているか
  □ 特定の個人・キャラクターに酷似していないか
  □ 既存のブランドロゴ・商標を侵害していないか
  □ 生成画像であることを適切に開示しているか（必要な場合）
  □ 社内のAI利用ガイドラインに準拠しているか
```

### よくある間違い

- **「AIが生成したから著作権フリー」と思い込む** → 生成画像にも様々な権利の問題が存在します
- **他人の作品に酷似した画像を商用利用する** → 著作権侵害のリスクがあります
- **利用規約を読まずにサービスを使う** → サービスごとに条件が異なるため、必ず確認しましょう
- **特定のアーティスト名をプロンプトに含める** → 権利上の問題になる可能性があります

> **ポイントまとめ**
> - AI生成画像の著作権は法整備が進行中であり、状況は変化し得る
> - 商用利用する場合はサービスの利用規約を必ず確認する
> - Adobe Fireflyのように著作権補償を提供するサービスもある
> - 特定の個人やキャラクターに酷似した画像の利用は避ける
> - 社内ガイドラインの整備も重要
