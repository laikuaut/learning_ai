> **学習目標**: バグだらけのC言語プログラムを読み解き、コンパイルエラー・実行時エラー・論理エラーをすべて特定して修正する力を養う

---

## 課題の説明

以下の「住所録プログラム」には **10個のバグ** が埋め込まれています。
プログラムをコンパイル・実行し、エラーメッセージや意図しない動作を手がかりにすべてのバグを見つけて修正してください。

### 進め方

1. 下のバグ入りコードを `address_book_buggy.c` として保存する
2. `gcc -o address_book address_book_buggy.c` でコンパイルを試みる
3. コンパイルエラーを修正する
4. 実行して動作を確認し、残りのバグを修正する
5. 全機能が正常に動作すれば完了

## バグ入りコード

以下のコードをそのまま `address_book_buggy.c` として保存してください。

```c
/* 住所録プログラム（バグ入り版） */
/* このコードには10個のバグが含まれています。すべて修正してください。 */

#include <stdio.h>
#include <string.h>

#define MAX_ENTRIES 100
#define NAME_LEN 50
#define PHONE_LEN 20
#define ADDR_LEN 100

typedef struct {
    char name[NAME_LEN];
    char phone[PHONE_LEN];
    char address[ADDR_LEN];
} Entry;

Entry entries[MAX_ENTRIES];
int count = 0;

/* エントリを追加する */
void add_entry(void) {
    if (count > MAX_ENTRIES) {  /* バグ1 */
        printf("住所録がいっぱいです。\n");
        return;
    }

    printf("名前: ");
    fgets(entries[count].name, NAME_LEN, stdin);

    printf("電話番号: ");
    fgets(entries[count].phone, PHONE_LEN, stdin);

    printf("住所: ");
    fgets(entries[count].address, ADDR_LEN, stdin);

    /* 改行を除去 */
    entries[count].name[strcspn(entries[count].name, "\n")] = '\0';
    entries[count].phone[strcspn(entries[count].phone, "\n")] = '\0';
    entries[count].address[strcspn(entries[count].address "\n")] = '\0';  /* バグ2 */

    count++;
    printf("追加しました。（現在 %d 件）\n", count);
}

/* 一覧表示 */
void show_all(void) {
    int i;
    if (count == 0) {
        printf("データがありません。\n");
        return;
    }

    printf("\n--- 住所録一覧 ---\n");
    for (i = 0; i <= count; i++) {  /* バグ3 */
        printf("%d. [%s] %s / %s\n",
               i + 1,
               entries[i].name,
               entries[i].phone,
               entries[i].address);
    }
}

/* 名前で検索 */
void search(void) {
    char keyword[NAME_LEN];
    int i;
    int found = 0;

    printf("検索キーワード: ");
    fgets(keyword, NAME_LEN, stdin);
    keyword[strcspn(keyword, "\n")] = '\0';

    for (i = 0; i < count; i++) {
        if (strcmp(entries[i].name, keyword) == 0) {  /* バグ4 */
            printf("  %s / %s / %s\n",
                   entries[i].name,
                   entries[i].phone,
                   entries[i].address);
            found++;
        }
    }

    if (!found) {
        printf("見つかりませんでした。\n");
    }
}

/* エントリを削除する */
void delete_entry(void) {
    int num;
    int i;

    printf("削除する番号: ");
    scanf("%d", &num);
    while (getchar() != '\n');

    if (num < 1 || num > count) {
        printf("無効な番号です。\n");
        return;
    }

    /* 削除：後ろの要素を前に詰める */
    for (i = num - 1; i < count; i++) {
        entries[i] = entries[i + 1];  /* バグ5 */
    }

    count--;
    printf("削除しました。（残り %d 件）\n", count);
}

/* ファイルに保存 */
void save_file(void) {
    FILE *fp;
    int i;

    fp = fopen("address_book.dat", "w");
    if (fp == NULL) {
        printf("ファイルを開けませんでした。\n");
        return;
    }

    fprintf(fp, "%d\n", count);
    for (i = 0; i < count; i++) {
        fprintf(fp, "%s\n%s\n%s\n",
                entries[i].name,
                entries[i].phone,
                entries[i].address);
    }

    /* バグ6: fclose が抜けている */
    printf("保存しました。\n");
}

/* ファイルから読み込み */
void load_file(void) {
    FILE *fp;
    int i, n;

    fp = fopen("address_book.dat", "r");
    if (fp == NULL) {
        printf("ファイルが見つかりません。\n");
        return;
    }

    fscanf(fp, "%d\n", &n);  /* バグ7: \n を含めると問題が起きるケースがある */

    for (i = 0; i < n; i++) {
        fgets(entries[i].name, NAME_LEN, fp);
        fgets(entries[i].phone, PHONE_LEN, fp);
        fgets(entries[i].address, ADDR_LEN, fp);
        entries[i].name[strcspn(entries[i].name, "\n")] = '\0';
        entries[i].phone[strcspn(entries[i].phone, "\n")] = '\0';
        entries[i].address[strcspn(entries[i].address, "\n")] = '\0';
    }

    count = n;
    fclose(fp);
    printf("%d 件読み込みました。\n", count);
}

/* メイン */
int main(void) {
    int choice;
    char line[16];  /* バグ8: 後で使うが宣言が不適切な位置にあるわけではない */

    printf("===== 住所録 =====\n");

    while (1) {
        printf("\n1.追加 2.一覧 3.検索 4.削除 5.保存 6.読込 0.終了\n> ");
        fgets(line, sizeof(line), stdin);
        choice = line[0] - '0';  /* バグ9: 文字→数値変換 */

        switch (choice) {
            case 1: add_entry(); break;
            case 2: show_all(); break;
            case 3: search(); break;
            case 4: delete_entry(); break;
            case 5: save_file(); break;
            case 6: load_file(); break;
            case 0:
                printf("終了します。\n");
                return 0;
            default:
                printf("無効な選択です。\n");
                break;
        }
        /* バグ10: delete_entry 内の scanf の後に getchar で改行を消費しているが、
           fgets ベースのメインループと混在してバッファに不整合が起きる可能性 */
    }

    return 0;
}
```

## バグの種類（ヒントレベル別）

<details>
<summary>ヒント（軽め）：バグの種類だけ知りたい</summary>

10個のバグは以下のカテゴリに分類されます。

| カテゴリ | 件数 |
|---|:---:|
| コンパイルエラー（構文エラー） | 1 |
| 境界条件エラー（off-by-one） | 2 |
| 論理エラー（意図と実装の不一致） | 2 |
| リソースリーク | 1 |
| バッファ関連の問題 | 2 |
| 入出力の混在問題 | 2 |

</details>

<details>
<summary>ヒント（詳細）：各バグの場所と簡単な説明</summary>

| # | 場所 | ヒント |
|---|---|---|
| 1 | `add_entry` の上限チェック | `>` と `>=` の違いは？ |
| 2 | `add_entry` の `strcspn` 呼び出し | 関数呼び出しの引数区切りが… |
| 3 | `show_all` のループ条件 | `<=` だと配列の範囲外にアクセスしてしまう |
| 4 | `search` の文字列比較 | 完全一致でなく部分一致にすべきでは？ |
| 5 | `delete_entry` の詰め処理 | ループの終了条件が `count` だと最後に範囲外アクセス |
| 6 | `save_file` | ファイルを開いたら必ず閉じること |
| 7 | `load_file` の `fscanf` | `%d` の後に `fgetc` で改行を消費すべき |
| 8 | メインの入力方式 | `fgets` で読んだ行の変換方式自体は問題ないが、`atoi` の方が安全 |
| 9 | 文字→数値変換 | `'0'` 以外の文字（改行のみなど）が来た場合の処理がない |
| 10 | `delete_entry` の `scanf` | `scanf` と `fgets` の混在でバッファ不整合が起きる |

</details>

## 解答例

<details>
<summary>全10個のバグと修正方法</summary>

### バグ1：境界条件 ─ 上限チェックが甘い

```c
/* 修正前 */
if (count > MAX_ENTRIES) {
/* 修正後 */
if (count >= MAX_ENTRIES) {
```

`count` が `MAX_ENTRIES` と等しいとき、配列の範囲外 `entries[MAX_ENTRIES]` にアクセスしてしまいます。

### バグ2：構文エラー ─ カンマの欠落

```c
/* 修正前 */
entries[count].address[strcspn(entries[count].address "\n")] = '\0';
/* 修正後 */
entries[count].address[strcspn(entries[count].address, "\n")] = '\0';
```

`strcspn` の第1引数と第2引数の間のカンマが抜けています。

### バグ3：境界条件 ─ ループが1回多い

```c
/* 修正前 */
for (i = 0; i <= count; i++) {
/* 修正後 */
for (i = 0; i < count; i++) {
```

`<=` だと `entries[count]` にアクセスし、未初期化のデータを読むことになります。

### バグ4：論理エラー ─ 部分一致にすべき

```c
/* 修正前 */
if (strcmp(entries[i].name, keyword) == 0) {
/* 修正後 */
if (strstr(entries[i].name, keyword) != NULL) {
```

`strcmp` は完全一致のみです。部分一致検索には `strstr` を使います。

### バグ5：境界外アクセス ─ 削除時のループ条件

```c
/* 修正前 */
for (i = num - 1; i < count; i++) {
    entries[i] = entries[i + 1];
}
/* 修正後 */
for (i = num - 1; i < count - 1; i++) {
    entries[i] = entries[i + 1];
}
```

`i < count` だと `entries[count]` にアクセスしてしまいます。

### バグ6：リソースリーク ─ fclose の欠落

```c
/* 修正前 */
    /* fclose が抜けている */
    printf("保存しました。\n");
/* 修正後 */
    fclose(fp);
    printf("保存しました。\n");
```

`fopen` したファイルは必ず `fclose` で閉じます。閉じないとデータが書き込まれない場合があります。

### バグ7：入出力 ─ fscanf と fgets の混在

```c
/* 修正前 */
fscanf(fp, "%d\n", &n);
/* 修正後 */
fscanf(fp, "%d", &n);
fgetc(fp);  /* 改行を消費 */
```

`fscanf` の `%d\n` は空白文字をすべて読み飛ばすため、次の `fgets` と干渉します。

### バグ8・9：入力の堅牢性 ─ 文字変換の安全性

```c
/* 修正前 */
choice = line[0] - '0';
/* 修正後（より安全なバージョン） */
if (line[0] >= '0' && line[0] <= '9') {
    choice = line[0] - '0';
} else {
    choice = -1;  /* 無効な入力 */
}
```

改行のみの入力や英字入力で意図しない値になることを防ぎます。

### バグ10：入出力混在 ─ scanf と fgets の混在

`delete_entry` 内で `scanf` を使い、`while (getchar() != '\n');` で改行を消費していますが、これはメインの `fgets` ベースの入力と整合が取りにくくなります。`fgets` + `sscanf` に統一するのが安全です。

```c

    int num;
    printf("削除する番号: ");
    scanf("%d", &num);

/* 修正後 */

    int num;
    char buf[16];
    printf("削除する番号: ");
    fgets(buf, sizeof(buf), stdin);
```
