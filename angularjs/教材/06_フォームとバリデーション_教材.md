# 第6章：フォームとバリデーション

## この章のゴール

- AngularJS のフォーム管理機能を理解する
- フォームの状態プロパティ（`$pristine`、`$dirty`、`$valid` など）を使いこなす
- 組み込みバリデータでフォーム入力を検証できる
- CSSクラスを使ったビジュアルフィードバックを実装できる
- カスタムバリデータを作成できる
- 完全なフォーム送信処理を実装できる

---

## 6.1 AngularJS のフォーム管理

AngularJS は `<form>` 要素と `ng-model` を組み合わせると、**フォームの状態を自動的に追跡**します。

### フォームの命名と参照

```html
<!-- name 属性を付けると、フォームオブジェクトとしてスコープに登録される -->
<form name="myForm">
  <input type="text" name="userName" ng-model="user.name" required>
</form>

<!-- フォーム全体の状態 -->
{{ myForm.$valid }}      <!-- true/false -->
{{ myForm.$invalid }}

<!-- 個別フィールドの状態 -->
{{ myForm.userName.$valid }}
{{ myForm.userName.$error }}
```

### フォームの状態プロパティ

| プロパティ | 説明 |
|-----------|------|
| `$pristine` | まだ一度も変更されていない |
| `$dirty` | 一度でも変更された |
| `$valid` | すべてのバリデーションをパス |
| `$invalid` | 一つ以上のバリデーションに失敗 |
| `$touched` | フォーカスが当たって外れた |
| `$untouched` | まだフォーカスが当たっていない |
| `$error` | エラー情報のオブジェクト |
| `$submitted` | フォームが送信された |

```
【フォームの状態遷移】

初期状態: $pristine, $untouched
       │
       │ ユーザーが入力欄をクリック
       ▼
       $untouched → $touched（フォーカスが外れた時）
       │
       │ ユーザーが文字を入力
       ▼
       $pristine → $dirty
       │
       │ バリデーション実行
       ▼
       $valid or $invalid
```

## 6.2 組み込みバリデータ

### 主なバリデータ

| バリデータ | 使い方 | 説明 |
|-----------|--------|------|
| `required` | `<input required>` | 必須入力 |
| `ng-minlength` | `ng-minlength="3"` | 最小文字数 |
| `ng-maxlength` | `ng-maxlength="20"` | 最大文字数 |
| `ng-pattern` | `ng-pattern="/^[0-9]+$/"` | 正規表現パターン |
| `type="email"` | `<input type="email">` | メール形式 |
| `type="number"` | `<input type="number">` | 数値 |
| `type="url"` | `<input type="url">` | URL形式 |
| `min` | `min="0"` | 最小値（数値） |
| `max` | `max="100"` | 最大値（数値） |

### バリデーションの基本例

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
  <style>
    .form-group { margin: 12px 0; }
    label { display: block; font-weight: bold; margin-bottom: 4px; }
    input, select { padding: 8px; border: 2px solid #ddd; border-radius: 4px; width: 300px; font-size: 14px; }
    input.ng-invalid.ng-touched { border-color: #e74c3c; }
    input.ng-valid.ng-touched { border-color: #42b883; }
    .error { color: #e74c3c; font-size: 13px; margin-top: 4px; }
    .btn { padding: 10px 24px; background: #42b883; color: white; border: none; border-radius: 4px; font-size: 15px; cursor: pointer; }
    .btn:disabled { background: #ccc; cursor: not-allowed; }
  </style>
</head>
<body ng-controller="FormCtrl">
  <h2>ユーザー登録</h2>
  
  <form name="regForm" ng-submit="submit()" novalidate>
    <!-- 名前（必須、2〜20文字） -->
    <div class="form-group">
      <label>名前</label>
      <input type="text" name="userName" ng-model="user.name"
             required ng-minlength="2" ng-maxlength="20">
      <div class="error" ng-show="regForm.userName.$touched && regForm.userName.$invalid">
        <p ng-show="regForm.userName.$error.required">名前は必須です</p>
        <p ng-show="regForm.userName.$error.minlength">2文字以上で入力してください</p>
        <p ng-show="regForm.userName.$error.maxlength">20文字以内で入力してください</p>
      </div>
    </div>
    
    <!-- メール（必須、メール形式） -->
    <div class="form-group">
      <label>メールアドレス</label>
      <input type="email" name="userEmail" ng-model="user.email" required>
      <div class="error" ng-show="regForm.userEmail.$touched && regForm.userEmail.$invalid">
        <p ng-show="regForm.userEmail.$error.required">メールアドレスは必須です</p>
        <p ng-show="regForm.userEmail.$error.email">正しいメールアドレスを入力してください</p>
      </div>
    </div>
    
    <!-- 年齢（数値、18〜120） -->
    <div class="form-group">
      <label>年齢</label>
      <input type="number" name="userAge" ng-model="user.age"
             required min="18" max="120">
      <div class="error" ng-show="regForm.userAge.$touched && regForm.userAge.$invalid">
        <p ng-show="regForm.userAge.$error.required">年齢は必須です</p>
        <p ng-show="regForm.userAge.$error.min">18歳以上で入力してください</p>
        <p ng-show="regForm.userAge.$error.max">120以下で入力してください</p>
      </div>
    </div>
    
    <!-- 送信ボタン（フォームが有効な場合のみ） -->
    <button class="btn" type="submit" ng-disabled="regForm.$invalid">
      登録する
    </button>
  </form>
  
  <!-- デバッグ情報 -->
  <div style="margin-top: 20px; background: #f5f5f5; padding: 10px; border-radius: 4px;">
    <h4>フォーム状態</h4>
    <p>$valid: {{ regForm.$valid }} | $dirty: {{ regForm.$dirty }} | $submitted: {{ regForm.$submitted }}</p>
  </div>

  <script>
    angular.module('myApp', [])
      .controller('FormCtrl', function($scope) {
        $scope.user = {};
        $scope.submit = function() {
          if ($scope.regForm.$valid) {
            alert('登録成功: ' + JSON.stringify($scope.user));
          }
        };
      });
  </script>
</body>
</html>
```

## 6.3 CSS クラスによるスタイリング

AngularJS はフォーム要素に**自動的にCSSクラスを付与**します。

### 自動付与されるCSSクラス

| クラス | 意味 |
|--------|------|
| `ng-pristine` | 未変更 |
| `ng-dirty` | 変更済み |
| `ng-valid` | バリデーションOK |
| `ng-invalid` | バリデーションNG |
| `ng-touched` | フォーカスが外れた |
| `ng-untouched` | まだフォーカスされていない |
| `ng-valid-required` | required をパス |
| `ng-invalid-required` | required に失敗 |

### よく使うCSSパターン

```css
/* 触った後で無効な場合に赤枠 */
input.ng-invalid.ng-touched {
  border-color: #e74c3c;
  background-color: #ffeaea;
}

/* 触った後で有効な場合に緑枠 */
input.ng-valid.ng-touched {
  border-color: #42b883;
}

/* 触っていない場合はデフォルト */
input.ng-pristine {
  border-color: #ddd;
}
```

> **よくある間違い：** `ng-invalid` だけでスタイルを適用すると、フォーム表示直後から赤くなってしまいます。必ず `ng-touched` や `ng-dirty` と組み合わせましょう。

## 6.4 ng-pattern — 正規表現バリデーション

```html
<!-- 電話番号（ハイフン付き） -->
<input type="text" name="phone" ng-model="user.phone"
       ng-pattern="/^0\d{1,4}-\d{1,4}-\d{4}$/"
       placeholder="03-1234-5678">

<!-- 郵便番号 -->
<input type="text" name="zip" ng-model="user.zip"
       ng-pattern="/^\d{3}-?\d{4}$/"
       placeholder="123-4567">

<!-- 半角英数字のみ -->
<input type="text" name="userId" ng-model="user.id"
       ng-pattern="/^[a-zA-Z0-9]+$/"
       placeholder="英数字のみ">
```

## 6.5 カスタムバリデータ

組み込みバリデータで対応できない場合、ディレクティブとしてカスタムバリデータを作成します。

### パスワード一致チェックの例

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
  <style>
    .form-group { margin: 12px 0; }
    label { display: block; font-weight: bold; margin-bottom: 4px; }
    input { padding: 8px; border: 2px solid #ddd; border-radius: 4px; width: 300px; }
    input.ng-invalid.ng-dirty { border-color: #e74c3c; }
    input.ng-valid.ng-dirty { border-color: #42b883; }
    .error { color: #e74c3c; font-size: 13px; }
    .btn { padding: 10px 24px; background: #42b883; color: white; border: none; border-radius: 4px; cursor: pointer; }
    .btn:disabled { background: #ccc; }
  </style>
</head>
<body ng-controller="PasswordCtrl">
  <h2>パスワード設定</h2>
  
  <form name="pwForm" novalidate>
    <div class="form-group">
      <label>パスワード</label>
      <input type="password" name="password" ng-model="pw.password"
             required ng-minlength="8">
      <div class="error" ng-show="pwForm.password.$dirty && pwForm.password.$invalid">
        <p ng-show="pwForm.password.$error.required">パスワードは必須です</p>
        <p ng-show="pwForm.password.$error.minlength">8文字以上で入力してください</p>
      </div>
    </div>
    
    <div class="form-group">
      <label>パスワード（確認）</label>
      <input type="password" name="confirm" ng-model="pw.confirm"
             required password-match="pw.password">
      <div class="error" ng-show="pwForm.confirm.$dirty && pwForm.confirm.$invalid">
        <p ng-show="pwForm.confirm.$error.required">確認用パスワードは必須です</p>
        <p ng-show="pwForm.confirm.$error.passwordMatch">パスワードが一致しません</p>
      </div>
    </div>
    
    <button class="btn" ng-disabled="pwForm.$invalid">設定する</button>
  </form>

  <script>
    angular.module('myApp', [])
      .controller('PasswordCtrl', function($scope) {
        $scope.pw = {};
      })
      // カスタムバリデータディレクティブ
      .directive('passwordMatch', function() {
        return {
          require: 'ngModel',
          scope: {
            passwordMatch: '='
          },
          link: function(scope, elem, attrs, ngModel) {
            ngModel.$validators.passwordMatch = function(modelValue) {
              return modelValue === scope.passwordMatch;
            };
            
            // 比較対象が変わったら再バリデーション
            scope.$watch('passwordMatch', function() {
              ngModel.$validate();
            });
          }
        };
      });
  </script>
</body>
</html>
```

### カスタムバリデータの構造

```javascript
.directive('myValidator', function() {
  return {
    require: 'ngModel',  // ngModelController を取得
    link: function(scope, elem, attrs, ngModel) {
      // $validators にバリデーション関数を追加
      ngModel.$validators.myValidator = function(modelValue, viewValue) {
        // true を返せば valid、false を返せば invalid
        return /* バリデーション条件 */;
      };
    }
  };
})
```

## 6.6 フォーム送信の処理

### ng-submit と novalidate

```html
<!-- novalidate: ブラウザのデフォルトバリデーションを無効化 -->
<!-- ng-submit: AngularJS でフォーム送信を処理 -->
<form name="myForm" ng-submit="onSubmit()" novalidate>
  ...
  <button type="submit">送信</button>
</form>
```

### 送信時にバリデーションメッセージを表示

```javascript
$scope.onSubmit = function() {
  // $submitted フラグを使って未入力フィールドのエラーも表示
  if ($scope.myForm.$invalid) {
    return;  // 送信しない
  }
  // 有効な場合の処理
};
```

```html
<!-- $submitted と組み合わせたエラー表示 -->
<div class="error" ng-show="(myForm.field.$touched || myForm.$submitted) && myForm.field.$invalid">
  エラーメッセージ
</div>
```

### ポイントまとめ

- `novalidate` でブラウザのバリデーションを無効化し、AngularJS に任せる
- `ng-submit` でフォーム送信を処理する
- `$submitted` フラグで送信時に全エラーを表示できる
- `ng-disabled="formName.$invalid"` で無効なフォームの送信ボタンを無効化
