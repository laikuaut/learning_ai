## 8.1 エラーとは？

Pythonのエラーは大きく2種類あります。

### 構文エラー（SyntaxError）

プログラムの書き方が間違っている場合に発生します。実行前に検出されます。

```python
# 構文エラーの例
# print("Hello"   # ← 閉じ括弧がない
# if True          # ← コロンがない
```

### 例外（Exception）

プログラムの実行中に発生するエラーです。

```python
# ZeroDivisionError: 0で割り算
# print(10 / 0)

# TypeError: 型の不一致
# print("年齢: " + 20)

# ValueError: 値が不正
# int("abc")

# FileNotFoundError: ファイルがない
# open("存在しないファイル.txt")

# IndexError: インデックス範囲外
# lst = [1, 2, 3]
# print(lst[10])

# KeyError: 辞書にキーがない
# d = {"a": 1}
# print(d["b"])
```

## 8.2 try / except の基本

```python
try:
    # エラーが起きるかもしれない処理
    num = int(input("数値を入力: "))
    result = 100 / num
    print(f"結果: {result}")
except ValueError:
    print("数値を入力してください")
except ZeroDivisionError:
    print("0で割ることはできません")
```

### 複数の例外をまとめてキャッチ

```python
try:
    value = int("abc")
except (ValueError, TypeError) as e:
    print(f"エラーが発生: {e}")
```

### 全ての例外をキャッチ（注意して使う）

```python
try:
    result = 10 / 0
except Exception as e:
    print(f"予期しないエラー: {type(e).__name__}: {e}")
```

> **注意**: `except:` や `except Exception:` で全てをキャッチするのは、デバッグを困難にするため推奨されません。可能な限り具体的な例外を指定しましょう。

## 8.3 else と finally

```python
try:
    num = int(input("数値を入力: "))
    result = 100 / num
except ValueError:
    print("数値を入力してください")
except ZeroDivisionError:
    print("0で割ることはできません")
else:
    # エラーが発生しなかった場合のみ実行
    print(f"計算成功！結果: {result}")
finally:
    # エラーの有無に関わらず必ず実行
    print("処理を終了します")
```

### 処理の流れ

```
try → 成功 → else → finally
try → 失敗 → except → finally
```

### finally の活用例

```python
f = None
try:
    f = open("data.txt", "r", encoding="utf-8")
    content = f.read()
except FileNotFoundError:
    print("ファイルが見つかりません")
finally:
    if f:
        f.close()  # エラーが起きてもファイルを閉じる
    print("ファイル処理終了")

# ↑ ただし、with文を使えばfinallyは不要
```

## 8.4 raise で例外を発生させる

自分のプログラムで意図的にエラーを発生させることができます。

```python
def set_age(age):
    if age < 0:
        raise ValueError("年齢は0以上でなければなりません")
    if age > 150:
        raise ValueError("年齢が大きすぎます")
    return age

try:
    age = set_age(-5)
except ValueError as e:
    print(f"エラー: {e}")  # エラー: 年齢は0以上でなければなりません
```

### 例外の再送出

```python
try:
    result = 10 / 0
except ZeroDivisionError:
    print("エラーをログに記録しました")
    raise  # 同じ例外を再度発生させる
```

## 8.5 カスタム例外クラス

自分のプログラム専用のエラーを定義できます。

```python
class InsufficientFundsError(Exception):
    """残高不足エラー"""
    def __init__(self, balance, amount):
        self.balance = balance
        self.amount = amount
        super().__init__(
            f"残高不足: 残高{balance}円に対して{amount}円の引き出し"
        )


class InvalidAccountError(Exception):
    """不正な口座番号エラー"""
    pass


def withdraw(balance, amount):
    if amount <= 0:
        raise ValueError("引き出し額は正の数でなければなりません")
    if amount > balance:
        raise InsufficientFundsError(balance, amount)
    return balance - amount


# 使用例
try:
    new_balance = withdraw(1000, 5000)
except InsufficientFundsError as e:
    print(f"エラー: {e}")
    print(f"  残高: {e.balance}円")
    print(f"  要求額: {e.amount}円")
except ValueError as e:
    print(f"入力エラー: {e}")
```

## 8.6 よくあるエラーと対処法

### TypeError

```python
# 原因: 型の不一致
# "年齢: " + 20  → "年齢: " + str(20)

# 原因: 呼び出し可能でないオブジェクトを呼んだ
# x = 5
# x()  → xは関数ではない
```

### ValueError

```python
# 原因: 不正な値
try:
    num = int("abc")
except ValueError:
    print("数値に変換できません")
```

### KeyError

```python
# 原因: 辞書にキーがない
data = {"name": "太郎"}

# 対策1: get() を使う
print(data.get("age", "不明"))  # 不明

# 対策2: in で確認
if "age" in data:
    print(data["age"])
```

### IndexError

```python
# 原因: リストの範囲外
items = [1, 2, 3]

# 対策: 範囲チェック
index = 5
if 0 <= index < len(items):
    print(items[index])
else:
    print("範囲外です")
```

### FileNotFoundError

```python
from pathlib import Path

# 対策: 存在確認
filepath = Path("data.txt")
if filepath.exists():
    content = filepath.read_text(encoding="utf-8")
else:
    print("ファイルが見つかりません")
```

### AttributeError

```python
# 原因: 存在しない属性やメソッド
text = None
# text.upper()  → NoneType には upper がない

# 対策: None チェック
if text is not None:
    print(text.upper())
```

## 8.7 実践的なエラー処理パターン

### 入力値の検証ループ

```python
def get_positive_integer(prompt):
    """正の整数を入力させる（エラーなら再入力）"""
    while True:
        try:
            value = int(input(prompt))
            if value <= 0:
                print("正の整数を入力してください")
                continue
            return value
        except ValueError:
            print("数値を入力してください")

age = get_positive_integer("年齢を入力: ")
print(f"入力された年齢: {age}")
```

### ファイル操作のエラー処理

```python
import json

def load_data(filepath):
    """JSONファイルを安全に読み込む"""
    try:
        with open(filepath, "r", encoding="utf-8") as f:
            return json.load(f)
    except FileNotFoundError:
        print(f"ファイルが見つかりません: {filepath}")
        return {}
    except json.JSONDecodeError as e:
        print(f"JSONの形式が不正です: {e}")
        return {}
    except PermissionError:
        print(f"ファイルへのアクセス権がありません: {filepath}")
        return {}
```

### LBYL vs EAFP

Pythonでは「許可を求めるより許しを請う」（EAFP）スタイルが推奨されます。

```python
# LBYL (Look Before You Leap) - 事前チェック
if "key" in dictionary:
    value = dictionary["key"]

# EAFP (Easier to Ask Forgiveness than Permission) - 例外処理（Python推奨）
try:
    value = dictionary["key"]
except KeyError:
    value = "デフォルト値"
```

## まとめ

| 構文 | 説明 |
|------|------|
| `try / except` | エラーをキャッチして処理する |
| `except 型 as e` | 特定の例外をキャッチし変数に格納 |
| `else` | エラーが起きなかったときの処理 |
| `finally` | 必ず実行される処理 |
| `raise` | 例外を意図的に発生させる |
| `class MyError(Exception)` | カスタム例外を定義する |
