# 実践課題08：デバッグ演習 ─ 崩れたレイアウト ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第1章〜第5章（基本・ボックスモデル・Flexbox・Grid・レスポンシブ）
> **課題の種類**: デバッグ
> **学習目標**: CSSのバグを発見・修正する力を養う。ボックスモデルの計算ミス、Flexbox/Gridの誤用、レスポンシブ対応の不備を特定できるようになる

---

## 課題の説明

以下のHTMLファイルは「会社のランディングページ」ですが、**10個のCSSバグ** が含まれています。
ブラウザで表示すると、レイアウトが大きく崩れます。

### 進め方

1. 下のバグ入りコードをHTMLファイルとして保存する
2. ブラウザで開き、崩れている箇所を確認する
3. 開発者ツール（DevTools）のインスペクタを活用して原因を特定する
4. 1つずつバグを修正し、ブラウザをリロードして確認する
5. 全10個のバグを見つけたら完了

### ヒント：よくあるCSSバグのパターン

- `box-sizing` を忘れて幅が溢れる
- Flexbox の軸方向を間違える
- `position` の基準要素が設定されていない
- `z-index` が効かない
- メディアクエリの条件が逆
- 単位の付け忘れ（`0` 以外の値に単位がない）
- `display` の値が間違っている
- セレクタのスペル間違い

## バグ一覧と修正方法

<details>
<summary>バグ1〜5の解答</summary>

### バグ1: `box-sizing` が設定されていない

**問題:** `padding` や `border` を加えると要素が想定より大きくなり、はみ出す。

**修正:**
```css
/* 先頭に追加 */
*, *::before, *::after {
  box-sizing: border-box;
}
```

### バグ2: `flex-direction: column` が間違い

**問題:** ヘッダーのロゴとナビリンクが縦並びになっている。横に並べたい。

**修正:**
```css
.header {
  flex-direction: row;  /* column → row に変更（またはこの行を削除） */
}
```

### バグ3: `margin` と `padding` のリセット忘れ

**問題:** `<ul>` にはブラウザデフォルトの `margin` と `padding` があるため、ナビリンクの位置がずれる。

**修正:**
```css
.nav-links {
  margin: 0;   /* 追加 */
  padding: 0;  /* 追加 */
}
```

### バグ4: 高さの単位がない

**問題:** `height: 400` は無効なCSS（`0` 以外の数値には単位が必要）。ヒーローセクションの高さが効いていない。

**修正:**
```css
.hero {
  height: 400px;  /* 400 → 400px */
}
```

### バグ5: `repeat()` の括弧が閉じていない

**問題:** `repeat(3, 1fr;` → 閉じ括弧 `)` がないため、Grid全体が無効になる。

**修正:**
```css
.feature-grid {
  grid-template-columns: repeat(3, 1fr);  /* 閉じ括弧を追加 */
}
```

</details>

<details>
<summary>バグ6〜10の解答</summary>

### バグ6: `width: 100%` + `padding` で溢れる

**問題:** `box-sizing: border-box` を設定していない場合（バグ1）、`width: 100%` に `padding: 32px` が加算されて溢れる。バグ1を修正すれば解消するが、そもそも Grid の子要素に `width: 100%` は不要。

**修正:**
```css
.feature-card {
  /* width: 100%; を削除（Gridの子は自動的に幅が決まる） */
  padding: 32px;
}
```

### バグ7: `<span>` はインライン要素なので配置が正しく効かない

**問題:** `.cta-badge` は `<span>` 要素（インライン）。`position: absolute` は効くが、`padding` の上下や寸法が期待どおりにならない場合がある。明示的に `display: inline-block` または `display: block` にすべき。

**修正:**
```css
.cta-badge {
  display: inline-block;  /* 追加 */
}
```

### バグ8: セレクタのスペルミス

**問題:** HTMLでは `testimonial-author`（正しいスペル）だが、CSSでは `.tesimonial-author`（`t` が抜けている）。スタイルが適用されない。

**修正:**
```css
.testimonial-author {  /* tesimonial → testimonial */
  font-weight: bold;
  margin-top: 12px;
  color: #3498db;
}
```

### バグ9: フッターの `flex-direction` が間違い

**問題:** 3つのフッターセクションを横に並べたいのに `flex-direction: column` になっている。

**修正:**
```css
.footer {
  flex-direction: row;  /* column → row */
}
```

### バグ10: メディアクエリの条件が逆

**問題:** モバイル用のスタイル（1列レイアウト）を `@media (max-width: 768px)` で書いているが、フッターは元が `column`（バグ9修正前）なので条件は正しいように見える。しかし、バグ9を修正して `row` にした後、768px以下で再び `column` にする意図のため、メディアクエリ自体は正しい。

**実際の問題:** デスクトップ（768px超）で `.feature-grid` が3列のままでよいが、バグ5の修正後にタブレット幅（768px〜1024px程度）ではカードが狭すぎる。中間のブレークポイントを追加すべき。

**修正:**
```css
@media (max-width: 1024px) {
  .feature-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (max-width: 768px) {
  .feature-grid {
    grid-template-columns: 1fr;
  }

  .footer {
    flex-direction: column;
    gap: 24px;
  }
}
```
