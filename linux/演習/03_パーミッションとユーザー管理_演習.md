## 基本問題

### 問題1: パーミッションの読み方
以下の `ls -l` の出力から、パーミッションの意味を読み取ってください。

```
-rwxr-xr-- 1 tanaka developers 4096 Mar 20 15:00 deploy.sh
```

1. ファイルの種類は何ですか？
2. 所有者の権限は何ですか？
3. グループの権限は何ですか？
4. その他のユーザーの権限は何ですか？
5. 所有者とグループは誰ですか？

<details>
<summary>解答</summary>

```
パーミッション: -rwxr-xr--

1. ファイルの種類: - （通常ファイル）
   ※ d ならディレクトリ、l ならシンボリックリンク

2. 所有者の権限: rwx（読み取り・書き込み・実行すべて可能）

3. グループの権限: r-x（読み取り・実行は可能、書き込みは不可）

4. その他のユーザーの権限: r--（読み取りのみ可能）

5. 所有者: tanaka、グループ: developers
```

</details>

### 問題2: 数値表記への変換
以下のパーミッションを数値（8進数）表記に変換してください。

1. `rwxr-xr-x`
2. `rw-r--r--`
3. `rwx------`
4. `rw-rw-r--`

<details>
<summary>ヒント</summary>

r=4, w=2, x=1 で各桁を合計します。

</details>

<details>
<summary>解答</summary>

```
1. rwxr-xr-x
   rwx = 4+2+1 = 7
   r-x = 4+0+1 = 5
   r-x = 4+0+1 = 5
   → 755

2. rw-r--r--
   rw- = 4+2+0 = 6
   r-- = 4+0+0 = 4
   r-- = 4+0+0 = 4
   → 644

3. rwx------
   rwx = 4+2+1 = 7
   --- = 0+0+0 = 0
   --- = 0+0+0 = 0
   → 700

4. rw-rw-r--
   rw- = 4+2+0 = 6
   rw- = 4+2+0 = 6
   r-- = 4+0+0 = 4
   → 664
```

</details>

### 問題3: chmod コマンド
以下の要件を満たす `chmod` コマンドを書いてください。

1. `script.sh` に所有者のみ実行権限を追加する
2. `config.txt` を所有者は読み書き可能、その他は読み取りのみにする
3. `secret.txt` を所有者のみ読み書き可能にする（他は一切アクセス不可）
4. `public/` ディレクトリ以下のすべてのファイルに全員の読み取り権限を追加する

<details>
<summary>解答</summary>

```bash
# 1. 所有者に実行権限を追加（シンボリック表記）
$ chmod u+x script.sh

# 2. 所有者は rw、グループとその他は r のみ（数値表記: 644）
$ chmod 644 config.txt

# 3. 所有者のみ rw、他は一切不可（数値表記: 600）
$ chmod 600 secret.txt

# 4. ディレクトリ以下すべてに読み取り権限を追加（-R で再帰的）
$ chmod -R a+r public/
```

</details>

### 問題5: chown コマンド
以下の要件を満たす `chown` コマンドを書いてください。

1. `app.log` の所有者を `www-data` に変更する
2. `project/` ディレクトリの所有者を `tanaka`、グループを `developers` に変更する
3. `data/` ディレクトリ以下のすべてのファイルのグループを `staff` に変更する

<details>
<summary>解答</summary>

```bash
# 1. 所有者を変更（sudo が必要）
$ sudo chown www-data app.log

# 2. 所有者とグループを同時に変更
$ sudo chown tanaka:developers project/

# 3. 再帰的にグループを変更（-R オプション）
$ sudo chgrp -R staff data/
# または
$ sudo chown -R :staff data/
```

</details>

## 応用問題

### 問題6: sudo の使い方
以下の操作について、なぜ `sudo` が必要か、または不要かを説明してください。

1. `cat /etc/shadow`
2. `ls -la /home/tanaka/`
3. `systemctl restart nginx`
4. `chmod 600 ~/secret.txt`

<details>
<summary>解答</summary>

```
1. cat /etc/shadow
   → sudo が必要。/etc/shadow はパスワードハッシュが格納されており、
     root のみ読み取り可能（パーミッション: ----------）。

2. ls -la /home/tanaka/
   → tanaka ユーザー本人なら sudo は不要。他のユーザーの場合は
     パーミッションによっては sudo が必要。

3. systemctl restart nginx
   → sudo が必要。サービスの起動・停止はシステム全体に影響する
     操作であり、root 権限が必要。

4. chmod 600 ~/secret.txt
   → sudo は不要。自分のホームディレクトリのファイルは自分が
     所有者なので、権限変更が可能。
```

</details>

### 問題7: パーミッションのトラブルシューティング
以下の状況で、何が原因でエラーが発生しているかを特定し、解決コマンドを書いてください。

**状況**: `tanaka` ユーザーが `./deploy.sh` を実行しようとしたが、以下のエラーが出た。

```bash
$ ls -l deploy.sh
-rw-r--r-- 1 tanaka tanaka 256 Mar 20 15:00 deploy.sh

$ ./deploy.sh
bash: ./deploy.sh: Permission denied
```

<details>
<summary>解答</summary>

```
【原因】
deploy.sh に実行権限（x）が付与されていない。
現在のパーミッションは rw-r--r-- (644) で、誰にも実行権限がない。

【解決方法】
```

```bash
# 実行権限を追加
$ chmod +x deploy.sh
# または
$ chmod 755 deploy.sh

# 確認
$ ls -l deploy.sh
-rwxr-xr-x 1 tanaka tanaka 256 Mar 20 15:00 deploy.sh

# 実行
$ ./deploy.sh
```

</details>

### 問題8: ユーザー管理コマンド
以下の操作を行うコマンドを書いてください。

1. `newuser` というユーザーを作成する（ホームディレクトリも作成）
2. `newuser` のパスワードを設定する
3. `newuser` を `developers` グループに追加する
4. `newuser` のアカウントを無効化する

<details>
<summary>解答</summary>

```bash
# 1. ユーザーの作成（-m でホームディレクトリも作成）
$ sudo useradd -m newuser

# 2. パスワードの設定
$ sudo passwd newuser
New password: ********
Retype new password: ********
passwd: password updated successfully

# 3. グループに追加（-aG で既存グループを維持しつつ追加）
$ sudo usermod -aG developers newuser

# 4. アカウントの無効化（ロック）
$ sudo usermod -L newuser
# または
$ sudo passwd -l newuser
```

</details>

## チャレンジ問題

### 問題9: Webサーバーのパーミッション設定
Webサーバー（nginx）で公開するディレクトリを以下の要件で設定してください。

- ディレクトリパス: `/var/www/html`
- 所有者: `www-data`
- グループ: `www-data`
- ディレクトリのパーミッション: 所有者とグループは読み取り・実行可能、その他は読み取り・実行可能
- ファイルのパーミッション: 所有者は読み書き可能、グループとその他は読み取りのみ

<details>
<summary>ヒント</summary>

`chown -R`、`find` + `chmod` を組み合わせて、ディレクトリとファイルに異なるパーミッションを設定します。

</details>

<details>
<summary>解答</summary>

```bash
# 所有者とグループを設定
$ sudo chown -R www-data:www-data /var/www/html

# ディレクトリのパーミッション設定（755）
$ sudo find /var/www/html -type d -exec chmod 755 {} \;

# ファイルのパーミッション設定（644）
$ sudo find /var/www/html -type f -exec chmod 644 {} \;

# 確認
$ ls -la /var/www/html/
drwxr-xr-x 2 www-data www-data 4096 Mar 20 15:00 .
-rw-r--r-- 1 www-data www-data  612 Mar 20 15:00 index.html
```

解説：
- ディレクトリに x（実行）権限がないと、中のファイル一覧を取得できない
- ファイルには通常 x（実行）権限は不要
- `find -type d` でディレクトリのみ、`find -type f` でファイルのみを対象にする

</details>

### 問題10: /etc/passwd の解析
`/etc/passwd` ファイルの以下のエントリについて、各フィールドの意味を答えてください。

```
tanaka:x:1000:1000:Tanaka Taro:/home/tanaka:/bin/bash
```

<details>
<summary>解答</summary>

```
tanaka:x:1000:1000:Tanaka Taro:/home/tanaka:/bin/bash
│      │ │    │    │           │             │
│      │ │    │    │           │             └─ ログインシェル
│      │ │    │    │           └─ ホームディレクトリ
│      │ │    │    └─ GECOS（フルネーム等のコメント）
│      │ │    └─ GID（グループID）
│      │ └─ UID（ユーザーID）
│      └─ パスワード（x は /etc/shadow に格納されていることを示す）
└─ ユーザー名

各フィールドの値:
- ユーザー名: tanaka
- パスワード: x（/etc/shadow を参照）
- UID: 1000（一般ユーザー）
- GID: 1000
- コメント: Tanaka Taro
- ホームディレクトリ: /home/tanaka
- ログインシェル: /bin/bash
```

</details>
