## 基本問題

### 問題1: SCPの基本操作
以下の操作をSCPコマンドで書いてください。

1. ローカルの `report.pdf` をリモートサーバー `server` の `~/documents/` にコピー
2. リモートサーバー `server` の `~/data.csv` をローカルのカレントディレクトリにダウンロード
3. ローカルの `project/` ディレクトリをリモートサーバー `server` の `~/` に再帰的にコピー

<details>
<summary>解答</summary>

```bash
# 1. アップロード
$ scp report.pdf user@server:~/documents/

# 2. ダウンロード
$ scp user@server:~/data.csv ./

# 3. ディレクトリの再帰的コピー
$ scp -r project/ user@server:~/
```

</details>

### 問題2: SCPのオプション
以下の条件でSCPコマンドを書いてください。

1. ポート2222で、`file.txt` をリモートの `~/` にコピー
2. 秘密鍵 `~/.ssh/id_ed25519_work` を指定して、`file.txt` をコピー
3. 帯域幅を1000Kbit/sに制限して、`large.iso` をコピー

<details>
<summary>ヒント</summary>

SCPのポート指定は大文字の `-P` です（sshの小文字 `-p` と異なります）。

</details>

<details>
<summary>解答</summary>

```bash
# 1. ポート指定（大文字の -P）
$ scp -P 2222 file.txt user@server:~/

# 2. 秘密鍵指定
$ scp -i ~/.ssh/id_ed25519_work file.txt user@server:~/

# 3. 帯域制限
$ scp -l 1000 large.iso user@server:~/
```

</details>

### 問題3: SFTPの基本コマンド
SFTPセッション内で以下の操作を行うコマンドを書いてください。

1. リモートのファイル一覧を表示する
2. リモートの `/var/www/html` に移動する
3. ローカルの `index.html` をアップロードする
4. リモートの `backup.tar.gz` をダウンロードする
5. ローカルの現在のディレクトリを表示する

<details>
<summary>解答</summary>

```bash
sftp> ls              # 1. リモートのファイル一覧
sftp> cd /var/www/html  # 2. リモートのディレクトリ移動
sftp> put index.html  # 3. アップロード
sftp> get backup.tar.gz  # 4. ダウンロード
sftp> lpwd            # 5. ローカルの現在ディレクトリ
```

リモート操作のコマンドにはプレフィックスなし、ローカル操作は `l` プレフィックスが付きます。

</details>

### 問題4: 転送方式の比較
SCP・SFTP・rsyncの特徴について、以下の表の空欄（○/×）を埋めてください。

| 機能 | SCP | SFTP | rsync |
|------|-----|------|-------|
| 対話的操作 | (1) | (2) | (3) |
| 差分転送 | (4) | (5) | (6) |
| 除外パターン | (7) | (8) | (9) |

<details>
<summary>解答</summary>

| 機能 | SCP | SFTP | rsync |
|------|-----|------|-------|
| 対話的操作 | × | ○ | × |
| 差分転送 | × | × | ○ |
| 除外パターン | × | × | ○ |

```
SCP:   シンプルなコピーのみ。対話操作や高度な機能はなし
SFTP:  FTPのような対話的操作が可能。ただし差分転送は未対応
rsync: 差分転送と除外パターンに対応。大量ファイルの同期に最適
```

</details>

## 応用問題

### 問題5: rsyncの基本操作
以下の操作をrsyncコマンドで書いてください。

1. ローカルの `myproject/` をリモートの `~/projects/myproject/` に同期
2. 上記の操作で `.git` と `node_modules` を除外
3. 実際にはコピーせず、何が転送されるか確認（ドライラン）

<details>
<summary>解答</summary>

```bash
# 1. 基本的な同期
$ rsync -avz myproject/ user@server:~/projects/myproject/

# 2. 除外パターン付き
$ rsync -avz --exclude '.git' --exclude 'node_modules' \
    myproject/ user@server:~/projects/myproject/

# 3. ドライラン（-n オプション）
$ rsync -avzn --exclude '.git' --exclude 'node_modules' \
    myproject/ user@server:~/projects/myproject/
```

</details>

### 問題6: rsyncのスラッシュ
以下の2つのコマンドの違いを説明してください。

```bash
# コマンドA
$ rsync -avz mydir/ user@server:~/dest/

# コマンドB
$ rsync -avz mydir user@server:~/dest/
```

<details>
<summary>解答</summary>

```
コマンドA（スラッシュあり）:
  mydir/ の中身を dest/ にコピーする
  結果: dest/file1, dest/file2 ...

コマンドB（スラッシュなし）:
  mydir ディレクトリ自体を dest/ にコピーする
  結果: dest/mydir/file1, dest/mydir/file2 ...

つまり、コピー元のスラッシュの有無で、ディレクトリ階層が1段変わります。
意図しない構造にならないよう注意が必要です。
```

</details>

### 問題7: SFTPバッチモード
以下の操作をSFTPバッチファイルで自動化してください。

- リモートの `/var/www/html` に移動
- `index.html`、`style.css`、`script.js` の3ファイルをアップロード
- アップロード後のファイル一覧を確認

バッチファイルの内容と実行コマンドを書いてください。

<details>
<summary>解答</summary>

バッチファイル（`deploy.sftp`）:
```
cd /var/www/html
put index.html
put style.css
put script.js
ls -la
```

実行コマンド:
```bash
$ sftp -b deploy.sftp user@server
```

</details>

## チャレンジ問題

### 問題8: 用途に応じた選択
以下の各シーンでは SCP、SFTP、rsync のどれを使うべきですか？理由も答えてください。

1. 1つの設定ファイルをサーバーにアップロードする
2. サーバー上のファイルを対話的に探索しながらダウンロードする
3. 毎日のバックアップでローカルとリモートのディレクトリを同期する
4. サーバー上の大量の画像ファイルを、新しいものだけダウンロードする

<details>
<summary>解答</summary>

```
1. → SCP（またはrsync）
   理由: 1ファイルの単純なコピーなので、最もシンプルなSCPで十分。

2. → SFTP
   理由: ls、cd でリモートを探索しながらgetでダウンロードできる。
   対話的な操作が必要な場面ではSFTPが最適。

3. → rsync
   理由: 差分転送により、変更されたファイルだけを転送する。
   毎日の同期で全ファイルを転送するのは非効率。
   --delete オプションで削除も反映できる。

4. → rsync
   理由: 差分転送により、既にダウンロード済みのファイルをスキップする。
   大量ファイルの部分的な転送に最適。
```

</details>

### 問題9: tar + SSH による転送
ディスク容量が少ない環境で、中間ファイルを作成せずにディレクトリを転送する方法を説明してください。以下の操作をコマンドで書いてください。

1. ローカルの `myproject/` をリモートの `~/projects/` に転送
2. リモートの `~/backup/data/` をローカルのカレントディレクトリに取得

<details>
<summary>解答</summary>

```bash
# 1. ローカル → リモート
$ tar czf - myproject/ | ssh user@server "tar xzf - -C ~/projects/"

# 解説:
# tar czf -     : myproject/を圧縮して標準出力に出す
# |              : パイプで次のコマンドに渡す
# ssh ... "..."  : リモートで tar xzf - を実行（標準入力から展開）
# -C ~/projects/ : 展開先ディレクトリを指定

# 2. リモート → ローカル
$ ssh user@server "tar czf - -C ~/backup data/" | tar xzf -

# 解説:
# ssh ... "..."  : リモートで tar czf - を実行（標準出力に出す）
# -C ~/backup    : ~/backup ディレクトリ基準で
# data/          : data/ を圧縮
# | tar xzf -    : ローカルで標準入力から展開
```

この方法では中間ファイル（.tar.gz）を作成せず、パイプで直接転送するため、ディスク容量を節約できます。

</details>
