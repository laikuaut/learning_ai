## 基本問題

### 問題1：try...catch の基本

以下のコードにエラー処理を追加してください。JSON文字列をパースし、パースに失敗した場合はエラーメッセージを表示してデフォルト値を返すようにします。

```javascript
const jsonStrings = [
  '{"name": "太郎", "age": 25}',
  '{ invalid json }',
  '{"name": "花子"}'
];
```

**期待される出力：**
```
成功: { name: "太郎", age: 25 }
パースエラー: Expected property name or '}' ...
デフォルト値を使用: {}
成功: { name: "花子" }
```

<details>
<summary>解答例</summary>

```javascript
const jsonStrings = [
  '{"name": "太郎", "age": 25}',
  '{ invalid json }',
  '{"name": "花子"}'
];

function safeParseJSON(str) {
  try {
    const result = JSON.parse(str);
    console.log("成功:", JSON.stringify(result));
    return result;
  } catch (error) {
    console.log(`パースエラー: ${error.message}`);
    console.log("デフォルト値を使用: {}");
    return {};
  }
}

jsonStrings.forEach(str => safeParseJSON(str));
```

</details>

### 問題2：throw でエラーを投げる

年齢を検証する関数 `validateAge` を作成してください。以下の条件でエラーを投げます。

- 引数が数値でない: `TypeError`
- 0未満または150超: `RangeError`

**期待される出力：**
```
25 → 有効な年齢です
"abc" → TypeError: 年齢は数値で入力してください
-5 → RangeError: 年齢は0〜150の範囲で入力してください
200 → RangeError: 年齢は0〜150の範囲で入力してください
```

<details>
<summary>解答例</summary>

```javascript
function validateAge(age) {
  if (typeof age !== "number" || isNaN(age)) {
    throw new TypeError("年齢は数値で入力してください");
  }
  if (age < 0 || age > 150) {
    throw new RangeError("年齢は0〜150の範囲で入力してください");
  }
  return true;
}

const testValues = [25, "abc", -5, 200];

testValues.forEach(value => {
  try {
    validateAge(value);
    console.log(`${value} → 有効な年齢です`);
  } catch (error) {
    console.log(`${JSON.stringify(value)} → ${error.name}: ${error.message}`);
  }
});
```

</details>

### 問題3：エラーの型による分岐

以下のコードに、エラーの型に応じたメッセージを表示する処理を追加してください。

```javascript
function riskyOperation(input) {
  if (input === null) {
    throw new TypeError("nullは受け付けません");
  }
  if (input < 0) {
    throw new RangeError("負の値は受け付けません");
  }
  return JSON.parse(input);
}
```

**期待される出力：**
```
入力 null → 型エラー: nullは受け付けません
入力 -1 → 範囲エラー: 負の値は受け付けません
入力 "invalid" → 構文エラー: JSONのパースに失敗しました
入力 "42" → 結果: 42
```

<details>
<summary>解答例</summary>

```javascript
function riskyOperation(input) {
  if (input === null) {
    throw new TypeError("nullは受け付けません");
  }
  if (input < 0) {
    throw new RangeError("負の値は受け付けません");
  }
  return JSON.parse(input);
}

const testInputs = [null, -1, "invalid", "42"];

testInputs.forEach(input => {
  try {
    const result = riskyOperation(input);
    console.log(`入力 ${JSON.stringify(input)} → 結果: ${result}`);
  } catch (error) {
    if (error instanceof TypeError) {
      console.log(`入力 ${JSON.stringify(input)} → 型エラー: ${error.message}`);
    } else if (error instanceof RangeError) {
      console.log(`入力 ${JSON.stringify(input)} → 範囲エラー: ${error.message}`);
    } else if (error instanceof SyntaxError) {
      console.log(`入力 ${JSON.stringify(input)} → 構文エラー: JSONのパースに失敗しました`);
    } else {
      console.log(`入力 ${JSON.stringify(input)} → 不明なエラー: ${error.message}`);
    }
  }
});
```

</details>

### 問題4：カスタムエラーの作成

以下のカスタムエラークラスを作成し、ユーザー登録の検証で使用してください。

1. `ValidationError`: フィールド名とメッセージを持つ
2. `DuplicateError`: リソース名を持つ

```javascript
// 既存ユーザーリスト
const existingUsers = ["taro@example.com", "hanako@example.com"];
```

**期待される出力：**
```
--- テスト1 ---
登録成功: jiro@example.com
--- テスト2 ---
入力エラー (name): 名前は2文字以上で入力してください
--- テスト3 ---
重複エラー: メールアドレス taro@example.com は既に登録されています
```

<details>
<summary>解答例</summary>

```javascript
// カスタムエラークラス
class ValidationError extends Error {
  constructor(field, message) {
    super(message);
    this.name = "ValidationError";
    this.field = field;
  }
}

class DuplicateError extends Error {
  constructor(resource, value) {
    super(`${resource} ${value} は既に登録されています`);
    this.name = "DuplicateError";
    this.resource = resource;
    this.value = value;
  }
}

// 既存ユーザー
const existingUsers = ["taro@example.com", "hanako@example.com"];

// ユーザー登録関数
function registerUser({ name, email }) {
  if (!name || name.length < 2) {
    throw new ValidationError("name", "名前は2文字以上で入力してください");
  }
  if (!email || !/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
    throw new ValidationError("email", "正しいメールアドレスを入力してください");
  }
  if (existingUsers.includes(email)) {
    throw new DuplicateError("メールアドレス", email);
  }
  existingUsers.push(email);
  return `登録成功: ${email}`;
}

// テスト
const tests = [
  { name: "次郎", email: "jiro@example.com" },
  { name: "A", email: "a@example.com" },
  { name: "太郎", email: "taro@example.com" }
];

tests.forEach((data, i) => {
  console.log(`--- テスト${i + 1} ---`);
  try {
    console.log(registerUser(data));
  } catch (error) {
    if (error instanceof ValidationError) {
      console.log(`入力エラー (${error.field}): ${error.message}`);
    } else if (error instanceof DuplicateError) {
      console.log(`重複エラー: ${error.message}`);
    } else {
      console.log(`予期しないエラー: ${error.message}`);
    }
  }
});
```

</details>

### 問題5：非同期処理のエラーハンドリング

APIからデータを取得し、エラーの種類に応じて適切に処理する関数を作成してください。

- ネットワークエラー: 「ネットワークに接続できません」
- 404エラー: 「リソースが見つかりません」
- その他のHTTPエラー: 「サーバーエラー（コード: XXX）」

**期待される出力：**
```
--- 正常なURL ---
ユーザー: Leanne Graham
--- 存在しないURL ---
リソースが見つかりません
--- 不正なURL ---
ネットワークに接続できません
```

<details>
<summary>解答例</summary>

```javascript
async function safeFetch(url) {
  try {
    const response = await fetch(url);

    if (!response.ok) {
      if (response.status === 404) {
        throw new Error("リソースが見つかりません");
      }
      throw new Error(`サーバーエラー（コード: ${response.status}）`);
    }

    return await response.json();
  } catch (error) {
    // fetch自体が失敗した場合（ネットワークエラー）
    if (error instanceof TypeError) {
      console.log("ネットワークに接続できません");
    } else {
      console.log(error.message);
    }
    return null;
  }
}

async function main() {
  console.log("--- 正常なURL ---");
  const user = await safeFetch("https://jsonplaceholder.typicode.com/users/1");
  if (user) console.log(`ユーザー: ${user.name}`);

  console.log("--- 存在しないURL ---");
  await safeFetch("https://jsonplaceholder.typicode.com/users/999");

  console.log("--- 不正なURL ---");
  await safeFetch("https://invalid-domain-12345.example.com");
}

main();
```

</details>

### 問題6：Result型パターン

エラーを値として扱う「Result型」パターンを実装してください。成功時は `{ success: true, data }` を、失敗時は `{ success: false, error }` を返す関数群を作成します。

```javascript
// Result型のユーティリティ
function ok(data) { /* ... */ }
function err(message) { /* ... */ }
function isOk(result) { /* ... */ }
```

これを使って、JSON解析 → データ検証 → データ変換のパイプラインを作成してください。

**期待される出力：**
```
--- テスト1: 正常データ ---
成功: { name: "太郎", age: 25, isAdult: true }
--- テスト2: 不正なJSON ---
失敗: JSONのパースに失敗しました
--- テスト3: 必須フィールド欠如 ---
失敗: nameフィールドは必須です
```

<details>
<summary>解答例</summary>

```javascript
// Result型ユーティリティ
function ok(data) {
  return { success: true, data };
}

function err(message) {
  return { success: false, error: message };
}

function isOk(result) {
  return result.success === true;
}

// パイプライン関数
function parseJSON(str) {
  try {
    return ok(JSON.parse(str));
  } catch {
    return err("JSONのパースに失敗しました");
  }
}

function validateData(result) {
  if (!isOk(result)) return result;
  const data = result.data;
  if (!data.name) return err("nameフィールドは必須です");
  if (data.age !== undefined && typeof data.age !== "number") {
    return err("ageフィールドは数値でなければなりません");
  }
  return ok(data);
}

function transformData(result) {
  if (!isOk(result)) return result;
  const data = result.data;
  return ok({
    ...data,
    isAdult: (data.age || 0) >= 18
  });
}

// テスト
const tests = [
  { label: "正常データ", input: '{"name": "太郎", "age": 25}' },
  { label: "不正なJSON", input: "invalid" },
  { label: "必須フィールド欠如", input: '{"age": 20}' }
];

tests.forEach(({ label, input }, i) => {
  console.log(`--- テスト${i + 1}: ${label} ---`);
  const result = transformData(validateData(parseJSON(input)));
  if (isOk(result)) {
    console.log(`成功: ${JSON.stringify(result.data)}`);
  } else {
    console.log(`失敗: ${result.error}`);
  }
});
```

</details>
