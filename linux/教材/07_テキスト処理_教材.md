# 第7章：テキスト処理

## この章のゴール

- grep、sed、awk といった強力なテキスト処理コマンドを使いこなせるようになること
- sort、uniq、cut、tr、wc などのフィルタコマンドを理解し、組み合わせて使えるようになること
- パイプラインでコマンドを連結し、実務で役立つデータ加工ができるようになること

## 7.1 テキスト処理の考え方

Linuxでは「すべてはファイル」という思想があり、設定ファイル、ログファイル、コマンド出力のほとんどがテキストデータです。テキスト処理コマンドを使いこなすことは、Linux管理の生産性を大きく左右します。

### テキスト処理の基本パターン

```
┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
│  入力   │───→│ フィルタ │───→│ フィルタ │───→│  出力   │
│(ファイル)│ │  │  (grep) │ │  │ (sort)  │    │(画面/   │
│         │ │  │         │ │  │         │    │ ファイル)│
└────────┘ │  └────────┘ │  └────────┘    └────────┘
           パイプ         パイプ
```

この章で学ぶコマンドは、すべてパイプで連結して使うことを前提に設計されています。

## 7.2 grep - パターン検索

`grep`（Global Regular Expression Print）は、テキストの中から特定のパターンに一致する行を検索して表示するコマンドです。Linux で最も使用頻度の高いコマンドの一つです。

### 基本的な使い方

```bash
# ファイル内のパターンを検索
$ grep "error" /var/log/syslog
Mar 20 10:15:23 myserver kernel: [error] disk I/O timeout
Mar 20 11:30:45 myserver nginx: [error] connection refused

# 標準入力から検索（パイプと組み合わせ）
$ cat /var/log/syslog | grep "error"

# 複数のファイルを検索
$ grep "error" /var/log/*.log
```

### 主要なオプション

| オプション | 説明 | 例 |
|-----------|------|-----|
| `-i` | 大文字小文字を無視 | `grep -i "error" log.txt` |
| `-n` | 行番号を表示 | `grep -n "error" log.txt` |
| `-c` | マッチした行数を表示 | `grep -c "error" log.txt` |
| `-v` | マッチしない行を表示（逆マッチ） | `grep -v "comment" config` |
| `-r` | ディレクトリを再帰的に検索 | `grep -r "TODO" /src/` |
| `-l` | マッチしたファイル名のみ表示 | `grep -rl "error" /var/log/` |
| `-w` | 単語単位でマッチ | `grep -w "log" file.txt` |
| `-A N` | マッチ行の後N行も表示 | `grep -A 3 "error" log.txt` |
| `-B N` | マッチ行の前N行も表示 | `grep -B 3 "error" log.txt` |
| `-C N` | マッチ行の前後N行を表示 | `grep -C 3 "error" log.txt` |
| `-E` | 拡張正規表現を使用 | `grep -E "error|warn" log.txt` |
| `--color` | マッチ部分を色付き表示 | `grep --color "error" log.txt` |

### 使用例

```bash
# 大文字小文字を無視して検索
$ grep -i "error" /var/log/syslog
Mar 20 10:15:23 Error: disk full
Mar 20 11:30:45 ERROR: connection refused

# 行番号付きで検索
$ grep -n "server" /etc/nginx/nginx.conf
10:    server_name example.com;
15:    server {
20:        listen 80;

# マッチしない行を表示（コメント行を除外する例）
$ grep -v "^#" /etc/ssh/sshd_config | grep -v "^$"
Port 22
PermitRootLogin no
PasswordAuthentication yes

# ディレクトリ内を再帰的に検索
$ grep -rn "TODO" /home/tanaka/project/
/home/tanaka/project/app.py:15:# TODO: エラーハンドリングを追加
/home/tanaka/project/utils.py:42:# TODO: パフォーマンス改善

# 前後の行も表示（コンテキストの確認に便利）
$ grep -C 2 "error" /var/log/syslog
Mar 20 10:15:21 myserver systemd: Starting nginx...
Mar 20 10:15:22 myserver systemd: nginx started
Mar 20 10:15:23 myserver nginx: [error] config test failed
Mar 20 10:15:23 myserver systemd: nginx failed
Mar 20 10:15:24 myserver systemd: Stopping nginx...
```

### 正規表現との組み合わせ

```bash
# 行頭が "error" で始まる行
$ grep "^error" log.txt

# 行末が ".conf" で終わる行
$ grep "\.conf$" filelist.txt

# IPアドレスのパターンにマッチ
$ grep -E "[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+" access.log

# error または warning を含む行
$ grep -E "error|warning" /var/log/syslog

# 空行以外を表示
$ grep -v "^$" config.txt
```

### ポイントまとめ

- `grep` はテキスト検索の最重要コマンド
- `-i`（大文字小文字無視）、`-n`（行番号）、`-r`（再帰検索）が頻出
- `-v` で逆マッチ（除外フィルタ）ができる
- `-E` で拡張正規表現（`|` 等）が使える
- パイプと組み合わせて、他のコマンドの出力をフィルタリングする

## 7.3 sed - ストリームエディタ

`sed`（Stream Editor）は、テキストの置換・削除・挿入などを行うストリームエディタです。ファイルを開かずにテキストを加工できます。

### 基本的な置換

```bash
# 基本構文: sed 's/検索パターン/置換文字列/' ファイル名

# 各行の最初の "old" を "new" に置換
$ echo "old is old, very old" | sed 's/old/new/'
new is old, very old

# すべての "old" を "new" に置換（g フラグ）
$ echo "old is old, very old" | sed 's/old/new/g'
new is new, very new

# 大文字小文字を無視して置換（I フラグ）
$ echo "Hello HELLO hello" | sed 's/hello/hi/gI'
hi hi hi
```

### sedの主要な操作

| 操作 | 構文 | 説明 |
|------|------|------|
| 置換 | `s/パターン/置換/` | パターンに一致する文字列を置換 |
| 削除 | `d` | 行を削除 |
| 挿入 | `i\テキスト` | 行の前にテキストを挿入 |
| 追加 | `a\テキスト` | 行の後にテキストを追加 |
| 表示 | `p` | 行を表示 |

### 置換の実用例

```bash
# ファイル内の文字列を置換（結果を標準出力に表示）
$ sed 's/localhost/192.168.1.100/g' config.txt

# ファイルを直接書き換える（-i オプション）
$ sed -i 's/localhost/192.168.1.100/g' config.txt

# バックアップを作成しながら書き換え
$ sed -i.bak 's/localhost/192.168.1.100/g' config.txt
# → config.txt が更新され、config.txt.bak に元のファイルが残る

# 区切り文字を変更（パスにスラッシュが含まれる場合に便利）
$ sed 's|/usr/local/bin|/opt/bin|g' config.txt
$ sed 's#/old/path#/new/path#g' config.txt
```

### 行の操作

```bash
# 特定の行を削除
$ sed '3d' file.txt             # 3行目を削除
$ sed '2,5d' file.txt           # 2〜5行目を削除
$ sed '/^#/d' file.txt          # コメント行（#で始まる行）を削除
$ sed '/^$/d' file.txt          # 空行を削除

# 特定の行だけ置換
$ sed '3s/old/new/' file.txt    # 3行目だけ置換
$ sed '1,5s/old/new/g' file.txt # 1〜5行目を置換

# 特定の行を表示（-n と p の組み合わせ）
$ sed -n '5p' file.txt          # 5行目だけ表示
$ sed -n '10,20p' file.txt      # 10〜20行目を表示
$ sed -n '/error/p' file.txt    # "error" を含む行を表示
```

### 実務での使用例

```bash
# 設定ファイルのコメント行と空行を除外して表示
$ sed '/^#/d; /^$/d' /etc/ssh/sshd_config

# ログファイルからIPアドレスをマスクする
$ sed -E 's/[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+/xxx.xxx.xxx.xxx/g' access.log

# 複数の置換を一度に実行（-e オプション）
$ sed -e 's/foo/bar/g' -e 's/baz/qux/g' file.txt

# HTMLタグを除去
$ sed 's/<[^>]*>//g' page.html
```

### ポイントまとめ

- `sed 's/検索/置換/g'` が最も基本的な使い方
- `g` フラグで行内の全一致を置換（なしだと最初の1つだけ）
- `-i` でファイルを直接書き換え（`-i.bak` でバックアップ付き）
- `/^#/d` でコメント行を削除、`/^$/d` で空行を削除
- 区切り文字は `/` 以外も使える（`|`, `#` など）

## 7.4 awk - テキスト処理言語

`awk` は、テキストを列（フィールド）単位で処理するためのプログラミング言語です。ログの解析やデータの集計に非常に強力です。

### 基本的な使い方

```bash
# 基本構文: awk 'パターン { アクション }' ファイル名

# デフォルトでは空白区切りでフィールドを分割
$ echo "tanaka 30 tokyo" | awk '{print $1}'
tanaka

$ echo "tanaka 30 tokyo" | awk '{print $1, $3}'
tanaka tokyo

$ echo "tanaka 30 tokyo" | awk '{print $0}'    # $0 は行全体
tanaka 30 tokyo
```

### フィールド変数

| 変数 | 意味 |
|------|------|
| `$0` | 行全体 |
| `$1` | 1番目のフィールド |
| `$2` | 2番目のフィールド |
| `$NF` | 最後のフィールド |
| `NF` | フィールド数 |
| `NR` | 現在の行番号 |

```bash
# 最後のフィールドを表示
$ echo "a b c d e" | awk '{print $NF}'
e

# 行番号を付けて表示
$ awk '{print NR, $0}' file.txt
1 first line
2 second line
3 third line

# フィールド数を表示
$ echo "a b c" | awk '{print NF}'
3
```

### 区切り文字の指定

```bash
# コロン区切りのファイル（例: /etc/passwd）
$ awk -F: '{print $1, $3}' /etc/passwd
root 0
daemon 1
bin 2
tanaka 1000

# カンマ区切り（CSV）
$ echo "tanaka,30,tokyo" | awk -F, '{print $1, $3}'
tanaka tokyo

# 出力の区切り文字を指定（OFS）
$ echo "tanaka,30,tokyo" | awk -F, -v OFS="\t" '{print $1, $2, $3}'
tanaka	30	tokyo
```

### パターンマッチ

```bash
# 特定のパターンを含む行だけ処理
$ awk '/error/ {print $0}' /var/log/syslog

# 特定のフィールドで条件フィルタ
$ awk -F: '$3 >= 1000 {print $1, $3}' /etc/passwd
tanaka 1000
suzuki 1001

# 条件に応じて異なるアクションを実行
$ awk -F: '{
    if ($3 == 0) print $1, "は root ユーザーです"
    else if ($3 >= 1000) print $1, "は一般ユーザーです"
    else print $1, "はシステムユーザーです"
}' /etc/passwd
```

### BEGIN / END ブロック

```bash
# BEGIN: 処理の前に1回実行、END: 処理の後に1回実行
$ awk 'BEGIN {print "=== ユーザー一覧 ==="} {print $1} END {print "=== 以上 ==="}' /etc/passwd
=== ユーザー一覧 ===
root
daemon
...
tanaka
=== 以上 ===

# 合計の計算
$ echo -e "100\n200\n300" | awk '{sum += $1} END {print "合計:", sum}'
合計: 600

# 平均の計算
$ echo -e "80\n90\n70\n85" | awk '{sum += $1; count++} END {print "平均:", sum/count}'
平均: 81.25
```

### 実務での使用例

```bash
# ディスク使用率の取得
$ df -h | awk 'NR>1 {print $5, $6}'
45% /
12% /boot
78% /home

# アクセスログからステータスコード別の件数を集計
$ awk '{count[$9]++} END {for (code in count) print code, count[code]}' access.log
200 15234
301 423
404 89
500 12

# プロセスのメモリ使用量トップ5
$ ps aux | awk 'NR>1 {print $4, $11}' | sort -rn | head -5
5.2 /usr/bin/firefox
3.1 /usr/bin/mysqld
1.8 /usr/sbin/nginx
1.2 /usr/bin/python3
0.8 /usr/sbin/sshd
```

### ポイントまとめ

- `awk` はフィールド（列）単位の処理に最適
- `-F` で区切り文字を指定（デフォルトは空白）
- `$1`, `$2`, ... でフィールドを参照、`$0` で行全体
- `BEGIN` / `END` ブロックでヘッダー・フッター・集計処理
- パターンマッチで条件に合う行だけ処理できる

## 7.5 sort - 並べ替え

`sort` コマンドは、テキストを行単位で並べ替えます。

### 基本的な使い方

```bash
# アルファベット順にソート（デフォルト）
$ echo -e "cherry\napple\nbanana" | sort
apple
banana
cherry

# 逆順（降順）にソート（-r）
$ echo -e "cherry\napple\nbanana" | sort -r
cherry
banana
apple

# 数値としてソート（-n）
$ echo -e "10\n2\n30\n1" | sort -n
1
2
10
30

# 数値の降順
$ echo -e "10\n2\n30\n1" | sort -rn
30
10
2
1
```

### 主要なオプション

| オプション | 説明 | 例 |
|-----------|------|-----|
| `-n` | 数値としてソート | `sort -n` |
| `-r` | 逆順（降順） | `sort -r` |
| `-k N` | N番目のフィールドでソート | `sort -k 2` |
| `-t 区切り` | フィールドの区切り文字を指定 | `sort -t: -k3 -n` |
| `-u` | 重複行を除去 | `sort -u` |
| `-h` | 人が読める数値でソート（1K, 2M等） | `sort -h` |

### 使用例

```bash
# 特定のフィールドでソート（例: 5列目のファイルサイズ順）
$ ls -l | sort -k5 -n
-rw-r--r-- 1 tanaka tanaka    256 Mar 18 09:15 memo.txt
-rw-r--r-- 1 tanaka tanaka   1024 Mar 19 10:00 data.csv
drwxr-xr-x 2 tanaka tanaka   4096 Mar 15 10:00 Desktop

# コロン区切りの3列目（UID）で数値ソート
$ sort -t: -k3 -n /etc/passwd
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin

# ディスク使用量を人が読める形式でソート
$ du -sh /var/log/* | sort -h
4.0K    /var/log/alternatives.log
12K     /var/log/dpkg.log
256K    /var/log/auth.log
1.5M    /var/log/syslog
```

### ポイントまとめ

- `sort` は行単位の並べ替え
- 数値ソートには `-n` が必須（文字列ソートだと "10" < "2" になる）
- `-k` でソート対象のフィールドを指定
- `-t` でフィールド区切り文字を指定

## 7.6 uniq - 重複行の処理

`uniq` は、連続する重複行を処理するコマンドです。通常 `sort` と組み合わせて使います。

### 基本的な使い方

```bash
# 連続する重複行を1つにまとめる
$ echo -e "apple\napple\nbanana\napple" | uniq
apple
banana
apple

# ↑ 注意: 3番目の apple は残っている（連続していないため）

# sort と組み合わせるのが正しい使い方
$ echo -e "apple\napple\nbanana\napple" | sort | uniq
apple
banana
```

### 主要なオプション

```bash
# 重複回数を表示（-c）← 最もよく使う
$ echo -e "apple\nbanana\napple\napple\ncherry\nbanana" | sort | uniq -c
      3 apple
      2 banana
      1 cherry

# 回数でソートして頻度順に表示
$ echo -e "apple\nbanana\napple\napple\ncherry\nbanana" | sort | uniq -c | sort -rn
      3 apple
      2 banana
      1 cherry

# 重複している行だけ表示（-d）
$ echo -e "apple\nbanana\napple\ncherry" | sort | uniq -d
apple

# 重複していない行だけ表示（-u）
$ echo -e "apple\nbanana\napple\ncherry" | sort | uniq -u
banana
cherry
```

### ポイントまとめ

- `uniq` は **連続する** 重複行だけ処理する（先に `sort` が必要）
- `sort | uniq -c | sort -rn` は頻度集計の定番パイプライン
- `-d` で重複行のみ、`-u` で非重複行のみ表示

## 7.7 cut - フィールドの切り出し

`cut` は、各行から特定のフィールドや文字位置を切り出すコマンドです。

### 基本的な使い方

```bash
# 区切り文字とフィールドを指定して切り出し（-d と -f）
$ echo "tanaka:1000:Tokyo" | cut -d: -f1
tanaka

$ echo "tanaka:1000:Tokyo" | cut -d: -f1,3
tanaka:Tokyo

# /etc/passwd からユーザー名とホームディレクトリを取得
$ cut -d: -f1,6 /etc/passwd
root:/root
daemon:/usr/sbin
tanaka:/home/tanaka
```

### 主要なオプション

| オプション | 説明 | 例 |
|-----------|------|-----|
| `-d 区切り文字` | フィールド区切り文字を指定 | `cut -d: -f1` |
| `-f フィールド` | 取り出すフィールド番号 | `cut -f1,3` / `cut -f2-4` |
| `-c 文字位置` | 文字位置で切り出し | `cut -c1-5` |

```bash
# 文字位置で切り出し
$ echo "Hello, World!" | cut -c1-5
Hello

# フィールドの範囲指定
$ echo "a:b:c:d:e" | cut -d: -f2-4
b:c:d

# タブ区切り（デフォルト）
$ echo -e "name\tage\tcity" | cut -f2
age
```

### ポイントまとめ

- `cut` は単純なフィールド切り出しに最適
- `-d` で区切り文字、`-f` でフィールド番号を指定
- 複雑な処理が必要な場合は `awk` を使う

## 7.8 tr - 文字の変換・削除

`tr`（translate）は、文字の変換や削除を行うコマンドです。常に標準入力から読み取ります。

### 基本的な使い方

```bash
# 小文字を大文字に変換
$ echo "hello world" | tr 'a-z' 'A-Z'
HELLO WORLD

# 大文字を小文字に変換
$ echo "HELLO WORLD" | tr 'A-Z' 'a-z'
hello world

# スペースをタブに変換
$ echo "a b c" | tr ' ' '\t'
a	b	c

# 改行をスペースに変換（1行にまとめる）
$ echo -e "apple\nbanana\ncherry" | tr '\n' ' '
apple banana cherry
```

### 文字の削除

```bash
# 特定の文字を削除（-d）
$ echo "Hello, World! 123" | tr -d '0-9'
Hello, World!

$ echo "Hello, World! 123" | tr -d ' ,'
HelloWorld!123

# 数字以外を削除（-dc の組み合わせ = -c で補集合）
$ echo "Phone: 090-1234-5678" | tr -dc '0-9\n'
09012345678
```

### 文字の圧縮

```bash
# 連続する同一文字を1つにまとめる（-s）
$ echo "hello     world" | tr -s ' '
hello world

# 連続する改行を1つにまとめる（空行の除去）
$ cat file_with_blanks.txt | tr -s '\n'
```

### ポイントまとめ

- `tr` は1文字単位の変換・削除に使う
- `-d` で文字を削除、`-s` で連続する同一文字を圧縮
- ファイルを直接引数に取れない（常に標準入力から読み取る）
- 大文字小文字変換は `tr 'a-z' 'A-Z'` が定番

## 7.9 wc - テキストのカウント

`wc`（Word Count）は、行数・単語数・バイト数をカウントするコマンドです。

### 基本的な使い方

```bash
# 行数、単語数、バイト数を表示
$ wc /etc/passwd
  42  68  2345 /etc/passwd
  │   │   │
  │   │   └─ バイト数
  │   └───── 単語数
  └───────── 行数

# 行数のみ表示（-l）← 最もよく使う
$ wc -l /etc/passwd
42 /etc/passwd

# 単語数のみ表示（-w）
$ wc -w /etc/passwd
68 /etc/passwd

# バイト数のみ表示（-c）
$ wc -c /etc/passwd
2345 /etc/passwd

# 文字数のみ表示（-m）
$ wc -m /etc/passwd
2345 /etc/passwd
```

### パイプとの組み合わせ

```bash
# ファイル数をカウント
$ ls /var/log/*.log | wc -l
15

# プロセス数をカウント
$ ps aux | wc -l
128

# 特定の文字列を含む行数をカウント
$ grep -c "error" /var/log/syslog
23

# 空行の数をカウント
$ grep -c "^$" file.txt
5
```

### ポイントまとめ

- `wc -l` は行数のカウントで最も使用頻度が高い
- パイプと組み合わせて結果の件数を数えるのが定番
- `grep -c` でもパターンに一致する行数を数えられる

## 7.10 head / tail - ファイルの先頭・末尾の表示

### head - 先頭を表示

```bash
# 先頭10行を表示（デフォルト）
$ head /var/log/syslog

# 先頭N行を表示
$ head -n 5 /var/log/syslog
$ head -5 /var/log/syslog       # 短縮形
```

### tail - 末尾を表示

```bash
# 末尾10行を表示（デフォルト）
$ tail /var/log/syslog

# 末尾N行を表示
$ tail -n 20 /var/log/syslog

# リアルタイムで追記を監視（-f）← ログ監視で必須
$ tail -f /var/log/syslog
Mar 20 15:30:00 myserver nginx: access from 192.168.1.50
Mar 20 15:30:01 myserver nginx: access from 192.168.1.51
...
# Ctrl+C で停止

# 複数ファイルの同時監視
$ tail -f /var/log/syslog /var/log/auth.log
```

### ポイントまとめ

- `head -n N` で先頭N行、`tail -n N` で末尾N行を表示
- `tail -f` はリアルタイムログ監視の必須コマンド
- パイプと組み合わせて出力の先頭・末尾だけを取り出す

## 7.11 実践的なパイプライン例

これまで学んだコマンドを組み合わせた実践例を紹介します。

### アクセスログ解析

```bash
# アクセス数の多いIPアドレスのトップ10
$ awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -10
   1523 192.168.1.50
    892 203.0.113.100
    456 198.51.100.25
    ...

# ステータスコード別の集計
$ awk '{print $9}' access.log | sort | uniq -c | sort -rn
  15234 200
    423 301
     89 404
     12 500

# 404エラーのURLを集計
$ awk '$9 == 404 {print $7}' access.log | sort | uniq -c | sort -rn | head -10
```

### システム管理

```bash
# ディスク使用量のトップ10ディレクトリ
$ du -sh /var/* 2>/dev/null | sort -rh | head -10

# メモリを多く使っているプロセスのトップ5
$ ps aux --sort=-%mem | head -6

# ログインしているユーザーの一覧（重複なし）
$ who | awk '{print $1}' | sort -u

# 開いているポートとプロセスの一覧
$ sudo ss -tlnp | awk 'NR>1 {print $4, $6}' | sort
```

### データ加工

```bash
# CSVファイルの特定列の合計
$ cut -d, -f3 sales.csv | tail -n +2 | awk '{sum+=$1} END {print "合計:", sum}'

# テキストファイルの単語頻度集計
$ cat document.txt | tr -s ' ' '\n' | tr 'A-Z' 'a-z' | sort | uniq -c | sort -rn | head -20

# 設定ファイルの有効行だけ抽出（コメントと空行を除外）
$ grep -v '^#' config.conf | grep -v '^$' | sed 's/[[:space:]]*$//'
```

## 7.12 よくある間違い

### 間違い1: uniq を sort なしで使う

```bash
# 【誤り】uniq は連続する重複行しか処理しない
$ echo -e "a\nb\na" | uniq
a
b
a        # ← 重複が残る

# 【正解】先にソートする
$ echo -e "a\nb\na" | sort | uniq
a
b
```

### 間違い2: grep の正規表現で特殊文字をエスケープし忘れる

```bash
# 【誤り】ドットが任意の1文字にマッチしてしまう
$ grep "192.168.1.1" file.txt    # 192x168x1x1 にもマッチする

# 【正解】ドットをエスケープする
$ grep "192\.168\.1\.1" file.txt
```

### 間違い3: sed -i でバックアップを取らない

```bash
# 【危険】元のファイルが消える
$ sed -i 's/old/new/g' important.conf

# 【安全】バックアップを残す
$ sed -i.bak 's/old/new/g' important.conf
```

## 7.13 まとめ

| コマンド | 用途 | 代表的な使い方 |
|---------|------|---------------|
| `grep` | パターン検索 | `grep -rn "error" /var/log/` |
| `sed` | 文字列置換 | `sed 's/old/new/g' file.txt` |
| `awk` | フィールド処理 | `awk -F: '{print $1}' /etc/passwd` |
| `sort` | 並べ替え | `sort -rn`（数値の降順） |
| `uniq` | 重複処理 | `sort \| uniq -c`（頻度集計） |
| `cut` | フィールド切り出し | `cut -d: -f1,3` |
| `tr` | 文字の変換・削除 | `tr 'a-z' 'A-Z'` |
| `wc` | カウント | `wc -l`（行数カウント） |
| `head` | 先頭表示 | `head -n 10` |
| `tail` | 末尾表示 | `tail -f`（リアルタイム監視） |

### 定番パイプライン

```bash
# 頻度集計パイプライン
... | sort | uniq -c | sort -rn | head -10

# ログ解析パイプライン
awk '{print $フィールド}' ログファイル | sort | uniq -c | sort -rn

# 設定ファイルの有効行抽出
grep -v '^#' ファイル | grep -v '^$'
```
