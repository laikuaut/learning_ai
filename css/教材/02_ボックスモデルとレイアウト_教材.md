# 第2章：ボックスモデルとレイアウト

## この章で学ぶこと

- ボックスモデルの仕組み
- margin（外側の余白）
- padding（内側の余白）
- border（枠線）
- width / height（幅と高さ）
- box-sizing プロパティ
- display プロパティ
- overflow（はみ出しの制御）

---

## 1. ボックスモデルとは？

HTMLのすべての要素は「箱（ボックス）」として扱われます。この箱は4つの領域で構成されています。

```
┌─────────────────────────────────────┐
│            margin（外側の余白）         │
│  ┌───────────────────────────────┐  │
│  │         border（枠線）          │  │
│  │  ┌───────────────────────┐    │  │
│  │  │   padding（内側の余白）   │    │  │
│  │  │  ┌─────────────────┐  │    │  │
│  │  │  │   content        │  │    │  │
│  │  │  │  （中身・コンテンツ） │  │    │  │
│  │  │  └─────────────────┘  │    │  │
│  │  └───────────────────────┘    │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

**内側から順に：**
1. **content** ─ テキストや画像などの中身
2. **padding** ─ コンテンツと枠線の間の余白
3. **border** ─ 枠線
4. **margin** ─ 他の要素との間の余白

---

## 2. margin（外側の余白）

要素の外側に余白を作ります。他の要素との距離を調整するのに使います。

### 基本的な書き方

```css
.box {
  margin-top: 20px;      /* 上 */
  margin-right: 10px;    /* 右 */
  margin-bottom: 20px;   /* 下 */
  margin-left: 10px;     /* 左 */
}
```

### ショートハンド（省略記法）

```css
/* 上下左右すべて同じ */
margin: 20px;

/* 上下 / 左右 */
margin: 20px 10px;

/* 上 / 左右 / 下 */
margin: 20px 10px 30px;

/* 上 / 右 / 下 / 左（時計回り） */
margin: 20px 10px 30px 5px;
```

### 中央揃えのテクニック

ブロック要素を水平方向に中央揃えするには `margin: auto` を使います。

```css
.container {
  width: 800px;
  margin: 0 auto;  /* 上下は0、左右はauto（均等配分）→ 中央揃え */
}
```

> **注意：** `margin: auto` で中央揃えするには `width` の指定が必要です。

### マージンの相殺（margin collapsing）

上下に並ぶ要素のマージンは、大きい方の値に「相殺」されます。

```css
.box-a { margin-bottom: 30px; }
.box-b { margin-top: 20px; }
/* 間の余白は30px + 20px = 50pxではなく、大きい方の30pxになる */
```

> **覚えておこう：** これは上下方向のみで起こります。左右では相殺されません。

---

## 3. padding（内側の余白）

要素の内側に余白を作ります。コンテンツと枠線の間を調整します。

### 基本的な書き方

```css
.box {
  padding-top: 20px;
  padding-right: 15px;
  padding-bottom: 20px;
  padding-left: 15px;
}
```

### ショートハンド

marginと同じ記法で書けます。

```css
/* 上下左右すべて同じ */
padding: 20px;

/* 上下 / 左右 */
padding: 20px 15px;

/* 上 / 右 / 下 / 左（時計回り） */
padding: 10px 15px 20px 15px;
```

### margin と padding の違い

```css
.card {
  margin: 20px;    /* カードの外側（他の要素との距離） */
  padding: 16px;   /* カードの内側（テキストと枠の距離） */
  border: 1px solid #ddd;
  background-color: #fff;
}
```

| 比較項目 | margin | padding |
|---------|--------|---------|
| 場所 | ボックスの外側 | ボックスの内側 |
| 背景色 | 適用されない | 適用される |
| 負の値 | 使える（`margin: -10px`） | 使えない |
| 用途 | 要素間の距離調整 | 要素内のコンテンツ余白 |

---

## 4. border（枠線）

要素に枠線を付けます。

### 基本的な書き方

```css
.box {
  border: 1px solid #333;  /* 太さ 種類 色 */
}
```

### 枠線の種類

```css
border: 1px solid #333;    /* 実線 */
border: 2px dashed #333;   /* 破線 */
border: 3px dotted #333;   /* 点線 */
border: 2px double #333;   /* 二重線 */
border: none;               /* 枠線なし */
```

### 個別の辺に指定

```css
.box {
  border-top: 2px solid red;
  border-bottom: 1px solid gray;
  border-left: none;
  border-right: none;
}
```

### 角を丸くする（border-radius）

```css
.box {
  border-radius: 8px;        /* 全ての角を丸める */
  border-radius: 50%;         /* 正方形なら円になる */
  border-radius: 10px 0 10px 0;  /* 左上 右上 右下 左下 */
}
```

## 5. width と height（幅と高さ）

### 基本的な指定

```css
.box {
  width: 300px;     /* 幅を300pxに固定 */
  height: 200px;    /* 高さを200pxに固定 */
}
```

### 最大値・最小値

レスポンシブデザインで重要になります。

```css
.container {
  width: 100%;          /* 親要素の幅いっぱい */
  max-width: 1200px;    /* ただし最大は1200px */
  min-width: 320px;     /* 最小は320px */
}

.box {
  min-height: 200px;    /* 最低200pxの高さを確保 */
  /* heightは指定しない → コンテンツに応じて伸びる */
}
```

> **ベストプラクティス：** `height` は固定せず、`min-height` を使うことで、コンテンツが増えても対応できます。

## 6. box-sizing（超重要！）

### デフォルトの問題

デフォルト（`content-box`）では、`width` はコンテンツ部分のみの幅です。paddingとborderが加算されます。

```css
.box {
  width: 300px;
  padding: 20px;
  border: 5px solid #333;
  /* 実際の表示幅 = 300 + 20×2 + 5×2 = 350px！ */
}
```

### border-box で解決

`box-sizing: border-box` にすると、`width` にpaddingとborderが含まれます。

```css
.box {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 5px solid #333;
  /* 実際の表示幅 = 300px（paddingとborderを含む） */
}
```

### 全要素にborder-boxを適用（定番のリセット）

ほぼすべてのプロジェクトで使われる定番の設定です。

```css
*, *::before, *::after {
  box-sizing: border-box;
}
```

> **強く推奨：** この設定をCSSの冒頭に書く習慣をつけましょう。計算がシンプルになります。

### content-box vs border-box

```
【content-box（デフォルト）】
width: 300px / padding: 20px / border: 5px の場合

  表示幅 = 300 + 20 + 20 + 5 + 5 = 350px

  ├─5px─┤──20px──┤───300px───┤──20px──┤─5px─┤
  border  padding   content    padding  border


【border-box（推奨）】
width: 300px / padding: 20px / border: 5px の場合

  表示幅 = 300px（すべて含む）

  ├─5px─┤──20px──┤───250px───┤──20px──┤─5px─┤
  border  padding   content    padding  border
```

## 7. display プロパティ

要素の表示方法を制御する最も重要なプロパティの1つです。

### 7.1 block（ブロック要素）

```css
div { display: block; }
```

- 横幅いっぱいに広がる
- 次の要素は下に配置される（改行される）
- width / height を指定できる
- **デフォルトがblockの要素：** `<div>`, `<p>`, `<h1>`〜`<h6>`, `<ul>`, `<section>` など

### 7.2 inline（インライン要素）

```css
span { display: inline; }
```

- コンテンツの幅だけ占める
- 横に並ぶ（改行されない）
- width / height を指定できない
- 上下のmarginが効かない
- **デフォルトがinlineの要素：** `<span>`, `<a>`, `<strong>`, `<em>`, `<img>` など

### 7.3 inline-block

```css
.badge { display: inline-block; }
```

- 横に並ぶ（inlineの特徴）
- width / height を指定できる（blockの特徴）
- 両方の良いとこ取り

### 7.4 none（非表示）

```css
.hidden { display: none; }
```

- 要素が完全に消える（スペースも占めない）

### displayの比較表

| display | 横幅 | 並び方 | width/height | 上下margin |
|---------|------|--------|-------------|-----------|
| `block` | 親の幅いっぱい | 縦に並ぶ | 指定可 | 有効 |
| `inline` | コンテンツ幅 | 横に並ぶ | 指定不可 | 無効 |
| `inline-block` | コンテンツ幅 | 横に並ぶ | 指定可 | 有効 |
| `none` | ─ | 非表示 | ─ | ─ |

## 8. overflow（はみ出しの制御）

コンテンツがボックスからはみ出した場合の動作を指定します。

```css
.box {
  width: 300px;
  height: 200px;
  overflow: visible;   /* はみ出して表示（デフォルト） */
  overflow: hidden;    /* はみ出した部分を隠す */
  overflow: scroll;    /* 常にスクロールバーを表示 */
  overflow: auto;      /* 必要な時だけスクロールバーを表示 */
}
```

縦横別々に指定もできます。

```css
.box {
  overflow-x: hidden;  /* 横方向のはみ出しを隠す */
  overflow-y: auto;    /* 縦方向は必要時にスクロール */
}
```

## 9. 実践例：カードコンポーネント

ここまで学んだ知識を使って、カードを作ってみましょう。

```html
<div class="card">
  <div class="card-image">
    <img src="photo.jpg" alt="写真">
  </div>
  <div class="card-body">
    <h2 class="card-title">カードタイトル</h2>
    <p class="card-text">カードの説明文がここに入ります。ボックスモデルを活用したレイアウトです。</p>
  </div>
</div>
```

```css
*, *::before, *::after {
  box-sizing: border-box;
}

.card {
  width: 300px;
  border: 1px solid #ddd;
  border-radius: 12px;
  overflow: hidden;           /* 角丸からはみ出す画像を隠す */
  margin: 20px;
}

.card-image img {
  width: 100%;                /* 親要素の幅いっぱい */
  height: 200px;
  display: block;             /* imgのデフォルトinlineを解除 */
}

.card-body {
  padding: 16px;
}

.card-title {
  margin: 0 0 8px 0;         /* 上・右・下・左 */
  font-size: 18px;
  color: #333;
}

.card-text {
  margin: 0;
  font-size: 14px;
  color: #666;
  line-height: 1.6;
}
```

**見た目のイメージ：**
```
┌──────────────────────┐
│                      │
│       [画像]          │
│                      │
├──────────────────────┤
│ カードタイトル         │
│                      │
│ カードの説明文が       │
│ ここに入ります。       │
└──────────────────────┘
```

## まとめ

| 項目 | 要点 |
|------|------|
| ボックスモデル | content → padding → border → margin の4層構造 |
| margin | 外側の余白。`auto` で中央揃え。上下で相殺される |
| padding | 内側の余白。背景色が適用される |
| border | 枠線。`border-radius` で角丸 |
| box-sizing | `border-box` を常に使う（paddingとborderを含む計算） |
| display | `block`（縦並び）/ `inline`（横並び）/ `inline-block`（両方の特徴） |
| overflow | はみ出しの制御。`hidden` か `auto` をよく使う |
