## 問題1：typeof ガード（基本）

`string | number | boolean` のユニオン型を受け取り、型に応じた変換を行う `formatValue` 関数を実装してください。

- `string` → 大文字に変換
- `number` → 小数点2桁に整形
- `boolean` → `"はい"` / `"いいえ"`

**期待される出力：**
```
formatValue("hello") = HELLO
formatValue(3.14159) = 3.14
formatValue(true) = はい
formatValue(false) = いいえ
```

<details>
<summary>解答例</summary>

```typescript
function formatValue(value: string | number | boolean): string {
  if (typeof value === "string") {
    return value.toUpperCase();
  } else if (typeof value === "number") {
    return value.toFixed(2);
  } else {
    return value ? "はい" : "いいえ";
  }
}

console.log(`formatValue("hello") = ${formatValue("hello")}`);
console.log(`formatValue(3.14159) = ${formatValue(3.14159)}`);
console.log(`formatValue(true) = ${formatValue(true)}`);
console.log(`formatValue(false) = ${formatValue(false)}`);
```

</details>

## 問題2：instanceof ガード（基本）

カスタムエラークラスを作成し、`instanceof` で型を絞り込んでエラー処理を行ってください。

1. `HttpError` クラス：`statusCode: number` を持つ
2. `ValidationError` クラス：`fields: string[]` を持つ
3. `handleError(error: Error)` 関数：エラーの種類に応じたメッセージを返す

**期待される出力：**
```
HTTP 404: ページが見つかりません
バリデーションエラー (name, email): 入力が不正です
一般エラー: 何かがおかしい
```

<details>
<summary>解答例</summary>

```typescript
// カスタムエラークラス
class HttpError extends Error {
  constructor(
    public statusCode: number,
    message: string
  ) {
    super(message);
  }
}

class ValidationError extends Error {
  constructor(
    public fields: string[],
    message: string
  ) {
    super(message);
  }
}

// instanceofで型を絞り込む
function handleError(error: Error): string {
  if (error instanceof HttpError) {
    return `HTTP ${error.statusCode}: ${error.message}`;
  }
  if (error instanceof ValidationError) {
    return `バリデーションエラー (${error.fields.join(", ")}): ${error.message}`;
  }
  return `一般エラー: ${error.message}`;
}

// テスト
console.log(handleError(new HttpError(404, "ページが見つかりません")));
console.log(handleError(new ValidationError(["name", "email"], "入力が不正です")));
console.log(handleError(new Error("何かがおかしい")));
```

</details>

## 問題3：in ガードと判別プロパティ（基本）

支払い方法の判別共用体を定義し、`in` ガードまたは判別プロパティで型を絞り込んでください。

- `CreditCard`: `{ method: "credit"; cardNumber: string; expiryDate: string }`
- `BankTransfer`: `{ method: "bank"; accountNumber: string; bankName: string }`
- `Cash`: `{ method: "cash"; amount: number }`

**期待される出力：**
```
クレジットカード: ****-****-****-1234
銀行振込: みずほ銀行 口座 1234567
現金: ¥5,000
```

<details>
<summary>解答例</summary>

```typescript
interface CreditCard {
  method: "credit";
  cardNumber: string;
  expiryDate: string;
}

interface BankTransfer {
  method: "bank";
  accountNumber: string;
  bankName: string;
}

interface Cash {
  method: "cash";
  amount: number;
}

type Payment = CreditCard | BankTransfer | Cash;

function describePayment(payment: Payment): string {
  switch (payment.method) {
    case "credit":
      // paymentはCreditCard型に絞り込まれる
      const masked = "****-****-****-" + payment.cardNumber.slice(-4);
      return `クレジットカード: ${masked}`;
    case "bank":
      // paymentはBankTransfer型に絞り込まれる
      return `銀行振込: ${payment.bankName} 口座 ${payment.accountNumber}`;
    case "cash":
      // paymentはCash型に絞り込まれる
      return `現金: ¥${payment.amount.toLocaleString()}`;
  }
}

const payments: Payment[] = [
  { method: "credit", cardNumber: "4111111111111234", expiryDate: "12/28" },
  { method: "bank", accountNumber: "1234567", bankName: "みずほ銀行" },
  { method: "cash", amount: 5000 },
];

payments.forEach((p) => console.log(describePayment(p)));
```

</details>

## 問題4：カスタム型ガード関数（応用）

`is` 述語を使ったカスタム型ガード関数を実装してください。

1. `isString(value: unknown): value is string`
2. `isNumber(value: unknown): value is number`（NaN除外）
3. `isNonNull<T>(value: T | null | undefined): value is T`

これらを使って、配列のフィルタリングを行ってください。

**期待される出力：**
```
元の配列: [1, "hello", null, 42, undefined, "world", NaN, true]
文字列のみ: ["hello", "world"]
数値のみ: [1, 42]
null除去: [1, "hello", 42, "world", NaN, true]
```

<details>
<summary>解答例</summary>

```typescript
// カスタム型ガード関数
function isString(value: unknown): value is string {
  return typeof value === "string";
}

function isNumber(value: unknown): value is number {
  return typeof value === "number" && !Number.isNaN(value);
}

function isNonNull<T>(value: T | null | undefined): value is T {
  return value !== null && value !== undefined;
}

// テストデータ
const mixed: unknown[] = [1, "hello", null, 42, undefined, "world", NaN, true];

console.log(`元の配列: ${JSON.stringify(mixed)}`);

// 型ガードでフィルタリング
const strings = mixed.filter(isString);
console.log(`文字列のみ: ${JSON.stringify(strings)}`);
// strings の型は string[]

const numbers = mixed.filter(isNumber);
console.log(`数値のみ: ${JSON.stringify(numbers)}`);
// numbers の型は number[]

const nonNulls = mixed.filter(isNonNull);
console.log(`null除去: ${JSON.stringify(nonNulls)}`);
```

</details>

## 問題5：アサーション関数（応用）

`asserts` 述語を使ったバリデーション関数を実装してください。

1. `assertNonNull<T>(value, message?)` — null/undefinedでないことを保証
2. `assertIsUser(value)` — User型であることを保証

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}
```

**期待される出力：**
```
ユーザー: 太郎 (taro@example.com)
エラー: 値がnullです
エラー: nameは文字列でなければなりません
```

<details>
<summary>解答例</summary>

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

// assertNonNull: null/undefinedでないことを保証
function assertNonNull<T>(
  value: T | null | undefined,
  message?: string
): asserts value is T {
  if (value === null || value === undefined) {
    throw new Error(message ?? "値がnullまたはundefinedです");
  }
}

// assertIsUser: User型であることを保証
function assertIsUser(value: unknown): asserts value is User {
  if (typeof value !== "object" || value === null) {
    throw new Error("オブジェクトでなければなりません");
  }
  const obj = value as Record<string, unknown>;
  if (typeof obj.id !== "number") {
    throw new Error("idは数値でなければなりません");
  }
  if (typeof obj.name !== "string") {
    throw new Error("nameは文字列でなければなりません");
  }
  if (typeof obj.email !== "string") {
    throw new Error("emailは文字列でなければなりません");
  }
}

// テスト1: 正常なケース
try {
  const data: unknown = { id: 1, name: "太郎", email: "taro@example.com" };
  assertIsUser(data);
  // ここでdataはUser型として使える
  console.log(`ユーザー: ${data.name} (${data.email})`);
} catch (e) {
  console.log(`エラー: ${(e as Error).message}`);
}

// テスト2: nullチェック
try {
  const value: string | null = null;
  assertNonNull(value, "値がnullです");
  console.log(value.toUpperCase());
} catch (e) {
  console.log(`エラー: ${(e as Error).message}`);
}

// テスト3: 不正なデータ
try {
  const badData: unknown = { id: 1, name: 42, email: "test@test.com" };
  assertIsUser(badData);
} catch (e) {
  console.log(`エラー: ${(e as Error).message}`);
}
```

</details>

## 問題6：早期リターンによる絞り込み（応用）

ガード節（早期リターン）パターンを使って、ユーザー情報の表示関数を実装してください。

各チェックで早期リターンすることで、後続のコードでは型が自動的に絞り込まれます。

**期待される出力：**
```
太郎 <taro@example.com>（admin）
ユーザー名が未設定です
メールアドレスが未設定です
ゲストユーザーにはアクセス権がありません
```

<details>
<summary>解答例</summary>

```typescript
interface UserProfile {
  name?: string;
  email?: string;
  role?: "admin" | "user" | "guest";
}

// 早期リターンで段階的に型を絞り込む
function displayUser(user: UserProfile | null | undefined): string {
  if (!user) {
    return "ユーザー情報がありません";
  }
  // ここから先、userはUserProfile型

  if (!user.name) {
    return "ユーザー名が未設定です";
  }
  // ここから先、user.nameはstring型

  if (!user.email) {
    return "メールアドレスが未設定です";
  }
  // ここから先、user.emailはstring型

  if (user.role === "guest") {
    return "ゲストユーザーにはアクセス権がありません";
  }

  return `${user.name} <${user.email}>（${user.role ?? "不明"}）`;
}

// テスト
console.log(displayUser({ name: "太郎", email: "taro@example.com", role: "admin" }));
console.log(displayUser({ email: "test@test.com" }));
console.log(displayUser({ name: "花子" }));
console.log(displayUser({ name: "次郎", email: "jiro@test.com", role: "guest" }));
```

</details>

## 問題7：複合的な型ガード（チャレンジ）

APIから受け取ったデータを型安全に処理する関数を実装してください。

1. `ApiResponse` 型：成功/エラーの判別共用体
2. `isSuccessResponse` 型ガード関数
3. `isUserArray` 型ガード関数：`unknown` が `User[]` であることを確認
4. `processApiResponse` 関数：全ての型チェックを組み合わせてデータを処理

**期待される出力：**
```
ユーザー一覧:
  #1 太郎 (taro@example.com)
  #2 花子 (hanako@example.com)
エラー: サーバーエラー (コード: 500)
不正なデータ形式です
```

<details>
<summary>解答例</summary>

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

type ApiResponse =
  | { success: true; data: unknown }
  | { success: false; error: { code: number; message: string } };

// 型ガード: 成功レスポンスか
function isSuccessResponse(
  res: ApiResponse
): res is { success: true; data: unknown } {
  return res.success === true;
}

// 型ガード: User型か
function isUser(value: unknown): value is User {
  if (typeof value !== "object" || value === null) return false;
  const obj = value as Record<string, unknown>;
  return (
    typeof obj.id === "number" &&
    typeof obj.name === "string" &&
    typeof obj.email === "string"
  );
}

// 型ガード: User配列か
function isUserArray(value: unknown): value is User[] {
  return Array.isArray(value) && value.every(isUser);
}

// APIレスポンスの処理
function processApiResponse(response: ApiResponse): string {
  if (!isSuccessResponse(response)) {
    return `エラー: ${response.error.message} (コード: ${response.error.code})`;
  }

  if (!isUserArray(response.data)) {
    return "不正なデータ形式です";
  }

  // ここでresponse.dataはUser[]型
  const lines = response.data.map(
    (u) => `  #${u.id} ${u.name} (${u.email})`
  );
  return `ユーザー一覧:\n${lines.join("\n")}`;
}

// テスト
const successRes: ApiResponse = {
  success: true,
  data: [
    { id: 1, name: "太郎", email: "taro@example.com" },
    { id: 2, name: "花子", email: "hanako@example.com" },
  ],
};

const errorRes: ApiResponse = {
  success: false,
  error: { code: 500, message: "サーバーエラー" },
};

const badDataRes: ApiResponse = {
  success: true,
  data: "不正なデータ",
};

console.log(processApiResponse(successRes));
console.log(processApiResponse(errorRes));
console.log(processApiResponse(badDataRes));
```

</details>

## 問題8：型ガードユーティリティの作成（チャレンジ）

プロジェクト共通の型ガードユーティリティモジュールを設計してください。

以下の型ガード関数を全て実装し、テストしてください。

1. `isString`, `isNumber`, `isBoolean` — プリミティブ型ガード
2. `isNonNull<T>` — null/undefined除外
3. `hasProperty<K>(obj, key)` — プロパティ存在チェック
4. `isArrayOf<T>(arr, guard)` — 特定の型の配列であることを確認

**期待される出力：**
```
isString("hello") = true
isNumber(42) = true
isNumber(NaN) = false
hasProperty({name: "太郎"}, "name") = true
hasProperty({name: "太郎"}, "age") = false
isArrayOf([1, 2, 3], isNumber) = true
isArrayOf([1, "2", 3], isNumber) = false
```

<details>
<summary>解答例</summary>

```typescript
// プリミティブ型ガード
function isString(value: unknown): value is string {
  return typeof value === "string";
}

function isNumber(value: unknown): value is number {
  return typeof value === "number" && !Number.isNaN(value);
}

function isBoolean(value: unknown): value is boolean {
  return typeof value === "boolean";
}

// null/undefined除外
function isNonNull<T>(value: T | null | undefined): value is T {
  return value !== null && value !== undefined;
}

// プロパティ存在チェック
function hasProperty<K extends string>(
  obj: unknown,
  key: K
): obj is Record<K, unknown> {
  return typeof obj === "object" && obj !== null && key in obj;
}

// 特定型の配列チェック
function isArrayOf<T>(
  arr: unknown,
  guard: (value: unknown) => value is T
): arr is T[] {
  return Array.isArray(arr) && arr.every(guard);
}

// テスト
console.log(`isString("hello") = ${isString("hello")}`);
console.log(`isNumber(42) = ${isNumber(42)}`);
console.log(`isNumber(NaN) = ${isNumber(NaN)}`);

console.log(
  `hasProperty({name: "太郎"}, "name") = ${hasProperty({ name: "太郎" }, "name")}`
);
console.log(
  `hasProperty({name: "太郎"}, "age") = ${hasProperty({ name: "太郎" }, "age")}`
);

console.log(`isArrayOf([1, 2, 3], isNumber) = ${isArrayOf([1, 2, 3], isNumber)}`);
console.log(`isArrayOf([1, "2", 3], isNumber) = ${isArrayOf([1, "2", 3], isNumber)}`);
```

</details>
