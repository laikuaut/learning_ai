# 第8章：ネットワークセキュリティ基礎

## この章のゴール

- ファイアウォールの種類（パケットフィルタリング、ステートフル、アプリケーションゲートウェイ）を理解すること
- DMZ（非武装地帯）の設計思想と構成を説明できること
- VPN（Virtual Private Network）の種類と仕組みを理解すること
- IDS（侵入検知システム）とIPS（侵入防止システム）の違いを説明できること
- プロキシサーバの役割と種類を理解すること

## 8.1 ファイアウォール（Firewall）

### 8.1.1 ファイアウォールとは

**ファイアウォール（FW）**は、ネットワーク間の境界に設置され、通信の可否をルールに基づいて制御する仕組みです。「防火壁」の名の通り、外部からの不正な通信を遮断し、内部ネットワークを保護します。

```
【ファイアウォールの基本配置】

  インターネット                    内部ネットワーク
  (外部)                          (社内LAN)
      │                              │
      │     ┌────────────────┐      │
      │     │  ファイアウォール  │      │
      ├────→│                 │────→│  許可された通信
      │     │  ルール:         │      │
      ├──×→│  80/443 → 許可  │      │
      │     │  22 → 特定IPのみ│      │
      │     │  その他 → 拒否  │      │
      │     └────────────────┘      │
```

### 8.1.2 ファイアウォールの種類

**パケットフィルタリング型**

IPアドレス、ポート番号、プロトコル等のヘッダ情報に基づいて通信の可否を判断します。

```
【パケットフィルタリングのルール例（ACL）】

  番号  アクション  送信元IP        宛先IP          プロトコル  宛先ポート
  1     ALLOW      任意            192.168.1.10    TCP        80
  2     ALLOW      任意            192.168.1.10    TCP        443
  3     ALLOW      10.0.0.100      192.168.1.10    TCP        22
  4     DENY       任意            任意             任意       任意
                                                              ↑ 暗黙のDeny
```

| 長所 | 短所 |
|------|------|
| 高速処理 | 通信内容は検査しない |
| 設定がシンプル | アプリケーション層の攻撃は検出できない |
| 低コスト | 戻りの通信を個別に許可する必要がある |

**ステートフルパケットインスペクション（SPI）型**

通信の **状態（セッション）** を追跡し、正当な戻りの通信を自動的に許可します。現在のファイアウォールの主流です。

```
【ステートフルインスペクションの動作】

  ① 内部 → 外部: TCP SYN（接続開始）
     FWがセッションテーブルに記録
     ┌──────────────────────────────────┐
     │ 送信元: 192.168.1.10:50001       │
     │ 宛先:   93.184.216.34:80         │
     │ 状態:   SYN_SENT                 │
     └──────────────────────────────────┘

  ② 外部 → 内部: TCP SYN-ACK（応答）
     セッションテーブルと照合 → 正当な戻り → 許可
     明示的なルールなしに戻りの通信を通す

  ③ セッション終了後、テーブルから削除
```

**アプリケーションゲートウェイ型（プロキシ型）**

アプリケーション層まで通信内容を検査します。HTTPの内容を分析して不正なリクエストを遮断するなど、より高度な制御が可能です。

| 種類 | 検査対象 | 速度 | 防御レベル |
|------|---------|------|-----------|
| **パケットフィルタリング** | L3/L4ヘッダ | 高速 | 低 |
| **ステートフルインスペクション** | L3/L4 + セッション状態 | 中速 | 中 |
| **アプリケーションゲートウェイ** | L3～L7（内容含む） | 低速 | 高 |

**次世代ファイアウォール（NGFW）**

従来のファイアウォール機能に加え、以下の機能を統合したものです。

- アプリケーション識別（SNS、動画等のアプリ単位で制御）
- IPS（侵入防止機能）
- マルウェア検知
- URLフィルタリング
- SSL/TLS復号検査

## 8.2 DMZ（DeMilitarized Zone / 非武装地帯）

### 8.2.1 DMZとは

**DMZ** は、外部ネットワーク（インターネット）と内部ネットワーク（社内LAN）の間に設置される中間的なネットワークセグメントです。外部に公開するサーバ（Webサーバ、メールサーバ等）を配置します。

```
【DMZの構成（シングルファイアウォール方式）】

  インターネット
       │
  ┌────┴────┐
  │ FW       │
  │  │  │   │
  │  │  └─────→ DMZ
  │  │        │  ├── Webサーバ
  │  │        │  ├── メールサーバ
  │  │        │  └── DNSサーバ（外部公開用）
  │  │        │
  │  └────────→ 内部LAN
  │           │  ├── 社内PC
  │           │  ├── ファイルサーバ
  │           │  └── 社内DB
  └──────────┘

  通信ポリシー:
  ・インターネット → DMZ:  許可（HTTP/HTTPS等の限定ポート）
  ・インターネット → 内部: 拒否
  ・内部 → DMZ:           許可
  ・内部 → インターネット: 許可（プロキシ経由等）
  ・DMZ → 内部:           原則拒否（最小限のみ許可）
  ・DMZ → インターネット: 限定許可
```

### 8.2.2 デュアルファイアウォール方式

```
【デュアルファイアウォール方式】

  インターネット
       │
  ┌────┴────┐
  │ FW1      │  ← 外部ファイアウォール
  └────┬────┘
       │
     DMZ
   ┌──┼──┐
   Web Mail DNS
   └──┼──┘
       │
  ┌────┴────┐
  │ FW2      │  ← 内部ファイアウォール
  └────┬────┘
       │
    内部LAN

  メリット: FW1が突破されても、FW2で内部LANを保護できる
  デメリット: コストと管理の負荷が増える
```

## 8.3 VPN（Virtual Private Network）

### 8.3.1 VPNとは

**VPN（Virtual Private Network）**は、インターネットなどの公衆回線上に、暗号化された仮想的な専用線を構築する技術です。物理的な専用線に比べて低コストで、拠点間接続やリモートアクセスを安全に実現します。

```
【VPNの基本イメージ】

  物理的な接続（実際のインターネット）:
  [拠点A]───[ISP]───[インターネット]───[ISP]───[拠点B]

  論理的な接続（VPNトンネル）:
  [拠点A]═══════════ 暗号化トンネル ═══════════[拠点B]
           盗聴されても内容が解読できない
```

### 8.3.2 VPNの種類

| 種類 | 用途 | 接続形態 | 代表的なプロトコル |
|------|------|---------|------------------|
| **サイト間VPN（拠点間VPN）** | 拠点同士の接続 | ルータ対ルータ | IPsec |
| **リモートアクセスVPN** | 在宅勤務者の社内接続 | 端末対ルータ | SSL-VPN, L2TP/IPsec |
| **SSL-VPN** | Webブラウザ経由のリモートアクセス | 端末対サーバ | SSL/TLS |

### 8.3.3 IPsec（Internet Protocol Security）

IPsecはネットワーク層（L3）で動作する暗号化・認証プロトコルスイートです。

```
【IPsec の構成要素】

  ┌──────────────────────────────────────┐
  │ IPsec                                │
  │                                      │
  │  ┌────────────┐  ┌────────────┐     │
  │  │ AH          │  │ ESP         │     │
  │  │(認証のみ)    │  │(認証+暗号化) │     │
  │  └────────────┘  └────────────┘     │
  │                                      │
  │  ┌────────────┐  ┌────────────┐     │
  │  │ IKE         │  │ SA          │     │
  │  │(鍵交換)     │  │(セキュリティ │     │
  │  │             │  │ アソシエーション)│  │
  │  └────────────┘  └────────────┘     │
  └──────────────────────────────────────┘
```

| コンポーネント | 説明 |
|--------------|------|
| **AH（Authentication Header）** | 認証（改ざん検知）のみ。暗号化なし |
| **ESP（Encapsulating Security Payload）** | 認証 + 暗号化。一般的にこちらを使用 |
| **IKE（Internet Key Exchange）** | 暗号鍵の交換と SA の確立を行う |
| **SA（Security Association）** | 暗号化と認証に使うパラメータの合意 |

**IPsecのモード**

```
【トランスポートモード】
  元IPヘッダ │ ESP/AHヘッダ │ ペイロード（暗号化）│ ESPトレーラ
  ← 端末間の通信で使用。IPヘッダは暗号化しない →

【トンネルモード】
  新IPヘッダ │ ESP/AHヘッダ │ 元IPヘッダ+ペイロード（暗号化）│ ESPトレーラ
  ← 拠点間VPNで使用。元のIPヘッダごと暗号化 →
```

| モード | 暗号化範囲 | 用途 |
|--------|-----------|------|
| **トランスポートモード** | ペイロードのみ | ホスト間通信 |
| **トンネルモード** | 元のIPヘッダ+ペイロード | 拠点間VPN（一般的） |

### 8.3.4 SSL-VPN

SSL/TLS（ポート443）を使用するVPNです。Webブラウザだけでリモートアクセスが可能な手軽さが特徴です。

| 方式 | 説明 | 利点 |
|------|------|------|
| **リバースプロキシ方式** | Webアプリケーションへのアクセスのみ | 導入が容易 |
| **ポートフォワーディング方式** | 特定ポートの通信を転送 | 特定アプリケーションに対応 |
| **L2フォワーディング方式** | 全トラフィックをVPN経由 | IPsecに近い機能 |

## 8.4 IDS / IPS

### 8.4.1 IDS（Intrusion Detection System / 侵入検知システム）

ネットワークやホストの通信を監視し、不正なアクセスや攻撃を **検知してアラートを発報** するシステムです。

### 8.4.2 IPS（Intrusion Prevention System / 侵入防止システム）

IDSの検知機能に加え、不正な通信を **自動的に遮断** する機能を持つシステムです。

```
【IDSとIPSの違い】

  IDS（検知のみ）:
  攻撃者 ──→ [IDS] ──→ サーバ
                │
                └─→ 管理者にアラート通知
                    （攻撃は通過してしまう）

  IPS（検知 + 遮断）:
  攻撃者 ──→ [IPS] ──×  サーバ
                │
                └─→ 攻撃を遮断 + 管理者に通知
                    （攻撃を防御できる）
```

### 8.4.3 配置方式

| 方式 | 説明 | IDS/IPS |
|------|------|---------|
| **ネットワーク型（NIDS/NIPS）** | ネットワーク上を流れるパケットを監視 | IDS/IPS共に |
| **ホスト型（HIDS/HIPS）** | 個々のサーバやPCを監視 | IDS/IPS共に |

```
【ネットワーク型IDSの配置】

  インターネット
       │
  ┌────┴────┐
  │ FW       │
  └────┬────┘
       │
       ├──── [NIDS]  ← ミラーポートでトラフィックを監視
       │              （インラインではない = 通信を遮断しない）
       │
    内部LAN

【ネットワーク型IPSの配置】

  インターネット
       │
  ┌────┴────┐
  │ FW       │
  └────┬────┘
       │
  ┌────┴────┐
  │ NIPS     │  ← インラインに設置（通信経路上に配置）
  └────┬────┘     不正通信を直接遮断できる
       │
    内部LAN
```

### 8.4.4 検知方式

| 検知方式 | 説明 | 長所 | 短所 |
|---------|------|------|------|
| **シグネチャ型（不正検知）** | 既知の攻撃パターンとの一致で検知 | 誤検知が少ない | 未知の攻撃は検知できない |
| **アノマリ型（異常検知）** | 通常の通信パターンからの逸脱で検知 | 未知の攻撃も検知可能 | 誤検知（フォールスポジティブ）が多い |

| 用語 | 説明 |
|------|------|
| **フォールスポジティブ（偽陽性）** | 正常な通信を攻撃と誤判断 |
| **フォールスネガティブ（偽陰性）** | 攻撃を正常な通信と誤判断（見逃し） |

## 8.5 プロキシサーバ

### 8.5.1 フォワードプロキシ

クライアントの代理としてインターネットにアクセスするサーバです。

```
【フォワードプロキシ】

  社内PC ──→ [プロキシサーバ] ──→ インターネット
              │
              ├── アクセスログの記録
              ├── コンテンツキャッシュ
              ├── URLフィルタリング
              └── ウイルスチェック
```

### 8.5.2 リバースプロキシ

外部からのアクセスを受け、内部のサーバに転送するサーバです。

```
【リバースプロキシ】

  インターネット ──→ [リバースプロキシ] ──→ Webサーバ1
                      │                   ──→ Webサーバ2
                      │                   ──→ Webサーバ3
                      │
                      ├── 負荷分散（ロードバランシング）
                      ├── SSL終端（SSL Offload）
                      ├── キャッシュ
                      └── WAF機能
```

| 種類 | 用途 | 保護対象 |
|------|------|---------|
| **フォワードプロキシ** | クライアントの代理で外部接続 | 内部ユーザ |
| **リバースプロキシ** | 外部からのアクセスを内部サーバに中継 | 内部サーバ |

## 8.6 その他のセキュリティ技術

### 8.6.1 WAF（Web Application Firewall）

Webアプリケーション層（L7）の攻撃を防御する専用のファイアウォールです。SQLインジェクション、XSS等の攻撃パターンを検知・遮断します。

### 8.6.2 UTM（Unified Threat Management / 統合脅威管理）

ファイアウォール、IDS/IPS、アンチウイルス、URLフィルタリング、VPN等のセキュリティ機能を1台の機器に統合したものです。

```
【UTMの統合機能】

  ┌─────────────────────────────┐
  │           UTM               │
  │  ┌─────────┐ ┌──────────┐ │
  │  │ FW       │ │ IDS/IPS  │ │
  │  ├─────────┤ ├──────────┤ │
  │  │ アンチ   │ │ URL      │ │
  │  │ ウイルス │ │ フィルタ │ │
  │  ├─────────┤ ├──────────┤ │
  │  │ VPN      │ │ アンチ   │ │
  │  │          │ │ スパム   │ │
  │  └─────────┘ └──────────┘ │
  └─────────────────────────────┘
```

### 8.6.3 ネットワークセキュリティの全体像

```
【多層防御のネットワーク構成例】

  インターネット
       │
  [UTM / NGFW]  ← パケットフィルタリング + IPS
       │
  ┌────┴────┐
  │  DMZ     │ ← 公開サーバ（Webサーバ、メールサーバ）
  │ [WAF]    │ ← Webアプリケーション防御
  └────┬────┘
       │
  [FW（内部）]  ← 内部セグメント間の制御
       │
  ┌────┴────────────┐
  │  内部LAN          │
  │ [NIDS] ← 監視    │
  │ [プロキシ] ← Web制御│
  │ [HIDS] ← サーバ監視│
  └──────────────────┘
```

## 8.7 まとめ

### ポイントまとめ

1. **ファイアウォール** はネットワーク境界で通信を制御する基本的なセキュリティ機器です。パケットフィルタリング型、ステートフル型、アプリケーションゲートウェイ型があります
2. **DMZ** は外部公開サーバを配置する中間セグメントで、内部ネットワークへの直接アクセスを防ぎます
3. **VPN** は公衆回線上に暗号化された仮想専用線を構築します。IPsec（拠点間）とSSL-VPN（リモートアクセス）が代表的です
4. **IDS** は侵入を検知してアラートを発報、**IPS** は検知に加えて自動遮断を行います
5. **プロキシサーバ** にはフォワードプロキシ（クライアント側）とリバースプロキシ（サーバ側）があります
6. **多層防御** の考え方に基づき、FW・IDS/IPS・WAF・プロキシ等を組み合わせて防御します

### 試験対策キーワード

- ファイアウォール（パケットフィルタリング、SPI、アプリケーションゲートウェイ、NGFW）
- DMZ（非武装地帯）、デュアルファイアウォール
- VPN（IPsec、SSL-VPN）、トンネルモード、トランスポートモード
- IDS / IPS、シグネチャ型、アノマリ型、NIDS / NIPS
- フォールスポジティブ、フォールスネガティブ
- フォワードプロキシ、リバースプロキシ
- WAF、UTM

## 確認問題

1. ステートフルパケットインスペクション（SPI）がパケットフィルタリングより優れている点を説明してください。
2. DMZを設ける理由と、DMZに配置すべきサーバの例を挙げてください。
3. IPsecのトランスポートモードとトンネルモードの違いを説明してください。
4. IDSとIPSの違いを、配置方法と機能の観点から説明してください。
5. フォワードプロキシとリバースプロキシの違いを説明してください。
