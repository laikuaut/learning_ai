## 基本問題

### 問題1：複合フォームの入力と送信

以下の要件を満たすテストを作成してください。ユーザー登録フォームに対して、複数のフォーム要素を操作し、送信後の結果を検証します。

**要件：**
- テキスト入力（名前、メールアドレス）を入力する
- セレクトボックスで「東京」を選択する
- チェックボックス（利用規約に同意）をチェックする
- ラジオボタン（性別：男性）を選択する
- 送信ボタンをクリックし、「登録完了」メッセージが表示されることを確認する

**期待される結果：**
```
テスト実行後、フォームの各フィールドに値が入力され、
送信後に「登録完了」のメッセージが画面上に表示される
```

<details>
<summary>ヒント</summary>

Playwrightのフォーム操作メソッドを使い分けましょう。

```typescript
// テキスト入力
await page.getByLabel('名前').fill('テスト太郎');

// セレクトボックス
await page.getByLabel('都道府県').selectOption('tokyo');

// チェックボックス
await page.getByLabel('利用規約に同意').check();

// ラジオボタン
await page.getByLabel('男性').check();
```

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ユーザー登録フォームの入力と送信', async ({ page }) => {
  await page.goto('https://example.com/register');

  // ========================================
  // テキストフィールドの入力
  // fill() は既存の値をクリアしてから入力する
  // ========================================
  await page.getByLabel('名前').fill('テスト太郎');
  await page.getByLabel('メールアドレス').fill('test@example.com');

  // ========================================
  // セレクトボックスの選択
  // selectOption() にはvalue属性またはラベルテキストを渡せる
  // ========================================
  await page.getByLabel('都道府県').selectOption('tokyo');

  // ========================================
  // チェックボックスの操作
  // check() はすでにチェック済みでもエラーにならない
  // ========================================
  await page.getByLabel('利用規約に同意').check();

  // ========================================
  // ラジオボタンの選択
  // ラジオボタンもcheck()で操作する
  // ========================================
  await page.getByLabel('男性').check();

  // ========================================
  // フォーム送信と結果確認
  // ========================================
  await page.getByRole('button', { name: '登録' }).click();
  await expect(page.getByText('登録完了')).toBeVisible();
});
```

**ポイント：**
- `fill()` はテキスト入力に使い、既存の値を自動でクリアします
- `selectOption()` は `value` 属性、表示テキスト、インデックスのいずれかで指定できます
- `check()` / `uncheck()` はチェックボックスとラジオボタンの両方に使えます

</details>

### 問題2：alertダイアログの処理

ボタンをクリックすると `alert('保存しました')` が表示されるページに対して、ダイアログのメッセージを検証するテストを書いてください。

**要件：**
- ダイアログイベントを事前にリスンする
- ダイアログのメッセージが「保存しました」であることを確認する
- ダイアログを閉じる（accept）

**期待される結果：**
```
ダイアログのメッセージが「保存しました」と一致し、
ダイアログが正常に閉じられてテストが通る
```

<details>
<summary>ヒント</summary>

`page.on('dialog')` でダイアログイベントをリスンします。ダイアログのイベントハンドラは、ダイアログをトリガーする操作の **前に** 登録する必要があります。

```typescript
page.on('dialog', async (dialog) => {
  // dialog.message() でメッセージを取得
  await dialog.accept();
});
```

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('alertダイアログのメッセージを検証する', async ({ page }) => {
  await page.goto('https://example.com/save-page');

  // ========================================
  // ダイアログのイベントハンドラを登録
  // 重要：ダイアログを発生させる操作の「前に」登録する
  // ========================================
  let dialogMessage = '';
  page.on('dialog', async (dialog) => {
    // ダイアログの種類を確認（alert / confirm / prompt）
    expect(dialog.type()).toBe('alert');
    // メッセージを保存
    dialogMessage = dialog.message();
    // accept() でダイアログを閉じる
    await dialog.accept();
  });

  // ========================================
  // ダイアログをトリガーする操作
  // ========================================
  await page.getByRole('button', { name: '保存' }).click();

  // ========================================
  // ダイアログのメッセージを検証
  // ========================================
  expect(dialogMessage).toBe('保存しました');
});
```

**ポイント：**
- Playwrightはダイアログを自動で閉じませんので、`accept()` または `dismiss()` を明示的に呼ぶ必要があります
- ダイアログハンドラはダイアログが表示される前に登録しないと、テストがタイムアウトします

</details>

### 問題3：confirmダイアログのOK/キャンセル

削除ボタンを押すと `confirm('本当に削除しますか？')` が表示されるページに対して、以下の2つのシナリオをテストしてください。

**要件：**
- シナリオ1：OKを押した場合 → 「削除しました」と表示される
- シナリオ2：キャンセルを押した場合 → 項目がそのまま残る

**期待される結果：**
```
シナリオ1：「削除しました」メッセージが表示される
シナリオ2：対象の項目が画面上に残っている
```

<details>
<summary>ヒント</summary>

`dialog.accept()` でOK、`dialog.dismiss()` でキャンセルに対応します。`page.once()` を使うと1回だけイベントを処理できます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('confirmダイアログでOKを押すと削除される', async ({ page }) => {
  await page.goto('https://example.com/items');

  // ========================================
  // once() を使って1回だけダイアログを処理する
  // accept() でOKボタンをクリックする動作に対応
  // ========================================
  page.once('dialog', async (dialog) => {
    expect(dialog.type()).toBe('confirm');
    expect(dialog.message()).toBe('本当に削除しますか？');
    await dialog.accept();
  });

  await page.getByRole('button', { name: '削除' }).click();
  await expect(page.getByText('削除しました')).toBeVisible();
});

test('confirmダイアログでキャンセルすると削除されない', async ({ page }) => {
  await page.goto('https://example.com/items');

  // ========================================
  // dismiss() でキャンセルボタンをクリックする動作に対応
  // ========================================
  page.once('dialog', async (dialog) => {
    await dialog.dismiss();
  });

  await page.getByRole('button', { name: '削除' }).click();

  // ========================================
  // 項目がまだ画面に残っていることを確認
  // ========================================
  await expect(page.getByText('テスト項目')).toBeVisible();
});
```

**ポイント：**
- `page.once()` は一度だけイベントを処理するため、複数のダイアログがある場合は注意が必要です
- `accept()` はOK、`dismiss()` はキャンセルに対応します

</details>

### 問題4：iframe内の要素を操作する

メインページ内にiframeが埋め込まれており、iframe内にフォームがあります。iframe内のフォームを操作するテストを書いてください。

**要件：**
- `#editor-frame` というIDのiframe内にあるテキストエリアに文字を入力する
- iframe内の送信ボタンをクリックする
- iframe内に「送信完了」というメッセージが表示されることを確認する

**期待される結果：**
```
iframe内のフォームに正しく値が入力され、
送信後にiframe内に「送信完了」メッセージが表示される
```

<details>
<summary>ヒント</summary>

`page.frameLocator()` を使うとiframe内の要素にアクセスできます。

```typescript
const frame = page.frameLocator('#editor-frame');
await frame.getByRole('textbox').fill('テスト');
```

`frameLocator()` で取得したオブジェクトには、通常のLocatorと同じメソッドが使えます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('iframe内のフォームを操作する', async ({ page }) => {
  await page.goto('https://example.com/embedded-editor');

  // ========================================
  // frameLocator() でiframeの参照を取得
  // CSSセレクタでiframeを指定する
  // ========================================
  const frame = page.frameLocator('#editor-frame');

  // ========================================
  // iframe内の要素を操作
  // frameLocator経由で通常のLocatorメソッドが使える
  // ========================================
  await frame.getByRole('textbox').fill('Playwrightからの入力テスト');
  await frame.getByRole('button', { name: '送信' }).click();

  // ========================================
  // iframe内の結果を検証
  // ========================================
  await expect(frame.getByText('送信完了')).toBeVisible();
});
```

**ポイント：**
- `frameLocator()` はLocatorではなくFrameLocatorを返します
- ネストしたiframeの場合は `frameLocator().frameLocator()` のように連鎖できます
- `page.frame()` でもiframeにアクセスできますが、`frameLocator()` のほうが自動待機が効くため推奨されます

</details>

### 問題5：フォームバリデーションのテスト

お問い合わせフォームのバリデーションを検証するテストを書いてください。以下のバリデーションルールがあります。

**バリデーションルール：**
- 名前：必須、空欄の場合「名前は必須です」と表示
- メールアドレス：必須かつ形式チェック、不正な形式の場合「正しいメールアドレスを入力してください」と表示
- メッセージ：10文字以上必須、足りない場合「メッセージは10文字以上で入力してください」と表示

**要件：**
- 空のフォームを送信してバリデーションエラーを確認する
- 不正なメールアドレスで送信してエラーを確認する
- 短いメッセージで送信してエラーを確認する
- 正しい入力で送信して成功メッセージを確認する

**期待される結果：**
```
各バリデーションケースで適切なエラーメッセージが表示され、
正しい入力の場合は「送信完了」メッセージが表示される
```

<details>
<summary>ヒント</summary>

バリデーションテストでは、1つずつフィールドの条件を変えてテストするのがポイントです。`toBeVisible()` と `toHaveText()` を組み合わせて検証しましょう。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test.describe('お問い合わせフォームのバリデーション', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('https://example.com/contact');
  });

  test('空のフォームを送信するとエラーが表示される', async ({ page }) => {
    // ========================================
    // 何も入力せずに送信する
    // ========================================
    await page.getByRole('button', { name: '送信' }).click();

    // ========================================
    // 必須フィールドのエラーメッセージを確認
    // ========================================
    await expect(page.getByText('名前は必須です')).toBeVisible();
    await expect(page.getByText('正しいメールアドレスを入力してください')).toBeVisible();
    await expect(page.getByText('メッセージは10文字以上で入力してください')).toBeVisible();
  });

  test('不正なメールアドレスでエラーが表示される', async ({ page }) => {
    await page.getByLabel('名前').fill('テスト太郎');
    // ========================================
    // 不正な形式のメールアドレスを入力
    // ========================================
    await page.getByLabel('メールアドレス').fill('invalid-email');
    await page.getByLabel('メッセージ').fill('これは十分な長さのメッセージです');
    await page.getByRole('button', { name: '送信' }).click();

    await expect(page.getByText('正しいメールアドレスを入力してください')).toBeVisible();
    // 他のフィールドにはエラーが出ないことも確認
    await expect(page.getByText('名前は必須です')).not.toBeVisible();
  });

  test('短いメッセージでエラーが表示される', async ({ page }) => {
    await page.getByLabel('名前').fill('テスト太郎');
    await page.getByLabel('メールアドレス').fill('test@example.com');
    // ========================================
    // 10文字未満のメッセージを入力
    // ========================================
    await page.getByLabel('メッセージ').fill('短い');
    await page.getByRole('button', { name: '送信' }).click();

    await expect(page.getByText('メッセージは10文字以上で入力してください')).toBeVisible();
  });

  test('正しい入力で送信が成功する', async ({ page }) => {
    // ========================================
    // 全フィールドに正しい値を入力
    // ========================================
    await page.getByLabel('名前').fill('テスト太郎');
    await page.getByLabel('メールアドレス').fill('test@example.com');
    await page.getByLabel('メッセージ').fill('これはテスト用のお問い合わせメッセージです。');
    await page.getByRole('button', { name: '送信' }).click();

    await expect(page.getByText('送信完了')).toBeVisible();
  });
});
```

**ポイント：**
- `test.describe()` で関連するテストをグループ化し、`beforeEach` で共通のセットアップを行います
- バリデーションテストでは「エラーが出ないこと」も検証すると品質が上がります
- `not.toBeVisible()` で要素が表示されていないことを確認できます

</details>

### 問題6：ホバーで表示されるツールチップのテスト

アイコンにマウスをホバーするとツールチップが表示されるUIをテストしてください。

**要件：**
- ヘルプアイコン（`?` マーク）にホバーする
- ツールチップに「詳しくはヘルプページをご覧ください」というテキストが表示されることを確認する
- ホバーを外すとツールチップが消えることを確認する

**期待される結果：**
```
ホバー中にツールチップが表示され、
ホバーを外すとツールチップが非表示になる
```

<details>
<summary>ヒント</summary>

`locator.hover()` でマウスホバーを再現できます。ホバーを外すには別の要素にホバーするか、ページの空白部分にホバーします。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ホバーでツールチップが表示・非表示になる', async ({ page }) => {
  await page.goto('https://example.com/tooltip-demo');

  const helpIcon = page.getByRole('button', { name: 'ヘルプ' });
  const tooltip = page.getByRole('tooltip');

  // ========================================
  // ホバー前：ツールチップが非表示であることを確認
  // ========================================
  await expect(tooltip).not.toBeVisible();

  // ========================================
  // hover() でマウスオーバーを再現
  // ========================================
  await helpIcon.hover();

  // ========================================
  // ツールチップの表示と内容を検証
  // ========================================
  await expect(tooltip).toBeVisible();
  await expect(tooltip).toHaveText('詳しくはヘルプページをご覧ください');

  // ========================================
  // 別の要素にホバーしてツールチップを消す
  // ========================================
  await page.getByRole('heading').first().hover();
  await expect(tooltip).not.toBeVisible();
});
```

**ポイント：**
- `hover()` はマウスを要素の中央に移動させます
- ツールチップの実装方法（CSS `:hover` やJavaScript）によってはタイミングに注意が必要です
- ホバーを外すには、画面上の別の要素に `hover()` するのが確実です

</details>

## チャレンジ問題

### 問題7：ドラッグ＆ドロップのテスト

タスクボード（カンバン風UI）で、タスクカードをドラッグして別のカラムに移動するテストを書いてください。

**要件：**
- 「TODO」カラムにあるタスク「レビュー依頼」を「In Progress」カラムにドラッグ＆ドロップする
- 移動後、「In Progress」カラムに「レビュー依頼」が含まれていることを確認する
- 「TODO」カラムから「レビュー依頼」が消えていることを確認する

**期待される結果：**
```
「レビュー依頼」タスクがTODOカラムからIn Progressカラムに移動し、
各カラムの内容が正しく更新される
```

<details>
<summary>ヒント</summary>

Playwrightには `dragTo()` メソッドがあります。ドラッグ元の要素からドラッグ先の要素に対して呼び出します。

```typescript
await source.dragTo(target);
```

HTML5のDrag and Drop APIを使っている場合は、`dispatchEvent` を使う方法もあります。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('タスクカードをドラッグ＆ドロップで移動する', async ({ page }) => {
  await page.goto('https://example.com/kanban');

  // ========================================
  // ドラッグ元：TODOカラム内の「レビュー依頼」カード
  // ========================================
  const taskCard = page.locator('[data-column="todo"]')
    .getByText('レビュー依頼');

  // ========================================
  // ドラッグ先：In Progressカラム
  // ========================================
  const targetColumn = page.locator('[data-column="in-progress"]');

  // ========================================
  // dragTo() でドラッグ＆ドロップを実行
  // 内部的にはmousedown → mousemove → mouseupが順に実行される
  // ========================================
  await taskCard.dragTo(targetColumn);

  // ========================================
  // 移動後の検証
  // ========================================
  // In Progressカラムに移動していることを確認
  await expect(
    page.locator('[data-column="in-progress"]').getByText('レビュー依頼')
  ).toBeVisible();

  // TODOカラムから消えていることを確認
  await expect(
    page.locator('[data-column="todo"]').getByText('レビュー依頼')
  ).not.toBeVisible();
});
```

**ポイント：**
- `dragTo()` は自動待機が効くため、要素が表示されるまで待ってから操作します
- 一部のライブラリ（React DnDなど）では `dragTo()` がうまく動かない場合があり、その際はマウスイベントを手動でディスパッチする必要があります
- `{ force: true }` オプションで強制的にドラッグすることも可能です

</details>

### 問題8：ファイルダウンロードの検証

「レポートをダウンロード」ボタンをクリックするとCSVファイルがダウンロードされるページで、ダウンロードされたファイルの名前と内容を検証するテストを書いてください。

**要件：**
- ダウンロードイベントを待ち受ける
- ダウンロードされたファイル名が `report.csv` であることを確認する
- ファイルの中身に「名前,メール,部署」というヘッダー行が含まれることを確認する

**期待される結果：**
```
ファイル名: report.csv
ファイル内容の先頭行: "名前,メール,部署"
```

<details>
<summary>ヒント</summary>

`page.waitForEvent('download')` でダウンロードイベントを待ち受けます。ダウンロードしたファイルは `download.path()` で一時ファイルのパスを取得し、`fs` モジュールで読み取ることができます。

```typescript
const [download] = await Promise.all([
  page.waitForEvent('download'),
  page.getByRole('button', { name: 'ダウンロード' }).click(),
]);
```

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';
import fs from 'fs';

test('CSVファイルのダウンロードを検証する', async ({ page }) => {
  await page.goto('https://example.com/reports');

  // ========================================
  // Promise.all でダウンロードイベントとクリックを同時に待つ
  // waitForEvent はクリックの「前に」呼び出す必要がある
  // ========================================
  const [download] = await Promise.all([
    page.waitForEvent('download'),
    page.getByRole('button', { name: 'レポートをダウンロード' }).click(),
  ]);

  // ========================================
  // ファイル名の検証
  // suggestedFilename() でサーバーが提示したファイル名を取得
  // ========================================
  expect(download.suggestedFilename()).toBe('report.csv');

  // ========================================
  // ファイル内容の検証
  // path() で一時保存されたファイルのパスを取得
  // ========================================
  const filePath = await download.path();
  const content = fs.readFileSync(filePath!, 'utf-8');

  // ヘッダー行が含まれていることを確認
  expect(content).toContain('名前,メール,部署');

  // ========================================
  // 特定のパスにファイルを保存する場合
  // ========================================
  // await download.saveAs('./downloads/report.csv');
});
```

**ポイント：**
- `download.suggestedFilename()` はContent-Dispositionヘッダーから取得したファイル名を返します
- `download.path()` は一時ディレクトリに保存されたファイルのパスを返します（テスト終了後に削除されます）
- `download.saveAs()` で任意のパスにファイルを保存できます

</details>

### 問題9：promptダイアログへの入力

「名前を変更」ボタンを押すと `prompt('新しい名前を入力してください', '現在の名前')` が表示されるページで、ダイアログに新しい名前を入力して反映されることを検証してください。

**要件：**
- promptダイアログに「山田花子」と入力してOKを押す
- 画面上の名前表示が「山田花子」に更新されることを確認する
- キャンセルした場合は名前が変わらないことも確認する

**期待される結果：**
```
シナリオ1：promptにOKで応答後、表示名が「山田花子」に変わる
シナリオ2：promptをキャンセル後、表示名は元のまま
```

<details>
<summary>ヒント</summary>

`dialog.accept('入力値')` でpromptダイアログに値を入力してOKを押せます。`dialog.defaultValue()` でデフォルト値を取得できます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('promptダイアログに入力して名前を変更する', async ({ page }) => {
  await page.goto('https://example.com/profile');

  // ========================================
  // promptダイアログのハンドラを登録
  // accept() に文字列を渡すとpromptへの入力値になる
  // ========================================
  page.once('dialog', async (dialog) => {
    expect(dialog.type()).toBe('prompt');
    // デフォルト値を確認
    expect(dialog.defaultValue()).toBe('現在の名前');
    // 新しい名前を入力してOK
    await dialog.accept('山田花子');
  });

  await page.getByRole('button', { name: '名前を変更' }).click();

  // ========================================
  // 名前が更新されたことを確認
  // ========================================
  await expect(page.getByTestId('display-name')).toHaveText('山田花子');
});

test('promptダイアログをキャンセルすると名前は変わらない', async ({ page }) => {
  await page.goto('https://example.com/profile');

  // ========================================
  // dismiss() でキャンセルする
  // ========================================
  page.once('dialog', async (dialog) => {
    await dialog.dismiss();
  });

  await page.getByRole('button', { name: '名前を変更' }).click();

  // ========================================
  // 名前が元のまま変わらないことを確認
  // ========================================
  await expect(page.getByTestId('display-name')).toHaveText('現在の名前');
});
```

**ポイント：**
- `dialog.accept(value)` でpromptに値を入力できます（alert/confirmでは引数不要）
- `dialog.defaultValue()` はpromptの第2引数で指定されたデフォルト値を返します
- promptのテストでは「OKで入力」「キャンセル」「空文字でOK」の3パターンをテストするのが理想的です

</details>

### 問題10：日付ピッカーの操作

日付選択UIで特定の日付を選択し、選択結果が正しく反映されることを検証するテストを書いてください。

**要件：**
- `input[type="date"]` フィールドに「2026-04-15」を設定する
- カレンダーUIから日付を選択するパターンも試す
- 選択した日付が「2026年4月15日」と表示されることを確認する

**期待される結果：**
```
日付フィールドに2026-04-15が設定され、
フォーマットされた表示が「2026年4月15日」になる
```

<details>
<summary>ヒント</summary>

`input[type="date"]` には `fill()` で `YYYY-MM-DD` 形式の値を直接入力できます。カスタムの日付ピッカーUIの場合は、月の移動ボタンと日付セルのクリックで操作します。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('input[type="date"]に直接値を設定する', async ({ page }) => {
  await page.goto('https://example.com/date-form');

  // ========================================
  // fill() で日付を直接入力する方法
  // ブラウザネイティブのdate inputは YYYY-MM-DD 形式を受け付ける
  // ========================================
  await page.getByLabel('予約日').fill('2026-04-15');

  // 表示が正しいことを確認
  await expect(page.getByTestId('formatted-date')).toHaveText('2026年4月15日');
});

test('カスタム日付ピッカーから日付を選択する', async ({ page }) => {
  await page.goto('https://example.com/date-picker');

  // ========================================
  // カレンダーアイコンをクリックしてピッカーを開く
  // ========================================
  await page.getByRole('button', { name: 'カレンダーを開く' }).click();

  // ========================================
  // 目的の月まで移動する
  // 「次の月」ボタンを必要な回数クリック
  // ========================================
  const monthLabel = page.getByTestId('calendar-month');
  while (await monthLabel.textContent() !== '2026年4月') {
    await page.getByRole('button', { name: '次の月' }).click();
  }

  // ========================================
  // 日付セルをクリックして選択
  // ========================================
  await page.getByRole('gridcell', { name: '15' }).click();

  // ========================================
  // 選択結果の検証
  // ========================================
  await expect(page.getByTestId('formatted-date')).toHaveText('2026年4月15日');
});
```

**ポイント：**
- ネイティブの `input[type="date"]` には `fill('YYYY-MM-DD')` が最もシンプルです
- カスタム日付ピッカーはライブラリごとにDOM構造が異なるため、実際のHTML構造に合わせてセレクタを調整してください
- `while` ループで月を移動する場合は、無限ループを避けるためにカウンタや上限を設けることをお勧めします

</details>
