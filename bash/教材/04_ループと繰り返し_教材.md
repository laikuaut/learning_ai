# 第4章：ループと繰り返し

## この章のゴール

- for、while、untilの3種類のループ構文を使い分けられるようになること
- ファイルの行読み取りループをマスターすること
- breakとcontinueで制御フローを柔軟に操れるようになること
- select文で対話的メニューを作れるようになること
- ループとパイプ・プロセス置換を組み合わせた実践パターンを習得すること

---

## 4.1 forループ

### リストを使ったforループ

```bash
#!/usr/bin/env bash

# 基本的な for ループ
for name in Alice Bob Charlie; do
    echo "Hello, $name!"
done
```

実行結果：
```
Hello, Alice!
Hello, Bob!
Hello, Charlie!
```

### さまざまなリストの指定方法

```bash
#!/usr/bin/env bash

# ブレース展開
for i in {1..5}; do
    echo "番号: $i"
done
# 出力: 番号: 1 〜 番号: 5

# ステップ付きブレース展開
for i in {0..10..2}; do
    echo "偶数: $i"
done
# 出力: 偶数: 0, 2, 4, 6, 8, 10

# グロブパターン（ファイル一覧）
for file in *.sh; do
    echo "スクリプト: $file"
done

# コマンドの出力をリストに
for user in $(cut -d: -f1 /etc/passwd | head -5); do
    echo "ユーザー: $user"
done
```

### C言語スタイルのforループ

```bash
#!/usr/bin/env bash

# C言語スタイル（(( )) を使う）
for ((i = 0; i < 5; i++)); do
    echo "カウント: $i"
done

# 複数の変数を同時に操作
for ((i = 0, j = 10; i < 5; i++, j -= 2)); do
    echo "i=$i, j=$j"
done
```

### 実践例：ファイルの一括処理

```bash
#!/usr/bin/env bash

# JPEG ファイルの一括リネーム
count=1
for file in *.jpg; do
    [[ -f "$file" ]] || continue  # ファイルが存在しない場合はスキップ
    new_name=$(printf "photo_%03d.jpg" "$count")
    echo "リネーム: $file → $new_name"
    # mv "$file" "$new_name"  # 実際にリネームする場合はコメント解除
    ((count++))
done
```

### ポイントまとめ

- `for var in リスト` が基本形
- `{1..10}` でブレース展開、`{0..100..5}` でステップ指定
- `for ((i=0; i<N; i++))` でC言語スタイルのカウントループ
- グロブ `*.txt` でファイル一覧を回せる

## 4.2 whileループ

### 基本構文

```bash
#!/usr/bin/env bash

# カウンタを使った while ループ
count=1
while ((count <= 5)); do
    echo "カウント: $count"
    ((count++))
done
```

### ファイルの行読み取り（最重要パターン）

```bash
#!/usr/bin/env bash

# ファイルを1行ずつ読み取る（最も使うパターン）
while IFS= read -r line; do
    echo "行: $line"
done < /etc/hosts
```

> **解説**:
> - `IFS=` : 行頭・行末の空白を保持
> - `read -r` : バックスラッシュをエスケープとして扱わない
> - `< ファイル` : ファイルをリダイレクトで入力

### さまざまな入力パターン

```bash
#!/usr/bin/env bash

# パイプからの入力
echo -e "apple\nbanana\ncherry" | while IFS= read -r fruit; do
    echo "フルーツ: $fruit"
done

# CSVの処理（区切り文字を指定）
echo "Tanaka,30,Tokyo
Suzuki,25,Osaka
Sato,35,Nagoya" | while IFS=',' read -r name age city; do
    echo "$name さん（$age歳）は $city 在住"
done

# コマンド出力を行ごとに処理
while IFS= read -r proc; do
    echo "プロセス: $proc"
done < <(ps aux | head -5)
```

### 無限ループ

```bash
#!/usr/bin/env bash

# 無限ループ（while true）
while true; do
    read -p "コマンドを入力（quit で終了）: " cmd
    [[ "$cmd" = "quit" ]] && break
    echo "実行: $cmd"
done

# 別の書き方
while :; do
    echo "ループ中..."
    sleep 1
    break  # 1回で終了（デモ用）
done
```

### ポイントまとめ

- `while IFS= read -r line; do ... done < file` がファイル読み取りの定番
- `while true` または `while :` で無限ループ
- パイプからの入力は `while read` で行ごとに処理

## 4.3 untilループ

untilループは、条件が**偽の間**繰り返します（whileの逆）。

```bash
#!/usr/bin/env bash

# untilループ（条件が真になるまで繰り返す）
count=1
until ((count > 5)); do
    echo "カウント: $count"
    ((count++))
done
```

### 実践例：サービスの起動待ち

```bash
#!/usr/bin/env bash

# サーバーが応答するまで待つ
max_attempts=10
attempt=1

until curl -s http://localhost:8080/health > /dev/null 2>&1; do
    echo "サーバー起動待ち... ($attempt/$max_attempts)"
    if ((attempt >= max_attempts)); then
        echo "タイムアウト: サーバーが起動しませんでした" >&2
        exit 1
    fi
    ((attempt++))
    sleep 2
done

echo "サーバーが起動しました！"
```

### ポイントまとめ

- `until` は条件が真になるまで繰り返す（whileの逆）
- サービスの起動待ちやリトライ処理に向いている

## 4.4 break と continue

### break：ループを抜ける

```bash
#!/usr/bin/env bash

# 特定の条件でループを終了
for i in {1..100}; do
    if ((i * i > 50)); then
        echo "$i の二乗が50を超えました"
        break
    fi
    echo "$i の二乗 = $((i * i))"
done
```

### continue：次のイテレーションへ

```bash
#!/usr/bin/env bash

# 奇数だけを表示
for i in {1..10}; do
    ((i % 2 == 0)) && continue  # 偶数はスキップ
    echo "奇数: $i"
done
```

### ネストしたループでのbreak

```bash
#!/usr/bin/env bash

# break N で N 段階外側のループを抜ける
for i in {1..3}; do
    for j in {1..3}; do
        echo "i=$i, j=$j"
        if ((i == 2 && j == 2)); then
            echo "外側のループも終了"
            break 2  # 2つ外側のループまで抜ける
        fi
    done
done
```

### ポイントまとめ

- `break` でループを即座に終了
- `continue` で残りの処理をスキップし次のイテレーションへ
- `break N` でN段階外のループまで抜けられる

## 4.5 select文

select文は、対話的なメニューを簡単に作れるBash独自の構文です。

```bash
#!/usr/bin/env bash

echo "好きなプログラミング言語を選んでください:"
select lang in Python JavaScript TypeScript Bash "終了"; do
    case "$lang" in
        Python)
            echo "Pythonはデータサイエンスに強いです"
            ;;
        JavaScript)
            echo "JavaScriptはWeb開発の必須言語です"
            ;;
        TypeScript)
            echo "TypeScriptは型安全なJavaScriptです"
            ;;
        Bash)
            echo "Bashはシステム管理に欠かせません"
            ;;
        "終了")
            echo "終了します"
            break
            ;;
        *)
            echo "無効な選択です。もう一度選んでください"
            ;;
    esac
done
```

実行例：
```
好きなプログラミング言語を選んでください:
1) Python
2) JavaScript
3) TypeScript
4) Bash
5) 終了
#? 1
Pythonはデータサイエンスに強いです
#? 5
終了します
```

### PS3でプロンプトをカスタマイズ

```bash
#!/usr/bin/env bash

PS3="番号を選択してください: "
select option in "ファイル一覧" "ディスク使用量" "プロセス一覧" "終了"; do
    case "$REPLY" in
        1) ls -la ;;
        2) df -h ;;
        3) ps aux | head -10 ;;
        4) break ;;
        *) echo "1〜4の番号を入力してください" ;;
    esac
done
```

### ポイントまとめ

- `select` は番号付きメニューを自動生成
- `PS3` でプロンプト文字列をカスタマイズ
- `$REPLY` で入力された番号を取得
- `break` でメニューを終了

## 4.6 ループとパイプ

### パイプラインでのループ

```bash
#!/usr/bin/env bash

# パイプでループに入力
find /tmp -name "*.log" -mtime +7 2>/dev/null | while IFS= read -r file; do
    echo "古いログ: $file"
done
```

> **注意**: パイプの右側のwhileループはサブシェルで実行されるため、ループ内で設定した変数はループの外では参照できません。

```bash
#!/usr/bin/env bash

# パイプ版（サブシェル問題あり）
count=0
echo -e "a\nb\nc" | while IFS= read -r line; do
    ((count++))
done
echo "カウント: $count"  # → 0（サブシェルなので反映されない）

# プロセス置換版（問題なし）
count=0
while IFS= read -r line; do
    ((count++))
done < <(echo -e "a\nb\nc")
echo "カウント: $count"  # → 3（正しい）
```

### プロセス置換（Process Substitution）

```bash
#!/usr/bin/env bash

# <() でコマンドの出力をファイルとして扱う
while IFS= read -r line; do
    echo "行: $line"
done < <(ls -la /tmp)

# 2つのファイル（コマンド出力）を比較
diff <(sort file1.txt) <(sort file2.txt)
```

### ポイントまとめ

- パイプの右側はサブシェルになるので変数が保持されない
- `< <(command)` のプロセス置換で回避できる
- ファイル読み取りにはプロセス置換を使うのが安全

## 4.7 実践例

### ファイル一括処理

```bash
#!/usr/bin/env bash
# 指定ディレクトリ内の全テキストファイルの行数を集計

target_dir="${1:-.}"
total_lines=0

for file in "$target_dir"/*.txt; do
    [[ -f "$file" ]] || continue
    lines=$(wc -l < "$file")
    echo "$file: $lines 行"
    ((total_lines += lines))
done

echo "---"
echo "合計: $total_lines 行"
```

### リトライ処理

```bash
#!/usr/bin/env bash
# コマンドを最大N回リトライする

max_retries=3
retry_delay=2

for ((i = 1; i <= max_retries; i++)); do
    echo "試行 $i/$max_retries..."
    if curl -s --max-time 5 http://example.com > /dev/null 2>&1; then
        echo "成功しました！"
        break
    fi
    if ((i < max_retries)); then
        echo "失敗。${retry_delay}秒後にリトライします..."
        sleep "$retry_delay"
        ((retry_delay *= 2))  # 指数バックオフ
    else
        echo "最大リトライ回数に達しました" >&2
        exit 1
    fi
done
```

### 進捗表示

```bash
#!/usr/bin/env bash
# シンプルなプログレスバー

total=50
for ((i = 1; i <= total; i++)); do
    # プログレスバーの計算
    percent=$((i * 100 / total))
    filled=$((i * 40 / total))
    empty=$((40 - filled))

    # \r でカーソルを行頭に戻して上書き
    printf "\r["
    printf "%0.s#" $(seq 1 $filled)
    ((empty > 0)) && printf "%0.s-" $(seq 1 $empty)
    printf "] %3d%%" "$percent"

    sleep 0.05  # 処理の代わり
done
echo ""  # 改行
echo "完了！"
```

### ポイントまとめ

- ファイル一括処理はforループ + グロブパターンが定番
- リトライは指数バックオフ（待ち時間を倍々に）が実務的
- `printf "\r"` で進捗バーを上書き表示

## 4.8 章全体のまとめ

| 構文 | 用途 | 基本形 |
|------|------|--------|
| `for` | リストの反復 | `for i in リスト; do ... done` |
| `for (())` | カウントループ | `for ((i=0; i<N; i++)); do ... done` |
| `while` | 条件が真の間 | `while 条件; do ... done` |
| `while read` | ファイル行読み取り | `while IFS= read -r line; do ... done < file` |
| `until` | 条件が真になるまで | `until 条件; do ... done` |
| `select` | メニュー表示 | `select var in リスト; do ... done` |
| `break` | ループ脱出 | ループ内で `break` |
| `continue` | 次のイテレーション | ループ内で `continue` |
