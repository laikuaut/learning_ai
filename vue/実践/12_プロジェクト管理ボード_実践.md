## 課題の要件

### 必須機能

1. **3カラムのカンバンボード**: TODO / 進行中 / 完了
2. **タスク追加**: タイトル・説明・優先度・担当者を設定するモーダル
3. **ステータス移動**: ボタンクリックでタスクを次/前のカラムに移動
4. **タスク削除**: 個別削除（確認ダイアログ付き）
5. **タスク編集**: ダブルクリックでタイトルをインライン編集
6. **統計表示**: 各カラムのタスク数、全体の進捗率
7. **フィルタ**: 担当者・優先度でフィルタリング
8. **ダーク/ライトモード**: テーマ切替

### 設計要件

- **composable** を2つ以上使用する（`useBoard`、`useTheme`）
- **provide / inject** でテーマとフィルタ設定を共有する
- コンポーネントは最低7つに分割する
- `computed` を4つ以上使用する（フィルタ、カラムごとのタスク、統計、進捗率）

```js
function useBoard() {
  const tasks = ref([...初期データ])

  function addTask(taskData) {
    tasks.value.push({ ...taskData, id: Date.now(), status: "todo" })
  }

  function moveTask(id, direction) {
    const statuses = ["todo", "inProgress", "done"]
    const task = tasks.value.find(t => t.id === id)
    if (!task) return
    const idx = statuses.indexOf(task.status) + direction
    if (idx >= 0 && idx < statuses.length) {
      task.status = statuses[idx]
    }
  }

  function deleteTask(id) {
    tasks.value = tasks.value.filter(t => t.id !== id)
  }

  function updateTask(id, changes) {
    const task = tasks.value.find(t => t.id === id)
    if (task) Object.assign(task, changes)
  }

  return { tasks, addTask, moveTask, deleteTask, updateTask }
}
```

<details>
<summary>設計ポイントの解説</summary>

### 1. composable の設計

| composable | 責務 |
|---|---|
| `useTheme` | ダーク/ライト切替、localStorage永続化 |
| `useBoard` | タスクのCRUD操作 |
| `useFilter` | フィルタ状態管理（担当者・優先度） |

### 2. computed チェーン

```
tasks（生データ）
  ↓ useFilter のフィルタ適用
filteredTasks
  ↓ getColumnTasks でステータス分類
columnTasks（カラムごとのタスク）
  ↓ StatsBar で集計
統計情報（total, doneCount, percent）
```

`computed` が数珠つなぎで依存しており、どこかが変わると下流が自動で再計算されます。

### 3. コンポーネント間通信のパターン

| パターン | 使用箇所 |
|---|---|
| **props (親→子)** | BoardColumn → TaskCard（task） |
| **emit (子→親)** | TaskCard → BoardColumn → App（move, delete, update） |
| **provide/inject** | App → FilterBar, AppHeader（theme, filter） |

### 4. よくある設計判断

**Q: タスクの状態管理をPiniaにすべきか？**

CDN版では`useBoard` composable + `provide/inject` で十分です。Piniaを使う場合はVite環境でのビルドが前提となります。本課題の規模なら composable で管理する方がシンプルです。

**Q: TaskCard の `confirm()` は setup と methods どちらに書くべきか？**

`confirm()` はブラウザのネイティブダイアログで、Vue のリアクティビティとは関係ないため、`methods` に書いても `setup` に書いても動作します。本例ではテンプレートからの `this.$emit` を使うため `methods` に書いています。
