# 第5章 演習：キーマッピング

---

## 演習の目標

この演習では、以下のスキルを確認します。

- `map` と `noremap` の違いを理解している
- `nnoremap`、`inoremap`、`vnoremap` を使い分けられる
- Leader キーを設定し、Leader キーを使ったマッピングを作成できる
- 実用的なキーマッピング（`jj` で Esc、ウィンドウ移動、バッファ切替など）を設定できる
- `:verbose map` で既存のマッピングを調査できる

---

## 基本問題

### 問題5-1：map と noremap の違い

以下の `.vimrc` の設定を読み、それぞれの動作を説明してください。

```vim
" 設定A
nmap j gj
nmap gj j

" 設定B
nnoremap j gj
nnoremap gj j
```

(1) 設定Aで `j` を押すとどうなりますか。
(2) 設定Bで `j` を押すとどうなりますか。
(3) なぜ `noremap` を使うべきなのか、理由を説明してください。

**期待される出力例：**

```
(1) 設定Aでは j → gj → j → gj → ... と無限ループに陥る。
    map は再帰的にマッピングを展開するため。

(2) 設定Bでは j は表示行単位の移動（gj の本来の動作）になる。
    noremap は再帰的な展開をしないため、gj のオリジナルの機能が呼ばれる。

(3) noremap を使うことで、他のマッピングの影響を受けず、
    常にVim本来のコマンドにマッピングできるため。
    意図しない再帰やマッピングの連鎖を防ぐことができる。
```

<details>
<summary>ヒント</summary>

- `map` は**再帰的（recursive）**に展開されます。つまり、マッピング先がさらにマッピングされていれば、それも展開されます
- `noremap` は**非再帰的（non-recursive）**です。マッピング先をそのままVim本来のコマンドとして解釈します
- 通常は `noremap` 系（`nnoremap`、`inoremap`、`vnoremap`）を使うのがベストプラクティスです

</details>

<details>
<summary>解答例</summary>

```
(1) 設定Aでは j → gj → j → gj → ... と無限ループに陥る。
    nmap は再帰的マッピングのため、j が gj に展開された後、
    gj の中の j がさらに gj に展開され、無限ループになる。

(2) 設定Bでは j を押すと、Vimオリジナルの gj（表示行単位の下移動）が
    実行される。nnoremap は非再帰的なので、マッピング先の gj は
    そのままVim本来のコマンドとして実行される。

(3) noremap を使うべき理由：
    - 他のマッピングやプラグインの影響を受けない
    - 意図しない無限ループを防止できる
    - 動作が予測可能で安全
    - map を使うのはプラグインの <Plug> マッピングを利用する場合など、
      再帰展開が必要な特殊なケースに限られる
```

**ポイント：** `.vimrc` では原則として `noremap` 系を使い、`map` 系は `<Plug>` マッピングなど特別な理由がある場合にのみ使用します。

</details>

### 問題5-2：モード別マッピングの作成

以下の要件を満たすキーマッピングを `.vimrc` に記述してください。

1. ノーマルモードで `<Space>w` を押すとファイルを保存する（`:w<CR>`）
2. インサートモードで `jj` を押すとノーマルモードに戻る
3. ビジュアルモードで選択範囲を `J`（Shift+j） で1行下に移動、`K`（Shift+k）で1行上に移動する

**期待される出力例：**

```vim
" 1. ノーマルモード：Space + w でファイル保存
nnoremap <Space>w :w<CR>

" 2. インサートモード：jj でノーマルモードに戻る
inoremap jj <Esc>

" 3. ビジュアルモード：選択行を上下に移動
vnoremap J :m '>+1<CR>gv=gv
vnoremap K :m '<-2<CR>gv=gv
```

<details>
<summary>ヒント</summary>

- ノーマルモード専用のマッピングには `nnoremap` を使います
- インサートモード専用のマッピングには `inoremap` を使います
- ビジュアルモード専用のマッピングには `vnoremap` を使います
- `<CR>` は Enter キーを意味します
- `<Esc>` は Escape キーを意味します
- ビジュアルモードの行移動では `:m` コマンド（move）を使います

</details>

<details>
<summary>解答例</summary>

```vim
" 1. ノーマルモードでSpaceキー + w を押すとファイルを保存
" <Space> はスペースキーを表す特殊表記
" :w はファイル保存コマンド、<CR> は Enter キーで実行確定
nnoremap <Space>w :w<CR>

" 2. インサートモードで jj を素早く押すと Esc と同じ効果
" ホームポジションから手を動かさずにノーマルモードに戻れる
" j を1回押してから少し待つと通常の j が入力される
inoremap jj <Esc>

" 3. ビジュアルモードで選択行を上下に移動
" J で選択範囲を1行下に移動
"   :m '>+1  → 選択範囲の末尾('>')の1行下に移動
"   <CR>     → コマンド実行
"   gv       → 直前の選択範囲を再選択
"   =        → 自動インデント
"   gv       → 再度選択状態に戻す
vnoremap J :m '>+1<CR>gv=gv
" K で選択範囲を1行上に移動
"   :m '<-2  → 選択範囲の先頭('<')の2行上に移動
vnoremap K :m '<-2<CR>gv=gv
```

</details>

### 問題5-3：Leader キーの設定

以下の要件を満たす Leader キーの設定とマッピングを記述してください。

1. Leader キーをスペースキーに設定する
2. `<Leader>q` で現在のバッファを閉じる（`:q<CR>`）
3. `<Leader>s` でファイルを保存する（`:w<CR>`）
4. `<Leader>x` で保存して閉じる（`:wq<CR>`）

**期待される出力例：**

```vim
let mapleader = "\<Space>"

nnoremap <Leader>q :q<CR>
nnoremap <Leader>s :w<CR>
nnoremap <Leader>x :wq<CR>
```

<details>
<summary>ヒント</summary>

- `mapleader` 変数に Leader キーとして使うキーを設定します
- スペースキーを設定する場合は `"\<Space>"` と記述します
- Leader キーの設定は、マッピングの定義よりも**前**に記述する必要があります
- `<Leader>` はマッピング定義時に `mapleader` の値に置き換えられます

</details>

<details>
<summary>解答例</summary>

```vim
" Leader キーをスペースキーに設定
" デフォルトの Leader キーは \ (バックスラッシュ) だが、
" スペースキーは押しやすい位置にあるため、Leader に設定されることが多い
" 重要：この設定はすべての <Leader> マッピングよりも前に記述すること
let mapleader = "\<Space>"

" <Leader>q（Space + q）で現在のバッファを閉じる
nnoremap <Leader>q :q<CR>

" <Leader>s（Space + s）でファイルを保存
nnoremap <Leader>s :w<CR>

" <Leader>x（Space + x）で保存して閉じる
nnoremap <Leader>x :wq<CR>
```

**注意点：**

- `let mapleader` はマッピング定義よりも前に記述しないと、デフォルトの `\` が使われてしまいます
- `<Leader>` はマッピング定義時に展開されるため、後から `mapleader` を変更しても既存のマッピングには影響しません

</details>

## 応用問題

### 問題5-4：ウィンドウ操作のマッピング

Vimのウィンドウ（分割画面）操作を効率化するマッピングを作成してください。以下の要件を満たしてください。

1. `Ctrl+h/j/k/l` でウィンドウ間を移動する（デフォルトの `Ctrl+w h/j/k/l` の省略版）
2. `<Leader>v` で垂直分割（`:vsplit<CR>`）、`<Leader>h` で水平分割（`:split<CR>`）
3. `<Leader>=` でウィンドウサイズを均等にする

**期待される出力例：**

```vim
" ウィンドウ間移動
nnoremap <C-h> <C-w>h
nnoremap <C-j> <C-w>j
nnoremap <C-k> <C-w>k
nnoremap <C-l> <C-w>l

" ウィンドウ分割
nnoremap <Leader>v :vsplit<CR>
nnoremap <Leader>h :split<CR>

" ウィンドウサイズ均等化
nnoremap <Leader>= <C-w>=
```

<details>
<summary>ヒント</summary>

- `<C-h>` は Ctrl+h を意味する特殊表記です
- `<C-w>` はウィンドウ操作のプレフィックスキーです
- `<C-w>h` は左のウィンドウに移動、`<C-w>j` は下のウィンドウに移動、のように方向に対応しています
- `<C-w>=` はすべてのウィンドウを均等なサイズにします

</details>

<details>
<summary>解答例</summary>

```vim
" ウィンドウ間の移動を Ctrl + hjkl で行う
" デフォルトでは Ctrl+w を押してから方向キーを押す必要があるが、
" このマッピングにより1ステップで移動できる
nnoremap <C-h> <C-w>h    " 左のウィンドウに移動
nnoremap <C-j> <C-w>j    " 下のウィンドウに移動
nnoremap <C-k> <C-w>k    " 上のウィンドウに移動
nnoremap <C-l> <C-w>l    " 右のウィンドウに移動

" ウィンドウ分割のショートカット
nnoremap <Leader>v :vsplit<CR>    " 垂直分割（左右に分割）
nnoremap <Leader>h :split<CR>     " 水平分割（上下に分割）

" ウィンドウサイズの均等化
" 分割を繰り返すとサイズがばらつくため、一発で均等にできると便利
nnoremap <Leader>= <C-w>=
```

**補足：** ターミナルによっては `<C-h>` が Backspace として認識される場合があります。その場合は以下の設定を `.vimrc` に追加してください。

```vim
" ターミナルで Ctrl+h が Backspace になる問題の対策
if has('unix')
  nnoremap <BS> <C-w>h
endif
```

</details>

### 問題5-5：バッファ操作のマッピング

複数ファイルを同時に扱うためのバッファ操作マッピングを作成してください。

1. `<Leader>bn` で次のバッファに移動する
2. `<Leader>bp` で前のバッファに移動する
3. `<Leader>bd` で現在のバッファを閉じる（ウィンドウは閉じない）
4. `<Leader>bl` でバッファ一覧を表示する

**期待される出力例：**

```vim
nnoremap <Leader>bn :bnext<CR>
nnoremap <Leader>bp :bprevious<CR>
nnoremap <Leader>bd :bdelete<CR>
nnoremap <Leader>bl :ls<CR>
```

<details>
<summary>ヒント</summary>

- `:bnext` は次のバッファ、`:bprevious` は前のバッファに切り替えます
- `:bdelete` はバッファを削除しますが、ウィンドウ自体は閉じません
- `:ls` はバッファ一覧を表示します（`:buffers` と同じ）
- バッファ番号を指定して切り替えるには `:buffer N`（N はバッファ番号）を使います

</details>

<details>
<summary>解答例</summary>

```vim
" バッファ操作のマッピング
" バッファとは、Vimに読み込まれたファイルのこと
" 複数ファイルを開いている場合にバッファ間を移動して編集する

" 次のバッファに移動
nnoremap <Leader>bn :bnext<CR>

" 前のバッファに移動
nnoremap <Leader>bp :bprevious<CR>

" 現在のバッファを閉じる（ウィンドウは維持）
" :quit との違い：bdelete はバッファリストから削除するが、
" ウィンドウ自体は閉じない（他のバッファが表示される）
nnoremap <Leader>bd :bdelete<CR>

" バッファ一覧を表示
" % は現在のバッファ、# は直前のバッファを示す
nnoremap <Leader>bl :ls<CR>
```

**よくある改良版：** `Tab` と `Shift+Tab` でバッファを切り替える設定も人気があります。

```vim
nnoremap <Tab> :bnext<CR>
nnoremap <S-Tab> :bprevious<CR>
```

</details>

### 問題5-6：verbose map でマッピングの調査

以下の操作を Vim のコマンドラインで実行し、結果の読み方を説明してください。

1. ノーマルモードのすべてのマッピングを一覧表示するコマンドを書いてください
2. `<Leader>s` に割り当てられたマッピングの詳細（定義元ファイル含む）を確認するコマンドを書いてください
3. インサートモードの `jj` マッピングを確認するコマンドを書いてください

**期待される出力例：**

```vim
" 1. ノーマルモードのすべてのマッピングを表示
:nmap

" 2. <Leader>s のマッピングを詳細表示（定義元ファイルも表示）
:verbose nmap <Leader>s

" 3. インサートモードの jj マッピングを確認
:verbose imap jj
```

**出力の読み方の例：**

```
n  <Space>s     :w<CR>
        Last set from ~/.vimrc line 42
```

- `n` はノーマルモードのマッピング
- `<Space>s` がキー入力
- `:w<CR>` がマッピング先の動作
- `Last set from ~/.vimrc line 42` は定義元のファイルと行番号

<details>
<summary>ヒント</summary>

- `:nmap` はノーマルモードのマッピング一覧を表示します
- `:imap` はインサートモードのマッピング一覧を表示します
- `:verbose` を前に付けると、そのマッピングがどのファイルの何行目で定義されたかが表示されます
- プラグインが設定したマッピングとの競合を調べる際に非常に役立ちます

</details>

<details>
<summary>解答例</summary>

```vim
" 1. ノーマルモードのマッピング一覧
"    :nmap  → ノーマルモードの全マッピング（再帰的なものも含む）
"    :map   → ノーマル + ビジュアル + オペレータ待機モードの全マッピング
:nmap

" 2. 特定のマッピングの詳細を表示
"    :verbose を付けると、どのファイルの何行目で定義されたか表示される
"    プラグインとの競合調査に非常に便利
:verbose nmap <Leader>s

" 3. インサートモードの特定マッピングを確認
:verbose imap jj
```

**各モードのマッピング確認コマンド一覧：**

```
┌──────────────────────┬──────────────────────┐
│ コマンド             │ 表示対象モード       │
├──────────────────────┼──────────────────────┤
│ :nmap                │ ノーマルモード       │
│ :imap                │ インサートモード     │
│ :vmap                │ ビジュアルモード     │
│ :cmap                │ コマンドラインモード │
│ :map                 │ ノーマル+ビジュアル  │
│                      │ +オペレータ待機      │
└──────────────────────┴──────────────────────┘
```

**トラブルシューティングの例：** マッピングが期待通りに動作しない場合、`:verbose nmap <キー>` で確認すると、プラグインが上書きしているケースが見つかることがあります。

</details>

## チャレンジ問題

### 問題5-7：実用的なキーマッピング集の作成

以下の要件をすべて満たすキーマッピングのセットを `.vimrc` に記述してください。

1. **Leader キー**をスペースに設定
2. **検索ハイライトの解除**：`<Leader><CR>` で検索ハイライトを消す
3. **行の移動**：ノーマルモードで `Alt+j` で現在行を1行下に移動、`Alt+k` で1行上に移動
4. **ヤンクの改善**：`Y` を行末までのヤンクにする（`D` や `C` と同じ挙動に統一）
5. **画面中央維持**：`n`/`N`（検索の次/前）で画面を中央に保つ
6. **インデント維持**：ビジュアルモードで `<` / `>` を押してもビジュアル選択を解除しない

**期待される出力例：**

```vim
" Leader キー設定
let mapleader = "\<Space>"

" 検索ハイライト解除
nnoremap <Leader><CR> :nohlsearch<CR>

" ノーマルモードで行の上下移動
nnoremap <A-j> :m .+1<CR>==
nnoremap <A-k> :m .-2<CR>==

" Y を行末までのヤンクに変更（D, C との一貫性）
nnoremap Y y$

" 検索時に画面中央を維持
nnoremap n nzzzv
nnoremap N Nzzzv

" ビジュアルモードでインデント変更後も選択を維持
vnoremap < <gv
vnoremap > >gv
```

<details>
<summary>ヒント</summary>

- `:nohlsearch` は検索ハイライトを一時的に解除するコマンドです（`:noh` と省略可能）
- `:m .+1` は現在行を1行下に移動するコマンドです（`.` は現在行を意味します）
- `==` は自動インデントを適用します
- `zz` は現在行を画面中央に移動します。`zv` は折りたたみを開きます
- `gv` は直前のビジュアル選択範囲を再選択します
- `<A-j>` は Alt+j を意味します（macOS のターミナルでは追加設定が必要な場合があります）

</details>

<details>
<summary>解答例</summary>

```vim
" ==============================================================================
" 実用的なキーマッピング集
" ==============================================================================

" --- Leader キー ---
" スペースキーを Leader に設定（最も押しやすいキー）
let mapleader = "\<Space>"

" --- 検索ハイライト解除 ---
" 検索後にハイライトが残り続けて邪魔な場合に Space + Enter で消す
" :nohlsearch は検索パターン自体は保持するので、n で再度ハイライトされる
nnoremap <Leader><CR> :nohlsearch<CR>

" --- ノーマルモードで行の上下移動 ---
" Alt+j で現在行を1行下に移動
"   :m .+1  → 現在行(.)を1行下(+1)に移動
"   ==      → 移動後の自動インデント調整
nnoremap <A-j> :m .+1<CR>==
" Alt+k で現在行を1行上に移動
"   :m .-2  → 現在行(.)を2行上(-2)に移動（-1だと現在位置のまま）
nnoremap <A-k> :m .-2<CR>==

" --- Y の挙動を D, C と統一 ---
" デフォルトの Y は yy（行全体ヤンク）と同じ動作で、
" D（行末まで削除）や C（行末まで変更）と一貫性がない
" y$ にマッピングすることで、カーソル位置から行末までヤンクする
nnoremap Y y$

" --- 検索時に画面中央を維持 ---
" n（次の検索結果）の後に zz（画面中央）と zv（折りたたみを開く）
" 検索結果が画面の端に表示されて見失うことを防ぐ
nnoremap n nzzzv
nnoremap N Nzzzv

" --- ビジュアルモードでインデント変更後も選択を維持 ---
" デフォルトでは < や > でインデントを変更するとビジュアル選択が解除される
" gv を付けることで選択範囲を維持し、連続してインデント操作ができる
vnoremap < <gv
vnoremap > >gv
```

**macOS での Alt キーに関する注意：**

macOS のターミナル（Terminal.app）では Alt キーが特殊文字の入力に使われるため、`<A-j>` が動作しないことがあります。iTerm2 を使用している場合は、Preferences → Profiles → Keys で「Left Option Key」を「Esc+」に設定してください。

</details>
