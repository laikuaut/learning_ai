# 実践課題08：デバッグ演習 ─ 設定ファイルの誤り修正 ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第1章〜第7章（環境構築、基本操作、ファイル操作、Git連携、MCP、フック、CLAUDE.md）
> **課題の種類**: デバッグ
> **学習目標**: Claude Codeの設定ファイル（settings.json、CLAUDE.md、フック設定）に含まれるバグを発見・修正する力を養う

---

## 課題の説明

以下のプロジェクトには **10個のバグ** が各種設定ファイルに埋め込まれています。
各ファイルの内容を読み解き、すべてのバグを見つけて修正してください。

### 進め方

1. 下のバグ入りファイルをすべてプロジェクトに配置する
2. Claude Codeを起動してエラーや問題を確認する
3. 1つずつバグを修正し、動作を検証する
4. すべての設定が正常に機能すれば完了

## 解答例

<details>
<summary>全10個のバグと修正方法</summary>

### バグ1：JSONキー名のタイプミス ─ `comand` → `command`

```json
// 修正前
"comand": "npx",
// 修正後
"command": "npx",
```

### バグ2：空のGitHubトークン

```json
// 修正前
"GITHUB_PERSONAL_ACCESS_TOKEN": ""
// 修正後 - 実際のトークンを設定するか、settings.local.jsonに移動
"GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxxxxxxxx"
```

なお、トークンをsettings.jsonに直接書くのはセキュリティ上のリスクがあります。`.claude/settings.local.json`に移動し、`.gitignore`に追加するのがベストプラクティスです。

### バグ3：filesystemサーバーにパスの引数がない

```json
// 修正前
"args": ["-y", "@modelcontextprotocol/server-filesystem"]
// 修正後 - アクセスを許可するディレクトリパスを指定
"args": ["-y", "@modelcontextprotocol/server-filesystem", "."]
```

filesystemサーバーにはアクセスを許可するディレクトリパスの指定が必要です。

### バグ4：PostToolUseフックのmatcherにWriteが含まれていない

```json
// 修正前
"matcher": "Edit",
// 修正後
"matcher": "Edit|Write",
```

Writeツールで新規ファイルを作成した場合もリントすべきです。

### バグ5：リンターエラー時にフック全体が失敗する

```json
// 修正前
"command": "cd $CLAUDE_PROJECT_DIR && npx eslint $CLAUDE_FILE_PATH"
// 修正後
"command": "cd \"$CLAUDE_PROJECT_DIR\" && npx eslint \"$CLAUDE_FILE_PATH\" 2>&1 || true"
```

`|| true` がないと、リンターがエラーを検出した際にフック自体が失敗ステータスになります。また `2>&1` で標準エラー出力もClaude Codeにフィードバックします。

### バグ6：PreToolUseフックの終了コードが間違っている

```json
// 修正前 - exit 0 は「許可」を意味する
"command": "if echo $CLAUDE_TOOL_INPUT | grep -q 'rm -rf /'; then exit 0; fi"
// 修正後 - exit 2 で「ブロック」を意味する
"command": "if echo \"$CLAUDE_TOOL_INPUT\" | grep -q 'rm -rf /'; then echo 'BLOCKED: 危険なコマンド' >&2; exit 2; fi"
```

PreToolUseフックでは、終了コード `0` は「実行を許可」、`2` は「実行をブロック」です。元のコードでは危険なコマンドを許可してしまいます。

### バグ7：存在しないフックタイプ `OnStop`

```json
// 修正前
"OnStop": [...]
// 修正後
"Stop": [...]
```

正しいフックタイプ名は `Stop` です。`OnStop` というタイプは存在しません。

### バグ8：TypeScriptのコーディング規約がsnake_caseになっている

```markdown
<!-- 修正前 -->
- すべての変数名はsnake_caseにしてください

<!-- 修正後 -->
- 変数名・関数名はcamelCaseにしてください
- クラス名・型名はPascalCaseにしてください
- 定数はUPPER_SNAKE_CASEにしてください
```

TypeScript/JavaScriptの標準的な命名規約はcamelCaseです。snake_caseはPythonの慣例です。

### バグ9：テストディレクトリ名の誤り

```markdown
<!-- 修正前 -->
- テストファイルは __tests__ ディレクトリに配置

<!-- 修正後 -->
- テストファイルは tests/ ディレクトリに配置（Jestの設定に合わせる）
```

package.jsonのJest設定や一般的なNode.jsプロジェクトでは `tests/` や `__tests__/` を使いますが、プロジェクト内の実態と一致させる必要があります（ここでは一般的な `tests/` に合わせます）。

### バグ10：PostToolUseフックのファイルパスに引用符がない

```json
// 修正前
"command": "cd $CLAUDE_PROJECT_DIR && npx eslint $CLAUDE_FILE_PATH"
// 修正後
"command": "cd \"$CLAUDE_PROJECT_DIR\" && npx eslint \"$CLAUDE_FILE_PATH\" 2>&1 || true"
```

ファイルパスにスペースが含まれる場合、引用符がないとコマンドが正しく動作しません。バグ5の修正と合わせて対応します。
