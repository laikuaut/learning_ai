## 4.1 VLAN（Virtual LAN）

### 4.1.1 VLANとは

**VLAN（仮想LAN / Virtual LAN）** は、物理的なネットワーク構成に関わらず、論理的にLANを分割する技術です。

```
【VLANなし（物理構成=論理構成）】
SW──PC1(営業)
  ├─PC2(営業)
  ├─PC3(開発)        ← 全員同じブロードキャストドメイン
  └─PC4(開発)

【VLANあり（論理的に分割）】
SW──PC1(営業)  ─┐
  ├─PC2(営業)  ─┤ VLAN10（営業部）
  ├─PC3(開発)  ─┐
  └─PC4(開発)  ─┤ VLAN20（開発部）

  → ブロードキャストドメインが分割される
  → VLAN間通信にはルータ（L3スイッチ）が必要
```

### 4.1.2 VLANの種類

| 種類 | 分割基準 | 特徴 |
|------|---------|------|
| ポートVLAN | スイッチのポート番号 | 最も一般的、設定が簡単 |
| タグVLAN(IEEE 802.1Q) | フレームにタグを付与 | スイッチ間でVLAN情報を伝達 |
| MACアドレスVLAN | MACアドレス | ポート移動に柔軟 |
| プロトコルVLAN | プロトコル種別 | IPv4/IPv6の分離等 |

### 4.1.3 IEEE 802.1Q（タグVLAN）

スイッチ間を接続する**トランクポート**で、フレームにVLAN IDタグを付与します。

```
元のイーサネットフレーム:
┌──────┬──────┬──────┬────────┬─────┐
│宛先MAC│送元MAC│ Type │ データ  │ FCS │
└──────┴──────┴──────┴────────┴─────┘

802.1Qタグ付きフレーム:
┌──────┬──────┬────────┬──────┬────────┬─────┐
│宛先MAC│送元MAC│802.1Qタグ│ Type │ データ  │ FCS │
└──────┴──────┴────────┴──────┴────────┴─────┘
                 │
                 ├── TPID(0x8100): タグ付きフレームの識別
                 ├── PCP: 優先度（0-7）
                 ├── DEI: 破棄適格
                 └── VID: VLAN ID（0-4095、実質4094個）
```

**アクセスポートとトランクポート:**
- **アクセスポート:** 1つのVLANに所属、端末を接続
- **トランクポート:** 複数のVLANのフレームを伝送、スイッチ間接続

**ポイントまとめ:**
- VLANはブロードキャストドメインの論理的な分割技術
- 802.1QタグのVLAN IDは12ビット（最大4094個）
- VLAN間通信にはL3スイッチまたはルータが必要

## 4.2 STP（スパニングツリープロトコル）

### 4.2.1 STPの必要性

スイッチ間の冗長経路があると**ブロードキャストストーム**が発生します。

```
【ループ構成】
  SW1 ──── SW2
   │  ╲  ╱  │
   │   ╲╱   │
   │   ╱╲   │
   │  ╱  ╲  │
  SW3 ──── SW4

  ブロードキャストフレームが永久にループし続ける
  → ネットワークが麻痺（ブロードキャストストーム）
```

### 4.2.2 STPの動作（IEEE 802.1D）

STPはループを検出し、冗長経路を**論理的にブロック**します。

```
【STPによるループ除去】
  SW1(ルートブリッジ)
   │ RP        │ RP
  SW2         SW3
   │ DP        │ BP（ブロック）
  ─┴───────────┴─

  RP: ルートポート（ルートブリッジへの最短経路のポート）
  DP: 指定ポート（セグメント上でルートに最も近いポート）
  BP: ブロッキングポート（ループ防止で無効化）
```

**STPの動作手順:**
1. **ルートブリッジの選出:** ブリッジIDが最小のスイッチ
2. **ルートポートの決定:** 各スイッチでルートブリッジへの最短経路ポート
3. **指定ポートの決定:** 各セグメントでルートに最も近いポート
4. **ブロッキングポート:** 上記以外のポートをブロック

### 4.2.3 STPの改良版

| プロトコル | 特徴 | 収束時間 |
|-----------|------|---------|
| STP (802.1D) | 基本、収束が遅い | 30-50秒 |
| RSTP (802.1w) | 高速収束 | 1-2秒 |
| MSTP (802.1s) | VLAN毎に異なるツリー | 1-2秒 |

**ポイントまとめ:**
- STPはL2ネットワークのループ防止プロトコル
- ルートブリッジ→ルートポート→指定ポート→ブロッキングポートの順で決定
- RSTPはSTPの高速版で収束時間が大幅に短縮

## 4.3 ルーティングプロトコル

### 4.3.1 ルーティングの基本

```
┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐
│ LAN1 │───│  R1  │───│  R2  │───│ LAN2 │
└──────┘   └──┬───┘   └───┬──┘   └──────┘
               │           │
             ┌─┴───────────┴─┐
             │      R3       │
             └───────┬───────┘
                     │
               ┌─────┴─────┐
               │   LAN3    │
               └───────────┘

ルーティング: パケットをどの経路で転送するか決定する
```

| 分類 | 説明 |
|------|------|
| 静的ルーティング | 管理者が手動でルーティングテーブルを設定 |
| 動的ルーティング | ルーティングプロトコルで自動的に経路を学習 |

### 4.3.2 OSPF（Open Shortest Path First）

**リンクステート型**のルーティングプロトコルで、IGP（Interior Gateway Protocol）の代表です。

**特徴:**
- ダイクストラ法（SPF: Shortest Path First）で最短経路を計算
- エリア分割による階層的ルーティング
- コンバージェンス（収束）が高速
- コストベースの経路選択

```
【OSPFエリア構成】
        ┌──────────────────────┐
        │     エリア0（バックボーン）│
        │  R1 ──── R2 ──── R3 │
        └──┬────────┬──────┬──┘
           │        │      │
      ┌────┴───┐┌───┴───┐┌─┴──────┐
      │エリア1 ││エリア2 ││エリア3  │
      │        ││       ││        │
      └────────┘└───────┘└────────┘

  ABR(エリア境界ルータ): エリア間を接続
  バックボーンエリア(エリア0): 全エリアが接続必須
```

**OSPFのコスト計算:**
```
コスト = 基準帯域幅 / インタフェース帯域幅

例: 基準帯域幅=100Mbps
  100Mbps回線: コスト = 100/100 = 1
  10Mbps回線:  コスト = 100/10 = 10
  1Gbps回線:   コスト = 100/1000 = 0.1 → 1（最小1）
```

### 4.3.3 BGP（Border Gateway Protocol）

**パスベクタ型**のルーティングプロトコルで、EGP（Exterior Gateway Protocol）の代表です。インターネット上のAS（Autonomous System: 自律システム）間のルーティングに使用されます。

```
┌─────────┐     eBGP      ┌─────────┐
│  AS100  │───────────────│  AS200  │
│ (ISP A) │               │ (ISP B) │
│  iBGP   │               │  iBGP   │
└────┬────┘               └────┬────┘
     │         eBGP            │
     └──────────┬──────────────┘
           ┌────┴────┐
           │  AS300  │
           │ (ISP C) │
           └─────────┘

eBGP: AS間のBGP接続
iBGP: AS内部のBGP接続
```

**BGPの特徴:**
- AS番号（16ビットまたは32ビット）で識別
- パス属性（AS_PATH等）に基づく経路選択
- ポリシーベースのルーティング（経済的・政治的な判断も可能）

### 4.3.4 ルーティングプロトコルの比較

| 項目 | RIP | OSPF | BGP |
|------|-----|------|-----|
| 種別 | IGP | IGP | EGP |
| アルゴリズム | 距離ベクタ | リンクステート | パスベクタ |
| メトリック | ホップ数(最大15) | コスト | パス属性 |
| 収束速度 | 遅い | 速い | 中程度 |
| 規模 | 小規模 | 中〜大規模 | インターネット規模 |

**ポイントまとめ:**
- OSPFはIGPのリンクステート型、BGPはEGPのパスベクタ型
- OSPFはエリア構成とコスト計算が試験で問われる
- RIPのホップ数上限15は午前の定番

## 4.4 IPv6

### 4.4.1 IPv6アドレス

```
IPv4: 32ビット  → 約43億アドレス（枯渇済み）
IPv6: 128ビット → 約340澗(かん)アドレス（事実上無限）
```

**IPv6アドレスの表記:**
```
完全表記: 2001:0db8:0000:0000:0000:0000:0000:0001
省略表記: 2001:db8::1  （連続する0のグループを::で省略）
```

### 4.4.2 IPv6アドレスの種類

| 種類 | 説明 | プレフィックス例 |
|------|------|----------------|
| グローバルユニキャスト | インターネット上で一意 | 2000::/3 |
| リンクローカル | 同一リンク内のみ有効 | fe80::/10 |
| ユニークローカル | 組織内で使用 | fc00::/7 |
| マルチキャスト | 複数宛先への同報通信 | ff00::/8 |
| エニーキャスト | 最も近い宛先に配送 | （特定のプレフィックスなし） |

**注意:** IPv6には**ブロードキャスト**がありません。代わりにマルチキャストを使用します。

### 4.4.3 IPv6の主要な機能

| 機能 | 説明 |
|------|------|
| SLAAC | ステートレスアドレス自動設定（DHCPサーバ不要） |
| NDP | 近隣探索プロトコル（ARPの代替、ルータ発見等） |
| IPsec | 標準でサポート（IPv4ではオプション） |
| フローラベル | QoSのためのトラフィック識別 |

**SLAAC（Stateless Address Autoconfiguration）:**
```
1. リンクローカルアドレスを自動生成（MACアドレスベース or ランダム）
2. ルータ広告（RA）からプレフィックス情報を取得
3. プレフィックス + インタフェースID = グローバルアドレス

例:
  プレフィックス: 2001:db8:1::/64（RAから取得）
  インタフェースID: ::1a2b:3c4d:5e6f（自動生成）
  → 2001:db8:1::1a2b:3c4d:5e6f
```

### 4.4.4 IPv4からIPv6への移行技術

| 技術 | 説明 |
|------|------|
| デュアルスタック | IPv4/IPv6の両方を同時に動作 |
| トンネリング | IPv6パケットをIPv4パケットで包んで転送 |
| トランスレーション（NAT64） | IPv4とIPv6間のアドレス変換 |

**ポイントまとめ:**
- IPv6は128ビット、ブロードキャストなし
- リンクローカル(fe80::)とグローバルユニキャスト(2001::)を区別する
- SLAACはDHCP不要のアドレス自動設定

## 4.5 SDN（Software-Defined Networking）

### 4.5.1 SDNとは

**SDN（ソフトウェア定義ネットワーキング）** は、ネットワーク機器の**制御プレーン**と**データプレーン**を分離し、ソフトウェアで集中管理する技術です。

```
【従来のネットワーク】           【SDN】
┌──────────┐                ┌──────────────┐
│ ルータ/SW │                │ SDNコントローラ │ ← 制御プレーン
│┌────────┐│                └──────┬───────┘
││制御機能 ││                       │ OpenFlow等
│├────────┤│                ┌──────┼───────┐
││転送機能 ││                │      │       │
│└────────┘│              ┌─┴─┐ ┌─┴─┐ ┌─┴─┐
└──────────┘              │SW1│ │SW2│ │SW3│ ← データプレーン
                          └───┘ └───┘ └───┘
制御と転送が一体            制御と転送を分離
```

### 4.5.2 OpenFlow

SDNを実現する代表的なプロトコルです。

**構成要素:**
- **OpenFlowコントローラ:** ネットワーク全体のフロー制御を集中管理
- **OpenFlowスイッチ:** コントローラの指示に従いパケットを転送
- **フローテーブル:** パケットの転送ルールの集合

```
フローテーブルのエントリ:
┌──────────┬──────────┬──────────┐
│マッチフィールド│ カウンタ   │ アクション  │
│(条件)     │(統計情報) │(動作)     │
├──────────┼──────────┼──────────┤
│送元IP=10.0.0.1│ 1523パケット│ ポート2に転送│
│宛先ポート=80  │          │ キューイング │
└──────────┴──────────┴──────────┘
```

### 4.5.3 NFV（Network Functions Virtualization）

ルータ・ファイアウォール・ロードバランサなどの**ネットワーク機能を仮想化**し、汎用サーバ上で実行する技術です。

| 項目 | 従来 | NFV |
|------|------|-----|
| ハードウェア | 専用アプライアンス | 汎用サーバ |
| コスト | 高い | 低い |
| 柔軟性 | 低い（物理機器の増設が必要） | 高い（VM/コンテナで追加） |
| 管理 | 機器ごとに個別管理 | 集中管理 |

**ポイントまとめ:**
- SDNは制御プレーンとデータプレーンの分離が核心
- OpenFlowはフローテーブルに基づくパケット転送
- NFVはネットワーク機能の仮想化

## 4.6 QoS（Quality of Service）

### 4.6.1 QoSとは

**QoS（サービス品質）** は、ネットワーク上のトラフィックに優先度をつけ、重要な通信の品質を保証する技術です。

### 4.6.2 QoSの指標

| 指標 | 説明 | 許容値の例（VoIP） |
|------|------|-------------------|
| 帯域幅 | 利用可能な通信速度 | 80kbps以上/通話 |
| 遅延（レイテンシ） | パケットの到着時間 | 150ms以下 |
| ジッタ | 遅延の揺らぎ | 30ms以下 |
| パケットロス | パケットの損失率 | 1%以下 |

### 4.6.3 QoSの実現方式

| 方式 | 説明 |
|------|------|
| IntServ（統合サービス） | RSVPプロトコルで帯域を事前予約。個別フロー保証だがスケーラビリティ低 |
| DiffServ（差別化サービス） | パケットのDSCPフィールドでクラス分け。スケーラビリティ高 |
| ベストエフォート | QoS保証なし。通常のインターネット通信 |

**DiffServの動作:**
```
パケット分類 → マーキング(DSCP設定) → キューイング → スケジューリング

優先度の例:
  EF(Expedited Forwarding): 音声通信（最優先）
  AF(Assured Forwarding): 業務データ（保証あり）
  BE(Best Effort): 一般トラフィック（保証なし）
```

### 4.6.4 トラフィック制御技術

| 技術 | 説明 |
|------|------|
| シェーピング | トラフィックを一定速度に平滑化（バッファに蓄積） |
| ポリシング | 超過トラフィックを即座に破棄/マーキング |
| キューイング | 優先度に基づくキュー管理（WFQ, PQ, CQ等） |

**ポイントまとめ:**
- QoSの4指標（帯域幅、遅延、ジッタ、パケットロス）を覚える
- IntServ vs DiffServの違いが午前で問われる
- VoIPやテレビ会議はQoS要件が厳しい

## 4.7 ネットワーク設計

### 4.7.1 ネットワークの階層設計

```
┌──────────────────────────────────┐
│         コア層（Core Layer）       │  高速バックボーン
│         L3スイッチ / ルータ       │  高速転送、冗長性
├──────────────────────────────────┤
│   ディストリビューション層         │  ルーティング、ACL
│   (Distribution Layer)           │  VLAN間ルーティング
│   L3スイッチ                     │  ポリシー制御
├──────────────────────────────────┤
│      アクセス層（Access Layer）    │  端末の接続
│      L2スイッチ                  │  ポートセキュリティ
│                                  │  VLAN割当
└──────────────────────────────────┘
```

### 4.7.2 冗長化技術

| 技術 | 対象 | 説明 |
|------|------|------|
| VRRP/HSRP | デフォルトGW | 仮想IPアドレスでGWを冗長化 |
| リンクアグリゲーション | 物理リンク | 複数リンクを束ねて帯域拡大＋冗長化 |
| STP/RSTP | L2スイッチ | ループ防止＋冗長経路の確保 |

**VRRP（Virtual Router Redundancy Protocol）:**
```
        仮想IP: 192.168.1.1
       ┌───────────┐
       │           │
  ┌────┴───┐  ┌───┴────┐
  │マスタR1 │  │バックアップR2│
  │優先度200│  │優先度100│
  └────────┘  └────────┘
       │           │
  ─────┴───────────┴─────
         端末群
   (デフォルトGW: 192.168.1.1)

  マスタ障害時 → バックアップが仮想IPを引き継ぐ
```

### 4.7.3 帯域計算

**例題:**
1,000人の社員がいるオフィスで、同時利用率30%、1人あたり平均2Mbps必要な場合の必要帯域:

```
同時利用ユーザ数 = 1,000 × 0.3 = 300人
必要帯域 = 300 × 2Mbps = 600Mbps
余裕率1.5倍 → 600 × 1.5 = 900Mbps
→ 1Gbps回線が必要
```
