## 基本問題

### 問題1：テストレベルの分類

以下のテスト項目が、どのテストレベル（単体/結合/システム/受入）に該当するか分類してください。

1. ログイン関数にメールアドレスとパスワードを渡し、正しいJWTが返ることを確認する
2. 注文APIを呼んだ後、在庫数が正しく減算されていることを確認する
3. 100人が同時にログインした場合の応答時間が3秒以内であることを確認する
4. 経理部門の担当者が、月次の勤怠集計レポートを出力し、内容が正しいことを確認する
5. パスワードハッシュ化関数がbcryptで正しくハッシュを生成することを確認する
6. フロントエンドからAPIを経由してDBにデータが保存されることを確認する
7. 不正なSQLを入力してもSQLインジェクションが発生しないことを確認する
8. 顧客が実際の業務データを使って一連の業務フローを実行し、問題ないことを確認する

**期待される出力例：**

```
1. 単体テスト - 個別の関数の動作を検証している
2. 結合テスト - 注文APIと在庫管理の連携を検証している
```

<details>
<summary>ヒント</summary>

- 単体テスト：個々の関数・メソッドの動作
- 結合テスト：モジュール間の連携
- システムテスト：システム全体の品質（性能、セキュリティ含む）
- 受入テスト：顧客視点での業務要件の検証

</details>

<details>
<summary>解答例</summary>

```
1. 単体テスト
   理由：ログイン関数という個別の関数の入出力を検証している。
   他のモジュール（DB等）はモックで代替可能。

2. 結合テスト
   理由：注文API（注文モジュール）と在庫サービス（在庫モジュール）
   の連携が正しく動作するかを検証している。

3. システムテスト（性能テスト）
   理由：システム全体の性能要件（同時接続100人、応答3秒以内）
   を検証している。個別の機能ではなく非機能要件のテスト。

4. 受入テスト
   理由：実際のエンドユーザー（経理部門）が業務要件を
   満たしているかを検証している。

5. 単体テスト
   理由：パスワードハッシュ化関数という個別の関数の
   動作を検証している。

6. 結合テスト
   理由：フロントエンド → API → DBという複数レイヤーの
   連携（エンドツーエンドの一部）を検証している。

7. システムテスト（セキュリティテスト）
   理由：SQLインジェクションというセキュリティ上の脆弱性が
   ないことをシステムレベルで検証している。

8. 受入テスト
   理由：顧客が実際の業務データを使い、業務フロー全体が
   問題なく動作することを確認している。
```

</details>

### 問題2：境界値分析

以下の仕様に基づいて、境界値分析を行い、テストケースを作成してください。

**仕様：** パスワード設定
- 文字数：8文字以上、64文字以下
- 英大文字を1文字以上含む
- 英小文字を1文字以上含む
- 数字を1文字以上含む

文字数に関する境界値テストケースを作成してください。

**期待される出力例：**

```
| TC-ID | 入力値         | 文字数 | 期待結果 |
|-------|---------------|-------|---------|
| TC-001| "Aa1xxxx"     | 7     | エラー   |
```

<details>
<summary>ヒント</summary>

境界値は以下の4つです。
- 7文字（下限-1：無効）
- 8文字（下限：有効）
- 64文字（上限：有効）
- 65文字（上限+1：無効）

各テストデータは英大文字・英小文字・数字の要件を満たすようにしてください（文字数のテストに集中するため）。

</details>

<details>
<summary>解答例</summary>

```
【境界値分析：パスワード文字数】

       無効     │   有効              │  無効
  ◀────────────┤─────────────────────┤──────────▶
           7文字 8文字            64文字 65文字

| TC-ID   | 入力値                    | 文字数 | 期待結果 | テスト観点     |
|---------|--------------------------|-------|---------|-------------|
| TC-PW-001| "Aa1xxxx"               | 7     | エラー   | 下限-1（無効）|
| TC-PW-002| "Aa1xxxxx"              | 8     | 成功     | 下限（有効）  |
| TC-PW-003| "Aa1xxxxxxxxx"          | 9     | 成功     | 下限+1（有効）|
| TC-PW-004| "Aa1" + "x"×61          | 64    | 成功     | 上限（有効）  |
| TC-PW-005| "Aa1" + "x"×62          | 65    | エラー   | 上限+1（無効）|
| TC-PW-006| ""                       | 0     | エラー   | 空文字（無効）|

※ すべてのテストデータは英大文字(A)、英小文字(a,x)、
  数字(1)を含み、文字種の条件を満たしている。
  これにより文字数のみを検証している。

【エラーメッセージの確認】
| TC-ID   | 期待されるエラーメッセージ                      |
|---------|----------------------------------------------|
| TC-PW-001| 「パスワードは8文字以上で入力してください」       |
| TC-PW-005| 「パスワードは64文字以内で入力してください」      |
| TC-PW-006| 「パスワードを入力してください」                  |
```

</details>

### 問題3：同値分割法

以下の仕様に基づいて、同値分割によるテストケースを作成してください。

**仕様：** ECサイトの送料計算
- 注文金額が5,000円以上：送料無料
- 注文金額が3,000円以上5,000円未満：送料300円
- 注文金額が3,000円未満：送料500円
- 注文金額が0円以下：エラー

**期待される出力例：**

```
| TC-ID | 同値クラス         | 入力値   | 期待結果     |
|-------|-------------------|---------|-------------|
| TC-001| 5,000円以上（有効） | 6,000円  | 送料無料     |
```

<details>
<summary>ヒント</summary>

同値クラスを4つに分割し、各クラスから代表値を1つ選びます。さらに境界値も組み合わせると効果的です。

</details>

<details>
<summary>解答例</summary>

```
【同値クラスの分割】

  エラー     │  500円    │  300円      │  無料
 ◀──────────┤──────────┤────────────┤──────────▶
         0円  1円    2,999円 3,000円  4,999円 5,000円

【同値分割によるテストケース（代表値）】

| TC-ID   | 同値クラス                | 入力値（注文金額）| 期待結果        |
|---------|--------------------------|----------------|----------------|
| TC-SH-001| 5,000円以上（送料無料）    | 6,000円         | 送料: 0円      |
| TC-SH-002| 3,000〜4,999円（送料300円）| 4,000円         | 送料: 300円    |
| TC-SH-003| 1〜2,999円（送料500円）    | 1,500円         | 送料: 500円    |
| TC-SH-004| 0円以下（エラー）          | -100円          | エラー          |

【境界値を含めた拡張テストケース】

| TC-ID   | テスト観点  | 入力値   | 期待結果      |
|---------|-----------|---------|--------------|
| TC-SH-005| 境界値    | 0円      | エラー        |
| TC-SH-006| 境界値    | 1円      | 送料: 500円   |
| TC-SH-007| 境界値    | 2,999円  | 送料: 500円   |
| TC-SH-008| 境界値    | 3,000円  | 送料: 300円   |
| TC-SH-009| 境界値    | 4,999円  | 送料: 300円   |
| TC-SH-010| 境界値    | 5,000円  | 送料: 0円     |
```

</details>

### 問題4：デシジョンテーブルの作成

以下の仕様に基づいて、デシジョンテーブルを作成してください。

**仕様：** 会員登録時の特典
- 条件1：紹介コードの有無（あり/なし）
- 条件2：キャンペーン期間中か（期間中/期間外）
- 条件3：年齢（18歳以上/18歳未満）

- 18歳未満 → 登録不可
- 18歳以上 + 紹介コードあり → 500ポイント付与
- 18歳以上 + キャンペーン期間中 → 300ポイント付与
- 18歳以上 + 紹介コードあり + キャンペーン期間中 → 500 + 300 = 800ポイント付与
- 18歳以上 + 紹介コードなし + 期間外 → 100ポイント付与（通常特典）

<details>
<summary>ヒント</summary>

条件が3つで各2値なので、組み合わせは2×2×2=8通りです。18歳未満の場合は他の条件に関わらず登録不可になります。

</details>

<details>
<summary>解答例</summary>

```
【デシジョンテーブル：会員登録特典】

┌───────────────┬────┬────┬────┬────┬────┬────┬────┬────┐
│ 条件           │ R1 │ R2 │ R3 │ R4 │ R5 │ R6 │ R7 │ R8 │
├───────────────┼────┼────┼────┼────┼────┼────┼────┼────┤
│ 紹介コードあり  │ Y  │ Y  │ Y  │ Y  │ N  │ N  │ N  │ N  │
│ キャンペーン中  │ Y  │ Y  │ N  │ N  │ Y  │ Y  │ N  │ N  │
```

### 問題5：テスト設計書の作成（総合演習）

ECサイトの「商品をカートに追加する」機能について、テスト設計書を作成してください。

以下を含めること：
1. テスト観点の一覧
2. テストケース（正常系3件以上、異常系5件以上）
3. テストデータ
4. 合格基準

**機能仕様：**
- ログインユーザーのみカートに追加できる
- 数量は1〜99の範囲で指定できる
- 在庫数以内であること
- 同じ商品が既にカートにある場合は数量を加算する
- カートに入れられる商品種類の上限は50種類
- カート内商品の保持期間は30日間

<summary>ヒント</summary>

- 正常系：基本的なカート追加、数量指定、同一商品の加算
- 異常系：未ログイン、在庫不足、数量上限超過、カート上限超過
- 境界値：数量1（最小）、99（最大）、在庫ちょうど

</details>
