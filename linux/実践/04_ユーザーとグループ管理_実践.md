# 実践課題04：ユーザーとグループ管理 ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第3章（パーミッションとユーザー管理）
> **課題の種類**: ミニプロジェクト
> **学習目標**: ユーザーとグループの作成・管理を行い、適切なアクセス制御を設計できるようになる

---

## 完成イメージ

小規模なチーム環境を想定し、ユーザーとグループを作成してアクセス制御を設定します。

```
【チーム構成】

グループ: developers
  ├── dev1 (開発者1)
  └── dev2 (開発者2)

グループ: designers
  └── designer1 (デザイナー1)

グループ: project-team  ← dev1, designer1 が両方所属
  ├── dev1
  └── designer1

【共有ディレクトリ】
/tmp/team_project/
├── shared/       ← project-team メンバーが読み書き可能
├── dev/          ← developers のみ読み書き可能
├── design/       ← designers のみ読み書き可能
└── public/       ← 全員が読み取り可能
```

---

## 課題の要件

1. 3つのグループ（`developers`, `designers`, `project-team`）を作成する
2. 3人のユーザー（`dev1`, `dev2`, `designer1`）を作成する
3. 各ユーザーを適切なグループに所属させる
4. 共有ディレクトリを作成し、グループベースのアクセス制御を設定する
5. 各ユーザーとしてファイルの読み書きをテストする

> **注意**: この課題は `sudo` 権限が必要です。実行環境で `sudo` が利用可能であることを確認してください。テスト環境やVMでの実行を推奨します。

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

```bash
#!/bin/bash
# ユーザーとグループ管理の実践
# 学べる内容：useradd, groupadd, usermod, chgrp, chmod
# 実行方法：sudo bash user_management.sh
# 注意：root権限が必要です

# root権限チェック
if [ "$(id -u)" -ne 0 ]; then
    echo "エラー: このスクリプトはroot権限で実行してください。"
    echo "使い方: sudo bash user_management.sh"
    exit 1
fi

echo "===== ユーザー・グループ管理 実践 ====="
echo ""

# --- グループ作成 ---
echo "--- グループ作成 ---"
groupadd developers 2>/dev/null && echo "グループ 'developers' を作成しました" || echo "グループ 'developers' は既に存在します"
groupadd designers 2>/dev/null && echo "グループ 'designers' を作成しました" || echo "グループ 'designers' は既に存在します"
groupadd project-team 2>/dev/null && echo "グループ 'project-team' を作成しました" || echo "グループ 'project-team' は既に存在します"
echo ""

# --- ユーザー作成 ---
echo "--- ユーザー作成 ---"
for user in dev1 dev2 designer1; do
    if id "$user" > /dev/null 2>&1; then
        echo "ユーザー '$user' は既に存在します"
    else
        useradd -m -s /bin/bash "$user"
        echo "password123" | passwd --stdin "$user" 2>/dev/null || echo "$user:password123" | chpasswd
        echo "ユーザー '$user' を作成しました"
    fi
done
echo ""

# --- グループ割り当て ---
echo "--- グループ割り当て ---"
usermod -aG developers dev1
usermod -aG developers dev2
usermod -aG designers designer1
usermod -aG project-team dev1
usermod -aG project-team designer1

echo "dev1    → developers, project-team"
echo "dev2    → developers"
echo "designer1 → designers, project-team"
echo ""

# --- 所属グループ確認 ---
echo "--- 所属グループ確認 ---"
for user in dev1 dev2 designer1; do
    echo "$user: $(id "$user")"
done
echo ""

# --- 共有ディレクトリ作成 ---
echo "--- 共有ディレクトリ作成 ---"
BASE="/tmp/team_project"
mkdir -p "$BASE"/{shared,dev,design,public}

# shared: project-team グループが読み書き可
chgrp project-team "$BASE/shared"
chmod 770 "$BASE/shared"

# dev: developers グループが読み書き可
chgrp developers "$BASE/dev"
chmod 770 "$BASE/dev"

# design: designers グループが読み書き可
chgrp designers "$BASE/design"
chmod 770 "$BASE/design"

# public: 全員が読み取り可能
chmod 755 "$BASE/public"

echo "ディレクトリ構造:"
ls -la "$BASE/"
echo ""

# --- テスト ---
echo "--- アクセステスト ---"

# dev1 が shared にファイルを作成
su -c "echo 'shared file by dev1' > $BASE/shared/test.txt" dev1 && \
    echo "OK: dev1 は shared に書き込み可能" || \
    echo "NG: dev1 は shared に書き込み不可"

# dev2 が shared にアクセス（project-team に入っていないので不可のはず）
su -c "cat $BASE/shared/test.txt" dev2 2>/dev/null && \
    echo "OK: dev2 は shared を読み取り可能" || \
    echo "期待通り: dev2 は shared にアクセス不可"

# designer1 が dev にアクセス（developers に入っていないので不可のはず）
su -c "echo 'test' > $BASE/dev/test.txt" designer1 2>/dev/null && \
    echo "注意: designer1 が dev に書き込めてしまいました" || \
    echo "期待通り: designer1 は dev に書き込み不可"

echo ""
echo "===== 完了 ====="
```

<details>
<summary>解答例（改良版 ─ SGIDビットとクリーンアップ付き）</summary>

```bash
#!/bin/bash
# ユーザーとグループ管理の実践（改良版）
# 学べる内容：SGID、ACL、スクリプトのクリーンアップ
# 実行方法：sudo bash user_management_v2.sh
# クリーンアップ：sudo bash user_management_v2.sh --cleanup

set -e

BASE="/tmp/team_project"
USERS=(dev1 dev2 designer1)
GROUPS=(developers designers project-team)

# --- クリーンアップモード ---
if [ "${1}" = "--cleanup" ]; then
    echo "クリーンアップを実行します..."
    for user in "${USERS[@]}"; do
        userdel -r "$user" 2>/dev/null && echo "ユーザー '$user' を削除" || true
    done
    for group in "${GROUPS[@]}"; do
        groupdel "$group" 2>/dev/null && echo "グループ '$group' を削除" || true
    done
    rm -rf "$BASE"
    echo "クリーンアップ完了"
    exit 0
fi

# root権限チェック
if [ "$(id -u)" -ne 0 ]; then
    echo "エラー: sudo で実行してください" >&2
    exit 1
fi

echo "===== ユーザー・グループ管理（改良版） ====="

# --- グループ作成 ---
for group in "${GROUPS[@]}"; do
    groupadd "$group" 2>/dev/null || true
done
echo "グループ作成完了: ${GROUPS[*]}"

# --- ユーザー作成 ---
for user in "${USERS[@]}"; do
    if ! id "$user" > /dev/null 2>&1; then
        useradd -m -s /bin/bash "$user"
        echo "$user:password123" | chpasswd
    fi
done
echo "ユーザー作成完了: ${USERS[*]}"

# --- グループ割り当て（配列で管理） ---
declare -A USER_GROUPS=(
    [dev1]="developers project-team"
    [dev2]="developers"
    [designer1]="designers project-team"
)

for user in "${!USER_GROUPS[@]}"; do
    for group in ${USER_GROUPS[$user]}; do
        usermod -aG "$group" "$user"
    done
    echo "$user → ${USER_GROUPS[$user]}"
done

# --- ディレクトリ作成とパーミッション設定 ---
mkdir -p "$BASE"/{shared,dev,design,public}

# SGID ビット（Set Group ID）の設定
# SGIDを設定すると、そのディレクトリ内で作成されたファイルは
# ディレクトリのグループを自動的に継承する
configure_dir() {
    local dir="$1" group="$2" mode="$3"
    chgrp "$group" "$dir"
    chmod "$mode" "$dir"
    # SGIDビットを追加（2をモードの先頭に付ける）
    chmod g+s "$dir"
}

configure_dir "$BASE/shared" "project-team" 770
configure_dir "$BASE/dev" "developers" 770
configure_dir "$BASE/design" "designers" 770
chmod 755 "$BASE/public"

echo ""
echo "=== ディレクトリ設定 ==="
ls -la "$BASE/"
echo ""

# --- テスト ---
echo "=== アクセステスト ==="

run_test() {
    local user="$1" action="$2" path="$3" expected="$4"
    local result
    if result=$(su -c "$action" "$user" 2>&1); then
        if [ "$expected" = "success" ]; then
            echo "  PASS: $user → $path ($action)"
        else
            echo "  FAIL: $user が $path にアクセスできてしまいました"
        fi
    else
        if [ "$expected" = "fail" ]; then
            echo "  PASS: $user → $path アクセス拒否（期待通り）"
        else
            echo "  FAIL: $user が $path にアクセスできませんでした"
        fi
    fi
}

# dev1: shared に書き込み可能（project-team メンバー）
run_test dev1 "echo 'test' > $BASE/shared/file.txt" "$BASE/shared" "success"

# dev2: shared に書き込み不可（project-team メンバーではない）
run_test dev2 "echo 'test' > $BASE/shared/file2.txt" "$BASE/shared" "fail"

# dev1: dev に書き込み可能（developers メンバー）
run_test dev1 "echo 'test' > $BASE/dev/file.txt" "$BASE/dev" "success"

# designer1: dev に書き込み不可（developers メンバーではない）
run_test designer1 "echo 'test' > $BASE/dev/file2.txt" "$BASE/dev" "fail"

# designer1: design に書き込み可能（designers メンバー）
run_test designer1 "echo 'test' > $BASE/design/file.txt" "$BASE/design" "success"

echo ""

# SGID の効果を確認
echo "=== SGID の効果 ==="
su -c "touch $BASE/shared/sgid_test.txt" dev1
ls -l "$BASE/shared/sgid_test.txt"
echo "↑ グループが project-team になっていれば SGID が有効です"

echo ""
echo "クリーンアップするには: sudo bash $0 --cleanup"
```

**初心者向けとの違い:**
- SGID（Set Group ID）ビットを活用 → ディレクトリ内で作成されたファイルが自動的にグループを継承
- 連想配列（`declare -A`）でユーザーとグループの対応を管理
- テスト関数（`run_test`）で自動テストを実行 → 手動確認が不要
- `--cleanup` オプションで環境を元に戻せる → テスト後の片付けが簡単
