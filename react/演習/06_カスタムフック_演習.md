## 演習1（基本）：useToggleを作ろう

`true` / `false` の切り替えを行うカスタムフック `useToggle` を作成してください。

**要件：**
- `useToggle(initialValue)` で初期値を受け取る（デフォルトは `false`）
- 戻り値は `[value, toggle]` の配列
- `toggle()` を呼ぶと `true` ↔ `false` が切り替わる
- このフックを使って、FAQのアコーディオン（開閉パネル）を作成する

<details>
<summary>ヒント</summary>

カスタムフックは「useで始まる関数」の中でReactのフックを使うだけです。

```jsx
function useToggle(initialValue = false) {
  const [value, setValue] = useState(initialValue);
  const toggle = () => setValue(prev => !prev);
  return [value, toggle];
}
```

</details>

<details>
<summary>解答例</summary>

```jsx
import { useState } from 'react';

// ========================================
// カスタムフック：useToggle
// true / false を切り替えるシンプルなフック
// モーダルの開閉、メニューの表示/非表示、アコーディオンなど
// ON/OFF の切り替えが必要な場面で再利用できる
// ========================================
function useToggle(initialValue = false) {
  const [value, setValue] = useState(initialValue);

  // prev => !prev で関数型更新を使い、常に最新のstateを反転させる
  const toggle = () => setValue(prev => !prev);

  return [value, toggle];
}

// ========================================
// 使用例：FAQアコーディオン
// ========================================
function FAQItem({ question, answer }) {
  const [isOpen, toggle] = useToggle(false);

  return (
    <div style={{
      borderBottom: '1px solid #ddd',
      padding: '12px 0',
    }}>
      <button
        onClick={toggle}
        style={{
          width: '100%',
          textAlign: 'left',
          padding: '8px',
          background: 'none',
          border: 'none',
          fontSize: '16px',
          cursor: 'pointer',
          display: 'flex',
          justifyContent: 'space-between',
        }}
      >
        <span>{question}</span>
        <span>{isOpen ? '▲' : '▼'}</span>
      </button>
      {isOpen && (
        <div style={{
          padding: '8px 8px 0',
          color: '#555',
          lineHeight: '1.6',
        }}>
          {answer}
        </div>
      )}
    </div>
  );
}

function FAQ() {
  const faqs = [
    { q: "Reactとは何ですか？", a: "ReactはUIを構築するためのJavaScriptライブラリです。" },
    { q: "カスタムフックとは？", a: "Reactのフックを使ったロジックを再利用可能な関数に切り出したものです。" },
    { q: "なぜuseで始める必要がありますか？", a: "Reactがフックのルールを検証するために、命名規則が必要です。" },
  ];

  return (
    <div style={{ maxWidth: '600px', margin: '0 auto', padding: '20px' }}>
      <h2>よくある質問</h2>
      {faqs.map((faq, i) => (
        <FAQItem key={i} question={faq.q} answer={faq.a} />
      ))}
    </div>
  );
}

export default FAQ;
```

**ポイント：**
- カスタムフックは特別な仕組みではなく「useで始まる関数の中でフックを使う」だけです
- 同じ `useToggle` をモーダル、メニュー、アコーディオンなど様々な場所で再利用できます
- 各コンポーネントが `useToggle` を呼ぶと、それぞれ独立したstateが作られます

</details>

## 演習2（基本）：useLocalStorageを作ろう

ローカルストレージとstateを自動同期するカスタムフック `useLocalStorage` を作成してください。

**要件：**
- `useLocalStorage(key, initialValue)` でキーと初期値を受け取る
- 戻り値は `[storedValue, setValue]`（useStateと同じ使い勝手）
- 値が変わったら自動的にlocalStorageに保存
- ページリロード時にlocalStorageから値を復元

<details>
<summary>ヒント</summary>

- `useState` の遅延初期化で、localStorageから値を読み込みます
- `useEffect` で値が変わるたびにlocalStorageに保存します
- `JSON.stringify` / `JSON.parse` でオブジェクトも保存できるようにします

</details>

<details>
<summary>解答例</summary>

```jsx
import { useState, useEffect } from 'react';

// ========================================
// カスタムフック：useLocalStorage
// useStateと同じ感覚で使えて、自動的にlocalStorageと同期する
// ページをリロードしてもデータが消えない永続化フック
// ========================================
function useLocalStorage(key, initialValue) {
  // ========================================
  // 遅延初期化：初回レンダリング時のみlocalStorageを読み込む
  // localStorageにデータがあればそれを使い、なければinitialValueを使う
  // ========================================
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.error(`localStorage読み込みエラー (${key}):`, error);
      return initialValue;
    }
  });

  // ========================================
  // storedValueが変わるたびにlocalStorageに保存する
  // JSON.stringifyで文字列化してから保存するので、
  // オブジェクトや配列も保存できる
  // ========================================
  useEffect(() => {
    try {
      localStorage.setItem(key, JSON.stringify(storedValue));
    } catch (error) {
      console.error(`localStorage保存エラー (${key}):`, error);
    }
  }, [key, storedValue]);

  return [storedValue, setStoredValue];
}

// ========================================
// 使用例：設定画面
// ========================================
function SettingsPage() {
  // useStateと全く同じ使い勝手で、永続化が自動的に行われる
  const [username, setUsername] = useLocalStorage('username', '');
  const [fontSize, setFontSize] = useLocalStorage('fontSize', 16);
  const [darkMode, setDarkMode] = useLocalStorage('darkMode', false);

  return (
    <div style={{
      maxWidth: '400px',
      margin: '0 auto',
      padding: '20px',
      backgroundColor: darkMode ? '#333' : '#fff',
      color: darkMode ? '#fff' : '#333',
      fontSize: `${fontSize}px`,
    }}>
      <h2>設定</h2>
      <div style={{ marginBottom: '12px' }}>
        <label>ユーザー名: </label>
        <input
          value={username}
          onChange={(e) => setUsername(e.target.value)}
          style={{ padding: '6px' }}
        />
      </div>
      <div style={{ marginBottom: '12px' }}>
        <label>文字サイズ: {fontSize}px </label>
        <input
          type="range"
          min="12"
          max="24"
          value={fontSize}
          onChange={(e) => setFontSize(Number(e.target.value))}
        />
      </div>
      <div>
        <label>
          <input
            type="checkbox"
            checked={darkMode}
            onChange={(e) => setDarkMode(e.target.checked)}
          />
          ダークモード
        </label>
      </div>
      <p style={{ color: '#888', fontSize: '14px', marginTop: '16px' }}>
        設定はリロードしても保持されます
      </p>
    </div>
  );
}

export default SettingsPage;
```

**ポイント：**
- `try-catch` でlocalStorageのエラーをハンドリングしています（プライベートブラウジングなどで失敗する場合がある）
- `JSON.stringify` / `JSON.parse` を使うことで、文字列以外（数値、配列、オブジェクト）も保存できます
- `useLocalStorage` は実務でも非常によく使われる定番カスタムフックです

</details>

## 演習3（基本）：useFetchを作ろう

APIからデータを取得するカスタムフック `useFetch` を作成してください。

**要件：**
- `useFetch(url)` でURLを受け取る
- 戻り値は `{ data, loading, error }` オブジェクト
- URLが変わったら再フェッチ
- エラーハンドリングも含める

<details>
<summary>ヒント</summary>

`data`、`loading`、`error` の3つのstateを管理し、`useEffect` でURLが変わるたびにフェッチを実行します。

</details>

<details>
<summary>解答例</summary>

```jsx
import { useState, useEffect } from 'react';

// ========================================
// カスタムフック：useFetch
// URLを渡すだけでデータ取得・ローディング・エラーを管理する
// 複数のコンポーネントでAPI取得ロジックを再利用できる
// ========================================
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    // ========================================
    // 競合状態（race condition）を防ぐフラグ
    // URLが素早く変わったとき、古いリクエストの結果を無視する
    // ========================================
    let ignore = false;

    const fetchData = async () => {
      setLoading(true);
      setError(null);

      try {
        const response = await fetch(url);
        if (!response.ok) {
          throw new Error(`HTTP Error: ${response.status}`);
        }
        const json = await response.json();
        if (!ignore) {
          setData(json);
        }
      } catch (err) {
        if (!ignore) {
          setError(err.message);
        }
      } finally {
        if (!ignore) {
          setLoading(false);
        }
      }
    };

    fetchData();

    // クリーンアップ：URLが変わったら古いリクエストの結果を無視
    return () => {
      ignore = true;
    };
  }, [url]);  // URLが変わるたびに再実行

  return { data, loading, error };
}

// ========================================
// 使用例：ユーザー一覧
// useFetchを使えば、データ取得のボイラープレートが不要になる
// ========================================
function UserDirectory() {
  const { data: users, loading, error } = useFetch(
    'https://jsonplaceholder.typicode.com/users'
  );

  if (loading) return <p>読み込み中...</p>;
  if (error) return <p style={{ color: 'red' }}>エラー: {error}</p>;

  return (
    <div style={{ maxWidth: '500px', margin: '0 auto', padding: '20px' }}>
      <h2>ユーザー一覧</h2>
      {users.map(user => (
        <div key={user.id} style={{
          padding: '12px',
          borderBottom: '1px solid #eee',
        }}>
          <strong>{user.name}</strong>
          <p style={{ color: '#666', margin: '4px 0 0' }}>{user.email}</p>
        </div>
      ))}
    </div>
  );
}

export default UserDirectory;
```

**ポイント：**
- `useFetch` は最もよく実装されるカスタムフックの一つです
- 実務では `SWR` や `TanStack Query（React Query）` などのライブラリが同様の機能を提供します
- `ignore` フラグによる競合状態の防止は、非同期処理のuseEffectで必須のパターンです

</details>

## 演習4（応用）：バグを見つけて修正しよう

以下のカスタムフックにはバグがあります。2つの問題を見つけて修正してください。

```jsx
// バグのあるカスタムフック
function useWindowWidth() {
  const width = window.innerWidth;

  function handleResize() {
    width = window.innerWidth;
  }

  window.addEventListener('resize', handleResize);

  return width;
}

function ResponsiveLayout() {
  const width = useWindowWidth();

  return (
    <div>
      <p>ウィンドウ幅: {width}px</p>
      {width < 768 ? <MobileMenu /> : <DesktopMenu />}
    </div>
  );
}
```

<details>
<summary>ヒント</summary>

- **バグ1：** `width` は普通の変数なので、値を変更してもReactは再レンダリングしません。`useState` を使う必要があります。
- **バグ2：** `addEventListener` がレンダリングのたびに追加され続け、`removeEventListener` も呼ばれません。`useEffect` の中で処理しましょう。

</details>

<details>
<summary>解答例</summary>

```jsx
import { useState, useEffect } from 'react';

// ========================================
// 修正後のカスタムフック：useWindowWidth
// ========================================
function useWindowWidth() {
  // ========================================
  // 修正1：普通の変数ではなくuseStateを使う
  // Reactでは、画面の再描画を起こすにはstateを更新する必要がある
  // 普通の変数 (let width = ...) を書き換えても再レンダリングされない
  // ========================================
  const [width, setWidth] = useState(window.innerWidth);

  // ========================================
  // 修正2：useEffectの中でイベントリスナーを登録し、
  // クリーンアップで解除する
  //
  // 修正前の問題点：
  // - レンダリングのたびにaddEventListenerが呼ばれ、
  //   リスナーが無限に蓄積していた
  // - removeEventListenerが呼ばれないのでメモリリーク発生
  //
  // useEffectの中で登録することで：
  // - マウント時に1回だけリスナーを登録
  // - アンマウント時にクリーンアップで解除
  // ========================================
  useEffect(() => {
    const handleResize = () => {
      setWidth(window.innerWidth);
    };

    window.addEventListener('resize', handleResize);

    return () => {
      window.removeEventListener('resize', handleResize);
    };
  }, []);

  return width;
}

function ResponsiveLayout() {
  const width = useWindowWidth();

  return (
    <div style={{ padding: '20px' }}>
      <p>ウィンドウ幅: {width}px</p>
      {width < 768 ? (
        <p>モバイルメニュー表示中</p>
      ) : (
        <p>デスクトップメニュー表示中</p>
      )}
    </div>
  );
}

export default ResponsiveLayout;
```

**2つのバグのまとめ：**
1. **普通の変数を使っていた** → `useState` で状態管理することで、値の変化で再レンダリングが起きる
2. **useEffectなしでイベントリスナーを登録** → レンダリングのたびにリスナーが蓄積し、クリーンアップも行われないためメモリリーク

</details>

## 演習5（応用）：useCountdownを作ろう

指定した秒数からカウントダウンするフック `useCountdown` を作成してください。

**要件：**
- `useCountdown(initialSeconds)` で初期秒数を受け取る
- 戻り値は `{ seconds, isFinished, start, pause, reset }`
- 0になったら自動停止し、`isFinished` が `true` になる
- このフックを使って「クイズの制限時間タイマー」を作成する

<details>
<summary>ヒント</summary>

`useEffect` の中で `setInterval` を使い、`isRunning` の状態に応じてタイマーを制御します。秒数が0になったら自動的に `isRunning` を `false` にします。

</details>

<details>
<summary>解答例</summary>

```jsx
import { useState, useEffect, useCallback } from 'react';

// ========================================
// カスタムフック：useCountdown
// 指定秒数からカウントダウンし、0になったら停止する
// クイズの制限時間、セール終了タイマーなどに使える
// ========================================
function useCountdown(initialSeconds) {
  const [seconds, setSeconds] = useState(initialSeconds);
  const [isRunning, setIsRunning] = useState(false);

  // 0になったら完了
  const isFinished = seconds <= 0;

  useEffect(() => {
    // 動作中でないか、カウントが0以下なら何もしない
    if (!isRunning || seconds <= 0) {
      if (seconds <= 0) setIsRunning(false);
      return;
    }

    const intervalId = setInterval(() => {
      setSeconds(prev => {
        if (prev <= 1) {
          setIsRunning(false);
          return 0;
        }
        return prev - 1;
      });
    }, 1000);

    return () => clearInterval(intervalId);
  }, [isRunning, seconds]);

  // ========================================
  // 操作関数をuseCallbackでメモ化（任意だが推奨）
  // 子コンポーネントに渡す場合に不要な再レンダリングを防ぐ
  // ========================================
  const start = useCallback(() => {
    if (seconds > 0) setIsRunning(true);
  }, [seconds]);

  const pause = useCallback(() => {
    setIsRunning(false);
  }, []);

  const reset = useCallback(() => {
    setIsRunning(false);
    setSeconds(initialSeconds);
  }, [initialSeconds]);

  return { seconds, isFinished, isRunning, start, pause, reset };
}

// ========================================
// 使用例：クイズ制限時間タイマー
// ========================================
function QuizTimer() {
  const { seconds, isFinished, isRunning, start, pause, reset } = useCountdown(30);

  // 残り時間に応じた色を計算（派生値なのでstateにしない）
  const getColor = () => {
    if (seconds > 20) return '#27ae60';
    if (seconds > 10) return '#f39c12';
    return '#e74c3c';
  };

  return (
    <div style={{ textAlign: 'center', padding: '20px' }}>
      <h2>クイズタイマー</h2>

      <div style={{
        fontSize: '64px',
        fontFamily: 'monospace',
        color: isFinished ? '#999' : getColor(),
        margin: '20px 0',
      }}>
        {seconds}
      </div>

      {isFinished ? (
        <div>
          <p style={{ color: '#e74c3c', fontSize: '24px' }}>時間切れ！</p>
          <button onClick={reset} style={{ padding: '10px 24px', fontSize: '16px' }}>
            もう一度
          </button>
        </div>
      ) : (
        <div style={{ display: 'flex', gap: '8px', justifyContent: 'center' }}>
          {!isRunning ? (
            <button onClick={start} style={{ padding: '10px 24px' }}>
              {seconds === 30 ? 'スタート' : '再開'}
            </button>
          ) : (
            <button onClick={pause} style={{ padding: '10px 24px' }}>一時停止</button>
          )}
          <button onClick={reset} style={{ padding: '10px 24px' }}>リセット</button>
        </div>
      )}
    </div>
  );
}

export default QuizTimer;
```

**ポイント：**
- カスタムフックから「状態」と「操作関数」の両方を返すことで、UIから完全にロジックを分離できます
- 同じ `useCountdown` をクイズ、オークション、ポモドーロタイマーなど様々な場所で再利用できます
- `useCallback` は第10章で学ぶ最適化テクニックですが、カスタムフックが返す関数にはよく使われます

</details>

## 演習6（応用）：useFormを作ろう

フォームの入力値・バリデーション・送信を管理するカスタムフック `useForm` を作成してください。

**要件：**
- `useForm(initialValues, validate)` で初期値とバリデーション関数を受け取る
- 戻り値は `{ values, errors, handleChange, handleSubmit, isSubmitting }`
- 入力値が変わるたびにバリデーションを実行
- 送信時にバリデーションエラーがなければ `onSubmit` を実行

<details>
<summary>ヒント</summary>

- `values` はオブジェクトで管理します（`{ name: "", email: "" }` のように）
- `handleChange` では `e.target.name` をキーにして値を更新します
- `validate` 関数は `values` を受け取り、エラーオブジェクト `{ name: "名前は必須です" }` を返します

</details>

<details>
<summary>解答例</summary>

```jsx
import { useState } from 'react';

// ========================================
// カスタムフック：useForm
// フォームの状態管理・バリデーション・送信を一元管理する
// 実務でもよく使われるパターン（React Hook Formなどの
// ライブラリも同様のAPIを提供している）
// ========================================
function useForm(initialValues, validate) {
  const [values, setValues] = useState(initialValues);
  const [errors, setErrors] = useState({});
  const [isSubmitting, setIsSubmitting] = useState(false);

  // ========================================
  // 入力値の変更ハンドラ
  // e.target.name でどのフィールドかを判定し、
  // スプレッド構文で該当フィールドだけを更新する
  // ========================================
  const handleChange = (e) => {
    const { name, value, type, checked } = e.target;
    const newValues = {
      ...values,
      [name]: type === 'checkbox' ? checked : value,
    };
    setValues(newValues);

    // リアルタイムバリデーション：値が変わるたびにエラーを再計算
    if (validate) {
      const validationErrors = validate(newValues);
      setErrors(validationErrors);
    }
  };

  // ========================================
  // 送信ハンドラ
  // バリデーションエラーがなければonSubmitを実行する
  // ========================================
  const handleSubmit = (onSubmit) => async (e) => {
    e.preventDefault();

    // 送信前にバリデーションを実行
    const validationErrors = validate ? validate(values) : {};
    setErrors(validationErrors);

    // エラーがなければ送信
    if (Object.keys(validationErrors).length === 0) {
      setIsSubmitting(true);
      await onSubmit(values);
      setIsSubmitting(false);
    }
  };

  // フォームをリセットする関数
  const reset = () => {
    setValues(initialValues);
    setErrors({});
  };

  return { values, errors, handleChange, handleSubmit, isSubmitting, reset };
}

// ========================================
// 使用例：ユーザー登録フォーム
// ========================================

// バリデーション関数
const validateRegistration = (values) => {
  const errors = {};
  if (!values.name.trim()) errors.name = "名前は必須です";
  if (!values.email.trim()) {
    errors.email = "メールアドレスは必須です";
  } else if (!/\S+@\S+\.\S+/.test(values.email)) {
    errors.email = "メールアドレスの形式が正しくありません";
  }
  if (values.password.length < 8) {
    errors.password = "パスワードは8文字以上必要です";
  }
  return errors;
};

function RegistrationForm() {
  const { values, errors, handleChange, handleSubmit, isSubmitting, reset } = useForm(
    { name: '', email: '', password: '' },
    validateRegistration
  );

  const onSubmit = async (formData) => {
    // 実際にはここでAPIにデータを送信する
    await new Promise(resolve => setTimeout(resolve, 1000));
    alert(`登録完了: ${formData.name} (${formData.email})`);
    reset();
  };

  return (
    <form
      onSubmit={handleSubmit(onSubmit)}
      style={{ maxWidth: '400px', margin: '0 auto', padding: '20px' }}
    >
      <h2>ユーザー登録</h2>
      {['name', 'email', 'password'].map(field => (
        <div key={field} style={{ marginBottom: '16px' }}>
          <label style={{ display: 'block', marginBottom: '4px' }}>
            {field === 'name' ? '名前' : field === 'email' ? 'メール' : 'パスワード'}
          </label>
          <input
            name={field}
            type={field === 'password' ? 'password' : 'text'}
            value={values[field]}
            onChange={handleChange}
            style={{
              width: '100%',
              padding: '8px',
              borderColor: errors[field] ? '#e74c3c' : '#ddd',
              borderWidth: '1px',
              borderStyle: 'solid',
              borderRadius: '4px',
            }}
          />
          {errors[field] && (
            <p style={{ color: '#e74c3c', fontSize: '14px', margin: '4px 0 0' }}>
              {errors[field]}
            </p>
          )}
        </div>
      ))}
      <button
        type="submit"
        disabled={isSubmitting}
        style={{ padding: '10px 24px', width: '100%' }}
      >
        {isSubmitting ? '送信中...' : '登録'}
      </button>
    </form>
  );
}

export default RegistrationForm;
```

**ポイント：**
- `handleSubmit` はカリー化（関数を返す関数）を使っています。`handleSubmit(onSubmit)` で送信処理を受け取り、イベントハンドラを返します
- バリデーション関数をフックの外で定義することで、テストしやすく再利用しやすい設計になります
- 実務では `React Hook Form` や `Formik` などのライブラリが同様の機能を提供しますが、仕組みを理解しておくことが重要です

1. `useFetch(url)` — データ取得
2. `useSearch(items, query, key)` — 検索フィルタリング
3. `usePagination(items, itemsPerPage)` — ページネーション

```jsx
import { useState, useEffect, useMemo } from 'react';

// ========================================
// カスタムフック1：useFetch
// データ取得の汎用フック（演習3と同じもの）
// ========================================
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let ignore = false;
    setLoading(true);
    fetch(url)
      .then(res => res.json())
      .then(json => { if (!ignore) { setData(json); setLoading(false); } })
      .catch(err => { if (!ignore) { setError(err.message); setLoading(false); } });
    return () => { ignore = true; };
  }, [url]);

  return { data, loading, error };
}

// ========================================
// カスタムフック2：useSearch
// 配列の中から検索クエリに一致する要素をフィルタリングする
// ========================================
function useSearch(items, key) {
  const [query, setQuery] = useState("");

  // useMemoで検索結果をメモ化（itemsやqueryが変わらなければ再計算しない）
  const filtered = useMemo(() => {
    if (!items) return [];
    if (!query.trim()) return items;
    return items.filter(item =>
      item[key].toLowerCase().includes(query.toLowerCase())
    );
  }, [items, query, key]);

  return { query, setQuery, filtered };
}

// ========================================
// カスタムフック3：usePagination
// 配列をページ分割して現在のページのデータを返す
// ========================================
function usePagination(items, itemsPerPage = 5) {
  const [currentPage, setCurrentPage] = useState(1);

  const totalPages = Math.ceil(items.length / itemsPerPage);
  const paginated = items.slice(
    (currentPage - 1) * itemsPerPage,
    currentPage * itemsPerPage
  );

  // itemsが変わったら1ページ目に戻る
  useEffect(() => {
    setCurrentPage(1);
  }, [items.length]);

  const goToPage = (page) => {
    setCurrentPage(Math.min(Math.max(1, page), totalPages));
  };

  return { currentPage, totalPages, paginated, goToPage };
}

// ========================================
// リファクタリング後のコンポーネント
// 各カスタムフックが独立した責務を持ち、
// コンポーネントはUIの組み立てだけに集中できる
// ========================================
function UserSearch() {
  // データ取得
  const { data: users, loading, error } = useFetch(
    'https://jsonplaceholder.typicode.com/users'
  );

  // 検索
  const { query, setQuery, filtered } = useSearch(users, 'name');

  // ページネーション
  const { currentPage, totalPages, paginated, goToPage } = usePagination(filtered);

  if (loading) return <p>読み込み中...</p>;
  if (error) return <p>エラー: {error}</p>;

  return (
    <div style={{ maxWidth: '500px', margin: '0 auto', padding: '20px' }}>
      <h2>ユーザー検索</h2>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="名前で検索..."
        style={{ width: '100%', padding: '8px', marginBottom: '16px' }}
      />

      {paginated.map(user => (
        <div key={user.id} style={{ padding: '10px', borderBottom: '1px solid #eee' }}>
          <strong>{user.name}</strong>
          <span style={{ color: '#888', marginLeft: '8px' }}>{user.email}</span>
        </div>
      ))}

      <div style={{ display: 'flex', justifyContent: 'center', gap: '8px', marginTop: '16px' }}>
        <button onClick={() => goToPage(currentPage - 1)} disabled={currentPage === 1}>前へ</button>
        <span>{currentPage} / {totalPages}</span>
        <button onClick={() => goToPage(currentPage + 1)} disabled={currentPage === totalPages}>次へ</button>
      </div>
    </div>
  );
}

export default UserSearch;
```

**カスタムフック分離のメリット：**
1. **単一責任原則**：各フックが1つの関心事だけを担当する
2. **再利用性**：`useFetch`、`useSearch`、`usePagination` はどのコンポーネントでも使える
3. **テスト容易性**：各フックを独立してテストできる
4. **可読性**：コンポーネントが「UIの組み立て」だけに集中できる
