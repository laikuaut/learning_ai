# 第6章：構造体と列挙型

## この章のゴール

- 構造体（struct）を定義し、データをまとめて管理できる
- タプル構造体とユニット構造体の用途を理解する
- `impl` ブロックでメソッドと関連関数を定義できる
- 列挙型（enum）で複数のバリアントを持つ型を定義できる
- enum にデータを持たせて柔軟な型設計ができる
- `Option<T>` 型を使って「値がないかもしれない」状態を安全に扱える
- `match` と enum を組み合わせたパターンマッチングを活用できる
- `Display` と `Debug` トレイトの基礎を理解する

## 6.1 構造体（struct）の基本

### 6.1.1 構造体の定義とインスタンス化

構造体（struct）は、関連する複数の値をひとまとめにした独自の型です。他の言語のクラスやレコードに相当します。

```
┌────────────────────────────────────┐
│  構造体 User                        │
│  ┌──────────┬───────────────┐      │
│  │ name     │ String        │      │
│  ├──────────┼───────────────┤      │
│  │ email    │ String        │      │
│  ├──────────┼───────────────┤      │
│  │ age      │ u32           │      │
│  ├──────────┼───────────────┤      │
│  │ active   │ bool          │      │
│  └──────────┴───────────────┘      │
└────────────────────────────────────┘
```

```rust
// 構造体の定義
struct User {
    name: String,
    email: String,
    age: u32,
    active: bool,
}

fn main() {
    // インスタンスの作成
    let user1 = User {
        name: String::from("田中太郎"),
        email: String::from("tanaka@example.com"),
        age: 30,
        active: true,
    };

    // フィールドへのアクセス
    println!("名前: {}", user1.name);
    println!("メール: {}", user1.email);
    println!("年齢: {}", user1.age);
    println!("有効: {}", user1.active);
}
```

**実行結果：**
```
名前: 田中太郎
メール: tanaka@example.com
年齢: 30
有効: true
```

### 6.1.2 可変な構造体

構造体のフィールドを変更するには、インスタンス全体を `mut` にします。Rustでは特定のフィールドだけを可変にすることはできません。

```rust
struct User {
    name: String,
    email: String,
    age: u32,
    active: bool,
}

fn main() {
    let mut user = User {
        name: String::from("田中太郎"),
        email: String::from("tanaka@example.com"),
        age: 30,
        active: true,
    };

    // フィールドの変更（mut が必要）
    user.email = String::from("tanaka_new@example.com");
    user.age = 31;
    println!("更新後メール: {}", user.email);
    println!("更新後年齢: {}", user.age);
}
```

**実行結果：**
```
更新後メール: tanaka_new@example.com
更新後年齢: 31
```

### 6.1.3 フィールド初期化の省略記法

変数名とフィールド名が同じ場合、省略記法（field init shorthand）が使えます。

```rust
struct User {
    name: String,
    email: String,
    age: u32,
    active: bool,
}

fn create_user(name: String, email: String, age: u32) -> User {
    User {
        name,      // name: name と同じ
        email,     // email: email と同じ
        age,       // age: age と同じ
        active: true,
    }
}

fn main() {
    let user = create_user(
        String::from("佐藤花子"),
        String::from("sato@example.com"),
        25,
    );
    println!("{}さん（{}歳）", user.name, user.age);
}
```

**実行結果：**
```
佐藤花子さん（25歳）
```

### 6.1.4 構造体更新構文

既存のインスタンスをベースに、一部のフィールドだけ変更した新しいインスタンスを作る構文があります。

```rust
struct User {
    name: String,
    email: String,
    age: u32,
    active: bool,
}

fn main() {
    let user1 = User {
        name: String::from("田中太郎"),
        email: String::from("tanaka@example.com"),
        age: 30,
        active: true,
    };

    // user1をベースに一部だけ変更
    let user2 = User {
        email: String::from("new_tanaka@example.com"),
        ..user1  // 残りのフィールドはuser1から
    };

    // 注意: user1.nameはムーブされたので使えない
    // println!("{}", user1.name);  // コンパイルエラー！
    println!("user2: {} <{}>", user2.name, user2.email);

    // ただしCopy型のフィールドはコピーされる
    println!("user1.age: {}", user1.age);      // OK: u32はCopy
    println!("user1.active: {}", user1.active); // OK: boolはCopy
}
```

**実行結果：**
```
user2: 田中太郎 <new_tanaka@example.com>
user1.age: 30
user1.active: true
```

> **よくある間違い：** 構造体更新構文 `..user1` を使うと、`String` のようなムーブセマンティクスの型は所有権が移動します。元のインスタンスのそのフィールドは使えなくなります。

### ポイントまとめ

- `struct` で独自の型を定義：関連するデータをまとめる
- フィールドへのアクセスは `.` で行う
- `mut` はインスタンス全体に適用（フィールド単位では不可）
- フィールド初期化の省略記法で簡潔に書ける
- 構造体更新構文 `..other` で既存インスタンスをベースにできる

## 6.2 タプル構造体とユニット構造体

### 6.2.1 タプル構造体（tuple struct）

フィールド名を持たない構造体です。型に名前を付けたいが、フィールド名は不要な場合に使います。

```rust
// タプル構造体の定義
struct Color(u8, u8, u8);
struct Point(f64, f64);
struct Meters(f64);
struct Seconds(f64);

fn main() {
    let red = Color(255, 0, 0);
    let origin = Point(0.0, 0.0);

    println!("赤: ({}, {}, {})", red.0, red.1, red.2);
    println!("原点: ({}, {})", origin.0, origin.1);

    // ニュータイプパターン（Newtype Pattern）
    // 同じf64でもMetersとSecondsを間違えない
    let distance = Meters(100.0);
    let time = Seconds(9.58);

    // calculate_speed(time, distance);  // 型が違うのでコンパイルエラー！
    let speed = calculate_speed(distance, time);
    println!("速度: {:.2} m/s", speed);
}

fn calculate_speed(distance: Meters, time: Seconds) -> f64 {
    distance.0 / time.0
}
```

**実行結果：**
```
赤: (255, 0, 0)
原点: (0, 0)
速度: 10.44 m/s
```

> **現場のプロの知見：** ニュータイプパターン（Newtype Pattern）は、同じ基本型の値を区別するための強力なテクニックです。単位の取り違え（メートルと秒、円とドルなど）をコンパイル時に防げます。

### 6.2.2 ユニット構造体（unit struct）

フィールドを持たない構造体です。トレイトを実装するためだけに使うことが多いです。

```rust
struct AlwaysTrue;
struct AlwaysFalse;

// 後の章で学ぶトレイトの予習
// ここでは「ユニット構造体にも振る舞いを持たせられる」ことだけ理解しましょう
impl AlwaysTrue {
    fn check(&self) -> bool {
        true
    }
}

impl AlwaysFalse {
    fn check(&self) -> bool {
        false
    }
}

fn main() {
    let t = AlwaysTrue;
    let f = AlwaysFalse;
    println!("AlwaysTrue: {}", t.check());
    println!("AlwaysFalse: {}", f.check());
}
```

**実行結果：**
```
AlwaysTrue: true
AlwaysFalse: false
```

### ポイントまとめ

| 種類 | 構文 | 用途 |
|---|---|---|
| 通常の構造体 | `struct S { field: Type }` | 名前付きフィールドでデータを管理 |
| タプル構造体 | `struct S(Type1, Type2)` | 位置ベースのフィールド、ニュータイプ |
| ユニット構造体 | `struct S;` | トレイト実装、マーカー |

## 6.3 メソッドと impl ブロック

### 6.3.1 メソッドの定義

`impl` ブロック内に関数を定義することで、構造体にメソッド（method）を追加できます。メソッドの第1引数は必ず `self`（自分自身）です。

```
┌──────────────────────────────────────────┐
│  impl ブロックの構造                       │
│                                          │
│  impl 構造体名 {                          │
│      // メソッド（&self を取る）            │
│      fn method(&self) -> 戻り値 { ... }   │
│                                          │
│      // 可変メソッド（&mut self を取る）    │
│      fn mut_method(&mut self) { ... }     │
│                                          │
│      // 消費メソッド（self を取る）         │
│      fn consume(self) { ... }             │
│                                          │
│      // 関連関数（selfを取らない）          │
│      fn new() -> Self { ... }             │
│  }                                        │
└──────────────────────────────────────────┘
```

```rust
struct Rectangle {
    width: f64,
    height: f64,
}

impl Rectangle {
    // 不変参照メソッド：読み取りのみ
    fn area(&self) -> f64 {
        self.width * self.height
    }

    fn perimeter(&self) -> f64 {
        2.0 * (self.width + self.height)
    }

    fn is_square(&self) -> bool {
        (self.width - self.height).abs() < f64::EPSILON
    }

    // 他の構造体を引数に取るメソッド
    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}

fn main() {
    let rect = Rectangle { width: 10.0, height: 5.0 };
    println!("面積: {}", rect.area());
    println!("周囲: {}", rect.perimeter());
    println!("正方形: {}", rect.is_square());

    let small = Rectangle { width: 3.0, height: 2.0 };
    println!("rect は small を含められる: {}", rect.can_hold(&small));
}
```

**実行結果：**
```
面積: 50
周囲: 30
正方形: false
rect は small を含められる: true
```

### 6.3.2 &self, &mut self, self の使い分け

```rust
struct Counter {
    value: i32,
    name: String,
}

impl Counter {
    // &self: 読み取りのみ（最も一般的）
    fn get(&self) -> i32 {
        self.value
    }

    // &mut self: フィールドを変更する
    fn increment(&mut self) {
        self.value += 1;
    }

    fn reset(&mut self) {
        self.value = 0;
    }

    // self: 所有権を消費する（変換や破棄に使う）
    fn into_name(self) -> String {
        // selfの所有権がここに移る → 以後、元のインスタンスは使えない
        self.name
    }
}

fn main() {
    let mut counter = Counter {
        value: 0,
        name: String::from("メインカウンター"),
    };

    counter.increment();
    counter.increment();
    counter.increment();
    println!("カウント: {}", counter.get());

    counter.reset();
    println!("リセット後: {}", counter.get());

    // 所有権を消費するメソッド
    let name = counter.into_name();
    println!("名前: {}", name);
    // println!("{}", counter.get());  // コンパイルエラー！counterは消費済み
}
```

**実行結果：**
```
カウント: 3
リセット後: 0
名前: メインカウンター
```

### 6.3.3 関連関数（associated function）

`self` を取らない関数を関連関数と呼びます。`構造体名::関数名()` で呼び出します。コンストラクタとして使うのが最も一般的です。

```rust
struct Circle {
    radius: f64,
}

impl Circle {
    // 関連関数（コンストラクタ）：selfを取らない
    fn new(radius: f64) -> Circle {
        Circle { radius }
    }

    // 別のコンストラクタ
    fn unit() -> Circle {
        Circle { radius: 1.0 }
    }

    // メソッド
    fn area(&self) -> f64 {
        std::f64::consts::PI * self.radius * self.radius
    }

    fn circumference(&self) -> f64 {
        2.0 * std::f64::consts::PI * self.radius
    }

    // 拡大した新しい円を返す
    fn scale(&self, factor: f64) -> Circle {
        Circle::new(self.radius * factor)
    }
}

fn main() {
    let c1 = Circle::new(5.0);       // 関連関数の呼び出し
    let c2 = Circle::unit();          // 別のコンストラクタ
    let c3 = c1.scale(2.0);          // メソッドの呼び出し

    println!("c1: 半径={}, 面積={:.2}", c1.radius, c1.area());
    println!("c2: 半径={}, 面積={:.2}", c2.radius, c2.area());
    println!("c3: 半径={}, 面積={:.2}", c3.radius, c3.area());
    println!("c1の円周: {:.2}", c1.circumference());
}
```

**実行結果：**
```
c1: 半径=5, 面積=78.54
c2: 半径=1, 面積=3.14
c3: 半径=10, 面積=314.16
c1の円周: 31.42
```

> **現場のプロの知見：** Rustには `new` という特別なキーワードはありませんが、慣例としてコンストラクタには `new` という名前を付けます。これはRustコミュニティの強い慣例です。

### 6.3.4 複数のimplブロック

同じ構造体に対して複数の `impl` ブロックを定義できます。機能をグループ分けするのに便利です。

```rust
struct Player {
    name: String,
    hp: i32,
    max_hp: i32,
    attack: i32,
}

// コンストラクタ群
impl Player {
    fn new(name: &str, max_hp: i32, attack: i32) -> Player {
        Player {
            name: String::from(name),
            hp: max_hp,
            max_hp,
            attack,
        }
    }
}

// 戦闘関連メソッド
impl Player {
    fn take_damage(&mut self, damage: i32) {
        self.hp = (self.hp - damage).max(0);
        println!("{}は{}ダメージを受けた！ (HP: {}/{})",
            self.name, damage, self.hp, self.max_hp);
    }

    fn heal(&mut self, amount: i32) {
        self.hp = (self.hp + amount).min(self.max_hp);
        println!("{}は{}回復した！ (HP: {}/{})",
            self.name, amount, self.hp, self.max_hp);
    }

    fn is_alive(&self) -> bool {
        self.hp > 0
    }
}

// 表示関連メソッド
impl Player {
    fn status(&self) {
        println!("=== {} ===", self.name);
        println!("  HP: {}/{}", self.hp, self.max_hp);
        println!("  攻撃力: {}", self.attack);
        let bar_len = 20;
        let filled = (self.hp as f64 / self.max_hp as f64 * bar_len as f64) as usize;
        let bar: String = "█".repeat(filled) + &"░".repeat(bar_len - filled);
        println!("  [{}]", bar);
    }
}

fn main() {
    let mut hero = Player::new("勇者", 100, 25);
    hero.status();

    hero.take_damage(30);
    hero.take_damage(50);
    hero.heal(20);
    hero.status();

    println!("生存: {}", hero.is_alive());
}
```

**実行結果：**
```
=== 勇者 ===
  HP: 100/100
  攻撃力: 25
  [████████████████████]
勇者は30ダメージを受けた！ (HP: 70/100)
勇者は50ダメージを受けた！ (HP: 20/100)
勇者は20回復した！ (HP: 40/100)
=== 勇者 ===
  HP: 40/100
  攻撃力: 25
  [████████░░░░░░░░░░░░]
生存: true
```

### ポイントまとめ

| self の形式 | 意味 | 用途 |
|---|---|---|
| `&self` | 不変参照 | 読み取りメソッド（最も一般的） |
| `&mut self` | 可変参照 | フィールドを変更するメソッド |
| `self` | 所有権取得 | 型変換、消費系メソッド |
| なし | 関連関数 | コンストラクタ（`new` など） |

## 6.4 列挙型（enum）の基本

### 6.4.1 列挙型の定義

列挙型（enum、enumeration）は、取り得る値のバリアント（variant）を列挙する型です。

```rust
// シンプルな列挙型
enum Direction {
    North,
    South,
    East,
    West,
}

enum Season {
    Spring,
    Summer,
    Autumn,
    Winter,
}

fn main() {
    let dir = Direction::North;
    let season = Season::Summer;

    // matchで全バリアントを網羅的に処理
    match dir {
        Direction::North => println!("北に進む"),
        Direction::South => println!("南に進む"),
        Direction::East  => println!("東に進む"),
        Direction::West  => println!("西に進む"),
    }

    let greeting = match season {
        Season::Spring => "桜が綺麗ですね",
        Season::Summer => "暑いですね",
        Season::Autumn => "食欲の秋ですね",
        Season::Winter => "寒いですね",
    };
    println!("{}", greeting);
}
```

**実行結果：**
```
北に進む
暑いですね
```

### 6.4.2 データを持つ列挙型

Rustのenumは各バリアントが異なる型のデータを持てます。これはC言語のenumにはない強力な機能です。

```
┌──────────────────────────────────────────────┐
│  enum Message                                 │
│  ┌──────────────┬──────────────────────┐      │
│  │ Quit         │ （データなし）          │      │
│  ├──────────────┼──────────────────────┤      │
│  │ Echo         │ String               │      │
│  ├──────────────┼──────────────────────┤      │
│  │ Move         │ { x: i32, y: i32 }   │      │
│  ├──────────────┼──────────────────────┤      │
│  │ Color        │ (u8, u8, u8)         │      │
│  └──────────────┴──────────────────────┘      │
│  → 1つの型で4種類の異なるデータを表現！         │
└──────────────────────────────────────────────┘
```

```rust
enum Message {
    Quit,                        // データなし
    Echo(String),                // String を1つ持つ
    Move { x: i32, y: i32 },    // 名前付きフィールド
    Color(u8, u8, u8),           // タプル形式で3つ持つ
}

fn process_message(msg: Message) {
    match msg {
        Message::Quit => {
            println!("終了します");
        }
        Message::Echo(text) => {
            println!("エコー: {}", text);
        }
        Message::Move { x, y } => {
            println!("移動: ({}, {})", x, y);
        }
        Message::Color(r, g, b) => {
            println!("色: RGB({}, {}, {})", r, g, b);
        }
    }
}

fn main() {
    let messages = vec![
        Message::Echo(String::from("Hello!")),
        Message::Move { x: 10, y: 20 },
        Message::Color(255, 128, 0),
        Message::Quit,
    ];

    for msg in messages {
        process_message(msg);
    }
}
```

**実行結果：**
```
エコー: Hello!
移動: (10, 20)
色: RGB(255, 128, 0)
終了します
```

### 6.4.3 enumにメソッドを実装

構造体と同様に、enumにもメソッドを定義できます。

```rust
enum TrafficLight {
    Red,
    Yellow,
    Green,
}

impl TrafficLight {
    fn duration(&self) -> u32 {
        match self {
            TrafficLight::Red    => 60,
            TrafficLight::Yellow => 5,
            TrafficLight::Green  => 45,
        }
    }

    fn next(&self) -> TrafficLight {
        match self {
            TrafficLight::Red    => TrafficLight::Green,
            TrafficLight::Yellow => TrafficLight::Red,
            TrafficLight::Green  => TrafficLight::Yellow,
        }
    }

    fn name(&self) -> &str {
        match self {
            TrafficLight::Red    => "赤",
            TrafficLight::Yellow => "黄",
            TrafficLight::Green  => "青",
        }
    }
}

fn main() {
    let mut light = TrafficLight::Red;

    for _ in 0..6 {
        println!("信号: {} ({}秒)", light.name(), light.duration());
        light = light.next();
    }
}
```

**実行結果：**
```
信号: 赤 (60秒)
信号: 青 (45秒)
信号: 黄 (5秒)
信号: 赤 (60秒)
信号: 青 (45秒)
信号: 黄 (5秒)
```

### ポイントまとめ

- `enum` は取り得る値のバリアントを列挙する型
- 各バリアントは異なる型のデータを持てる（Rustの強力な機能）
- `match` で全バリアントを網羅的に処理できる
- `impl` でメソッドを定義できる

## 6.5 Option<T> 型

### 6.5.1 Null がない言語

Rustには `null` がありません。代わりに「値がないかもしれない」ことを `Option<T>` 型で表現します。これは標準ライブラリで以下のように定義されたenumです。

```
┌──────────────────────────────────┐
│  enum Option<T> {                │
│      Some(T),   // 値がある       │
│      None,      // 値がない       │
│  }                               │
│                                  │
│  Option<i32>の例:                │
│    Some(42)  → 42という値がある   │
│    None      → 値がない           │
└──────────────────────────────────┘
```

```rust
fn main() {
```

### 6.5.4 Optionの便利メソッド

```rust
fn main() {
    let some_val: Option<i32> = Some(42);
    let none_val: Option<i32> = None;

    // unwrap_or: Noneの場合のデフォルト値
    println!("unwrap_or: {}", some_val.unwrap_or(0));
    println!("unwrap_or: {}", none_val.unwrap_or(0));

    // unwrap_or_else: Noneの場合にクロージャで値を生成
    let default = || {
        println!("デフォルト値を計算中...");
        -1
    };
    println!("unwrap_or_else: {}", none_val.unwrap_or_else(default));

    // map: Someの中身を変換
    let doubled = some_val.map(|x| x * 2);
    let doubled_none = none_val.map(|x| x * 2);
    println!("map Some: {:?}", doubled);     // Some(84)
    println!("map None: {:?}", doubled_none); // None

    // and_then: Someの中身で別のOptionを返す（チェーン）
    let result = some_val
        .map(|x| x * 2)
        .and_then(|x| if x > 50 { Some(x) } else { None })
        .unwrap_or(0);
    println!("チェーン結果: {}", result);

    // is_some / is_none
    println!("some_val is Some: {}", some_val.is_some());
    println!("none_val is None: {}", none_val.is_none());
}
```

**実行結果：**
```
unwrap_or: 42
unwrap_or: 0
デフォルト値を計算中...
unwrap_or_else: -1
map Some: Some(84)
map None: None
チェーン結果: 84
some_val is Some: true
none_val is None: true
```

### 6.6.1 ネストしたパターンマッチ

```rust
enum Shape {
    Circle(f64),
    Rectangle(f64, f64),
    Triangle(f64, f64, f64),  // 3辺の長さ
}

fn describe_shape(shape: &Shape) -> String {
    match shape {
        Shape::Circle(r) if *r <= 0.0 => {
            String::from("無効な円")
        }
        Shape::Circle(r) => {
            format!("半径{}の円（面積: {:.2}）",
                r, std::f64::consts::PI * r * r)
        }
        Shape::Rectangle(w, h) if (w - h).abs() < f64::EPSILON => {
            format!("一辺{}の正方形（面積: {:.2}）", w, w * h)
        }
        Shape::Rectangle(w, h) => {
            format!("{}x{}の長方形（面積: {:.2}）", w, h, w * h)
        }
        Shape::Triangle(a, b, c) => {
            // ヘロンの公式
            let s = (a + b + c) / 2.0;
            let area = (s * (s - a) * (s - b) * (s - c)).sqrt();
            format!("三辺({},{},{})の三角形（面積: {:.2}）", a, b, c, area)
        }
    }
}

        Shape::Circle(5.0),
        Shape::Rectangle(4.0, 4.0),
        Shape::Rectangle(3.0, 7.0),
        Shape::Triangle(3.0, 4.0, 5.0),
    ];

    for shape in &shapes {
        println!("{}", describe_shape(shape));
    }
}
```

## 6.7 Debug と Display トレイトの基礎

### 6.7.1 #[derive(Debug)]

`{:?}` フォーマットで構造体やenumの内容を表示するには、`Debug` トレイトが必要です。`#[derive(Debug)]` を付けると自動実装されます。

```rust
#[derive(Debug)]
struct Point {
    x: f64,
    y: f64,
}

#[derive(Debug)]
enum Color {
    Red,
    Green,
    Blue,
    Custom(u8, u8, u8),
}

fn main() {
    let p = Point { x: 3.0, y: 4.0 };
    let c1 = Color::Red;
    let c2 = Color::Custom(128, 64, 255);

    // {:?} で Debug表示
    println!("{:?}", p);
    println!("{:?}", c1);
    println!("{:?}", c2);

    // {:#?} で整形表示（Pretty Print）
    println!("{:#?}", p);
}
```

**実行結果：**
```
Point { x: 3.0, y: 4.0 }
Red
Custom(128, 64, 255)
Point {
    x: 3.0,
    y: 4.0,
}
```

### 6.7.2 Display トレイトの手動実装

ユーザー向けの表示には `Display` トレイトを実装します。`{}` フォーマットで使用されます。

```rust
use std::fmt;

struct Temperature {
    celsius: f64,
}

impl Temperature {
    fn new(celsius: f64) -> Temperature {
        Temperature { celsius }
    }

    fn to_fahrenheit(&self) -> f64 {
        self.celsius * 9.0 / 5.0 + 32.0
    }
}

// Displayトレイトの実装
impl fmt::Display for Temperature {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "{:.1}°C ({:.1}°F)", self.celsius, self.to_fahrenheit())
    }
}

// Debugはderiveで自動実装
#[derive(Debug)]
struct Point {
    x: f64,
    y: f64,
}

impl fmt::Display for Point {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "({}, {})", self.x, self.y)
    }
}

fn main() {
    let temp = Temperature::new(36.5);
    let point = Point { x: 3.0, y: 4.0 };

    // Display: {} で表示
    println!("体温: {}", temp);
    println!("座標: {}", point);

    // Debug: {:?} で表示
    println!("Debug: {:?}", point);

    // Displayを実装するとto_string()も使える
    let s = temp.to_string();
    println!("文字列: {}", s);
}
```

**実行結果：**
```
体温: 36.5°C (97.7°F)
座標: (3, 4)
Debug: Point { x: 3.0, y: 4.0 }
文字列: 36.5°C (97.7°F)
```

### 6.7.3 enumにDisplayを実装

```rust
use std::fmt;

#[derive(Debug)]
enum Suit {
    Spade,
    Heart,
    Diamond,
    Club,
}

#[derive(Debug)]
struct Card {
    suit: Suit,
    rank: u8,  // 1=A, 11=J, 12=Q, 13=K
}

impl fmt::Display for Suit {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Suit::Spade   => write!(f, "♠"),
            Suit::Heart   => write!(f, "♥"),
            Suit::Diamond => write!(f, "♦"),
            Suit::Club    => write!(f, "♣"),
        }
    }
}

impl fmt::Display for Card {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        let rank_str = match self.rank {
            1  => String::from("A"),
            11 => String::from("J"),
            12 => String::from("Q"),
            13 => String::from("K"),
            n  => n.to_string(),
        };
        write!(f, "{}{}", self.suit, rank_str)
    }
}

impl Card {
    fn new(suit: Suit, rank: u8) -> Card {
        Card { suit, rank }
    }
}

fn main() {
    let hand = vec![
        Card::new(Suit::Spade, 1),
        Card::new(Suit::Heart, 13),
        Card::new(Suit::Diamond, 7),
        Card::new(Suit::Club, 10),
    ];

    print!("手札: ");
    for (i, card) in hand.iter().enumerate() {
        if i > 0 { print!(", "); }
        print!("{}", card);
    }
    println!();

    // Debug表示
    println!("Debug: {:?}", hand[0]);
}
```

**実行結果：**
```
手札: ♠A, ♥K, ♦7, ♣10
Debug: Card { suit: Spade, rank: 1 }
```

### 6.7.4 よく使うderiveマクロ

```rust
// よく使うderiveの組み合わせ
#[derive(Debug, Clone, PartialEq)]
struct Point {
    x: f64,
    y: f64,

fn main() {

    let p2 = p1.clone();

    // PartialEq: == で比較
    println!("p1 == p2: {}", p1 == p2);

    let p3 = Point { x: 3.0, y: 4.0 };
```

### ポイントまとめ

| トレイト | 用途 | フォーマット | 自動実装 |
|---|---|---|---|
| `Debug` | デバッグ表示 | `{:?}`, `{:#?}` | `#[derive(Debug)]` |
| `Display` | ユーザー向け表示 | `{}` | 手動実装が必要 |
| `Clone` | 明示的な複製 | `.clone()` | `#[derive(Clone)]` |
| `PartialEq` | 等値比較 | `==`, `!=` | `#[derive(PartialEq)]` |

## 6.8 実践例：タスク管理システム

学んだ知識を組み合わせた実践的な例を見てみましょう。

```rust
use std::fmt;

// タスクの優先度
#[derive(Debug, Clone, PartialEq)]
enum Priority {
    Low,
    Medium,
    High,
    Critical,
}

impl fmt::Display for Priority {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Priority::Low      => write!(f, "低"),
            Priority::Medium   => write!(f, "中"),
            Priority::High     => write!(f, "高"),
            Priority::Critical => write!(f, "緊急"),
        }
    }
}

// タスクの状態
#[derive(Debug, Clone, PartialEq)]
enum Status {
    Todo,
    InProgress,
    Done,
    Cancelled(String),  // キャンセル理由を持つ
}

impl fmt::Display for Status {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Status::Todo       => write!(f, "未着手"),
            Status::InProgress => write!(f, "進行中"),
            Status::Done       => write!(f, "完了"),
            Status::Cancelled(reason) => write!(f, "中止({})", reason),
        }
    }
}

// タスク
#[derive(Debug, Clone)]
struct Task {
    id: u32,
    title: String,
    priority: Priority,
    status: Status,
    assignee: Option<String>,
}

impl Task {
    fn new(id: u32, title: &str, priority: Priority) -> Task {
        Task {
            id,
            title: String::from(title),
            priority,
            status: Status::Todo,
            assignee: None,
        }
    }

    fn assign(&mut self, name: &str) {
        self.assignee = Some(String::from(name));
        self.status = Status::InProgress;
    }

    fn complete(&mut self) {
        self.status = Status::Done;
    }

    fn cancel(&mut self, reason: &str) {
        self.status = Status::Cancelled(String::from(reason));
    }

    fn is_active(&self) -> bool {
        matches!(self.status, Status::Todo | Status::InProgress)
    }
}

impl fmt::Display for Task {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        let assignee = self.assignee.as_deref().unwrap_or("未割り当て");
        write!(f, "[#{}] {} | 優先度:{} | 状態:{} | 担当:{}",
            self.id, self.title, self.priority, self.status, assignee)
    }
}

// タスクリスト
struct TaskList {
    tasks: Vec<Task>,
    next_id: u32,
}

impl TaskList {
    fn new() -> TaskList {
        TaskList {
            tasks: Vec::new(),
            next_id: 1,
        }
    }

    fn add(&mut self, title: &str, priority: Priority) -> u32 {
        let id = self.next_id;
        self.tasks.push(Task::new(id, title, priority));
        self.next_id += 1;
        id
    }

    fn find_mut(&mut self, id: u32) -> Option<&mut Task> {
        self.tasks.iter_mut().find(|t| t.id == id)
    }

    fn active_tasks(&self) -> Vec<&Task> {
        self.tasks.iter().filter(|t| t.is_active()).collect()
    }

    fn count_by_status(&self) -> (usize, usize, usize, usize) {
        let mut todo = 0;
        let mut in_progress = 0;
        let mut done = 0;
        let mut cancelled = 0;
        for task in &self.tasks {
            match task.status {
                Status::Todo       => todo += 1,
                Status::InProgress => in_progress += 1,
                Status::Done       => done += 1,
                Status::Cancelled(_) => cancelled += 1,
            }
        }
        (todo, in_progress, done, cancelled)
    }

    fn display_all(&self) {
        println!("=== タスク一覧 ({}) ===", self.tasks.len());
        for task in &self.tasks {
            println!("  {}", task);
        }
        let (todo, prog, done, cancel) = self.count_by_status();
        println!("--- 未着手:{} | 進行中:{} | 完了:{} | 中止:{} ---",
            todo, prog, done, cancel);
    }
}

fn main() {
    let mut list = TaskList::new();

    // タスクを追加
    let id1 = list.add("ログイン機能の実装", Priority::High);
    let id2 = list.add("READMEの更新", Priority::Low);
    let id3 = list.add("セキュリティ脆弱性の修正", Priority::Critical);
    let id4 = list.add("UIデザインの改善", Priority::Medium);

    // タスクの状態を更新
    if let Some(task) = list.find_mut(id1) {
        task.assign("田中");
    }
    if let Some(task) = list.find_mut(id3) {
        task.assign("佐藤");
        task.complete();
    }
    if let Some(task) = list.find_mut(id2) {
        task.cancel("優先度見直しにより延期");
    }

    // 一覧表示
    list.display_all();

    println!();

    // アクティブなタスクのみ表示
    println!("=== アクティブなタスク ===");
    for task in list.active_tasks() {
        println!("  {}", task);
    }
}
```

**実行結果：**
```
=== タスク一覧 (4) ===
  [#1] ログイン機能の実装 | 優先度:高 | 状態:進行中 | 担当:田中
  [#2] READMEの更新 | 優先度:低 | 状態:中止(優先度見直しにより延期) | 担当:未割り当て
```
