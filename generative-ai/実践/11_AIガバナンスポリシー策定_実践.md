## ステップガイド

<details>
<summary>ステップ1：AIガバナンスの基本原則を定義する</summary>

組織のAI利用を方向付ける基本原則を定めましょう。

国内外のAIガバナンスフ��ームワークを参考にします。
- 経済産業省「AI事業者ガイドライン」
- OECD AI原則
- EU AI Act

一般的な原則の例：
1. **透明性（Transparency）**: AI利用を明示する
2. **公平性（Fairness）**: バイアスを最小限に抑える
3. **説明責任（Accountability）**: AI利用の結果に責任を持つ
4. **安全性（Safety）**: 情報漏洩やリスクを防止する
5. **人間中心（Human-Centric）**: 最終判断は人間が行う

</details>

<details>
<summary>ステップ2：データ分類とAI利用ルールを定義する</summary>

社内データを機密レベルで分類し、各レベルのAI利用ルールを定めましょう。

```
機密レベル → AI利用可否 → 使用可能なAIサービス
```

分類の例：
- レベル1（公開情報）: 制限なし
- レベル2（社内限定）: 利用可能（条件付き）
- レベル3（機密）: 社内AIのみ利用可能
- レベル4（極秘）: AI利用禁止

**よくある間違い**: 「すべて禁止」にしてしまうこと。禁止すると社員は隠れて使い、かえってリスクが高まります（シャドーIT（Shadow IT））。

</details>

<details>
<summary>ステップ3：ガイドラインの具体的なルールを策定する</summary>

以下の観点でルールを策定しましょう。

- **利用可能なAIサービスのリスト（ホワイトリスト）**
- **入力禁止情報のリスト**
- **出力結果の扱い（レビュー必須、著作権表示等）**
- **利用記録の保持**
- **違反時のペナルティ**

実効性のあるルールにするため、「禁止」だけでなく「推奨される使い方」も示しましょう。

</details>

<details>
<summary>ステップ4：���ンシデント対応と見直しサイクルを設計する</summary>

**インシデント対応:**
- 発見 → 報告 → 初動対応 → 調査 → 再発防止 の流れ
- 報告ルート（誰に、何時間以内に）
- 深刻度の分類と対応レベル

**定期見直し:**
- 見直しの頻度（四半期ごと推奨）
- 見直し観点（技術動向、法規制変更、インシデント事例）
- 見直しの承認フロー

### レベル1：AIガバナンス基本方針

---

**株式会社〇〇 AIガバナンス基本方針**

制定日：2026年4月1日
最終更新：2026年4月1日
承認者：代表取締役社長

#### 1. 目的

本方針は、当社における生成AI（Generative AI）の利用に関する基本的な考え方を定め、安全かつ効果的なAI活用を推進することを目的とします。

#### 2. 基本原則

当社のAI利用は、以下の5つの原則に基づきます。

1. **安全性の確保**: 情報セキュリティを最優先し、機密情報の保護を徹底します
2. **透明性の担保**: AI利用であることを適切に開示し、隠蔽しません
3. **人間中心の意思決定**: AIの出力は参考情報とし、最終判断は必ず人間が行います
4. **公平性の追求**: AIによるバイアスや差別を防止する措置を講じます
5. **継続的改善**: 技術動向や法規制の変化に応じて、本方針を定期的に見直します

#### 3. 適用範囲

本方針は、当社の全役員・全従業員（派遣社員・業務委託先を含む）に適用されます。

#### 4. 推進体制

```

#### ポリシー成熟度モデル

```
レベル1：反応的（Reactive）← 現在地
  「問題が起きてから対応」
  ・ルールなし、個人任せ
  ・インシデント発生で慌てて対応

     ▼ Phase 1後

レベル2：基礎的（Basic）
  「最低限のルールがある」
  ・利用ガイドラインが存在

  ・教育が年1回実施

     ▼ Phase 2後

  「体系的なガバナンスが機能」
  ・リスクアセスメントが定着
  ・承認プロセスが運用中
  ・定期的な監査を実施
  ・KPIで効果を測定

     ▼ Phase 3後

レベル4：最適化（Optimized）
  「AIガバナンスが競争力に」
  ・自動化された監視・検知
  ・フィードバックによる継続改善
  ・業界のベストプラクティスをリード
  ・外部認証の取得
