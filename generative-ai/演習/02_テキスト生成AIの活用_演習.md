# 第2章 演習：テキスト生成AIの活用

## 基本問題

### 問題1：テキスト生成AIの適切な活用シーンの判断

以下の業務シーンについて、テキスト生成AIの活用が「適切」「条件付きで適切」「不適切」のいずれかを判断し、理由を述べてください。

1. 社内向けの週次報告メールの下書き作成
2. 患者への処方薬の服用量の決定
3. プログラムのエラーメッセージの原因調査と修正案の提示
4. 裁判の判決文の作成
5. ブレインストーミング会議でのアイデア出し
6. 社外向けプレスリリースの草稿作成
7. 顧客の個人情報を含むデータベースの分析結果の要約
8. 新入社員向け研修資料のドラフト作成

**期待される出力例：**
```
1. 適切
   → 社内向けの定型的な報告であり、人間がレビューしてから送信すればリスクは低い。

2. 不適切
   → 医療行為に関わる判断であり、誤りが人命に直結する。
     医師の専門知識と判断が不可欠。
...
```

<details>
<summary>解答例</summary>

```
1. 適切
   → 社内向けの定型報告であり、ポイントを入力すればAIが文章に整えてくれます。
     最終確認を人間が行えば十分実用的です。

2. 不適切
   → 医療行為に関わる意思決定であり、AIの誤りが患者の健康・生命に直結します。
     処方は必ず医師が行うべきであり、AIはあくまで参考情報の提供に留めるべきです。

3. 条件付きで適切
   → エラーの原因分析や修正案の提示にAIは有用ですが、
     提案されたコードを検証せずに本番環境に適用するのは危険です。
     テスト環境での動作確認を必ず行いましょう。

4. 不適切
   → 法的拘束力を持つ文書であり、誤りが重大な権利侵害につながります。
     法律の解釈は専門家（裁判官・弁護士）が行うべきです。
     ただし、判例調査の補助としてなら活用の余地はあります。

5. 適切
   → アイデアの発散フェーズではAIの多様な提案が有効です。
     「正解」を求めるタスクではないため、ハルシネーションのリスクも低いです。

6. 条件付きで適切
   → 草稿作成の効率化にAIは有効ですが、社外向け文書のため
     事実確認・法務チェック・広報部門の承認プロセスが必須です。
     数値データや固有名詞は特に入念な確認が必要です。

7. 条件付きで適切
   → 個人情報をAIに入力する際はセキュリティリスクがあります。
     データの匿名化を事前に行い、API経由（学習に使用されない設定）で
     利用することが前提条件です。社内規定やプライバシーポリシーの遵守が必須です。

8. 適切
   → 研修資料のドラフト作成はAIの得意分野です。
     構成の提案、説明文の作成、クイズの生成などに活用できます。
     専門知識の正確性は人間が最終確認してください。
```
</details>

### 問題2：基本的なプロンプトの構造理解

以下の要素を正しい順序に並べ替えて、効果的なプロンプトの構造を作成してください。

**要素（順不同）：**
- A: 「日本語で回答してください」
- B: 「あなたは10年の経験を持つマーケティング専門家です」
- C: 「以下の商品情報を基に、SNS投稿文を3パターン作成してください」
- D: 「商品名: エコボトル / 特徴: 再生素材100% / 価格: 2,980円 / ターゲット: 20-30代女性」
- E: 「各パターンは100文字以内、ハッシュタグを3つ含めること」

**期待される出力例：**
```
1番目: B（役割設定）
2番目: C（タスク指示）
3番目: D（入力情報）
4番目: E（出力条件）
5番目: A（言語指定）
```

<details>
<summary>ヒント</summary>

効果的なプロンプトの基本構造は次のとおりです。
1. 役割（Role）: AIにどのような専門家として振る舞うかを指定
2. タスク（Task）: 何をしてほしいかを明確に指示
3. コンテキスト（Context）: 参考にすべき情報を提供
4. 制約条件（Constraints）: 出力形式や制限を指定
5. その他の指定: 言語、トーンなど
</details>

<details>
<summary>解答例</summary>

```
1番目: B（役割設定 / Role）
   「あなたは10年の経験を持つマーケティング専門家です」
   → 最初にAIの役割を定義することで、回答の方向性と専門性が決まります。

2番目: C（タスク指示 / Task）
   「以下の商品情報を基に、SNS投稿文を3パターン作成してください」
   → 具体的に何をしてほしいかを明確に伝えます。

3番目: D（入力情報 / Context）
   「商品名: エコボトル / 特徴: 再生素材100% / ...」
   → タスクを遂行するために必要な情報を構造化して提供します。

4番目: E（出力条件 / Constraints）
   「各パターンは100文字以内、ハッシュタグを3つ含めること」
   → 出力の形式や品質に関する制約を指定します。

5番目: A（言語指定）
   「日本語で回答してください」
   → 出力言語の指定です。明示しなくても推測されますが、
     多言語環境では明示が重要です。

【組み立てた完成プロンプト】
---
あなたは10年の経験を持つマーケティング専門家です。

以下の商品情報を基に、SNS投稿文を3パターン作成してください。

商品名: エコボトル
特徴: 再生素材100%
価格: 2,980円
ターゲット: 20-30代女性

各パターンは100文字以内、ハッシュタグを3つ含めてください。

日本語で回答してください。
---
```
</details>

### 問題3：プロンプトの改善

以下の「悪いプロンプト」を改善してください。何が問題かを説明し、改善版を作成してください。

**悪いプロンプト①：**
```
いい感じのメール書いて
```

**悪いプロンプト②：**
```
この文章を要約して。あと翻訳もして。ついでに改善点も教えて。
文章: 弊社の新サービスについてご説明いたします...（長文）
```

**悪いプロンプト③：**
```
Pythonのコードを書いてください
```

**期待される出力例：**
```
【悪いプロンプト①の問題点】
- メールの目的、宛先、トーンが不明
- 「いい感じ」は主観的で曖昧

【改善版①】
あなたはビジネスマナーに精通した社会人です。
以下の条件でビジネスメールを作成してください。
...
```

<details>
<summary>ヒント</summary>

プロンプト改善の5つのチェックポイント:
1. 目的は明確か？
2. 必要な情報（コンテキスト）は提供されているか？
3. 出力形式は指定されているか？
4. タスクは1つに絞られているか（複数ある場合は優先順位がついているか）？
5. 曖昧な表現を使っていないか？
</details>

<details>
<summary>解答例</summary>

```

- メールの目的（お礼？依頼？謝罪？）が不明
- 宛先（上司？取引先？同僚？）が不明
- 「いい感じ」が曖昧で、品質基準がない
- 必要な情報がまったく提供されていない

【改善版①】
あなたはビジネスマナーに精通した社会人です。
以下の条件でビジネスメールを作成してください。

目的: プロジェクト完了のお礼
宛先: 取引先の担当者（山田様）
関係性: 半年間のプロジェクトで協力いただいた
含める内容:
  - プロジェクト成功への感謝
  - 具体的に助かった点（納期調整への柔軟な対応）
  - 今後の関係継続への期待
トーン: 丁寧かつ温かみのある表現
長さ: 200〜300文字程度

---

【悪いプロンプト②の問題点】
- 1つのプロンプトに3つのタスク（要約・翻訳・改善）を同時に要求
- 各タスクの優先順位や出力形式が不明確
- 翻訳先の言語が指定されていない
- 要約の長さが指定されていない

【改善版②】
以下の文章について、3つのタスクを順番に実行してください。

## タスク1: 要約
- 元の文章を3〜5文で要約してください
- 重要なポイントを箇条書きで示してください

## タスク2: 英語翻訳
- 要約した内容を英語に翻訳してください
- ビジネス英語のトーンで記述してください

## タスク3: 改善提案
- 元の文章の改善点を3つ挙げてください
- 各改善点に具体的な修正例を示してください

文章:
弊社の新サービスについてご説明いたします...（長文）

---

【悪いプロンプト③の問題点】
- 何を実現するコードか不明
- 入出力の仕様がない
- 使用するライブラリの制約がない
- 対象のPythonバージョンが不明

【改善版③】
Python 3.12で動作するスクリプトを作成してください。

目的: CSVファイルを読み込み、月別の売上合計を集計する

入力:
  - CSVファイル（列: date, product_name, amount）
  - dateの形式: YYYY-MM-DD

出力:
  - 月別（YYYY-MM形式）の売上合計を表示
  - 合計金額が大きい順にソート

制約:
  - 標準ライブラリのみ使用（csvモジュール）
  - エラーハンドリングを含める（ファイル不在、不正データ）
  - コメントで処理内容を説明
```
</details>

### 問題6：文章要約タスクのプロンプト設計

以下の長文を「3種類の要約」に変換するプロンプトをそれぞれ作成してください。

```
デジタルトランスフォーメーション（DX）は、企業がデジタル技術を活用して、
ビジネスモデルや業務プロセスを根本的に変革する取り組みです。
経済産業省の調査によると、日本企業の約70%がDXに取り組んでいるものの、
成果を実感している企業は約30%にとどまっています。

DX推進の主な課題として、レガシーシステムの存在、DX人材の不足、
```

### 問題9：プロンプトチェーニングによる複雑タスクの分解

「新規サービスの企画書作成」という複雑なタスクを、テキスト生成AIへの複数回のプロンプト（チェーン）に分解してください。

**タスクの前提：**
```
あなたはスタートアップ企業の企画担当者です。
「シニア世代向けのオンライン趣味マッチングサービス」の企画書を
作成する必要があります。
```

**要件：**
- 最低4ステップのプロンプトチェーンを設計すること
- 各ステップの入力（前のステップの出力を含む）と出力を明確にすること
- なぜこの順序でタスクを分解したのか理由を述べること

**期待される出力例：**
```
【プロンプトチェーン設計】

ステップ1: 市場調査・ニーズ分析
  入力: サービスの基本コンセプト
  プロンプト: ...
  出力: ターゲット分析と市場規模の概算

ステップ2: ...
```

<details>
<summary>ヒント</summary>

- プロンプトチェーニングとは、複雑なタスクを小さなステップに分割し、各ステップの出力を次のステップの入力に使う手法です。
- 企画書作成の一般的なフロー: 調査 → アイデア → 設計 → 計画 → 文書化
- 各ステップを独立させることで、途中で方向修正がしやすくなります。
</details>

<summary>解答例</summary>

```
効果的です。各ステップを独立させることで以下のメリットがあります。
  - 各ステップの出力品質を個別に確認・修正できる
  - 方向性が間違っていた場合、該当ステップからやり直せる
```
