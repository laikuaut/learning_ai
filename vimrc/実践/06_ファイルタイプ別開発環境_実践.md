# 実践課題06：ファイルタイプ別開発環境 ★3

> **難易度**: ★★★☆☆（中級）
> **前提知識**: 第3章（インデント設定）、第5章（キーマッピング）、第6章（autocmdとファイルタイプ別設定）
> **課題の種類**: ミニプロジェクト
> **学習目標**: `autocmd` と `filetype` を活用して、Python・HTML・Makefileなど言語ごとに最適な設定を自動適用する仕組みを構築する

---

## 完成イメージ

ファイルを開くだけで、言語に応じた設定が自動適用されます。

```
Python (.py) を開いた場合:
┌──────────────────────────────────────────────┐
│  1 def hello():                              │
│  2 ····print("Hello")   ← 4スペースインデント│
│  3                                           │
│  ~                                           │
│ example.py [python] [utf-8]  indent:4sp      │
└──────────────────────────────────────────────┘

HTML (.html) を開いた場合:
┌──────────────────────────────────────────────┐
│  1 <div>                                     │
│  2 ··<p>Hello</p>       ← 2スペースインデント│
│  3 </div>                                    │
│  ~                                           │
│ index.html [html] [utf-8]  indent:2sp        │
└──────────────────────────────────────────────┘

Makefile を開いた場合:
┌──────────────────────────────────────────────┐
│  1 all:                                      │
│  2 →   gcc main.c       ← タブ文字          │
│  3                                           │
│  ~                                           │
│ Makefile [make] [utf-8]  indent:tab          │
└──────────────────────────────────────────────┘
```

---

## 課題の要件

1. `augroup` を使ってautocmdをグループ化する
2. Python用の設定を定義する（インデント4スペース、行の長さ目安表示）
3. HTML/CSS用の設定を定義する（インデント2スペース）
4. JavaScript/TypeScript用の設定を定義する（インデント2スペース）
5. Makefile用の設定を定義する（タブ文字を使用）
6. Markdown用の設定を定義する（折り返し、スペルチェック）
7. ファイル保存時に行末の空白を自動削除する

---

## ステップガイド

<details>
<summary>ステップ1：augroupの基本構造</summary>

`autocmd` を書くとき、必ず `augroup` で囲みます。これをしないと、vimrcを再読み込みするたびに同じ `autocmd` が重複登録されてしまいます。

```vim
" augroup のテンプレート
augroup MyFileTypes
    autocmd!              " グループ内の既存autocmdをすべて削除
    autocmd FileType python  setlocal ...
    autocmd FileType html    setlocal ...
augroup END
```

**`autocmd!`（感嘆符付き）を必ず書きましょう。** これがないと再読み込みで設定が蓄積します。

</details>

<details>
<summary>ステップ2：Python用設定</summary>

```vim
augroup PythonSettings
    autocmd!
    autocmd FileType python setlocal tabstop=4
    autocmd FileType python setlocal shiftwidth=4
    autocmd FileType python setlocal softtabstop=4
    autocmd FileType python setlocal expandtab
    autocmd FileType python setlocal colorcolumn=80
    autocmd FileType python setlocal textwidth=79
augroup END
```

- `colorcolumn=80`: 80列目にガイドラインを表示（PEP 8の推奨）
- `textwidth=79`: 自動折り返しの目安
- `setlocal` を使うとバッファ固有の設定になります（`set` だとグローバルに影響）

</details>

<details>
<summary>ステップ3：HTML/CSS/JavaScript用設定</summary>

```vim
augroup WebSettings
    autocmd!
    " HTML
    autocmd FileType html setlocal tabstop=2 shiftwidth=2 softtabstop=2 expandtab

    " CSS
    autocmd FileType css setlocal tabstop=2 shiftwidth=2 softtabstop=2 expandtab

    " JavaScript / TypeScript
    autocmd FileType javascript,typescript setlocal tabstop=2 shiftwidth=2 softtabstop=2 expandtab
    autocmd FileType javascript,typescript setlocal colorcolumn=100
augroup END
```

カンマ区切りで複数のファイルタイプに同じ設定を適用できます。

</details>

<details>
<summary>ステップ4：Makefile用設定</summary>

```vim
augroup MakefileSettings
    autocmd!
    " Makefileではタブ文字が必須
    autocmd FileType make setlocal noexpandtab
    autocmd FileType make setlocal tabstop=8
    autocmd FileType make setlocal shiftwidth=8
augroup END
```

**重要**: Makefileのインデントは**タブ文字でなければならない**というルールがあります。スペースを使うとエラーになるため、`noexpandtab` が必須です。

</details>

<details>
<summary>ステップ5：Markdown用設定</summary>

```vim
augroup MarkdownSettings
    autocmd!
    autocmd FileType markdown setlocal wrap
    autocmd FileType markdown setlocal linebreak
    autocmd FileType markdown setlocal spell spelllang=en,cjk
    autocmd FileType markdown setlocal conceallevel=0
    autocmd FileType markdown setlocal tabstop=2 shiftwidth=2 expandtab
augroup END
```

- `spell`: スペルチェックを有効化
- `spelllang=en,cjk`: 英語のスペルをチェック、CJK（日本語等）は無視
- `conceallevel=0`: Markdownの記号（`**`等）を隠さずに表示

</details>

<details>
<summary>ステップ6：ファイル保存時の自動処理</summary>

```vim
augroup AutoCommands
    autocmd!
    " 保存時に行末の空白を自動削除
    autocmd BufWritePre * %s/\s\+$//e

    " 保存時にファイル末尾の空行を削除
    autocmd BufWritePre * %s/\($\n\s*\)\+\%$//e
augroup END
```

`BufWritePre` はファイルが保存される直前に実行されるイベントです。`e` フラグはパターンが見つからなくてもエラーにしないための指定です。

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

```vim
" ===========================================
" ファイルタイプ別設定
" ===========================================

set nocompatible
filetype plugin indent on
syntax enable

" --- デフォルトのインデント ---
set tabstop=4
set shiftwidth=4
set softtabstop=4
set expandtab
set autoindent

" --- Python ---
augroup PythonSettings
    autocmd!
    autocmd FileType python setlocal tabstop=4 shiftwidth=4 expandtab
    autocmd FileType python setlocal colorcolumn=80
augroup END

" --- Web系（HTML/CSS/JS/TS）---
augroup WebSettings
    autocmd!
    autocmd FileType html,css setlocal tabstop=2 shiftwidth=2 expandtab
    autocmd FileType javascript,typescript setlocal tabstop=2 shiftwidth=2 expandtab
augroup END

" --- Makefile ---
augroup MakefileSettings
    autocmd!
    autocmd FileType make setlocal noexpandtab tabstop=8 shiftwidth=8
augroup END

" --- Markdown ---
augroup MarkdownSettings
    autocmd!
    autocmd FileType markdown setlocal wrap linebreak spell spelllang=en,cjk
augroup END

" --- 保存時の自動処理 ---
augroup AutoTrim
    autocmd!
    autocmd BufWritePre * %s/\s\+$//e
augroup END
```

<details>
<summary>解答例（改良版：ヘルパー関数と拡張設定）</summary>

```vim
" ===========================================
" ファイルタイプ別設定 - 改良版
" ===========================================

set nocompatible
filetype plugin indent on
syntax enable

" --- デフォルト ---
set tabstop=4 shiftwidth=4 softtabstop=4 expandtab autoindent

" ------------------------------------------
" インデント設定のヘルパー関数
" ------------------------------------------
function! s:SetIndent(width, use_tabs) abort
    let &l:tabstop = a:use_tabs ? a:width : a:width
    let &l:shiftwidth = a:width
    let &l:softtabstop = a:width
    let &l:expandtab = !a:use_tabs
endfunction

" ------------------------------------------
" ファイルタイプ別設定
" ------------------------------------------
augroup MyFileTypeSettings
    autocmd!

    " --- Python ---
    autocmd FileType python call s:SetIndent(4, 0)
    autocmd FileType python setlocal colorcolumn=80 textwidth=79
    autocmd FileType python setlocal formatoptions+=r  " コメント継続

    " --- Web系 ---
    autocmd FileType html,css,scss,vue call s:SetIndent(2, 0)
    autocmd FileType javascript,typescript,json call s:SetIndent(2, 0)
    autocmd FileType javascript,typescript setlocal colorcolumn=100

    " --- Makefile ---
    autocmd FileType make call s:SetIndent(8, 1)

    " --- Shell script ---
    autocmd FileType sh,bash,zsh call s:SetIndent(4, 0)
    autocmd FileType sh,bash,zsh setlocal colorcolumn=80

    " --- Markdown ---
    autocmd FileType markdown call s:SetIndent(2, 0)
    autocmd FileType markdown setlocal wrap linebreak
    autocmd FileType markdown setlocal spell spelllang=en,cjk
    autocmd FileType markdown setlocal conceallevel=0

    " --- Ruby ---
    autocmd FileType ruby call s:SetIndent(2, 0)

    " --- Go ---
    autocmd FileType go call s:SetIndent(4, 1)

    " --- YAML ---
    autocmd FileType yaml call s:SetIndent(2, 0)
    autocmd FileType yaml setlocal cursorcolumn  " インデントミスを発見しやすく

augroup END

" ------------------------------------------
" 自動処理
" ------------------------------------------
augroup AutoCommands
    autocmd!
    " 保存時に行末空白を削除（ただしMarkdownは除外）
    autocmd BufWritePre * if &filetype !=# 'markdown' | %s/\s\+$//e | endif

    " 新しいファイルを開いたときにディレクトリを自動作成
    autocmd BufWritePre * call s:MkdirIfNeeded()

    " 前回の編集位置を復元
    autocmd BufReadPost *
        \ if line("'\"") >= 1 && line("'\"") <= line("$") |
        \     execute "normal! g`\"" |
        \ endif
augroup END

" ディレクトリが存在しなければ作成する
function! s:MkdirIfNeeded() abort
    let l:dir = expand('%:p:h')
    if !isdirectory(l:dir)
        call mkdir(l:dir, 'p')
    endif
endfunction
```

**初心者向けとの違い**:

- `s:SetIndent()` ヘルパー関数でインデント設定を簡潔に
- Go、Ruby、YAML、Shell script の設定を追加
- Markdownファイルは行末空白を削除しない（Markdownでは末尾スペース2つが改行の意味）
- ファイル保存時にディレクトリを自動作成
- 前回の編集位置を自動復元

</details>

## よくある間違い

| ミス | 正しい書き方 | 説明 |
|------|-------------|------|
| `augroup` 内で `autocmd!` を書き忘れ | `autocmd!` を先頭に書く | vimrc再読み込みで設定が重複する |
| `set` を使う | `setlocal` を使う | `set` は全バッファに影響する |
| `BufRead` で設定を書く | `FileType` イベントを使う | `FileType` の方がVimのファイルタイプ検出に従って正確 |
| Makefileで `expandtab` のまま | `noexpandtab` を設定 | Makefileはタブ文字が必須 |
