# 実践課題08：デバッグ演習 ─ ショッピングカート ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第1章〜第5章（基本・リアクティビティ・テンプレート構文・コンポーネント・Composition API）
> **課題の種類**: デバッグ
> **学習目標**: Vue.js特有のバグ（リアクティビティの罠、`v-for` のkey問題、`watch` の使い方、`computed` と `methods` の違い）を発見・修正する力を養う

## 課題の説明

以下の「ショッピングカート」アプリには **8個のバグ** が埋め込まれています。
ブラウザで開いて動作を確認しながら、すべてのバグを見つけて修正してください。

## バグ入りコード

```html

<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>ショッピングカート（バグ入り）</title>
  <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

      setup() {
        const products = ref([
          { id: 1, name: "ノートPC", price: 89000 },
          { id: 2, name: "マウス", price: 3500 },
          { id: 3, name: "キーボード", price: 12000 },
          { id: 4, name: "モニター", price: 45000 },

        const cart = ref([])

        // バグ6: addToCart の実装
        function addToCart(productId) {
          const product = products.value.find(p => p.id === productId)
          const existing = cart.value.find(item => item.id === productId)
          if (existing) {
            existing.quantity = existing.quantity + 1  // これは動くが...
          } else {
            cart.value.push({ id: product.id, name: product.name, price: product.price, quantity: 1 })
          }
        }

        function increment(id) {
          const item = cart.value.find(i => i.id === id)
          if (item) item.quantity++
        }

        // バグ7: decrement で0以下になる
        function decrement(id) {
          const item = cart.value.find(i => i.id === id)
          if (item) {
            item.quantity--
          }
        }

        // バグ8: removeItem の filter 条件が逆

          cart.value = cart.value.filter(item => item.id === id)

          products, cart,
          addToCart, increment, decrement, removeItem,
          totalPrice
        }
      }
```

## バグの種類

<details>
<summary>ヒント（軽め）：バグのカテゴリ</summary>

| カテゴリ | 件数 |
|---|:---:|
| 表示フォーマットの問題 | 1 |
| イベント引数の問題 | 1 |
| `v-for` のkey不足 | 1 |
| 重複表示の問題 | 1 |
| `computed` vs 関数 | 1 |
| ロジックエラー（数量） | 1 |
| ロジックエラー（filter条件） | 1 |
| `watch` のオプション不足 | 1 |

</details>

<details>
<summary>ヒント（詳細）：各バグの場所</summary>

| # | 場所 | ヒント |
|---|---|---|
| 1 | 商品一覧の価格表示 | `toLocaleString()` でカンマ区切りにすべき |
| 2 | `addToCart` の引数 | テンプレートでは `product.id` を渡しているが、関数側でproductオブジェクトを期待していないか確認 → 実は引数は`id`で正しいが... |
| 3 | カートの `v-for` | `:key` が指定されていない |
| 4 | カート小計の表示 | 小計がカンマ区切りでなく、名前欄に「×数量」と金額表示が重複 |
| 5 | `totalPrice` | `computed` ではなく通常関数として定義 → テンプレートで `totalPrice()` と呼ぶか `computed` にすべき |
| 6 | `totalPrice` 内部 | `item.price` だけで `quantity` を掛けていない |
| 7 | `decrement` | 数量が0以下になれる |
| 8 | `removeItem` の `filter` | `===` が `!==` であるべき |
| (追加) | `watch` | `{ deep: true }` が必要（配列内のオブジェクトの変更を検知するため） |

</details>

## 解答例

<details>
<summary>全バグの修正</summary>

### バグ1：価格表示のフォーマット

```html
<!-- 修正前 -->
<div>{{ product.price }}円</div>

<!-- 修正後 — toLocaleString()でカンマ区切り -->
<div>{{ product.price.toLocaleString() }}円</div>
```

### バグ3：v-for に key がない

```html
<!-- 修正前 — keyがないとVueが要素の再利用を正しくできない -->
<div class="cart-item" v-for="item in cart">

<!-- 修正後 -->
<div class="cart-item" v-for="item in cart" :key="item.id">
```

### バグ4：カート表示の重複と小計フォーマット

```html
<!-- 修正前 — 名前欄に×数量と金額欄に小計が両方表示 -->
<span class="name">{{ item.name }} ×{{ item.quantity }}</span>
<span>{{ item.price * item.quantity }}円</span>

<!-- 修正後 — 名前欄はシンプルに、小計にはカンマ区切り -->
<span class="name">{{ item.name }}</span>
<span>{{ (item.price * item.quantity).toLocaleString() }}円</span>
```

### バグ5・6：totalPrice を computed にし、quantity を掛ける

```js
// 修正前 — 通常関数かつquantity未考慮
function totalPrice() {
  return cart.value.reduce((sum, item) => sum + item.price, 0)
}

// 修正後 — computed にしてquantityを掛ける
const totalPrice = computed(() => {
  return cart.value.reduce((sum, item) => sum + item.price * item.quantity, 0)
})
```

テンプレート側も修正が必要です。

```html
<!-- 修正前 — 関数だった場合 totalPrice() と書く必要があるが... -->
<div class="total">合計: {{ totalPrice }}円</div>

<!-- 修正後 — computed なので .value は不要、テンプレートではそのまま使える -->
<div class="total">合計: {{ totalPrice.toLocaleString() }}円</div>
```

### バグ7：decrement で0以下になる

```js
// 修正前
function decrement(id) {
  const item = cart.value.find(i => i.id === id)
  if (item) {
    item.quantity--
  }
}

// 修正後 — 1未満にしない、0になったら削除
function decrement(id) {
  const item = cart.value.find(i => i.id === id)
  if (item) {
    if (item.quantity <= 1) {
      // 1個以下なら削除
      cart.value = cart.value.filter(i => i.id !== id)
    } else {
      item.quantity--
    }
  }
}
```

### バグ8：removeItem の filter 条件が逆

```js
// 修正前 — 削除対象だけ残してしまう
cart.value = cart.value.filter(item => item.id === id)

// 修正後 — 削除対象以外を残す
cart.value = cart.value.filter(item => item.id !== id)
```

### 追加修正：watch に deep オプション

```js
// 修正前 — 配列内オブジェクトのプロパティ変更を検知できない
watch(cart, (newCart) => {
  document.title = "カート（" + newCart.length + "種類）"
})

// 修正後 — deep: true で深い変更も検知
watch(cart, (newCart) => {
  document.title = "カート（" + newCart.length + "種類）"
}, { deep: true })
```

**ポイント**: Vue 3 の `watch` でリアクティブな配列を監視する場合、配列内のオブジェクトのプロパティ変更（例: `item.quantity++`）は `{ deep: true }` がないと検知されません。配列への `push` や `filter` による再代入は検知されます。

</details>
