# 実践課題10：生成AI導入計画策定 ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第1章（生成AI概論）、第6章（業務プロセスへの統合）、第7章（API連携と自動化）、第8章（倫理・法律・リスク管理）
> **課題の種類**: ミニプロジェクト
> **学習目標**: 組織に生成AIを導入するためのプロジェクト計画（導入戦略・ロードマップ・体制・KPI）を策定する力を身につける

## 課題の要件

1. 以下の前提条件で、生成AI導入のプロジェクト計画書を作成する
   - **企業規模**: 従業員1,000名の中堅IT企業
   - **現状**: 一部の社員が個人的に生成AIを利用しているが、公式なルールや環境はない
   - **経営方針**: DX推進の一環として生成AIを活用し、業務効率化を図りたい
   - **予算**: 初年度 2,000万円
   - **期間**: 6ヶ月で全社展開を目指す
2. 4フェーズのロードマップを作成する（調査・企画 → PoC → パイロット → 全社展開）
3. プロジェクト体制図を作成する（役割と責任を明確に）
4. 予算配分計画を作成する
5. リスク一覧と対策を策定する
6. 効果測定のKPI（Key Performance Indicator：重要業績評価指標）を設計する

## ステップガイド

<details>
<summary>ステップ1：現状分析と導入ゴールを定義する</summary>

まず「何のために生成AIを導入するのか」を明確にしましょう。

**分析すべき項目：**
- 現在の業務で非効率な作業は何か
- 既に個人利用している社員はどの部署に多いか
- セキュリティポリシーや社内規定の現状
- 競合他社の生成AI活用状況

**ゴール設定のフレームワーク:**
```
定量目標：〇〇業務の工数を△△%削減する
定性目標：社員が生成AIを安全に活用できる環境を整備する
```

**よくある間違い**: 「とりあえず導入する」が目的になってしまうこと。具体的な業務課題と紐付けて導入しないと、使われないツールになってしまいます。

</details>

<details>
<summary>ステップ2：4フェーズのロードマップを設計する</summary>

各フェーズの目的・成果物・判断基準を明確にしましょう。

```
Phase 1：調査・企画（ゴール：経営承認を得る）
  └─ 成果物：現状分析報告書、導入企画書、予算案

Phase 2：PoC実施（ゴール：技術的実現性と効果を検証する）
  └─ 成果物：PoC結果報告書、ガイドライン草案

Phase 3：パイロット導入（ゴール：実運用の課題を洗い出す）
  └─ 成果物：運用マニュアル、教育資料、効果測定レポート

Phase 4：全社展開（ゴール：全社員が安全に活用できる状態）
  └─ 成果物：全社展開レポート、継続改善計画
```

各フェーズ間に「Go/No-Go判断」を設けます。

</details>

<details>
<summary>ステップ3：体制と予算を設計する</summary>

**体制の考え方：**
- プロジェクトオーナー（経営層）
- プロジェクトマネージャー
- 技術チーム（AI・インフラ担当）
- 業務チーム（各部門の代表者）
- セキュリティ・法務チーム

**予算の配分:**
- ツール・ライセンス費
- 開発・構築費
- 教育・研修費
- 外部コンサルティング費
- 予備費（不測の事態用）

</details>

<details>
<summary>ステップ4：リスク管理とKPIを設計する</summary>

**リスクカテゴリ：**
- セキュリティリスク（情報漏洩）
- コンプライアンスリスク（著作権、個人情報）
- 技術リスク（APIの可用性、性能）
- 組織リスク（抵抗勢力、スキル不足）
- 財務リスク（コスト超過）

**KPI設計のポイント：**
- 先行指標（Leading Indicator）と遅行指標（Lagging Indicator）を組み合わせる
- 定量的に測定可能な指標を設定する

</details>

### 生成AI導入プロジェクト計画書

**プロジェクト名**: AI Boost プロジェクト
**期間**: 2026年5月〜2026年10月（6ヶ月間）
**予算**: 2,000万円

---

#### 1. 導入ゴール

**定量目標:**
- 対象業務の作業時間を30%削減する
- 全社員の50%以上が月1回以上生成AIを業務で活用する

**定性目標:**
- 全社員が安全に生成AIを活用できる環境と知識を提供する
- 生成AI活用のガバナンス体制を確立する

**対象業務（優先順位）:**
1. 社内文書作成（報告書、議事録、メール）
2. プログラミング支援（コード生成、レビュー）
3. 翻訳・多言語対応
4. データ分析・要約

---

#### 2. ロードマップ

```
2026年

├───Phase 1──┤──Phase 2──┤──Phase 3────────────┤─Phase 4────┤

│            │           │                     │            │

### 改良版の特徴

- **チェンジマネジメント（Change Management）戦略**を追加
- **段階的なガバナンスモデル**を設計
- **ベンダー選定の評価フレームワーク**を追加
- **継続的改善サイクル**を設計

---

#### チェンジマネジメント戦略

技術導入だけでなく、組織の「人」の変化を管理する戦略です。

```
┌──────────────────────────────────────────┐
│       チェンジマネジメント施策            │
├──────────────────────────────────────────┤
│                                          │
│  【認知】(Phase 1)                       │
│  ・経営層からのメッセージ発信             │
│  ・全社タウンホールミーティング           │
│  ・FAQ公開（「仕事を奪われるの？」等に回答）│
│                                          │
│  【興味】(Phase 2)                       │
│  ・PoC結果の社内デモ                      │
│  ・成功事例の共有（Before/After）          │
│  ・AIアンバサダーの任命（各部門1名）       │
│                                          │
│  【試行】(Phase 3)                       │
│  ・ハンズオン研修（全社員必須）            │
│  ・メンター制度（経験者→初心者）          │
│  ・「AI活用コンテスト」で優秀事例を表彰   │
│                                          │
│  【定着】(Phase 4)                       │
│  ・活用事例の定期的な社内共有             │
│  ・部門別の活用目標設定                   │
│  ・人事評価への反映（活用度ではなく成果）  │
│                                          │
└──────────────────────────────────────────┘
```

**よくある間違い**: 研修だけ実施して「あとは各自で使ってください」と放置すること。継続的な支援と成功体験の共有が定着の鍵です。

---

#### 段階的ガバナンスモデル

```
【レベル1：個人利用】（Phase 1-2）
  ├─ 利用可能ツール：承認済みサービスのみ
  ├─ 禁止事項：機密情報の入力
  └─ 責任：個人

【レベル2：部門利用】（Phase 3）
  ├─ 部門ごとの利用ルール策定
  ├─ API連携の承認プロセス
  └─ 責任：部門マネージャー

【レベル3：全社利用】（Phase 4）
  ├─ 全社統一ガイドライン
  ├─ 集中管理（利用量、コスト、セキュリティ）
  ├─ 定期監査
  └─ 責任：AI推進チーム + 各部門

【レベル4：外部連携】（Phase 4以降）
  ├─ 顧客向けサービスへの組み込み
  ├─ パートナー企業との連携
  ├─ 厳格な品質保証・法務チェック
  └─ 責任：経営層 + 法務
```

---

#### ベンダー選定の評価フレームワーク

| 評価項目 | 重み | ベンダーA | ベンダーB | ベンダーC |
|---------|:----:|:---------:|:---------:|:---------:|
| セキュリティ（データ保護） | 30% | 4/5 | 5/5 | 3/5 |
| 日本語性能 | 20% | 4/5 | 4/5 | 5/5 |
| コスト（TCO） | 20% | 3/5 | 4/5 | 4/5 |
| API・連携性 | 15% | 5/5 | 3/5 | 4/5 |
| サポート体制 | 15% | 3/5 | 5/5 | 3/5 |
| **加重平均** | | **3.85** | **4.25** | **3.80** |

TCO（Total Cost of Ownership：総所有コスト）には、ライセンス費だけでなく、導入・運用・教育にかかる全コストを含めます。

---

#### 継続的改善サイクル

```
        ┌─────────┐
        │ Plan    │  KPI未達の原因分析、改善施策の立案
        │ (計画)  │
        └────┬────┘
             │
    ┌────────┴────────┐
    │                  │
┌───┴───┐         ┌───┴───┐
│ Act   │         │ Do    │  施策の実行（プロンプト改善、
│(改善) │         │(実行) │  研修追加、ツール変更等）

**初心者向けとの違い**:
- チェンジマネジメント戦略で「人の変化」を管理する視点を追加
- ガバナンスを段階的に強化するモデルで、最初から厳しすぎない導入を実現
- ベンダー選定を定量的な評価フレームワークで実施
- PDCAサイクルで導入後の継続的改善を仕組み化
