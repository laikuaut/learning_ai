## 7.1 オブジェクト指向プログラミングとは

### OOPの基本概念

オブジェクト指向プログラミング（Object-Oriented Programming、OOP）は、データとそれに対する操作を「オブジェクト」としてまとめるプログラミングパラダイム（paradigm）です。

```
┌──────────────────────────────────────────────────┐
│  オブジェクト指向の4大原則                          │
│                                                  │
│  1. カプセル化（Encapsulation）                    │
│     → データと操作をまとめ、内部を隠す              │
│                                                  │
│  2. 抽象化（Abstraction）                         │
│     → 複雑さを隠し、必要な部分だけ公開する          │
│                                                  │
│  3. 継承（Inheritance）                           │
│     → 既存クラスの機能を受け継ぐ（第8章で学習）     │
│                                                  │
│  4. ポリモーフィズム（Polymorphism）                │
│     → 同じ操作で異なる動作をする（第8章で学習）     │
└──────────────────────────────────────────────────┘
```

### クラスとオブジェクトの関係

クラスは「設計図」、オブジェクトは「設計図から作られた実体」です。

```
┌─────────────────┐
│  クラス（設計図）  │
│  Car             │
│  ─────────────── │
│  - color         │     new Car()        new Car()
│  - speed         │ ──────────────→  ──────────────→
│  ─────────────── │
│  + accelerate()  │   ┌──────────┐    ┌──────────┐
│  + brake()       │   │ myCar    │    │ yourCar  │
└─────────────────┘   │ 赤, 60km │    │ 青, 0km  │
                       └──────────┘    └──────────┘
                       オブジェクト1     オブジェクト2
```

### ポイントまとめ

- クラスはオブジェクトの設計図（テンプレート）
- オブジェクトはクラスから作られた実体（インスタンス）
- OOPはデータと操作をまとめて管理する考え方

## 7.2 クラスの定義

### 基本的なクラス定義

```java
public class Dog {
    // フィールド（field）: オブジェクトの状態を表すデータ
    String name;
    String breed;
    int age;

    // メソッド（method）: オブジェクトの動作
    void bark() {
        System.out.println(name + "がワンワン吠えた！");
    }

    void introduce() {
        System.out.println("名前: " + name + ", 犬種: " + breed + ", 年齢: " + age + "歳");
    }

    // mainメソッド（テスト用）
    public static void main(String[] args) {
        // オブジェクトの生成
        Dog dog1 = new Dog();
        dog1.name = "ポチ";
        dog1.breed = "柴犬";
        dog1.age = 3;

        Dog dog2 = new Dog();
        dog2.name = "タロウ";
        dog2.breed = "ゴールデンレトリバー";
        dog2.age = 5;

        // メソッドの呼び出し
        dog1.introduce();  // 名前: ポチ, 犬種: 柴犬, 年齢: 3歳
        dog1.bark();       // ポチがワンワン吠えた！

        dog2.introduce();  // 名前: タロウ, 犬種: ゴールデンレトリバー, 年齢: 5歳
        dog2.bark();       // タロウがワンワン吠えた！
    }
}
```

### オブジェクトのメモリ構造

```
┌─────────────────────────────────────────────┐
│  スタック領域           ヒープ領域             │
│                                             │
│  dog1 ──────────→  ┌───────────────────┐    │
│                     │ Dog オブジェクト     │    │
│                     │ name = "ポチ"      │    │
│                     │ breed = "柴犬"     │    │
│                     │ age = 3           │    │
│                     └───────────────────┘    │
│                                             │
│  dog2 ──────────→  ┌───────────────────┐    │
│                     │ Dog オブジェクト     │    │
│                     │ name = "タロウ"    │    │
│                     │ breed = "ゴールデン" │   │
│                     │ age = 5           │    │
│                     └───────────────────┘    │
└─────────────────────────────────────────────┘
```

### ポイントまとめ

- クラスはフィールド（データ）とメソッド（動作）で構成される
- `new` キーワードでオブジェクトを生成する
- 変数にはオブジェクトへの参照（reference）が格納される

## 7.3 コンストラクタ

コンストラクタ（constructor）は、オブジェクト生成時に自動的に呼ばれる特別なメソッドです。

### 基本的なコンストラクタ

```java
public class Student {
    String name;
    int age;
    String studentId;

    // コンストラクタ（クラス名と同じ名前、戻り値なし）
    Student(String name, int age, String studentId) {
        this.name = name;           // this でフィールドを参照
        this.age = age;
        this.studentId = studentId;
    }

    void introduce() {
        System.out.println("学籍番号: " + studentId + ", 名前: " + name + ", 年齢: " + age + "歳");
    }

    public static void main(String[] args) {
        // コンストラクタで初期化
        Student s1 = new Student("田中太郎", 20, "S001");
        Student s2 = new Student("鈴木花子", 19, "S002");

        s1.introduce();  // 学籍番号: S001, 名前: 田中太郎, 年齢: 20歳
        s2.introduce();  // 学籍番号: S002, 名前: 鈴木花子, 年齢: 19歳
    }
}
```

### this キーワード

`this` は「今のオブジェクト自身」を指すキーワードです。フィールド名と引数名が同じ場合に区別するために使います。

```java
public class Point {
    int x;
    int y;

    // this でフィールドと引数を区別
    Point(int x, int y) {
        this.x = x;  // this.x はフィールド、x は引数
        this.y = y;
    }

    // this で自分自身を返す（メソッドチェーン用）
    Point move(int dx, int dy) {
        this.x += dx;
        this.y += dy;
        return this;
    }

    void print() {
        System.out.println("(" + x + ", " + y + ")");
    }

    public static void main(String[] args) {
        Point p = new Point(0, 0);
        p.move(3, 4).move(1, 2).print();  // (4, 6)
    }
}
```

### コンストラクタのオーバーロード

同じクラスに引数の異なる複数のコンストラクタを定義できます。これをオーバーロード（overload）と呼びます。

```java
public class Book {
    String title;
    String author;
    int price;
    int pages;

    // コンストラクタ1: 全項目指定
    Book(String title, String author, int price, int pages) {
        this.title = title;
        this.author = author;
        this.price = price;
        this.pages = pages;
    }

    // コンストラクタ2: ページ数なし
    Book(String title, String author, int price) {
        this(title, author, price, 0);  // this() で他のコンストラクタを呼ぶ
    }

    // コンストラクタ3: タイトルのみ
    Book(String title) {
        this(title, "不明", 0, 0);
    }

    void showInfo() {
        System.out.printf("『%s』著者: %s, %d円, %dページ%n",
            title, author, price, pages);
    }

    public static void main(String[] args) {
        Book b1 = new Book("Java入門", "山田太郎", 2800, 400);
        Book b2 = new Book("Python入門", "佐藤花子", 2500);
        Book b3 = new Book("不明な本");

        b1.showInfo();  // 『Java入門』著者: 山田太郎, 2800円, 400ページ
        b2.showInfo();  // 『Python入門』著者: 佐藤花子, 2500円, 0ページ
        b3.showInfo();  // 『不明な本』著者: 不明, 0円, 0ページ
    }
}
```

> **よくある間違い：** `this()` によるコンストラクタ呼び出しは、コンストラクタの**最初の行**に書く必要があります。2行目以降に書くとコンパイルエラーになります。

### ポイントまとめ

- コンストラクタはクラス名と同じ名前で、戻り値を書かない
- `this` はオブジェクト自身を参照するキーワード
- `this()` で同じクラスの別のコンストラクタを呼べる
- オーバーロードで引数違いの複数コンストラクタを定義できる

## 7.4 アクセス修飾子とカプセル化

### アクセス修飾子（Access Modifier）

| 修飾子 | クラス内 | 同パッケージ | サブクラス | すべて |
|---|:---:|:---:|:---:|:---:|
| `private` | ○ | × | × | × |
| （なし）default | ○ | ○ | × | × |
| `protected` | ○ | ○ | ○ | × |
| `public` | ○ | ○ | ○ | ○ |

### カプセル化（Encapsulation）

カプセル化とは、フィールドを `private` にして外部から直接アクセスできないようにし、Getter/Setterメソッドを通じて安全にアクセスする設計パターンです。

```java
public class BankAccount {
    // フィールドを private で隠す
    private String owner;
    private double balance;
    private String accountNumber;

    // コンストラクタ
    public BankAccount(String owner, String accountNumber, double initialBalance) {
        this.owner = owner;
        this.accountNumber = accountNumber;
        if (initialBalance >= 0) {
            this.balance = initialBalance;
        } else {
            this.balance = 0;
            System.out.println("警告: 初期残高が負のため、0に設定しました。");
        }
    }

    // Getter: 値を取得するメソッド
    public String getOwner() {
        return owner;
    }

    public double getBalance() {
        return balance;
    }

    public String getAccountNumber() {
        return accountNumber;
    }

    // Setter: 値を設定するメソッド（バリデーション付き）
    public void setOwner(String owner) {
        if (owner != null && !owner.isEmpty()) {
            this.owner = owner;
        } else {
            System.out.println("エラー: 名前が空です。");
        }
    }

    // 入金
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.printf("%s: ¥%,.0f 入金しました。残高: ¥%,.0f%n",
                owner, amount, balance);
        } else {
            System.out.println("エラー: 入金額は正の数でなければなりません。");
        }
    }

    // 出金
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.printf("%s: ¥%,.0f 出金しました。残高: ¥%,.0f%n",
                owner, amount, balance);
        } else if (amount > balance) {
            System.out.println("エラー: 残高不足です。");
        } else {
            System.out.println("エラー: 出金額は正の数でなければなりません。");
        }
    }

    // 口座情報表示
    public void showInfo() {
        System.out.printf("口座番号: %s, 名義: %s, 残高: ¥%,.0f%n",
            accountNumber, owner, balance);
    }

    public static void main(String[] args) {
        BankAccount account = new BankAccount("田中太郎", "1234-5678", 100000);
        account.showInfo();

        account.deposit(50000);    // 入金
        account.withdraw(30000);   // 出金
        account.withdraw(200000);  // 残高不足エラー

        // account.balance = -999;  // コンパイルエラー！ private なのでアクセスできない
        System.out.println("残高: ¥" + String.format("%,.0f", account.getBalance()));
    }
}
```

> **よくある間違い：** フィールドを `public` にしてしまうと、外部から自由に値を変更できてしまい、不正な値が設定される可能性があります。カプセル化で**データの整合性**を守りましょう。

### ポイントまとめ

- フィールドは `private` にしてカプセル化する
- Getter で値を取得、Setter で値を設定する
- Setter にバリデーション（validation）を入れてデータを保護する
- アクセス修飾子は4段階の公開範囲がある

## 7.5 staticフィールドとstaticメソッド

### static の意味

`static` を付けたフィールドやメソッドは、オブジェクトではなく**クラスに紐づきます**。

```
┌────────────────────────────────────────────┐
│  通常のフィールド    → オブジェクトごとに別    │
│  static フィールド  → クラスで1つだけ共有      │
│                                            │
│  Counter クラス                              │
│  static count = 3                           │
│                                            │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │ obj1     │ │ obj2     │ │ obj3     │   │
│  │ id = 1   │ │ id = 2   │ │ id = 3   │   │
│  └──────────┘ └──────────┘ └──────────┘   │
│  個別のフィールド  個別のフィールド  個別のフィールド │
└────────────────────────────────────────────┘
```

```java
public class Counter {
    // static フィールド: 全オブジェクトで共有
    private static int totalCount = 0;

    // インスタンスフィールド: オブジェクトごとに別
    private int id;
    private String name;

    public Counter(String name) {
        totalCount++;          // クラス全体のカウントを増加
        this.id = totalCount;  // 現在のカウントをIDとして割り当て
        this.name = name;
    }

    // static メソッド: オブジェクトなしで呼べる
    public static int getTotalCount() {
        return totalCount;
    }

    // インスタンスメソッド
    public void showInfo() {
        System.out.println("ID: " + id + ", 名前: " + name
            + " (全" + totalCount + "件中)");
    }

    public static void main(String[] args) {
        System.out.println("初期カウント: " + Counter.getTotalCount());  // 0

        Counter c1 = new Counter("Alice");
        Counter c2 = new Counter("Bob");
        Counter c3 = new Counter("Charlie");

        c1.showInfo();  // ID: 1, 名前: Alice (全3件中)
        c2.showInfo();  // ID: 2, 名前: Bob (全3件中)
        c3.showInfo();  // ID: 3, 名前: Charlie (全3件中)

        // static メソッドはクラス名で呼ぶ（推奨）
        System.out.println("合計: " + Counter.getTotalCount());  // 3
    }
}
```

### static メソッドの注意点

```java
public class StaticExample {
    private int instanceVar = 10;
    private static int staticVar = 20;

    // static メソッドからはインスタンスメンバにアクセスできない
    public static void staticMethod() {
        System.out.println(staticVar);      // OK
        // System.out.println(instanceVar);  // コンパイルエラー！
        // instanceMethod();                 // コンパイルエラー！
    }

    // インスタンスメソッドからは両方アクセスできる
    public void instanceMethod() {
        System.out.println(instanceVar);  // OK
        System.out.println(staticVar);    // OK
    }

    public static void main(String[] args) {
        staticMethod();  // static メソッドは直接呼べる

        StaticExample obj = new StaticExample();
        obj.instanceMethod();  // インスタンスメソッドはオブジェクト経由
    }
}
```

> **よくある間違い：** `main` メソッドは `static` なので、同じクラス内のインスタンスメソッドを直接呼べません。オブジェクトを生成してから呼び出してください。

### ポイントまとめ

- `static` フィールドはクラスに1つだけ存在し、全オブジェクトで共有する
- `static` メソッドはオブジェクトを作らずにクラス名で呼べる
- `static` メソッドからインスタンスメンバには直接アクセスできない
- ユーティリティメソッドや定数に `static` を使う

## 7.6 toStringメソッドのオーバーライド

`toString()` メソッドをオーバーライド（override）すると、オブジェクトの文字列表現をカスタマイズできます。

```java
public class Product {
    private String name;
    private int price;
    private int stock;

    public Product(String name, int price, int stock) {
        this.name = name;
        this.price = price;
        this.stock = stock;
    }

    // toString() をオーバーライド
    @Override
    public String toString() {
        return String.format("Product{name='%s', price=%d, stock=%d}", name, price, stock);
    }

    public static void main(String[] args) {
        Product p1 = new Product("ノートPC", 89800, 15);
        Product p2 = new Product("マウス", 2980, 50);

        // toString() が自動的に呼ばれる
        System.out.println(p1);  // Product{name='ノートPC', price=89800, stock=15}
        System.out.println(p2);  // Product{name='マウス', price=2980, stock=50}

        // 文字列結合でも自動呼び出し
        System.out.println("商品情報: " + p1);
    }
}
```

### ポイントまとめ

- `toString()` をオーバーライドするとオブジェクトの表示をカスタマイズできる
- `System.out.println()` や文字列結合で自動的に呼ばれる
- `@Override` アノテーションを付けて意図を明示する

## 7.7 実践例：総合クラス設計

### 学生管理システム

```java
public class StudentManager {
    // --- Student クラス（内部クラスとして定義） ---
    static class Student {
        private static int nextId = 1;

        private int id;
        private String name;
        private int[] scores;

        public Student(String name, int[] scores) {
            this.id = nextId++;
            this.name = name;
            this.scores = scores.clone();  // 防御的コピー
        }

        public String getName() {
            return name;
        }

        public double getAverage() {
            if (scores.length == 0) return 0;
            int sum = 0;
            for (int score : scores) {
                sum += score;
            }
            return (double) sum / scores.length;
        }

        public int getHighestScore() {
            int max = scores[0];
            for (int score : scores) {
                if (score > max) max = score;
            }
            return max;
        }

        public String getGrade() {
            double avg = getAverage();
            if (avg >= 90) return "A";
            if (avg >= 80) return "B";
            if (avg >= 70) return "C";
            if (avg >= 60) return "D";
            return "F";
        }

        @Override
        public String toString() {
            return String.format("ID:%d %-8s 平均:%.1f 評定:%s",
                id, name, getAverage(), getGrade());
        }
    }

    // --- メイン処理 ---
    public static void main(String[] args) {
        Student[] students = {
            new Student("田中太郎", new int[]{85, 90, 78, 92, 88}),
            new Student("鈴木花子", new int[]{95, 98, 92, 96, 94}),
            new Student("佐藤次郎", new int[]{70, 65, 72, 68, 75}),
            new Student("高橋美咲", new int[]{88, 82, 90, 85, 87}),
            new Student("山田一郎", new int[]{55, 60, 48, 52, 58})
        };

        System.out.println("===== 成績一覧 =====");
        for (Student s : students) {
            System.out.println(s);
        }

        // クラス平均の計算
        double classTotal = 0;
        Student topStudent = students[0];
        for (Student s : students) {
            classTotal += s.getAverage();
            if (s.getAverage() > topStudent.getAverage()) {
                topStudent = s;
            }
        }

        System.out.println("\n===== 統計情報 =====");
        System.out.printf("クラス平均: %.1f点%n", classTotal / students.length);
        System.out.println("最高平均点: " + topStudent.getName()
            + " (" + String.format("%.1f", topStudent.getAverage()) + "点)");
    }
}
```

### ポイントまとめ

- 実践的なクラス設計では、データの保護（private）とメソッドの公開（public）を意識する
- `clone()` による防御的コピーで外部からの変更を防ぐ
- `toString()` のオーバーライドでデバッグや表示が楽になる
- 静的フィールドでID自動採番のような共通機能を実現できる

## まとめ

この章では、Javaのクラスとオブジェクト指向の基礎を学びました。

1. **クラスとオブジェクト** - クラスは設計図、オブジェクトは実体
