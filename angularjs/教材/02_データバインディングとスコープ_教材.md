# 第2章：データバインディングとスコープ

## この章のゴール

- 双方向データバインディング（Two-way Data Binding）の仕組みを理解する
- `$scope` の役割とスコープ階層を把握する
- ダイジェストサイクル（Digest Cycle）の基本を知る
- スコープ継承の落とし穴と「ドット記法ルール」を習得する
- `Controller As` 構文でよりモダンな書き方を学ぶ

---

## 2.1 データバインディングとは

データバインディング（Data Binding）とは、**モデル（JavaScript のデータ）とビュー（HTML）を自動的に同期する仕組み**です。

### 単方向バインディング vs 双方向バインディング

| 種類 | 方向 | 説明 |
|------|------|------|
| 単方向（One-way） | モデル → ビュー | データの変更がHTMLに反映される |
| 双方向（Two-way） | モデル ⇄ ビュー | データとHTMLが相互に同期する |

```
【単方向バインディング】
  モデル ──────→ ビュー
  (JavaScript)      (HTML)
  
  {{ message }}  →  画面に表示

【双方向バインディング】
  モデル ⇄ ビュー
  (JavaScript)      (HTML)

  ng-model="name"
  入力欄に文字を打つ → モデルが更新 → 他の表示も更新
```

### 双方向バインディングの例

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body ng-controller="MainCtrl">
  <!-- 入力（ビュー → モデル） -->
  <input type="text" ng-model="userName" placeholder="名前を入力">
  
  <!-- 表示（モデル → ビュー） -->
  <h2>こんにちは、{{ userName }}さん！</h2>
  
  <!-- 別の入力でも同じモデルを共有 -->
  <input type="text" ng-model="userName">
  <p>↑ どちらの入力欄を変更しても同期されます</p>

  <script>
    angular.module('myApp', [])
      .controller('MainCtrl', function($scope) {
        $scope.userName = 'ゲスト';
      });
  </script>
</body>
</html>
```

> **ポイント：** `ng-model` を使うと、入力フォームの値とJavaScriptの変数が自動的に同期されます。明示的にイベントリスナーを書く必要がありません。

## 2.2 $scope の基本

`$scope`（スコープ）は、**コントローラとビュー（テンプレート）をつなぐオブジェクト**です。

### $scope の役割

```
┌─────────────────────────────┐
│         HTML（ビュー）        │
│  {{ message }}               │
│  ng-model="name"             │
│           ↕                  │
│      ┌─────────┐            │
│      │ $scope   │            │
│      │ .message │            │
│      │ .name    │            │
│      └─────────┘            │
│           ↕                  │
│    Controller（ロジック）     │
└─────────────────────────────┘
```

### 基本的な使い方

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body ng-controller="ProfileCtrl">
  <h2>{{ profile.name }}</h2>
  <p>年齢: {{ profile.age }}歳</p>
  <p>趣味: {{ profile.hobbies.join(', ') }}</p>
  
  <button ng-click="incrementAge()">1歳増やす</button>

  <script>
    angular.module('myApp', [])
      .controller('ProfileCtrl', function($scope) {
        // $scope にデータを設定
        $scope.profile = {
          name: '田中太郎',
          age: 25,
          hobbies: ['読書', 'プログラミング', '映画鑑賞']
        };
        
        // $scope にメソッドを設定
        $scope.incrementAge = function() {
          $scope.profile.age++;
        };
      });
  </script>
</body>
</html>
```

### ポイントまとめ

- `$scope` はコントローラの第一引数として注入される
- `$scope` にプロパティを追加すると、テンプレートから `{{ }}` や `ng-model` でアクセスできる
- `$scope` にメソッドを追加すると、`ng-click` などから呼び出せる

## 2.3 スコープの階層構造

AngularJS のスコープは**ツリー構造**を形成します。各コントローラは独自の `$scope` を持ち、親子関係が生まれます。

```
$rootScope（ルートスコープ）
├── $scope（HeaderCtrl）
├── $scope（MainCtrl）
│   ├── $scope（SidebarCtrl）
│   └── $scope（ContentCtrl）
└── $scope（FooterCtrl）
```

### $rootScope

`$rootScope` はアプリケーション全体で共有される最上位のスコープです。

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body>
  <!-- 親コントローラ -->
  <div ng-controller="ParentCtrl">
    <h2>親: {{ parentMessage }}</h2>
    <p>共有データ: {{ sharedData }}</p>
    
    <!-- 子コントローラ -->
    <div ng-controller="ChildCtrl" style="margin-left: 20px; border-left: 3px solid #42b883; padding-left: 10px;">
      <h3>子: {{ childMessage }}</h3>
      <p>親のデータにアクセス: {{ parentMessage }}</p>
      <p>共有データ: {{ sharedData }}</p>
    </div>
  </div>

  <script>
    angular.module('myApp', [])
      .run(function($rootScope) {
        // $rootScope はアプリ全体で共有される
        $rootScope.sharedData = '全コントローラから見える';
      })
      .controller('ParentCtrl', function($scope) {
        $scope.parentMessage = '親コントローラのデータ';
      })
      .controller('ChildCtrl', function($scope) {
        $scope.childMessage = '子コントローラのデータ';
        // parentMessage は親の $scope から継承されてアクセス可能
      });
  </script>
</body>
</html>
```

### スコープ継承のルール

| ルール | 説明 |
|--------|------|
| 子は親を参照できる | プロトタイプ継承により、子スコープから親のプロパティを読める |
| 親は子を参照できない | 子スコープのプロパティは親からは見えない |
| 書き込みは別 | 子スコープでプリミティブ値を変更すると、子にローカルコピーが作られる |

## 2.4 スコープ継承の落とし穴 — ドット記法ルール

AngularJS で最もよくある間違いの一つが、**プリミティブ値のスコープ継承問題**です。

### よくある間違い：プリミティブ値の罠

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body>
  <div ng-controller="ParentCtrl">
    <h3>親の name: {{ name }}</h3>
    <input ng-model="name" placeholder="親で変更">
    
    <div ng-controller="ChildCtrl" style="margin-left: 20px; padding: 10px; background: #f0f0f0;">
      <h3>子の name: {{ name }}</h3>
      <input ng-model="name" placeholder="子で変更">
      <p style="color: red;">↑ 子で変更しても親に反映されない！</p>
    </div>
  </div>

  <script>
    angular.module('myApp', [])
      .controller('ParentCtrl', function($scope) {
        $scope.name = '太郎';  // プリミティブ値（文字列）
      })
      .controller('ChildCtrl', function($scope) {
        // 何も定義しない — 親のnameを継承
      });
  </script>
</body>
</html>
```

**問題：** 子コントローラの `input` で `name` を変更すると、子スコープに新しい `name` プロパティが作られ（シャドウイング）、親の `name` とは切り離されます。

### 解決策：ドット記法（Dot Notation）ルール

```
★ AngularJS の黄金ルール ★
「ng-model にはドット（.）を含めるべし」

✗ ng-model="name"          → プリミティブ値、継承が壊れる
✓ ng-model="user.name"     → オブジェクトのプロパティ、正しく共有される
```

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body>
  <div ng-controller="ParentCtrl">
    <h3>親の user.name: {{ user.name }}</h3>
    <input ng-model="user.name" placeholder="親で変更">
    
    <div ng-controller="ChildCtrl" style="margin-left: 20px; padding: 10px; background: #e8f5e9;">
      <h3>子の user.name: {{ user.name }}</h3>
      <input ng-model="user.name" placeholder="子で変更">
      <p style="color: green;">↑ 子で変更しても親と同期される！</p>
    </div>
  </div>

  <script>
    angular.module('myApp', [])
      .controller('ParentCtrl', function($scope) {
        $scope.user = { name: '太郎' };  // オブジェクトに包む
      })
      .controller('ChildCtrl', function($scope) {
        // user オブジェクトは参照で共有されるため、正しく同期する
      });
  </script>
</body>
</html>
```

### ポイントまとめ

- プリミティブ値（文字列、数値）を `ng-model` に直接使うと、子スコープで書き込み時にシャドウイングが発生する
- **常にオブジェクトのプロパティ（ドット記法）を使う**ことで問題を回避できる
- これは AngularJS で最も重要なベストプラクティスの一つ

## 2.5 ダイジェストサイクル（Digest Cycle）

AngularJS がデータ変更を検知する仕組みが**ダイジェストサイクル**です。

### ダイジェストサイクルの流れ

```
ユーザー操作（クリック、入力など）
       │
       ▼
  AngularJS がイベントを検知
       │
       ▼
  $digest サイクル開始
       │
       ▼
  すべての $watch を巡回
  （値が変わったかチェック）
       │
       ├── 変更あり → ウォッチャーのコールバック実行
       │                   │
       │                   ▼
       │              もう一度 $digest を実行
       │              （最大10回まで）
       │
       └── 変更なし → $digest 終了
                           │
                           ▼
                      DOM を更新（画面に反映）
```

### $watch — 値の変更を監視

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body ng-controller="WatchCtrl">
  <input ng-model="searchText" placeholder="検索ワードを入力">
  <p>入力文字数: {{ charCount }}</p>
  <p>変更回数: {{ changeCount }}</p>

  <script>
    angular.module('myApp', [])
      .controller('WatchCtrl', function($scope) {
        $scope.searchText = '';
        $scope.charCount = 0;
        $scope.changeCount = 0;
        
        // searchText の変更を監視
        $scope.$watch('searchText', function(newVal, oldVal) {
          if (newVal !== oldVal) {
            $scope.charCount = newVal.length;
            $scope.changeCount++;
          }
        });
      });
  </script>
</body>
</html>
```

### $apply — AngularJS の外からの変更を通知

AngularJS が管理していないコード（`setTimeout`、サードパーティライブラリなど）からデータを変更した場合、`$apply` で通知する必要があります。

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body ng-controller="ApplyCtrl">
  <h3>現在時刻: {{ currentTime }}</h3>
  <button ng-click="startClock()">時計を開始</button>
  <button ng-click="stopClock()">時計を停止</button>

  <script>
    angular.module('myApp', [])
      .controller('ApplyCtrl', function($scope) {
        $scope.currentTime = new Date().toLocaleTimeString();
        var timer = null;
        
        $scope.startClock = function() {
          timer = setInterval(function() {
            // setInterval は AngularJS の外なので $apply が必要
            $scope.$apply(function() {
              $scope.currentTime = new Date().toLocaleTimeString();
            });
          }, 1000);
        };
        
        $scope.stopClock = function() {
          if (timer) {
            clearInterval(timer);
            timer = null;
          }
        };
      });
  </script>
</body>
</html>
```

> **よくある間違い：** `setTimeout` や `setInterval` でデータを変更しても画面が更新されない場合は、`$apply` を忘れている可能性があります。代わりに AngularJS の `$timeout` や `$interval` サービスを使えば `$apply` は不要です。

### ポイントまとめ

- AngularJS は `$digest` サイクルで全ての `$watch` をチェックし、変更を検知する
- `$watch` で特定の値の変更を監視できる
- AngularJS 外部のコードからの変更には `$apply` が必要
- **できるだけ `$timeout`、`$interval` などの AngularJS サービスを使う**ことで `$apply` を避けられる

## 2.6 Controller As 構文

`Controller As` は `$scope` を直接使わず、コントローラ自体をビューモデルとして使う**モダンな書き方**です。

### 従来の $scope 方式 vs Controller As

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body>
  <h2>$scope 方式 vs Controller As 方式</h2>

  <!-- 従来の $scope 方式 -->
  <div ng-controller="OldCtrl">
    <h3>$scope 方式</h3>
    <p>{{ message }}</p>
    <button ng-click="greet()">挨拶</button>
  </div>

  <!-- Controller As 方式 -->
  <div ng-controller="NewCtrl as vm">
    <h3>Controller As 方式</h3>
    <p>{{ vm.message }}</p>
    <button ng-click="vm.greet()">挨拶</button>
  </div>

  <script>
    angular.module('myApp', [])
      // 従来の方式: $scope を注入して使う
      .controller('OldCtrl', function($scope) {
        $scope.message = 'こんにちは（$scope）';
        $scope.greet = function() {
          $scope.message = 'Hello from $scope!';
        };
      })
      // Controller As 方式: this を使う
      .controller('NewCtrl', function() {
        var vm = this;  // vm = ViewModel の略
        vm.message = 'こんにちは（Controller As）';
        vm.greet = function() {
          vm.message = 'Hello from Controller As!';
        };
      });
  </script>
</body>
</html>
```

### Controller As のメリット

| メリット | 説明 |
|----------|------|
| スコープ階層が明確 | `parent.name` vs `child.name` のようにどのコントローラのデータか明確 |
| ドット記法が自然に | `vm.property` で自然にドットが含まれ、継承問題を回避 |
| `this` ベース | 普通のJavaScriptオブジェクトとして扱える |
| テストしやすい | `$scope` への依存を減らせる |

### ネストしたコントローラでの比較

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body>
  <div ng-controller="ParentCtrl as parent">
    <h3>親: {{ parent.name }}</h3>
    <input ng-model="parent.name">
    
    <div ng-controller="ChildCtrl as child" style="margin-left: 20px; padding: 10px; background: #e3f2fd;">
      <h3>子: {{ child.name }}</h3>
      <input ng-model="child.name">
      
      <p>親のデータ: {{ parent.name }}</p>
      <p>↑ どのコントローラのデータか一目瞭然！</p>
    </div>
  </div>

  <script>
    angular.module('myApp', [])
      .controller('ParentCtrl', function() {
        this.name = '親太郎';
      })
      .controller('ChildCtrl', function() {
        this.name = '子次郎';
      });
  </script>
</body>
</html>
```

### ポイントまとめ

- `Controller As` 構文は `$scope` の代わりに `this` を使うモダンな書き方
- テンプレートで `vm.property` のようにプレフィックスが付くため、データの出所が明確になる
- ネストしたコントローラでの名前衝突を防げる
- **新規プロジェクトでは `Controller As` 構文を推奨**

## 2.7 まとめ

| トピック | ポイント |
|----------|----------|
| 双方向バインディング | `ng-model` でモデルとビューが自動同期 |
| $scope | コントローラとビューをつなぐオブジェクト |
| スコープ階層 | プロトタイプ継承、子は親を参照可能 |
| ドット記法ルール | `ng-model` にはオブジェクトのプロパティを使う |
| ダイジェストサイクル | `$watch` → `$digest` → DOM更新 |
| $apply | Angular外のコードからの変更通知に使用 |
| Controller As | `this` ベースのモダンな書き方、推奨 |
