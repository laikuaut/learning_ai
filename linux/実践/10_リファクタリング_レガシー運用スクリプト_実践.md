# 実践課題10：リファクタリング ─ レガシー運用スクリプト ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第2章〜第5章（ファイル操作、パーミッション、プロセス管理、シェルスクリプト基礎）、第7章（テキスト処理）、第8章（システム管理）
> **課題の種類**: リファクタリング
> **学習目標**: 動くが問題の多いスクリプトを読み解き、保守性・安全性・可読性を改善するリファクタリング力を養う

---

## 課題の説明

以下の「サーバーデプロイスクリプト」は実際に動作しますが、多数の問題を抱えています。
コードを読み、問題点を特定し、より良いスクリプトにリファクタリングしてください。

### リファクタリングのポイント

以下の観点から問題を見つけてください。

1. **安全性**: エラー処理、権限チェック、入力検証
2. **保守性**: ハードコーディング、コードの重複、関数化
3. **可読性**: 変数名、コメント、コードの構造
4. **堅牢性**: 一時ファイル、ロック、シグナル処理
5. **移植性**: 環境依存のコマンド、パス

## レガシーコード

```bash
#!/bin/bash
# deploy script
# last updated: 2019-03-15

d=/var/www/myapp
b=/var/www/backup
l=/var/log/deploy.log
r=https://github.com/myteam/myapp.git

echo "deploying..."
echo "$(date)" >> $l

cd $d
git pull origin master

cp -r $d $b/backup_$(date +%s)

cd $d
npm install
npm run build

rm -rf $d/node_modules/.cache
rm -rf /tmp/*

sudo systemctl restart nginx
sudo systemctl restart myapp

sleep 5

c=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:3000)
if [ $c == 200 ]; then
    echo "deploy ok" >> $l
else
    echo "deploy failed" >> $l
    cd $b
    latest=$(ls -t | head -1)
    cp -r $latest $d
    sudo systemctl restart myapp
    sudo systemctl restart nginx
fi

echo "done"
```

## 解答例

<details>
<summary>設問1の解答：問題点一覧</summary>

### 安全性の問題

1. **`set -e` がない**: コマンドが失敗しても処理が続行される。`git pull` が失敗してもビルドが実行される
2. **変数のクォートがない**: `$d`, `$b`, `$l` がクォートされておらず、パスにスペースが含まれると壊れる
3. **`rm -rf /tmp/*` が非常に危険**: 他のプロセスが使用している一時ファイルまで削除される。本番環境でこれを実行すると障害の原因になる
4. **入力検証がない**: 引数やブランチの指定ができず、常に `master` を pull する
5. **root権限チェックがない**: `sudo systemctl` を使っているが、sudo権限がなければ失敗する

### 保守性の問題

6. **変数名が不明瞭**: `d`, `b`, `l`, `r`, `c` では何を表すか分からない
7. **ハードコーディング**: URL、ポート番号（3000）、ブランチ名（master）がスクリプト内に直接書かれている
8. **関数化されていない**: すべてがフラットに書かれており、処理の単位が分かりにくい
9. **エラーメッセージが不十分**: 「deploy ok」「deploy failed」だけでは原因を特定できない

### 可読性の問題

10. **コメントがほぼない**: 各処理が何をしているか説明がない
11. **コードの構造が不明瞭**: セクション分けがなく、一本道で処理が流れている
12. **日付が古い**: 「last updated: 2019-03-15」で更新履歴が管理されていない

### 堅牢性の問題

13. **ロールバック処理が不完全**: `cp -r $latest $d` ではバックアップで上書きするが、`node_modules` や設定が正しく復元されるか不明
14. **多重実行防止がない**: 同時に2回実行されると破壊的な結果になる
15. **バックアップの世代管理がない**: バックアップが際限なく増え続け、ディスクを圧迫する
16. **ヘルスチェックが不十分**: 1回のHTTPチェックだけでは不安定。リトライ機能がない

```bash
#!/bin/bash
# =============================================================
# deploy.sh - アプリケーションデプロイスクリプト
# 用途: Gitリポジトリから最新コードを取得し、ビルド・デプロイを行う
# 使い方: sudo bash deploy.sh [ブランチ名]
# =============================================================

set -e

# --- 設定 ---
APP_DIR="/var/www/myapp"
BACKUP_DIR="/var/www/backup"
LOG_FILE="/var/log/deploy.log"
REPO_URL="https://github.com/myteam/myapp.git"
BRANCH="${1:-main}"         # デフォルトはmain
HEALTH_CHECK_URL="http://localhost:3000"
BACKUP_RETENTION=5          # 保持するバックアップ世代数

# --- ログ関数 ---
log() {
    local message="[$(date '+%Y-%m-%d %H:%M:%S')] $1"
    echo "$message"
    echo "$message" >> "$LOG_FILE"
}

# --- root権限チェック ---
if [ "$(id -u)" -ne 0 ]; then
    echo "エラー: root権限で実行してください（sudo bash deploy.sh）"
    exit 1
fi

# --- ディレクトリ確認 ---
if [ ! -d "$APP_DIR" ]; then
    echo "エラー: アプリケーションディレクトリが見つかりません: $APP_DIR"
    exit 1
fi

mkdir -p "$BACKUP_DIR"

log "===== デプロイ開始（ブランチ: $BRANCH） ====="

# --- バックアップ ---
BACKUP_NAME="backup_$(date '+%Y%m%d_%H%M%S')"
log "バックアップ作成: $BACKUP_NAME"
cp -r "$APP_DIR" "$BACKUP_DIR/$BACKUP_NAME"

# --- コード更新 ---
log "コード更新中..."
cd "$APP_DIR"
git fetch origin
git checkout "$BRANCH"
git pull origin "$BRANCH"

# --- ビルド ---
log "ビルド中..."
npm install
npm run build

# --- キャッシュクリア（安全に） ---
if [ -d "$APP_DIR/node_modules/.cache" ]; then
    rm -rf "$APP_DIR/node_modules/.cache"
    log "キャッシュをクリアしました"
fi

# --- サービス再起動 ---
log "サービス再起動中..."
systemctl restart myapp
systemctl restart nginx

# --- ヘルスチェック ---
log "ヘルスチェック中..."
sleep 5

HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" "$HEALTH_CHECK_URL")
if [ "$HTTP_CODE" = "200" ]; then
    log "デプロイ成功（HTTP $HTTP_CODE）"
else
    log "エラー: ヘルスチェック失敗（HTTP $HTTP_CODE）"
    log "ロールバック開始..."

    # ロールバック
    rm -rf "$APP_DIR"
    cp -r "$BACKUP_DIR/$BACKUP_NAME" "$APP_DIR"
    systemctl restart myapp
    systemctl restart nginx

    log "ロールバック完了"
    exit 1
fi

# --- 古いバックアップの削除 ---
BACKUP_COUNT=$(ls -1d "$BACKUP_DIR"/backup_* 2>/dev/null | wc -l)
if [ "$BACKUP_COUNT" -gt "$BACKUP_RETENTION" ]; then
    DELETE_COUNT=$((BACKUP_COUNT - BACKUP_RETENTION))
    ls -1d "$BACKUP_DIR"/backup_* | sort | head -"$DELETE_COUNT" | while read -r old_backup; do
        rm -rf "$old_backup"
        log "古いバックアップ削除: $(basename "$old_backup")"
    done
fi

log "===== デプロイ完了 ====="
```

<summary>設問2の解答：リファクタリング済みコード（改良版）</summary>

```bash
#!/bin/bash
# =============================================================
# deploy.sh - アプリケーションデプロイスクリプト（改良版）
# 用途: Gitリポジトリから最新コードを取得し、ビルド・デプロイを行う
# 使い方: sudo bash deploy.sh [--branch BRANCH] [--dry-run]
# =============================================================

set -euo pipefail

# --- 設定（環境変数またはデフォルト値） ---
readonly APP_DIR="${DEPLOY_APP_DIR:-/var/www/myapp}"
readonly BACKUP_DIR="${DEPLOY_BACKUP_DIR:-/var/www/backup}"
readonly LOG_FILE="${DEPLOY_LOG_FILE:-/var/log/deploy.log}"
readonly LOCK_FILE="/tmp/deploy.lock"
readonly HEALTH_CHECK_URL="${DEPLOY_HEALTH_URL:-http://localhost:3000}"
readonly HEALTH_CHECK_RETRIES=3
readonly HEALTH_CHECK_INTERVAL=5
readonly BACKUP_RETENTION=5

BRANCH="main"
DRY_RUN=false

# --- 引数解析 ---
while [ $# -gt 0 ]; do
    case "$1" in
        --branch|-b) BRANCH="$2"; shift 2 ;;
        --dry-run)   DRY_RUN=true; shift ;;
        --help|-h)
            echo "使い方: sudo $0 [--branch BRANCH] [--dry-run]"
            echo "  --branch, -b BRANCH  デプロイするブランチ（デフォルト: main）"
            echo "  --dry-run            実際にはデプロイせず、手順だけ表示"
            exit 0
            ;;
        *) echo "不明なオプション: $1"; exit 1 ;;
    esac
done

# --- ユーティリティ関数 ---
log() {
    local level="$1"
    shift
    local message="[$(date '+%Y-%m-%d %H:%M:%S')] [$level] $*"
    echo "$message"
    echo "$message" >> "$LOG_FILE"
}

die() {
    log "ERROR" "$1"
    exit "${2:-1}"
}

run() {
    if [ "$DRY_RUN" = true ]; then
        log "DRY-RUN" "実行予定: $*"
    else
        "$@"
    fi
}

# --- 前提条件チェック ---
preflight_check() {
    # root権限チェック
    [ "$(id -u)" -eq 0 ] || die "root権限で実行してください"

    # 必要なコマンドの確認
    local required_commands=(git npm curl systemctl)
    for cmd in "${required_commands[@]}"; do
        command -v "$cmd" > /dev/null 2>&1 || die "必要なコマンドが見つかりません: $cmd"
    done

    # ディレクトリ確認
    [ -d "$APP_DIR" ] || die "アプリディレクトリが見つかりません: $APP_DIR"
    [ -d "$APP_DIR/.git" ] || die "Gitリポジトリではありません: $APP_DIR"

    mkdir -p "$BACKUP_DIR"
}

# --- 排他ロック ---
acquire_lock() {
    exec 9>"$LOCK_FILE"
    if ! flock -n 9; then
        die "別のデプロイが実行中です"
    fi
    log "INFO" "ロック取得 (PID: $$)"
}

release_lock() {
    flock -u 9 2>/dev/null || true
    rm -f "$LOCK_FILE"
}

# --- バックアップ ---
create_backup() {
    local backup_name="backup_$(date '+%Y%m%d_%H%M%S')"
    local backup_path="$BACKUP_DIR/$backup_name"
    log "INFO" "バックアップ作成: $backup_name"
    run cp -r "$APP_DIR" "$backup_path"
    echo "$backup_path"
}

cleanup_old_backups() {
    local count
    count=$(ls -1d "$BACKUP_DIR"/backup_* 2>/dev/null | wc -l)
    if [ "$count" -gt "$BACKUP_RETENTION" ]; then
        local delete_count=$((count - BACKUP_RETENTION))
        ls -1d "$BACKUP_DIR"/backup_* | sort | head -"$delete_count" | while read -r old; do
            run rm -rf "$old"
            log "INFO" "古いバックアップ削除: $(basename "$old")"
        done
    fi
}

# --- デプロイ ---
update_code() {
    log "INFO" "コード更新: ブランチ $BRANCH"
    cd "$APP_DIR"
    run git fetch origin
    run git checkout "$BRANCH"
    run git pull origin "$BRANCH"
    local commit
    commit=$(git rev-parse --short HEAD)
    log "INFO" "最新コミット: $commit"
}

build_app() {
    log "INFO" "ビルド開始"
    cd "$APP_DIR"
    run npm ci --production=false    # npm ci は npm install より再現性が高い
    run npm run build
    # 安全なキャッシュクリア（アプリディレクトリ内のみ）
    [ -d "$APP_DIR/node_modules/.cache" ] && run rm -rf "$APP_DIR/node_modules/.cache"
    log "INFO" "ビルド完了"
}

restart_services() {
    log "INFO" "サービス再起動"
    run systemctl restart myapp
    run systemctl restart nginx
}

# --- ヘルスチェック（リトライ付き） ---
health_check() {
    log "INFO" "ヘルスチェック開始（最大 ${HEALTH_CHECK_RETRIES} 回）"
    local i
    for ((i = 1; i <= HEALTH_CHECK_RETRIES; i++)); do
        sleep "$HEALTH_CHECK_INTERVAL"
        local http_code
        http_code=$(curl -s -o /dev/null -w "%{http_code}" --max-time 10 "$HEALTH_CHECK_URL" 2>/dev/null || echo "000")
        if [ "$http_code" = "200" ]; then
            log "INFO" "ヘルスチェック成功（HTTP $http_code, 試行 $i/$HEALTH_CHECK_RETRIES）"
            return 0
        fi
        log "WARN" "ヘルスチェック失敗（HTTP $http_code, 試行 $i/$HEALTH_CHECK_RETRIES）"
    done
    return 1
}

# --- ロールバック ---
rollback() {
    local backup_path="$1"
    log "WARN" "ロールバック開始: $(basename "$backup_path")"
    if [ -d "$backup_path" ]; then
        run rm -rf "$APP_DIR"
        run cp -r "$backup_path" "$APP_DIR"
        run systemctl restart myapp
        run systemctl restart nginx
        log "INFO" "ロールバック完了"
    else
        die "バックアップが見つかりません: $backup_path"
    fi
}

# --- クリーンアップ ---
on_exit() {
    local exit_code=$?
    release_lock
    if [ "$exit_code" -ne 0 ]; then
        log "ERROR" "デプロイが異常終了しました（終了コード: $exit_code）"
    fi
}

# --- メイン処理 ---
main() {
    trap on_exit EXIT

    preflight_check
    acquire_lock

    log "INFO" "===== デプロイ開始（ブランチ: $BRANCH, DRY_RUN: $DRY_RUN） ====="

    local backup_path
    backup_path=$(create_backup)

    update_code
    build_app
    restart_services

    if [ "$DRY_RUN" = true ]; then
        log "INFO" "DRY-RUN モードのため、ヘルスチェックをスキップ"
    elif health_check; then
        log "INFO" "===== デプロイ成功 ====="
    else
        log "ERROR" "ヘルスチェック失敗。ロールバックします"
        rollback "$backup_path"
        die "デプロイ失敗（ロールバック済み）" 2
    fi

    cleanup_old_backups
    log "INFO" "===== デプロイ完了 ====="
}

main "$@"
```

**レガシー版との主な違い:**

| 観点 | レガシー | 改良版 |
|------|---------|--------|
| エラー処理 | なし | `set -euo pipefail` + `die` 関数 |
| 変数名 | `d`, `b`, `l` | `APP_DIR`, `BACKUP_DIR`, `LOG_FILE` |
| コード構造 | フラット | 関数に分割 |
| 多重実行 | 防止なし | `flock` によるロック |
| ヘルスチェック | 1回のみ | リトライ機能付き |
| バックアップ | 世代管理なし | 自動クリーンアップ |
| 危険操作 | `rm -rf /tmp/*` | アプリキャッシュのみ削除 |
| 引数 | なし | ブランチ指定、--dry-run |
| ロールバック | 不完全 | 完全な復元 |
| ログ | 最低限 | レベル付き構造化ログ |
