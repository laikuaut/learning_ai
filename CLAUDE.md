# プログラミング学習教材プロジェクト

## 概要
初心者向けプログラミング学習教材。各技術分野をフォルダごとに整理し、教材・演習問題・動作サンプルを提供する。

## ディレクトリ構造
```
learning/
├── CLAUDE.md
├── python/           # Python基礎〜応用
├── javascript/       # JavaScript基礎〜応用
├── typescript/       # TypeScript
├── html/             # HTML
├── css/              # CSS
├── react/            # React
├── vue/              # Vue.js
├── nextjs/           # Next.js
├── linux/            # Linux基礎
├── network/          # ネットワーク基礎
├── security/         # 情報セキュリティ
├── project-management/ # プロジェクトマネジメント
├── prompt-engineering/ # プロンプトエンジニアリング
├── 基本情報技術者/      # 基本情報技術者試験対策
└── 応用情報技術者/      # 応用情報技術者試験対策
```

## 各フォルダの構成
```
<topic>/
├── 教材/           # 教材Markdownファイル（XX_トピック名_教材.md）
├── 演習/           # 演習問題（XX_トピック名_演習.md）
└── サンプル/       # 動作するコードサンプル（XX_サンプル名.拡張子）
```

## 教材作成ルール

### 教材ファイル（_教材.md）
- 各章の冒頭に「この章のゴール」を明記
- 概念説明 → コード例 → 解説 の順で構成
- 必ず動作するコード例を含める（コピペで動くこと）
- 現場で実際に使われるパターンを優先して紹介
- アスキーアートや図表で視覚的に説明
- 各セクション末に「ポイントまとめ」を入れる

### 演習ファイル（_演習.md）
- 基本→応用→チャレンジの3段階構成
- 全問題に期待される出力例を記載
- `<details>` タグでヒントと解答例を折りたたみ表示
- 解答例にはコメントで解説を付加

### サンプルファイル
- **そのまま実行できる完成コード**であること（コピペで即動作）
- 外部ライブラリに依存しない（標準ライブラリのみ使用）
- 冒頭コメントに「学べる内容」「実行方法」を記載
- 現実的なユースケースを題材にする
- input() 等を使った対話型も含める
- HTMLファイルはブラウザで開くだけで動作すること

### 文体・スタイル
- 敬体（です・ます調）で統一
- 専門用語は初出時に日本語と英語を併記（例：変数（variable））
- 初心者がつまずきやすいポイントは「よくある間違い」として明示
- 現場のプロの知見・ベストプラクティスを積極的に盛り込む

### 品質基準
- 全コード例が実際に動作すること
- 説明の論理的な飛躍がないこと
- 前の章の知識だけで理解できる構成にすること（前方参照を最小限に）
- 実務で使わない古い書き方は避け、モダンな書き方を優先
