# 第8章：実践的なVim活用テクニック

## この章のゴール

- マクロを記録・実行できる
- レジスタを理解して活用できる
- マークとジャンプリストを使える
- 外部コマンドとの連携ができる
- 効率的なワークフローを構築できる

## 1. マクロの記録と実行

マクロは一連のキー操作を記録し、何度でも再生できる機能です。反復作業の自動化に非常に強力です。

### マクロの基本

```
  q{レジスタ}  ← マクロの記録開始（a〜z のレジスタを指定）
  q            ← マクロの記録終了
  @{レジスタ}  ← マクロの実行
  @@           ← 直前のマクロを再実行
  {数値}@{レジスタ} ← 指定回数だけマクロを実行
```

### マクロの記録と実行の流れ

```
  1. q を押す（画面左下に "recording @a" と表示される）
  2. レジスタ名を入力（例: a）
  3. 操作を行う（通常通りに編集）
  4. q を押して記録終了
  5. @a で再生

  ┌─────────┐     ┌─────────┐     ┌─────────┐
  │ qa      │ ──▶ │ 操作を  │ ──▶ │ q       │
  │ 記録開始 │     │ 実行    │     │ 記録終了 │
  └─────────┘     └─────────┘     └─────────┘
                                       │
                                       ▼
                                  ┌─────────┐
                                  │ @a      │
                                  │ 再生    │
                                  └─────────┘
```

### 実践例1：行のフォーマット変換

```
  元データ（CSVの各行を SQL の INSERT 文に変換したい）:
  Alice,30,Tokyo
  Bob,25,Osaka
  Charlie,35,Nagoya

  手順:
  1. 1行目にカーソルを置く
  2. qa で記録開始
  3. I INSERT INTO users VALUES(' で行頭に追記
  4. ESC で抜ける
  5. f, でカンマへ移動 → s','  でカンマを置換
  6. f, でもう1つのカンマ → s','  で同様に置換
  7. A '); で行末に追記
  8. ESC → j → 0 で次の行頭に移動
  9. q で記録終了
  10. 2@a で残り2行に適用

  結果:
  INSERT INTO users VALUES('Alice','30','Tokyo');
  INSERT INTO users VALUES('Bob','25','Osaka');
  INSERT INTO users VALUES('Charlie','35','Nagoya');
```

### 実践例2：連番の生成

```
  元:
  item

  手順:
  1. yy で行をヤンク
  2. 9p で9行ペースト（計10行になる）
  3. 1行目に移動して qa で記録開始
  4. A1 ESC（行末に1を追加）
  5. j で次の行へ
  6. q で記録終了
  7. @a で次の行に 1 が追加される
     ... しかし番号が全部1になってしまう

  より良い方法:
  1. 10行の item を用意
  2. 1行目に "item1" と入力
  3. 2行目にカーソルを置いて qa で記録開始
  4. A ESC（行末へ）
  5. Ctrl+a は前の行を参照できないので、別の方法で:
     :%s/item/\='item' . line('.')/g
  → item1, item2, item3 ... item10 が生成される
```

### ポイントまとめ

- `qa` で記録開始、`q` で終了、`@a` で再生です
- `@@` で直前のマクロを手軽に再実行できます
- `100@a` で大量の反復作業を一瞬で処理できます
- マクロは「行頭から始めて行末で終わる」設計にすると再利用しやすいです

## 2. レジスタの活用

レジスタは Vim のクリップボードで、テキストを一時保存する場所です。複数のレジスタを使い分けることで、複数のテキストを同時に保持できます。

### レジスタの基本操作

```
  "{レジスタ}y{モーション}  ← レジスタに保存（ヤンク）
  "{レジスタ}d{モーション}  ← レジスタに保存（削除）
  "{レジスタ}p              ← レジスタから貼り付け

  例:
  "ayy    ← 現在行をレジスタ a にヤンク
  "bdd    ← 現在行を削除してレジスタ b に保存
  "ap     ← レジスタ a から貼り付け
  "bp     ← レジスタ b から貼り付け
```

### レジスタの種類

```
  :reg  または :registers  ← レジスタ一覧を表示

  主なレジスタ:
  ""     ← 無名レジスタ（d, y, c, s の結果が自動的に入る）
  "0     ← ヤンクレジスタ（y の結果が入る。d の結果は入らない）
  "1〜"9 ← 削除履歴レジスタ（d, c の結果が順番に入る）
  "a〜"z ← 名前付きレジスタ（ユーザーが自由に使える）
  "A〜"Z ← 名前付きレジスタに追記（大文字で追記モード）
  "+     ← システムクリップボード
  "*     ← X11選択バッファ（Linux）
  "/     ← 最後の検索パターン
  ":     ← 最後に実行した Ex コマンド
  "%     ← 現在のファイル名
  "#     ← 直前のファイル名
  "_     ← ブラックホールレジスタ（破棄用）
  "=     ← 式レジスタ（計算結果を挿入）
```

### よくある使い方

```
  問題: テキストをヤンクして別の場所に貼り付けたいが、
        途中で dd してしまうと無名レジスタが上書きされる

  解決方法1: "0p でヤンクレジスタから貼り付け
  解決方法2: 事前に名前付きレジスタに保存 "ayy → "ap
  解決方法3: "_dd でブラックホールレジスタに削除（レジスタを汚さない）
```

### 挿入モードでのレジスタ貼り付け

```
  挿入モード中に Ctrl+r {レジスタ} で貼り付け可能:

  Ctrl+r a    ← レジスタ a の内容を挿入
  Ctrl+r "    ← 無名レジスタの内容を挿入
  Ctrl+r +    ← クリップボードの内容を挿入
  Ctrl+r /    ← 最後の検索パターンを挿入
  Ctrl+r %    ← 現在のファイル名を挿入
  Ctrl+r =    ← 式の評価結果を挿入（例: =2+3 Enter → 5）
```

### ポイントまとめ

- `"ayy` でレジスタ a に保存、`"ap` で貼り付けです
- `"0` はヤンク専用レジスタで、削除で上書きされません
- `"_` はブラックホールレジスタで、レジスタを汚さず削除できます
- 挿入モード中は `Ctrl+r {レジスタ}` で貼り付けできます

## 3. マークとジャンプ

### マークの設定と移動

```
  m{a-z}   ← 現在位置にマーク（小文字: ファイル内ローカル）
  m{A-Z}   ← 現在位置にマーク（大文字: グローバル、ファイル間で有効）
  '{マーク} ← マークの行の先頭に移動
  `{マーク} ← マークの正確な位置に移動
  :marks   ← マーク一覧を表示
```

### 特殊マーク（自動設定）

```
  ''  または ``   ← 直前のジャンプ位置
  '.  または `.   ← 最後に変更した位置
  '^  または `^   ← 最後に挿入モードを抜けた位置
  '[  または `[   ← 最後にヤンク/変更した範囲の先頭
  ']  または `]   ← 最後にヤンク/変更した範囲の末尾
  '<  または `<   ← ビジュアル選択の先頭
  '>  または `>   ← ビジュアル選択の末尾
```

### ジャンプリスト

```
  Ctrl+o  ← ジャンプリストを戻る（older）
  Ctrl+i  ← ジャンプリストを進む（newer、Tab と同じ）
  :jumps  ← ジャンプリストを表示

  ジャンプリストに記録される操作:
  - 検索（/, ?, *, #）
  - gg, G, {数字}G
  - ''{マーク}
  - {, }（段落移動）
  - :数字（行番号指定の移動）
  など
```

### 実践的な使い方

```
  シナリオ: 関数を編集中に別の関数を確認したい

  1. 現在位置にマークを設定: ma
  2. 別の場所に移動して確認: /function_name Enter
  3. 元の位置に戻る: 'a または Ctrl+o

  シナリオ: 最後に編集した場所に戻りたい
  → '. で最後の変更位置にジャンプ
```

### ポイントまとめ

- `ma` でマーク設定、`'a` でマーク行に移動します
- `'.` で最後の変更位置、`''` で直前のジャンプ位置に戻れます
- `Ctrl+o` / `Ctrl+i` でジャンプ履歴を行き来できます

## 4. 外部コマンドとの連携

### 外部コマンドの実行

```vim
" 外部コマンドを実行（結果を表示）
:!ls -la
:!python3 %          " 現在のファイルを Python で実行
:!bash %             " 現在のファイルを Bash で実行

" 外部コマンドの結果をバッファに挿入
:r !date             " 現在の日時を挿入
:r !ls               " ファイル一覧を挿入
:r !curl -s URL      " URL の内容を挿入

" バッファの内容を外部コマンドに渡す
:w !wc -l            " 行数をカウント
:w !python3          " バッファの内容を Python で実行
```

### フィルタコマンド

バッファの内容を外部コマンドで加工（フィルタ）できます。

```vim
" ファイル全体をフィルタ
:%!sort              " 全行をソート
:%!sort -u           " 重複を除いてソート
:%!sort -n           " 数値としてソート
:%!uniq              " 連続する重複行を除去
:%!python3 -m json.tool  " JSON を整形

" 特定範囲をフィルタ
:10,20!sort          " 10〜20行目をソート

" ビジュアル選択した範囲をフィルタ
" V で行を選択してから :!sort
```

### ノーマルモードでのフィルタ

```
  !{モーション}{コマンド}

  例:
  !ip sort    ← 現在の段落をソート
  !G sort     ← 現在行からファイル末尾までソート
```

### 実践例

```vim
" JSON ファイルを整形
:%!python3 -m json.tool

" CSV をカラムでソート（2列目で数値ソート）
:%!sort -t, -k2 -n

" 選択範囲の行を逆順にする
:'<,'>!tac

" 現在のファイルを実行して結果を別バッファに表示
:new | r !python3 #
```

### ポイントまとめ

- `:!コマンド` で外部コマンドを実行します
- `:r !コマンド` で実行結果をバッファに挿入します
- `:%!コマンド` でバッファ全体をフィルタ処理します
- `:%!sort` や `:%!python3 -m json.tool` は特に実用的です

## 5. 便利な小技集

### 数値の増減

```
  Ctrl+a  ← カーソル上/カーソル以降の数値を +1
  Ctrl+x  ← カーソル上/カーソル以降の数値を -1
  {数値}Ctrl+a  ← 指定した数だけ加算

  例:
  "width: 100px" で100にカーソルを合わせて
  Ctrl+a → "width: 101px"
  5Ctrl+a → "width: 106px"
  Ctrl+x → "width: 105px"
```

### ファイルを開く

```
  gf      ← カーソル上のファイル名のファイルを開く (goto file)
  gx      ← カーソル上の URL をブラウザで開く

  例: import "./utils.js" の上で gf を押すと utils.js が開く
```

### その他の便利な操作

```
  J       ← 現在行と次の行を結合
  gJ      ← 行を結合（スペースを挿入しない）
  ~       ← カーソル上の文字の大文字/小文字を反転
  g~iw    ← 単語の大文字/小文字を反転
  gUiw    ← 単語を大文字に変換
  guiw    ← 単語を小文字に変換
  Ctrl+n  ← 補完候補（次）
  Ctrl+p  ← 補完候補（前）
  ga      ← カーソル上の文字のコード情報を表示
  g Ctrl+g ← カーソル位置の情報（行、単語数、バイト数）を表示
```

### ペーストモード

外部からテキストを貼り付ける際、自動インデントで崩れることがあります。

```vim
:set paste          " ペーストモード ON（自動インデントを無効化）
" テキストを貼り付け
:set nopaste        " ペーストモード OFF

" トグルキーを設定しておくと便利
set pastetoggle=<F2>
```

### アンドゥの時間指定

```vim
:earlier 5m         " 5分前の状態に戻す
:earlier 1h         " 1時間前の状態に戻す
:earlier 3f         " 3回前のファイル保存時の状態に戻す
:later 5m           " 5分後の状態に進む
```

### ポイントまとめ

- `Ctrl+a`/`Ctrl+x` で数値の増減ができます
- `gf` でファイル名からファイルを開けます
- `:set paste` で外部からの貼り付けが崩れません
- `:earlier`/`:later` で時間指定のアンドゥができます

## 6. Vim のプラグイン管理

### vim-plug の紹介

vim-plug は、最も人気のあるVimプラグインマネージャーです。

```bash
# vim-plug のインストール
curl -fLo ~/.vim/autoload/plug.vim --create-dirs \
    https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim
```

### .vimrc でのプラグイン設定

```vim
call plug#begin('~/.vim/plugged')

" ファイルエクスプローラ
Plug 'preservim/nerdtree'

" ファジーファインダー
Plug 'junegunn/fzf', { 'do': { -> fzf#install() } }
Plug 'junegunn/fzf.vim'

" 括弧やクォートの操作
Plug 'tpope/vim-surround'

" コメントアウト
Plug 'tpope/vim-commentary'

" Git 連携
Plug 'tpope/vim-fugitive'

" ステータスライン
Plug 'itchyny/lightline.vim'

" カラースキーム
Plug 'morhetz/gruvbox'

call plug#end()
```

```vim
" プラグインのインストール・管理コマンド
:PlugInstall        " プラグインをインストール
:PlugUpdate         " プラグインを更新
:PlugClean          " 未使用プラグインを削除
:PlugStatus         " プラグインの状態を表示
```

### おすすめプラグイン紹介

| プラグイン | 説明 |
|-----------|------|
| **vim-surround** | 括弧やクォートの追加・変更・削除（`cs"'` で `"` を `'` に変更） |
| **vim-commentary** | `gcc` でコメントアウト、`gc` + モーションで範囲コメント |
| **fzf.vim** | 高速なファジーファインダー（ファイル検索、grep 等） |
| **NERDTree** | ツリー形式のファイルエクスプローラ |
| **vim-fugitive** | Vim 内での Git 操作 |
| **lightline.vim** | 軽量なステータスラインプラグイン |

### ポイントまとめ

- vim-plug で簡単にプラグインを管理できます
- まずは vim-surround と vim-commentary から始めるのがおすすめです
- プラグインを入れすぎると起動が遅くなるので、本当に必要なものだけ入れましょう

## 7. 効率的なVimワークフロー

### Vim 内ターミナル

Vim 8.1 以降では、Vim 内でターミナルを開けます。

```vim
:terminal           " 水平分割でターミナルを開く（:term も可）
:vertical terminal  " 垂直分割でターミナルを開く
:tab terminal       " 新しいタブでターミナルを開く

" ターミナルモードでの操作
Ctrl+w N            " ターミナルノーマルモード（スクロール可能に）
i または a          " ターミナルジョブモードに戻る
Ctrl+w Ctrl+w       " ウィンドウ切り替え
exit                " ターミナルを終了
```

### tmux との連携

tmux（ターミナルマルチプレクサ）と Vim を組み合わせると、さらに効率的な開発環境が構築できます。

```
  tmux のペイン構成例:

  ┌─────────────────────┬──────────────┐
  │                     │              │
  │                     │  ターミナル   │
  │     Vim             │ （テスト実行）│
  │                     │              │
  │                     ├──────────────┤
  │                     │              │
  │                     │  ターミナル   │
  │                     │ （Git操作）   │
  │                     │              │
  └─────────────────────┴──────────────┘
```

### 効率的な作業のヒント

```
  1. ホームポジションから手を離さない
     → hjkl 移動、Ctrl+[ でESC、jj マッピング

  2. 繰り返しを減らす
     → . コマンド、マクロ、:s 置換を活用

  3. 移動は最小限に
     → f/t、テキストオブジェクト、/ 検索で目的地に直行

  4. 操作は原子的に
     → ciw、dap など「意味のある単位」で操作する
     → ドットコマンドで繰り返しやすい操作を心がける

  5. Vim を「言語」として考える
     → d（削除）+ iw（単語の中）= diw
     → 新しい動詞（オペレータ）や名詞（モーション）を1つ覚えるたびに
        組み合わせが指数的に増える
```

### ポイントまとめ

- `:terminal` で Vim 内からコマンドを実行できます
- tmux と組み合わせるとさらに効率的です
- 「繰り返しを減らす」「意味のある単位で操作する」が上達のコツです
- Vim はすぐにはマスターできませんが、毎日少しずつ新しいコマンドを覚えていきましょう
