# 第1章 vimrcの基本と仕組み

## この章のゴール

- vimrc が何であり、Vim においてどのような役割を果たすかを理解する
- vimrc の配置場所をOS別に把握する
- Vim 起動時の設定ファイル読み込み順序を理解する
- vimrc の作成・編集・再読み込みができるようになる
- `set` コマンドと `let` コマンドの基本的な使い方を習得する

---

## 1.1 vimrcとは

### Vimの設定ファイルとしての役割

vimrc（ブイアイエムアールシー）とは、Vim の起動時に自動的に読み込まれる**設定ファイル**（configuration file）です。Vim のデフォルトの動作を自分好みにカスタマイズするために使用します。

vimrc には以下のような設定を記述できます。

- エディタの見た目（行番号の表示、カラースキームなど）
- 編集動作（インデント幅、タブの扱いなど）
- キーマッピング（ショートカットキーの変更・追加）
- プラグインの読み込みと設定
- 自作関数やコマンドの定義

```
┌─────────────────────────────────────────────────────┐
│                  Vimの起動フロー                      │
├─────────────────────────────────────────────────────┤
│                                                      │
│   vim コマンド実行                                   │
│        │                                             │
│        ▼                                             │
│   vimrc を読み込む                                   │
│   （set number, colorscheme, map ... など）           │
│        │                                             │
│        ▼                                             │
│   カスタマイズされた状態で                           │
│   エディタが起動する                                 │
│                                                      │
└─────────────────────────────────────────────────────┘
```

vimrc がない状態で Vim を起動すると、すべてデフォルトの設定で動作します。vimrc を用意することで、毎回同じ快適な環境でコーディングができるようになります。

### Vi互換モードとの関係

Vim は歴史的に vi エディタの後継として開発されました。vimrc が**存在しない**場合、Vim は vi との互換性を保つ「互換モード」（compatible mode）で起動します。互換モードでは、Vim 独自の便利な機能の多くが無効になります。

vimrc を作成するだけで、Vim は自動的に `nocompatible`（非互換モード）に切り替わり、Vim のフル機能が利用できるようになります。

```vim
" vimrc の先頭に書くことが推奨される設定
" （vimrc が存在する時点で自動的に nocompatible になりますが、
"   明示的に書いておくと意図が明確になります）
set nocompatible
```

> **よくある間違い**: 「set compatible」を vimrc に書いてしまうと、Vim 独自の機能（ビジュアルモード、複数アンドゥなど）が使えなくなります。特別な理由がない限り、`set nocompatible` を使いましょう。

### ポイントまとめ

- vimrc は Vim の設定ファイルであり、起動時に自動的に読み込まれます
- エディタの見た目、編集動作、キーマッピングなどをカスタマイズできます
- vimrc が存在するだけで Vi 互換モードが無効になり、Vim のフル機能が使えます

## 1.2 vimrcの配置場所

### OS別の配置場所

vimrc の場所はOSによって異なります。以下の表で確認しましょう。

| OS | ユーザー用 vimrc | 備考 |
|-----|------------------|------|
| **Linux** | `~/.vimrc` | 最も一般的 |
| **macOS** | `~/.vimrc` | Linux と同じ |
| **Windows** | `$HOME/_vimrc` または `$HOME/.vimrc` | `_vimrc` が優先される |

```bash
# Linux / macOS でホームディレクトリを確認する
echo $HOME
# 出力例: /home/username

# Windows（PowerShell）でホームディレクトリを確認する
echo $HOME
# 出力例: C:\Users\username
```

### ~/.vimrc と ~/.vim/vimrc

Vim 8.0 以降では、設定ファイルを `~/.vim/vimrc` に置くこともできます。この方法には大きなメリットがあります。

```
従来の方法:
  ~/.vimrc          ← 設定ファイル（ホームディレクトリ直下）
  ~/.vim/            ← プラグイン等のディレクトリ

推奨される方法（Vim 8.0以降）:
  ~/.vim/
  ├── vimrc          ← 設定ファイル（.vim ディレクトリの中）
  ├── pack/          ← プラグイン
  ├── colors/        ← カラースキーム
  └── after/         ← 後から読み込む設定
```

`~/.vim/vimrc` に配置すると、`.vim` ディレクトリごと Git で管理でき、別のマシンへの移行が容易になります。

```bash
# ~/.vim/vimrc を使う場合の構成例
mkdir -p ~/.vim
mv ~/.vimrc ~/.vim/vimrc
```

> **よくある間違い**: `~/.vimrc` と `~/.vim/vimrc` の両方が存在する場合、`~/.vimrc` が優先されます。設定が反映されないと思ったら、両方にファイルがないか確認しましょう。

### Neovimとの違い

Neovim（nvim）は Vim のフォーク（fork）であり、設定ファイルの場所が異なります。

| エディタ | 設定ファイル | 設定ディレクトリ |
|----------|-------------|-----------------|
| **Vim** | `~/.vimrc` または `~/.vim/vimrc` | `~/.vim/` |
| **Neovim** | `~/.config/nvim/init.vim` | `~/.config/nvim/` |
| **Neovim (Lua)** | `~/.config/nvim/init.lua` | `~/.config/nvim/` |

```bash
# Neovim の設定ディレクトリを作成する
mkdir -p ~/.config/nvim

# Neovim で Vim の vimrc を読み込みたい場合
# ~/.config/nvim/init.vim に以下を記述する
# source ~/.vimrc
```

Neovim は Vim の設定ファイルを直接読み込まないため、移行する場合は `init.vim` から `source` で読み込むか、設定を書き直す必要があります。

### ポイントまとめ

- Linux / macOS では `~/.vimrc` が標準的な配置場所です
- Vim 8.0 以降は `~/.vim/vimrc` に置くと Git 管理がしやすくなります
- Neovim は `~/.config/nvim/init.vim` を使い、Vim とは場所が異なります

## 1.3 vimrcの読み込み順序

### 起動時の読み込みフロー

Vim は起動時に複数の設定ファイルを**決まった順序**で読み込みます。全体の流れを理解しておくと、設定が意図通りに反映されないときのトラブルシューティングに役立ちます。

```
┌──────────────────────────────────────────────────────────┐
│              Vim 起動時の設定読み込み順序                  │
├──────────────────────────────────────────────────────────┤
│                                                           │
│  1. system vimrc                                          │
│     /etc/vim/vimrc や /usr/share/vim/vimrc                │
│     （システム管理者が全ユーザー共通で設定）              │
│           │                                               │
│           ▼                                               │
│  2. user vimrc                                            │
│     ~/.vimrc または ~/.vim/vimrc                          │
│     （ユーザー個別の設定 ← ここを編集する）              │
│           │                                               │
│           ▼                                               │
│  3. user exrc（あまり使わない）                           │
│     ~/.exrc                                               │
│           │                                               │
│           ▼                                               │
│  4. $VIMRUNTIME/defaults.vim                              │
│     （user vimrc がない場合のみ読み込まれる）             │
│           │                                               │
│           ▼                                               │
│  5. プラグインファイル                                    │
│     plugin/, ftplugin/ 等                                 │
│           │                                               │
│           ▼                                               │
│  6. gvimrc（GVim使用時のみ）                             │
│     ~/.gvimrc                                             │
│                                                           │
└──────────────────────────────────────────────────────────┘
```

後から読み込まれた設定が、先に読み込まれた設定を**上書き**します。つまり、user vimrc の設定は system vimrc の設定より優先されます。

### :version で確認する方法

Vim がどの設定ファイルを読み込むかは `:version` コマンドで確認できます。

```vim
" Vim 内で以下を実行する
:version
```

出力の中から以下のような行を探します。

```
   system vimrc file: "/etc/vim/vimrc"
     user vimrc file: "$HOME/.vimrc"
 2nd user vimrc file: "~/.vim/vimrc"
      user exrc file: "$HOME/.exrc"
       defaults file: "$VIMRUNTIME/defaults.vim"
  fall-back for $VIM: "/usr/share/vim"
```

この情報を見れば、お使いの環境で Vim がどのパスの設定ファイルを期待しているかがわかります。

```vim
" 実際に読み込まれたスクリプトの一覧を確認する
:scriptnames
```

`:scriptnames` を実行すると、現在の Vim セッションで**実際に読み込まれた**すべてのスクリプトが読み込み順に表示されます。

### ポイントまとめ

- Vim は system vimrc → user vimrc → プラグイン → gvimrc の順で設定を読み込みます
- 後から読み込まれた設定が優先されます
- `:version` で設定ファイルのパスを、`:scriptnames` で実際に読み込まれたファイルを確認できます

## 1.4 vimrcの作成と編集

### vimrc ファイルを作成する

まだ vimrc がない場合は、Vim 自身を使って作成しましょう。

```bash
# ターミナルで以下を実行する
vim ~/.vimrc
```

新規ファイルとして開かれるので、設定を記述して保存します。

### コメントの書き方

vimrc ではダブルクォーテーション `"` でコメントを記述します。

```vim
" これはコメントです（行全体がコメント）
set number    " 行頭以外でもコメントを書ける

" 良い vimrc の習慣: セクションごとにコメントで区切る
" ==================================================
" 基本設定
" ==================================================
set number
set ruler

" ==================================================
" 検索設定
" ==================================================
set hlsearch
set incsearch
```

> **よくある間違い**: 一部のコマンド（`map` コマンドなど）では、行末の `"` がコメントではなく入力文字として解釈される場合があります。マッピング定義の後にコメントを書きたい場合は、次の行にコメントを書くのが安全です。

### :source で再読み込み

vimrc を編集した後、Vim を再起動しなくても `:source` コマンドで設定を反映できます。

```vim
" vimrc を再読み込みする
:source ~/.vimrc

" 短縮形
:so ~/.vimrc

" 現在編集中のファイルを source する（vimrc を開いている場合）
:source %
```

vimrc に以下のようなキーマッピングを追加しておくと、素早く再読み込みできます。

```vim
" vimrc を素早く開く
nnoremap <Leader>ev :edit $MYVIMRC<CR>

" vimrc を素早く再読み込みする
nnoremap <Leader>sv :source $MYVIMRC<CR>
```

`$MYVIMRC` は Vim が認識した vimrc のパスを保持する特殊な環境変数です。

### :set で一時的に設定を確認する

Vim のコマンドラインモードで `:set` を使うと、vimrc を編集しなくても一時的に設定を変更・確認できます。これはテスト用途に便利です。

```vim
" 現在の行番号設定を確認する
:set number?
" 出力例: number  または  nonumber

" 一時的に行番号を有効にする（Vim を閉じるとリセットされる）
:set number

" 現在変更されているすべての設定を表示する
:set

" すべての設定を表示する
:set all
```

### ポイントまとめ

- `vim ~/.vimrc` で vimrc を作成・編集します
- コメントは `"` で記述します
- `:source ~/.vimrc` で Vim を再起動せずに設定を反映できます
- `:set option?` で現在の設定値を確認できます
- `$MYVIMRC` は Vim が認識した vimrc のパスを示す特殊変数です

## 1.5 set コマンドの基本

`set` コマンドは Vim のオプション（option）を設定するための最も基本的なコマンドです。vimrc の大部分は `set` コマンドで構成されます。

### set の書式パターン

Vim のオプションには主に3種類があり、それぞれ `set` の使い方が異なります。

| 種類 | 設定方法 | 例 |
|------|---------|-----|
| **ブーリアン型** | `set option` / `set nooption` | `set number` / `set nonumber` |
| **数値型** | `set option=値` | `set tabstop=4` |
| **文字列型** | `set option=値` | `set statusline=%f\ %m` |

```vim
" ──────────────────────────────────
" ブーリアン型（ON/OFF の切り替え）
" ──────────────────────────────────

" 行番号を表示する（ON）
set number

" 行番号を非表示にする（OFF）
set nonumber

" 行番号の表示状態を切り替える（トグル）
set number!

" 現在の状態を確認する
set number?
" 出力: number  または  nonumber

" ──────────────────────────────────
" 数値型
" ──────────────────────────────────

" タブ幅を4に設定する
set tabstop=4

" 現在の値を確認する
set tabstop?
" 出力: tabstop=4

" ──────────────────────────────────
" 文字列型
" ──────────────────────────────────

" ファイルのエンコーディングを設定する
set fileencoding=utf-8

" 現在の値を確認する
set fileencoding?
" 出力: fileencoding=utf-8
```

### デフォルト値に戻す

設定を試行錯誤しているうちに元に戻したくなることがあります。`set option&` でデフォルト値にリセットできます。

```vim
" タブ幅をデフォルトに戻す
set tabstop&

" 確認する
set tabstop?
" 出力: tabstop=8  （Vim のデフォルトは 8）
```

### 複数のオプションをまとめて設定する

```vim
" 1行にまとめて書くこともできる
set number ruler cursorline

" 可読性のために1行ずつ書くことが推奨される
set number
set ruler
set cursorline
```

### 実用的な set コマンドの例

以下は実務で頻繁に使われる基本設定です。

```vim
" ==================================================
" 基本的な vimrc 設定の例
" ==================================================

" Vi 互換モードを無効にする
set nocompatible

" 文字コード
set encoding=utf-8
set fileencoding=utf-8

" 行番号を表示する
set number

" 入力中のコマンドを右下に表示する
set showcmd

" 対応する括弧をハイライトする
set showmatch

" バックスペースの挙動を直感的にする
set backspace=indent,eol,start

" ビープ音を無効にする
set belloff=all
```

### ポイントまとめ

- `set option` で ON、`set nooption` で OFF にします（ブーリアン型）
- `set option=value` で数値型・文字列型のオプションを設定します
- `set option?` で現在の値を確認、`set option&` でデフォルトに戻せます
- `set option!` でブーリアン型の ON/OFF をトグルできます

## 1.6 let コマンドの基本

`let` コマンドは Vim スクリプト（Vim script）における**変数**を設定するためのコマンドです。`set` がオプション（Vim の組み込み設定）を扱うのに対し、`let` はより汎用的な変数の操作に使います。

### set と let の違い

```
┌──────────────────────────────────────────────────────┐
│              set と let の使い分け                     │
├─────────────────────┬────────────────────────────────┤
│       set           │           let                  │
├─────────────────────┼────────────────────────────────┤
│ Vim オプションの設定│ 変数の設定                     │
│ set number          │ let g:my_var = 1               │
│ set tabstop=4       │ let g:color = "blue"           │
│ Vim 組み込みの設定  │ プラグイン設定や自作変数       │
│ : の前置は不要      │ スコープ指定が可能             │
└─────────────────────┴────────────────────────────────┘
```

### 変数のスコープ

Vim の変数にはスコープ（scope）があり、プレフィックス（prefix）で区別します。

| プレフィックス | スコープ | 説明 |
|:---:|---|---|
| `g:` | グローバル | どこからでもアクセスできる。最もよく使う |
| `b:` | バッファローカル | 現在のバッファ（ファイル）内でのみ有効 |
| `w:` | ウィンドウローカル | 現在のウィンドウ内でのみ有効 |
| `t:` | タブページローカル | 現在のタブページ内でのみ有効 |
| `l:` | 関数ローカル | 関数内でのみ有効 |
| `s:` | スクリプトローカル | そのスクリプトファイル内でのみ有効 |
| `v:` | Vim 定義変数 | Vim が定義する特殊変数（読み取り専用が多い） |

```vim
" ──────────────────────────────────
" グローバル変数
" ──────────────────────────────────

" グローバル変数を設定する
let g:my_name = "Vim User"

" 変数の値を確認する
echo g:my_name
" 出力: Vim User

" 数値変数
let g:max_count = 100

" リスト（配列）
let g:my_list = [1, 2, 3, 4, 5]

" 辞書（連想配列）
let g:my_dict = {"name": "Vim", "version": 9}

" ──────────────────────────────────
" Vim 定義変数（v: スコープ）
" ──────────────────────────────────

" Vim のバージョンを確認する
echo v:version
" 出力例: 900  （Vim 9.0 の場合）

" 直前の検索パターン
echo v:searchforward

" 直前のコマンドの終了ステータス
echo v:shell_error
```

### プラグイン設定での使用例

`let` コマンドは、プラグインの設定で頻繁に使われます。プラグインはグローバル変数を読み取って動作を変更する設計が一般的です。

```vim
" ==================================================
" プラグイン設定の例（よくあるパターン）
" ==================================================

" netrw（Vim 組み込みのファイラー）の設定
let g:netrw_banner = 0        " バナーを非表示にする
let g:netrw_liststyle = 3     " ツリー表示にする
let g:netrw_winsize = 25      " ウィンドウ幅を25%にする
```

### 変数の削除と確認

```vim
" 変数を削除する
unlet g:my_name

" 変数が存在するか確認してから削除する（エラー防止）
unlet! g:my_name

" 変数が存在するか確認する
if exists("g:my_name")
  echo "変数は存在します: " . g:my_name
else
  echo "変数は存在しません"
endif
```

### オプション値を let で操作する

`set` で設定するオプションも、`let` と `&` プレフィックスを使って操作できます。

```vim
" set tabstop=4 と同じ意味
let &tabstop = 4

" set number と同じ意味
let &number = 1

" 条件に応じてオプションを変えたい場合に便利
if has("gui_running")
  let &lines = 40
  let &columns = 120
endif
```

### ポイントまとめ

- `let` は変数を設定するコマンドで、`set` とは用途が異なります
- 変数にはスコープがあり、`g:`（グローバル）が最もよく使われます
- プラグインの設定は `let g:plugin_option = value` の形で行います
- `unlet` で変数を削除、`exists()` で存在確認ができます
- `let &option = value` でオプション値を変数のように操作できます
