<details>
<summary>解答例</summary>

```
以下の技術ブログ記事を、Slack投稿用に要約してください。

## 要約のルール
- 1行目：記事タイトルと一言コメント（30文字以内）
- 2行目以降：主要ポイントを3つの箇条書き
- 最終行：「こんな人におすすめ」を1文で

## 出力フォーマット
📝 [タイトル] - [一言コメント]
• ポイント1
• ポイント2
• ポイント3
👉 [おすすめの対象者]

## 記事
```

### 問題2：コード生成プロンプトの作成

以下の仕様を満たすコード生成プロンプトを作成してください。

仕様：Pythonで、CSVファイルを読み込み、指定された列の統計情報（平均、中央値、最大値、最小値、標準偏差）を計算して結果をJSON形式で出力する関数。

<details>
<summary>解答例</summary>

```
以下の仕様に基づいてPython関数を実装してください。

## 関数仕様
関数名：analyze_csv_column
入力：
  - file_path: str（CSVファイルのパス）
  - column_name: str（分析対象の列名）
出力：dict（統計情報を含む辞書）

## 出力例
{
  "column": "売上",
  "count": 12,
  "mean": 1250000.0,
  "median": 1175000.0,
  "max": 1800000,
  "min": 980000,
  "std": 245000.5
}

## 制約
- 標準ライブラリのみ使用（csv, statistics, json）
- 型ヒント（type hints）を使用する
- docstringを含める
- 以下のエラーハンドリングを含める：
  - ファイルが存在しない場合：FileNotFoundError
  - 指定した列が存在しない場合：ValueError
  - 数値以外のデータが含まれる場合：スキップして処理を続行
- テストケースを3つ含める（正常系1、異常系2）
```
</details>

## 応用問題

### 問題4：障害分析（RCA）プロンプト

以下の障害情報を基に、RCA（根本原因分析）を行うプロンプトを作成してください。

障害情報：
- 発生：2026/03/24 09:00 JST
- 症状：Webアプリのレスポンスが5秒以上に悪化
- 影響：全ユーザー（推定5,000人が影響）
- 復旧：2026/03/24 10:30 JST
- 直前の変更：前日夜にDB接続プール設定を変更するデプロイを実施

<details>
<summary>解答例</summary>

```
あなたはSRE（Site Reliability Engineer）です。
以下の障害について根本原因分析（RCA）を行ってください。

## 障害概要
- 発生日時：2026/03/24 09:00 JST
- 復旧日時：2026/03/24 10:30 JST（ダウンタイム：1.5時間）
- 症状：Webアプリのレスポンスが通常200ms → 5秒以上に悪化
- 影響範囲：全ユーザー（推定5,000人）
- 直前の変更：前日夜にDB接続プール設定を変更するデプロイ実施

## 分析フレームワーク

### 1. タイムライン
事象を時系列で整理してください。

### 2. 5 Whys分析
「なぜ？」を5回繰り返して根本原因に到達してください。

### 3. 直接原因と根本原因
- 直接原因（何が起きたか）
- 根本原因（なぜ起きたか）
- 寄与要因（起きやすくした要因）

### 4. 再発防止策
以下の3つのカテゴリで提案してください：
- 短期（1週間以内）：即座に対応すべきこと
- 中期（1ヶ月以内）：プロセスの改善
- 長期（3ヶ月以内）：仕組みの構築

### 5. アクションアイテム
| # | アクション | 担当 | 期限 | 優先度 |
```
</details>

## チャレンジ問題

### 問題5：複合パターンの設計

あなたは技術ブログの編集長です。エンジニアが書いた技術記事のドラフトを受け取り、公開前に品質チェックと改善提案を行うプロンプトを設計してください。

以下のパターンを組み合わせてください：
- 分析パターン（記事の品質評価）
- 校正パターン（文章の改善）
- コード生成パターン（コード例の検証）
- 要約パターン（SNS共有用の要約生成）

<details>
<summary>解答例</summary>

```
あなたは技術ブログの編集長です。
以下の技術記事のドラフトを品質チェックし、
改善提案を行ってください。

## 記事ドラフト
"""
（記事の内容をここに貼り付け）
"""

## チェック項目と出力形式

### 1. 品質評価（分析パターン）
以下の基準で5段階評価してください：
| 基準 | スコア(1-5) | コメント |
- 技術的正確性
- 構成の論理性
- 読みやすさ
- 実用性（読者が再現可能か）
- オリジナリティ

### 2. 文章の改善（校正パターン）
以下の観点で改善箇所を5つまで指摘してください：
- 誤字・脱字
- 表現の不自然さ
- 冗長な表現
- 専門用語の説明不足
各指摘：修正前 → 修正後 の形式で

### 3. コード例の検証（コード生成パターン）
記事内のコード例について：
- 文法エラーはないか
- 実行可能か
- ベストプラクティスに沿っているか
- 改善案があれば提示

### 4. SNS共有用テキスト（要約パターン）
以下の3パターンを生成してください：
- X（140文字以内）
- はてなブックマーク用コメント（100文字以内）
- 社内Slack投稿用（200文字以内、箇条書き）

### 5. 総合判断
□ 公開OK（そのまま公開可能）
□ 軽微な修正後に公開（指摘箇所を修正すれば公開可能）
□ 大幅な修正が必要（構成や内容の見直しが必要）
```
</details>
