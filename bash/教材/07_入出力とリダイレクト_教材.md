# 第7章：入出力とリダイレクト

## この章のゴール

- 標準入力・標準出力・標準エラー出力の概念を理解すること
- リダイレクトの各種パターンを使いこなせるようになること
- ヒアドキュメントとヒアストリングを活用できるようになること
- パイプとteeを組み合わせた実践的なデータ処理ができるようになること
- プロセス置換とファイルディスクリプタを理解すること

---

## 7.1 標準入出力

Unix/Linuxでは、すべてのプロセスに3つの標準的な入出力チャネルがあります。

```
┌──────────────────────────────────────────────┐
│                 プロセス                       │
│                                               │
│  stdin (0)  ──→ ┌─────────┐ ──→ stdout (1)  │
│  （標準入力）    │  処理    │    （標準出力）  │
│                  └─────────┘                  │
│                       │                       │
│                       └──→ stderr (2)         │
│                           （標準エラー出力）   │
└──────────────────────────────────────────────┘
```

| チャネル | ファイルディスクリプタ | 説明 | デフォルト |
|----------|:---:|------|---------|
| 標準入力（stdin） | 0 | プログラムへの入力 | キーボード |
| 標準出力（stdout） | 1 | プログラムの通常出力 | 画面 |
| 標準エラー出力（stderr） | 2 | エラーメッセージ | 画面 |

```bash
#!/usr/bin/env bash

# stdout に出力
echo "これは標準出力です"

# stderr に出力
echo "これはエラー出力です" >&2

# stdin から読み取り
read -p "入力してください: " input
echo "入力: $input"
```

### ポイントまとめ

- stdin(0)、stdout(1)、stderr(2) の3つが基本
- `>&2` で標準エラー出力に書き込み
- エラーメッセージは必ず stderr に出すのがマナー

## 7.2 リダイレクト

### 出力リダイレクト

```bash
#!/usr/bin/env bash

# stdout をファイルに書き込み（上書き）
echo "Hello" > output.txt

# stdout をファイルに追記
echo "World" >> output.txt

# stderr をファイルに書き込み
ls /nonexistent 2> error.log

# stdout と stderr を別々のファイルに
command > stdout.log 2> stderr.log

# stdout と stderr を同じファイルに
command > all.log 2>&1
# または（Bash 4以降の省略形）
command &> all.log

# 出力を捨てる（/dev/null）
command > /dev/null 2>&1
# または
command &> /dev/null
```

### 入力リダイレクト

```bash
#!/usr/bin/env bash

# ファイルから入力
wc -l < /etc/hosts

# sort にファイルを入力
sort < unsorted.txt > sorted.txt
```

### リダイレクトの順序

```bash
#!/usr/bin/env bash

# 【正しい】stderr を stdout にリダイレクトしてからファイルへ
command > output.txt 2>&1

# 【誤り】順序が逆だと stderr はファイルに行かない
command 2>&1 > output.txt
```

> **覚え方**: `2>&1` は「2番（stderr）を1番（stdout）と同じ先に向ける」という意味です。リダイレクトは左から右に評価されるので、先に stdout の向き先を決めてから stderr を合流させます。

### ポイントまとめ

- `>` は上書き、`>>` は追記
- `2>` で stderr をリダイレクト
- `&>` で stdout と stderr を同時にリダイレクト
- `/dev/null` は出力の捨て場所

## 7.3 ヒアドキュメントとヒアストリング

### ヒアドキュメント（Here Document）

```bash
#!/usr/bin/env bash

# 基本形（変数展開あり）
name="田中"
cat <<EOF
こんにちは、${name}さん。
今日は $(date +%Y年%m月%d日) です。
お元気ですか？
EOF
```

実行結果：
```
こんにちは、田中さん。
今日は 2026年03月27日 です。
お元気ですか？
```

### 変数展開なしのヒアドキュメント

```bash
#!/usr/bin/env bash

# デリミタをクォートすると変数展開しない
cat <<'EOF'
変数展開されません: $HOME
コマンド置換もされません: $(date)
そのまま表示されます
EOF
```

### インデント付きヒアドキュメント

```bash
#!/usr/bin/env bash

# <<- を使うとタブインデントが除去される
if true; then
    cat <<-EOF
	これはインデントされたヒアドキュメントです
	タブインデントが除去されます
	EOF
fi
```

### ヒアストリング（Here String）

```bash
#!/usr/bin/env bash

# <<< で文字列を stdin に渡す
read -r first last <<< "Taro Tanaka"
echo "名: $first, 姓: $last"

# 変数の内容を渡す
data="apple,banana,cherry"
IFS=',' read -r a b c <<< "$data"
echo "$a, $b, $c"
```

### 実践例：設定ファイルの生成

```bash
#!/usr/bin/env bash

db_host="localhost"
db_port=5432
db_name="myapp"

cat <<EOF > /tmp/config.yml
database:
  host: ${db_host}
  port: ${db_port}
  name: ${db_name}
  pool_size: 10
EOF

echo "設定ファイルを生成しました"
cat /tmp/config.yml
rm -f /tmp/config.yml
```

### ポイントまとめ

- `<<EOF ... EOF` で複数行テキストを扱う
- `<<'EOF'` で変数展開を無効にする
- `<<<` で1行の文字列を stdin に渡す
- 設定ファイルの生成に非常に便利

## 7.4 パイプ

### パイプの基本

```bash
#!/usr/bin/env bash

# コマンドの出力を次のコマンドの入力に
ls -la | grep ".sh"

# 複数のパイプを連結
cat /etc/passwd | cut -d: -f1 | sort | head -10

# 行数をカウント
find . -name "*.sh" | wc -l
```

### パイプラインの実践パターン

```bash
#!/usr/bin/env bash

# プロセスのメモリ使用量上位5件
ps aux | sort -k4 -rn | head -5

# ファイルの拡張子別カウント
find . -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn

# アクセスログからIPアドレス上位10件
# cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -10
```

### tee コマンド

`tee` は stdout をファイルに保存しつつ、次のパイプにも渡します。

```bash
#!/usr/bin/env bash

# 画面にも表示しつつファイルにも保存
echo "Hello, World!" | tee output.txt

# 追記モード
echo "追加行" | tee -a output.txt

# 複数ファイルに同時書き込み
echo "共通データ" | tee file1.txt file2.txt > /dev/null

# パイプラインの途中でログを取る
cat /etc/passwd | tee /tmp/passwd_backup.txt | wc -l

# クリーンアップ
rm -f output.txt file1.txt file2.txt /tmp/passwd_backup.txt
```

```
┌──────┐    ┌──────┐    ┌───────┐
│ echo │───→│ tee  │───→│ wc -l │ → 画面
└──────┘    └──┬───┘    └───────┘
               │
               └──→ ファイル
```

### PIPESTATUS

```bash
#!/usr/bin/env bash

# パイプライン中の各コマンドの終了ステータスを取得
false | true | false
echo "PIPESTATUS: ${PIPESTATUS[@]}"  # → 1 0 1

# set -o pipefail でパイプラインのエラーを検出
set -o pipefail
if ! false | true; then
    echo "パイプラインの一部が失敗しました"
fi
```

### ポイントまとめ

- `|` でコマンドの出力を次の入力に接続
- `tee` で出力を分岐（ファイルとパイプの両方に）
- `${PIPESTATUS[@]}` でパイプライン中の各ステータスを確認
- `set -o pipefail` でパイプのエラーを検出

## 7.5 プロセス置換

プロセス置換（process substitution）は、コマンドの出力をファイルのように扱う機能です。

```bash
#!/usr/bin/env bash

# <() は出力をファイルとして読む
diff <(sort file1.txt) <(sort file2.txt)

# 2つのコマンド出力を比較
diff <(ls /usr/bin) <(ls /usr/local/bin)

# while read と組み合わせ（サブシェル問題の回避）
count=0
while IFS= read -r line; do
    ((count++))
done < <(find . -name "*.sh")
echo "シェルスクリプト数: $count"
```

### >() で出力先にプロセスを指定

```bash
#!/usr/bin/env bash

# 出力を複数のコマンドに同時に渡す
echo "Hello, World!" | tee >(wc -c > /tmp/char_count.txt) >(wc -w > /tmp/word_count.txt) > /dev/null

echo "文字数: $(cat /tmp/char_count.txt)"
echo "単語数: $(cat /tmp/word_count.txt)"
rm -f /tmp/char_count.txt /tmp/word_count.txt
```

### ポイントまとめ

- `<(command)` でコマンド出力をファイルとして読み込み
- `>(command)` で出力先としてコマンドを指定
- `diff <(cmd1) <(cmd2)` はよく使うパターン

## 7.6 ファイルディスクリプタ

### exec でファイルディスクリプタを操作

```bash
#!/usr/bin/env bash

# カスタムファイルディスクリプタ（3番以降）
exec 3> /tmp/custom_output.txt  # fd3 をファイルに紐付け

echo "標準出力に表示"
echo "fd3 に書き込み" >&3
echo "これも標準出力"
echo "これもfd3" >&3

exec 3>&-  # fd3 を閉じる

cat /tmp/custom_output.txt
rm -f /tmp/custom_output.txt
```

### 入力用ファイルディスクリプタ

```bash
#!/usr/bin/env bash

# fd3 から読み取り
exec 3< /etc/hosts

while IFS= read -r line <&3; do
    echo "読み取り: $line"
done

exec 3<&-  # fd3 を閉じる
```

### stdout と stderr の入れ替え

```bash
#!/usr/bin/env bash

# stdout と stderr を入れ替えるテクニック
# 3>&1  : fd3 に stdout を保存
# 1>&2  : stdout を stderr に向ける
# 2>&3  : stderr を fd3（元の stdout）に向ける
# 3>&-  : fd3 を閉じる

some_command() {
    echo "標準出力"
    echo "エラー出力" >&2
}

# stderr だけをパイプに渡す
some_command 3>&1 1>&2 2>&3 3>&- | grep "エラー"
```

### ポイントまとめ

- `exec N> file` でカスタムファイルディスクリプタを開く
- `>&N` でファイルディスクリプタに書き込み
- `exec N>&-` でファイルディスクリプタを閉じる

## 7.7 実践例

### ログ出力の実装

```bash
#!/usr/bin/env bash

LOG_FILE="/tmp/script_$(date +%Y%m%d).log"

# ログ関数（画面とファイルの両方に出力）
log() {
    local level="$1"
    shift
    local message="[$(date '+%Y-%m-%d %H:%M:%S')] [$level] $*"
    echo "$message" | tee -a "$LOG_FILE"
}

log "INFO" "スクリプトを開始しました"
log "WARN" "ディスク容量が80%を超えています"
log "ERROR" "ファイルが見つかりません"

echo "ログファイル: $LOG_FILE"
rm -f "$LOG_FILE"
```

### 一時ファイルの安全な使用

```bash
#!/usr/bin/env bash

# mktemp で安全な一時ファイルを作成
tmpfile=$(mktemp)

# trap で確実にクリーンアップ
trap 'rm -f "$tmpfile"' EXIT

# 一時ファイルを使った処理
echo "一時データ" > "$tmpfile"
cat "$tmpfile"
# スクリプト終了時に自動的に削除される
```

### ポイントまとめ

- `tee -a` でログの画面出力とファイル保存を同時に
- `mktemp` + `trap EXIT` で一時ファイルの安全な管理
- エラーメッセージは必ず `>&2` で stderr に出力

## 7.8 章全体のまとめ

| 操作 | 構文 | 説明 |
|------|------|------|
| stdout リダイレクト | `> file` | ファイルに上書き |
| stdout 追記 | `>> file` | ファイルに追記 |
| stderr リダイレクト | `2> file` | エラーをファイルに |
| 両方リダイレクト | `&> file` | stdout+stderr |
| 出力を捨てる | `> /dev/null 2>&1` | 完全に無音 |
| ヒアドキュメント | `<<EOF ... EOF` | 複数行テキスト |
| ヒアストリング | `<<< "text"` | 1行テキスト |
| パイプ | `cmd1 \| cmd2` | 出力を入力に |
| tee | `cmd \| tee file` | 出力を分岐 |
| プロセス置換 | `<(cmd)` | 出力をファイルとして |
