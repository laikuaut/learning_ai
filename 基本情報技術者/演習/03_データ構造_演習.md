## 基本問題

### 問題1：配列のアドレス計算
先頭アドレスが 2000、要素サイズが 4バイトの1次元配列 A[0]〜A[9] において、A[7] のアドレスを求めてください。

<details>
<summary>解答</summary>

```
A[7] のアドレス = 先頭アドレス + 7 × 要素サイズ
               = 2000 + 7 × 4
               = 2000 + 28
               = 2028
```
答え：**2028**
</details>

### 問題2：スタックの操作
スタックに対して以下の操作を順に行ったとき、最終的にスタックに残っているデータを底から順に答えてください。

push(A) → push(B) → push(C) → pop() → push(D) → pop() → pop()

<details>
<summary>解答</summary>

```
push(A) → [A]
push(B) → [A, B]
push(C) → [A, B, C]
pop()   → C を取り出し → [A, B]
push(D) → [A, B, D]
pop()   → D を取り出し → [A, B]
pop()   → B を取り出し → [A]
```
答え：スタックに残っているのは **A** のみ
</details>

### 問題3：キューの操作
キューに対して以下の操作を順に行ったとき、取り出されたデータを順に答えてください。

enqueue(X) → enqueue(Y) → enqueue(Z) → dequeue() → dequeue() → enqueue(W) → dequeue()

<details>
<summary>解答</summary>

```
enqueue(X) → [X]
enqueue(Y) → [X, Y]
enqueue(Z) → [X, Y, Z]
dequeue()  → X を取り出し → [Y, Z]
dequeue()  → Y を取り出し → [Z]
enqueue(W) → [Z, W]
dequeue()  → Z を取り出し → [W]
```
取り出された順：**X, Y, Z**
</details>

### 問題4：四択問題
後入れ先出し（LIFO）の特徴を持つデータ構造はどれですか。

ア. キュー　　イ. スタック　　ウ. リスト　　エ. 配列

<details>
<summary>解答</summary>

答え：**イ. スタック**

スタックはLIFO（Last In, First Out）、キューはFIFO（First In, First Out）です。

### 問題5：二分木の基本
深さ3の完全二分木のノード数と葉の数を求めてください。

<details>
<summary>ヒント</summary>
深さ d の完全二分木のノード数は 2^(d+1) - 1、葉の数は 2^d です。
</details>

<details>
<summary>解答</summary>

```
ノード数 = 2^(3+1) - 1 = 2⁴ - 1 = 16 - 1 = 15
葉の数   = 2³ = 8
```
答え：ノード数 **15**、葉の数 **8**
</details>

## 応用問題

### 問題6：逆ポーランド記法
後置記法（逆ポーランド記法）「4 5 + 3 2 - ×」の計算結果を、スタックを使って求めてください。

<details>
<summary>ヒント</summary>
数値はスタックに push し、演算子がきたら2つ pop して演算し、結果を push します。
</details>

<details>
<summary>解答</summary>

```
4 → push(4)        スタック: [4]
5 → push(5)        スタック: [4, 5]
+ → pop 5,4 → 4+5=9 → push(9)  スタック: [9]
3 → push(3)        スタック: [9, 3]
2 → push(2)        スタック: [9, 3, 2]
- → pop 2,3 → 3-2=1 → push(1)  スタック: [9, 1]
× → pop 1,9 → 9×1=9 → push(9)  スタック: [9]
```
答え：**9**

元の式（中置記法）：(4 + 5) × (3 - 2) = 9 × 1 = 9
</details>

### 問題7：2次元配列のアドレス計算
2次元配列 A[5][4]（5行4列）で、先頭アドレスが 1000、要素サイズが 2バイト、行優先で格納されているとき、A[3][2] のアドレスを求めてください。（添字は0始まり）

<details>
<summary>ヒント</summary>
行優先：A[i][j] のアドレス = 先頭アドレス + (i × 列数 + j) × 要素サイズ
</details>

<details>
<summary>解答</summary>

```
A[3][2] のアドレス = 1000 + (3 × 4 + 2) × 2
                   = 1000 + (12 + 2) × 2
                   = 1000 + 14 × 2
                   = 1000 + 28
                   = 1028
```
答え：**1028**
</details>

### 問題8：ハッシュ法
ハッシュ関数 h(key) = key mod 7 を使って、キー {15, 22, 8, 36, 29} をハッシュテーブル（サイズ7）に格納してください。衝突が起きた場合は線形探索法（オープンアドレス法）で次の空き位置に格納します。

<details>
<summary>解答</summary>

```
h(15) = 15 mod 7 = 1 → テーブル[1] = 15
h(22) = 22 mod 7 = 1 → 衝突！ → テーブル[2] = 22（次の空き位置）
h(8)  = 8 mod 7  = 1 → 衝突！ → [1],[2]使用中 → テーブル[3] = 8
h(36) = 36 mod 7 = 1 → 衝突！ → [1],[2],[3]使用中 → テーブル[4] = 36
h(29) = 29 mod 7 = 1 → 衝突！ → [1]〜[4]使用中 → テーブル[5] = 29

最終的なハッシュテーブル：
[0]: 空
[1]: 15
[2]: 22
[3]: 8
[4]: 36
[5]: 29
[6]: 空
```

この例ではすべてのキーが h(key)=1 となり（シノニム）、線形探索法で順に格納されました。
</details>

### 問題9：木の走査
以下の二分探索木を中順（inorder）で走査した結果を答えてください。

```
        10
       /  \
      5    15
     / \     \
    3   7    20
```

<details>
<summary>ヒント</summary>
中順走査の順序：左 → 根 → 右
</details>

<details>
<summary>解答</summary>

中順走査（左 → 根 → 右）：

```
1. 左部分木(5)の左部分木 → 3
2. 5
3. 左部分木(5)の右部分木 → 7
4. 10（根）
5. 右部分木(15) → 15の左は空なので15
6. 右部分木(15)の右部分木 → 20
```

答え：**3, 5, 7, 10, 15, 20**

二分探索木の中順走査は昇順（ソート順）になります。
</details>

### 問題10：四択問題
ハッシュ法における衝突（コリジョン）の解決法のうち、同じハッシュ値を持つデータをリンクリストでつなぐ方法を何というか。

ア. 線形探索法　　イ. 二分探索法　　ウ. チェイン法　　エ. オープンアドレス法

<details>
<summary>解答</summary>

答え：**ウ. チェイン法（連鎖法）**

チェイン法は同じハッシュ値のデータをリストでつなぎます。オープンアドレス法（線形探索法はその一種）は空きスロットを探して格納します。
</details>

## チャレンジ問題

### 問題11：木の走査（総合）
以下の二分木を前順（preorder）、中順（inorder）、後順（postorder）でそれぞれ走査した結果を答えてください。

```
        A
       / \
      B   C
     /   / \
    D   E   F
```

<details>
<summary>解答</summary>

**前順（根→左→右）：** A, B, D, C, E, F

```
A → B → D → C → E → F
```

**中順（左→根→右）：** D, B, A, E, C, F

```
D → B → A → E → C → F
```

**後順（左→右→根）：** D, B, E, F, C, A

```
D → B → E → F → C → A
```
</details>

### 問題12：計算量の比較
以下の操作について、配列とリスト（単方向リスト）の計算量をそれぞれ答えてください。

(1) i番目の要素へのアクセス
(2) 先頭への要素の挿入
(3) 要素の探索（未ソート）

<details>
<summary>解答</summary>

| 操作 | 配列 | リスト |
|------|------|--------|
| (1) i番目の要素へのアクセス | **O(1)** | **O(n)** |
| (2) 先頭への要素の挿入 | **O(n)** | **O(1)** |
| (3) 要素の探索（未ソート） | **O(n)** | **O(n)** |

配列はインデックスで直接アクセスできるのでO(1)ですが、先頭に挿入すると全要素をずらす必要があります。リストは先頭の挿入がO(1)ですが、i番目へのアクセスは先頭から辿る必要があります。
</details>

### 問題13：ヒープの構築
最大ヒープにデータ {4, 10, 3, 5, 1, 8} を順に挿入したとき、最終的なヒープの形を示してください。

<details>
<summary>ヒント</summary>
要素を末尾に追加し、親より大きければ交換する（上方修復）操作を繰り返します。
</details>

<details>
<summary>解答</summary>

```
挿入4:    4

挿入10:    4       → 10 > 4 なので交換 →   10
          /                                /
         10                               4

挿入3:    10
         / \
        4   3

挿入5:    10       → 5 > 4 なので交換
         / \
        4   3      →   10
       /               / \
      5               5   3
                     /
                    4

挿入1:    10
         / \
        5   3
       / \
      4   1

挿入8:    10       → 8 > 3 なので交換
         / \
        5   3      →   10
       / \ /           / \
      4  1 8          5   8
                     / \ /
                    4  1 3
```

最終的な最大ヒープ：
```
        10
       /  \
      5    8
     / \  /
    4   1 3
```

配列表現：[10, 5, 8, 4, 1, 3]
</details>
