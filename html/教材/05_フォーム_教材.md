# 第5章：フォーム

## この章のゴール
- HTMLフォームの基本構造（form要素）を理解し、実際にデータ送信の仕組みを把握する
- 主要な input type（text, email, password, number, date 等）を使い分けられるようになる
- textarea, select, button などのフォーム要素を適切に配置できる
- バリデーション属性（required, pattern, min/max 等）でクライアント側の入力チェックができる
- fieldset / legend でフォームをグループ化し、アクセシビリティに配慮したフォームを作れる

---

## 5.1 form要素：フォームの基本

フォームは、ユーザーからの入力をサーバーに送信するための仕組みです。`<form>` 要素がフォーム全体を囲むコンテナになります。

### 基本構文
```html
<form action="/submit" method="post">
  <!-- フォーム要素をここに配置 -->
  <label for="name">名前：</label>
  <input type="text" id="name" name="name">
  <button type="submit">送信</button>
</form>
```

### form要素の主な属性
| 属性 | 説明 | 例 |
|---|---|---|
| `action` | 送信先のURL | `action="/api/submit"` |
| `method` | HTTP メソッド | `method="post"` |
| `enctype` | エンコーディング方式 | `enctype="multipart/form-data"` |
| `autocomplete` | オートコンプリート | `autocomplete="off"` |
| `novalidate` | バリデーション無効化 | `novalidate` |

### method属性の値
| 値 | 特徴 | 用途 |
|---|---|---|
| `get` | URLのクエリパラメータとして送信。データがURLに表示される | 検索フォーム |
| `post` | リクエストボディとして送信。データがURLに表示されない | ログイン、登録フォーム |

```html
<!-- GET：検索フォーム -->
<form action="/search" method="get">
  <input type="text" name="q" placeholder="検索キーワード">
  <button type="submit">検索</button>
</form>
<!-- 送信すると /search?q=キーワード のURLに遷移 -->

<!-- POST：ログインフォーム -->
<form action="/login" method="post">
  <input type="email" name="email">
  <input type="password" name="password">
  <button type="submit">ログイン</button>
</form>
```

### enctype属性の値
| 値 | 用途 |
|---|---|
| `application/x-www-form-urlencoded` | デフォルト。通常のテキストデータ |
| `multipart/form-data` | ファイルアップロードを含む場合（**必須**） |
| `text/plain` | デバッグ用（ほとんど使わない） |

```html
<!-- ファイルアップロードがある場合 -->
<form action="/upload" method="post" enctype="multipart/form-data">
  <input type="file" name="photo">
  <button type="submit">アップロード</button>
</form>
```

> **重要**: ファイルアップロードを行う場合、`enctype="multipart/form-data"` を必ず指定してください。指定しないとファイルが正しく送信されません。

## 5.2 label要素：入力欄のラベル

`<label>` 要素はフォーム要素に対するラベル（説明文）です。**アクセシビリティの観点で非常に重要**です。

### 関連付けの方法

#### 方法1：for属性とid属性の紐付け（推奨）
```html
<label for="email">メールアドレス：</label>
<input type="email" id="email" name="email">
```

#### 方法2：label要素で囲む
```html
<label>
  メールアドレス：
  <input type="email" name="email">
</label>
```

### labelを付けるメリット
1. **スクリーンリーダー**がラベルを読み上げ、入力欄の目的がわかる
2. **ラベルをクリック**すると、対応する入力欄にフォーカスが移る（クリック領域が広がる）
3. チェックボックスやラジオボタンでは特に操作性が向上する

```html
<!-- labelがないと、小さなチェックボックスだけをクリックしなければならない -->
<input type="checkbox" id="agree" name="agree">
<label for="agree">利用規約に同意します</label>
<!-- ↑ テキスト部分をクリックしてもチェックが入る -->
```

> **ポイント**: すべての入力欄に `<label>` を付けることは、アクセシビリティの基本中の基本です。

## 5.3 input要素：入力フィールド

`<input>` は最も多く使われるフォーム要素です。`type` 属性によって外見と動作が大きく変わります。

### テキスト系の input type

```html
<!-- テキスト入力（1行） -->
<input type="text" name="username" placeholder="ユーザー名">

<!-- パスワード入力（文字がマスクされる） -->
<input type="password" name="password" placeholder="パスワード">

<!-- メールアドレス（スマホではメール用キーボードが表示される） -->
<input type="email" name="email" placeholder="example@mail.com">

<!-- URL入力 -->
<input type="url" name="website" placeholder="https://example.com">

<!-- 電話番号（スマホではテンキーが表示される） -->
<input type="tel" name="phone" placeholder="090-1234-5678">

<!-- 検索（×ボタンでクリア可能） -->
<input type="search" name="q" placeholder="検索...">
```

### 数値・日時系の input type

```html
<!-- 数値入力（上下ボタン付き） -->
<input type="number" name="quantity" min="1" max="99" step="1" value="1">

<!-- 範囲（スライダー） -->
<input type="range" name="volume" min="0" max="100" value="50">

<!-- 日付 -->
<input type="date" name="birthday" value="2000-01-01">

<!-- 時刻 -->
<input type="time" name="meeting-time" value="09:00">

<!-- 日時（ローカル） -->
<input type="datetime-local" name="event-date">

<!-- 月 -->
<input type="month" name="start-month">

<!-- 週 -->
<input type="week" name="target-week">
```

### 選択系の input type

```html
<!-- チェックボックス（複数選択可） -->
<fieldset>
  <legend>好きな果物（複数選択可）</legend>
  <label><input type="checkbox" name="fruit" value="apple"> りんご</label>
  <label><input type="checkbox" name="fruit" value="orange"> みかん</label>
  <label><input type="checkbox" name="fruit" value="banana"> バナナ</label>
</fieldset>

<!-- ラジオボタン（1つだけ選択） -->
<fieldset>
  <legend>性別</legend>
  <label><input type="radio" name="gender" value="male"> 男性</label>
  <label><input type="radio" name="gender" value="female"> 女性</label>
  <label><input type="radio" name="gender" value="other"> その他</label>
</fieldset>
```

> **ポイント**: ラジオボタンは同じ `name` 属性のグループから1つだけ選択できます。`name` が異なると別グループとして扱われます。

### その他の input type

```html
<!-- 色選択（カラーピッカー） -->
<input type="color" name="fav-color" value="#ff0000">

<!-- ファイル選択 -->
<input type="file" name="document" accept=".pdf,.doc,.docx">

<!-- 複数ファイル選択 -->
<input type="file" name="photos" multiple accept="image/*">

<!-- 隠しフィールド（画面に表示されない） -->
<input type="hidden" name="csrf_token" value="abc123">
```

### input type 一覧表
| type | 説明 | スマホキーボード |
|---|---|---|
| `text` | テキスト（デフォルト） | 通常 |
| `password` | パスワード | 通常 |
| `email` | メールアドレス | @付きキーボード |
| `url` | URL | .com付きキーボード |
| `tel` | 電話番号 | テンキー |
| `search` | 検索テキスト | 通常（検索ボタン付き） |
| `number` | 数値 | テンキー |
| `range` | スライダー | ― |
| `date` | 日付 | 日付ピッカー |
| `time` | 時刻 | 時刻ピッカー |
| `datetime-local` | 日時 | 日時ピッカー |
| `month` | 年月 | 月ピッカー |
| `week` | 年と週 | 週ピッカー |
| `checkbox` | チェックボックス | ― |
| `radio` | ラジオボタン | ― |
| `color` | カラーピッカー | ― |
| `file` | ファイル選択 | ― |
| `hidden` | 隠しフィールド | ― |

## 5.4 input要素の共通属性

```html
<input type="text"
       name="username"
       id="username"
       value="初期値"
       placeholder="入力のヒント"
       required
       readonly
       disabled
       maxlength="50"
       minlength="3"
       autofocus
       autocomplete="name">
```

| 属性 | 説明 |
|---|---|
| `name` | サーバーに送信されるパラメータ名（**必須**） |
| `id` | labelとの紐付けやJavaScriptでの参照用 |
| `value` | 初期値 |
| `placeholder` | 入力欄が空の時に表示されるヒントテキスト |
| `required` | 入力必須にする |
| `readonly` | 読み取り専用（送信はされる） |
| `disabled` | 無効化（送信もされない） |
| `maxlength` | 最大文字数 |
| `minlength` | 最小文字数 |
| `autofocus` | ページ読み込み時に自動フォーカス |
| `autocomplete` | オートコンプリートのヒント |

### readonlyとdisabledの違い
| 項目 | readonly | disabled |
|---|---|---|
| 見た目 | 通常と同じ | グレーアウト |
| フォーカス | できる | できない |
| テキスト選択 | できる | できない |
| フォーム送信 | **される** | **されない** |

## 5.5 textarea要素：複数行テキスト

```html
<label for="message">メッセージ：</label>
<textarea id="message" name="message" rows="5" cols="40"
          placeholder="お問い合わせ内容をご記入ください"></textarea>
```

### textarea の属性
| 属性 | 説明 |
|---|---|
| `rows` | 表示行数（高さの目安） |
| `cols` | 表示列数（幅の目安） |
| `maxlength` | 最大文字数 |
| `minlength` | 最小文字数 |
| `wrap` | 折り返しモード（`soft` / `hard`） |
| `placeholder` | ヒントテキスト |

> **注意**: `<textarea>` は空要素ではありません。初期値はタグの間に記述します。`value` 属性はありません。

```html
<!-- 初期値あり -->
<textarea name="bio">ここに初期テキストが入ります</textarea>

<!-- 間に空白を入れない（不要な空白が初期値になる） -->
<textarea name="note"></textarea>  <!-- 正しい -->
<textarea name="note">
</textarea>  <!-- 改行が初期値になってしまう -->
```

## 5.6 select要素：ドロップダウンリスト

```html
<label for="prefecture">都道府県：</label>
<select id="prefecture" name="prefecture">
  <option value="">-- 選択してください --</option>
  <option value="tokyo">東京都</option>
  <option value="osaka">大阪府</option>
  <option value="kyoto">京都府</option>
  <option value="hokkaido">北海道</option>
</select>
```

### optgroup でグループ化
```html
<label for="car">車種：</label>
<select id="car" name="car">
  <optgroup label="日本車">
    <option value="toyota">トヨタ</option>
    <option value="honda">ホンダ</option>
    <option value="nissan">日産</option>
  </optgroup>
  <optgroup label="外国車">
    <option value="bmw">BMW</option>
    <option value="benz">メルセデス・ベンツ</option>
    <option value="audi">アウディ</option>
  </optgroup>
</select>
```

### 複数選択（multiple）
```html
<label for="skills">スキル（複数選択可、Ctrl+クリック）：</label>
<select id="skills" name="skills" multiple size="5">
  <option value="html">HTML</option>
  <option value="css">CSS</option>
  <option value="js">JavaScript</option>
  <option value="python">Python</option>
  <option value="java">Java</option>
</select>
```

### option要素の属性
| 属性 | 説明 |
|---|---|
| `value` | 送信される値（省略時はテキスト内容が送信される） |
| `selected` | 初期選択状態にする |
| `disabled` | 選択不可にする |

## 5.7 datalist要素：入力候補リスト

`<datalist>` はテキスト入力にオートコンプリート候補を表示します。ドロップダウンとテキスト入力のハイブリッドです。

```html
<label for="browser">ブラウザ：</label>
<input type="text" id="browser" name="browser" list="browsers">
<datalist id="browsers">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
  <option value="Edge">
  <option value="Opera">
</datalist>
```

- `<input>` の `list` 属性と `<datalist>` の `id` を一致させる
- ユーザーは候補から選ぶことも、自由に入力することもできる
- `<select>` とは異なり、候補にないテキストも入力可能

## 5.8 button要素

```html
<!-- 送信ボタン -->
<button type="submit">送信する</button>

<!-- リセットボタン（フォームを初期状態に戻す） -->
<button type="reset">リセット</button>

<!-- 通常のボタン（JavaScriptで動作を定義） -->
<button type="button">クリック</button>
```

### button要素の type 属性
| 値 | 動作 |
|---|---|
| `submit` | フォームを送信する（**デフォルト**） |
| `reset` | フォームを初期値に戻す |
| `button` | 何もしない（JSで制御） |

> **注意**: `<form>` 内の `<button>` はデフォルトで `type="submit"` になります。送信したくないボタンには必ず `type="button"` を指定してください。

### input type="submit" との違い
```html
<!-- input版：テキストしか入れられない -->
<input type="submit" value="送信">

<!-- button版：HTML（アイコン等）を入れられる -->
<button type="submit">
  <img src="icon-send.png" alt="" width="16" height="16">
  送信する
</button>
```

`<button>` の方が柔軟性が高いため、現在は `<button>` の使用が推奨されています。

## 5.9 fieldset要素とlegend要素：フォームのグループ化

`<fieldset>` は関連するフォーム要素をグループ化し、`<legend>` はそのグループのタイトルを表示します。

```html
<form action="/register" method="post">
  <fieldset>
    <legend>個人情報</legend>
    <p>
      <label for="fullname">氏名：</label>
      <input type="text" id="fullname" name="fullname" required>
    </p>
    <p>
      <label for="email">メール：</label>
      <input type="email" id="email" name="email" required>
    </p>
  </fieldset>

  <fieldset>
    <legend>アカウント情報</legend>
    <p>
      <label for="user-id">ユーザーID：</label>
      <input type="text" id="user-id" name="user-id" required>
    </p>
    <p>
      <label for="pw">パスワード：</label>
      <input type="password" id="pw" name="pw" required>
    </p>
  </fieldset>

  <button type="submit">登録</button>
</form>
```

### fieldsetのメリット
1. **視覚的なグループ化**：デフォルトで枠線が表示される
2. **アクセシビリティ**：スクリーンリーダーがグループの構造を認識できる
3. **一括無効化**：`<fieldset disabled>` で中のすべての要素を無効にできる

```html
<!-- fieldset全体を無効化 -->
<fieldset disabled>
  <legend>この項目は現在利用できません</legend>
  <input type="text" name="promo" placeholder="プロモコード">
  <button type="button">適用</button>
</fieldset>
```

## 5.10 バリデーション属性

HTMLのバリデーション（入力検証）属性を使うと、JavaScriptなしでクライアント側の入力チェックが行えます。

### 主なバリデーション属性

```html
<form action="/register" method="post">
  <!-- 必須入力 -->
  <input type="text" name="name" required>

  <!-- 最小・最大文字数 -->
  <input type="text" name="username" minlength="3" maxlength="20">

  <!-- 正規表現パターン -->
  <input type="text" name="zipcode" pattern="\d{3}-\d{4}"
         title="000-0000の形式で入力してください">

  <!-- 数値の範囲 -->
  <input type="number" name="age" min="0" max="150">

  <!-- ステップ値 -->
  <input type="number" name="quantity" step="5" min="0" max="100">

  <button type="submit">送信</button>
</form>
```

### バリデーション属性一覧
| 属性 | 対象 | 説明 |
|---|---|---|
| `required` | すべて | 入力必須 |
| `minlength` | text系 | 最小文字数 |
| `maxlength` | text系 | 最大文字数 |
| `min` | number, date等 | 最小値 |
| `max` | number, date等 | 最大値 |
| `step` | number, range | 刻み値 |
| `pattern` | text系 | 正規表現によるパターンチェック |
| `title` | pattern使用時 | エラーメッセージに表示されるヒント |

### type属性による自動バリデーション
`type="email"` や `type="url"` は、形式が正しいかを自動でチェックします。

```html
<!-- @を含まないと送信できない -->
<input type="email" name="email" required>

<!-- https:// 等のURLでないと送信できない -->
<input type="url" name="website" required>
```

### patternの活用例

```html
<!-- 電話番号（ハイフン付き） -->
<input type="tel" name="phone"
       pattern="0\d{1,4}-\d{1,4}-\d{4}"
       title="例：03-1234-5678">

<!-- 郵便番号 -->
<input type="text" name="zip"
       pattern="\d{3}-\d{4}"
       title="例：100-0001">

<!-- 英数字のみ -->
<input type="text" name="code"
       pattern="[A-Za-z0-9]+"
       title="英数字のみ入力できます">

<!-- パスワード（8文字以上、大小英字と数字を含む） -->
<input type="password" name="password"
       pattern="(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}"
       title="8文字以上で、大文字・小文字・数字を含めてください">
```

> **重要**: クライアント側のバリデーションだけでは不十分です。ユーザーはDevToolsで`required`属性を削除できるため、**サーバー側でも必ずバリデーション**を行ってください。

## 5.11 output要素：計算結果の表示

`<output>` 要素は計算結果やユーザー操作の結果を表示するための要素です。

```html
<form oninput="result.value = parseInt(a.value) + parseInt(b.value)">
  <input type="number" id="a" name="a" value="0"> +
  <input type="number" id="b" name="b" value="0"> =
  <output name="result" for="a b">0</output>
</form>
```

```html
      <select id="c-category" name="category" required>

        <option value="general">一般的なお問い合わせ</option>
        <option value="support">技術サポート</option>
        <option value="billing">お支払いについて</option>
        <option value="other">その他</option>
      </select>

      <label for="c-message">メッセージ（必須）：</label>
      <textarea id="c-message" name="message" rows="6" required
                minlength="10" maxlength="1000"
                placeholder="お問い合わせ内容をご記入ください（10文字以上）"></textarea>
    </fieldset>

    <fieldset>
      <legend>確認</legend>
      <label>
        <input type="checkbox" name="agree" required>
        プライバシーポリシーに同意します（必須）
      </label>
    </fieldset>

    <button type="submit">送信する</button>
    <button type="reset">リセット</button>
  </form>
</body>
</html>
```

## 5.13 よくある間違い

### 1. name属性の付け忘れ
```html
<!-- 悪い例：nameがないのでサーバーにデータが送られない -->
<input type="text" id="username">

<!-- 良い例 -->
<input type="text" id="username" name="username">
```

### 2. labelの紐付け忘れ
```html
<!-- 悪い例：forとidが一致していない -->
<label for="user">名前：</label>
<input type="text" id="username" name="username">

<!-- 良い例 -->
<label for="username">名前：</label>
<input type="text" id="username" name="username">
```

### 3. ラジオボタンのname属性が異なる
```html
<!-- 悪い例：nameが異なるので両方選択できてしまう -->
<input type="radio" name="gender1" value="male"> 男性
<input type="radio" name="gender2" value="female"> 女性

<!-- 良い例：同じnameで排他制御される -->
<input type="radio" name="gender" value="male"> 男性
<input type="radio" name="gender" value="female"> 女性
```

### 4. button の type 指定忘れ
```html
<!-- 悪い例：form内でtype未指定 → 送信されてしまう -->
<button onclick="doSomething()">処理実行</button>

<!-- 良い例 -->
<button type="button" onclick="doSomething()">処理実行</button>
```

### 5. ファイルアップロードのenctype忘れ
```html
<!-- 悪い例：enctypeがないのでファイルが送信されない -->
<form method="post">
  <input type="file" name="photo">
</form>

<!-- 良い例 -->
<form method="post" enctype="multipart/form-data">
  <input type="file" name="photo">
</form>
```

## 5.14 アクセシビリティのベストプラクティス

### チェックリスト
1. **すべての入力欄に `<label>` を付ける**
2. **`<fieldset>` と `<legend>` でグループ化する**（特にラジオボタン・チェックボックス）
3. **`required` 属性のある項目に「必須」と表記する**
4. **エラーメッセージを明確に表示する**
5. **`autocomplete` 属性を適切に設定する**（ブラウザの自動入力を支援）
6. **キーボードだけで操作できる**ことを確認する（Tabキーで移動可能）
7. **`placeholder` をラベルの代わりにしない**（入力を始めると見えなくなる）

### autocomplete属性の主な値
| 値 | 用途 |
|---|---|
| `name` | 氏名 |
| `email` | メールアドレス |
| `tel` | 電話番号 |
| `address-line1` | 住所1行目 |
| `postal-code` | 郵便番号 |
| `cc-number` | クレジットカード番号 |
| `new-password` | 新しいパスワード |
| `current-password` | 現在のパスワード |

```html
<input type="text" name="name" autocomplete="name">
<input type="email" name="email" autocomplete="email">
<input type="tel" name="phone" autocomplete="tel">
```

## ポイントまとめ

| 概念 | 説明 |
|---|---|
| `<form>` | フォーム全体のコンテナ。action と method を指定 |
| `<input>` | 各種入力フィールド。type属性で種類が変わる |
| `<label>` | 入力欄の説明。for属性でinputのidと紐付け |
| `<textarea>` | 複数行テキスト入力 |
| `<select>` / `<option>` | ドロップダウンリスト |
| `<datalist>` | 入力候補の提示 |
| `<button>` | ボタン。type を明示することが重要 |
| `<fieldset>` / `<legend>` | フォーム要素のグループ化とタイトル |
| `required` | 入力必須のバリデーション |
| `pattern` | 正規表現によるバリデーション |
| `min` / `max` | 数値・日付の範囲指定 |
| `enctype` | ファイル送信時は `multipart/form-data` が必須 |
