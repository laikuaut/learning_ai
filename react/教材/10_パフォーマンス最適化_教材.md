## 1. Reactの再レンダリングを理解する

### 再レンダリングが起きるタイミング

| トリガー | 説明 |
|---------|------|
| stateの更新 | `setState` が呼ばれた時 |
| propsの変更 | 親から渡されるpropsが変わった時 |
| 親の再レンダリング | 親コンポーネントが再レンダリングされた時 |
| Contextの値の変更 | useContextで参照している値が変わった時 |

### 問題：不要な再レンダリング

```jsx
function App() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>カウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
      {/* countが変わるたびにExpensiveListも再レンダリングされる */}
      <ExpensiveList />
    </div>
  );
}

function ExpensiveList() {
  console.log('ExpensiveList がレンダリングされた');  // 毎回呼ばれる
  const items = Array.from({ length: 1000 }, (_, i) => `アイテム ${i}`);
  return (
    <ul>
      {items.map(item => <li key={item}>{item}</li>)}
    </ul>
  );
}
```

## 2. React.memo：コンポーネントのメモ化

`React.memo`でラップしたコンポーネントは、propsが変わらない限り再レンダリングをスキップします。

```jsx
import { memo, useState } from 'react';

// React.memoでラップ
const ExpensiveList = memo(function ExpensiveList({ items }) {
  console.log('ExpensiveList がレンダリングされた');

  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
});

function App() {
  const [count, setCount] = useState(0);
  const [items] = useState([
    { id: 1, name: 'React' },
    { id: 2, name: 'Vue' },
    { id: 3, name: 'Angular' },
  ]);

  return (
    <div>
      <p>カウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
      {/* itemsが変わっていないので再レンダリングされない */}
      <ExpensiveList items={items} />
    </div>
  );
}
```

### カスタム比較関数

```jsx
const MemoizedComponent = memo(MyComponent, (prevProps, nextProps) => {
  // trueを返すと再レンダリングをスキップ
  // falseを返すと再レンダリングする
  return prevProps.id === nextProps.id;
});
```

### React.memoの注意点

| 注意 | 説明 |
|------|------|
| propsにオブジェクトを渡す場合 | 毎回新しい参照が作られるとmemoが効かない |
| propsに関数を渡す場合 | useCallbackと組み合わせる必要がある |
| 全てにmemoを付けるべきではない | memoのコスト（比較処理）もある |

## 3. useMemo：計算結果のメモ化

重い計算の結果をキャッシュし、依存する値が変わらない限り再計算をスキップします。

```jsx
import { useMemo, useState } from 'react';

function FilteredList() {
  const [query, setQuery] = useState('');
  const [count, setCount] = useState(0);

  // 大量のデータ
  const allItems = Array.from({ length: 10000 }, (_, i) => ({
    id: i,
    name: `アイテム${i}`,
    category: i % 3 === 0 ? 'A' : i % 3 === 1 ? 'B' : 'C',
  }));

  // useMemoでフィルタ結果をメモ化
  // queryが変わった時だけ再計算される
  const filteredItems = useMemo(() => {
    console.log('フィルタリング実行');
    return allItems.filter(item =>
      item.name.includes(query)
    );
  }, [query]); // queryが変わった時だけ再計算

  return (
    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="検索..."
      />

      {/* countを変えてもフィルタリングは再実行されない */}
      <button onClick={() => setCount(count + 1)}>
        カウント: {count}
      </button>

      <p>{filteredItems.length} 件</p>
      <ul>
        {filteredItems.slice(0, 20).map(item => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
}
```

### useMemoの使いどころ

```jsx
// OK：重い計算
const sorted = useMemo(() => {
  return [...items].sort((a, b) => a.price - b.price);
}, [items]);

// OK：オブジェクト/配列の参照を安定させる（React.memoと併用）
const style = useMemo(() => ({
  backgroundColor: theme === 'dark' ? '#333' : '#fff',
  color: theme === 'dark' ? '#fff' : '#333',
}), [theme]);

// 不要：単純な計算にはuseMemoは不要
// const doubled = useMemo(() => count * 2, [count]);
// ↓ これで十分
const doubled = count * 2;
```

## 4. useCallback：関数のメモ化

`useCallback`は関数をメモ化します。React.memoと組み合わせて、子コンポーネントの不要な再レンダリングを防ぎます。

```jsx
import { memo, useCallback, useState } from 'react';

// React.memoでメモ化された子コンポーネント
const TodoItem = memo(function TodoItem({ todo, onToggle, onDelete }) {
  console.log(`TodoItem ${todo.id} がレンダリングされた`);

  return (
    <div style={{ display: 'flex', gap: '8px', alignItems: 'center' }}>
      <input
        type="checkbox"
        checked={todo.completed}
        onChange={() => onToggle(todo.id)}
      />
      <span style={{ textDecoration: todo.completed ? 'line-through' : 'none' }}>
        {todo.text}
      </span>
      <button onClick={() => onDelete(todo.id)}>削除</button>
    </div>
  );
});

function TodoApp() {
  const [todos, setTodos] = useState([
    { id: 1, text: '買い物', completed: false },
    { id: 2, text: '洗濯', completed: false },
    { id: 3, text: '掃除', completed: false },
  ]);
  const [input, setInput] = useState('');

  // useCallbackで関数をメモ化
  const handleToggle = useCallback((id) => {
    setTodos(prev => prev.map(todo =>
      todo.id === id ? { ...todo, completed: !todo.completed } : todo
    ));
  }, []); // 依存なし：setTodosの関数形式を使っているため

  const handleDelete = useCallback((id) => {
    setTodos(prev => prev.filter(todo => todo.id !== id));
  }, []);

  const handleAdd = () => {
    if (input.trim() === '') return;
    setTodos(prev => [...prev, { id: Date.now(), text: input, completed: false }]);
    setInput('');
  };

  return (
    <div>
      <div style={{ display: 'flex', gap: '8px' }}>
        <input value={input} onChange={(e) => setInput(e.target.value)} />
        <button onClick={handleAdd}>追加</button>
      </div>

      {todos.map(todo => (
        <TodoItem
          key={todo.id}
          todo={todo}
          onToggle={handleToggle}
          onDelete={handleDelete}
        />
      ))}
    </div>
  );
}
```

### なぜuseCallbackが必要？

```jsx
// useCallbackなしの場合
function Parent() {
  const [count, setCount] = useState(0);

  // 毎回のレンダリングで新しい関数が作られる
  const handleClick = () => { console.log('click'); };

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>+1</button>
      {/* handleClickが毎回新しい参照なので、
          React.memoが効かず毎回再レンダリングされる */}
      <MemoizedChild onClick={handleClick} />
    </div>
  );
}
```

## 5. useRef：再レンダリングなしに値を保持

`useRef`は再レンダリングを発生させずに値を保持できます。DOM要素への参照にも使えます。

### DOM要素への参照

```jsx
import { useRef } from 'react';

function FocusInput() {
  const inputRef = useRef(null);

  const handleClick = () => {
    inputRef.current.focus();
  };

  return (
    <div>
      <input ref={inputRef} placeholder="ここにフォーカス" />
      <button onClick={handleClick}>フォーカスを当てる</button>
    </div>
  );
}
```

### 再レンダリングなしの値の保持

```jsx
import { useRef, useState, useEffect } from 'react';

function StopWatch() {
  const [time, setTime] = useState(0);
  const [isRunning, setIsRunning] = useState(false);
  const intervalRef = useRef(null);  // タイマーIDを保持（再レンダリング不要）
  const startTimeRef = useRef(0);

  const start = () => {
    if (isRunning) return;
    setIsRunning(true);
    startTimeRef.current = Date.now() - time;
    intervalRef.current = setInterval(() => {
      setTime(Date.now() - startTimeRef.current);
    }, 10);
  };

  const stop = () => {
    setIsRunning(false);
    clearInterval(intervalRef.current);
  };

  const reset = () => {
    stop();
    setTime(0);
  };

  useEffect(() => {
    return () => clearInterval(intervalRef.current);
  }, []);

  const formatTime = (ms) => {
    const minutes = Math.floor(ms / 60000);
    const seconds = Math.floor((ms % 60000) / 1000);
    const centiseconds = Math.floor((ms % 1000) / 10);
    return `${String(minutes).padStart(2, '0')}:${String(seconds).padStart(2, '0')}.${String(centiseconds).padStart(2, '0')}`;
  };

  return (
    <div style={{ textAlign: 'center' }}>
      <p style={{ fontSize: '48px', fontFamily: 'monospace' }}>{formatTime(time)}</p>
      <button onClick={start} disabled={isRunning}>開始</button>
      <button onClick={stop} disabled={!isRunning}>停止</button>
      <button onClick={reset}>リセット</button>
    </div>
  );
}
```

### useRef vs useState

| | useRef | useState |
|---|--------|---------|
| 値の更新 | `.current` を直接変更 | set関数を使う |
| 再レンダリング | 発生しない | 発生する |
| 用途 | DOM参照、タイマーID、前回の値 | UIに表示する値 |

## 6. React.lazy と Suspense：遅延読み込み

大きなコンポーネントを必要な時にだけ読み込むことで、初期ロード時間を短縮できます。

```jsx
import { lazy, Suspense, useState } from 'react';

// 遅延読み込み：このコンポーネントは必要になるまで読み込まれない
const HeavyChart = lazy(() => import('./HeavyChart'));
const AdminPanel = lazy(() => import('./AdminPanel'));

function App() {
  const [showChart, setShowChart] = useState(false);
  const [showAdmin, setShowAdmin] = useState(false);

  return (
    <div>
      <h1>ダッシュボード</h1>

      <button onClick={() => setShowChart(true)}>グラフを表示</button>
      <button onClick={() => setShowAdmin(true)}>管理画面</button>

      {showChart && (
        <Suspense fallback={<p>グラフを読み込み中...</p>}>
          <HeavyChart />
        </Suspense>
      )}

      {showAdmin && (
        <Suspense fallback={<p>管理画面を読み込み中...</p>}>
          <AdminPanel />
        </Suspense>
      )}
    </div>
  );
}
```

### React Routerとの組み合わせ

```jsx
import { lazy, Suspense } from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';

// 各ページを遅延読み込み
const Home = lazy(() => import('./pages/Home'));
const Products = lazy(() => import('./pages/Products'));
const About = lazy(() => import('./pages/About'));

function App() {
  return (
    <BrowserRouter>
      <Suspense fallback={<div style={{ padding: '20px' }}>ページを読み込み中...</div>}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/products" element={<Products />} />
          <Route path="/about" element={<About />} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}
```

## 7. key propによる最適化

### keyでコンポーネントをリセットする

```jsx
function App() {
  const [userId, setUserId] = useState(1);

  return (
    <div>
      <button onClick={() => setUserId(1)}>ユーザー1</button>
      <button onClick={() => setUserId(2)}>ユーザー2</button>

      {/* keyが変わるとProfileが完全にリマウント（リセット）される */}
      <Profile key={userId} userId={userId} />
    </div>
  );
}

function Profile({ userId }) {
  const [edit, setEdit] = useState(false);
  const [name, setName] = useState('');

  // keyが変わるたびにstateがリセットされるので、
  // 前のユーザーの編集状態が残らない
  return (
    <div>
      <p>ユーザーID: {userId}</p>
      <input value={name} onChange={(e) => setName(e.target.value)} />
    </div>
  );
}
```

## 8. パフォーマンス最適化のガイドライン

### やるべきこと

| テクニック | 使いどころ |
|-----------|-----------|
| React.memo | propsが変わらないのに再レンダリングされる重いコンポーネント |
| useMemo | 重い計算（ソート、フィルタ、集計）の結果をキャッシュ |
| useCallback | React.memoされた子に渡す関数を安定させる |
| React.lazy | 初期表示に不要な大きなコンポーネントの遅延読み込み |
| key | リストの効率的な更新、コンポーネントのリセット |

### やってはいけないこと

| アンチパターン | 理由 |
|--------------|------|
| 全コンポーネントにReact.memo | 比較コストが増え、逆に遅くなることもある |
| 全ての値にuseMemo | メモリを無駄に消費し、コードも複雑になる |
| 計測なしの最適化 | 実際にボトルネックでない場所を最適化しても効果がない |

### 最適化の手順

1. **まずは計測する**：React DevTools の Profiler を使う
2. **ボトルネックを特定する**：どのコンポーネントが遅いか
3. **原因を分析する**：不要な再レンダリング？重い計算？
4. **適切なテクニックを適用する**
5. **再度計測して効果を確認する**

## まとめ

| 項目 | ポイント |
|------|---------|
| React.memo | propsが変わらなければ再レンダリングをスキップ |
| useMemo | 計算結果をメモ化。依存値が変わるまでキャッシュ |
| useCallback | 関数の参照をメモ化。React.memoと併用 |
| useRef | 再レンダリングなしに値を保持。DOM参照にも使用 |
| React.lazy | コンポーネントの遅延読み込み |
| Suspense | 遅延読み込み中のフォールバック表示 |
| key prop | リスト更新の効率化、コンポーネントのリセット |
| 原則 | 計測してからのみ最適化する |
