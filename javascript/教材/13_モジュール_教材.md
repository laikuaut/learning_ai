# 第13章: モジュール

## この章のゴール

- モジュールの概念と必要性を理解する
- import/export の各構文を使い分けられるようになる
- デフォルトエクスポートと名前付きエクスポートの違いを把握する
- モジュール設計のベストプラクティスを知る
- CommonJS（Node.js）との違いを理解する

---

## 13.1 モジュールとは

モジュールは、コードを**ファイル単位で分割・管理**する仕組みです。

```
モジュールがない場合（グローバル汚染）:

  <script src="utils.js"></script>     → グローバルに変数・関数が登録
  <script src="api.js"></script>       → 名前の衝突が起きる可能性
  <script src="app.js"></script>       → 読み込み順序に依存

モジュールがある場合:

  math.js    → export { add, multiply }
  api.js     → export { fetchUsers }
  app.js     → import { add } from './math.js'
               import { fetchUsers } from './api.js'

  ・各ファイルが独立したスコープを持つ
  ・必要なものだけを明示的にインポート
  ・名前の衝突を防止
  ・依存関係が明確
```

## 13.2 名前付きエクスポート（Named Export）

### エクスポート側

```javascript
// math.js

// 方法1: 宣言時にexport
export const PI = 3.14159;

export function add(a, b) {
  return a + b;
}

export function multiply(a, b) {
  return a * b;
}

export class Calculator {
  constructor() {
    this.result = 0;
  }

  add(n) {
    this.result += n;
    return this;
  }

  getResult() {
    return this.result;
  }
}

// 方法2: まとめてexport（ファイル末尾で）
// const PI = 3.14159;
// function add(a, b) { return a + b; }
// function multiply(a, b) { return a * b; }
// export { PI, add, multiply };
```

### インポート側

```javascript
// app.js

// 必要なものだけインポート
import { add, multiply } from './math.js';
console.log(add(3, 5));      // 8
console.log(multiply(3, 5)); // 15

// 全てインポート（名前空間として）
import * as math from './math.js';
console.log(math.add(3, 5));      // 8
console.log(math.PI);             // 3.14159

// 別名でインポート
import { add as sum, multiply as mul } from './math.js';
console.log(sum(3, 5));  // 8
console.log(mul(3, 5));  // 15
```

## 13.3 デフォルトエクスポート（Default Export）

1つのモジュールに1つだけ設定できる「主要なエクスポート」です。

```javascript
// User.js — クラスをデフォルトエクスポート
export default class User {
  constructor(name, email) {
    this.name = name;
    this.email = email;
  }

  greet() {
    return `こんにちは、${this.name}です`;
  }
}

// api.js — 関数をデフォルトエクスポート
export default async function fetchData(url) {
  const response = await fetch(url);
  return response.json();
}
```

```javascript
// インポート側（名前は自由に付けられる）
import User from './User.js';
import fetchData from './api.js';

const user = new User("太郎", "taro@example.com");
console.log(user.greet());
```

### デフォルト + 名前付きの併用

```javascript
// logger.js
export default function log(message) {
  console.log(`[LOG] ${message}`);
}

export function warn(message) {
  console.warn(`[WARN] ${message}`);
}

export function error(message) {
  console.error(`[ERROR] ${message}`);
}

export const LOG_LEVELS = { DEBUG: 0, INFO: 1, WARN: 2, ERROR: 3 };
```

```javascript
// インポート
import log, { warn, error, LOG_LEVELS } from './logger.js';

log("アプリ開始");
warn("メモリ使用量が高い");
error("接続に失敗");
```

## 13.4 HTMLでのモジュール使用

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>モジュールの例</title>
</head>
<body>
  <!-- type="module" を指定する -->
  <script type="module">
    import { add } from './math.js';
    console.log(add(3, 5)); // 8
  </script>

  <!-- 外部ファイルの場合も同様 -->
  <script type="module" src="./app.js"></script>
</body>
</html>
```

```
type="module" の特徴:

  ・自動的にstrictモードになる
  ・各モジュールが独自のスコープを持つ（グローバル汚染なし）
  ・同じモジュールを複数回importしても1回だけ実行される
  ・defer属性と同じタイミングで実行される（DOM構築後）
  ・CORSが適用される（file://では動かない→ローカルサーバーが必要）
```

## 13.5 動的インポート

```javascript
// 通常のimportはファイル先頭で静的に記述する必要がある
// 動的importは条件分岐やイベントハンドラ内で使える

// 条件による動的読み込み
async function loadModule(type) {
  if (type === "chart") {
    const { renderChart } = await import('./chart.js');
    renderChart(data);
  } else if (type === "table") {
    const { renderTable } = await import('./table.js');
    renderTable(data);
  }
}

// ボタンクリック時に読み込み（コード分割/遅延読み込み）
document.querySelector("#loadBtn").addEventListener("click", async () => {
  const module = await import('./heavyModule.js');
  module.default(); // デフォルトエクスポートの実行
});

// エラーハンドリング
try {
  const module = await import('./someModule.js');
  module.doSomething();
} catch (error) {
  console.error("モジュールの読み込みに失敗:", error);
}
```

## 13.6 再エクスポート

```javascript
// index.js — 複数モジュールをまとめて再エクスポート
export { add, multiply } from './math.js';
export { default as User } from './User.js';
export { fetchData, postData } from './api.js';
export * from './utils.js'; // 全てを再エクスポート
```

```javascript
// 使う側はindex.jsからまとめてインポートできる
import { add, User, fetchData } from './index.js';
```

```
ディレクトリ構成例:

  lib/
    ├── index.js        ← 再エクスポート（エントリーポイント）
    ├── math.js
    ├── User.js
    ├── api.js
    └── utils.js

  app.js
    import { add, User } from './lib/index.js';
    // または
    import { add, User } from './lib/';  （バンドラー使用時）
```

## 13.7 CommonJS（Node.js）

Node.jsでは歴史的にCommonJSモジュールシステムが使われています。

```javascript
// math.js（CommonJS）
function add(a, b) {
  return a + b;
}

function multiply(a, b) {
  return a * b;
}

module.exports = { add, multiply };
// または
// exports.add = add;
// exports.multiply = multiply;
```

```javascript
// app.js（CommonJS）
const { add, multiply } = require('./math');
// または
const math = require('./math');
console.log(math.add(3, 5));
```

```
ESModules vs CommonJS:

  特徴          ESModules          CommonJS
  ──────────────────────────────────────────
  構文          import/export      require/module.exports
  読み込み      静的（コンパイル時） 動的（実行時）
  トップレベル  import文のみ       どこでもrequire可能
  非同期読み込み import()          なし
  環境          ブラウザ/Node.js   Node.js
  拡張子        .mjs / .js        .cjs / .js
  strictモード  自動               手動で'use strict'
```

### Node.jsでESModulesを使う方法

```json
// package.json に "type": "module" を追加
{
  "name": "my-project",
  "type": "module"
}
```

```javascript
// または .mjs 拡張子を使う
// utils.mjs
export function greet(name) {
  return `Hello, ${name}!`;
}

// app.mjs
import { greet } from './utils.mjs';
```

### ユーティリティモジュール

```javascript
// utils/string.js
export function capitalize(str) {
  return str.charAt(0).toUpperCase() + str.slice(1);
}

export function truncate(str, maxLength, suffix = "...") {
  if (str.length <= maxLength) return str;
  return str.slice(0, maxLength - suffix.length) + suffix;
}

export function slugify(str) {
  return str
    .toLowerCase()
    .replace(/[^\w\s-]/g, "")
    .replace(/\s+/g, "-");
}

// utils/array.js
export function chunk(arr, size) {
  const chunks = [];
  for (let i = 0; i < arr.length; i += size) {
    chunks.push(arr.slice(i, i + size));
  }
  return chunks;
}

export function shuffle(arr) {
  const result = [...arr];
  for (let i = result.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [result[i], result[j]] = [result[j], result[i]];
  }
  return result;
}

// utils/index.js — まとめてエクスポート
export * from './string.js';
export * from './array.js';
```

### ファクトリーパターン

```javascript
// database.js
class Database {
  constructor(config) {
    this.config = config;

  return instance;
```

## よくある間違い

### 1. import/export の構文の混同

```javascript
// NG: 名前付きをデフォルトとしてインポート
// import add from './math.js'; // undefinedになる

// OK: 名前付きエクスポートは {} で受け取る
import { add } from './math.js';

// デフォルトエクスポートは {} なしで受け取る
import Calculator from './Calculator.js';
```

### 2. 循環依存

```javascript
// a.js
import { b } from './b.js';
export const a = "A" + b; // bがまだundefinedの可能性

// b.js
import { a } from './a.js';
export const b = "B" + a; // aがまだundefinedの可能性

// → 循環依存は避けるべき。共通部分を別モジュールに切り出す
```

### 3. ファイルパスの指定

```javascript
// NG: 拡張子やパスの省略（ブラウザのESModulesでは動かない）
// import { add } from 'math';       // パッケージ名と勘違い
// import { add } from './math';     // 拡張子なし

// OK: 相対パスと拡張子を明記
import { add } from './math.js';
import { User } from '../models/User.js';

// ※ バンドラー（Webpack, Vite等）を使う場合は省略可能な場合もある
```

### 4. type="module"の忘れ

```html
<!-- NG: importが使えない -->
<script src="app.js"></script>

<!-- OK -->
<script type="module" src="app.js"></script>
```

## ポイントまとめ

| エクスポート | インポート | 用途 |
|-------------|-----------|------|
| `export const x` | `import { x } from` | 名前付き（複数可） |
| `export default x` | `import x from` | デフォルト（1つだけ） |
| `export { x, y }` | `import { x, y } from` | まとめて名前付き |
| `export * from` | — | 再エクスポート |
| — | `import * as mod from` | 名前空間インポート |
| — | `await import()` | 動的インポート |

- モジュールはコードを**ファイル単位で分割・管理**する仕組みです
- **名前付きエクスポート**（`export { }`）は複数のものを公開する場合に使います
- **デフォルトエクスポート**（`export default`）はモジュールの主要な機能を公開する場合に使います
- HTMLでは `<script type="module">` が必要です
- **動的インポート**（`import()`）でコード分割・遅延読み込みが可能です
- **バレルファイル**（index.js）でモジュールをまとめると管理しやすくなります
- Node.jsではCommonJS（`require`）とESModules（`import`）の両方が使えます
