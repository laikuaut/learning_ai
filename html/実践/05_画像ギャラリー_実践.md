# 実践課題05：画像ギャラリー ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第1章（HTMLの基本構造）、第2章（テキストとリンク）、第3章（画像とマルチメディア）、第4章（リストとテーブル）
> **課題の種類**: ミニプロジェクト
> **学習目標**: 画像（img）、キャプション（figure / figcaption）、リンク（a）を組み合わせて、ギャラリーページを構築する

---

## 完成イメージ

ブラウザで開くと以下のような画像ギャラリーが表示されます。

```
====================================
      旅の思い出フォトギャラリー
====================================

[画像1]       [画像2]       [画像3]
東京タワー    富士山        京都・金閣寺

[画像4]       [画像5]       [画像6]
沖縄の海      北海道の雪景色  大阪城

撮影情報
| 写真      | 撮影日     | 場所     | カメラ   |
|-----------|-----------|----------|---------|
| 東京タワー | 2025-01-15| 東京都   | iPhone  |
| 富士山     | 2025-02-20| 静岡県   | Canon   |
| ...       | ...       | ...      | ...     |

※ 画像をクリックすると拡大表示されます
```

---

## 課題の要件

1. 正しいHTML基本構造を持つこと
2. `<figure>` と `<figcaption>` で画像にキャプションを付けること
3. `<img>` の `alt` 属性に適切な代替テキストを設定すること
4. `<img>` の `width` と `height` 属性を設定すること（レイアウトシフト防止）
5. 画像を `<a>` タグで囲んでクリック可能にすること
6. 撮影情報を `<table>` で一覧表示すること
7. 最低6枚の画像を配置すること（プレースホルダー画像でOK）

---

## ステップガイド

<details>
<summary>ステップ1：figure と figcaption を使う</summary>

```html
<figure>
  <img src="https://via.placeholder.com/200x150" alt="東京タワー"
       width="200" height="150">
  <figcaption>東京タワー（2025年1月撮影）</figcaption>
</figure>
```

`<figure>` は「独立したコンテンツ」を表します。画像とそのキャプションを1つにまとめる役割があります。

</details>

<details>
<summary>ステップ2：画像をクリック可能にする</summary>

`<a>` タグで `<img>` を囲むと、画像がリンクになります。

```html
<figure>
  <a href="https://via.placeholder.com/800x600" target="_blank">
    <img src="https://via.placeholder.com/200x150" alt="東京タワー"
         width="200" height="150">
  </a>
  <figcaption>東京タワー</figcaption>
</figure>
```

サムネイル画像をクリックすると、大きな画像が新しいタブで開きます。

</details>

<details>
<summary>ステップ3：テーブルで撮影情報を表示する</summary>

```html
<table border="1">
  <thead>
    <tr>
      <th>写真</th>
      <th>撮影日</th>
      <th>場所</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>東京タワー</td>
      <td>2025-01-15</td>
      <td>東京都港区</td>
    </tr>
  </tbody>
</table>
```

</details>

**初心者向けとの違い:**
- CSSの `display: flex` と `flex-wrap: wrap` で画像を横並びに配置 → グリッドレイアウト
- `loading="lazy"` で画面外の画像を遅延読み込み → ページの読み込み速度が向上
- `object-fit: cover` で画像のアスペクト比を保ちながらトリミング → 統一された見た目
- ホバー時に `opacity` を変えてクリック可能であることを示す → ユーザビリティ向上
- `box-shadow` でカード風のデザイン → 現代的なWebデザイン
- テーブルの偶数行に背景色を交互に設定（ゼブラストライプ）→ 見やすさ向上
