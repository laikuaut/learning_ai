## 5.1 動画生成AIの基本

### 動画生成AIとは

動画生成AI（Video Generation AI）とは、テキストや画像などの入力から動画を自動生成する技術です。2024年以降、急速に品質が向上し、実用レベルに達しつつあります。

### 主な生成方式

動画生成AIには、いくつかの入力方式があります。

```
┌─────────────────────────────────────────────────────┐
│              動画生成AIの入力方式                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ① Text-to-Video（T2V）                             │
│     テキスト説明 ──→ 動画                            │
│     「海辺を歩く犬の映像」→ 🎬                       │
│                                                     │
│  ② Image-to-Video（I2V）                            │
│     静止画 ──→ 動画                                  │
│     写真1枚 → その写真が動き出す映像                  │
│                                                     │
│  ③ Video-to-Video（V2V）                            │
│     既存動画 ──→ スタイル変換された動画               │
│     実写動画 → アニメ風動画                          │
│                                                     │
│  ④ Image-to-Video + カメラ制御                       │
│     静止画 + カメラワーク指定 ──→ 動画               │
│     写真 +「パン・ズーム」→ シネマティック映像        │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 動画生成の基本的な仕組み

現在の主流は拡散モデル（Diffusion Model）をベースにした手法です。画像生成と同様にノイズから徐々にクリアな映像を生成しますが、時間軸方向の一貫性を保つ仕組みが追加されています。

```
ノイズ ──→ [拡散モデル + 時間的整合性モジュール] ──→ 動画フレーム群
                    ↑
              テキスト/画像の条件
```

### 動画生成で指定できる主なパラメータ

| パラメータ | 説明 | 一般的な選択肢 |
|---|---|---|
| アスペクト比（Aspect Ratio） | 動画の縦横比 | 16:9, 9:16, 1:1 |
| 解像度（Resolution） | 画質 | 720p, 1080p, 4K |
| 長さ（Duration） | 動画の秒数 | 5秒〜60秒 |
| FPS | フレームレート | 24fps, 30fps |
| カメラワーク | カメラの動き | パン、ズーム、回転 |

## 5.2 主要な動画生成サービス

### サービス比較一覧（2026年3月時点）

| サービス | 開発元 | 最大長 | 解像度 | 特徴 |
|---|---|---|---|---|
| Sora | OpenAI | 〜60秒 | 1080p | 物理法則の理解が高い、テキスト理解が正確 |
| Runway Gen-3 | Runway | 〜16秒 | 4K | プロ向け編集機能が充実、V2V対応 |
| Pika | Pika Labs | 〜10秒 | 1080p | 手軽さが魅力、エフェクト機能が豊富 |
| Kling | Kuaishou | 〜120秒 | 1080p | 長尺生成が可能、人物の動きが自然 |
| Veo 2 | Google | 〜60秒 | 4K | 映画品質、物理シミュレーションが正確 |

### 各サービスの詳細

#### Sora（OpenAI）

OpenAIが開発した動画生成AIです。テキストの意味理解に優れ、物理法則に沿った自然な映像を生成できます。

**得意分野：**
- 複雑なシーンの描写（複数の被写体が相互作用する場面）
- テキスト指示の正確な反映
- 一貫性のあるキャラクター表現

**プロンプト例：**
```
A golden retriever puppy playing in autumn leaves in a park.
The camera slowly orbits around the puppy as it jumps
and catches falling leaves. Warm afternoon sunlight creates
a bokeh effect in the background. Cinematic quality, 24fps.
```

#### Runway Gen-3（Runway）

クリエイター向けの機能が充実したサービスです。動画編集ソフトとの連携が強く、プロの映像制作ワークフローに組み込みやすいのが特徴です。

**主な機能：**
- Motion Brush：画像の特定部分だけを動かす
- カメラコントロール：詳細なカメラワーク指定
- スタイルリファレンス：参照画像のスタイルを適用
- Green Screen：背景除去・合成

#### Pika

手軽さと楽しさを重視したサービスです。特殊エフェクト（爆発、溶解、膨張など）のテンプレートが豊富で、SNS向けのコンテンツ制作に向いています。

#### Kling（Kuaishou）

中国のKuaishou社が開発したサービスで、最大120秒の長尺動画を生成できる点が大きな強みです。人物の自然な動きの表現に優れています。

#### Veo 2（Google DeepMind）

Googleが開発した動画生成AIで、映画品質の映像を生成できます。物理シミュレーションの正確さと4K対応が特徴です。

### よくある間違い

> **「動画生成AIだけで完成品が作れる」と考えてしまう**
>
> 現時点の動画生成AIは、素材作成ツールとして捉えるのが正確です。生成された動画は「ベース素材」であり、編集ソフト（Premiere Pro、DaVinci Resolveなど）での後処理が前提となります。BGMの追加、テロップ挿入、カット編集などは別途行う必要があります。

## 5.3 音声生成AIの基本

### 音声生成AIの種類

音声生成AIは大きく3つのカテゴリに分かれます。

```
┌─────────────────────────────────────────────────────┐
│              音声生成AIの3つのカテゴリ                 │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ① TTS（Text-to-Speech / テキスト音声合成）          │
│     テキスト ──→ 自然な読み上げ音声                   │
│     用途：ナレーション、読み上げ、アクセシビリティ     │
│                                                     │
│  ② Voice Cloning（音声クローン）                     │
│     サンプル音声 + テキスト ──→ その声で読み上げ      │
│     用途：一貫したブランドボイス、多言語化             │
│                                                     │
│  ③ Music Generation（音楽生成）                      │
│     テキスト説明 ──→ 楽曲                            │
│     用途：BGM制作、ジングル、プロトタイプ             │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### TTSの進化

従来のTTSは機械的な読み上げでしたが、最新のAI TTSは人間と区別がつかないレベルの自然さを実現しています。

**最新TTSの特徴：**
- 感情表現（喜び、悲しみ、興奮など）の制御が可能
- 間（ま）の取り方が自然
- 多言語対応（同じ声で複数言語を話せる）
- リアルタイム生成に対応

### 音声クローンの仕組み

音声クローン（Voice Cloning）は、短い音声サンプル（数秒〜数分）から話者の声の特徴を学習し、任意のテキストをその声で読み上げる技術です。

```
音声サンプル（10秒〜数分）
        ↓
  [声の特徴を抽出]  ← 話し方、声質、アクセント
        ↓
  音声モデル生成
        ↓
  テキスト入力 ──→ クローン音声出力
```

## 5.4 主要な音声生成サービス

### サービス比較一覧

| サービス | カテゴリ | 特徴 | 日本語対応 |
|---|---|---|---|
| ElevenLabs | TTS / 音声クローン | 業界最高品質のTTS、29言語対応 | あり |
| OpenAI TTS | TTS | ChatGPTと統合、APIで利用可能 | あり |
| Suno | 音楽生成 | テキストから楽曲生成、歌詞+歌唱 | あり |
| Udio | 音楽生成 | 高品質な音楽生成、ジャンル指定が細かい | 一部対応 |
| NotebookLM Audio Overview | TTS / 要約 | ドキュメントからポッドキャスト風音声を自動生成 | あり |

### 各サービスの詳細

#### ElevenLabs

音声合成の品質で業界をリードするサービスです。特に感情表現の自然さとVoice Cloning精度が高く評価されています。

**主な機能：**
- 高品質TTS（29言語対応）
- Instant Voice Cloning（短時間サンプルから音声クローン）
- Professional Voice Cloning（高精度版）
- Sound Effects生成
- 音声の感情・スタイル制御

**APIでの利用例（Python）：**
```python
# ElevenLabs APIを使ったTTSの例
import requests

ELEVENLABS_API_KEY = "your-api-key"
VOICE_ID = "21m00Tcm4TlvDq8ikWAM"  # デフォルト音声「Rachel」

url = f"https://api.elevenlabs.io/v1/text-to-speech/{VOICE_ID}"

headers = {
    "Accept": "audio/mpeg",
    "Content-Type": "application/json",
    "xi-api-key": ELEVENLABS_API_KEY,
}

data = {
    "text": "こんにちは。本日の会議の要点をお伝えします。",
    "model_id": "eleven_multilingual_v2",
    "voice_settings": {
        "stability": 0.5,       # 安定性（高いほど一貫した声）
        "similarity_boost": 0.8  # 類似度（高いほど元の声に近い）
    }
}

response = requests.post(url, json=data, headers=headers)

with open("output.mp3", "wb") as f:
    f.write(response.content)

print("音声ファイルを生成しました: output.mp3")
```

#### OpenAI TTS

ChatGPTと同じOpenAIが提供するTTSサービスです。APIで手軽に利用でき、6種類の組み込み音声から選択できます。

**APIでの利用例（Python）：**
```python
from openai import OpenAI

client = OpenAI()

response = client.audio.speech.create(
    model="tts-1",        # tts-1（高速）または tts-1-hd（高品質）
    voice="nova",          # alloy, echo, fable, onyx, nova, shimmer
    input="本日の売上レポートをお伝えします。"
)

response.stream_to_file("report_narration.mp3")
```

#### Suno

テキストプロンプトから楽曲（歌詞+メロディ+歌唱）を丸ごと生成できるサービスです。

**プロンプト例：**
```
明るいポップスで、企業のCM向けのジングル。
テーマは「新しい朝、新しい挑戦」。
30秒程度、キャッチーなサビ付き。
```

#### Udio

Sunoと同様の音楽生成サービスですが、音質やジャンルの再現性に定評があります。クラシック、ジャズ、エレクトロニカなど幅広いジャンルに対応しています。

#### NotebookLM Audio Overview（Google）

Google NotebookLMの機能で、アップロードしたドキュメントの内容を2人の話者によるポッドキャスト風の音声に自動変換します。

**特徴：**
- 複数のドキュメントを要約してポッドキャスト化
- 2人の話者が対話形式で内容を解説
- 日本語にも対応
- カスタム指示で焦点を絞ることが可能

**活用例：**
```
1. 社内の長文レポート（PDF）をアップロード
2. 「技術者向けに要点を解説して」とカスタム指示
3. 15分程度のポッドキャスト音声が自動生成
4. 通勤中に聞ける社内ナレッジ共有コンテンツに
```

### よくある間違い

> **「音声クローンは誰の声でも自由に使える」と考えてしまう**
>
> 他人の声を無断でクローンすることは、肖像権・パブリシティ権の侵害となる可能性があります。音声クローンを利用する際は、必ず本人の明示的な同意を得てください。多くのサービスでは利用規約で同意の取得を義務付けています。

## 5.5 実践的な活用シーン

### 活用シーン一覧

```
┌────────────────────────────────────────────────────────────┐
│          動画・音声生成AIの主な活用シーン                     │
├────────────────┬───────────────────────────────────────────┤
│ プレゼン動画    │ スライドに動画素材を追加、製品デモ映像     │
│ 研修動画        │ 社内教育用のナレーション付き解説動画       │
│ ナレーション    │ マニュアルの音声ガイド、アプリ音声案内     │
│ ポッドキャスト  │ 社内報のポッドキャスト化、市場レポート音声 │
│ SNSコンテンツ  │ ショート動画、リール素材の大量生成         │
│ プロトタイプ    │ 映像企画のラフ動画、CM候補の素案           │
│ 多言語展開     │ 同一コンテンツの多言語ナレーション         │
└────────────────┴───────────────────────────────────────────┘
```

### 活用例1：社内研修動画の制作

従来、社内研修動画の制作には撮影・編集で数日〜数週間かかっていました。生成AIを活用すると、大幅な時間短縮が可能です。

**制作フロー：**
```
① 台本作成（ChatGPT等で下書き → 人間がレビュー）
      ↓
② ナレーション生成（ElevenLabs / OpenAI TTS）
      ↓
③ スライド・図表作成（PowerPoint / Canva）
      ↓
④ 補助的な動画素材生成（Sora / Runway）
      ↓
⑤ 動画編集ソフトで統合（BGM + テロップ追加）
      ↓
⑥ レビュー・修正
```

### 活用例2：多言語ナレーション

1つの音声を録音すれば、Voice Cloningと多言語TTSを組み合わせることで、同じ声色で複数言語のナレーションを生成できます。

**従来の方法 vs AI活用：**

| 項目 | 従来の方法 | AI活用 |
|---|---|---|
| 必要な声優 | 言語ごとに1名 | 1名分の音声サンプルのみ |
| 制作期間 | 言語数 × 数日 | 数時間 |
| コスト | 数十万〜数百万円 | 数千〜数万円 |
| 声の一貫性 | 声優が異なるため低い | 同一声色で高い |
| 修正対応 | 再録音が必要 | テキスト修正で再生成 |

### 活用例3：NotebookLMで社内ナレッジをポッドキャスト化

長文の社内ドキュメントをNotebookLM Audio Overviewでポッドキャスト化し、移動中や隙間時間に「ながら聞き」できるようにする活用法です。

**手順：**
1. NotebookLMに社内レポートやマニュアルをアップロード
2. Audio Overview機能を選択
3. カスタム指示で「新入社員向けにわかりやすく」などと指定
4. 生成されたポッドキャスト音声を社内共有

## 5.6 品質向上のコツと制約

### 動画生成のプロンプト設計

動画生成AIで高品質な結果を得るためのプロンプトの書き方を紹介します。

**効果的なプロンプトの構成要素：**

```
┌─────────────────────────────────────────────┐
│         動画プロンプトの5要素                  │
├─────────────────────────────────────────────┤
│ ① 被写体  ：何が映っているか                 │
│ ② アクション：何をしているか                  │
│ ③ 環境    ：どこで、いつ（時間帯・天候）      │
│ ④ カメラ  ：カメラワーク、アングル            │
│ ⑤ スタイル ：映像の雰囲気、品質              │
└─────────────────────────────────────────────┘
```

**良いプロンプトの例：**
```
A professional Japanese businesswoman giving a presentation
in a modern conference room. She gestures toward a large screen
showing graphs. The camera slowly dollies in from a medium shot
to a close-up. Soft natural lighting from large windows.
Corporate documentary style, shallow depth of field, 4K quality.
```

**改善が必要なプロンプトの例：**
```
会議のシーン
```
このような短いプロンプトでは、AIが多くの要素を「推測」するため、意図した映像になりにくくなります。

### 音声生成の品質向上テクニック

| テクニック | 説明 |
|---|---|
| SSML記法の活用 | 間の長さ、強調、読み方を細かく指定 |
| 句読点の工夫 | 「、」「。」の位置で間を制御 |
| stability値の調整 | 低いと表現豊か、高いと安定的 |
| 分割生成 | 長文は段落ごとに生成して結合 |
| 後処理 | ノイズ除去、音量正規化をDAWで実施 |

### 現時点の限界と制約

動画・音声生成AIは急速に進化していますが、2026年3月時点ではまだいくつかの制約があります。

**動画生成の制約：**
- 長尺動画の一貫性維持が難しい（キャラクターの外見が途中で変わるなど）
- 細かい手指の動きや文字の描写が苦手
- 物理法則に反する動きが発生することがある
- 生成に時間がかかる（1本あたり数分〜数十分）
- 商用利用のライセンス条件がサービスごとに異なる

**音声生成の制約：**
- 固有名詞や専門用語の読み間違い
- 長文での感情の一貫性維持
- リアルタイム会話レベルの低遅延生成はまだ発展途上
- 音声クローンの悪用リスク（ディープフェイク問題）

### よくある間違い

> **「高解像度・長尺で生成すれば良い動画になる」と考えてしまう**
>
> 解像度や長さを上げると生成時間とコストが増大するだけでなく、品質が不安定になる場合があります。まずは短尺・低解像度でプロトタイプを作り、方向性を確認してから本番品質で再生成するのが効率的です。
