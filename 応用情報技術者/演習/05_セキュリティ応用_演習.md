# 第5章 セキュリティ応用 演習問題

## 基本問題

### 問1（四択・PKI）
PKIにおいて、デジタル証明書の失効状態をリアルタイムで確認できるプロトコルはどれか。

ア CRL　　イ OCSP　　ウ LDAP　　エ RADIUS

<details><summary>解答・解説</summary>

**正解: イ OCSP（Online Certificate Status Protocol）**

- CRL: 証明書失効リストで、定期的に公開されるため最新性に欠ける
- OCSP: 個別の証明書の有効性をリアルタイムに問い合わせるプロトコル
- LDAP: ディレクトリサービスのアクセスプロトコル
- RADIUS: 認証サーバのプロトコル

</details>

---

### 問2（四択・OAuth/OIDC）
OAuth 2.0とOpenID Connect（OIDC）に関する記述として、正しいものはどれか。

ア OAuth 2.0は認証プロトコルであり、OIDCは認可プロトコルである
イ OAuth 2.0は認可フレームワークであり、OIDCはOAuth 2.0を拡張した認証プロトコルである
ウ OAuth 2.0とOIDCはどちらも認証のみを目的としたプロトコルである
エ OIDCはOAuth 2.0とは独立した別のプロトコルである

<details><summary>解答・解説</summary>

**正解: イ**

OAuth 2.0は**認可**（Authorization）のフレームワークで、「何ができるか」を制御します。OIDCはOAuth 2.0を拡張した**認証**（Authentication）プロトコルで、IDトークン（JWT形式）により「誰であるか」を確認できます。

</details>

### 問3（四択・ゼロトラスト）
ゼロトラストモデルの基本的な考え方として、最も適切なものはどれか。

ア 社内ネットワークからのアクセスは信頼し、社外からのアクセスのみを検証する
イ ファイアウォールで境界を守れば内部ネットワークは安全である
ウ ネットワークの場所に関わらず、すべてのアクセスを検証する
エ VPNを使用していれば、アクセス元の検証は不要である

<details><summary>解答・解説</summary>

**正解: ウ**

ゼロトラストは「何も信頼しない」を前提とし、ネットワークの場所（社内/社外）に関わらず、全てのアクセスに対して認証・認可を行います。これは従来の境界防御モデル（ア、イ、エの考え方）とは根本的に異なるアプローチです。

</details>

### 問4（四択・フォレンジック）
デジタルフォレンジックにおいて、証拠の収集順序として正しいものはどれか。

ア ディスクイメージ → メモリダンプ → ネットワーク情報
イ メモリダンプ → ネットワーク情報 → ディスクイメージ
ウ ネットワーク情報 → ディスクイメージ → メモリダンプ
エ ディスクイメージ → ネットワーク情報 → メモリダンプ

<details><summary>解答・解説</summary>

**正解: イ**

フォレンジックでは揮発性の高いデータから先に収集します。

```
揮発性: 高 → 低
メモリ（RAM） → ネットワーク接続情報 → ディスク上のファイル
```

電源を切るとメモリの内容は消失するため、最優先で収集します。

</details>

### 問5（四択・リスク）
リスク対応のうち、保険に加入してリスクの財務的影響を他者に移す方法は何か。

ア リスク回避　　イ リスク低減　　ウ リスク移転　　エ リスク保有

<details><summary>解答・解説</summary>

**正解: ウ リスク移転**

- リスク回避: リスクの原因となる活動そのものをやめる
- リスク低減: セキュリティ対策でリスクの発生確率や影響を軽減する
- リスク移転: 保険加入や外部委託でリスクの影響を他者に転嫁する
- リスク保有: リスクを認識したうえで対策をとらず受容する

</details>

## 応用問題

### 問6（四択・ALE計算）
ある組織のWebサーバの資産価値が2,000万円、情報漏洩時の露出係数が30%、年間発生率が0.2回の場合、年間予想損失額（ALE）はどれか。

ア 60万円　　イ 120万円　　ウ 200万円　　エ 600万円

<details><summary>ヒント</summary>

ALE = SLE × ARO、SLE = 資産価値 × 露出係数(EF)

</details>

<details><summary>解答・解説</summary>

**正解: イ 120万円**

```
SLE = 資産価値 × EF = 2,000万円 × 0.3 = 600万円
ALE = SLE × ARO = 600万円 × 0.2 = 120万円/年
```

この結果は、セキュリティ対策への投資判断の材料になります。年間120万円の損失が見込まれるため、それ以下のコストで実施できる対策は投資価値があると判断できます。

</details>

### 問7（四択・Web攻撃）
SQLインジェクション攻撃に対する最も効果的な対策はどれか。

ア WAF（Web Application Firewall）の導入
イ プリペアドステートメント（バインド機構）の使用
ウ ファイアウォールでポート80/443をブロック
エ サーバ証明書の導入

<details><summary>解答・解説</summary>

**正解: イ プリペアドステートメント（バインド機構）の使用**

プリペアドステートメントは、SQL文のテンプレートとパラメータを分離して処理するため、入力値がSQL文の構造を変更することが原理的に不可能になります。これが根本的な対策です。

- ア: WAFは補助的な対策としては有効ですが、バイパスされる可能性がある
- ウ: Webサービスが利用できなくなる
- エ: 通信の暗号化であり、SQLインジェクションとは無関係

</details>

### 問8（記述・認証フロー）
OAuth 2.0の認可コードフローにおける、以下の各ステップの役割を説明せよ。

1. 認可リクエスト
2. 認可コードの発行
3. トークンリクエスト
4. アクセストークンの発行
5. APIアクセス

<details><summary>解答・解説</summary>

1. **認可リクエスト:** クライアントアプリがユーザを認可サーバの認可エンドポイントにリダイレクトし、リソースへのアクセス許可を要求します。ユーザはログインし、アクセスを許可する同意画面で承認します。

2. **認可コードの発行:** ユーザが承認すると、認可サーバは一時的な認可コードを発行し、事前に登録されたリダイレクトURIにコードを付与してクライアントアプリに戻します。この認可コードは短時間で失効します。

3. **トークンリクエスト:** クライアントアプリはバックエンドサーバから認可サーバのトークンエンドポイントに対し、認可コード、クライアントID、クライアントシークレットを送信してアクセストークンを要求します。

4. **アクセストークンの発行:** 認可サーバはクライアントの認証と認可コードの検証を行い、問題がなければアクセストークン（必要に応じてリフレッシュトークンも）を発行します。

5. **APIアクセス:** クライアントアプリは取得したアクセストークンをHTTPヘッダ（Authorization: Bearer トークン）に含めて、リソースサーバのAPIにアクセスします。リソースサーバはトークンを検証し、アクセスを許可します。

</details>

## チャレンジ問題

### 問9（記述・午後試験形式）
C社のセキュリティインシデント対応について、以下の設問に答えなさい。

C社のSOCチームは、SIEMのアラートにより社内の1台のPCから外部の不審なサーバへの通信（C&C通信の疑い）を検知した。

**設問1:** インシデント対応の手順を、検知から事後対応まで時系列で6つのステップに分けて記述せよ。

**設問2:** 該当PCのフォレンジック調査を行う際、最初に実施すべき作業と、その理由を述べよ。

**設問3:** 今後の再発防止策として、技術的対策と組織的対策をそれぞれ2つずつ挙げよ。

<details><summary>解答・解説</summary>

**設問1の解答:**

1. **検知:** SIEMアラートの内容を確認し、C&C通信の疑いがあることを認識する
2. **トリアージ:** 通信先のIPアドレス/ドメインを脅威インテリジェンスで照合し、影響範囲（他に感染PCがないか）を調査して優先度を判定する
3. **封じ込め:** 該当PCをネットワークから隔離し、不審な通信先をファイアウォールでブロックする。ただし、証拠保全のためPCの電源は切らない
4. **根絶:** マルウェアの特定と除去、感染経路の特定、同じ脆弱性を持つ他のPCへの対策実施
5. **復旧:** クリーンなバックアップからのPC復元、または再インストール後、正常動作を確認してネットワークに再接続する
6. **事後対応:** インシデント報告書の作成、再発防止策の策定、関係者への情報共有、必要に応じて外部への報告（個人情報漏洩の場合など）

**設問2の解答:**

最初に実施すべき作業: **メモリダンプ（メモリイメージ）の取得**

理由: メモリ上にはマルウェアのプロセス情報、暗号化キー、ネットワーク接続情報、復号されたC&C通信の内容など、揮発性の高い重要な証拠が存在します。電源を切るとこれらの情報は全て消失するため、ディスクイメージの取得よりも先にメモリダンプを優先する必要があります。メモリダンプの取得後にハッシュ値を記録し、証拠の同一性を確保します。

**設問3の解答:**

技術的対策:
1. **EDR（Endpoint Detection and Response）の導入:** エンドポイントレベルで不審な挙動（C&C通信パターン、異常なプロセス起動など）をリアルタイムに検知・対応できる仕組みを導入する
2. **ネットワークセグメンテーションの強化:** 重要なサーバセグメントと端末セグメントを分離し、セグメント間通信をファイアウォールで制御することで、万一感染しても横展開を防止する

組織的対策:
1. **セキュリティ教育の実施:** 全社員に対して標的型メール攻撃の手口やマルウェア感染経路に関する教育・訓練を定期的に実施し、不審なメールやリンクへの注意を喚起する
2. **インシデント対応手順書の整備:** 今回の対応を踏まえ、インシデント発生時の初動対応手順、エスカレーション基準、連絡体制を文書化し、定期的に訓練を実施する

</details>

### 問10（記述・午後試験形式）
ゼロトラストアーキテクチャへの移行計画について、以下の設問に答えなさい。

D社は従来の境界防御モデル（VPN＋ファイアウォール）で社内システムを保護していたが、リモートワークの拡大に伴いゼロトラストモデルへの移行を検討している。

**設問1:** 従来の境界防御モデルの限界を3つ挙げよ。

**設問2:** ゼロトラスト実現のために導入すべき技術要素を4つ挙げ、それぞれの役割を簡潔に説明せよ。

<details><summary>解答・解説</summary>

**設問1の解答:**

1. **内部脅威に対する防御が不十分:** 境界防御モデルは外部からの脅威を前提としているため、社内ネットワークに侵入された後の横展開（ラテラルムーブメント）に対する防御が弱い。内部犯行者やマルウェア感染端末からの攻撃に対して脆弱である

2. **リモートワークへの対応が困難:** VPNは全トラフィックを社内に集約するため、利用者が増えるとVPN装置がボトルネックとなり性能が低下する。また、クラウドサービスへのアクセスも社内経由となり非効率である

3. **クラウド環境の保護が不十分:** 境界の概念がないクラウドサービス（SaaS等）に対して、従来のファイアウォールでは適切なアクセス制御ができない。シャドーITの管理も困難である

**設問2の解答:**

1. **IAM（Identity and Access Management）:** ユーザのID管理、認証、認可を一元管理するプラットフォーム。全アクセスの起点となるID基盤として、シングルサインオン（SSO）や多要素認証（MFA）を提供する

2. **EDR（Endpoint Detection and Response）:** エンドポイント（PC・スマートフォン等）の振る舞いを監視し、マルウェアや不正活動を検知・対応する。デバイスの健全性をリアルタイムに確認し、アクセス制御の判断材料を提供する

3. **CASB（Cloud Access Security Broker）:** 従業員のクラウドサービス利用を可視化・制御するプロキシ。シャドーITの発見、データ漏洩防止（DLP）、コンプライアンス確保を実現する

4. **マイクロセグメンテーション:** ネットワークを細かいセグメントに分割し、セグメント間の通信を最小限に制御する技術。万一1つのセグメントが侵害されても、他のセグメントへの影響を限定できる

</details>
