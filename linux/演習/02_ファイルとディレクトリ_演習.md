## 基本問題

### 問題1: パスの種類
以下のパスを「絶対パス」と「相対パス」に分類してください。

1. `/home/tanaka/documents`
2. `./scripts/backup.sh`
3. `/var/log/syslog`
4. `../config/settings.conf`
5. `~/Desktop`

<details>
<summary>解答</summary>

```
【絶対パス】（/ から始まるパス）
1. /home/tanaka/documents
3. /var/log/syslog

【相対パス】（現在のディレクトリからの相対的なパス）
2. ./scripts/backup.sh（. = 現在のディレクトリ）
4. ../config/settings.conf（.. = 一つ上のディレクトリ）

【特殊なパス】
5. ~/Desktop（~ = ホームディレクトリ。展開されると絶対パスになる）
```

</details>

### 問題2: ディレクトリ移動
現在 `/home/tanaka/projects/webapp` にいるとき、以下の各操作の後のカレントディレクトリを答えてください。

1. `cd /etc`
2. `cd ..`（元の位置に戻った状態から）
3. `cd ~`
4. `cd -`（3の直後に実行）

<details>
<summary>解答</summary>

```bash
# 初期位置: /home/tanaka/projects/webapp

# 1. 絶対パスで移動
$ cd /etc
# → /etc

# 2. 元の位置に戻った後、一つ上に移動
# （/home/tanaka/projects/webapp から）
$ cd ..
# → /home/tanaka/projects

# 3. ホームディレクトリに移動
$ cd ~
# → /home/tanaka

# 4. 直前のディレクトリに戻る
# （cd ~ の直前にいた /home/tanaka/projects に戻る）
$ cd -
# → /home/tanaka/projects
```

</details>

### 問題3: ls コマンドのオプション
以下の各要件を満たす `ls` コマンドを書いてください。

1. 隠しファイルを含めて詳細表示する
2. ファイルサイズを人が読みやすい形式で表示する
3. 更新日時の新しい順にソートして表示する
4. `/var/log` 配下のすべての `.log` ファイルを一覧表示する

<details>
<summary>解答</summary>

```bash
# 1. 隠しファイルを含めて詳細表示
$ ls -la

# 2. ファイルサイズを人が読みやすい形式で表示
$ ls -lh

# 3. 更新日時の新しい順にソートして表示
$ ls -lt

# 4. /var/log 配下のすべての .log ファイルを一覧表示
$ ls /var/log/*.log
```

</details>

### 問題4: ファイルとディレクトリの作成
以下のディレクトリ構造を作成するコマンドを書いてください。

```
myproject/
├── src/
│   ├── main.py
│   └── utils.py
├── tests/
│   └── test_main.py
└── README.md
```

<details>
<summary>ヒント</summary>

`mkdir -p` で深いディレクトリを一度に作成できます。`touch` でファイルを作成します。

</details>

<details>
<summary>解答</summary>

```bash
# ディレクトリを作成（-p で親ディレクトリも同時に作成）
$ mkdir -p myproject/src myproject/tests

# ファイルを作成
$ touch myproject/src/main.py
$ touch myproject/src/utils.py
$ touch myproject/tests/test_main.py
$ touch myproject/README.md

# 確認
$ ls -R myproject/
myproject/:
README.md  src  tests

myproject/src:
main.py  utils.py

myproject/tests:
test_main.py
```

</details>

### 問題5: ファイルのコピーと移動
以下の操作を行うコマンドを書いてください。

1. `report.txt` を `backup/` ディレクトリにコピーする
2. `old_name.txt` を `new_name.txt` にリネームする
3. `data/` ディレクトリを中身ごと `archive/` にコピーする
4. `temp.log` を `/var/log/` に移動する

<details>
<summary>解答</summary>

```bash
# 1. ファイルをコピー
$ cp report.txt backup/

# 2. ファイルのリネーム（mvコマンドを使用）
$ mv old_name.txt new_name.txt

# 3. ディレクトリを中身ごとコピー（-r で再帰的にコピー）
$ cp -r data/ archive/

# 4. ファイルを移動
$ mv temp.log /var/log/
```

</details>

### 問題6: ファイルの削除
以下の操作を行うコマンドを書いてください。ただし、安全に行うことを意識してください。

1. `unwanted.txt` を確認しながら削除する
2. `old_dir/` ディレクトリを中身ごと削除する
3. カレントディレクトリの `.tmp` ファイルをすべて削除する

<details>
<summary>解答</summary>

```bash
# 1. 確認しながら削除（-i オプション）
$ rm -i unwanted.txt
rm: remove regular file 'unwanted.txt'? y

# 2. ディレクトリを中身ごと削除（-r で再帰的に削除）
$ rm -r old_dir/
# ※ 安全のため -ri で確認しながら削除することを推奨
$ rm -ri old_dir/

# 3. .tmp ファイルをすべて削除
$ rm *.tmp
# ※ 安全のため先に ls で確認してから削除するのが良い
$ ls *.tmp    # 対象を確認
$ rm *.tmp    # 削除実行
```

</details>

### 問題7: find コマンド
以下の条件でファイルを検索する `find` コマンドを書いてください。

1. `/home` 以下から `.txt` ファイルをすべて検索する
2. `/var/log` 以下から7日以上前に更新されたファイルを検索する
3. `/tmp` 以下からサイズが10MB以上のファイルを検索する
4. カレントディレクトリ以下から空のディレクトリを検索する

<details>
<summary>解答</summary>

```bash
# 1. .txt ファイルを検索
$ find /home -name "*.txt"

# 2. 7日以上前に更新されたファイル
$ find /var/log -mtime +7

# 3. 10MB以上のファイル
$ find /tmp -size +10M

# 4. 空のディレクトリ
$ find . -type d -empty
```

</details>

### 問題8: ファイル内容の表示
以下の要件を満たすコマンドを書いてください。

1. `/etc/passwd` の先頭5行を表示する
2. `/var/log/syslog` の末尾20行を表示する
3. `/var/log/syslog` をリアルタイムで監視する
4. `config.txt` の内容を行番号付きで表示する

<details>
<summary>解答</summary>

```bash
# 1. 先頭5行を表示
$ head -n 5 /etc/passwd
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin
sys:x:3:3:sys:/dev:/usr/sbin/nologin
sync:x:4:65534:sync:/bin:/bin/sync

# 2. 末尾20行を表示
$ tail -n 20 /var/log/syslog

# 3. リアルタイムで監視（-f オプション）
$ tail -f /var/log/syslog
# Ctrl+C で停止

# 4. 行番号付きで表示（cat -n または nl）
$ cat -n config.txt
     1  server_name=web01
     2  port=8080
     3  environment=production
# または
$ nl config.txt
```

</details>

### 問題9: リダイレクトとワイルドカード
以下の操作を行うコマンドを書いてください。

1. `ls -la` の結果を `filelist.txt` に保存する
2. `date` の出力を `filelist.txt` に追記する
3. カレントディレクトリの `.sh` ファイルの数を数える

<details>
<summary>解答</summary>

```bash
# 1. ls -la の結果をファイルに保存（リダイレクト >）
$ ls -la > filelist.txt

# 2. date の出力を追記（追記リダイレクト >>）
$ date >> filelist.txt

# 3. .sh ファイルの数を数える（ls + wc のパイプ）
$ ls *.sh | wc -l
# または
$ find . -maxdepth 1 -name "*.sh" | wc -l
```

</details>

### 問題10: 総合問題
以下の一連の操作をコマンドで実行してください。

1. ホームディレクトリに `exercise` ディレクトリを作成する
2. `exercise` に移動する
3. `file1.txt`、`file2.txt`、`file3.txt` を作成する
4. `file1.txt` に "Hello, Linux!" と書き込む
5. `file1.txt` を `backup.txt` という名前でコピーする
6. 全ファイルの詳細一覧を表示する
7. `file3.txt` を削除する
8. 最終的なファイル一覧を確認する

<details>
<summary>解答</summary>

```bash
# 1. ディレクトリを作成
$ mkdir ~/exercise

# 2. 移動
$ cd ~/exercise

# 3. ファイルを作成
$ touch file1.txt file2.txt file3.txt

# 4. file1.txt に書き込む
$ echo "Hello, Linux!" > file1.txt

# 5. コピー
$ cp file1.txt backup.txt

# 6. 詳細一覧を表示
$ ls -la
total 16
drwxr-xr-x 2 tanaka tanaka 4096 Mar 20 15:30 .
drwxr-xr-x 8 tanaka tanaka 4096 Mar 20 15:30 ..

-rw-r--r-- 1 tanaka tanaka    0 Mar 20 15:30 file2.txt
-rw-r--r-- 1 tanaka tanaka    0 Mar 20 15:30 file3.txt

# 7. 削除
$ rm file3.txt

# 8. 最終確認
$ ls -la
total 16
drwxr-xr-x 2 tanaka tanaka 4096 Mar 20 15:30 .
drwxr-xr-x 8 tanaka tanaka 4096 Mar 20 15:30 ..
-rw-r--r-- 1 tanaka tanaka   14 Mar 20 15:30 backup.txt

```

### 問題11: find と実行の組み合わせ
`/var/log` 以下から、サイズが100KB以上で、拡張子が `.log` のファイルを検索し、そのファイルパスとサイズを表示するコマンドを書いてください。

<details>
<summary>ヒント</summary>

`find` コマンドの `-exec` オプション、または `find` + `xargs` + `ls -lh` を組み合わせます。

</details>

<details>
<summary>解答</summary>

```bash
# 方法1: find + -exec
$ find /var/log -name "*.log" -size +100k -exec ls -lh {} \;

# 方法2: find + xargs
$ find /var/log -name "*.log" -size +100k | xargs ls -lh

# 方法3: find の -printf オプション
$ find /var/log -name "*.log" -size +100k -printf "%s %p\n"
```

</details>
