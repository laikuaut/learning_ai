## 課題2：バグのあるプログラムを修正する

### 問題3：最大値の探索（バグあり）

以下のプログラムには2つのバグがあります。見つけて修正してください。

```python
# 配列の最大値を探すプログラム
def find_max(data):
    max_value = 0          # バグ1はこの行にあります
    max_index = 0

    for i in range(1, len(data)):   # バグ2はこの行にあります
        if data[i] >= max_value:
            max_value = data[i]
            max_index = i

    return max_value, max_index

# テスト
print(find_max([3, 7, 2, 9, 1]))   # 期待: (9, 3)
print(find_max([-5, -2, -8, -1]))   # 期待: (-1, 3)
print(find_max([5, 5, 5, 5]))       # 期待: (5, 0) ← 最初に見つかった位置
```

<details>
<summary>ヒント</summary>

- バグ1: 初期値が0だと、全要素が負のときに正しく動きません
- バグ2: ループの開始位置と比較演算子を確認してください

</details>

### 問題4：二分探索（バグあり）

以下のプログラムには1つのバグがあります。特定の条件で無限ループになります。

```python
def binary_search(data, target):
    low = 0
    high = len(data) - 1

    while low <= high:
        mid = (low + high) // 2
        if data[mid] == target:
            return mid
        elif data[mid] < target:
            low = mid         # バグはこの行
        else:
            high = mid        # バグはこの行
    return -1

# テスト
data = [1, 3, 5, 7, 9, 11, 13]
print(binary_search(data, 7))    # 期待: 3
print(binary_search(data, 4))    # 期待: -1（無限ループ？）
```

<details>
<summary>ヒント</summary>

`low = mid` だと、`data[mid] < target` のとき探索範囲が1つも狭まらない場合があります。
例: low=2, high=3, mid=2 のとき、low=2 のまま変化しません。

</details>

### 問題5：スタックによる括弧チェック（バグあり）

以下のプログラムには論理的なバグがあります。

```python
def check_brackets(text):
    """括弧の対応が正しいか判定する"""
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}

    for char in text:
        if char in '([{':
            stack.append(char)
        elif char in ')]}':
            if len(stack) == 0:
                return False
            top = stack.pop()
            if top != pairs[char]:
                return True     # バグ: ここの返り値が逆

    return len(stack) == 0

# テスト
print(check_brackets("(a + b) * [c - d]"))  # 期待: True
print(check_brackets("(a + b"))              # 期待: False
print(check_brackets("(a + b]"))             # 期待: False ← バグで True になる
print(check_brackets(")("))                  # 期待: False
```

<summary>問題3の解答</summary>

**バグ1:** `max_value = 0` → 全要素が負のとき、0より大きい要素がないため正しく動かない。
**修正:** `max_value = data[0]`

**バグ2:** `if data[i] >= max_value` → `>=` だと同値のとき後ろの位置を返してしまう。「最初に見つかった位置」を返すなら `>` にする。

```python
def find_max(data):
    max_value = data[0]    # 修正: 最初の要素で初期化
    max_index = 0

    for i in range(1, len(data)):
        if data[i] > max_value:    # 修正: > に変更
            max_value = data[i]
            max_index = i

    return max_value, max_index
```

<details>
<summary>問題4の解答</summary>

**バグ:** `low = mid` と `high = mid` → 探索範囲が狭まらず無限ループになる。

**修正:** `low = mid + 1`, `high = mid - 1`

```python
def binary_search(data, target):
    low = 0
    high = len(data) - 1

    while low <= high:
        mid = (low + high) // 2
        if data[mid] == target:
            return mid
        elif data[mid] < target:
            low = mid + 1       # 修正: +1 を追加
        else:
            high = mid - 1      # 修正: -1 を追加
    return -1
```

**無限ループが発生する例:** data=[1,3,5,7], target=4
- low=1, high=2, mid=1 → data[1]=3 < 4 → low=mid=1（変化なし！）
- low=1, high=2, mid=1 → 同じ状態の繰り返し...

</details>

<details>
<summary>問題7の解答</summary>

式: `3 4 + 2 * 5 -`

| トークン | 操作 | スタック状態 |
|---------|------|------------|
| 3 | push(3) | [3] |
| 4 | push(4) | [3, 4] |
| + | pop→4, pop→3, push(3+4=7) | [7] |
| 2 | push(2) | [7, 2] |
| * | pop→2, pop→7, push(7*2=14) | [14] |
| 5 | push(5) | [14, 5] |
| - | pop→5, pop→14, push(14-5=9) | [9] |

**計算結果: 9**

中置記法に直すと: `(3 + 4) × 2 - 5 = 7 × 2 - 5 = 14 - 5 = 9`
