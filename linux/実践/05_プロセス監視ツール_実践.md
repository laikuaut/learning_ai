> **前提知識**: 第1章（Linuxの基本）、第4章（プロセス管理）

## 課題の要件

1. システムのロードアベレージ（load average）を表示する
2. メモリの使用状況を表示する
3. 実行中のプロセス総数を表示する
4. CPU使用率の上位5プロセスを表示する
5. メモリ使用率の上位5プロセスを表示する
6. 結果をファイルに保存する

---

## ステップガイド

<details>
<summary>ステップ1：プロセス関連コマンドを確認する</summary>

```bash
# プロセスの一覧（全ユーザー、詳細表示）
$ ps aux

# CPU使用率順にソート（上位5件）
$ ps aux --sort=-%cpu | head -6

# メモリ使用率順にソート（上位5件）
$ ps aux --sort=-%mem | head -6

# ロードアベレージ
$ uptime
# または
$ cat /proc/loadavg

# メモリ使用状況
$ free -h

# プロセス数のカウント
$ ps aux | wc -l
```

</details>

<details>
<summary>ステップ2：必要な情報を抽出する</summary>

`ps` の出力をフォーマットするには `ps -eo` で表示列を指定できます。

```bash
# 表示列を指定（PID、ユーザー、CPU%、メモリ%、コマンド）
$ ps -eo pid,user,%cpu,%mem,comm --sort=-%cpu | head -6

# ロードアベレージの数値だけ取得
$ uptime | awk -F'load average:' '{print $2}'

# メモリ使用率を計算
$ free | awk '/Mem:/ {printf "使用率: %.0f%%\n", $3/$2*100}'
```

</details>

<details>
<summary>ステップ3：スクリプトにまとめる</summary>

各コマンドの出力を `echo` と組み合わせてレポート形式にまとめます。
リダイレクトでファイルに保存する部分を追加しましょう。

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

```bash
#!/bin/bash
# プロセス監視スクリプト
# 学べる内容：ps, free, uptime, プロセスの見方
# 実行方法：bash process_monitor.sh

REPORT="process_report.txt"

echo "===== プロセス監視レポート =====" | tee "$REPORT"
echo "日時: $(date '+%Y-%m-%d %H:%M:%S')" | tee -a "$REPORT"
echo "" | tee -a "$REPORT"

# --- システム負荷 ---
echo "【システム負荷】" | tee -a "$REPORT"
echo "ロードアベレージ: $(uptime | awk -F'load average:' '{print $2}')" | tee -a "$REPORT"
echo "CPU数: $(nproc)" | tee -a "$REPORT"
echo "" | tee -a "$REPORT"

# --- メモリ使用状況 ---
echo "【メモリ使用状況】" | tee -a "$REPORT"
free -h | head -2 | tee -a "$REPORT"
echo "" | tee -a "$REPORT"

# --- プロセス概要 ---
echo "【プロセス概要】" | tee -a "$REPORT"
total=$(ps aux | wc -l)
running=$(ps aux | awk '$8 ~ /R/ {count++} END {print count+0}')
sleeping=$(ps aux | awk '$8 ~ /S/ {count++} END {print count+0}')
echo "総プロセス数: $total" | tee -a "$REPORT"
echo "実行中(R): $running" | tee -a "$REPORT"
echo "スリープ(S): $sleeping" | tee -a "$REPORT"
echo "" | tee -a "$REPORT"

# --- CPU使用率 TOP5 ---
echo "【CPU使用率 TOP5】" | tee -a "$REPORT"
ps -eo pid,user,%cpu,comm --sort=-%cpu | head -6 | tee -a "$REPORT"
echo "" | tee -a "$REPORT"

# --- メモリ使用率 TOP5 ---
echo "【メモリ使用率 TOP5】" | tee -a "$REPORT"
ps -eo pid,user,%mem,rss,comm --sort=-%mem | head -6 | tee -a "$REPORT"
echo "" | tee -a "$REPORT"

echo "レポートを ${REPORT} に保存しました。"
```

</details>

<details>
<summary>解答例（改良版 ─ 繰り返し監視と閾値アラート）</summary>

```bash
#!/bin/bash
# プロセス監視スクリプト（改良版）
# 学べる内容：ループ監視、閾値チェック、kill、シグナル
# 実行方法：bash process_monitor_v2.sh [間隔(秒)] [回数]
# 例：bash process_monitor_v2.sh 5 3  ← 5秒間隔で3回監視

INTERVAL="${1:-5}"   # 監視間隔（デフォルト5秒）
COUNT="${2:-1}"      # 監視回数（デフォルト1回）
LOG_DIR="process_logs"
CPU_THRESHOLD=80     # CPU使用率の警告閾値(%)
MEM_THRESHOLD=80     # メモリ使用率の警告閾値(%)

mkdir -p "$LOG_DIR"

# メモリ使用率を計算する関数
get_memory_usage() {
    free | awk '/Mem:/ {printf "%.1f", $3/$2*100}'
}

# ロードアベレージを取得する関数
get_load_average() {
    cat /proc/loadavg | awk '{print $1, $2, $3}'
}

# 警告チェック関数
check_alerts() {
    local mem_usage
    mem_usage=$(get_memory_usage)

    # メモリ使用率チェック
    if [ "$(echo "$mem_usage > $MEM_THRESHOLD" | bc 2>/dev/null || echo 0)" -eq 1 ]; then
        echo "  [警告] メモリ使用率が ${mem_usage}% です（閾値: ${MEM_THRESHOLD}%）"
    fi

    # CPU使用率の高いプロセスチェック
    ps -eo pid,user,%cpu,comm --sort=-%cpu --no-headers | head -5 | while read -r pid user cpu comm; do
        cpu_int=${cpu%.*}
        if [ "${cpu_int:-0}" -ge "$CPU_THRESHOLD" ]; then
            echo "  [警告] プロセス ${comm}(PID:${pid}) の CPU使用率が ${cpu}% です"
        fi
    done
}

# メインの監視ループ
for ((i = 1; i <= COUNT; i++)); do
    TIMESTAMP=$(date '+%Y-%m-%d_%H%M%S')
    REPORT="$LOG_DIR/report_${TIMESTAMP}.txt"

    {
        echo "===== プロセス監視レポート ($i/$COUNT) ====="
        echo "日時: $(date '+%Y-%m-%d %H:%M:%S')"
        echo ""

        echo "【システム負荷】"
        echo "ロードアベレージ: $(get_load_average)"
        echo "CPU数: $(nproc)"
        echo ""

        echo "【メモリ使用状況】"
        free -h | grep -E "^(      |Mem:|Swap:)"
        echo "メモリ使用率: $(get_memory_usage)%"
        echo ""

        echo "【プロセス概要】"
        ps aux | awk '
        BEGIN {total=0; run=0; sleep=0; zombie=0}
        NR>1 {
            total++
            if ($8 ~ /R/) run++
            else if ($8 ~ /S/) sleep++
            else if ($8 ~ /Z/) zombie++
        }
        END {
            printf "総プロセス数: %d\n", total
            printf "実行中(R): %d\n", run
            printf "スリープ(S): %d\n", sleep
            printf "ゾンビ(Z): %d\n", zombie
        }'
        echo ""

        echo "【CPU使用率 TOP5】"
        printf "  %-8s %-12s %6s  %s\n" "PID" "USER" "%CPU" "COMMAND"
        ps -eo pid,user,%cpu,comm --sort=-%cpu --no-headers | head -5 | \
            while read -r pid user cpu comm; do
                printf "  %-8s %-12s %6s  %s\n" "$pid" "$user" "$cpu" "$comm"
            done
        echo ""

        echo "【メモリ使用率 TOP5】"
        printf "  %-8s %-12s %6s %10s  %s\n" "PID" "USER" "%MEM" "RSS" "COMMAND"
        ps -eo pid,user,%mem,rss,comm --sort=-%mem --no-headers | head -5 | \
            while read -r pid user mem rss comm; do
                # RSS をKBから人間が読みやすい形式に変換
                if [ "$rss" -ge 1048576 ]; then
                    rss_fmt="$(echo "scale=1; $rss/1048576" | bc)G"
                elif [ "$rss" -ge 1024 ]; then
                    rss_fmt="$(echo "scale=1; $rss/1024" | bc)M"
                else
                    rss_fmt="${rss}K"
                fi
                printf "  %-8s %-12s %6s %10s  %s\n" "$pid" "$user" "$mem" "$rss_fmt" "$comm"
            done
        echo ""

        echo "【アラート】"
        alerts=$(check_alerts)
        if [ -n "$alerts" ]; then
            echo "$alerts"
        else
            echo "  異常なし"
        fi
    } | tee "$REPORT"

    echo ""
    echo "レポート保存: $REPORT"

    # 最後のループでなければ待機
    if [ "$i" -lt "$COUNT" ]; then
        echo "次の監視まで ${INTERVAL} 秒待機..."
        sleep "$INTERVAL"
        echo ""
    fi
done

echo ""
echo "監視完了（全 $COUNT 回）。ログ: $LOG_DIR/"
```

**初心者向けとの違い:**
- コマンドライン引数で監視間隔と回数を指定可能
- CPU/メモリ使用率の閾値チェックとアラート機能
- ゾンビプロセス（zombie process）の検出
- RSS（Resident Set Size）を人間が読みやすい形式（KB/MB/GB）に変換
- タイムスタンプ付きのログファイルで履歴を保存

</details>
