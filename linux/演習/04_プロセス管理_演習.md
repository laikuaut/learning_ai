## 基本問題

### 問題1: プロセスの基本概念
以下の質問に答えてください。

1. PID とは何ですか？
2. PID 1 のプロセスは通常何ですか？
3. プロセスの状態で `R`、`S`、`Z` はそれぞれ何を意味しますか？

<details>
<summary>解答</summary>

```
1. PID（Process ID）は、プロセスに割り振られる一意の識別番号です。
   カーネルがプロセスを管理するために使用します。

2. PID 1 は systemd（または init）プロセスです。
   すべてのプロセスの親であり、システム起動時に最初に実行されます。

3. プロセスの状態:
   R（Running）  = 実行中、またはCPU実行待ちの状態
   S（Sleeping） = 入出力やイベントを待機中（割り込み可能なスリープ）
   Z（Zombie）   = プロセスは終了したが、親プロセスがまだ終了を回収していない状態
```

</details>

### 問題2: ps コマンド
以下の要件を満たす `ps` コマンドを書いてください。

1. 全ユーザーの全プロセスを詳細表示する
2. 特定のユーザー（tanaka）のプロセスだけを表示する
3. nginx に関連するプロセスを検索する

<details>
<summary>解答</summary>

```bash
# 1. 全プロセスを詳細表示（BSD形式）
$ ps aux
# または（System V形式）
$ ps -ef

# 2. 特定ユーザーのプロセス
$ ps -u tanaka

# 3. nginx 関連のプロセスを検索（ps + grep の組み合わせ）
$ ps aux | grep nginx
root       900  0.0  0.1  65432  5432 ?  Ss   10:00   0:00 nginx: master process
www-data   901  0.0  0.2  65832  8234 ?  S    10:00   0:01 nginx: worker process
```

</details>

### 問題3: ps aux の出力の読み方
以下の `ps aux` の出力の各フィールドの意味を答えてください。

```
USER       PID %CPU %MEM    VSZ   RSS TTY    STAT START   TIME COMMAND
root         1  0.0  0.3 169436 12048 ?      Ss   Feb15   2:30 /sbin/init
```

<details>
<summary>解答</summary>

```
USER    = プロセスを実行しているユーザー（root）
PID     = プロセスID（1）
%CPU    = CPU使用率（0.0%）
%MEM    = メモリ使用率（0.3%）
VSZ     = 仮想メモリサイズ（169436 KB）
RSS     = 実メモリ使用量（12048 KB）
TTY     = 制御端末（? = 端末なし、デーモン等）
STAT    = プロセスの状態（Ss = スリープ中のセッションリーダー）
START   = 開始日時（Feb15）
TIME    = CPU累積使用時間（2分30秒）
COMMAND = 実行コマンド（/sbin/init）
```

</details>

### 問題4: プロセスの停止
以下のシナリオでどのコマンドを使うべきか答えてください。

1. PID 12345 のプロセスを正常に終了させたい
2. PID 12345 のプロセスが応答しないので強制終了させたい
3. 名前が `python3` のプロセスをすべて終了させたい

<details>
<summary>解答</summary>

```bash
# 1. 正常終了（SIGTERM シグナルを送信）
$ kill 12345
# または明示的にシグナルを指定
$ kill -15 12345

# 2. 強制終了（SIGKILL シグナルを送信）
$ kill -9 12345

# 3. 名前でプロセスを終了
$ killall python3
# または
$ pkill python3
```

シグナルの違い:
- SIGTERM (15): 正常終了を要求。プロセスは後処理を行ってから終了できる
- SIGKILL (9): 強制終了。プロセスは一切の後処理なしに即座に終了する
- まず SIGTERM を試し、応答しない場合のみ SIGKILL を使うのが正しい手順

</details>

### 問題5: top コマンド
`top` コマンドの実行中に以下の操作をするにはどのキーを押すか答えてください。

1. CPU使用率でソートする
2. メモリ使用率でソートする
3. top を終了する
4. プロセスを終了する

<details>
<summary>解答</summary>

```
1. CPU使用率でソート: P キー（Shift + p）
2. メモリ使用率でソート: M キー（Shift + m）
3. top を終了: q キー
4. プロセスを終了: k キーを押してPIDを入力
```

</details>

### 問題6: バックグラウンド実行
以下のシナリオで使うコマンドを書いてください。

1. `long_process.sh` をバックグラウンドで実行する
2. 実行中のフォアグラウンドプロセスを一時停止する
3. 一時停止したプロセスをバックグラウンドで再開する
4. バックグラウンドジョブの一覧を表示する
5. バックグラウンドのジョブ番号1をフォアグラウンドに戻す

<details>
<summary>解答</summary>

```bash
# 1. バックグラウンドで実行（末尾に & を付ける）
$ ./long_process.sh &
[1] 12345

# 2. フォアグラウンドプロセスを一時停止
# Ctrl + Z を押す
[1]+  Stopped                 ./long_process.sh

# 3. バックグラウンドで再開
$ bg
[1]+ ./long_process.sh &

# 4. ジョブ一覧の表示
$ jobs
[1]+  Running                 ./long_process.sh &

# 5. フォアグラウンドに戻す
$ fg %1
```

</details>

### 問題7: プロセスの監視スクリプト
特定のプロセスが動いているか確認し、動いていなければメッセージを表示するコマンドを書いてください。確認対象は `nginx` です。

期待される出力例（nginx が動いている場合）：
```
nginx は動作中です (PID: 900)
```

期待される出力例（nginx が動いていない場合）：
```
nginx は動作していません！
```

<details>
<summary>ヒント</summary>

`pgrep` コマンドを使うと、プロセス名からPIDを取得できます。

</details>

<details>
<summary>解答</summary>

```bash
# 方法1: pgrep を使用
$ if pgrep -x nginx > /dev/null; then
    echo "nginx は動作中です (PID: $(pgrep -x nginx | head -1))"
  else
    echo "nginx は動作していません！"
  fi

# 方法2: ps + grep を使用
$ if ps aux | grep -v grep | grep -q nginx; then
    echo "nginx は動作中です"
  else
    echo "nginx は動作していません！"
  fi

# 方法3: systemctl を使用（サービスとして管理されている場合）
$ if systemctl is-active --quiet nginx; then
    echo "nginx は動作中です"
  else
    echo "nginx は動作していません！"
  fi
```

</details>

### 問題8: nohup と画面切断
SSH接続中に時間のかかる処理を実行し、SSH切断後も継続させるコマンドを書いてください。

<details>
<summary>解答</summary>

```bash
# nohup を使用（SSH切断後もプロセスが継続する）
$ nohup ./long_task.sh > output.log 2>&1 &
[1] 12345

# 確認
$ jobs
[1]+  Running  nohup ./long_task.sh > output.log 2>&1 &

# SSH 切断後にログを確認
$ tail -f output.log
```

解説：
- `nohup`: HUPシグナル（ターミナル切断時に送られる）を無視する
- `> output.log 2>&1`: 標準出力と標準エラーをファイルに保存
- `&`: バックグラウンドで実行

</details>

### 問題9: メモリ使用量トップ5
メモリ使用率が高い順にプロセスのトップ5を表示するコマンドを書いてください。
表示する情報は、ユーザー名、PID、メモリ使用率、コマンド名です。

期待される出力例：
```
www-data  900  5.2  nginx
mysql    1000  3.1  mysqld
tanaka   1200  1.8  python3
root      850  0.8  sshd
root        1  0.3  systemd
```

<details>
<summary>ヒント</summary>

`ps aux` の出力を `sort` でソートし、`head` で上位を切り出します。`awk` でフィールドを選択します。

</details>

<details>
<summary>解答</summary>

```bash
# 方法1: ps + sort + head + awk
$ ps aux --sort=-%mem | awk 'NR>1 && NR<=6 {print $1, $2, $4, $11}'

# 方法2: ps のカスタムフォーマット
$ ps -eo user,pid,%mem,comm --sort=-%mem | head -6

# 方法3: より整形された出力
$ ps aux --sort=-%mem | head -6 | awk '{printf "%-10s %6s %5s  %s\n", $1, $2, $4, $11}'
```

</details>

### 問題10: プロセスツリーの確認
現在のシェルから起動したプロセスの親子関係をツリー形式で表示するコマンドを書いてください。

<details>
<summary>解答</summary>

```bash
# プロセスツリーを表示
$ pstree -p $$
bash(12346)───pstree(12500)

# 全体のプロセスツリー
$ pstree -p
systemd(1)─┬─cron(950)
            ├─nginx(900)─┬─nginx(901)
            │             └─nginx(902)
            └─sshd(850)───sshd(12345)───bash(12346)───pstree(12500)

# $$ は現在のシェルの PID を表す特殊変数
$ echo $$
12346
```

</details>
