# 第6章 演習：配列と連想配列

---

## 演習の目標

この演習では、以下のスキルを確認します。

- 配列を宣言し、要素の追加・参照・削除ができる
- 配列とループを組み合わせて反復処理ができる
- 連想配列（associative array）を使ってキーと値のペアを管理できる
- 配列の長さ取得、スライス、検索などの操作ができる
- 実践的な場面で配列を活用できる

---

## 基本問題

### 問題1：配列の基本操作

以下の要件を満たすスクリプトを作成してください。

1. 果物の名前を5つ格納した配列 `fruits` を作成する
2. 配列の全要素を表示する
3. 配列の要素数を表示する
4. インデックス0番目と2番目の要素を個別に表示する
5. 最後の要素を表示する

**期待される出力例：**

```
$ bash array_basic.sh
全要素: りんご みかん ぶどう バナナ いちご
要素数: 5
0番目: りんご
2番目: ぶどう
最後の要素: いちご
```

<details>
<summary>ヒント</summary>

- 配列は `array=(要素1 要素2 ...)` で宣言します
- 全要素は `${array[@]}` で展開します
- 要素数は `${#array[@]}` で取得します
- 個別の要素は `${array[インデックス]}` で参照します
- 最後の要素は `${array[-1]}`（Bash 4.2以降）で取得できます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 配列の宣言（スペース区切りで要素を列挙）
fruits=("りんご" "みかん" "ぶどう" "バナナ" "いちご")

# 全要素を表示（@ で全要素を展開）
echo "全要素: ${fruits[@]}"

# 要素数を表示（# で長さを取得）
echo "要素数: ${#fruits[@]}"

# インデックス指定で個別の要素を表示（0始まり）
echo "0番目: ${fruits[0]}"
echo "2番目: ${fruits[2]}"

# 最後の要素を表示（Bash 4.2以降で負のインデックスが使える）
echo "最後の要素: ${fruits[-1]}"
```

</details>

### 問題2：配列の追加・削除・置換

以下の操作を順番に行うスクリプトを作成してください。

1. `colors=("赤" "青" "緑")` で配列を初期化する
2. 末尾に「黄」を追加する
3. 末尾に「紫」と「白」を一度に追加する
4. インデックス1番目の要素（「青」）を「水色」に置換する
5. インデックス2番目の要素（「緑」）を削除する
6. 各操作の後に配列の状態を表示する

**期待される出力例：**

```
$ bash array_modify.sh
初期状態: 赤 青 緑
追加後: 赤 青 緑 黄
複数追加後: 赤 青 緑 黄 紫 白
置換後: 赤 水色 緑 黄 紫 白
削除後: 赤 水色 黄 紫 白
```

<details>
<summary>ヒント</summary>

- 末尾への追加は `array+=("要素")` で行います
- 要素の置換は `array[インデックス]="新しい値"` で行います
- 要素の削除は `unset 'array[インデックス]'` の後、`array=("${array[@]}")` で再構築が必要です

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 配列の初期化
colors=("赤" "青" "緑")
echo "初期状態: ${colors[@]}"

# 末尾に1つ追加（+= で配列に要素を追加）
colors+=("黄")
echo "追加後: ${colors[@]}"

# 末尾に複数追加（括弧内にスペース区切りで列挙）
colors+=("紫" "白")
echo "複数追加後: ${colors[@]}"

# インデックス1番目を置換（直接代入で上書き）
colors[1]="水色"
echo "置換後: ${colors[@]}"

# インデックス2番目を削除
# unset で要素を消すと歯抜けになるため、再構築が必要
unset 'colors[2]'
colors=("${colors[@]}")  # インデックスを詰め直す
echo "削除後: ${colors[@]}"
```

</details>

### 問題3：配列とループ

以下の要件を満たすスクリプトを作成してください。

1. サーバー名の配列 `servers=("web01" "web02" "db01" "db02" "cache01")` を定義する
2. `for` ループで全サーバー名を1行ずつ表示する
3. インデックス付きで全サーバーを表示する（例：`[0] web01`）
4. 「db」で始まるサーバーだけをフィルタリングして表示する

**期待される出力例：**

```
$ bash array_loop.sh
=== 全サーバー一覧 ===
web01
web02
db01
db02
cache01

=== インデックス付き一覧 ===
[0] web01
[1] web02
[2] db01
[3] db02
[4] cache01

=== DBサーバーのみ ===
db01
db02
```

<details>
<summary>ヒント</summary>

- `for item in "${array[@]}"; do ... done` で全要素をループできます
- インデックス付きループには `for i in "${!array[@]}"` を使います（`!` でインデックス一覧を取得）
- 文字列の前方一致には `[[ "$str" == db* ]]` が使えます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# サーバー名の配列
servers=("web01" "web02" "db01" "db02" "cache01")

# 1. for-in ループで全要素を表示
# "${array[@]}" をダブルクォートで囲むことで、スペースを含む要素も正しく扱える
echo "=== 全サーバー一覧 ==="
for server in "${servers[@]}"; do
    echo "$server"
done

echo ""

# 2. インデックス付きで表示
# ${!array[@]} でインデックスの一覧を取得する
echo "=== インデックス付き一覧 ==="
for i in "${!servers[@]}"; do
    echo "[$i] ${servers[$i]}"
done

echo ""

# 3. 「db」で始まるサーバーをフィルタリング
# パターンマッチング（== db*）で前方一致を判定
echo "=== DBサーバーのみ ==="
for server in "${servers[@]}"; do
    if [[ "$server" == db* ]]; then
        echo "$server"
    fi
done
```

</details>

### 問題4：連想配列の基本

以下の要件を満たすスクリプトを作成してください。

1. 連想配列 `user` を宣言し、以下のキーと値を設定する
   - `name` → `"田中太郎"`
   - `age` → `"30"`
   - `email` → `"tanaka@example.com"`
   - `role` → `"エンジニア"`
2. 特定のキーの値を個別に表示する
3. 全キーと全値を表示する
4. キーと値のペアを一覧表示する
5. キーの存在チェックを行う

**期待される出力例：**

```
$ bash assoc_basic.sh
名前: 田中太郎
メール: tanaka@example.com

=== 全キー ===
name age email role

=== 全値 ===
田中太郎 30 tanaka@example.com エンジニア

=== キーと値のペア ===
name: 田中太郎
age: 30
email: tanaka@example.com
role: エンジニア

=== キーの存在チェック ===
キー 'name' は存在します
キー 'phone' は存在しません
```

<details>
<summary>ヒント</summary>

- 連想配列は `declare -A array` で宣言します
- 値の設定は `array[キー]="値"` で行います
- 全キーは `${!array[@]}`、全値は `${array[@]}` で取得します
- キーの存在チェックには `${array[キー]+exists}` という書き方が使えます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 連想配列の宣言（declare -A が必須）
declare -A user

# キーと値の設定
user[name]="田中太郎"
user[age]="30"
user[email]="tanaka@example.com"
user[role]="エンジニア"

# 特定のキーの値を表示
echo "名前: ${user[name]}"
echo "メール: ${user[email]}"

echo ""

# 全キーを表示（${!array[@]} でキーの一覧を展開）
echo "=== 全キー ==="
echo "${!user[@]}"

echo ""

# 全値を表示（${array[@]} で値の一覧を展開）
echo "=== 全値 ==="
echo "${user[@]}"

echo ""

# キーと値のペアを一覧表示
echo "=== キーと値のペア ==="
for key in "${!user[@]}"; do
    echo "${key}: ${user[$key]}"
done

echo ""

# キーの存在チェック
# ${array[キー]+exists} は、キーが存在すれば "exists" に展開される
echo "=== キーの存在チェック ==="
if [[ -n "${user[name]+exists}" ]]; then
    echo "キー 'name' は存在します"
else
    echo "キー 'name' は存在しません"
fi

if [[ -n "${user[phone]+exists}" ]]; then
    echo "キー 'phone' は存在します"
else
    echo "キー 'phone' は存在しません"
fi
```

</details>

### 問題5：配列の検索とスライス

以下の要件を満たすスクリプトを作成してください。

1. 数値の配列 `numbers=(10 25 3 47 8 31 15 42)` を定義する
2. 配列から最大値と最小値を求める関数 `find_max` と `find_min` を作成する
3. 配列のスライス（部分配列）を表示する（インデックス2から3つ分）
4. 特定の値（`31`）が配列に含まれるか検索する関数 `array_contains` を作成する

**期待される出力例：**

```
$ bash array_search.sh
配列: 10 25 3 47 8 31 15 42
最大値: 47
最小値: 3
スライス [2:3]: 3 47 8
31 は配列に含まれています
99 は配列に含まれていません
```

<details>
<summary>ヒント</summary>

- スライスは `${array[@]:開始位置:個数}` で取得します
- 最大値・最小値は配列をループしながら比較して求めます
- 検索は `for` ループで一致する要素があるか確認します
- 関数の引数に配列を渡すには `"${array[@]}"` を使います

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 配列の中から最大値を求める関数
find_max() {
    local max="$1"
    shift
    for num in "$@"; do
        if (( num > max )); then
            max="$num"
        fi
    done
    echo "$max"
}

# 配列の中から最小値を求める関数
find_min() {
    local min="$1"
    shift
    for num in "$@"; do
        if (( num < min )); then
            min="$num"
        fi
    done
    echo "$min"
}

# 配列に特定の値が含まれるか検索する関数
# 引数: $1=検索値, $2以降=配列の要素
array_contains() {
    local target="$1"
    shift
    for item in "$@"; do
        if [[ "$item" == "$target" ]]; then
            return 0  # 見つかった
        fi
    done
    return 1  # 見つからなかった
}

# 数値の配列
numbers=(10 25 3 47 8 31 15 42)

echo "配列: ${numbers[@]}"

# 最大値・最小値を求める
max=$(find_max "${numbers[@]}")
min=$(find_min "${numbers[@]}")
echo "最大値: ${max}"
echo "最小値: ${min}"

# スライス（部分配列の取得）
# ${array[@]:開始位置:個数} で部分配列を取得
slice=("${numbers[@]:2:3}")
echo "スライス [2:3]: ${slice[@]}"

# 値の検索
if array_contains 31 "${numbers[@]}"; then
    echo "31 は配列に含まれています"
else
    echo "31 は配列に含まれていません"
fi

if array_contains 99 "${numbers[@]}"; then
    echo "99 は配列に含まれています"
else
    echo "99 は配列に含まれていません"
fi
```

</details>

### 問題6：連想配列で単語の出現回数をカウント

文字列中の各単語の出現回数を連想配列でカウントするスクリプトを作成してください。

1. 以下の文字列を処理対象とする：`"apple banana apple cherry banana apple cherry cherry banana banana"`
2. 連想配列を使って各単語の出現回数を数える
3. 結果を「単語: 回数」の形式で表示する

**期待される出力例：**

```
$ bash word_count.sh
=== 単語の出現回数 ===
apple: 3
banana: 4
cherry: 3
```

<details>
<summary>ヒント</summary>

- 文字列を配列に変換するには `read -ra array <<< "$string"` が使えます
- 連想配列の値を算術インクリメントするには `(( array[key]++ ))` が使えます
- 連想配列の初期値は空文字列ですが、算術式では 0 として扱われます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 処理対象の文字列
text="apple banana apple cherry banana apple cherry cherry banana banana"

# 連想配列を宣言
declare -A word_count

# 文字列を配列に変換
# read -ra で文字列をスペース区切りで配列に読み込む
read -ra words <<< "$text"

# 各単語の出現回数をカウント
# (( )) 内では未定義の変数は 0 として扱われる
for word in "${words[@]}"; do
    (( word_count[$word]++ ))
done

# 結果を表示
echo "=== 単語の出現回数 ==="
for key in "${!word_count[@]}"; do
    echo "${key}: ${word_count[$key]}"
done
```

</details>

### 問題7：配列を使った簡易タスクマネージャー

配列を使って、以下の機能を持つ簡易タスクマネージャーを作成してください。

- `add_task "タスク名"` : タスクを追加する
- `remove_task 番号` : 指定番号のタスクを削除する
- `list_tasks` : 全タスクを番号付きで一覧表示する
- `count_tasks` : タスクの総数を表示する

スクリプト内でいくつかのタスクを追加・削除してデモ動作させてください。

**期待される出力例：**

```
$ bash task_manager.sh
タスク追加: レポートを書く
タスク追加: メールを送る
タスク追加: ミーティング準備
タスク追加: コードレビュー

=== タスク一覧 (4件) ===
  [1] レポートを書く
  [2] メールを送る
  [3] ミーティング準備
  [4] コードレビュー

タスク削除: [2] メールを送る

=== タスク一覧 (3件) ===
  [1] レポートを書く
  [2] ミーティング準備
  [3] コードレビュー
```

<details>
<summary>ヒント</summary>

- グローバル配列を複数の関数から操作する設計にします
- 削除後はインデックスを詰め直す（`array=("${array[@]}")`）と管理が楽です
- 表示用の番号はインデックス+1にすると直感的です
- 削除時は範囲チェックを忘れずに行いましょう

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# タスクを格納するグローバル配列
tasks=()

# タスクを追加する関数
add_task() {
    local task_name="$1"
    if [[ -z "$task_name" ]]; then
        echo "エラー: タスク名を指定してください" >&2
        return 1
    fi
    tasks+=("$task_name")
    echo "タスク追加: ${task_name}"
}

# 指定番号のタスクを削除する関数（表示番号=インデックス+1）
remove_task() {
    local display_num="$1"
    local index=$(( display_num - 1 ))

    # 範囲チェック
    if (( index < 0 || index >= ${#tasks[@]} )); then
        echo "エラー: 無効な番号です" >&2
        return 1
    fi

    echo "タスク削除: [${display_num}] ${tasks[$index]}"
    unset 'tasks[$index]'
    # インデックスを詰め直す（歯抜けを防ぐ）
    tasks=("${tasks[@]}")
}

# 全タスクを一覧表示する関数
list_tasks() {
    local count=${#tasks[@]}
    echo ""
    echo "=== タスク一覧 (${count}件) ==="

    if (( count == 0 )); then
        echo "  （タスクはありません）"
        return
    fi

    for i in "${!tasks[@]}"; do
        echo "  [$(( i + 1 ))] ${tasks[$i]}"
    done
}

# タスク数を表示する関数
count_tasks() {
    echo "タスク数: ${#tasks[@]}件"
}

# === デモ動作 ===
add_task "レポートを書く"
add_task "メールを送る"
add_task "ミーティング準備"
add_task "コードレビュー"

list_tasks

echo ""
remove_task 2

list_tasks
```

</details>

### 問題8：連想配列で設定ファイルをパース

以下の形式の設定文字列を連想配列にパースするスクリプトを作成してください。

```
# データベース設定
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_USER=admin

# アプリケーション設定
APP_PORT=8080
APP_ENV=production
```

1. 上記の設定文字列（ヒアドキュメントで定義）を1行ずつ読み取る
2. コメント行（`#` で始まる行）と空行をスキップする
3. `キー=値` の形式をパースして連想配列に格納する
4. 格納した設定を一覧表示する
5. 特定のキー（`DB_HOST` と `APP_PORT`）の値を取得して表示する

**期待される出力例：**

```
$ bash parse_config.sh
=== 設定値一覧 ===
DB_HOST = localhost
DB_PORT = 5432
DB_NAME = myapp
DB_USER = admin
APP_PORT = 8080
APP_ENV = production

=== 個別取得 ===
DB_HOST: localhost
APP_PORT: 8080

設定項目数: 6
```

<details>
<summary>ヒント</summary>

- ヒアドキュメント `<<'EOF'` でシングルクォートを使うと変数展開を抑制できます
- `while IFS= read -r line` で1行ずつ読み取ります
- 行頭の `#` 判定は `[[ "$line" == \#* ]]` で行えます
- `=` での分割は `${line%%=*}`（キー）と `${line#*=}`（値）で行えます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 設定文字列をヒアドキュメントで定義
# 'EOF' とシングルクォートにすると変数展開が行われない
config_text=$(cat <<'EOF'
# データベース設定
DB_HOST=localhost
DB_PORT=5432
DB_NAME=myapp
DB_USER=admin

# アプリケーション設定
APP_PORT=8080
APP_ENV=production
EOF
)

# 連想配列を宣言
declare -A config

# 設定文字列を1行ずつ読み取ってパース
while IFS= read -r line; do
    # 空行をスキップ（-z は文字列が空かチェック）
    [[ -z "$line" ]] && continue

    # コメント行をスキップ（# で始まる行）
    [[ "$line" == \#* ]] && continue

    # = で分割してキーと値を取得
    # ${line%%=*} : 最初の = より前（キー）
    # ${line#*=}  : 最初の = より後（値）
    key="${line%%=*}"
    value="${line#*=}"

    # 連想配列に格納
    config["$key"]="$value"
done <<< "$config_text"

# 設定値を一覧表示
echo "=== 設定値一覧 ==="
for key in "${!config[@]}"; do
    echo "${key} = ${config[$key]}"
done

echo ""

# 特定のキーの値を取得
echo "=== 個別取得 ==="
echo "DB_HOST: ${config[DB_HOST]}"
echo "APP_PORT: ${config[APP_PORT]}"

echo ""

# 設定項目数
echo "設定項目数: ${#config[@]}"
```

</details>
