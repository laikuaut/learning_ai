> **難易度**: ★★★★★（総合）

> **課題の種類**: ミニプロジェクト（総合）

## 課題の要件

### 必須要件

1. **4つのルート**を設定する
   - `/dashboard` : 統計ダッシュボード
   - `/employees` : 社員一覧（検索・フィルタ付き）
   - `/employees/new` : 新規社員登録
   - `/employees/:id` : 社員詳細・編集
2. **サービス層**を実装する
   - `EmployeeService` : 社員データのCRUD操作
   - `StatsService` : 統計データの計算
3. **コンポーネント（またはディレクティブ）**を3つ以上作成する
   - 統計カード（`<stat-card>`）
   - 社員リストアイテム（`<employee-row>`）
   - スキルタグ入力（`<skill-input>`）
4. **フォームバリデーション**を実装する
   - 必須チェック、メール形式チェック、文字数制限
   - カスタムバリデータ（メールアドレスの重複チェック）
5. **カスタムフィルター**を2つ以上作成する
   - 勤続年数計算フィルター
   - スキル検索フィルター
6. **Controller As** 構文を使用する
7. 初期データとして8人以上の社員データを持つ

### 発展要件（余力があれば）

- 社員データのソート機能（名前順・入社年順・部署順）
- ページネーション（1ページ5件）
- 社員の削除確認ダイアログ
- ダッシュボードに棒グラフ風の表示
- データのバリデーション結果をリアルタイムに表示

<summary>ステップ5：フォームバリデーションを実装する</summary>

```javascript
// カスタムバリデータ: メール重複チェック
app.directive('uniqueEmail', ['EmployeeService', function(EmployeeService) {
  return {
    require: 'ngModel',
    scope: { excludeId: '=' },
    link: function(scope, elem, attrs, ngModel) {
      ngModel.$asyncValidators.uniqueEmail = function(modelValue) {
        // 同期チェックで十分だが、$asyncValidators の例として
        return new Promise(function(resolve, reject) {
          if (EmployeeService.isEmailUnique(modelValue, scope.excludeId)) {
            resolve();
          } else {
            reject();
          }
        });
      };
    }
  };
}]);
```

改良版では以下の点をさらに追加します。初心者向けコードをベースに、以下の変更を加えてください。

### 1. ページネーションの追加

社員一覧コントローラに以下を追加します。

```javascript
vm.page = 1;
vm.perPage = 5;

vm.pagedEmployees = function() {
  var filtered = vm.filtered();
  var sorted = orderByFilter(filtered, vm.sort);
  var start = (vm.page - 1) * vm.perPage;
  return sorted.slice(start, start + vm.perPage);
};

vm.totalPages = function() {
  return Math.ceil(vm.filtered().length / vm.perPage);
};

vm.nextPage = function() {
  if (vm.page < vm.totalPages()) vm.page++;
};

vm.prevPage = function() {
  if (vm.page > 1) vm.page--;
};
```

コントローラに `orderByFilter` を注入します:
```javascript
app.controller('EmployeeListCtrl', ['$location', 'EmployeeService', 'orderByFilter',
  function($location, ES, orderByFilter) { /* ... */ }
]);
```

### 2. ワンタイムバインディング

変更されないデータにはワンタイムバインディング `::` を使います:

```html
<!-- 社員一覧で名前やメールは変わらないのでワンタイム -->
<div class="emp-name">{{ ::emp.name }}</div>
<div class="emp-dept">{{ ::emp.department }} ・ {{ ::emp.role }}</div>
```

### 3. resolve でデータ事前読み込み

```javascript
.when('/employees/:id', {
  templateUrl: 'employee-detail.html',
  controller: 'EmployeeDetailCtrl',
  controllerAs: 'vm',
  resolve: {
    employee: ['$route', 'EmployeeService', function($route, ES) {
      return ES.getById(parseInt($route.current.params.id));
    }]
  }
})
```

コントローラで `resolve` の値を注入:
```javascript
app.controller('EmployeeDetailCtrl', ['$location', 'EmployeeService', 'employee',
  function($location, ES, employee) {
    var vm = this;
    vm.emp = employee;
    // ...
  }
]);
```

**初心者向けとの違い:**
- ページネーションで大量データに対応（`orderByFilter` をプログラム的に使用）
- ワンタイムバインディング `::` でパフォーマンス向上
- `resolve` でルート遷移前にデータを読み込み、存在しない場合のハンドリングが容易に
- `track by` で `ng-repeat` のDOM再利用を促進
