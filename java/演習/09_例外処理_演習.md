## 基本問題

### 問題1：安全な割り算

2つの整数を受け取り、割り算を行うメソッドを作成してください。ゼロ除算の場合は `ArithmeticException` をキャッチして適切なメッセージを表示してください。

**期待される出力例：**
```
10 ÷ 3 = 3
10 ÷ 0 = エラー: ゼロで割ることはできません
```

<details>
<summary>ヒント</summary>

- `try-catch` で `ArithmeticException` をキャッチします
- 整数の割り算でゼロ除算すると `ArithmeticException` が発生します

</details>

<details>
<summary>解答例</summary>

```java
public class SafeDivision {
    /* ゼロ除算を安全に処理するメソッド */
    static void divide(int a, int b) {
        try {
            int result = a / b;
            System.out.println(a + " ÷ " + b + " = " + result);
        } catch (ArithmeticException e) {
            System.out.println(a + " ÷ " + b + " = エラー: ゼロで割ることはできません");
        }
    }

    public static void main(String[] args) {
        divide(10, 3);
        divide(10, 0);
    }
}
```

</details>

### 問題2：数値変換の例外処理

文字列を整数に変換するメソッドを作成してください。変換できない場合は `NumberFormatException` をキャッチしてデフォルト値を返してください。

**期待される出力例：**
```
"123" → 123
"abc" → 0（変換失敗：デフォルト値を使用）
"456" → 456
"12.3" → 0（変換失敗：デフォルト値を使用）
```

<details>
<summary>ヒント</summary>

- `Integer.parseInt()` は数値に変換できない文字列で `NumberFormatException` を投げます
- デフォルト値（0）を返すようにします

</details>

<details>
<summary>解答例</summary>

```java
public class SafeParser {
    /* 文字列を安全に整数に変換するメソッド */
    static int safeParseInt(String str, int defaultValue) {
        try {
            return Integer.parseInt(str);
        } catch (NumberFormatException e) {
            System.out.printf("\"%s\" → %d（変換失敗：デフォルト値を使用）%n", str, defaultValue);
            return defaultValue;
        }
    }

    public static void main(String[] args) {
        String[] inputs = {"123", "abc", "456", "12.3"};

        for (String input : inputs) {
            int value = safeParseInt(input, 0);
            if (value != 0 || input.equals("0")) {
                System.out.printf("\"%s\" → %d%n", input, value);
            }
        }
    }
}
```

</details>

### 問題3：配列の安全なアクセス

配列のインデックスを指定して要素を取得するメソッドを作成してください。範囲外アクセスの場合は `ArrayIndexOutOfBoundsException` をキャッチして処理してください。

**期待される出力例：**
```
scores[0] = 85
scores[2] = 92
scores[5] = エラー: インデックス5は範囲外です（配列サイズ: 4）
scores[-1] = エラー: インデックス-1は範囲外です（配列サイズ: 4）
```

<details>
<summary>ヒント</summary>

- 配列の範囲外にアクセスすると `ArrayIndexOutOfBoundsException` が発生します
- 配列の長さは `.length` で取得できます

</details>

<details>
<summary>解答例</summary>

```java
public class SafeArrayAccess {
    /* 安全に配列要素を取得するメソッド */
    static void getElement(int[] arr, int index) {
        try {
            System.out.println("scores[" + index + "] = " + arr[index]);
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.printf("scores[%d] = エラー: インデックス%dは範囲外です（配列サイズ: %d）%n",
                index, index, arr.length);
        }
    }

    public static void main(String[] args) {
        int[] scores = {85, 78, 92, 88};

        getElement(scores, 0);
        getElement(scores, 2);
        getElement(scores, 5);
        getElement(scores, -1);
    }
}
```

</details>

## 応用問題

### 問題4：カスタム例外 - InvalidAgeException

年齢のバリデーションを行うカスタム例外 `InvalidAgeException` を作成してください。年齢が0未満または150超の場合に例外を投げてください。

**期待される出力例：**
```
年齢 25 は有効です。
年齢 -5 は無効です: 年齢は0〜150の範囲で入力してください（入力値: -5）
年齢 200 は無効です: 年齢は0〜150の範囲で入力してください（入力値: 200）
年齢 0 は有効です。
```

<details>
<summary>ヒント</summary>

- `Exception` を継承してカスタム例外クラスを作ります
- バリデーションメソッドで条件を満たさない場合に `throw new InvalidAgeException(...)` します
- 呼び出し側で `try-catch` でキャッチします

</details>

<details>
<summary>解答例</summary>

```java
/* カスタム例外クラス */
class InvalidAgeException extends Exception {
    int age;

    InvalidAgeException(int age) {
        super("年齢は0〜150の範囲で入力してください（入力値: " + age + "）");
        this.age = age;
    }
}

public class AgeValidator {
    /* 年齢を検証するメソッド */
    static void validateAge(int age) throws InvalidAgeException {
        if (age < 0 || age > 150) {
            throw new InvalidAgeException(age);
        }
        System.out.println("年齢 " + age + " は有効です。");
    }

    public static void main(String[] args) {
        int[] testAges = {25, -5, 200, 0};

        for (int age : testAges) {
            try {
                validateAge(age);
            } catch (InvalidAgeException e) {
                System.out.println("年齢 " + age + " は無効です: " + e.getMessage());
            }
        }
    }
}
```

</details>

### 問題5：複数の例外をキャッチ

ユーザー入力をシミュレーションし、以下の例外を適切に処理する計算プログラムを作成してください。
- `NumberFormatException`（数値変換エラー）
- `ArithmeticException`（ゼロ除算）
- `ArrayIndexOutOfBoundsException`（引数不足）

**期待される出力例：**
```
計算: 10 / 2 = 5
計算: 10 / 0 → 算術エラー: / by zero
計算: abc / 2 → 変換エラー: "abc"は数値ではありません
引数不足エラー: 2つの数値を指定してください
```

<details>
<summary>ヒント</summary>

- 複数の `catch` ブロックを使います
- `catch` は具体的な例外から順に書きます
- `finally` ブロックで後処理を行えます

</details>

<details>
<summary>解答例</summary>

```java
public class MultiCatch {
    /* 文字列引数から計算を実行 */
    static void calculate(String[] args) {
        try {
            String a = args[0];
            String b = args[1];
            int num1 = Integer.parseInt(a);
            int num2 = Integer.parseInt(b);
            int result = num1 / num2;
            System.out.println("計算: " + a + " / " + b + " = " + result);
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("引数不足エラー: 2つの数値を指定してください");
        } catch (NumberFormatException e) {
            System.out.println("計算: " + args[0] + " / " + args[1]
                + " → 変換エラー: \"" + args[0] + "\"は数値ではありません");
        } catch (ArithmeticException e) {
            System.out.println("計算: " + args[0] + " / " + args[1]
                + " → 算術エラー: " + e.getMessage());
        }
    }

    public static void main(String[] args) {
        /* さまざまなケースをテスト */
        calculate(new String[]{"10", "2"});
        calculate(new String[]{"10", "0"});
        calculate(new String[]{"abc", "2"});
        calculate(new String[]{});
    }
}
```

</details>

### 問題6：try-with-resources の実践

ファイルにデータを書き込み、その後読み込んで表示するプログラムを作成してください。`try-with-resources` を使ってリソースを自動的に閉じてください。

**期待される出力例：**
```
ファイルに書き込み中...
書き込み完了！

ファイルから読み込み中...
1: Java例外処理の練習
2: try-with-resourcesを使用
3: リソースは自動的に閉じられます
読み込み完了！
```

<details>
<summary>ヒント</summary>

- `BufferedWriter` で書き込み、`BufferedReader` で読み込みます
- どちらも `try-with-resources` で囲みます
- `java.io.*` をインポートします

</details>

<details>
<summary>解答例</summary>

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.FileReader;
import java.io.FileWriter;
import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Path;

public class TryWithResources {
    public static void main(String[] args) {
        String filename = "try_test.txt";

        /* 書き込み */
        System.out.println("ファイルに書き込み中...");
        try (BufferedWriter writer = new BufferedWriter(new FileWriter(filename))) {
            writer.write("Java例外処理の練習");
            writer.newLine();
            writer.write("try-with-resourcesを使用");
            writer.newLine();
            writer.write("リソースは自動的に閉じられます");
            writer.newLine();
            System.out.println("書き込み完了！");
        } catch (IOException e) {
            System.err.println("書き込みエラー: " + e.getMessage());
            return;
        }

        /* 読み込み */
        System.out.println("\nファイルから読み込み中...");
        try (BufferedReader reader = new BufferedReader(new FileReader(filename))) {
            String line;
            int lineNum = 1;
            while ((line = reader.readLine()) != null) {
                System.out.println(lineNum + ": " + line);
                lineNum++;
            }
            System.out.println("読み込み完了！");
        } catch (IOException e) {
            System.err.println("読み込みエラー: " + e.getMessage());
        }

        /* クリーンアップ */
        try {
            Files.deleteIfExists(Path.of(filename));
        } catch (IOException e) {
            /* 無視 */
        }
    }
}
```

</details>

## チャレンジ問題

### 問題7：堅牢な電卓プログラム

4つの演算（+, -, *, /）に対応した電卓プログラムを作成してください。以下の例外をすべて適切に処理してください。

- 数値変換エラー
- ゼロ除算エラー
- 不正な演算子エラー（カスタム例外 `InvalidOperatorException`）

**期待される出力例：**
```
=== 電卓プログラム ===
10 + 5 = 15.00
10 - 3 = 7.00
10 * 4 = 40.00
10 / 3 = 3.33
10 / 0 → エラー: ゼロで割ることはできません
10 % 3 → エラー: 不正な演算子です: %
abc + 5 → エラー: 数値の形式が不正です: abc
```

<details>
<summary>ヒント</summary>

- カスタム例外 `InvalidOperatorException` を `Exception` の子クラスとして定義します
- `switch` 文で演算子を判定し、不正な場合にカスタム例外を投げます
- `double` で計算して精度を保ちます

</details>

<details>
<summary>解答例</summary>

```java
/* カスタム例外：不正な演算子 */
class InvalidOperatorException extends Exception {
    InvalidOperatorException(String operator) {
        super("不正な演算子です: " + operator);
    }
}

public class RobustCalculator {
    /* 計算を実行するメソッド */
    static double calculate(String numStr1, String operator, String numStr2)
            throws InvalidOperatorException {
        double a = Double.parseDouble(numStr1);
        double b = Double.parseDouble(numStr2);

        switch (operator) {
            case "+": return a + b;
            case "-": return a - b;
            case "*": return a * b;
            case "/":
                if (b == 0) {
                    throw new ArithmeticException("ゼロで割ることはできません");
                }
                return a / b;
            default:
                throw new InvalidOperatorException(operator);
        }
    }

    /* 計算を安全に実行して結果を表示 */
    static void safeCalculate(String a, String op, String b) {
        try {
            double result = calculate(a, op, b);
            System.out.printf("%s %s %s = %.2f%n", a, op, b, result);
        } catch (NumberFormatException e) {
            System.out.printf("%s %s %s → エラー: 数値の形式が不正です: %s%n", a, op, b, a);
        } catch (ArithmeticException e) {
            System.out.printf("%s %s %s → エラー: %s%n", a, op, b, e.getMessage());
        } catch (InvalidOperatorException e) {
            System.out.printf("%s %s %s → エラー: %s%n", a, op, b, e.getMessage());
        }
    }

    public static void main(String[] args) {
        System.out.println("=== 電卓プログラム ===");
        safeCalculate("10", "+", "5");
        safeCalculate("10", "-", "3");
        safeCalculate("10", "*", "4");
        safeCalculate("10", "/", "3");
        safeCalculate("10", "/", "0");
        safeCalculate("10", "%", "3");
        safeCalculate("abc", "+", "5");
    }
}
```

</details>

### 問題8：例外チェーンと再スロー

データ処理パイプラインをシミュレーションし、低レベルの例外をキャッチしてより意味のあるカスタム例外に包んで再スローする仕組みを実装してください。

**期待される出力例：**
```
データ処理開始: "42"
  → 変換成功: 42
  → 処理成功: 84

データ処理開始: "invalid"
  → データ処理エラー: 入力データの変換に失敗しました
  → 原因: For input string: "invalid"
```

<details>
<summary>ヒント</summary>

- カスタム例外のコンストラクタで `cause`（原因となった例外）を受け取ります
- `new CustomException(message, cause)` の形で例外チェーンを構築します
- `getCause()` で元の例外を取得できます

</details>

<details>
<summary>解答例</summary>

```java
/* カスタム例外（例外チェーン対応） */
class DataProcessingException extends Exception {
    DataProcessingException(String message, Throwable cause) {
        super(message, cause);
    }
}

public class ExceptionChain {
    /* データ変換処理 */
    static int parseData(String input) throws DataProcessingException {
        try {
            return Integer.parseInt(input);
        } catch (NumberFormatException e) {
            /* 低レベルの例外を包んで再スロー */
            throw new DataProcessingException("入力データの変換に失敗しました", e);
        }
    }

    /* データ処理パイプライン */
    static void processData(String input) {
        System.out.println("データ処理開始: \"" + input + "\"");
        try {
            int value = parseData(input);
            System.out.println("  → 変換成功: " + value);
            int processed = value * 2;
            System.out.println("  → 処理成功: " + processed);
        } catch (DataProcessingException e) {
            System.out.println("  → データ処理エラー: " + e.getMessage());
            System.out.println("  → 原因: " + e.getCause().getMessage());
        }
    }

    public static void main(String[] args) {
        processData("42");
        System.out.println();
        processData("invalid");
    }
}
```

</details>
