## 基本問題

### 問題1: マクロの記録と実行

以下のテキストの各行の末尾にセミコロン（`;`）を追加するマクロを記録・実行してください。マクロの記録開始から終了まで、入力するキー操作を全て答えてください。

```
let a = 1
let b = 2
let c = 3
let d = 4
```

期待される結果：
```
let a = 1;
let b = 2;
let c = 3;
let d = 4;
```

<details>
<summary>解答</summary>

```
1. 1行目にカーソルを置く
2. qa           ← レジスタ a にマクロ記録開始
3. A;           ← 行末に移動して ; を挿入
4. ESC          ← ノーマルモードに戻る
5. j            ← 次の行に移動
6. q            ← マクロ記録終了
7. 3@a          ← 残り3行にマクロを適用

記録されるマクロ: A;[ESC]j
```

- `qa` でレジスタ a にマクロの記録を開始します
- マクロは「次の行に移動する」（`j`）で終わるように設計すると、繰り返し実行しやすいです
- `3@a` で3回繰り返し実行します
- `@@` で直前のマクロを再実行することもできます

</details>

### 問題2: レジスタ操作

以下の操作で使うレジスタに関する問題です。

a) `yy` でヤンクした内容が保存されるレジスタ名を2つ答えてください
b) `dd` で削除した後に、以前ヤンクした内容を貼り付けたい場合、使うコマンドは？
c) 削除した内容をどのレジスタにも保存したくない場合のコマンドは？（`dd` の代わり）

<details>
<summary>解答</summary>

```
a) "" （無名レジスタ）と "0 （ヤンクレジスタ）
   yy の結果は両方に保存されます

b) "0p
   "0（ヤンクレジスタ）は削除操作では上書きされないため、
   dd した後でも以前にヤンクした内容を貼り付けられます

c) "_dd
   "_（ブラックホールレジスタ）に削除すると、
   どのレジスタも汚しません
```

- 無名レジスタ `""` は `d`, `y`, `c`, `s` の全ての結果が入ります
- ヤンクレジスタ `"0` は `y` の結果のみが入り、`d` では上書きされません
- ブラックホールレジスタ `"_` は削除した内容を破棄します

</details>

### 問題3: マークとジャンプ

以下の各シナリオで使うコマンドを答えてください。

a) 現在のカーソル位置にマーク `a` を設定する
b) マーク `a` の正確な位置にジャンプする
c) 最後に変更を行った位置にジャンプする
d) 直前のジャンプ位置に戻る

<details>
<summary>解答</summary>

```
a) ma          ← 現在位置にマーク a を設定
b) `a          ← マーク a の正確な位置にジャンプ
               ← 'a だとマーク a の行の先頭にジャンプ
c) `.          ← 最後に変更した位置にジャンプ
               ← '. だと最後に変更した行の先頭
d) ``          ← 直前のジャンプ位置に戻る
               ← '' だと直前のジャンプ行の先頭
   または Ctrl+o でジャンプリストを戻る
```

- `` ` ``（バッククォート）は正確な位置、`'`（シングルクォート）は行の先頭にジャンプします
- `Ctrl+o` / `Ctrl+i` でジャンプ履歴を行き来できます
- `:marks` でマーク一覧を確認できます

</details>

## 応用問題

### 問題4: マクロを使ったCSVデータ整形

以下のCSVデータをマクロを使って Markdown テーブル形式に変換してください。マクロの記録手順を答えてください。

元データ：
```
Name,Age,City
Alice,30,Tokyo
Bob,25,Osaka
Charlie,35,Nagoya
```

期待される結果：
```
| Name | Age | City |
| Alice | 30 | Tokyo |
| Bob | 25 | Osaka |
| Charlie | 35 | Nagoya |
```

<details>
<summary>ヒント</summary>

各行に対して以下の操作を行うマクロを記録します:
1. 行頭に `| ` を追加
2. カンマを ` | ` に置換
3. 行末に ` |` を追加

</details>

<details>
<summary>解答</summary>

```
方法1: マクロを使う場合
1. 1行目にカーソルを置く
2. qa で記録開始
3. I| ESC       ← 行頭に | を挿入
4. :s/,/ | /g Enter  ← カンマを | に置換
5. A |ESC       ← 行末に | を追加
6. j0           ← 次の行の先頭に移動
7. q            ← 記録終了
8. 3@a          ← 残り3行に適用

方法2: 置換コマンドだけで一括処理
:%s/^/| /           ← 全行の先頭に | を追加
:%s/,/ | /g         ← 全カンマを | に置換
:%s/$/ |/           ← 全行の末尾に | を追加
```

- マクロは1行単位の処理を繰り返すのに適しています
- 全行に同じ処理をする場合は `:%s` の方が効率的です
- 実務ではどちらの方法も使えるようにしておくと便利です

</details>

### 問題5: 外部コマンドとの連携

以下の各操作を実現する Vim コマンドを答えてください。

a) 現在のバッファの内容を行数カウントして表示する（ファイルに変更を加えない）
b) 現在の日時をカーソル位置に挿入する
c) バッファ全体を JSON として整形する（Python3 が使える環境）
d) 選択した行をアルファベット順にソートする

<details>
<summary>解答</summary>

```vim
" a) 行数カウント
:w !wc -l
" バッファの内容を wc -l に渡し、結果を表示します
" ファイル自体は変更されません

" b) 現在の日時をカーソル位置に挿入
:r !date
" :r はコマンドの出力をカーソルの下の行に挿入します

" c) JSON 整形
:%!python3 -m json.tool
" :%! でバッファ全体を外部コマンドのフィルタに通します

" d) 選択行をソート
" V で行を選択した後:
:'<,'>!sort
" ビジュアル選択範囲に対して sort を実行します
" または Vim 内蔵の :sort コマンドも使えます:
:'<,'>sort
```

- `:w !コマンド` はバッファ内容をコマンドに渡す（書き込み先がコマンド）
- `:r !コマンド` はコマンド出力をバッファに読み込む
- `:%!コマンド` はバッファ全体をコマンドでフィルタ（置き換え）

</details>

## チャレンジ問題

### 問題6: 複雑なテキスト変換

以下のJSONデータをVimの機能（マクロ、置換、外部コマンドなど）を組み合わせて、環境変数の export 文に変換してください。複数の方法を考えて、最も効率的だと思う方法を答えてください。

元データ：
```json
{
  "DB_HOST": "localhost",
  "DB_PORT": "5432",
  "DB_NAME": "myapp",
  "API_KEY": "abc123",
  "DEBUG": "true"
}
```

期待される結果：
```bash
export DB_HOST="localhost"
export DB_PORT="5432"
export DB_NAME="myapp"
export API_KEY="abc123"
export DEBUG="true"
```

<details>
<summary>ヒント</summary>

- まず不要な行（`{` と `}`）を削除
- 各行のフォーマットを変換（`: ` を `=` に、先頭の空白や末尾のカンマを処理）
- 行頭に `export ` を追加

</details>

<details>
<summary>解答</summary>

```
方法1: 置換コマンドの組み合わせ（おすすめ）

1. 波括弧の行を削除
   :g/^[{}]/d

2. 前後の空白とカンマを処理して形式を変換
   :%s/^\s*"\(\w\+\)": "\(.*\)",\?$/export \1="\2"/

3. 完了！

方法2: ステップごとの置換

1. 波括弧の行を削除
   :g/^[{}]/d

2. 行頭の空白を削除
   :%s/^\s\+//

3. 最初のキー名のクォートを外す
   :%s/"\(\w\+\)": /\1=/

4. 末尾のカンマを削除
   :%s/,$//

5. 行頭に export を追加
   :%s/^/export /

方法3: マクロ + 手動操作

1. dd で { の行を削除
2. G dd で } の行を削除
3. 1行目にカーソルを置いて qa で記録開始
4. ^ で行頭の非空白文字へ → d0 で先頭の空白を削除
5. f" x で最初の " を削除 → f" x で次の " を削除
6. f: で : に移動 → cl= で : を = に変更 → ESC
7. $ でend → F, x で末尾のカンマがあれば削除
8. I export ESC で行頭に追加
9. j で次の行へ → q で記録終了
10. 4@a で残りの行に適用
```

- 方法1が最も効率的です（正規表現のキャプチャグループを活用）
- 方法2は理解しやすいステップバイステップの方法です
- 方法3はマクロの練習になりますが、この場合は置換の方が効率的です
- 実務では状況に応じて最適な方法を選びましょう

</details>
