# 実践課題04：検索設定の最適化 ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第1章（vimrcの基本）、第4章（検索と置換の設定）
> **課題の種類**: ミニプロジェクト
> **学習目標**: インクリメンタルサーチ・ハイライト・大文字小文字の扱いを設定し、効率的な検索環境を構築する

## 課題の要件

1. インクリメンタルサーチ（入力しながら検索）を有効にする
2. 検索結果をハイライト表示する
3. 大文字小文字を状況に応じて自動判定する（`smartcase`）
4. 検索ハイライトを簡単に消すキーマッピングを設定する
5. 検索がファイル末尾に達したら先頭に戻る設定にする
6. 置換時の便利な設定を追加する

---

## ステップガイド

<details>
<summary>ステップ1：インクリメンタルサーチの設定</summary>

```vim
" --- インクリメンタルサーチ ---
set incsearch           " 入力しながらリアルタイムで検索
```

`incsearch` を有効にすると、`/` で検索文字を1文字入力するたびにマッチ箇所にカーソルが移動します。目的の場所が見えた時点で `Enter` を押せばよいので、検索が格段に効率的になります。

</details>

<details>
<summary>ステップ2：検索結果のハイライト</summary>

```vim
" --- 検索ハイライト ---
set hlsearch            " 検索結果をハイライト表示
```

`hlsearch` はすべてのマッチ箇所を色付きで表示します。便利ですが、検索後にハイライトが残り続けるので、消すためのキーマッピングが必要です（ステップ4で設定）。

</details>

<details>
<summary>ステップ3：大文字・小文字の自動判定</summary>

```vim
" --- 大文字・小文字 ---
set ignorecase          " 検索時に大文字・小文字を区別しない
set smartcase           " ただし大文字を含む場合は区別する
```

この2つを組み合わせると:

| 検索パターン | 動作 | 理由 |
|-------------|------|------|
| `/hello` | 大文字小文字を無視 | すべて小文字なので |
| `/Hello` | 大文字小文字を区別 | 大文字 `H` を含むので |
| `/HELLO` | 大文字小文字を区別 | 大文字を含むので |

**`ignorecase` と `smartcase` は必ずセットで設定してください。** `smartcase` は `ignorecase` が有効なときだけ機能します。

</details>

<details>
<summary>ステップ4：ハイライトを消すキーマッピング</summary>

```vim
" --- ハイライト解除 ---
nnoremap <Esc><Esc> :nohlsearch<CR>
```

`Esc` を2回押すと検索ハイライトが消えます。`:nohlsearch` はハイライトを一時的に消すコマンドで、次の検索で再びハイライトされます。

よく使われる他のキーマッピング例:
```vim
" Ctrl+L でハイライト解除（画面再描画と兼用）
" nnoremap <C-l> :nohlsearch<CR><C-l>
```

</details>

<details>
<summary>ステップ5：検索のループと置換設定</summary>

```vim
" --- 検索のループ ---
set wrapscan            " ファイル末尾に達したら先頭に戻って検索

" --- 置換の便利設定 ---
set gdefault            " 置換時に g フラグをデフォルトにする
```

`gdefault` を設定すると、`:%s/foo/bar/` だけで行内の全マッチが置換されます（通常は `:%s/foo/bar/g` と `g` が必要）。

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

```vim
" ===========================================
" 検索と置換の設定
" ===========================================

set nocompatible

" --- インクリメンタルサーチ ---
set incsearch

" --- 検索ハイライト ---
set hlsearch

" --- 大文字・小文字の扱い ---
set ignorecase
set smartcase

" --- 検索のループ ---
set wrapscan

" --- ハイライト解除のキーマッピング ---
nnoremap <Esc><Esc> :nohlsearch<CR>
```

</details>

<details>
<summary>解答例（改良版：置換プレビューと検索強化）</summary>

```vim
" ===========================================
" 検索と置換の設定 - 改良版
" ===========================================

set nocompatible

" --- 基本検索設定 ---
set incsearch           " インクリメンタルサーチ
set hlsearch            " 検索ハイライト
set ignorecase          " 大文字小文字を無視
set smartcase           " 大文字を含む場合は区別
set wrapscan            " ファイル末尾で先頭に戻る

" --- 置換設定 ---
set gdefault            " 置換のgフラグをデフォルトに

" --- ハイライト解除 ---
nnoremap <Esc><Esc> :nohlsearch<CR>

" --- 検索中のカーソル位置を画面中央に ---
nnoremap n nzz          " 次の検索結果を画面中央に
nnoremap N Nzz          " 前の検索結果を画面中央に
nnoremap * *zz          " カーソル下の単語検索（中央）
nnoremap # #zz          " カーソル下の単語検索・逆方向（中央）

" --- ビジュアルモードで選択範囲を検索 ---
vnoremap * "zy/\V<C-r>z<CR>
vnoremap # "zy?\V<C-r>z<CR>

" --- 検索結果の件数を表示 ---
nnoremap <Leader>c :%s///gn<CR>
```

**初心者向けとの違い**:

- `nzz`, `Nzz` で検索結果を常に画面中央に表示（見失わない）
- `*zz`, `#zz` でカーソル下の単語検索でも中央表示
- ビジュアルモードで選択したテキストをそのまま検索可能に
- `<Leader>c` で現在の検索パターンの一致件数を表示

</details>

## 確認方法

1. **インクリメンタルサーチ**: `/` を押して文字を入力し、1文字ごとにカーソルが移動することを確認
2. **ハイライト**: 検索確定後、すべてのマッチ箇所が色付きになることを確認
3. **smartcase**: `/hello` で大文字小文字無視、`/Hello` で区別されることを確認
4. **ハイライト解除**: `Esc` を2回押してハイライトが消えることを確認

---

## よくある間違い

| ミス | 正しい設定 | 説明 |
|------|-----------|------|
| `smartcase` だけ設定して `ignorecase` を忘れる | 両方設定する | `smartcase` は `ignorecase` が前提 |
| `nohlsearch` を `set` で書く | `nnoremap` でコマンドとして実行 | `set nohlsearch` は機能自体を無効にする |
| `nnoremap` と `nmap` を混同 | `nnoremap` を使う | `nmap` は再帰的で意図しない動作になりうる |
| `gdefault` 設定後に `:s/foo/bar/g` と書く | `:s/foo/bar/` （`g` 不要） | `gdefault` と `g` フラグで二重反転になる |
