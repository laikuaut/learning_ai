# 第5章 演習：動画・音声生成AIの活用

## はじめに

この演習では、動画生成AI・音声生成AIの特徴を理解し、目的に応じたツール選定やプロンプト設計ができるようになることを目指します。基本問題でサービスの分類と選定力を養い、応用問題で実務的な企画・設計に取り組み、チャレンジ問題でマルチメディア制作の全体像を設計します。

---

## 基本問題

### 問題1：動画生成AIサービスの分類

以下の動画生成AIサービスを、**テキストから動画生成（Text-to-Video）**、**画像から動画生成（Image-to-Video）**、**動画編集補助** の3つのカテゴリに分類してください。

```
a) Runway Gen-3
b) Pika Labs
c) Stable Video Diffusion
d) HeyGen（アバター動画）
e) Descript（動画編集）
f) Sora（OpenAI）
```

**期待される出力例：**

```
【テキストから動画生成（Text-to-Video）】
- Runway Gen-3：テキストプロンプトから短尺動画を生成
- Sora（OpenAI）：高品質なテキストベースの動画生成
...

【画像から動画生成（Image-to-Video）】
- Stable Video Diffusion：静止画をもとにモーションを付与
...

【動画編集補助】
- Descript：文字起こしベースで動画を編集
...
```

<details>
<summary>ヒント</summary>

各サービスの「入力」と「出力」に注目しましょう。テキストだけで動画ができるのか、既存の画像が必要なのか、既存の動画を加工するのかで分類できます。なお、一部のサービスは複数のカテゴリにまたがる機能を持っています。

</details>

<details>
<summary>解答例</summary>

```
【テキストから動画生成（Text-to-Video）】
- Runway Gen-3：テキストプロンプトから短尺動画を生成。カメラワーク指定も可能
- Pika Labs：テキストから短い動画クリップを生成。スタイル変換にも対応
- Sora（OpenAI）：テキスト指示から高解像度・長尺の動画を生成

【画像から動画生成（Image-to-Video）】
- Stable Video Diffusion：静止画を入力にモーション付き動画を生成
- Runway Gen-3：画像入力からの動画生成にも対応（複数カテゴリにまたがる）
- Pika Labs：画像からの動画変換にも対応（複数カテゴリにまたがる）

【動画編集補助】
- HeyGen：アバターを使った動画生成。既存スクリプトからプレゼン動画を作成
- Descript：文字起こしをベースにテキスト編集感覚で動画を編集

# ポイント：Runway Gen-3やPika Labsのように複数カテゴリにまたがるサービスもあります。
# 実務では「何を入力として持っているか」でツールを選ぶのが効率的です。
```

</details>

### 問題2：音声生成AIサービスの特徴整理

以下の音声生成AIサービスについて、**主な用途**、**対応言語**、**特徴** をそれぞれ簡潔にまとめてください。

```
a) ElevenLabs
b) VOICEVOX
c) OpenAI TTS（Text-to-Speech API）
d) Microsoft Azure Speech Service
```

**期待される出力例：**

```
| サービス名 | 主な用途 | 対応言語 | 特徴 |
|---|---|---|---|
| ElevenLabs | ナレーション・吹き替え | 多言語対応 | 声のクローニングが可能、感情表現が豊か |
| ... | ... | ... | ... |
```

<details>
<summary>ヒント</summary>

それぞれのサービスが「商用利用可能か」「日本語対応しているか」「カスタム音声が作れるか」という観点で比較すると整理しやすいです。

</details>

<details>
<summary>解答例</summary>

```
| サービス名 | 主な用途 | 対応言語 | 特徴 |
|---|---|---|---|
| ElevenLabs | ナレーション・吹き替え・オーディオブック | 29言語以上（日本語含む） | 声のクローニング、感情豊かな表現、高品質 |
| VOICEVOX | 日本語ナレーション・読み上げ | 日本語特化 | 無料・オープンソース、多数のキャラクター音声 |
| OpenAI TTS | アプリ組み込み・ナレーション | 多言語対応 | API経由で利用、6種類の音声、コスト効率が良い |
| Azure Speech Service | 業務システム組み込み | 100言語以上 | エンタープライズ向け、カスタムニューラル音声、SLA保証 |

# ポイント：用途によって最適なサービスは異なります。
# 個人制作ならVOICEVOXやElevenLabs、業務システム組み込みならAzureやOpenAI TTSが適しています。
# コスト・品質・ライセンスの3軸で判断しましょう。
```

</details>

### 問題3：適切なツール選定

以下の各シナリオに対して、最も適切な動画・音声生成AIツールを選び、選定理由を述べてください。

```
シナリオA：社内研修用に、スライド資料にナレーションを付けた5分程度の解説動画を作りたい
シナリオB：SNS広告用に、テキストの商品説明から15秒のプロモーション動画を作りたい
シナリオC：多言語対応のカスタマーサポート用に、自動音声応答システムを構築したい
シナリオD：個人のYouTubeチャンネルで、日本語のキャラクターナレーションを使いたい
```

**期待される出力例：**

```
【シナリオA】
推奨ツール：HeyGen
選定理由：アバター付きのプレゼンテーション動画を、スクリプトから自動生成できるため。
スライドとナレーションの同期も容易で、社内研修に最適。
...
```

<details>
<summary>ヒント</summary>

各シナリオの要件を「動画か音声か」「長さ」「用途（商用・個人）」「言語」「予算」の観点で分解してからツールを選びましょう。

</details>

<details>
<summary>解答例</summary>

```
【シナリオA】社内研修用ナレーション付き解説動画
推奨ツール：HeyGen
選定理由：スクリプトを入力するだけでアバター付きの解説動画を生成できます。
スライドを背景に設定し、自然なナレーションを付けられるため、
研修動画の制作工数を大幅に削減できます。

【シナリオB】SNS広告用プロモーション動画
推奨ツール：Runway Gen-3 または Sora
選定理由：テキストプロンプトから視覚的にインパクトのある短尺動画を生成できます。
商品の特徴をプロンプトに記述するだけで、広告向けの映像素材を作成可能です。

【シナリオC】多言語カスタマーサポート音声応答
推奨ツール：Microsoft Azure Speech Service
選定理由：100以上の言語に対応し、エンタープライズ向けのSLAが保証されています。
APIでの組み込みが容易で、カスタムニューラル音声で企業ブランドに合った
音声を構築できます。

【シナリオD】YouTubeキャラクターナレーション
推奨ツール：VOICEVOX
選定理由：無料で利用可能（クレジット表記が必要）で、多数のキャラクター音声から
選択できます。日本語に特化しており、個人クリエイターに最適です。

# ポイント：ツール選定では「必須要件」を最初に洗い出すことが重要です。
# 予算・言語・品質・ライセンスの優先度を明確にしてから比較しましょう。
```

</details>

## 応用問題

### 問題4：研修動画の企画・プロンプト設計

あなたは企業の教育担当者です。「新入社員向け情報セキュリティ研修」の動画（全3本、各5分程度）を生成AIを活用して制作することになりました。

以下の内容を設計してください。

1. 全3本の動画タイトルと各動画の概要（3〜5行）
2. 第1本目の動画について、動画生成AIに渡すプロンプト（シーンごとに3つ以上）
3. 使用するツールの組み合わせと選定理由

**期待される出力例：**

```
【動画企画】
第1本「パスワード管理の基本」
  概要：安全なパスワードの作り方、パスワードマネージャーの使い方、
       二要素認証の設定方法を解説する。
...

【第1本目のプロンプト設計】
シーン1（導入・30秒）：
  プロンプト：「オフィスでパソコンに向かう新入社員。画面にログイン画面が
  表示されている。明るく清潔感のあるオフィス環境。カメラはゆっくりズームイン。」
...

【ツール構成】
- 動画生成：HeyGen（アバター解説パート）+ Runway Gen-3（イメージ映像パート）
- 音声生成：ElevenLabs（ナレーション）
- 選定理由：...
```

<details>
<summary>ヒント</summary>

研修動画では「導入（課題提示）→ 解説（知識）→ まとめ（行動指針）」の流れが効果的です。プロンプト設計では、映像の雰囲気・カメラワーク・登場人物の動きを具体的に記述しましょう。

</details>

<details>
<summary>解答例</summary>

```
【動画企画】
第1本「パスワード管理の基本〜あなたのアカウントを守る〜」
  概要：弱いパスワードのリスクを具体例で示し、安全なパスワードの作成ルール、
       パスワードマネージャーの導入手順、二要素認証の有効化方法を解説する。
       実際の被害事例を冒頭で紹介し、危機意識を持たせる構成。

第2本「フィッシング詐欺を見抜く〜怪しいメールの見分け方〜」
  概要：フィッシングメールの典型的な特徴、URLの確認方法、添付ファイルの
       取り扱いルールを解説する。実際のフィッシングメール例を再現して
       注意すべきポイントを視覚的に示す。

第3本「情報漏えいを防ぐ日常の習慣」
  概要：クリアデスク・クリアスクリーンの実践、公共Wi-Fiの危険性、
       USBメモリの取り扱い、SNSへの業務情報投稿の禁止など、
       日常業務で守るべきセキュリティ習慣をチェックリスト形式で紹介。

【第1本目のプロンプト設計】
シーン1（導入・問題提起 40秒）：
  映像プロンプト：「暗い画面に『あなたのパスワードは大丈夫？』というテキストが
  浮かび上がる。続いて、ニュースヘッドラインのように『大手企業で10万件の
  アカウント情報が漏えい』というテロップが流れる。心配そうな表情の
  ビジネスパーソンのクローズアップ。」

シーン2（解説パート1・パスワードの強度 90秒）：
  アバタープロンプト：「こんにちは。今日はパスワード管理について学びましょう。
  まず、よく使われる危険なパスワードを見てみます。『123456』『password』
  『会社名+生年月日』……これらは攻撃者に数秒で破られてしまいます。
  安全なパスワードの条件は、12文字以上、大文字・小文字・数字・記号の
  組み合わせ、そして辞書に載っている単語を避けることです。」

シーン3（解説パート2・パスワードマネージャー 80秒）：
  映像プロンプト：「パソコン画面にパスワードマネージャーのインターフェースが
  表示されている。マスターパスワードを入力すると、各サービスのパスワードが
  安全に管理されている様子。画面録画風のチュートリアル映像。」
  アバタープロンプト：「パスワードを全部覚えるのは現実的ではありません。
  そこで便利なのがパスワードマネージャーです。1つのマスターパスワードを
  覚えるだけで、すべてのサービスに異なる強力なパスワードを設定できます。」

シーン4（まとめ・行動指針 50秒）：
  アバタープロンプト：「今日のポイントをまとめます。1つ目、パスワードは
  12文字以上で複雑に。2つ目、パスワードマネージャーを活用する。
  3つ目、二要素認証を必ず設定する。今日から実践してみてください。」

【ツール構成】
- 動画生成：HeyGen（アバターによる解説パート）
         Runway Gen-3（導入部のイメージ映像、シーン転換用の素材）
- 音声生成：ElevenLabs（日本語ナレーション、落ち着いたビジネストーン）
- 編集統合：Descript（各素材の結合、字幕の自動生成、最終編集）
- 選定理由：HeyGenは一貫した講師アバターで統一感を出せる。
  Runway Gen-3で視覚的なインパクトのある導入映像を補完。
  ElevenLabsの自然な日本語音声でプロフェッショナルな印象を確保。
  Descriptで素材を統合し字幕も自動生成することで、編集工数を削減。

# ポイント：研修動画では「飽きさせない構成」が重要です。
# アバター解説だけでなく、イメージ映像やテロップを組み合わせることで
# 視聴者の集中力を維持できます。
```

</details>

### 問題5：ナレーション生成の設計

以下の3つのシナリオそれぞれについて、音声生成AIで作成するナレーションの **設計書** を作成してください。設計書には、使用ツール、音声の特徴指定（トーン・速度・感情）、サンプルスクリプト（100文字程度）を含めてください。

```
シナリオ1：企業のIR（投資家向け）動画のナレーション
シナリオ2：子ども向け学習アプリの読み聞かせ音声
シナリオ3：社内ポッドキャストの司会進行
```

**期待される出力例：**

```
【シナリオ1：IR動画ナレーション】
使用ツール：ElevenLabs
音声特徴：
  - トーン：落ち着いた、信頼感のある男性の声
  - 速度：やや遅め（0.85倍速程度）
  - 感情：ニュートラル〜やや自信を持った表現
サンプルスクリプト：
  「当社は2025年度、売上高前年比12%増を達成しました。
   中期経営計画の2年目として、着実な成長を遂げております。」
...
```

<details>
<summary>ヒント</summary>

ターゲットの聴衆（投資家・子ども・社員）によって、声のトーン、速度、語彙レベルが大きく異なります。それぞれの聴衆が「心地よい」と感じる音声の特徴を考えましょう。

</details>

<details>
<summary>解答例</summary>

```
【シナリオ1：IR動画ナレーション】
使用ツール：ElevenLabs（高品質な多言語音声、プロフェッショナルなトーン）
音声特徴：
  - トーン：落ち着いた、信頼感と知性を感じさせる男性または女性の声
  - 速度：やや遅め（0.85倍速）。数字やデータが多いため聞き取りやすさを重視
  - 感情：ニュートラルを基本とし、成果報告部分ではやや自信を込めた表現
  - その他：滑舌が明瞭で、専門用語もはっきり発音。BGMとの共存を考慮した音量
サンプルスクリプト：
  「当社は2025年度、連結売上高1,200億円、前年比12%の増収を達成しました。
   中期経営計画で掲げた重点3領域への投資が着実に成果を上げております。」

【シナリオ2：子ども向け学習アプリの読み聞かせ音声】
使用ツール：VOICEVOX（親しみやすいキャラクター音声、無料で利用可能）
音声特徴：
  - トーン：明るく優しい、親しみやすい声。高めのピッチ
  - 速度：ゆっくり（0.75倍速）。単語ごとに適度な間を入れる
  - 感情：楽しげで、励ましを感じさせる表現。褒める場面では特に明るく
  - その他：漢字の読み仮名を正確に指定。効果音との連携も考慮
サンプルスクリプト：
  「すごい！正解だよ！りんごは英語で apple って言うんだね。
   じゃあ次のもんだい。この動物は英語でなんて言うかな？」

【シナリオ3：社内ポッドキャストの司会進行】
使用ツール：OpenAI TTS（自然な会話調、API組み込みで定期配信を自動化しやすい）
音声特徴：
  - トーン：カジュアルだが品のある声。フレンドリーな印象
  - 速度：標準速度（1.0倍速）。テンポよく進行する
  - 感情：明るく前向き。話題転換時にはワクワク感を込める
  - その他：会話形式を想定し、相槌や間のタイミングを設計
サンプルスクリプト：
  「みなさん、こんにちは！社内ポッドキャスト『イノベーションラボ』、
   今週のテーマは『リモートワーク3年目の気づき』です。
   今日はゲストに開発部の田中さんをお迎えしています。」

# ポイント：音声設計では「誰に聞かせるか」を最初に定義することが最重要です。
# ターゲットに合わせてトーン・速度・語彙を調整し、
# テスト音声を生成して関係者レビューを受けるプロセスを組み込みましょう。
```

</details>

## チャレンジ問題

### 問題6：マルチメディアコンテンツ制作の全体ワークフロー設計

あなたは中小企業のマーケティング担当者です。新製品（スマートウォッチ）の発売に合わせて、生成AIを活用したマルチメディアプロモーションキャンペーンを企画します。

以下の **全5種類のコンテンツ** を生成AIで制作するための、全体ワークフローを設計してください。

```
制作するコンテンツ：
1. 30秒のプロモーション動画（SNS広告用）
2. 製品紹介ナレーション音声（WebサイトLP用）
3. 製品画像バリエーション5枚（ECサイト・SNS投稿用）
4. 15秒の縦型ショート動画（TikTok / Instagram Reels用）
5. 製品説明のポッドキャスト音声（5分、自社メディア用）
```

設計には以下を含めてください。

- 各コンテンツで使用するAIツールの選定と理由
- 制作の順序と依存関係（どのコンテンツを先に作るべきか）
- 品質管理のチェックポイント（各工程で確認すべき項目）
- 想定スケジュール（5営業日以内を想定）
- リスクと対策（生成AIならではの注意点）

**期待される出力例：**

```
【全体ワークフロー図】
Day1: 企画・素材準備 → Day2: 画像・音声生成 → Day3: 動画生成 → ...

【ツール選定一覧】
| コンテンツ | 使用ツール | 選定理由 |
|---|---|---|
| プロモーション動画 | Runway Gen-3 + Descript | ... |
...

【制作順序と依存関係】
1. 製品画像（他のコンテンツの素材として使用するため最初に制作）
2. ナレーション音声（動画のオーディオトラックに使用）
...

【品質管理チェックポイント】
□ 製品の外観が実物と一致しているか（画像生成時）
□ ブランドガイドラインに沿った色使いか
...

【スケジュール】
Day1：...
...

【リスクと対策】
| リスク | 影響 | 対策 |
|---|---|---|
| 生成画像が実物と異なる | ブランド毀損 | 実物写真との比較レビュー必須 |
...
```

<details>
<summary>ヒント</summary>

ワークフロー設計のコツは「依存関係の整理」です。画像は動画の素材になり、ナレーションは動画の音声トラックになります。素材→部品→完成品の順で制作すると手戻りが少なくなります。また、生成AIの出力は必ず人間がレビューする工程を入れましょう。

</details>

<details>
<summary>解答例</summary>

```
【全体ワークフロー図】
Day1: 企画確定・プロンプト設計・素材収集

| # | コンテンツ | 使用ツール | 選定理由 |

| 1 | プロモーション動画（30秒） | Runway Gen-3 + Descript | テキストから高品質映像を生成し、Descriptで音声・字幕と統合 |
| 2 | ナレーション音声（LP用） | ElevenLabs | 高品質な日本語音声、製品の高級感に合うトーン設定が可能 |

【制作順序と依存関係】
Step 1（Day1）：全コンテンツ共通のプロンプト設計・ブランドガイドライン確認
  → 製品のキーメッセージ、ターゲット、トーン&マナーを統一

Step 2（Day2午前）：製品画像バリエーション5枚を生成
  → 理由：動画素材・SNS投稿・ECサイトすべての基盤となるため最優先
  → 依存：なし（最初に着手可能）

Step 3（Day2午後）：ナレーション音声 + ポッドキャスト音声を生成
  → 理由：動画のオーディオトラックとして必要
  → 依存：スクリプトの確定（Day1で準備済み）
  → ナレーションとポッドキャストは並行制作可能

Step 4（Day3）：プロモーション動画（30秒）を制作
  → 依存：製品画像（Step 2）+ ナレーション音声（Step 3）
  → 画像をベースにモーションを追加し、ナレーションを重ねる

Step 5（Day4午前）：ショート動画（15秒）を制作
  → 依存：プロモーション動画の素材を流用・再編集
  → 縦型フォーマットに変換し、テンポの速い編集に調整

Step 6（Day4午後〜Day5）：全コンテンツ最終レビュー・修正・納品

【品質管理チェックポイント】
＜画像生成時＞
  □ 製品の外観（色・形状・ロゴ配置）が実物と一致しているか
  □ ブランドカラーが正確に再現されているか
  □ 不自然なアーティファクト（指の本数、文字の歪み等）がないか
  □ 背景やライティングが製品の高級感を損なっていないか

＜音声生成時＞
  □ 製品名・社名の読みが正確か
  □ トーンとスピードがターゲット層に適しているか
  □ 不自然な抑揚やイントネーションがないか
  □ 技術用語の発音が正確か

＜動画生成時＞
  □ 映像と音声のタイミングが同期しているか
  □ ブランドロゴの表示タイミングと位置が適切か
  □ 各プラットフォームの推奨フォーマット（解像度・アスペクト比）に準拠しているか
  □ 法的に問題のある表現（誇大広告等）が含まれていないか

＜全体＞
  □ 5種類のコンテンツ間でメッセージの一貫性があるか
  □ ブランドガイドラインに準拠しているか
  □ 著作権・肖像権の問題がないか
```
