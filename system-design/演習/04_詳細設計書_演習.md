```

**期待される出力例：**

```
この処理は、ポイント利用処理を表している。

<details>
<summary>解答例</summary>

```
この処理は、注文時のポイント利用処理を表している。

1. ユーザーのポイント残高をデータベースから取得する。

2. ポイント残高と利用しようとしているポイント数を比較する。

3. ポイント残高が利用ポイント以上の場合（正常系）：
   a. ポイント残高から利用ポイント数を減算する。
   b. 注文金額から利用ポイント分を割引する。
   c. ポイント利用履歴をデータベースに記録する。

4. ポイント残高が利用ポイント未満の場合（異常系）：
   「ポイントが不足しています」というエラーメッセージを
   返し、処理を終了する。

※ このフローチャートでは、ポイント減算と注文金額割引が
   別々の処理になっているが、実装時にはトランザクションで
   まとめて実行し、途中で失敗した場合にロールバックする
   必要がある。
```

1. ユーザーがメールアドレスを入力する

### 問題4：擬似コードの作成

以下の処理を擬似コードで記述してください。

**処理：ECサイトのクーポン適用処理**
- 入力：注文情報（商品リスト、合計金額）、クーポンコード
- クーポンには「割引率型」と「固定金額型」の2種類がある
- クーポンの有効期限、利用回数上限、最低注文金額のチェックが必要
- 出力：割引後の金額

<details>
<summary>ヒント</summary>

以下の順序で処理を考えましょう。
1. クーポンコードからクーポン情報を取得
2. 各種バリデーション（有効期限、利用回数、最低金額）
3. クーポン種別に応じた割引計算
4. 割引後金額の計算と返却

</details>

<details>
<summary>解答例</summary>

```
関数: クーポン適用(注文情報, クーポンコード)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

入力: 注文情報（商品リスト、合計金額）
      クーポンコード（文字列）
出力: 割引結果（割引金額、割引後金額、適用メッセージ）

処理:
  1. クーポンコードでDBからクーポン情報を取得する
     もし クーポンが見つからない場合:
       → エラー「無効なクーポンコードです」

  2. バリデーションチェック:
     2.1 もし 現在日時 > クーポン.有効期限 の場合:
         → エラー「このクーポンは有効期限が切れています」

     2.2 もし クーポン.利用回数 ≧ クーポン.利用上限 の場合:
         → エラー「このクーポンは利用上限に達しています」

     2.3 もし 注文情報.合計金額 < クーポン.最低注文金額 の場合:
         → エラー「注文金額が○○円以上で利用できます」

     2.4 もし クーポン.対象カテゴリ が指定されている場合:
         もし 注文情報.商品リスト に対象カテゴリの商品がない場合:
           → エラー「このクーポンは対象商品がありません」

  3. 割引金額を計算する:
     もし クーポン.種別 = "割引率型" の場合:
       割引金額 = 注文情報.合計金額 × クーポン.割引率
       もし 割引金額 > クーポン.割引上限額 の場合:
         割引金額 = クーポン.割引上限額
       割引金額 = 小数点以下切り捨て

     もし クーポン.種別 = "固定金額型" の場合:
       割引金額 = クーポン.割引金額
       もし 割引金額 > 注文情報.合計金額 の場合:
         割引金額 = 注文情報.合計金額
         （※ 割引後金額が0円未満にならないようにする）

  4. 割引後金額を計算する:
     割引後金額 = 注文情報.合計金額 - 割引金額

  5. クーポンの利用回数を1増加する

  6. 結果を返す:
     {
       割引金額: 割引金額,
       割引後金額: 割引後金額,
       メッセージ: "クーポン「{コード}」を適用しました。
                   {割引金額}円割引"
     }

例外処理:
  ・DBアクセスエラーの場合:
    エラーログを記録し「システムエラーが発生しました」を返す
  ・同時利用による利用回数の競合:
    楽観的排他制御でリトライ（最大3回）
```

</details>

### 問題5：状態遷移図の作成

ヘルプデスクのチケットについて、状態遷移図を作成してください。

**チケットのステータス：**
- 新規（New）
- 対応中（In Progress）
- 保留中（On Hold）
- 解決済（Resolved）
- 完了（Closed）
- 再オープン（Reopened）

**遷移ルール：**
- 新規 → 対応中（担当者がチケットを受け取った時）
- 対応中 → 保留中（ユーザーからの情報待ちの時）
- 対応中 → 解決済（対応が完了した時）
- 保留中 → 対応中（ユーザーから情報が来た時）
- 解決済 → 完了（ユーザーが解決を確認した時）
- 解決済 → 再オープン（ユーザーが未解決と報告した時）
- 再オープン → 対応中（担当者が再対応を開始した時）

<details>
<summary>ヒント</summary>

状態を四角で、遷移を矢印で表現してください。各矢印にはトリガー（遷移条件）を記載します。

</details>

<details>
<summary>解答例</summary>
