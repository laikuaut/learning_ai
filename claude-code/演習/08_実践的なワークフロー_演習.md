## 基本問題

### 問題1：ワークフローの手順

以下のワークフローの手順を正しい順番に並べ替えてください。

**バグ修正ワークフロー:**
A. テストを実行して修正を確認
B. コミットを作成
C. エラーログを共有して原因を調査依頼
D. 修正内容を確認して承認
E. 回帰テストを追加
F. Claude Codeに修正を依頼

**期待される解答：**
```
1番目: ___
2番目: ___
3番目: ___
4番目: ___
5番目: ___
6番目: ___
```

<details>
<summary>ヒント</summary>

基本的な流れ：調査 → 修正 → 確認 → テスト → コミットです。

</details>

<details>
<summary>解答例</summary>

```
1番目: C（エラーログを共有して原因を調査依頼）
2番目: F（Claude Codeに修正を依頼）
3番目: D（修正内容を確認して承認）
4番目: E（回帰テストを追加）
5番目: A（テストを実行して修正を確認）
6番目: B（コミットを作成）
```

- まず原因を特定し、修正を依頼します
- 修正内容を人間の目で確認してから承認します
- 同じバグが再発しないよう回帰テストを追加します
- 全テストを実行して問題がないことを確認します
- 最後にコミットを作成します

</details>

### 問題3：TDDサイクルの理解

TDD（テスト駆動開発）の3つのステップを正しい順番で答え、各ステップでClaude Codeにどのような指示を出すか記述してください。

**期待される解答：**
```
Step 1: ___ （  ）
  プロンプト: ___

Step 2: ___ （  ）
  プロンプト: ___

Step 3: ___ （  ）
  プロンプト: ___
```

<details>
<summary>ヒント</summary>

TDDの3ステップは色で表されます：Red（赤：失敗するテスト）→ Green（緑：テスト通過）→ Refactor（リファクタリング）

</details>

<details>
<summary>解答例</summary>

```
Step 1: Red（テストを書く）
  プロンプト: 「calculateTax 関数のテストを先に書いてください。
  税率10%で計算し、小数点以下は切り捨てる仕様です。
  まだ関数の実装は作らないでください。」

Step 2: Green（実装する）
  プロンプト: 「テストがすべて通る最小限の
  calculateTax 関数を実装してください。」

Step 3: Refactor（リファクタリング）
  プロンプト: 「実装をリファクタリングして可読性を改善してください。
  テストが引き続き通ることを確認してください。」
```

- Red: まず失敗するテストを書く（仕様の明確化）
- Green: テストを通す最小限の実装を書く
- Refactor: テストを壊さずにコードを改善する

</details>

### 問題4：バグ修正プロンプト設計

以下のバグ報告に対して、Claude Codeに出す調査・修正のプロンプトを設計してください。

**バグ報告：**
```
タイトル: 商品価格が0円で表示される
環境: 本番環境
再現手順:
  1. 商品一覧ページを開く
  2. カテゴリ「セール品」でフィルタリング
  3. 一部の商品の価格が「¥0」と表示される
期待する動作: セール価格（元の価格の20%オフ）が表示される
技術スタック: React + Express + MongoDB
```

プロンプトを3ステップで設計してください。

<details>
<summary>ヒント</summary>

1. まずバグの原因を調査する（フロントエンド？バックエンド？データ？）
2. 原因を特定したら修正を依頼
3. テストを追加して再発防止

</details>

<details>
<summary>解答例</summary>

```
Step 1: 原因調査
> 商品一覧でカテゴリ「セール品」のフィルタリング時に、
> 一部の商品価格が0円と表示されるバグがあります。
> セール価格は元の価格の20%オフで計算されるはずです。
> 以下を調査してください：
> 1. バックエンドのセール価格計算ロジック
> 2. フロントエンドの価格表示コンポーネント
> 3. MongoDBのセール品データの状態
> 原因の可能性を特定してください。

Step 2: 修正
> （原因が「セール価格計算で元の価格がnullの場合に
>  0を返している」と判明した場合）
> セール価格の計算ロジックを修正してください。
> 元の価格がnullまたは0の場合はエラーとして処理し、
> 通常価格を表示するようにフォールバックしてください。
> フロントエンドでも価格が0の場合に「価格未設定」と
> 表示するガード処理を追加してください。

Step 3: テスト追加
> セール価格計算のテストを追加してください：
> - 正常系: 元の価格から20%オフが正しく計算される
> - 異常系: 元の価格がnullの場合の処理
> - 異常系: 元の価格が0の場合の処理
> - 境界値: 小数点の端数処理
> テストを実行して、すべて通ることを確認してください。
```

</details>

### 問題5：自動化スクリプトの設計

Claude Codeの`-p`フラグを使って、以下の作業を自動化するシェルスクリプトの設計を考えてください。

**要件：** PRを作成する前に、以下のチェックを自動実行したい
1. コードのリント
2. テストの実行
3. 変更内容のセキュリティチェック（機密情報の混入がないか）

各チェックの結果を1つのファイル（`pr-check-report.md`）にまとめて出力してください。

<details>
<summary>ヒント</summary>

- `claude -p "プロンプト"` でワンショット実行
- 結果をファイルに追記するには `>>` を使う
- 各チェックを順番に実行する

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash
# PR前チェック自動化スクリプト

REPORT="pr-check-report.md"

# レポートの初期化
echo "# PR前チェックレポート" > $REPORT
echo "実行日時: $(date)" >> $REPORT
echo "" >> $REPORT

# 1. リントチェック
echo "## リントチェック" >> $REPORT
npm run lint >> $REPORT 2>&1
if [ $? -eq 0 ]; then
  echo "結果: ✅ 問題なし" >> $REPORT
else
  echo "結果: ❌ エラーあり" >> $REPORT
fi
echo "" >> $REPORT

# 2. テスト実行
echo "## テスト結果" >> $REPORT
npm test >> $REPORT 2>&1
if [ $? -eq 0 ]; then
  echo "結果: ✅ 全テスト通過" >> $REPORT
else
  echo "結果: ❌ テスト失敗あり" >> $REPORT
fi
echo "" >> $REPORT

# 3. セキュリティチェック（Claude Codeで分析）
echo "## セキュリティチェック" >> $REPORT
claude -p "git diff --staged の変更内容を確認して、
以下の観点でセキュリティチェックをしてください：
- APIキーやパスワードのハードコーディング
- .env ファイルの変更
- 機密情報を含む可能性のあるログ出力
問題がなければ「問題なし」、
あれば具体的な箇所と対処法を報告してください。" >> $REPORT

echo "" >> $REPORT
echo "---" >> $REPORT
echo "チェック完了" >> $REPORT

echo "レポートを $REPORT に出力しました"
```

ポイント：
- Claude Codeのワンショットモードをセキュリティチェックに活用
- リントとテストは通常のCLIコマンドで実行（高速）
- AIが必要な分析はClaude Codeに任せる
- `--max-turns` を追加してコスト制御も検討

</details>

### 問題6：総合ワークフロー設計

あなたは新しいプロジェクト「オンライン書店API」の開発リーダーです。Claude Codeを最大限活用して、以下のタスクを効率的に進めるワークフローを設計してください。

**プロジェクト情報：**
- 技術スタック：TypeScript + NestJS + Prisma + PostgreSQL
- チーム：開発者3人
- 期間：2週間スプリント

**タスク一覧：**
1. プロジェクトの初期セットアップ（CLAUDE.md、設定ファイル）
2. 書籍のCRUD API実装
3. ユーザー認証機能
4. 検索機能
5. テスト・コードレビュー

各タスクについて、Claude Codeにどのような指示を出すか具体的に記述してください。

<details>
<summary>ヒント</summary>

- タスク1はCLAUDE.mdと.claude/settings.jsonの作成
- タスク2-4は機能開発ワークフロー（計画→実装→テスト→コミット）
- タスク5はClaude Codeのレビュー機能を活用

</details>

<details>
<summary>解答例</summary>

```
■ タスク1: プロジェクト初期セットアップ

プロンプト1（CLAUDE.md作成）:
> 以下の情報でCLAUDE.mdを作成してください：
> - プロジェクト名: OnlineBookStore API
> - 技術スタック: TypeScript, NestJS, Prisma, PostgreSQL
> - ディレクトリ構成: src/ 配下に modules/, common/, config/
> - 開発コマンド: npm run start:dev, npm test, npm run lint
> - ブランチ戦略: Git Flow（main, develop, feature/*, fix/*）
> - コミット規約: Conventional Commits
> - 注意事項: prisma/generated は編集禁止

プロンプト2（settings.json作成）:
> .claude/settings.json を作成してください。
> Read, Glob, Grep は自動許可、
> npm test と npm run lint の実行も自動許可、
> rm -rf は拒否してください。

■ タスク2: 書籍CRUD API

プロンプト1（設計相談）:
> 書籍のCRUD APIを設計してください。
> エンドポイント: GET/POST/PUT/DELETE /api/books

プロンプト2（実装）:
> 設計に基づいて、以下を作成してください：
> - src/modules/books/ 配下にmodule, controller, service

プロンプト3（テスト）:
> BooksService と BooksController のユニットテストを
> 作成して実行してください。

■ タスク3: ユーザー認証

プロンプト:
> JWT認証を実装してください。
> POST /api/auth/register（ユーザー登録）
> POST /api/auth/login（ログイン、JWTトークン発行）
> Guard で認証が必要なエンドポイントを保護
> テストも作成してください。

■ タスク4: 検索機能

プロンプト:
> 書籍の検索APIを実装してください。

> テストも作成してください。

■ タスク5: テスト・レビュー

プロンプト1（総合テスト）:
> プロジェクト全体のテストを実行して、
> カバレッジレポートを表示してください。

> 追加テストを提案してください。

プロンプト2（コードレビュー）:
> src/ ディレクトリ全体をレビューしてください。
> セキュリティ、パフォーマンス、エラーハンドリング、
> コードの一貫性の観点で問題点を指摘してください。
```

</details>
