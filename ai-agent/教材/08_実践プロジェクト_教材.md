## 8.1 プロジェクト設計の進め方

### 要件定義

エージェントプロジェクトでは、通常のソフトウェア開発に加えて「エージェント特有の要件」を定義する必要があります。

```
┌──────────────────────────────────────────────────────┐
│        エージェントプロジェクトの要件定義               │
├──────────────────────────────────────────────────────┤
│                                                      │
│  【機能要件】                                         │
│  ・どんなタスクを自律的に実行するか                    │
│  ・どのツールを使えるようにするか                      │
│  ・人間の承認が必要な操作は何か                        │
│                                                      │
│  【非機能要件】                                       │
│  ・応答速度の目標（例：30秒以内）                     │
│  ・1タスクあたりのコスト上限（例：$0.10以内）          │
│  ・稼働率（例：99.5%）                               │
│  ・セキュリティ要件（PII の扱い等）                   │
│                                                      │
│  【エージェント固有の要件】                            │
│  ・最大ステップ数の上限                               │
│  ・フォールバック（Fallback）戦略                      │
│  ・人間へのエスカレーション条件                        │
│  ・許容するエラー率                                   │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### アーキテクチャ選定

タスクの複雑さに応じて適切なアーキテクチャを選びます。

| アーキテクチャ | 適するケース | 複雑さ | 例 |
|---------------|-------------|--------|-----|
| シンプルループ | 単一ツールの反復 | 低 | FAQ検索ボット |
| ReAct パターン | 推論+行動の交互実行 | 中 | カスタマーサポート |
| マルチエージェント | 専門性が異なる複数タスク | 高 | リサーチ+レポート |
| 階層型 | 大規模・長時間タスク | 高 | プロジェクト管理 |

### 技術スタック（Technology Stack）の決定

```python
# プロジェクト構成の例
project_structure = {
    "LLMプロバイダ": "OpenAI GPT-4o / Anthropic Claude",
    "フレームワーク": "LangGraph / 自作ループ",
    "ベクトルDB": "ChromaDB / Pinecone（RAG利用時）",
    "モニタリング": "LangSmith / Langfuse",
    "テスト": "pytest + 自作ベンチマーク",
    "デプロイ": "FastAPI + Docker",
}
```

> **ポイントまとめ**
> - 要件定義ではエージェント固有の要件（最大ステップ数、コスト上限等）を明記する
> - タスクの複雑さに応じてアーキテクチャを選定する
> - 技術スタックは運用・監視まで含めて決定する

## 8.2 プロジェクト1：カスタマーサポートエージェント

### 概要と設計

FAQ検索とチケット作成を自律的に行うカスタマーサポートエージェントを構築します。

```
┌──────────────────────────────────────────────────────────┐
│  カスタマーサポートエージェント アーキテクチャ             │
│                                                          │
│  ユーザー                                                │
│    │                                                     │
│    ▼                                                     │
│  ┌──────────────┐                                        │
│  │   Agent      │──→ search_faq()      FAQ検索           │
│  │  (ReAct)     │──→ get_order_info()  注文情報取得      │
│  │              │──→ create_ticket()   チケット作成       │
│  │              │──→ escalate()        人間へエスカレート  │
│  └──────────────┘                                        │
│    │                                                     │
│    ▼                                                     │
│  ユーザーへの回答                                         │
└──────────────────────────────────────────────────────────┘
```

### 実装

```python
import json
from datetime import datetime

# =============================================
# ツール定義
# =============================================

# FAQ データベース（本番ではベクトルDBを使用）
FAQ_DATABASE = [
    {
        "id": "faq-001",
        "category": "返品",
        "question": "商品を返品したい",
        "answer": "購入から30日以内であれば返品可能です。マイページの注文履歴から返品申請を行ってください。",
        "keywords": ["返品", "返送", "キャンセル"],
    },
    {
        "id": "faq-002",
        "category": "配送",
        "question": "商品が届かない",
        "answer": "通常3-5営業日で配送されます。マイページの注文履歴から配送状況をご確認ください。7営業日経っても届かない場合はサポートチケットを作成します。",
        "keywords": ["届かない", "配送", "発送", "遅延"],
    },
    {
        "id": "faq-003",
        "category": "アカウント",
        "question": "パスワードをリセットしたい",
        "answer": "ログイン画面の「パスワードを忘れた方」リンクから、登録メールアドレス宛にリセットメールを送信できます。",
        "keywords": ["パスワード", "ログイン", "リセット"],
    },
]

# 注文データベース（本番ではDBやAPIから取得）
ORDERS_DATABASE = {
    "ORD-12345": {"status": "配送中", "shipped_date": "2026-03-20", "eta": "2026-03-27"},
    "ORD-67890": {"status": "処理中", "shipped_date": None, "eta": "2026-03-30"},
}


def search_faq(query: str) -> str:
    """FAQデータベースからキーワード検索を行います"""
    results = []
    for faq in FAQ_DATABASE:
        if any(kw in query for kw in faq["keywords"]):
            results.append(faq)
    if not results:
        return json.dumps({"results": [], "message": "該当するFAQが見つかりませんでした"}, ensure_ascii=False)
    return json.dumps({"results": results}, ensure_ascii=False)


def get_order_info(order_id: str) -> str:
    """注文情報を取得します"""
    order = ORDERS_DATABASE.get(order_id)
    if not order:
        return json.dumps({"error": f"注文 {order_id} が見つかりません"}, ensure_ascii=False)
    return json.dumps({"order_id": order_id, **order}, ensure_ascii=False)


def create_ticket(title: str, description: str, priority: str = "medium") -> str:
    """サポートチケットを作成します"""
    ticket_id = f"TKT-{datetime.now().strftime('%Y%m%d%H%M%S')}"
    return json.dumps({
        "ticket_id": ticket_id,
        "title": title,
        "description": description,
        "priority": priority,
        "status": "open",
    }, ensure_ascii=False)


def escalate(reason: str) -> str:
    """人間のオペレーターにエスカレートします"""
    return json.dumps({
        "escalated": True,
        "reason": reason,
        "message": "人間のオペレーターに転送しました。しばらくお待ちください。",
    }, ensure_ascii=False)


# =============================================
# エージェント本体
# =============================================

TOOLS = {
    "search_faq": {
        "function": search_faq,
        "description": "FAQデータベースを検索します",
        "parameters": {"query": "検索キーワード（文字列）"},
    },
    "get_order_info": {
        "function": get_order_info,
        "description": "注文IDから注文情報を取得します",
        "parameters": {"order_id": "注文ID（例: ORD-12345）"},
    },
    "create_ticket": {
        "function": create_ticket,
        "description": "サポートチケットを作成します",
        "parameters": {"title": "件名", "description": "詳細", "priority": "優先度(low/medium/high)"},
    },
    "escalate": {
        "function": escalate,
        "description": "人間のオペレーターにエスカレートします",
        "parameters": {"reason": "エスカレート理由"},
    },
}

SYSTEM_PROMPT = """あなたはカスタマーサポートエージェントです。

## 対応手順
1. ユーザーの問い合わせ内容を正確に把握する
2. search_faq でFAQを検索する
3. 注文に関する問い合わせは get_order_info で注文情報を確認する
4. FAQで解決できない場合は create_ticket でチケットを作成する
5. 感情的なクレームや複雑な問題は escalate で人間に引き継ぐ

## 注意事項
- 推測で回答しない（必ずFAQまたは注文情報を確認する）
- 丁寧な言葉遣いを使う
- 解決策を具体的に提示する
- 1回の対応で解決を目指す
"""


class CustomerSupportAgent:
    """カスタマーサポートエージェント"""

    def __init__(self, llm_client, max_steps=8):
        self.llm = llm_client
        self.max_steps = max_steps
        self.tools = TOOLS

    def run(self, user_message: str) -> dict:
        messages = [
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "user", "content": user_message},
        ]
        steps = []

        for step_num in range(self.max_steps):
            # LLMに判断を委ねる
            response = self.llm.chat(
                messages=messages,
                tools=self._get_tool_definitions(),
            )

            # ツール呼び出しがなければ最終回答
            if not response.tool_calls:
                return {
                    "answer": response.content,
                    "steps": steps,
                    "total_steps": step_num + 1,
                }

            # ツールを実行
            for tool_call in response.tool_calls:
                tool_name = tool_call.name
                tool_args = tool_call.arguments
                tool_fn = self.tools[tool_name]["function"]
                result = tool_fn(**tool_args)

                steps.append({
                    "step": step_num + 1,
                    "tool": tool_name,
                    "args": tool_args,
                    "result": result,
                })

                # 結果をメッセージに追加
                messages.append({"role": "assistant", "tool_calls": [tool_call]})
                messages.append({"role": "tool", "content": result})

        return {"answer": "申し訳ございません。問題を解決できませんでした。", "steps": steps}

    def _get_tool_definitions(self):
        return [
            {"name": name, "description": info["description"], "parameters": info["parameters"]}
            for name, info in self.tools.items()
        ]
```

## 8.3 プロジェクト2：コードレビューエージェント

### 概要と設計

Gitリポジトリの差分を読み取り、静的解析の結果と合わせてレビューコメントを自動生成するエージェントです。

```
┌─────────────────────────────────────────────────────────┐
│  コードレビューエージェント アーキテクチャ                │
│                                                         │
│  Git PR / Diff                                          │
│    │                                                    │
│    ▼                                                    │
│  ┌────────────────┐                                     │
│  │  Orchestrator  │                                     │
│  │  (調整役)       │                                     │
│  └────────────────┘                                     │
│    │          │          │                              │
│    ▼          ▼          ▼                              │
│  ┌──────┐  ┌──────┐  ┌──────────┐                      │
│  │ Diff │  │ Lint │  │ Security │                      │
│  │解析  │  │ 解析 │  │  Check   │                      │
│  └──────┘  └──────┘  └──────────┘                      │
│    │          │          │                              │
│    └──────────┴──────────┘                              │
│              │                                          │
│              ▼                                          │
│    ┌──────────────────┐                                 │
│    │ レビューコメント   │                                 │
│    │ 生成 (LLM)        │                                 │
│    └──────────────────┘                                 │
│              │                                          │
│              ▼                                          │
│    GitHub PR にコメント投稿                              │
└─────────────────────────────────────────────────────────┘
```

### 実装

```python
import subprocess
import json
import re
from dataclasses import dataclass, field

# =============================================
# データ構造
# =============================================

@dataclass
class DiffFile:
    """差分ファイル1つ分の情報"""
    filename: str
    additions: list = field(default_factory=list)  # (行番号, 内容)
    deletions: list = field(default_factory=list)
    raw_diff: str = ""

@dataclass
class ReviewComment:
    """レビューコメント1件"""
    file: str
    line: int
    severity: str   # "error", "warning", "suggestion"
    category: str   # "bug", "style", "security", "performance"
    message: str
    suggestion: str = ""  # 修正案

# =============================================
# ツール定義
# =============================================

def get_git_diff(base_branch: str = "main") -> list[DiffFile]:
    """Gitの差分を取得して解析します"""
    result = subprocess.run(
        ["git", "diff", base_branch, "--unified=3"],
        capture_output=True, text=True, encoding="utf-8"
    )
    return parse_diff(result.stdout)


def parse_diff(diff_text: str) -> list[DiffFile]:
    """unified diff 形式をパースします"""
    files = []
    current_file = None
    current_line = 0

    for line in diff_text.split("\n"):
        if line.startswith("+++ b/"):
            filename = line[6:]
            current_file = DiffFile(filename=filename, raw_diff="")
            files.append(current_file)
        elif line.startswith("@@ "):
            # @@ -a,b +c,d @@ から新ファイルの行番号を取得
            match = re.search(r"\+(\d+)", line)
            if match:
                current_line = int(match.group(1))
        elif current_file is not None:
            if line.startswith("+") and not line.startswith("+++"):
                current_file.additions.append((current_line, line[1:]))
                current_line += 1
            elif line.startswith("-") and not line.startswith("---"):
                current_file.deletions.append((current_line, line[1:]))
            else:
                current_line += 1
            current_file.raw_diff += line + "\n"

    return files


def run_linter(filename: str) -> list[dict]:
    """静的解析ツール（linter）を実行します"""
    # Python ファイルの場合
    if filename.endswith(".py"):
        try:
            result = subprocess.run(
                ["python", "-m", "py_compile", filename],
                capture_output=True, text=True
            )
            issues = []
            if result.returncode != 0:
                issues.append({
                    "type": "syntax_error",
                    "message": result.stderr.strip(),
                })
            return issues
        except FileNotFoundError:
            return []
    return []


def check_security_patterns(diff_files: list[DiffFile]) -> list[dict]:
    """セキュリティ上の問題パターンを検出します"""
    security_patterns = [
        (r"password\s*=\s*['\"]", "ハードコードされたパスワードが検出されました"),
        (r"api_key\s*=\s*['\"]", "ハードコードされたAPIキーが検出されました"),
        (r"eval\(", "eval() の使用はコードインジェクションのリスクがあります"),
        (r"exec\(", "exec() の使用はコードインジェクションのリスクがあります"),
        (r"SELECT.*\+.*(?:user_input|request)", "SQLインジェクションの可能性があります"),
        (r"subprocess\.call\(.*shell=True", "shell=True はコマンドインジェクションのリスクがあります"),
    ]

    findings = []
    for diff_file in diff_files:
        for line_num, content in diff_file.additions:
            for pattern, message in security_patterns:
                if re.search(pattern, content, re.IGNORECASE):
                    findings.append({
                        "file": diff_file.filename,
                        "line": line_num,
                        "content": content.strip(),
                        "message": message,
                    })
    return findings


# =============================================
# レビューエージェント本体
# =============================================

REVIEW_SYSTEM_PROMPT = """あなたは熟練のコードレビュアーです。
以下の情報をもとに、建設的なレビューコメントを作成してください。

## レビュー観点
1. バグの可能性（ロジックエラー、エッジケース未考慮）
2. コードの可読性（命名、構造、コメント）
3. セキュリティ（入力バリデーション、機密情報の扱い）
4. パフォーマンス（不要なループ、N+1問題）
5. ベストプラクティス（言語固有の慣習、設計パターン）

## 出力形式
各コメントをJSON配列で出力してください:
[{"file": "ファイル名", "line": 行番号, "severity": "error|warning|suggestion",
  "category": "bug|style|security|performance", "message": "指摘内容",
  "suggestion": "修正案（あれば）"}]
"""


class CodeReviewAgent:
    """コードレビューエージェント"""

    def __init__(self, llm_client, base_branch="main"):
        self.llm = llm_client
        self.base_branch = base_branch

    def review(self) -> list[ReviewComment]:
        """PRの差分をレビューします"""
        # ステップ1: 差分を取得
        diff_files = get_git_diff(self.base_branch)
        if not diff_files:
            return []

        # ステップ2: 静的解析
        lint_issues = []
        for df in diff_files:
            lint_issues.extend(run_linter(df.filename))

        # ステップ3: セキュリティチェック
        security_issues = check_security_patterns(diff_files)

        # ステップ4: LLMによるレビュー
        review_input = self._build_review_input(diff_files, lint_issues, security_issues)
        response = self.llm.chat(
            messages=[
                {"role": "system", "content": REVIEW_SYSTEM_PROMPT},
                {"role": "user", "content": review_input},
            ]
        )

        # ステップ5: コメントをパース
        comments = self._parse_comments(response.content)
        return comments

    def _build_review_input(self, diff_files, lint_issues, security_issues) -> str:
        sections = ["## 差分"]
        for df in diff_files:
            sections.append(f"### {df.filename}")
            sections.append(f"```\n{df.raw_diff[:2000]}\n```")

        if lint_issues:
            sections.append("\n## 静的解析の結果")
            for issue in lint_issues:
                sections.append(f"- {issue['type']}: {issue['message']}")

        if security_issues:
            sections.append("\n## セキュリティチェックの結果")
            for issue in security_issues:
                sections.append(f"- {issue['file']}:{issue['line']} - {issue['message']}")

        return "\n".join(sections)

    def _parse_comments(self, llm_output: str) -> list[ReviewComment]:
        try:
            # JSON部分を抽出
            match = re.search(r"\[.*\]", llm_output, re.DOTALL)
            if match:
                data = json.loads(match.group())
                return [ReviewComment(**item) for item in data]
        except (json.JSONDecodeError, TypeError):
            pass
        return []


# =============================================
# GitHub PR へのコメント投稿
# =============================================

def post_review_to_github(comments: list[ReviewComment], pr_number: int, repo: str):
    """GitHubのPRにレビューコメントを投稿します"""
    # gh CLI を使用（本番では GitHub API を直接呼ぶことも）
    for comment in comments:
        severity_emoji = {"error": "x", "warning": "!", "suggestion": "?"}
        emoji = severity_emoji.get(comment.severity, "?")
        body = f"**[{comment.severity.upper()}] {comment.category}**\n\n{comment.message}"
        if comment.suggestion:
            body += f"\n\n**修正案:**\n```suggestion\n{comment.suggestion}\n```"

        subprocess.run([
            "gh", "api",
            f"repos/{repo}/pulls/{pr_number}/comments",
            "-f", f"body={body}",
            "-f", f"path={comment.file}",
            "-f", f"line={comment.line}",
            "-f", "side=RIGHT",
        ])
```

> **よくある間違い**
> - 差分全体を1回のLLM呼び出しで処理しようとする → 大きなPRではトークン制限に引っかかります。ファイルごとに分割してレビューしましょう
> - セキュリティチェックをLLMだけに頼る → パターンマッチによる静的解析と組み合わせることで検出精度が向上します

> **ポイントまとめ**
> - 静的解析 + セキュリティチェック + LLMレビューを組み合わせる
> - 大きなPRはファイル単位で分割処理する
> - GitHub API / gh CLI で自動的にコメントを投稿する

## 8.4 プロジェクト3：リサーチエージェント

### 概要と設計

Web検索で情報を収集し、要約してレポートを生成する複合エージェント（Compound Agent）です。マルチエージェント構成で役割を分担します。

```
┌──────────────────────────────────────────────────────────┐
│  リサーチエージェント アーキテクチャ（マルチエージェント）  │
```

### 実装

```python
import json
import urllib.request
import urllib.parse
from dataclasses import dataclass, field
from datetime import datetime

# =============================================
# データ構造
# =============================================

@dataclass
class SearchResult:
    """Web検索結果1件"""
    title: str
    url: str
    snippet: str

@dataclass
class ResearchPlan:
    """調査計画"""
    topic: str
    queries: list  # 検索クエリのリスト
    focus_areas: list  # 重点調査項目

@dataclass
class CollectedInfo:
    """収集した情報"""
    source_url: str
    title: str
    content: str  # 要約済みテキスト
    relevance: float  # 0.0-1.0

@dataclass
class ResearchReport:
    """最終レポート"""
    title: str
    summary: str
    sections: list  # [{"heading": "...", "content": "..."}]
    sources: list    # [{"title": "...", "url": "..."}]
    generated_at: str = field(default_factory=lambda: datetime.now().isoformat())

# =============================================
# ツール定義
# =============================================

def search_web(query: str, num_results: int = 5) -> list[SearchResult]:
    """Web検索を実行します（実装はAPI依存）"""
    # 本番では Google Custom Search API や Bing Search API を使用
    # ここではインターフェースを示します
    encoded = urllib.parse.quote(query)
    # api_url = f"https://api.search.example.com/search?q={encoded}&n={num_results}"
    # response = urllib.request.urlopen(api_url)
    # data = json.loads(response.read())

    # デモ用の戻り値
    return [
        SearchResult(
            title=f"検索結果: {query}",
            url=f"https://example.com/result",
            snippet=f"{query}に関する情報...",
        )
    ]


def fetch_page(url: str) -> str:
    """Webページの本文テキストを取得します"""
    try:
        req = urllib.request.Request(url, headers={"User-Agent": "ResearchAgent/1.0"})
        with urllib.request.urlopen(req, timeout=10) as response:
            html = response.read().decode("utf-8", errors="ignore")
            # HTMLタグを簡易的に除去（本番では BeautifulSoup 等を使用）
            import re
            text = re.sub(r"<[^>]+>", "", html)
            text = re.sub(r"\s+", " ", text).strip()
            return text[:5000]  # 最初の5000文字
    except Exception as e:
        return f"ページ取得エラー: {str(e)}"


# =============================================
# エージェント群
# =============================================

class PlannerAgent:
    """調査計画を立てるエージェント"""

    PROMPT = """あなたはリサーチの計画を立てる専門家です。
ユーザーのリサーチトピックに対して、効果的な調査計画を立ててください。

JSON形式で出力してください:
{"queries": ["検索クエリ1", "検索クエリ2", ...], "focus_areas": ["重点項目1", ...]}

注意:
- 検索クエリは3〜5個（多角的な視点で）
- 日本語と英語の両方のクエリを含める
- 最新情報を得るために年号を含めるクエリも追加する
"""

    def __init__(self, llm_client):
        self.llm = llm_client

    def create_plan(self, topic: str) -> ResearchPlan:
        response = self.llm.chat(
            messages=[
                {"role": "system", "content": self.PROMPT},
                {"role": "user", "content": f"リサーチトピック: {topic}"},
            ]
        )
        data = json.loads(response.content)
        return ResearchPlan(
            topic=topic,
            queries=data["queries"],
            focus_areas=data["focus_areas"],
        )


class ResearcherAgent:
    """Web検索と情報収集を行うエージェント"""

    SUMMARIZE_PROMPT = """以下のWebページの内容を、リサーチトピックに関連する部分を
200文字以内で要約してください。関連がない場合は「関連情報なし」と回答してください。

トピック: {topic}
ページ内容: {content}
"""

    def __init__(self, llm_client):
        self.llm = llm_client

    def research(self, plan: ResearchPlan) -> list[CollectedInfo]:
        all_info = []

        for query in plan.queries:
            # 検索実行
            results = search_web(query, num_results=3)

            for result in results:
                # ページ取得
                page_content = fetch_page(result.url)

                # LLMで要約・関連性判定
                response = self.llm.chat(
                    messages=[{
                        "role": "user",
                        "content": self.SUMMARIZE_PROMPT.format(
                            topic=plan.topic, content=page_content[:3000]
                        ),
                    }]
                )

                if "関連情報なし" not in response.content:
                    all_info.append(CollectedInfo(
                        source_url=result.url,
                        title=result.title,
                        content=response.content,
                        relevance=0.8,  # 本番ではLLMに評価させる
                    ))

        # 関連性でソート
        all_info.sort(key=lambda x: x.relevance, reverse=True)
        return all_info[:10]  # 上位10件


class WriterAgent:
    """レポートを生成するエージェント"""

    PROMPT = """あなたはリサーチレポートの執筆者です。
以下の収集情報をもとに、構造化されたレポートを作成してください。

## 出力形式（JSON）
{{
  "title": "レポートタイトル",
  "summary": "エグゼクティブサマリー（3-5文）",
  "sections": [
    {{"heading": "セクション見出し", "content": "セクション内容"}},
    ...
  ]
}}

## 注意
- 客観的な記述を心がける
- 出典を明記する
- 最新の動向を優先する
- 3〜5セクション構成にする
"""

    def __init__(self, llm_client):
        self.llm = llm_client

    def write_report(self, topic: str, collected_info: list[CollectedInfo]) -> ResearchReport:
        info_text = "\n\n".join([
            f"### {info.title}\n出典: {info.source_url}\n{info.content}"
            for info in collected_info
        ])

        response = self.llm.chat(
            messages=[
                {"role": "system", "content": self.PROMPT},
                {"role": "user", "content": f"トピック: {topic}\n\n## 収集情報\n{info_text}"},
            ]
        )

        data = json.loads(response.content)
        return ResearchReport(
            title=data["title"],
            summary=data["summary"],
            sections=data["sections"],
            sources=[{"title": i.title, "url": i.source_url} for i in collected_info],
        )


# =============================================
# オーケストレータ（全体制御）
# =============================================

class ResearchOrchestrator:
    """リサーチエージェント群を統括するオーケストレータ"""

    def __init__(self, llm_client):
        self.planner = PlannerAgent(llm_client)
        self.researcher = ResearcherAgent(llm_client)
        self.writer = WriterAgent(llm_client)

    def run(self, topic: str) -> ResearchReport:
        print(f"[1/3] 調査計画を作成中...")
        plan = self.planner.create_plan(topic)
        print(f"  - 検索クエリ: {len(plan.queries)}件")
        print(f"  - 重点項目: {plan.focus_areas}")

        print(f"[2/3] 情報収集中...")
        collected = self.researcher.research(plan)
        print(f"  - 収集情報: {len(collected)}件")

        print(f"[3/3] レポート生成中...")
        report = self.writer.write_report(topic, collected)
        print(f"  - レポートタイトル: {report.title}")
        print(f"  - セクション数: {len(report.sections)}")

        return report

    def export_markdown(self, report: ResearchReport) -> str:
        """レポートをMarkdown形式で出力します"""
        lines = [f"# {report.title}", "", report.summary, ""]
        for section in report.sections:
            lines.append(f"## {section['heading']}")
            lines.append(section["content"])
            lines.append("")
        lines.append("## 出典")
        for i, src in enumerate(report.sources, 1):
            lines.append(f"{i}. [{src['title']}]({src['url']})")
        lines.append(f"\n---\n*生成日時: {report.generated_at}*")
        return "\n".join(lines)
```

> **よくある間違い**
> - 1つのエージェントに全ての役割を持たせる → 計画・調査・執筆を分離することで品質と保守性が向上します
> - 検索結果を無条件に信頼する → LLMで関連性を判定し、ノイズを除去しましょう

> **ポイントまとめ**
> - 計画→調査→執筆の3段階をマルチエージェントで分担する
> - 各エージェントは単一の責務（Single Responsibility）を持つ
> - 情報の関連性判定でノイズを除去する

## 8.5 本番運用のポイント

### モニタリング（Monitoring）

本番環境ではエージェントの状態を常時監視する仕組みが必要です。

```python
import time
from dataclasses import dataclass, field

@dataclass
class AgentMetrics:
    """エージェントの運用メトリクス"""
    total_requests: int = 0
    successful_requests: int = 0
    failed_requests: int = 0
    total_cost_usd: float = 0.0
    total_tokens: int = 0
    latency_history: list = field(default_factory=list)

    @property
    def success_rate(self) -> float:
        if self.total_requests == 0:
            return 0.0
        return self.successful_requests / self.total_requests

    @property
    def avg_latency(self) -> float:
        if not self.latency_history:
            return 0.0
        return sum(self.latency_history) / len(self.latency_history)

    def record(self, success: bool, cost: float, tokens: int, latency: float):
        self.total_requests += 1
        if success:
            self.successful_requests += 1
        else:
            self.failed_requests += 1
        self.total_cost_usd += cost
        self.total_tokens += tokens
        self.latency_history.append(latency)

    def check_alerts(self) -> list[str]:
        """アラート条件をチェックします"""
        alerts = []
        if self.success_rate < 0.9:
            alerts.append(f"成功率が低下しています: {self.success_rate:.1%}")
        if self.avg_latency > 30:
            alerts.append(f"平均レイテンシが高すぎます: {self.avg_latency:.1f}秒")
        if self.total_cost_usd > 100:
            alerts.append(f"コストが閾値を超えています: ${self.total_cost_usd:.2f}")
        return alerts
```

### コスト管理

| 管理項目 | 手法 | 効果 |
|---------|------|------|
| トークン制限 | 入出力の最大トークン数を設定 | コスト上限を制御 |
| キャッシュ | 同一クエリの結果をキャッシュ | API呼び出し回数を削減 |
| モデル使い分け | 簡単なタスクには軽量モデルを使用 | 単価を削減 |
| ステップ上限 | max_steps を設定 | 暴走を防止 |
| 日次予算 | 日あたりの費用上限を設定 | 予算超過を防止 |

```python
class CostManager:
    """コスト管理マネージャー"""

    # モデル別の料金（$/1Kトークン, 2026年目安）
    PRICING = {
        "gpt-4o":       {"input": 0.0025, "output": 0.010},
        "gpt-4o-mini":  {"input": 0.00015, "output": 0.0006},
        "claude-sonnet": {"input": 0.003, "output": 0.015},
    }

    def __init__(self, daily_budget_usd: float = 10.0):
        self.daily_budget = daily_budget_usd
        self.daily_spent = 0.0

    def estimate_cost(self, model: str, input_tokens: int, output_tokens: int) -> float:
        pricing = self.PRICING.get(model, {"input": 0.01, "output": 0.03})
        return (input_tokens * pricing["input"] + output_tokens * pricing["output"]) / 1000

    def can_proceed(self, estimated_cost: float) -> bool:
        return (self.daily_spent + estimated_cost) <= self.daily_budget

    def record_usage(self, cost: float):
        self.daily_spent += cost
```

## 8.6 AIエージェントの未来

### 自律性の向上

現在のAIエージェントは比較的限定的なタスクを扱いますが、今後は以下のような進化が予想されます。

```
┌──────────────────────────────────────────────────────┐
│  AIエージェントの自律性レベル                          │
│                                                      │
│  Level 1: ツール実行（現在の主流）                    │
│           人間が指示 → エージェントがツールを選んで実行 │
│                                                      │
│  Level 2: 計画と実行                                  │
│           人間がゴールを設定 → エージェントが計画を立て │
│           て自律的に実行                               │
│                                                      │
│  Level 3: 目標設定と計画                              │
│           人間が大方針を提示 → エージェントがサブゴール │
│           を設定し、計画・実行・評価を自律的に回す      │
│                                                      │
│  Level 4: 完全自律（将来の展望）                      │
│           エージェントが自ら課題を発見し、解決策を     │
│           立案・実行する                               │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 汎用エージェント（General-Purpose Agent）

特定タスクに特化したエージェントから、幅広いタスクに対応できる汎用エージェントへの発展が進んでいます。

| 特化型エージェント | 汎用エージェント |
|-------------------|----------------|
| 1つのドメインに最適化 | 複数ドメインを横断 |
| 固定のツールセット | 動的にツールを学習・追加 |
| 事前定義のワークフロー | 状況に応じてワークフローを生成 |
| 高い精度（限定領域） | 広い適用範囲（精度は発展途上） |

### 人間との協働（Human-AI Collaboration）

AIエージェントの究極的な目標は、人間を置き換えることではなく、人間の能力を拡張する（Augment）ことです。

```
┌────────────────────────────────────────────────────────┐
│  人間-AIエージェント協働モデル                          │
│                                                        │
│  ┌──────────┐                    ┌──────────┐          │
│  │  人間    │  ← 承認依頼 ←     │  AI      │          │
│  │          │  → ゴール設定 →   │ エージェント │       │
│  │  得意:   │  ← 結果報告 ←     │          │          │
│  │  創造性  │  → フィードバック → │ 得意:    │          │
│  │  判断    │                    │ 速度     │          │
│  │  倫理    │                    │ 一貫性   │          │
│  │  共感    │                    │ 網羅性   │          │
│  └──────────┘                    └──────────┘          │
│                                                        │
│  重要なのは「人間がループに入っている」こと              │
│  (Human-in-the-Loop)                                   │
└────────────────────────────────────────────────────────┘
```

### 学習を終えたあなたへ

この教材を通じて、AIエージェントの基本概念から実践的な構築方法までを学んできました。この分野は日々急速に進化しています。重要なのは以下の3点です。

1. **基礎を固める** - LLMの仕組み、プロンプト設計、ツール連携の基本は変わりません
2. **手を動かす** - 小さなエージェントから始めて、徐々に複雑なものに挑戦しましょう
3. **コミュニティに参加する** - 最新の論文、OSS、ベストプラクティスを追い続けましょう

> **よくある間違い**
> - 最初から完璧なエージェントを作ろうとする → まずは最小限の機能で動くものを作り、改善を重ねましょう（MVP: Minimum Viable Product）
> - AIエージェントに全てを任せようとする → 人間の判断が必要な場面を見極め、適切にHuman-in-the-Loopを設計しましょう

> **ポイントまとめ**
> - AIエージェントの自律性は段階的に向上していく
> - 汎用エージェントは発展途上だが、急速に進化している
> - 人間との協働（Human-in-the-Loop）が実用化の鍵
> - 基礎を固め、手を動かし、最新動向を追い続けることが大切
