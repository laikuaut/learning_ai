# 第1章：変数と所有権 - 演習問題

この演習では、Rustの変数宣言、型指定、可変性（mutability）、そしてRust最大の特徴である所有権（ownership）と借用（borrowing）について実践的に学びます。

---

## 基本問題

### 問題1：変数の宣言と型指定

以下の要件に従って変数を宣言し、それぞれの値と型を出力するプログラムを作成してください。

1. 整数型（`i32`）の変数 `age` に値 `25` を代入
2. 浮動小数点型（`f64`）の変数 `height` に値 `170.5` を代入
3. 文字列スライス型（`&str`）の変数 `name` に `"田中太郎"` を代入
4. 真偽値型（`bool`）の変数 `is_student` に `true` を代入
5. 文字型（`char`）の変数 `initial` に `'T'` を代入

**期待される出力例：**
```
名前: 田中太郎
年齢: 25歳
身長: 170.5cm
学生: true
イニシャル: T
```

<details><summary>ヒント</summary>

- Rustでは `let 変数名: 型 = 値;` で型を明示的に指定できます
- `println!` マクロでは `{}` をプレースホルダとして使います
- Rustは型推論があるため、型注釈を省略することもできますが、この問題では明示してください

</details>

<details><summary>解答例</summary>

```rust
fn main() {
    // 整数型の変数を宣言
    let age: i32 = 25;

    // 浮動小数点型の変数を宣言
    let height: f64 = 170.5;

    // 文字列スライス型の変数を宣言
    let name: &str = "田中太郎";

    // 真偽値型の変数を宣言
    let is_student: bool = true;

    // 文字型の変数を宣言
    let initial: char = 'T';

    // 各変数を出力
    println!("名前: {}", name);
    println!("年齢: {}歳", age);
    println!("身長: {}cm", height);
    println!("学生: {}", is_student);
    println!("イニシャル: {}", initial);
}
```

**補足：** Rustの基本的な型には `i8`, `i16`, `i32`, `i64`, `i128`（符号付き整数）、`u8`, `u16`, `u32`, `u64`, `u128`（符号なし整数）、`f32`, `f64`（浮動小数点）、`bool`、`char`、`&str`（文字列スライス）、`String`（所有文字列）などがあります。型を明示しない場合、整数は `i32`、浮動小数点は `f64` がデフォルトとして推論されます。

</details>

### 問題2：可変変数と再代入

変数 `score` を `0` で初期化し、以下の手順で値を更新して各段階の値を出力するプログラムを作成してください。

1. 初期値 `0` を出力
2. `score` に `50` を加算して出力
3. `score` に `30` を加算して出力
4. `score` を `2` で割って出力
5. 最終スコアを出力

**期待される出力例：**
```
初期スコア: 0
+50点: 50
+30点: 80
÷2: 40
最終スコア: 40
```

<details><summary>ヒント</summary>

- Rustでは変数はデフォルトで不変（immutable）です
- 値を変更したい場合は `let mut` を使って可変（mutable）にする必要があります
- `+=` や `/=` などの複合代入演算子が使えます

</details>

<details><summary>解答例</summary>

```rust
fn main() {
    // mut を付けて可変変数として宣言
    let mut score: i32 = 0;
    println!("初期スコア: {}", score);

    // 50を加算
    score += 50;
    println!("+50点: {}", score);

    // 30を加算
    score += 30;
    println!("+30点: {}", score);

    // 2で割る
    score /= 2;
    println!("÷2: {}", score);

    // 最終スコアを出力
    println!("最終スコア: {}", score);
}
```

**補足：** `let` で宣言した変数は不変です。`mut` を忘れると、再代入しようとした時点でコンパイルエラーになります。Rustが不変をデフォルトにしているのは、意図しない値の変更を防ぎ、安全なコードを書くためです。「変更が必要な変数だけ `mut` を付ける」という設計思想を覚えておきましょう。

</details>

### 問題3：シャドーイングの活用

シャドーイング（shadowing）を使って、以下の変換を同じ変数名 `value` で行うプログラムを作成してください。

1. `value` を文字列 `"42"` で初期化
2. `value` を整数 `42` に変換（シャドーイングで型を変更）
3. `value` を2倍にして `84` にする（シャドーイングで新しい値を束縛）
4. `value` を文字列に変換して `"結果: 84"` にする

各段階で値を出力してください。

**期待される出力例：**
```
文字列の value: "42"
整数の value: 42
2倍の value: 84
最終の value: 結果: 84
```

<details><summary>ヒント</summary>

- シャドーイングは `let` で同じ名前の変数を再宣言することです
- `mut` とは異なり、型を変更することもできます
- 文字列から整数への変換は `.parse::<i32>().unwrap()` を使います
- 整数から文字列への変換は `format!` マクロを使います

</details>

<details><summary>解答例</summary>

```rust
fn main() {
    // 文字列として初期化
    let value = "42";
    println!("文字列の value: {:?}", value);

    // シャドーイングで整数に変換（型が &str → i32 に変わる）
    let value: i32 = value.parse().unwrap();
    println!("整数の value: {}", value);

    // シャドーイングで2倍にする
    let value = value * 2;
    println!("2倍の value: {}", value);

    // シャドーイングで再び文字列に変換
    let value = format!("結果: {}", value);
    println!("最終の value: {}", value);
}
```

**補足：** シャドーイングは `let` で同名の新しい変数を作ることです。`mut` による変更とは本質的に異なります。`mut` は同じメモリ位置の値を変更しますが、シャドーイングは新しい変数を作り、古い変数を隠します。そのため型を変えることも可能です。これはRustでよく使われるパターンで、特に文字列を数値に変換する場面で頻繁に登場します。

</details>

## 応用問題

### 問題4：所有権の移動を理解する

以下のコードはコンパイルエラーになります。エラーの理由を説明し、正しく動作するように修正してください。

```rust
fn main() {
    let message = String::from("こんにちは、Rust!");
    let greeting = message;
    println!("message: {}", message);
    println!("greeting: {}", greeting);
}
```

修正後のプログラムは、`message` と `greeting` の両方を出力できるようにしてください。

**期待される出力例：**
```
message: こんにちは、Rust!
greeting: こんにちは、Rust!
```

<details><summary>ヒント</summary>

- `String` 型はヒープにデータを持つため、代入時に所有権が移動（ムーブ）します
- 所有権が移動した後の変数は使用できません
- 解決方法は主に2つあります：
  1. `.clone()` で値を複製する
  2. 参照（`&`）を使って借用する

</details>

<details><summary>解答例</summary>

**方法1：clone() で複製する**

```rust
fn main() {
    let message = String::from("こんにちは、Rust!");
    // clone() でデータのディープコピーを作成
    let greeting = message.clone();
    println!("message: {}", message);
    println!("greeting: {}", greeting);
}
```

**方法2：参照を使って借用する**

```rust
fn main() {
    let message = String::from("こんにちは、Rust!");
    // & で参照を作成（所有権は移動しない）
    let greeting = &message;
    println!("message: {}", message);
    println!("greeting: {}", greeting);
}
```

**補足：** `String` 型の変数を別の変数に代入すると、所有権が移動（ムーブ）します。これはRustの重要なルールで、メモリの二重解放を防ぐための仕組みです。

```
ムーブ前:
  message → [ヒープ上の "こんにちは、Rust!"]

ムーブ後:
  message → （無効）
  greeting → [ヒープ上の "こんにちは、Rust!"]
```

`clone()` を使うとデータの完全なコピーが作られるため両方の変数が有効になります。参照（`&`）を使うと所有権を移動せずにデータにアクセスできます。どちらを使うかは状況によりますが、不必要な `clone()` はメモリを余計に使うため、できるだけ参照を使うのがRustのベストプラクティスです。

</details>

### 問題5：参照を使った関数

文字列の長さを返す関数 `count_chars` と、文字列を大文字の説明に変換する関数 `describe` を作成してください。

要件：
- `count_chars` は `&String` を受け取り、文字数（`usize`）を返す
- `describe` は `&String` を受け取り、説明文（`String`）を返す
- どちらの関数も所有権を奪わない（呼び出し後も元の変数を使える）

**期待される出力例：**
```
テキスト: Rustは安全で高速な言語です
文字数: 14
説明: 「Rustは安全で高速な言語です」（14文字）
テキスト（まだ使える）: Rustは安全で高速な言語です
```

<details><summary>ヒント</summary>

- 関数の引数に `&String` を使うと、所有権を移動せずに値を借用できます
- 文字数は `.chars().count()` で取得できます（バイト数の `.len()` とは異なります）
- `format!` マクロで新しい `String` を作成できます

</details>

<details><summary>解答例</summary>

```rust
// 文字列の文字数を返す関数（参照で借用するので所有権は移動しない）
fn count_chars(text: &String) -> usize {
    // chars() でUnicode文字単位のイテレータを作り、count() で数える
    text.chars().count()
}

// 文字列の説明文を返す関数
fn describe(text: &String) -> String {
    let len = text.chars().count();
    // format! で新しい String を作成して返す
    format!("「{}」（{}文字）", text, len)
}

fn main() {
    let text = String::from("Rustは安全で高速な言語です");

    println!("テキスト: {}", text);

    // & を付けて参照を渡す（所有権は移動しない）
    let char_count = count_chars(&text);
    println!("文字数: {}", char_count);

    let description = describe(&text);
    println!("説明: {}", description);

    // 所有権が移動していないので、まだ text を使える
    println!("テキスト（まだ使える）: {}", text);
}
```

**補足：** 関数に `String` をそのまま渡すと所有権が移動してしまい、関数呼び出し後にその変数は使えなくなります。`&String`（参照）を使うことで、所有権を移動させずにデータにアクセスできます。これを「借用（borrowing）」と呼びます。

なお、実務では `&String` よりも `&str`（文字列スライス）を引数の型にするのがより慣用的です。`&str` にすると `String` も `&str` もどちらも受け取れるようになるためです。

```rust
// より慣用的な書き方
fn count_chars(text: &str) -> usize {
    text.chars().count()
}
```

</details>

### 問題6：可変参照を使った値の変更

整数のベクタ（`Vec<i32>`）を受け取り、全要素を2倍にする関数 `double_all` を作成してください。

要件：
- `double_all` は可変参照（`&mut Vec<i32>`）を受け取る
- 元のベクタを直接変更する（新しいベクタを返さない）
- 変更前と変更後の状態を出力する

**期待される出力例：**
```
変更前: [1, 2, 3, 4, 5]
変更後: [2, 4, 6, 8, 10]
```

<details><summary>ヒント</summary>

- 可変参照を受け取るには引数を `&mut Vec<i32>` にします
- 関数に渡す際は `&mut 変数名` とします
- ベクタの要素を変更するには `for item in vec.iter_mut()` を使います
- `*item` でデリファレンスして値を変更できます

</details>

<details><summary>解答例</summary>

```rust
// 可変参照でベクタを受け取り、全要素を2倍にする
fn double_all(numbers: &mut Vec<i32>) {
    // iter_mut() で可変イテレータを取得
    for num in numbers.iter_mut() {
        // * でデリファレンスして値を変更
        *num *= 2;
    }
}

fn main() {
    // mut で可変変数として宣言
    let mut numbers = vec![1, 2, 3, 4, 5];

    println!("変更前: {:?}", numbers);

    // &mut で可変参照を渡す
    double_all(&mut numbers);

    println!("変更後: {:?}", numbers);
}
```

**補足：** Rustの借用ルールでは、ある変数に対して以下のいずれかしか同時に存在できません。

1. 不変参照（`&T`）はいくつでも同時に存在できる
2. 可変参照（`&mut T`）は同時に1つだけ

```
不変参照（OK: 複数同時に可能）:
  &numbers ──→ [1, 2, 3, 4, 5]  ← 読み取り専用
  &numbers ──→ [1, 2, 3, 4, 5]  ← 読み取り専用

可変参照（OK: 1つだけ）:
  &mut numbers ──→ [1, 2, 3, 4, 5]  ← 読み書き可能

不変参照と可変参照の同時使用（NG: コンパイルエラー）:
  &numbers ──→ [1, 2, 3, 4, 5]
  &mut numbers ──→ [1, 2, 3, 4, 5]  ← 同時にはできない!
```

このルールにより、データ競合が原理的に発生しないことがコンパイル時に保証されます。

</details>

## チャレンジ問題

### 問題7：借用ルールを正しく適用する

以下の要件を満たすプログラムを作成してください。

1. 構造体 `BankAccount` を定義する（フィールド：`owner: String`, `balance: f64`）
2. 口座情報を表示する関数 `display_account`（不変参照で受け取る）
3. 入金する関数 `deposit`（可変参照で受け取る）
4. 出金する関数 `withdraw`（可変参照で受け取る、残高不足ならエラーメッセージを表示）
5. 残高を比較する関数 `has_more_balance`（2つの不変参照を受け取り、`bool` を返す）

**期待される出力例：**
```
=== 口座情報 ===
口座名義: 田中太郎
残高: 10000.00円

--- 入金: 5000.00円 ---
口座名義: 田中太郎
残高: 15000.00円

--- 出金: 3000.00円 ---
口座名義: 田中太郎
残高: 12000.00円

--- 出金: 20000.00円 ---
エラー: 残高不足です（現在の残高: 12000.00円）
口座名義: 田中太郎
残高: 12000.00円

=== 口座比較 ===
口座名義: 佐藤花子
残高: 20000.00円
田中太郎より佐藤花子の方が残高が多いです
```

<details><summary>ヒント</summary>

- 構造体は `struct BankAccount { ... }` で定義します
- 不変参照 `&BankAccount` で口座情報を読み取り、可変参照 `&mut BankAccount` で残高を変更します
- 浮動小数点の表示は `{:.2}` で小数点以下2桁に指定できます
- 2つの口座を比較する際は、両方を不変参照で受け取ります（同時に複数の不変参照が可能）

</details>

<details><summary>解答例</summary>

```rust
// 銀行口座を表す構造体
struct BankAccount {
    owner: String,
    balance: f64,
}

// 口座情報を表示する関数（不変参照で借用）
fn display_account(account: &BankAccount) {
    println!("口座名義: {}", account.owner);
    println!("残高: {:.2}円", account.balance);
}

// 入金する関数（可変参照で借用して残高を変更）
fn deposit(account: &mut BankAccount, amount: f64) {
    println!("\n--- 入金: {:.2}円 ---", amount);
    account.balance += amount;
    display_account(account);
}

// 出金する関数（残高不足チェック付き）
fn withdraw(account: &mut BankAccount, amount: f64) {
    println!("\n--- 出金: {:.2}円 ---", amount);
    if account.balance >= amount {
        account.balance -= amount;
    } else {
        println!("エラー: 残高不足です（現在の残高: {:.2}円）", account.balance);
    }
    display_account(account);
}

// 2つの口座の残高を比較する関数（両方を不変参照で借用）
fn has_more_balance(a: &BankAccount, b: &BankAccount) -> bool {
    a.balance > b.balance
}

fn main() {
    // 田中太郎の口座を作成
    let mut account1 = BankAccount {
        owner: String::from("田中太郎"),
        balance: 10000.0,
    };

    println!("=== 口座情報 ===");
    display_account(&account1);

    // 入金と出金
    deposit(&mut account1, 5000.0);
    withdraw(&mut account1, 3000.0);
    withdraw(&mut account1, 20000.0);

    // 佐藤花子の口座を作成
    let account2 = BankAccount {
        owner: String::from("佐藤花子"),
        balance: 20000.0,
    };

    println!("\n=== 口座比較 ===");
    display_account(&account2);

    // 2つの不変参照を同時に渡す（不変参照は複数同時にOK）
    if has_more_balance(&account1, &account2) {
        println!("{}より{}の方が残高が多いです", account2.owner, account1.owner);
    } else {
        println!("{}より{}の方が残高が多いです", account1.owner, account2.owner);
    }
}
```

**補足：** この問題では、Rustの借用ルールの3つのパターンを全て使っています。

1. **不変参照（`&T`）** : `display_account` と `has_more_balance` で使用。データを読み取るだけなので不変参照で十分です。
2. **可変参照（`&mut T`）** : `deposit` と `withdraw` で使用。残高を変更するため可変参照が必要です。
3. **複数の不変参照の同時使用** : `has_more_balance` で2つの口座を同時に不変参照で受け取っています。不変参照は複数同時に存在できるため、これは問題ありません。

重要なポイントは、可変参照を使っている間は不変参照を作れないことです。例えば、`deposit` の呼び出し中に別の場所で `&account1` を使うことはできません。Rustコンパイラがこのルールを強制することで、データ競合を防いでいます。

</details>

### 問題8：ライフタイムの基礎

2つの文字列スライスのうち、長い方を返す関数 `longer` を作成してください。

要件：
- 関数 `longer` は2つの `&str` を受け取り、文字数が多い方の `&str` を返す
- 文字数が同じ場合は最初の引数を返す
- ライフタイム注釈を正しく付ける
- 様々な文字列のペアで動作を確認する

**期待される出力例：**
```
比較: "Rust" vs "Python"
長い方: Python

比較: "こんにちは" vs "Hi"
長い方: こんにちは

比較: "ABC" vs "XYZ"
長い方（同じ長さ）: ABC
```

<details><summary>ヒント</summary>

- 参照を返す関数では、返される参照のライフタイムを指定する必要があります
- ライフタイム注釈は `'a` のように書き、関数シグネチャに付けます
- `fn longer<'a>(s1: &'a str, s2: &'a str) -> &'a str` のように書きます
- 文字数は `.chars().count()` で取得します

</details>

<details><summary>解答例</summary>

```rust
// ライフタイム 'a を使って、返される参照が引数と同じ期間有効であることを示す
fn longer<'a>(s1: &'a str, s2: &'a str) -> &'a str {
    // chars().count() でUnicode文字数を比較
    if s1.chars().count() >= s2.chars().count() {
        s1
    } else {
        s2
    }
}

fn main() {
    // テスト1: 英語の文字列
    let s1 = "Rust";
    let s2 = "Python";
    println!("比較: {:?} vs {:?}", s1, s2);
    println!("長い方: {}", longer(s1, s2));

    println!();

    // テスト2: 日本語と英語
    let s3 = "こんにちは";
    let s4 = "Hi";
    println!("比較: {:?} vs {:?}", s3, s4);
    println!("長い方: {}", longer(s3, s4));

    println!();

    // テスト3: 同じ長さ
    let s5 = "ABC";
    let s6 = "XYZ";
    println!("比較: {:?} vs {:?}", s5, s6);
    println!("長い方（同じ長さ）: {}", longer(s5, s6));
}
```

**補足：** ライフタイム注釈 `'a` は、「この関数が返す参照は、引数 `s1` と `s2` のうち短い方のライフタイムと同じ期間だけ有効である」ということをコンパイラに伝えます。

```
ライフタイムの概念図:

fn longer<'a>(s1: &'a str, s2: &'a str) -> &'a str
              ^^^           ^^^            ^^^
              |             |              |
              +-------------+--------------+
              これら全てが同じライフタイム 'a

s1 のスコープ: |===================|
s2 のスコープ:     |==============|
'a の有効期間:     |==============|  ← 短い方に合わせる
戻り値の有効期間:   |==============|  ← 'a と同じ
```

ライフタイムはRust初心者にとって最も難しい概念の一つですが、「参照が無効なメモリを指すこと（ダングリング参照）を防ぐ仕組み」と理解すれば、その意義がわかります。コンパイラはライフタイム注釈を使って、参照が常に有効なデータを指していることを保証します。
