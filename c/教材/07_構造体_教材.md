# 第7章：構造体（struct）

## この章のゴール

- 構造体（struct）を使って複数のデータを1つにまとめられるようになる
- typedef による型エイリアスを理解し活用できる
- 構造体の配列やポインタを使った操作ができる
- 構造体を関数に渡す方法（値渡し・ポインタ渡し）を理解する
- 共用体（union）と列挙型（enum）の基本を理解する
- 実用的なプログラム（住所録・学生管理）を構造体で設計できる

---

## 1. 構造体とは

これまで学んだ配列は「同じ型のデータの集まり」でした。しかし、実際のプログラムでは「名前（文字列）」「年齢（整数）」「身長（小数）」のように**異なる型のデータをひとまとめに**したい場面が多くあります。

構造体（structure）は、異なる型の変数をまとめて1つの新しいデータ型として扱う仕組みです。

```
┌─────────────────────────────────┐
│         構造体 Student          │
├─────────────┬───────────────────┤
│ name        │ "田中太郎"        │  char[50]
├─────────────┼───────────────────┤
│ age         │ 20                │  int
├─────────────┼───────────────────┤
│ gpa         │ 3.5               │  double
└─────────────┴───────────────────┘
```

### 構造体の定義と使い方

```c
#include <stdio.h>
#include <string.h>

/* 構造体の定義 */
struct Student {
    char name[50];
    int age;
    double gpa;
};

int main(void) {
    /* 構造体変数の宣言と初期化 */
    struct Student s1 = {"田中太郎", 20, 3.5};

    /* メンバへのアクセス（ドット演算子） */
    printf("名前: %s\n", s1.name);
    printf("年齢: %d\n", s1.age);
    printf("GPA:  %.1f\n", s1.gpa);

    /* メンバへの代入 */
    struct Student s2;
    strcpy(s2.name, "佐藤花子");
    s2.age = 22;
    s2.gpa = 3.8;

    printf("\n名前: %s\n", s2.name);
    printf("年齢: %d\n", s2.age);
    printf("GPA:  %.1f\n", s2.gpa);

    return 0;
}
```

構造体の各変数のことを**メンバ（member）**と呼びます。メンバにアクセスするには**ドット演算子（`.`）**を使います。

> **注意：** 文字列メンバへの代入は `=` ではなく `strcpy()` を使います。`s2.name = "佐藤花子"` はコンパイルエラーになります。

### ポイントまとめ
- 構造体は `struct タグ名 { メンバ; };` で定義する
- メンバへは `.`（ドット演算子）でアクセスする
- 宣言時に `{}` で初期化できる（メンバの順番通り）
- 文字列メンバへの代入には `strcpy()` を使う

---

## 2. typedef による型エイリアス

毎回 `struct Student` と書くのは冗長です。`typedef` を使うと短い名前を付けられます。

```c
#include <stdio.h>

/* typedef で型エイリアスを定義 */
typedef struct {
    char name[50];
    int age;
    double gpa;
} Student;

int main(void) {
    /* struct を省略して宣言できる */
    Student s1 = {"鈴木一郎", 21, 3.2};

    printf("名前: %s, 年齢: %d, GPA: %.1f\n",
           s1.name, s1.age, s1.gpa);

    return 0;
}
```

`typedef` を使うと、以降は `Student` だけで型として使えます。C言語の実務コードでは `typedef` を使うのが一般的です。

### ポイントまとめ
- `typedef struct { ... } 型名;` で簡潔な型名を定義できる
- 実務では typedef を使うのが標準的なスタイル

---

## 3. 構造体の配列

同じ構造体型の変数を複数扱うには、構造体の配列を使います。

```c
#include <stdio.h>

typedef struct {
    char name[50];
    int score;
} Student;

int main(void) {
    Student class[] = {
        {"田中", 85},
        {"佐藤", 92},
        {"鈴木", 78},
        {"山田", 95},
        {"高橋", 88}
    };
    int n = sizeof(class) / sizeof(class[0]);

    /* 全員の情報を表示 */
    printf("%-10s %s\n", "名前", "点数");
    printf("--------------------\n");
    for (int i = 0; i < n; i++) {
        printf("%-10s %d\n", class[i].name, class[i].score);
    }

    /* 平均点を計算 */
    int total = 0;
    for (int i = 0; i < n; i++) {
        total += class[i].score;
    }
    printf("--------------------\n");
    printf("平均点:    %.1f\n", (double)total / n);

    return 0;
}
```

```
メモリ上のイメージ（構造体配列）:

class[0]         class[1]         class[2]
┌──────┬──────┐ ┌──────┬──────┐ ┌──────┬──────┐
│ name │score │ │ name │score │ │ name │score │ ...
│"田中"│  85  │ │"佐藤"│  92  │ │"鈴木"│  78  │
└──────┴──────┘ └──────┴──────┘ └──────┴──────┘
```

### ポイントまとめ
- 構造体も配列にできる：`Student arr[10];`
- 各要素へは `arr[i].メンバ名` でアクセスする
- `sizeof` で要素数を計算するテクニックは配列と同じ

---

## 4. 構造体とポインタ（アロー演算子）

第6章で学んだポインタは構造体にも使えます。構造体へのポインタからメンバにアクセスするには**アロー演算子（`->`）**を使います。

```c
#include <stdio.h>

typedef struct {
    char name[50];
    int age;
} Person;

int main(void) {
    Person p = {"渡辺直美", 35};
    Person *ptr = &p;

    /* ドット演算子（変数から直接アクセス） */
    printf("名前: %s\n", p.name);

    /* アロー演算子（ポインタ経由でアクセス） */
    printf("名前: %s\n", ptr->name);
    printf("年齢: %d\n", ptr->age);

    /* ptr->age は (*ptr).age と同じ意味 */
    printf("年齢: %d\n", (*ptr).age);

    /* ポインタ経由でメンバを変更 */
    ptr->age = 36;
    printf("更新後の年齢: %d\n", p.age);

    return 0;
}
```

```
変数とポインタの関係:

  ptr              p
┌──────┐    ┌──────────────┐
│ &p ──│───>│ name: "渡辺" │
└──────┘    │ age:  35     │
            └──────────────┘

  ptr->age  は  (*ptr).age  と同じ
```

`->` 演算子は「ポインタが指す先の構造体のメンバにアクセスする」ための省略記法です。`(*ptr).age` と書いても同じですが、`ptr->age` の方が読みやすいため、こちらが一般的に使われます。

### ポイントまとめ
- 構造体ポインタからのアクセスには `->` を使う
- `ptr->member` は `(*ptr).member` と同じ意味
- 実務では `->` が圧倒的に多く使われる

## 5. 構造体を関数に渡す

### 値渡し（コピーが渡される）

```c
#include <stdio.h>

typedef struct {
    char name[50];
    int age;
} Person;

void print_person(Person p) {
    printf("名前: %s, 年齢: %d\n", p.name, p.age);
}

/* 値渡しでは元の構造体は変わらない */
void try_change_age(Person p, int new_age) {
    p.age = new_age;  /* コピーを変更しているだけ */
}

int main(void) {
    Person p = {"田中", 25};
    print_person(p);

    try_change_age(p, 30);
    printf("変更後: 年齢 = %d（変わっていない）\n", p.age);

    return 0;
}
```

### ポインタ渡し（元のデータを直接操作）

```c
#include <stdio.h>

typedef struct {
    char name[50];
    int age;
} Person;

void print_person(const Person *p) {
    printf("名前: %s, 年齢: %d\n", p->name, p->age);
}

/* ポインタ渡しでは元の構造体を変更できる */
void change_age(Person *p, int new_age) {
    p->age = new_age;
}

int main(void) {
    Person p = {"田中", 25};
    print_person(&p);

    change_age(&p, 30);
    printf("変更後: 年齢 = %d（変更された）\n", p.age);

    return 0;
}
```

```
値渡し:                          ポインタ渡し:

main        print_person         main        print_person
┌──────┐    ┌──────┐            ┌──────┐    ┌──────┐
│ p    │    │ p    │ (コピー)   │ p    │<───│ *p   │
│ 田中 │    │ 田中 │            │ 田中 │    └──────┘
│ 25   │    │ 25   │            │ 25   │
└──────┘    └──────┘            └──────┘
  別々のメモリ                    同じメモリを参照
```

**実務では構造体はポインタ渡しが基本です。** 理由は2つあります。

1. **効率性：** 構造体のコピーが発生しないため高速
2. **メモリ節約：** 大きな構造体でもポインタ（8バイト）だけ渡せばよい

変更しない場合は `const` を付けて安全性を確保します。

### ポイントまとめ
- 値渡し：構造体のコピーが作られる（元データは変わらない）
- ポインタ渡し：元データを直接操作できる（`->` でアクセス）
- 実務ではポインタ渡し + `const` が定番パターン

## 6. ネストした構造体

構造体のメンバとして別の構造体を含めることができます。

```c
#include <stdio.h>

typedef struct {
    int year;
    int month;
    int day;
} Date;

typedef struct {
    char postal_code[10];
    char prefecture[20];
    char city[50];
    char street[100];
} Address;

typedef struct {
    char name[50];
    Date birthday;
    Address address;
} Person;

int main(void) {
    Person p = {
        "山田太郎",
        {1995, 4, 15},
        {"100-0001", "東京都", "千代田区", "千代田1-1-1"}
    };

    printf("名前:     %s\n", p.name);
    printf("誕生日:   %d年%d月%d日\n",
           p.birthday.year, p.birthday.month, p.birthday.day);
    printf("住所:     〒%s %s%s%s\n",
           p.address.postal_code,
           p.address.prefecture,
           p.address.city,
           p.address.street);

    return 0;
}
```

ネストした構造体のメンバへは、ドット演算子を連結してアクセスします（例：`p.birthday.year`）。

### ポイントまとめ
- 構造体の中に構造体を含められる（ネスト）
- アクセスはドット演算子を連結：`変数.外側メンバ.内側メンバ`
- データを論理的にグループ化して整理できる

## 7. 共用体（union）

共用体（union）は構造体に似ていますが、**全メンバが同じメモリ領域を共有**します。一度に使えるメンバは1つだけです。

```c
#include <stdio.h>

typedef union {
    int i;
    float f;
    char str[20];
} Data;

int main(void) {
    Data d;

    d.i = 42;
    printf("整数: %d\n", d.i);

    d.f = 3.14f;
    printf("小数: %.2f\n", d.f);
    printf("整数（壊れている）: %d\n", d.i);  /* f を書き込んだので不正な値 */

    printf("\nsizeof(int)   = %zu\n", sizeof(int));
    printf("sizeof(float) = %zu\n", sizeof(float));
    printf("sizeof(Data)  = %zu（最大メンバのサイズ）\n", sizeof(Data));

    return 0;
}
```

```
構造体 (struct) のメモリ:      共用体 (union) のメモリ:

┌──────┐                       ┌──────────────┐
│ int  │  4バイト              │              │
├──────┤                       │  全メンバが   │
│float │  4バイト              │  同じ領域を   │  最大メンバの
├──────┤                       │  共有する     │  サイズだけ確保
│str[20]│ 20バイト             │              │
└──────┘                       └──────────────┘
合計: 28バイト以上              合計: 20バイト
```

共用体は、同じメモリ領域を異なる型で解釈したい場合に使います。使用頻度は構造体ほど高くありませんが、組み込みプログラミングやプロトコル解析で活躍します。

### ポイントまとめ
- `union` は全メンバが同じメモリを共有する
- サイズは最大メンバのサイズになる
- 一度に有効なメンバは1つだけ

## 8. 列挙型（enum）

列挙型（enumeration）は、関連する定数に名前を付けてまとめる仕組みです。

```c
#include <stdio.h>

/* 列挙型の定義 */
typedef enum {
    MON, TUE, WED, THU, FRI, SAT, SUN
} Weekday;
/* MON=0, TUE=1, WED=2, ... と自動的に連番が振られる */

/* 値を明示的に指定することもできる */
typedef enum {
    RED   = 1,
    GREEN = 2,
    BLUE  = 4
} Color;

const char* weekday_name(Weekday day) {
    const char *names[] = {
        "月曜日", "火曜日", "水曜日", "木曜日",
        "金曜日", "土曜日", "日曜日"
    };
    return names[day];
}

int main(void) {
    Weekday today = WED;
    printf("今日は%sです\n", weekday_name(today));

    /* enum の値は整数として扱える */
    printf("WED の値: %d\n", WED);

    /* switch 文との相性が良い */
    Color c = GREEN;
    switch (c) {
        case RED:   printf("赤\n"); break;
        case GREEN: printf("緑\n"); break;
        case BLUE:  printf("青\n"); break;
    }

    return 0;
}
```

列挙型を使うと、マジックナンバー（意味不明な数値）を避けてコードの可読性を高められます。`#define` で定数を定義するよりも型安全で、デバッガでも名前が表示されるため便利です。

### ポイントまとめ
- `enum` は関連する整数定数に名前を付ける仕組み
- デフォルトは 0 から連番、明示的に値を指定することも可能
- `switch` 文との組み合わせが定番パターン
- マジックナンバーの排除に有効

## 9. 実用例：住所録プログラム

学んだ内容を組み合わせた実用的なプログラムです。

```c
#include <stdio.h>
#include <string.h>

#define MAX_CONTACTS 100

typedef struct {
    char name[50];
    char phone[20];
    char email[50];
} Contact;

typedef struct {
    Contact entries[MAX_CONTACTS];
    int count;
} AddressBook;

/* 連絡先を追加する */
void add_contact(AddressBook *book, const char *name,
                 const char *phone, const char *email) {
    if (book->count >= MAX_CONTACTS) {
        printf("エラー: 住所録がいっぱいです\n");
        return;
    }
    Contact *c = &book->entries[book->count];
    strcpy(c->name, name);
    strcpy(c->phone, phone);
    strcpy(c->email, email);
    book->count++;
    printf("追加しました: %s\n", name);
}

/* 名前で検索する */
void search_by_name(const AddressBook *book, const char *keyword) {
    int found = 0;
    printf("\n--- 検索結果: \"%s\" ---\n", keyword);
    for (int i = 0; i < book->count; i++) {
        if (strstr(book->entries[i].name, keyword) != NULL) {
            const Contact *c = &book->entries[i];
            printf("名前: %s\n", c->name);
            printf("電話: %s\n", c->phone);
            printf("Email: %s\n\n", c->email);
            found++;
        }
    }
    if (found == 0) {
        printf("該当する連絡先が見つかりません\n");
    }
}

/* 全件表示 */
void print_all(const AddressBook *book) {
    printf("\n=== 住所録一覧（%d件） ===\n", book->count);
    printf("%-15s %-15s %s\n", "名前", "電話番号", "Email");
    printf("----------------------------------------------\n");
    for (int i = 0; i < book->count; i++) {
        const Contact *c = &book->entries[i];
        printf("%-15s %-15s %s\n", c->name, c->phone, c->email);
    }
}

int main(void) {
    AddressBook book = {.count = 0};

    add_contact(&book, "田中太郎", "090-1234-5678", "tanaka@example.com");
    add_contact(&book, "佐藤花子", "080-9876-5432", "sato@example.com");
    add_contact(&book, "田中次郎", "070-1111-2222", "jiro@example.com");
    add_contact(&book, "鈴木一郎", "090-3333-4444", "suzuki@example.com");

    print_all(&book);
    search_by_name(&book, "田中");

    return 0;
}
```

## 10. 実用例：学生成績管理プログラム

構造体・列挙型・関数を組み合わせた総合的な例です。

```c
#include <stdio.h>
#include <string.h>

#define MAX_STUDENTS 50

typedef enum {
    GRADE_S,  /* 90点以上 */
    GRADE_A,  /* 80点以上 */
    GRADE_B,  /* 70点以上 */
    GRADE_C,  /* 60点以上 */
    GRADE_F   /* 60点未満（不合格） */
} Grade;

typedef struct {
    int student_id;
    char name[50];
    int scores[3];  /* 国語, 数学, 英語 */
    double average;
    Grade grade;
} Student;

/* 成績を判定する */
Grade calc_grade(double avg) {
    if (avg >= 90) return GRADE_S;
    if (avg >= 80) return GRADE_A;
    if (avg >= 70) return GRADE_B;
    if (avg >= 60) return GRADE_C;
    return GRADE_F;
}

/* 成績をアルファベットに変換する */
const char* grade_to_str(Grade g) {
    const char *labels[] = {"S", "A", "B", "C", "F"};
    return labels[g];
}

/* 学生データを初期化する */
void init_student(Student *s, int id, const char *name,
                  int jp, int math, int eng) {
    s->student_id = id;
    strcpy(s->name, name);
    s->scores[0] = jp;
    s->scores[1] = math;
    s->scores[2] = eng;
    s->average = (jp + math + eng) / 3.0;
    s->grade = calc_grade(s->average);
}

/* 一覧表示 */
void print_students(const Student students[], int count) {
    printf("\n%-4s %-12s %4s %4s %4s %6s %s\n",
           "ID", "名前", "国語", "数学", "英語", "平均", "評価");
    printf("---------------------------------------------------\n");
    for (int i = 0; i < count; i++) {
        const Student *s = &students[i];
        printf("%-4d %-12s %4d %4d %4d %6.1f   %s\n",
               s->student_id, s->name,
               s->scores[0], s->scores[1], s->scores[2],
               s->average, grade_to_str(s->grade));
    }
}

int main(void) {
    Student students[MAX_STUDENTS];
    int count = 0;

    init_student(&students[count++], 1, "田中太郎", 85, 90, 78);
    init_student(&students[count++], 2, "佐藤花子", 92, 88, 95);
    init_student(&students[count++], 3, "鈴木一郎", 70, 65, 72);
    init_student(&students[count++], 4, "山田美咲", 55, 48, 60);
    init_student(&students[count++], 5, "高橋健太", 80, 85, 82);

    print_students(students, count);

    /* 最高平均点の学生を探す */
    int best = 0;
    for (int i = 1; i < count; i++) {
        if (students[i].average > students[best].average) {
            best = i;
        }
    }
    printf("\n最高成績: %s（平均 %.1f点、評価 %s）\n",
           students[best].name,
           students[best].average,
           grade_to_str(students[best].grade));

    return 0;
}
```

### ポイントまとめ
- 構造体に関数を組み合わせることでデータと操作を整理できる
- `const` ポインタで読み取り専用を明示する習慣をつける
- 列挙型で状態や分類を表現するとコードが読みやすくなる

## よくある間違い

### 1. 文字列メンバへの `=` 代入

```c
/* NG: コンパイルエラー */
Student s;
s.name = "田中";

/* OK: strcpy を使う */
strcpy(s.name, "田中");
```

### 2. 構造体の比較に `==` を使う

```c
/* NG: 構造体同士の == 比較はできない */
if (s1 == s2) { ... }

/* OK: メンバごとに比較する */
if (strcmp(s1.name, s2.name) == 0 && s1.age == s2.age) { ... }
```

### 3. ポインタなのにドット演算子を使う

```c
Student *ptr = &s;

/* NG: ポインタにはドット演算子は使えない */
printf("%s\n", ptr.name);

/* OK: アロー演算子を使う */
printf("%s\n", ptr->name);
```

### 4. 構造体定義の末尾のセミコロン忘れ

```c
/* NG: セミコロンがない */
struct Student {
    char name[50];
    int age;
}

/* OK */
struct Student {
    char name[50];
    int age;
};  /* ← この ; を忘れがち */
```

### 5. 値渡しで元データが変わると思い込む

```c
void update(Student s) {
    s.age = 30;  /* コピーを変更しているだけ！ */
}

/* 元のデータを変更したい場合はポインタ渡しにする */
void update(Student *s) {
    s->age = 30;  /* 元データが変わる */
}
```

## 章のまとめ

| 項目 | 説明 |
|------|------|
| `struct` | 異なる型のデータをまとめた新しい型 |
| `.` 演算子 | 構造体変数からメンバにアクセス |
| `->` 演算子 | 構造体ポインタからメンバにアクセス |
| `typedef` | 型に別名を付けて簡潔に書けるようにする |
| `union` | 全メンバが同じメモリを共有する型 |
| `enum` | 関連する整数定数に名前を付ける型 |
| 値渡し | 構造体のコピーが渡される（元は不変） |
| ポインタ渡し | 元の構造体を直接操作できる |

次の章では、ファイル入出力（file I/O）について学びます。構造体のデータをファイルに保存・読み込みする方法を扱います。
