# 実践課題05：リモートリポジトリ連携 ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第2章（基本操作）、第3章（ブランチとマージ）、第4章（リモートリポジトリ）
> **課題の種類**: ミニプロジェクト
> **学習目標**: ローカルとリモートの関係を理解し、push・pull・fetch・cloneの操作を確実に使えるようにする

## 完成イメージ

```
$ git remote -v
origin  /tmp/remote-repo.git (fetch)
origin  /tmp/remote-repo.git (push)

$ git branch -a
* main
  remotes/origin/main

$ git log --oneline --all
b2c3d4e (HEAD -> main, origin/main) ローカルで追加した変更
a1b2c3d 初回コミット
```

---

## 課題の要件

この課題では、GitHubを使わずにローカルマシン上でリモートリポジトリをシミュレーションします。
（`--bare` リポジトリをリモートとして使います。）

1. ベアリポジトリ（bare repository）をリモートとして作成する
2. 別のディレクトリで `git clone` する
3. クローンしたリポジトリで変更を加えて `git push` する
4. 元のディレクトリから `git fetch` / `git pull` で変更を取得する
5. `git remote` コマンドでリモートの情報を確認する

---

## ステップガイド

<details>
<summary>ステップ1：ベアリポジトリ（リモート役）を作成する</summary>

ベアリポジトリ（bare repository）は、作業ディレクトリを持たないリポジトリです。GitHubやGitLabのサーバー上のリポジトリはこの形式です。

```bash
# リモート役のベアリポジトリを作成
git init --bare /tmp/remote-repo.git
```

</details>

<details>
<summary>ステップ2：ローカルリポジトリを作成してリモートに push する</summary>

```bash
# 作業用ディレクトリを作成
mkdir /tmp/local-repo-A
cd /tmp/local-repo-A
git init

# ファイルを作成してコミット
echo "# 共同プロジェクト" > README.md
git add README.md
git commit -m "初回コミット"

# リモートを登録して push
git remote add origin /tmp/remote-repo.git
git push -u origin main
```

</details>

<details>
<summary>ステップ3：別のディレクトリで clone する</summary>

```bash
# 別の「開発者B」のつもりで clone
git clone /tmp/remote-repo.git /tmp/local-repo-B
cd /tmp/local-repo-B

# リモートの情報を確認
git remote -v

# ファイルがあることを確認
cat README.md
```

</details>

<details>
<summary>ステップ4：clone 先で変更して push する</summary>

```bash
cd /tmp/local-repo-B

# ファイルを追加
echo "## 機能一覧" >> README.md
echo "- ユーザー管理" >> README.md
git add README.md
git commit -m "機能一覧を追加"

# リモートに push
git push origin main
```

</details>

<details>
<summary>ステップ5：元のリポジトリで変更を取得する</summary>

```bash
cd /tmp/local-repo-A

# 現在の状態を確認（まだ古いまま）
cat README.md

# リモートの変更を取得
git fetch origin
# fetch はダウンロードするだけで、作業ディレクトリには反映しない

# リモートとの差分を確認
git diff main origin/main

# 作業ディレクトリに反映（マージ）
git pull origin main

# 変更が反映されたことを確認
cat README.md
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け：各ステップで状態を確認）</summary>

```bash
# ===== 1. リモート役のベアリポジトリ作成 =====
git init --bare /tmp/remote-repo.git
# 出力例: Initialized empty Git repository in /tmp/remote-repo.git/

# ===== 2. 開発者A のローカルリポジトリ =====
mkdir /tmp/local-repo-A
cd /tmp/local-repo-A
git init

echo "# 共同プロジェクト" > README.md
git add README.md
git commit -m "初回コミット"

# リモートを登録
git remote add origin /tmp/remote-repo.git

# リモート情報を確認
git remote -v
# 出力例:
# origin  /tmp/remote-repo.git (fetch)
# origin  /tmp/remote-repo.git (push)

# 初回 push（-u でトラッキングブランチを設定）
git push -u origin main

# ===== 3. 開発者B として clone =====
git clone /tmp/remote-repo.git /tmp/local-repo-B
cd /tmp/local-repo-B

# 状態確認
git remote -v
git log --oneline
cat README.md

# ===== 4. 開発者B が変更して push =====
echo "## 機能一覧" >> README.md
echo "- ユーザー管理" >> README.md
echo "- 商品管理" >> README.md

git add README.md
git commit -m "機能一覧を追加"
git push origin main

# ===== 5. 開発者A が変更を取得 =====
cd /tmp/local-repo-A

# まだ古い内容
cat README.md
# 出力: # 共同プロジェクト

# fetch で取得（作業ディレクトリはまだ変わらない）
git fetch origin

# リモートとの差分を確認
git log --oneline main..origin/main
# 出力例: b2c3d4e 機能一覧を追加

git diff main origin/main

# pull で作業ディレクトリに反映
git pull origin main

# 変更が反映された
cat README.md
# 出力:
# # 共同プロジェクト
# ## 機能一覧
# - ユーザー管理
# - 商品管理

# ===== 後片付け =====
rm -rf /tmp/remote-repo.git /tmp/local-repo-A /tmp/local-repo-B
```

</details>

<details>
<summary>解答例（改良版 ─ fetch と pull の違いを深く理解する）</summary>

```bash
# ===== 準備（初心者向けと同じ手順でリポジトリを作成）=====
git init --bare /tmp/remote-repo.git

mkdir /tmp/local-repo-A
cd /tmp/local-repo-A
git init
echo "# 共同プロジェクト" > README.md
git add README.md
git commit -m "初回コミット"
git remote add origin /tmp/remote-repo.git
git push -u origin main

git clone /tmp/remote-repo.git /tmp/local-repo-B

# ===== 開発者B が変更 =====
cd /tmp/local-repo-B
echo "B が追加した行" >> README.md
git add README.md
git commit -m "B: README を更新"
git push origin main

# ===== 開発者A：fetch vs pull の違いを体験 =====
cd /tmp/local-repo-A

# --- fetch の動作 ---
git fetch origin

# fetch 後の状態を確認
git log --oneline --all
# 出力例:
# b2c3d4e (origin/main) B: README を更新
# a1b2c3d (HEAD -> main) 初回コミット
# ※ HEAD -> main は古いまま、origin/main だけ進んでいる

# ローカルの main とリモートの main の差を見る
git log main..origin/main --oneline
# 出力: b2c3d4e B: README を更新

# 差分を見てからマージするかどうか決められる
git diff main origin/main

# 手動でマージ
git merge origin/main

# --- pull = fetch + merge のショートカット ---
# 次回からは git pull origin main で一発

# ===== fetch が安全な理由 =====
# pull は即座にマージするため、コンフリクトが予期せず発生することがある
# fetch → diff で確認 → merge の手順なら安全にレビューしてからマージできる

# ===== リモートブランチの管理 =====
# リモートのブランチ一覧
git branch -r

# すべてのブランチ（ローカル＋リモート）
git branch -a

# リモートの詳細情報
git remote show origin

# ===== 後片付け =====
rm -rf /tmp/remote-repo.git /tmp/local-repo-A /tmp/local-repo-B
```

**初心者向けとの違い:**
- `fetch` → `diff` → `merge` の安全な手順を詳しく解説
- `pull` は `fetch + merge` のショートカットであることを体験で理解
- `git branch -r` や `git remote show` でリモートの状態を把握する方法を紹介

</details>
