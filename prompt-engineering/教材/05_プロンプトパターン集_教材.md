# 第5章：プロンプトパターン集

## この章のゴール

- 頻出タスク別のプロンプトパターンを習得する
- 要約、翻訳、コード生成、分析、クリエイティブの各パターンを使いこなす
- タスクに応じた最適なプロンプト構造を選べるようになる
- パターンを組み合わせて複雑なタスクに対応できるようになる

---

## 5.1 要約パターン

### 基本の要約プロンプト

```
以下のテキストを要約してください。

制約：
- 3〜5文で要約する
- 重要なキーワードを保持する
- 原文にない情報を追加しない

テキスト：
"""
（要約したいテキスト）
"""
```

### 段階的要約（長文向け）

```
以下の長文を段階的に要約してください。

ステップ1：各セクションのキーポイントを箇条書きで抽出
ステップ2：キーポイントを統合して200文字の要約を作成
ステップ3：要約を1文（50文字以内）のエグゼクティブサマリーに凝縮

テキスト：
"""
（長文テキスト）
"""
```

### 目的別の要約バリエーション

| 目的 | プロンプトの追加指示 |
|------|-------------------|
| 経営層向けレポート | 「ビジネスインパクトに焦点を当てて」 |
| 技術者向け | 「技術的な詳細を保持して」 |
| SNS投稿向け | 「140文字以内で、興味を引く表現で」 |
| 議事録の要約 | 「決定事項とアクションアイテムを中心に」 |

> **ポイントまとめ**: 要約は「誰に向けて」「何の目的で」「どの程度の長さで」を明示するのがコツです。

---

## 5.2 翻訳パターン

### 基本の翻訳プロンプト

```
以下の英語テキストを日本語に翻訳してください。

制約：
- 技術用語は一般的なカタカナ表記を使用
- 自然な日本語になるよう意訳を許可
- 原文の段落構成を保持

テキスト：
"""
（翻訳対象テキスト）
"""
```

### 専門分野別の翻訳

```
あなたはIT分野の専門翻訳者です。
以下の技術文書を日本語に翻訳してください。

翻訳ルール：
- 以下の用語はカタカナ表記を使用：
  deploy → デプロイ、container → コンテナ、
  microservice → マイクロサービス
- 以下の用語は英語のまま使用：
  API、HTTP、JSON、Docker、Kubernetes
- 「〜されます」ではなく「〜します」の能動態を優先
```

### 多言語対応パターン

```
以下のUIテキストを日本語、中国語（簡体字）、韓国語に翻訳してください。

出力形式：
| 英語 | 日本語 | 中国語 | 韓国語 |

制約：
- UIラベルとして自然な表現を使用
- 文字数が大幅に増減しないよう配慮
- 敬語は使用しない（カジュアルなアプリUI）

テキスト：
- "Sign Up"
- "Forgot Password?"
- "Add to Cart"
- "Your order has been confirmed"
```

> **ポイントまとめ**: 翻訳では用語集の提供と文体の指定が品質を大きく左右します。

## 5.3 コード生成パターン

### 基本のコード生成プロンプト

```
以下の仕様に基づいてPythonの関数を実装してください。

関数名：calculate_shipping_fee
入力：
  - weight: float（重量、kg）
  - distance: int（距離、km）
  - is_express: bool（速達かどうか）
出力：int（送料、円）

仕様：
- 基本料金 = 重量 × 100円/kg + 距離 × 5円/km
- 速達の場合は基本料金の1.5倍
- 最低料金は500円
- 送料は10円単位で切り上げ

制約：
- 型ヒント（type hints）を使用する
- docstringを含める
- エラーハンドリング（負の値チェック）を含める
- テストケースを3つ以上含める
```

### コードレビュー依頼パターン

```
あなたはシニアエンジニアです。以下のコードをレビューしてください。

レビュー観点：
1. バグや論理的な誤り
2. セキュリティの問題
3. パフォーマンスの問題
4. 可読性・保守性
5. エラーハンドリング

出力形式：
各指摘を以下の形式で記述してください：
- 行番号：
- 重要度：高/中/低
- カテゴリ：バグ/セキュリティ/パフォーマンス/可読性/エラー処理
- 指摘内容：
- 改善案：

コード：
```python
（レビュー対象コード）
```
```

### リファクタリングパターン

```
以下のコードをリファクタリングしてください。

リファクタリングの方針：
- 関数を単一責任の原則（SRP）に基づいて分割する
- マジックナンバーを名前付き定数に置き換える
- ネストの深い条件分岐を早期リターンで平坦化する
- 重複コードをDRYにする

制約：
- 外部からの入出力（インターフェース）を変更しない
- 既存のテストが通ることを確認できるよう、テストも更新する
- 変更内容を箇条書きで説明する
```

> **ポイントまとめ**: コード生成では入出力の仕様・制約・品質基準を明確にすることが重要です。

## 5.4 分析パターン

### データ分析プロンプト

```
以下のデータを分析し、インサイトを抽出してください。

データ：
"""
（CSVデータやテーブルデータ）
"""

分析してほしいこと：
1. データの概要（行数、カラム、データ型）
2. 基本統計量（平均、中央値、最頻値）
3. 注目すべきトレンドやパターン
4. 異常値の有無
5. ビジネス上の示唆（アクションにつながる洞察）

出力形式：
各分析項目について見出し付きで回答してください。
グラフが適切な場合は、Pythonのmatplotlibコードも提供してください。
```

### 比較分析パターン

```
以下の2つの選択肢を比較分析してください。

選択肢A：PostgreSQL
選択肢B：MongoDB

比較の観点：
1. パフォーマンス（読み取り/書き込み）
2. スケーラビリティ
3. データの整合性
4. 学習コスト
5. 運用コスト
6. エコシステム（ツール、ライブラリ）

出力形式：
| 観点 | PostgreSQL | MongoDB | 推奨 |

最後に、以下のユースケースごとの推奨を述べてください：
- ECサイト
- IoTデータ収集
- SNSアプリ
```

### 障害分析（RCA）パターン

```
以下の障害について、根本原因分析（RCA）を行ってください。

障害概要：
- 発生日時：2026/03/20 14:30
- 症状：APIの応答時間が通常200msから5秒に悪化
- 影響範囲：全ユーザー
- 復旧日時：2026/03/20 16:00

ログ情報：
"""
（関連するログ）
"""

分析フレームワーク：
1. タイムライン（時系列で何が起きたか）
2. 5 Whys分析（なぜ？を5回繰り返す）
3. 根本原因の特定
4. 再発防止策（短期・中期・長期）
5. 改善アクションアイテム（担当者・期限付き）
```

> **ポイントまとめ**: 分析タスクでは観点と出力形式を明確に指定し、ビジネス上の示唆まで含めるよう指示しましょう。

## 5.5 クリエイティブパターン

### ブレインストーミングパターン

```
以下のテーマについてブレインストーミングを行ってください。

テーマ：「エンジニアの社内勉強会をもっと参加者を増やすには」

制約：
- 10個以上のアイデアを出す
- 実現可能性は考えず、自由に発想する
- 各アイデアに1行の説明を付ける

その後、以下の基準で上位3つを選んでください：
- 実現容易性（高/中/低）
- 期待効果（高/中/低）
- コスト（高/中/低）
```

### 文章リライトパターン

```
以下の文章を指定されたトーンでリライトしてください。

原文：「本サービスは、ユーザーの検索クエリに基づいて
最適な商品を推薦するアルゴリズムを実装しています。」

リライトのバリエーション：
1. カジュアル（SNS投稿向け）
2. フォーマル（プレスリリース向け）
3. テクニカル（技術ブログ向け）
4. ユーザーフレンドリー（ヘルプページ向け）
```

### 構造化ライティングパターン

```
以下のトピックについて技術ブログ記事の構成案を作成してください。

トピック：「React Server Components入門」
想定読者：Reactの基本は理解しているフロントエンドエンジニア
目標文字数：3,000文字

出力形式：
1. タイトル案（3つ）
2. 構成案（H2/H3見出し一覧）
3. 各セクションの概要（2〜3文）
4. 記事で使用するコード例の概要
5. 読者に持ち帰ってほしいこと（3つ）
```

> **ポイントまとめ**: クリエイティブタスクでは「自由に」と「制約」のバランスが大切です。完全な自由よりも適度な制約がある方が質の高い出力になります。

---

## 5.6 パターンの組み合わせ

### 複合タスクの例

実際の業務では、複数のパターンを組み合わせて使うことが多いです。

```
■ 要約 + 翻訳：
「英語の技術記事を読み、日本語で3行の要約を作成してください」

■ 分析 + コード生成：
「このCSVデータの傾向を分析し、可視化するPythonスクリプトを生成してください」

■ コードレビュー + リファクタリング：
「このコードの問題点を指摘し、改善したバージョンを提供してください」

■ ブレインストーミング + 比較分析：
「新機能のアイデアを10個出し、実現性と価値で評価してください」
```
