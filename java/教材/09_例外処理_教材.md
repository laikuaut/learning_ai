# 第9章：例外処理

## この章のゴール

- 例外（exception）の概念と必要性を理解する
- Javaの例外階層を把握する
- `try-catch-finally` を使ったエラー処理を書けるようになる
- `try-with-resources` でリソースを安全に扱えるようになる
- 検査例外（checked exception）と非検査例外（unchecked exception）の違いを理解する
- `throw` / `throws` を使った例外の送出と宣言を学ぶ
- 独自例外クラスを作成できるようになる
- 例外処理のベストプラクティスを身につける

## 9.1 例外とは

### なぜ例外処理が必要か

プログラムの実行中には、さまざまなエラーが発生する可能性があります。

```java
public class WhyExceptions {
    public static void main(String[] args) {
        // 例外が発生する場面の例

        // 1. ゼロ除算
        // int result = 10 / 0;  // ArithmeticException

        // 2. null参照
        // String s = null;
        // s.length();  // NullPointerException

        // 3. 配列の範囲外
        // int[] arr = {1, 2, 3};
        // arr[5] = 10;  // ArrayIndexOutOfBoundsException

        // 4. 型変換の失敗
        // int num = Integer.parseInt("abc");  // NumberFormatException

        // 例外処理がないとプログラムが強制終了してしまう
        System.out.println("例外処理の学習を始めましょう！");
    }
}
```

### Javaの例外階層

```
┌───────────────────────────────────────────────────┐
│                   Throwable                        │
│              ┌────────┴────────┐                   │
│           Error              Exception             │
│          (回復不能)         (回復可能)               │
│             │            ┌────┴────────┐           │
│     OutOfMemoryError     │        RuntimeException │
│     StackOverflowError   │          (非検査例外)    │
│         (JVMエラー)       │              │          │
│                     IOException    NullPointerException
│                     SQLException   ArrayIndexOutOf...
│                    (検査例外)       NumberFormatException
│                                   IllegalArgument...
│                                   ClassCastException
└───────────────────────────────────────────────────┘

  検査例外（Checked Exception）:
    → コンパイル時にチェックされる。catchまたはthrowsが必須

  非検査例外（Unchecked Exception）:
    → RuntimeExceptionのサブクラス。catchは任意
```

### ポイントまとめ

- 例外は実行時に発生するエラーを表すオブジェクト
- `Error` はJVMレベルのエラーで通常はキャッチしない
- `Exception` はプログラムで対処可能なエラー
- `RuntimeException` は非検査例外で、catchを強制されない

## 9.2 try-catch-finally

### 基本構文

```java
public class TryCatchBasics {
    public static void main(String[] args) {
        // 基本的な try-catch
        try {
            int result = 10 / 0;
            System.out.println("結果: " + result);  // ここは実行されない
        } catch (ArithmeticException e) {
            System.out.println("エラー発生: " + e.getMessage());
            // エラー発生: / by zero
        }
        System.out.println("プログラムは続行します。");

        System.out.println("---");

        // try-catch-finally
        try {
            String text = null;
            int length = text.length();  // NullPointerException
        } catch (NullPointerException e) {
            System.out.println("nullエラー: " + e.getMessage());
        } finally {
            // finally は例外の有無に関わらず必ず実行される
            System.out.println("finallyブロック実行（常に実行）");
        }

        System.out.println("---");

        // 例外が発生しない場合もfinallyは実行
        try {
            int result = 10 / 2;
            System.out.println("結果: " + result);
        } catch (ArithmeticException e) {
            System.out.println("エラー: " + e.getMessage());
        } finally {
            System.out.println("finallyは常に実行されます。");
        }
    }
}
```

### 複数のcatchブロック

```java
public class MultipleCatch {
    public static void main(String[] args) {
        String[] inputs = {"42", "abc", null, "0"};

        for (String input : inputs) {
            try {
                System.out.print("入力: " + input + " → ");
                int number = Integer.parseInt(input);
                int result = 100 / number;
                System.out.println("結果: " + result);
            } catch (NumberFormatException e) {
                System.out.println("数値変換エラー: " + e.getMessage());
            } catch (ArithmeticException e) {
                System.out.println("算術エラー: " + e.getMessage());
            } catch (NullPointerException e) {
                System.out.println("nullエラー");
            }
        }
        // 入力: 42 → 結果: 2
        // 入力: abc → 数値変換エラー: For input string: "abc"
        // 入力: null → nullエラー
        // 入力: 0 → 算術エラー: / by zero

        System.out.println("\n--- マルチキャッチ構文（Java 7以降）---");
        try {
            String text = "abc";
            int num = Integer.parseInt(text);
        } catch (NumberFormatException | ArithmeticException e) {
            // 複数の例外を1つのcatchで処理
            System.out.println("エラー: " + e.getMessage());
        }
    }
}
```

> **よくある間違い：** 複数の `catch` ブロックは、**具体的な例外を先に**書きます。親クラスの例外（`Exception`）を先に書くと、子クラスの `catch` に到達できずコンパイルエラーになります。

```java
// 悪い例（コンパイルエラー）
// try {
//     ...
// } catch (Exception e) {           // 親が先 → 全てここでキャッチ
//     ...
// } catch (IOException e) {         // 到達不能コード
//     ...
// }

// 良い例
// try {
//     ...
// } catch (IOException e) {         // 具体的な例外が先
//     ...
// } catch (Exception e) {           // 汎用的な例外が後
//     ...
// }
```

### ポイントまとめ

- `try` ブロック内で例外が発生すると、対応する `catch` に処理が移る
- `finally` は例外の有無に関わらず必ず実行される
- 複数の `catch` は具体的な例外を先に書く
- マルチキャッチ（`|`）で複数の例外を1つの `catch` で処理できる

## 9.3 try-with-resources

### リソースの自動クロース

`try-with-resources`（Java 7以降）を使うと、`AutoCloseable` インターフェースを実装したリソースが自動的にクローズされます。

```java
import java.io.BufferedReader;
import java.io.StringReader;
import java.io.IOException;

public class TryWithResources {
    public static void main(String[] args) {
        // 従来の方法（finally で明示的にクローズ）
        BufferedReader reader1 = null;
        try {
            reader1 = new BufferedReader(new StringReader("Hello\nWorld"));
            String line;
            while ((line = reader1.readLine()) != null) {
                System.out.println("従来: " + line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            if (reader1 != null) {
                try {
                    reader1.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }

        System.out.println("---");

        // try-with-resources（推奨）
        try (BufferedReader reader2 = new BufferedReader(new StringReader("Hello\nWorld"))) {
            String line;
            while ((line = reader2.readLine()) != null) {
                System.out.println("新方式: " + line);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        // reader2 は自動的にクローズされる
    }
}
```

### 自作リソースクラス

```java
public class CustomResource implements AutoCloseable {
    private String name;

    public CustomResource(String name) {
        this.name = name;
        System.out.println(name + " をオープンしました。");
    }

    public void doWork() {
        System.out.println(name + " で作業中...");
    }

    @Override
    public void close() {
        System.out.println(name + " をクローズしました。");
    }

    public static void main(String[] args) {
        // 複数リソースも管理できる
        try (CustomResource r1 = new CustomResource("リソースA");
             CustomResource r2 = new CustomResource("リソースB")) {
            r1.doWork();
            r2.doWork();
        }
        // 出力:
        // リソースA をオープンしました。
        // リソースB をオープンしました。
        // リソースA で作業中...
        // リソースB で作業中...
        // リソースB をクローズしました。（後から開いたものが先にクローズ）
        // リソースA をクローズしました。
    }
}
```

### ポイントまとめ

- `try-with-resources` でリソースの閉じ忘れを防げる
- `AutoCloseable` インターフェースを実装したクラスが対象
- 複数リソースはセミコロンで区切って宣言できる
- クローズは宣言の逆順で行われる

## 9.4 検査例外と非検査例外

### 違いの一覧

| 特徴 | 検査例外（Checked） | 非検査例外（Unchecked） |
|---|---|---|
| 親クラス | `Exception`（RuntimeException以外） | `RuntimeException` |
| コンパイル時チェック | あり（catchかthrowsが必須） | なし（任意） |
| 代表例 | `IOException`, `SQLException` | `NullPointerException`, `NumberFormatException` |
| 原因 | 外部環境（ファイル、ネットワーク等） | プログラムのバグ |
| 対処方針 | try-catchで回復処理を書く | バグを修正する |

### throwsキーワード

メソッドが検査例外をスローする可能性がある場合、`throws` で宣言します。

```java
import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.List;

public class ThrowsDemo {
    // throws で検査例外を宣言
    public static String readFirstLine(String filename) throws IOException {
        List<String> lines = Files.readAllLines(Path.of(filename));
        if (lines.isEmpty()) {
            return "";
        }
        return lines.get(0);
    }

    public static void main(String[] args) {
        // 呼び出し側で try-catch するか、さらに throws する
        try {
            String line = readFirstLine("sample.txt");
            System.out.println("最初の行: " + line);
        } catch (IOException e) {
            System.out.println("ファイル読み込みエラー: " + e.getMessage());
        }
    }
}
```

### throwキーワード

`throw` で意図的に例外をスローします。

```java
public class ThrowDemo {
    public static void setAge(int age) {
        if (age < 0) {
            throw new IllegalArgumentException("年齢は0以上でなければなりません: " + age);
        }
        if (age > 150) {
            throw new IllegalArgumentException("年齢が不正です: " + age);
        }
        System.out.println("年齢を " + age + " に設定しました。");
    }

    public static double divide(double a, double b) {
        if (b == 0) {
            throw new ArithmeticException("ゼロで割ることはできません。");
        }
        return a / b;
    }

    public static void main(String[] args) {
        try {
            setAge(25);   // 年齢を 25 に設定しました。
            setAge(-5);   // IllegalArgumentException
        } catch (IllegalArgumentException e) {
            System.out.println("エラー: " + e.getMessage());
        }

        try {
            System.out.println("10 / 3 = " + divide(10, 3));
            System.out.println("10 / 0 = " + divide(10, 0));
        } catch (ArithmeticException e) {
            System.out.println("エラー: " + e.getMessage());
        }
    }
}
```

### ポイントまとめ

- 検査例外はコンパイル時にチェックされ、`catch` か `throws` が必須
- 非検査例外（`RuntimeException`）は `catch` を強制されない
- `throw` で例外を明示的にスローできる
- `throws` はメソッドが例外をスローする可能性を宣言する

## 9.5 カスタム例外クラス

### 独自例外の作成

```java
// カスタム検査例外
class InsufficientFundsException extends Exception {
    private double amount;
    private double balance;

    public InsufficientFundsException(double amount, double balance) {
        super(String.format("残高不足: 出金額=%.0f円, 残高=%.0f円, 不足額=%.0f円",
            amount, balance, amount - balance));
        this.amount = amount;
        this.balance = balance;
    }

    public double getShortage() {
        return amount - balance;
    }
}

// カスタム非検査例外
class InvalidAccountException extends RuntimeException {
    public InvalidAccountException(String accountNumber) {
        super("無効な口座番号: " + accountNumber);
    }
}

class SimpleBank {
    private String accountNumber;
    private double balance;

    public SimpleBank(String accountNumber, double initialBalance) {
        if (accountNumber == null || accountNumber.length() != 8) {
            throw new InvalidAccountException(accountNumber);
        }
        this.accountNumber = accountNumber;
        this.balance = initialBalance;
    }

    // 検査例外をスローするメソッド
    public void withdraw(double amount) throws InsufficientFundsException {
        if (amount <= 0) {
            throw new IllegalArgumentException("出金額は正の値である必要があります。");
        }
        if (amount > balance) {
            throw new InsufficientFundsException(amount, balance);
        }
        balance -= amount;
        System.out.printf("¥%,.0f 出金しました。残高: ¥%,.0f%n", amount, balance);
    }

    public double getBalance() {
        return balance;
    }
}

public class CustomExceptionDemo {
    public static void main(String[] args) {
        // 正常系
        try {
            SimpleBank account = new SimpleBank("12345678", 100000);
            account.withdraw(30000);
            account.withdraw(50000);
            account.withdraw(30000);  // 残高不足
        } catch (InsufficientFundsException e) {
            System.out.println("エラー: " + e.getMessage());
            System.out.printf("不足額: ¥%,.0f%n", e.getShortage());
        }

        System.out.println("---");

        // 無効な口座番号
        try {
            SimpleBank invalid = new SimpleBank("123", 50000);
        } catch (InvalidAccountException e) {
            System.out.println("エラー: " + e.getMessage());
        }
    }
}
```

### ポイントまとめ

- `Exception` を継承すると検査例外、`RuntimeException` を継承すると非検査例外になる
- カスタム例外にはエラーの詳細情報（フィールド）を持たせると便利
- 業務ロジック固有のエラーにはカスタム例外を使う

## 9.6 よくある例外と対処法

### 代表的な例外

```java
public class CommonExceptions {
    public static void main(String[] args) {
        // 1. NullPointerException
        System.out.println("--- NullPointerException ---");
        try {
            String text = null;
            int len = text.length();
        } catch (NullPointerException e) {
            System.out.println("対処: null チェックを行う");
            String text = null;
            int len = (text != null) ? text.length() : 0;
            System.out.println("安全な長さ: " + len);
        }

        // 2. ArrayIndexOutOfBoundsException
        System.out.println("\n--- ArrayIndexOutOfBoundsException ---");
        try {
            int[] arr = {10, 20, 30};
            int val = arr[5];
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("対処: インデックスの範囲を確認する");
            int[] arr = {10, 20, 30};
            int index = 5;
            if (index >= 0 && index < arr.length) {
                System.out.println("値: " + arr[index]);
            } else {
                System.out.println("インデックス " + index + " は範囲外です (0-" + (arr.length - 1) + ")");
            }
        }

        // 3. NumberFormatException
        System.out.println("\n--- NumberFormatException ---");
        try {
            int num = Integer.parseInt("abc");
        } catch (NumberFormatException e) {
            System.out.println("対処: 変換前に検証するか、try-catchで囲む");
            String input = "abc";
            try {
                int num = Integer.parseInt(input);
                System.out.println("数値: " + num);
            } catch (NumberFormatException ex) {
                System.out.println("'" + input + "' は数値に変換できません。");
            }
        }

        // 4. ClassCastException
        System.out.println("\n--- ClassCastException ---");
        try {
            Object obj = "Hello";
            Integer num = (Integer) obj;
        } catch (ClassCastException e) {
            System.out.println("対処: instanceof で型を確認してからキャストする");
            Object obj = "Hello";
            if (obj instanceof Integer) {
                Integer num = (Integer) obj;
            } else {
                System.out.println("Integer ではありません: " + obj.getClass().getSimpleName());
            }
        }

        // 5. StackOverflowError（再帰の終了条件忘れ）
        System.out.println("\n--- StackOverflowError ---");
        try {
            badRecursion(1);
        } catch (StackOverflowError e) {
            System.out.println("対処: 再帰の終了条件を必ず書く");
        }
    }

    static void badRecursion(int n) {
        badRecursion(n + 1);  // 終了条件がない！
    }
}
```

### ポイントまとめ

- `NullPointerException` は null チェックで予防する
- `ArrayIndexOutOfBoundsException` は範囲チェックで予防する
- `NumberFormatException` は変換前の検証か try-catch で対処する
- `StackOverflowError` は再帰の終了条件を必ず設定する

## 9.7 例外処理のベストプラクティス

### やるべきこと・避けるべきこと

```java
public class ExceptionBestPractices {
    public static void main(String[] args) {
        // ===== 良いプラクティス =====

        // 1. 具体的な例外をキャッチする
        try {
            int num = Integer.parseInt("abc");
        } catch (NumberFormatException e) {  // 具体的な例外
            System.out.println("数値変換エラー: " + e.getMessage());
        }
        // catch (Exception e) は最後の手段として使う

        // 2. 例外のメッセージを活用する
        try {
            int[] arr = new int[3];
            arr[5] = 100;
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("エラー内容: " + e.getMessage());
            // ログに記録するのが実務的
        }

        // 3. 例外の再スロー（情報を追加して投げ直す）
        try {
            processData("invalid");
        } catch (IllegalStateException e) {
            System.out.println("処理エラー: " + e.getMessage());
            if (e.getCause() != null) {
                System.out.println("原因: " + e.getCause().getMessage());
            }
        }

        // ===== 避けるべきプラクティス =====

        // 悪い例1: 空のcatchブロック（例外を握りつぶす）
        // try {
        //     riskyOperation();
        // } catch (Exception e) {
        //     // 何もしない → バグの原因が分からなくなる
        // }

        // 悪い例2: 制御フローに例外を使う
        // try {
        //     while (true) {
        //         array[i++] = value;  // 範囲外で例外 → ループ終了
        //     }
        // } catch (ArrayIndexOutOfBoundsException e) {}
        // → 普通の条件分岐を使うべき

        // 悪い例3: 広すぎる例外をキャッチ
        // catch (Exception e) を安易に使わない
    }

    static void processData(String data) {
        try {
            int value = Integer.parseInt(data);
        } catch (NumberFormatException e) {
            // 例外チェーン: 原因（cause）を含めて再スロー
            throw new IllegalStateException("データ処理に失敗: " + data, e);
        }
    }
}
```

### ベストプラクティスまとめ

```
┌────────────────────────────────────────────────────┐
│  例外処理のベストプラクティス                         │
│                                                    │
│  ○ 具体的な例外クラスをキャッチする                   │
│  ○ catchブロックで適切な対処をする                    │
│  ○ try-with-resourcesでリソースを管理する            │
│  ○ 例外メッセージに有用な情報を含める                 │
│  ○ 例外チェーンで原因を保持する                      │
│                                                    │
│  × 空のcatchブロック（握りつぶし）は避ける            │
│  × 制御フローに例外を使わない                        │
│  × catch (Exception e) を安易に使わない              │
│  × Errorをキャッチしない（JVMエラーは回復不能）       │
│  × catchブロックで新たな例外を隠さない               │
└────────────────────────────────────────────────────┘
```

### ポイントまとめ

- 具体的な例外をキャッチし、適切な対処を行う
- 空の `catch` ブロックは絶対に避ける
- 例外チェーン（`cause`）で元の例外情報を保持する
- `try-with-resources` でリソース管理を自動化する

## 9.8 実践例：入力検証システム

```java
class ValidationException extends Exception {
    private String fieldName;
    private String invalidValue;

    public ValidationException(String fieldName, String invalidValue, String message) {
        super(message);
        this.fieldName = fieldName;
        this.invalidValue = invalidValue;
    }

    public String getFieldName() { return fieldName; }
    public String getInvalidValue() { return invalidValue; }
}

class UserRegistration {
    private String name;
    private int age;
    private String email;

    public UserRegistration(String name, String ageStr, String email)
            throws ValidationException {
        this.name = validateName(name);
        this.age = validateAge(ageStr);
        this.email = validateEmail(email);
    }

    private String validateName(String name) throws ValidationException {
        if (name == null || name.trim().isEmpty()) {
            throw new ValidationException("名前", name, "名前は必須です。");
        }
        if (name.length() < 2 || name.length() > 50) {
            throw new ValidationException("名前", name, "名前は2〜50文字で入力してください。");
        }
        return name.trim();
    }

    private int validateAge(String ageStr) throws ValidationException {
        if (ageStr == null || ageStr.trim().isEmpty()) {
            throw new ValidationException("年齢", ageStr, "年齢は必須です。");
        }
        try {
            int age = Integer.parseInt(ageStr.trim());
            if (age < 0 || age > 150) {
                throw new ValidationException("年齢", ageStr, "年齢は0〜150の範囲で入力してください。");
            }
            return age;
        } catch (NumberFormatException e) {
            throw new ValidationException("年齢", ageStr, "年齢は数値で入力してください。");
        }
    }

    private String validateEmail(String email) throws ValidationException {
        if (email == null || email.trim().isEmpty()) {
            throw new ValidationException("メール", email, "メールアドレスは必須です。");
        }
        if (!email.matches("^[\\w.-]+@[\\w.-]+\\.[a-zA-Z]{2,}$")) {
            throw new ValidationException("メール", email, "メールアドレスの形式が不正です。");
        }
        return email.trim();
    }

    @Override
    public String toString() {
        return String.format("ユーザー{名前='%s', 年齢=%d, メール='%s'}", name, age, email);
    }
}
```

## まとめ

この章では、Javaの例外処理について学びました。

1. **例外とは** - 実行時に発生するエラーを表すオブジェクト
2. **例外階層** - `Throwable` → `Error` / `Exception` → `RuntimeException`
3. **try-catch-finally** - 例外をキャッチして適切に処理する
4. **try-with-resources** - リソースを自動的にクローズする
5. **検査例外 vs 非検査例外** - コンパイル時チェックの有無
6. **throw / throws** - 例外のスローと宣言
7. **カスタム例外** - 業務ロジック固有の例外クラスを作成
8. **ベストプラクティス** - 具体的にキャッチし、握りつぶさない
