## 読解対象コード

```typescript
// ===== 状態管理ライブラリ: MiniStore =====

// ---------- 型定義 ----------

type Listener<S> = (state: S, prevState: S) => void;

type Selector<S, R> = (state: S) => R;

type ActionHandler<S, P = void> = P extends void
  ? (state: S) => Partial<S>
  : (state: S, payload: P) => Partial<S>;

interface StoreOptions<S> {
  initialState: S;
  onError?: (error: unknown) => void;
}

interface Store<S extends Record<string, unknown>> {
  getState(): Readonly<S>;
  setState(updater: Partial<S> | ((prev: S) => Partial<S>)): void;
  subscribe(listener: Listener<S>): () => void;
  select<R>(selector: Selector<S, R>): R;
  addAction<P = void>(name: string, handler: ActionHandler<S, P>): void;
  dispatch(name: string, payload?: unknown): void;
  reset(): void;
}

// ---------- ミドルウェアの型 ----------

type Middleware<S> = (
  store: Pick<Store<S>, "getState" | "setState">,
  next: () => void,
  actionName: string,
  payload: unknown
) => void;

// ---------- ストア生成関数 ----------

function createStore<S extends Record<string, unknown>>(
  options: StoreOptions<S>
): Store<S> {
  let state: S = { ...options.initialState };
  const initialState: Readonly<S> = Object.freeze({ ...options.initialState });
  const listeners: Set<Listener<S>> = new Set();
  const actions: Map<string, ActionHandler<S, any>> = new Map();
  const middlewares: Middleware<S>[] = [];

  function notify(prevState: S): void {
    for (const listener of listeners) {
      try {
        listener(state, prevState);
      } catch (error) {
        options.onError?.(error);
      }
    }
  }

  function getState(): Readonly<S> {
    return Object.freeze({ ...state });
  }

  function setState(updater: Partial<S> | ((prev: S) => Partial<S>)): void {
    const prevState: S = { ...state };
    const partial: Partial<S> =
      typeof updater === "function" ? updater(state) : updater;
    state = { ...state, ...partial };
    notify(prevState);
  }

  function subscribe(listener: Listener<S>): () => void {
    listeners.add(listener);
    return () => {
      listeners.delete(listener);
    };
  }

  function select<R>(selector: Selector<S, R>): R {
    return selector(state);
  }

  function addAction<P = void>(
    name: string,
    handler: ActionHandler<S, P>
  ): void {
    actions.set(name, handler as ActionHandler<S, any>);
  }

  function dispatch(name: string, payload?: unknown): void {
    const handler = actions.get(name);
    if (!handler) {
      throw new Error(`Action "${name}" is not registered`);
    }

    const executeDispatch = (): void => {
      const partial = (handler as Function)(state, payload);
      setState(partial);
    };

    if (middlewares.length === 0) {
      executeDispatch();
      return;
    }

    let index = 0;
    const next = (): void => {
      if (index < middlewares.length) {
        const mw = middlewares[index++];
        mw({ getState, setState }, next, name, payload);
      } else {
        executeDispatch();
      }
    };
    next();
  }

  function reset(): void {
    const prevState: S = { ...state };
    state = { ...initialState } as S;
    notify(prevState);
  }

  return {
    getState,
    setState,
    subscribe,
    select,
    addAction,
    dispatch,
    reset,
  };
}

// ---------- ユーティリティ ----------

function createSelector<S extends Record<string, unknown>, R>(
  selectors: Selector<S, unknown>[],
  combiner: (...args: unknown[]) => R
): Selector<S, R> {
  let lastArgs: unknown[] | null = null;
  let lastResult: R;

  return (state: S): R => {
    const currentArgs = selectors.map((sel) => sel(state));
    const changed =
      lastArgs === null ||
      currentArgs.some((arg, i) => arg !== lastArgs![i]);

    if (changed) {
      lastResult = combiner(...currentArgs);
      lastArgs = currentArgs;
    }

    return lastResult;
  };
}

type ExtractState<T> = T extends Store<infer S> ? S : never;

// ---------- 使用例 ----------

interface TodoState {
  todos: { id: number; text: string; done: boolean }[];
  filter: "all" | "active" | "done";
  nextId: number;
}

const todoStore = createStore<TodoState>({
  initialState: {
    todos: [],
    filter: "all",
    nextId: 1,
  },
  onError: (error) => console.error("Store error:", error),
});

todoStore.addAction<string>("addTodo", (state, text) => ({
  todos: [...state.todos, { id: state.nextId, text, done: false }],
  nextId: state.nextId + 1,
}));

todoStore.addAction<number>("toggleTodo", (state, id) => ({
  todos: state.todos.map((todo) =>
    todo.id === id ? { ...todo, done: !todo.done } : todo
  ),
}));

todoStore.addAction<TodoState["filter"]>("setFilter", (_state, filter) => ({
  filter,
}));

const selectFilteredTodos = createSelector<TodoState, TodoState["todos"]>(
  [
    (state) => state.todos,
    (state) => state.filter,
  ],
  (todos, filter) => {
    const allTodos = todos as TodoState["todos"];
    const currentFilter = filter as TodoState["filter"];
    switch (currentFilter) {
      case "all":
        return allTodos;
      case "active":
        return allTodos.filter((t) => !t.done);
      case "done":
        return allTodos.filter((t) => t.done);
    }
  }
);
```

## 設問

各設問に答えてください。コードをじっくり読んで、型の流れを追ってみましょう。

### 設問1（基礎）

`Listener<S>` と `Selector<S, R>` の型を日本語で説明してください。それぞれどのような「契約」を表していますか？

### 設問2（基礎）

`ActionHandler<S, P = void>` の型定義には条件付き型（conditional type）が使われています。`P = void` の `= void` は何を意味しますか？ また、`P extends void` が true / false の場合、それぞれどんな関数シグネチャになりますか？

### 設問3（読解）

### 設問4（読解）

const state = todoStore.getState();

### 設問6（型の追跡）

`todoStore.addAction<string>("addTodo", (state, text) => {...})` において、`state` と `text` の型はそれぞれ何ですか？ `ActionHandler<S, P>` の型定義から型がどう決まるかを追跡してください。

### 設問7（設計判断）

`Middleware<S>` の型定義で、ストアへのアクセスが `Pick<Store<S>, "getState" | "setState">` に制限されています。`Store<S>` 全体を渡さず一部だけを渡す設計にしている理由を推測してください。

## 解答例

<details>
<summary>設問1の解答</summary>

- **`Listener<S>`**: 「状態 `S` の変化を受け取るコールバック関数」の型です。新しい状態（`state`）と変更前の状態（`prevState`）を引数として受け取り、何も返しません（`void`）。「状態が変わったときに通知を受ける関数の契約」を表しています。

- **`Selector<S, R>`**: 「状態 `S` から特定の値 `R` を取り出す関数」の型です。状態全体を受け取り、必要な部分だけを返す「抽出器」の契約を表しています。例えば `(state: TodoState) => state.todos` のような関数です。

</details>

<details>
<summary>設問2の解答</summary>

`= void` は**型パラメータのデフォルト値**です。`P` を指定しなかった場合、自動的に `void` になります。

- **`P extends void` が true（P = void）の場合**: `(state: S) => Partial<S>` — ペイロードなしの関数。状態だけを受け取って新しい部分状態を返します。
- **`P extends void` が false（例: P = string）の場合**: `(state: S, payload: P) => Partial<S>` — ペイロードありの関数。状態とペイロードを受け取って新しい部分状態を返します。

これにより、`addAction("reset", (state) => ({...}))` のようにペイロードなし、`addAction<string>("addTodo", (state, text) => ({...}))` のようにペイロードありの両方をサポートできます。

</details>

<details>
<summary>設問3の解答</summary>

`setState` は以下の2つの使い方ができます。

1. **部分的な状態オブジェクトを直接渡す**:
```typescript
todoStore.setState({ filter: "done" }); // Partial<TodoState>
```

2. **前の状態を受け取る関数を渡す**:
```typescript
todoStore.setState((prev) => ({ nextId: prev.nextId + 1 })); // (prev: S) => Partial<S>
```

関数版は「前の状態に基づいて新しい状態を計算する」場合に使います。Reactの `setState` と同じパターンです。`typeof updater === "function"` で判別しています。

</details>

<details>
<summary>設問4の解答</summary>

戻り値の `() => void` は**購読解除（unsubscribe）関数**です。

```typescript
const unsubscribe = todoStore.subscribe((state) => console.log(state));
// ... 必要なくなったら解除
unsubscribe();
```

この設計にする理由は以下の通りです。
- リスナーを解除するために、リスナー自体の参照を管理する必要がない
- `unsubscribe` 関数は購読時の `listener` をクロージャで保持しているため、確実に正しいリスナーを解除できる
- ReactのuseEffectのクリーンアップ関数と同じパターンで使える

</details>

<details>
<summary>設問5の解答</summary>

`state` の型は `Readonly<TodoState>` です。

型推論の流れ:
1. `todoStore` の型は `Store<TodoState>`（`createStore<TodoState>(...)` で作成されたため）
2. `Store<TodoState>` の `getState()` メソッドの戻り値型は `Readonly<S>` = `Readonly<TodoState>`
3. よって `state` は `Readonly<TodoState>` 型

`Readonly<TodoState>` なので、`state.filter = "done"` のようなプロパティの直接変更はコンパイルエラーになります。ただし `Readonly` はシャロー（浅い）なので、`state.todos.push(...)` のような深いレベルの変更は防げません。

</details>

<details>
<summary>設問6の解答</summary>

- `state` の型: `TodoState`
- `text` の型: `string`

型の追跡:
1. `addAction<string>` と呼んでいるので、型パラメータ `P = string`
2. `ActionHandler<TodoState, string>` が展開される
3. 条件付き型: `string extends void` → **false**
4. よって `(state: TodoState, payload: string) => Partial<TodoState>` になる
5. コールバックの引数名は `text` だが、型は `payload` の型 = `string`

</details>

<details>
<summary>設問7の解答</summary>

`Pick<Store<S>, "getState" | "setState">` に制限する理由:

1. **最小権限の原則**: ミドルウェアに必要なのは「現在の状態を読む」「状態を更新する」の2つだけ。`subscribe`, `dispatch`, `addAction` などはミドルウェア内で使うべきではない。
2. **無限ループの防止**: もしミドルウェア内で `dispatch` を呼べると、ミドルウェアが再帰的に呼ばれて無限ループになる可能性がある。
3. **テスト容易性**: 限定されたインターフェースのモックは、完全な `Store` のモックより簡単。
4. **型安全性**: 制限されたAPIしか使えないことがコンパイル時に保証される。

</details>

<details>
<summary>設問8の解答</summary>

`ExtractState<T>` は**条件付き型と `infer` を使って、`Store<S>` の型パラメータ `S` を抽出する型**です。

```typescript
type ExtractState<T> = T extends Store<infer S> ? S : never;
```

- `T` が `Store<SomeType>` にマッチする場合、`S` に `SomeType` が推論（infer）され、その `S` が返される
- マッチしない場合は `never` が返される

`ExtractState<typeof todoStore>` の結果は `TodoState` です。`typeof todoStore` は `Store<TodoState>` なので、`infer S` に `TodoState` がマッチします。

`infer` は「この位置にどんな型が入っているか推論して、その型を変数 `S` に束縛する」という意味です。

</details>

<details>
<summary>設問9の解答</summary>

**クロージャの仕組み**: `createSelector` は関数を返しますが、返された関数は外側のスコープの `lastArgs` と `lastResult` を参照し続けます（クロージャ）。関数が呼ばれるたびに以前の結果と比較し、変化がなければキャッシュを返します。

**`!` の意味**: `lastArgs!` の `!` は**非nullアサーション演算子**（non-null assertion operator）です。TypeScriptに「この値は `null` や `undefined` ではないと保証する」と伝えます。ここでは、`lastArgs === null` のチェックが `||` の前にあるため、`some` が実行される時点では `lastArgs` は `null` ではないことが論理的に保証されています。しかしTypeScriptの型の絞り込みが `||` をまたいで追跡できないため、`!` で明示しています。

</details>

<details>
<summary>設問10の解答</summary>

**改善案1: `any` の排除**

`actions` の型が `Map<string, ActionHandler<S, any>>` で、`dispatch` 内部で `(handler as Function)` とキャストしています。アクション名とペイロードの型を紐づける型マップを導入すれば、より型安全になります。

```typescript
interface ActionMap {
  addTodo: string;
  toggleTodo: number;
  setFilter: TodoState["filter"];
}
// dispatch<K extends keyof ActionMap>(name: K, payload: ActionMap[K]): void
```

**改善案2: `createSelector` の型安全性向上**

現在 `selectors` の型は `Selector<S, unknown>[]` で、`combiner` の引数も `unknown[]` です。タプル型を使って、各セレクタの戻り値型を正確に追跡できます。

```typescript
function createSelector<S, R1, R2, Result>(
  selectors: [Selector<S, R1>, Selector<S, R2>],
  combiner: (r1: R1, r2: R2) => Result
): Selector<S, Result>;
```

**改善案3: パフォーマンス改善**

`getState()` で毎回 `Object.freeze({ ...state })` を呼んでいるため、頻繁な読み取りでオーバーヘッドが生じます。`Proxy` を使った遅延フリーズや、変更フラグで最適化できます。

</details>
