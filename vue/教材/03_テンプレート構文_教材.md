# 第3章：テンプレート構文

## 3.1 条件付きレンダリング

### v-if / v-else-if / v-else

条件に基づいて要素の表示・非表示を切り替えます。**条件がfalseのとき、要素はDOMから完全に削除されます。**

```vue
<script setup>
import { ref } from 'vue'

const score = ref(85)
const isLoggedIn = ref(false)
const user = ref(null)
</script>

<template>
  <!-- 基本的な v-if -->
  <p v-if="isLoggedIn">ようこそ！</p>

  <!-- v-if / v-else -->
  <div v-if="isLoggedIn">
    <p>ログイン済みです</p>
  </div>
  <div v-else>
    <p>ログインしてください</p>
  </div>

  <!-- v-if / v-else-if / v-else チェーン -->
  <p v-if="score >= 90">評価: A（優秀）</p>
  <p v-else-if="score >= 70">評価: B（良好）</p>
  <p v-else-if="score >= 50">評価: C（普通）</p>
  <p v-else>評価: D（要改善）</p>

  <!-- null チェック -->
  <div v-if="user">
    <p>{{ user.name }}</p>
  </div>

  <!-- template で複数要素をグループ化（余計なDOMノードが追加されない） -->
  <template v-if="isLoggedIn">
    <h2>ダッシュボード</h2>
    <p>お知らせが3件あります</p>
    <nav>メニュー</nav>
  </template>
</template>
```

### v-show

`v-show` は要素を**常にDOMに保持**し、CSSの `display` プロパティで表示を切り替えます。

```vue
<script setup>
import { ref } from 'vue'

const isVisible = ref(true)
</script>

<template>
  <!-- v-show: display: none で非表示にする -->
  <div v-show="isVisible">
    この要素は常にDOMにあり、表示/非表示が切り替わる
  </div>

  <button @click="isVisible = !isVisible">
    {{ isVisible ? '非表示' : '表示' }}
  </button>
</template>
```

### v-if vs v-show の使い分け

| 特徴 | `v-if` | `v-show` |
|------|--------|----------|
| DOM操作 | 要素の追加/削除 | display の切り替え |
| 初期コスト | 低い（非表示時はレンダリングしない） | 高い（常にレンダリング） |
| 切り替えコスト | 高い（DOM操作が発生） | 低い（CSSのみ） |
| 適する場面 | 切り替え頻度が低い場合 | 頻繁に切り替える場合 |
| `<template>` | 使える | 使えない |

## 3.2 リストレンダリング（v-for）

配列やオブジェクトの各要素を繰り返し描画します。

```vue
<script setup>
import { ref } from 'vue'

const fruits = ref(['りんご', 'バナナ', 'みかん', 'ぶどう'])

const users = ref([
  { id: 1, name: '太郎', age: 20 },
  { id: 2, name: '花子', age: 25 },
  { id: 3, name: '次郎', age: 18 }
])

const settings = ref({
  theme: 'dark',
  language: 'ja',
  fontSize: 14
})
</script>

<template>
  <!-- 配列の繰り返し（indexも取得可能） -->
  <ul>
    <li v-for="(fruit, index) in fruits" :key="index">
      {{ index + 1 }}. {{ fruit }}
    </li>
  </ul>

  <!-- オブジェクト配列（:key にユニークな値を指定） -->
  <div v-for="user in users" :key="user.id">
    <p>{{ user.name }}（{{ user.age }}歳）</p>
  </div>

  <!-- オブジェクトのプロパティを繰り返し -->
  <ul>
    <li v-for="(value, key, index) in settings" :key="key">
      {{ index }}: {{ key }} = {{ value }}
    </li>
  </ul>

  <!-- 数値の範囲（1から始まる） -->
  <span v-for="n in 5" :key="n">{{ n }} </span>
  <!-- 出力: 1 2 3 4 5 -->

  <!-- template で複数要素をグループ化 -->
  <template v-for="user in users" :key="user.id">
    <h3>{{ user.name }}</h3>
    <p>年齢: {{ user.age }}</p>
    <hr />
  </template>
</template>
```

### :key の重要性

`:key` は Vue が各要素を一意に識別するために必要です。**常に指定してください。**

```vue
<template>
  <!-- ✅ ユニークなIDを key に使う（推奨） -->
  <div v-for="item in items" :key="item.id">
    {{ item.name }}
  </div>

  <!-- ⚠️ index を key に使うのは非推奨（並び替えや削除で問題が起きる） -->
  <div v-for="(item, index) in items" :key="index">
    {{ item.name }}
  </div>
</template>
```

### 配列の変更検知

Vue は以下の配列メソッドの呼び出しを検知してUIを更新します（ミューテーションメソッド）：

```js
const items = ref(['a', 'b', 'c'])

// ✅ 検知される
items.value.push('d')       // 末尾に追加
items.value.pop()           // 末尾を削除
items.value.shift()         // 先頭を削除
items.value.unshift('x')    // 先頭に追加
items.value.splice(1, 1)    // 要素の削除/挿入
items.value.sort()          // ソート
items.value.reverse()       // 逆順

// ✅ 配列の置き換えも検知される
items.value = items.value.filter(item => item !== 'b')
items.value = [...items.value, 'e']
```

### v-for と v-if の併用

**同じ要素に `v-for` と `v-if` を同時に使わないでください。** `v-if` の方が優先度が高く、`v-for` の変数にアクセスできません。

```vue
<template>
  <!-- ❌ v-for と v-if を同じ要素に使わない -->
  <li v-for="user in users" v-if="user.isActive" :key="user.id">
    {{ user.name }}
  </li>

  <!-- ✅ computed で事前にフィルタリング -->
  <li v-for="user in activeUsers" :key="user.id">
    {{ user.name }}
  </li>

  <!-- ✅ または template で分離 -->
  <template v-for="user in users" :key="user.id">
    <li v-if="user.isActive">
      {{ user.name }}
    </li>
  </template>
</template>

<script setup>
import { ref, computed } from 'vue'

const users = ref([/* ... */])
const activeUsers = computed(() => users.value.filter(u => u.isActive))
</script>
```

## 3.3 イベントハンドリング（v-on / @）

### 基本的なイベント処理

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)
const message = ref('')

// メソッドハンドラ
function increment() {
  count.value++
}

function decrement() {
  count.value--
}

// イベントオブジェクトを受け取る
function handleClick(event) {
  console.log('クリック位置:', event.clientX, event.clientY)
  console.log('対象要素:', event.target)
}

// 引数を受け取る
function greet(name) {
  message.value = `こんにちは、${name}さん！`
}

// 引数とイベントオブジェクトの両方を受け取る
function handleItemClick(itemId, event) {
  console.log('ID:', itemId)
  console.log('イベント:', event)
}
</script>

<template>
  <!-- v-on:click="..." -->
  <button v-on:click="increment">+1</button>

  <!-- 省略記法 @click="..."（実務ではこちらが主流） -->
  <button @click="decrement">-1</button>

  <!-- インライン式 -->
  <button @click="count = 0">リセット</button>
  <button @click="count += 10">+10</button>

  <!-- メソッド参照（イベントオブジェクトが自動で渡される） -->
  <div @click="handleClick">クリックエリア</div>

  <!-- 引数を渡す -->
  <button @click="greet('太郎')">挨拶</button>

  <!-- 引数とイベントオブジェクト($event) -->
  <button @click="handleItemClick(123, $event)">アイテム</button>

  <p>カウント: {{ count }}</p>
  <p>{{ message }}</p>
</template>
```

### さまざまなイベント

```vue
<template>
  <!-- マウスイベント -->
  <div @click="onClick">クリック</div>
  <div @dblclick="onDoubleClick">ダブルクリック</div>
  <div @mouseenter="onMouseEnter">マウスオーバー</div>
  <div @mouseleave="onMouseLeave">マウスアウト</div>

  <!-- キーボードイベント -->
  <input @keyup="onKeyUp" />
  <input @keydown="onKeyDown" />
  <input @keypress="onKeyPress" />

  <!-- フォームイベント -->
  <input @input="onInput" />
  <input @change="onChange" />
  <input @focus="onFocus" />
  <input @blur="onBlur" />
  <form @submit="onSubmit">...</form>
</template>
```

## 3.4 イベント修飾子

Vueはイベントハンドリングを簡潔にする修飾子を提供しています。

```vue
<template>
  <!-- .stop — event.stopPropagation() と同等 -->
  <div @click="parentClick">
    <button @click.stop="childClick">
      親に伝播しない
    </button>
  </div>

  <!-- .prevent — event.preventDefault() と同等 -->
  <form @submit.prevent="onSubmit">
    <button type="submit">送信</button>
  </form>

  <!-- .self — event.target が要素自身の場合のみ発火 -->
  <div @click.self="onDivClick">
    <button>このボタンをクリックしても onDivClick は発火しない</button>
  </div>

  <!-- .once — 一度だけ発火 -->
  <button @click.once="onClickOnce">一度だけ</button>

  <!-- .passive — スクロール性能の最適化 -->
  <div @scroll.passive="onScroll">スクロール</div>

  <!-- 修飾子のチェーン -->
  <a @click.stop.prevent="onLink">リンク</a>
</template>
```

### キー修飾子

```vue
<template>
  <!-- 特定のキーだけに反応 -->
  <input @keyup.enter="onEnter" placeholder="Enterで送信" />
  <input @keyup.tab="onTab" />
  <input @keyup.delete="onDelete" />  <!-- Delete と Backspace -->
  <input @keyup.esc="onEscape" />
  <input @keyup.space="onSpace" />
  <input @keyup.up="onUp" />
  <input @keyup.down="onDown" />
  <input @keyup.left="onLeft" />
  <input @keyup.right="onRight" />

  <!-- システム修飾子キー -->
  <input @keyup.ctrl.enter="onCtrlEnter" />
  <input @keyup.alt.s="onAltS" />
  <input @keyup.shift.tab="onShiftTab" />
  <input @keyup.meta.a="onMetaA" />  <!-- Cmd (Mac) / Win (Windows) -->

  <!-- .exact — 指定したキーの組み合わせのみ -->
  <button @click.ctrl.exact="onCtrlClick">
    Ctrl+クリックのみ（Shift+Ctrl+クリックは無視）
  </button>

  <!-- マウスボタン修飾子 -->
  <button @click.left="onLeftClick">左クリック</button>
  <button @click.right="onRightClick">右クリック</button>
  <button @click.middle="onMiddleClick">中クリック</button>
</template>
```

## 3.5 フォーム入力バインディング（v-model）

`v-model` は入力要素とデータの**双方向バインディング**を実現します。

```vue
<script setup>
import { ref } from 'vue'

const text = ref('')
const message = ref('')
const selected = ref('')
const checked = ref(false)
const checkedNames = ref([])
const picked = ref('')
const multiSelected = ref([])
</script>

<template>
  <!-- テキスト入力 -->
  <input v-model="text" placeholder="テキスト入力" />
  <p>入力値: {{ text }}</p>

  <!-- テキストエリア -->
  <textarea v-model="message" placeholder="メッセージ"></textarea>

  <!-- チェックボックス（単一: boolean） -->
  <label>
    <input type="checkbox" v-model="checked" />
    同意する
  </label>
  <p>同意: {{ checked }}</p>

  <!-- チェックボックス（複数: 配列） -->
  <label><input type="checkbox" v-model="checkedNames" value="太郎" /> 太郎</label>
  <label><input type="checkbox" v-model="checkedNames" value="花子" /> 花子</label>
  <label><input type="checkbox" v-model="checkedNames" value="次郎" /> 次郎</label>
  <p>選択: {{ checkedNames }}</p>

  <!-- ラジオボタン -->
  <label><input type="radio" v-model="picked" value="A" /> プランA</label>
  <label><input type="radio" v-model="picked" value="B" /> プランB</label>
  <label><input type="radio" v-model="picked" value="C" /> プランC</label>
  <p>選択: {{ picked }}</p>

  <!-- セレクト（単一） -->
  <select v-model="selected">
    <option disabled value="">選択してください</option>
    <option>東京</option>
    <option>大阪</option>
    <option>名古屋</option>
  </select>
  <p>都市: {{ selected }}</p>

  <!-- セレクト（複数） -->
  <select v-model="multiSelected" multiple>
    <option>JavaScript</option>
    <option>TypeScript</option>
    <option>Python</option>
  </select>
  <p>スキル: {{ multiSelected }}</p>
</template>
```

### v-model の修飾子

```vue
<template>
  <!-- .lazy — input イベントではなく change イベントで同期 -->
  <input v-model.lazy="message" />

  <!-- .number — 入力値を数値に変換 -->
  <input v-model.number="age" type="number" />

  <!-- .trim — 前後の空白を除去 -->
  <input v-model.trim="username" />

  <!-- 修飾子の組み合わせ -->
  <input v-model.lazy.trim="email" />
</template>
```

## 3.6 v-bind の応用

### 動的な属性名

```vue
<script setup>
import { ref } from 'vue'

const attrName = ref('title')
const attrValue = ref('ツールチップ')
</script>

<template>
  <!-- 動的な属性名 -->
  <div v-bind:[attrName]="attrValue">ホバーしてみてください</div>

  <!-- 省略記法 -->
  <div :[attrName]="attrValue">ホバーしてみてください</div>
</template>
```

### 複数の属性を一括バインド

```vue
<script setup>
const inputAttrs = {
  id: 'username',
  type: 'text',
  placeholder: 'ユーザー名を入力',
  class: 'form-input',
  maxlength: 20
}
</script>

<template>
  <!-- v-bind でオブジェクトの全プロパティを属性としてバインド -->
  <input v-bind="inputAttrs" />

  <!-- 上記は以下と同等 -->
  <input
    id="username"
    type="text"
    placeholder="ユーザー名を入力"
    class="form-input"
    maxlength="20"
  />
</template>
```

## 3.7 テンプレート参照（ref属性）

DOM要素やコンポーネントインスタンスに直接アクセスする方法です。

```vue
<script setup>
import { ref, onMounted } from 'vue'

// テンプレート内の ref="inputEl" と同じ名前の変数を宣言
const inputEl = ref(null)
const containerEl = ref(null)

onMounted(() => {
  // マウント後にDOM要素にアクセスできる
  inputEl.value.focus()  // input にフォーカス
  console.log(containerEl.value.offsetWidth)  // 幅を取得
})

function scrollToTop() {
  containerEl.value.scrollTop = 0
}
</script>

<template>
  <input ref="inputEl" placeholder="自動フォーカス" />
  <div ref="containerEl" class="scroll-container">
    <!-- コンテンツ -->
  </div>
  <button @click="scrollToTop">トップへ</button>
</template>
```

### v-for 内での ref

```vue
<script setup>
import { ref, onMounted } from 'vue'

const items = ref(['A', 'B', 'C'])
const itemRefs = ref([])

onMounted(() => {
  console.log(itemRefs.value)  // DOM要素の配列
})
</script>

<template>
  <ul>
    <li v-for="item in items" :key="item" ref="itemRefs">
      {{ item }}
    </li>
  </ul>
</template>
```

## 3.8 本章のまとめ

| ディレクティブ | 用途 | 省略記法 |
|----------------|------|----------|
| `v-if` / `v-else-if` / `v-else` | 条件付き表示（DOM追加/削除） | — |
| `v-show` | 条件付き表示（CSS切り替え） | — |
| `v-for` | リスト描画 | — |
| `v-on:event` | イベントハンドリング | `@event` |
| `v-bind:attr` | 属性バインディング | `:attr` |
| `v-model` | 双方向バインディング | — |
| `ref="name"` | テンプレート参照 | — |

### 修飾子一覧

| 修飾子 | 対象 | 説明 |
|--------|------|------|
| `.stop` | イベント | 伝播を停止 |
| `.prevent` | イベント | デフォルト動作を阻止 |
| `.self` | イベント | 自身が対象の場合のみ |
| `.once` | イベント | 一度だけ発火 |
| `.passive` | イベント | パッシブリスナー |
| `.enter` `.esc` 等 | キー | 特定のキー |
| `.ctrl` `.alt` 等 | キー | システム修飾子 |
| `.lazy` | v-model | change イベントで同期 |
| `.number` | v-model | 数値変換 |
| `.trim` | v-model | 空白除去 |
