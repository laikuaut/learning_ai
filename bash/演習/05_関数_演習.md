この演習では、以下のスキルを確認します。

## 基本問題

### 問題1：基本的な関数定義と呼び出し

名前を引数として受け取り、「こんにちは、○○さん！」と表示する関数 `greet` を作成してください。
引数が指定されなかった場合は「こんにちは、ゲストさん！」と表示するようにしてください。

**期待される出力例：**

```
$ bash greet.sh
こんにちは、ゲストさん！
こんにちは、太郎さん！
こんにちは、花子さん！
```

<details>
<summary>ヒント</summary>

- Bashの関数は `function 関数名 { ... }` または `関数名() { ... }` で定義します
- デフォルト値は `${1:-デフォルト値}` の構文で設定できます
- 関数の引数は `$1`, `$2`, ... で参照します

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 関数定義（引数のデフォルト値を設定）
greet() {
    # $1 が未指定の場合は "ゲスト" をデフォルト値にする
    local name="${1:-ゲスト}"
    echo "こんにちは、${name}さん！"
}

# 関数の呼び出し
greet          # 引数なし → デフォルト値を使用
greet "太郎"   # 引数あり
greet "花子"   # 引数あり
```

</details>

### 問題2：戻り値を返す関数（echo パターン）

2つの整数を引数に取り、その合計を返す関数 `add` と、積を返す関数 `multiply` を作成してください。
それぞれの関数の結果をコマンド置換 `$(...)` で受け取って表示してください。

**期待される出力例：**

```
$ bash calc.sh
10 + 3 = 13
10 * 3 = 30
```

<details>
<summary>ヒント</summary>

- Bashの `return` はステータスコード（0〜255）しか返せません
- 文字列や数値を返すには `echo` を使い、呼び出し側で `$(関数名 引数)` で受け取ります
- 算術演算は `$(( ))` で行えます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# echo で結果を出力し、呼び出し側で $(関数名) で受け取るパターン
add() {
    # 2つの引数を加算した結果を echo で返す
    echo $(( $1 + $2 ))
}

multiply() {
    # 2つの引数を乗算した結果を echo で返す
    echo $(( $1 * $2 ))
}

# コマンド置換で関数の出力を変数に格納
sum=$(add 10 3)
product=$(multiply 10 3)

echo "10 + 3 = ${sum}"
echo "10 * 3 = ${product}"
```

</details>

### 問題3：return によるステータスコードの活用

引数として渡されたファイルパスが存在して読み取り可能かどうかを判定する関数 `is_readable_file` を作成してください。
ファイルが存在して読み取り可能なら `return 0`（成功）、そうでなければ `return 1`（失敗）を返すようにしてください。

**期待される出力例：**

```
$ bash check_file.sh /etc/hostname
/etc/hostname は読み取り可能です
$ bash check_file.sh /nonexistent
/nonexistent は読み取れません
```

<details>
<summary>ヒント</summary>

- `return 0` は成功、`return 1` は失敗を意味します
- `-f` でファイルの存在確認、`-r` で読み取り権限の確認ができます
- 関数の戻り値は `if 関数名; then ...` や `関数名 && ...` で判定します

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# ファイルの読み取り可能性を判定する関数
is_readable_file() {
    local filepath="$1"

    # ファイルが存在し、かつ読み取り可能かチェック
    if [[ -f "$filepath" && -r "$filepath" ]]; then
        return 0  # 成功（読み取り可能）
    else
        return 1  # 失敗（読み取り不可）
    fi
}

# 引数からファイルパスを取得
target="${1:-/etc/hostname}"

# if文で関数の戻り値を判定
if is_readable_file "$target"; then
    echo "${target} は読み取り可能です"
else
    echo "${target} は読み取れません"
fi
```

</details>

### 問題4：ローカル変数とスコープ

以下の要件を満たすスクリプトを作成してください。

1. グローバル変数 `count` を `0` に初期化する
2. 関数 `increment` は `local` を使ったローカル変数 `count` を `100` に設定して表示する
3. 関数 `increment_global` は `local` を使わずに `count` を `+1` する
4. 各関数の呼び出し前後でグローバルの `count` を表示して、スコープの違いを確認する

**期待される出力例：**

```
$ bash scope.sh
グローバル count: 0
increment 内の count: 100
グローバル count: 0
increment_global 呼び出し後の count: 1
increment_global 呼び出し後の count: 2
```

<details>
<summary>ヒント</summary>

- `local` キーワードを付けると、変数は関数のスコープ内だけで有効になります
- `local` を付けないと、関数内でもグローバル変数を直接変更します
- Bashではデフォルトですべての変数がグローバルスコープです

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# グローバル変数の初期化
count=0

# local を使う関数（グローバル変数に影響しない）
increment() {
    local count=100  # ローカル変数として宣言
    echo "increment 内の count: ${count}"
}

# local を使わない関数（グローバル変数を直接変更する）
increment_global() {
    count=$(( count + 1 ))  # グローバル変数を変更
}

# 動作確認
echo "グローバル count: ${count}"
increment
echo "グローバル count: ${count}"      # local のおかげで 0 のまま

increment_global
echo "increment_global 呼び出し後の count: ${count}"  # 1 に増加

increment_global
echo "increment_global 呼び出し後の count: ${count}"  # 2 に増加
```

</details>

### 問題5：可変長引数の処理

ユーザー情報を表示する関数 `show_user_info` を作成してください。
この関数は可変長引数を受け取り、以下のように処理します。

- 第1引数：ユーザー名（必須）
- 第2引数：年齢（省略時は「不明」）
- 第3引数以降：趣味（0個以上）

**期待される出力例：**

```
$ bash user_info.sh
=== ユーザー情報 ===
名前: 太郎
年齢: 25
趣味: 読書, プログラミング, 映画鑑賞
=== ユーザー情報 ===
名前: 花子
年齢: 不明
趣味: なし
```

<details>
<summary>ヒント</summary>

- `$#` で引数の数を取得できます
- `shift` で引数を1つずらすことができます
- `$@` で残りのすべての引数を取得できます
- 配列を `IFS` とともに使うと、カンマ区切りの文字列を作れます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

show_user_info() {
    # 第1引数：ユーザー名（必須チェック）
    local name="${1:?ユーザー名は必須です}"
    shift  # 第1引数を消費

    # 第2引数：年齢（デフォルト値「不明」）
    local age="${1:-不明}"
    # 引数が残っている場合のみ shift
    [[ $# -gt 0 ]] && shift

    # 残りの引数を趣味として配列に格納
    local hobbies=("$@")

    echo "=== ユーザー情報 ==="
    echo "名前: ${name}"
    echo "年齢: ${age}"

    # 趣味がある場合はカンマ区切りで表示
    if [[ ${#hobbies[@]} -gt 0 ]]; then
        # IFS を一時的に変更してカンマ区切りに
        local IFS=", "
        echo "趣味: ${hobbies[*]}"
    else
        echo "趣味: なし"
    fi
}

# 趣味ありのパターン
show_user_info "太郎" 25 "読書" "プログラミング" "映画鑑賞"

# 年齢・趣味なしのパターン
show_user_info "花子"
```

</details>

### 問題6：再帰関数

再帰関数を使って、指定された正の整数の階乗（factorial）を計算する関数 `factorial` を作成してください。
また、入力が負の数や数値でない場合はエラーメッセージを表示してください。

**期待される出力例：**

```
$ bash factorial.sh
5! = 120
10! = 3628800
0! = 1
エラー: 正の整数を指定してください
```

<details>
<summary>ヒント</summary>

- 再帰関数は、関数の中から自分自身を呼び出す関数です
- 階乗の定義: `n! = n * (n-1)!`、`0! = 1`（基底条件）
- Bashの再帰呼び出しは他の言語と同様に書けます
- 入力値のバリデーションは正規表現 `=~` で行えます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 再帰関数で階乗を計算
factorial() {
    local n="$1"

    # 入力値のバリデーション
    if ! [[ "$n" =~ ^[0-9]+$ ]]; then
        echo "エラー: 正の整数を指定してください" >&2
        return 1
    fi

    # 基底条件: 0! = 1
    if [[ "$n" -le 1 ]]; then
        echo 1
        return 0
    fi

    # 再帰呼び出し: n * (n-1)!
    local sub
    sub=$(factorial $(( n - 1 )))
    echo $(( n * sub ))
}

# テスト実行
result=$(factorial 5)
echo "5! = ${result}"

result=$(factorial 10)
echo "10! = ${result}"

result=$(factorial 0)
echo "0! = ${result}"

# エラーケース
factorial "abc"
```

</details>

### 問題7：関数ライブラリの作成と読み込み

以下の2つのファイルを作成してください。

**ファイル1：`string_utils.sh`（関数ライブラリ）**
- `to_upper` : 引数の文字列を大文字に変換して出力
- `to_lower` : 引数の文字列を小文字に変換して出力
- `str_repeat` : 第1引数の文字列を第2引数の回数だけ繰り返して出力

**ファイル2：`main.sh`（メインスクリプト）**
- `source` で `string_utils.sh` を読み込み、各関数を呼び出して結果を表示

**期待される出力例：**

```
$ bash main.sh
大文字: HELLO WORLD
小文字: hello world
繰り返し: =-==-==-=
```

<details>
<summary>ヒント</summary>

- Bashの大文字変換は `${変数^^}`、小文字変換は `${変数,,}` で行えます
- 他のファイルの関数を読み込むには `source ファイル名` または `. ファイル名` を使います
- `source` するファイルのパスは、スクリプト自身のディレクトリを基準にすると安全です
- スクリプト自身のディレクトリは `$(cd "$(dirname "$0")" && pwd)` で取得できます

</details>

<details>
<summary>解答例</summary>

```bash
# === string_utils.sh（関数ライブラリ） ===
#!/bin/bash
# 文字列ユーティリティ関数集
# source で読み込んで使用する

# 大文字に変換
to_upper() {
    local str="$1"
    echo "${str^^}"
}

# 小文字に変換
to_lower() {
    local str="$1"
    echo "${str,,}"
}

# 文字列を指定回数繰り返す
str_repeat() {
    local str="$1"
    local count="${2:-1}"
    local result=""
    local i

    for (( i = 0; i < count; i++ )); do
        result+="${str}"
    done
    echo "$result"
}
```

```bash
# === main.sh（メインスクリプト） ===
#!/bin/bash

# スクリプト自身のディレクトリを取得（相対パス問題を回避）
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"

# 関数ライブラリを読み込み
source "${SCRIPT_DIR}/string_utils.sh"

# 各関数を呼び出し
upper=$(to_upper "hello world")
echo "大文字: ${upper}"

lower=$(to_lower "HELLO WORLD")
echo "小文字: ${lower}"

repeated=$(str_repeat "=-" 4)
echo "繰り返し: ${repeated}="
```

</details>

### 問題8：汎用ログ関数の作成

以下の機能を持つログ出力関数群を作成してください。

- `log_info` : `[INFO]` プレフィックス付きで標準出力にメッセージを表示
- `log_warn` : `[WARN]` プレフィックス付きで標準エラー出力にメッセージを表示
- `log_error` : `[ERROR]` プレフィックス付きで標準エラー出力にメッセージを表示
- すべてのログに現在時刻（`YYYY-MM-DD HH:MM:SS` 形式）を付加する
- 内部的にはプライベート関数 `_log` を1つ作り、各関数はそれを呼び出す構成にする

**期待される出力例（時刻は実行時のもの）：**

```
$ bash logger.sh
[2026-03-27 10:30:45] [INFO] アプリケーションを起動しました
[2026-03-27 10:30:45] [WARN] 設定ファイルが見つかりません。デフォルト値を使用します
[2026-03-27 10:30:45] [ERROR] データベースへの接続に失敗しました
--- 標準出力のみ確認 ---
[2026-03-27 10:30:45] [INFO] アプリケーションを起動しました
```

<details>
<summary>ヒント</summary>

- `date '+%Y-%m-%d %H:%M:%S'` で現在時刻を取得できます
- 標準エラー出力は `>&2` でリダイレクトします
- 共通処理は内部関数にまとめると、コードの重複を減らせます
- 最後のテストでは `2>/dev/null` で標準エラーを非表示にして、標準出力だけを確認しています

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# プライベートなログ出力関数（内部用）
# 引数: $1=ログレベル, $2=メッセージ, $3=出力先fd（1:stdout, 2:stderr）
_log() {
    local level="$1"
    local message="$2"
    local fd="${3:-1}"  # デフォルトは標準出力

    # 現在時刻を取得
    local timestamp
    timestamp=$(date '+%Y-%m-%d %H:%M:%S')

    # 指定のファイルディスクリプタに出力
    echo "[${timestamp}] [${level}] ${message}" >&${fd}
}

# INFO レベル（標準出力）
log_info() {
    _log "INFO" "$1" 1
}

# WARN レベル（標準エラー出力）
log_warn() {
    _log "WARN" "$1" 2
}

# ERROR レベル（標準エラー出力）
log_error() {
    _log "ERROR" "$1" 2
}

# 使用例
log_info "アプリケーションを起動しました"
log_warn "設定ファイルが見つかりません。デフォルト値を使用します"
log_error "データベースへの接続に失敗しました"

# 標準出力のみ確認（標準エラーを捨てる）
echo "--- 標準出力のみ確認 ---"
log_info "アプリケーションを起動しました"
log_warn "これは表示されません（標準エラーのため）" 2>/dev/null
log_error "これも表示されません（標準エラーのため）" 2>/dev/null
```

</details>
