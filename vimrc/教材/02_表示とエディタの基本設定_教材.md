# 第2章：表示とエディタの基本設定

## この章のゴール

- 行番号の表示方法（絶対行番号・相対行番号・ハイブリッド）を設定できる
- カーソルや画面表示に関する設定を使いこなせる
- シンタックスハイライトとファイルタイプ検出を有効化できる
- カラースキームを変更し、ターミナルの色設定を理解する
- ステータスラインをカスタマイズできる
- 不可視文字（タブ・スペース・改行）を可視化できる

---

## 1. 行番号の表示

### 3つの行番号モード

Vim には3種類の行番号表示があります。

```vim
set number              " 絶対行番号を表示
set relativenumber      " 相対行番号を表示
set number relativenumber  " ハイブリッド（両方を組み合わせ）
```

### 表示の違い

```
  number のみ:           relativenumber のみ:     ハイブリッド:
  ┌────┬──────────┐     ┌────┬──────────┐       ┌────┬──────────┐
  │  1 │ aaa      │     │  3 │ aaa      │       │  3 │ aaa      │
  │  2 │ bbb      │     │  2 │ bbb      │       │  2 │ bbb      │
  │  3 │ ccc      │     │  1 │ ccc      │       │  1 │ ccc      │
  │  4 │ ddd ←カーソル│  │  0 │ ddd ←カーソル│   │  4 │ ddd ←カーソル│
  │  5 │ eee      │     │  1 │ eee      │       │  1 │ eee      │
  │  6 │ fff      │     │  2 │ fff      │       │  2 │ fff      │
  │  7 │ ggg      │     │  3 │ ggg      │       │  3 │ ggg      │
  └────┴──────────┘     └────┴──────────┘       └────┴──────────┘

  ハイブリッド = 現在行は絶対行番号、他の行は相対行番号
```

### なぜ相対行番号が便利なのか

相対行番号を使うと、`5j`（5行下に移動）や `3dk`（3行上まで削除）などの操作で、何行移動すればよいか一目でわかります。

```vim
" 推奨: ハイブリッド表示
set number
set relativenumber
```

### 行番号の表示を切り替えるマッピング

```vim
" Ctrl+n で行番号表示を切り替え
nnoremap <C-n> :set relativenumber!<CR>
```

### ポイントまとめ

- `number` + `relativenumber` のハイブリッド表示が最も実用的です
- 相対行番号があると `5j` `10dd` などの行数指定操作が楽になります

## 2. カーソルと表示

### カーソル表示の設定

```vim
set cursorline          " 現在行をハイライト表示
" set cursorcolumn      " 現在列をハイライト表示（好みで）
```

```
  cursorline の効果:

  │  1 │ def hello():              │
  │  2 │ ████████████████████████  │  ← 現在行がハイライトされる
  │  3 │     return "world"        │
```

### 入力中のコマンド表示

```vim
set showcmd             " 入力中のコマンドを右下に表示
set showmode            " 現在のモードを左下に表示（デフォルトで ON）
set ruler               " カーソル位置（行:列）を右下に表示
```

```
  showcmd の効果:
  ノーマルモードで d を押した時:
  ┌──────────────────────────────────────┐
  │                                      │
  │  -- INSERT --              d  42,10  │
  │  ↑ showmode        showcmd↑  ↑ruler  │
  └──────────────────────────────────────┘
```

### 折り返しとスクロール

```vim
set wrap                " 長い行を折り返して表示（デフォルト）
set nowrap              " 折り返さない（横スクロールで表示）
set linebreak           " 単語の途中で折り返さない
set breakindent         " 折り返し行にもインデントを適用

set scrolloff=5         " カーソルの上下に常に5行表示する
set sidescrolloff=5     " カーソルの左右に常に5列表示する（nowrap 時）
```

```
  scrolloff の効果:

  scrolloff=0:              scrolloff=5:
  ┌──────────────────┐     ┌──────────────────┐
  │ line 1           │     │ line 5           │
  │ line 2           │     │ line 6           │
  │ line 3           │     │ line 7           │
  │ cursor ←画面端   │     │ line 8           │
  └──────────────────┘     │ line 9           │
                           │ cursor ←余白あり │
                           │ line 11          │
                           │ line 12          │
                           │ line 13          │
                           │ line 14          │
                           │ line 15          │
                           └──────────────────┘
```

### signcolumn の設定

```vim
set signcolumn=yes      " サイン列を常に表示（Git差分やLintの印に使用）
" set signcolumn=auto   " サインがある時だけ表示（デフォルト）
" set signcolumn=number " 行番号列にサインを重ねて表示
```

### ポイントまとめ

- `cursorline` で現在行をハイライトすると視認性が上がります
- `scrolloff=5` でカーソル周辺に常に余白を確保できます
- `signcolumn=yes` で画面のガタつきを防げます

## 3. シンタックスハイライト

### syntax on

```vim
syntax on               " シンタックスハイライトを有効化
" syntax off            " 無効化する場合
" syntax enable         " on と同等（ユーザー設定を保持する点が異なる）
```

`syntax on` は、ファイルの内容をプログラミング言語の構文に基づいて色分け表示します。キーワード、文字列、コメント、数値などが異なる色で表示されます。

### filetype の設定

```vim
filetype on             " ファイルタイプの検出を有効化
filetype plugin on      " ファイルタイプ別プラグインを有効化
filetype indent on      " ファイルタイプ別インデントを有効化

" 上記3つをまとめて有効化（推奨）
filetype plugin indent on
```

### filetype の各オプション

```
  ┌─────────────────────────┬──────────────────────────────────┐
  │ コマンド                │ 効果                             │
  ├─────────────────────────┼──────────────────────────────────┤
  │ filetype on             │ .py → python のように拡張子で    │
  │                         │ ファイルタイプを自動判別         │
  ├─────────────────────────┼──────────────────────────────────┤
  │ filetype plugin on      │ ファイルタイプに応じたプラグイン │
  │                         │ （補完、コメント等）を読み込む   │
  ├─────────────────────────┼──────────────────────────────────┤
  │ filetype indent on      │ ファイルタイプに応じたインデント │
  │                         │ ルールを読み込む                 │
  ├─────────────────────────┼──────────────────────────────────┤
  │ filetype plugin indent on │ 上記3つを全て有効化（推奨）   │
  └─────────────────────────┴──────────────────────────────────┘
```

### ポイントまとめ

- `syntax on` でシンタックスハイライトを有効化します
- `filetype plugin indent on` で言語ごとの設定を自動化します
- この2つは vimrc の冒頭に書くのが定番です

## 4. カラースキーム

### カラースキームの設定

```vim
colorscheme desert      " カラースキームを設定
set background=dark     " 暗い背景用の配色にする
" set background=light  " 明るい背景用の配色にする
```

### 組み込みカラースキーム一覧

```vim
" Vim に標準で含まれるカラースキーム:
" blue, darkblue, default, delek, desert, elflord,
" evening, habamax, industry, koehler, lunaperche,
" morning, murphy, pablo, peachpuff, quiet, ron,
" shine, slate, torte, wildcharm, zaibatsu

" 利用可能なカラースキームを確認
:colorscheme <Tab>      " Tab で候補を表示

" おすすめ（暗い背景向け）:
" desert    - 見やすく定番
" slate     - 落ち着いた色合い
" industry  - コントラスト高め
" habamax   - モダンな見た目（Vim 9.0+）
```

### ターミナルの色数設定

```vim
" 256色を有効化
set t_Co=256

" True Color（24ビットカラー）を有効化
if has('termguicolors')
  set termguicolors
endif
```

```
  色数の違い:

  8色       → 基本的な色のみ
  256色     → 十分な色数（ほとんどのターミナルで対応）
  True Color → 1677万色（モダンなターミナルで対応）

  ターミナルが対応していれば termguicolors を有効にしましょう。
  対応状況は以下で確認できます:
  $ echo $TERM
  $ tput colors
```

### ポイントまとめ

- `colorscheme` でVimの見た目を変更できます
- `set background=dark/light` で背景の明暗に合わせた配色になります
- `termguicolors` を設定すると、より多くの色を使えます

## 5. ステータスラインの設定

### ステータスラインの有効化

```vim
set laststatus=2        " ステータスラインを常に表示
                        " 0: 表示しない
                        " 1: ウィンドウが2つ以上の時のみ
                        " 2: 常に表示（推奨）
```

### statusline の書式要素

```
  ┌──────────┬──────────────────────────────────────┐
  │ 要素     │ 説明                                 │
  ├──────────┼──────────────────────────────────────┤
  │ %f       │ ファイルパス（相対）                  │
  │ %F       │ ファイルパス（絶対）                  │
  │ %t       │ ファイル名のみ                       │
  │ %m       │ 変更フラグ [+]                       │
  │ %r       │ 読み取り専用フラグ [RO]              │
  │ %h       │ ヘルプフラグ [Help]                  │
  │ %w       │ プレビューフラグ [Preview]            │
  │ %y       │ ファイルタイプ [python]               │
  │ %{&encoding} │ エンコーディング                  │
  │ %{&ff}   │ ファイルフォーマット（unix/dos/mac）  │
  │ %l       │ 現在の行番号                         │
  │ %L       │ 全行数                               │
  │ %c       │ 現在のカラム番号                     │
  │ %p       │ ファイル内の位置（パーセント）        │
  │ %%       │ リテラルの %                         │
  │ %=       │ 左右の区切り（以降は右寄せ）         │
  └──────────┴──────────────────────────────────────┘
```

### 実用的なステータスライン設定

```vim
" シンプルで情報量の多いステータスライン
set statusline=
set statusline+=\ %f                    " ファイルパス
set statusline+=\ %m%r%h%w              " 各種フラグ
set statusline+=%=                      " 右寄せ開始
set statusline+=\ %y                    " ファイルタイプ
set statusline+=\ [%{&encoding}]        " エンコーディング
set statusline+=\ [%{&fileformat}]      " ファイルフォーマット
set statusline+=\ %l:%c                 " 行:列
set statusline+=\ [%p%%]               " 位置（パーセント）
set statusline+=\                       " 末尾スペース
```

```
  表示例:
  ┌──────────────────────────────────────────────────────────┐
  │  src/main.py [+]       [python] [utf-8] [unix] 42:10 [65%] │
  └──────────────────────────────────────────────────────────┘
```

### ステータスラインの色設定

```vim
" ステータスラインの色をカスタマイズ
highlight StatusLine   ctermfg=White ctermbg=DarkBlue
highlight StatusLineNC ctermfg=Gray  ctermbg=DarkGray   " 非アクティブウィンドウ
```

### ポイントまとめ

- `set laststatus=2` でステータスラインを常に表示します
- `%f` `%l` `%c` などの書式要素を組み合わせてカスタマイズします
- `%=` で左寄せと右寄せを区切ります

## 6. タブラインの設定

### タブラインの表示

```vim
set showtabline=2       " タブラインを常に表示
                        " 0: 表示しない
                        " 1: タブが2つ以上の時のみ（デフォルト）
                        " 2: 常に表示
```

### tabline のカスタマイズ

```vim
" シンプルなタブライン設定
set tabline=%!MyTabLine()

function! MyTabLine()
  let s = ''
  for i in range(tabpagenr('$'))
    " タブ番号（1始まり）
    let tabnr = i + 1
    " アクティブなタブかどうか
    if tabnr == tabpagenr()
      let s .= '%#TabLineSel#'
    else
      let s .= '%#TabLine#'
    endif
    " タブ番号とファイル名
    let bufnr = tabpagebuflist(tabnr)[tabpagewinnr(tabnr) - 1]
    let fname = fnamemodify(bufname(bufnr), ':t')
    if fname == ''
      let fname = '[No Name]'
    endif
    let s .= ' ' . tabnr . ':' . fname . ' '
  endfor
  let s .= '%#TabLineFill#'
  return s
endfunction
```

### ポイントまとめ

- `showtabline=2` でタブラインを常に表示できます
- `tabline` で表示内容をカスタマイズ可能です

## 7. 不可視文字の表示

### list と listchars

不可視文字（タブ、末尾スペース、改行など）を表示するには `list` オプションを使います。

```vim
set list                " 不可視文字を表示する
set listchars=tab:▸\ ,trail:·,eol:¬,extends:❯,precedes:❮,nbsp:␣
```

### listchars の各要素

```
  ┌──────────┬────────────────────────────────────────┐
  │ 要素     │ 説明                                   │
  ├──────────┼────────────────────────────────────────┤
  │ tab:▸\   │ タブ文字を ▸ とスペースで表示          │
  │ trail:·  │ 行末の余分なスペースを · で表示        │
  │ eol:¬    │ 改行文字を ¬ で表示                    │
  │ extends:❯│ 画面右端で切れた行を ❯ で表示          │
  │ precedes:❮│ 画面左端で切れた行を ❮ で表示         │
  │ nbsp:␣   │ ノーブレークスペースを ␣ で表示        │
  │ space:·  │ スペースを · で表示（Vim 8.0+）        │
  └──────────┴────────────────────────────────────────┘
```

### 表示例

```
  list が OFF の場合:
  │ def hello():          │
  │     return "world"    │
  │                       │

  list が ON の場合（listchars 設定済み）:
  │ def·hello():¬         │
  │ ▸   return·"world"··¬ │  ← 末尾スペースが · で見える
  │ ¬                     │     タブが ▸ で見える
```

### ターミナルが特殊文字に対応しない場合

```vim
" ASCII 文字のみを使う安全な設定
set listchars=tab:>-,trail:~,eol:$,extends:>,precedes:<,nbsp:+
```

### list 表示の切り替え

```vim
" Leader + l で不可視文字表示を切り替え
nnoremap <Leader>l :set list!<CR>
```

### ポイントまとめ

- `set list` で不可視文字を可視化できます
- `listchars` で表示する文字をカスタマイズします
- `trail` で末尾スペースを検出できるのが特に便利です
- 特殊文字が表示できない環境では ASCII 文字を使いましょう
