# 第7章 演習：入出力とリダイレクト

---

## 演習の目標

この演習では、以下のスキルを確認します。

- 標準出力・標準エラー出力のリダイレクトを正しく使い分けられる
- ヒアドキュメントを使って複数行の入力を渡せる
- パイプでコマンドを連結し、データを加工できる
- `tee` コマンドで出力を分岐できる
- プロセス置換（process substitution）を活用できる
- ファイルディスクリプタを操作してエラー出力を制御できる

## 基本問題

### 問題1：リダイレクトの基本

以下の操作を行うスクリプトを作成してください。

1. `echo` で「Hello, World!」を `output.txt` に書き込む（上書き）
2. `echo` で「追加の行です」を `output.txt` に追記する
3. `output.txt` の内容を `cat` で表示する
4. 存在しないファイルを `cat` で読もうとし、エラーメッセージを `error.log` にリダイレクトする
5. 最後に作成したファイルを削除して後片付けする

**期待される出力例：**

```
$ bash redirect_basic.sh
=== output.txt の内容 ===
Hello, World!
追加の行です

=== エラーを error.log に記録 ===
error.log の内容: cat: /tmp/nonexistent_file: No such file or directory

後片付け完了
```

<details>
<summary>ヒント</summary>

- 上書きリダイレクトは `>` 、追記リダイレクトは `>>` を使います
- 標準エラー出力のリダイレクトは `2>` を使います
- ファイルの削除は `rm -f` で行います（`-f` はエラーを抑制）

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 作業用の一時ディレクトリを使用
tmpdir="/tmp/redirect_test_$$"
mkdir -p "$tmpdir"

# 1. 上書きリダイレクト（> でファイルに書き込み。既存内容は消える）
echo "Hello, World!" > "$tmpdir/output.txt"

# 2. 追記リダイレクト（>> でファイル末尾に追加）
echo "追加の行です" >> "$tmpdir/output.txt"

# 3. ファイルの内容を表示
echo "=== output.txt の内容 ==="
cat "$tmpdir/output.txt"

echo ""

# 4. 標準エラー出力をファイルにリダイレクト
# 2> は標準エラー出力（fd 2）をリダイレクトする
echo "=== エラーを error.log に記録 ==="
cat /tmp/nonexistent_file 2> "$tmpdir/error.log"
echo "error.log の内容: $(cat "$tmpdir/error.log")"

echo ""

# 5. 後片付け
rm -rf "$tmpdir"
echo "後片付け完了"
```

</details>

### 問題2：ヒアドキュメント

以下の要件を満たすスクリプトを作成してください。

1. ヒアドキュメントを使って、複数行のテキストを画面に表示する（変数展開あり）
2. ヒアドキュメントを使って、変数展開を抑制した状態でテキストを表示する
3. ヒアドキュメントを使って、`cat` 経由でファイルに内容を書き込む

**期待される出力例：**

```
$ bash heredoc.sh
=== 変数展開あり ===
サーバー情報レポート
ホスト名: myhost
日付: 2026-03-27
ユーザー: user

=== 変数展開なし ===
変数 $HOSTNAME はそのまま表示されます
コマンド置換 $(date) も展開されません

=== ファイルに書き込み ===
[config]
host=localhost
port=8080
```

<details>
<summary>ヒント</summary>

- 変数展開あり：`cat <<EOF ... EOF`
- 変数展開なし：`cat <<'EOF' ... EOF`（区切り文字をシングルクォートで囲む）
- ファイルへの書き込み：`cat <<EOF > ファイル名`

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 1. 変数展開ありのヒアドキュメント
# EOF をクォートしないと、$変数 や $(コマンド) が展開される
echo "=== 変数展開あり ==="
cat <<EOF
サーバー情報レポート
ホスト名: $(hostname)
日付: $(date +%Y-%m-%d)
ユーザー: $USER
EOF

echo ""

# 2. 変数展開なしのヒアドキュメント
# 'EOF' とシングルクォートで囲むと展開が抑制される
echo "=== 変数展開なし ==="
cat <<'EOF'
変数 $HOSTNAME はそのまま表示されます
コマンド置換 $(date) も展開されません
EOF

echo ""

# 3. ヒアドキュメントでファイルに書き込む
tmpfile="/tmp/heredoc_test_$$.conf"
cat <<EOF > "$tmpfile"
[config]
host=localhost
port=8080
EOF

echo "=== ファイルに書き込み ==="
cat "$tmpfile"

# 後片付け
rm -f "$tmpfile"
```

</details>

### 問題3：パイプの活用

以下の要件を満たすワンライナーまたはスクリプトを作成してください。

1. `/etc/passwd` の内容からユーザー名（第1フィールド）だけを抽出し、アルファベット順にソートして最初の5件を表示する
2. `echo` で出力した数値列をパイプで `sort` と `head` に渡して処理する
3. `ps aux` の出力から、メモリ使用量（第4フィールド）が最も大きいプロセスの上位3件を表示する

**期待される出力例：**

```
$ bash pipe_practice.sh
=== /etc/passwd のユーザー名（先頭5件） ===
_apt
backup
bin
daemon
games

=== 数値のソート ===
元データ: 42 17 83 5 61 29
昇順: 5 17 29 42 61 83

=== メモリ使用量 上位3プロセス ===
USER       %MEM  COMMAND
user        2.1  /usr/bin/python3
root        1.5  /usr/sbin/mysqld
user        0.8  bash
```

<details>
<summary>ヒント</summary>

- フィールド抽出は `cut -d: -f1` または `awk -F: '{print $1}'` で行えます
- 数値ソートは `sort -n`、逆順は `sort -rn` を使います
- `ps aux --sort=-%mem` でメモリ順にソートできます
- `head -n 件数` で先頭行を取得します

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

# 1. /etc/passwd からユーザー名を抽出してソート
# cut -d: -f1 : コロン区切りの第1フィールドを取得
# sort : アルファベット順にソート
# head -n 5 : 先頭5行だけ表示
echo "=== /etc/passwd のユーザー名（先頭5件） ==="
cut -d: -f1 /etc/passwd | sort | head -n 5

echo ""

# 2. 数値のソート
# tr でスペースを改行に変換し、sort -n で数値として昇順ソート
echo "=== 数値のソート ==="
data="42 17 83 5 61 29"
echo "元データ: $data"
sorted=$(echo "$data" | tr ' ' '\n' | sort -n | tr '\n' ' ')
echo "昇順: $sorted"

echo ""

# 3. メモリ使用量の上位プロセス
# ps aux で全プロセスを表示し、awk で必要な列を抽出
# sort -rnk4 で第4フィールド（%MEM）を数値として逆順ソート
# head で上位3件を取得
echo "=== メモリ使用量 上位3プロセス ==="
echo "USER       %MEM  COMMAND"
ps aux --sort=-%mem | awk 'NR>1 {printf "%-10s %4s  %s\n", $1, $4, $11}' | head -n 3
```

</details>

### 問題4：tee コマンドの活用

以下の要件を満たすスクリプトを作成してください。

1. コマンドの出力を画面に表示しながら、同時にファイルにも保存する（`tee`）
2. `tee -a` で追記モードの動作を確認する
3. パイプラインの途中で `tee` を使い、中間結果を保存しながら次の処理に渡す

**期待される出力例：**

```
$ bash tee_practice.sh
=== tee で画面とファイルに同時出力 ===
現在の日時: 2026-03-27 10:30:45
ホスト名: myhost
ユーザー: user

保存されたファイルの内容:
現在の日時: 2026-03-27 10:30:45
ホスト名: myhost
ユーザー: user

=== パイプライン途中の tee ===
最終結果（先頭3件）:
bin
daemon
games

中間結果（ソート前の全ユーザー名）の行数: 25
```

<details>
<summary>ヒント</summary>

- `コマンド | tee ファイル名` で画面表示とファイル保存を同時に行えます
- `tee -a` は追記モードです（`>>`と同じ効果）
- パイプラインの途中で `tee 中間ファイル` を挟むと、中間結果を保存しつつ次に渡せます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

tmpdir="/tmp/tee_test_$$"
mkdir -p "$tmpdir"

# 1. tee で画面とファイルに同時出力
# パイプで tee に渡すと、標準出力（画面）とファイルの両方に書き込む
echo "=== tee で画面とファイルに同時出力 ==="
{
    echo "現在の日時: $(date '+%Y-%m-%d %H:%M:%S')"
    echo "ホスト名: $(hostname)"
    echo "ユーザー: $USER"
} | tee "$tmpdir/system_info.txt"

echo ""
echo "保存されたファイルの内容:"
cat "$tmpdir/system_info.txt"

echo ""

# 2. パイプライン途中の tee
# cut → tee（中間結果を保存）→ sort → head の流れ
echo "=== パイプライン途中の tee ==="
echo "最終結果（先頭3件）:"
cut -d: -f1 /etc/passwd \
    | tee "$tmpdir/all_users.txt" \
    | sort \
    | head -n 3

echo ""
# 中間結果の行数を確認
line_count=$(wc -l < "$tmpdir/all_users.txt")
echo "中間結果（ソート前の全ユーザー名）の行数: $line_count"

# 後片付け
rm -rf "$tmpdir"
```

</details>

### 問題5：標準出力と標準エラー出力の分離・結合

以下の要件を満たすスクリプトを作成してください。

1. 標準出力と標準エラー出力の両方を生成する関数 `mixed_output` を作成する
2. 標準出力だけをファイルに保存し、標準エラー出力は画面に表示する
3. 標準エラー出力だけをファイルに保存し、標準出力は画面に表示する
4. 両方を同じファイルにリダイレクトする
5. 標準エラー出力を標準出力にマージしてパイプに渡す

**期待される出力例：**

```
$ bash fd_practice.sh
=== 標準出力のみファイルに保存 ===
画面（stderr）: [ERROR] ファイルが見つかりません
[INFO] 処理を開始します
[INFO] 処理が完了しました

=== 標準エラー出力のみファイルに保存 ===
画面（stdout）: [INFO] 処理を開始します
画面（stdout）: [INFO] 処理が完了しました

=== 両方を同じファイルに保存 ===
all_output.log の内容:
[INFO] 処理を開始します
[ERROR] ファイルが見つかりません
[INFO] 処理が完了しました

=== stderr を stdout にマージして grep ===
ERROR行のみ: [ERROR] ファイルが見つかりません
```

<details>
<summary>ヒント</summary>

- 標準出力のリダイレクト：`> file` または `1> file`
- 標準エラー出力のリダイレクト：`2> file`
- 両方を同じ場所：`> file 2>&1` または `&> file`
- stderr を stdout にマージ：`2>&1`（パイプの前に記述）

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

tmpdir="/tmp/fd_test_$$"
mkdir -p "$tmpdir"

# 標準出力と標準エラー出力の両方を生成する関数
mixed_output() {
    echo "[INFO] 処理を開始します"        # stdout（fd 1）
    echo "[ERROR] ファイルが見つかりません" >&2  # stderr（fd 2）
    echo "[INFO] 処理が完了しました"       # stdout（fd 1）
}

# 1. 標準出力のみファイルに保存（stderr は画面に表示される）
echo "=== 標準出力のみファイルに保存 ==="
echo -n "画面（stderr）: "
mixed_output > "$tmpdir/stdout.log"
cat "$tmpdir/stdout.log"

echo ""

# 2. 標準エラー出力のみファイルに保存（stdout は画面に表示される）
echo "=== 標準エラー出力のみファイルに保存 ==="
mixed_output 2> "$tmpdir/stderr.log" | while IFS= read -r line; do
    echo "画面（stdout）: $line"
done

echo ""

# 3. 両方を同じファイルにリダイレクト
# &> は stdout と stderr の両方をリダイレクトする省略記法
echo "=== 両方を同じファイルに保存 ==="
mixed_output &> "$tmpdir/all_output.log"
echo "all_output.log の内容:"
cat "$tmpdir/all_output.log"

echo ""

# 4. stderr を stdout にマージしてパイプに渡す
# 2>&1 で fd 2 を fd 1 に複製（マージ）する
echo "=== stderr を stdout にマージして grep ==="
result=$(mixed_output 2>&1 | grep "ERROR")
echo "ERROR行のみ: $result"

# 後片付け
rm -rf "$tmpdir"
```

</details>

### 問題6：プロセス置換

以下の要件を満たすスクリプトを作成してください。

1. プロセス置換 `<(コマンド)` を使って、2つのコマンドの出力を `diff` で比較する
2. プロセス置換を使って、ソート済みの2つのリストの共通要素を `comm` で抽出する
3. プロセス置換 `>(コマンド)` を使って、出力を複数の処理に同時に渡す

**期待される出力例：**

```
$ bash process_sub.sh
=== 2つのディレクトリ内容の差分 ===
（diffの出力 - 環境によって異なります）

=== 2つのリストの共通要素 ===
banana
cherry

=== 出力を複数の処理に分岐 ===
大文字変換結果: HELLO WORLD FOO BAR
行数カウント: 4行
```

<details>
<summary>ヒント</summary>

- `<(コマンド)` はコマンドの出力をファイルのように扱えます
- `diff <(コマンド1) <(コマンド2)` で2つのコマンドの出力を直接比較できます
- `comm` は2つのソート済みファイルの共通・差分を表示します
- `>(コマンド)` は書き込み先をコマンドの標準入力にリダイレクトします

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

tmpdir="/tmp/procsub_test_$$"
mkdir -p "$tmpdir"

# 1. プロセス置換で2つのコマンドの出力を diff で比較
# <(コマンド) はコマンドの出力を一時的なファイルのように扱う
echo "=== 2つのディレクトリ内容の差分 ==="
diff <(ls /usr/bin | head -5) <(ls /usr/sbin | head -5) || true
# diff は差分があると終了コード1を返すので || true でエラーを抑制

echo ""

# 2. ソート済みリストの共通要素を comm で抽出
# comm -12 は「両方に存在する行」だけを表示する
echo "=== 2つのリストの共通要素 ==="
list_a=$(printf "apple\nbanana\ncherry\ndate\n" | sort)
list_b=$(printf "banana\ncherry\nelderberry\nfig\n" | sort)
comm -12 <(echo "$list_a") <(echo "$list_b")

echo ""

# 3. 出力を複数の処理に同時に分岐
# tee と >(コマンド) を組み合わせて、出力を2つの処理に渡す
echo "=== 出力を複数の処理に分岐 ==="
printf "hello\nworld\nfoo\nbar\n" \
    | tee >(tr '[:lower:]' '[:upper:]' > "$tmpdir/upper.txt") \
    | wc -l > "$tmpdir/count.txt"

# 少し待ってからサブプロセスの結果を表示
# （プロセス置換は非同期に実行されるため）
sleep 0.1
echo "大文字変換結果: $(tr '\n' ' ' < "$tmpdir/upper.txt")"
echo "行数カウント: $(cat "$tmpdir/count.txt" | tr -d ' ')行"

# 後片付け
rm -rf "$tmpdir"
```

</details>

### 問題7：ファイルディスクリプタの操作

以下の要件を満たすスクリプトを作成してください。

1. `exec` を使ってファイルディスクリプタ3番を開き、ファイルに書き込む
2. 複数の書き込みをファイルディスクリプタ3番経由で行う
3. ファイルディスクリプタ3番を閉じる
4. `exec` を使ってファイルディスクリプタ4番を読み取り用に開き、1行ずつ読み取る
5. ファイルディスクリプタ4番を閉じる

**期待される出力例：**

```
$ bash fd_advanced.sh
=== fd 3 に書き込み ===
fd 3 経由で3行書き込みました

=== 書き込んだファイルの内容 ===
1行目: ログエントリ A
2行目: ログエントリ B
3行目: ログエントリ C

=== fd 4 から読み取り ===
読み取り: 1行目: ログエントリ A
読み取り: 2行目: ログエントリ B
読み取り: 3行目: ログエントリ C

すべてのファイルディスクリプタを閉じました
```

<details>
<summary>ヒント</summary>

- `exec 3> ファイル` でファイルディスクリプタ3番を書き込み用に開きます
- `echo "..." >&3` でファイルディスクリプタ3番に書き込みます
- `exec 3>&-` でファイルディスクリプタ3番を閉じます
- `exec 4< ファイル` でファイルディスクリプタ4番を読み取り用に開きます
- `read -r line <&4` でファイルディスクリプタ4番から1行読み取ります

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash

tmpfile="/tmp/fd_advanced_$$.log"

# 1. exec でファイルディスクリプタ3番を書き込み用に開く
# exec N> ファイル でファイルディスクリプタNをファイルに関連付ける
exec 3> "$tmpfile"

# 2. fd 3 に複数回書き込む
# >&3 でファイルディスクリプタ3番にリダイレクト
echo "=== fd 3 に書き込み ==="
echo "1行目: ログエントリ A" >&3
echo "2行目: ログエントリ B" >&3
echo "3行目: ログエントリ C" >&3
echo "fd 3 経由で3行書き込みました"

# 3. fd 3 を閉じる（3>&- で閉じる）
exec 3>&-

echo ""

# 書き込んだ内容を確認
echo "=== 書き込んだファイルの内容 ==="
cat "$tmpfile"

echo ""

# 4. fd 4 を読み取り用に開く
# exec N< ファイル で読み取り用に関連付ける
exec 4< "$tmpfile"

echo "=== fd 4 から読み取り ==="
# read -r line <&4 で fd 4 から1行ずつ読み取る
while read -r line <&4; do
    echo "読み取り: $line"
done

# 5. fd 4 を閉じる（4<&- で閉じる）
exec 4<&-

echo ""
echo "すべてのファイルディスクリプタを閉じました"

# 後片付け
rm -f "$tmpfile"
```

</details>

### 問題8：ログ分離スクリプト

実践的なスクリプトとして、コマンドの標準出力と標準エラー出力を別々のファイルに記録しながら、画面にはすべて表示するスクリプトを作成してください。

要件：
1. 関数 `run_with_logging` を作成する。この関数は引数で渡されたコマンドを実行する
2. 標準出力は `stdout.log` に記録する
3. 標準エラー出力は `stderr.log` に記録する
4. 画面には標準出力と標準エラー出力の両方をリアルタイムで表示する
5. テスト用に正常出力とエラー出力の両方を生成するコマンドを実行する

**期待される出力例：**

```
$ bash log_separator.sh
=== コマンド実行（画面にはすべて表示）===
[stdout] 処理1: データ読み込み完了
[stderr] 警告: 古い形式のデータが見つかりました
[stdout] 処理2: データ変換完了
[stderr] 警告: 一部のフィールドがスキップされました
[stdout] 処理3: 出力完了

=== stdout.log の内容 ===
処理1: データ読み込み完了
処理2: データ変換完了
処理3: 出力完了

=== stderr.log の内容 ===
警告: 古い形式のデータが見つかりました
警告: 一部のフィールドがスキップされました
```

<details>
<summary>ヒント</summary>

- プロセス置換 `>(tee ファイル)` を使うとファイルに記録しながら画面にも出力できます
- `exec` でファイルディスクリプタをリダイレクトすると、以降のすべての出力に適用されます
- `sed` で行にプレフィックスを付けると、stdout/stderr の区別がしやすくなります
- 名前付きパイプ（`mkfifo`）を使う方法もあります

</details>

<summary>解答例</summary>

```bash
#!/bin/bash

tmpdir="/tmp/log_separator_$$"
mkdir -p "$tmpdir"

# テスト用の関数（stdout と stderr の両方を生成）
test_command() {
    echo "処理1: データ読み込み完了"
    echo "警告: 古い形式のデータが見つかりました" >&2
    echo "処理2: データ変換完了"
    echo "警告: 一部のフィールドがスキップされました" >&2
    echo "処理3: 出力完了"
}

# ログ分離実行関数
# stdout → stdout.log + 画面（[stdout] プレフィックス付き）
# stderr → stderr.log + 画面（[stderr] プレフィックス付き）
run_with_logging() {
    local stdout_log="$tmpdir/stdout.log"
    local stderr_log="$tmpdir/stderr.log"

    # プロセス置換を使って stdout/stderr を分離
    # 1> >(tee ...) : stdout を tee でファイルと画面の両方に出力
    # 2> >(tee ...) : stderr を tee でファイルと画面の両方に出力
    # sed でプレフィックスを付けて画面出力を区別する
    "$@" \
        1> >(tee "$stdout_log" | sed 's/^/[stdout] /') \
        2> >(tee "$stderr_log" | sed 's/^/[stderr] /' >&2)
}

echo "=== コマンド実行（画面にはすべて表示）==="
run_with_logging test_command

# プロセス置換は非同期なので少し待つ
sleep 0.2

echo ""
echo "=== stdout.log の内容 ==="
cat "$tmpdir/stdout.log"

echo ""
echo "=== stderr.log の内容 ==="
cat "$tmpdir/stderr.log"

# 後片付け
rm -rf "$tmpdir"
```

## まとめ

この演習では以下の内容を確認しました。

| カテゴリ | 学んだこと |
|---|---|
| リダイレクト | `>`（上書き）、`>>`（追記）、`2>`（stderr） |
| ヒアドキュメント | `<<EOF`（変数展開あり）、`<<'EOF'`（展開なし） |
| パイプ | `\|` によるコマンド連結、`cut`・`sort`・`head` の組み合わせ |
| tee | 画面とファイルの同時出力、パイプライン途中での分岐 |
| プロセス置換 | `<(cmd)` で入力、`>(cmd)` で出力をコマンドに渡す |
| ファイルディスクリプタ | `exec` による fd の開閉、`>&N`・`<&N` による操作 |
| 実践応用 | ログの分離記録、stdout/stderr の同時制御 |
