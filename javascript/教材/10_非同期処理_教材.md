# 第10章: 非同期処理

## この章のゴール

- 同期処理と非同期処理の違いを理解する
- コールバック、Promise、async/await の3つの非同期パターンを使い分けられる
- fetch API でHTTPリクエストを送れるようになる
- 並行処理（Promise.all 等）を活用できる
- エラーハンドリングを含む実践的な非同期処理を書ける

---

## 10.1 同期処理と非同期処理

```
同期処理（順番に実行）:
  処理A ━━━━━┓
              ┃ 待ち
  処理B ━━━━━┛━━━━━┓
                    ┃ 待ち
  処理C             ┗━━━━━┓
                          完了

非同期処理（待たずに次へ進む）:
  処理A ━━━━━━━━━━━━━━━━━━━━━→ 完了
  処理B ━━━━━━━━━━━━━→ 完了
  処理C ━━━━━→ 完了
  ※ 並行して進む（実際にはイベントループで管理）
```

```javascript
// 同期処理の例
console.log("1: 開始");
console.log("2: 処理中");
console.log("3: 完了");
// 出力: 1 → 2 → 3（順番通り）

// 非同期処理の例
console.log("1: 開始");
setTimeout(() => {
  console.log("2: 非同期処理完了");
}, 1000); // 1秒後に実行
console.log("3: 次の処理");
// 出力: 1 → 3 → 2（2は後から実行される！）
```

## 10.2 コールバック関数

非同期処理の最も基本的なパターンです。

```javascript
// setTimeout — 指定時間後に実行
setTimeout(() => {
  console.log("2秒後に実行");
}, 2000);

// setInterval — 一定間隔で繰り返し実行
let count = 0;
const intervalId = setInterval(() => {
  count++;
  console.log(`${count}回目`);
  if (count >= 5) {
    clearInterval(intervalId); // 停止
  }
}, 1000);

// コールバック地獄（Callback Hell）— これが問題
function getUser(id, callback) {
  setTimeout(() => callback({ id, name: "太郎" }), 1000);
}

function getOrders(userId, callback) {
  setTimeout(() => callback([{ id: 1, item: "本" }]), 1000);
}

function getOrderDetail(orderId, callback) {
  setTimeout(() => callback({ id: orderId, price: 1500 }), 1000);
}

// ネストが深くなり、読みにくい...
getUser(1, (user) => {
  console.log(user);
  getOrders(user.id, (orders) => {
    console.log(orders);
    getOrderDetail(orders[0].id, (detail) => {
      console.log(detail);
      // さらにネストが続く...
    });
  });
});
```

## 10.3 Promise

Promiseは非同期処理の結果を表すオブジェクトです。コールバック地獄を解消します。

```
Promiseの3つの状態:

  pending（保留中）
    ↓ 成功
  fulfilled（解決済み）→ then() で結果を受け取る
    ↓ 失敗
  rejected（拒否済み）→ catch() でエラーを受け取る

  ┌──────────┐    resolve(値)    ┌────────────┐
  │ pending  │ ─────────────→ │ fulfilled  │→ .then(値)
  │ (保留中)  │                  │ (解決済み)  │
  └──────────┘    reject(理由)   ┌────────────┐
       │ ──────────────────→   │ rejected   │→ .catch(理由)
                               │ (拒否済み)  │
                               └────────────┘
```

```javascript
// Promiseの作成
const promise = new Promise((resolve, reject) => {
  // 非同期処理
  setTimeout(() => {
    const success = true;
    if (success) {
      resolve("成功しました！"); // 成功時
    } else {
      reject("失敗しました..."); // 失敗時
    }
  }, 1000);
});

// Promiseの使用
promise
  .then((result) => {
    console.log(result); // "成功しました！"
  })
  .catch((error) => {
    console.error(error); // エラー時
  })
  .finally(() => {
    console.log("処理完了（成功でも失敗でも実行）");
  });
```

### Promiseチェーン

```javascript
// thenをつなげて順序制御（コールバック地獄の解消）
function getUser(id) {
  return new Promise((resolve) => {
    setTimeout(() => resolve({ id, name: "太郎" }), 500);
  });
}

function getOrders(userId) {
  return new Promise((resolve) => {
    setTimeout(() => resolve([{ id: 1, item: "本", userId }]), 500);
  });
}

function getOrderDetail(orderId) {
  return new Promise((resolve) => {
    setTimeout(() => resolve({ id: orderId, price: 1500 }), 500);
  });
}

// フラットなチェーンで読みやすい
getUser(1)
  .then((user) => {
    console.log("ユーザー:", user.name);
    return getOrders(user.id);
  })
  .then((orders) => {
    console.log("注文:", orders[0].item);
    return getOrderDetail(orders[0].id);
  })
  .then((detail) => {
    console.log("価格:", detail.price);
  })
  .catch((error) => {
    console.error("エラー:", error);
  });
```

## 10.4 async/await

Promiseをより直感的に書ける構文です。**現在の主流パターン**です。

```javascript
// async関数の宣言
async function fetchUserData() {
  // awaitでPromiseの解決を待つ（同期的に見える！）
  const user = await getUser(1);
  console.log("ユーザー:", user.name);

  const orders = await getOrders(user.id);
  console.log("注文:", orders[0].item);

  const detail = await getOrderDetail(orders[0].id);
  console.log("価格:", detail.price);

  return detail;
}

// async関数はPromiseを返す
fetchUserData()
  .then((result) => console.log("結果:", result))
  .catch((error) => console.error("エラー:", error));
```

### エラーハンドリング

```javascript
async function fetchData() {
  try {
    const user = await getUser(1);
    const orders = await getOrders(user.id);
    return orders;
  } catch (error) {
    console.error("データ取得に失敗:", error);
    return []; // フォールバック値
  } finally {
    console.log("処理完了");
  }
}
```

### アロー関数でのasync

```javascript
// アロー関数でも使える
const fetchUser = async (id) => {
  const response = await fetch(`/api/users/${id}`);
  return response.json();
};

// 即時実行関数（IIFE）
(async () => {
  const data = await fetchData();
  console.log(data);
})();
```

## 10.5 fetch API

ネットワークリクエストを送る標準APIです。

```javascript
// 基本的なGETリクエスト
async function getUsers() {
  const response = await fetch("https://jsonplaceholder.typicode.com/users");

  // レスポンスのステータスチェック
  if (!response.ok) {
    throw new Error(`HTTP Error: ${response.status}`);
  }

  const users = await response.json(); // JSONをパース
  console.log(users);
  return users;
}

// POSTリクエスト
async function createUser(userData) {
  const response = await fetch("https://jsonplaceholder.typicode.com/users", {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify(userData)
  });

  if (!response.ok) {
    throw new Error(`HTTP Error: ${response.status}`);
  }

  return response.json();
}

// 使用例
async function main() {
  try {
    // GET
    const users = await getUsers();
    console.log(`${users.length}人のユーザーを取得`);

    // POST
    const newUser = await createUser({
      name: "太郎",
      email: "taro@example.com"
    });
    console.log("作成:", newUser);
  } catch (error) {
    console.error("エラー:", error.message);
  }
}

main();
```

### レスポンスの型

```javascript
const response = await fetch(url);

// JSON
const jsonData = await response.json();

// テキスト
const textData = await response.text();

// Blob（画像など）
const blobData = await response.blob();

// レスポンスヘッダー
console.log(response.headers.get("Content-Type"));
console.log(response.status);     // 200, 404, 500 など
console.log(response.statusText); // "OK", "Not Found" など
console.log(response.ok);         // status が 200-299 なら true
```

## 10.6 並行処理

### Promise.all — すべて完了を待つ

```javascript
async function fetchAllData() {
  // 3つのリクエストを同時に開始
  const [users, posts, comments] = await Promise.all([
    fetch("https://jsonplaceholder.typicode.com/users").then(r => r.json()),
    fetch("https://jsonplaceholder.typicode.com/posts").then(r => r.json()),
    fetch("https://jsonplaceholder.typicode.com/comments").then(r => r.json())
  ]);

  console.log(`ユーザー: ${users.length}件`);
  console.log(`投稿: ${posts.length}件`);
  console.log(`コメント: ${comments.length}件`);
}

// ★1つでも失敗すると全体がrejectされる
```

```
Promise.all の動き:

  Promise A ━━━━━━━━━━━ 完了
  Promise B ━━━━━ 完了
  Promise C ━━━━━━━━━━━━━━━ 完了
                               ↓
  全て完了した時点で結果の配列を返す
```

### Promise.allSettled — 成功/失敗に関わらず全完了を待つ

```javascript
const results = await Promise.allSettled([
  fetch("https://api.example.com/data1").then(r => r.json()),
  fetch("https://invalid-url.example.com").then(r => r.json()),
  fetch("https://api.example.com/data3").then(r => r.json())
]);

results.forEach((result, index) => {
  if (result.status === "fulfilled") {
    console.log(`${index}: 成功 -`, result.value);
  } else {
    console.log(`${index}: 失敗 -`, result.reason);
  }
});
```

### Promise.race — 最初の1つが完了した時点で返す

```javascript
// タイムアウト処理の実装
function withTimeout(promise, ms) {
  const timeout = new Promise((_, reject) =>
    setTimeout(() => reject(new Error("タイムアウト")), ms)
  );
  return Promise.race([promise, timeout]);
}

try {
  const data = await withTimeout(
    fetch("https://api.example.com/data").then(r => r.json()),
    5000 // 5秒でタイムアウト
  );
  console.log(data);
} catch (error) {
  console.error(error.message); // "タイムアウト" or ネットワークエラー
}
```

### Promise.any — 最初に成功した1つを返す

```javascript
// 複数のAPIエンドポイントから最速で応答したものを使う
const fastest = await Promise.any([
  fetch("https://api1.example.com/data").then(r => r.json()),
  fetch("https://api2.example.com/data").then(r => r.json()),
  fetch("https://api3.example.com/data").then(r => r.json())
]);
console.log("最速:", fastest);
```

## 10.7 非同期の反復処理

```javascript
// 順次実行（1つずつ待つ）
async function sequential(urls) {
  const results = [];
  for (const url of urls) {
    const response = await fetch(url);
    results.push(await response.json());
  }
  return results;
}

// 並行実行（全て同時に）
async function parallel(urls) {
  const promises = urls.map(url =>
    fetch(url).then(r => r.json())
  );
  return Promise.all(promises);
}

// 並行数制限（一度にN個まで）
async function parallelLimit(urls, limit = 3) {
  const results = [];
  for (let i = 0; i < urls.length; i += limit) {
    const batch = urls.slice(i, i + limit);
    const batchResults = await Promise.all(
      batch.map(url => fetch(url).then(r => r.json()))
    );
    results.push(...batchResults);
  }
  return results;
}
```

## 10.8 現場で使われるパターン

### API通信のラッパー関数

```javascript
class ApiClient {
  constructor(baseUrl) {
    this.baseUrl = baseUrl;
  }

  async request(endpoint, options = {}) {
    const url = `${this.baseUrl}${endpoint}`;

    const config = {
      headers: {
        "Content-Type": "application/json",
        ...options.headers
      },
      ...options
    };

    if (config.body && typeof config.body === "object") {
      config.body = JSON.stringify(config.body);
    }

    const response = await fetch(url, config);

    if (!response.ok) {
      const errorBody = await response.text();
      throw new Error(`API Error ${response.status}: ${errorBody}`);
    }

    return response.json();
  }

  get(endpoint) {
    return this.request(endpoint);
  }

  post(endpoint, data) {
    return this.request(endpoint, { method: "POST", body: data });
  }

  put(endpoint, data) {
    return this.request(endpoint, { method: "PUT", body: data });
  }

  delete(endpoint) {
    return this.request(endpoint, { method: "DELETE" });
  }
}

// 使用例
const api = new ApiClient("https://jsonplaceholder.typicode.com");

async function main() {
  const users = await api.get("/users");
  console.log(users);

  const newPost = await api.post("/posts", {
    title: "新しい投稿",
    body: "本文です"
  });
  console.log(newPost);
}
```

### リトライ処理

```javascript
async function fetchWithRetry(url, maxRetries = 3, delay = 1000) {
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      const response = await fetch(url);
      if (!response.ok) throw new Error(`HTTP ${response.status}`);
      return await response.json();
    } catch (error) {
      console.warn(`試行 ${attempt}/${maxRetries} 失敗: ${error.message}`);
      if (attempt === maxRetries) throw error;
      await new Promise(resolve => setTimeout(resolve, delay * attempt));
    }
  }
}
```

## よくある間違い

### 1. awaitを忘れる

```javascript
// NG: awaitを忘れるとPromiseオブジェクトが入る
async function getUser() {
  const response = fetch("/api/users"); // awaitがない！
  console.log(response); // Promise { <pending> }
}

// OK
async function getUser() {
  const response = await fetch("/api/users");
  const data = await response.json(); // これもawaitが必要
  console.log(data);
}
```

### 2. async関数の外でawaitを使おうとする

```javascript
// NG: トップレベルawait（モジュールでないファイル）
// const data = await fetch("/api/data"); // SyntaxError

// OK: async関数で囲む
async function main() {
  const data = await fetch("/api/data");
}
main();

// OK: ESモジュールならトップレベルawaitが使える
// <script type="module"> の中や .mjs ファイル
```

### 3. forEachの中でawaitが効かない

```javascript
const urls = ["/api/1", "/api/2", "/api/3"];

// NG: forEachの中のawaitは待ってくれない
urls.forEach(async (url) => {
  const data = await fetch(url); // 各iterationが独立して実行される
});

// OK: for...of を使う（順次実行）
for (const url of urls) {
  const data = await fetch(url);
}

// OK: Promise.all を使う（並行実行）
const results = await Promise.all(
  urls.map(url => fetch(url).then(r => r.json()))
);
```

## ポイントまとめ

| パターン | 特徴 | 使い所 |
|----------|------|--------|
| コールバック | 最も基本的 | setTimeout等の単純な処理 |
| Promise | チェーンで順序制御 | ライブラリのAPI |
| async/await | 同期的に見える書き方 | 現在の主流、推奨 |

- 非同期処理は「待たずに次へ進む」処理です
- **async/await** が最も読みやすく、現在の主流です
- `fetch` API でHTTPリクエストを送り、`response.json()` でパースします
- `try...catch` でエラーハンドリングを行います
- `Promise.all` で複数の非同期処理を並行実行できます
- `forEach` 内の `await` は期待通りに動きません。`for...of` か `Promise.all` を使いましょう
