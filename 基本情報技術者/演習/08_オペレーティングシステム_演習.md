# 第8章：オペレーティングシステム 演習問題

## 基本問題

### 問題1：プロセスの状態遷移
プロセスの3つの基本状態を挙げ、各状態の意味を説明してください。

<details>
<summary>解答</summary>

1. **実行可能状態（Ready）：** CPUの割り当てを待っている状態
2. **実行状態（Running）：** CPUを使って処理を実行中の状態
3. **待ち状態（Waiting / Blocked）：** 入出力完了などのイベントを待っている状態
</details>

---

### 問題2：四択問題
プロセスの状態遷移として正しくないものはどれですか。

ア. 実行可能状態 → 実行状態
イ. 実行状態 → 待ち状態
ウ. 待ち状態 → 実行可能状態
エ. 待ち状態 → 実行状態

<details>
<summary>解答</summary>

答え：**エ. 待ち状態 → 実行状態**

待ち状態から直接実行状態に遷移することはありません。必ず「待ち状態 → 実行可能状態 → 実行状態」の順に遷移します。
</details>

---

### 問題3：メモリ管理の断片化
内部断片化と外部断片化の違いを説明し、それぞれどのメモリ管理方式で発生するかを答えてください。

<details>
<summary>解答</summary>

- **内部断片化：** 区画内の未使用領域。プロセスサイズが区画サイズより小さいとき発生。**固定区画方式**で発生する。
- **外部断片化：** メモリ上に散在する小さな空き領域。個々は空いているが連続した空きが確保できない。**可変区画方式**で発生する。
</details>

---

### 問題4：ディスクアクセス時間
ディスクアクセス時間を構成する3つの要素を挙げてください。

<details>
<summary>解答</summary>

```
ディスクアクセス時間 = シーク時間 + 回転待ち時間 + データ転送時間
```

1. **シーク時間（Seek Time）：** 磁気ヘッドを目的のトラックに移動する時間
2. **回転待ち時間（Rotational Latency）：** 目的のセクタがヘッドの下に来るまでの回転待ち時間
3. **データ転送時間（Transfer Time）：** データの読み書きにかかる時間
</details>

---

### 問題5：四択問題
ページング方式の仮想記憶において、アクセスしたいページが主記憶上にないときに発生する割り込みを何というか。

ア. ページアウト　　イ. ページイン　　ウ. ページフォールト　　エ. ページスワップ

<details>
<summary>解答</summary>

答え：**ウ. ページフォールト**

ページフォールトが発生すると、OSが補助記憶から該当ページを主記憶に読み込みます。
</details>

## 応用問題

### 問題6：ターンアラウンドタイムの計算（FCFS）
3つのジョブが時刻0に同時到着し、FCFS（到着順）で A → B → C の順に実行されます。各ジョブの実行時間が A=4, B=2, C=6 のとき、平均ターンアラウンドタイムを求めてください。

<details>
<summary>解答</summary>

```
|----A----|--B--|------C------|
0        4    6            12

ジョブA: TAT = 4 - 0 = 4
ジョブB: TAT = 6 - 0 = 6
ジョブC: TAT = 12 - 0 = 12

平均TAT = (4 + 6 + 12) / 3 = 22 / 3 ≒ 7.33
```
答え：平均ターンアラウンドタイム **約 7.33**
</details>

---

### 問題7：ターンアラウンドタイムの計算（SJF）
問題6と同じ条件で、SJF（最短ジョブ優先）で実行した場合の平均ターンアラウンドタイムを求めてください。

<details>
<summary>解答</summary>

```
SJF順: B(2) → A(4) → C(6)

|--B--|----A----|------C------|
0    2        6            12

ジョブB: TAT = 2 - 0 = 2
ジョブA: TAT = 6 - 0 = 6
ジョブC: TAT = 12 - 0 = 12

平均TAT = (2 + 6 + 12) / 3 = 20 / 3 ≒ 6.67
```
答え：平均ターンアラウンドタイム **約 6.67**

SJFの方がFCFS（7.33）より平均TATが短くなっています。SJFは平均TATを最小化するスケジューリングです。
</details>

### 問題8：ページ置換（LRU）
ページフレーム数 = 3、参照列 = 3, 2, 1, 4, 3, 2 のとき、LRU（Least Recently Used）アルゴリズムでのページフォールト回数を求めてください。

<details>
<summary>ヒント</summary>
LRUは「最も長い間参照されていないページ」を置換します。
</details>

<details>
<summary>解答</summary>

```
参照  フレーム状態      ページフォールト
 3    [3, -, -]        ○（空きフレームに読込）
 2    [3, 2, -]        ○（空きフレームに読込）
 1    [3, 2, 1]        ○（空きフレームに読込）
 4    [4, 2, 1]        ○（3が最も古い参照→置換）
 3    [4, 3, 1]        ○（2が最も古い参照→置換）
 2    [4, 3, 2]        ○（1が最も古い参照→置換）
```

答え：ページフォールト回数 = **6回**（全てフォールト）
</details>

### 問題9：ディスクアクセス時間の計算
ディスクの回転速度が 7200rpm、シーク時間が 8ms、1トラックのデータ量が 60KB のとき、15KB のデータを読み取るアクセス時間を計算してください。

<details>
<summary>解答</summary>

```
1回転の時間 = 60秒 / 7200 = 1/120秒 ≒ 8.33ms
平均回転待ち時間 = 8.33 / 2 ≒ 4.17ms
データ転送時間 = (15 / 60) × 8.33 = 0.25 × 8.33 ≒ 2.08ms

ディスクアクセス時間 = 8 + 4.17 + 2.08 = 14.25ms
```
答え：約 **14.25ms**
</details>

### 問題10：四択問題
ページフォールトが頻発し、OSがページの入れ替え処理ばかり行って実際の処理がほとんど進まなくなる現象を何というか。

ア. フラグメンテーション　　イ. スラッシング　　ウ. スワッピング　　エ. スプーリング

<details>
<summary>解答</summary>

答え：**イ. スラッシング**

原因は、主記憶に対して実行するプロセスが多すぎること（多重度が高すぎること）です。対策として主記憶の増設や多重度の制限があります。
</details>

## チャレンジ問題

### 問題11：ラウンドロビンスケジューリング
3つのジョブが時刻0に同時到着し、タイムクォンタム = 3 でラウンドロビンスケジューリングを行います。実行時間が A=7, B=3, C=5 のとき、各ジョブの完了時刻と平均ターンアラウンドタイムを求めてください。

<details>
<summary>解答</summary>

```
残り時間: A=7, B=3, C=5
タイムクォンタム = 3

|---A---|---B---|---C---|--A--|---C--|--A--|
0      3      6      9    12    14   15

A: [0-3]残4 → [9-12]残1 → [14-15]完了  TAT=15
B: [3-6]完了                           TAT=6
C: [6-9]残2 → [12-14]完了              TAT=14

平均TAT = (15 + 6 + 14) / 3 = 35 / 3 ≒ 11.67
```
答え：
- A完了: 時刻15、B完了: 時刻6、C完了: 時刻14
- 平均TAT: **約 11.67**
</details>

### 問題12：セマフォ
セマフォの初期値が 2 のとき、以下の操作を順に実行した場合、各操作後のセマフォの値と、プロセスが待ち状態になるかどうかを答えてください。

(1) プロセスAが P操作
(2) プロセスBが P操作
(3) プロセスCが P操作
(4) プロセスAが V操作

<details>
<summary>解答</summary>

| 操作 | セマフォ値 | 状態 |
|------|----------|------|
| 初期状態 | 2 | - |
| (1) プロセスA: P操作 | 1 | 進入OK |
| (2) プロセスB: P操作 | 0 | 進入OK |
| (3) プロセスC: P操作 | -1 | **待ち状態**（資源なし） |
| (4) プロセスA: V操作 | 0 | プロセスCの待ちが解除 |

セマフォ値が0未満になるとプロセスは待ち状態になります。V操作で値が増加すると、待ちプロセスが1つ起こされます。
</details>

### 問題13：デッドロックの4条件
デッドロック発生のコフマン条件4つを挙げ、そのうち1つを崩すための具体的な対策を述べてください。

<details>
<summary>解答</summary>

**デッドロック発生の4条件（コフマン条件）：**

1. **相互排除：** 資源を同時に使えるのは1プロセスだけ
2. **保持と待ち：** 資源を保持したまま別の資源を要求する
3. **横取り不可：** 他のプロセスから強制的に資源を奪えない
4. **循環待ち：** 資源待ちが循環している

**対策例（循環待ちを崩す）：**
全ての資源に番号をつけ、プロセスは常に番号の小さい資源からロックするルールを設ける。これによりロック順序が統一され、循環待ちが発生しなくなる。
</details>
