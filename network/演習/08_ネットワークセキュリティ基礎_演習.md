# 第8章：ネットワークセキュリティ基礎 演習問題

## 基本問題

### 問題8-1：ファイアウォールの種類
以下の3種類のファイアウォールについて、検査対象と特徴を答えてください。

1. パケットフィルタリング型
2. ステートフルパケットインスペクション（SPI）型
3. アプリケーションゲートウェイ型

<details>
<summary>解答</summary>

| 種類 | 検査対象 | 特徴 |
|------|---------|------|
| パケットフィルタリング | L3/L4ヘッダ（IPアドレス、ポート番号） | 高速だが通信内容は検査しない。戻りの通信も個別にルール設定が必要 |
| ステートフルインスペクション | L3/L4ヘッダ + セッション状態 | 通信の状態を追跡し、正当な戻り通信を自動許可。現在の主流 |
| アプリケーションゲートウェイ | L3〜L7（通信内容を含む） | アプリケーション層まで検査可能で防御レベルが最も高いが、処理速度が遅い |

</details>

### 問題8-2：DMZ
以下の問いに答えてください。

(1) DMZ（非武装地帯）とは何ですか。設置する目的を説明してください。
(2) DMZに配置すべきサーバの例を3つ挙げてください。
(3) DMZの通信ポリシーとして「DMZ → 内部LAN」の通信を原則拒否する理由を述べてください。

<details>
<summary>解答</summary>

(1) **DMZ** は、外部ネットワーク（インターネット）と内部ネットワーク（社内LAN）の間に設置される中間的なネットワークセグメントです。外部に公開するサーバを配置し、万が一公開サーバが侵害されても内部LANへの被害拡大を防止する目的があります。

(2) DMZに配置すべきサーバの例：
- **Webサーバ**：外部からHTTP/HTTPSアクセスを受ける
- **メールサーバ（MTA）**：外部からSMTP通信を受ける
- **外部向けDNSサーバ**：外部からのDNS問い合わせに応答
- （他にリバースプロキシ、VPNゲートウェイ等）

(3) DMZのサーバが攻撃者に侵害された場合、DMZから内部LANへの通信を許可していると、攻撃者が内部LANに侵入する踏み台として利用される恐れがあります。原則拒否にすることで、DMZサーバの侵害が内部LANに波及することを防ぎます。

</details>

### 問題8-3：VPNの種類
以下のVPN方式の違いを表で比較してください。

1. IPsec VPN（サイト間）
2. SSL-VPN（リモートアクセス）

<details>
<summary>解答</summary>

| 項目 | IPsec VPN | SSL-VPN |
|------|----------|---------|
| 動作層 | L3（ネットワーク層） | L4〜L7（トランスポート〜アプリケーション層） |
| 主な用途 | 拠点間VPN（ルータ対ルータ） | リモートアクセス（端末対サーバ） |
| クライアント | 専用クライアントまたはルータ | Webブラウザで利用可能（方式による） |
| 暗号化範囲 | IPパケット全体（トンネルモード） | SSL/TLSセッション内 |
| ポート | ESP（プロトコル番号50）、IKE（UDP 500/4500） | TCP 443（HTTPS） |
| 導入の容易さ | ルータ/VPN装置の設定が必要 | 比較的容易（ブラウザベース） |
| FW通過 | NATやFWの設定変更が必要な場合がある | HTTPS（443）を使うためFWを通過しやすい |

</details>

### 問題8-4：IDS と IPS
IDSとIPSの違いを以下の3つの観点から説明してください。

1. 配置方法
2. 検知後の動作
3. メリット・デメリット

<details>
<summary>解答</summary>

| 観点 | IDS（侵入検知システム） | IPS（侵入防止システム） |
|------|---------------------|---------------------|
| 配置方法 | ミラーポート（通信経路の外）に接続し、トラフィックのコピーを監視 | インラインに設置（通信経路上に配置） |
| 検知後の動作 | 管理者にアラートを通知するのみ。攻撃通信は通過してしまう | 不正な通信を自動的に遮断・破棄。攻撃を防御できる |
| メリット | ネットワーク性能に影響しない。誤検知時もサービスが止まらない | リアルタイムで攻撃を防御できる |
| デメリット | 攻撃を防御できない（検知のみ） | 誤検知（フォールスポジティブ）時に正常通信が遮断される。障害時にネットワークが切断される可能性 |

</details>

## 応用問題

### 問題8-5：ファイアウォールルール設計
以下の要件を満たすファイアウォールルール（ACL）を作成してください。

- 外部からDMZのWebサーバ（10.0.1.10）へのHTTP/HTTPSアクセスを許可
- 外部からDMZのメールサーバ（10.0.1.20）へのSMTPアクセスを許可
- 内部LAN（192.168.1.0/24）から外部へのHTTP/HTTPSアクセスを許可
- 管理者PC（192.168.1.100）からDMZのサーバへのSSHアクセスを許可
- その他の通信は全て拒否

<details>
<summary>解答</summary>

| 番号 | アクション | 送信元 | 宛先 | プロトコル | 宛先ポート | 説明 |
|------|----------|--------|------|----------|-----------|------|
| 1 | ALLOW | Any | 10.0.1.10 | TCP | 80 | 外部→Webサーバ（HTTP） |
| 2 | ALLOW | Any | 10.0.1.10 | TCP | 443 | 外部→Webサーバ（HTTPS） |
| 3 | ALLOW | Any | 10.0.1.20 | TCP | 25 | 外部→メールサーバ（SMTP） |
| 4 | ALLOW | 192.168.1.0/24 | Any | TCP | 80 | 内部→外部（HTTP） |
| 5 | ALLOW | 192.168.1.0/24 | Any | TCP | 443 | 内部→外部（HTTPS） |
| 6 | ALLOW | 192.168.1.100 | 10.0.1.0/24 | TCP | 22 | 管理者→DMZ（SSH） |
| 7 | DENY | Any | Any | Any | Any | 暗黙のDeny（その他全拒否） |

※ ステートフルファイアウォールの場合、許可したルールの戻り通信は自動で許可されます。パケットフィルタリング型の場合は戻り通信のルールも個別に追加する必要があります。

</details>

### 問題8-6：IPsecの構成要素
IPsecの以下の構成要素について説明してください。

1. AH と ESP の違い
2. トランスポートモードとトンネルモードの違い
3. IKE の役割

<details>
<summary>解答</summary>

**1. AH と ESP の違い**

| 項目 | AH（Authentication Header） | ESP（Encapsulating Security Payload） |
|------|--------------------------|--------------------------------------|
| 認証 | あり | あり |
| 暗号化 | なし | あり |
| 保護範囲 | IPヘッダを含む（改ざん検知） | ペイロードのみ暗号化・認証 |
| 用途 | 改ざん検知のみが必要な場合 | 一般的なVPN通信（暗号化+認証） |

実際にはESPが圧倒的に使用されます（暗号化が必要な場面がほとんど）。

**2. トランスポートモードとトンネルモードの違い**

| 項目 | トランスポートモード | トンネルモード |
|------|-------------------|-------------|
| 暗号化範囲 | ペイロードのみ（元のIPヘッダは平文） | 元のIPヘッダ+ペイロード全体 |
| 新IPヘッダ | 付加しない | 新しいIPヘッダを付加 |
| 用途 | ホスト間通信 | 拠点間VPN（一般的） |

**3. IKE（Internet Key Exchange）の役割**
- IPsec通信で使用する暗号アルゴリズム、認証方式、暗号鍵を安全に交換・合意するプロトコル
- SA（Security Association）を確立する
- フェーズ1：IKE SA の確立（相互認証と暗号化通信路の確立）
- フェーズ2：IPsec SA の確立（実際のデータ通信用のパラメータ合意）

</details>

### 問題8-7：検知方式の比較
IDSの検知方式であるシグネチャ型とアノマリ型について、以下の問いに答えてください。

(1) それぞれの検知方式の仕組みを説明してください。
(2) フォールスポジティブとフォールスネガティブの意味を説明してください。
(3) 未知の攻撃（ゼロデイ攻撃）を検知できるのはどちらですか。理由も述べてください。

<details>
<summary>解答</summary>

**(1) 検知方式の仕組み**

**シグネチャ型（不正検知）**：既知の攻撃パターン（シグネチャ）をデータベースとして保持し、通信をそのパターンと照合して一致すれば攻撃と判定する。ウイルス対策ソフトのパターンマッチングと同様の考え方。

**アノマリ型（異常検知）**：正常な通信のパターン（ベースライン）を学習し、そこから大きく逸脱する通信を異常（攻撃の可能性あり）と判定する。統計的手法や機械学習が使用される。

**(2)**
- **フォールスポジティブ（偽陽性）**：正常な通信を攻撃と誤って判定すること。IPSの場合、正常な通信が遮断されてサービスに影響する
- **フォールスネガティブ（偽陰性）**：実際の攻撃を正常な通信と誤って判定すること（見逃し）。攻撃が検知されずに通過してしまう

**(3)** **アノマリ型** が未知の攻撃を検知できる可能性があります。理由：アノマリ型は通信パターンの逸脱を検知するため、シグネチャが存在しない未知の攻撃でも、通常と異なる挙動を示せば検知可能です。一方、シグネチャ型は既知の攻撃パターンとの一致でのみ判定するため、データベースに登録されていない未知の攻撃は検知できません。

</details>

## チャレンジ問題

### 問題8-8：ネットワークセキュリティ設計
ある中小企業のネットワークセキュリティを設計してください。

要件：
- 社員50名、PC50台
- 外部公開Webサーバ1台
- 社内ファイルサーバ1台
- テレワーク社員10名がリモートアクセス
- 予算は限られている

(1) ネットワーク構成図を描いてください（DMZ含む）。
(2) 使用するセキュリティ機器・技術を列挙し、理由を述べてください。
(3) リモートアクセスVPNの方式を選択し、理由を述べてください。

<details>
<summary>解答例</summary>

**(1) ネットワーク構成図**
```
  インターネット
       │
  ┌────┴────┐
  │  UTM     │  ← FW + IPS + アンチウイルス + VPN を統合
  │  ├───────┤
  │  │ DMZ   │── Webサーバ（198.51.100.10）
  │  ├───────┤
  │  │ 内部  │── L2スイッチ ── 社内PC × 50
  │  │       │              └── ファイルサーバ
  └──────────┘
       ↑
  テレワーク社員（SSL-VPN接続）
```

**(2) セキュリティ機器・技術**

| 機器・技術 | 理由 |
|-----------|------|
| **UTM** | 予算が限られているため、FW・IPS・アンチウイルス・VPN等を1台に統合。個別導入より低コスト |
| **DMZ** | Webサーバが侵害されても内部LANへの被害拡大を防止 |
| **SSL-VPN** | テレワーク用のリモートアクセス |
| **ウイルス対策ソフト** | 各PCにエンドポイントセキュリティを導入 |
| **URLフィルタリング** | UTMの機能で不適切なWebサイトへのアクセスを制限 |

**(3) リモートアクセスVPNの方式**
**SSL-VPN** が適切です。理由：
- Webブラウザベースでアクセスできるため、テレワーク社員のPCに専用ソフトの導入が不要（または簡易クライアントのみ）
- HTTPS（ポート443）を使用するため、自宅やカフェのWi-Fiからでもファイアウォールに遮断されにくい
- UTMにSSL-VPN機能が統合されていることが多く、追加コストを抑えられる
- IPsec VPNに比べて導入・運用が容易

</details>

### 問題8-9：セキュリティ機器の選択
以下の攻撃に対して、最も効果的なセキュリティ機器/技術を選択し、理由を述べてください。

(a) SQLインジェクション攻撃
(b) DoS/DDoS攻撃
(c) ポートスキャン
(d) 内部からの機密情報持ち出し
(e) マルウェアのダウンロード

<details>
<summary>解答</summary>

(a) **SQLインジェクション → WAF（Web Application Firewall）**
理由：WAFはHTTPリクエストの内容をアプリケーション層で検査し、SQLインジェクションのパターンを検知・遮断できます。通常のファイアウォールではHTTPの内容まで検査できません。

(b) **DoS/DDoS攻撃 → IPS + CDN/DDoS緩和サービス**
理由：IPSはトラフィックの異常を検知して遮断できます。大規模なDDoS攻撃に対しては、クラウドベースのDDoS緩和サービス（Cloudflare、AWS Shield等）が有効です。

(c) **ポートスキャン → ファイアウォール + IDS**
理由：ファイアウォールで不要なポートを閉じ、IDSでポートスキャンの試行を検知してアラートを発報します。

(d) **内部からの機密情報持ち出し → DLP（Data Loss Prevention）+ フォワードプロキシ**
理由：DLPは機密情報を含むデータの外部送信を検知・遮断します。フォワードプロキシでWeb経由のアップロードを監視・制限します。

(e) **マルウェアのダウンロード → UTM（アンチウイルス + URLフィルタリング）+ エンドポイント保護**
理由：UTMのゲートウェイアンチウイルスで通信上のマルウェアを検知し、URLフィルタリングで既知の不正サイトをブロックします。各PCのエンドポイント保護と併用して多層防御とします。

</details>
