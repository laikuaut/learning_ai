# 実践課題03：公開鍵認証の設定 ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第2章（SSHクライアントの基本操作）、第3章（公開鍵認証の設定）
> **課題の種類**: ミニプロジェクト
> **学習目標**: 鍵ペアの生成からサーバーへの公開鍵登録、鍵認証でのログインまでの一連の手順を実践し、パスワード認証との違いを体感する

---

## 完成イメージ

```
# パスワード認証（設定前）
$ ssh user@server
user@server's password: ********    ← 毎回パスワードを入力

# 公開鍵認証（設定後）
$ ssh user@server
user@server:~$                      ← パスワードなしでログイン成功！
```

---

## 課題の要件

1. サーバー接続用の Ed25519 鍵ペアを生成する
2. `ssh-copy-id` を使って公開鍵をサーバーに登録する
3. 鍵認証でログインできることを確認する
4. サーバー側の `~/.ssh/authorized_keys` の内容を確認する
5. パーミッションの検証スクリプトを作成する

---

## ステップガイド

<details>
<summary>ステップ1：鍵ペアを生成する</summary>

```bash
# Ed25519鍵を生成（パスフレーズ付き推奨）
$ ssh-keygen -t ed25519 -C "myuser@server-practice" -f ~/.ssh/id_ed25519_practice

# パスフレーズを聞かれるので入力する
Enter passphrase (empty for no passphrase): （安全な文字列を入力）
Enter same passphrase again: （同じ文字列を再入力）
```

</details>

<details>
<summary>ステップ2：公開鍵をサーバーに登録する</summary>

`ssh-copy-id` コマンドが最も簡単な方法です。

```bash
# ssh-copy-id で公開鍵を登録
$ ssh-copy-id -i ~/.ssh/id_ed25519_practice.pub user@server

# パスワードを聞かれるので入力する（最後のパスワード入力！）
user@server's password: ********

# 成功メッセージ
Number of key(s) added: 1
```

`ssh-copy-id` が使えない環境では手動で登録します。

```bash
# 手動登録の場合
$ cat ~/.ssh/id_ed25519_practice.pub | ssh user@server "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

</details>

<details>
<summary>ステップ3：鍵認証でログインする</summary>

```bash
# 鍵を指定して接続
$ ssh -i ~/.ssh/id_ed25519_practice user@server

# パスフレーズを設定した場合はパスフレーズが聞かれる
# （サーバーのパスワードではなく、鍵のパスフレーズ）
Enter passphrase for key '/home/user/.ssh/id_ed25519_practice':
```

</details>

<details>
<summary>ステップ4：サーバー側の設定を確認する</summary>

```bash
# サーバーに接続した後、authorized_keys を確認
$ cat ~/.ssh/authorized_keys

# パーミッションを確認
$ ls -la ~/.ssh/
$ ls -la ~/.ssh/authorized_keys
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け）─ 手順通りに設定</summary>

```bash
#!/bin/bash
# 公開鍵認証の設定スクリプト
# 学べる内容：鍵生成、ssh-copy-id、パーミッション確認
# 実行方法：bash setup_pubkey_auth.sh <ユーザー名> <サーバー>

USER="$1"
SERVER="$2"
KEY_NAME="id_ed25519_practice"
KEY_PATH="$HOME/.ssh/$KEY_NAME"

if [ -z "$USER" ] || [ -z "$SERVER" ]; then
    echo "使い方: $0 <ユーザー名> <サーバー>"
    echo "例: $0 myuser 192.168.1.100"
    exit 1
fi

echo "===== 公開鍵認証セットアップ ====="
echo "対象: ${USER}@${SERVER}"
echo ""

# --- ステップ1: 鍵ペアの生成 ---
echo "--- ステップ1: 鍵ペアの生成 ---"
if [ -f "$KEY_PATH" ]; then
    echo "鍵は既に存在します: $KEY_PATH"
else
    ssh-keygen -t ed25519 -C "${USER}@${SERVER}" -f "$KEY_PATH"
    echo "鍵を生成しました。"
fi

# --- ステップ2: パーミッション確認 ---
echo ""
echo "--- ステップ2: ローカルのパーミッション確認 ---"
chmod 600 "$KEY_PATH"
chmod 644 "${KEY_PATH}.pub"
ls -la "$KEY_PATH" "${KEY_PATH}.pub"

# --- ステップ3: 公開鍵の登録 ---
echo ""
echo "--- ステップ3: 公開鍵をサーバーに登録 ---"
echo "パスワードの入力を求められます。"
ssh-copy-id -i "${KEY_PATH}.pub" "${USER}@${SERVER}"

# --- ステップ4: 接続テスト ---
echo ""
echo "--- ステップ4: 鍵認証で接続テスト ---"
ssh -i "$KEY_PATH" "${USER}@${SERVER}" "echo '鍵認証での接続に成功しました！'; hostname"

echo ""
echo "===== セットアップ完了 ====="
```

</details>

<details>
<summary>解答例（改良版）─ パーミッション検証とトラブルシューティング付き</summary>

```bash
#!/bin/bash
# 公開鍵認証 セットアップ＆検証ツール（改良版）
# 学べる内容：パーミッション検証、ssh-copy-idの代替手段、トラブルシューティング
# 実行方法：bash setup_pubkey_auth_v2.sh <ユーザー名> <サーバー>

set -euo pipefail

USER="${1:-}"
SERVER="${2:-}"
KEY_NAME="id_ed25519_practice"
KEY_PATH="$HOME/.ssh/$KEY_NAME"

# === ユーティリティ関数 ===

check_permission() {
    local file="$1"
    local expected="$2"
    local actual
    actual=$(stat -c '%a' "$file" 2>/dev/null || stat -f '%Lp' "$file" 2>/dev/null)

    if [ "$actual" = "$expected" ]; then
        echo "  [OK] $file: $actual"
    else
        echo "  [NG] $file: $actual (期待値: $expected)"
        chmod "$expected" "$file"
        echo "       → $expected に修正しました。"
    fi
}

verify_remote_permissions() {
    local remote="$1"
    echo "--- リモート側のパーミッション検証 ---"
    ssh -i "$KEY_PATH" "$remote" << 'EOF'
echo "  ~/.ssh ディレクトリ:"
ls -ld ~/.ssh 2>/dev/null || echo "    [エラー] ~/.ssh が存在しません"
echo ""
echo "  authorized_keys:"
ls -la ~/.ssh/authorized_keys 2>/dev/null || echo "    [エラー] authorized_keys が存在しません"
echo ""
echo "  ホームディレクトリ:"
ls -ld ~ | awk '{print "    パーミッション: " $1}'

# パーミッションチェック
home_perm=$(stat -c '%a' ~ 2>/dev/null || stat -f '%Lp' ~ 2>/dev/null)
ssh_perm=$(stat -c '%a' ~/.ssh 2>/dev/null || stat -f '%Lp' ~/.ssh 2>/dev/null)
auth_perm=$(stat -c '%a' ~/.ssh/authorized_keys 2>/dev/null || stat -f '%Lp' ~/.ssh/authorized_keys 2>/dev/null)

echo ""
echo "  パーミッション検証結果:"

# ホームディレクトリは755以下であること
if [ "$home_perm" -le 755 ] 2>/dev/null; then
    echo "    [OK] ホームディレクトリ: $home_perm"
else
    echo "    [NG] ホームディレクトリ: $home_perm (755以下にしてください)"
fi

if [ "$ssh_perm" = "700" ]; then
    echo "    [OK] ~/.ssh: $ssh_perm"
else
    echo "    [NG] ~/.ssh: $ssh_perm (700が必要)"
fi

if [ "$auth_perm" = "600" ]; then
    echo "    [OK] authorized_keys: $auth_perm"
else
    echo "    [NG] authorized_keys: $auth_perm (600が必要)"
fi
EOF
}

# === 引数チェック ===

if [ -z "$USER" ] || [ -z "$SERVER" ]; then
    echo "使い方: $0 <ユーザー名> <サーバー>"
    echo "例: $0 myuser 192.168.1.100"
    exit 1
fi

REMOTE="${USER}@${SERVER}"

echo "===== 公開鍵認証 セットアップ＆検証ツール ====="
echo "対象: $REMOTE"
echo ""

# --- ステップ1: ローカル環境の準備 ---
echo "--- ステップ1: ローカル環境の準備 ---"
mkdir -p "$HOME/.ssh"
check_permission "$HOME/.ssh" "700"

# --- ステップ2: 鍵ペアの生成 ---
echo ""
echo "--- ステップ2: 鍵ペアの生成 ---"
if [ -f "$KEY_PATH" ]; then
    echo "  鍵は既に存在します。スキップします。"
    echo "  フィンガープリント: $(ssh-keygen -l -f "${KEY_PATH}.pub")"
else
    ssh-keygen -t ed25519 -C "${USER}@${SERVER}-$(date +%Y%m%d)" -f "$KEY_PATH"
fi

check_permission "$KEY_PATH" "600"
check_permission "${KEY_PATH}.pub" "644"

# --- ステップ3: 公開鍵の登録 ---
echo ""
echo "--- ステップ3: 公開鍵の登録 ---"

# ssh-copy-id が利用可能か確認
if command -v ssh-copy-id > /dev/null 2>&1; then
    echo "ssh-copy-id を使用して登録します。"
    ssh-copy-id -i "${KEY_PATH}.pub" "$REMOTE"
else
    echo "ssh-copy-id が見つかりません。手動で登録します。"
    cat "${KEY_PATH}.pub" | ssh "$REMOTE" \
        "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
fi

# --- ステップ4: 接続テスト ---
echo ""
echo "--- ステップ4: 接続テスト ---"
if ssh -i "$KEY_PATH" -o BatchMode=yes "$REMOTE" "echo '鍵認証テスト成功'" 2>/dev/null; then
    echo "  [OK] 鍵認証で接続できました。"
else
    echo "  [NG] 鍵認証での接続に失敗しました。"
    echo ""
    echo "  トラブルシューティング:"
    echo "  1. ssh -vvv -i $KEY_PATH $REMOTE で詳細ログを確認"
    echo "  2. サーバー側の /var/log/auth.log を確認"
    echo "  3. パーミッションを確認（次のステップへ）"
fi

# --- ステップ5: リモート側の検証 ---
echo ""
verify_remote_permissions "$REMOTE"

echo ""
echo "===== 完了 ====="
```

**初心者向けとの違い:**
- パーミッションを自動検証・修正する関数を用意 → 設定ミスによるトラブルを未然に防止
- `ssh-copy-id` が使えない場合の代替手段を用意 → どの環境でも動作
- `-o BatchMode=yes` で非対話的にテスト → スクリプトの自動化に対応
- リモート側のパーミッションまで検証 → 「鍵認証が通らない」問題の原因特定が容易
