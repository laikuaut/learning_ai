# 第3章 演習：サーバーコンポーネントとクライアントコンポーネント

---

## 演習1（基本）：サーバーコンポーネントとクライアントコンポーネントを見分けよう

以下のコードを見て、サーバーコンポーネントかクライアントコンポーネントか、また正しく動作するかどうかを判定してください。

**コードA:**
```tsx
export default function Welcome() {
  return <h1>ようこそ</h1>
}
```

**コードB:**
```tsx
import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```

**コードC:**
```tsx
'use client'
import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```

<details>
<summary>ヒント</summary>

App Router では、`'use client'` がないファイルはすべてサーバーコンポーネントとして扱われます。サーバーコンポーネントでは `useState` やイベントハンドラは使用できません。

</details>

<details>
<summary>解答例</summary>

- **コードA**: サーバーコンポーネント。正しく動作します。静的な JSX のみを返しており、サーバーコンポーネントとして問題ありません。
- **コードB**: サーバーコンポーネント（`'use client'` がないため）。**エラーになります**。`useState` はクライアントコンポーネントでのみ使用可能です。
- **コードC**: クライアントコンポーネント。正しく動作します。`'use client'` ディレクティブにより、`useState` やイベントハンドラが使用可能です。

</details>

## 演習2（基本）：クライアントコンポーネントでカウンターを作ろう

クリックするたびに数字が増えるカウンターコンポーネントを作成してください。

**要件：**
- `app/components/Counter.tsx` にクライアントコンポーネントとして作成する
- `useState` を使って状態を管理する
- 「+1」「-1」「リセット」ボタンを配置する

<details>
<summary>ヒント</summary>

ファイルの先頭に `'use client'` を記述することで、そのファイルはクライアントコンポーネントとして扱われます。`useState` や `onClick` が使えるようになります。

</details>

<details>
<summary>解答例</summary>

```tsx
// app/components/Counter.tsx
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div style={{
      display: 'flex',
      alignItems: 'center',
      gap: '1rem',
      padding: '1rem',
      border: '1px solid #e2e8f0',
      borderRadius: '8px',
    }}>
      <button
        onClick={() => setCount(count - 1)}
        style={{ padding: '0.5rem 1rem', cursor: 'pointer' }}
      >
        -1
      </button>
      <span style={{ fontSize: '1.5rem', fontWeight: 'bold', minWidth: '3rem', textAlign: 'center' }}>
        {count}
      </span>
      <button
        onClick={() => setCount(count + 1)}
        style={{ padding: '0.5rem 1rem', cursor: 'pointer' }}
      >
        +1
      </button>
      <button
        onClick={() => setCount(0)}
        style={{ padding: '0.5rem 1rem', cursor: 'pointer', marginLeft: '1rem' }}
      >
        リセット
      </button>
    </div>
  )
}
```

```tsx
// app/page.tsx（サーバーコンポーネント）
import Counter from './components/Counter'

export default function HomePage() {
  return (
    <div style={{ padding: '2rem' }}>
      <h1>カウンター</h1>
      <Counter />
    </div>
  )
}
```

サーバーコンポーネント（`page.tsx`）からクライアントコンポーネント（`Counter`）をインポートして使用しています。ページ全体ではなく、インタラクティブな部分だけをクライアントコンポーネントにするのがベストプラクティスです。

</details>

## 演習3（基本）：サーバーコンポーネントで非同期データを扱おう

`async` なサーバーコンポーネントを作成し、データを取得して表示してください。

**要件：**
- サーバーコンポーネント内で `async/await` を使ってデータを取得する
- 取得したデータを画面に表示する
- 外部 API（`https://jsonplaceholder.typicode.com/users`）を使用する

<details>
<summary>ヒント</summary>

サーバーコンポーネントは `async` 関数として定義できます。`useEffect` や `useState` を使わずに、直接 `await fetch(...)` でデータを取得できます。

</details>

<details>
<summary>解答例</summary>

```tsx
// app/users/page.tsx（サーバーコンポーネント）
type User = {
  id: number
  name: string
  email: string
  company: {
    name: string
  }
}

export default async function UsersPage() {
  // サーバー上で直接データを取得（useEffect 不要）
  const res = await fetch('https://jsonplaceholder.typicode.com/users')
  const users: User[] = await res.json()

  return (
    <div style={{ padding: '2rem' }}>
      <h1>ユーザー一覧</h1>
      <p style={{ color: '#666' }}>
        このデータはサーバーで取得され、HTML に含まれた状態でクライアントに送信されます。
      </p>
      <ul style={{ listStyle: 'none', padding: 0 }}>
        {users.map((user) => (
          <li
            key={user.id}
            style={{
              padding: '1rem',
              borderBottom: '1px solid #eaeaea',
            }}
          >
            <strong>{user.name}</strong>
            <br />
            <span style={{ color: '#666' }}>{user.email}</span>
            <br />
            <span style={{ color: '#999', fontSize: '0.9rem' }}>
              {user.company.name}
            </span>
          </li>
        ))}
      </ul>
    </div>
  )
}
```

サーバーコンポーネントで `async/await` を使ってデータを取得すると、HTML に結果が含まれた状態でクライアントに送信されます。これにより SEO 対応済みで、ウォーターフォールも発生しません。

</details>

## 演習4（応用）：サーバーからクライアントへデータを渡すパターン

サーバーコンポーネントでデータを取得し、クライアントコンポーネントに渡してインタラクティブな表示を行ってください。

**要件：**
- サーバーコンポーネントで TODO データを取得する
- クライアントコンポーネントでフィルタリング機能を実装する
- 「完了」「未完了」でフィルタできるようにする

<details>
<summary>ヒント</summary>

サーバーコンポーネントでデータを取得し、Props としてクライアントコンポーネントに渡します。渡す Props はシリアライズ可能（プリミティブ値、配列、プレーンオブジェクト）である必要があります。

</details>

<details>
<summary>解答例</summary>

```tsx
// app/components/TodoFilter.tsx
'use client'

import { useState } from 'react'

type Todo = {
  id: number
  title: string
  completed: boolean
}

type Props = {
  todos: Todo[]
}

export default function TodoFilter({ todos }: Props) {
  const [filter, setFilter] = useState<'all' | 'completed' | 'pending'>('all')

  const filtered = todos.filter((todo) => {
    if (filter === 'completed') return todo.completed
    if (filter === 'pending') return !todo.completed
    return true
  })

  return (
    <div>
      {/* フィルタボタン */}
      <div style={{ display: 'flex', gap: '0.5rem', marginBottom: '1rem' }}>
        {(['all', 'completed', 'pending'] as const).map((f) => (
          <button
            key={f}
            onClick={() => setFilter(f)}
            style={{
              padding: '0.5rem 1rem',
              backgroundColor: filter === f ? '#0070f3' : '#f0f0f0',
              color: filter === f ? 'white' : '#333',
              border: 'none',
              borderRadius: '4px',
              cursor: 'pointer',
            }}
          >
            {f === 'all' ? 'すべて' : f === 'completed' ? '完了' : '未完了'}
          </button>
        ))}
      </div>

      {/* TODO リスト */}
      <p style={{ color: '#666' }}>{filtered.length}件表示中</p>
      <ul style={{ listStyle: 'none', padding: 0 }}>
        {filtered.map((todo) => (
          <li key={todo.id} style={{ padding: '0.5rem 0' }}>
            <span style={{ marginRight: '0.5rem' }}>
              {todo.completed ? '✅' : '⬜'}
            </span>
            <span style={{
              textDecoration: todo.completed ? 'line-through' : 'none',
              color: todo.completed ? '#999' : '#333',
            }}>
              {todo.title}
            </span>
          </li>
        ))}
      </ul>
    </div>
  )
}
```

```tsx
// app/todos/page.tsx（サーバーコンポーネント）
import TodoFilter from '../components/TodoFilter'

type Todo = {
  id: number
  title: string
  completed: boolean
}

export default async function TodosPage() {
  // サーバーでデータを取得
  const res = await fetch('https://jsonplaceholder.typicode.com/todos?_limit=20')
  const todos: Todo[] = await res.json()

  return (
    <div style={{ padding: '2rem', maxWidth: '600px', margin: '0 auto' }}>
      <h1>TODO リスト</h1>
      {/* サーバーで取得したデータをクライアントコンポーネントに渡す */}
      <TodoFilter todos={todos} />
    </div>
  )
}
```

データの取得はサーバーコンポーネントで行い、インタラクティブなフィルタリング機能はクライアントコンポーネントに分離しています。これにより、データ取得のセキュリティとインタラクティブ性を両立できます。

</details>

## 演習5（応用）：children パターンでプロバイダーを実装しよう

テーマを切り替えるプロバイダーコンポーネントを `children` パターンで実装してください。

**要件：**
- `app/providers/ThemeProvider.tsx` をクライアントコンポーネントとして作成する
- `children` を受け取ってレンダリングする
- `layout.tsx` でプロバイダーをラップする
- `children`（サーバーコンポーネント）はサーバーコンポーネントのまま動作することを確認する

<details>
<summary>ヒント</summary>

`children` パターンでは、クライアントコンポーネント内に `children` としてサーバーコンポーネントを渡します。`children` は既にサーバーでレンダリング済みの React ノードなので、クライアントコンポーネントの中にあってもサーバーコンポーネントのままです。

</details>

<details>
<summary>解答例</summary>

```tsx
// app/providers/ThemeProvider.tsx
'use client'

import { createContext, useContext, useState } from 'react'

type Theme = 'light' | 'dark'

const ThemeContext = createContext<{
  theme: Theme
  toggleTheme: () => void
}>({
  theme: 'light',
  toggleTheme: () => {},
})

export function useTheme() {
  return useContext(ThemeContext)
}

export default function ThemeProvider({
  children,
}: {
  children: React.ReactNode
}) {
  const [theme, setTheme] = useState<Theme>('light')

  const toggleTheme = () => {
    setTheme((prev) => (prev === 'light' ? 'dark' : 'light'))
  }

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      <div style={{
        backgroundColor: theme === 'light' ? '#ffffff' : '#1a202c',
        color: theme === 'light' ? '#1a202c' : '#ffffff',
        minHeight: '100vh',
        transition: 'all 0.3s ease',
      }}>
        {children}
      </div>
    </ThemeContext.Provider>
  )
}
```

```tsx
// app/components/ThemeToggle.tsx
'use client'

import { useTheme } from '../providers/ThemeProvider'

export default function ThemeToggle() {
  const { theme, toggleTheme } = useTheme()

  return (
    <button onClick={toggleTheme} style={{ padding: '0.5rem 1rem', cursor: 'pointer' }}>
      {theme === 'light' ? 'ダークモードに切替' : 'ライトモードに切替'}
    </button>
  )
}
```

```tsx
// app/layout.tsx（サーバーコンポーネント）
import ThemeProvider from './providers/ThemeProvider'
import ThemeToggle from './components/ThemeToggle'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="ja">
      <body>
        <ThemeProvider>
          <header style={{ padding: '1rem', display: 'flex', justifyContent: 'flex-end' }}>
            <ThemeToggle />
          </header>
          {/* children はサーバーコンポーネントのまま */}
          {children}
        </ThemeProvider>
      </body>
    </html>
  )
}
```

`ThemeProvider` はクライアントコンポーネントですが、`children` として渡されたサーバーコンポーネントはサーバーコンポーネントのまま動作します。これが `children` パターンの重要なポイントです。

</details>

## 演習6（応用）：Suspense で段階的な表示を実装しよう

`<Suspense>` を使って、複数のデータを段階的に表示するダッシュボードを作成してください。

**要件：**
- 3つの非同期サーバーコンポーネントを作成する
- 各コンポーネントを `<Suspense>` で囲み、個別のフォールバックを設定する
- 取得完了した順にコンテンツが表示されることを確認する

<details>
<summary>ヒント</summary>

`<Suspense fallback={<p>読み込み中...</p>}>` で各コンポーネントを囲みます。各コンポーネントは `async` 関数として `fetch` を行います。

</details>

<details>
<summary>解答例</summary>

```tsx
// app/dashboard/page.tsx（サーバーコンポーネント）
import { Suspense } from 'react'

export default function DashboardPage() {
  return (
    <div style={{ padding: '2rem' }}>
      <h1>ダッシュボード</h1>

      <Suspense fallback={<p style={{ color: '#999' }}>ユーザー情報を読み込み中...</p>}>
        <UserInfo />
      </Suspense>

      <Suspense fallback={<p style={{ color: '#999' }}>投稿一覧を読み込み中...</p>}>
        <RecentPosts />
      </Suspense>

      <Suspense fallback={<p style={{ color: '#999' }}>統計を読み込み中...</p>}>
        <Statistics />
      </Suspense>
    </div>
  )
}

// 非同期サーバーコンポーネント
async function UserInfo() {
  const res = await fetch('https://jsonplaceholder.typicode.com/users/1')
  const user = await res.json()

  return (
    <div style={{ padding: '1rem', border: '1px solid #eaeaea', borderRadius: '8px', marginBottom: '1rem' }}>
      <h2>ユーザー情報</h2>
      <p>名前: {user.name}</p>
      <p>メール: {user.email}</p>
    </div>
  )
}

async function RecentPosts() {
  const res = await fetch('https://jsonplaceholder.typicode.com/posts?_limit=5')
  const posts = await res.json()

  return (
    <div style={{ padding: '1rem', border: '1px solid #eaeaea', borderRadius: '8px', marginBottom: '1rem' }}>
      <h2>最近の投稿</h2>
      <ul>
        {posts.map((post: any) => (
          <li key={post.id}>{post.title}</li>
        ))}
      </ul>
    </div>
  )
}

async function Statistics() {
  const res = await fetch('https://jsonplaceholder.typicode.com/todos')
  const todos = await res.json()
  const completed = todos.filter((t: any) => t.completed).length

  return (
    <div style={{ padding: '1rem', border: '1px solid #eaeaea', borderRadius: '8px' }}>
      <h2>統計情報</h2>
      <p>総 TODO 数: {todos.length}</p>
      <p>完了数: {completed}</p>
      <p>完了率: {Math.round((completed / todos.length) * 100)}%</p>
    </div>
  )
}
```

各セクションが独立した `<Suspense>` で囲まれているため、データ取得が完了した順にストリーミングで表示されます。ページ全体のローディングを待つ必要がないため、ユーザー体験が向上します。

</details>

## 演習7（チャレンジ）：クライアント境界の最適化を実践しよう

以下の「悪い例」を、クライアント境界を最小化する「良い例」に書き換えてください。

**悪い例（ページ全体がクライアントコンポーネント）:**
```tsx
'use client'
import { useState } from 'react'

export default function ProductPage() {
  const [isWished, setIsWished] = useState(false)

  return (
    <div>
      <h1>商品名: プログラミング入門書</h1>
      <p>価格: 2,980円</p>
      <p>著者: 山田太郎</p>
      <p>出版社: テック出版</p>
      <p>ページ数: 320ページ</p>
      <p>ISBN: 978-4-xxx-xxxxx-x</p>
      <div>
        <h2>商品説明</h2>
        <p>初心者向けのプログラミング入門書です。基本的な概念から実践的なプロジェクトまで段階的に学べます。</p>
      </div>
      <button onClick={() => setIsWished(!isWished)}>
        {isWished ? 'ウィッシュリストから削除' : 'ウィッシュリストに追加'}
      </button>
    </div>
  )
}
```

**要件：**
- 静的なコンテンツはサーバーコンポーネントのままにする
- インタラクティブな部分（ウィッシュリストボタン）のみをクライアントコンポーネントに分離する

<details>
<summary>ヒント</summary>

インタラクティブな部分を別のクライアントコンポーネントに切り出し、ページ本体はサーバーコンポーネントのままにします。これにより、静的な商品情報は JavaScript バンドルに含まれなくなります。

</details>

<details>
<summary>解答例</summary>

```tsx
// app/components/WishlistButton.tsx — インタラクティブな部分のみクライアント
'use client'

import { useState } from 'react'

export default function WishlistButton() {
  const [isWished, setIsWished] = useState(false)

  return (
    <button
      onClick={() => setIsWished(!isWished)}
      style={{
        padding: '0.75rem 1.5rem',
        backgroundColor: isWished ? '#e53e3e' : '#3182ce',
        color: 'white',
        border: 'none',
        borderRadius: '6px',
        cursor: 'pointer',
        fontSize: '1rem',
      }}
    >
      {isWished ? 'ウィッシュリストから削除' : 'ウィッシュリストに追加'}
    </button>
  )
}
```

```tsx
// app/product/page.tsx — サーバーコンポーネント（デフォルト）
import WishlistButton from '../components/WishlistButton'

export default function ProductPage() {
  // この部分はすべてサーバーでレンダリングされる
  // → JavaScript バンドルに含まれない
  return (
    <div style={{ padding: '2rem', maxWidth: '600px', margin: '0 auto' }}>
      <h1>商品名: プログラミング入門書</h1>
      <p>価格: 2,980円</p>
      <p>著者: 山田太郎</p>
      <p>出版社: テック出版</p>
      <p>ページ数: 320ページ</p>
      <p>ISBN: 978-4-xxx-xxxxx-x</p>
      <div>
        <h2>商品説明</h2>
        <p>
          初心者向けのプログラミング入門書です。
          基本的な概念から実践的なプロジェクトまで段階的に学べます。
        </p>
      </div>
      {/* インタラクティブな部分だけクライアントコンポーネント */}
      <WishlistButton />
    </div>
  )
}
```

ページ全体を `'use client'` にする代わりに、ウィッシュリストボタンだけをクライアントコンポーネントに分離しました。これにより:
- 商品の静的情報は JavaScript バンドルに含まれず、ページの読み込みが高速になる
- SEO にも有利（サーバーでレンダリングされた HTML が返される）
- クライアントに送信する JavaScript のサイズが最小限になる

</details>

## 演習8（チャレンジ）：動的インポートで重いコンポーネントを遅延読み込みしよう

`next/dynamic` を使って、重いクライアントコンポーネントの遅延読み込みを実装してください。

**要件：**
- ボタンをクリックしたときにのみ、重いコンポーネント（エディタなど）を読み込む
- ローディング中のフォールバックを表示する
- `ssr: false` オプションでサーバーサイドレンダリングを無効化する

<details>
<summary>ヒント</summary>

`import dynamic from 'next/dynamic'` を使い、`dynamic(() => import('./HeavyComponent'), { ssr: false, loading: () => <p>...</p> })` のように記述します。

</details>

<details>
<summary>解答例</summary>

```tsx
// app/components/RichEditor.tsx — 重いクライアントコンポーネント
'use client'

import { useState } from 'react'

export default function RichEditor() {
  const [content, setContent] = useState('')

  return (
    <div style={{
      border: '1px solid #ccc',
      borderRadius: '8px',
      padding: '1rem',
    }}>
      <h3>リッチエディタ</h3>
      <textarea
        value={content}
        onChange={(e) => setContent(e.target.value)}
        placeholder="テキストを入力..."
        style={{ width: '100%', minHeight: '200px', padding: '0.5rem' }}
      />
      <p style={{ color: '#666' }}>文字数: {content.length}</p>
    </div>
  )
}
```

```tsx
// app/editor/page.tsx — 動的インポートでページを構成
'use client'

import { useState } from 'react'
import dynamic from 'next/dynamic'

// 遅延読み込み: ssr: false でサーバーサイドレンダリングを無効化
const RichEditor = dynamic(() => import('../components/RichEditor'), {
  ssr: false,
  loading: () => (
    <div style={{ padding: '2rem', textAlign: 'center', color: '#999' }}>
      エディタを読み込み中...
    </div>
  ),
})

export default function EditorPage() {
  const [showEditor, setShowEditor] = useState(false)

  return (
    <div style={{ padding: '2rem' }}>
      <h1>エディタページ</h1>
      <p>ボタンをクリックするとエディタが読み込まれます。</p>

      {!showEditor ? (
        <button
          onClick={() => setShowEditor(true)}
          style={{
            padding: '0.75rem 1.5rem',
            backgroundColor: '#0070f3',
            color: 'white',
            border: 'none',
            borderRadius: '6px',
            cursor: 'pointer',
          }}
        >
          エディタを開く
        </button>
      ) : (
        <RichEditor />
      )}
    </div>
  )
}
```

`dynamic` を使った遅延読み込みにより、エディタの JavaScript は必要になるまで読み込まれません。ブラウザ API に依存するコンポーネントでは `ssr: false` を設定して、サーバーサイドレンダリングをスキップします。
