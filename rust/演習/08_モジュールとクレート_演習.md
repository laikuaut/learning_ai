### 問題1：モジュールの定義とpub

1つのファイル内でモジュールを定義し、アクセス制御（`pub`）を理解する問題です。

以下の構造でモジュールを作成してください。

- `math` モジュール：基本的な数学関数（加算、減算、乗算、除算）
- `math::advanced` サブモジュール：累乗、階乗、最大公約数
- 一部の関数は非公開にして、公開関数からのみ利用する

**期待される出力例：**
```
=== 基本演算 ===
10 + 3 = 13
10 - 3 = 7
10 * 3 = 30
10 / 3 = 3（余り: 1）

=== 高度な演算 ===
2^10 = 1024
5! = 120
GCD(48, 18) = 6
LCM(12, 8) = 24
```

<details><summary>ヒント</summary>

- `mod math { ... }` でモジュールを定義します
- `pub fn` で関数を公開します
- サブモジュールは `mod advanced { ... }` のようにネストします
- 非公開のヘルパー関数は `fn`（`pub` なし）で定義します
- `use crate::math::advanced;` でモジュールをインポートできます

</details>

<details><summary>解答例</summary>

```rust
// モジュールの定義とアクセス制御

// 数学モジュール
mod math {
    // 公開: 加算
    pub fn add(a: i64, b: i64) -> i64 {
        a + b
    }

    // 公開: 減算
    pub fn sub(a: i64, b: i64) -> i64 {
        a - b
    }

    // 公開: 乗算
    pub fn mul(a: i64, b: i64) -> i64 {
        a * b
    }

    // 公開: 除算（商と余りを返す）
    pub fn div(a: i64, b: i64) -> (i64, i64) {
        (a / b, a % b)
    }

    // 高度な数学関数のサブモジュール
    pub mod advanced {
        // 公開: 累乗
        pub fn power(base: i64, exp: u32) -> i64 {
            // 非公開のヘルパー関数を使用
            power_recursive(base, exp)
        }

        // 非公開: 再帰的な累乗計算（ヘルパー）
        fn power_recursive(base: i64, exp: u32) -> i64 {
            if exp == 0 {
                return 1;
            }
            base * power_recursive(base, exp - 1)
        }

        // 公開: 階乗
        pub fn factorial(n: u64) -> u64 {
            // 非公開のヘルパー関数を使用
            factorial_iter(n)
        }

        // 非公開: 反復的な階乗計算（ヘルパー）
        fn factorial_iter(n: u64) -> u64 {
            (1..=n).product()
        }

        // 公開: 最大公約数
        pub fn gcd(a: i64, b: i64) -> i64 {
            let mut a = a.abs();
            let mut b = b.abs();
            while b != 0 {
                let temp = b;
                b = a % b;
                a = temp;
            }
            a
        }

        // 公開: 最小公倍数
        pub fn lcm(a: i64, b: i64) -> i64 {
            (a * b).abs() / gcd(a, b)
        }
    }
}

fn main() {
    // useでモジュールをインポート
    use math::advanced;

    // === 基本演算 ===
    println!("=== 基本演算 ===");
    println!("10 + 3 = {}", math::add(10, 3));
    println!("10 - 3 = {}", math::sub(10, 3));
    println!("10 * 3 = {}", math::mul(10, 3));

    let (quotient, remainder) = math::div(10, 3);
    println!("10 / 3 = {}（余り: {}）", quotient, remainder);

    println!();

    // === 高度な演算 ===
    println!("=== 高度な演算 ===");
    println!("2^10 = {}", advanced::power(2, 10));
    println!("5! = {}", advanced::factorial(5));
    println!("GCD(48, 18) = {}", advanced::gcd(48, 18));
    println!("LCM(12, 8) = {}", advanced::lcm(12, 8));

    // 以下はコンパイルエラーになる（非公開関数へのアクセス）
    // advanced::power_recursive(2, 10);  // エラー!
    // advanced::factorial_iter(5);        // エラー!
}
```

**補足：** Rustのモジュールシステムでは、デフォルトですべてが非公開です。`pub` キーワードを付けることで外部に公開します。これにより、内部実装の詳細を隠蔽し、安定したAPIだけを公開する「カプセル化」が実現できます。サブモジュールの公開関数は、親モジュールも `pub mod` で公開されている必要があります。

</details>

### 問題2：useとパスの省略

`use` 文を使ったモジュールのインポートと、パスの省略方法を練習します。さまざまな `use` の書き方を試してください。

**期待される出力例：**
```
=== useの使い方 ===

--- フルパス ---
赤 の RGB: (255, 0, 0)

--- use でインポート ---
青 の RGB: (0, 0, 255)

--- ネストしたuse ---
円の面積: 78.54
円周: 31.42

--- エイリアス ---
HashMap のサイズ: 3
BTreeMap のサイズ: 3

--- glob import ---
PI = 3.141592653589793
E = 2.718281828459045
```

<details><summary>ヒント</summary>

- `use crate::module::function;` で関数を直接インポートできます
- `use module::{item1, item2};` でまとめてインポートできます
- `use module as alias;` でエイリアス（別名）を付けられます
- `use module::*;` でモジュール内の全公開アイテムをインポートできます（glob import）

</details>

<details><summary>解答例</summary>

```rust
// useとパスの省略

// カラーモジュール
mod color {
    pub struct Color {
        pub name: String,
        pub r: u8,
        pub g: u8,
        pub b: u8,
    }

    impl Color {
        pub fn new(name: &str, r: u8, g: u8, b: u8) -> Self {
            Color {
                name: name.to_string(),
                r,
                g,
                b,
            }
        }

        pub fn rgb(&self) -> String {
            format!("({}, {}, {})", self.r, self.g, self.b)
        }
    }

    // プリセットカラーのサブモジュール
    pub mod preset {
        use super::Color;

        pub fn red() -> Color {
            Color::new("赤", 255, 0, 0)
        }

        pub fn green() -> Color {
            Color::new("緑", 0, 255, 0)
        }

        pub fn blue() -> Color {
            Color::new("青", 0, 0, 255)
        }
    }
}

// 数学定数モジュール
mod constants {
    pub const PI: f64 = std::f64::consts::PI;
    pub const E: f64 = std::f64::consts::E;
    pub const TAU: f64 = std::f64::consts::TAU;

    pub mod geometry {
        pub fn circle_area(radius: f64) -> f64 {
            super::PI * radius * radius
        }

        pub fn circumference(radius: f64) -> f64 {
            super::TAU * radius
        }
    }
}

fn main() {
    println!("=== useの使い方 ===\n");

    // --- フルパスでアクセス ---
    println!("--- フルパス ---");
    let red = color::preset::red();
    println!("{} の RGB: {}", red.name, red.rgb());
    println!();

    // --- use でインポート ---
    println!("--- use でインポート ---");
    use color::preset::blue;
    let b = blue();
    println!("{} の RGB: {}", b.name, b.rgb());
    println!();

    // --- ネストしたuse ---
    println!("--- ネストしたuse ---");
    use constants::geometry::{circle_area, circumference};
    println!("円の面積: {:.2}", circle_area(5.0));
    println!("円周: {:.2}", circumference(5.0));
    println!();

    // --- エイリアス ---
    println!("--- エイリアス ---");
    use std::collections::HashMap as HMap;
    use std::collections::BTreeMap as BMap;

    let mut hmap = HMap::new();
    hmap.insert("a", 1);
    hmap.insert("b", 2);
    hmap.insert("c", 3);
    println!("HashMap のサイズ: {}", hmap.len());

    let mut bmap = BMap::new();
    bmap.insert("x", 10);
    bmap.insert("y", 20);
    bmap.insert("z", 30);
    println!("BTreeMap のサイズ: {}", bmap.len());
    println!();

    // --- glob import ---
    println!("--- glob import ---");
    use constants::*;
    println!("PI = {}", PI);
    println!("E = {}", E);
}
```

**補足：** `use` 文のベストプラクティスは以下の通りです。

- **関数**: 親モジュールまでインポート（`use module;` → `module::function()`）
- **構造体・列挙型**: フルパスでインポート（`use module::Struct;`）
- **名前の衝突**: `as` でエイリアスを付ける
- **glob import (`*`)**: テストコード以外ではなるべく避ける（名前の衝突や、どこから来たか分かりにくくなるため）

`super::` は親モジュールを、`crate::` はクレートのルートを参照します。

</details>

### 問題3：構造体のフィールド公開制御

構造体のフィールドの公開・非公開を適切に設定し、コンストラクタとアクセサ（getter/setter）メソッドを通じてのみデータにアクセスできるようにする問題です。

**期待される出力例：**
```
=== 銀行口座 ===
口座作成: 田中太郎 (ID: ACC-001)
残高: 10000円

入金: 5000円 → 残高: 15000円
出金: 3000円 → 残高: 12000円
出金: 20000円 → エラー: 残高不足（残高: 12000円, 出金額: 20000円）

=== 取引履歴 ===
1. 入金: +5000円 (残高: 15000円)
2. 出金: -3000円 (残高: 12000円)
3. 出金失敗: -20000円 (残高不足)
```

<details><summary>ヒント</summary>

- フィールドを非公開にして、メソッド経由でのみアクセス可能にします
- `pub fn balance(&self) -> i64` のようなgetterメソッドを定義します
- 不正な操作（マイナス残高など）をメソッド内で防止します
- 取引履歴は `Vec` で管理します

</details>

<details><summary>解答例</summary>

```rust
// 構造体のフィールド公開制御

mod bank {
    use std::fmt;

    // 取引の種類
    #[derive(Debug)]
    pub enum TransactionType {
        Deposit,          // 入金
        Withdrawal,       // 出金
        WithdrawalFailed, // 出金失敗
    }

    // 取引記録
    #[derive(Debug)]
    pub struct Transaction {
        transaction_type: TransactionType,
        amount: i64,
        balance_after: i64,
    }

    impl fmt::Display for Transaction {
        fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
            match self.transaction_type {
                TransactionType::Deposit => {
                    write!(
                        f,
                        "入金: +{}円 (残高: {}円)",
                        self.amount, self.balance_after
                    )
                }
                TransactionType::Withdrawal => {
                    write!(
                        f,
                        "出金: -{}円 (残高: {}円)",
                        self.amount, self.balance_after
                    )
                }
                TransactionType::WithdrawalFailed => {
                    write!(
                        f,
                        "出金失敗: -{}円 (残高不足)",
                        self.amount
                    )
                }
            }
        }
    }

    // 銀行口座（フィールドは全て非公開）
    pub struct BankAccount {
        id: String,
        owner: String,
        balance: i64,
        history: Vec<Transaction>,
    }

    impl BankAccount {
        // コンストラクタ
        pub fn new(
            id: &str,
            owner: &str,
            initial_balance: i64,
        ) -> Self {
            BankAccount {
                id: id.to_string(),
                owner: owner.to_string(),
                balance: initial_balance,
                history: Vec::new(),
            }
        }

        // getter: 口座ID
        pub fn id(&self) -> &str {
            &self.id
        }

        // getter: 口座名義人
        pub fn owner(&self) -> &str {
            &self.owner
        }

        // getter: 残高
        pub fn balance(&self) -> i64 {
            self.balance
        }

        // 入金
        pub fn deposit(&mut self, amount: i64) -> Result<i64, String> {
            if amount <= 0 {
                return Err(
                    "入金額は正の数である必要があります".to_string()
                );
            }

            self.balance += amount;
            self.history.push(Transaction {
                transaction_type: TransactionType::Deposit,
                amount,
                balance_after: self.balance,
            });

            println!(
                "入金: {}円 → 残高: {}円",
                amount, self.balance
            );
            Ok(self.balance)
        }

        // 出金
        pub fn withdraw(
            &mut self,
            amount: i64,
        ) -> Result<i64, String> {
            if amount <= 0 {
                return Err(
                    "出金額は正の数である必要があります".to_string()
                );
            }

            if self.balance < amount {
                self.history.push(Transaction {
                    transaction_type: TransactionType::WithdrawalFailed,
                    amount,
                    balance_after: self.balance,
                });

                let msg = format!(
                    "残高不足（残高: {}円, 出金額: {}円）",
                    self.balance, amount
                );
                println!("出金: {}円 → エラー: {}", amount, msg);
                return Err(msg);
            }

            self.balance -= amount;
            self.history.push(Transaction {
                transaction_type: TransactionType::Withdrawal,
                amount,
                balance_after: self.balance,
            });

            println!(
                "出金: {}円 → 残高: {}円",
                amount, self.balance
            );
            Ok(self.balance)
        }

        // 取引履歴を表示
        pub fn print_history(&self) {
            println!("=== 取引履歴 ===");
            for (i, tx) in self.history.iter().enumerate() {
                println!("{}. {}", i + 1, tx);
            }
        }
    }
}

fn main() {
    // bankモジュールからBankAccountをインポート
    use bank::BankAccount;

    println!("=== 銀行口座 ===");

    let mut account = BankAccount::new("ACC-001", "田中太郎", 10000);
    println!(
        "口座作成: {} (ID: {})",
        account.owner(),
        account.id()
    );
    println!("残高: {}円\n", account.balance());

    // 入金・出金操作
    let _ = account.deposit(5000);
    let _ = account.withdraw(3000);
    let _ = account.withdraw(20000);

    println!();

    // 取引履歴
    account.print_history();

    // 以下はコンパイルエラーになる（フィールドに直接アクセスできない）
    // account.balance = 999999;    // エラー!
    // account.id = "HACK".into();  // エラー!
}
```

**補足：** 構造体のフィールドを非公開にし、メソッド経由でのみアクセスさせることで、データの不整合を防止できます。例えば、残高を直接変更できると、取引履歴と残高が不一致になる可能性がありますが、`deposit` と `withdraw` メソッドを通せば常に整合性が保たれます。これは「情報隠蔽」の原則で、Rustのモジュールシステムがこれを言語レベルでサポートしています。

</details>

### 問題4：ファイル分割を想定したモジュール設計

実際のプロジェクトで使うような、ファイル分割を想定したモジュール設計を1つのファイルで練習します。オンラインストアのドメインモデルを設計してください。

以下のモジュール構造を実装してください。

```
store
├── product    （商品）
├── cart       （ショッピングカート）
└── order      （注文）
```

**期待される出力例：**
```
=== オンラインストア ===

--- 商品カタログ ---
[P001] Rust入門書 - 3000円 (在庫: 10)
[P002] プログラミングキーボード - 15000円 (在庫: 5)
[P003] USBケーブル - 500円 (在庫: 100)

--- ショッピング ---
カートに追加: Rust入門書 x 2
カートに追加: USBケーブル x 3

カート内容:
  Rust入門書 x 2 = 6000円
  USBケーブル x 3 = 1500円
合計: 7500円 (3商品)

--- 注文 ---
注文番号: ORD-001
注文が確定しました
注文合計: 7500円
ステータス: 処理中
```

<details><summary>ヒント</summary>

- 各モジュールは `pub` で公開し、構造体やメソッドも必要に応じて `pub` にします
- `product` モジュールの `Product` 構造体を `cart` モジュールで使うには `use super::product::Product;` とします
- `Clone` トレイトを導入すると、所有権の問題を解決しやすくなります
- 注文ステータスは `enum` で表現します

</details>

<details><summary>解答例</summary>

```rust
// ファイル分割を想定したモジュール設計
// オンラインストアのドメインモデル

mod store {
    // 商品モジュール
    pub mod product {
        #[derive(Debug, Clone)]
        pub struct Product {
            id: String,
            name: String,
            price: u32,
            stock: u32,
        }

        impl Product {
            pub fn new(
                id: &str,
                name: &str,
                price: u32,
                stock: u32,
            ) -> Self {
                Product {
                    id: id.to_string(),
                    name: name.to_string(),
                    price,
                    stock,
                }
            }

            pub fn id(&self) -> &str {
                &self.id
            }

            pub fn name(&self) -> &str {
                &self.name
            }

            pub fn price(&self) -> u32 {
                self.price
            }

            pub fn stock(&self) -> u32 {
                self.stock
            }

            pub fn display(&self) {
                println!(
                    "[{}] {} - {}円 (在庫: {})",
                    self.id, self.name, self.price, self.stock
                );
            }

            // 在庫を減らす
            pub fn reduce_stock(
                &mut self,
                quantity: u32,
            ) -> Result<(), String> {
                if self.stock < quantity {
                    return Err(format!(
                        "在庫不足: {} (在庫: {}, 要求: {})",
                        self.name, self.stock, quantity
                    ));
                }
                self.stock -= quantity;
                Ok(())
            }
        }
    }

    // ショッピングカートモジュール
    pub mod cart {
        use super::product::Product;

        // カート内のアイテム
        #[derive(Debug, Clone)]
        pub struct CartItem {
            product: Product,
            quantity: u32,
        }

        impl CartItem {
            pub fn subtotal(&self) -> u32 {
                self.product.price() * self.quantity
            }

            pub fn display(&self) {
                println!(
                    "  {} x {} = {}円",
                    self.product.name(),
                    self.quantity,
                    self.subtotal()
                );
            }
        }

        // ショッピングカート
        pub struct ShoppingCart {
            items: Vec<CartItem>,
        }

        impl ShoppingCart {
            pub fn new() -> Self {
                ShoppingCart { items: Vec::new() }
            }

            // 商品をカートに追加
            pub fn add(
                &mut self,
                product: &Product,
                quantity: u32,
            ) {
                // 既存のアイテムがあれば数量を加算
                for item in &mut self.items {
                    if item.product.id() == product.id() {
                        item.quantity += quantity;
                        println!(
                            "カートに追加: {} x {}",
                            product.name(),
                            item.quantity
                        );
                        return;
                    }
                }

                // 新規アイテム追加
                println!(
                    "カートに追加: {} x {}",
                    product.name(),
                    quantity
                );
                self.items.push(CartItem {
                    product: product.clone(),
                    quantity,
                });
            }

            // 合計金額
            pub fn total(&self) -> u32 {
                self.items.iter().map(|item| item.subtotal()).sum()
            }

            // 合計アイテム数
            pub fn total_items(&self) -> u32 {
                self.items.iter().map(|item| item.quantity).sum()
            }

            // カート内容を表示
            pub fn display(&self) {
                println!("カート内容:");
                for item in &self.items {
                    item.display();
                }
                println!(
                    "合計: {}円 ({}商品)",
                    self.total(),
                    self.total_items()
                );
            }

            // カート内のアイテムを取得（注文生成用）
            pub fn items(&self) -> &[CartItem] {
                &self.items
            }

            // カートを空にする
            pub fn clear(&mut self) {
                self.items.clear();
            }
        }
    }

    // 注文モジュール
    pub mod order {
        use super::cart::ShoppingCart;

        // 注文ステータス
        #[derive(Debug)]
        pub enum OrderStatus {
            Pending,    // 保留
            Processing, // 処理中
            Shipped,    // 発送済み
            Delivered,  // 配達完了
            Cancelled,  // キャンセル
        }

        impl OrderStatus {
            pub fn name(&self) -> &str {
                match self {
                    OrderStatus::Pending => "保留",
                    OrderStatus::Processing => "処理中",
                    OrderStatus::Shipped => "発送済み",
                    OrderStatus::Delivered => "配達完了",
                    OrderStatus::Cancelled => "キャンセル",
                }
            }
        }

        // 注文
        pub struct Order {
            id: String,
            total: u32,
            status: OrderStatus,
        }

        impl Order {
            // カートから注文を生成
            pub fn from_cart(
                order_id: &str,
                cart: &ShoppingCart,
            ) -> Result<Self, String> {
                if cart.total_items() == 0 {
                    return Err(
                        "カートが空です".to_string()
                    );
                }

                let order = Order {
                    id: order_id.to_string(),
                    total: cart.total(),
                    status: OrderStatus::Pending,
                };

                println!("注文番号: {}", order.id);
                Ok(order)
            }

            // 注文を確定
            pub fn confirm(&mut self) {
                self.status = OrderStatus::Processing;
                println!("注文が確定しました");
            }

            // 注文合計を取得
            pub fn total(&self) -> u32 {
                self.total
            }

            // ステータスを表示
            pub fn display_status(&self) {
                println!("ステータス: {}", self.status.name());
            }
        }
    }
}

fn main() {
    use store::cart::ShoppingCart;
    use store::order::Order;
    use store::product::Product;
```

### 問題5：テスト記述の基本

`#[cfg(test)]` と `#[test]` を使ったユニットテストを書く練習をします。文字列処理ライブラリにテストを追加してください。

以下の関数にテストを書いてください。

1. `reverse_string`: 文字列を反転する
2. `count_vowels`: 母音（a, e, i, o, u）の数を数える
3. `is_palindrome`: 回文かどうか判定する
4. `truncate`: 指定文字数で切り詰めて「...」を付ける

**期待される出力例（テスト実行結果）：**

- `#[cfg(test)]` でテストモジュールを定義します
- `#[test]` で各テスト関数をマークします
- `assert_eq!(actual, expected)` で値の一致を確認します
- `assert!(condition)` で条件の真偽を確認します

```rust
// テスト記述の基本 - 文字列処理ライブラリ

mod string_utils {
    // 文字列を反転する
    pub fn reverse_string(s: &str) -> String {
        s.chars().rev().collect()
    }

    // 母音の数を数える（英語の母音: a, e, i, o, u）
    pub fn count_vowels(s: &str) -> usize {
        s.to_lowercase()
            .chars()
            .filter(|c| "aeiou".contains(*c))
            .count()
    }

    // 回文かどうか判定する（大文字小文字を無視）
    pub fn is_palindrome(s: &str) -> bool {
        let cleaned: String = s
            .to_lowercase()
            .chars()
            .filter(|c| c.is_alphanumeric())
            .collect();
        let reversed: String = cleaned.chars().rev().collect();
        cleaned == reversed
    }

    // 指定文字数で切り詰めて "..." を付ける
    pub fn truncate(s: &str, max_len: usize) -> String {
        if s.len() <= max_len {
            s.to_string()
        } else {
            let truncated: String =
                s.chars().take(max_len).collect();
            format!("{}...", truncated)
        }
    }

    // テストモジュール
    #[cfg(test)]
    mod tests {
        // 親モジュールの全公開アイテムをインポート
        use super::*;

            assert_eq!(reverse_string(""), "");

        #[test]
        fn test_reverse_single_char() {
            assert_eq!(reverse_string("a"), "a");

            assert!(!is_palindrome("hello"));
```

### 問題6：再エクスポートとAPIの整理

モジュール内の型を `pub use` で再エクスポートし、ユーザーに使いやすいAPIを提供する方法を練習します。

ログシステムのライブラリを設計し、内部構造を隠蔽しつつ、必要な型だけを公開してください。

**期待される出力例：**
```
=== ログシステム ===

[2024-01-15 10:30:00] [INFO] アプリケーションが起動しました
[2024-01-15 10:30:01] [DEBUG] 設定ファイルを読み込み中
[2024-01-15 10:30:02] [WARN] メモリ使用量が80%を超えています
[2024-01-15 10:30:03] [ERROR] データベース接続に失敗しました

--- ログ統計 ---
INFO: 1件
DEBUG: 1件
WARN: 1件
ERROR: 1件
合計: 4件

--- ERRORのみフィルタ ---
[ERROR] データベース接続に失敗しました
```

<details><summary>ヒント</summary>

- `pub use` で内部モジュールの型を親モジュールから直接アクセス可能にします
- 例：`pub use self::level::LogLevel;` とすると `logging::LogLevel` でアクセスできます
- 内部モジュールの詳細は隠蔽し、公開APIのみをエクスポートします

</details>

<details><summary>解答例</summary>

```rust
// 再エクスポートとAPIの整理 - ログシステム

mod logging {
    // --- 内部モジュール: ログレベル ---
    mod level {
        #[derive(Debug, Clone, Copy, PartialEq, PartialOrd)]
        pub enum LogLevel {
            Debug,
            Info,
            Warn,
            Error,
        }

        impl LogLevel {
            pub fn label(&self) -> &str {
                match self {
                    LogLevel::Debug => "DEBUG",
                    LogLevel::Info => "INFO",
                    LogLevel::Warn => "WARN",
                    LogLevel::Error => "ERROR",
                }
            }
        }
    }

    // --- 内部モジュール: ログエントリ ---
    mod entry {
        use super::level::LogLevel;

        #[derive(Debug, Clone)]
        pub struct LogEntry {
            pub timestamp: String,
            pub level: LogLevel,
            pub message: String,
        }

        impl LogEntry {
            pub fn new(
                timestamp: &str,
                level: LogLevel,
                message: &str,
            ) -> Self {
                LogEntry {
                    timestamp: timestamp.to_string(),
                    level,
                    message: message.to_string(),
                }
            }

            pub fn format(&self) -> String {
                format!(
                    "[{}] [{}] {}",
                    self.timestamp,
                    self.level.label(),
                    self.message
                )
            }

            pub fn format_short(&self) -> String {
                format!(
                    "[{}] {}",
                    self.level.label(),
                    self.message
                )
            }
        }
    }

    // --- 内部モジュール: ロガー ---
    mod logger {
        use super::entry::LogEntry;
        use super::level::LogLevel;

        pub struct Logger {
            entries: Vec<LogEntry>,
            min_level: LogLevel,
        }

        impl Logger {
            pub fn new(min_level: LogLevel) -> Self {
                Logger {
                    entries: Vec::new(),
                    min_level,
                }
            }

            pub fn log(
                &mut self,
                timestamp: &str,
                level: LogLevel,
                message: &str,
            ) {
                let entry =
                    LogEntry::new(timestamp, level, message);

                // 最小レベル以上のみ記録・表示
                if level >= self.min_level {
                    println!("{}", entry.format());
                    self.entries.push(entry);
                }
            }

            pub fn info(&mut self, timestamp: &str, message: &str) {
                self.log(timestamp, LogLevel::Info, message);
            }

            pub fn debug(
                &mut self,
                timestamp: &str,
                message: &str,
            ) {
                self.log(timestamp, LogLevel::Debug, message);
            }

            pub fn warn(&mut self, timestamp: &str, message: &str) {
                self.log(timestamp, LogLevel::Warn, message);
            }

            pub fn error(
                &mut self,
                timestamp: &str,
                message: &str,
            ) {
                self.log(timestamp, LogLevel::Error, message);
            }

            // ログレベルごとの件数を集計
            pub fn stats(&self) {
                println!("--- ログ統計 ---");

                let levels = vec![
                    LogLevel::Info,
                    LogLevel::Debug,
                    LogLevel::Warn,
                    LogLevel::Error,
                ];

                for level in &levels {
                    let count = self
                        .entries
                        .iter()
                        .filter(|e| e.level == *level)
                        .count();
                    if count > 0 {
                        println!("{}: {}件", level.label(), count);
                    }
                }

                println!("合計: {}件", self.entries.len());
            }

            // 指定レベルのログをフィルタ
            pub fn filter_by_level(
                &self,
                level: LogLevel,
            ) -> Vec<String> {
                self.entries
                    .iter()
                    .filter(|e| e.level == level)
                    .map(|e| e.format_short())
                    .collect()
            }
        }
    }

    // ======================================
    // pub use で公開APIを整理
    // ユーザーは logging::LogLevel, logging::Logger で
    // アクセスできる（内部モジュール名を知らなくてよい）
    // ======================================
    pub use self::level::LogLevel;
    pub use self::logger::Logger;
    // LogEntry は内部実装なので再エクスポートしない
}

fn main() {
    // pub use のおかげで、logging:: から直接アクセスできる
    // logging::level::LogLevel ではなく logging::LogLevel
    use logging::{LogLevel, Logger};

    println!("=== ログシステム ===\n");

    // DEBUGレベル以上のログを記録するロガーを作成
    let mut logger = Logger::new(LogLevel::Debug);

    logger.info(
        "2024-01-15 10:30:00",
        "アプリケーションが起動しました",
    );
    logger.debug(
        "2024-01-15 10:30:01",
        "設定ファイルを読み込み中",
    );
    logger.warn(
        "2024-01-15 10:30:02",
        "メモリ使用量が80%を超えています",
    );
    logger.error(
        "2024-01-15 10:30:03",
        "データベース接続に失敗しました",
    );

    println!();

    // 統計
    logger.stats();

    println!();

    // フィルタ
    println!("--- ERRORのみフィルタ ---");
    let errors = logger.filter_by_level(LogLevel::Error);
    for entry in errors {
        println!("{}", entry);
    }
}
```

**補足：** `pub use`（再エクスポート）は、ライブラリの公開APIを設計する上で非常に重要なテクニックです。内部のモジュール構造を変更しても、`pub use` を適切に設定すれば、ユーザーから見えるAPIを変えずに済みます。

例えば、`logging::level::LogLevel` を `logging::LogLevel` として再エクスポートすることで、ユーザーは内部構造を意識せずに使えます。Rustの標準ライブラリでもこのパターンが多用されています（例：`std::io::Error` は実際には内部モジュールからの再エクスポートです）。

</details>

### 問題7：テスト駆動で小さなライブラリを作成

テスト駆動開発（TDD）の流れで、シンプルなスタック（LIFO）データ構造ライブラリを作成してください。

手順：
1. まずテストを書く
2. テストが通る最小限の実装を書く
3. リファクタリングする

以下の機能をテスト付きで実装してください。

- `new()`: 空のスタックを作成
- `push(item)`: 要素を追加
- `pop()`: 最後の要素を取り出す（`Option<T>`）
- `peek()`: 最後の要素を参照する（`Option<&T>`）
- `is_empty()`: 空かどうか
- `size()`: 要素数
- `clear()`: 全要素を削除

**期待される出力例：**
```
=== Stack ライブラリ ===

--- 基本操作 ---
push: 10
push: 20
push: 30
サイズ: 3, 空?: false
peek: Some(30)
pop: Some(30)
pop: Some(20)
サイズ: 1

--- クリア ---
clear 前: サイズ = 1
clear 後: サイズ = 0, 空?: true

--- 文字列スタック ---
push: "Hello"
push: "World"
pop: Some("World")
pop: Some("Hello")
pop: None

=== テスト結果のシミュレーション ===
test_new_stack_is_empty ... ok
test_push_and_pop ... ok
test_peek ... ok
test_pop_empty_stack ... ok
test_size ... ok
test_clear ... ok
test_multiple_types ... ok
全 7 テスト: ok
```

<details><summary>ヒント</summary>

- ジェネリクス `Stack<T>` で任意の型に対応します
- 内部では `Vec<T>` を使って実装します
- `peek()` は `self.items.last()` で実装できます
- テストでは `assert_eq!`、`assert!`、`assert_ne!` を使います

</details>

<details><summary>解答例</summary>

```rust
// テスト駆動で作る Stack ライブラリ

mod collections {
    // ジェネリックなスタック
    pub struct Stack<T> {
        items: Vec<T>,
    }

    impl<T> Stack<T> {
        // 空のスタックを作成
        pub fn new() -> Self {
            Stack { items: Vec::new() }
        }

        // 要素を追加
        pub fn push(&mut self, item: T) {
            self.items.push(item);
        }

        // 最後の要素を取り出す
        pub fn pop(&mut self) -> Option<T> {
            self.items.pop()
        }

        // 最後の要素を参照する（取り出さない）
        pub fn peek(&self) -> Option<&T> {
            self.items.last()
        }

        // 空かどうか
        pub fn is_empty(&self) -> bool {
            self.items.is_empty()
        }

        // 要素数
        pub fn size(&self) -> usize {
            self.items.len()
        }

        // 全要素を削除
        pub fn clear(&mut self) {
            self.items.clear();
        }
    }

    // ================================
    // テストモジュール
    // ================================
    #[cfg(test)]
    mod tests {
        use super::*;

        #[test]
        fn test_new_stack_is_empty() {
            let stack: Stack<i32> = Stack::new();
            assert!(stack.is_empty());
            assert_eq!(stack.size(), 0);
        }

        #[test]
        fn test_push_and_pop() {
            let mut stack = Stack::new();
            stack.push(1);
            stack.push(2);
            stack.push(3);

            // LIFO: 最後に入れたものが最初に出る
            assert_eq!(stack.pop(), Some(3));
            assert_eq!(stack.pop(), Some(2));
            assert_eq!(stack.pop(), Some(1));
            assert_eq!(stack.pop(), None);
        }

        #[test]
        fn test_peek() {
            let mut stack = Stack::new();
            // 空のスタックでpeek
            assert_eq!(stack.peek(), None);

            stack.push(42);
            // peekは値を取り出さない
            assert_eq!(stack.peek(), Some(&42));
            assert_eq!(stack.peek(), Some(&42));
            assert_eq!(stack.size(), 1); // サイズは変わらない
        }

        #[test]
        fn test_pop_empty_stack() {
            let mut stack: Stack<i32> = Stack::new();
            assert_eq!(stack.pop(), None);
            assert_eq!(stack.pop(), None); // 連続でNone
        }

        #[test]
        fn test_size() {
            let mut stack = Stack::new();
            assert_eq!(stack.size(), 0);

            stack.push("a");
            assert_eq!(stack.size(), 1);

            stack.push("b");
            assert_eq!(stack.size(), 2);

            stack.pop();
            assert_eq!(stack.size(), 1);
        }

        #[test]
        fn test_clear() {
            let mut stack = Stack::new();
            stack.push(1);
            stack.push(2);
            stack.push(3);

            assert_eq!(stack.size(), 3);
            stack.clear();
            assert!(stack.is_empty());
            assert_eq!(stack.size(), 0);
            assert_eq!(stack.pop(), None);
        }

        #[test]
        fn test_multiple_types() {
            // 整数スタック
            let mut int_stack = Stack::new();
            int_stack.push(1);
            int_stack.push(2);
            assert_eq!(int_stack.pop(), Some(2));

            // 文字列スタック
            let mut str_stack = Stack::new();
            str_stack.push("hello".to_string());
            str_stack.push("world".to_string());
            assert_eq!(
                str_stack.pop(),
                Some("world".to_string())
            );

            // f64スタック
            let mut float_stack = Stack::new();
            float_stack.push(3.14);
            assert_eq!(float_stack.peek(), Some(&3.14));
        }
    }
}

    // --- クリア ---
    println!("--- クリア ---");
    println!("clear 前: サイズ = {}", stack.size());
    stack.clear();
    println!(
        "clear 後: サイズ = {}, 空?: {}",
        stack.size(),
        stack.is_empty()
    );

    println!();

    // --- 文字列スタック ---
    println!("--- 文字列スタック ---");
    let mut str_stack: Stack<&str> = Stack::new();

    for s in &["Hello", "World"] {
        println!("push: \"{}\"", s);
        str_stack.push(s);
    }
    println!("pop: {:?}", str_stack.pop());
    println!("pop: {:?}", str_stack.pop());
    println!("pop: {:?}", str_stack.pop());

    println!();

    simulate_tests();
}
```

**補足：** テスト駆動開発（TDD）は「Red → Green → Refactor」のサイクルで進めます。

1. **Red**: 失敗するテストを書く
2. **Green**: テストが通る最小限の実装をする
3. **Refactor**: コードを改善する

Rustでは `cargo test` でテストを実行できます。`#[cfg(test)]` モジュールのテストは単体テスト（unit test）と呼ばれ、プライベートな関数にもアクセスできます。結合テスト（integration test）は `tests/` ディレクトリに配置し、公開APIのみをテストします。

</details>

### 問題8：モジュールを活用した完成度の高いライブラリ設計

<details><summary>ヒント</summary>

- `pub use` でユーザーが `tasklib::Task` のようにアクセスできるようにします
- タスクIDは `storage` モジュール内で自動採番します
- `formatter` モジュールはタスクの表示ロジックを担当します

<details><summary>解答例</summary>

```rust
// タスク管理ライブラリ
// モジュールを活用した実践的な設計

mod tasklib {
    // ========================================
    // モデルモジュール: データ型の定義
    // ========================================
    pub mod model {
        // 優先度
        #[derive(Debug, Clone, Copy, PartialEq)]
        pub enum Priority {
            Low,
            Medium,
            High,
        }

        impl Priority {
            pub fn label(&self) -> &str {
                match self {
                    Priority::Low => "低",
                    Priority::Medium => "中",
                    Priority::High => "高",
                }
            }
        }

        // ステータス
        #[derive(Debug, Clone, Copy, PartialEq)]
        pub enum Status {
            Todo,        // 未着手
            InProgress,  // 進行中
            Done,        // 完了
        }

        impl Status {
            pub fn label(&self) -> &str {
                match self {
                    Status::Todo => "未着手",
                    Status::InProgress => "進行中",
                    Status::Done => "完了",
                }
            }
        }

        // タスク
        #[derive(Debug, Clone)]
        pub struct Task {
            id: u32,
            title: String,
            priority: Priority,
            status: Status,
        }

        impl Task {
            pub fn new(
                id: u32,
                title: &str,
                priority: Priority,
            ) -> Self {
                Task {
                    id,
                    title: title.to_string(),
                    priority,
                    status: Status::Todo,
                }
            }

            pub fn id(&self) -> u32 {
                self.id
            }

            pub fn title(&self) -> &str {
                &self.title
            }

            pub fn priority(&self) -> Priority {
                self.priority
            }

            pub fn status(&self) -> Status {
                self.status
            }

            pub fn set_status(&mut self, status: Status) {
                self.status = status;
            }
        }
    }

    // ========================================
    // ストレージモジュール: タスクの保存・検索
    // ========================================
    pub mod storage {
        use super::model::{Priority, Status, Task};

        pub struct TaskStore {
            tasks: Vec<Task>,
            next_id: u32,
        }

        impl TaskStore {
            pub fn new() -> Self {
                TaskStore {
                    tasks: Vec::new(),
                    next_id: 1,
                }
            }

            // タスクを追加
            pub fn add(
                &mut self,
                title: &str,
                priority: Priority,
            ) -> &Task {
                let task =
                    Task::new(self.next_id, title, priority);
                println!(
                    "タスクを追加しました: {} (優先度: {})",
                    title,
                    priority.label()
                );
                self.tasks.push(task);
                self.next_id += 1;
                self.tasks.last().unwrap()
            }

            // ステータス更新
            pub fn update_status(
                &mut self,
                id: u32,
                status: Status,
            ) -> Result<(), String> {
                match self.tasks.iter_mut().find(|t| t.id() == id) {
                    Some(task) => {
                        task.set_status(status);
                        println!(
                            "タスク #{} のステータスを '{}' に更新しました",
                            id,
                            status.label()
                        );
                        Ok(())
                    }
                    None => Err(format!(
                        "タスク #{} が見つかりません",
                        id
                    )),
                }
            }

            // 全タスクを取得
            pub fn all(&self) -> &[Task] {
                &self.tasks
            }

            // 優先度でフィルタ
            pub fn filter_by_priority(
                &self,
                priority: Priority,
            ) -> Vec<&Task> {
                self.tasks
                    .iter()
                    .filter(|t| t.priority() == priority)
                    .collect()
            }

            // 未完了タスクを取得
            pub fn incomplete(&self) -> Vec<&Task> {
                self.tasks
                    .iter()
                    .filter(|t| t.status() != Status::Done)
                    .collect()
            }

            // ステータスごとの件数
            pub fn count_by_status(&self, status: Status) -> usize {
                self.tasks
                    .iter()
                    .filter(|t| t.status() == status)
                    .count()
            }

            // 全タスク数
            pub fn total(&self) -> usize {
                self.tasks.len()
            }
        }
    }

    // ========================================
    // フォーマッタモジュール: 表示ロジック
    // ========================================
    pub mod formatter {
        use super::model::{Status, Task};
        use super::storage::TaskStore;

        // タスク1件のフォーマット
        pub fn format_task(task: &Task) -> String {
            format!(
                "  #{} [{}] {} ({})",
                task.id(),
                task.priority().label(),
                task.title(),
                task.status().label()
            )
        }

        // タスクリストの表示
        pub fn print_tasks(tasks: &[&Task]) {
            for task in tasks {
                println!("{}", format_task(task));
            }
        }

        // 全タスクの表示
        pub fn print_all(store: &TaskStore) {
            for task in store.all() {
                println!("{}", format_task(task));
            }
        }

        // サマリーの表示
        pub fn print_summary(store: &TaskStore) {
            println!("全タスク: {}件", store.total());
            println!(
                "  未着手: {}件",
                store.count_by_status(Status::Todo)
            );
            println!(
                "  進行中: {}件",
                store.count_by_status(Status::InProgress)
            );
            println!(
                "  完了: {}件",
                store.count_by_status(Status::Done)
            );
        }
    }

    // ========================================
    // pub use で公開APIを整理
    // ========================================
    pub use self::model::{Priority, Status, Task};
    pub use self::storage::TaskStore;
    pub use self::formatter::{
        format_task, print_all, print_summary, print_tasks,
    };

    // ========================================
    // テストモジュール
    // ========================================
    #[cfg(test)]
    mod tests {
        use super::*;

        #[test]
        fn test_add_task() {
            let mut store = TaskStore::new();
            store.add("テストタスク", Priority::High);
            assert_eq!(store.total(), 1);
            assert_eq!(store.all()[0].title(), "テストタスク");
            assert_eq!(
                store.all()[0].priority(),
                Priority::High
            );
        }

        #[test]
        fn test_update_status() {
            let mut store = TaskStore::new();
            store.add("タスク1", Priority::Medium);

            let result =
                store.update_status(1, Status::InProgress);
            assert!(result.is_ok());
            assert_eq!(
                store.all()[0].status(),
                Status::InProgress
            );

            // 存在しないID
            let result = store.update_status(99, Status::Done);
            assert!(result.is_err());
        }

        #[test]
        fn test_filter_by_priority() {
            let mut store = TaskStore::new();
            store.add("高1", Priority::High);
            store.add("低1", Priority::Low);
            store.add("高2", Priority::High);

            let high_tasks =
                store.filter_by_priority(Priority::High);
            assert_eq!(high_tasks.len(), 2);
        }

        #[test]
        fn test_incomplete_tasks() {
            let mut store = TaskStore::new();
            store.add("タスク1", Priority::High);
            store.add("タスク2", Priority::Medium);
            let _ = store.update_status(1, Status::Done);

            let incomplete = store.incomplete();
            assert_eq!(incomplete.len(), 1);
            assert_eq!(incomplete[0].title(), "タスク2");
        }

        #[test]
        fn test_summary() {
            let mut store = TaskStore::new();
            store.add("タスク1", Priority::High);
            store.add("タスク2", Priority::Medium);
            store.add("タスク3", Priority::Low);
            let _ = store.update_status(1, Status::InProgress);
            let _ = store.update_status(2, Status::Done);

            assert_eq!(
                store.count_by_status(Status::Todo),
                1
            );
            assert_eq!(
                store.count_by_status(Status::InProgress),
                1
            );
            assert_eq!(
                store.count_by_status(Status::Done),
                1
            );
        }
    }
}

        "test_update_status",
        "test_filter_by_priority",

fn main() {
    // pub use のおかげで tasklib:: から直接アクセスできる
    use tasklib::{
        print_all, print_summary, print_tasks, Priority,
        Status, TaskStore,
    };

    println!("=== タスク管理システム ===\n");

    let mut store = TaskStore::new();

    // タスクを追加
    store.add("Rustの学習", Priority::High);
    store.add("ドキュメント作成", Priority::Medium);
    store.add("コードレビュー", Priority::High);
    store.add("ミーティング準備", Priority::Low);

    println!();

    // 全タスク
    println!("--- 全タスク ---");
    print_all(&store);

    println!();

    // ステータス更新
    println!("--- ステータス更新 ---");
    let _ = store.update_status(1, Status::InProgress);
    let _ = store.update_status(3, Status::Done);

    println!();

    // 優先度「高」でフィルタ
    println!("--- 優先度「高」のタスク ---");
    let high_tasks = store.filter_by_priority(Priority::High);
    print_tasks(&high_tasks);

    println!();

    // 未完了タスク
    println!("--- 未完了タスク ---");
    let incomplete = store.incomplete();
    print_tasks(&incomplete);

    println!();

    // サマリー
    println!("--- サマリー ---");
    print_summary(&store);

    println!();

    simulate_tests();
}
```

**補足：** この問題では、実際のRustプロジェクトで使われるモジュール設計のパターンを総合的に学べます。

**設計のポイント：**
1. **関心の分離**: `model`（データ型）、`storage`（永続化）、`formatter`（表示）を分離
2. **カプセル化**: 構造体のフィールドを非公開にし、メソッド経由でアクセス
3. **再エクスポート**: `pub use` で使いやすいAPIを提供
4. **テスト**: 各機能のユニットテストを記述

実際のプロジェクトでは、これらのモジュールを別ファイルに分割し、`mod.rs` や `lib.rs` で管理します。`cargo` プロジェクトでは `src/lib.rs` がライブラリクレートのルート、`src/main.rs` がバイナリクレートのルートになります。
