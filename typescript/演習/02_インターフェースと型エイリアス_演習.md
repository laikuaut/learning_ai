## 問題1：基本的なインターフェース（基本）

以下の要件を満たす `Book` インターフェースを定義し、それに従ったオブジェクトを2つ作成してください。

- `title: string` — 書名
- `author: string` — 著者
- `pages: number` — ページ数
- `isbn?: string` — ISBN（オプショナル）

**期待される出力：**
```
TypeScript入門（田中太郎）- 350ページ [ISBN: 978-4-xxx]
JavaScript基礎（山田花子）- 280ページ [ISBN: なし]
```

<details>
<summary>ヒント</summary>

- `interface` キーワードでインターフェースを定義します。
- オプショナルプロパティは `?` を付けます。
- `??` 演算子でデフォルト値を設定できます。

</details>

<details>
<summary>解答例</summary>

```typescript
// インターフェースの定義
interface Book {
  title: string;
  author: string;
  pages: number;
  isbn?: string; // オプショナルプロパティ
}

// インターフェースに従ったオブジェクト
const book1: Book = {
  title: "TypeScript入門",
  author: "田中太郎",
  pages: 350,
  isbn: "978-4-xxx",
};

const book2: Book = {
  title: "JavaScript基礎",
  author: "山田花子",
  pages: 280,
  // isbnは省略（オプショナルなのでOK）
};

// 表示関数
function formatBook(book: Book): string {
  const isbnStr = book.isbn ?? "なし";
  return `${book.title}（${book.author}）- ${book.pages}ページ [ISBN: ${isbnStr}]`;
}

console.log(formatBook(book1));
console.log(formatBook(book2));
```

</details>

## 問題2：readonlyプロパティ（基本）

座標を表す `Point` インターフェースを定義してください。`x` と `y` はどちらも `readonly` にします。

1. `Point` インターフェースを定義
2. 2つの座標 `p1` と `p2` を作成
3. 2点間の距離を計算する関数 `distance` を実装

**期待される出力：**
```
点1: (3, 4)
点2: (7, 1)
距離: 5.00
```

<details>
<summary>ヒント</summary>

- `readonly` を付けると、プロパティの変更が禁止されます。
- 2点間の距離: `Math.sqrt((x2-x1)^2 + (y2-y1)^2)`

</details>

<details>
<summary>解答例</summary>

```typescript
// readonlyプロパティを持つインターフェース
interface Point {
  readonly x: number;
  readonly y: number;
}

const p1: Point = { x: 3, y: 4 };
const p2: Point = { x: 7, y: 1 };

// p1.x = 10; // エラー！readonlyプロパティは変更できない

// 2点間の距離を計算
function distance(a: Point, b: Point): number {
  return Math.sqrt((b.x - a.x) ** 2 + (b.y - a.y) ** 2);
}

console.log(`点1: (${p1.x}, ${p1.y})`);
console.log(`点2: (${p2.x}, ${p2.y})`);
console.log(`距離: ${distance(p1, p2).toFixed(2)}`);
```

</details>

## 問題3：インターフェースの拡張（基本）

以下のインターフェースの継承関係を実装してください。

1. `Animal` インターフェース：`name: string`, `age: number`
2. `Pet` インターフェース：`Animal` を拡張し、`owner: string` を追加
3. `Dog` インターフェース：`Pet` を拡張し、`breed: string` を追加

それぞれのインターフェースに従ったオブジェクトを作成し、情報を表示してください。

**期待される出力：**
```
ポチは柴犬の3歳で、飼い主は山田さんです。
```

<details>
<summary>解答例</summary>

```typescript
// 基本のインターフェース
interface Animal {
  name: string;
  age: number;
}

// Animalを拡張
interface Pet extends Animal {
  owner: string;
}

// Petをさらに拡張
interface Dog extends Pet {
  breed: string;
}

// Dogインターフェースに従ったオブジェクト
const myDog: Dog = {
  name: "ポチ",
  age: 3,
  owner: "山田",
  breed: "柴犬",
};

console.log(
  `${myDog.name}は${myDog.breed}の${myDog.age}歳で、飼い主は${myDog.owner}さんです。`
);
```

</details>

## 問題4：型エイリアスの活用（応用）

以下の型エイリアスを定義してください。

1. `ID` 型：`string` または `number`
2. `Status` 型：`"active"` | `"inactive"` | `"pending"` のリテラルユニオン
3. `User` 型：`{ id: ID; name: string; status: Status }` のオブジェクト型

さらに、ユーザーの配列からステータスが `"active"` のユーザーだけを取得する関数を実装してください。

**期待される出力：**
```
全ユーザー: 太郎(active), 花子(inactive), 次郎(active), 美咲(pending)
アクティブユーザー: 太郎, 次郎
```

<details>
<summary>ヒント</summary>

- `type` キーワードで型エイリアスを定義します。
- ユニオン型は `|` で結合します。
- `filter()` で配列を絞り込めます。

</details>

<details>
<summary>解答例</summary>

```typescript
// 型エイリアスの定義
type ID = string | number;
type Status = "active" | "inactive" | "pending";

type User = {
  id: ID;
  name: string;
  status: Status;
};

// ユーザーデータ
const users: User[] = [
  { id: 1, name: "太郎", status: "active" },
  { id: "u-002", name: "花子", status: "inactive" },
  { id: 3, name: "次郎", status: "active" },
  { id: "u-004", name: "美咲", status: "pending" },
];

// アクティブユーザーを取得
function getActiveUsers(users: User[]): User[] {
  return users.filter((user) => user.status === "active");
}

// 表示
const allStr = users.map((u) => `${u.name}(${u.status})`).join(", ");
console.log(`全ユーザー: ${allStr}`);

const activeUsers = getActiveUsers(users);
console.log(`アクティブユーザー: ${activeUsers.map((u) => u.name).join(", ")}`);
```

</details>

## 問題5：インデックスシグネチャ（応用）

英和辞書を表す `Dictionary` インターフェースを定義してください。

- キーは英単語（`string`）、値は日本語訳（`string`）
- 辞書に単語を追加する関数 `addWord`
- 辞書から単語を検索する関数 `lookup`（見つからない場合は `"未登録"` を返す）

**期待される出力：**
```
apple → りんご
banana → バナナ
cherry → さくらんぼ
grape → 未登録
登録単語数: 3
```

<details>
<summary>ヒント</summary>

- インデックスシグネチャは `[key: string]: string` のように定義します。
- `Object.keys()` で登録数を取得できます。

</details>

<details>
<summary>解答例</summary>

```typescript
// インデックスシグネチャを持つインターフェース
interface Dictionary {
  [word: string]: string;
}

// 辞書に単語を追加
function addWord(dict: Dictionary, word: string, meaning: string): void {
  dict[word] = meaning;
}

// 辞書から検索
function lookup(dict: Dictionary, word: string): string {
  return dict[word] ?? "未登録";
}

// 辞書の作成と操作
const dictionary: Dictionary = {};

addWord(dictionary, "apple", "りんご");
addWord(dictionary, "banana", "バナナ");
addWord(dictionary, "cherry", "さくらんぼ");

// 検索
console.log(`apple → ${lookup(dictionary, "apple")}`);
console.log(`banana → ${lookup(dictionary, "banana")}`);
console.log(`cherry → ${lookup(dictionary, "cherry")}`);
console.log(`grape → ${lookup(dictionary, "grape")}`);

console.log(`登録単語数: ${Object.keys(dictionary).length}`);
```

</details>

## 問題6：交差型による拡張（応用）

型エイリアスの交差型（`&`）を使って、以下の型を合成してください。

1. `Timestamped` 型：`createdAt: Date`, `updatedAt: Date`
2. `HasId` 型：`id: number`
3. `Article` 型：`HasId & Timestamped & { title: string; body: string; published: boolean }`

`Article` 型のオブジェクトを作成し、情報を表示してください。

**期待される出力：**
```
記事 #1: TypeScriptの基本
作成日: 2026-01-15
更新日: 2026-03-20
状態: 公開済み
```

<details>
<summary>解答例</summary>

```typescript
// 共通の型を定義
type Timestamped = {
  createdAt: Date;
  updatedAt: Date;
};

type HasId = {
  id: number;
};

// 交差型で合成
type Article = HasId & Timestamped & {
  title: string;
  body: string;
  published: boolean;
};

// Article型のオブジェクトを作成
const article: Article = {
  id: 1,
  title: "TypeScriptの基本",
  body: "TypeScriptはJavaScriptのスーパーセットです...",
  published: true,
  createdAt: new Date("2026-01-15"),
  updatedAt: new Date("2026-03-20"),
};

// 表示
console.log(`記事 #${article.id}: ${article.title}`);
console.log(`作成日: ${article.createdAt.toISOString().split("T")[0]}`);
console.log(`更新日: ${article.updatedAt.toISOString().split("T")[0]}`);
console.log(`状態: ${article.published ? "公開済み" : "下書き"}`);
```

</details>

## 問題7：interface と type の使い分け（チャレンジ）

以下の要件を `interface` と `type` を使い分けて実装してください。

1. **interface** で定義: `Product`（オブジェクトの形状定義）
   - `id: number`, `name: string`, `price: number`
2. **type** で定義: `SortOrder`（ユニオン型） → `"asc" | "desc"`
3. **type** で定義: `ProductComparator`（関数型） → `(a: Product, b: Product) => number`
4. 関数 `sortProducts`: `Product[]` と `SortOrder` を受け取り、価格でソートして返す

**期待される出力：**
```
=== 昇順 ===
コーヒー: ¥300
紅茶: ¥350
ケーキ: ¥500

=== 降順 ===
ケーキ: ¥500
紅茶: ¥350
コーヒー: ¥300
```

<details>
<summary>ヒント</summary>

- オブジェクトの形状は `interface`、ユニオン型と関数型は `type` が推奨されます。
- `Array.sort()` はコンパレータ関数を受け取ります。

</details>

<details>
<summary>解答例</summary>

```typescript
// オブジェクトの形状 → interface
interface Product {
  id: number;
  name: string;
  price: number;
}

// ユニオン型 → type
type SortOrder = "asc" | "desc";

// 関数型 → type
type ProductComparator = (a: Product, b: Product) => number;

// ソート関数
function sortProducts(products: Product[], order: SortOrder): Product[] {
  // コンパレータ関数を作成
  const comparator: ProductComparator = (a, b) => {
    return order === "asc" ? a.price - b.price : b.price - a.price;
  };

  // 元の配列を変更しないようスプレッド構文でコピー
  return [...products].sort(comparator);
}

// 商品データ
const products: Product[] = [
  { id: 1, name: "ケーキ", price: 500 },
  { id: 2, name: "コーヒー", price: 300 },
  { id: 3, name: "紅茶", price: 350 },
];

// 昇順ソート
console.log("=== 昇順 ===");
sortProducts(products, "asc").forEach((p) => {
  console.log(`${p.name}: ¥${p.price}`);
});

// 降順ソート
console.log("\n=== 降順 ===");
sortProducts(products, "desc").forEach((p) => {
  console.log(`${p.name}: ¥${p.price}`);
});
```

</details>

## 問題8：APIレスポンス型の設計（チャレンジ）

以下の型を設計し、ダミーデータで動作確認してください。

1. `ApiResponse<T>` インターフェース
   - `status: number`
   - `message: string`
   - `data: T`
2. `PaginationInfo` インターフェース
   - `currentPage: number`, `totalPages: number`, `totalItems: number`
3. `PaginatedResponse<T>` インターフェース
   - `ApiResponse<T[]>` を拡張し、`pagination: PaginationInfo` を追加

ユーザー一覧のページネーション付きレスポンスのダミーデータを作成して表示してください。

**期待される出力：**
```
ステータス: 200 OK
ユーザー一覧（ページ 1/3, 全30件）:
  - 太郎 (taro@example.com)
  - 花子 (hanako@example.com)
```

<details>
<summary>解答例</summary>

```typescript
// ジェネリックなAPIレスポンス
interface ApiResponse<T> {
  status: number;
  message: string;
  data: T;
}

// ページネーション情報
interface PaginationInfo {
  currentPage: number;
  totalPages: number;
  totalItems: number;
}

// ページネーション付きレスポンス（ApiResponseを拡張）
interface PaginatedResponse<T> extends ApiResponse<T[]> {
  pagination: PaginationInfo;
}

// ユーザー型
interface User {
  id: number;
  name: string;
  email: string;
}

// ダミーレスポンスデータ
const response: PaginatedResponse<User> = {
  status: 200,
  message: "OK",
  data: [
    { id: 1, name: "太郎", email: "taro@example.com" },
    { id: 2, name: "花子", email: "hanako@example.com" },
  ],
  pagination: {
    currentPage: 1,
    totalPages: 3,
    totalItems: 30,
  },
};

// レスポンスの表示
console.log(`ステータス: ${response.status} ${response.message}`);
const { currentPage, totalPages, totalItems } = response.pagination;
console.log(`ユーザー一覧（ページ ${currentPage}/${totalPages}, 全${totalItems}件）:`);
response.data.forEach((user) => {
  console.log(`  - ${user.name} (${user.email})`);
});
```

</details>
