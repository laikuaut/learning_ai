5人のチームで開発するフルスタックWebアプリケーションに対して、Claude Codeを最大限活用した開発環境を構築します。

```

## 課題の要件

### 必須要件（すべて実装すること）

1. **CLAUDE.md設計**（3ファイル以上）
   - プロジェクトルートのCLAUDE.md
   - frontend用のCLAUDE.md
   - backend用のCLAUDE.md
   
2. **設定ファイル**
   - `.claude/settings.json`（チーム共有設定）
   - `.claude/settings.local.json.example`（個人設定テンプレート）
   
3. **フック設定**（4つ以上）
   - PreToolUse: 危険操作の防止（最低2つ）
   - PostToolUse: コード品質の自動チェック（最低1つ）
   - Stop: セッション終了時の処理（最低1つ）
   
4. **MCP設定**（2つ以上）
   - GitHub MCP
   - その他1つ以上（filesystem、postgres等）
   
5. **自動化スクリプト**（2つ以上）
   - 新メンバー用セットアップスクリプト
   - PRレビューまたはドキュメント生成スクリプト
   
6. **Git連携設定**
   - .gitignoreの適切な設定
   - ブランチ保護のフック

### プロジェクトシナリオ

**オンライン学習プラットフォーム**を開発するプロジェクトです。

| 項目 | 内容 |
|---|---|
| プロジェクト名 | LearnFlow |
| チーム | 5人（フロントエンド2名、バックエンド2名、フルスタック1名） |
| Frontend | React 18 + TypeScript + Vite |
| Backend | Python FastAPI + SQLAlchemy |
| Database | PostgreSQL 15 |
| Cache | Redis |
| Test | Vitest (frontend), pytest (backend) |
| CI/CD | GitHub Actions |
| パッケージ管理 | pnpm (frontend), uv (backend) |

# LearnFlow - オンライン学習プラットフォーム

## 概要
動画講座、クイズ、進捗管理を提供するオンライン学習プラットフォームです。
フロントエンド（React）とバックエンド（FastAPI）のモノレポ構成です。

## 技術スタック
### Frontend
- React 18 + TypeScript + Vite
- 状態管理: Zustand
- UIライブラリ: Tailwind CSS
- テスト: Vitest + Testing Library
- パッケージ管理: pnpm

### Backend
- Python 3.12 + FastAPI
- ORM: SQLAlchemy 2.0
- DB: PostgreSQL 15
- Cache: Redis
- テスト: pytest
- パッケージ管理: uv

### 共通
- CI/CD: GitHub Actions
- コンテナ: Docker Compose（開発環境）

## モノレポ構成
```
packages/
├── frontend/     # React SPA（ブラウザで動作）
├── backend/      # FastAPI（REST API提供）
└── shared/       # 共有型定義（TypeScript）
```

## 開発コマンド
- `pnpm dev` : 全サービス起動（フロントエンド + バックエンド）
- `pnpm test` : 全テスト実行
- `pnpm test:frontend` : フロントエンドテストのみ
- `pnpm test:backend` : バックエンドテストのみ（pytest）
- `pnpm lint` : 全パッケージのリント
- `pnpm build` : プロダクションビルド

## コーディング規約

### 共通
- コミットメッセージ: Conventional Commits必須（feat:, fix:, chore:, docs:, refactor:, test:）
- PR: テンプレートに従って説明を記載、1名以上のレビュー承認必須
- `/commit` コマンドでAIにメッセージ生成を推奨

### Frontend (TypeScript/React)
- 命名: camelCase (変数/関数), PascalCase (コンポーネント/型)
- コンポーネント: 関数コンポーネント + Hooks（classコンポーネント禁止）
- スタイル: Tailwind CSSのユーティリティクラスを使用
- 状態管理: ローカル状態はuseState、グローバル状態はZustand

### Backend (Python/FastAPI)
- 命名: snake_case (変数/関数), PascalCase (クラス), UPPER_SNAKE_CASE (定数)
- 型ヒント: すべての関数に型アノテーション必須
- API設計: RESTful、レスポンスは `{"success": bool, "data": any, "error": str?}` 形式
- DBアクセス: SQLAlchemy ORMを通す（生SQLは原則禁止）

## 重要なルール
- `packages/shared/` の変更はfrontend・backend両方に影響するため、両方のテストを実行すること
- `.env.production` は絶対に変更しない（インフラチームが管理）
- DBマイグレーションは `alembic upgrade head` で行い、手動SQLは禁止
- 認証関連の変更（`packages/backend/src/api/auth.py`）は必ずセキュリティレビューを受けること
- パフォーマンスに影響する変更はPRに明記すること
- セッションが長くなったら `/compact` でコンテキストを圧縮すること
MDEOF
```

cat > scripts/setup-claude.sh << 'SCRIPT'
#!/bin/bash
set -euo pipefail

echo "===== LearnFlow - Claude Code セットアップ ====="
echo ""

# 1. 前提条件チェック
echo "[1/5] 前提条件をチェック中..."

check_command() {
    if command -v "$1" &> /dev/null; then
        echo "  ✓ $1 ($(eval "$2" 2>/dev/null || echo 'バージョン不明'))"
        return 0
    else
        echo "  ✗ $1 が見つかりません"
        return 1
    fi
}

MISSING=0
check_command "node" "node --version" || MISSING=1
check_command "pnpm" "pnpm --version" || MISSING=1
check_command "python" "python --version" || MISSING=1
check_command "claude" "claude --version" || MISSING=1
check_command "gh" "gh --version | head -1" || MISSING=1

if [ "$MISSING" -eq 1 ]; then
    echo ""
    echo "ERROR: 不足しているツールをインストールしてください"
    exit 1
fi

# 2. ローカル設定ファイルの作成
echo ""
echo "[2/5] ローカル設定ファイルを作成中..."

if [ ! -f .claude/settings.local.json ]; then
    cp .claude/settings.local.json.example .claude/settings.local.json
    echo "  .claude/settings.local.json を作成しました"
    echo "  → GitHubトークンとDB接続情報を設定してください"
else
    echo "  .claude/settings.local.json は既に存在します（スキップ）"
fi

# 3. 依存パッケージのインストール
echo ""
echo "[3/5] 依存パッケージをインストール中..."
pnpm install 2>/dev/null || echo "  （pnpm installをスキップ）"

if [ -f packages/backend/pyproject.toml ]; then
    cd packages/backend
    uv sync 2>/dev/null || echo "  （uv syncをスキップ）"
    cd ../..
fi

# 4. Claude Code動作確認
echo ""
echo "[4/5] Claude Codeの動作を確認中..."
CLAUDE_TEST=$(claude -p "こんにちは。1+1は？" --max-turns 1 2>/dev/null || echo "FAILED")
if echo "$CLAUDE_TEST" | grep -qi "2"; then
    echo "  ✓ Claude Codeが正常に動作しています"
else
    echo "  ✗ Claude Codeの動作確認に失敗しました"
    echo "  → APIキーまたはOAuth認証を確認してください"
fi

# 5. MCP接続確認
echo ""
echo "[5/5] MCP設定を確認中..."
claude mcp list 2>/dev/null || echo "  MCPサーバーの一覧を取得できませんでした"

echo ""
echo "===== セットアップ完了 ====="
echo ""
echo "次のステップ:"
echo "  1. .claude/settings.local.json のトークンとDB情報を設定してください"
echo "  2. 'claude' コマンドで起動して動作確認してください"
echo "  3. '/mcp' コマンドでMCPサーバーの接続を確認してください"
SCRIPT

**PRレビュースクリプト:**

```bash
cat > scripts/review-pr.sh << 'SCRIPT'
#!/bin/bash
set -euo pipefail

PR_NUMBER=${1:?'使い方: ./scripts/review-pr.sh <PR番号>'}
REPO=${GITHUB_REPOSITORY:-$(gh repo view --json nameWithOwner -q '.nameWithOwner' 2>/dev/null || echo '')}

if [ -z "$REPO" ]; then
    echo "ERROR: リポジトリ情報を取得できませんでした"
    exit 1
fi

echo "=== PR #${PR_NUMBER} の自動レビュー ==="

# PR情報取得
PR_TITLE=$(gh pr view "$PR_NUMBER" --repo "$REPO" --json title -q '.title' 2>/dev/null || echo '不明')
DIFF=$(gh pr diff "$PR_NUMBER" --repo "$REPO" 2>/dev/null | head -n 500)
FILES=$(gh pr view "$PR_NUMBER" --repo "$REPO" --json files -q '.files[].path' 2>/dev/null | head -20)

if [ -z "$DIFF" ]; then
    echo "ERROR: PRの差分を取得できませんでした"
    exit 1
fi

# レビュー実行
REVIEW=$(echo "$DIFF" | claude -p "
あなたはLearnFlowプロジェクトのシニアエンジニアです。以下のPRをレビューしてください。

タイトル: ${PR_TITLE}
変更ファイル:
${FILES}

レビュー観点:
1. バグの可能性（ロジックエラー、エッジケース）
2. セキュリティ（認証漏れ、インジェクション）
3. パフォーマンス（N+1クエリ、不要な再レンダリング）
4. コーディング規約への準拠
5. テストの十分性
6. sharedパッケージ変更の影響範囲

出力は日本語で、以下の形式でお願いします：
### 概要（1-2行）
### 指摘事項（重要度付き）
### 良い点
### 総合判定（LGTM / 修正推奨 / 要修正）
" --max-turns 3 --output-format text 2>/dev/null) || {
    echo "ERROR: レビューの生成に失敗しました"
    exit 1
}

# 結果出力
echo "$REVIEW"

# PRにコメント（--post-commentオプションがある場合）
if [ "${POST_COMMENT:-false}" = "true" ]; then
    gh pr comment "$PR_NUMBER" --repo "$REPO" --body "## 自動コードレビュー

${REVIEW}

---
*Claude Codeによる自動レビュー（$(date '+%Y-%m-%d %H:%M')）*"
    echo "=== PRにコメントを投稿しました ==="
fi
SCRIPT

chmod +x scripts/review-pr.sh
```

name: Claude Code PR Review

on:
  pull_request:

permissions:
  contents: read
  pull-requests: write

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          GITHUB_REPOSITORY: ${{ github.repository }}
          POST_COMMENT: 'true'
        run: ./scripts/review-pr.sh ${{ github.event.pull_request.number }}
        timeout-minutes: 5

<summary>解答例（初心者向け）── 最低限の設定で始める</summary>

最低限の設定として、以下の3ファイルから始められます。

```bash
# 1. CLAUDE.md（プロジェクト概要のみ）
cat > CLAUDE.md << 'EOF'
# LearnFlow

## 概要
オンライン学習プラットフォーム。React + FastAPI。

## コマンド
- `pnpm dev` : 開発サーバー
- `pnpm test` : テスト

## ルール
- mainへの直接push禁止
- .envはコミットしない
EOF

# 2. settings.json（基本的なフック）
mkdir -p .claude
cat > .claude/settings.json << 'EOF'
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "command": "echo 'ファイルが変更されました: $CLAUDE_FILE_PATH'"
      }
    ]
  }
}
EOF

# 3. .gitignore
cat > .gitignore << 'EOF'
node_modules/
__pycache__/
.env
.claude/settings.local.json
EOF
```

初心者はまずこの3ファイルから始めて、チームの開発フローに慣れてきたら段階的にフック、MCP、CIスクリプトを追加していくのがおすすめです。

<details>
<summary>解答例（改良版）── プロダクション品質の統合設計</summary>

改良版はステップ2〜6のすべての設定を含む完全版です。各ファイルの要点を整理します。

### 設計のポイント

**1. CLAUDE.mdの階層設計:**

```
CLAUDE.md（ルート）
  → プロジェクト全体の概要、技術スタック、コーディング規約
  → チーム全員が読む「プロジェクトの憲法」

packages/frontend/CLAUDE.md
  → React/TypeScript固有のルール（camelCase、関数コンポーネント必須等）
  → フロントエンド担当が主に参照

packages/backend/CLAUDE.md
  → Python/FastAPI固有のルール（snake_case、型ヒント必須等）
  → バックエンド担当が主に参照
```

階層化することで、Claude Codeは作業中のディレクトリに応じた適切なルールを適用します。

**2. フックの多層防御:**

```
PreToolUse（防御層）:
  ├── 破壊的コマンドのブロック（rm -rf, DROP TABLE等）
  ├── mainブランチへの直接push防止
  └── 機密ファイルの変更防止

PostToolUse（品質層）:
  ├── TypeScript/JavaScript → ESLint自動実行
  ├── Python → 構文チェック自動実行
  └── shared変更時 → テスト推奨の警告表示

Stop（レポート層）:
  └── 未コミット変更の一覧表示
```

**3. 設定のスコープ分離:**

```
.claude/settings.json（チーム共有、Gitにコミット）
  → MCPサーバー名、フック設定
  → 全員に適用される共通ルール

.claude/settings.local.json（個人、Gitに含めない）
  → GitHubトークン、DB接続情報
  → .gitignoreに登録

settings.local.json.example（テンプレート、Gitにコミット）
  → 新メンバーがコピーして使う
```

**4. 自動化の段階的な導入:**

| 段階 | 内容 | ファイル |
|---|---|---|
| 第1段階 | CLAUDE.md + 基本フック | CLAUDE.md, settings.json |
| 第2段階 | MCP連携 + ローカル設定分離 | settings.local.json |
| 第3段階 | セットアップ自動化 | scripts/setup-claude.sh |
| 第4段階 | CI/CDへの統合 | .github/workflows/, scripts/review-pr.sh |

**初心者向けとの違い:**
- 初心者向けは「最小限で動く」ことを目指し、3ファイルでスタートします
- 改良版は「チーム全体の生産性を最大化する」ことを目指し、CLAUDE.mdの階層化、多層フック、MCP連携、CI/CD統合、オンボーディング自動化まで含みます
- 実務では第1段階から始めて、チームの成熟度に応じて段階的に改良版に近づけていくのが現実的です
