# 第5章：ファイル転送（SCP・SFTP）

## この章のゴール
- SCPとSFTPの違いと使い分けを理解する
- SCPでファイルやディレクトリを転送できるようになる
- SFTPの対話的操作をマスターする
- rsync over SSH で効率的な同期ができるようになる

---

## 5.1 SSH経由のファイル転送の概要

SSH経由でファイルを転送する方法は複数あります。

| 方法 | 特徴 | 推奨度 |
|------|------|--------|
| **SFTP** | 対話的操作、レジューム対応、現在の推奨 | ★★★ |
| **rsync** | 差分転送、大量ファイルの同期に最適 | ★★★ |
| **SCP** | シンプルだが機能が限定的、レガシー | ★★ |

> **注意**: OpenSSH 9.0 以降、SCP はデフォルトで内部的に SFTP プロトコルを使用するようになりました。ただし従来のSCPプロトコルも `-O` オプションで利用可能です。

---

## 5.2 SCP（Secure Copy）

### 基本構文

```bash
scp [オプション] コピー元 コピー先
```

コピー元・コピー先の指定形式：
```
ローカルファイル:  /path/to/file
リモートファイル:  ユーザー名@ホスト名:/path/to/file
```

### ファイルのコピー

```bash
# ローカル → リモート（アップロード）
$ scp myfile.txt user@example.com:/home/user/
$ scp myfile.txt user@example.com:~/         # ~ はホームディレクトリ
$ scp myfile.txt user@example.com:~/newname.txt  # リネームしてコピー

# リモート → ローカル（ダウンロード）
$ scp user@example.com:/home/user/data.csv ./
$ scp user@example.com:~/data.csv ./local_data.csv  # リネームしてコピー

# リモート → リモート（サーバー間コピー）
$ scp user@server1:/path/file user@server2:/path/
```

### ディレクトリのコピー

```bash
# ディレクトリを再帰的にコピー（-r オプション）
$ scp -r myproject/ user@example.com:~/projects/

# リモートからディレクトリをダウンロード
$ scp -r user@example.com:~/projects/myapp ./
```

### よく使うオプション

```bash
# -P: ポート番号を指定（大文字の P に注意！sshは小文字の -p）
$ scp -P 2222 myfile.txt user@example.com:~/

# -i: 秘密鍵を指定
$ scp -i ~/.ssh/id_ed25519_work myfile.txt user@example.com:~/

# -C: 通信を圧縮
$ scp -C largefile.tar user@example.com:~/

# -l: 帯域幅を制限（Kbit/s）
$ scp -l 1000 largefile.iso user@example.com:~/  # 約125KB/sに制限

# -p: タイムスタンプとパーミッションを保持
$ scp -p myfile.txt user@example.com:~/

# -q: プログレス表示を抑制
$ scp -q myfile.txt user@example.com:~/

# 複数オプションの組み合わせ
$ scp -rpC -P 2222 myproject/ user@example.com:~/
```

### SCPのオプション一覧

| オプション | 説明 |
|-----------|------|
| `-r` | ディレクトリを再帰的にコピー |
| `-P` | ポート番号を指定（大文字に注意） |
| `-i` | 秘密鍵を指定 |
| `-C` | 圧縮を有効化 |
| `-l` | 帯域幅を制限（Kbit/s） |
| `-p` | タイムスタンプとパーミッションを保持 |
| `-q` | プログレス非表示 |
| `-v` | 詳細表示（デバッグ） |
| `-O` | 従来のSCPプロトコルを使用 |

## 5.3 SFTP（SSH File Transfer Protocol）

SFTPはSSH上で動作する対話型のファイル転送プロトコルです。FTPに似た操作感で、より多機能です。

### SFTP セッションの開始

```bash
# 接続
$ sftp user@example.com
Connected to example.com.
sftp>

# ポート番号を指定
$ sftp -P 2222 user@example.com

# ~/.ssh/config のエイリアスを使用
$ sftp dev
```

### SFTP の基本コマンド

```bash
sftp> help   # コマンド一覧を表示

# --- ファイル操作（リモート側） ---
sftp> ls                    # ファイル一覧
sftp> ls -la                # 詳細一覧
sftp> cd /var/log           # ディレクトリ移動
sftp> pwd                   # 現在のディレクトリ
sftp> mkdir newdir          # ディレクトリ作成
sftp> rmdir emptydir        # 空ディレクトリ削除
sftp> rm oldfile.txt        # ファイル削除
sftp> rename old.txt new.txt # リネーム
sftp> chmod 644 file.txt    # パーミッション変更

# --- ファイル操作（ローカル側） ---
sftp> lls                   # ローカルのファイル一覧
sftp> lcd /tmp              # ローカルのディレクトリ移動
sftp> lpwd                  # ローカルの現在ディレクトリ
sftp> lmkdir localdir       # ローカルにディレクトリ作成

# --- ファイル転送 ---
sftp> get remote_file.txt                    # ダウンロード
sftp> get remote_file.txt local_name.txt     # リネームしてダウンロード
sftp> get -r remote_dir/                     # ディレクトリをダウンロード
sftp> put local_file.txt                     # アップロード
sftp> put local_file.txt remote_name.txt     # リネームしてアップロード
sftp> put -r local_dir/                      # ディレクトリをアップロード

# --- セッション終了 ---
sftp> exit   # または quit, bye
```

### SFTP コマンド一覧

| コマンド | 説明 | 対象 |
|---------|------|------|
| `ls` / `dir` | ファイル一覧 | リモート |
| `cd` | ディレクトリ移動 | リモート |
| `pwd` | 現在のディレクトリ | リモート |
| `mkdir` | ディレクトリ作成 | リモート |
| `rm` | ファイル削除 | リモート |
| `rename` | リネーム | リモート |
| `chmod` | パーミッション変更 | リモート |
| `lls` | ファイル一覧 | ローカル |
| `lcd` | ディレクトリ移動 | ローカル |
| `lpwd` | 現在のディレクトリ | ローカル |
| `get` | ダウンロード | リモート→ローカル |
| `put` | アップロード | ローカル→リモート |
| `!command` | ローカルでコマンド実行 | ローカル |

### SFTP バッチモード

対話的操作ではなく、バッチファイルで自動実行することもできます。

```bash
# バッチファイルを作成
$ cat > sftp_commands.txt << 'EOF'
cd /var/www/html
put index.html
put style.css
put script.js
ls -la
EOF

# バッチモードで実行
$ sftp -b sftp_commands.txt user@example.com

# エラーがあっても続行する場合
$ sftp -b sftp_commands.txt user@example.com || true
```

## 5.4 rsync over SSH

rsyncは**差分転送**を行うため、大量のファイルや大きなファイルの同期に最適です。

### 基本構文

```bash
rsync [オプション] コピー元 コピー先
```

### 基本的な使い方

```bash
# ローカル → リモート（アップロード）
$ rsync -avz myproject/ user@example.com:~/projects/myproject/

# リモート → ローカル（ダウンロード）
$ rsync -avz user@example.com:~/projects/myproject/ ./myproject/

# ドライラン（実際にはコピーしない、何が転送されるか確認）
$ rsync -avzn myproject/ user@example.com:~/projects/myproject/
```

### よく使うオプション

| オプション | 説明 |
|-----------|------|
| `-a` | アーカイブモード（-rlptgoD と同等、パーミッション等を保持） |
| `-v` | 詳細表示 |
| `-z` | 転送時に圧縮 |
| `-n` | ドライラン（テスト実行） |
| `--delete` | コピー元にないファイルをコピー先から削除 |
| `--exclude` | 除外パターンを指定 |
| `-P` | プログレス表示＋中断時の再開対応 |
| `-e` | 使用するリモートシェルを指定 |

### 実践的な使用例

```bash
# プロジェクトの同期（.gitやnode_modulesを除外）
$ rsync -avz --exclude '.git' --exclude 'node_modules' \
    myproject/ user@example.com:~/projects/myproject/

# SSHポートを指定
$ rsync -avz -e "ssh -p 2222" myproject/ user@example.com:~/projects/

# 特定の秘密鍵を使用
$ rsync -avz -e "ssh -i ~/.ssh/id_ed25519_work" \
    myproject/ user@example.com:~/projects/myproject/

# ミラーリング（コピー先をコピー元と完全に同じにする）
$ rsync -avz --delete myproject/ user@example.com:~/projects/myproject/

# 帯域幅を制限（KB/s）
$ rsync -avz --bwlimit=500 largedir/ user@example.com:~/backup/

# プログレス表示付き
$ rsync -avzP largefile.iso user@example.com:~/
largefile.iso
    524,288,000  45%   12.34MB/s    0:00:25
```

### rsync のトレイリングスラッシュの注意

```bash
# スラッシュあり: ディレクトリの中身をコピー
$ rsync -avz mydir/ user@example.com:~/dest/
# 結果: dest/file1, dest/file2 ...

# スラッシュなし: ディレクトリ自体をコピー
$ rsync -avz mydir user@example.com:~/dest/
# 結果: dest/mydir/file1, dest/mydir/file2 ...
```

> **よくある間違い**: rsyncのコピー元のスラッシュ有無で結果が変わります。意図しないディレクトリ構造にならないよう注意しましょう。

## 5.5 SCP・SFTP・rsync の比較

| 機能 | SCP | SFTP | rsync |
|------|-----|------|-------|
| 対話的操作 | × | ○ | × |
| 差分転送 | × | × | ○ |
| レジューム（中断再開） | × | △ | ○ |
| ディレクトリ一覧 | × | ○ | × |
| シンボリックリンク対応 | △ | ○ | ○ |
| 帯域制限 | ○ | × | ○ |
| 除外パターン | × | × | ○ |
| バッチ処理 | × | ○ | ○ |
| 速度（大量ファイル） | 遅い | 普通 | 速い |

### 使い分けの目安

```
1ファイルの簡単なコピー          → SCP
対話的にファイルを操作したい      → SFTP
大量のファイルを効率的に同期      → rsync
定期的なバックアップ             → rsync
初回は全転送、以降は差分のみ     → rsync
```

## 5.6 SSH経由のtar転送

SCPやSFTPを使わず、tarとSSHを組み合わせてディレクトリを転送する方法もあります。

```bash
# ディレクトリをtarで圧縮しながらSSH経由で転送
$ tar czf - myproject/ | ssh user@example.com "tar xzf - -C ~/projects/"

# リモートからローカルにディレクトリを取得
$ ssh user@example.com "tar czf - -C ~/projects myproject/" | tar xzf -

# 進捗を表示（pvが必要）
$ tar czf - myproject/ | pv | ssh user@example.com "tar xzf - -C ~/projects/"
```

> **ポイント**: tar + SSH は中間ファイルを作成せず、パイプで直接転送するため、ディスク容量が少ない場合に便利です。

---

## ポイントまとめ

```
1. SFTP が現在の推奨ファイル転送方法（対話的操作も可能）
2. rsync は差分転送で大量ファイルの同期に最適
3. SCP はシンプルだが機能が限定的（レガシー扱い）
4. SCP の -P は大文字（ssh の -p は小文字）に注意
5. rsync のトレイリングスラッシュ（/）で挙動が変わる
6. SFTP のバッチモードで自動化が可能
7. tar + SSH でディスク容量を節約しながら転送できる
```
