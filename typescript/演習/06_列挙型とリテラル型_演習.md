## 問題1：文字列enum（基本）

ログレベルを表す文字列enum `LogLevel` を定義し、ログ出力関数を実装してください。

- `Debug = "DEBUG"`, `Info = "INFO"`, `Warn = "WARN"`, `Error = "ERROR"`

`log(level, message)` 関数はフォーマットされたログメッセージを出力します。

**期待される出力：**
```
[DEBUG] デバッグ情報を表示
[INFO] アプリケーション起動
[WARN] メモリ使用量が高い
[ERROR] 接続に失敗しました
```

<details>
<summary>解答例</summary>

```typescript
// 文字列enum
enum LogLevel {
  Debug = "DEBUG",
  Info = "INFO",
  Warn = "WARN",
  Error = "ERROR",
}

function log(level: LogLevel, message: string): void {
  console.log(`[${level}] ${message}`);
}

log(LogLevel.Debug, "デバッグ情報を表示");
log(LogLevel.Info, "アプリケーション起動");
log(LogLevel.Warn, "メモリ使用量が高い");
log(LogLevel.Error, "接続に失敗しました");
```

</details>

## 問題2：数値enumの活用（基本）

HTTPステータスコードを数値enumで定義し、ステータスメッセージを返す関数を実装してください。

**期待される出力：**
```
200: OK - リクエスト成功
201: Created - リソースが作成されました
400: Bad Request - 不正なリクエスト
404: Not Found - リソースが見つかりません
500: Internal Server Error - サーバーエラー
```

<details>
<summary>解答例</summary>

```typescript
// 数値enum
enum HttpStatus {
  OK = 200,
  Created = 201,
  BadRequest = 400,
  NotFound = 404,
  InternalError = 500,
}

function getStatusMessage(status: HttpStatus): string {
  switch (status) {
    case HttpStatus.OK:
      return "OK - リクエスト成功";
    case HttpStatus.Created:
      return "Created - リソースが作成されました";
    case HttpStatus.BadRequest:
      return "Bad Request - 不正なリクエスト";
    case HttpStatus.NotFound:
      return "Not Found - リソースが見つかりません";
    case HttpStatus.InternalError:
      return "Internal Server Error - サーバーエラー";
  }
}

const statuses: HttpStatus[] = [
  HttpStatus.OK, HttpStatus.Created, HttpStatus.BadRequest,
  HttpStatus.NotFound, HttpStatus.InternalError,
];

statuses.forEach((s) => {
  console.log(`${s}: ${getStatusMessage(s)}`);
});
```

</details>

## 問題3：as const による定数定義（基本）

`as const` を使って、色の定数オブジェクトとそのユニオン型を定義してください。

1. `COLORS` オブジェクト（Red, Green, Blue のHEX値）
2. `Color` 型（値のユニオン型を `typeof` + `keyof` で抽出）
3. 色名からHEX値を返す関数

**期待される出力：**
```
Red → #FF0000
Green → #00FF00
Blue → #0000FF
```

<details>
<summary>ヒント</summary>

- `as const` でオブジェクトをリテラル型にします。
- `(typeof COLORS)[keyof typeof COLORS]` で値の型を取得できます。

</details>

<details>
<summary>解答例</summary>

```typescript
// as const で定数オブジェクトを定義
const COLORS = {
  Red: "#FF0000",
  Green: "#00FF00",
  Blue: "#0000FF",
} as const;

// 値のユニオン型を抽出
type Color = (typeof COLORS)[keyof typeof COLORS];
// "#FF0000" | "#00FF00" | "#0000FF"

// キーのユニオン型
type ColorName = keyof typeof COLORS;
// "Red" | "Green" | "Blue"

// 色名からHEX値を返す関数
function getHexColor(name: ColorName): Color {
  return COLORS[name];
}

// 表示
const colorNames: ColorName[] = ["Red", "Green", "Blue"];
colorNames.forEach((name) => {
  console.log(`${name} → ${getHexColor(name)}`);
});
```

</details>

## 問題4：テンプレートリテラル型（応用）

テンプレートリテラル型を使って、CSSの値を型安全に扱えるようにしてください。

1. `CSSUnit` 型：`"px" | "em" | "rem" | "%"`
2. `CSSValue` 型：`${number}${CSSUnit}`（テンプレートリテラル型）
3. `cssValue(value, unit)` 関数：CSSValue型の文字列を返す

**期待される出力：**
```
width: 100px
fontSize: 1.5rem
margin: 2em
opacity: 50%
```

<details>
<summary>解答例</summary>

```typescript
// テンプレートリテラル型
type CSSUnit = "px" | "em" | "rem" | "%";
type CSSValue = `${number}${CSSUnit}`;

// CSSValue型の文字列を生成
function cssValue(value: number, unit: CSSUnit): CSSValue {
  return `${value}${unit}` as CSSValue;
}

// 使用例
const width: CSSValue = cssValue(100, "px");
const fontSize: CSSValue = cssValue(1.5, "rem");
const margin: CSSValue = cssValue(2, "em");
const opacity: CSSValue = cssValue(50, "%");

console.log(`width: ${width}`);
console.log(`fontSize: ${fontSize}`);
console.log(`margin: ${margin}`);
console.log(`opacity: ${opacity}`);

// 型エラーの例（コメントを外すとエラー）
// const bad: CSSValue = "auto"; // エラー！パターンに合わない
```

</details>

## 問題5：定数配列からユニオン型を導出（応用）

`as const` を使って、サポートされる言語のリストからユニオン型を導出してください。

1. `SUPPORTED_LANGUAGES` 配列を `as const` で定義（`"ja"`, `"en"`, `"zh"`, `"ko"`）
2. `SupportedLanguage` 型を導出
3. `isSupported(lang)` 型ガード関数を実装
4. `getGreeting(lang)` 言語別挨拶関数を実装

**期待される出力：**
```
ja: こんにちは
en: Hello
zh: 你好
ko: 안녕하세요
fr はサポートされていません
```

<details>
<summary>解答例</summary>

```typescript
// as const で配列を定義
const SUPPORTED_LANGUAGES = ["ja", "en", "zh", "ko"] as const;

// 配列の要素からユニオン型を導出
type SupportedLanguage = (typeof SUPPORTED_LANGUAGES)[number];
// "ja" | "en" | "zh" | "ko"

// 型ガード関数
function isSupported(lang: string): lang is SupportedLanguage {
  return (SUPPORTED_LANGUAGES as readonly string[]).includes(lang);
}

// 言語別挨拶
function getGreeting(lang: SupportedLanguage): string {
  const greetings: Record<SupportedLanguage, string> = {
    ja: "こんにちは",
    en: "Hello",
    zh: "你好",
    ko: "안녕하세요",
  };
  return greetings[lang];
}

// 表示
SUPPORTED_LANGUAGES.forEach((lang) => {
  console.log(`${lang}: ${getGreeting(lang)}`);
});

// サポートされていない言語
const testLang = "fr";
if (isSupported(testLang)) {
  console.log(getGreeting(testLang));
} else {
  console.log(`${testLang} はサポートされていません`);
}
```

</details>

## 問題6：enum vs as const の比較（応用）

同じ「ユーザーロール」を `enum` と `as const` の両方で定義し、それぞれの使い方を比較してください。

1. `UserRoleEnum` — enum版
2. `USER_ROLES` — as const版
3. それぞれを使った権限チェック関数を実装

**期待される出力：**
```
=== enum版 ===
ADMIN → 管理者権限あり
USER → 一般ユーザー
GUEST → ゲストユーザー

=== as const版 ===
ADMIN → 管理者権限あり
USER → 一般ユーザー
GUEST → ゲストユーザー
```

<details>
<summary>解答例</summary>

```typescript
// === enum版 ===
enum UserRoleEnum {
  Admin = "ADMIN",
  User = "USER",
  Guest = "GUEST",
}

function describeRoleEnum(role: UserRoleEnum): string {
  switch (role) {
    case UserRoleEnum.Admin:
      return "管理者権限あり";
    case UserRoleEnum.User:
      return "一般ユーザー";
    case UserRoleEnum.Guest:
      return "ゲストユーザー";
  }
}

// === as const版 ===
const USER_ROLES = {
  Admin: "ADMIN",
  User: "USER",
  Guest: "GUEST",
} as const;

type UserRole = (typeof USER_ROLES)[keyof typeof USER_ROLES];

function describeRoleConst(role: UserRole): string {
  switch (role) {
    case USER_ROLES.Admin:
      return "管理者権限あり";
    case USER_ROLES.User:
      return "一般ユーザー";
    case USER_ROLES.Guest:
      return "ゲストユーザー";
  }
}

// enum版の出力
console.log("=== enum版 ===");
console.log(`${UserRoleEnum.Admin} → ${describeRoleEnum(UserRoleEnum.Admin)}`);
console.log(`${UserRoleEnum.User} → ${describeRoleEnum(UserRoleEnum.User)}`);
console.log(`${UserRoleEnum.Guest} → ${describeRoleEnum(UserRoleEnum.Guest)}`);

// as const版の出力
console.log("\n=== as const版 ===");
console.log(`${USER_ROLES.Admin} → ${describeRoleConst(USER_ROLES.Admin)}`);
console.log(`${USER_ROLES.User} → ${describeRoleConst(USER_ROLES.User)}`);
console.log(`${USER_ROLES.Guest} → ${describeRoleConst(USER_ROLES.Guest)}`);
```

</details>

## 問題7：イベントシステムの型定義（チャレンジ）

テンプレートリテラル型と `as const` を活用して、型安全なイベントシステムを設計してください。

1. イベントタイプを `as const` で定義
2. イベントハンドラ名をテンプレートリテラル型で自動生成（`on${Capitalize<EventType>}`）
3. イベント→ハンドラのマッピングオブジェクトを作成

**期待される出力：**
```
イベントハンドラ:
  click → onClick
  focus → onFocus
  blur → onBlur
  submit → onSubmit
```

<details>
<summary>解答例</summary>

```typescript
// イベントタイプの定数
const EVENT_TYPES = ["click", "focus", "blur", "submit"] as const;
type EventType = (typeof EVENT_TYPES)[number];

// テンプレートリテラル型でハンドラ名を生成
type EventHandlerName = `on${Capitalize<EventType>}`;
// "onClick" | "onFocus" | "onBlur" | "onSubmit"

// イベントとハンドラのマッピングを生成する関数
function createHandlerMap(events: readonly string[]): Record<string, string> {
  const map: Record<string, string> = {};
  events.forEach((event) => {
    const handlerName = `on${event.charAt(0).toUpperCase()}${event.slice(1)}`;
    map[event] = handlerName;
  });
  return map;
}

// マッピングの生成と表示
const handlerMap = createHandlerMap(EVENT_TYPES);

console.log("イベントハンドラ:");
EVENT_TYPES.forEach((event) => {
  console.log(`  ${event} → ${handlerMap[event]}`);
});
```

</details>

## 問題8：型安全なルーティング（チャレンジ）

`as const` とテンプレートリテラル型を使って、型安全なルーティングシステムを構築してください。

1. ルートの定数定義（`as const`）
2. `Route` 型（値のユニオン）
3. パスパラメータ付きルート（`/users/:id` など）の型定義
4. `navigate(route)` 関数

**期待される出力：**
```
ナビゲート: /
ナビゲート: /about
ナビゲート: /users/42
ナビゲート: /posts/hello-world
```

<details>
<summary>解答例</summary>

```typescript
// ルートの定義
const ROUTES = {
  HOME: "/",
  ABOUT: "/about",
  USERS: "/users",
  POSTS: "/posts",
} as const;

type StaticRoute = (typeof ROUTES)[keyof typeof ROUTES];

// パスパラメータ付きルート
type DynamicRoute =
  | `${typeof ROUTES.USERS}/${number}`
  | `${typeof ROUTES.POSTS}/${string}`;

// 全ルート
type Route = StaticRoute | DynamicRoute;

// ナビゲーション関数
function navigate(route: Route): void {
  console.log(`ナビゲート: ${route}`);
}

// ユーザーページのURL生成
function userPath(id: number): `${typeof ROUTES.USERS}/${number}` {
  return `${ROUTES.USERS}/${id}`;
}

// 投稿ページのURL生成
function postPath(slug: string): `${typeof ROUTES.POSTS}/${string}` {
  return `${ROUTES.POSTS}/${slug}`;
}

// 使用例
navigate(ROUTES.HOME);
navigate(ROUTES.ABOUT);
navigate(userPath(42));
navigate(postPath("hello-world"));

// navigate("/invalid"); // エラー！Route型に含まれない
```

</details>
