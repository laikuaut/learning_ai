### 問題1：書籍管理構造体

書籍情報を管理する構造体 `Book` を作成してください。以下のフィールドとメソッドを実装してください。

- フィールド：タイトル（String）、著者（String）、価格（u32）、在庫数（u32）
- `new` 関連関数：新しいBookを作成する
- `display_info` メソッド：書籍情報を表示する
- `is_available` メソッド：在庫があるかどうかを返す
- `apply_discount` メソッド：指定した割引率（%）を適用して価格を更新する

**期待される出力例：**
```
『Rustプログラミング入門』 著者: 山田太郎 / 3000円 (在庫: 5)
在庫あり: true
20%割引後: 2400円
『Rustプログラミング入門』 著者: 山田太郎 / 2400円 (在庫: 5)
```

<details><summary>ヒント</summary>

- `impl Book { ... }` でメソッドを定義します
- `new` は `Self` を返す関連関数です（`&self` を取りません）
- `&self` は不変参照、`&mut self` は可変参照のメソッドです
- 割引後の価格は `price * (100 - discount) / 100` で計算できます

</details>

<details><summary>解答例</summary>

```rust
// 書籍管理構造体

struct Book {
    title: String,
    author: String,
    price: u32,
    stock: u32,
}

impl Book {
    // コンストラクタ（関連関数）
    fn new(title: &str, author: &str, price: u32, stock: u32) -> Self {
        Book {
            title: title.to_string(),
            author: author.to_string(),
            price,
            stock,
        }
    }

    // 書籍情報を表示
    fn display_info(&self) {
        println!(
            "『{}』 著者: {} / {}円 (在庫: {})",
            self.title, self.author, self.price, self.stock
        );
    }

    // 在庫があるかどうか
    fn is_available(&self) -> bool {
        self.stock > 0
    }

    // 割引を適用（価格を更新）
    fn apply_discount(&mut self, discount_percent: u32) {
        self.price = self.price * (100 - discount_percent) / 100;
    }
}

fn main() {
    let mut book = Book::new("Rustプログラミング入門", "山田太郎", 3000, 5);

    book.display_info();
    println!("在庫あり: {}", book.is_available());

    // 20%割引を適用
    book.apply_discount(20);
    println!("20%割引後: {}円", book.price);
    book.display_info();
}
```

**補足：** `&self` は `self: &Self` の省略形で、構造体への不変参照を取ります。`&mut self` は可変参照を取るため、フィールドの値を変更できます。`new` のような関連関数は `Self` を返すのが慣例で、`Book::new(...)` のように呼び出します。

</details>

### 問題2：メソッドチェーン対応の構造体

メソッドチェーン（ビルダーパターン）で設定できる `Config` 構造体を作成してください。

- フィールド：ホスト名（String）、ポート番号（u16）、最大接続数（u32）、デバッグモード（bool）
- 各フィールドを設定するメソッドは `self` を返すことでチェーンを可能にする
- `build` メソッドで設定内容を表示する

**期待される出力例：**
```
=== サーバー設定 ===
ホスト: localhost
ポート: 8080
最大接続数: 100
デバッグモード: 有効

=== 本番設定 ===
ホスト: example.com
ポート: 443
最大接続数: 10000
デバッグモード: 無効
```

<details><summary>ヒント</summary>

- メソッドチェーンを実現するには、各メソッドが `self` を返します
- `fn host(mut self, host: &str) -> Self` のようにシグネチャを書きます
- `Default` トレイトを実装すると初期値を簡単に設定できます

</details>

<details><summary>解答例</summary>

```rust
// メソッドチェーン対応のConfig構造体（ビルダーパターン）

struct Config {
    host: String,
    port: u16,
    max_connections: u32,
    debug: bool,
}

impl Config {
    // デフォルト値で初期化
    fn new() -> Self {
        Config {
            host: "localhost".to_string(),
            port: 8080,
            max_connections: 100,
            debug: false,
        }
    }

    // ホスト名を設定（メソッドチェーン対応）
    fn host(mut self, host: &str) -> Self {
        self.host = host.to_string();
        self
    }

    // ポート番号を設定
    fn port(mut self, port: u16) -> Self {
        self.port = port;
        self
    }

    // 最大接続数を設定
    fn max_connections(mut self, max: u32) -> Self {
        self.max_connections = max;
        self
    }

    // デバッグモードの設定
    fn debug(mut self, enabled: bool) -> Self {
        self.debug = enabled;
        self
    }

    // 設定内容を表示
    fn build(self) {
        println!("ホスト: {}", self.host);
        println!("ポート: {}", self.port);
        println!("最大接続数: {}", self.max_connections);
        println!(
            "デバッグモード: {}",
            if self.debug { "有効" } else { "無効" }
        );
    }
}

fn main() {
    // デフォルト設定 + デバッグモード有効
    println!("=== サーバー設定 ===");
    Config::new()
        .debug(true)
        .build();

    println!();

    // 本番設定
    println!("=== 本番設定 ===");
    Config::new()
        .host("example.com")
        .port(443)
        .max_connections(10000)
        .debug(false)
        .build();
}
```

**補足：** ビルダーパターンは、多数のオプションパラメータを持つオブジェクトの構築に便利です。各メソッドが `Self` を返すことで、`.host("...").port(8080).build()` のようにチェーンできます。Rustでは `mut self` で所有権を受け取って変更するのが一般的なビルダーの書き方です。

</details>

### 問題3：列挙型の基本

交通信号を表す列挙型 `TrafficLight` を作成してください。各信号色に対応する動作メッセージと待ち時間を返すメソッドを実装してください。

**期待される出力例：**
```
🔴 赤信号: 止まれ！ (待ち時間: 60秒)
🟡 黄信号: 注意！まもなく変わります (待ち時間: 5秒)
🟢 青信号: 進め！ (待ち時間: 45秒)

信号のサイクル: 赤 → 青 → 黄 → 赤
合計サイクル時間: 110秒
```

<details><summary>ヒント</summary>

- `enum TrafficLight { Red, Yellow, Green }` で定義します
- `match` 式で各バリアントに対する処理を記述します
- メソッドは `impl TrafficLight { ... }` で定義します
- タプルで `(&str, &str, u32)` を返すと情報をまとめて返せます

</details>

<details><summary>解答例</summary>

```rust
// 交通信号の列挙型

enum TrafficLight {
    Red,
    Yellow,
    Green,
}

impl TrafficLight {
    // 信号の情報を返す (アイコン, メッセージ, 待ち時間)
    fn info(&self) -> (&str, &str, u32) {
        match self {
            TrafficLight::Red => ("🔴", "止まれ！", 60),
            TrafficLight::Yellow => ("🟡", "注意！まもなく変わります", 5),
            TrafficLight::Green => ("🟢", "進め！", 45),
        }
    }

    // 信号の色名を返す
    fn color_name(&self) -> &str {
        match self {
            TrafficLight::Red => "赤",
            TrafficLight::Yellow => "黄",
            TrafficLight::Green => "青",
        }
    }

    // 表示用メソッド
    fn display(&self) {
        let (icon, message, duration) = self.info();
        println!(
            "{} {}信号: {} (待ち時間: {}秒)",
            icon,
            self.color_name(),
            message,
            duration
        );
    }

    // 待ち時間を返す
    fn duration(&self) -> u32 {
        let (_, _, duration) = self.info();
        duration
    }
}

fn main() {
    // 各信号を表示
    let lights = vec![
        TrafficLight::Red,
        TrafficLight::Yellow,
        TrafficLight::Green,
    ];

    for light in &lights {
        light.display();
    }
    println!();

    // 信号のサイクル
    let cycle = vec![
        TrafficLight::Red,
        TrafficLight::Green,
        TrafficLight::Yellow,
        TrafficLight::Red,
    ];

    let cycle_names: Vec<&str> = cycle.iter().map(|l| l.color_name()).collect();
    println!("信号のサイクル: {}", cycle_names.join(" → "));

    // 合計サイクル時間（最後のRedを除く）
    let total: u32 = cycle[..3].iter().map(|l| l.duration()).sum();
    println!("合計サイクル時間: {}秒", total);
}
```

**補足：** Rustの `enum` は `match` 式と組み合わせることで、すべてのバリアントを網羅的に処理できます。コンパイラがパターンの網羅性をチェックするため、バリアントの追加時に対応漏れを防げます。これはRustの安全性を支える重要な機能の1つです。

</details>

### 問題4：Option型を活用した安全なデータアクセス

ユーザーリストを管理するプログラムを作成してください。`Option` 型を使って安全にデータにアクセスする方法を実践します。

以下の機能を実装してください。

1. `User` 構造体（ID、名前、メールアドレス（Option<String>））
2. IDでユーザーを検索する関数（`Option<&User>` を返す）
3. メールアドレスのドメインを取得する関数（`Option<String>` を返す）
4. ユーザー情報の安全な表示

**期待される出力例：**
```
=== ユーザー一覧 ===
ID: 1, 名前: 田中太郎, メール: tanaka@example.com
ID: 2, 名前: 山田花子, メール: (未設定)
ID: 3, 名前: 佐藤一郎, メール: sato@company.co.jp

=== ユーザー検索 ===
ID 1 のユーザー: 田中太郎
ID 1 のドメイン: example.com
ID 2 のドメイン: メールアドレスが未設定です
ID 99 のユーザー: 見つかりません

=== メールアドレスがある全ユーザー ===
田中太郎: tanaka@example.com
佐藤一郎: sato@company.co.jp
```

<details><summary>ヒント</summary>

- `Option<String>` を使うと「値があるかもしれない」ことを型で表現できます
- `as_ref()` で `Option<String>` を `Option<&String>` に変換できます
- `map()` で `Option` の中身を変換できます
- `and_then()` で `Option` を返す関数をチェーンできます
- `unwrap_or()` で `None` のときのデフォルト値を指定できます

</details>

<details><summary>解答例</summary>

```rust
// Option型を活用した安全なデータアクセス

struct User {
    id: u32,
    name: String,
    email: Option<String>, // メールアドレスは任意
}

impl User {
    fn new(id: u32, name: &str, email: Option<&str>) -> Self {
        User {
            id,
            name: name.to_string(),
            email: email.map(|e| e.to_string()),
        }
    }

    // メールアドレスの表示用文字列を返す
    fn email_display(&self) -> &str {
        // as_ref()でOption<String>を&Option<String>から
        // Option<&String>に変換し、map_orでデフォルト値を設定
        self.email.as_deref().unwrap_or("(未設定)")
    }

    // メールアドレスのドメイン部分を取得
    fn email_domain(&self) -> Option<String> {
        // and_thenでOptionをチェーン
        self.email.as_ref().and_then(|email| {
            email.find('@').map(|pos| email[pos + 1..].to_string())
        })
    }

    // ユーザー情報を表示
    fn display(&self) {
        println!(
            "ID: {}, 名前: {}, メール: {}",
            self.id,
            self.name,
            self.email_display()
        );
    }
}

// IDでユーザーを検索
fn find_user_by_id(users: &[User], id: u32) -> Option<&User> {
    users.iter().find(|u| u.id == id)
}

fn main() {
    // ユーザーリストの作成
    let users = vec![
        User::new(1, "田中太郎", Some("tanaka@example.com")),
        User::new(2, "山田花子", None),
        User::new(3, "佐藤一郎", Some("sato@company.co.jp")),
    ];

    // === ユーザー一覧 ===
    println!("=== ユーザー一覧 ===");
    for user in &users {
        user.display();
    }
    println!();

    // === ユーザー検索 ===
    println!("=== ユーザー検索 ===");

    // ID 1 のユーザーを検索
    match find_user_by_id(&users, 1) {
        Some(user) => println!("ID 1 のユーザー: {}", user.name),
        None => println!("ID 1 のユーザー: 見つかりません"),
    }

    // ID 1 のドメインを取得
    let domain_1 = find_user_by_id(&users, 1)
        .and_then(|u| u.email_domain());
    match domain_1 {
        Some(domain) => println!("ID 1 のドメイン: {}", domain),
        None => println!("ID 1 のドメイン: メールアドレスが未設定です"),
    }

    // ID 2 のドメインを取得（メール未設定）
    let domain_2 = find_user_by_id(&users, 2)
        .and_then(|u| u.email_domain());
    match domain_2 {
        Some(domain) => println!("ID 2 のドメイン: {}", domain),
        None => println!("ID 2 のドメイン: メールアドレスが未設定です"),
    }

    // 存在しないIDの検索
    match find_user_by_id(&users, 99) {
        Some(user) => println!("ID 99 のユーザー: {}", user.name),
        None => println!("ID 99 のユーザー: 見つかりません"),
    }

    println!();

    // === メールアドレスがあるユーザーのみ表示 ===
    println!("=== メールアドレスがある全ユーザー ===");
    for user in &users {
        // if letでOptionをパターンマッチ
        if let Some(email) = &user.email {
            println!("{}: {}", user.name, email);
        }
    }
}
```

**補足：** `Option<T>` はRustで「値がないかもしれない」ことを型で表現する方法です。他言語の `null` や `nil` と異なり、コンパイル時に `None` の可能性を見落とすことがありません。`map()`、`and_then()`、`unwrap_or()` などのメソッドを使うことで、安全かつ簡潔にOption値を扱えます。

</details>

### 問題5：トレイト実装 - 図形の面積と表示

異なる図形（円、長方形、三角形）を共通のインターフェースで扱うために、トレイトを実装してください。

以下を実装してください。

1. `Shape` トレイト：`area()` と `description()` メソッドを定義
2. `Circle`、`Rectangle`、`Triangle` の3つの構造体
3. 各構造体に `Shape` トレイトを実装
4. `std::fmt::Display` トレイトも実装して `println!` で表示可能にする
5. トレイトオブジェクトを使って異なる図形をまとめて処理する

**期待される出力例：**
```
=== 各図形の情報 ===
円 (半径: 5.0) → 面積: 78.54
長方形 (幅: 4.0, 高さ: 6.0) → 面積: 24.00
三角形 (底辺: 3.0, 高さ: 8.0) → 面積: 12.00

=== 図形リスト（面積順） ===
1. 円 (半径: 5.0) - 面積: 78.54
2. 長方形 (幅: 4.0, 高さ: 6.0) - 面積: 24.00
3. 三角形 (底辺: 3.0, 高さ: 8.0) - 面積: 12.00

合計面積: 114.54
```

<details><summary>ヒント</summary>

- トレイトは `trait Shape { fn area(&self) -> f64; }` のように定義します
- `impl Shape for Circle { ... }` で各構造体にトレイトを実装します
- `std::fmt::Display` を実装すると `{}` でフォーマットできます
- `Vec<Box<dyn Shape>>` でトレイトオブジェクトのリストを作れます
- 円周率は `std::f64::consts::PI` を使います

</details>

<details><summary>解答例</summary>

```rust
// トレイトを使った図形の面積計算
use std::f64::consts::PI;
use std::fmt;

// Shapeトレイト: 面積計算と説明文を提供
trait Shape: fmt::Display {
    fn area(&self) -> f64;
    fn description(&self) -> String;
}

// 円
struct Circle {
    radius: f64,
}

impl Circle {
    fn new(radius: f64) -> Self {
        Circle { radius }
    }
}

impl Shape for Circle {
    fn area(&self) -> f64 {
        PI * self.radius * self.radius
    }

    fn description(&self) -> String {
        format!("円 (半径: {})", self.radius)
    }
}

impl fmt::Display for Circle {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "{} → 面積: {:.2}", self.description(), self.area())
    }
}

// 長方形
struct Rectangle {
    width: f64,
    height: f64,
}

impl Rectangle {
    fn new(width: f64, height: f64) -> Self {
        Rectangle { width, height }
    }
}

impl Shape for Rectangle {
    fn area(&self) -> f64 {
        self.width * self.height
    }

    fn description(&self) -> String {
        format!("長方形 (幅: {}, 高さ: {})", self.width, self.height)
    }
}

impl fmt::Display for Rectangle {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "{} → 面積: {:.2}", self.description(), self.area())
    }
}

// 三角形
struct Triangle {
    base: f64,
    height: f64,
}

impl Triangle {
    fn new(base: f64, height: f64) -> Self {
        Triangle { base, height }
    }
}

impl Shape for Triangle {
    fn area(&self) -> f64 {
        self.base * self.height / 2.0
    }

    fn description(&self) -> String {
        format!("三角形 (底辺: {}, 高さ: {})", self.base, self.height)
    }
}

impl fmt::Display for Triangle {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "{} → 面積: {:.2}", self.description(), self.area())
    }
}

fn main() {
    // 各図形を作成
    let circle = Circle::new(5.0);
    let rect = Rectangle::new(4.0, 6.0);
    let tri = Triangle::new(3.0, 8.0);

    // === 各図形の情報 ===
    println!("=== 各図形の情報 ===");
    println!("{}", circle);
    println!("{}", rect);
    println!("{}", tri);
    println!();

    // トレイトオブジェクトのベクタに格納
    let mut shapes: Vec<Box<dyn Shape>> = vec![
        Box::new(Circle::new(5.0)),
        Box::new(Rectangle::new(4.0, 6.0)),
        Box::new(Triangle::new(3.0, 8.0)),
    ];

    // 面積の大きい順にソート
    shapes.sort_by(|a, b| {
        b.area()
            .partial_cmp(&a.area())
            .unwrap_or(std::cmp::Ordering::Equal)
    });

    // === 図形リスト（面積順） ===
    println!("=== 図形リスト（面積順） ===");
    for (i, shape) in shapes.iter().enumerate() {
        println!(
            "{}. {} - 面積: {:.2}",
            i + 1,
            shape.description(),
            shape.area()
        );
    }
    println!();

    // 合計面積
    let total: f64 = shapes.iter().map(|s| s.area()).sum();
    println!("合計面積: {:.2}", total);
}
```

**補足：** トレイトはRustにおけるポリモーフィズム（多態性）の実現手段です。`Box<dyn Shape>` はトレイトオブジェクトと呼ばれ、異なる型を同じコレクションに格納できます。`trait Shape: fmt::Display` のようにスーパートレイトを指定すると、`Shape` を実装する型は `Display` も実装する必要があります。

</details>

### 問題6：データ付き列挙型 - コマンドシステム

テキストエディタのコマンドを表す列挙型を作成してください。各バリアントにはデータを持たせ、コマンドの実行をシミュレートします。

以下を実装してください。

1. `Command` 列挙型（Insert, Delete, Replace, MoveCursor, Undo）
2. 各バリアントは必要なデータを保持する
3. コマンドを実行するメソッド
4. コマンドの説明を返すメソッド

**期待される出力例：**
```
=== テキストエディタ コマンド履歴 ===

実行: カーソルを位置 5 に移動
実行: 位置 5 に "Hello" を挿入
実行: 位置 3 から 2 文字を削除
実行: "World" を "Rust" に置換
実行: 直前の操作を取り消し

--- コマンド履歴 ---
1. MoveCursor(5)
2. Insert(5, "Hello")
3. Delete(3, 2)
4. Replace("World" → "Rust")
5. Undo
```

<details><summary>ヒント</summary>

- データ付き列挙型は `Insert(usize, String)` のように定義します
- `match` でパターンマッチし、データを分解（destructure）できます
- `Command::Insert(5, "Hello".to_string())` のようにインスタンスを作成します

</details>

<details><summary>解答例</summary>

```rust
// データ付き列挙型 - テキストエディタのコマンドシステム

// コマンド列挙型: 各バリアントが異なるデータを保持
enum Command {
    Insert(usize, String),         // (位置, テキスト)
    Delete(usize, usize),          // (位置, 文字数)
    Replace(String, String),       // (検索文字列, 置換文字列)
    MoveCursor(usize),             // (移動先位置)
    Undo,                          // データなし
}

impl Command {
    // コマンドを実行（シミュレーション）
    fn execute(&self) {
        match self {
            Command::Insert(pos, text) => {
                println!(
                    "実行: 位置 {} に \"{}\" を挿入",
                    pos, text
                );
            }
            Command::Delete(pos, count) => {
                println!(
                    "実行: 位置 {} から {} 文字を削除",
                    pos, count
                );
            }
            Command::Replace(from, to) => {
                println!(
                    "実行: \"{}\" を \"{}\" に置換",
                    from, to
                );
            }
            Command::MoveCursor(pos) => {
                println!("実行: カーソルを位置 {} に移動", pos);
            }
            Command::Undo => {
                println!("実行: 直前の操作を取り消し");
            }
        }
    }

    // コマンドの簡潔な説明を返す
    fn summary(&self) -> String {
        match self {
            Command::Insert(pos, text) => {
                format!("Insert({}, \"{}\")", pos, text)
            }
            Command::Delete(pos, count) => {
                format!("Delete({}, {})", pos, count)
            }
            Command::Replace(from, to) => {
                format!("Replace(\"{}\" → \"{}\")", from, to)
            }
            Command::MoveCursor(pos) => {
                format!("MoveCursor({})", pos)
            }
            Command::Undo => "Undo".to_string(),
        }
    }
}

fn main() {
    // コマンド履歴を作成
    let commands = vec![
        Command::MoveCursor(5),
        Command::Insert(5, "Hello".to_string()),
        Command::Delete(3, 2),
        Command::Replace("World".to_string(), "Rust".to_string()),
        Command::Undo,
    ];

    println!("=== テキストエディタ コマンド履歴 ===\n");

    // コマンドを順番に実行
    for cmd in &commands {
        cmd.execute();
    }

    // コマンド履歴を表示
    println!("\n--- コマンド履歴 ---");
    for (i, cmd) in commands.iter().enumerate() {
        println!("{}. {}", i + 1, cmd.summary());
    }
}
```

**補足：** Rustの `enum` はデータを持てる点が、他の多くの言語の列挙型と異なります。これは代数的データ型（Algebraic Data Type）と呼ばれ、関数型プログラミングの概念です。各バリアントが異なる型・数のデータを保持できるため、コマンドパターンやメッセージパッシングの実装に非常に適しています。

</details>

### 問題7：enumとmatchを駆使した状態管理 - 自動販売機

自動販売機をシミュレートするプログラムを作成してください。状態遷移を `enum` で表現し、`match` で処理を分岐させます。

以下の仕様で実装してください。

1. `DrinkType` 列挙型：Cola（120円）、Tea（100円）、Water（80円）、Coffee（150円）
2. `VendingState` 列挙型：Idle（待機）、CoinInserted（投入額）、Dispensing（払い出し中）
3. `VendingMachine` 構造体：状態、投入金額、在庫、売上を管理
4. 操作：コイン投入、ドリンク選択、おつり返却
5. 在庫管理と売上集計

**期待される出力例：**
```
=== 自動販売機シミュレーション ===

[待機中] コインを投入してください
100円投入 → 合計: 100円
50円投入 → 合計: 150円

購入可能な商品:
  Tea (100円) - 在庫: 3
  Water (80円) - 在庫: 5

Cola を選択 → 残高不足です（必要: 120円, 残高: 150円 → OK!）
Cola を購入しました！おつり: 30円

200円投入 → 合計: 200円
Coffee を選択 → Coffee を購入しました！おつり: 50円

=== 売上レポート ===
Cola: 1本 (120円)
Coffee: 1本 (150円)
合計売上: 270円
残り在庫: Cola=2, Tea=3, Water=5, Coffee=1
```

<details><summary>ヒント</summary>

- `DrinkType` に価格を返すメソッドを実装します
- `VendingMachine` は `state: VendingState` フィールドで状態を管理します
- 状態遷移は `match` で現在の状態を確認してから行います
- `HashMap` の代わりに、構造体のフィールドで在庫を管理しても構いません
- 売上記録には `Vec<(DrinkType, u32)>` を使えます

</details>

<details><summary>解答例</summary>

```rust
// 自動販売機シミュレーション
// enumとmatchを使った状態管理

use std::collections::HashMap;

// ドリンクの種類
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
enum DrinkType {
    Cola,
    Tea,
    Water,
    Coffee,
}

impl DrinkType {
    // 価格を返す
    fn price(&self) -> u32 {
        match self {
            DrinkType::Cola => 120,
            DrinkType::Tea => 100,
            DrinkType::Water => 80,
            DrinkType::Coffee => 150,
        }
    }

    // 名前を返す
    fn name(&self) -> &str {
        match self {
            DrinkType::Cola => "Cola",
            DrinkType::Tea => "Tea",
            DrinkType::Water => "Water",
            DrinkType::Coffee => "Coffee",
        }
    }

    // 全種類のリストを返す
    fn all() -> Vec<DrinkType> {
        vec![
            DrinkType::Cola,
            DrinkType::Tea,
            DrinkType::Water,
            DrinkType::Coffee,
        ]
    }
}

// 自動販売機の状態
#[derive(Debug)]
enum VendingState {
    Idle,                // 待機中
    CoinInserted(u32),   // コイン投入済み（投入額）
}

// 自動販売機
struct VendingMachine {
    state: VendingState,
    stock: HashMap<DrinkType, u32>,       // 在庫
    sales: Vec<(DrinkType, u32)>,         // 売上記録
}

impl VendingMachine {
    fn new() -> Self {
        let mut stock = HashMap::new();
        stock.insert(DrinkType::Cola, 3);
        stock.insert(DrinkType::Tea, 3);
        stock.insert(DrinkType::Water, 5);
        stock.insert(DrinkType::Coffee, 2);

        VendingMachine {
            state: VendingState::Idle,
            stock,
            sales: Vec::new(),
        }
    }

    // コインを投入
    fn insert_coin(&mut self, amount: u32) {
        match self.state {
            VendingState::Idle => {
                println!("{}円投入 → 合計: {}円", amount, amount);
                self.state = VendingState::CoinInserted(amount);
            }
            VendingState::CoinInserted(current) => {
                let total = current + amount;
                println!("{}円投入 → 合計: {}円", amount, total);
                self.state = VendingState::CoinInserted(total);
            }
        }
    }

    // 購入可能な商品を表示
    fn show_available(&self) {
        match &self.state {
            VendingState::Idle => {
                println!("[待機中] コインを投入してください");
            }
            VendingState::CoinInserted(amount) => {
                println!("購入可能な商品:");
                for drink in DrinkType::all() {
                    let stock = self.stock.get(&drink).unwrap_or(&0);
                    if *stock > 0 && drink.price() <= *amount {
                        println!(
                            "  {} ({}円) - 在庫: {}",
                            drink.name(),
                            drink.price(),
                            stock
                        );
                    }
                }
            }
        }
    }

    // ドリンクを選択して購入
    fn select(&mut self, drink: DrinkType) {
        let amount = match self.state {
            VendingState::Idle => {
                println!("先にコインを投入してください");
                return;
            }
            VendingState::CoinInserted(amount) => amount,
        };

        let price = drink.price();
        let stock = *self.stock.get(&drink).unwrap_or(&0);

        if stock == 0 {
            println!("{} は売り切れです", drink.name());
            return;
        }

        if amount < price {
            println!(
                "{} を選択 → 残高不足です（必要: {}円, 残高: {}円）",
                drink.name(),
                price,
                amount
            );
            return;
        }

        // 購入処理
        let change = amount - price;
        *self.stock.get_mut(&drink).unwrap() -= 1;
        self.sales.push((drink, price));

        println!(
            "{} を購入しました！おつり: {}円",
            drink.name(),
            change
        );

        // 状態をリセット
        self.state = VendingState::Idle;
    }

    // おつりを返却
    fn return_change(&mut self) {
        match self.state {
            VendingState::CoinInserted(amount) => {
                println!("{}円を返却しました", amount);
                self.state = VendingState::Idle;
            }
            VendingState::Idle => {
                println!("返却するコインはありません");
            }
        }
    }

    // 売上レポートを表示
    fn sales_report(&self) {
        println!("=== 売上レポート ===");

        // ドリンク種類ごとの売上を集計
        let mut report: HashMap<DrinkType, (u32, u32)> = HashMap::new();
        for (drink, price) in &self.sales {
            let entry = report.entry(*drink).or_insert((0, 0));
            entry.0 += 1;      // 本数
            entry.1 += price;   // 金額
        }

        let mut total_sales = 0;
        for drink in DrinkType::all() {
            if let Some((count, amount)) = report.get(&drink) {
                println!("{}: {}本 ({}円)", drink.name(), count, amount);
                total_sales += amount;
            }
        }

        println!("合計売上: {}円", total_sales);

        // 在庫状況
        let stock_str: Vec<String> = DrinkType::all()
            .iter()
            .map(|d| {
                format!(
                    "{}={}",
                    d.name(),
                    self.stock.get(d).unwrap_or(&0)
                )
            })
            .collect();
        println!("残り在庫: {}", stock_str.join(", "));
    }
}

fn main() {
    println!("=== 自動販売機シミュレーション ===\n");

    let mut machine = VendingMachine::new();

    // 待機状態を表示
    machine.show_available();

    // コインを投入
    machine.insert_coin(100);
    machine.insert_coin(50);
    println!();

    // 購入可能な商品を表示
    machine.show_available();
    println!();

    // Colaを購入（150円で120円の商品）
    machine.select(DrinkType::Cola);
    println!();

    // もう一度購入
    machine.insert_coin(200);
    machine.select(DrinkType::Coffee);
    println!();

    // 売上レポート
    machine.sales_report();
}
```

**補足：** この問題では、`enum` を使った状態管理パターンを学びます。`VendingState` で自動販売機の状態を型として表現し、`match` で状態に応じた処理を分岐させます。このパターンは「ステートマシン」と呼ばれ、ゲーム開発やネットワークプロトコルの実装で広く使われます。

`#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]` は構造体や列挙型に自動的にトレイトを実装するマクロです。`Hash` を導入することで `HashMap` のキーとして使用できるようになります。

</details>

### 問題8：ジェネリクスとトレイトを組み合わせたコレクション

ジェネリクスとトレイトを組み合わせて、型に依存しない汎用的なデータコンテナ `SortedList` を作成してください。

以下の機能を実装してください。

1. `SortedList<T>` 構造体：挿入時に自動的にソートされるリスト
2. `add` メソッド：要素を追加して自動ソート
3. `get` メソッド：インデックスでアクセス（`Option<&T>` を返す）
4. `contains` メソッド：要素の存在確認
5. `Display` トレイトの実装：リストの内容を表示
6. 整数型と文字列型の両方でテスト

**期待される出力例：**
```
=== 整数のSortedList ===
追加: 30 → [30]
追加: 10 → [10, 30]
追加: 50 → [10, 30, 50]
追加: 20 → [10, 20, 30, 50]
追加: 40 → [10, 20, 30, 50, 40... → [10, 20, 30, 40, 50]

要素数: 5
先頭: 10
末尾: 50
20を含む: true
25を含む: false

=== 文字列のSortedList ===
Sorted: ["Apple", "Banana", "Cherry", "Date", "Fig"]

=== 統計情報（整数リスト） ===
最小値: 10
最大値: 50
合計: 150
```

<details><summary>ヒント</summary>

- ジェネリクスの型パラメータに `T: Ord + std::fmt::Display` のようにトレイト境界を指定します
- ソート済みリストへの挿入は `binary_search` メソッドが便利です
- `binary_search` は `Result<usize, usize>` を返し、見つからない場合は挿入位置を返します
- `where` 句を使ってトレイト境界を見やすく書けます

</details>

<details><summary>解答例</summary>

```rust
// ジェネリクスとトレイトを組み合わせた
// 自動ソートリスト

use std::fmt;

// ソート済みリスト構造体
struct SortedList<T>
where
    T: Ord + fmt::Display,
{
    items: Vec<T>,
}

impl<T> SortedList<T>
where
    T: Ord + fmt::Display,
{
    // 新しい空のリストを作成
    fn new() -> Self {
        SortedList { items: Vec::new() }
    }

    // 要素を追加（自動的にソート位置に挿入）
    fn add(&mut self, item: T) {
        // binary_searchでソート位置を見つける
        let pos = match self.items.binary_search(&item) {
            Ok(pos) => pos,     // 同じ値が見つかった場合、その位置
            Err(pos) => pos,    // 見つからない場合、挿入すべき位置
        };
        self.items.insert(pos, item);
    }

    // インデックスでアクセス
    fn get(&self, index: usize) -> Option<&T> {
        self.items.get(index)
    }

    // 要素の存在確認
    fn contains(&self, item: &T) -> bool {
        self.items.binary_search(item).is_ok()
    }

    // 要素数を返す
    fn len(&self) -> usize {
        self.items.len()
    }

    // 先頭要素を返す
    fn first(&self) -> Option<&T> {
        self.items.first()
    }

    // 末尾要素を返す
    fn last(&self) -> Option<&T> {
        self.items.last()
    }
}

// Display トレイトの実装
impl<T> fmt::Display for SortedList<T>
where
    T: Ord + fmt::Display,
{
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        let items_str: Vec<String> =
            self.items.iter().map(|item| format!("{}", item)).collect();
        write!(f, "[{}]", items_str.join(", "))
    }
}

// 整数リスト用の統計機能
fn stats(list: &SortedList<i32>) {
    println!("=== 統計情報（整数リスト） ===");
    if let Some(min) = list.first() {
        println!("最小値: {}", min);
    }
    if let Some(max) = list.last() {
        println!("最大値: {}", max);
    }
    // 内部データにアクセスして合計を計算
    let sum: i32 = (0..list.len())
        .filter_map(|i| list.get(i))
        .sum();
    println!("合計: {}", sum);
}

fn main() {
    // === 整数のSortedList ===
    println!("=== 整数のSortedList ===");
    let mut int_list = SortedList::new();

    for &num in &[30, 10, 50, 20, 40] {
        int_list.add(num);
        println!("追加: {} → {}", num, int_list);
    }
    println!();

    println!("要素数: {}", int_list.len());
    if let Some(first) = int_list.first() {
        println!("先頭: {}", first);
    }
    if let Some(last) = int_list.last() {
        println!("末尾: {}", last);
    }
    println!("20を含む: {}", int_list.contains(&20));
    println!("25を含む: {}", int_list.contains(&25));
    println!();

    // === 文字列のSortedList ===
    println!("=== 文字列のSortedList ===");
    let mut str_list = SortedList::new();
    for name in ["Cherry", "Apple", "Fig", "Banana", "Date"] {
        str_list.add(name.to_string());
    }
    // Displayトレイトで表示（""で囲む）
    let display: Vec<String> = (0..str_list.len())
        .filter_map(|i| str_list.get(i))
        .map(|s| format!("\"{}\"", s))
        .collect();
    println!("Sorted: [{}]", display.join(", "));
    println!();

    // === 統計情報 ===
    stats(&int_list);
}
```

**補足：** ジェネリクス `<T>` とトレイト境界 `T: Ord + Display` を組み合わせることで、「順序比較可能で表示可能な任意の型」に対して動作する汎用的なデータ構造を作れます。`binary_search` は O(log n) で効率的に検索・挿入位置を見つけられます。

`where` 句は型パラメータが複雑になった場合に読みやすく書くための構文で、`impl<T: Ord + Display>` と同等ですが、関数シグネチャが長くなる場合に推奨されます。

</details>
