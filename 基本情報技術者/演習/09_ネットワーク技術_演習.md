## 基本問題

### 問題1：OSI参照モデル
OSI参照モデルの第1層から第4層までの名称を順に答えてください。

<details>
<summary>解答</summary>

1. 第1層：**物理層**
2. 第2層：**データリンク層**
3. 第3層：**ネットワーク層**
4. 第4層：**トランスポート層**
</details>

---

### 問題2：ネットワーク機器と対応層
以下のネットワーク機器が動作するOSI参照モデルの層を答えてください。

(1) ルータ
(2) L2スイッチ（ブリッジ）
(3) リピータ（ハブ）

<details>
<summary>解答</summary>

(1) ルータ：**第3層（ネットワーク層）** — IPアドレスによるパケット転送
(2) L2スイッチ：**第2層（データリンク層）** — MACアドレスによるフレーム転送
(3) リピータ/ハブ：**第1層（物理層）** — 電気信号の増幅・中継
</details>

---

### 問題3：TCP と UDP の比較
TCP と UDP の違いを、接続方式と信頼性の観点から説明してください。

<details>
<summary>解答</summary>

- **TCP：** コネクション型（3ウェイハンドシェイクで接続確立）。再送制御により信頼性あり。低速だがデータの確実な到達を保証。
- **UDP：** コネクションレス型（事前接続なし）。信頼性なし（再送制御なし）。高速でリアルタイム通信に適する。
</details>

---

### 問題4：四択問題
IPアドレスからMACアドレスを取得するプロトコルはどれですか。

ア. DNS　　イ. DHCP　　ウ. ARP　　エ. ICMP

<details>
<summary>解答</summary>

答え：**ウ. ARP（Address Resolution Protocol）**

ARPはIP（論理アドレス）→ MAC（物理アドレス）の変換を行います。
</details>

## 応用問題

### 問題6：IPアドレスのクラス判定
以下のIPアドレスのクラスを答えてください。

(1) 10.0.0.1
(2) 172.20.1.5
(3) 192.168.0.100

<details>
<summary>解答</summary>

(1) 10.0.0.1 → 第1オクテット 10（1〜126）→ **クラスA**
(2) 172.20.1.5 → 第1オクテット 172（128〜191）→ **クラスB**
(3) 192.168.0.100 → 第1オクテット 192（192〜223）→ **クラスC**

※ いずれもプライベートIPアドレスの範囲内です。
</details>

### 問題7：ホスト数の計算
IPアドレス 192.168.10.0/28 のネットワークで使用可能なホスト数を求めてください。

<details>
<summary>ヒント</summary>
ホスト部のビット数 = 32 - プレフィックス長。使用可能ホスト数 = 2^n - 2 です。
</details>

<details>
<summary>解答</summary>

```
ホスト部のビット数 = 32 - 28 = 4ビット
使用可能ホスト数 = 2⁴ - 2 = 16 - 2 = 14台
```
答え：**14台**

ネットワークアドレス（ホスト部が全て0）とブロードキャストアドレス（ホスト部が全て1）を除くため、2を引きます。
</details>

### 問題8：サブネットマスクの計算
IPアドレス 192.168.5.130 / サブネットマスク 255.255.255.192 のとき、ネットワークアドレスとブロードキャストアドレスを求めてください。

<details>
<summary>ヒント</summary>
255.255.255.192 = /26。ホスト部は6ビット。IPとサブネットマスクのAND演算でネットワークアドレスを求めます。
</details>

<details>
<summary>解答</summary>

```
IPアドレス:       192.168.5.130 = 192.168.5.10000010
サブネットマスク:  255.255.255.192 = 255.255.255.11000000

第4オクテット：
  10000010（130）
AND 11000000（192）
= 10000000（128）

ネットワークアドレス: 192.168.5.128

ホスト部（6ビット）を全て1にするとブロードキャスト：
10000000 → 10111111 = 191

ブロードキャストアドレス: 192.168.5.191
```

答え：
- ネットワークアドレス：**192.168.5.128**
- ブロードキャストアドレス：**192.168.5.191**
- 使用可能範囲：192.168.5.129 〜 192.168.5.190（62台）
</details>

### 問題9：3ウェイハンドシェイク
TCPの3ウェイハンドシェイクの手順を、クライアントとサーバー間のやりとりとして説明してください。

<details>
<summary>解答</summary>

```
1. クライアント → サーバー: SYN（接続要求）
2. サーバー → クライアント: SYN + ACK（接続要求の確認 + サーバーからの接続要求）
3. クライアント → サーバー: ACK（確認応答）
```

この3回のやりとりで TCPコネクションが確立され、以降データの送受信が可能になります。
</details>

### 問題10：四択問題
ルーティングプロトコル RIP が経路選択の基準とするメトリック（指標）はどれですか。

ア. 帯域幅　　イ. 遅延時間　　ウ. ホップ数　　エ. コスト

<details>
<summary>解答</summary>

答え：**ウ. ホップ数**

RIPは経由するルータの数（ホップ数）で経路を選択します。最大15ホップまでです。
一方、OSPFは帯域幅（コスト）を基準に経路を選択します。
</details>

## チャレンジ問題

### 問題11：サブネット分割
192.168.1.0/24 のネットワークを8つのサブネットに分割する場合、各サブネットのプレフィックス長、サブネットマスク、使用可能ホスト数を求めてください。

<details>
<summary>解答</summary>

```
8つに分割 → 2³ = 8 → サブネットビット数 = 3
新しいプレフィックス長 = 24 + 3 = 27（/27）
サブネットマスク = 255.255.255.224（11100000）

ホスト部 = 32 - 27 = 5ビット
各サブネットのアドレス数 = 2⁵ = 32
使用可能ホスト数 = 32 - 2 = 30台

サブネット一覧：
1: 192.168.1.0   〜 192.168.1.31   (ホスト: .1〜.30)
2: 192.168.1.32  〜 192.168.1.63   (ホスト: .33〜.62)
3: 192.168.1.64  〜 192.168.1.95   (ホスト: .65〜.94)
4: 192.168.1.96  〜 192.168.1.127  (ホスト: .97〜.126)
5: 192.168.1.128 〜 192.168.1.159  (ホスト: .129〜.158)
6: 192.168.1.160 〜 192.168.1.191  (ホスト: .161〜.190)
7: 192.168.1.192 〜 192.168.1.223  (ホスト: .193〜.222)
8: 192.168.1.224 〜 192.168.1.255  (ホスト: .225〜.254)
```

答え：
- プレフィックス長：**/27**
- サブネットマスク：**255.255.255.224**
- 使用可能ホスト数：**30台**
</details>

### 問題12：伝送時間の計算
20MBのファイルを 50Mbps の回線で転送する場合、伝送時間を求めてください。回線利用率は 80% とします。

<details>
<summary>解答</summary>

```
データ量 = 20MB = 20 × 8 = 160Mビット（1バイト = 8ビット）
実効伝送速度 = 50 × 0.8 = 40 Mbps

伝送時間 = 160Mビット / 40Mbps = 4秒
```
答え：**4秒**
</details>

### 問題13：DNS名前解決
DNS の名前解決における「再帰問い合わせ」と「反復問い合わせ」の違いを説明してください。

<details>
<summary>解答</summary>

- **再帰問い合わせ：** クライアントがキャッシュDNSサーバーに問い合わせる方式。キャッシュDNSサーバーが代わりに名前解決の全過程を行い、最終的な回答をクライアントに返す。
- **反復問い合わせ：** キャッシュDNSサーバーが各権威DNSサーバーに対して行う問い合わせ方式。問い合わせ先のDNSサーバーが答えを知らない場合、次に問い合わせるべきDNSサーバーの情報を返す。キャッシュDNSサーバーが順番に問い合わせを繰り返して名前を解決する。

クライアント→キャッシュDNS が再帰問い合わせ、キャッシュDNS→各権威DNS が反復問い合わせです。
</details>
