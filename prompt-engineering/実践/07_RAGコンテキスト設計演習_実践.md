## 課題の要件

### 背景

RAGシステムでは、外部データベースから検索した情報（コンテキスト）をプロンプトに注入し、LLMに回答させます。この演習では、RAGのプロンプト部分の設計に焦点を当てます。

```
RAGの基本フロー：
ユーザーの質問 → 検索エンジン → 関連文書を取得 → プロンプトに注入 → LLMが回答
                                    ↑
                            この部分を設計する
```

### シナリオ（4つ）

| シナリオ | 業務内容 | コンテキストの種類 |
|---------|---------|-------------------|
| A | 社内FAQシステム | Q&A形式の文書 |
| B | 製品マニュアル検索 | 技術文書（手順書） |
| C | 社内規程の問い合わせ | 規程・規則文書 |
| D | 議事録からの情報検索 | 会議の議事録 |

### 各シナリオで設計すること

1. **システムプロンプト**: LLMの役割と振る舞いのルールを定義
2. **コンテキスト注入テンプレート**: 検索結果をプロンプトに組み込むフォーマット
3. **回答生成プロンプト**: ユーザーの質問とコンテキストを組み合わせた最終プロンプト
4. **フォールバック処理**: コンテキストに回答がない場合の対応方法
5. **テストケース**: 3つの質問で動作を検証（正常系2つ、異常系1つ）

## ステップガイド

<details>
<summary>ステップ1：RAGプロンプトの基本構造を理解する</summary>

RAGプロンプトは通常、以下の構造を持ちます。

```
┌─────────────────────────────────────────────┐
│ 1. システムプロンプト（固定）                 │
│    → LLMの役割・ルール・制約を定義           │
├─────────────────────────────────────────────┤
│ 2. コンテキスト（動的に注入）                 │
│    → 検索エンジンから取得した関連文書         │
├─────────────────────────────────────────────┤
│ 3. ユーザーの質問（動的）                     │
│    → ユーザーが入力した質問文                 │
├─────────────────────────────────────────────┤
│ 4. 回答指示（固定）                           │
│    → 回答の形式・制約を指定                   │
└─────────────────────────────────────────────┘
```

重要な原則：
- **コンテキストに基づいて回答する**（LLMの事前知識に頼らない）
- **回答できない場合は正直に伝える**（ハルシネーション防止）
- **情報の出典を明示する**（どのドキュメントから回答したかを示す）

</details>

<details>
<summary>ステップ2：シナリオAのプロンプトを設計する</summary>

シナリオA「社内FAQシステム」を例に設計しましょう。

**サンプルのFAQデータ**（コンテキストとして使用）:

```
FAQ-001: 有給休暇の申請方法
Q: 有給休暇はどうやって申請しますか？
A: 社内ポータルの「勤怠管理」→「休暇申請」から申請してください。
   申請期限は休暇日の3営業日前までです。上長の承認が必要です。

FAQ-002: リモートワークのルール
Q: リモートワークは週に何日まで可能ですか？
A: 原則として週3日までです。ただし、部門長の承認があれば
   週5日のフルリモートも可能です。申請は月初に行ってください。

FAQ-003: 経費精算の方法
Q: 経費精算はどこで行いますか？
A: 経費精算システム「ExpenseHub」から申請してください。
   領収書の画像添付が必須です。申請期限は支出日から1ヶ月以内です。
```

この3つのFAQをコンテキストとして使い、プロンプトを設計してみましょう。

</details>

<details>
<summary>ステップ3：フォールバック処理を設計する</summary>

コンテキストに回答が含まれていない場合の対応は、RAGシステムの品質を左右する重要な要素です。

**フォールバックの3段階**:

1. **部分回答**: 関連する情報が一部ある場合
   → 「○○については次の情報があります。ただし、△△については情報が見つかりませんでした」

2. **回答不可の明示**: 関連情報がまったくない場合
   → 「申し訳ございませんが、ご質問の内容に関する情報が見つかりませんでした」

3. **代替案の提示**: 回答できない場合の次のアクション
   → 「詳しくは○○部門（内線：XXXX）にお問い合わせください」

</details>

<details>
<summary>ステップ4：テストケースで検証する</summary>

各シナリオで以下の3パターンのテストケースを作成しましょう。

| パターン | 説明 | テスト内容 |
|---------|------|-----------|
| 正常系1 | 完全一致 | コンテキストに直接回答がある質問 |
| 正常系2 | 推論が必要 | コンテキストの情報を組み合わせて回答する質問 |
| 異常系 | 情報なし | コンテキストに回答がない質問 |

テスト結果を「期待する出力」と「実際の出力」で比較してください。

</details>

### シナリオA：社内FAQシステム

#### 1. システムプロンプト

```
あなたは社内FAQシステムのアシスタントです。

以下のルールに従って回答してください：

1. 提供されたFAQ文書の情報のみに基づいて回答してください
2. FAQ文書に記載がない内容については「該当する情報が見つかりませんでした」と回答してください
3. 回答の末尾に、参照したFAQ番号を記載してください（例：参照: FAQ-001）
4. 丁寧語（です・ます調）で回答してください
5. 回答は簡潔に、200文字以内を目安にしてください
```

#### 2. コンテキスト注入テンプレート

```
【参照FAQ文書】
以下のFAQ文書を参考にして回答してください。

---
{検索結果として取得されたFAQ文書をここに挿入}
---
```

### シナリオB：製品マニュアル検索（改良版）

改良版では、**コンテキストの前処理**と**信頼度スコア**を導入します。

#### 1. システムプロンプト（改良版）

```
あなたは製品サポートのAIアシスタントです。

## 基本ルール
1. 提供されたマニュアル文書の情報のみに基づいて回答してください
2. 回答に含める情報の信頼度を3段階で自己評価してください
   - 高：マニュアルに明記されている内容
   - 中：マニュアルの情報から論理的に推論した内容
   - 低：マニュアルに直接関連しないが一般的な知識から補足した内容
3. 信頼度「低」の情報を含む場合は、必ず「※一般的な情報として」と注釈をつけてください

## 回答形式
- 手順の説明は番号付きリストで記述
- 注意事項がある場合は「⚠ 注意」として明示
- 参照元を「出典: [文書名] セクション[番号]」の形式で記載

## 回答できない場合
- 「該当する情報がマニュアルに見つかりませんでした」と明示
- 関連しそうなトピックがあれば「関連情報として、[トピック名]をご確認ください」と案内
- サポート窓口の連絡先を提示（電話: 0120-XXX-XXX、メール: support@example.com）
```

#### 2. コンテキスト注入テンプレート（改良版）

```
【参照マニュアル文書】
※ 検索スコアの高い順に表示しています。

[文書1] スコア: 0.95 / 出典: セットアップガイド セクション3.2
---
{文書内容}
---

[文書2] スコア: 0.82 / 出典: トラブルシューティング セクション5.1
---
{文書内容}
---

[文書3] スコア: 0.71 / 出典: FAQ集 Q-042
---
{文書内容}
---

※ スコア0.7未満の文書は信頼性が低い可能性があります。
```

**改良ポイント**:
- 検索スコアを表示し、LLMが情報の信頼性を判断できるようにした
- 出典情報を文書レベルで付与し、トレーサビリティ（traceability）を確保
- スコアの閾値を明示し、低スコア文書への注意を促した

#### 3. サンプルコンテキストとテスト

**サンプルマニュアル文書**:

```
[文書1] スコア: 0.95 / 出典: セットアップガイド セクション3.2
---
Wi-Fi接続の設定方法：
1. 本体の電源を入れ、ホーム画面が表示されるまで待ちます（約30秒）
2. 設定アイコン（歯車マーク）をタップします
3. 「ネットワーク」→「Wi-Fi」を選択します
4. 接続するネットワーク名を選択し、パスワードを入力します
5. 「接続」をタップし、画面上部にWi-Fiアイコンが表示されれば完了です

注意：5GHz帯のWi-Fiには対応していません。2.4GHz帯をご使用ください。
---

[文書2] スコア: 0.78 / 出典: トラブルシューティング セクション5.1
---
Wi-Fiに接続できない場合：
- ルーターの電源を切り、30秒後に再度入れてください
- 本体を再起動してください（電源ボタンを10秒長押し）
- パスワードに全角文字が含まれていないか確認してください
- 上記で解決しない場合、ネットワーク設定を初期化してください
  （設定 → ネットワーク → 詳細設定 → ネットワーク初期化）
---
```

**テスト1（正常系・手順の説明）**:
- 質問：「Wi-Fiの設定方法を教えてください」
- 期待する出力：
```
Wi-Fi接続の設定手順は以下の通りです。

1. 本体の電源を入れ、ホーム画面が表示されるまで待ちます（約30秒）
2. 設定アイコン（歯車マーク）をタップします
3. 「ネットワーク」→「Wi-Fi」を選択します
4. 接続するネットワーク名を選択し、パスワードを入力します
5. 「接続」をタップし、画面上部にWi-Fiアイコンが表示されれば完了です

⚠ 注意：5GHz帯のWi-Fiには対応していません。2.4GHz帯をご使用ください。

信頼度：高（マニュアルに明記された手順）
出典: セットアップガイド セクション3.2
```

**テスト2（正常系・複数文書の統合）**:
- 質問：「Wi-Fiに繋がらないのですが、設定方法と対処法を教えてください」
- 期待する出力：設定手順（文書1）とトラブルシューティング（文書2）の両方を統合した回答

**テスト3（異常系）**:
- 質問：「Bluetooth機器との接続方法を教えてください」
- 期待する出力：「Bluetooth接続に関する情報がマニュアルに見つかりませんでした。関連情報として、Wi-Fi接続の設定手順はございます。Bluetoothについては、サポート窓口（電話: 0120-XXX-XXX）にお問い合わせください。」

### コンテキスト最適化のベストプラクティス

```
┌─────────────────────────────────────────────────────────┐
│         RAGコンテキスト最適化 5つの原則                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 検索スコアを活用する                                 │
│     → 高スコアの文書を優先的に参照するよう指示           │
│                                                         │
│  2. 文書の出典を明示する                                 │
│     → トレーサビリティの確保、ユーザーの信頼性向上       │
│                                                         │
│  3. 回答の信頼度を自己評価させる                         │
│     → 推論と事実の区別を明確にする                       │
│                                                         │
│  4. フォールバックを段階的に設計する                     │
│     → 完全回答 → 部分回答 → 回答不可 → 代替案の提示    │
│                                                         │
│  5. コンテキストの量を制御する                           │
│     → 多すぎると「Lost in the Middle」問題が発生        │
│     → 3〜5文書を目安に、関連度順で提供                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**初心者向けとの違い**:
- 検索スコアの概念を導入し、コンテキストの信頼度を制御
- 回答の信頼度を3段階で自己評価させる仕組みを設計
- 複数文書を統合する際の優先順位付けを明示
- 「Lost in the Middle」問題への対策を考慮したコンテキスト量の制御
