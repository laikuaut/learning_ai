> **難易度**: ★★★★☆（上級）
> **前提知識**: 第2章（ファイルとディレクトリ）、第3章（パーミッション）、第5章（シェルスクリプト基礎）、第7章（テキスト処理）
> **課題の種類**: デバッグ
> **学習目標**: バグだらけのシェルスクリプトを読み解き、原因を特定して修正する力を養う

---

## 課題の説明

以下の「ファイル整理スクリプト」には **10個のバグ** が埋め込まれています。
スクリプトを実行し、エラーメッセージや意図しない動作を手がかりにすべてのバグを見つけて修正してください。

### 進め方

1. まず下のバグ入りコードをファイルに保存する
2. テスト用のファイルを作成する（テストデータ作成スクリプトを先に実行）
3. バグ入りスクリプトを実行してエラーを確認する
4. 1つずつバグを修正し、再実行する
5. 全機能が正常に動作すれば完了

## バグ入りコード

以下のコードを `file_organizer_buggy.sh` として保存してください。

```bash
#!/bin/bash
# ファイル整理スクリプト（バグ入り版）
# 指定ディレクトリ内のファイルを拡張子別にフォルダに整理する
# このスクリプトには10個のバグが含まれています

SOURCE_DIR=$1   # バグ1

# ディレクトリの存在チェック
if [ ! -d $SOURCE_DIR ]; then   # バグ2
    echo "エラー: ディレクトリが見つかりません: $SOURCE_DIR"
    exit 1
fi

echo "===== ファイル整理スクリプト ====="
echo "対象: $SOURCE_DIR"
echo ""

# ファイル数のカウント
file_count=0
for f in $SOURCE_DIR/*; do   # バグ3
    if [ -f "$f" ]; then
        file_count=$file_count+1   # バグ4
    fi
done
echo "整理対象ファイル数: $file_count"
echo ""

# 拡張子の分類ルール
classify_file() {
    local filename=$1
    local ext="${filename##*.}"

    case $ext in
        txt|md|csv)
            echo "documents"
            ;;
        html|css|js)
            echo "web"
            ;;
        sh|py|rb)
            echo "scripts"
            ;;
        jpg|png|gif)
            echo "images"
            ;;
        log)
            echo "logs"
            ;;
        *)
            echo "others"
    esac   # バグ5
}

# ファイル整理のメイン処理
moved_count=0
error_count=0

for filepath in $SOURCE_DIR/*; do   # バグ6（バグ3と同じ問題）
    if [ -f "$filepath" ]; then
        filename=$(basename "$filepath")
        category=$(classify_file "$filename")

        # カテゴリ別ディレクトリを作成
        target_dir="${SOURCE_DIR}/${category}"
        mkdir -p "$target_dir"

        # ファイルを移動
        mv $filepath "$target_dir/"   # バグ7
        if [ $? -eq 0 ]; then
            echo "  移動: $filename → ${category}/"
            moved_count=$((moved_count + 1))
        else
            echo "  エラー: $filename の移動に失敗"
            error_count=$((error_count + 1))
        fi
    fi
done

echo ""
echo "===== 整理完了 ====="
echo "移動したファイル: $moved_count"
echo "エラー: $error_count"

# 整理後のディレクトリ構造を表示
echo ""
echo "--- 整理後のディレクトリ構造 ---"
for dir in $SOURCE_DIR/*/; do   # バグ8
    if [ -d "$dir" ]; then
        dir_name=$(basename "$dir")
        count=ls "$dir" | wc -l   # バグ9
        echo "  ${dir_name}/ (${count} ファイル)"
        ls "$dir" | while read file; do
            echo "    - $file"
        done
    fi
done

# ログ出力
LOG_FILE="${SOURCE_DIR}/organize.log"
echo "[$(date)] 整理完了 - 移動:${moved_count} エラー:${error_count}" > "$LOG_FILE"
echo ""
echo "ログ: $LOG_FILE"   # バグ10
```

| # | 場所 | ヒント |
|---|---|---|

## 解答例

<details>
<summary>全10個のバグと修正方法</summary>

### バグ1：引数チェックがない

```bash
# 修正前
SOURCE_DIR=$1

# 修正後
if [ -z "$1" ]; then
    echo "使い方: $0 <ディレクトリパス>"
    exit 1
fi
SOURCE_DIR="$1"
```

引数なしで実行すると `$1` が空になり、後続の処理で予期しない動作をします。

### バグ2：変数のクォート漏れ（test文）

```bash
# 修正前
if [ ! -d $SOURCE_DIR ]; then

# 修正後
if [ ! -d "$SOURCE_DIR" ]; then
```

パスにスペースが含まれる場合、クォートなしだと複数の引数として解釈されます。

### バグ3・6：forループのグロブにクォートがない

```bash
# 修正前
for f in $SOURCE_DIR/*; do

# 修正後
for f in "$SOURCE_DIR"/*; do
```

`$SOURCE_DIR` にスペースが含まれる場合に問題になります。ただし `/*` の部分はクォートの外に出す必要があります。

### バグ4：数値の加算が文字列連結になっている

```bash
# 修正前
file_count=$file_count+1     # "0+1+1+1..." という文字列になる

# 修正後
file_count=$((file_count + 1))  # 算術展開を使用
```

### バグ5：関数の閉じ括弧がない

```bash
# 修正前
    esac

# 修正後
    esac
}
```

`classify_file()` 関数の定義が `}` で閉じられていません。

### バグ7：mvコマンドのクォート漏れ

```bash
# 修正前
mv $filepath "$target_dir/"

# 修正後
mv "$filepath" "$target_dir/"
```

スペースや特殊文字を含むファイル名を正しく扱うにはクォートが必要です。

### バグ8：ディレクトリ一覧のクォート漏れ

```bash
# 修正前
for dir in $SOURCE_DIR/*/; do

# 修正後
for dir in "$SOURCE_DIR"/*/; do
```

### バグ9：コマンド置換の構文エラー

```bash
# 修正前
count=ls "$dir" | wc -l       # 代入ではなくコマンド実行になる

# 修正後
count=$(ls "$dir" | wc -l)    # $() でコマンド置換
```

### バグ10：ログファイルが整理対象になる

```bash
# 修正前（ログを対象ディレクトリ内に作成）
LOG_FILE="${SOURCE_DIR}/organize.log"

# 修正後（ログを対象ディレクトリの外に作成、または整理前にログを書く）
LOG_FILE="/tmp/organize_$(date '+%Y%m%d_%H%M%S').log"
```

ログファイルが `SOURCE_DIR` 内に作成されると、次回実行時にログ自体が整理対象になってしまいます。
