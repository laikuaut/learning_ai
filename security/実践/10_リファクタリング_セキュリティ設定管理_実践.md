# 実践課題10：リファクタリング ─ セキュリティ設定管理 ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第1章（情報セキュリティの基本）、第5章（ネットワークセキュリティ）、第6章（セキュリティマネジメント）
> **課題の種類**: リファクタリング
> **学習目標**: 「動くが保守しにくい」セキュリティ設定管理コードを、安全で保守しやすい構造にリファクタリングする力を養う

---

## 課題の説明

以下は「セキュリティ設定チェッカー」のコードです。サーバーのセキュリティ設定を検査してレポートを出力します。**正しく動作しますが、以下の問題を抱えています。**

- 全処理が1つの巨大な関数に詰め込まれている
- 設定値がハードコードされていて変更しにくい
- 同じようなチェック処理がコピー&ペーストで繰り返されている
- エラー処理がない
- 検査結果のフォーマットが統一されていない
- 新しい検査項目の追加が困難

**このコードを段階的にリファクタリングしてください。**

### ゴール

1. **ステップ1**: 検査項目ごとに関数を分離する
2. **ステップ2**: 設定値を定数や辞書で管理する
3. **ステップ3**: 検査結果を統一的なデータ構造で管理する

# セキュリティ設定チェッカー（改良版）
# 学べる内容：クラス設計、戦略パターン、拡張性のある設計
# 実行方法：python security_checker_v2.py

from datetime import datetime


class CheckResult:
    """1件の検査結果を表すクラスです。"""
    OK = "OK"
    WARN = "WARN"
    NG = "NG"

    def __init__(self, category, item, level, message):
        self.category = category
        self.item = item
        self.level = level
        self.message = message

    def __str__(self):
        return f"[{self.level:<4}] {self.item}: {self.message}"


class SecurityChecker:
    """セキュリティ検査を管理するクラスです。"""

    def __init__(self):
        self.results = []
        self.checks = []  # 登録された検査関数のリスト

    def add_check(self, check_func):
        """検査関数を登録します。"""
        self.checks.append(check_func)

    def run_all(self):
        """登録されたすべての検査を実行します。"""
        self.results = []
        for check_func in self.checks:
            try:
                results = check_func()
                self.results.extend(results)
            except Exception as e:
                self.results.append(CheckResult(
                    "エラー", check_func.__name__, CheckResult.NG,
                    f"検査中にエラー: {e}"
                ))

    def get_summary(self):
        """検査結果のサマリを返します。"""
        ok = sum(1 for r in self.results if r.level == CheckResult.OK)
        warn = sum(1 for r in self.results if r.level == CheckResult.WARN)
        ng = sum(1 for r in self.results if r.level == CheckResult.NG)
        total = ok + warn + ng

        if ng == 0:
            verdict = "合格"
        elif ng <= 2:
            verdict = "要改善"
        else:
            verdict = "不合格"

        return {"ok": ok, "warn": warn, "ng": ng, "total": total, "verdict": verdict}

    def print_report(self):
        """レポートを表示します。"""
        print("=" * 50)
        print("  セキュリティ設定チェッカー")
        print(f"  実行日時: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")
        print("=" * 50)

        # カテゴリごとにグループ化
        categories = {}
        for r in self.results:
            categories.setdefault(r.category, []).append(r)

        for category, items in categories.items():
            print(f"\n--- {category} ---")
            for item in items:
                print(f"  {item}")

        # サマリ
        s = self.get_summary()
        print("\n" + "=" * 50)
        print(f"  合格: {s['ok']}  警告: {s['warn']}  不合格: {s['ng']}  計: {s['total']}")
        print(f"  総合判定: {s['verdict']}")
        print("=" * 50)

    def export_report(self, filename):
        """レポートをファイルに出力します。"""
        lines = []
        lines.append("セキュリティ設定チェッカー レポート")
        lines.append(f"実行日時: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")
        lines.append("")

        for r in self.results:
            lines.append(f"[{r.level}] {r.category} > {r.item}: {r.message}")

        s = self.get_summary()
        lines.append("")
        lines.append(f"合格: {s['ok']} / 警告: {s['warn']} / 不合格: {s['ng']}")
        lines.append(f"総合判定: {s['verdict']}")

        with open(filename, "w", encoding="utf-8") as f:
            f.write("\n".join(lines))
        print(f"\nレポートを {filename} に保存しました。")

        ("/etc/ssh/sshd_config", 0o600),

    simulated = {"/etc/passwd": 0o644, "/etc/shadow": 0o666, "/etc/ssh/sshd_config": 0o600}
    results = []

    for path, expected in configs:
        actual = simulated.get(path, 0o644)
        if actual == expected:
            results.append(CheckResult(
                category, path, CheckResult.OK, f"正常 ({oct(actual)})"))
        else:
            results.append(CheckResult(
                category, path, CheckResult.NG,
                f"不正 (期待: {oct(expected)}, 実際: {oct(actual)})"))

    return results

**レガシー版との違い:**
- `CheckResult` クラスで検査結果を統一的に管理 → 集計やフィルタリングが容易
- `SecurityChecker` クラスで検査の実行とレポート生成を分離 → テストしやすい
- 検査関数をプラグイン的に `add_check()` で登録 → 新しい検査項目の追加が1関数分の作業で済む
- `try-except` でエラーが発生しても他の検査を続行 → 堅牢な設計
- 辞書ベースの設定（`DANGEROUS_PORTS`, `tls_levels`）→ ハードコードされた `if-elif` の連鎖を排除
