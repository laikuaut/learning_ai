# 第2章：OSI参照モデルとTCP/IP 演習問題

## 基本問題

### 問題2-1：プロトコルと層の対応
以下のプロトコルが動作するOSI参照モデルの層を答えてください。

1. HTTP
2. TCP
3. IP
4. Ethernet
5. FTP
6. UDP
7. ARP
8. ICMP

<details>
<summary>解答</summary>

1. HTTP → 第7層（アプリケーション層）
2. TCP → 第4層（トランスポート層）
3. IP → 第3層（ネットワーク層）
4. Ethernet → 第2層（データリンク層）
5. FTP → 第7層（アプリケーション層）
6. UDP → 第4層（トランスポート層）
7. ARP → 第2層/第3層の境界（MACアドレスとIPアドレスを対応付ける）
8. ICMP → 第3層（ネットワーク層）

</details>

### 問題2-2：ウェルノウンポート
以下のプロトコルのデフォルトポート番号を答えてください。

1. HTTP
2. HTTPS
3. FTP（データ/制御）
4. SSH
5. DNS
6. SMTP
7. POP3
8. DHCP（サーバ/クライアント）

<details>
<summary>解答</summary>

1. HTTP → **80**
2. HTTPS → **443**
3. FTP → **20**（データ）/ **21**（制御）
4. SSH → **22**
5. DNS → **53**
6. SMTP → **25**
7. POP3 → **110**
8. DHCP → **67**（サーバ側）/ **68**（クライアント側）

ポート番号の範囲：
- ウェルノウンポート：0〜1023
- 登録済みポート：1024〜49151
- 動的/プライベートポート：49152〜65535

</details>

### 問題2-3：ネットワーク機器と対応層
以下のネットワーク機器が主に動作するOSI層を答えてください。

1. リピータ / ハブ
2. L2スイッチ / ブリッジ
3. ルータ / L3スイッチ
4. ファイアウォール（L4）
5. ロードバランサー（L7）

<details>
<summary>解答</summary>

1. リピータ / ハブ → **第1層（物理層）** ：電気信号を増幅・中継
2. L2スイッチ / ブリッジ → **第2層（データリンク層）** ：MACアドレスを基にフレームを転送
3. ルータ / L3スイッチ → **第3層（ネットワーク層）** ：IPアドレスを基にパケットをルーティング
4. ファイアウォール（L4） → **第4層（トランスポート層）** ：ポート番号を基にフィルタリング
5. ロードバランサー（L7） → **第7層（アプリケーション層）** ：HTTPの内容等を基に振り分け

</details>

### 問題2-4：プロトコルの3要素
プロトコルの3要素を挙げ、それぞれHTTPを例に具体的に説明してください。

<details>
<summary>解答</summary>

1. **構文（Syntax）**：データの形式・構造
   - HTTPの場合：リクエスト行（メソッド、URI、バージョン）、ヘッダフィールドの書式、空行、ボディの構造

2. **意味（Semantics）**：各フィールドの意味
   - HTTPの場合：GETは「リソースの取得」、POSTは「データの送信」、ステータスコード200は「成功」、404は「未検出」

3. **タイミング（Timing）**：送受信のタイミング・順序
   - HTTPの場合：クライアントがリクエストを送信し、サーバがレスポンスを返す（リクエスト/レスポンスモデル）。HTTP/1.1ではリクエストの順序通りにレスポンスを返す

</details>

## 応用問題

### 問題2-5：通信の流れ
PCからWebサーバにアクセスする際、ブラウザのアドレスバーにURLを入力してからWebページが表示されるまでの通信の流れを、OSI参照モデルの各層の処理と対応付けて説明してください。

<details>
<summary>解答</summary>

**送信側（PC）の処理：**

1. **第7層（アプリケーション層）**：ブラウザがHTTP GETリクエストを作成。まずDNSで宛先のIPアドレスを解決
2. **第4層（トランスポート層）**：TCPヘッダを付加（送信元ポート：ランダム、宛先ポート：80）。3ウェイハンドシェイクでコネクション確立
3. **第3層（ネットワーク層）**：IPヘッダを付加（送信元IP、宛先IP）。ルーティングテーブルを参照して次ホップを決定
4. **第2層（データリンク層）**：MACヘッダを付加（送信元MAC、次ホップのMAC）。ARPで次ホップのMACアドレスを取得。FCSを付加
5. **第1層（物理層）**：フレームを電気信号に変換してケーブルに送出

**受信側（サーバ）の処理：**

逆の順序で各層のヘッダを確認・除去し、最終的にHTTPリクエストを取得。HTMLデータを含むHTTPレスポンスを同様にカプセル化して返送する。

</details>

### 問題2-6：ルータの動作
PCが異なるネットワークにあるサーバにアクセスする際、ルータはどのように動作しますか。MACアドレスとIPアドレスの変化に着目して説明してください。

```
[PC-A] ──── [ルータ] ──── [サーバ]
192.168.1.10   │   │    10.0.0.20
MAC-A       MAC-R1 MAC-R2   MAC-S
```

<details>
<summary>解答</summary>

**ルータの動作：**

1. PC-Aがパケットを送信する際：
   - **IPヘッダ**：送信元IP = 192.168.1.10、宛先IP = 10.0.0.20（**変化しない**）
   - **MACヘッダ**：送信元MAC = MAC-A、宛先MAC = MAC-R1（デフォルトゲートウェイのMAC）

2. ルータがパケットを受信して転送する際：
   - **IPヘッダ**：送信元IP = 192.168.1.10、宛先IP = 10.0.0.20（**変化しない**）
   - **MACヘッダ**：送信元MAC = MAC-R2、宛先MAC = MAC-S（**書き換えられる**）

**重要なポイント：**
- IPアドレスはエンドツーエンドで変化しない（NATを使わない場合）
- MACアドレスはルータを通過するたびに書き換えられる（ホップごとに変化）
- ルータはIPヘッダの宛先を見てルーティングテーブルを参照し、適切なインターフェースから送出する

</details>

## チャレンジ問題

### 問題2-7：トラブルシューティング
社内のPCからインターネット上のWebサイトにアクセスできない場合、OSI参照モデルの下位層から順に確認すべき項目を挙げてください。

<details>
<summary>解答例</summary>

**第1層（物理層）の確認：**
- LANケーブルが正しく接続されているか
- NICのLEDが点灯しているか
- ケーブルの断線はないか

**第2層（データリンク層）の確認：**
- スイッチのポートが有効か
- MACアドレスがスイッチに学習されているか
- `arp -a` で同一ネットワーク内のMACアドレスが解決できるか

**第3層（ネットワーク層）の確認：**
- IPアドレスとサブネットマスクが正しく設定されているか（`ipconfig` / `ifconfig`）
- デフォルトゲートウェイが正しいか
- `ping` でデフォルトゲートウェイに到達できるか
- `ping` でDNSサーバに到達できるか
- `ping` で外部サイトのIPアドレスに到達できるか

**第4層（トランスポート層）の確認：**
- ファイアウォールでポート80/443がブロックされていないか
- `telnet ホスト 80` などでポートの疎通を確認

**第7層（アプリケーション層）の確認：**
- DNS名前解決が正常に動作しているか（`nslookup`）
- プロキシ設定が正しいか
- ブラウザの設定に問題がないか

このように下位層から順番に確認することで、問題箇所を効率的に特定できます。

</details>

### 問題2-8：プロトコル選択
以下の各シナリオで使用されるプロトコルとポート番号を答えてください。

1. 自宅のPCからリモートサーバに安全にログインしたい
2. メールを送信したい
3. ドメイン名からIPアドレスを調べたい
4. IPアドレスを自動的に取得したい
5. ネットワーク機器の時刻を正確に同期したい

<details>
<summary>解答</summary>

1. **SSH**（ポート22）：暗号化されたリモートアクセスプロトコル。Telnet（ポート23）は平文のため現在は使用を避ける
2. **SMTP**（ポート25、または587）：メール送信プロトコル。ポート587はサブミッションポート（認証付き送信）
3. **DNS**（ポート53）：ドメイン名とIPアドレスの相互変換。UDP（通常の問い合わせ）とTCP（ゾーン転送等）を使用
4. **DHCP**（サーバ：ポート67、クライアント：ポート68）：IPアドレス、サブネットマスク、デフォルトゲートウェイ、DNSサーバ等を自動配布
5. **NTP**（ポート123）：ネットワーク上の機器の時刻を正確に同期するプロトコル

</details>
