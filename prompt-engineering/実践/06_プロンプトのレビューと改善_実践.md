## 課題の要件

### レビュー対象プロンプト（6本）

以下のプロンプトにはそれぞれ改善すべき点が含まれています。各プロンプトをレビューし、改善してください。

---

**プロンプト①（ビジネスメール生成）**:
```
メールを書いてください。
取引先に送ります。
新しい製品の紹介です。
丁寧にお願いします。
```

**プロンプト②（コードレビュー依頼）**:
```
あなたは世界最高のプログラマーです。
100%完璧なコードレビューをしてください。
バグは絶対に見逃さないでください。
以下のコードをチェックしてください。

def calc(x, y):
    return x / y
```

**プロンプト③（マーケティング戦略）**:
```
あなたはマーケティングの専門家です。新しいスマートフォンアプリのマーケティング戦略を考えてください。ターゲットは若者です。予算は少なめです。SNSを使いたいです。いい感じの戦略をお願いします。できるだけ詳しく教えてください。あと、競合分析もしてください。KPIも設定してください。タイムラインも作ってください。リスク分析もお願いします。
```

**プロンプト④（技術解説）**:
```
機械学習とディープラーニングとAIの違いを説明してください。
それぞれのメリット・デメリットも教えてください。
具体例も出してください。
将来の展望も述べてください。
初心者向けにお願いします。
でも専門的な内容も入れてください。
```

**プロンプト⑤（翻訳依頼）**:
```
以下の文章を翻訳してください。

弊社は2020年に設立され、AIを活用したソリューションを提供しています。
主力製品である「SmartAssist」は、カスタマーサポートの効率化を実現し、

```

**プロンプト⑥（データ分析）**:
```
あなたはデータ分析の専門家です。

以下のデータを分析してください。

月: 1月, 2月, 3月, 4月, 5月, 6月
売上: 100, 95, 110, 108, 130, 125
顧客数: 50, 48, 55, 52, 65, 60

何かインサイトをください。
```

### 評価基準（5つの観点）

各プロンプトを以下の5つの基準で5段階評価してください。

| 評価基準 | 観点 |
|---------|------|
| **明確性（Clarity）** | 指示が具体的で曖昧さがないか |
| **完全性（Completeness）** | 必要な情報（文脈・制約・形式）が含まれているか |
| **構造性（Structure）** | 情報が論理的に整理されているか |
| **現実性（Realism）** | 実現可能な指示になっているか（過度な期待がないか） |
| **効率性（Efficiency）** | トークンの無駄なく、簡潔に書かれているか |

### 各プロンプトで実施すること

1. 5つの基準で1〜5のスコアをつける
2. 問題点を3つ以上具体的に指摘する
3. 改善提案を理由とともに記述する
4. 改善版プロンプトを作成する
5. 改善前と改善後の出力を比較する

## ステップガイド

<details>
<summary>ステップ1：評価スコアリングの方法</summary>

各基準のスコアリング基準は以下の通りです。

| スコア | 意味 | 判断基準 |
|:------:|------|---------|
| 5 | 優秀 | 改善の余地がほぼない |
| 4 | 良好 | 小さな改善点が1〜2個ある |
| 3 | 普通 | いくつかの改善点がある |
| 2 | 不十分 | 大きな改善が必要 |
| 1 | 要改善 | 根本的な見直しが必要 |

**プロンプト①の評価例**:

| 基準 | スコア | 理由 |
|------|:------:|------|
| 明確性 | 2 | 「丁寧に」が主観的、具体的な指示がない |
| 完全性 | 1 | 製品情報、宛先、目的の詳細が欠如 |
| 構造性 | 2 | 箇条書きだが論理的な順序がない |
| 現実性 | 3 | 実現可能だが出力品質は低い |
| 効率性 | 3 | 短いが、短すぎて情報不足 |

</details>

<details>
<summary>ステップ2：問題点の分析パターン</summary>

よくある問題パターンとその見つけ方を紹介します。

**パターン1：情報不足**
- 文脈が足りない（誰に？何のために？）
- 制約条件がない（文字数、形式、トーンなど）
- プロンプト⑤が典型例（翻訳先の言語すら未指定）

**パターン2：過剰な期待**
- 「100%完璧」「絶対に見逃さない」等の非現実的な要求
- 1つのプロンプトに多すぎるタスクを詰め込む
- プロンプト②、③が典型例

**パターン3：矛盾する指示**
- 「初心者向けだが専門的に」のような相反する要求
- プロンプト④が典型例

**パターン4：構造の欠如**
- 指示が1文に詰め込まれている
- 優先順位が不明
- プロンプト③が典型例

**パターン5：出力形式の未定義**
- どのような形式で返してほしいかが不明
- プロンプト⑥が典型例（「何かインサイトを」は曖昧）

</details>

<details>
<summary>ステップ3：改善版を作成する</summary>

改善する際は以下の手順で進めましょう。

1. **問題点の優先順位を決める**: 最も影響が大きい問題から改善する
2. **不足情報を補う**: 文脈、制約、出力形式を追加する
3. **構造を整理する**: セクション分け、番号付けで可読性を向上させる
4. **矛盾を解消する**: 相反する指示を統合または優先順位をつける
5. **現実的な期待に調整する**: 過剰な要求を実行可能なレベルに

プロンプト②を例にすると：
- 「世界最高」「100%完璧」「絶対に」→ 非現実的な期待を除去
- コードレビューの観点（セキュリティ、パフォーマンス等）を具体化
- 出力形式（指摘ごとに深刻度・修正案を記載）を追加

</details>

<details>
<summary>ステップ4：Before/After比較をまとめる</summary>

各プロンプトについて、以下の形式で比較を記録しましょう。

```markdown
### プロンプト○：△△△

**評価スコア**: 明確性○ / 完全性○ / 構造性○ / 現実性○ / 効率性○（合計○/25）

**問題点**:
1. ...
2. ...
3. ...

**改善版プロンプト**: （改善版を記載）

**出力の変化**:
- Before: （改善前の出力の特徴を要約）
- After: （改善後の出力の特徴を要約）
- 改善効果: （具体的にどう良くなったか）
```

</details>

### プロンプト①（ビジネスメール生成）のレビュー

**評価スコア**:

| 基準 | スコア |
|------|:------:|
| 明確性 | 2 |
| 完全性 | 1 |
| 構造性 | 2 |
| 現実性 | 3 |
| 効率性 | 3 |
| **合計** | **11/25** |

**問題点**:
1. 宛先の情報がない（どの取引先の誰に送るのか）
2. 製品の具体的な情報がない（製品名、特徴、価格帯など）
3. 「丁寧にお願いします」が主観的（どの程度の丁寧さか不明）
4. メールの目的が不明確（情報提供のみ？商談の申し込み？）
5. 出力形式の指定がない（件名は必要か、署名は含めるかなど）

**改善版プロンプト**:
```
以下の条件でビジネスメールを作成してください。

【宛先】
- 株式会社ABC商事 営業部 山田太郎様（過去に3回の取引実績あり）

【目的】
- 新製品「SmartWidget Pro」の紹介と、デモ実施のご案内

【製品情報】
- 製品名：SmartWidget Pro
- 特徴：従来品と比べて処理速度が2倍、消費電力30%削減
- 価格帯：月額5万円〜（プランにより変動）
- リリース日：2026年5月1日

【メールの条件】
- トーン：丁寧なビジネス敬語（既存取引先への親近感も含める）
- 長さ：300文字程度
- 構成：件名 → 挨拶 → 製品紹介 → デモのご案内 → 結び → 署名
- 署名の名前は「鈴木一郎」
```

**出力の変化**:
- Before: 汎用的で内容の薄いメール、宛先も製品名も不明なまま
- After: 具体的な製品情報を含み、デモ案内まで含む実用的なメール
- 改善効果: そのまま送信に近い品質のメールが生成された

### プロンプト③（マーケティング戦略）のレビュー（改良版）

**評価スコア**:

| 基準 | スコア | 詳細 |
|------|:------:|------|
| 明確性 | 2 | 「いい感じの」が主観的、ターゲットの「若者」が曖昧 |
| 完全性 | 3 | 情報はあるが具体性が不足 |
| 構造性 | 1 | すべて1段落に詰め込まれ、優先順位不明 |
| 現実性 | 2 | 1つのプロンプトで7つのタスクを要求（過剰） |
| 効率性 | 2 | 冗長で整理されていない |
| **合計** | **10/25** |

**問題点の詳細分析**:

1. **構造の欠如（最重要）**: 7つの要求（戦略・SNS活用・競合分析・KPI・タイムライン・リスク分析・詳細説明）が1段落に詰め込まれています。これでは各タスクの境界が不明確で、LLMがどこに注力すべきか判断できません。

2. **具体性の不足**: 
   - 「若者」→ 年齢層は？（15〜20歳と25〜30歳では戦略が全く異なる）
   - 「少なめの予算」→ 月10万円？100万円？
   - 「スマートフォンアプリ」→ ジャンルは？（ゲーム？ツール？SNS？）

3. **タスクの過剰さ**: 戦略、競合分析、KPI設定、タイムライン、リスク分析はそれぞれ独立したタスクです。1つのプロンプトで全てを高品質に出力させるのは無理があります。

4. **矛盾する指示**: 「SNSを使いたい」（手段の限定）と「マーケティング戦略を考えて」（自由な立案）が矛盾する可能性があります。

**改善戦略**: 1つの巨大プロンプトを**3段階のプロンプトチェーン**（prompt chain）に分割します。

**改善版プロンプト（3段階構成）**:

**ステップ1：状況分析と戦略の方向性**
```
あなたはモバイルアプリのマーケティング戦略の専門家です。

以下の条件に基づいて、マーケティング戦略の方向性を提案してください。

【アプリ情報】
- カテゴリ：家計簿アプリ
- 主な機能：AI自動分類、支出予測、貯蓄目標管理
- 対象プラットフォーム：iOS / Android
- リリース済み（ユーザー数：月間5,000人、成長率：月10%）

【ターゲット】
- 年齢：22〜30歳の社会人
- 特徴：初めての一人暮らし、家計管理に不慣れ、SNS利用率が高い

【予算】
- マーケティング月額予算：30万円
- 期間：3ヶ月

【出力形式】
1. ターゲットのペルソナ（1人分、200文字以内）
2. 戦略の方向性（3つの案を箇条書きで、各50文字以内）
3. 推奨案とその理由（100文字以内）
```

**ステップ2：具体的な施策とKPI**
```
（ステップ1の出力結果を貼り付けた上で）

上記の戦略方向性に基づいて、以下を作成してください。

1. SNS施策の詳細（使用するSNS、投稿頻度、コンテンツ種類を表形式で）
2. KPI設定（指標名、目標値、測定方法を表形式で、3〜5個）
3. 月別タイムライン（3ヶ月分を表形式で）
```

**ステップ3：リスク分析と競合比較**
```
（ステップ1・2の結果を踏まえて）

1. 競合アプリ3つとの差別化ポイントを表形式で整理してください
2. 上記の戦略におけるリスクを3つ挙げ、各リスクに対する
   対応策を記述してください
```

### プロンプト④（技術解説）のレビュー（改良版）

**問題点の核心**: 「初心者向けにお願いします。でも専門的な内容も入れてください。」は矛盾する指示です。

**改善戦略**: 矛盾を解消するために「レイヤー構造」を採用します。

**改善版プロンプト**:
```
機械学習、ディープラーニング、AIの違いを説明してください。

【構成】2層構造で記述してください。

■ レイヤー1：初心者向けの概要（各技術100文字以内）
- 日常的なたとえ話を1つ使う
- 専門用語は使わない

■ レイヤー2：もう少し詳しく（各技術200文字以内）
- 技術的な仕組みを1つ説明する
- 専門用語は初出時に（ ）で補足する

【比較表】
以下の列を持つ表を作成してください。
| 項目 | AI | 機械学習 | ディープラーニング |
- 定義（1文）
- 具体例（1つ）
- 得意なこと
- 苦手なこと

【注意】
- 3つの概念の包含関係（AI ⊃ 機械学習 ⊃ ディープラーニング）を最初に図示
- 「将来の展望」は含めない（範囲を限定）
```

**改善ポイント**:
- 矛盾する「初心者向け」と「専門的」を2層構造で両立
- 「メリット・デメリット」の曖昧な要求を、比較表の具体的な列に変換
- 「将来の展望」は範囲外として明示的に除外（スコープの制御）
