### 問題1：Result型の基本操作

文字列を整数に変換する関数 `parse_number` を作成してください。変換に成功したら値を、失敗したらエラーメッセージを返します。さらに、`unwrap`、`expect`、`match` の3通りの方法でResult型を処理してください。

**期待される出力例：**
```
=== parse_number のテスト ===
"42" → Ok(42)
"hello" → Err(無効な数値です: hello)
"" → Err(空文字列です)
"999999999999999999999" → Err(数値が大きすぎます)

=== 3つの処理方法 ===
match: 42を取得しました
unwrap_or: デフォルト値 0 を使用
expect: パニックを避けてunwrap_orを使用 → 0
```

<details><summary>ヒント</summary>

- `Result<T, E>` は `Ok(T)` または `Err(E)` のどちらかを持ちます
- `str::parse::<i64>()` は `Result<i64, ParseIntError>` を返します
- `match` で `Ok(value)` と `Err(e)` をパターンマッチできます
- `unwrap_or(default)` はエラー時にデフォルト値を返します
- `is_empty()` で空文字列を判定できます

</details>

<details><summary>解答例</summary>

```rust
// Result型の基本操作

fn parse_number(input: &str) -> Result<i64, String> {
    // 空文字列チェック
    if input.is_empty() {
        return Err("空文字列です".to_string());
    }

    // 文字列を整数に変換
    match input.parse::<i64>() {
        Ok(n) => Ok(n),
        Err(e) => {
            // エラーの種類に応じたメッセージ
            if input.len() > 18 {
                Err("数値が大きすぎます".to_string())
            } else {
                Err(format!("無効な数値です: {}", input))
            }
        }
    }
}

fn main() {
    // === parse_number のテスト ===
    println!("=== parse_number のテスト ===");
    let test_cases = vec!["42", "hello", "", "999999999999999999999"];

    for input in test_cases {
        let result = parse_number(input);
        match &result {
            Ok(n) => println!("\"{}\" → Ok({})", input, n),
            Err(e) => println!("\"{}\" → Err({})", input, e),
        }
    }

    println!();

    // === 3つの処理方法 ===
    println!("=== 3つの処理方法 ===");

    // 1. match で処理
    match parse_number("42") {
        Ok(value) => println!("match: {}を取得しました", value),
        Err(e) => println!("match: エラー - {}", e),
    }

    // 2. unwrap_or でデフォルト値を使用
    let value = parse_number("hello").unwrap_or(0);
    println!("unwrap_or: デフォルト値 {} を使用", value);

    // 3. expect の代わりに unwrap_or を使用（安全な方法）
    let value = parse_number("hello").unwrap_or(0);
    println!(
        "expect: パニックを避けてunwrap_orを使用 → {}",
        value
    );
}
```

**補足：** `unwrap()` と `expect()` はエラー時にパニック（プログラム異常終了）を起こすため、プロダクションコードでは避けるべきです。代わりに `match`、`unwrap_or()`、`unwrap_or_else()` を使うのが安全です。`expect` はデバッグ目的やプロトタイプで使われることがあります。

</details>

### 問題2：ファイル読み込みのエラーハンドリング

ファイルの読み込みをシミュレートする関数を作成してください。実際のファイルI/Oの代わりに、ファイル名に応じて成功・失敗を返す関数で練習します。

以下の関数を実装してください。
- `read_file`: ファイル名を受け取り、内容を返す（`Result<String, String>`）
- `read_first_line`: ファイルの最初の行を返す
- `count_words`: ファイル内の単語数を数える

**期待される出力例：**
```
=== ファイル読み込みテスト ===
hello.txt → 内容: "Hello, World!\nThis is a test file."
missing.txt → エラー: ファイルが見つかりません: missing.txt
secret.txt → エラー: アクセス権限がありません: secret.txt

=== 最初の行 ===
hello.txt の最初の行: "Hello, World!"

=== 単語数カウント ===
hello.txt の単語数: 8
```

<details><summary>ヒント</summary>

- ファイル名に応じて結果を返すモック関数を作ります
- `?` 演算子を使うと、エラーを呼び出し元に伝搬できます
- 文字列の分割には `.lines()` や `.split_whitespace()` が使えます

</details>

<details><summary>解答例</summary>

```rust
// ファイル読み込みのエラーハンドリング（シミュレーション）

// ファイル読み込みシミュレーション
fn read_file(filename: &str) -> Result<String, String> {
    match filename {
        "hello.txt" => Ok("Hello, World!\nThis is a test file.".to_string()),
        "data.txt" => Ok("1,2,3\n4,5,6\n7,8,9".to_string()),
        "empty.txt" => Ok("".to_string()),
        "secret.txt" => {
            Err(format!("アクセス権限がありません: {}", filename))
        }
        _ => Err(format!("ファイルが見つかりません: {}", filename)),
    }
}

// 最初の行を返す（?演算子でエラー伝搬）
fn read_first_line(filename: &str) -> Result<String, String> {
    // ?演算子: read_fileがErrなら即座にErrを返す
    let content = read_file(filename)?;

    // 最初の行を取得
    content
        .lines()
        .next()
        .map(|line| line.to_string())
        .ok_or_else(|| "ファイルが空です".to_string())
}

// 単語数を数える（?演算子でエラー伝搬）
fn count_words(filename: &str) -> Result<usize, String> {
    let content = read_file(filename)?;
    Ok(content.split_whitespace().count())
}

fn main() {
    // === ファイル読み込みテスト ===
    println!("=== ファイル読み込みテスト ===");
    let filenames = vec!["hello.txt", "missing.txt", "secret.txt"];

    for filename in filenames {
        match read_file(filename) {
            Ok(content) => println!(
                "{} → 内容: \"{}\"",
                filename, content
            ),
            Err(e) => println!("{} → エラー: {}", filename, e),
        }
    }

    println!();

    // === 最初の行 ===
    println!("=== 最初の行 ===");
    match read_first_line("hello.txt") {
        Ok(line) => println!(
            "hello.txt の最初の行: \"{}\"",
            line
        ),
        Err(e) => println!("エラー: {}", e),
    }

    println!();

    // === 単語数カウント ===
    println!("=== 単語数カウント ===");
    match count_words("hello.txt") {
        Ok(count) => println!("hello.txt の単語数: {}", count),
        Err(e) => println!("エラー: {}", e),
    }
}
```

**補足：** `?` 演算子はRustのエラーハンドリングで最も重要な機能の1つです。`Result` が `Err` の場合に自動的にその `Err` を呼び出し元に返し、`Ok` の場合は中身の値を取り出します。これにより、エラー処理のボイラープレートコードを大幅に削減できます。

</details>

### 問題3：Option型とResult型の変換

`Option` 型と `Result` 型を相互に変換する練習をします。ユーザーの設定値を取得・変換する関数を作成してください。

**期待される出力例：**
```
=== 設定値の取得と変換 ===
"max_retries" → Ok(3)
"timeout" → Ok(30)
"missing_key" → Err(設定が見つかりません: missing_key)
"invalid" → Err(数値に変換できません: abc)

=== チェーン処理 ===
max_retries の2倍: 6
unknown の2倍: デフォルト値 0 を使用
```

<details><summary>ヒント</summary>

- `Option<T>` を `Result<T, E>` に変換するには `.ok_or()` や `.ok_or_else()` を使います
- `Result<T, E>` を `Option<T>` に変換するには `.ok()` を使います
- `and_then()` でチェーン処理ができます

</details>

<details><summary>解答例</summary>

```rust
// Option型とResult型の変換

use std::collections::HashMap;

// 設定値を保持する構造体
struct Config {
    settings: HashMap<String, String>,
}

impl Config {
    fn new() -> Self {
        let mut settings = HashMap::new();
        settings.insert("max_retries".to_string(), "3".to_string());
        settings.insert("timeout".to_string(), "30".to_string());
        settings.insert("invalid".to_string(), "abc".to_string());

        Config { settings }
    }

    // 設定値を文字列で取得（Option<&String>を返す）
    fn get_raw(&self, key: &str) -> Option<&String> {
        self.settings.get(key)
    }

    // 設定値を整数で取得（Result<i32, String>を返す）
    fn get_int(&self, key: &str) -> Result<i32, String> {
        // Option → Result に変換（ok_or_else）
        let value = self
            .get_raw(key)
            .ok_or_else(|| format!("設定が見つかりません: {}", key))?;

        // 文字列 → 整数に変換
        value
            .parse::<i32>()
            .map_err(|_| format!("数値に変換できません: {}", value))
    }

    // 設定値を取得して変換する（チェーン処理）
    fn get_int_doubled(&self, key: &str) -> Result<i32, String> {
        self.get_int(key).map(|n| n * 2)
    }
}

fn main() {
    let config = Config::new();

    // === 設定値の取得と変換 ===
    println!("=== 設定値の取得と変換 ===");

    let keys = vec!["max_retries", "timeout", "missing_key", "invalid"];
    for key in keys {
        match config.get_int(key) {
            Ok(value) => println!("\"{}\" → Ok({})", key, value),
            Err(e) => println!("\"{}\" → Err({})", key, e),
        }
    }

    println!();

    // === チェーン処理 ===
    println!("=== チェーン処理 ===");

    match config.get_int_doubled("max_retries") {
        Ok(value) => println!("max_retries の2倍: {}", value),
        Err(e) => println!("エラー: {}", e),
    }

    // unwrap_or でデフォルト値を使用
    let value = config.get_int_doubled("unknown").unwrap_or(0);
    println!("unknown の2倍: デフォルト値 {} を使用", value);
}
```

**補足：** `Option` と `Result` の変換は実務で頻繁に行います。主な変換メソッドは以下の通りです。

- `Option → Result`: `.ok_or(err)` / `.ok_or_else(|| err)`
- `Result → Option`: `.ok()` で `Ok` の値を `Some` に、`.err()` で `Err` の値を `Some` に変換
- `.map()`: 中身の値を変換
- `.and_then()`: 中身の値を使って新しい `Result`/`Option` を返す
- `.map_err()`: エラーの型を変換

</details>

### 問題4：?演算子とカスタムエラー型

設定ファイル（INI形式）のパーサーを作成してください。カスタムエラー型を定義し、`?` 演算子でエラーを伝搬させます。

INI形式の例：
```
[server]
host=localhost
port=8080

[database]
name=mydb
port=5432
```

**期待される出力例：**
```
=== 設定ファイルパーサー ===

有効な設定ファイル:
  [server] host = "localhost"
  [server] port = "8080"
  [database] name = "mydb"
  [database] port = "5432"

=== エラーケース ===
空の入力: EmptyInput
不正な行: InvalidLine(行 1: "not a valid line")
不正なセクション: InvalidSection(行 1: "[unclosed")

=== 値の取得 ===
server.host = "localhost"
server.port = 8080
database.unknown → キーが見つかりません: unknown
```

<details><summary>ヒント</summary>

- `enum` でカスタムエラー型を定義します
- `std::fmt::Display` を実装するとエラーメッセージを表示できます
- `HashMap<String, HashMap<String, String>>` でセクション→キー→値を管理します
- 各行を `.trim()` して空行やコメント行を飛ばします

</details>

<details><summary>解答例</summary>

```rust
// INI形式の設定ファイルパーサー
// カスタムエラー型と?演算子の活用

use std::collections::HashMap;
use std::fmt;

// カスタムエラー型
#[derive(Debug)]
enum ConfigError {
    EmptyInput,
    InvalidLine(usize, String),      // (行番号, 内容)
    InvalidSection(usize, String),   // (行番号, 内容)
    KeyNotFound(String),             // キー名
    SectionNotFound(String),         // セクション名
    ParseError(String, String),      // (キー名, エラー内容)
}

// Displayトレイトの実装
impl fmt::Display for ConfigError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            ConfigError::EmptyInput => write!(f, "EmptyInput"),
            ConfigError::InvalidLine(line, content) => {
                write!(f, "InvalidLine(行 {}: \"{}\")", line, content)
            }
            ConfigError::InvalidSection(line, content) => {
                write!(f, "InvalidSection(行 {}: \"{}\")", line, content)
            }
            ConfigError::KeyNotFound(key) => {
                write!(f, "キーが見つかりません: {}", key)
            }
            ConfigError::SectionNotFound(section) => {
                write!(f, "セクションが見つかりません: {}", section)
            }
            ConfigError::ParseError(key, detail) => {
                write!(f, "変換エラー: {} ({})", key, detail)
            }
        }
    }
}

// 設定データ構造体
struct IniConfig {
    sections: HashMap<String, HashMap<String, String>>,
}

impl IniConfig {
    // INI形式の文字列をパース
    fn parse(input: &str) -> Result<Self, ConfigError> {
        let input = input.trim();
        if input.is_empty() {
            return Err(ConfigError::EmptyInput);
        }

        let mut sections: HashMap<String, HashMap<String, String>> =
            HashMap::new();
        let mut current_section = String::new();

        for (i, line) in input.lines().enumerate() {
            let line = line.trim();
            let line_num = i + 1;

            // 空行とコメントをスキップ
            if line.is_empty() || line.starts_with('#') || line.starts_with(';') {
                continue;
            }

            // セクションヘッダー [section]
            if line.starts_with('[') {
                if !line.ends_with(']') {
                    return Err(ConfigError::InvalidSection(
                        line_num,
                        line.to_string(),
                    ));
                }
                current_section =
                    line[1..line.len() - 1].trim().to_string();
                sections
                    .entry(current_section.clone())
                    .or_insert_with(HashMap::new);
                continue;
            }

            // キー=値
            if let Some(pos) = line.find('=') {
                let key = line[..pos].trim().to_string();
                let value = line[pos + 1..].trim().to_string();

                if current_section.is_empty() {
                    return Err(ConfigError::InvalidLine(
                        line_num,
                        line.to_string(),
                    ));
                }

                sections
                    .get_mut(&current_section)
                    .unwrap()
                    .insert(key, value);
            } else {
                return Err(ConfigError::InvalidLine(
                    line_num,
                    line.to_string(),
                ));
            }
        }

        Ok(IniConfig { sections })
    }

    // 文字列値を取得
    fn get(
        &self,
        section: &str,
        key: &str,
    ) -> Result<&String, ConfigError> {
        let sec = self
            .sections
            .get(section)
            .ok_or_else(|| {
                ConfigError::SectionNotFound(section.to_string())
            })?;

        sec.get(key)
            .ok_or_else(|| ConfigError::KeyNotFound(key.to_string()))
    }

    // 整数値を取得
    fn get_int(
        &self,
        section: &str,
        key: &str,
    ) -> Result<i32, ConfigError> {
        let value = self.get(section, key)?;
        value.parse::<i32>().map_err(|e| {
            ConfigError::ParseError(key.to_string(), e.to_string())
        })
    }

    // 全設定を表示
    fn display_all(&self) {
        for (section, values) in &self.sections {
            for (key, value) in values {
                println!("  [{}] {} = \"{}\"", section, key, value);
            }
        }
    }
}

fn main() {
    // === 有効な設定ファイル ===
    println!("=== 設定ファイルパーサー ===\n");

    let valid_input = r#"
[server]
host=localhost
port=8080

[database]
name=mydb
port=5432
"#;

    println!("有効な設定ファイル:");
    match IniConfig::parse(valid_input) {
        Ok(config) => {
            config.display_all();

            // === 値の取得 ===
            println!("\n=== 値の取得 ===");

            match config.get("server", "host") {
                Ok(value) => {
                    println!("server.host = \"{}\"", value)
                }
                Err(e) => println!("エラー: {}", e),
            }

            match config.get_int("server", "port") {
                Ok(value) => println!("server.port = {}", value),
                Err(e) => println!("エラー: {}", e),
            }

            match config.get("database", "unknown") {
                Ok(value) => {
                    println!("database.unknown = \"{}\"", value)
                }
                Err(e) => println!("database.unknown → {}", e),
            }
        }
        Err(e) => println!("パースエラー: {}", e),
    }

    // === エラーケース ===
    println!("\n=== エラーケース ===");

    // 空の入力
    match IniConfig::parse("") {
        Err(e) => println!("空の入力: {}", e),
        Ok(_) => println!("成功（想定外）"),
    }

    // 不正な行
    match IniConfig::parse("[section]\nnot a valid line") {
        Err(e) => println!("不正な行: {}", e),
        Ok(_) => println!("成功（想定外）"),
    }

    // 不正なセクション
    match IniConfig::parse("[unclosed") {
        Err(e) => println!("不正なセクション: {}", e),
        Ok(_) => println!("成功（想定外）"),
    }
}
```

**補足：** カスタムエラー型を `enum` で定義することで、エラーの種類を型安全に表現できます。`?` 演算子は `From` トレイトと組み合わせることで、自動的にエラー型を変換することも可能です。実務では `thiserror` クレートを使うと、エラー型の定義がさらに簡潔になります。

</details>

### 問題5：複数のエラー型を統合する

異なるエラー型を扱う複数の関数を組み合わせて処理を行うプログラムを作成してください。`From` トレイトを実装してエラー型の自動変換を実現します。

以下のシナリオを実装してください。
- 文字列から数値への変換（`ParseIntError`）
- 範囲チェック（カスタムエラー）
- 計算処理（ゼロ除算エラー）

**期待される出力例：**
```
=== 計算テスト ===
calculate("100", "4") → Ok(25)
calculate("100", "0") → Err(計算エラー: ゼロで除算できません)
calculate("abc", "4") → Err(変換エラー: invalid digit found in string)
calculate("100", "-1") → Err(範囲エラー: 除数は1以上である必要があります（入力値: -1）)

=== バッチ処理 ===
[1] "100" / "4" = 25 ✓
[2] "200" / "5" = 40 ✓
[3] "abc" / "4" = エラー: 変換エラー: invalid digit found in string
[4] "100" / "0" = エラー: 計算エラー: ゼロで除算できません

成功: 2件, 失敗: 2件
```

<details><summary>ヒント</summary>

- カスタムエラーの `enum` に `ParseError`、`RangeError`、`CalcError` のバリアントを定義します
- `impl From<std::num::ParseIntError> for MyError` で自動変換を実装します
- `From` を実装すると `?` 演算子が自動的にエラー型を変換してくれます
- バッチ処理では `Result` を1つずつ処理し、成功・失敗を集計します

</details>

<details><summary>解答例</summary>

```rust
// 複数のエラー型を統合する
use std::fmt;
use std::num::ParseIntError;

// 統合エラー型
#[derive(Debug)]
enum CalcError {
    ParseError(ParseIntError),    // 文字列→数値変換エラー
    RangeError(String),           // 範囲チェックエラー
    DivisionByZero,               // ゼロ除算エラー
}

// Displayトレイトの実装
impl fmt::Display for CalcError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            CalcError::ParseError(e) => {
                write!(f, "変換エラー: {}", e)
            }
            CalcError::RangeError(msg) => {
                write!(f, "範囲エラー: {}", msg)
            }
            CalcError::DivisionByZero => {
                write!(f, "計算エラー: ゼロで除算できません")
            }
        }
    }
}

// ParseIntError → CalcError の自動変換
impl From<ParseIntError> for CalcError {
    fn from(e: ParseIntError) -> Self {
        CalcError::ParseError(e)
    }
}

// 文字列を数値に変換（?演算子でParseIntErrorを自動変換）
fn parse_value(s: &str) -> Result<i64, CalcError> {
    // ?演算子がFrom<ParseIntError>を使って自動変換
    let value = s.parse::<i64>()?;
    Ok(value)
}

// 範囲チェック
fn validate_divisor(n: i64) -> Result<i64, CalcError> {
    if n == 0 {
        Err(CalcError::DivisionByZero)
    } else if n < 0 {
        Err(CalcError::RangeError(format!(
            "除数は1以上である必要があります（入力値: {}）",
            n
        )))
    } else {
        Ok(n)
    }
}

// メインの計算関数（複数のエラー型を?で伝搬）
fn calculate(
    dividend_str: &str,
    divisor_str: &str,
) -> Result<i64, CalcError> {
    let dividend = parse_value(dividend_str)?;
    let divisor = parse_value(divisor_str)?;
    let validated_divisor = validate_divisor(divisor)?;
    Ok(dividend / validated_divisor)
}

fn main() {
    // === 計算テスト ===
    println!("=== 計算テスト ===");

    let test_cases = vec![
        ("100", "4"),
        ("100", "0"),
        ("abc", "4"),
        ("100", "-1"),
    ];

    for (a, b) in &test_cases {
        match calculate(a, b) {
            Ok(result) => {
                println!("calculate(\"{}\", \"{}\") → Ok({})", a, b, result)
            }
            Err(e) => {
                println!("calculate(\"{}\", \"{}\") → Err({})", a, b, e)
            }
        }
    }

    println!();

    // === バッチ処理 ===
    println!("=== バッチ処理 ===");

    let batch = vec![
        ("100", "4"),
        ("200", "5"),
        ("abc", "4"),
        ("100", "0"),
    ];

    let mut success_count = 0;
    let mut error_count = 0;

    for (i, (a, b)) in batch.iter().enumerate() {
        match calculate(a, b) {
            Ok(result) => {
                println!(
                    "[{}] \"{}\" / \"{}\" = {} ✓",
                    i + 1,
                    a,
                    b,
                    result
                );
                success_count += 1;
            }
            Err(e) => {
                println!(
                    "[{}] \"{}\" / \"{}\" = エラー: {}",
                    i + 1,
                    a,
                    b,
                    e
                );
                error_count += 1;
            }
        }
    }

    println!(
        "\n成功: {}件, 失敗: {}件",
        success_count, error_count
    );
}
```

**補足：** `From` トレイトを実装すると、`?` 演算子がエラー型を自動変換してくれます。これにより、異なるライブラリから返される異なるエラー型を、1つの統合エラー型にまとめることができます。実務では `thiserror` クレートの `#[from]` アトリビュートを使うと、この `From` 実装を自動生成できます。

// エラーチェーンと回復処理（フォールバック）

// データの取得元を表す列挙型
#[derive(Debug)]
enum DataSource {
    Main,
    Cache,
    Default,
}

impl DataSource {
    fn name(&self) -> &str {
        match self {
            DataSource::Main => "メイン",
            DataSource::Cache => "キャッシュ",
            DataSource::Default => "デフォルト",
        }
    }
}

// 取得結果（値と取得元）
struct FetchResult {
    value: String,
    source: DataSource,
}

// メインデータソースから取得
fn fetch_from_main(key: &str) -> Result<String, String> {
    println!("[メイン] {} を取得中...", key);

    // シミュレーション: user_1のみ成功
    match key {
        "user_1" => {
            println!("[メイン] 成功: Alice");
            Ok("Alice".to_string())
        }
        _ => {
            let err = "メインDBに接続できません".to_string();
            println!("[メイン] 失敗: {}", err);
            Err(err)
        }
    }
}

// キャッシュから取得
fn fetch_from_cache(key: &str) -> Result<String, String> {
    println!("[キャッシュ] {} を取得中...", key);

    // シミュレーション: user_2のみキャッシュにある
    match key {
        "user_2" => {
            println!("[キャッシュ] 成功: Bob (キャッシュ)");
            Ok("Bob (キャッシュ)".to_string())
        }
        _ => {
            let err = "キャッシュにデータがありません".to_string();
            println!("[キャッシュ] 失敗: {}", err);
            Err(err)
        }
    }
}

// フォールバック付きのデータ取得
fn fetch_with_fallback(key: &str) -> FetchResult {
    // 1. メインから取得を試みる
    if let Ok(value) = fetch_from_main(key) {
        return FetchResult {
            value,
            source: DataSource::Main,
        };
    }

    // 2. キャッシュから取得を試みる
    if let Ok(value) = fetch_from_cache(key) {
        return FetchResult {
            value,
            source: DataSource::Cache,
        };
    }

    // 3. デフォルト値を返す
    println!("[デフォルト] デフォルト値を使用");
    FetchResult {
        value: "Unknown User".to_string(),
        source: DataSource::Default,
    }
}

fn main() {
    println!("=== データ取得（フォールバック付き） ===\n");

    let keys = vec!["user_1", "user_2", "user_3"];

    for key in keys {
        println!("--- \"{}\" の取得 ---", key);

        let result = fetch_with_fallback(key);
        println!(
            "結果: \"{}\"（ソース: {}）\n",
            result.value,
            result.source.name()
        );
    }
}
```

**補足：** フォールバックパターンは、信頼性の高いシステムを構築する際に重要な設計パターンです。メインのデータソースが利用できない場合にキャッシュやデフォルト値で対応することで、システム全体の可用性を高めます。Rustの `Result` 型と `or_else()` メソッドを使うと、このようなフォールバックチェーンを型安全に実装できます。

// 実践的なCSVパーサー
// 複数のエラー型を適切に処理する

use std::fmt;

// CSVパースエラー
#[derive(Debug)]
enum CsvError {
    FieldCount {
        line: usize,
        expected: usize,
        actual: usize,
    },
    InvalidAge {
        line: usize,
        value: String,
    },
    InvalidScore {
        line: usize,
        value: String,
    },
    ScoreOutOfRange {
        line: usize,
        value: i32,
    },
    EmptyName {
        line: usize,
    },
    EmptyLine {
        line: usize,
    },
}

impl fmt::Display for CsvError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            CsvError::FieldCount {
                line,
                expected,
                actual,
            } => write!(
                f,
                "行 {}: フィールド数が不正です（期待: {}, 実際: {}）",
                line, expected, actual
            ),
            CsvError::InvalidAge { line, value } => {
                write!(f, "行 {}: 年齢が無効です: \"{}\"", line, value)
            }
            CsvError::InvalidScore { line, value } => {
                write!(
                    f,
                    "行 {}: スコアが無効です: \"{}\"",
                    line, value
                )
            }
            CsvError::ScoreOutOfRange { line, value } => {
                write!(
                    f,
                    "行 {}: スコアが範囲外です: {}（0-100）",
                    line, value
                )
            }
            CsvError::EmptyName { line } => {
                write!(f, "行 {}: 名前が空です", line)
            }
            CsvError::EmptyLine { line } => {
                write!(f, "行 {}: 空行です", line)
            }
        }
    }
}

// レコード構造体
#[derive(Debug)]
struct Record {
    name: String,
    age: u32,
    score: i32,
}

impl Record {
    fn display(&self) -> String {
        format!(
            "{} ({}歳, スコア: {})",
            self.name, self.age, self.score
        )
    }
}

// 1行をパースする
fn parse_line(
    line: &str,
    line_number: usize,
) -> Result<Record, CsvError> {
    let line = line.trim();

    // 空行チェック
    if line.is_empty() {
        return Err(CsvError::EmptyLine {
            line: line_number,
        });
    }

    // フィールド分割
    let fields: Vec<&str> = line.split(',').collect();
    if fields.len() != 3 {
        return Err(CsvError::FieldCount {
            line: line_number,
            expected: 3,
            actual: fields.len(),
        });
    }

    // 名前のバリデーション
    let name = fields[0].trim().to_string();
    if name.is_empty() {
        return Err(CsvError::EmptyName {
            line: line_number,
        });
    }

    // 年齢のパースとバリデーション
    let age_str = fields[1].trim();
    let age: u32 = age_str.parse().map_err(|_| CsvError::InvalidAge {
        line: line_number,
        value: age_str.to_string(),
    })?;

    // スコアのパースとバリデーション
    let score_str = fields[2].trim();
    let score: i32 =
        score_str.parse().map_err(|_| CsvError::InvalidScore {
            line: line_number,
            value: score_str.to_string(),
        })?;

    // スコア範囲チェック
    if score < 0 || score > 100 {
        return Err(CsvError::ScoreOutOfRange {
            line: line_number,
            value: score,
        });
    }

    Ok(Record { name, age, score })
}

// CSVデータ全体を処理する
fn process_csv(csv_data: &str) {
    println!("--- 処理結果 ---");

    let mut records: Vec<Record> = Vec::new();
    let mut errors: Vec<CsvError> = Vec::new();

    for (i, line) in csv_data.lines().enumerate() {
        let line_number = i + 1;

        // ヘッダー行をスキップ（必要に応じて）
        if i == 0 && line.contains("名前") {
            continue;
        }

        match parse_line(line, line_number) {
            Ok(record) => {
                println!(
                    "[行 {}] ✓ {}",
                    line_number,
                    record.display()
                );
                records.push(record);
            }
            Err(e) => {
                println!("[行 {}] ✗ エラー: {}", line_number, e);
                errors.push(e);
            }
        }
    }

    // --- 集計結果 ---
    println!("\n--- 集計結果 ---");
    println!("処理成功: {}件", records.len());
    println!("エラー: {}件", errors.len());

    if !records.is_empty() {
        println!("\n有効データの統計:");

        // 平均年齢
        let avg_age: f64 = records.iter().map(|r| r.age as f64).sum::<f64>()
            / records.len() as f64;
        println!("  平均年齢: {:.1}歳", avg_age);

        // 平均スコア
        let avg_score: f64 =
            records.iter().map(|r| r.score as f64).sum::<f64>()
                / records.len() as f64;
        println!("  平均スコア: {:.1}点", avg_score);

        // 最高スコア
        if let Some(best) =
            records.iter().max_by_key(|r| r.score)
        {
            println!(
                "  最高スコア: {} ({}点)",
                best.name, best.score
            );
        }

        // 最低スコア
        if let Some(worst) =
            records.iter().min_by_key(|r| r.score)
        {
            println!(
                "  最低スコア: {} ({}点)",
                worst.name, worst.score
            );
        }
    }

    // --- エラーレポート ---
    if !errors.is_empty() {
        println!("\n--- エラーレポート ---");
        for (i, error) in errors.iter().enumerate() {
            println!("{}. {}", i + 1, error);
        }
    }
}

fn main() {
    println!("=== CSV パーサー ===\n");

    let csv_data = "\
田中太郎,25,85
山田花子,30,92
佐藤次郎,abc,70
鈴木,40
鈴木一郎,28,78
高橋健太,22,150
,35,60";

    process_csv(csv_data);
}
```

**補足：** このCSVパーサーでは、エラーが発生しても処理を続行する「寛容なパース」パターンを使っています。各行の結果を個別に処理し、エラーを蓄積してあとからレポートします。実務では、データの品質問題を一括で報告する方が、1つずつエラーを修正するよりも効率的です。

`map_err()` は `Result` のエラー型を別の型に変換するメソッドで、`?` 演算子と組み合わせると、低レベルのエラー（`ParseIntError`）を業務ドメインのエラー（`CsvError`）に変換できます。

<details><summary>解答例</summary>

```rust
// リトライ処理とエラー回復のシミュレーション

use std::fmt;

// ネットワークエラー
#[derive(Debug, Clone)]
enum NetworkError {
    Timeout,
    ServerError(u16),
    NotFound,
    ConnectionRefused,
}

impl fmt::Display for NetworkError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            NetworkError::Timeout => write!(f, "接続タイムアウト"),
            NetworkError::ServerError(code) => {
                write!(f, "サーバーエラー: {}", code)
            }
            NetworkError::NotFound => write!(f, "リソースが見つかりません"),
            NetworkError::ConnectionRefused => {
                write!(f, "接続が拒否されました")
            }
        }
    }
}

// レスポンス
struct Response {
    status: u16,
    body: String,
}

impl Response {
    fn description(&self) -> String {
        let status_text = match self.status {
            200 => "OK",
            201 => "Created",
            _ => "Unknown",
        };
        format!("{} {}", self.status, status_text)
    }
}

// シミュレーション用の擬似乱数カウンター
struct SimulatedRequest {
    url: String,
    attempt_count: usize,
    // 何回目の試行で成功するか（0 = 常に失敗）
    success_at: usize,
    // 失敗時のエラーパターン
    error_pattern: Vec<NetworkError>,
}

impl SimulatedRequest {
    fn new(
        url: &str,
        success_at: usize,
        errors: Vec<NetworkError>,
    ) -> Self {
        SimulatedRequest {
            url: url.to_string(),
            attempt_count: 0,
            success_at,
            error_pattern: errors,
        }
    }

    // リクエストを実行（シミュレーション）
    fn execute(&mut self) -> Result<Response, NetworkError> {
        self.attempt_count += 1;

        if self.success_at > 0
            && self.attempt_count >= self.success_at
        {
            Ok(Response {
                status: 200,
                body: "データ取得成功".to_string(),
            })
        } else {
            // エラーパターンからエラーを返す
            let error_index =
                (self.attempt_count - 1) % self.error_pattern.len();
            Err(self.error_pattern[error_index].clone())
        }
    }
}

// リトライ付きリクエスト実行
fn execute_with_retry(
    request: &mut SimulatedRequest,
    max_retries: usize,
) -> Result<Response, NetworkError> {
    let mut last_error = NetworkError::Timeout;

    for attempt in 1..=max_retries {
        print!("  試行 {}/{}: ", attempt, max_retries);

        match request.execute() {
            Ok(response) => {
                println!("成功！");
                return Ok(response);
            }
            Err(e) => {
                println!("失敗 ({})", e);
                last_error = e;
            }
        }
    }

    Err(last_error)
}

// リクエスト結果
struct RequestResult {
    url: String,
    success: bool,
    message: String,
}

fn main() {
    println!("=== リトライ処理シミュレーション ===\n");

    // テスト用のリクエストを準備
    let mut requests = vec![
        // 3回目で成功
        SimulatedRequest::new(
            "https://api.example.com/users",
            3,
            vec![
                NetworkError::Timeout,
                NetworkError::ServerError(503),
            ],
        ),
        // 1回目で成功
        SimulatedRequest::new(
            "https://api.example.com/posts",
            1,
            vec![],
        ),
        // 常に失敗
        SimulatedRequest::new(
            "https://api.unstable.com/data",
            0, // 0 = 常に失敗
            vec![
                NetworkError::Timeout,
                NetworkError::Timeout,
                NetworkError::ServerError(503),
            ],
        ),
    ];

    let max_retries = 3;
    let mut results: Vec<RequestResult> = Vec::new();

    for request in &mut requests {
        println!("--- {} ---", request.url);

        let result = execute_with_retry(request, max_retries);

        match &result {
            Ok(response) => {
                let msg = format!(
                    "データ取得成功（レスポンス: {}）",
                    response.description()
                );
                println!("結果: ✓ {}\n", msg);
                results.push(RequestResult {
                    url: request.url.clone(),
                    success: true,
                    message: msg,
                });
            }
            Err(e) => {
                let msg = format!(
                    "{}回試行後に失敗: {}",
                    max_retries, e
                );
                println!("結果: ✗ {}\n", msg);
                results.push(RequestResult {
                    url: request.url.clone(),
                    success: false,
                    message: e.to_string(),
                });
            }
        }
    }

    // === サマリーレポート ===
    println!("=== サマリーレポート ===");

    let total = results.len();
    let success_count = results.iter().filter(|r| r.success).count();
    let fail_count = total - success_count;

    println!("成功: {}/{}", success_count, total);
    println!("失敗: {}/{}", fail_count, total);

    // 失敗したリクエストの詳細
    let failures: Vec<&RequestResult> =
        results.iter().filter(|r| !r.success).collect();
    if !failures.is_empty() {
        for failure in failures {
            println!("  - {}: {}", failure.url, failure.message);
        }
    }
}
```

**補足：** リトライパターンは、ネットワーク通信やファイルI/Oなど、一時的な障害が発生しうる処理で広く使われます。実務では以下の要素も考慮します。

- **指数バックオフ**: リトライ間隔を徐々に長くする（1秒→2秒→4秒）
- **ジッター**: リトライ間隔にランダムなゆらぎを加え、サーバーへの同時リクエストを分散する
- **サーキットブレーカー**: 連続失敗が一定回数を超えたら、一定時間リクエストを停止する

Rustでは `Result` 型とクロージャを組み合わせることで、型安全なリトライロジックを簡潔に実装できます。
