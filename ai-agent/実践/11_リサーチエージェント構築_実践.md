## 課題の要件

1. 以下の**4つのフェーズ**を含むリサーチエージェントを設計すること
   - **フェーズ1: クエリ分析・計画立案** - ユーザーのリサーチ要求を分析し、調査計画を立てる
   - **フェーズ2: 情報収集** - Web検索を実行し、関連情報を収集する
   - **フェーズ3: 分析・統合** - 収集した情報を整理・分析する
   - **フェーズ4: レポート生成** - 構造化されたレポートを生成する
2. 以下の設計書を作成すること
   - **システムアーキテクチャ図**（全コンポーネントとデータフロー）
   - **各フェーズの詳細設計**（入出力、処理ロジック、使用ツール）
   - **エージェント定義**（各エージェントのロール、プロンプト、ツール）
   - **データモデル**（各フェーズ間で受け渡すデータの構造）
   - **品質管理設計**（情報の信頼性評価、ファクトチェック）
   - **エラーハンドリング設計**（各フェーズでの失敗と復旧）
3. **擬似コード**でメインのオーケストレーションロジックを記述すること
4. **出力レポートのテンプレート**を設計すること
5. 具体的なリサーチ例（「量子コンピュータの実用化の最新動向」）を用いて、全フェーズの**実行トレース例**を記述すること

## ステップガイド

<details>
<summary>ステップ1：リサーチ要求の分解方法を設計する</summary>

ユーザーの漠然とした要求を、検索可能なサブクエリに分解する方法を設計しましょう。

例: 「量子コンピュータの実用化の最新動向」

分解結果:
1. 「量子コンピュータ 実用化 2025 2026 最新」
2. 「量子コンピュータ 商用利用 事例 企業」
3. 「量子コンピュータ 技術的課題 エラー訂正」
4. 「量子コンピュータ 市場規模 予測」

分解の観点:
- **What**: 何について調べるか
- **When**: いつの情報か（最新動向なら直近1〜2年）
- **Who**: 誰（どの企業・研究機関）が関わっているか
- **How**: 技術的にどう進展しているか

</details>

<details>
<summary>ステップ2：情報収集の並列化を設計する</summary>

複数のサブクエリに対して並列にWeb検索を実行し、結果を集約する方法を設計しましょう。

考慮すべきポイント:
- 各検索結果の上位何件を取得するか
- Webページのコンテンツをどう抽出するか（スクレイピング）
- レート制限への対処
- 検索結果の重複をどう扱うか

</details>

<details>
<summary>ステップ3：情報の信頼性評価を設計する</summary>

収集した情報すべてが正確とは限りません。信頼性を評価する仕組みが必要です。

評価基準の例:
- ソースの種類（学術論文 > 公式発表 > ニュース記事 > ブログ）
- 発行日（新しい方が高スコア）
- 情報の一致度（複数ソースで確認できるか）

**よくある間違い**: Web検索結果をそのまま信頼してしまうこと。特にAIが生成したコンテンツが検索結果に含まれる可能性が増えており、ソースの質を評価する仕組みが不可欠です。

</details>

<details>
<summary>ステップ4：レポートの構造と品質チェックを設計する</summary>

最終レポートの品質を担保するために、以下を設計しましょう。

- レポートのテンプレート（セクション構成）
- 各主張に参考文献を紐づける仕組み
- 矛盾する情報がある場合の表記方法
- レポートのセルフレビュー（生成後に自己チェック）

</details>

```python
class ResearchOrchestrator:
    def __init__(self, llm, search_tool, fetch_tool):
        self.planner = QueryPlanner(llm)
        self.collector = DataCollector(llm, search_tool, fetch_tool)
        self.analyzer = DataAnalyzer(llm)
        self.writer = ReportWriter(llm)

    def research(self, user_query):
        # フェーズ1: 計画立案
        plan = self.planner.create_plan(user_query)

        # フェーズ2: 情報収集（並列）
        collected = self.collector.collect(plan.sub_queries)

        analysis = self.analyzer.analyze(collected, plan)

        # フェーズ4: レポート生成
        report = self.writer.generate(analysis, plan)

        return report
```

### 2. 詳細擬似コード

```python
class ResearchOrchestrator:
    def __init__(self, llm, tools, config=None):
        self.llm = llm
        self.tools = tools
        self.config = config or ResearchConfig()
        self.state = ResearchState()

    def research(self, user_query):
        """メインのリサーチフロー"""

        # Phase 1: クエリ分析・計画立案
        plan = self._phase1_plan(user_query)
        self.state.plan = plan

        # 調査ループ
        all_collected = []
        for iteration in range(self.config.max_research_iterations):
            # Phase 2: 情報収集
            collected = self._phase2_collect(plan.sub_queries)
            all_collected.extend(collected)

            # Phase 3: 分析
            analysis = self._phase3_analyze(all_collected, plan)
            self.state.analysis = analysis

            # 充足度チェック
            sufficiency = self._check_sufficiency(analysis, plan)
            if sufficiency.is_sufficient:
                break

            # 追加サブクエリ生成
            additional_queries = self._generate_additional_queries(
                sufficiency.gaps,
                plan
            )
            plan.sub_queries = additional_queries

        # Phase 4: レポート生成
        report = self._phase4_report(analysis, plan)

        # Phase 5: 品質チェック
        for check_round in range(self.config.max_quality_checks):
            quality = self._phase5_quality_check(report, all_collected)
            if quality.passed:
                break
            report = self._revise_report(report, quality.feedback)

        return report

    def _phase1_plan(self, user_query):
        """Phase 1: クエリ分析・計画立案"""
        prompt = f"""
        以下のリサーチ要求を分析し、調査計画を立ててください。

        リサーチ要求: {user_query}

        以下のJSON形式で回答:
        {{
          "topic": "調査テーマ",
          "scope": "調査範囲の説明",
          "time_range": "対象期間",
          "sub_queries": [
            {{"id": "sq1", "query": "検索クエリ", "aspect": "調査観点", "priority": "high/medium/low"}}
          ],
          "expected_sections": ["セクション名のリスト"],
          "success_criteria": "何がわかれば調査完了か"
        }}
        """
        return self.llm.generate_structured(prompt, schema=ResearchPlan)

    def _phase2_collect(self, sub_queries):
        """Phase 2: 情報収集（並列実行）"""
        import asyncio

        async def collect_for_query(sq):
            # Web検索
            search_results = await self.tools.web_search(
                query=sq.query,
                num_results=5,
                time_range=self.state.plan.time_range
            )

            collected_items = []
            for result in search_results:
                # ページ内容の取得
                try:
                    page_content = await self.tools.fetch_page(
                        url=result.url,
                        timeout=10
                    )
                except Exception:
                    continue

                # 関連情報の抽出
                extracted = self._extract_relevant_info(
                    page_content,
                    sq.aspect
                )

                # 信頼性評価
                reliability = self._evaluate_reliability(
                    url=result.url,
                    publish_date=result.date,
                    source_type=self._classify_source(result.url)
                )

                collected_items.append(CollectedItem(
                    sub_query_id=sq.id,
                    source_url=result.url,
                    title=result.title,
                    content=extracted,
                    publish_date=result.date,
                    reliability=reliability
                ))

            return collected_items

        # 全サブクエリを並列実行
        tasks = [collect_for_query(sq) for sq in sub_queries]
        results = asyncio.run(asyncio.gather(*tasks))

        # 結果をフラット化
        return [item for sublist in results for item in sublist]

    def _phase3_analyze(self, collected, plan):
        """Phase 3: 分析・統合"""

        # 1. 重複除去
        unique_items = self._deduplicate(collected)

        # 2. テーマ別クラスタリング
        clusters = self._cluster_by_theme(unique_items, plan.expected_sections)

        # 3. 矛盾検出
        contradictions = self._detect_contradictions(unique_items)

        # 4. Key Findingsの抽出
        key_findings = self._extract_key_findings(clusters)

        return AnalysisResult(
            clusters=clusters,
            contradictions=contradictions,
            key_findings=key_findings,
            total_sources=len(unique_items),
            coverage=self._calculate_coverage(clusters, plan.expected_sections)
        )

    def _check_sufficiency(self, analysis, plan):
        """情報の充足度をチェックする"""
        prompt = f"""
        調査計画と収集された情報を比較し、充足度を評価してください。

        調査計画のセクション: {plan.expected_sections}
        各セクションのカバー率: {analysis.coverage}
        成功基準: {plan.success_criteria}

        JSON形式:
        {{
          "is_sufficient": true/false,
          "overall_coverage": 0.0-1.0,
          "gaps": ["不足している情報の説明"],
          "suggested_additional_queries": ["追加すべき検索クエリ"]
        }}
        """
        return self.llm.generate_structured(prompt, schema=SufficiencyCheck)

    def _evaluate_reliability(self, url, publish_date, source_type):
        """情報源の信頼性を評価する"""

        # 基本スコア（ソースの種類に基づく）
        base_scores = {
            "academic": 0.95,
            "official_press": 0.90,
            "major_media": 0.80,
            "tech_blog_official": 0.75,
            "general_news": 0.60,
            "personal_blog": 0.40,
            "social_media": 0.20,
            "unknown": 0.30
        }
        score = base_scores.get(source_type, 0.30)

        # 日付による補正（新しいほど高スコア）
        if publish_date:
            days_old = (datetime.now() - publish_date).days
            if days_old <= 30:
                score *= 1.0
            elif days_old <= 90:
                score *= 0.95
            elif days_old <= 365:
                score *= 0.85
            else:
                score *= 0.70

        return min(score, 1.0)
```

### 3. ファクトチェック機構

```python
def _phase5_quality_check(self, report, collected_items):
    """Phase 5: 品質チェック・ファクトチェック"""

    checks = []

    # 1. 主張と参照元の整合性チェック
    claims = self._extract_claims(report)
    for claim in claims:
        supporting_sources = self._find_supporting_sources(
            claim, collected_items
        )
        checks.append({
            "type": "claim_verification",
            "claim": claim.text,
            "supported": len(supporting_sources) > 0,
            "sources": [s.source_url for s in supporting_sources],
            "confidence": max(s.reliability for s in supporting_sources)
                         if supporting_sources else 0.0
        })

    # 2. 構造チェック
    structure_check = self._check_report_structure(report)
    checks.append({
        "type": "structure",
        "has_summary": structure_check.has_summary,
        "has_references": structure_check.has_references,
        "section_balance": structure_check.section_balance
    })

    # 3. 一貫性チェック
    consistency = self._check_consistency(report)
    checks.append({
        "type": "consistency",
        "contradictions_found": consistency.contradictions,
        "tone_consistent": consistency.tone_consistent
    })

    # 合否判定
    unsupported_claims = [c for c in checks
                         if c["type"] == "claim_verification"
                         and not c["supported"]]

    passed = (
        len(unsupported_claims) == 0
        and structure_check.has_summary
        and structure_check.has_references
        and len(consistency.contradictions) == 0
    )

    feedback = []
    if unsupported_claims:
        feedback.append(f"根拠のない主張が{len(unsupported_claims)}件あります")
    if not structure_check.has_references:
        feedback.append("参考文献リストがありません")

    return QualityResult(passed=passed, checks=checks, feedback=feedback)
```
