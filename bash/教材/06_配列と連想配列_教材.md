# 第6章：配列と連想配列

## この章のゴール

- 配列（indexed array）の宣言、代入、参照を習得すること
- 配列の追加、削除、スライスなどの操作をマスターすること
- 連想配列（associative array）の使い方を理解すること
- 配列とループを組み合わせた実践パターンを身につけること

---

## 6.1 配列の基本

### 配列の宣言と代入

```bash
#!/usr/bin/env bash

# 方法1: () で一括代入
fruits=("apple" "banana" "cherry" "date")

# 方法2: インデックスを指定して代入
colors[0]="red"
colors[1]="green"
colors[2]="blue"

# 方法3: declare で宣言
declare -a numbers=(10 20 30 40 50)
```

### 配列の参照

```bash
#!/usr/bin/env bash
fruits=("apple" "banana" "cherry" "date")

# 特定の要素を参照
echo "${fruits[0]}"    # → apple（最初の要素）
echo "${fruits[2]}"    # → cherry

# 全要素を参照
echo "${fruits[@]}"    # → apple banana cherry date（個別の単語として）
echo "${fruits[*]}"    # → apple banana cherry date（1つの文字列として）

# 要素数を取得
echo "${#fruits[@]}"   # → 4

# 特定の要素の文字数
echo "${#fruits[0]}"   # → 5（"apple"の文字数）

# 全インデックスを取得
echo "${!fruits[@]}"   # → 0 1 2 3
```

### "@" と "*" の違い

```bash
#!/usr/bin/env bash
items=("hello world" "foo bar" "baz")

# "$@" はスペース入り要素を保持
echo '--- "${items[@]}" ---'
for item in "${items[@]}"; do
    echo "  [$item]"
done
# 出力:
#   [hello world]
#   [foo bar]
#   [baz]

# "$*" は全要素を1つに結合
echo '--- "${items[*]}" ---'
for item in "${items[*]}"; do
    echo "  [$item]"
done
# 出力:
#   [hello world foo bar baz]
```

### ポイントまとめ

- 配列は `arr=(val1 val2 ...)` で宣言
- `${arr[i]}` で要素参照、`${arr[@]}` で全要素
- `${#arr[@]}` で要素数
- ループでは `"${arr[@]}"` を使う（スペース入り要素を保持）

## 6.2 配列の操作

### 要素の追加

```bash
#!/usr/bin/env bash
fruits=("apple" "banana")

# 末尾に追加
fruits+=("cherry")
fruits+=("date" "elderberry")

echo "${fruits[@]}"  # → apple banana cherry date elderberry
echo "要素数: ${#fruits[@]}"  # → 5
```

### 要素の削除

```bash
#!/usr/bin/env bash
fruits=("apple" "banana" "cherry" "date")

# 特定のインデックスを削除
unset 'fruits[1]'
echo "${fruits[@]}"    # → apple cherry date
echo "${!fruits[@]}"   # → 0 2 3（インデックスが詰められない！）

# インデックスを詰め直す
fruits=("${fruits[@]}")
echo "${!fruits[@]}"   # → 0 1 2
```

### スライス（部分配列）

```bash
#!/usr/bin/env bash
numbers=(10 20 30 40 50 60 70)

# ${arr[@]:offset:length}
echo "${numbers[@]:2:3}"   # → 30 40 50（インデックス2から3個）
echo "${numbers[@]:4}"     # → 50 60 70（インデックス4以降すべて）

# 新しい配列に代入
sub=("${numbers[@]:1:3}")
echo "${sub[@]}"  # → 20 30 40
```

### 配列の結合

```bash
#!/usr/bin/env bash
arr1=(1 2 3)
arr2=(4 5 6)

# 結合
combined=("${arr1[@]}" "${arr2[@]}")
echo "${combined[@]}"  # → 1 2 3 4 5 6
```

### 配列の検索

```bash
#!/usr/bin/env bash
fruits=("apple" "banana" "cherry" "date")

# 特定の値が含まれるか確認
contains() {
    local target="$1"
    shift
    for item in "$@"; do
        [[ "$item" = "$target" ]] && return 0
    done
    return 1
}

if contains "cherry" "${fruits[@]}"; then
    echo "cherry が見つかりました"
fi
```

### ポイントまとめ

- `arr+=("val")` で末尾に追加
- `unset 'arr[i]'` で削除（インデックスは詰まらない）
- `${arr[@]:offset:length}` でスライス
- 結合は `combined=("${arr1[@]}" "${arr2[@]}")`

## 6.3 配列とループ

### 基本的な配列ループ

```bash
#!/usr/bin/env bash
servers=("web-01" "web-02" "db-01" "cache-01")

# 値でループ
for server in "${servers[@]}"; do
    echo "サーバー: $server"
done
```

### インデックス付きループ

```bash
#!/usr/bin/env bash
tasks=("ビルド" "テスト" "デプロイ" "検証")

# インデックスと値の両方を使う
for i in "${!tasks[@]}"; do
    echo "ステップ$((i + 1)): ${tasks[$i]}"
done
```

実行結果：
```
ステップ1: ビルド
ステップ2: テスト
ステップ3: デプロイ
ステップ4: 検証
```

### 実践例：並列pingチェック

```bash
#!/usr/bin/env bash
hosts=("google.com" "github.com" "example.com")

for host in "${hosts[@]}"; do
    if ping -c 1 -W 2 "$host" > /dev/null 2>&1; then
        echo "✓ $host: 到達可能"
    else
        echo "✗ $host: 到達不可"
    fi
done
```

### ポイントまとめ

- `for item in "${arr[@]}"` で全要素をループ
- `for i in "${!arr[@]}"` でインデックス付きループ
- `"${arr[@]}"` のダブルクォートを忘れない

## 6.4 連想配列

連想配列（associative array）は、文字列をキーとして値を格納できる配列です（Bash 4.0以降）。

### 宣言と基本操作

```bash
#!/usr/bin/env bash

# 連想配列は declare -A で宣言が必須
declare -A user
user[name]="田中太郎"
user[age]="30"
user[city]="東京"

echo "名前: ${user[name]}"
echo "年齢: ${user[age]}"
echo "都市: ${user[city]}"
```

### 一括代入

```bash
#!/usr/bin/env bash

declare -A colors=(
    [red]="#FF0000"
    [green]="#00FF00"
    [blue]="#0000FF"
    [white]="#FFFFFF"
    [black]="#000000"
)

echo "赤のカラーコード: ${colors[red]}"
```

### キーと値の一覧

```bash
#!/usr/bin/env bash

declare -A config=(
    [host]="localhost"
    [port]="8080"
    [debug]="true"
)

# 全キーを取得
echo "キー: ${!config[@]}"  # → host port debug

# 全値を取得
echo "値: ${config[@]}"     # → localhost 8080 true

# 要素数
echo "項目数: ${#config[@]}"  # → 3

# キーと値をループ
for key in "${!config[@]}"; do
    echo "  $key = ${config[$key]}"
done
```

### キーの存在確認

```bash
#!/usr/bin/env bash

declare -A settings=([debug]="true" [verbose]="false")

# キーが存在するか確認
if [[ -v settings[debug] ]]; then
    echo "debug キーが存在します: ${settings[debug]}"
fi

if [[ ! -v settings[log_level] ]]; then
    echo "log_level キーは存在しません"
fi
```

### 実践例：設定ファイルの読み込み

```bash
#!/usr/bin/env bash

# key=value 形式の設定ファイルを連想配列に読み込む
declare -A config

load_config() {
    local file="$1"
    while IFS='=' read -r key value; do
        # コメント行と空行をスキップ
        [[ "$key" =~ ^[[:space:]]*# ]] && continue
        [[ -z "$key" ]] && continue
        # 前後の空白を除去
        key=$(echo "$key" | xargs)
        value=$(echo "$value" | xargs)
        config["$key"]="$value"
    done < "$file"
}

# 設定ファイルの例（テスト用に作成）
cat > /tmp/test_config.conf <<'EOF'
# データベース設定
db_host = localhost
db_port = 5432
db_name = myapp
# ログ設定
log_level = info
EOF

load_config "/tmp/test_config.conf"

for key in "${!config[@]}"; do
    echo "$key = ${config[$key]}"
done

# クリーンアップ
rm -f /tmp/test_config.conf
```

### ポイントまとめ

- `declare -A` で連想配列を宣言（必須）
- `${!arr[@]}` で全キー、`${arr[@]}` で全値
- `[[ -v arr[key] ]]` でキーの存在確認
- 設定ファイルの読み込みに便利

## 6.5 配列の実践パターン

### コマンドライン引数の配列処理

```bash
#!/usr/bin/env bash

# 引数をカテゴリ分けして配列に格納
declare -a files=()
declare -a options=()

for arg in "$@"; do
    if [[ "$arg" == -* ]]; then
        options+=("$arg")
    else
        files+=("$arg")
    fi
done

echo "オプション: ${options[*]}"
echo "ファイル: ${files[*]}"
```

### スタック（後入れ先出し）

```bash
#!/usr/bin/env bash

declare -a stack=()

push() { stack+=("$1"); }
pop() {
    local last=$((${#stack[@]} - 1))
    echo "${stack[$last]}"
    unset 'stack[$last]'
}
peek() { echo "${stack[-1]}"; }

push "first"
push "second"
push "third"
echo "トップ: $(peek)"   # → third
echo "ポップ: $(pop)"    # → third
echo "ポップ: $(pop)"    # → second
echo "残り: ${stack[*]}"  # → first
```

### ポイントまとめ

- 配列でデータ構造（スタック等）を実装できる
- 引数のカテゴリ分けに配列が便利
- `${arr[-1]}` で末尾要素にアクセス（Bash 4.3以降）

## 6.6 章全体のまとめ

| 操作 | 配列 | 連想配列 |
|------|------|----------|
| 宣言 | `arr=(v1 v2)` | `declare -A arr=([k]=v)` |
| 参照 | `${arr[0]}` | `${arr[key]}` |
| 全要素 | `${arr[@]}` | `${arr[@]}` |
| 全キー | `${!arr[@]}` | `${!arr[@]}` |
| 要素数 | `${#arr[@]}` | `${#arr[@]}` |
| 追加 | `arr+=(val)` | `arr[key]=val` |
| 削除 | `unset 'arr[i]'` | `unset 'arr[key]'` |
| 存在確認 | - | `[[ -v arr[key] ]]` |
