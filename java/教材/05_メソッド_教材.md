# 第5章：メソッド

## この章のゴール

- メソッド（method）の定義方法と呼び出し方を理解する

## 5.1 メソッドとは

メソッドとは、**処理のまとまりに名前をつけたもの**です。同じ処理を何度も書く代わりに、メソッドとして定義しておけば再利用できます。

```
┌─────────────────────────────────────────────┐
│  メソッドのイメージ                           │
│                                             │
│  入力（引数）→ [  処理  ] → 出力（戻り値）    │
│                                             │
│  例：                                        │
│  2, 3  →  [ 足し算 ]  →  5                  │
└─────────────────────────────────────────────┘
```

### メソッドを使うメリット

| メリット | 説明 |
|---|---|
| 再利用性 | 同じ処理を何度も書かなくてよい |
| 可読性 | 処理に名前がつくのでコードが読みやすくなる |
| 保守性 | 修正が1箇所で済む |
| テスト容易性 | 小さな単位でテストしやすい |

## 5.2 メソッドの定義と呼び出し

### 基本構文

```java
アクセス修飾子 static 戻り値の型 メソッド名(引数の型 引数名, ...) {
    // 処理
    return 戻り値;
}
```

### 最初のメソッド

```java
public class MethodBasic {
    // メソッドの定義
    static int add(int a, int b) {
        return a + b;
    }

    public static void main(String[] args) {
        // メソッドの呼び出し
        int result = add(3, 5);
        System.out.println("3 + 5 = " + result);  // 3 + 5 = 8

        // 何度でも呼び出せる
        System.out.println("10 + 20 = " + add(10, 20));  // 10 + 20 = 30
    }
}
```

### 各部分の解説

```
static int add(int a, int b) {
  │     │   │    │        │
  │     │   │    └── 引数（パラメータ）：メソッドに渡す値
  │     │   └──────── メソッド名：処理を表す名前
  │     └──────────── 戻り値の型：メソッドが返す値の型
  └────────────────── static：クラスメソッド（後述）
```

### ポイントまとめ
- メソッドは「処理のまとまり」に名前をつけたもの
- `return` で値を呼び出し元に返す
- 引数で外部からデータを受け取る

## 5.3 戻り値のないメソッド（void）

値を返す必要がないメソッドは、戻り値の型に `void` を指定します。

```java
public class VoidMethod {
    // 戻り値なしのメソッド
    static void greet(String name) {
        System.out.println("こんにちは、" + name + "さん！");
        // return は省略可能（書く場合は return; のみ）
    }

    // 罫線を表示するメソッド
    static void printLine(int length) {
        for (int i = 0; i < length; i++) {
            System.out.print("-");
        }
        System.out.println();
    }

    public static void main(String[] args) {
        greet("田中");       // こんにちは、田中さん！
        printLine(20);       // --------------------
        greet("鈴木");       // こんにちは、鈴木さん！
        printLine(20);       // --------------------
    }
}
```

> **よくある間違い：** `void` メソッドで `return 値;` を書くとコンパイルエラーになります。逆に、戻り値の型を指定したメソッドで `return` を忘れてもエラーになります。

## 5.4 引数の詳細

### 複数の引数

```java
public class MultipleParams {
    // 複数の引数を受け取るメソッド
    static double calculateBMI(double weightKg, double heightM) {
        return weightKg / (heightM * heightM);
    }

    static void printPersonInfo(String name, int age, String city) {
        System.out.printf("%sさん（%d歳）- %s在住%n", name, age, city);
    }

    public static void main(String[] args) {
        double bmi = calculateBMI(65.0, 1.72);
        System.out.printf("BMI: %.1f%n", bmi);  // BMI: 22.0

        printPersonInfo("田中", 30, "東京");
        // 田中さん（30歳）- 東京在住
    }
}
```

### 引数なしのメソッド

```java
static String getCurrentDate() {
    java.time.LocalDate today = java.time.LocalDate.now();
    return today.toString();
}
```

### 引数の渡し方：値渡し

Javaでは引数は**値渡し（pass by value）**です。プリミティブ型の場合、メソッド内で引数の値を変更しても呼び出し元には影響しません。

```java
public class PassByValue {
    static void tryToChange(int x) {
        x = 100;  // メソッド内でのみ有効
        System.out.println("メソッド内: x = " + x);  // 100
    }

    public static void main(String[] args) {
        int num = 5;
        tryToChange(num);
        System.out.println("呼び出し後: num = " + num);  // 5（変わらない）
    }
}
```

> **📖 コラム：** 配列やオブジェクトの場合、参照の値がコピーされます。そのため、メソッド内で配列の要素を変更すると呼び出し元にも影響します。これは「参照の値渡し」と呼ばれます。

```java
static void doubleArray(int[] arr) {
    for (int i = 0; i < arr.length; i++) {
        arr[i] *= 2;  // 呼び出し元の配列にも反映される
    }
}
```

### ポイントまとめ
- 引数は複数指定可能（型と名前のペアをカンマで区切る）
- Javaの引数は「値渡し」
- プリミティブ型は値のコピー、参照型は参照のコピー

## 5.5 メソッドのオーバーロード

**オーバーロード（overload）**とは、同じ名前で引数の型や数が異なるメソッドを定義することです。

```java
public class Overload {
    // int × 2
    static int add(int a, int b) {
        return a + b;
    }

    // double × 2
    static double add(double a, double b) {
        return a + b;
    }

    // int × 3
    static int add(int a, int b, int c) {
        return a + b + c;
    }

    // String × 2（文字列連結）
    static String add(String a, String b) {
        return a + b;
    }

    public static void main(String[] args) {
        System.out.println(add(1, 2));           // 3（int版）
        System.out.println(add(1.5, 2.3));       // 3.8（double版）
        System.out.println(add(1, 2, 3));        // 6（int×3版）
        System.out.println(add("Hello", " World")); // Hello World（String版）
    }
}
```

### オーバーロードのルール

| ルール | OK / NG |
|---|---|
| 引数の数が異なる | OK |
| 引数の型が異なる | OK |
| 引数の順序が異なる | OK（ただし可読性に注意） |
| 戻り値の型だけ異なる | NG（コンパイルエラー） |
| 引数名だけ異なる | NG（同じメソッドと見なされる） |

> **よくある間違い：** 戻り値の型だけが違うメソッドはオーバーロードできません。`int calc(int a)` と `double calc(int a)` は共存できません。

### ポイントまとめ
- 同名メソッドを引数違いで複数定義できる
- 引数の「型」「数」「順序」で区別される
- 戻り値の型だけではオーバーロードできない

## 5.6 変数のスコープ

変数には**有効範囲（スコープ）**があります。

```java
public class ScopeExample {
    // クラス変数（static変数）：クラス全体で共有
    static int classVar = 100;

    static void methodA() {
        // ローカル変数：このメソッド内でのみ有効
        int localVar = 10;
        System.out.println("classVar = " + classVar);   // OK
        System.out.println("localVar = " + localVar);   // OK
    }

    static void methodB() {
        System.out.println("classVar = " + classVar);   // OK
        // System.out.println(localVar);  // エラー！methodAのローカル変数
    }

    public static void main(String[] args) {
        methodA();
        methodB();

        // ブロックスコープ
        for (int i = 0; i < 3; i++) {
            int temp = i * 2;
            System.out.println(temp);
        }
        // System.out.println(i);     // エラー！forブロックの外
        // System.out.println(temp);  // エラー！forブロックの外
    }
}
```

### スコープの図解

```
┌─ クラス ──────────────────────────────┐
│  static int classVar ← どこからでもOK   │
│                                       │
│  ┌─ methodA ──────────────┐           │
│  │  int localVar ← ここだけ │           │
│  └────────────────────────┘           │
│                                       │
│  ┌─ methodB ──────────────┐           │
│  │  localVar は使えない     │           │
│  └────────────────────────┘           │
│                                       │
│  ┌─ main ─────────────────┐           │
│  │  ┌─ for ─────────┐     │           │
│  │  │ int i ← ここだけ│     │           │
│  │  └───────────────┘     │           │
│  └────────────────────────┘           │
└───────────────────────────────────────┘
```

### ポイントまとめ
- ローカル変数はメソッドやブロック内でのみ有効
- クラス変数（static）はクラス全体で共有
- スコープが狭いほど安全で管理しやすい

## 5.7 static メソッドとインスタンスメソッド

Javaのメソッドには2種類あります。

| 種類 | キーワード | 呼び出し方 | 用途 |
|---|---|---|---|
| static メソッド | `static` あり | `クラス名.メソッド名()` | ユーティリティ処理 |
| インスタンスメソッド | `static` なし | `オブジェクト.メソッド名()` | オブジェクト固有の処理 |

```java
public class StaticVsInstance {
    // インスタンスフィールド
    String name;

    // コンストラクタ
    StaticVsInstance(String name) {
        this.name = name;
    }

    // インスタンスメソッド — name にアクセスできる
    void sayHello() {
        System.out.println("こんにちは、" + this.name + "です！");
    }

    // static メソッド — インスタンスなしで呼べる
    static int multiply(int a, int b) {
        return a * b;
    }

    public static void main(String[] args) {
        // static メソッドの呼び出し
        int result = StaticVsInstance.multiply(3, 4);
        System.out.println("3 × 4 = " + result);

        // インスタンスメソッドの呼び出し
        StaticVsInstance person = new StaticVsInstance("田中");
        person.sayHello();
    }
}
```

> **よくある間違い：** `static` メソッドから直接インスタンスフィールドやインスタンスメソッドを呼び出すことはできません。`main` メソッドは `static` なので、インスタンスを生成してから呼び出す必要があります。

## 5.8 可変長引数（varargs）

引数の数が決まっていない場合、**可変長引数**を使えます。

```java
public class Varargs {
    // 可変長引数：int... で任意個の int を受け取れる
    static int sum(int... numbers) {
        int total = 0;
        for (int n : numbers) {
            total += n;
        }
        return total;
    }

    // 通常の引数と組み合わせ可能（可変長引数は最後に置く）
    static void printScores(String name, int... scores) {
        System.out.print(name + "さんの得点: ");
        for (int s : scores) {
            System.out.print(s + " ");
        }
        System.out.println();
    }

    public static void main(String[] args) {
        System.out.println(sum(1, 2, 3));           // 6
        System.out.println(sum(10, 20, 30, 40));    // 100
        System.out.println(sum());                   // 0

        printScores("田中", 80, 90, 75);
        printScores("鈴木", 95, 88);
    }
}
```

### 可変長引数のルール
- `型... 引数名` の形式で宣言する
- メソッドの引数リストの**最後**に1つだけ置ける
- メソッド内では配列として扱われる
- 0個以上の引数を渡せる

### ポイントまとめ
- `int... numbers` で任意個の引数を受け取れる
- 通常引数との併用時は最後に配置する
- 内部的には配列として扱われる

## 5.9 実践例：ユーティリティメソッド集

実用的なメソッドをまとめた例です。

```java
public class MathUtils {
    // 最大値を返す
    static int max(int a, int b) {
        return (a > b) ? a : b;
    }

    // 最小値を返す
    static int min(int a, int b) {
        return (a < b) ? a : b;
    }

    // 配列の合計
    static int sumArray(int[] arr) {
        int total = 0;
        for (int value : arr) {
            total += value;
        }
        return total;
    }

    // 配列の平均
    static double averageArray(int[] arr) {
        if (arr.length == 0) return 0.0;
        return (double) sumArray(arr) / arr.length;
    }

    // 素数判定
    static boolean isPrime(int n) {
        if (n <= 1) return false;
        if (n <= 3) return true;
        if (n % 2 == 0) return false;
        for (int i = 3; i * i <= n; i += 2) {
            if (n % i == 0) return false;
        }
        return true;
    }

    // 階乗
    static long factorial(int n) {
        if (n <= 1) return 1;
        long result = 1;
        for (int i = 2; i <= n; i++) {
            result *= i;
        }
        return result;
    }

    public static void main(String[] args) {
        System.out.println("max(10, 20) = " + max(10, 20));
        System.out.println("min(10, 20) = " + min(10, 20));

        int[] scores = {80, 90, 75, 85, 95};
        System.out.println("合計: " + sumArray(scores));
        System.out.printf("平均: %.1f%n", averageArray(scores));

        System.out.println("7は素数？ " + isPrime(7));
        System.out.println("10は素数？ " + isPrime(10));

        System.out.println("5! = " + factorial(5));
        System.out.println("10! = " + factorial(10));
    }
}
```

実行結果：
```
max(10, 20) = 20
min(10, 20) = 10
合計: 425
平均: 85.0
7は素数？ true
10は素数？ false
5! = 120
10! = 3628800
```

## 5.10 よくある間違いとトラブルシューティング

| ミス | 症状 | 対処法 |
|---|---|---|
| `return` を忘れる | コンパイルエラー | 戻り値の型を指定したら必ず `return` する |
| void で値を返す | コンパイルエラー | `void` メソッドでは `return;` のみ可 |
| 引数の型不一致 | コンパイルエラー | 定義と呼び出しの型を揃える |
| static から非 static を呼ぶ | コンパイルエラー | インスタンスを生成してから呼ぶ |
| スコープ外の変数を使う | コンパイルエラー | 変数の有効範囲を確認する |
| 可変長引数が最後でない | コンパイルエラー | 引数リストの最後に配置する |
