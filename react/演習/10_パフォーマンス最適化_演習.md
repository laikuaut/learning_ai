## 演習1（基本）：React.memoで不要な再レンダリングを防ごう

以下のコードでは、カウンターのボタンをクリックするたびに `ExpensiveList` も再レンダリングされてしまいます。`React.memo` を使って不要な再レンダリングを防いでください。

**修正前のコード：**
```jsx
import { useState } from 'react';

function ExpensiveList({ items }) {
  console.log('ExpensiveList がレンダリングされた');
  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>{item.name} - ¥{item.price}</li>
      ))}
    </ul>
  );
}

function App() {
  const [count, setCount] = useState(0);
  const items = [
    { id: 1, name: 'りんご', price: 200 },
    { id: 2, name: 'バナナ', price: 150 },
    { id: 3, name: 'みかん', price: 100 },
  ];

  return (
    <div>
      <p>カウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <ExpensiveList items={items} />
    </div>
  );
}
```

**要件：**
- `ExpensiveList` を `React.memo` でラップして、`items` が変わらなければ再レンダリングされないようにする
- `items` の定義場所にも注意する（毎レンダリングで新しい参照が作られていないか？）

<details>
<summary>ヒント</summary>

`React.memo` でラップしても、`items` が毎回新しい配列として作成されていると効果がありません。
`items` を `useState` か `useMemo` で安定させましょう。

```jsx
const [items] = useState([...]);
// または
const items = useMemo(() => [...], []);
```

</details>

<details>
<summary>解答例</summary>

```jsx
import { memo, useState } from 'react';

// ========================================
// React.memo でラップ：propsが変わらなければ再レンダリングをスキップ
// consoleを見て、カウントを変更してもこのコンポーネントが
// レンダリングされないことを確認しよう
// ========================================
const ExpensiveList = memo(function ExpensiveList({ items }) {
  console.log('ExpensiveList がレンダリングされた');
  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>{item.name} - ¥{item.price}</li>
      ))}
    </ul>
  );
});

function App() {
  const [count, setCount] = useState(0);

  // ========================================
  // useState で items を定義すると参照が安定する
  // 毎レンダリングで const items = [...] と書くと
  // 新しい配列が作られて React.memo が効かない
  // ========================================
  const [items] = useState([
    { id: 1, name: 'りんご', price: 200 },
    { id: 2, name: 'バナナ', price: 150 },
    { id: 3, name: 'みかん', price: 100 },
  ]);

  return (
    <div style={{ padding: '20px' }}>
      <p>カウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
      {/* items の参照が変わらないので、React.memo が効いて再レンダリングされない */}
      <ExpensiveList items={items} />
    </div>
  );
}

export default App;
```

**ポイント：**
- `React.memo` はコンポーネントの props を浅い比較（shallow comparison）して、変化がなければレンダリングをスキップします
- 配列やオブジェクトは毎回新しい参照が作られると `React.memo` が効かなくなるため、`useState` や `useMemo` で安定させます
- コンソールに `console.log` を仕込んで、再レンダリングの有無を確認する癖を付けましょう

</details>

## 演習2（基本）：useMemoで重い計算をメモ化しよう

大量のデータからフィルタリングとソートを行う処理を `useMemo` でメモ化してください。

**要件：**
- 10000件のアイテムデータを生成
- 検索キーワードでフィルタリング
- 価格順でソート（昇順/降順の切り替え）
- 別のstateを変更してもフィルタリング/ソートが再実行されないことを確認
- `console.log` で再計算のタイミングを確認

<details>
<summary>ヒント</summary>

`useMemo` の第2引数（依存配列）に指定した値が変わった時だけ再計算されます。

```jsx
const result = useMemo(() => {
  console.log('再計算された');
  // 重い計算...
  return computedValue;
}, [dependency1, dependency2]);
```

</details>

<details>
<summary>解答例</summary>

```jsx
import { useState, useMemo } from 'react';

// ========================================
// 大量のテストデータを生成
// ========================================
function generateItems(count) {
  return Array.from({ length: count }, (_, i) => ({
    id: i + 1,
    name: `商品${String(i + 1).padStart(5, '0')}`,
    price: Math.floor(Math.random() * 10000) + 100,
    category: ['電子機器', '書籍', '食品', '衣類'][i % 4],
  }));
}

function App() {
  const [allItems] = useState(() => generateItems(10000));
  const [query, setQuery] = useState('');
  const [sortOrder, setSortOrder] = useState('asc');
  const [theme, setTheme] = useState('light'); // フィルタと無関係なstate

  // ========================================
  // useMemo：queryまたはsortOrderが変わった時だけ再計算
  // themeを変更しても再計算されない
  // ========================================
  const filteredAndSorted = useMemo(() => {
    console.log('フィルタリング&ソート実行');

    const filtered = allItems.filter(item =>
      item.name.includes(query) || item.category.includes(query)
    );

    return [...filtered].sort((a, b) =>
      sortOrder === 'asc' ? a.price - b.price : b.price - a.price
    );
  }, [allItems, query, sortOrder]); // themeは依存配列に含まない

  // ========================================
  // 合計金額もメモ化（filteredAndSortedが変わった時だけ再計算）
  // ========================================
  const totalPrice = useMemo(() => {
    return filteredAndSorted.reduce((sum, item) => sum + item.price, 0);
  }, [filteredAndSorted]);

  return (
    <div style={{
      padding: '20px',
      backgroundColor: theme === 'dark' ? '#333' : '#fff',
      color: theme === 'dark' ? '#fff' : '#333',
      minHeight: '100vh',
    }}>
      <h2>商品検索（{allItems.length.toLocaleString()}件）</h2>

      <div style={{ display: 'flex', gap: '8px', marginBottom: '16px' }}>
        <input
          value={query}
          onChange={(e) => setQuery(e.target.value)}
          placeholder="キーワードで検索..."
          style={{ flex: 1, padding: '8px' }}
        />
        <select
          value={sortOrder}
          onChange={(e) => setSortOrder(e.target.value)}
          style={{ padding: '8px' }}
        >
          <option value="asc">価格：安い順</option>
          <option value="desc">価格：高い順</option>
        </select>
        <button onClick={() => setTheme(t => t === 'light' ? 'dark' : 'light')}>
          テーマ切替（再計算されないことを確認）
        </button>
      </div>

      <p>{filteredAndSorted.length}件表示 / 合計: ¥{totalPrice.toLocaleString()}</p>

      {/* 表示は先頭50件のみ */}
      <table style={{ width: '100%', borderCollapse: 'collapse' }}>
        <thead>
          <tr style={{ borderBottom: '2px solid #ddd' }}>
            <th style={{ textAlign: 'left', padding: '8px' }}>商品名</th>
            <th style={{ textAlign: 'left', padding: '8px' }}>カテゴリ</th>
            <th style={{ textAlign: 'right', padding: '8px' }}>価格</th>
          </tr>
        </thead>
        <tbody>
          {filteredAndSorted.slice(0, 50).map(item => (
            <tr key={item.id} style={{ borderBottom: '1px solid #eee' }}>
              <td style={{ padding: '8px' }}>{item.name}</td>
              <td style={{ padding: '8px' }}>{item.category}</td>
              <td style={{ textAlign: 'right', padding: '8px' }}>¥{item.price.toLocaleString()}</td>
            </tr>
          ))}
        </tbody>
      </table>
    </div>
  );
}

export default App;
```

**ポイント：**
- `useMemo` は依存配列の値が変わった時だけ再計算します。それ以外の state 変更では前回の計算結果を再利用します
- テーマ変更ボタンを押してもコンソールに「フィルタリング&ソート実行」が出ないことを確認してください
- `useState(() => generateItems(10000))` のように関数を渡すと、初回レンダリング時にだけ実行されます（遅延初期化）

</details>

## 演習3（基本）：useCallbackで子コンポーネントの再レンダリングを防ごう

`React.memo` された子コンポーネントに関数を渡す際、`useCallback` で関数をメモ化して不要な再レンダリングを防いでください。

**要件：**
- `TodoItem`（React.memo でラップ済み）に `onToggle` と `onDelete` 関数を渡す
- 新しいTodoを追加しても、既存の `TodoItem` が再レンダリングされないことを確認
- `console.log` でレンダリングを追跡

<details>
<summary>ヒント</summary>

`useCallback` を使わない場合、コンポーネントが再レンダリングされるたびに新しい関数が作られるため、`React.memo` が効きません。

```jsx
// 毎回新しい関数 → React.memo が効かない
const handleToggle = (id) => { ... };

// useCallback でメモ化 → 参照が安定する
const handleToggle = useCallback((id) => { ... }, []);
```

`setTodos` に関数形式（`prev => ...`）を使うと、依存配列に `todos` を含めなくて済みます。

</details>

<details>
<summary>解答例</summary>

```jsx
import { memo, useState, useCallback } from 'react';

// ========================================
// React.memoでラップした子コンポーネント
// propsが変わらなければレンダリングをスキップ
// ========================================
const TodoItem = memo(function TodoItem({ todo, onToggle, onDelete }) {
  console.log(`TodoItem "${todo.text}" がレンダリングされた`);

  return (
    <div style={{
      display: 'flex', alignItems: 'center', gap: '8px',
      padding: '8px', borderBottom: '1px solid #eee',
    }}>
      <input
        type="checkbox"
        checked={todo.completed}
        onChange={() => onToggle(todo.id)}
      />
      <span style={{
        flex: 1,
        textDecoration: todo.completed ? 'line-through' : 'none',
        color: todo.completed ? '#999' : '#333',
      }}>
        {todo.text}
      </span>
      <button onClick={() => onDelete(todo.id)}>削除</button>
    </div>
  );
});

function App() {
  const [todos, setTodos] = useState([
    { id: 1, text: '買い物', completed: false },
    { id: 2, text: '洗濯', completed: false },
    { id: 3, text: '掃除', completed: true },
  ]);
  const [input, setInput] = useState('');

  // ========================================
  // useCallback でメモ化：依存配列が [] なので関数の参照が安定する
  // setTodos(prev => ...) の関数形式を使うことで
  // todos を依存配列に含めなくて済む
  // ========================================
  const handleToggle = useCallback((id) => {
    setTodos(prev => prev.map(todo =>
      todo.id === id ? { ...todo, completed: !todo.completed } : todo
    ));
  }, []);

  const handleDelete = useCallback((id) => {
    setTodos(prev => prev.filter(todo => todo.id !== id));
  }, []);

  const handleAdd = () => {
    if (!input.trim()) return;
    setTodos(prev => [...prev, {
      id: Date.now(),
      text: input.trim(),
      completed: false,
    }]);
    setInput('');
  };

  return (
    <div style={{ maxWidth: '500px', margin: '0 auto', padding: '20px' }}>
      <h2>Todo（最適化版）</h2>

      <div style={{ display: 'flex', gap: '8px', marginBottom: '16px' }}>
        <input
          value={input}
          onChange={(e) => setInput(e.target.value)}
          onKeyDown={(e) => e.key === 'Enter' && handleAdd()}
          placeholder="新しいタスク..."
          style={{ flex: 1, padding: '8px' }}
        />
        <button onClick={handleAdd}>追加</button>
      </div>

      <p style={{ color: '#666' }}>
        コンソールを確認：新しいTodoを追加した時、既存のTodoItemは再レンダリングされません
      </p>

      {todos.map(todo => (
        <TodoItem
          key={todo.id}
          todo={todo}
          onToggle={handleToggle}
          onDelete={handleDelete}
        />
      ))}
    </div>
  );
}

export default App;
```

**ポイント：**
- `useCallback` は `useMemo` の関数版です。`useMemo(() => fn, deps)` と `useCallback(fn, deps)` は同じ効果です
- `setTodos(prev => ...)` の関数形式を使うことで、常に最新の `todos` を参照できます。これにより依存配列を `[]` にでき、関数の参照が安定します
- `React.memo` + `useCallback` のセットで初めて子コンポーネントの再レンダリングを防げます

</details>

## 演習4（応用）：useRefでストップウォッチを作ろう

`useRef` を使って、再レンダリングなしにタイマーIDを保持するストップウォッチを作成してください。

**要件：**
- 開始/停止/リセット の3つのボタン
- ミリ秒単位の表示（MM:SS.CC 形式）
- ラップタイム記録機能（「ラップ」ボタンで現在のタイムを記録）
- `useRef` でインターバルIDを保持（`useState` だと不要な再レンダリングが発生する）

<details>
<summary>ヒント</summary>

`useRef` の `.current` を変更しても再レンダリングは発生しません。タイマーIDのような「UIに表示しないが保持したい値」に最適です。

```jsx
const intervalRef = useRef(null);

// 開始
intervalRef.current = setInterval(() => { ... }, 10);

// 停止
clearInterval(intervalRef.current);
```

</details>

<details>
<summary>解答例</summary>

```jsx
import { useState, useRef, useEffect } from 'react';

function StopWatch() {
  const [time, setTime] = useState(0);
  const [isRunning, setIsRunning] = useState(false);
  const [laps, setLaps] = useState([]);

  // ========================================
  // useRef：再レンダリングを起こさずに値を保持
  // intervalRef → setInterval のIDを保持（UIに表示しない値）
  // startTimeRef → 開始時刻を保持
  // ========================================
  const intervalRef = useRef(null);
  const startTimeRef = useRef(0);

  const start = () => {
    if (isRunning) return;
    setIsRunning(true);
    startTimeRef.current = Date.now() - time;
    intervalRef.current = setInterval(() => {
      setTime(Date.now() - startTimeRef.current);
    }, 10);
  };

  const stop = () => {
    setIsRunning(false);
    clearInterval(intervalRef.current);
  };

  const reset = () => {
    stop();
    setTime(0);
    setLaps([]);
  };

  const lap = () => {
    if (!isRunning) return;
    setLaps(prev => [...prev, time]);
  };

  // コンポーネントがアンマウントされたらタイマーをクリーンアップ
  useEffect(() => {
    return () => clearInterval(intervalRef.current);
  }, []);

  // 時間のフォーマット
  const formatTime = (ms) => {
    const minutes = Math.floor(ms / 60000);
    const seconds = Math.floor((ms % 60000) / 1000);
    const centiseconds = Math.floor((ms % 1000) / 10);
    return `${String(minutes).padStart(2, '0')}:${String(seconds).padStart(2, '0')}.${String(centiseconds).padStart(2, '0')}`;
  };

  const buttonStyle = {
    padding: '10px 20px', margin: '4px', fontSize: '16px',
    borderRadius: '4px', border: 'none', cursor: 'pointer',
  };

  return (
    <div style={{ textAlign: 'center', padding: '20px' }}>
      <h2>ストップウォッチ</h2>

      <p style={{
        fontSize: '64px', fontFamily: 'monospace', margin: '20px 0',
        color: isRunning ? '#2c3e50' : '#999',
      }}>
        {formatTime(time)}
      </p>

      <div>
        {!isRunning ? (
          <button onClick={start} style={{ ...buttonStyle, backgroundColor: '#2ecc71', color: '#fff' }}>
            {time === 0 ? '開始' : '再開'}
          </button>
        ) : (
          <button onClick={stop} style={{ ...buttonStyle, backgroundColor: '#e74c3c', color: '#fff' }}>
            停止
          </button>
        )}
        <button onClick={lap} disabled={!isRunning}
          style={{ ...buttonStyle, backgroundColor: '#3498db', color: '#fff', opacity: isRunning ? 1 : 0.5 }}>
          ラップ
        </button>
        <button onClick={reset}
          style={{ ...buttonStyle, backgroundColor: '#95a5a6', color: '#fff' }}>
          リセット
        </button>
      </div>

      {laps.length > 0 && (
        <div style={{ marginTop: '20px' }}>
          <h3>ラップタイム</h3>
          <table style={{ margin: '0 auto', borderCollapse: 'collapse' }}>
            <thead>
              <tr style={{ borderBottom: '2px solid #ddd' }}>
                <th style={{ padding: '8px 16px' }}>#</th>
                <th style={{ padding: '8px 16px' }}>タイム</th>
                <th style={{ padding: '8px 16px' }}>スプリット</th>
              </tr>
            </thead>
            <tbody>
              {laps.map((lapTime, index) => (
                <tr key={index} style={{ borderBottom: '1px solid #eee' }}>
                  <td style={{ padding: '8px 16px' }}>{index + 1}</td>
                  <td style={{ padding: '8px 16px', fontFamily: 'monospace' }}>
                    {formatTime(lapTime)}
                  </td>
                  <td style={{ padding: '8px 16px', fontFamily: 'monospace' }}>
                    {formatTime(index === 0 ? lapTime : lapTime - laps[index - 1])}
                  </td>
                </tr>
              ))}
            </tbody>
          </table>
        </div>
      )}
    </div>
  );
}

export default StopWatch;
```

**ポイント：**
- `intervalRef` を `useState` で管理すると、タイマーIDを保存するたびに再レンダリングが発生してしまいます
- `useRef` の `.current` を変更しても再レンダリングは起きないので、UIに表示しない値の保持に最適です
- `useEffect` のクリーンアップ関数でタイマーをクリアするのを忘れないようにしましょう（メモリリーク防止）

</details>

## 演習5（応用）：最適化のバグを見つけよう

以下のコードには、パフォーマンス最適化に関するバグが3つあります。見つけて修正してください。

```jsx
import { memo, useState, useMemo, useCallback } from 'react';

const ProductItem = memo(function ProductItem({ product, onAddToCart }) {
  console.log(`ProductItem ${product.name} がレンダリングされた`);
  return (
    <div>
      <span>{product.name} - ¥{product.price}</span>
      <button onClick={() => onAddToCart(product)}>カートに追加</button>
    </div>
  );
});

function App() {
  const [cart, setCart] = useState([]);
  const [query, setQuery] = useState('');

  const products = [                         // バグ1
    { id: 1, name: 'React本', price: 2800 },
    { id: 2, name: 'Vue本', price: 3200 },
    { id: 3, name: 'Angular本', price: 2500 },
  ];

  const filtered = products.filter(p =>      // バグ2
    p.name.includes(query)
  );

  const addToCart = (product) => {            // バグ3
    setCart([...cart, product]);
  };

  return (
    <div>
      <input value={query} onChange={(e) => setQuery(e.target.value)} />
      <p>カート: {cart.length}件</p>
      {filtered.map(p => (
        <ProductItem key={p.id} product={p} onAddToCart={addToCart} />
      ))}
    </div>
  );
}
```

<details>
<summary>ヒント</summary>

- **バグ1：** `products` が毎レンダリングで新しい配列として作られていませんか？
- **バグ2：** フィルタリング結果が毎回再計算されていませんか？
- **バグ3：** `addToCart` が毎レンダリングで新しい関数になっていませんか？また、`cart` をクロージャで参照していると依存配列の問題が起きます

</details>

<details>
<summary>解答例</summary>

```jsx
import { memo, useState, useMemo, useCallback } from 'react';

const ProductItem = memo(function ProductItem({ product, onAddToCart }) {
  console.log(`ProductItem ${product.name} がレンダリングされた`);
  return (
    <div style={{ padding: '8px', borderBottom: '1px solid #eee', display: 'flex', justifyContent: 'space-between' }}>
      <span>{product.name} - ¥{product.price}</span>
      <button onClick={() => onAddToCart(product)}>カートに追加</button>
    </div>
  );
});

function App() {
  const [cart, setCart] = useState([]);
  const [query, setQuery] = useState('');

  // ========================================
  // 修正1：useState で products を定義
  //
  // 修正前：const products = [...] （毎レンダリングで新しい配列が作られる）
  // → filtered の useMemo が効かない（productsの参照が毎回変わるため）
  //
  // 修正後：useState で初期化すれば参照が安定する
  // ========================================
  const [products] = useState([
    { id: 1, name: 'React本', price: 2800 },
    { id: 2, name: 'Vue本', price: 3200 },
    { id: 3, name: 'Angular本', price: 2500 },
  ]);

  // ========================================
  // 修正2：useMemo でフィルタリング結果をメモ化
  //
  // 修正前：const filtered = products.filter(...)（毎回再計算）
  // → query以外のstate変更でも再計算される
  //
  // 修正後：useMemo で query と products が変わった時だけ再計算
  // ========================================
  const filtered = useMemo(() => {
    return products.filter(p => p.name.includes(query));
  }, [products, query]);

  // ========================================
  // 修正3：useCallback で関数をメモ化 + 関数形式の setState
  //
  // 修正前：const addToCart = (product) => { setCart([...cart, product]) }
  // → 毎回新しい関数が作られ、React.memo が効かない
  // → さらに cart をクロージャで参照しているため、
  //   古い cart を参照するバグが発生する可能性もある
  //
  // 修正後：useCallback + setCart(prev => ...) で安全かつ安定
  // ========================================
  const addToCart = useCallback((product) => {
    setCart(prev => [...prev, product]);
  }, []);

  return (
    <div style={{ maxWidth: '500px', margin: '0 auto', padding: '20px' }}>
      <h2>商品一覧</h2>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="検索..."
        style={{ width: '100%', padding: '8px', marginBottom: '12px' }}
      />
      <p>カート: {cart.length}件</p>
      {filtered.map(p => (
        <ProductItem key={p.id} product={p} onAddToCart={addToCart} />
      ))}
    </div>
  );
}

export default App;
```

**3つのバグのまとめ：**
1. **`products` の参照不安定** → `useState` で安定させる
2. **フィルタリングの無駄な再計算** → `useMemo` でメモ化
3. **`addToCart` の参照不安定 + クロージャの問題** → `useCallback` + `setCart(prev => ...)` で解決

</details>

## 演習6（チャレンジ）：useMemoとuseCallbackの使い分けを判断しよう

以下のシナリオで、`useMemo`、`useCallback`、`React.memo`、`useRef` のどれが適切か判断し、理由を説明してください。

**シナリオA：** 1万件のデータを価格順にソートして表示する

**シナリオB：** React.memoされた子コンポーネントに渡すイベントハンドラ

**シナリオC：** setIntervalのタイマーIDを保持する

**シナリオD：** 数値 `x` と `y` から合計値を計算する（`x + y`）

**シナリオE：** ユーザーが入力するたびにAPIを呼ぶデバウンス処理のタイマーID

<details>
<summary>解答例</summary>

### シナリオA：1万件のソート → **useMemo が適切**

```
理由：
- ソートは O(n log n) の重い計算
- データが変わらなければ再計算は無駄
- useMemo で依存配列にデータとソート条件を指定する

推奨：
const sorted = useMemo(() => {
  return [...data].sort((a, b) => a.price - b.price);
}, [data, sortOrder]);
```

### シナリオB：React.memoされた子への関数 → **useCallback が適切**

```
理由：
- 関数は毎レンダリングで新しい参照になる
- React.memo は props の参照を比較するので、
  関数の参照が変わると子コンポーネントが再レンダリングされる
- useCallback で関数の参照を安定させることで React.memo が効く

推奨：
const handleClick = useCallback((id) => {
  setItems(prev => prev.filter(item => item.id !== id));
}, []);
```

### シナリオC：setIntervalのタイマーID → **useRef が適切**

```
理由：
- タイマーIDはUIに表示しない
- useStateだと値を更新するたびに再レンダリングが起きてしまう
- useRefなら再レンダリングなしに値を保持できる

推奨：
const timerRef = useRef(null);
timerRef.current = setInterval(() => { ... }, 1000);
clearInterval(timerRef.current);
```

### シナリオD：x + y の計算 → **useMemoは不要**

```
理由：
- 足し算は O(1) の単純な計算
- useMemo のオーバーヘッド（依存配列の比較）の方が計算より重い
- 普通に計算すればよい

推奨：
const total = x + y;  // これで十分
// NG: const total = useMemo(() => x + y, [x, y]); // 過剰
```

### シナリオE：デバウンスのタイマーID → **useRef が適切**

```
理由：
- デバウンスの setTimeout ID はUIに表示しない
- 入力のたびに前のタイマーをクリアして新しいタイマーをセットする
- useStateだと不要な再レンダリングが発生する

推奨：
const debounceRef = useRef(null);

const handleInput = (value) => {
  clearTimeout(debounceRef.current);
  debounceRef.current = setTimeout(() => {
    fetchAPI(value);
  }, 300);
};
```

### 判断のフローチャート

```
「最適化が必要？」
  │
  ├── UIに表示しない値を保持 → useRef
  │
  ├── 重い計算結果をキャッシュ → useMemo
  │
  ├── React.memo した子に渡す関数 → useCallback
  │
  ├── 子コンポーネントの不要な再レンダリング → React.memo
  │
  └── 単純な計算 → 何もしない（普通に書く）
```

</details>
