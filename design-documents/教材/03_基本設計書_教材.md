# 第3章：基本設計書（外部設計）

## この章のゴール

- 基本設計書の目的と位置づけを理解する
- システム構成図（アーキテクチャ図）の描き方を習得する
- 機能一覧・機能設計の記述方法を学ぶ
- 画面遷移図の作成方法を把握する
- 外部インターフェース設計の考え方を理解する

---

## 3.1 基本設計書とは

### 目的

基本設計書（外部設計書）は、**ユーザーやステークホルダーから見たシステムの全体像と振る舞い**を定義するドキュメントです。

### 要件定義書との違い

| 項目 | 要件定義書 | 基本設計書 |
|------|-----------|-----------|
| **問い** | 何を作るか | どのような構造で作るか |
| **視点** | ビジネス・ユーザー視点 | システム全体の視点 |
| **粒度** | 機能の概要 | 機能の構造・画面・データ |
| **技術** | 技術に依存しない | 技術選定を含む |
| **読者** | 顧客、PM | 顧客、開発チーム |

```
要件定義書                    基本設計書
「商品を検索できる」  →  「検索画面のレイアウト」
                          「検索APIの仕様」
                          「検索結果の表示方法」
                          「絞り込み条件の種類」
```

### 基本設計書の構成

```markdown
1. システム概要
   1.1 システム構成図
   1.2 技術スタック
   1.3 開発・運用環境
2. 機能設計
   2.1 機能一覧
   2.2 各機能の詳細
3. 画面設計
   3.1 画面一覧
   3.2 画面遷移図
   3.3 各画面のレイアウト
4. データ設計（概要）
   4.1 主要エンティティ
   4.2 ER図（概要）
5. 外部インターフェース設計
   5.1 API一覧
   5.2 外部システム連携
6. セキュリティ設計
7. 運用設計
```

## 3.2 システム構成図

### システム構成図とは

システム構成図は、**システムを構成するサーバー、ネットワーク、外部サービスの関係**を図示したものです。開発チーム全員がシステムの全体像を共有するために作成します。

### Webアプリケーションの典型的な構成例

```
                    ┌─────────────┐
                    │   ユーザー    │
                    └──────┬──────┘
                           │ HTTPS
                    ┌──────▼──────┐
                    │     CDN      │
                    │ (CloudFront) │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  ロードバランサ │
                    │    (ALB)     │
                    └──┬───────┬──┘
                       │       │
              ┌────────▼─┐ ┌──▼────────┐
              │ Webサーバー│ │ Webサーバー │
              │  (App 1)  │ │  (App 2)  │
              └────┬──────┘ └──────┬────┘
                   │               │
              ┌────▼───────────────▼────┐
              │      データベース         │
              │    (RDS PostgreSQL)      │
              │   Primary ── Replica     │
              └────────────┬────────────┘
                           │
              ┌────────────▼────────────┐
              │       キャッシュ          │
              │     (ElastiCache)       │
              └─────────────────────────┘
```

### システム構成図に含める要素

| 要素 | 記載内容 | 例 |
|------|---------|-----|
| **サーバー/サービス** | 名前、役割、スペック | Webサーバー (EC2 t3.medium) |
| **ネットワーク** | プロトコル、ポート | HTTPS (443), WebSocket |
| **データストア** | DB、キャッシュ、ストレージ | PostgreSQL, Redis, S3 |
| **外部サービス** | 連携するSaaS・API | Stripe (決済), SendGrid (メール) |
| **セキュリティ境界** | VPC、ファイアウォール | VPC, Security Group |

## 3.3 技術スタック

### 技術スタックの記述例

基本設計書には使用する技術の一覧と選定理由を記載します。

```markdown
## 技術スタック

### フロントエンド
| 技術 | バージョン | 用途 | 選定理由 |
|------|-----------|------|---------|
| React | 18.x | UIフレームワーク | コンポーネントベースで再利用性が高い |
| TypeScript | 5.x | 型付き言語 | 型安全により品質が向上する |
| Tailwind CSS | 3.x | スタイリング | ユーティリティファーストで開発効率が良い |

### バックエンド
| 技術 | バージョン | 用途 | 選定理由 |
|------|-----------|------|---------|
| Node.js | 20.x LTS | ランタイム | フロントと同一言語で開発効率が良い |
| Express | 4.x | Webフレームワーク | 軽量で柔軟性が高い |
| Prisma | 5.x | ORM | 型安全なDB操作が可能 |

### インフラ
| 技術 | 用途 | 選定理由 |
|------|------|---------|
| AWS ECS | コンテナ実行 | Dockerベースでスケーラブル |
| RDS PostgreSQL | データベース | 信頼性が高く機能が豊富 |
| ElastiCache Redis | キャッシュ | 高速なインメモリデータストア |
```

## 3.4 機能設計

### 機能一覧

各機能をID付きで整理します。要件定義書の機能要件をさらに具体化したものです。

```markdown
## 機能一覧

| 機能ID | 機能カテゴリ | 機能名 | 概要 |
|--------|------------|--------|------|
| F-001 | 認証 | ユーザー登録 | メールアドレスとパスワードで新規登録 |
| F-002 | 認証 | ログイン | メール/パスワード認証 |
| F-003 | 認証 | パスワードリセット | メール経由でパスワードを再設定 |
| F-010 | 商品 | 商品一覧表示 | カテゴリ・ソート付き一覧表示 |
| F-011 | 商品 | 商品詳細表示 | 商品の詳細情報と画像を表示 |
| F-012 | 商品 | 商品検索 | キーワード・フィルターで検索 |
| F-020 | カート | カート表示 | カート内商品の一覧表示 |
| F-021 | カート | カート追加 | 商品をカートに追加 |
| F-022 | カート | カート編集 | 数量変更・商品削除 |
| F-030 | 注文 | 注文手続き | 配送先・支払方法選択 |
| F-031 | 注文 | 注文確定 | 決済実行・注文確定 |
| F-032 | 注文 | 注文履歴 | 過去の注文一覧表示 |
```

### 機能の詳細設計

各機能について、入出力や処理概要を記載します。

```markdown
## F-012 商品検索

### 概要
ユーザーがキーワードやフィルター条件を指定して商品を検索する機能。

### 入力
| 項目 | 型 | 必須 | 説明 |
|------|-----|------|------|
| keyword | string | × | 検索キーワード |
| category_id | integer | × | カテゴリID |
| min_price | integer | × | 最低価格 |
| max_price | integer | × | 最高価格 |
| sort | string | × | ソート条件（price_asc, price_desc, newest） |
| page | integer | × | ページ番号（デフォルト：1） |

### 出力
- 商品一覧（商品名、価格、画像サムネイル、評価）
- 検索結果件数
- ページネーション情報

### 処理概要
1. 入力パラメータのバリデーション
2. キーワードが指定された場合、商品名・説明文で部分一致検索
3. フィルター条件で絞り込み
4. 指定のソート条件で並び替え
5. ページネーション処理（1ページ20件）
6. 検索結果を返却

### ビジネスルール
- 非公開商品は検索結果に含めない
- 在庫切れ商品は検索結果の末尾に表示する
- キーワード・フィルターが未指定の場合は新着順で全商品を表示
```

## 3.5 画面遷移図

### 画面遷移図とは

画面遷移図は、**ユーザーがどの画面からどの画面に移動できるか**を図示したものです。

### 画面遷移図の例（ECサイト）

```
┌──────────┐    検索     ┌──────────┐    商品選択   ┌──────────┐
│  トップ   │──────────→│ 検索結果  │──────────→│ 商品詳細  │
│  ページ   │           │  一覧    │           │          │
└────┬─────┘           └──────────┘           └────┬─────┘
     │                                              │
     │ ログイン                          カートに追加 │
     ▼                                              ▼
┌──────────┐                              ┌──────────┐
│ ログイン  │                              │  カート   │
│  画面    │                              │          │
└────┬─────┘                              └────┬─────┘
     │                                         │
     │ 認証成功                        注文手続き │
     ▼                                         ▼
┌──────────┐                              ┌──────────┐
│ マイページ │                              │ 注文手続き │
│          │                              │          │
└──────────┘                              └────┬─────┘
                                                │
                                        注文確定  │
                                                ▼
                                          ┌──────────┐
                                          │ 注文完了  │
                                          │          │
                                          └──────────┘
```

### 画面一覧

| 画面ID | 画面名 | URL | 認証 | 説明 |
|--------|--------|-----|------|------|
| SC-001 | トップページ | / | 不要 | サイトのトップ画面 |
| SC-002 | 商品一覧 | /products | 不要 | カテゴリ別商品一覧 |
| SC-003 | 商品詳細 | /products/:id | 不要 | 商品の詳細情報 |
| SC-004 | 検索結果 | /search | 不要 | 検索結果の一覧 |
| SC-010 | ログイン | /login | 不要 | ログインフォーム |
| SC-011 | ユーザー登録 | /register | 不要 | 新規登録フォーム |
| SC-020 | カート | /cart | 必要 | カート内容の表示・編集 |
| SC-021 | 注文手続き | /checkout | 必要 | 配送先・支払方法の選択 |
| SC-022 | 注文完了 | /order/complete | 必要 | 注文完了の確認 |
| SC-030 | マイページ | /mypage | 必要 | ユーザー情報・注文履歴 |

## 3.6 外部インターフェース設計

### 外部システムとの連携

システムが連携する外部サービスやシステムを一覧化します。

```markdown
## 外部インターフェース一覧

| IF-ID | 連携先 | 連携方式 | 方向 | 用途 |
|-------|--------|---------|------|------|
| IF-001 | Stripe | REST API | 送信 | クレジットカード決済 |
| IF-002 | SendGrid | REST API | 送信 | メール送信（注文確認等） |
| IF-003 | S3 | AWS SDK | 送信 | 商品画像の保存・取得 |
| IF-004 | Google Analytics | JavaScript | 送信 | アクセス解析 |
| IF-005 | 基幹システム | バッチ連携（CSV） | 受信 | 商品マスタ同期 |
```

### 外部IF設計の記載項目

各インターフェースについて、以下を定義します。

| 項目 | 説明 |
|------|------|
| 通信プロトコル | HTTPS、WebSocket、SFTP等 |
| データ形式 | JSON、XML、CSV等 |
| 認証方式 | APIキー、OAuth2、Basic認証等 |
| タイムアウト | 接続・読み取りタイムアウト値 |
| リトライ | リトライ回数と間隔 |
| エラーハンドリング | 連携先障害時の動作 |

## 3.7 セキュリティ設計（概要）

### 基本設計段階でのセキュリティ方針

| セキュリティ領域 | 方針 |
|----------------|------|
| **認証** | JWTトークンベース認証。アクセストークン（15分）+ リフレッシュトークン（7日） |
| **認可** | ロールベースアクセス制御（RBAC）。一般ユーザー・管理者の2ロール |
| **通信** | 全通信をHTTPS（TLS 1.2以上）で暗号化 |
| **データ保護** | パスワードはbcryptでハッシュ化。個人情報はAES-256で暗号化 |
| **入力検証** | 全入力値をサーバーサイドでバリデーション。SQLインジェクション・XSS対策 |
| **ログ** | アクセスログ・操作ログを記録。個人情報はマスキング |

| 概念 | ポイント |
|------|---------|
| 基本設計書 | ユーザー・ステークホルダー視点でシステム全体の構造を定義する |

| 外部IF設計 | 外部システムとの連携方式・データ形式を定義する |
