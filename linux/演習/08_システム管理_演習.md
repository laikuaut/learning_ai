### 問題1: systemctl の基本操作
以下の操作を行う `systemctl` コマンドを書いてください。

1. nginx サービスの状態を確認する
2. nginx サービスを起動する
3. nginx サービスを停止する
4. nginx サービスをOS起動時に自動起動するように設定する
5. 起動に失敗したサービスの一覧を表示する

<details>
<summary>解答</summary>

```bash
# 1. 状態の確認
$ sudo systemctl status nginx

# 2. サービスの起動
$ sudo systemctl start nginx

# 3. サービスの停止
$ sudo systemctl stop nginx

# 4. 自動起動の有効化
$ sudo systemctl enable nginx
# ※ 有効化と同時に起動したい場合
$ sudo systemctl enable --now nginx

# 5. 起動に失敗したサービスの一覧
$ systemctl --failed
```

</details>

### 問題3: journalctl の基本
以下の要件を満たす `journalctl` コマンドを書いてください。

1. nginx サービスのログだけを表示する
2. 今日のログだけを表示する
3. エラー以上のログだけを表示する
4. 最新のログをリアルタイムで追跡する
5. 直近1時間のログを表示する

<details>
<summary>解答</summary>

```bash
# 1. 特定サービスのログ
$ journalctl -u nginx.service

# 2. 今日のログ
$ journalctl --since today

# 3. エラー以上のログ
$ journalctl -p err

# 4. リアルタイム追跡
$ journalctl -f

# 5. 直近1時間のログ
$ journalctl --since "1 hour ago"
```

</details>

### 問題4: cron のスケジュール設定
以下の実行タイミングを crontab の書式で書いてください。

1. 毎日午前3時に実行
2. 毎週月曜日の午前9時に実行
3. 5分ごとに実行
4. 平日（月〜金）の午前8時30分に実行
5. 毎月1日の午前0時に実行

<details>
<summary>解答</summary>

```bash
# crontab の書式: 分 時 日 月 曜日 コマンド

# 1. 毎日午前3時
0 3 * * * /path/to/script.sh

# 2. 毎週月曜日の午前9時
0 9 * * 1 /path/to/script.sh

# 3. 5分ごと
*/5 * * * * /path/to/script.sh

# 4. 平日の午前8時30分
30 8 * * 1-5 /path/to/script.sh

# 5. 毎月1日の午前0時
0 0 1 * * /path/to/script.sh
```

</details>

### 問題6: メモリとCPU
以下の質問に答えてください。

1. メモリ使用量を確認するコマンドは何ですか？
2. `free -h` の出力で `available` が示す意味は何ですか？
3. ロードアベレージを確認するコマンドは何ですか？
4. CPUが4コアのサーバーで、ロードアベレージが 8.0 の場合、サーバーの状態はどうですか？

<details>
<summary>解答</summary>

```bash
# 1. メモリ使用量の確認
$ free -h

# 2. available の意味
# 新しいプロセスが使用可能なメモリ量です。
# Linux はメモリを積極的にキャッシュに使うため、free が少なくても
# available が十分にあれば問題ありません。
# available = free + 解放可能なキャッシュ

# 3. ロードアベレージの確認
$ uptime
# または
$ top（で上部に表示される）

# 4. CPUコア数4でロードアベレージ8.0
# ロードアベレージがCPUコア数の2倍（4 x 2 = 8）に達しており、
# 高負荷の状態です。プロセスがCPUの処理を待っている状態であり、
# 要調査です。
```

</details>

### 問題7: cron の実践設定
以下の要件を満たす crontab を作成してください。出力は適切にログファイルにリダイレクトしてください。

1. 毎日午前3時にバックアップスクリプト `/home/tanaka/scripts/backup.sh` を実行
2. 10分ごとにヘルスチェック `/home/tanaka/scripts/health_check.sh` を実行
3. 毎週日曜日の午前2時にログクリーンアップ `/home/tanaka/scripts/cleanup.sh` を実行

<details>
<summary>解答</summary>

```bash
# crontab -e で以下を設定

# 環境変数の設定（推奨）
SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin

# 1. 毎日午前3時にバックアップ
0 3 * * * /home/tanaka/scripts/backup.sh >> /var/log/backup.log 2>&1

# 2. 10分ごとにヘルスチェック
*/10 * * * * /home/tanaka/scripts/health_check.sh >> /var/log/health_check.log 2>&1

# 3. 毎週日曜午前2時にクリーンアップ
0 2 * * 0 /home/tanaka/scripts/cleanup.sh >> /var/log/cleanup.log 2>&1
```

ポイント：
- コマンドはフルパスで記述する
- 出力をログファイルにリダイレクトする（`>> /var/log/xxx.log 2>&1`）
- SHELL と PATH を冒頭に設定すると安全

</details>

### 問題8: ログの調査
以下のシナリオで使うコマンドを書いてください。

**シナリオ**: 今朝の9時頃からWebサーバー（nginx）でエラーが発生しているとの報告がありました。原因を調査してください。

<details>
<summary>解答</summary>

```bash
# Step 1: nginx サービスの状態を確認
$ sudo systemctl status nginx

# Step 2: 今朝9時以降のnginxログを確認
$ journalctl -u nginx.service --since "today 09:00" --no-pager

# Step 3: エラーレベル以上のログに絞り込み
$ journalctl -u nginx.service --since "today 09:00" -p err --no-pager

# Step 4: nginx のエラーログを直接確認
$ tail -100 /var/log/nginx/error.log
# 9時以降のエラーに絞り込み
$ grep "$(date +%Y/%m/%d)" /var/log/nginx/error.log | grep -i "error"

# Step 5: アクセスログで異常なリクエストを確認
$ awk '$9 >= 400' /var/log/nginx/access.log | tail -20

# Step 6: サーバーリソースの確認
$ df -h          # ディスク使用量
$ free -h        # メモリ使用量
$ uptime         # ロードアベレージ
```

</details>

### 問題10: サーバー起動後のチェックスクリプト
サーバー起動後のヘルスチェックを自動化するスクリプトを作成してください。

チェック項目：
1. 起動に失敗したサービスがないか確認
2. ディスク使用率が80%を超えるパーティションがないか確認
3. メモリの使用状況を確認
4. ロードアベレージを確認
5. 直近30分以内のエラーログがないか確認

期待される出力例：
```
=== サーバーヘルスチェック ===
日時: 2024-03-20 15:30:00

[1] サービス状態の確認
  ✓ すべてのサービスが正常に動作しています

[2] ディスク使用率の確認
  ✓ / : 48% (正常)
  ! /home : 82% (警告: 80%超過)

[3] メモリ使用状況
  合計: 7.8G  使用中: 2.1G  空き: 5.2G (available)

[4] ロードアベレージ
  0.25, 0.18, 0.15 (CPUコア数: 4)

[5] 直近のエラーログ
  エラーなし
```

<details>
<summary>ヒント</summary>

`systemctl --failed`、`df -h`、`free -h`、`uptime`、`journalctl -p err --since "30 min ago"` を組み合わせます。

</details>

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# health_check.sh - サーバーヘルスチェック

echo "=== サーバーヘルスチェック ==="
echo "日時: $(date '+%Y-%m-%d %H:%M:%S')"
echo ""

# [1] サービス状態の確認
echo "[1] サービス状態の確認"
FAILED=$(systemctl --failed --no-legend | wc -l)
if [ "$FAILED" -eq 0 ]; then
    echo "  OK: すべてのサービスが正常に動作しています"
else
    echo "  NG: $FAILED 個のサービスが失敗しています"
    systemctl --failed --no-legend | while read -r line; do
        echo "    - $line"
    done
fi
echo ""

# [2] ディスク使用率の確認
echo "[2] ディスク使用率の確認"
df -h | awk 'NR>1 {
    usage = $5 + 0  # % を数値に変換
    if (usage >= 80) {
        printf "  NG: %-10s %s (警告: 80%%超過)\n", $6, $5
    } else {
        printf "  OK: %-10s %s (正常)\n", $6, $5
    }
}'
echo ""

# [3] メモリ使用状況
echo "[3] メモリ使用状況"
free -h | awk '/^Mem:/ {printf "  合計: %s  使用中: %s  空き: %s (available)\n", $2, $3, $7}'
echo ""

# [4] ロードアベレージ
echo "[4] ロードアベレージ"
LOAD=$(uptime | awk -F'load average:' '{print $2}')
CORES=$(nproc)
echo "  $LOAD (CPUコア数: $CORES)"
echo ""

# [5] 直近のエラーログ
echo "[5] 直近のエラーログ"
ERROR_COUNT=$(journalctl -p err --since "30 min ago" --no-pager 2>/dev/null | wc -l)
if [ "$ERROR_COUNT" -le 1 ]; then
    echo "  エラーなし"
else
    echo "  $ERROR_COUNT 件のエラーがあります"
    journalctl -p err --since "30 min ago" --no-pager 2>/dev/null | tail -5
fi
```

</details>

### 問題11: ジャーナルログの管理
以下の操作を行うコマンドを書いてください。

1. ジャーナルログのディスク使用量を確認する
2. ジャーナルログを100MB以下に制限する
3. 7日以上前のジャーナルログを削除する

<details>
<summary>解答</summary>

```bash
# 1. ディスク使用量の確認
$ journalctl --disk-usage
Archived and active journals take up 256.0M in the file system.

# 2. 100MB以下に制限
$ sudo journalctl --vacuum-size=100M

# 3. 7日以上前のログを削除
$ sudo journalctl --vacuum-time=7d
```

</details>
