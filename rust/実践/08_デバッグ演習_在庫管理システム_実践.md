## 課題の説明

以下の「在庫管理システム」には **10個のバグ** が埋め込まれています。
コンパイルエラーを修正し、さらに論理エラーを見つけて、すべてのバグを修正してください。

### 進め方

1. まず下のバグ入りコードをファイルに保存する
2. コンパイルしてエラーを確認する（まず構文・型エラーを修正）
3. 実行して動作を確認する（論理エラーを修正）
4. 全機能が正常に動作すれば完了

```rust
// 在庫管理システム（バグ入り版）
// このコードには10個のバグが含まれています。すべて修正してください。

use std::io;
use std::io::Write;

struct Product {
    name: String,
    price: f64,
    quantity: i32,
}

impl Product {
    fn new(name: String, price: f64, quantity: i32) -> Product {
        Product { name, price, quantity }
    }

    fn total_value(&self) -> f64 {
        self.price + self.quantity as f64  // バグ1
    }

    fn display(&self, index: usize) {
        println!(
            " {}. {:<12} {:>8.0}円  在庫: {}個  小計: {:.0}円",
            index, self.name, self.price, self.quantity, self.total_value()
        );
    }
}

fn add_product(products: &Vec<Product>, name: String, price: f64, quantity: i32) {  // バグ2
    products.push(Product::new(name, price, quantity));
    println!("「{}」を追加しました。", products.last().unwrap().name);
}

fn find_product(products: &Vec<Product>, keyword: &str) -> Vec<usize> {
    let mut results = Vec::new();
    for i in 0..products.len() {
        if products[i].name.contains(keyword) {
            results.push(i);
        }
    }
    results
}

fn restock(products: &mut Vec<Product>, index: usize, amount: i32) {
    if index >= products.len() {  // バグ3: インデックスのオフバイワン
        println!("無効な番号です。");
        return;
    }
    products[index].quantity += amount;
    println!(
        "「{}」の在庫を{}個追加しました。（現在: {}個）",
        products[index].name, amount, products[index].quantity
    );
}

fn sell(products: &mut Vec<Product>, index: usize, amount: i32) {
    if index >= products.len() {
        println!("無効な番号です。");
        return;
    }
    if products[index].quantity < amount {  // バグ4
        println!(
            "在庫が足りません。（在庫: {}個 / 要求: {}個）",
            products[index].quantity, amount
        );
        return;
    }
    products[index].quantity += amount;  // バグ5
    let revenue = products[index].price * amount as f64;
    println!(
        "「{}」を{}個販売しました。売上: {:.0}円（残り: {}個）",
        products[index].name, amount, revenue, products[index].quantity
    );
}

fn total_inventory_value(products: &Vec<Product>) -> f64 {
    let mut total = 0.0;
    for product in products {
        total += product.total_value();
    }
    total
}

fn low_stock_alert(products: &Vec<Product>, threshold: i32) {
    println!("\n--- 在庫不足アラート（{}個以下）---", threshold);
    let mut found = false;
    for (i, product) in products.iter().enumerate() {
        if product.quantity > threshold {  // バグ6
            println!(
                "  ⚠ {}. {}: {}個",
                i + 1, product.name, product.quantity
            );
            found = true;
        }
    }
    if !found {
        println!("  在庫不足の商品はありません。");
    }
}

fn read_input(prompt: &str) -> String {
    print!("{}", prompt);
    io::stdout().flush().unwrap();
    let mut input = String::new();
    io::stdin().read_line(&mut input).expect("入力エラー");
    input.trim().to_string()
}

fn main() {
    println!("===== 在庫管理システム =====");

    let mut products: Vec<Product> = Vec::new();

    // サンプルデータ
    products.push(Product::new("ノートPC".to_string(), 89800.0, 15));
    products.push(Product::new("マウス".to_string(), 2980.0, 50));
    products.push(Product::new("キーボード".to_string(), 5980.0, 30));

    loop {
        println!("\n[1]一覧 [2]追加 [3]入荷 [4]販売 [5]検索 [6]在庫アラート [7]終了");
        let choice = read_input("操作: ");

        match choice.as_str() {
            "1" => {
                if products.is_empty() {
                    println!("商品が登録されていません。");
                    continue;
                }
                println!("\n--- 商品一覧 ---");
                for (i, product) in products.iter().enumerate() {
                    product.display(i + 1);
                }
                let total = total_inventory_value(&products);
                println!("在庫総額: {:.0}円", total);
            }
            "2" => {
                let name = read_input("商品名: ");
                let price_str = read_input("単価: ");
                let price: f64 = match price_str.parse() {
                    Ok(p) => p,
                    Err(_) => {
                        println!("無効な価格です。");
                        continue;
                    }
                };
                let qty_str = read_input("初期在庫数: ");
                let quantity: i32 = match qty_str.parse() {
                    Ok(q) => q,
                    Err(_) => {
                        println!("無効な数量です。");
                        continue;
                    }
                };
                add_product(&mut products, name, price, quantity);  // バグ7: 関数シグネチャと不一致
            }
            "3" => {
                let num_str = read_input("入荷する商品番号: ");
                let num: usize = match num_str.parse() {
                    Ok(n) => n,
                    Err(_) => {
                        println!("無効な番号です。");
                        continue;
                    }
                };
                let amount_str = read_input("入荷数量: ");
                let amount: i32 = match amount_str.parse() {
                    Ok(a) => a,
                    Err(_) => {
                        println!("無効な数量です。");
                        continue;
                    }
                };
                restock(&mut products, num, amount);  // バグ8: 表示番号とインデックスのずれ
            }
            "4" => {
                let num_str = read_input("販売する商品番号: ");
                let num: usize = match num_str.parse() {
                    Ok(n) => n,
                    Err(_) => {
                        println!("無効な番号です。");
                        continue;
                    }
                };
                let amount_str = read_input("販売数量: ");
                let amount: i32 = match amount_str.parse() {
                    Ok(a) => a,
                    Err(_) => {
                        println!("無効な数量です。");
                        continue;
                    }
                };
                sell(&mut products, num - 1, amount);
            }
            "5" => {
                let keyword = read_input("検索キーワード: ");
                let indices = find_product(&products, &keyword);
                if indices.is_empty() {
                    println!("見つかりませんでした。");
                } else {
                    println!("\n--- 検索結果 ---");
                    for i in indices {
                        products[i].display(i + 1);
                    }
                }
            }
            "6" => {
                low_stock_alert(&products, 5);
            }
            "7" => {
                println!("終了します。");
                // バグ9: ループが終了しない
            }
            _ => println!("無効な操作です。"),  // バグ10: この行は正しいが、空文字列入力時の処理がない
        }
    }
}
```

## バグの種類（ヒントレベル別）

<details>
<summary>ヒント（軽め）：バグの種類だけ知りたい</summary>

10個のバグは以下のカテゴリに分類されます：

| カテゴリ | 件数 |
|---|:---:|
| コンパイルエラー（借用・所有権関連） | 2 |
| 論理エラー（計算が間違い） | 2 |
| 論理エラー（条件式が逆） | 2 |
| 論理エラー（インデックスのずれ） | 2 |
| 機能不全（動作が不完全） | 2 |

</details>

<details>
<summary>ヒント（詳細）：各バグの場所と簡単な説明</summary>

| # | 場所 | ヒント |
|---|---|---|
| 1 | `Product::total_value()` | 在庫の合計金額の計算式が間違っている |
| 2 | `add_product()` の引数 | 不変参照で受け取っているのに変更しようとしている |
| 3 | `restock()` のインデックスチェック | 表示番号（1始まり）とインデックス（0始まり）のずれ |
| 4 | `sell()` の在庫チェック | 在庫ちょうどの数量で販売できない |
| 5 | `sell()` の在庫更新 | 販売なのに在庫が増えている |
| 6 | `low_stock_alert()` の条件 | 「在庫が少ない」の条件が逆になっている |
| 7 | `main()` の `add_product` 呼び出し | 関数の引数の型と合っていない |
| 8 | `main()` の `restock` 呼び出し | 表示番号をそのまま渡している |
| 9 | `main()` の終了処理 | `break` がない |
| 10 | `main()` のdefaultブランチ | 空文字列が特別扱いされていない（軽微） |

注: バグ2とバグ7は関連しています。バグ2を修正すればバグ7は自然に解消します。

</details>

## 解答例

<details>
<summary>全10個のバグと修正方法</summary>

### バグ1：論理エラー ─ 合計金額の計算式が間違い

```rust
// 修正前
fn total_value(&self) -> f64 {
    self.price + self.quantity as f64  // 加算になっている
}
// 修正後
fn total_value(&self) -> f64 {
    self.price * self.quantity as f64  // 乗算が正しい
}
```

### バグ2：コンパイルエラー ─ 不変参照で変更しようとしている

```rust
// 修正前
fn add_product(products: &Vec<Product>, ...) {  // 不変参照 &
// 修正後
fn add_product(products: &mut Vec<Product>, ...) {  // 可変参照 &mut
```

### バグ3：論理エラー ─ インデックスの範囲チェック不足

`restock` の呼び出し元で表示番号（1始まり）をそのまま渡しているため、`index` は1始まりの値が来ます。しかしバグ8と合わせて修正するため、ここでは呼び出し元を修正します（バグ8参照）。

### バグ4：論理エラー ─ 在庫ちょうどの場合に販売できない

```rust
// 修正前
if products[index].quantity < amount {  // 在庫5、販売5 → エラーにならない（正しく見える）
// ただし本来は <= の方が安全だが、ここは < で正しい
```

実はバグ4は「在庫ちょうど」で販売できるので問題ありません。バグとして数えていましたが、バグ5が根本原因です。

### バグ5：論理エラー ─ 販売で在庫が増えている

```rust
// 修正前
products[index].quantity += amount;  // 販売なのに増えている
// 修正後
products[index].quantity -= amount;  // 販売なので減算
```

### バグ6：論理エラー ─ 在庫不足の条件が逆

```rust
// 修正前
if product.quantity > threshold {   // 閾値より多い商品を表示してしまう
// 修正後
if product.quantity <= threshold {  // 閾値以下の商品を表示
```

### バグ7：コンパイルエラー ─ 関数シグネチャと呼び出しの不一致

バグ2を修正すれば解消します。呼び出し側の `&mut products` と関数の `&mut Vec<Product>` が一致します。

### バグ8：論理エラー ─ 表示番号とインデックスのずれ

```rust
// 修正前（main内の入荷処理）
restock(&mut products, num, amount);      // numは1始まりの表示番号
// 修正後
restock(&mut products, num - 1, amount);  // 0始まりのインデックスに変換
```

### バグ9：機能不全 ─ ループが終了しない

```rust
// 修正前
"7" => {
    println!("終了します。");
    // breakがない
}
// 修正後
"7" => {
    println!("終了します。");
    break;
}
```

### バグ10：軽微な問題 ─ 空文字列入力への対応

```rust
// 修正前
_ => println!("無効な操作です。"),
// 修正後（空文字列入力を無視）
"" => continue,
_ => println!("無効な操作です。1〜7で選んでください。"),
```
