# 第6章：ポインタ

## この章のゴール

- メモリとアドレスの仕組みを理解する
- ポインタ変数の宣言・使い方をマスターする
- `&`（アドレス演算子）と `*`（間接参照演算子）を正しく使い分けられる
- ポインタと配列の関係を理解する
- 関数への参照渡し（ポインタ渡し）を使いこなす
- `malloc` / `free` による動的メモリ確保の基本を習得する

## 6.1 メモリとアドレス

コンピュータのメモリ（memory）は、データを格納する巨大な棚のようなものです。
すべてのメモリ上の場所には**アドレス（address）**という番号がついています。

```
  アドレス     メモリの中身
 ┌──────────┬──────────┐
 │ 0x1000   │   10     │  ← 変数 a の場所
 ├──────────┼──────────┤
 │ 0x1004   │   20     │  ← 変数 b の場所
 ├──────────┼──────────┤
 │ 0x1008   │  0x1000  │  ← ポインタ p の場所（a のアドレスを格納）
 ├──────────┼──────────┤
 │ 0x100C   │   ...    │
 └──────────┴──────────┘
```

変数を宣言すると、コンパイラがメモリ上の場所を確保します。
`&`（アドレス演算子（address-of operator））を使うと、その変数のアドレスを取得できます。

```c
#include <stdio.h>

int main(void) {
    int a = 10;
    int b = 20;

    printf("a の値: %d\n", a);
    printf("a のアドレス: %p\n", (void *)&a);
    printf("b の値: %d\n", b);
    printf("b のアドレス: %p\n", (void *)&b);

    return 0;
}
```

**実行例：**
```
a の値: 10
a のアドレス: 0x7ffd5a3b4c5c
b の値: 20
b のアドレス: 0x7ffd5a3b4c58
```

アドレスの値は実行するたびに変わります。`%p` はポインタ（アドレス）を表示するための書式指定子です。

## 6.2 ポインタ変数の宣言と使い方

**ポインタ（pointer）**とは、メモリのアドレスを格納するための変数です。

### 宣言の書き方

```c
int *p;    // int 型の値を指すポインタ
char *cp;  // char 型の値を指すポインタ
double *dp; // double 型の値を指すポインタ
```

`*` はポインタであることを示します。「`p` は `int` へのポインタ」と読みます。

### & と * の関係

| 演算子 | 名前 | 意味 |
|--------|------|------|
| `&x` | アドレス演算子 | 変数 `x` のアドレスを取得 |
| `*p` | 間接参照演算子（dereference operator） | ポインタ `p` が指す先の値を取得 |

```c
#include <stdio.h>

int main(void) {
    int a = 42;
    int *p = &a;  // p に a のアドレスを代入

    printf("a の値     : %d\n", a);
    printf("&a (アドレス): %p\n", (void *)&a);
    printf("p の値     : %p\n", (void *)p);   // a のアドレスと同じ
    printf("*p (指す先)  : %d\n", *p);         // a の値と同じ

    *p = 100;  // ポインタ経由で a の値を変更
    printf("変更後の a  : %d\n", a);  // 100

    return 0;
}
```

```
  &a と *p の関係図

    変数 a          ポインタ p
  ┌────────┐      ┌────────┐
  │   42   │ ←──  │ &a     │
  └────────┘  *p  └────────┘
   アドレス:         p の値 =
   0x1000          0x1000
```

> **よくある間違い：** `int *p;` と宣言しただけでは `p` はどこも指していません（未初期化）。
> 未初期化のポインタに `*p = 10;` とすると**未定義動作（undefined behavior）**になり、
> プログラムがクラッシュする原因となります。必ず有効なアドレスを代入してから使いましょう。

## 6.3 NULLポインタ

**NULLポインタ（null pointer）**は「どこも指していない」ことを明示するための特別な値です。

```c
#include <stdio.h>

int main(void) {
    int *p = NULL;  // どこも指していないことを明示

    if (p == NULL) {
        printf("p は NULL です。安全のため参照しません。\n");
    }

    // *p = 10;  // ← これは実行時エラー（セグメンテーション違反）になる！

    int x = 5;
    p = &x;  // 有効なアドレスを代入

    if (p != NULL) {
        printf("*p = %d\n", *p);  // 安全に参照できる
    }

    return 0;
}
```

ポインタを使う前に `NULL` チェックを行うのは、C言語の重要な習慣です。

## 6.4 ポインタと配列

C言語では、配列名はその先頭要素のアドレスとして扱われます。

```c
#include <stdio.h>

int main(void) {
    int arr[5] = {10, 20, 30, 40, 50};
    int *p = arr;  // arr は &arr[0] と同じ

    printf("arr[0] = %d,  *p = %d\n", arr[0], *p);

    // ポインタ演算で要素にアクセス
    for (int i = 0; i < 5; i++) {
        printf("arr[%d] = %d,  *(p+%d) = %d\n", i, arr[i], i, *(p + i));
    }

    return 0;
}
```

```
  配列とポインタの関係

  arr (= &arr[0])
   │
   ▼
  ┌────┬────┬────┬────┬────┐
  │ 10 │ 20 │ 30 │ 40 │ 50 │
  └────┴────┴────┴────┴────┘
  p    p+1  p+2  p+3  p+4
```

### ポインタ演算（pointer arithmetic）

ポインタに整数を足すと、「その型のサイズ分」だけアドレスが進みます。

```c
#include <stdio.h>

int main(void) {
    int arr[3] = {100, 200, 300};
    int *p = arr;

    printf("p   のアドレス: %p, 値: %d\n", (void *)p, *p);
    printf("p+1 のアドレス: %p, 値: %d\n", (void *)(p+1), *(p+1));
    printf("p+2 のアドレス: %p, 値: %d\n", (void *)(p+2), *(p+2));

    // int は通常4バイトなので、アドレスが4ずつ増える
    return 0;
}
```

> **ポイント：** `p[i]` と `*(p + i)` は完全に同じ意味です。
> 配列の添字アクセスは、実はポインタ演算の別の書き方に過ぎません。

## 6.5 関数への参照渡し（ポインタ渡し）

第5章で学んだように、C言語の関数引数は**値渡し（pass by value）**です。
しかしポインタを渡せば、呼び出し元の変数を直接変更できます。

### swap関数の例

```c
#include <stdio.h>

// 間違い：値渡しでは入れ替わらない
void swap_wrong(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
    // ここでの変更はコピーに対するもので、呼び出し元には反映されない
}

// 正しい：ポインタ渡しで入れ替わる
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main(void) {
    int x = 10, y = 20;

    printf("swap前: x=%d, y=%d\n", x, y);

    swap_wrong(x, y);
    printf("swap_wrong後: x=%d, y=%d\n", x, y);  // 変わらない

    swap(&x, &y);
    printf("swap後: x=%d, y=%d\n", x, y);  // 入れ替わる

    return 0;
}
```

**実行結果：**
```
swap前: x=10, y=20
swap_wrong後: x=10, y=20
swap後: x=20, y=10
```

```
  値渡し vs ポインタ渡し

  【値渡し】              【ポインタ渡し】
  main の x: 10           main の x: 10
      │ コピー                │ アドレスを渡す
      ▼                      ▼
  関数の a: 10 → 20       *a → x を直接変更
  （元の x は変わらない）   （元の x が変わる）
```

### 関数から複数の値を返す

C言語の関数は `return` で1つしか値を返せません。
ポインタを使えば、複数の結果を呼び出し元に返せます。

```c
#include <stdio.h>

// 商と余りを同時に返す
void divmod(int a, int b, int *quotient, int *remainder) {
    *quotient = a / b;
    *remainder = a % b;
}

int main(void) {
    int q, r;
    divmod(17, 5, &q, &r);
    printf("17 ÷ 5 = %d 余り %d\n", q, r);

    return 0;
}
```

## 6.6 動的メモリ確保：malloc と free

これまでの配列は**静的（static）**にサイズが決まっていました。
実行時にサイズを決めたい場合は、**動的メモリ確保（dynamic memory allocation）**を使います。

### malloc と free の基本

```c
#include <stdio.h>
#include <stdlib.h>  // malloc, free に必要

int main(void) {
    int n;
    printf("要素数を入力: ");
    scanf("%d", &n);

    // n個分の int 領域を確保
    int *arr = (int *)malloc(sizeof(int) * n);

    if (arr == NULL) {
        printf("メモリ確保に失敗しました\n");
        return 1;
    }

    // 配列として使用
    for (int i = 0; i < n; i++) {
        arr[i] = (i + 1) * 10;
    }

    // 内容を表示
    for (int i = 0; i < n; i++) {
        printf("arr[%d] = %d\n", i, arr[i]);
    }

    // 使い終わったら必ず free で解放
    free(arr);
    arr = NULL;  // 解放後は NULL を代入する（安全対策）

    return 0;
}
```

```
  malloc / free の流れ

  ┌──────────────────────────────┐
  │         ヒープ領域            │
  │                              │
  │  malloc(sizeof(int)*3)       │
  │  ┌────┬────┬────┐           │
  │  │    │    │    │  ← 確保   │
  │  └────┴────┴────┘           │
  │       ↑                      │
  │    arr が指す                 │
  │                              │
  │  free(arr)                   │
  │  ┌────┬────┬────┐           │
  │  │????│????│????│  ← 解放   │
  │  └────┴────┴────┘           │
  └──────────────────────────────┘
```

### メモリ管理の重要な注意点

| 問題 | 原因 | 対策 |
|------|------|------|
| **メモリリーク（memory leak）** | `free` を忘れる | `malloc` と `free` を必ずペアにする |
| **ダングリングポインタ（dangling pointer）** | `free` 後にポインタを使う | `free` 後に `NULL` を代入する |
| **二重解放（double free）** | 同じ領域を2回 `free` する | `NULL` チェックしてから `free` する |

```c
// ダングリングポインタの例（やってはいけない）
int *p = (int *)malloc(sizeof(int));
*p = 42;
free(p);
// printf("%d\n", *p);  // ← 未定義動作！解放済みの領域にアクセス

// 安全な書き方
free(p);
p = NULL;  // NULL にしておけば、誤って使ってもすぐ気づける
```

> **よくある間違い：** `malloc` で確保した領域を `free` し忘れると、プログラムが終了するまで
> そのメモリは確保されたままになります。長時間動くプログラムでは致命的な問題になります。

## 6.7 ポインタのポインタ

ポインタ自体もメモリ上の変数なので、そのアドレスを別のポインタに格納できます。
これを**ダブルポインタ（pointer to pointer）**と呼びます。

```c
#include <stdio.h>

int main(void) {
    int a = 5;
    int *p = &a;
    int **pp = &p;  // ポインタのポインタ

    printf("a   = %d\n", a);
    printf("*p  = %d\n", *p);
    printf("**pp = %d\n", **pp);  // すべて同じ値

    return 0;
}
```

```
  ダブルポインタの関係

  pp ──→ p ──→ a
  (int **)  (int *)  (int)
  **pp      *p       5
```

ダブルポインタは、関数内でポインタ自体を変更したいときや、
文字列の配列（`char **argv`）を扱うときに使われます。

## 6.8 ポインタまとめ

### ポイントまとめ

```
  ポインタの全体像

  ┌─────────────────────────────────────┐
  │  & 演算子：変数 → アドレス取得       │
  │  * 演算子：アドレス → 値を取得       │
  │                                     │
  │  配列名 ≒ 先頭要素へのポインタ       │
  │  p[i]  ≡  *(p + i)                  │
  │                                     │
  │  ポインタ渡し：呼び出し元を変更可能   │
  │  malloc/free：動的メモリ管理          │
  │  NULL チェック：安全なポインタ利用     │
  └─────────────────────────────────────┘
```

- ポインタはメモリのアドレスを格納する変数です
- `&` でアドレスを取得し、`*` で指す先の値を取得・変更します
- 配列名は先頭要素へのポインタとして扱われます
- 関数にポインタを渡すことで、呼び出し元の変数を変更できます
- `malloc` で確保したメモリは、必ず `free` で解放しましょう
- 未初期化ポインタや `free` 済みポインタへのアクセスは未定義動作です
- ポインタを使う前は必ず `NULL` チェックを行いましょう
