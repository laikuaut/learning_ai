# 第5章：関数

## この章のゴール

- 関数の定義と呼び出しの2つの書き方をマスターすること
- 関数の引数、戻り値、ローカル変数を正しく扱えるようになること
- 再帰関数の基本を理解すること
- sourceコマンドで関数ライブラリを読み込んで活用できるようになること
- 実務で使える関数パターン（ログ、バリデーション等）を習得すること

---

## 5.1 関数の定義と呼び出し

### 2つの定義方法

```bash
#!/usr/bin/env bash

# 方法1: function キーワードを使う
function greet() {
    echo "Hello, World!"
}

# 方法2: function キーワードを省略（POSIX互換、推奨）
greet2() {
    echo "Hello, World! (2)"
}

# 関数の呼び出し（カッコは不要）
greet
greet2
```

> **推奨**: `name() { }` の書き方がPOSIX互換で広く使えます。

### 関数は呼び出し前に定義が必要

```bash
#!/usr/bin/env bash

# 【誤り】定義前の呼び出し
# say_hello  # エラー: command not found

# 定義
say_hello() {
    echo "こんにちは！"
}

# 呼び出し（定義後）
say_hello
```

### ポイントまとめ

- `name() { ... }` の形式が推奨
- 関数は定義してから呼び出す
- 呼び出し時にカッコは付けない

## 5.2 引数の受け取り

関数の引数は、スクリプトの引数と同じ `$1`, `$2`, ... で受け取ります。

```bash
#!/usr/bin/env bash

greet() {
    local name="$1"
    local greeting="${2:-こんにちは}"
    echo "$greeting、${name}さん！"
}

# 関数に引数を渡す
greet "田中"              # → こんにちは、田中さん！
greet "鈴木" "おはよう"   # → おはよう、鈴木さん！
```

### 関数内の特殊変数

```bash
#!/usr/bin/env bash

show_args() {
    echo "関数名: ${FUNCNAME[0]}"
    echo "引数の数: $#"
    echo "第1引数: $1"
    echo "全引数: $@"
}

show_args "hello" "world" "bash"
```

実行結果：
```
関数名: show_args
引数の数: 3
第1引数: hello
全引数: hello world bash
```

### 可変長引数

```bash
#!/usr/bin/env bash

# すべての引数を処理
print_all() {
    local i=1
    for arg in "$@"; do
        echo "  引数$i: $arg"
        ((i++))
    done
}

print_all "apple" "banana" "cherry"
```

### ポイントまとめ

- 関数の引数は `$1`, `$2`, ... で受け取る
- `$#` で引数の個数、`$@` で全引数
- `${FUNCNAME[0]}` で現在の関数名を取得

## 5.3 戻り値とreturn

### return（終了ステータス）

`return` は終了ステータス（0〜255の整数）を返します。

```bash
#!/usr/bin/env bash

# return は成功/失敗を返す
is_even() {
    local num="$1"
    ((num % 2 == 0))
    return $?
}

if is_even 4; then
    echo "4は偶数です"
fi

if ! is_even 3; then
    echo "3は奇数です"
fi
```

### echoで値を返す

文字列や計算結果を返すには `echo` とコマンド置換を組み合わせます。

```bash
#!/usr/bin/env bash

# echo で値を返す
get_greeting() {
    local name="$1"
    local hour
    hour=$(date +%H)

    if ((hour < 12)); then
        echo "おはようございます、${name}さん"
    elif ((hour < 18)); then
        echo "こんにちは、${name}さん"
    else
        echo "こんばんは、${name}さん"
    fi
}

# コマンド置換で受け取る
message=$(get_greeting "田中")
echo "$message"
```

### 複数の値を返す

```bash
#!/usr/bin/env bash

# 複数の値を区切り文字で返す
get_user_info() {
    local username="$1"
    local uid gid home
    IFS=: read -r _ _ uid gid _ home _ < <(grep "^${username}:" /etc/passwd)
    echo "${uid}:${gid}:${home}"
}

# 受け取り側で分割
IFS=: read -r uid gid home <<< "$(get_user_info "root")"
echo "UID=$uid, GID=$gid, HOME=$home"
```

### グローバル変数で返す

```bash
#!/usr/bin/env bash

# グローバル変数に結果を格納（echo の代替）
_result=""

divide() {
    local a="$1" b="$2"
    if ((b == 0)); then
        return 1
    fi
    _result=$(echo "scale=2; $a / $b" | bc)
    return 0
}

if divide 10 3; then
    echo "結果: $_result"  # → 結果: 3.33
else
    echo "エラー: ゼロ除算"
fi
```

### ポイントまとめ

- `return` は終了ステータス（0=成功、1〜255=失敗）を返す
- 値を返すには `echo` + コマンド置換 `$()` を使う
- 関数内で余計な `echo` を入れないこと（戻り値が汚染される）

## 5.4 ローカル変数

### localキーワード

```bash
#!/usr/bin/env bash

greeting="グローバル"

change_greeting() {
    local greeting="ローカル"
    echo "関数内: $greeting"  # → ローカル
}

echo "関数前: $greeting"  # → グローバル
change_greeting
echo "関数後: $greeting"  # → グローバル（変更されない）
```

### localを使わないと

```bash
#!/usr/bin/env bash

counter=0

bad_function() {
    counter=100  # グローバル変数を書き換えてしまう！
}

echo "前: $counter"  # → 0
bad_function
echo "後: $counter"  # → 100（意図せず変更された）
```

> **ベストプラクティス**: 関数内の変数は必ず `local` を付けましょう。グローバル変数の意図しない変更を防げます。

### ポイントまとめ

- `local` で関数内スコープの変数を宣言
- `local` なしだとグローバル変数を書き換える危険がある
- 関数内の変数は原則すべて `local` にする

## 5.5 再帰関数

```bash
#!/usr/bin/env bash

# 階乗を計算する再帰関数
factorial() {
    local n="$1"
    if ((n <= 1)); then
        echo 1
    else
        local sub
        sub=$(factorial $((n - 1)))
        echo $((n * sub))
    fi
}

echo "5! = $(factorial 5)"    # → 120
echo "10! = $(factorial 10)"  # → 3628800
```

### ディレクトリの再帰的な探索

```bash
#!/usr/bin/env bash

# ディレクトリツリーを再帰的に表示
show_tree() {
    local dir="$1"
    local indent="${2:-}"

    for entry in "$dir"/*; do
        [[ -e "$entry" ]] || continue
        local name
        name=$(basename "$entry")

        if [[ -d "$entry" ]]; then
            echo "${indent}📁 $name/"
            show_tree "$entry" "  $indent"
        else
            echo "${indent}📄 $name"
        fi
    done
}

show_tree "${1:-.}"
```

### ポイントまとめ

- Bashでも再帰関数は書ける
- 深い再帰はパフォーマンスに注意（Bashは末尾再帰最適化なし）

## 5.6 関数ライブラリ

### sourceで外部ファイルを読み込む

```bash
# lib/utils.sh（ライブラリファイル）
log_info() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [INFO] $*"
}

log_error() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [ERROR] $*" >&2
}

is_number() {
    [[ "$1" =~ ^[0-9]+$ ]]
}
```

```bash
#!/usr/bin/env bash
# main.sh（メインスクリプト）

# ライブラリを読み込む
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
source "$SCRIPT_DIR/lib/utils.sh"
# または: . "$SCRIPT_DIR/lib/utils.sh"

# ライブラリの関数を使う
log_info "スクリプトを開始します"

if is_number "42"; then
    log_info "42は数値です"
fi

log_error "これはエラーメッセージです"
```

### ポイントまとめ

- `source` または `.` で外部ファイルの関数を読み込める
- `SCRIPT_DIR` を使って相対パスを解決するのが定番
- 共通処理をライブラリ化して再利用

## 5.7 実践的な関数パターン

### ログ関数

```bash
#!/usr/bin/env bash

# カラー付きログ関数
readonly RED='\033[0;31m'
readonly GREEN='\033[0;32m'
readonly YELLOW='\033[0;33m'
readonly NC='\033[0m'  # No Color

log_info()  { echo -e "${GREEN}[INFO]${NC}  $(date '+%H:%M:%S') $*"; }
log_warn()  { echo -e "${YELLOW}[WARN]${NC}  $(date '+%H:%M:%S') $*"; }
log_error() { echo -e "${RED}[ERROR]${NC} $(date '+%H:%M:%S') $*" >&2; }

log_info "処理を開始します"
log_warn "ディスク容量が少なくなっています"
log_error "ファイルが見つかりません"
```

### バリデーション関数

```bash
#!/usr/bin/env bash

# 必須引数チェック
require_arg() {
    local name="$1"
    local value="$2"
    if [[ -z "$value" ]]; then
        echo "エラー: $name は必須です" >&2
        return 1
    fi
}

# 使用例
main() {
    require_arg "ユーザー名" "$1" || exit 1
    require_arg "メールアドレス" "$2" || exit 1
    echo "ユーザー: $1, メール: $2"
}

main "$@"
```

### usage関数

```bash
#!/usr/bin/env bash

usage() {
    cat <<EOF
使い方: $(basename "$0") [オプション] <ファイル>

オプション:
  -v, --verbose    詳細な出力
  -o, --output     出力先ファイル
  -h, --help       このヘルプを表示

例:
  $(basename "$0") -v input.txt
  $(basename "$0") -o result.txt input.txt
EOF
}

# --help や引数なしで使い方を表示
[[ "$1" = "-h" || "$1" = "--help" || $# -eq 0 ]] && { usage; exit 0; }
```

### ポイントまとめ

- ログ関数は色付きで見やすくする
- バリデーション関数で引数チェックを共通化
- usage関数はヒアドキュメントで見やすく記述

## 5.8 章全体のまとめ

| 項目 | 構文・説明 |
|------|-----------|
| 関数定義 | `name() { ... }` |
| 引数 | `$1`, `$2`, ..., `$@`, `$#` |
| 戻り値 | `return N`（ステータス）、`echo`（値） |
| ローカル変数 | `local var=value` |
| ライブラリ読み込み | `source file` / `. file` |
| 再帰 | 関数内で自分自身を呼び出す |
| ベストプラクティス | 変数は `local` 宣言、余計な `echo` を入れない |
