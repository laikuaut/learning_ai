# 第6章：待機処理とネットワーク制御

## この章のゴール

- Playwrightの自動待機メカニズムを深く理解する
- 明示的な待機（`waitFor`系）を適切に使える
- APIリクエストの監視・待機ができる
- ネットワークリクエストのモック（差し替え）ができる
- APIレスポンスをインターセプトしてテストを安定化できる

---

## 1. Playwrightの自動待機

Playwrightのほとんどの操作は、要素が操作可能になるまで**自動で待機**します。

### 自動待機が行われる操作

```
┌──────────────────────────────────────────────────────┐
│ 操作                 │ 自動待機の内容                 │
├──────────────────────────────────────────────────────┤
│ click()              │ 要素が表示・安定・有効・       │
│                      │ 遮られていないことを確認       │
│ fill()               │ 要素が表示・有効・編集可能     │
│ check()              │ 要素が表示・有効・未チェック   │
│ selectOption()       │ 要素が表示・有効               │
│ expect().toBeVisible │ 要素が表示されるまで再試行     │
│ expect().toHaveText  │ テキストが一致するまで再試行   │
└──────────────────────────────────────────────────────┘
```

```typescript
// 自動待機のおかげで、以下のコードは安定���て動く
test('ボタンクリック', async ({ page }) => {
  await page.goto('/slow-page');

  // ボタンが表示されるまで自動で待ってからクリック
  // → sleep や waitFor は不要！
  await page.getByRole('button', { name: '送信' }).click();
});
```

> **重要**: Playwrightでは `await page.waitForTimeout(3000)` のような固定時間の待機は**アンチパターン**です。テストが遅くなり、環境によって不安定になります。

## 2. 明示的な待機

自動待機だけでは不十分な場合に、明示的な待機を使います。

### 要素の状態を待つ

```typescript
// 要素が表示されるのを待つ
await page.getByText('読み込み完了').waitFor({ state: 'visible' });

// 要素が非表示になるのを待つ（ローディング終了）
await page.locator('.loading-spinner').waitFor({ state: 'hidden' });

// 要素がDOMから消えるのを待つ
await page.locator('.modal').waitFor({ state: 'detached' });

// タイムアウトを指定
await page.getByText('処理完了').waitFor({
  state: 'visible',
  timeout: 15000,  // 15秒まで待つ
});
```

### ページのロード状態を待つ

```typescript
// DOM構築完了を待つ
await page.waitForLoadState('domcontentloaded');

// 全リソースの読み込み完了を待つ
await page.waitForLoadState('load');

// ネットワーク通信が落ち着くのを待つ
await page.waitForLoadState('networkidle');
```

### 特定の条件を待つ

```typescript
// 任意の条件が満たされるまで待つ
await page.waitForFunction(() => {
  return document.querySelectorAll('.item').length >= 10;
});

// 引数を渡す
await page.waitForFunction(
  (expectedCount) => document.querySelectorAll('.item').length >= expectedCount,
  5  // 引数
);
```

## 3. ネットワークリクエストの待機

### 特定のAPIレスポンスを待つ

```typescript
test('データ読み込みの完了を待つ', async ({ page }) => {
  // APIレスポンスを待ちながらページ遷移
  const [response] = await Promise.all([
    page.waitForResponse('**/api/users'),
    page.goto('/users'),
  ]);

  // レスポンスの検証
  expect(response.status()).toBe(200);

  const data = await response.json();
  expect(data.users).toHaveLength(10);
});
```

### リクエストの待機

```typescript
test('検索リクエストが送信される', async ({ page }) => {
  await page.goto('/search');

  // リクエストを待ちながら操��
  const [request] = await Promise.all([
    page.waitForRequest('**/api/search*'),
    page.getByRole('searchbox').fill('Playwright'),
    page.getByRole('searchbox').press('Enter'),
  ]);

  // リクエストの内容を検証
  expect(request.url()).toContain('q=Playwright');
  expect(request.method()).toBe('GET');
});
```

### URL パターンのマッチング

```typescript
// 文字列（部分一致）
page.waitForResponse('**/api/users');

// 正規表現
page.waitForResponse(/\/api\/users\?page=\d+/);

// 関数（細かい条件）
page.waitForResponse(
  (response) => response.url().includes('/api/users') && response.status() === 200
);
```

## 4. ネットワークのモック（route）

`page.route()` を使うと、ネットワークリクエストを**インターセプト**してレスポンスを差し替えられます。

### 基本的なモック

```typescript
test('APIレスポンスをモックする', async ({ page }) => {
  // /api/users へのリクエストをインターセプト
  await page.route('**/api/users', async (route) => {
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify({
        users: [
          { id: 1, name: '太郎', email: 'taro@example.com' },
          { id: 2, name: '花子', email: 'hanako@example.com' },
        ],
      }),
    });
  });

  await page.goto('/users');

  // モックデータが表示されることを確認
  await expect(page.getByText('太郎')).toBeVisible();
  await expect(page.getByText('花子')).toBeVisible();
});
```

### なぜモックするのか

```
┌──────────────────────────────────────────────────────┐
│ モックのメリット                                      │
├──────────────────────────────────────────────────────┤
│ ・バックエンドが未完成でもフロントのテストができる    │
│ ・テストが高速になる（実際のAPI呼び出しを省略）      │
│ ・エラーケースを簡単に再現できる                     │
│ ・テストデータを完全にコントロールできる             │
│ ���外部APIの制限（レートリミット等）を回避できる      │
└──────────────────────────────────────────────────────┘
```

### エラーレスポンスのモック

```typescript
test('APIエラー時にエラーメッセージが表示される', async ({ page }) => {
  await page.route('**/api/users', async (route) => {
    await route.fulfill({
      status: 500,
      contentType: 'application/json',
      body: JSON.stringify({ error: 'サーバーエラー' }),
    });
  });

  await page.goto('/users');
  await expect(page.getByText('データの取得に失敗しました')).toBeVisible();
});

test('ネットワークエラーのシミュレーション', async ({ page }) => {
  await page.route('**/api/users', async (route) => {
    await route.abort('connectionrefused');  // 接続拒否をシミュレート
  });

  await page.goto('/users');
  await expect(page.getByText('ネットワークエラー')).toBeVisible();
});
```

### レスポンスの一部だけ変更する

```typescript
test('実際のレスポンスを一部変更する', async ({ page }) => {
  await page.route('**/api/users', async (route) => {
    // 実際のリクエストを送信
    const response = await route.fetch();
    const json = await response.json();

    // レスポンスを一部変更
    json.users[0].name = 'テストユーザー';

    // 変更したレスポンスを返す
    await route.fulfill({
      response,
      body: JSON.stringify(json),
    });
  });

  await page.goto('/users');
});
```

### 遅延のシミュレーション

```typescript
test('レスポンスが遅い場合のローディング表示', async ({ page }) => {
  await page.route('**/api/users', async (route) => {
    // 3秒遅延させる
    await new Promise((resolve) => setTimeout(resolve, 3000));

    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify({ users: [] }),
    });
  });

  await page.goto('/users');

  // ローディング表示を確認
  await expect(page.getByText('読み込み中...')).toBeVisible();

  // 3秒後にローディングが消えることを確認
  await expect(page.getByText('読み込み中...')).not.toBeVisible({ timeout: 5000 });
});
```

## 5. ネットワークリクエストの監視

### リクエスト・レスポンスイベントの監視

```typescript
test('APIリクエストを監視する', async ({ page }) => {
  const requests: string[] = [];

  // すべてのリクエストを記録
  page.on('request', (request) => {
    if (request.url().includes('/api/')) {
      requests.push(`${request.method()} ${request.url()}`);
    }
  });

  await page.goto('/dashboard');
  await page.getByRole('button', { name: '更新' }).click();

  // 期待するAPIコールが行われたことを確認
  expect(requests).toContain('GET https://example.com/api/stats');
});
```

## 6. 実践：非同期UIのテスト

```typescript
import { test, expect } from '@playwright/test';

test.describe('商品一覧ページ', () => {
  test.beforeEach(async ({ page }) => {
    // APIをモック
    await page.route('**/api/products', async (route) => {
      await route.fulfill({
        status: 200,
        contentType: 'application/json',
        body: JSON.stringify({
          products: [
            { id: 1, name: 'ノートPC', price: 120000, stock: 5 },
            { id: 2, name: 'マウス', price: 3000, stock: 50 },
            { id: 3, name: 'キーボード', price: 8000, stock: 0 },
          ],
        }),
      });
    });
  });

  test('商品一覧が表示される', async ({ page }) => {
    await page.goto('/products');

    await expect(page.getByRole('listitem')).toHaveCount(3);
    await expect(page.getByText('ノートPC')).toBeVisible();
    await expect(page.getByText('¥120,000')).toBeVisible();
  });

  test('在庫切れ商品は購入ボタンが無効', async ({ page }) => {
    await page.goto('/products');

    const keyboard = page.getByRole('listitem').filter({ hasText: 'キーボード' });
    await expect(keyboard.getByRole('button', { name: '購入' })).toBeDisabled();
  });

  test('API障害時にエラー画面が表示される', async ({ page }) => {
    // beforeEachのモックを上書き
    await page.route('**/api/products', async (route) => {
      await route.fulfill({ status: 503 });
    });

    await page.goto('/products');
    await expect(page.getByText('サービスが一時的に利用できません')).toBeVisible();
    await expect(page.getByRole('button', { name: '再試行' })).toBeVisible();
  });
});
```

## まとめ

| 項目 | ポイント |
|------|---------|
| 自動待機 | Playwrightは操作前に自動で要素の状態を確認する |
| 固��時間の待機 | `waitForTimeout()` はアンチパターン。使わない |
| 要素の待機 | `.waitFor({ state: 'visible' })` で明示的に待つ |
| APIの待機 | `page.waitForResponse()` でレスポンスを待つ |
| モック | `page.route()` でAPIレスポンスを差し替え |
| エラーシミュレーション | `route.fulfill({ status: 500 })` / `route.abort()` |
| レスポンス変更 | `route.fetch()` で実際のレスポンスを取得→一部変更 |
| 遅延シミュレーション | `setTimeout` でレスポンスを遅延させる |
