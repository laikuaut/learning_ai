# 実践課題10：リファクタリング ─ 散らかったコミット履歴の整理 ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第2章（基本操作）、第3章（ブランチとマージ）、第8章（高度なGit操作）
> **課題の種類**: リファクタリング
> **学習目標**: 散らかったコミット履歴を rebase やコミットの整理で読みやすく改善する力を養う

---

## 課題の説明

以下のスクリプトは、「散らかったコミット履歴」を持つリポジトリを作成します。実務では、開発中に試行錯誤したコミットをプルリクエスト前に整理する作業が頻繁に発生します。

### 散らかった履歴の問題点

```
$ git log --oneline
h8i9j0k WIP
g7h8i9j fix typo
f6g7h8i fix typo again
e5f6g7h aaaa（テスト用の意味不明なメッセージ）
d4e5f6g ログイン機能を追加
c3d4e5f 忘れてたファイル追加
b2c3d4e ログイン画面のCSSを追加
a1b2c3d 初回コミット
```

### 整理後の理想的な履歴

```
$ git log --oneline
d4e5f6g feat: ログイン機能を実装
c3d4e5f feat: ログイン画面のUIを作成
b2c3d4e 初回コミット
```

---

## セットアップスクリプト

以下を実行して、散らかったリポジトリを作成してください。

```bash
#!/bin/bash
# 散らかったコミット履歴を持つリポジトリのセットアップ

REPO_DIR="/tmp/messy-history"
rm -rf "$REPO_DIR"
mkdir "$REPO_DIR"
cd "$REPO_DIR"
git init

# コミット1: 初回
echo "# ログインアプリ" > README.md
git add README.md
git commit -m "初回コミット"

# コミット2: CSSを追加
cat << 'EOF' > login.css
.login-form {
  max-width: 400px;
  margin: 0 auto;
  padding: 20px;
}
.login-btn {
  background: blue;
  color: white;
}
EOF
git add login.css
git commit -m "ログイン画面のCSSを追加"

# コミット3: ファイル追加忘れ
cat << 'EOF' > login.html
<form class="login-form">
  <input type="text" name="username">
  <input type="password" name="password">
  <button class="login-btn">ログイン</button>
</form>
EOF
git add login.html
git commit -m "忘れてたファイル追加"

# コミット4: ログイン機能
cat << 'EOF' > auth.js
function login(username, password) {
  if (username === "admin" && password == "pass") {
    return true;
  }
  return false;
}
EOF
git add auth.js
git commit -m "ログイン機能を追加"

# コミット5: 意味不明なコミット
echo "// TODO: バリデーション追加" >> auth.js
git add auth.js
git commit -m "aaaa"

# コミット6: タイポ修正
sed -i 's/password == "pass"/password === "pass"/' auth.js
git add auth.js
git commit -m "fix typo"

# コミット7: またタイポ修正
echo "" >> auth.js
echo "module.exports = { login };" >> auth.js
git add auth.js
git commit -m "fix typo again"

# コミット8: WIP（Work In Progress）
echo "// WIP: エラーハンドリング" >> auth.js
git add auth.js
git commit -m "WIP"

echo ""
echo "========================================="
echo "散らかったリポジトリの作成完了"
echo "cd $REPO_DIR で移動してください"
echo "========================================="
echo ""
echo "$ git log --oneline"
cd "$REPO_DIR" && git log --oneline
```

## 課題の要件

以下の手法を使って、コミット履歴を整理してください。

1. **方法A**: `git rebase` を使って履歴を整理する
2. **方法B**: 新しいブランチで `git reset --soft` を使って整理する

どちらの方法でも、最終的に以下のような整理された履歴にしてください。

### 整理のルール

- 「WIP」「aaaa」「fix typo」のような無意味なコミットをまとめる
- 意味のある単位（UI作成 / 機能実装）でコミットを分ける
- Conventional Commits 形式のメッセージを付ける
- 最終的なコードの内容は変えない

## ステップガイド

<details>
<summary>方法A：git rebase でコミットをまとめる（squash）</summary>

```bash
cd /tmp/messy-history

# 現在の履歴を確認
git log --oneline

# ===== rebase の準備 =====
# 安全のためにバックアップブランチを作成
git branch backup

# 初回コミット以降の7つのコミットを対象に rebase
# （エディタが開くので、pick / squash / fixup を設定する）
git rebase -i HEAD~7
```

エディタに以下が表示されます：

```
pick b2c3d4e ログイン画面のCSSを追加
pick c3d4e5f 忘れてたファイル追加
pick d4e5f6g ログイン機能を追加
pick e5f6g7h aaaa
pick f6g7h8i fix typo
pick g7h8i9j fix typo again
pick h8i9j0k WIP
```

これを以下のように変更します：

```
pick b2c3d4e ログイン画面のCSSを追加
squash c3d4e5f 忘れてたファイル追加
pick d4e5f6g ログイン機能を追加
squash e5f6g7h aaaa
squash f6g7h8i fix typo
squash g7h8i9j fix typo again
squash h8i9j0k WIP
```

- `squash` は前のコミットと統合します
- コミットメッセージの編集画面で、新しいメッセージに書き換えます

</details>

<details>
<summary>方法B：reset --soft で一からコミットし直す（初心者向け）</summary>

rebase が難しい場合は、この方法が簡単です。

```bash
cd /tmp/messy-history

# バックアップ
git branch backup

# 新しいブランチを作成
git switch -c clean-history

# 初回コミットまで soft reset（ファイルはそのまま、コミットだけ戻す）
git reset --soft HEAD~7

# 今、すべてのファイルがステージングされた状態
git status
# Changes to be committed:
#   login.css
#   login.html
#   auth.js

# UI 関連だけまずコミット
git reset HEAD auth.js  # auth.js をステージから外す
git commit -m "feat(ui): ログイン画面のHTMLとCSSを作成"

# 機能をコミット
git add auth.js
git commit -m "feat(auth): ログイン認証機能を実装"

# 確認
git log --oneline
```

</details>

## 解答例

<details>
<summary>解答例（方法B：reset --soft を使った整理）</summary>

```bash
cd /tmp/messy-history

# ===== 現在の散らかった状態を確認 =====
git log --oneline
# h8i9j0k WIP
# g7h8i9j fix typo again
# f6g7h8i fix typo
# e5f6g7h aaaa
# d4e5f6g ログイン機能を追加
# c3d4e5f 忘れてたファイル追加
# b2c3d4e ログイン画面のCSSを追加
# a1b2c3d 初回コミット

# ===== バックアップを取る（いつでも元に戻せるように）=====
git branch backup-before-cleanup

# ===== 方法B: reset --soft で整理 =====
git switch -c clean-history

# 初回コミット以降をすべて soft reset
git reset --soft HEAD~7

# 状態確認（全ファイルがステージされている）
git status

# ===== UI 関連のコミット =====
# auth.js を一旦ステージから外す
git restore --staged auth.js

# login.html と login.css だけコミット
git commit -m "feat(ui): ログイン画面のHTMLとCSSを作成

- login.html: ユーザー名・パスワードの入力フォーム
- login.css: フォームのスタイリング"

# ===== 機能のコミット =====
git add auth.js
git commit -m "feat(auth): ログイン認証機能を実装

- login() 関数を追加
- バリデーション用 TODO コメント
- module.exports でエクスポート"

# ===== 整理後の確認 =====
git log --oneline
# b2b2b2b (HEAD -> clean-history) feat(auth): ログイン認証機能を実装
# a1a1a1a feat(ui): ログイン画面のHTMLとCSSを作成
# a1b2c3d 初回コミット

# ファイルの内容が変わっていないことを確認
cat auth.js
cat login.html
cat login.css

# ===== main を更新する場合 =====
git switch main
git reset --hard clean-history
# ※ 注意: リモートに push 済みの場合は force push が必要になるため、
#   チームメンバーに事前に連絡してください
```

</details>

<details>
<summary>解答例（改良版 ─ rebase -i を使った整理）</summary>

対話的リベース（interactive rebase）は、より柔軟にコミットを操作できます。

```bash
cd /tmp/messy-history

# バックアップ
git branch backup-before-cleanup

# ===== 対話的リベースの実行 =====
# エディタが開くので編集する
# 環境変数でエディタを指定できる
export GIT_EDITOR="vi"  # お好みのエディタに変更可能

git rebase -i HEAD~7

# ===== エディタで以下のように編集 =====
# 元の内容:
#   pick b2c3d4e ログイン画面のCSSを追加
#   pick c3d4e5f 忘れてたファイル追加
#   pick d4e5f6g ログイン機能を追加
#   pick e5f6g7h aaaa
#   pick f6g7h8i fix typo
#   pick g7h8i9j fix typo again
#   pick h8i9j0k WIP
#
# 変更後:
#   pick b2c3d4e ログイン画面のCSSを追加
#   fixup c3d4e5f 忘れてたファイル追加
#   pick d4e5f6g ログイン機能を追加
#   fixup e5f6g7h aaaa
#   fixup f6g7h8i fix typo
#   fixup g7h8i9j fix typo again
#   fixup h8i9j0k WIP

# ===== rebase の操作の違い =====
# pick   - そのまま使う
# squash - 前のコミットと統合（メッセージ編集あり）
# fixup  - 前のコミットと統合（メッセージは前のものを使用）
# reword - メッセージだけ変更
# edit   - コミットを修正するために一時停止
# drop   - コミットを削除

# fixup を使うと、メッセージ編集の手間が省ける

# ===== rebase 完了後、メッセージを修正 =====
# 1つ目のコミットメッセージを修正
git rebase -i HEAD~2
# 各コミットを reword に変更してメッセージを書き換える

# 最終確認
git log --oneline

# ===== もし rebase に失敗した場合 =====
# git rebase --abort  # rebase を中断して元に戻す
# git reset --hard backup-before-cleanup  # バックアップから復元
```

**方法A（rebase）と方法B（reset --soft）の比較:**

| 観点 | rebase -i | reset --soft |
|---|---|---|
| 難易度 | やや高い | 低い |
| 柔軟性 | コミット単位で細かく操作可能 | 全部やり直し |
| メッセージ編集 | squash/reword で1つずつ | コミット時に自由に設定 |
| コミット順序変更 | 可能（行を入れ替える） | 手動でステージ順序を管理 |
| 実務での使用頻度 | 高い（PR前の整理で頻繁に使う） | 時々（大幅に作り直すとき） |

</details>
