# 第5章：ネットワークセキュリティ

## この章のゴール

- ファイアウォールの種類と適切な選択基準を理解すること
- DMZ（非武装地帯）の設計思想とネットワーク構成を説明できること
- VPNの種類（IPsec, SSL-VPN）と利用場面を把握すること
- 無線LANのセキュリティ規格（WPA2, WPA3）と脅威を理解すること
- IDS/IPS、WAF、プロキシサーバの役割と配置を説明できること

## 5.1 ファイアウォール

### 5.1.1 ファイアウォールの役割

ファイアウォール（Firewall, FW）は、ネットワーク境界に設置され、事前に定義されたルール（ポリシー）に基づいて通信の許可/拒否を判断するセキュリティ機器です。

```
【ファイアウォールの基本動作】

  外部（インターネット）         内部（社内LAN）
         │                        │
         │    ┌──────────┐       │
         ├──→│ FW       │──→   │  許可（ルールに合致）
         │    │ ルール:   │       │
         ├─×→│ 評価      │       │  拒否（ルールに不合致）
         │    └──────────┘       │
         │                        │

  デフォルトポリシー:
  ・ホワイトリスト方式: 明示的に許可した通信のみ通す（推奨）
  ・ブラックリスト方式: 明示的に拒否した通信のみ遮断する
```

### 5.1.2 ファイアウォールの種類

| 種類 | 検査対象 | OSI層 | 特徴 |
|------|---------|-------|------|
| **パケットフィルタリング** | IPアドレス、ポート番号 | L3/L4 | 高速・シンプルだが通信内容は見ない |
| **ステートフルインスペクション（SPI）** | ヘッダ + セッション状態 | L3/L4 | 戻り通信を自動許可。現在の主流 |
| **アプリケーションゲートウェイ** | 通信内容（ペイロード含む） | L7 | 高い防御力だが処理が遅い |
| **次世代FW（NGFW）** | アプリケーション識別 | L3～L7 | IPS・マルウェア検知等を統合 |

### 5.1.3 ステートフルインスペクションの動作

```
【ステートフルインスペクションの仕組み】

  ① 内部PC → 外部Webサーバ: HTTP接続開始（SYN）
     FWがセッションテーブルに登録
     ┌─────────────────────────────────┐
     │ 192.168.1.10:50001 → 93.0.0.1:80 │
     │ 状態: SYN_SENT                    │
     └─────────────────────────────────┘

  ② 外部Webサーバ → 内部PC: 応答（SYN-ACK）
     セッションテーブルと照合 → 正当な戻り → 自動許可

  ③ セッション終了 → テーブルからエントリ削除

  ポイント: 戻り通信の個別ルール設定が不要
```

## 5.2 DMZ（DeMilitarized Zone）

### 5.2.1 DMZの設計思想

DMZ（非武装地帯）は、外部と内部の中間に位置するネットワークセグメントです。外部に公開するサーバを配置し、万が一そのサーバが侵害されても内部ネットワークへの直接的な影響を最小限に抑えます。

```
【DMZ構成とトラフィックポリシー】

  インターネット
       │
  ┌────┴────┐
  │  FW      │
  │  ├──→ DMZ                      通信ポリシー:
  │  │    ├── Webサーバ              外部→DMZ:   制限付き許可（80,443等）
  │  │    ├── メールサーバ           外部→内部:  拒否
  │  │    └── DNS（外部公開用）      内部→DMZ:   許可
  │  │                              内部→外部:  許可（プロキシ経由推奨）
  │  └──→ 内部LAN                   DMZ→内部:   原則拒否
  │       ├── 社内PC                DMZ→外部:   制限付き許可
  │       ├── ファイルサーバ
  │       └── 社内DB
  └──────────┘
```

### 5.2.2 DMZ構成パターン

| 構成 | 特徴 | コスト |
|------|------|--------|
| **シングルFW（3レグ構成）** | 1台のFWに3つのインターフェース（外部・DMZ・内部） | 低 |
| **デュアルFW** | 外部FWと内部FWの2台構成。DMZは2つのFWの間 | 高（より安全） |

## 5.3 VPN（Virtual Private Network）

### 5.3.1 VPNの概要

VPNは公衆回線（インターネット）上に暗号化された仮想的な専用線（トンネル）を構築する技術です。

```
【VPNの3つの機能】

  1. 暗号化（Encryption）: 通信内容を暗号化し盗聴を防止
  2. 認証（Authentication）: 通信相手の正当性を確認
  3. 完全性（Integrity）: データの改ざんを検知

  ┌─────────────────────────────────────┐
  │         インターネット（公衆回線）     │
  │                                      │
  │  [拠点A]══ 暗号化トンネル ══[拠点B]  │
  │         盗聴されても解読不能          │
  └─────────────────────────────────────┘
```

### 5.3.2 IPsec VPN

ネットワーク層（L3）で動作する暗号化プロトコルスイートです。

| コンポーネント | 役割 |
|--------------|------|
| **ESP** | 暗号化 + 認証（最も一般的に使用） |
| **AH** | 認証のみ（暗号化なし） |
| **IKE** | 暗号鍵の交換と SA の確立 |
| **SA** | 暗号化パラメータの合意情報 |

```
【IPsecのモード】

  トランスポートモード（ホスト間通信）:
  [元IPヘッダ][ESPヘッダ][ペイロード（暗号化）][ESPトレーラ]

  トンネルモード（拠点間VPN、一般的）:
  [新IPヘッダ][ESPヘッダ][元IPヘッダ+ペイロード（暗号化）][ESPトレーラ]
```

### 5.3.3 SSL-VPN

SSL/TLS（ポート443）を使用するVPNで、Webブラウザからのリモートアクセスが容易です。

| SSL-VPN方式 | 説明 | 対象 |
|------------|------|------|
| リバースプロキシ方式 | Webアプリケーションへのアクセスのみ | Webアプリ |
| ポートフォワーディング方式 | 特定ポートの通信を転送 | 特定アプリ |
| L2フォワーディング方式 | 全トラフィックをVPN経由 | 全通信 |

### 5.3.4 IPsecとSSL-VPNの比較

| 項目 | IPsec VPN | SSL-VPN |
|------|----------|---------|
| 動作層 | L3 | L4～L7 |
| 用途 | 拠点間VPN | リモートアクセス |
| 導入の容易さ | 専用ソフト/機器が必要 | ブラウザで利用可能 |
| FW透過性 | NATやFWの設定変更が必要な場合あり | 443/TCPのため通過しやすい |

## 5.4 無線LANセキュリティ

### 5.4.1 無線LANの脅威

```
【無線LANの脅威】

  ┌─────────────────────────────────┐
  │  無線LAN電波の到達範囲            │
  │                                  │
  │  [正規AP]  ←→  [社内PC]         │
  │      ↑                           │
  │   [盗聴者]  ← 電波を傍受         │
  │                                  │
  │   [不正AP]  ← 偽のAPを設置       │
  │      ↓          （Evil Twin攻撃） │
  │   [被害者]  ← 偽APに接続          │
  └─────────────────────────────────┘
```

| 脅威 | 説明 |
|------|------|
| **盗聴** | 電波を傍受して通信内容を取得 |
| **不正アクセス** | パスワードを推測して無線LANに接続 |
| **Evil Twin攻撃** | 正規APと同じSSIDの偽APを設置し、接続してきた端末の通信を盗聴 |
| **WPSの脆弱性** | WPS PINの総当たり攻撃でパスワードを取得 |

### 5.4.2 無線LANセキュリティ規格

| 規格 | 暗号方式 | 認証 | 安全性 |
|------|---------|------|--------|
| **WEP** | RC4（64/128bit） | 共有鍵 | 危険（容易に解読可能） |
| **WPA** | TKIP（RC4ベース） | PSK / 802.1X | 脆弱性あり |
| **WPA2** | CCMP（AES） | PSK / 802.1X | 安全（現在の主流） |
| **WPA3** | SAE + CCMP/GCMP | SAE / 802.1X | より安全（推奨） |

> **重要**: WEPは数分で解読可能であり、絶対に使用してはいけません。WPA（TKIP）にも脆弱性が見つかっています。WPA2-AES以上を使用してください。

### 5.4.3 WPA2/WPA3の認証方式

| 方式 | 説明 | 用途 |
|------|------|------|
| **PSK（Pre-Shared Key）** | 事前共有鍵（パスワード）で認証。パーソナルモードとも呼ぶ | 家庭・小規模 |
| **802.1X / EAP** | RADIUSサーバによるユーザ個別認証。エンタープライズモードとも呼ぶ | 企業 |

```
【802.1X認証の流れ】

  [無線端末] ←→ [AP（認証者）] ←→ [RADIUSサーバ]
  (サプリカント)                    (認証サーバ)

  ① 端末がAPに接続要求
  ② APが端末の認証情報をRADIUSサーバに転送
  ③ RADIUSサーバが認証（ユーザ名/パスワード、証明書等）
  ④ 認証成功 → APがポートを開放 → 通信開始
```

## 5.5 IDS / IPS

### 5.5.1 IDS（Intrusion Detection System / 侵入検知システム）

ネットワーク上の通信を監視し、不正なアクセスや攻撃を **検知してアラートを発報** します。

### 5.5.2 IPS（Intrusion Prevention System / 侵入防止システム）

IDSの機能に加え、不正な通信を **自動的に遮断** します。

```
【IDS vs IPS の配置と動作】

  IDS: ミラーポート（スイッチのコピー）で監視
  攻撃者 ──→ [スイッチ] ──→ サーバ（攻撃は到達）
                  │
              [IDS] → アラート通知

  IPS: インラインに設置（通信経路上）
  攻撃者 ──→ [IPS] ──× サーバ（攻撃は遮断）
                │
            遮断 + アラート通知
```

### 5.5.3 検知方式

| 方式 | 仕組み | 長所 | 短所 |
|------|--------|------|------|
| **シグネチャ型** | 既知の攻撃パターンとの照合 | 誤検知が少ない | 未知の攻撃は検知不可 |
| **アノマリ型** | 正常パターンからの逸脱を検知 | 未知の攻撃も検知可能 | 誤検知が多い |

## 5.6 WAF・プロキシ・UTM

### 5.6.1 WAF（Web Application Firewall）

Webアプリケーション層（L7）への攻撃（SQLインジェクション、XSS等）を検知・遮断する専用ファイアウォールです。

### 5.6.2 プロキシサーバ

| 種類 | 方向 | 役割 |
|------|------|------|
| **フォワードプロキシ** | 内部 → 外部 | クライアントの代理でインターネットにアクセス。キャッシュ、URLフィルタリング、ログ記録 |
| **リバースプロキシ** | 外部 → 内部 | 外部からのリクエストを内部サーバに中継。負荷分散、SSL終端、キャッシュ |

### 5.6.3 UTM（Unified Threat Management）

ファイアウォール、IDS/IPS、アンチウイルス、URLフィルタリング、VPN等のセキュリティ機能を1台に統合した機器です。中小企業でコストを抑えてセキュリティを確保する場合に有効です。

## 5.7 ネットワークセキュリティの全体設計

### 5.7.1 多層防御の構成例

```
  インターネット
       │
  [UTM/NGFW]  ← パケットフィルタリング + IPS + アンチウイルス
       │
  ┌────┴────┐
  │  DMZ     │
  │ [WAF]    │── Webサーバ（公開用）
  │ [SMTP]   │── メールサーバ
  └────┬────┘
       │
  [内部FW]    ← セグメント間制御
       │
  ┌────┴────────────────┐
  │  内部LAN              │
  │ [フォワードプロキシ]   │── 社内PC（Web閲覧制御）
  │ [NIDS]               │── 不正通信の監視
  │ [RADIUS]             │── 無線LAN認証
  └──────────────────────┘
```

## 5.8 まとめ

### ポイントまとめ

1. **ファイアウォール** はネットワーク境界の基本的な防御装置です。ステートフルインスペクションが現在の主流です
2. **DMZ** は外部公開サーバを隔離し、内部LANへの侵害拡大を防止します
3. **VPN** は IPsec（拠点間）と SSL-VPN（リモートアクセス）が代表的です
4. **無線LAN** は WPA2-AES 以上を使用し、企業では 802.1X認証を導入してください
5. **IDS** は検知とアラート、**IPS** は検知に加え自動遮断を行います
6. **多層防御** により、FW + IDS/IPS + WAF + プロキシ等を組み合わせることが重要です

### 試験対策キーワード

- ファイアウォール（パケットフィルタリング、SPI、NGFW）
- DMZ、シングルFW、デュアルFW
- VPN（IPsec、ESP、AH、IKE、SA、SSL-VPN）
- WPA2、WPA3、802.1X、RADIUS、PSK
- IDS / IPS、シグネチャ型、アノマリ型
- WAF、フォワードプロキシ、リバースプロキシ、UTM
