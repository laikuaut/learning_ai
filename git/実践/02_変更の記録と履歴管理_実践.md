# 実践課題02：変更の記録と履歴管理 ★1

> **難易度**: ★☆☆☆☆（入門）
> **前提知識**: 第1章（Gitの基本概念とセットアップ）、第2章（基本操作）
> **課題の種類**: ミニプロジェクト
> **学習目標**: add・commit・diff・logの基本操作を繰り返し実行して、ステージング（staging）と履歴確認の流れを定着させる

---

## 完成イメージ

```
$ git log --oneline
f4e5d6c (HEAD -> main) 第3版：メニューに飲み物を追加
b2c3d4e 第2版：営業時間の情報を追加
a1b2c3d 初版：カフェメニューの基本情報を作成
```

```
$ git diff HEAD~1
  コーヒー    450円
  紅茶        400円
+ 抹茶ラテ    550円
+ ホットココア 500円
```

---

## 課題の要件

1. 新しいリポジトリを作成する
2. `menu.txt`（カフェのメニュー）を作成して初回コミットする
3. `menu.txt` を編集（営業時間を追記）して2回目のコミットをする
4. さらに `menu.txt` を編集（メニュー項目を追加）して3回目のコミットをする
5. 各ステップで `git diff` を使って変更内容を確認する
6. `git log` で履歴を確認する
7. `git show` で特定のコミットの内容を見る

---

## ステップガイド

<details>
<summary>ステップ1：リポジトリを作成して初回コミット</summary>

```bash
mkdir cafe-menu
cd cafe-menu
git init

# メニューファイルを作成
cat << 'EOF' > menu.txt
===== カフェメニュー =====

【ホットドリンク】
コーヒー    450円
紅茶        400円
カフェラテ  500円

【コールドドリンク】
アイスコーヒー  470円
アイスティー    420円
EOF

git add menu.txt
git commit -m "初版：カフェメニューの基本情報を作成"
```

</details>

<details>
<summary>ステップ2：変更して差分を確認し、2回目のコミット</summary>

```bash
# menu.txt の末尾に営業時間を追記
cat << 'EOF' >> menu.txt

===== 営業情報 =====
営業時間：9:00 - 18:00
定休日：毎週水曜日
EOF

# 変更差分を確認（ステージング前）
git diff

# ステージング
git add menu.txt

# ステージング後の差分を確認（--staged オプション）
git diff --staged

# コミット
git commit -m "第2版：営業時間の情報を追加"
```

</details>

<details>
<summary>ステップ3：さらに変更して3回目のコミット</summary>

ホットドリンクに2つ追加します。`sed` コマンドまたはエディタで編集してください。

```bash
# エディタで menu.txt を開いて、ホットドリンクのセクションに追記
# カフェラテの行の後に以下を追加：
#   抹茶ラテ    550円
#   ホットココア 500円

# 差分確認
git diff

# コミット
git add menu.txt
git commit -m "第3版：メニューに飲み物を追加"
```

</details>

<details>
<summary>ステップ4：履歴を確認する</summary>

```bash
# コミット履歴を確認
git log --oneline

# 詳細な履歴
git log

# 1つ前のコミットとの差分
git diff HEAD~1

# 特定のコミットの詳細を確認（ハッシュは git log で確認）
git show HEAD
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け：逐一確認しながら進める）</summary>

```bash
# ===== 準備 =====
mkdir cafe-menu
cd cafe-menu
git init

# ===== 第1回コミット =====
cat << 'EOF' > menu.txt
===== カフェメニュー =====

【ホットドリンク】
コーヒー    450円
紅茶        400円
カフェラテ  500円

【コールドドリンク】
アイスコーヒー  470円
アイスティー    420円
EOF

# 状態を確認（menu.txt が Untracked になっている）
git status

git add menu.txt
git status  # Changes to be committed に変わっていることを確認

git commit -m "初版：カフェメニューの基本情報を作成"
git log --oneline
# 出力例: a1b2c3d (HEAD -> main) 初版：カフェメニューの基本情報を作成

# ===== 第2回コミット =====
cat << 'EOF' >> menu.txt

===== 営業情報 =====
営業時間：9:00 - 18:00
定休日：毎週水曜日
EOF

# 変更差分を確認（追記された行が + で表示される）
git diff
# 出力例:
# +===== 営業情報 =====
# +営業時間：9:00 - 18:00
# +定休日：毎週水曜日

git add menu.txt
git diff --staged  # ステージ済みの差分を確認
git commit -m "第2版：営業時間の情報を追加"

git log --oneline
# 出力例:
# b2c3d4e (HEAD -> main) 第2版：営業時間の情報を追加
# a1b2c3d 初版：カフェメニューの基本情報を作成

# ===== 第3回コミット =====
# menu.txt を編集して、カフェラテの行の後に2行追加する
# sed を使う方法（macOS/Linux）：
sed -i '/カフェラテ/a\抹茶ラテ    550円\nホットココア 500円' menu.txt

# 差分確認
git diff
# 出力例:
# +抹茶ラテ    550円
# +ホットココア 500円

git add menu.txt
git commit -m "第3版：メニューに飲み物を追加"

# ===== 履歴の確認 =====
# 全コミットを1行ずつ表示
git log --oneline
# 出力例:
# f4e5d6c (HEAD -> main) 第3版：メニューに飲み物を追加
# b2c3d4e 第2版：営業時間の情報を追加
# a1b2c3d 初版：カフェメニューの基本情報を作成

# 直前のコミットとの差分を見る
git diff HEAD~1

# 最新コミットの内容を詳しく見る
git show HEAD

# 最初のコミットの内容を見る
git show HEAD~2
```

</details>

<details>
<summary>解答例（改良版 ─ 実務で役立つログ確認テクニック）</summary>

基本的なコミット手順は同じですが、より高度な履歴の確認方法を紹介します。

```bash
# （リポジトリ作成〜3回のコミットは初心者向けと同じ手順）

# ===== 実務で便利な git log のオプション =====

# グラフ表示（ブランチがあるときに威力を発揮）
git log --oneline --graph

# 変更されたファイル名も表示
git log --oneline --name-only

# 各コミットの変更行数を表示
git log --oneline --stat

# 特定のファイルに関するコミットだけ表示
git log --oneline -- menu.txt

# コミットメッセージで検索
git log --oneline --grep="メニュー"

# 日付で絞り込み
git log --oneline --since="1 hour ago"

# ===== git diff の活用 =====

# 最初のコミットから最新までの全差分
git diff HEAD~2..HEAD

# 特定の2つのコミット間の差分
git diff HEAD~2..HEAD~1

# 変更されたファイル名だけ表示
git diff --name-only HEAD~2..HEAD

# 変更の統計情報（追加・削除行数）
git diff --stat HEAD~2..HEAD
```

**初心者向けとの違い:**
- `git log` の豊富なオプションで、目的に応じた履歴検索ができる
- `git diff` でコミット間の比較範囲を柔軟に指定できる
- `--stat` や `--name-only` で変更の概要をすばやく把握する技術

</details>
