# 第7章: 型ガードと型の絞り込み

## この章のゴール

- 型ガード（Type Guard）の概念と、TypeScriptが型を自動的に絞り込む仕組みを理解する
- `typeof`, `instanceof`, `in`, 等値チェックなどの組み込み型ガードを使いこなせるようになる
- カスタム型ガード関数（`is` 述語）を自作できるようになる
- `asserts` 型述語を理解し、アサーション関数を実装できるようになる
- 実務で遭遇する複雑な型の絞り込みパターンを解決できるようになる

---

## 7.1 型の絞り込みとは

### 制御フロー解析（Control Flow Analysis）

TypeScriptのコンパイラは、コードの制御フロー（`if`, `switch`, `return` など）を解析して、変数の型を自動的に絞り込みます。これを**制御フロー解析**と呼びます。

```typescript
function example(value: string | number | null) {
  // この時点では value は string | number | null

  if (value === null) {
    // ここでは value は null
    return;
  }

  // ここでは value は string | number（nullは除外された）

  if (typeof value === "string") {
    // ここでは value は string
    console.log(value.toUpperCase());
  } else {
    // ここでは value は number
    console.log(value.toFixed(2));
  }
}
```

この仕組みの素晴らしい点は、**開発者がコードを自然に書くだけで、TypeScriptが自動的に型を正しく認識してくれる**ことです。特別な構文を覚える必要はなく、普通のJavaScriptの条件分岐がそのまま型ガードとして機能します。

```
┌──────────────────────────────────────────────────┐
│         型の絞り込みフロー                         │
│                                                    │
│  value: string | number | null                     │
│         │                                          │
│         ├─ if (value === null) → null               │
│         │    return; ← ここで処理を抜ける          │
│         │                                          │
│         ├─ value: string | number ← nullが除外     │
│         │                                          │
│         ├─ if (typeof value === "string")           │
│         │    → string                               │
│         │                                          │
│         └─ else                                     │
│              → number                               │
└──────────────────────────────────────────────────┘
```

### なぜ型の絞り込みが重要なのか

JavaScriptでは、ユニオン的な値に対して条件分岐を書くのはよくあることです。しかし、分岐を忘れたり間違えたりすると実行時エラーになります。TypeScriptの型の絞り込みは、この問題をコンパイル時に検出します。

```typescript
// ❌ JavaScriptの場合: 型チェックを忘れてもエラーにならない
function process(value) {
  return value.toUpperCase(); // numberが来たらランタイムエラー！
}

// ✅ TypeScriptの場合: 型チェックしないとコンパイルエラー
function processTS(value: string | number): string {
  // return value.toUpperCase(); // エラー！numberにtoUpperCaseはない

  if (typeof value === "string") {
    return value.toUpperCase();
  }
  return value.toString();
}
```

## 7.2 組み込みの型ガード

### typeof ガード

`typeof` 演算子は、プリミティブ型の判定に使います。

```typescript
function format(value: string | number | boolean): string {
  switch (typeof value) {
    case "string":
      return `"${value}"`; // value は string
    case "number":
      return value.toFixed(2); // value は number
    case "boolean":
      return value ? "真" : "偽"; // value は boolean
  }
}

// typeof が返す値の一覧:
// "string", "number", "bigint", "boolean",
// "symbol", "undefined", "object", "function"
```

**typeof の注意点:**

```typescript
// typeof null は "object" を返す（JavaScriptの歴史的バグ）
function handleValue(value: string | null): void {
  if (typeof value === "object") {
    // ここでは value は null（object型ではない！）
    // value.toUpperCase(); // 実行時エラー！
  }

  // 正しいやり方
  if (value !== null && typeof value === "string") {
    value.toUpperCase(); // OK
  }
}
```

### instanceof ガード

`instanceof` はクラスのインスタンスかどうかを判定します。

```typescript
class HttpError extends Error {
  constructor(
    public statusCode: number,
    message: string
  ) {
    super(message);
  }
}

class ValidationError extends Error {
  constructor(
    public fields: string[],
    message: string
  ) {
    super(message);
  }
}

function handleError(error: Error): string {
  if (error instanceof HttpError) {
    // error は HttpError
    return `HTTP ${error.statusCode}: ${error.message}`;
  }

  if (error instanceof ValidationError) {
    // error は ValidationError
    return `バリデーションエラー (${error.fields.join(", ")}): ${error.message}`;
  }

  // error は Error（基底型）
  return `不明なエラー: ${error.message}`;
}
```

### in ガード

`in` 演算子は、オブジェクトに特定のプロパティが存在するかを判定します。

```typescript
interface Dog {
  kind: "dog";
  bark(): string;
  fetch(item: string): void;
}

interface Cat {
  kind: "cat";
  meow(): string;
  purr(): void;
}

interface Fish {
  kind: "fish";
  swim(): void;
}

type Animal = Dog | Cat | Fish;

function interactWith(animal: Animal): string {
  // "bark" プロパティがあるか？
  if ("bark" in animal) {
    // animal は Dog
    return animal.bark();
  }

  // "meow" プロパティがあるか？
  if ("meow" in animal) {
    // animal は Cat
    return animal.meow();
  }

  // ここでは animal は Fish
  animal.swim();
  return "魚が泳いでいます";
}
```

### 等値チェック（Equality Narrowing）

`===`, `!==`, `==`, `!=` による比較も型を絞り込みます。

```typescript
function example(x: string | number, y: string | boolean) {
  if (x === y) {
    // x と y が等しいなら、両方とも string でなければならない
    // （他の型の組み合わせでは === が true にならないため）
    x.toUpperCase(); // OK: x は string
    y.toUpperCase(); // OK: y は string
  }
}

// null チェック
function processName(name: string | null | undefined): string {
  if (name == null) {
    // == null は null と undefined の両方にマッチ
    return "名前なし";
  }
  // name は string
  return name.trim();
}
```

### 真偽値チェック（Truthiness Narrowing）

JavaScriptの真偽値変換を利用した絞り込みです。

```typescript
function printLength(str: string | null | undefined): void {
  if (str) {
    // str は string（null, undefined, "" は除外）
    console.log(str.length);
  } else {
    // str は string | null | undefined
    // 注意: "" (空文字列) もここに来る！
    console.log("値がありません");
  }
}

// 配列の絞り込み
function processItems(items: string[] | undefined): void {
  if (items && items.length > 0) {
    // items は string[] かつ要素が1つ以上
    items.forEach((item) => console.log(item));
  }
}
```

> **注意:** 真偽値チェックは `0`, `""`, `NaN` も falsy として扱います。これらの値が有効なケースでは、`!== null` や `!== undefined` を使いましょう。

### ポイントまとめ

- `typeof`: プリミティブ型の判定に使う
- `instanceof`: クラスインスタンスの判定に使う
- `in`: オブジェクトのプロパティ存在チェックに使う
- `===` / `!==`: 特定の値との一致を判定
- 真偽値チェック: null/undefined の簡易チェック（falsy値に注意）

## 7.3 カスタム型ガード関数（型述語）

### なぜカスタム型ガードが必要か

組み込みの型ガードではカバーできないケースがあります。例えば、APIから受け取ったデータが特定の型かどうかを判定する場合です。

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

// ❌ 通常の関数では型が絞り込まれない
function isUser(value: unknown): boolean {
  return (
    typeof value === "object" &&
    value !== null &&
    "id" in value &&
    "name" in value &&
    "email" in value
  );
}

const data: unknown = { id: 1, name: "太郎", email: "taro@example.com" };

if (isUser(data)) {
  // data はまだ unknown型... 絞り込まれない！
  // data.name; // エラー！
}
```

### is 述語（Type Predicate）

`is` キーワードを使うと、関数の戻り値が `true` のとき、引数の型が特定の型であることをTypeScriptに伝えられます。

```typescript
// ✅ is 述語を使ったカスタム型ガード
function isUser(value: unknown): value is User {
  return (
    typeof value === "object" &&
    value !== null &&
    "id" in value &&
    typeof (value as User).id === "number" &&
    "name" in value &&
    typeof (value as User).name === "string" &&
    "email" in value &&
    typeof (value as User).email === "string"
  );
}

const data: unknown = { id: 1, name: "太郎", email: "taro@example.com" };

if (isUser(data)) {
  // data は User型に絞り込まれた！
  console.log(data.name);  // OK: "太郎"
  console.log(data.email); // OK: "taro@example.com"
}
```

### 配列のフィルタリングでの活用

カスタム型ガードは、配列の `filter` メソッドと組み合わせると非常に強力です。

```typescript
// null/undefinedを除外
function isNotNull<T>(value: T | null | undefined): value is T {
  return value !== null && value !== undefined;
}

const items: (string | null | undefined)[] = ["a", null, "b", undefined, "c"];

// ❌ 型ガードなしだと (string | null | undefined)[] のまま
const filtered1 = items.filter((item) => item !== null && item !== undefined);
// filtered1 の型: (string | null | undefined)[]  ← 絞り込まれない

// ✅ 型ガード関数を使うと string[] になる
const filtered2 = items.filter(isNotNull);
// filtered2 の型: string[]  ← 正しく絞り込まれた！

// 判別ユニオンの特定メンバーをフィルタ
type Result<T> = { ok: true; value: T } | { ok: false; error: string };

function isOk<T>(result: Result<T>): result is { ok: true; value: T } {
  return result.ok;
}

const results: Result<number>[] = [
  { ok: true, value: 1 },
  { ok: false, error: "失敗" },
  { ok: true, value: 3 },
];

const successes = results.filter(isOk);
// successes の型: { ok: true; value: number }[]
const values = successes.map((r) => r.value); // number[] → [1, 3]
```

### 判別ユニオン型の型ガード

```typescript
interface Circle {
  kind: "circle";
  radius: number;
}

interface Rectangle {
  kind: "rectangle";
  width: number;
  height: number;
}

type Shape = Circle | Rectangle;

function isCircle(shape: Shape): shape is Circle {
  return shape.kind === "circle";
}

function isRectangle(shape: Shape): shape is Rectangle {
  return shape.kind === "rectangle";
}

const shapes: Shape[] = [
  { kind: "circle", radius: 5 },
  { kind: "rectangle", width: 10, height: 20 },
  { kind: "circle", radius: 3 },
];

const circles = shapes.filter(isCircle);
// circles の型: Circle[]

const totalArea = circles.reduce(
  (sum, c) => sum + Math.PI * c.radius ** 2,
  0
);
```

### ポイントまとめ

- `value is Type` 構文で、関数の戻り値と型の絞り込みを紐付ける
- `filter()` と組み合わせると、配列の要素の型を正しく絞り込める
- カスタム型ガード内の判定ロジックは開発者の責任 -- 間違った判定をするとバグになる

## 7.4 アサーション関数（Assertion Functions）

### asserts 型述語

`asserts` キーワードを使うと、「この関数が正常に返った場合、引数は特定の型である」ことを表現できます。関数が `throw` した場合は条件が満たされなかったことを意味します。

```typescript
// asserts 型述語
function assertIsString(value: unknown): asserts value is string {
  if (typeof value !== "string") {
    throw new Error(`Expected string, got ${typeof value}`);
  }
}

function processInput(input: unknown): void {
  // input は unknown
  assertIsString(input);
  // ここでは input は string に絞り込まれている！
  console.log(input.toUpperCase());
}
```

### 実践例: バリデーション関数

```typescript
interface UserInput {
  name: string;
  age: number;
  email: string;
}

function assertValidUserInput(data: unknown): asserts data is UserInput {
  if (typeof data !== "object" || data === null) {
    throw new Error("データはオブジェクトでなければなりません");
  }

  const obj = data as Record<string, unknown>;

  if (typeof obj.name !== "string" || obj.name.length === 0) {
    throw new Error("nameは空でない文字列でなければなりません");
  }

  if (typeof obj.age !== "number" || obj.age < 0 || obj.age > 150) {
    throw new Error("ageは0〜150の数値でなければなりません");
  }

  if (typeof obj.email !== "string" || !obj.email.includes("@")) {
    throw new Error("emailは有効なメールアドレスでなければなりません");
  }
}

// 使用例
function handleRequest(body: unknown): void {
  assertValidUserInput(body);
  // body は UserInput型に絞り込まれている
  console.log(`ユーザー登録: ${body.name} (${body.age}歳)`);
}
```

### assertNonNull ヘルパー

```typescript
function assertNonNull<T>(
  value: T | null | undefined,
  message?: string
): asserts value is T {
  if (value === null || value === undefined) {
    throw new Error(message ?? "値がnullまたはundefinedです");
  }
}

// DOM操作での使用例
function setupUI(): void {
  const button = document.getElementById("submit-btn");
  assertNonNull(button, "送信ボタンが見つかりません");
  // button は HTMLElement型に絞り込まれている
  button.addEventListener("click", () => {
    console.log("クリックされました");
  });
}
```

### is 述語 vs asserts 述語

| 特徴 | `is` 述語 | `asserts` 述語 |
|------|-----------|----------------|
| 戻り値 | `boolean` | `void`（例外で失敗） |
| 使い方 | `if` 文の条件として使う | 関数呼び出し後に型が絞り込まれる |
| 失敗時 | `false` を返す | 例外を投げる |
| 用途 | 条件分岐 | 前提条件のバリデーション |

```typescript
// is述語: 分岐して処理する場合に
function isString(value: unknown): value is string {
  return typeof value === "string";
}

if (isString(data)) {
  // stringの場合の処理
} else {
  // string以外の場合の処理
}

// asserts述語: 前提条件を満たさなければ即エラーにしたい場合に
function assertString(value: unknown): asserts value is string {
  if (typeof value !== "string") throw new Error("文字列ではありません");
}

assertString(data);
// ここから先、data は string型
```

## 7.5 高度な絞り込みパターン

### 代入による絞り込み

```typescript
let value: string | number;

value = "hello"; // 代入後、valueはstring型
console.log(value.toUpperCase()); // OK

value = 42; // 代入後、valueはnumber型
console.log(value.toFixed(2)); // OK
```

### never を使った網羅性チェック

```typescript
type Shape =
  | { kind: "circle"; radius: number }
  | { kind: "rectangle"; width: number; height: number }
  | { kind: "triangle"; base: number; height: number };

function assertNever(value: never): never {
  throw new Error(`予期しない値: ${JSON.stringify(value)}`);
}

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "rectangle":
      return shape.width * shape.height;
    case "triangle":
      return (shape.base * shape.height) / 2;
    default:
      // 全てのケースを処理していれば、shapeはnever型
      return assertNever(shape);
  }
}

// Shapeに新しいメンバーを追加すると、
// default句でコンパイルエラーになる → 処理の追加忘れを防止
```

### オプショナルチェーンと型の絞り込み

```typescript
interface Company {
  name: string;
  address?: {
    city: string;
    postalCode?: string;
  };
}

function getCity(company: Company): string {
  // オプショナルチェーン（?.）は型を絞り込まない
  const city = company.address?.city;
  // city の型は string | undefined

  // 絞り込むには条件チェックが必要
  if (company.address) {
    // company.address は { city: string; postalCode?: string }
    return company.address.city;
  }

  return "不明";
}
```

### Null合体演算子（??）との組み合わせ

```typescript
function getDisplayName(
  user: { name?: string; nickname?: string } | null
): string {
  // ?? は null と undefined のみを除外（0 や "" は通す）
  return user?.nickname ?? user?.name ?? "匿名ユーザー";
}

getDisplayName({ name: "太郎", nickname: "タロちゃん" }); // "タロちゃん"
getDisplayName({ name: "太郎" });                         // "太郎"
getDisplayName(null);                                     // "匿名ユーザー"
```

## 7.6 よくある間違い

### 1. 型ガード関数の判定ロジックが不正確

```typescript
// ❌ 危険: 不十分な型チェック
function isUser(value: unknown): value is User {
  return typeof value === "object" && value !== null;
  // これだと {} も User扱いされてしまう！
}

// ✅ 安全: 全てのプロパティを検証
function isUserSafe(value: unknown): value is User {
  if (typeof value !== "object" || value === null) return false;
  const obj = value as Record<string, unknown>;
  return (
    typeof obj.id === "number" &&
    typeof obj.name === "string" &&
    typeof obj.email === "string"
  );
}
```

### 2. typeof null の罠

```typescript
// ❌ typeof null は "object"
function process(value: string | object | null) {
  if (typeof value === "object") {
    // value は object | null — nullが含まれている！
    // value.toString(); // null.toString() でランタイムエラーの可能性
  }
}

// ✅ nullを先にチェック
function processSafe(value: string | object | null) {
  if (value === null) return;
  if (typeof value === "object") {
    // value は object（nullは除外済み）
    value.toString(); // OK
  }
}
```

### 3. 型ガード後にスコープを超えて使おうとする

```typescript
function example(value: string | number) {
  if (typeof value === "string") {
    // ここでは value は string
    const upper = value.toUpperCase();
  }

  // ここでは value は再び string | number
  // 前の if ブロックの絞り込みは影響しない
  // value.toUpperCase(); // エラー！
}
```

## 7.7 現場でのベストプラクティス

### 1. ガード関数を共通ユーティリティとしてまとめる

```typescript
// utils/typeGuards.ts
export function isNonNull<T>(value: T | null | undefined): value is T {
  return value !== null && value !== undefined;
}

export function isString(value: unknown): value is string {
  return typeof value === "string";
}

export function isNumber(value: unknown): value is number {
  return typeof value === "number" && !Number.isNaN(value);
}

export function hasProperty<K extends string>(
  obj: unknown,
  key: K
): obj is Record<K, unknown> {
  return typeof obj === "object" && obj !== null && key in obj;
}
```

### 2. zodなどのバリデーションライブラリと型ガードを組み合わせる

```typescript
// 実務では、手書きの型ガードよりもバリデーションライブラリを使うことが多い
// zod, yup, io-ts などのライブラリは、
// バリデーションと型の絞り込みを同時に行ってくれる

// 手書きの型ガードが適切な場面:
// - ライブラリを導入したくない小さなプロジェクト
// - パフォーマンスが重要な場面
// - 型の判別が単純な場面（判別ユニオンなど）
```

### 3. 早期リターンパターンを活用する

```typescript
// ✅ 早期リターンで型を絞り込む（ガード節パターン）
function processUser(user: User | null | undefined): string {
  if (!user) return "ユーザーが見つかりません";
  // ここから先、user は User型

  if (!user.email) return "メールアドレスが未設定です";
  // ここから先、user.email は string型

  return `${user.name} <${user.email}>`;
}
```
