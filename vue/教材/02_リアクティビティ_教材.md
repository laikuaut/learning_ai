# 第2章：リアクティビティシステム

## 2.1 リアクティビティとは

リアクティビティ（反応性）とは、**データが変更されたとき、それに依存するUIが自動的に更新される仕組み**です。Vue.jsの根幹をなす機能であり、手動でDOM操作する必要がなくなります。

```
データ変更 → Vueが検知 → 依存する箇所を自動更新 → UIに反映
```

### Vue 3 のリアクティビティの仕組み

Vue 3 は JavaScript の **Proxy** オブジェクトを使ってリアクティビティを実現しています。

```js
// Vue内部のイメージ（概念的なコード）
const data = new Proxy(target, {
  get(target, key) {
    track(target, key)    // この値を「追跡」
    return target[key]
  },
  set(target, key, value) {
    target[key] = value
    trigger(target, key)  // 依存する処理を「発火」
    return true
  }
})
```

## 2.2 ref() — プリミティブ値のリアクティブ化

`ref()` は最も基本的なリアクティブ変数を作成する関数です。

```vue
<script setup>
import { ref } from 'vue'

// ref() でリアクティブな値を作成
const count = ref(0)          // 数値
const message = ref('hello')  // 文字列
const isVisible = ref(true)   // 真偽値
const user = ref(null)        // null（後からオブジェクトを代入可能）

// script 内では .value でアクセス
console.log(count.value)      // 0
count.value++                 // 1に増加
message.value = 'こんにちは'   // 文字列を変更
</script>

<template>
  <!-- template 内では .value は不要（自動的にアンラップされる） -->
  <p>カウント: {{ count }}</p>
  <p>メッセージ: {{ message }}</p>
  <button @click="count++">+1</button>
</template>
```

### ref の重要なポイント

```js
import { ref } from 'vue'

const count = ref(0)

// ✅ 正しい: .value を使う
count.value = 10

// ❌ 間違い: ref オブジェクト自体を置き換えてしまう
// count = 10  // リアクティビティが失われる！

// ref はオブジェクトも格納できる
const user = ref({ name: '太郎', age: 20 })
user.value.name = '花子'  // プロパティの変更も検知される
user.value = { name: '次郎', age: 25 }  // 丸ごと置き換えもOK
```

## 2.3 reactive() — オブジェクトのリアクティブ化

`reactive()` はオブジェクトや配列をリアクティブにします。`.value` が不要です。

```vue
<script setup>
import { reactive } from 'vue'

// オブジェクトをリアクティブに
const state = reactive({
  count: 0,
  message: 'hello',
  user: {
    name: '太郎',
    age: 20
  }
})

// .value 不要で直接アクセス
state.count++
state.message = 'こんにちは'
state.user.name = '花子'  // ネストしたプロパティもリアクティブ

// 配列もリアクティブに
const items = reactive(['りんご', 'バナナ'])
items.push('みかん')  // 配列メソッドも検知される
</script>

<template>
  <p>{{ state.count }}</p>
  <p>{{ state.message }}</p>
  <p>{{ state.user.name }}（{{ state.user.age }}歳）</p>
  <ul>
    <li v-for="item in items" :key="item">{{ item }}</li>
  </ul>
</template>
```

### ref と reactive の比較

| 特徴 | `ref()` | `reactive()` |
|------|---------|--------------|
| 対象 | あらゆる型 | オブジェクト/配列のみ |
| アクセス（script内） | `.value` が必要 | 直接アクセス |
| アクセス（template内） | 自動アンラップ | 直接アクセス |
| 再代入 | `ref.value = 新値` で可能 | 再代入するとリアクティビティ喪失 |
| 分割代入 | — | リアクティビティが失われる |

### reactive の注意点

```js
import { reactive } from 'vue'

const state = reactive({ count: 0 })

// ❌ 分割代入するとリアクティビティが失われる
let { count } = state
count++  // state.count は変わらない！

// ❌ 再代入するとリアクティビティが失われる
// state = reactive({ count: 1 })  // 元の参照が切れる

// ✅ プロパティを個別に更新する
state.count++

// ✅ Object.assign で一括更新
Object.assign(state, { count: 1 })
```

**結論**: 迷ったら `ref()` を使うのが安全です。Vue公式も `ref()` を推奨しています。

## 2.4 computed() — 算出プロパティ

`computed()` は、他のリアクティブな値から**派生する値**を定義します。依存する値が変わったときだけ再計算されます（キャッシュあり）。

```vue
<script setup>
import { ref, computed } from 'vue'

const firstName = ref('山田')
const lastName = ref('太郎')
const price = ref(1000)
const quantity = ref(3)
const items = ref([
  { name: 'りんご', done: true },
  { name: 'バナナ', done: false },
  { name: 'みかん', done: true }
])

// 算出プロパティ（読み取り専用）
const fullName = computed(() => {
  console.log('fullName が再計算された')  // 依存値が変わった時だけ実行
  return `${firstName.value} ${lastName.value}`
})

const total = computed(() => price.value * quantity.value)

const tax = computed(() => Math.floor(total.value * 0.1))

const totalWithTax = computed(() => total.value + tax.value)

// フィルタリング
const completedItems = computed(() =>
  items.value.filter(item => item.done)
)

const completedCount = computed(() => completedItems.value.length)
</script>

<template>
  <p>氏名: {{ fullName }}</p>
  <p>小計: {{ total.toLocaleString() }}円</p>
  <p>消費税: {{ tax.toLocaleString() }}円</p>
  <p>合計: {{ totalWithTax.toLocaleString() }}円</p>
  <p>完了済み: {{ completedCount }} / {{ items.length }}</p>
</template>
```

### 書き込み可能な computed

```js
import { ref, computed } from 'vue'

const firstName = ref('太郎')
const lastName = ref('山田')

// getter と setter を定義
const fullName = computed({
  get() {
    return `${lastName.value} ${firstName.value}`
  },
  set(newValue) {
    const parts = newValue.split(' ')
    lastName.value = parts[0]
    firstName.value = parts[1] || ''
  }
})

fullName.value = '佐藤 花子'
// → lastName = '佐藤', firstName = '花子'
```

### computed vs メソッド

```vue
<script setup>
import { ref, computed } from 'vue'

const count = ref(0)

// computed: キャッシュされる。count が変わった時だけ再計算
const doubled = computed(() => {
  console.log('computed 実行')
  return count.value * 2
})

// メソッド: テンプレートが再描画されるたびに実行
function getDoubled() {
  console.log('method 実行')
  return count.value * 2
}
</script>

<template>
  <!-- computed: 何度参照してもキャッシュから返す -->
  <p>{{ doubled }}</p>
  <p>{{ doubled }}</p>

  <!-- method: 呼ばれるたびに実行される -->
  <p>{{ getDoubled() }}</p>
  <p>{{ getDoubled() }}</p>
</template>
```

## 2.5 watch() — 値の変更を監視

`watch()` は特定のリアクティブな値を監視し、変更時にコールバックを実行します。

```vue
<script setup>
import { ref, watch } from 'vue'

const searchQuery = ref('')
const count = ref(0)
const user = ref({ name: '太郎', age: 20 })

// 基本形: ref を監視
watch(searchQuery, (newValue, oldValue) => {
  console.log(`検索: "${oldValue}" → "${newValue}"`)
  // API呼び出しなどの副作用を実行
})

// 即座に実行 + 監視
watch(count, (newValue) => {
  console.log(`count: ${newValue}`)
}, { immediate: true })  // マウント時にも実行

// ディープウォッチ（ネストしたプロパティの変更も検知）
watch(user, (newValue) => {
  console.log('user changed:', newValue)
}, { deep: true })

// 特定のプロパティだけ監視（ゲッター関数を使う）
watch(
  () => user.value.name,  // ゲッター関数
  (newName, oldName) => {
    console.log(`名前: ${oldName} → ${newName}`)
  }
)

// 複数のソースを同時に監視
watch(
  [searchQuery, count],
  ([newQuery, newCount], [oldQuery, oldCount]) => {
    console.log('どちらかが変更された')
  }
)
</script>
```

### watch のクリーンアップ

```js
import { ref, watch } from 'vue'

const searchQuery = ref('')

watch(searchQuery, (newQuery, oldQuery, onCleanup) => {
  // 非同期処理（前回のリクエストをキャンセルしたい場合）
  const controller = new AbortController()

  fetch(`/api/search?q=${newQuery}`, {
    signal: controller.signal
  })

  // 次回の watch 実行前、またはコンポーネント破棄時に呼ばれる
  onCleanup(() => {
    controller.abort()  // 前回のリクエストをキャンセル
  })
})
```

### watch の停止

```js
import { ref, watch } from 'vue'

const count = ref(0)

// watch は停止関数を返す
const stopWatch = watch(count, (newValue) => {
  console.log(newValue)
  if (newValue >= 10) {
    stopWatch()  // 10以上になったら監視を停止
  }
})
```

## 2.6 watchEffect() — 自動依存追跡

`watchEffect()` は、コールバック内で使われているリアクティブな値を**自動的に追跡**します。明示的な監視対象の指定が不要です。

```vue
<script setup>
import { ref, watchEffect } from 'vue'

const count = ref(0)
const message = ref('hello')

// count と message の両方を自動追跡
// どちらかが変更されると再実行される
watchEffect(() => {
  console.log(`count: ${count.value}, message: ${message.value}`)
})
// → 即座に実行される（watch の immediate: true 相当）

// DOM操作のタイミングを制御
watchEffect(() => {
  // DOM更新後に実行したい場合
}, { flush: 'post' })
</script>
```

### watch vs watchEffect

| 特徴 | `watch` | `watchEffect` |
|------|---------|---------------|
| 監視対象 | 明示的に指定 | 自動追跡 |
| 初回実行 | `immediate: true` が必要 | 常に即座実行 |
| 古い値 | 取得可能 | 取得不可 |
| 用途 | 特定の値の変更に応じた処理 | 複数の値に依存する副作用 |

## 2.7 toRef / toRefs — reactive からの変換

`reactive()` オブジェクトのプロパティを `ref` として取り出す関数です。

```vue
<script setup>
import { reactive, toRef, toRefs } from 'vue'

const state = reactive({
  name: '太郎',
  age: 20,
  email: 'taro@example.com'
})

// toRef: 1つのプロパティを ref に変換（元のオブジェクトとリンク）
const nameRef = toRef(state, 'name')
nameRef.value = '花子'  // state.name も '花子' に変わる

// toRefs: 全プロパティを ref に変換
const { name, age, email } = toRefs(state)
// name.value, age.value, email.value でアクセス
// 元の state ともリンクしている

age.value = 25  // state.age も 25 に変わる

// composable（再利用可能関数）で特に有用
function useUser() {
  const user = reactive({
    name: '',
    email: ''
  })

  // toRefs で返すことで、分割代入してもリアクティビティが維持される
  return toRefs(user)
}
</script>

<template>
  <p>{{ name }}（{{ age }}歳）</p>
  <p>{{ email }}</p>
</template>
```

## 2.8 shallowRef / shallowReactive

パフォーマンスが重要な場面では、浅い（shallow）リアクティビティを使えます。

```js
import { shallowRef, shallowReactive, triggerRef } from 'vue'

// shallowRef: .value の置き換えのみ検知（中のプロパティ変更は検知しない）
const state = shallowRef({ count: 0 })
state.value.count++           // ❌ UIは更新されない
state.value = { count: 1 }    // ✅ UIが更新される

// 手動でトリガーすることも可能
state.value.count++
triggerRef(state)  // 強制的にUIを更新

// shallowReactive: 第1階層のプロパティのみ検知
const obj = shallowReactive({
  nested: { count: 0 }
})
obj.nested = { count: 1 }  // ✅ 検知される
obj.nested.count++          // ❌ 検知されない
```

## 2.9 リアクティビティのデバッグ

```js
import { ref, watchEffect } from 'vue'

const count = ref(0)

watchEffect((onCleanup) => {
  console.log(count.value)
}, {
  onTrack(e) {
    // リアクティブな値が追跡されたとき
    console.log('tracked:', e)
  },
  onTrigger(e) {
    // リアクティブな値が変更されて再実行されるとき
    debugger  // ブラウザのデバッガで停止
    console.log('triggered:', e)
  }
})
```

## 2.10 本章のまとめ

| 関数 | 用途 | .value |
|------|------|--------|
| `ref()` | あらゆる値をリアクティブに | 必要 |
| `reactive()` | オブジェクト/配列をリアクティブに | 不要 |
| `computed()` | 派生値の算出（キャッシュあり） | 必要 |
| `watch()` | 特定の値の変更を監視 | — |
| `watchEffect()` | 依存する値を自動追跡して副作用実行 | — |
| `toRef()` | reactive の1プロパティを ref 化 | 必要 |
| `toRefs()` | reactive の全プロパティを ref 化 | 必要 |

### ベストプラクティス

1. **迷ったら `ref()` を使う** — 最も汎用的で安全
2. **派生値は `computed()` にする** — キャッシュにより効率的
3. **副作用（API呼び出しなど）は `watch()` / `watchEffect()` で** — テンプレートではなく明示的に
4. **`reactive()` の分割代入には `toRefs()` を使う** — リアクティビティの喪失を防ぐ
