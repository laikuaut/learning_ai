## 問題1：基本的なクラスの型（基本）

`User` クラスを作成してください。

- プロパティ：`name: string`, `age: number`, `email: string`
- メソッド：`greet(): string` — 挨拶文を返す
- メソッド：`toString(): string` — ユーザー情報の文字列表現

Parameter Properties（省略記法）を使ってください。

**期待される出力：**
```
こんにちは、太郎です（25歳）
User: 太郎 <taro@example.com>（25歳）
```

<details>
<summary>解答例</summary>

```typescript
class User {
  // Parameter Properties で宣言と代入を同時に
  constructor(
    public name: string,
    public age: number,
    public email: string
  ) {}

  greet(): string {
    return `こんにちは、${this.name}です（${this.age}歳）`;
  }

  toString(): string {
    return `User: ${this.name} <${this.email}>（${this.age}歳）`;
  }
}

const user = new User("太郎", 25, "taro@example.com");
console.log(user.greet());
console.log(user.toString());
```

</details>

## 問題2：アクセス修飾子（基本）

銀行口座クラス `BankAccount` を実装してください。

- `public ownerName: string` — 口座名義
- `private balance: number` — 残高（外部から直接アクセス不可）
- `private readonly accountId: string` — 口座番号（変更不可）
- `public deposit(amount): void` — 入金
- `public withdraw(amount): boolean` — 出金（残高不足なら `false`）
- `public getBalance(): number` — 残高照会

**期待される出力：**
```
口座: 太郎（残高: ¥10,000）
入金 ¥5,000 → 残高: ¥15,000
出金 ¥3,000 → 成功 → 残高: ¥12,000
出金 ¥20,000 → 失敗（残高不足）→ 残高: ¥12,000
```

<details>
<summary>解答例</summary>

```typescript
class BankAccount {
  private balance: number;
  private readonly accountId: string;

  constructor(
    public ownerName: string,
    initialBalance: number
  ) {
    this.balance = initialBalance;
    this.accountId = `ACC-${Date.now()}`;
  }

  deposit(amount: number): void {
    if (amount <= 0) throw new Error("入金額は正の数でなければなりません");
    this.balance += amount;
  }

  withdraw(amount: number): boolean {
    if (amount <= 0) return false;
    if (amount > this.balance) return false;
    this.balance -= amount;
    return true;
  }

  getBalance(): number {
    return this.balance;
  }
}

const account = new BankAccount("太郎", 10000);
console.log(`口座: ${account.ownerName}（残高: ¥${account.getBalance().toLocaleString()}）`);

account.deposit(5000);
console.log(`入金 ¥5,000 → 残高: ¥${account.getBalance().toLocaleString()}`);

const success1 = account.withdraw(3000);
console.log(`出金 ¥3,000 → ${success1 ? "成功" : "失敗"} → 残高: ¥${account.getBalance().toLocaleString()}`);

const success2 = account.withdraw(20000);
console.log(`出金 ¥20,000 → ${success2 ? "成功" : "失敗（残高不足）"} → 残高: ¥${account.getBalance().toLocaleString()}`);

// account.balance; // エラー！privateプロパティ
// account.accountId; // エラー！privateプロパティ
```

## 問題3：継承とprotected（基本）

`Vehicle` 基底クラスと `Car`, `Bicycle` サブクラスを作成してください。

- `Vehicle`: `protected speed: number`, `public accelerate()`, `public brake()`, `public getInfo()`
- `Car`: `honk()` メソッドを追加
- `Bicycle`: `ringBell()` メソッドを追加

**期待される出力：**
```
トヨタ カローラ: 時速 0km → 加速 → 時速 60km
プップー！
ジャイアント ロード: 時速 0km → 加速 → 時速 25km
チリンチリン！
```

<details>
<summary>解答例</summary>

```typescript
class Vehicle {
  protected speed: number = 0;

  constructor(
    public readonly brand: string,
    public readonly model: string,
    protected maxSpeed: number
  ) {}

  accelerate(amount: number): void {
    this.speed = Math.min(this.speed + amount, this.maxSpeed);
  }

  brake(amount: number): void {
    this.speed = Math.max(this.speed - amount, 0);
  }

  getInfo(): string {
    return `${this.brand} ${this.model}: 時速 ${this.speed}km`;
  }
}

class Car extends Vehicle {
  honk(): string {
    return "プップー！";
  }
}

class Bicycle extends Vehicle {
  ringBell(): string {
    return "チリンチリン！";
  }
}

const car = new Car("トヨタ", "カローラ", 180);
console.log(`${car.getInfo()} → 加速 → ${(car.accelerate(60), car.getInfo())}`);
console.log(car.honk());

const bike = new Bicycle("ジャイアント", "ロード", 40);
console.log(`${bike.getInfo()} → 加速 → ${(bike.accelerate(25), bike.getInfo())}`);
console.log(bike.ringBell());
```

</details>

## 問題4：抽象クラス（応用）

図形の抽象クラスを設計し、具象クラスを実装してください。

- `abstract class Shape`: 抽象メソッド `getArea()`, `getPerimeter()`, 通常メソッド `describe()`
- `Circle`: 半径を受け取る
- `Rectangle`: 幅と高さを受け取る

**期待される出力：**
```
円（半径: 5）: 面積 = 78.54, 周長 = 31.42
長方形（10 x 5）: 面積 = 50.00, 周長 = 30.00
```

<details>
<summary>解答例</summary>

```typescript
abstract class Shape {
  // 抽象メソッド（サブクラスで実装必須）
  abstract getArea(): number;
  abstract getPerimeter(): number;
  abstract getName(): string;

  // 通常メソッド（共通ロジック）
  describe(): string {
    return `${this.getName()}: 面積 = ${this.getArea().toFixed(2)}, 周長 = ${this.getPerimeter().toFixed(2)}`;
  }
}

class Circle extends Shape {
  constructor(private radius: number) {
    super();
  }

  getArea(): number {
    return Math.PI * this.radius ** 2;
  }

  getPerimeter(): number {
    return 2 * Math.PI * this.radius;
  }

  getName(): string {
    return `円（半径: ${this.radius}）`;
  }
}

class Rectangle extends Shape {
  constructor(
    private width: number,
    private height: number
  ) {
    super();
  }

  getArea(): number {
    return this.width * this.height;
  }

  getPerimeter(): number {
    return 2 * (this.width + this.height);
  }

  getName(): string {
    return `長方形（${this.width} x ${this.height}）`;
  }
}

// 抽象クラスの変数にサブクラスのインスタンスを代入
const shapes: Shape[] = [new Circle(5), new Rectangle(10, 5)];
shapes.forEach((s) => console.log(s.describe()));
```

</details>

## 問題5：インターフェースの実装（応用）

以下のインターフェースを定義し、クラスで実装してください。

- `Serializable`: `serialize(): string`, `deserialize(data: string): void`
- `Comparable<T>`: `compareTo(other: T): number`
- `Task` クラス: 両方のインターフェースを実装

**期待される出力：**
```
タスク: バグ修正（優先度: 3）
シリアライズ: {"title":"バグ修正","priority":3,"completed":false}
比較: バグ修正 > ドキュメント作成（優先度が高い）
ソート結果: テスト作成(1), ドキュメント作成(2), バグ修正(3)
```

<details>
<summary>解答例</summary>

```typescript
interface Serializable {
  serialize(): string;
  deserialize(data: string): void;
}

interface Comparable<T> {
  compareTo(other: T): number;
}

class Task implements Serializable, Comparable<Task> {
  constructor(
    public title: string,
    public priority: number,
    public completed: boolean = false
  ) {}

  // Serializableの実装
  serialize(): string {
    return JSON.stringify({
      title: this.title,
      priority: this.priority,
      completed: this.completed,
    });
  }

  deserialize(data: string): void {
    const parsed = JSON.parse(data);
    this.title = parsed.title;
    this.priority = parsed.priority;
    this.completed = parsed.completed;
  }

  // Comparableの実装（優先度で比較）
  compareTo(other: Task): number {
    return this.priority - other.priority;
  }

  toString(): string {
    return `${this.title}(${this.priority})`;
  }
}

// テスト
const task1 = new Task("バグ修正", 3);
const task2 = new Task("ドキュメント作成", 2);
const task3 = new Task("テスト作成", 1);

console.log(`タスク: ${task1.title}（優先度: ${task1.priority}）`);
console.log(`シリアライズ: ${task1.serialize()}`);

const comparison = task1.compareTo(task2);
const compStr = comparison > 0 ? ">" : comparison < 0 ? "<" : "=";
console.log(`比較: ${task1.title} ${compStr} ${task2.title}（優先度が高い）`);

// ソート
const tasks = [task1, task2, task3];
const sorted = [...tasks].sort((a, b) => a.compareTo(b));
console.log(`ソート結果: ${sorted.map((t) => t.toString()).join(", ")}`);
```

</details>

## 問題6：ジェネリッククラス（応用）

ジェネリックな `Queue<T>`（先入先出）クラスを実装してください。

- `enqueue(item: T): void`
- `dequeue(): T | undefined`
- `peek(): T | undefined`
- `size` ゲッター
- `isEmpty(): boolean`
- `toArray(): T[]`

**期待される出力：**
```
キューに追加: タスクA, タスクB, タスクC
先頭: タスクA
取り出し: タスクA
残り: [タスクB, タスクC]
サイズ: 2
```

<details>
<summary>解答例</summary>

```typescript
class Queue<T> {
  private items: T[] = [];

  enqueue(item: T): void {
    this.items.push(item);
  }

  dequeue(): T | undefined {
    return this.items.shift();
  }

  peek(): T | undefined {
    return this.items[0];
  }

  get size(): number {
    return this.items.length;
  }

  isEmpty(): boolean {
    return this.items.length === 0;
  }

  toArray(): T[] {
    return [...this.items];
  }
}

// 文字列のキュー
const queue = new Queue<string>();
queue.enqueue("タスクA");
queue.enqueue("タスクB");
queue.enqueue("タスクC");
console.log("キューに追加: タスクA, タスクB, タスクC");

console.log(`先頭: ${queue.peek()}`);

const dequeued = queue.dequeue();
console.log(`取り出し: ${dequeued}`);
console.log(`残り: [${queue.toArray().join(", ")}]`);
console.log(`サイズ: ${queue.size}`);
```

</details>

## 問題7：シングルトンパターン（チャレンジ）

設定管理クラスを「シングルトンパターン」で実装してください。

- `private constructor` でインスタンス化を制限
- `static getInstance()` で唯一のインスタンスを取得
- `get(key)` と `set(key, value)` で設定を読み書き

**期待される出力：**
```
同じインスタンス: true
設定 - theme: dark
設定 - language: ja
全設定: {"theme":"dark","language":"ja","fontSize":14}
```

<details>
<summary>解答例</summary>

```typescript
class AppSettings {
  private static instance: AppSettings | null = null;
  private settings: Record<string, unknown> = {};

  // privateコンストラクタ（外部からnewできない）
  private constructor() {}

  // 唯一のインスタンスを取得
  static getInstance(): AppSettings {
    if (!AppSettings.instance) {
      AppSettings.instance = new AppSettings();
    }
    return AppSettings.instance;
  }

  set(key: string, value: unknown): void {
    this.settings[key] = value;
  }

  get<T>(key: string): T | undefined {
    return this.settings[key] as T | undefined;
  }

  getAll(): Record<string, unknown> {
    return { ...this.settings };
  }
}

// テスト
const settings1 = AppSettings.getInstance();
const settings2 = AppSettings.getInstance();

console.log(`同じインスタンス: ${settings1 === settings2}`);

settings1.set("theme", "dark");
settings1.set("language", "ja");
settings2.set("fontSize", 14); // settings1と同じインスタンス

console.log(`設定 - theme: ${settings1.get<string>("theme")}`);
console.log(`設定 - language: ${settings2.get<string>("language")}`);
console.log(`全設定: ${JSON.stringify(settings1.getAll())}`);
```

</details>

## 問題8：総合問題 — ログシステム（チャレンジ）

以下の設計でログシステムを実装してください。

1. `Logger` インターフェース：`info()`, `warn()`, `error()` メソッド
2. `abstract class BaseLogger implements Logger`：共通のフォーマット処理
3. `ConsoleLogger extends BaseLogger`：コンソールに出力
4. `ArrayLogger extends BaseLogger`：配列にログを蓄積（テスト用）

**期待される出力：**
```
[INFO] 2026-03-25T... アプリケーション起動
[WARN] 2026-03-25T... メモリ使用量が80%を超えています
[ERROR] 2026-03-25T... データベース接続に失敗

ArrayLoggerの蓄積ログ: 3件
  [INFO] テスト開始
  [ERROR] テスト中のエラー
  [INFO] テスト完了
```

<details>
<summary>解答例</summary>

```typescript
// ログレベルの型
type LogLevel = "INFO" | "WARN" | "ERROR";

// Loggerインターフェース
interface Logger {
  info(message: string): void;
  warn(message: string): void;
  error(message: string): void;
}

// 抽象基底クラス（共通のフォーマットロジック）
abstract class BaseLogger implements Logger {
  protected formatMessage(level: LogLevel, message: string): string {
    const timestamp = new Date().toISOString();
    return `[${level}] ${timestamp} ${message}`;
  }

  // サブクラスが実装する出力メソッド
  protected abstract output(formattedMessage: string): void;

  info(message: string): void {
    this.output(this.formatMessage("INFO", message));
  }

  warn(message: string): void {
    this.output(this.formatMessage("WARN", message));
  }

  error(message: string): void {
    this.output(this.formatMessage("ERROR", message));
  }
}

// コンソール出力のロガー
class ConsoleLogger extends BaseLogger {
  protected output(formattedMessage: string): void {
    console.log(formattedMessage);
  }
}

// 配列蓄積のロガー（テスト用）
class ArrayLogger extends BaseLogger {
  private logs: string[] = [];

  protected output(formattedMessage: string): void {
    this.logs.push(formattedMessage);
  }

  getLogs(): string[] {
    return [...this.logs];
  }

  clear(): void {
    this.logs = [];
  }
}

// ConsoleLoggerのテスト
const consoleLogger: Logger = new ConsoleLogger();
consoleLogger.info("アプリケーション起動");
consoleLogger.warn("メモリ使用量が80%を超えています");
consoleLogger.error("データベース接続に失敗");

// ArrayLoggerのテスト
console.log("");
const arrayLogger = new ArrayLogger();
arrayLogger.info("テスト開始");
arrayLogger.error("テスト中のエラー");
arrayLogger.info("テスト完了");

const logs = arrayLogger.getLogs();
console.log(`ArrayLoggerの蓄積ログ: ${logs.length}件`);
logs.forEach((log) => {
  // タイムスタンプを除いて表示
  const simplified = log.replace(/\d{4}-\d{2}-\d{2}T[\d:.]+Z\s/, "");
  console.log(`  ${simplified}`);
});
```

</details>
