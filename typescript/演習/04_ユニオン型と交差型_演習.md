## 問題1：基本的なユニオン型（基本）

`string | number` 型の引数を受け取り、型に応じた処理を行う `doubleValue` 関数を実装してください。

- `string` の場合 → 文字列を2回繰り返す（`"abc"` → `"abcabc"`）
- `number` の場合 → 数値を2倍にする（`5` → `10`）

**期待される出力：**
```
doubleValue("hello") = hellohello
doubleValue(21) = 42
doubleValue("TypeScript") = TypeScriptTypeScript
doubleValue(3.14) = 6.28
```

<details>
<summary>ヒント</summary>

- `typeof` で型を絞り込みます。
- `typeof value === "string"` のブロック内では `value` は `string` 型として扱えます。

</details>

<details>
<summary>解答例</summary>

```typescript
// ユニオン型の引数をtypeofで絞り込む
function doubleValue(value: string | number): string | number {
  if (typeof value === "string") {
    // ここではvalueはstring型
    return value.repeat(2);
  } else {
    // ここではvalueはnumber型
    return value * 2;
  }
}

console.log(`doubleValue("hello") = ${doubleValue("hello")}`);
console.log(`doubleValue(21) = ${doubleValue(21)}`);
console.log(`doubleValue("TypeScript") = ${doubleValue("TypeScript")}`);
console.log(`doubleValue(3.14) = ${doubleValue(3.14)}`);
```

</details>

## 問題2：リテラル型（基本）

季節を表すリテラル型 `Season` と、季節に応じたメッセージを返す関数を実装してください。

- `"spring"` → `"桜が咲いています"`
- `"summer"` → `"暑い夏です"`
- `"autumn"` → `"紅葉がきれいです"`
- `"winter"` → `"雪が降っています"`

**期待される出力：**
```
spring: 桜が咲いています
summer: 暑い夏です
autumn: 紅葉がきれいです
winter: 雪が降っています
```

<details>
<summary>解答例</summary>

```typescript
// リテラル型のユニオン
type Season = "spring" | "summer" | "autumn" | "winter";

function getSeasonMessage(season: Season): string {
  switch (season) {
    case "spring":
      return "桜が咲いています";
    case "summer":
      return "暑い夏です";
    case "autumn":
      return "紅葉がきれいです";
    case "winter":
      return "雪が降っています";
  }
}

// 全ての季節のメッセージを表示
const seasons: Season[] = ["spring", "summer", "autumn", "winter"];
seasons.forEach((s) => {
  console.log(`${s}: ${getSeasonMessage(s)}`);
});
```

</details>

## 問題3：判別共用体（基本）

図形の面積を計算するプログラムを、判別共用体を使って実装してください。

- `Circle`: `kind: "circle"`, `radius: number`
- `Rectangle`: `kind: "rectangle"`, `width: number`, `height: number`
- `Triangle`: `kind: "triangle"`, `base: number`, `height: number`

**期待される出力：**
```
円（半径5）の面積: 78.54
長方形（10x20）の面積: 200.00
三角形（底辺6, 高さ8）の面積: 24.00
```

<details>
<summary>ヒント</summary>

- `kind` プロパティ（判別プロパティ）で型を絞り込みます。
- `switch (shape.kind)` で分岐できます。

</details>

<details>
<summary>解答例</summary>

```typescript
// 判別共用体（Discriminated Unions）
interface Circle {
  kind: "circle";
  radius: number;
}

interface Rectangle {
  kind: "rectangle";
  width: number;
  height: number;
}

interface Triangle {
  kind: "triangle";
  base: number;
  height: number;
}

type Shape = Circle | Rectangle | Triangle;

// kindプロパティで型を絞り込む
function getArea(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "rectangle":
      return shape.width * shape.height;
    case "triangle":
      return (shape.base * shape.height) / 2;
  }
}

// 図形の説明を返す
function describeShape(shape: Shape): string {
  switch (shape.kind) {
    case "circle":
      return `円（半径${shape.radius}）`;
    case "rectangle":
      return `長方形（${shape.width}x${shape.height}）`;
    case "triangle":
      return `三角形（底辺${shape.base}, 高さ${shape.height}）`;
  }
}

// 動作確認
const shapes: Shape[] = [
  { kind: "circle", radius: 5 },
  { kind: "rectangle", width: 10, height: 20 },
  { kind: "triangle", base: 6, height: 8 },
];

shapes.forEach((s) => {
  console.log(`${describeShape(s)}の面積: ${getArea(s).toFixed(2)}`);
});
```

</details>

## 問題4：交差型でミックスイン（応用）

以下の型を交差型で組み合わせて、タスク管理システムの型を定義してください。

1. `HasId`: `{ id: number }`
2. `Timestamped`: `{ createdAt: Date; updatedAt: Date }`
3. `TaskInfo`: `{ title: string; description: string; priority: "low" | "medium" | "high" }`
4. `Task` 型 = `HasId & Timestamped & TaskInfo`

タスクの配列を作成し、優先度の高い順に並べ替えて表示してください。

**期待される出力：**
```
=== タスク一覧（優先度順） ===
[HIGH] #2: バグ修正
[MEDIUM] #1: ドキュメント作成
[LOW] #3: リファクタリング
```

<details>
<summary>解答例</summary>

```typescript
type HasId = { id: number };
type Timestamped = { createdAt: Date; updatedAt: Date };
type TaskInfo = {
  title: string;
  description: string;
  priority: "low" | "medium" | "high";
};

// 交差型で全てを合成
type Task = HasId & Timestamped & TaskInfo;

// 優先度の重み
const priorityWeight: Record<Task["priority"], number> = {
  high: 3,
  medium: 2,
  low: 1,
};

// タスクデータ
const tasks: Task[] = [
  {
    id: 1, title: "ドキュメント作成", description: "APIドキュメントを書く",
    priority: "medium", createdAt: new Date(), updatedAt: new Date(),
  },
  {
    id: 2, title: "バグ修正", description: "ログイン画面のバグを修正",
    priority: "high", createdAt: new Date(), updatedAt: new Date(),
  },
  {
    id: 3, title: "リファクタリング", description: "共通処理を関数化",
    priority: "low", createdAt: new Date(), updatedAt: new Date(),
  },
];

// 優先度でソート（降順）
const sorted = [...tasks].sort(
  (a, b) => priorityWeight[b.priority] - priorityWeight[a.priority]
);

console.log("=== タスク一覧（優先度順） ===");
sorted.forEach((task) => {
  console.log(`[${task.priority.toUpperCase()}] #${task.id}: ${task.title}`);
});
```

</details>

## 問題5：APIレスポンスの判別共用体（応用）

API呼び出しの状態を判別共用体で型安全に管理してください。

1. `LoadingState`: `{ status: "loading" }`
2. `SuccessState<T>`: `{ status: "success"; data: T }`
3. `ErrorState`: `{ status: "error"; error: { code: number; message: string } }`
4. `ApiState<T>` = 上記3つのユニオン

`renderState` 関数を実装し、状態に応じたメッセージを返してください。

**期待される出力：**
```
読み込み中...
ユーザー一覧: 太郎, 花子, 次郎
エラー 404: ユーザーが見つかりません
```

<details>
<summary>解答例</summary>

```typescript
// 判別共用体でAPIの状態を表現
interface LoadingState {
  status: "loading";
}

interface SuccessState<T> {
  status: "success";
  data: T;
}

interface ErrorState {
  status: "error";
  error: { code: number; message: string };
}

type ApiState<T> = LoadingState | SuccessState<T> | ErrorState;

// 状態に応じた表示
function renderState<T>(state: ApiState<T>): string {
  switch (state.status) {
    case "loading":
      return "読み込み中...";
    case "success":
      return `データ: ${JSON.stringify(state.data)}`;
    case "error":
      return `エラー ${state.error.code}: ${state.error.message}`;
  }
}

// ユーザー名のリスト用のレンダリング
function renderUserList(state: ApiState<string[]>): string {
  switch (state.status) {
    case "loading":
      return "読み込み中...";
    case "success":
      return `ユーザー一覧: ${state.data.join(", ")}`;
    case "error":
      return `エラー ${state.error.code}: ${state.error.message}`;
  }
}

// テスト
const loading: ApiState<string[]> = { status: "loading" };
const success: ApiState<string[]> = {
  status: "success",
  data: ["太郎", "花子", "次郎"],
};
const error: ApiState<string[]> = {
  status: "error",
  error: { code: 404, message: "ユーザーが見つかりません" },
};

console.log(renderUserList(loading));
console.log(renderUserList(success));
console.log(renderUserList(error));
```

</details>

## 問題6：網羅性チェック（応用）

色のユニオン型に対して、網羅性チェック付きの `colorToJapanese` 関数を実装してください。

`type Color = "red" | "green" | "blue" | "yellow"`

全てのケースを処理し、`default` 句で `never` 型による網羅性チェックを行ってください。

**期待される出力：**
```
red → 赤
green → 緑
blue → 青
yellow → 黄
```

<details>
<summary>ヒント</summary>

- `default` 句で `const _exhaustive: never = color;` とすると、全ケースを処理していなければコンパイルエラーになります。

</details>

<details>
<summary>解答例</summary>

```typescript
type Color = "red" | "green" | "blue" | "yellow";

// 網羅性チェック用のヘルパー関数
function assertNever(value: never, message?: string): never {
  throw new Error(message ?? `Unexpected value: ${value}`);
}

function colorToJapanese(color: Color): string {
  switch (color) {
    case "red":
      return "赤";
    case "green":
      return "緑";
    case "blue":
      return "青";
    case "yellow":
      return "黄";
    default:
      // 全てのケースを処理していれば、ここはnever型
      // 新しい色を追加してcase文を書き忘れるとコンパイルエラーになる
      return assertNever(color);
  }
}

const colors: Color[] = ["red", "green", "blue", "yellow"];
colors.forEach((c) => {
  console.log(`${c} → ${colorToJapanese(c)}`);
});
```

</details>

## 問題7：Result型パターン（チャレンジ）

成功と失敗を型安全に表現する `Result` 型を実装し、JSON解析と数値変換に活用してください。

1. `Result<T, E>` 型を定義（判別共用体）
2. `ok<T>(value)` ヘルパー関数
3. `err<E>(error)` ヘルパー関数
4. `parseJSON(input)` 関数：JSONの解析結果を `Result` で返す
5. `parseNumber(input)` 関数：文字列を数値に変換した結果を `Result` で返す

**期待される出力：**
```
JSON解析成功: {"name":"太郎","age":25}
JSON解析失敗: 不正なJSON形式
数値変換成功: 42
数値変換失敗: "abc" は数値に変換できません
```

<details>
<summary>解答例</summary>

```typescript
// Result型の定義
type Result<T, E = string> =
  | { ok: true; value: T }
  | { ok: false; error: E };

// ヘルパー関数
function ok<T>(value: T): Result<T, never> {
  return { ok: true, value };
}

function err<E>(error: E): Result<never, E> {
  return { ok: false, error };
}

// JSON解析
function parseJSON(input: string): Result<unknown, string> {
  try {
    return ok(JSON.parse(input));
  } catch {
    return err("不正なJSON形式");
  }
}

// 数値変換
function parseNumber(input: string): Result<number, string> {
  const num = Number(input);
  if (Number.isNaN(num)) {
    return err(`"${input}" は数値に変換できません`);
  }
  return ok(num);
}

// 結果の表示ヘルパー
function showResult<T>(label: string, result: Result<T, string>): void {
  if (result.ok) {
    console.log(`${label}成功: ${JSON.stringify(result.value)}`);
  } else {
    console.log(`${label}失敗: ${result.error}`);
  }
}

// テスト
showResult("JSON解析", parseJSON('{"name":"太郎","age":25}'));
showResult("JSON解析", parseJSON("invalid json"));
showResult("数値変換", parseNumber("42"));
showResult("数値変換", parseNumber("abc"));
```

</details>

## 問題8：複合的な型の設計（チャレンジ）

通知システムの型を設計してください。通知には3種類あります。

- `EmailNotification`: `{ type: "email"; to: string; subject: string; body: string }`
- `SmsNotification`: `{ type: "sms"; phoneNumber: string; message: string }`
- `PushNotification`: `{ type: "push"; deviceId: string; title: string; body: string }`

1. 各通知型を定義
2. `Notification` ユニオン型を定義
3. 全通知型に共通の `BaseMeta` 型（`id: string`, `createdAt: Date`, `isRead: boolean`）を交差型で追加した `NotificationWithMeta` 型を定義
4. `sendNotification` 関数：通知の種類に応じたメッセージを返す
5. `getUnreadCount` 関数：未読通知の数を返す

**期待される出力：**
```
メール送信: user@example.com - 会議のお知らせ
SMS送信: 090-1234-5678 - 認証コード: 123456
プッシュ通知: device-001 - 新着メッセージ
未読通知: 2件
```

<details>
<summary>解答例</summary>

```typescript
// 各通知型
interface EmailNotification {
  type: "email";
  to: string;
  subject: string;
  body: string;
}

interface SmsNotification {
  type: "sms";
  phoneNumber: string;
  message: string;
}

interface PushNotification {
  type: "push";
  deviceId: string;
  title: string;
  body: string;
}

// ユニオン型
type Notification = EmailNotification | SmsNotification | PushNotification;

// メタ情報
type BaseMeta = {
  id: string;
  createdAt: Date;
  isRead: boolean;
};

// メタ情報付き通知（交差型）
type NotificationWithMeta = Notification & BaseMeta;

// 通知の送信処理
function sendNotification(notification: Notification): string {
  switch (notification.type) {
    case "email":
      return `メール送信: ${notification.to} - ${notification.subject}`;
    case "sms":
      return `SMS送信: ${notification.phoneNumber} - ${notification.message}`;
    case "push":
      return `プッシュ通知: ${notification.deviceId} - ${notification.title}`;
  }
}

// 未読数のカウント
function getUnreadCount(notifications: NotificationWithMeta[]): number {
  return notifications.filter((n) => !n.isRead).length;
}

// テストデータ
const notifications: NotificationWithMeta[] = [
  {
    type: "email", to: "user@example.com", subject: "会議のお知らせ",
    body: "明日10時から会議です", id: "n-1", createdAt: new Date(), isRead: false,
  },
  {
    type: "sms", phoneNumber: "090-1234-5678", message: "認証コード: 123456",
    id: "n-2", createdAt: new Date(), isRead: true,
  },
  {
    type: "push", deviceId: "device-001", title: "新着メッセージ",
    body: "太郎さんからメッセージが届きました", id: "n-3", createdAt: new Date(), isRead: false,
  },
];

// 表示
notifications.forEach((n) => console.log(sendNotification(n)));
console.log(`未読通知: ${getUnreadCount(notifications)}件`);
```

</details>
