### 問題1：BMI計算関数

身長（cm）と体重（kg）を受け取り、BMI値と判定結果をタプルで返す関数 `calculate_bmi` を作成してください。

BMIの計算式は `体重(kg) / (身長(m))^2` です。判定基準は以下の通りです。

| BMI | 判定 |
|---|---|
| 18.5未満 | 低体重 |
| 18.5以上25.0未満 | 普通体重 |
| 25.0以上30.0未満 | 肥満(1度) |
| 30.0以上 | 肥満(2度以上) |

**期待される出力例：**
```
身長: 170.0cm, 体重: 65.0kg
BMI: 22.49 → 普通体重

身長: 155.0cm, 体重: 42.0kg
BMI: 17.48 → 低体重

身長: 180.0cm, 体重: 95.0kg
BMI: 29.32 → 肥満(1度)
```

<details><summary>ヒント</summary>

- 身長をcmからmに変換するには `height / 100.0` を使います
- 戻り値はタプル `(f64, &str)` で返せます
- BMI値の小数点以下2桁は `format!("{:.2}", bmi)` で表示できます

</details>

<details><summary>解答例</summary>

```rust
// BMI計算関数
// 身長(cm)と体重(kg)からBMIと判定を返す

fn calculate_bmi(height_cm: f64, weight_kg: f64) -> (f64, &'static str) {
    // 身長をメートルに変換
    let height_m = height_cm / 100.0;
    // BMIを計算
    let bmi = weight_kg / (height_m * height_m);

    // 判定を決定
    let category = if bmi < 18.5 {
        "低体重"
    } else if bmi < 25.0 {
        "普通体重"
    } else if bmi < 30.0 {
        "肥満(1度)"
    } else {
        "肥満(2度以上)"
    };

    (bmi, category)
}

fn main() {
    // テストデータ: (身長cm, 体重kg)
    let test_data = vec![
        (170.0, 65.0),
        (155.0, 42.0),
        (180.0, 95.0),
    ];

    for (height, weight) in test_data {
        let (bmi, category) = calculate_bmi(height, weight);
        println!("身長: {:.1}cm, 体重: {:.1}kg", height, weight);
        println!("BMI: {:.2} → {}\n", bmi, category);
    }
}
```

**補足：** Rustでは関数の戻り値としてタプルを使うことで、複数の値を一度に返すことができます。`&'static str` は文字列リテラルの型で、プログラムの全期間にわたって有効な参照です。

</details>

### 問題2：参照渡しでベクタを操作する

整数のベクタを**可変参照**で受け取り、すべての要素を2倍にする関数 `double_elements` を作成してください。また、ベクタを**不変参照**で受け取り、合計値を返す関数 `sum_elements` も作成してください。

**期待される出力例：**
```
変更前: [1, 2, 3, 4, 5]
変更後: [2, 4, 6, 8, 10]
合計: 30
```

<details><summary>ヒント</summary>

- 可変参照は `&mut Vec<i32>` で受け取ります
- 不変参照は `&Vec<i32>` または `&[i32]`（スライス）で受け取ります
- ベクタの要素を変更するには `for elem in vec.iter_mut()` を使います

</details>

<details><summary>解答例</summary>

```rust
// 参照渡しでベクタを操作する関数

// 可変参照で受け取り、全要素を2倍にする
fn double_elements(vec: &mut Vec<i32>) {
    for elem in vec.iter_mut() {
        *elem *= 2; // デリファレンスして値を更新
    }
}

// 不変参照（スライス）で受け取り、合計を返す
fn sum_elements(slice: &[i32]) -> i32 {
    let mut total = 0;
    for &elem in slice {
        total += elem;
    }
    total
}

fn main() {
    let mut numbers = vec![1, 2, 3, 4, 5];
    println!("変更前: {:?}", numbers);

    // 可変参照を渡して要素を2倍にする
    double_elements(&mut numbers);
    println!("変更後: {:?}", numbers);

    // 不変参照を渡して合計を求める
    let total = sum_elements(&numbers);
    println!("合計: {}", total);
}
```

**補足：** Rustの所有権システムでは、可変参照（`&mut`）は同時に1つしか存在できません。関数に `&mut` を渡している間は、他の参照を作ることはできません。スライス `&[i32]` は `&Vec<i32>` よりも汎用的で、配列やベクタの一部にも対応できるため推奨されます。

</details>

### 問題3：複数の戻り値を返す関数

文字列のベクタを受け取り、以下の統計情報をタプルで返す関数 `string_stats` を作成してください。

- 最も短い文字列
- 最も長い文字列
- 平均文字数（f64）

**期待される出力例：**
```
入力: ["Rust", "Go", "JavaScript", "Python", "C"]
最短: "C" (1文字)
最長: "JavaScript" (10文字)
平均文字数: 4.60
```

<details><summary>ヒント</summary>

- 文字数は `.len()` で取得できます（バイト数ですが、ASCII文字列なら文字数と同じです）
- 戻り値はタプル `(&str, &str, f64)` で返せます
- 平均は `合計 as f64 / 個数 as f64` で計算します

</details>

<details><summary>解答例</summary>

```rust
// 文字列ベクタの統計情報を返す関数

fn string_stats(words: &[&str]) -> (&str, &str, f64) {
    // 最初の要素で初期化
    let mut shortest = words[0];
    let mut longest = words[0];
    let mut total_len = 0;

    for &word in words {
        // 最短の文字列を更新
        if word.len() < shortest.len() {
            shortest = word;
        }
        // 最長の文字列を更新
        if word.len() > longest.len() {
            longest = word;
        }
        // 文字数の合計を計算
        total_len += word.len();
    }

    // 平均を計算
    let average = total_len as f64 / words.len() as f64;

    (shortest, longest, average)
}

fn main() {
    let words = vec!["Rust", "Go", "JavaScript", "Python", "C"];
    println!("入力: {:?}", words);

    let (shortest, longest, avg) = string_stats(&words);
    println!("最短: \"{}\" ({}文字)", shortest, shortest.len());
    println!("最長: \"{}\" ({}文字)", longest, longest.len());
    println!("平均文字数: {:.2}", avg);
}
```

**補足：** 入力のスライス `&[&str]` からの参照を返す場合、Rustのライフタイム省略規則により、入力と同じライフタイムが自動的に推論されます。`Vec<&str>` は関数呼び出し時に自動的にスライス `&[&str]` に変換（Derefによる型強制）されます。

</details>

### 問題4：高階関数 - 数値変換器

整数のベクタと変換関数を受け取り、変換後のベクタを返す高階関数 `transform` を作成してください。この関数を使って以下の3つの変換を行ってください。

1. すべての要素を2乗する
2. すべての要素を文字列に変換する
3. 偶数のみを抽出して3倍にする

**期待される出力例：**
```
元のデータ: [1, 2, 3, 4, 5]

2乗: [1, 4, 9, 16, 25]
文字列化: ["1", "2", "3", "4", "5"]
偶数を3倍: [6, 12]
```

<details><summary>ヒント</summary>

- 高階関数は `fn transform<T>(data: &[i32], f: impl Fn(i32) -> T) -> Vec<T>` のように定義できます
- クロージャは `|x| x * x` のように書きます
- 偶数の抽出にはイテレータの `.filter()` メソッドが便利です
- `impl Fn(i32) -> T` はクロージャを受け取るジェネリックな書き方です

</details>

<details><summary>解答例</summary>

```rust
// 高階関数による数値変換

// 各要素に関数fを適用して新しいベクタを返す高階関数
fn transform<T>(data: &[i32], f: impl Fn(i32) -> T) -> Vec<T> {
    data.iter().map(|&x| f(x)).collect()
}

// フィルタと変換を組み合わせた高階関数
fn filter_and_transform(
    data: &[i32],
    predicate: impl Fn(i32) -> bool,
    transformer: impl Fn(i32) -> i32,
) -> Vec<i32> {
    data.iter()
        .filter(|&&x| predicate(x))
        .map(|&x| transformer(x))
        .collect()
}

fn main() {
    let numbers = vec![1, 2, 3, 4, 5];
    println!("元のデータ: {:?}\n", numbers);

    // 1. すべての要素を2乗
    let squared = transform(&numbers, |x| x * x);
    println!("2乗: {:?}", squared);

    // 2. すべての要素を文字列に変換
    let strings = transform(&numbers, |x| x.to_string());
    println!("文字列化: {:?}", strings);

    // 3. 偶数のみを抽出して3倍
    let filtered = filter_and_transform(
        &numbers,
        |x| x % 2 == 0,  // 偶数判定
        |x| x * 3,        // 3倍にする
    );
    println!("偶数を3倍: {:?}", filtered);
}
```

**補足：** `impl Fn(i32) -> T` は「`i32` を受け取って `T` を返すクロージャ」を意味します。Rustのクロージャには `Fn`、`FnMut`、`FnOnce` の3種類のトレイトがあります。`Fn` は環境を不変参照でキャプチャするクロージャで、最も制約が厳しい代わりに何度でも呼び出せます。

</details>

### 問題5：クロージャで状態を持つカウンター

クロージャを返す関数 `make_counter` を作成してください。この関数は初期値を受け取り、呼び出すたびに値をインクリメントして返すクロージャを返します。さらに、ステップ値を指定できる `make_step_counter` も作成してください。

**期待される出力例：**
```
カウンター1: 1
カウンター1: 2
カウンター1: 3

カウンター2（ステップ5）: 5
カウンター2（ステップ5）: 10
カウンター2（ステップ5）: 15

カウンター1の続き: 4
```

<details><summary>ヒント</summary>

- クロージャを返すには `impl FnMut() -> i32` を戻り値の型にします
- `move` キーワードを使うと、クロージャが変数の所有権を奪います
- クロージャ内で変数を変更するには `FnMut` が必要です

</details>

<details><summary>解答例</summary>

```rust
// クロージャを返す関数（カウンター）

// 1ずつインクリメントするカウンターを返す
fn make_counter(start: i32) -> impl FnMut() -> i32 {
    let mut count = start;
    // moveでcountの所有権をクロージャに移動
    move || {
        count += 1;
        count
    }
}

// 指定ステップでインクリメントするカウンターを返す
fn make_step_counter(start: i32, step: i32) -> impl FnMut() -> i32 {
    let mut count = start;
    move || {
        count += step;
        count
    }
}

fn main() {
    // カウンター1: 0から1ずつ
    let mut counter1 = make_counter(0);
    println!("カウンター1: {}", counter1());
    println!("カウンター1: {}", counter1());
    println!("カウンター1: {}", counter1());
    println!();

    // カウンター2: 0から5ずつ
    let mut counter2 = make_step_counter(0, 5);
    println!("カウンター2（ステップ5）: {}", counter2());
    println!("カウンター2（ステップ5）: {}", counter2());
    println!("カウンター2（ステップ5）: {}", counter2());
    println!();

    // カウンター1は独立して状態を保持している
    println!("カウンター1の続き: {}", counter1());
}
```

**補足：** `move` クロージャは外部の変数の所有権を奪い取ります。これにより、クロージャが関数のスコープを超えて生存しても安全に動作します。`impl FnMut() -> i32` は「可変な状態を持ち、`i32` を返すクロージャ」を意味する戻り値型です。各カウンターは独立した状態を持つため、互いに影響しません。

</details>

### 問題6：関数ポインタとクロージャの使い分け

以下の要件を満たす簡易計算機プログラムを作成してください。

1. 四則演算を行う関数を4つ定義する（`add`, `sub`, `mul`, `div`）
2. 演算子の文字列から対応する関数を返す関数 `get_operation` を作成する
3. 計算履歴を保存するクロージャ `make_calculator` を作成する

**期待される出力例：**
```
10 + 3 = 13
10 - 3 = 7
10 * 3 = 30
10 / 3 = 3.33

--- 計算履歴付き電卓 ---
計算: 100 + 50 = 150 (累計: 150)
計算: 150 * 2 = 300 (累計: 300)
計算: 300 - 80 = 220 (累計: 220)
```

<details><summary>ヒント</summary>

- 関数ポインタの型は `fn(f64, f64) -> f64` です
- `get_operation` は `Option<fn(f64, f64) -> f64>` を返すようにします
- 累計を保持するクロージャは `FnMut` で状態を管理します

</details>

<details><summary>解答例</summary>

```rust
// 簡易計算機 - 関数ポインタとクロージャの活用

// 四則演算の関数
fn add(a: f64, b: f64) -> f64 { a + b }
fn sub(a: f64, b: f64) -> f64 { a - b }
fn mul(a: f64, b: f64) -> f64 { a * b }
fn div(a: f64, b: f64) -> f64 { a / b }

// 演算子文字列から対応する関数ポインタを返す
fn get_operation(op: &str) -> Option<fn(f64, f64) -> f64> {
    match op {
        "+" => Some(add),
        "-" => Some(sub),
        "*" => Some(mul),
        "/" => Some(div),
        _ => None,
    }
}

// 計算履歴を持つ電卓クロージャを返す
fn make_calculator() -> impl FnMut(f64, &str, f64) -> f64 {
    let mut accumulator = 0.0;
    let mut is_first = true;

    move |a: f64, op: &str, b: f64| -> f64 {
        if let Some(operation) = get_operation(op) {
            let result = operation(a, b);
            if is_first {
                accumulator = result;
                is_first = false;
            } else {
                accumulator = result;
            }
            println!(
                "計算: {} {} {} = {} (累計: {})",
                a, op, b, result, accumulator
            );
            result
        } else {
            println!("不明な演算子: {}", op);
            a
        }
    }
}

fn main() {
    // 関数ポインタによる基本計算
    let operations = vec!["+", "-", "*", "/"];
    let a = 10.0;
    let b = 3.0;

    for op in &operations {
        if let Some(func) = get_operation(op) {
            let result = func(a, b);
            println!("{} {} {} = {:.2}", a as i32, op, b as i32, result);
        }
    }

    println!("\n--- 計算履歴付き電卓 ---");

    // クロージャを使った電卓
    let mut calc = make_calculator();
    let result1 = calc(100.0, "+", 50.0);
    let result2 = calc(result1, "*", 2.0);
    let _result3 = calc(result2, "-", 80.0);
}
```

**補足：** 関数ポインタ `fn(f64, f64) -> f64` とクロージャ `impl Fn(f64, f64) -> f64` は似ていますが、関数ポインタは状態を持てません。状態を保持したい場合はクロージャを使います。`get_operation` のように、実行時に関数を選択するパターンは、コマンドパターンやストラテジーパターンの簡易版として実務でもよく使われます。

</details>

### 問題7：クロージャとイテレータで成績処理システム

学生の名前とテストの点数リストを管理し、さまざまな統計処理を行うプログラムを作成してください。

以下の関数・クロージャを実装してください。

1. `Student` 構造体（名前と点数リスト）
2. 平均点を計算する関数
3. 条件に合う学生をフィルタリングする高階関数
4. 成績ランクを付与する関数（A: 90以上、B: 80以上、C: 70以上、D: 60以上、F: 60未満）
5. クロージャを使った並べ替え

**期待される出力例：**
```
=== 学生成績一覧 ===
田中太郎: 平均 85.0点 (ランク: B)
山田花子: 平均 92.3点 (ランク: A)
佐藤一郎: 平均 67.7点 (ランク: D)
鈴木美咲: 平均 78.3点 (ランク: C)
高橋健一: 平均 55.0点 (ランク: F)

=== 平均80点以上の学生 ===
田中太郎: 85.0点
山田花子: 92.3点

=== 平均点の高い順 ===
1位: 山田花子 (92.3点)
2位: 田中太郎 (85.0点)
3位: 鈴木美咲 (78.3点)
4位: 佐藤一郎 (67.7点)
5位: 高橋健一 (55.0点)

=== クラス統計 ===
クラス平均: 75.7点
最高点: 山田花子 (92.3点)
最低点: 高橋健一 (55.0点)
```

<details><summary>ヒント</summary>

- 構造体には `#[derive(Clone)]` を付けるとクローンが使えて便利です
- イテレータの `.map()`, `.filter()`, `.fold()`, `.enumerate()` を活用しましょう
- ソートには `.sort_by()` と `f64` の比較に `.partial_cmp()` を使います
- 高階関数のフィルタは `impl Fn(&Student) -> bool` で条件を受け取ります

</details>

<details><summary>解答例</summary>

```rust
// 学生成績処理システム
// クロージャとイテレータを組み合わせた実践的な例

#[derive(Clone)]
struct Student {
    name: String,
    scores: Vec<f64>,
}

impl Student {
    // コンストラクタ
    fn new(name: &str, scores: Vec<f64>) -> Self {
        Student {
            name: name.to_string(),
            scores,
        }
    }

    // 平均点を計算
    fn average(&self) -> f64 {
        if self.scores.is_empty() {
            return 0.0;
        }
        let sum: f64 = self.scores.iter().sum();
        // 小数点以下1桁に丸める
        (sum / self.scores.len() as f64 * 10.0).round() / 10.0
    }
}

// 成績ランクを返す関数
fn get_rank(average: f64) -> &'static str {
    if average >= 90.0 {
        "A"
    } else if average >= 80.0 {
        "B"
    } else if average >= 70.0 {
        "C"
    } else if average >= 60.0 {
        "D"
    } else {
        "F"
    }
}

// 条件に合う学生をフィルタリングする高階関数
fn filter_students(
    students: &[Student],
    predicate: impl Fn(&Student) -> bool,
) -> Vec<&Student> {
    students.iter().filter(|s| predicate(s)).collect()
}

// 平均点で並べ替えた学生リストを返す（高い順）
fn sort_by_average(students: &[Student]) -> Vec<Student> {
    let mut sorted = students.to_vec();
    sorted.sort_by(|a, b| {
        b.average()
            .partial_cmp(&a.average())
            .unwrap_or(std::cmp::Ordering::Equal)
    });
    sorted
}

fn main() {
    // テストデータの作成
    let students = vec![
        Student::new("田中太郎", vec![80.0, 85.0, 90.0]),
        Student::new("山田花子", vec![95.0, 88.0, 94.0]),
        Student::new("佐藤一郎", vec![70.0, 65.0, 68.0]),
        Student::new("鈴木美咲", vec![75.0, 82.0, 78.0]),
        Student::new("高橋健一", vec![50.0, 55.0, 60.0]),
    ];

    // === 学生成績一覧 ===
    println!("=== 学生成績一覧 ===");
    for student in &students {
        let avg = student.average();
        let rank = get_rank(avg);
        println!("{}: 平均 {:.1}点 (ランク: {})", student.name, avg, rank);
    }
    println!();

    // === 平均80点以上の学生 ===
    println!("=== 平均80点以上の学生 ===");
    let high_scorers = filter_students(&students, |s| s.average() >= 80.0);
    for student in &high_scorers {
        println!("{}: {:.1}点", student.name, student.average());
    }
    println!();

    // === 平均点の高い順 ===
    println!("=== 平均点の高い順 ===");
    let sorted = sort_by_average(&students);
    for (i, student) in sorted.iter().enumerate() {
        println!(
            "{}位: {} ({:.1}点)",
            i + 1,
            student.name,
            student.average()
        );
    }
    println!();

    // === クラス統計 ===
    println!("=== クラス統計 ===");

    // クラス平均をイテレータで計算
    let class_avg: f64 = students.iter().map(|s| s.average()).sum::<f64>()
        / students.len() as f64;
    println!("クラス平均: {:.1}点", class_avg);

    // 最高点の学生をクロージャで検索
    let best = students
        .iter()
        .max_by(|a, b| {
            a.average()
                .partial_cmp(&b.average())
                .unwrap_or(std::cmp::Ordering::Equal)
        })
        .unwrap();
    println!("最高点: {} ({:.1}点)", best.name, best.average());

    // 最低点の学生をクロージャで検索
    let worst = students
        .iter()
        .min_by(|a, b| {
            a.average()
                .partial_cmp(&b.average())
                .unwrap_or(std::cmp::Ordering::Equal)
        })
        .unwrap();
    println!("最低点: {} ({:.1}点)", worst.name, worst.average());
}
```

**補足：** この問題では、Rustのイテレータアダプタ（`map`, `filter`, `sum`, `max_by`, `min_by`）とクロージャを組み合わせた実践的なデータ処理を体験できます。`partial_cmp` は `f64` 型の比較に使われます。`f64` は `NaN` を含む可能性があるため、通常の `cmp` ではなく `partial_cmp` を使う必要があります。

実務では、このようなデータ変換処理をイテレータチェーンで書くことが多く、宣言的で読みやすいコードになります。

</details>

### 問題8：関数合成パイプライン

複数の変換関数を順番に適用する「パイプライン」を構築するプログラムを作成してください。

以下の機能を実装してください。

1. 文字列を受け取って文字列を返すクロージャを複数定義する
2. 複数のクロージャを順番に適用する `pipeline` 関数を作成する
3. パイプラインを使って文字列の加工を行う

変換処理：
- 前後の空白を除去
- 小文字に変換
- 特定の単語を置換
- 文字数を追記

**期待される出力例：**
```
=== 文字列変換パイプライン ===

入力: "  Hello, World!  "
処理1 (トリム): "Hello, World!"
処理2 (小文字化): "hello, world!"
処理3 (置換 world→rust): "hello, rust!"
処理4 (文字数追記): "hello, rust! [12文字]"

=== パイプライン一括処理 ===
"  Hello, World!  " → "hello, rust! [12文字]"
"  Goodbye, World!  " → "goodbye, rust! [13文字]"
"  Hey World  " → "hey rust [8文字]"
```

<details><summary>ヒント</summary>

- `Box<dyn Fn(String) -> String>` でクロージャをヒープに格納できます
- パイプライン関数は `Vec<Box<dyn Fn(String) -> String>>` を受け取ります
- `.fold()` を使うと、初期値に対して関数を順番に適用できます
- 文字列の置換は `.replace()` メソッドを使います

</details>

<details><summary>解答例</summary>

```rust
// 関数合成パイプライン
// 複数のクロージャを順番に適用する仕組み

// パイプライン: 複数の変換関数を順番に適用する
fn pipeline(input: String, transforms: &[Box<dyn Fn(String) -> String>]) -> String {
    transforms.iter().fold(input, |acc, f| f(acc))
}

// パイプラインの各ステップを表示しながら実行する
fn pipeline_verbose(
    input: String,
    transforms: &[(String, Box<dyn Fn(String) -> String>)],
) -> String {
    let mut result = input;
    for (name, f) in transforms {
        result = f(result);
        println!("処理{}: \"{}\"", name, result);
    }
    result
}

fn main() {
    // 変換クロージャを定義
    let trim: Box<dyn Fn(String) -> String> = Box::new(|s| s.trim().to_string());

    let to_lower: Box<dyn Fn(String) -> String> =
        Box::new(|s| s.to_lowercase());

    let replace_world: Box<dyn Fn(String) -> String> =
        Box::new(|s| s.replace("world", "rust"));

    let add_length: Box<dyn Fn(String) -> String> = Box::new(|s| {
        let len = s.len();
        format!("{} [{}文字]", s, len)
    });

    // === 詳細表示付きパイプライン ===
    println!("=== 文字列変換パイプライン ===\n");

    let input = "  Hello, World!  ".to_string();
    println!("入力: \"{}\"", input);

    let named_transforms: Vec<(String, Box<dyn Fn(String) -> String>)> = vec![
        ("1 (トリム)".to_string(), Box::new(|s| s.trim().to_string())),
        ("2 (小文字化)".to_string(), Box::new(|s| s.to_lowercase())),
        (
            "3 (置換 world→rust)".to_string(),
            Box::new(|s| s.replace("world", "rust")),
        ),
        ("4 (文字数追記)".to_string(), Box::new(|s| {
            let len = s.len();
            format!("{} [{}文字]", s, len)
        })),
    ];

    pipeline_verbose(input, &named_transforms);

    // === 一括処理 ===
    println!("\n=== パイプライン一括処理 ===");

    let transforms: Vec<Box<dyn Fn(String) -> String>> = vec![
        Box::new(|s| s.trim().to_string()),
        Box::new(|s| s.to_lowercase()),
        Box::new(|s| s.replace("world", "rust")),
        Box::new(|s| {
            let len = s.len();
            format!("{} [{}文字]", s, len)
        }),
    ];

    let test_inputs = vec![
        "  Hello, World!  ",
        "  Goodbye, World!  ",
        "  Hey World  ",
    ];

    for input_str in test_inputs {
        let result = pipeline(input_str.to_string(), &transforms);
        println!("\"{}\" → \"{}\"", input_str, result);
    }
}
```

**補足：** `Box<dyn Fn(String) -> String>` はトレイトオブジェクトと呼ばれ、異なるクロージャを同じ型として扱うことができます。`dyn` は動的ディスパッチを意味し、実行時に適切な関数が呼び出されます。`fold` はイテレータの強力なメソッドで、累積的な処理を簡潔に書けます。

関数合成パイプラインは、データ変換処理やミドルウェアの実装で広く使われるパターンです。各ステップを独立した関数として定義することで、テストしやすく再利用可能なコードになります。

</details>
