## 1.1 ネットワークとは

ネットワークとは、複数のコンピュータやデバイスを相互に接続し、データを共有・通信するための仕組みである。

### ネットワークの分類

| 分類 | 範囲 | 例 |
|------|------|-----|
| **PAN** (Personal Area Network) | 数メートル | Bluetooth接続 |
| **LAN** (Local Area Network) | 建物内～敷地内 | 社内ネットワーク |
| **MAN** (Metropolitan Area Network) | 都市規模 | 市内光ファイバ網 |
| **WAN** (Wide Area Network) | 国・大陸規模 | インターネット |

---

## 1.2 LAN（Local Area Network）

### LANの特徴
- **限られた範囲**（オフィス、ビル、キャンパス等）のネットワーク
- **高速通信**が可能（1Gbps～100Gbps）
- **管理者が自由に構築・管理**できる
- 通信コストが低い

### LANの構成要素

```
[PC-A]---+
          |
[PC-B]---+---[スイッチ]---[ルータ]===WAN===
          |
[PC-C]---+
          |
[サーバ]--+
```

| 機器 | 役割 |
|------|------|
| NIC（ネットワークインターフェースカード） | PCをネットワークに接続 |
| スイッチ（L2） | 同一LAN内のフレーム転送 |
| ルータ（L3） | 異なるネットワーク間のパケット転送 |
| ケーブル（UTP/光ファイバ） | 物理的な接続媒体 |

---

## 1.3 WAN（Wide Area Network）

### WANの特徴
- **広域**（都市間、国家間）のネットワーク
- 通信事業者（キャリア）の回線を利用
- LANに比べて**低速・高コスト**
- インターネットは世界最大のWAN

### WAN接続方式

| 方式 | 特徴 | 用途 |
|------|------|------|
| **専用線** | 帯域保証、高セキュリティ、高コスト | 基幹系 |
| **IP-VPN** | MPLS利用、閉域網 | 拠点間接続 |
| **広域イーサネット** | L2接続、柔軟性高い | 拠点間接続 |
| **インターネットVPN** | 低コスト、暗号化必須 | 中小企業 |
| **フレッツ等のブロードバンド** | 低コスト、ベストエフォート | 小規模拠点 |

### WANの典型的構成

```
  [本社LAN]                              [支社LAN]
      |                                      |
  [ルータ]---[WAN回線/インターネット]---[ルータ]
      |                                      |
  [スイッチ]                            [スイッチ]
   /  |  \                              /  |  \
 PC  PC  PC                           PC  PC  PC
```

## 1.4 ネットワークトポロジ

トポロジとは、ネットワークの接続形態（構成の形）のことである。

### 物理トポロジ

```
【バス型】                    【スター型】
                                    [HUB]
 PC--+--PC--+--PC--+--PC        /  / | \  \
     |      |      |          PC PC PC PC  PC
  (終端)          (終端)

【リング型】                  【メッシュ型（フルメッシュ）】
    PC---PC                    PC----PC
   /       \                   |\ /\ /|
  PC       PC                  | X  X |
   \       /                   |/ \/ \|
    PC---PC                    PC----PC

【ツリー型】                  【スター拡張型（実際の企業LAN）】
      [コア]                       [コアSW]
     /     \                      /        \
  [分配]   [分配]            [分配SW]    [分配SW]
  / \      / \               / \          / \
PC  PC   PC  PC          [アクセスSW] [アクセスSW]
                           / \          / \
                         PC  PC      PC  PC
```

### 論理トポロジ
物理的な配線形態とは別に、データがどのように流れるかを示すのが論理トポロジである。
- 例：物理的にはスター型だが、論理的にはバス型（イーサネット）

## 1.5 OSI参照モデル（7層モデル）

### 概要
**OSI参照モデル**（Open Systems Interconnection Reference Model）は、国際標準化機構（ISO）が策定した通信機能の階層モデルである。通信を7つの層（レイヤ）に分割し、各層の役割を明確にしている。

### 全7層の詳細

```
+---------------------------+
| 第7層 アプリケーション層   | ← ユーザに近い
+---------------------------+
| 第6層 プレゼンテーション層 |
+---------------------------+
| 第5層 セッション層         |
+---------------------------+
| 第4層 トランスポート層     |
+---------------------------+
| 第3層 ネットワーク層       |
+---------------------------+
| 第2層 データリンク層       |
+---------------------------+
| 第1層 物理層               | ← ハードウェアに近い
+---------------------------+
```

#### 第7層：アプリケーション層（Application Layer）

| 項目 | 内容 |
|------|------|
| **役割** | ユーザアプリケーションに対するネットワークサービスの提供 |
| **PDU** | データ（メッセージ） |
| **プロトコル例** | HTTP, HTTPS, FTP, SMTP, POP3, DNS, SSH, Telnet, SNMP |
| **機器** | ロードバランサー（L7）, WAF |

- ユーザが直接操作するWebブラウザやメールソフトが利用する層
- アプリケーション間のデータ交換の規約を定義

#### 第6層：プレゼンテーション層（Presentation Layer）

| 項目 | 内容 |
|------|------|
| **役割** | データ形式の変換、暗号化/復号、圧縮/伸張 |
| **PDU** | データ |
| **プロトコル例** | SSL/TLS（暗号化）, JPEG, MPEG, ASCII, EBCDIC |
| **機能** | 文字コード変換、データ圧縮、暗号化 |

- 異なるシステム間のデータ形式の差異を吸収
- 実際のプロトコルスタックでは、アプリケーション層に統合されることが多い

#### 第5層：セッション層（Session Layer）

| 項目 | 内容 |
|------|------|
| **役割** | 通信の開始・維持・終了（セッション管理） |
| **PDU** | データ |
| **プロトコル例** | NetBIOS, RPC, SQL |
| **機能** | セッション確立/解放、同期、チェックポイント |

- 通信の論理的な「会話」を管理
- 全二重/半二重通信の制御

#### 第4層：トランスポート層（Transport Layer）

| 項目 | 内容 |
|------|------|
| **役割** | エンドツーエンドの信頼性ある通信 |
| **PDU** | **セグメント**（TCP）/ **データグラム**（UDP） |
| **プロトコル例** | TCP, UDP |
| **機器** | ファイアウォール（L4） |

- ポート番号によるアプリケーション識別
- TCPによるフロー制御、エラー検出、再送制御
- UDPによる高速・軽量な通信

#### 第3層：ネットワーク層（Network Layer）

| 項目 | 内容 |
|------|------|
| **役割** | 異なるネットワーク間のルーティング（経路制御） |
| **PDU** | **パケット** |
| **プロトコル例** | IP, ICMP, ARP, OSPF, BGP, RIP |
| **機器** | **ルータ**, L3スイッチ |

- IPアドレスによる論理的なアドレッシング
- 最適経路の選択と転送
- パケットの分割（フラグメンテーション）と再構成

#### 第2層：データリンク層（Data Link Layer）

| 項目 | 内容 |
|------|------|
| **役割** | 隣接ノード間の信頼性ある通信 |
| **PDU** | **フレーム** |
| **プロトコル例** | Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11), PPP |
| **機器** | **スイッチ（L2）**, ブリッジ |

- MACアドレスによる物理的なアドレッシング
- フレームの生成と解析
- エラー検出（FCS/CRC）
- メディアアクセス制御（CSMA/CD等）

#### 第1層：物理層（Physical Layer）

| 項目 | 内容 |
|------|------|
| **役割** | ビット列の電気信号/光信号への変換と伝送 |
| **PDU** | **ビット** |
| **規格例** | RS-232C, IEEE 802.3 (物理仕様), DSL |
| **機器** | **リピータ**, ハブ, ケーブル, コネクタ |

- 電気的・機械的・手続き的な仕様を定義
- ケーブルの種類（UTP, STP, 光ファイバ）
- コネクタの形状（RJ-45, LC, SC）
- 伝送速度、符号化方式

### 覚え方（語呂合わせ）
**「ア（7）プ（6）セ（5）ト（4）ネ（3）デ（2）ブ（1）」**
→ アプリケーション・プレゼンテーション・セッション・トランスポート・ネットワーク・データリンク・物理

## 1.6 TCP/IPモデル（4層モデル）

### OSI参照モデルとの対応

```
  OSI参照モデル              TCP/IPモデル
+-------------------+    +-------------------+
| アプリケーション層 |    |                   |
+-------------------+    |  アプリケーション層 |
| プレゼンテーション |    |  (HTTP,DNS,SMTP   |
+-------------------+    |   FTP,SSH等)       |
| セッション層       |    |                   |
+-------------------+    +-------------------+
| トランスポート層   |    | トランスポート層   |
|                   |    | (TCP, UDP)        |
+-------------------+    +-------------------+
| ネットワーク層     |    | インターネット層   |
|                   |    | (IP, ICMP, ARP)   |
+-------------------+    +-------------------+
| データリンク層     |    |                   |
+-------------------+    | ネットワーク       |
| 物理層             |    | インターフェース層 |
+-------------------+    | (Ethernet等)      |
                         +-------------------+
```

### TCP/IPモデルの各層

| TCP/IP層 | 対応するOSI層 | 主なプロトコル |
|----------|--------------|---------------|
| アプリケーション層 | 第5～7層 | HTTP, DNS, SMTP, FTP, SSH |
| トランスポート層 | 第4層 | TCP, UDP |
| インターネット層 | 第3層 | IP, ICMP, ARP |
| ネットワークインターフェース層 | 第1～2層 | Ethernet, Wi-Fi, PPP |

## 1.7 プロトコルとは

### 定義
**プロトコル**とは、通信を行う際の規約（ルール）のことである。通信相手とデータを正しくやり取りするために、データ形式、通信手順、エラー処理方法などを事前に取り決めておく必要がある。

### プロトコルの3要素

| 要素 | 説明 | 例 |
|------|------|-----|
| **構文（Syntax）** | データの形式・構造 | ヘッダのフォーマット |
| **意味（Semantics）** | 各フィールドの意味 | フラグの意味 |
| **タイミング（Timing）** | 送受信のタイミング・順序 | ハンドシェイクの手順 |

### 主要プロトコル一覧

| プロトコル | 層 | ポート | 用途 |
|-----------|-----|--------|------|
| HTTP | L7 | 80 | Web閲覧 |
| HTTPS | L7 | 443 | 暗号化Web閲覧 |
| DNS | L7 | 53 | 名前解決 |
| SMTP | L7 | 25 | メール送信 |
| POP3 | L7 | 110 | メール受信 |
| IMAP | L7 | 143 | メール受信 |
| FTP | L7 | 20/21 | ファイル転送 |
| SSH | L7 | 22 | 暗号化リモート接続 |
| DHCP | L7 | 67/68 | IPアドレス自動割当 |
| NTP | L7 | 123 | 時刻同期 |
| TCP | L4 | - | 信頼性ある通信 |
| UDP | L4 | - | 高速な通信 |
| IP | L3 | - | パケット配送 |
| ICMP | L3 | - | 通信状態通知 |
| ARP | L2/L3 | - | MACアドレス解決 |

## 1.8 カプセル化と脱カプセル化

### カプセル化（Encapsulation）

送信側では、上位層から下位層へデータが渡される際に、各層のヘッダ（制御情報）が順に付加される。この過程を**カプセル化**という。

```
送信側の処理フロー：

  アプリケーション層 : [     データ     ]
                              ↓
  トランスポート層   : [TCPヘッダ][   データ   ]  ← セグメント
                              ↓
  ネットワーク層     : [IPヘッダ][TCPヘッダ][データ]  ← パケット
                              ↓
  データリンク層     : [MACヘッダ][IPヘッダ][TCPヘッダ][データ][FCS]  ← フレーム
                              ↓
  物理層             :  01101001110100101... ← ビット列（電気/光信号）
```

### 脱カプセル化（Decapsulation）

受信側では、下位層から上位層へデータが渡される際に、各層のヘッダが順に取り除かれる。この過程を**脱カプセル化**という。

```
受信側の処理フロー：

  物理層             :  01101001110100101...
                              ↓
  データリンク層     : [MACヘッダ][IPヘッダ][TCPヘッダ][データ][FCS]
                       ↑ MACヘッダを確認・除去        FCS確認 ↑
                              ↓
  ネットワーク層     : [IPヘッダ][TCPヘッダ][データ]
                       ↑ IPヘッダを確認・除去
                              ↓
  トランスポート層   : [TCPヘッダ][データ]
                       ↑ TCPヘッダを確認・除去
                              ↓
  アプリケーション層 : [     データ     ]
```

### PDU（Protocol Data Unit）の名称

| 層 | PDU名称 |
|----|---------|
| アプリケーション層 | メッセージ / データ |
| トランスポート層 | **セグメント**（TCP）/ **データグラム**（UDP） |
| ネットワーク層 | **パケット** |
| データリンク層 | **フレーム** |
| 物理層 | **ビット** |

### 通信全体の流れ

```
[送信PC]                                            [受信PC]
  |                                                    |
  | アプリ層：HTTPリクエスト作成                         |
  | トランスポート層：TCPヘッダ付加(ポート80)            |
  | ネットワーク層：IPヘッダ付加(宛先IP)                 |
  | データリンク層：MACヘッダ付加(次ホップMAC)           |
  | 物理層：電気信号に変換                               |
  |                                                    |
  |==========[スイッチ]=====[ルータ]======[スイッチ]====|
  |                          |                          |
  |              L2:MACヘッダ確認・書換                  |
  |              L3:IPヘッダ確認・ルーティング            |
  |              L2:新MACヘッダ付加                      |
  |                                                    |
  |                              物理層：信号受信        |
  |                              データリンク層：MAC確認  |
  |                              ネットワーク層：IP確認   |
  |                              トランスポート層：TCP確認|
  |                              アプリ層：データ取得     |
```

## 1.9 まとめ

### 重要ポイント
1. **ネットワークの基本分類**：PAN → LAN → MAN → WAN（範囲の拡大順）
2. **OSI参照モデル**は通信を7層に分割し、各層の責務を明確化
3. **TCP/IPモデル**は実際のインターネットで使用される4層モデル
4. **プロトコル**は通信の規約であり、構文・意味・タイミングの3要素で構成
5. **カプセル化**：送信時に各層のヘッダを付加
6. **脱カプセル化**：受信時に各層のヘッダを除去

### CCNA/ネスペ試験対策ポイント
- OSI参照モデルの各層の役割と対応プロトコルは**暗記必須**
- PDU名称（ビット→フレーム→パケット→セグメント→データ）の使い分け
- カプセル化の過程で何が付加されるかを正確に説明できること
- TCP/IPモデルとOSI参照モデルの対応関係
