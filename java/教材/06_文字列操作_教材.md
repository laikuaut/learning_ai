## 6.1 Stringクラスの基礎

### 文字列の生成

Javaでは文字列（String）はオブジェクトです。文字列リテラル（literal）を使う方法と、`new` キーワードを使う方法があります。

```java
public class StringBasics {
    public static void main(String[] args) {
        // 文字列リテラルで生成（推奨）
        String greeting = "こんにちは";
        String name = "太郎";

        // new キーワードで生成（通常は不要）
        String message = new String("Hello");

        System.out.println(greeting);  // こんにちは
        System.out.println(name);      // 太郎
        System.out.println(message);   // Hello
    }
}
```

### 文字列の不変性（Immutability）

Javaの `String` は不変（immutable）です。一度作成した文字列の中身は変更できません。文字列を操作するメソッドは、すべて**新しい文字列を返します**。

```
┌─────────────────────────────────────────────────┐
│  String s = "Hello";                            │
│                                                 │
│  メモリ上:                                       │
│  s ──→ [ "Hello" ]  ← この中身は変わらない       │
│                                                 │
│  s = s + " World";                              │
│                                                 │
│  s ──→ [ "Hello World" ]  ← 新しいオブジェクト    │
│         [ "Hello" ]       ← 古いものは残る(GC対象)│
└─────────────────────────────────────────────────┘
```

```java
public class StringImmutability {
    public static void main(String[] args) {
        String original = "Hello";
        String modified = original.toUpperCase();

        System.out.println(original);  // Hello（元の文字列は変わらない）
        System.out.println(modified);  // HELLO（新しい文字列が返される）

        // 文字列結合も新しいオブジェクトを生成する
        String a = "Java";
        String b = a;
        a = a + " Programming";

        System.out.println(a);  // Java Programming
        System.out.println(b);  // Java（bは影響を受けない）
    }
}
```

### 文字列プール（String Pool）

Javaには文字列プール（String Pool）という仕組みがあり、同じ内容のリテラル文字列はメモリ上で共有されます。

```java
public class StringPool {
    public static void main(String[] args) {
        String s1 = "Hello";
        String s2 = "Hello";
        String s3 = new String("Hello");

        // リテラル同士は同じオブジェクトを参照
        System.out.println(s1 == s2);      // true（同じ参照）
        // new で作ると別オブジェクト
        System.out.println(s1 == s3);      // false（別の参照）
        // 内容の比較は equals を使う
        System.out.println(s1.equals(s3)); // true（内容は同じ）
    }
}
```

### ポイントまとめ

- `String` は不変オブジェクトであり、操作すると新しい文字列が生成される
- 文字列リテラルはプールで共有される
- 文字列の比較には `==` ではなく `equals` を使う

## 6.2 主要な文字列メソッド

Stringクラスには多数の便利なメソッドが用意されています。

### 基本メソッド一覧

| メソッド | 説明 | 使用例 | 結果 |
|---|---|---|---|
| `length()` | 文字数を返す | `"Hello".length()` | `5` |
| `charAt(i)` | i番目の文字を返す | `"Hello".charAt(0)` | `'H'` |
| `substring(a, b)` | a〜b-1の部分文字列 | `"Hello".substring(1, 4)` | `"ell"` |
| `indexOf(s)` | 文字列の位置を返す | `"Hello".indexOf("ll")` | `2` |
| `contains(s)` | 含むかどうか | `"Hello".contains("ell")` | `true` |
| `replace(a, b)` | 置換 | `"Hello".replace("l", "r")` | `"Herro"` |
| `split(s)` | 分割 | `"a,b,c".split(",")` | `{"a","b","c"}` |
| `trim()` | 前後の空白除去 | `" Hi ".trim()` | `"Hi"` |
| `toUpperCase()` | 大文字変換 | `"hello".toUpperCase()` | `"HELLO"` |
| `toLowerCase()` | 小文字変換 | `"HELLO".toLowerCase()` | `"hello"` |

### 実践コード

```java
public class StringMethods {
    public static void main(String[] args) {
        String text = "  Java Programming Language  ";

        // length(): 文字数（空白含む）
        System.out.println("文字数: " + text.length());  // 29

        // trim(): 前後の空白除去
        String trimmed = text.trim();
        System.out.println("trim後: [" + trimmed + "]");
        // trim後: [Java Programming Language]

        // charAt(): 指定位置の文字を取得（0始まり）
        System.out.println("3番目の文字: " + trimmed.charAt(3));  // a

        // substring(): 部分文字列
        System.out.println("部分文字列: " + trimmed.substring(0, 4));  // Java
        System.out.println("5文字目以降: " + trimmed.substring(5));    // Programming Language

        // indexOf(): 検索
        System.out.println("'Program'の位置: " + trimmed.indexOf("Program"));  // 5
        System.out.println("'Python'の位置: " + trimmed.indexOf("Python"));    // -1（見つからない）

        // contains(): 含むかどうか
        System.out.println("'Java'を含む: " + trimmed.contains("Java"));  // true

        // replace(): 置換
        System.out.println("置換: " + trimmed.replace("Java", "Python"));
        // 置換: Python Programming Language

        // toUpperCase() / toLowerCase(): 大文字・小文字変換
        System.out.println("大文字: " + trimmed.toUpperCase());
        System.out.println("小文字: " + trimmed.toLowerCase());

        // split(): 分割
        String csv = "りんご,みかん,ぶどう,バナナ";
        String[] fruits = csv.split(",");
        for (String fruit : fruits) {
            System.out.println("果物: " + fruit);
        }

        // startsWith() / endsWith(): 前方・後方一致
        String filename = "report_2024.pdf";
        System.out.println("reportで始まる: " + filename.startsWith("report"));  // true
        System.out.println(".pdfで終わる: " + filename.endsWith(".pdf"));          // true

        // isEmpty() / isBlank(): 空判定
        System.out.println("空文字列: " + "".isEmpty());       // true
        System.out.println("空白のみ: " + "   ".isBlank());    // true（Java 11以降）
    }
}
```

> **よくある間違い：** `charAt()` や `substring()` でインデックスの範囲外を指定すると `StringIndexOutOfBoundsException` が発生します。常に `length()` で範囲を確認しましょう。

### ポイントまとめ

- `length()` は文字数を返す（配列の `.length` とは異なりメソッド呼び出し）
- `indexOf()` は見つからない場合 `-1` を返す
- `split()` は正規表現を引数に取るため、`.` で分割する場合は `"\\."` とエスケープが必要
- すべてのメソッドは新しい文字列を返し、元の文字列は変更しない

## 6.3 文字列の比較

### == と equals の違い

Javaの文字列比較は初心者が最もつまずきやすいポイントです。

```
┌──────────────────────────────────────────┐
│  ==       : 参照（アドレス）が同じか比較   │
│  equals() : 内容（中身）が同じか比較       │
│                                          │
│  文字列の比較には必ず equals() を使う！     │
└──────────────────────────────────────────┘
```

```java
public class StringComparison {
    public static void main(String[] args) {
        String s1 = "Hello";
        String s2 = "Hello";
        String s3 = new String("Hello");
        String s4 = "hello";

        // == は参照の比較
        System.out.println("s1 == s2: " + (s1 == s2));        // true（プール内の同じ参照）
        System.out.println("s1 == s3: " + (s1 == s3));        // false（別のオブジェクト）

        // equals() は内容の比較
        System.out.println("s1.equals(s3): " + s1.equals(s3));  // true

        // equalsIgnoreCase(): 大文字小文字を無視して比較
        System.out.println("s1.equals(s4): " + s1.equals(s4));                  // false
        System.out.println("s1.equalsIgnoreCase(s4): " + s1.equalsIgnoreCase(s4)); // true

        // compareTo(): 辞書順の比較（ソートに便利）
        System.out.println("compareTo: " + "apple".compareTo("banana"));  // 負の値（appleが先）
        System.out.println("compareTo: " + "banana".compareTo("apple"));  // 正の値（bananaが後）
        System.out.println("compareTo: " + "apple".compareTo("apple"));   // 0（同じ）

        // null 安全な比較のテクニック
        String input = null;
        // input.equals("test")  // NullPointerException が発生！
        System.out.println("null安全: " + "test".equals(input));  // false（安全）
    }
}
```

> **よくある間違い：** `==` で文字列を比較すると、リテラル同士では `true` になることがあるため、バグに気づきにくくなります。**常に `equals()` を使う習慣**をつけましょう。

### ポイントまとめ

- 文字列の内容比較には必ず `equals()` を使う
- 大文字小文字を無視する場合は `equalsIgnoreCase()` を使う
- `compareTo()` は辞書順比較で、ソート処理に役立つ
- `null` の可能性がある変数には、リテラル側から `equals()` を呼ぶ

## 6.4 StringBuilder / StringBuffer

### なぜ StringBuilder が必要か

`String` は不変なので、ループ内で文字列を繰り返し結合すると毎回新しいオブジェクトが生成され、パフォーマンスが悪化します。

```java
public class StringConcatPerformance {
    public static void main(String[] args) {
        // 悪い例：String の繰り返し結合
        long start1 = System.currentTimeMillis();
        String result1 = "";
        for (int i = 0; i < 100000; i++) {
            result1 = result1 + i;  // 毎回新しい String が生成される
        }
        long end1 = System.currentTimeMillis();
        System.out.println("String結合: " + (end1 - start1) + "ms");

        // 良い例：StringBuilder を使う
        long start2 = System.currentTimeMillis();
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < 100000; i++) {
            sb.append(i);  // 内部バッファに追加するだけ
        }
        String result2 = sb.toString();
        long end2 = System.currentTimeMillis();
        System.out.println("StringBuilder: " + (end2 - start2) + "ms");
    }
}
```

### StringBuilder の基本操作

```java
public class StringBuilderDemo {
    public static void main(String[] args) {
        StringBuilder sb = new StringBuilder("Hello");

        // append(): 末尾に追加
        sb.append(" ");
        sb.append("World");
        sb.append("!");
        System.out.println(sb.toString());  // Hello World!

        // insert(): 指定位置に挿入
        sb.insert(5, ",");
        System.out.println(sb.toString());  // Hello, World!

        // delete(): 指定範囲を削除
        sb.delete(5, 6);
        System.out.println(sb.toString());  // Hello World!

        // replace(): 指定範囲を置換
        sb.replace(6, 11, "Java");
        System.out.println(sb.toString());  // Hello Java!

        // reverse(): 逆順にする
        StringBuilder sb2 = new StringBuilder("abcde");
        sb2.reverse();
        System.out.println(sb2.toString());  // edcba

        // メソッドチェーン（連続して呼び出し）
        String result = new StringBuilder()
            .append("名前: ")
            .append("太郎")
            .append(", 年齢: ")
            .append(25)
            .toString();
        System.out.println(result);  // 名前: 太郎, 年齢: 25
    }
}
```

### StringBuilder と StringBuffer の違い

| 特徴 | StringBuilder | StringBuffer |
|---|---|---|
| スレッド安全性 | なし（非同期） | あり（同期） |
| パフォーマンス | 高速 | やや遅い |
| 推奨場面 | 単一スレッド | マルチスレッド |
| 導入バージョン | Java 5 | Java 1.0 |

通常は `StringBuilder` を使います。マルチスレッド環境で共有する場合のみ `StringBuffer` を検討してください。

### ポイントまとめ

- ループ内の文字列結合には `StringBuilder` を使う
- `append()`, `insert()`, `delete()`, `reverse()` などのメソッドがある
- 最終的に `toString()` で `String` に変換する
- 通常は `StringBuilder` を使い、`StringBuffer` はマルチスレッド用

## 6.5 文字列フォーマット

### String.format()

`String.format()` を使うと、書式指定子（format specifier）を使って文字列を整形できます。

```java
public class StringFormatDemo {
    public static void main(String[] args) {
        // 基本的な書式指定子
        // %s: 文字列, %d: 整数, %f: 浮動小数点, %b: 真偽値
        String name = "太郎";
        int age = 25;
        double height = 175.5;

        String info = String.format("名前: %s, 年齢: %d歳, 身長: %.1fcm", name, age, height);
        System.out.println(info);
        // 名前: 太郎, 年齢: 25歳, 身長: 175.5cm

        // 桁揃え
        System.out.println(String.format("|%-10s|%10s|", "左寄せ", "右寄せ"));
        // |左寄せ        |        右寄せ|

        // ゼロ埋め
        System.out.println(String.format("注文番号: %05d", 42));
        // 注文番号: 00042

        // printf: 直接出力（フォーマットして画面に表示）
        System.out.printf("消費税込み: ¥%,.0f%n", 10800.0);
        // 消費税込み: ¥10,800

        // 主要な書式指定子一覧
        System.out.printf("文字列:    %s%n", "Hello");
        System.out.printf("整数:      %d%n", 42);
        System.out.printf("浮動小数:  %.2f%n", 3.14159);
        System.out.printf("16進数:    %x%n", 255);
        System.out.printf("8進数:     %o%n", 255);
        System.out.printf("文字:      %c%n", 'A');
        System.out.printf("真偽値:    %b%n", true);

        // 表形式の出力
        System.out.println("\n--- 商品一覧 ---");
        System.out.printf("%-12s %8s %6s%n", "商品名", "単価", "数量");
        System.out.printf("%-12s %,8d %6d%n", "りんご", 150, 3);
        System.out.printf("%-12s %,8d %6d%n", "みかん", 80, 10);
        System.out.printf("%-12s %,8d %6d%n", "ぶどう", 500, 2);
    }
}
```

### ポイントまとめ

- `String.format()` で書式付き文字列を生成できる
- `System.out.printf()` で書式付き出力ができる
- `%s`（文字列）、`%d`（整数）、`%f`（小数）が基本
- `%n` は改行（OSに依存しない）

## 6.6 正規表現の基礎

正規表現（Regular Expression、regex）は、文字列のパターンマッチングに使われる強力な機能です。

### 基本的な正規表現

```java
public class RegexBasics {
    public static void main(String[] args) {
        // matches(): 文字列全体がパターンに一致するか
        System.out.println("12345".matches("\\d+"));        // true（数字のみ）
        System.out.println("abc123".matches("\\d+"));       // false
        System.out.println("hello".matches("[a-z]+"));      // true（小文字のみ）

        // 主要なパターン
        // .     : 任意の1文字
        // \\d   : 数字 [0-9]
        // \\w   : 英数字とアンダースコア [a-zA-Z0-9_]
        // \\s   : 空白文字
        // +     : 1回以上の繰り返し
        // *     : 0回以上の繰り返し
        // ?     : 0回または1回
        // {n}   : ちょうどn回
        // {n,m} : n回以上m回以下
        // ^     : 行頭
        // $     : 行末
        // []    : 文字クラス
        // |     : または

        // 電話番号の判定
        String phone1 = "090-1234-5678";
        String phone2 = "12-3456";
        String phonePattern = "\\d{2,4}-\\d{2,4}-\\d{4}";
        System.out.println(phone1 + " -> " + phone1.matches(phonePattern));  // true
        System.out.println(phone2 + " -> " + phone2.matches(phonePattern));  // false

        // 郵便番号の判定
        String zipPattern = "\\d{3}-\\d{4}";
        System.out.println("100-0001".matches(zipPattern));  // true
        System.out.println("1000001".matches(zipPattern));   // false
    }
}
```

### Pattern と Matcher クラス

```java
import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class PatternMatcherDemo {
    public static void main(String[] args) {
        // Pattern: 正規表現をコンパイルしたもの
        // Matcher: パターンを文字列に適用した結果
        String text = "私のメールは taro@example.com と jiro@test.org です。";
        Pattern pattern = Pattern.compile("[\\w.]+@[\\w.]+\\.[a-zA-Z]+");
        Matcher matcher = pattern.matcher(text);

        // find(): パターンに一致する部分を検索
        while (matcher.find()) {
            System.out.println("見つかったメール: " + matcher.group());
        }
        // 見つかったメール: taro@example.com
        // 見つかったメール: jiro@test.org

        // replaceAll(): 一致部分をすべて置換
        String censored = text.replaceAll("[\\w.]+@[\\w.]+\\.[a-zA-Z]+", "[メール非表示]");
        System.out.println(censored);
        // 私のメールは [メール非表示] と [メール非表示] です。

        // split() での正規表現使用
        String data = "apple  banana   cherry    date";
        String[] words = data.split("\\s+");  // 1つ以上の空白で分割
        for (String word : words) {
            System.out.println(word);
        }
    }
}
```

### ポイントまとめ

- `matches()` は文字列全体のパターン一致を判定する
- `Pattern` と `Matcher` を使うと部分一致の検索・抽出ができる
- `\\d`（数字）、`\\w`（英数字）、`\\s`（空白）が基本パターン
- Javaでは `\` を `\\` と書く必要がある（エスケープ）

## 6.7 実践例

### 例1：回文（palindrome）チェッカー

```java
public class PalindromeChecker {
    public static boolean isPalindrome(String text) {
        // 空白を除去し、小文字に統一
        String cleaned = text.replaceAll("\\s+", "").toLowerCase();
        // 逆順の文字列を作成
        String reversed = new StringBuilder(cleaned).reverse().toString();
        return cleaned.equals(reversed);
    }

    public static void main(String[] args) {
        String[] testWords = {"level", "racecar", "hello", "madam", "Java"};

        for (String word : testWords) {
            System.out.printf("%-10s -> %s%n", word,
                isPalindrome(word) ? "回文です" : "回文ではありません");
        }
        // level      -> 回文です
        // racecar    -> 回文です
        // hello      -> 回文ではありません
        // madam      -> 回文です
        // Java       -> 回文ではありません

        // スペース含みのテスト
        System.out.println(isPalindrome("was it a car or a cat i saw"));  // true
    }
}
```

### 例2：単語カウンター

```java
import java.util.HashMap;
import java.util.Map;

public class WordCounter {
    public static Map<String, Integer> countWords(String text) {
        Map<String, Integer> wordCount = new HashMap<>();
        // 句読点を除去し、小文字に統一して分割
        String[] words = text.toLowerCase()
                             .replaceAll("[.,!?;:]", "")
                             .split("\\s+");

        for (String word : words) {
            if (!word.isEmpty()) {
                wordCount.put(word, wordCount.getOrDefault(word, 0) + 1);
            }
        }
        return wordCount;
    }

    public static void main(String[] args) {
        String text = "Java is great. Java is powerful. I love Java programming. "
                     + "Programming in Java is fun.";

        Map<String, Integer> result = countWords(text);

        System.out.println("--- 単語出現回数 ---");
        for (Map.Entry<String, Integer> entry : result.entrySet()) {
            System.out.printf("%-15s: %d回%n", entry.getKey(), entry.getValue());
        }
    }
}
```

### 例3：簡易メールアドレスバリデーター

```java
import java.util.regex.Pattern;

public class EmailValidator {
    // メールアドレスの基本的なパターン
    private static final Pattern EMAIL_PATTERN =
        Pattern.compile("^[\\w.-]+@[\\w.-]+\\.[a-zA-Z]{2,}$");

    public static boolean isValidEmail(String email) {
        if (email == null || email.isEmpty()) {
            return false;
        }
        return EMAIL_PATTERN.matcher(email).matches();
    }

    public static void main(String[] args) {
        String[] emails = {
            "user@example.com",
            "taro.yamada@company.co.jp",
            "invalid-email",
            "@missing-local.com",
            "no-domain@",
            "user@.com",
            "valid_user123@test.org"
        };

        System.out.println("--- メールアドレス検証 ---");
        for (String email : emails) {
            System.out.printf("%-30s -> %s%n", email,
                isValidEmail(email) ? "有効" : "無効");
        }
    }
}
```

### ポイントまとめ

- 文字列操作は実務で頻繁に使うため、メソッドを覚えておくと効率が上がる
- `StringBuilder` + `reverse()` で回文判定が簡単にできる
- `HashMap` と文字列分割を組み合わせると単語カウンターが作れる
- 正規表現は入力バリデーションに非常に有効

4. **StringBuilder** - ループ内の結合には `StringBuilder` を使いパフォーマンスを改善する
5. **フォーマット** - `String.format()` と `printf()` で整形された出力ができる
6. **正規表現** - `Pattern` と `Matcher` でパターンマッチングができる
