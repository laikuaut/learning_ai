# 第8章：実践パターンとベストプラクティス

## この章のゴール

- `.component()` を使ったコンポーネントベースの設計を理解する
- 単方向バインディング（`<`）とコールバック（`&`）による安全なデータフローを学ぶ
- ライフサイクルフック（`$onInit`、`$onChanges`、`$onDestroy`）を活用できる
- パフォーマンス改善のテクニック（ワンタイムバインディング、`track by`）を知る
- よくあるアンチパターンを避けられる
- Angular 2+ への移行パスを理解する

## 8.1 .component() — コンポーネントベース設計

AngularJS 1.5 で導入された `.component()` は、**ディレクティブをシンプルに書けるショートカット**です。

### .directive() vs .component() の比較

| 特性 | .directive() | .component() |
|------|-------------|-------------|
| 書き方 | DDO を返す関数 | 設定オブジェクトを直接渡す |
| スコープ | 任意 | 常に isolated（分離） |
| バインディング | `scope: {}` | `bindings: {}` |
| コントローラ | オプション | 常に使用（`controller`） |
| `controllerAs` | 手動設定 | デフォルトで `$ctrl` |
| restrict | 任意 | 常に `'E'`（要素のみ） |

### 基本例

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
  <style>
    .user-card { border: 1px solid #ddd; border-radius: 8px; padding: 16px; margin: 8px; display: inline-block; min-width: 200px; }
    .user-card h3 { color: #42b883; margin: 0 0 8px; }
  </style>
</head>
<body>
  <user-card name="田中太郎" role="エンジニア"></user-card>
  <user-card name="鈴木花子" role="デザイナー"></user-card>
  <user-card name="佐藤一郎" role="マネージャー"></user-card>

  <script>
    angular.module('myApp', [])
      .component('userCard', {
        bindings: {
          name: '@',
          role: '@'
        },
        template: '<div class="user-card">' +
                  '  <h3>{{ $ctrl.name }}</h3>' +
                  '  <p>役職: {{ $ctrl.role }}</p>' +
                  '</div>',
        controller: function() {
          // $ctrl は自動的にテンプレートで使える
          this.$onInit = function() {
            console.log($ctrl.name + ' コンポーネント初期化');
          };
        }
      });
  </script>
</body>
</html>
```

> **ポイント：** `.component()` ではテンプレート内で `$ctrl` がデフォルトのプレフィックスになります。

## 8.2 バインディングのベストプラクティス

### 単方向バインディング `<` を推奨

```
【データフローの推奨パターン】

  親コンポーネント
       │
       │  < (単方向: データを下に流す)
       ▼
  子コンポーネント
       │
       │  & (コールバック: イベントを上に伝える)
       ▼
  親コンポーネント（親のメソッドが実行される）
```

| バインディング | 記号 | 用途 |
|---------------|------|------|
| 単方向（推奨） | `<` | 親から子へデータを渡す |
| 文字列 | `@` | 文字列値を渡す |
| コールバック | `&` | 子から親へイベントを通知 |
| 双方向 | `=` | **非推奨**（データフローが追いにくい） |

### 親子コンポーネント間通信の例

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
  <style>
    .container { max-width: 600px; margin: 20px auto; font-family: sans-serif; }
    .todo-item { display: flex; align-items: center; padding: 8px; border-bottom: 1px solid #eee; }
    .todo-item.done span { text-decoration: line-through; color: #999; }
    .btn-sm { padding: 4px 8px; border: none; border-radius: 4px; cursor: pointer; margin-left: 8px; }
    .btn-danger { background: #e74c3c; color: white; }
    .btn-add { background: #42b883; color: white; border: none; padding: 8px 16px; border-radius: 4px; cursor: pointer; }
    input[type="text"] { padding: 8px; border: 1px solid #ddd; border-radius: 4px; flex: 1; }
    .add-form { display: flex; gap: 8px; margin-bottom: 16px; }
  </style>
</head>
<body>
  <div class="container">
    <todo-app></todo-app>
  </div>

  <script>
    angular.module('myApp', [])
      // 親: TODOアプリ全体
      .component('todoApp', {
        template: '<h2>TODOアプリ（コンポーネント設計）</h2>' +
                  '<todo-form on-add="$ctrl.addTodo(text)"></todo-form>' +
                  '<todo-list items="$ctrl.todos" on-toggle="$ctrl.toggleTodo(item)" on-delete="$ctrl.deleteTodo(item)"></todo-list>' +
                  '<p>残り: {{ $ctrl.remaining() }}件</p>',
        controller: function() {
          var ctrl = this;
          ctrl.todos = [
            { text: 'AngularJSを学ぶ', done: false },
            { text: 'コンポーネントを作る', done: true }
          ];
          ctrl.addTodo = function(text) {
            ctrl.todos.push({ text: text, done: false });
          };
          ctrl.toggleTodo = function(item) {
            item.done = !item.done;
          };
          ctrl.deleteTodo = function(item) {
            var idx = ctrl.todos.indexOf(item);
            if (idx > -1) ctrl.todos.splice(idx, 1);
          };
          ctrl.remaining = function() {
            return ctrl.todos.filter(function(t) { return !t.done; }).length;
          };
        }
      })
      // 子: 入力フォーム
      .component('todoForm', {
        bindings: {
          onAdd: '&'   // コールバック
        },
        template: '<div class="add-form">' +
                  '  <input type="text" ng-model="$ctrl.newText" placeholder="新しいタスク" ng-keyup="$event.keyCode === 13 && $ctrl.add()">' +
                  '  <button class="btn-add" ng-click="$ctrl.add()">追加</button>' +
                  '</div>',
        controller: function() {
          this.newText = '';
          this.add = function() {
            if (this.newText.trim()) {
              this.onAdd({ text: this.newText });
              this.newText = '';
            }
          };
        }
      })
      // 子: リスト表示
      .component('todoList', {
        bindings: {
          items: '<',      // 単方向
          onToggle: '&',   // コールバック
          onDelete: '&'    // コールバック
        },
        template: '<div class="todo-item" ng-class="{ done: item.done }" ng-repeat="item in $ctrl.items">' +
                  '  <input type="checkbox" ng-checked="item.done" ng-click="$ctrl.onToggle({ item: item })">' +
                  '  <span style="flex:1; margin-left:8px;">{{ item.text }}</span>' +
                  '  <button class="btn-sm btn-danger" ng-click="$ctrl.onDelete({ item: item })">削除</button>' +
                  '</div>'
      });
  </script>
</body>
</html>
```

### & バインディングでのパラメータ渡し

```html
<!-- 親テンプレート -->
<child-component on-action="$ctrl.handleAction(param1, param2)"></child-component>

<!-- 子コンポーネントのコントローラ -->
this.onAction({ param1: 'value1', param2: 'value2' });
// → 親の handleAction('value1', 'value2') が呼ばれる
```

> **よくある間違い：** `&` バインディングのコールバック呼び出しでは、引数をオブジェクト `{ key: value }` で渡します。直接 `this.onAction(value)` と書いても動きません。

## 8.3 ライフサイクルフック

| フック | タイミング | 用途 |
|--------|-----------|------|
| `$onInit()` | コンポーネント初期化時 | 初期データの設定、API呼び出し |
| `$onChanges(changes)` | バインディングの値が変わった時 | 入力値の変更に対応 |
| `$onDestroy()` | コンポーネント破棄時 | クリーンアップ（タイマー停止など） |
| `$postLink()` | テンプレートがリンクされた後 | DOM操作 |

```javascript
.component('timerWidget', {
  template: '<p>経過時間: {{ $ctrl.seconds }}秒</p>',
  controller: ['$interval', function($interval) {
    var ctrl = this;
    var timer;
    
    ctrl.$onInit = function() {
      ctrl.seconds = 0;
      timer = $interval(function() {
        ctrl.seconds++;
      }, 1000);
    };
    
    ctrl.$onDestroy = function() {
      // タイマーを停止してメモリリークを防ぐ
      $interval.cancel(timer);
    };
  }]
})
```

### $onChanges の使い方

```javascript
.component('priceDisplay', {
  bindings: {
    price: '<',
    taxRate: '<'
  },
  template: '<p>税込: ¥{{ $ctrl.totalPrice }}</p>',
  controller: function() {
    var ctrl = this;
    
    ctrl.$onChanges = function(changes) {
      // バインディングの値が変わるたびに再計算
      ctrl.totalPrice = Math.floor(ctrl.price * (1 + ctrl.taxRate));
    };
  }
})
```

## 8.4 パフォーマンス改善テクニック

### ワンタイムバインディング `::` 

一度表示したら変わらないデータには `::` を付けると、ウォッチャーが作られずパフォーマンスが向上します。

```html
<!-- 通常のバインディング（常に監視） -->
<p>{{ user.name }}</p>

<!-- ワンタイムバインディング（一度表示したら監視しない） -->
<p>{{ ::user.name }}</p>

<!-- ng-repeat でも使える -->
<li ng-repeat="item in ::staticItems">{{ ::item.name }}</li>
```

### track by の指定

```html
<!-- NG: オブジェクト全体を比較（遅い） -->
<div ng-repeat="item in items">{{ item.name }}</div>

<!-- OK: ID で追跡（速い） -->
<div ng-repeat="item in items track by item.id">{{ item.name }}</div>
```

### その他のパフォーマンスTips

| テクニック | 効果 |
|-----------|------|
| `::` ワンタイムバインディング | ウォッチャー数を削減 |
| `track by` | ng-repeat のDOM再利用を最適化 |
| `ng-if` vs `ng-show` | 不要な要素をDOMから除去 |
| `$scope.$watchCollection` | 配列の浅い監視（`$watch` より軽量） |
| `ng-model-options` | debounce でダイジェスト頻度を減らす |

```html
<!-- debounce: 入力後300ms待ってから更新 -->
<input ng-model="search" ng-model-options="{ debounce: 300 }">
```

## 8.5 よくあるアンチパターン

### 1. コントローラにDOM操作を書く

```javascript
// ❌ アンチパターン
.controller('BadCtrl', function($scope) {
  document.getElementById('myBtn').addEventListener('click', ...);
})

// ✅ 正しい方法: ディレクティブの link 関数で行う
.directive('myBtn', function() {
  return {
    link: function(scope, element) {
      element.on('click', function() { ... });
    }
  };
})
```

### 2. $scope を使い続ける

```javascript
// ❌ 古い書き方
.controller('OldCtrl', function($scope) {
  $scope.name = '太郎';
})

// ✅ Controller As または .component() を使う
.component('newComp', {
  controller: function() {
    this.name = '太郎';
  }
})
```

### 3. $rootScope の乱用

```javascript
// ❌ グローバル変数的に使う
$rootScope.currentUser = user;

// ✅ サービスで管理する
.factory('AuthService', function() {
  var currentUser = null;
  return {
    setUser: function(user) { currentUser = user; },
    getUser: function() { return currentUser; }
  };
})
```

### 4. ウォッチャーの増えすぎ

```html
<!-- ❌ 大量のウォッチャー -->
<tr ng-repeat="row in largeData">
  <td>{{ row.col1 }}</td>
  <td>{{ row.col2 }}</td>
  <td>{{ row.col3 }}</td>
</tr>

<!-- ✅ ワンタイムバインディングで削減 -->
<tr ng-repeat="row in largeData track by row.id">
  <td>{{ ::row.col1 }}</td>
  <td>{{ ::row.col2 }}</td>
  <td>{{ ::row.col3 }}</td>
</tr>
```

## 8.6 Angular 2+ への移行パス

AngularJS（1.x）から Angular（2+）への移行を見据えた準備：

| 準備 | 説明 |
|------|------|
| `.component()` を使う | Angular 2+ のコンポーネントに近い構造 |
| TypeScript を導入 | Angular 2+ は TypeScript が前提 |
| 単方向データフロー | `<` と `&` を使い、`=` を避ける |
| ES6 モジュールを使う | `import/export` に移行 |
| サービスをクラスで書く | Angular 2+ の `@Injectable()` に対応 |

### 段階的な移行戦略

```
1. .directive() → .component() に書き換え
2. $scope → Controller As / $ctrl に移行
3. = バインディング → < と & に変更
4. ngUpgrade を使ってハイブリッド実行
5. コンポーネントを一つずつ Angular 2+ に変換
```

  'ngRoute',

## 8.8 まとめ

| トピック | ポイント |
|----------|----------|
| .component() | `.directive()` よりシンプル、推奨の書き方 |
| 単方向バインディング `<` | データは上から下へ流す |
| コールバック `&` | イベントは下から上へ伝える |
| ライフサイクルフック | `$onInit`, `$onChanges`, `$onDestroy` |
| ワンタイムバインディング | `::` で不要なウォッチャーを削減 |
| アンチパターン | DOM操作はディレクティブ、$rootScope は避ける |
| 移行準備 | .component() + 単方向フロー + TypeScript |
