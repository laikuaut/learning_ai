# 第1章: 基本構文と変数

## はじめに

JavaScriptは、Webブラウザ上で動作するプログラミング言語として誕生し、現在ではサーバーサイド（Node.js）、モバイルアプリ、デスクトップアプリなど、あらゆる場面で使われる世界で最も広く利用されている言語の一つです。

この章では、JavaScriptの最も基本的な構文と変数について学びます。

---

## 1.1 JavaScriptの実行環境

### ブラウザのコンソール
最も手軽な方法は、ブラウザの開発者ツール（DevTools）を使うことです。

1. ブラウザで `F12` キーを押す（または右クリック→「検証」）
2. 「Console」タブを選択
3. コードを入力して `Enter` で実行

### HTMLファイルからの実行
```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>JavaScript入門</title>
</head>
<body>
  <!-- 方法1: HTMLファイル内に直接記述 -->
  <script>
    console.log("Hello, JavaScript!");
  </script>

  <!-- 方法2: 外部ファイルを読み込む -->
  <script src="script.js"></script>
</body>
</html>
```

### Node.js
ターミナル（コマンドプロンプト）で実行する方法です。
```bash
node script.js
```

## 1.2 console.log() — 出力の基本

`console.log()` は、コンソールにメッセージを表示するための最も基本的な関数です。

```javascript
// 文字列を表示
console.log("こんにちは、JavaScript！");

// 数値を表示
console.log(42);

// 複数の値をカンマ区切りで表示
console.log("答えは", 42, "です");

// 計算結果を表示
console.log(3 + 5); // 8
```

### その他のconsoleメソッド（参考）
```javascript
console.warn("警告メッセージ");    // 黄色い警告として表示
console.error("エラーメッセージ");  // 赤いエラーとして表示
console.table([1, 2, 3]);          // 表形式で表示
console.clear();                    // コンソールをクリア
```

## 1.4 変数宣言: let, const, var

変数とは、データを一時的に保存する「名前の付いた箱」のようなものです。

### const（定数）— 最も推奨

`const` で宣言した変数は**再代入できません**。値が変わらないことが保証されるため、バグを防ぎやすくなります。

```javascript
const PI = 3.14159;
const APP_NAME = "MyApp";
const MAX_RETRY = 3;

// PI = 3.14; // エラー！ConstでTypeError: Assignment to constant variable.

// ただし、オブジェクトや配列の中身は変更できる（参照が変わらないため）
const user = { name: "太郎" };
user.name = "花子";  // これはOK！（オブジェクトの中身を変えているだけ）
console.log(user.name); // "花子"

const numbers = [1, 2, 3];
numbers.push(4);  // これもOK！
console.log(numbers); // [1, 2, 3, 4]

// numbers = [5, 6, 7]; // エラー！変数自体の再代入はできない
```

### let（変数）— 値が変わる場合に使用

`let` は再代入が可能な変数です。ループカウンタや、後から値が変わる場合に使います。

```javascript
let score = 0;
score = 100;        // 再代入OK
score = score + 50; // 150

let message;        // 宣言だけもできる（値はundefined）
message = "Hello";

// ブロックスコープ（後述）
if (true) {
  let x = 10;
  console.log(x); // 10
}
// console.log(x); // エラー！ブロックの外からはアクセスできない
```

### var（非推奨）— レガシーコード向け

`var` は古い書き方で、現代のJavaScriptでは**ほぼ使いません**。以下の問題があります。

```javascript
var oldStyle = "古い書き方";

// 問題1: 関数スコープ（ブロックスコープではない）
if (true) {
  var leaked = "漏れ出す！";
}
console.log(leaked); // "漏れ出す！" ← ブロックの外からアクセスできてしまう

// 問題2: 再宣言が可能
var x = 1;
var x = 2; // エラーにならない（letやconstならエラー）

// 問題3: ホイスティング（巻き上げ）
console.log(hoisted); // undefined（エラーにならない！）
var hoisted = "値";
// ↑ これは内部的に以下のように解釈される:
// var hoisted;
// console.log(hoisted); // undefined
// hoisted = "値";
```

### 使い分けのガイドライン

```
基本は const を使う
    ↓ 再代入が必要な場合のみ
let を使う
    ↓ 古いコードの保守時のみ
var（通常は使わない）
```

## 1.5 変数の命名規則

```javascript
// ✅ 良い命名（キャメルケース）
let userName = "太郎";
let isLoggedIn = true;

```

## 1.6 データ型

JavaScriptには以下の **8つのデータ型** があります。

### プリミティブ型（7つ）

値そのものが格納される基本的な型です。

```javascript
// 1. 数値 (Number)
let integer = 42;
let float = 3.14;
let negative = -10;
let infinity = Infinity;
let notANumber = NaN; // "Not a Number"（計算エラー時に発生）
console.log(0.1 + 0.2); // 0.30000000000000004（浮動小数点の注意点！）

// 2. 長整数 (BigInt) — 非常に大きな整数を扱う
let bigNumber = 9007199254740991n; // 末尾にnを付ける
let anotherBig = BigInt("99999999999999999999");

// 3. 文字列 (String)
let single = '文字列';      // シングルクォート
let double = "文字列";      // ダブルクォート
let backtick = `文字列`;    // バッククォート（テンプレートリテラル）
let empty = "";              // 空文字列

// 4. 真偽値 (Boolean)
let isActive = true;
let isDeleted = false;

// 5. undefined — 値が割り当てられていない状態
let notAssigned;
console.log(notAssigned); // undefined

// 6. null — 意図的に「値がない」ことを示す
let emptyValue = null;

// 7. シンボル (Symbol) — ユニークな識別子
let sym1 = Symbol("説明");
let sym2 = Symbol("説明");
console.log(sym1 === sym2); // false（同じ説明でも異なるシンボル）
```

### オブジェクト型（1つ）

複数のデータをまとめて管理する型です。

```javascript
// オブジェクト
let person = {
  name: "太郎",
  age: 25,
  isStudent: true
};

// 配列（特殊なオブジェクト）
let fruits = ["りんご", "みかん", "バナナ"];

// 関数（特殊なオブジェクト）
let greet = function() { return "Hello!"; };
```

## 1.7 typeof 演算子

`typeof` は変数のデータ型を文字列で返します。

```javascript
console.log(typeof 42);          // "number"
console.log(typeof 3.14);        // "number"
console.log(typeof "Hello");     // "string"
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object" ← 注意！歴史的なバグ
console.log(typeof Symbol());    // "symbol"
console.log(typeof 42n);         // "bigint"
console.log(typeof {});          // "object"
console.log(typeof []);          // "object" ← 配列もobject
console.log(typeof function(){}); // "function"

// nullの判定方法
let value = null;
console.log(value === null); // true（===で直接比較する）

// 配列の判定方法
let arr = [1, 2, 3];
console.log(Array.isArray(arr)); // true
```

## 1.8 テンプレートリテラル

バッククォート（`` ` ``）で囲む文字列は、**テンプレートリテラル**と呼ばれ、特別な機能を持ちます。

### 変数の埋め込み（${...}）
```javascript
const name = "太郎";
const age = 25;

// 従来の書き方（文字列連結）
console.log("私の名前は" + name + "で、" + age + "歳です。");

// テンプレートリテラル（推奨）
console.log(`私の名前は${name}で、${age}歳です。`);

// 式も埋め込める
console.log(`来年は${age + 1}歳になります。`);
console.log(`税込価格: ${(1000 * 1.1).toFixed(0)}円`);

// 条件式も使える
const score = 85;
console.log(`結果: ${score >= 60 ? "合格" : "不合格"}`);
```

### 複数行の文字列
```javascript
// 従来の書き方
const oldWay = "1行目\n" +
               "2行目\n" +
               "3行目";

// テンプレートリテラル（改行がそのまま反映される）
const newWay = `1行目
2行目
3行目`;

// HTMLテンプレートとして活用
const html = `
  <div class="card">
    <h2>${name}</h2>
    <p>年齢: ${age}</p>
  </div>
`;
```

### タグ付きテンプレートリテラル（発展）
```javascript
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) => {
    return result + str + (values[i] !== undefined ? `【${values[i]}】` : "");
  }, "");
}

const item = "JavaScript";
const level = "入門";
console.log(highlight`${item}の${level}コースへようこそ！`);
// "【JavaScript】の【入門】コースへようこそ！"
```

## 1.9 文字列の基本操作

```javascript
const str = "Hello, JavaScript!";

// 長さ
console.log(str.length); // 18

// 文字の取得
console.log(str[0]);        // "H"
console.log(str.charAt(7)); // "J"

// 検索
console.log(str.indexOf("Java"));    // 7
console.log(str.includes("Script")); // true
console.log(str.startsWith("Hello")); // true
console.log(str.endsWith("!"));       // true

// 切り出し
console.log(str.slice(7, 17));  // "JavaScript"
console.log(str.slice(-11));    // "JavaScript!"

// 変換
console.log(str.toUpperCase()); // "HELLO, JAVASCRIPT!"
console.log(str.toLowerCase()); // "hello, javascript!"

// 置換
console.log(str.replace("JavaScript", "World")); // "Hello, World!"

// 分割
console.log("a,b,c".split(","));  // ["a", "b", "c"]

// 空白除去
console.log("  hello  ".trim());      // "hello"
console.log("  hello  ".trimStart()); // "hello  "
console.log("  hello  ".trimEnd());   // "  hello"

// 繰り返し・パディング
console.log("abc".repeat(3));       // "abcabcabc"
console.log("5".padStart(3, "0"));  // "005"
console.log("5".padEnd(3, "0"));    // "500"
```

## 1.10 数値の基本操作

```javascript
// 特殊な値
console.log(Infinity);     // Infinity
console.log(-Infinity);    // -Infinity
console.log(NaN);          // NaN
console.log(isNaN(NaN));   // true
console.log(isFinite(42)); // true

// 数値変換
console.log(Number("42"));     // 42
console.log(Number(""));       // 0
console.log(Number("abc"));    // NaN
console.log(parseInt("42px")); // 42（先頭の数値部分を解析）
console.log(parseFloat("3.14em")); // 3.14

// Mathオブジェクト
console.log(Math.PI);           // 3.141592653589793
console.log(Math.round(4.5));   // 5（四捨五入）
console.log(Math.floor(4.9));   // 4（切り捨て）
console.log(Math.ceil(4.1));    // 5（切り上げ）
console.log(Math.trunc(4.9));   // 4（整数部分のみ）
console.log(Math.abs(-5));      // 5（絶対値）
console.log(Math.max(1, 5, 3)); // 5
console.log(Math.min(1, 5, 3)); // 1
console.log(Math.random());     // 0以上1未満のランダムな数
console.log(Math.pow(2, 3));    // 8（2の3乗）
console.log(Math.sqrt(16));     // 4（平方根）

// ランダムな整数を得る（例: 1～10）
const randomInt = Math.floor(Math.random() * 10) + 1;
```

## 1.11 セミコロンと文の区切り

```javascript
// セミコロンは文の終わりを示す
let a = 1;
let b = 2;

// 1行に複数の文を書く場合はセミコロンが必要
let c = 3; let d = 4;

// 自動セミコロン挿入（ASI）があるため、省略しても多くの場合動く
// しかし、明示的に書くことが推奨される
let e = 5  // セミコロンなしでも動くが...

// ASIが問題を起こすケース
// return
//   { key: "value" }
// ↑ return; と解釈され、undefinedが返る！
```
