## 課題の要件

以下の5つのテーマについて、プロンプトの4要素（役割（Role）・指示（Instruction）・文脈（Context）・出力形式（Output Format））を明示的に設計し、実際にLLMに入力して結果を確認してください。

### テーマ一覧

| テーマ | 内容 |
|--------|------|
| A | IT企業の会社説明会用の自己紹介文を作成する |
| B | 小学生向けに「インターネットの仕組み」を説明する |
| C | 上司へのプロジェクト進捗報告メールを作成する |
| D | 料理レシピ（カレーライス）を初心者向けに説明する |
| E | 英語の短い文章（3文程度）を自然な日本語に翻訳する |

### 各テーマで実施すること

1. 4要素をそれぞれ明記したプロンプトを設計する
2. 実際にLLMに入力し、出力結果を記録する
3. 出力結果を「期待通りか」「改善点はあるか」の観点で評価する
4. 全テーマ完了後、4要素それぞれの効果をまとめる

## ステップガイド

<details>
<summary>ステップ1：テーマAのプロンプトを設計する</summary>

テーマA「IT企業の会社説明会用の自己紹介文」を例に、4要素を設計しましょう。

**役割の設定例**:
- 「あなたは採用担当のプロフェッショナルです」
- 「あなたはキャリアコンサルタントです」

**指示の設定例**:
- 「自己紹介文を作成してください」
- 「会社説明会で使える自己紹介文を書いてください」

**文脈の設定例**:
- 「IT企業の新卒向け会社説明会で、エンジニア職を希望する学生に向けて話します」
- 「会社は従業員100名のWebサービス企業です」

**出力形式の設定例**:
- 「200文字以内で、です・ます調で書いてください」
- 「1分間で読み上げられる長さ（300文字程度）にしてください」

**よくある間違い**: 4要素をすべて1文にまとめてしまうこと。要素ごとに段落や行を分けると、LLMが正確に理解しやすくなります。

</details>

<details>
<summary>ステップ2：4要素の有無による違いを確認する</summary>

テーマAについて、以下の3パターンを試してみましょう。

**パターン1（指示のみ）**:
```
自己紹介文を書いてください。
```

**パターン2（指示＋文脈）**:
```
IT企業の新卒向け会社説明会で使う自己紹介文を書いてください。
```

**パターン3（4要素すべて）**:
```
あなたは採用経験10年のキャリアコンサルタントです。

IT企業（従業員100名のWebサービス企業）の新卒向け会社説明会で、
エンジニア職を希望する学生に向けて使う自己紹介文を作成してください。

背景として、当社はリモートワーク推進企業で、若手エンジニアの活躍が多いです。

以下の条件で出力してください：
- 300文字程度
- です・ます調
- 親しみやすさと専門性のバランスを意識
```

3パターンの出力を比較し、要素が増えることでどう変化するかを記録してください。

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

### テーマA：IT企業の会社説明会用の自己紹介文

**設計したプロンプト**:

```
あなたは採用担当のプロフェッショナルです。

IT企業の新卒向け会社説明会で使う自己紹介文を作成してください。

当社は従業員100名のWebサービス企業で、リモートワークを推進しています。
対象はエンジニア職を希望する大学生です。

以下の条件で出力してください：
- 300文字程度
- です・ます調
- 親しみやすさを重視
```

**出力結果の例**:

> 皆さん、こんにちは。本日はお忙しい中、弊社の説明会にお越しいただきありがとうございます。私は採用担当の田中と申します。弊社は従業員100名のWebサービス企業で、「技術で社会をもっと便利にする」をミッションに掲げています。リモートワークを全社的に推進しており、エンジニアの皆さんには自分のペースで最高のパフォーマンスを発揮できる環境を整えています。入社1年目から大規模なプロジェクトに参加できるチャンスがあり、若手エンジニアが多数活躍しています。本日は弊社の魅力をたっぷりお伝えしますので、気軽にご質問ください。

**評価**: 4要素をすべて指定したことで、ターゲット（新卒学生）に適したトーンと内容の自己紹介文が生成されました。「親しみやすさを重視」という指定が反映され、堅すぎない文章になっています。

---

### テーマB：小学生向けに「インターネットの仕組み」を説明する

**設計したプロンプト**:

```
あなたは小学校の理科の先生です。

小学4年生にインターネットの仕組みを説明してください。

授業の導入部分（5分程度）で使う説明です。
子供たちはスマートフォンやタブレットを日常的に使っていますが、
インターネットの仕組みは知りません。

以下の条件で出力してください：
- 200文字程度
- たとえ話を1つ以上使う
- 難しい漢字にはふりがなをつける
```

**評価**: 役割を「小学校の先生」にしたことで、分かりやすい言葉遣いになりました。「たとえ話を使う」という出力形式の指定が効果的でした。

---

### 4要素の効果まとめ

プロンプトの4要素を意識的に設計することで、LLMの出力品質が大幅に向上することを確認しました。「役割」を設定すると、専門性やトーンが適切に調整されます。例えば、同じ「説明してください」という指示でも、「小学校の先生」と「大学教授」では語彙レベルが明らかに異なりました。「指示」は具体的であるほど的確な出力になり、「文脈」を追加するとタスクの背景に合った内容が生成されます。「出力形式」は文字数やスタイルを制御するのに不可欠で、指定しないと冗長になりがちです。4要素すべてを使うことが常に必要なわけではありませんが、複雑なタスクほど要素を増やすことで期待通りの結果に近づきます。

### テーマA：IT企業の会社説明会用の自己紹介文（改良版）

**設計したプロンプト**:

```
# 役割
あなたは、IT企業で5年以上の採用経験を持つ人事マネージャーです。
新卒採用の説明会で高い参加満足度を維持してきた実績があります。

# 指示
以下の条件に基づいて、会社説明会の冒頭で使う自己紹介文を作成してください。
自己紹介文は「アイスブレイク」の役割も果たすものにしてください。

# 文脈
- 企業：従業員100名のWebサービス企業（BtoC向けアプリ開発）
- 対象：エンジニア職を志望する大学3年生〜大学院1年生
- 場所：オンライン説明会（Zoom）
- 企業の強み：リモートワーク推進、若手登用実績、技術ブログ運営
- 直近の実績：自社アプリが月間100万ダウンロード達成

# 出力形式
- 400文字以内
- です・ます調
- 以下の構成で出力：
  1. 挨拶と自己紹介（名前は「田中」とする）
  2. 会社の概要（1〜2文）
  3. 学生の関心を引くエピソード（1つ）
  4. 本日の説明会の流れへの導入（1文）
```

**改良ポイント**:
- Markdownの見出し（`#`）で4要素を明確に区切り、構造を視覚的に分かりやすくした
- 役割に「実績」を追加し、より具体的なペルソナ（persona）を設定した
- 文脈に場所（オンライン）や企業の具体的な実績を追加し、リアリティを向上させた
- 出力形式に構成の順序を指定し、文章の流れを制御した

### テーマC：進捗報告メール（改良版）

**設計したプロンプト**:

```
# 役割
あなたはビジネスライティングの専門家です。
IT企業で使われるプロジェクト管理の文書作成に精通しています。

# 指示
以下のプロジェクト情報に基づいて、上司（部長）への週次進捗報告メールを
作成してください。問題点がある場合は、対応策も併記してください。

# 文脈
- プロジェクト名：顧客管理システムリニューアル
- 期間：2026年1月〜2026年6月
- 現在の進捗：全体の45%（計画比 -5%の遅延）
- 今週の成果：データベース設計の完了、API仕様書の初版作成
- 問題点：フロントエンド開発者が1名病欠中（復帰未定）
- 来週の予定：API開発着手、UI設計レビュー

# 出力形式
以下の構成でメールを作成してください：
- 件名（15文字以内）
- 宛名
- 冒頭の挨拶（1文）
- 進捗サマリー（箇条書き）
- 問題点と対応策（表形式）
- 来週のアクション（箇条書き）
- 結びの挨拶
```

**改良ポイント**:
- 文脈に具体的な数値（進捗率、遅延日数）を含め、現実的なシナリオにした
- 出力形式を詳細に構成指定し、ビジネスメールとしての完成度を高めた
- 「問題点がある場合は対応策も併記」という条件付き指示を追加した

---

### 4要素の効果まとめ（改良版）

プロンプトの4要素は、それぞれが独立した役割を持ちつつ、組み合わせることで相乗効果を発揮します。

**役割の効果**: 単に「専門家です」と書くだけでなく、具体的な経験年数や実績を加えることで、出力の専門性と具体性が向上しました。テーマAでは「5年以上の採用経験」を追記しただけで、採用現場特有の表現（「アイスブレイク」「参加満足度」等）が自然に含まれるようになりました。

**指示の効果**: 「作成してください」だけでなく、「アイスブレイクの役割も果たすものに」のような意図の補足を加えると、単なる情報伝達ではなく目的に沿った文章が生成されます。

**文脈の効果**: 文脈の粒度を上げる（「IT企業」→「従業員100名のBtoC向けアプリ開発企業、月間100万DL達成」）と、出力のリアリティが格段に向上しました。

**出力形式の効果**: 構成の順序まで指定すると、LLMは指定された流れに従って文章を生成します。改良版では「構成指定」がもっとも効果が大きく、初心者向けより格段に整理された出力が得られました。

**初心者向けとの違い**:
- 4要素の区切りをMarkdown見出しで明示し、可読性を向上させた
- 役割に具体的な実績・経験年数を追加し、ペルソナの深度を増した
- 出力形式に構成順序を指定し、文章全体の流れを制御した
- 改良の「なぜ」を言語化し、再現可能なノウハウとして整理した
