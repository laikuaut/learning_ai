# 第6章：構造体と列挙型

## この章のゴール

この章を学び終えると、以下のことができるようになります。

- 構造体（struct）を定義し、インスタンスを生成できる
- タプル構造体とユニット構造体の用途を理解できる
- impl ブロックでメソッドと関連関数を定義できる
- 列挙型（enum）を定義し、バリアントにデータを持たせられる
- Option\<T\> 型を使って「値がないかもしれない」状態を安全に扱える
- match 式と enum を組み合わせたパターンマッチングができる
- Debug トレイトと Display トレイトを実装し、型を表示できる
- 構造体と列挙型を組み合わせた実践的なデータモデリングができる

---

## 6.1 構造体（struct）の定義と使用

### 構造体とは

構造体（struct）は、複数の関連する値をひとまとめにした独自のデータ型です。

```
┌────────────────────────────────────┐
│  なぜ構造体が必要？                   │
│                                    │
│  タプルだと:                         │
│    let user = ("Alice", 30, true); │
│    user.0  ← 何のデータか不明        │
│                                    │
│  構造体なら:                         │
│    user.name  ← 明確！              │
│    user.age   ← 明確！              │
│    user.active ← 明確！             │
└────────────────────────────────────┘
```

### 基本的な構造体の定義

```rust
// 構造体の定義
struct User {
    name: String,
    age: u32,
    active: bool,
}

fn main() {
    // インスタンスの生成
    let user1 = User {
        name: String::from("Alice"),
        age: 30,
        active: true,
    };

    println!("名前: {}", user1.name);
    println!("年齢: {}", user1.age);
    println!("有効: {}", user1.active);
}
```

```
名前: Alice
年齢: 30
有効: true
```

### フィールドの変更

構造体のフィールドを変更するには、インスタンス全体を `mut` にする必要があります。特定のフィールドだけを `mut` にすることはできません。

```rust
struct User {
    name: String,
    age: u32,
    active: bool,
}

fn main() {
    let mut user1 = User {
        name: String::from("Alice"),
        age: 30,
        active: true,
    };

    println!("変更前: {} ({}歳)", user1.name, user1.age);

    user1.age = 31;
    user1.name = String::from("Alice Smith");

    println!("変更後: {} ({}歳)", user1.name, user1.age);
}
```

```
変更前: Alice (30歳)
変更後: Alice Smith (31歳)
```

### フィールド初期化の省略記法

変数名とフィールド名が同じ場合、省略記法が使えます。

```rust
struct User {
    name: String,
    age: u32,
    active: bool,
}

fn create_user(name: String, age: u32) -> User {
    User {
        name,           // name: name の省略形
        age,            // age: age の省略形
        active: true,   // これは変数名と一致しないので省略不可
    }
}

fn main() {
    let user = create_user(String::from("Bob"), 25);
    println!("{} ({}歳, active={})", user.name, user.age, user.active);
}
```

```
Bob (25歳, active=true)
```

### 構造体更新構文

既存のインスタンスをベースに一部だけ変更した新しいインスタンスを作れます。

```rust
struct User {
    name: String,
    age: u32,
    active: bool,
    email: String,
}

fn main() {
    let user1 = User {
        name: String::from("Alice"),
        age: 30,
        active: true,
        email: String::from("alice@example.com"),
    };

    // user1 をベースに一部変更
    let user2 = User {
        name: String::from("Bob"),
        age: 25,
        ..user1  // 残りのフィールドは user1 から
    };

    println!("{} ({}歳) - {}", user2.name, user2.age, user2.email);
    // println!("{}", user1.email);  // エラー！email は String なのでムーブ済み
    println!("user1.age = {}", user1.age);  // u32 は Copy なので OK
}
```

```
Bob (25歳) - alice@example.com
user1.age = 30
```

> **よくある間違い：** 構造体更新構文 `..user1` で `String` 型のフィールドがムーブされると、元のインスタンスのそのフィールドは使えなくなります。Copy 型（i32, bool など）のフィールドは問題ありません。

### ポイントまとめ

| 項目 | 内容 |
|------|------|
| 定義 | `struct Name { field: Type, ... }` |
| 生成 | `Name { field: value, ... }` |
| アクセス | `instance.field` |
| 変更 | インスタンス全体を `mut` にする |
| 省略記法 | 変数名 = フィールド名なら `: value` 省略可 |
| 更新構文 | `..other_instance` で残りを埋める |

## 6.2 タプル構造体とユニット構造体

### タプル構造体（tuple struct）

フィールドに名前を付けずに、位置で区別する構造体です。型に意味を持たせたいときに使います。

```rust
// タプル構造体の定義
struct Color(u8, u8, u8);
struct Point(f64, f64);

fn main() {
    let red = Color(255, 0, 0);
    let origin = Point(0.0, 0.0);

    println!("赤: ({}, {}, {})", red.0, red.1, red.2);
    println!("原点: ({}, {})", origin.0, origin.1);

    // 異なる型として扱われる
    // let p: Point = Color(1, 2, 3);  // エラー！型が違う
}
```

```
赤: (255, 0, 0)
原点: (0, 0)
```

### ニュータイプパターン（newtype pattern）

フィールドが1つだけのタプル構造体は「ニュータイプパターン」と呼ばれ、型安全性を高めるために現場でよく使われます。

```rust
struct Meters(f64);
struct Kilograms(f64);

fn calculate_bmi(weight: Kilograms, height: Meters) -> f64 {
    weight.0 / (height.0 * height.0)
}

fn main() {
    let weight = Kilograms(70.0);
    let height = Meters(1.75);

    let bmi = calculate_bmi(weight, height);
    println!("BMI: {:.1}", bmi);

    // 引数を間違えるとコンパイルエラー！
    // let bmi = calculate_bmi(height, weight);  // 型が違うのでエラー
}
```

```
BMI: 22.9
```

### ユニット構造体（unit struct）

フィールドを持たない構造体です。トレイトの実装に使うことが多いです。

```rust
struct AlwaysEqual;

fn main() {
    let _subject = AlwaysEqual;
    // トレイトを実装して意味を持たせる（後述）
    println!("ユニット構造体を生成しました");
}
```

```
ユニット構造体を生成しました
```

### ポイントまとめ

| 種類 | 構文 | 用途 |
|------|------|------|
| 名前付き構造体 | `struct S { name: Type }` | 一般的なデータ型 |
| タプル構造体 | `struct S(Type1, Type2)` | 位置で区別する簡潔な型 |
| ニュータイプ | `struct S(Type)` | 型安全性の向上 |
| ユニット構造体 | `struct S;` | トレイト実装のマーカー |

## 6.3 メソッドと impl ブロック

### impl ブロックとメソッド

`impl`（implementation）ブロックで、構造体にメソッド（method）を定義できます。メソッドの第1引数は必ず `self`（自身のインスタンス）です。

```rust
struct Rectangle {
    width: f64,
    height: f64,
}

impl Rectangle {
    // メソッド: &self で不変参照を受け取る
    fn area(&self) -> f64 {
        self.width * self.height
    }

    fn perimeter(&self) -> f64 {
        2.0 * (self.width + self.height)
    }

    fn is_square(&self) -> bool {
        (self.width - self.height).abs() < f64::EPSILON
    }
}

fn main() {
    let rect = Rectangle {
        width: 10.0,
        height: 5.0,
    };

    println!("幅: {}, 高さ: {}", rect.width, rect.height);
    println!("面積: {}", rect.area());
    println!("周囲: {}", rect.perimeter());
    println!("正方形: {}", rect.is_square());
}
```

```
幅: 10, 高さ: 5
面積: 50
周囲: 30
正方形: false
```

### self の3つの形式

```
┌─────────────────────────────────────────────────────┐
│  &self      不変参照で借用（読み取りのみ）               │
│             → 最もよく使う                             │
│                                                     │
│  &mut self  可変参照で借用（変更可能）                   │
│             → フィールドを変更するメソッド               │
│                                                     │
│  self       所有権を奪う（インスタンスを消費）            │
│             → 変換メソッドなど特殊な場合                 │
└─────────────────────────────────────────────────────┘
```

```rust
struct Counter {
    count: i32,
}

impl Counter {
    // 読み取り: &self
    fn value(&self) -> i32 {
        self.count
    }

    // 変更: &mut self
    fn increment(&mut self) {
        self.count += 1;
    }

    // 消費: self（所有権を奪い、新しい値を返す）
    fn into_inner(self) -> i32 {
        self.count
    }
}

fn main() {
    let mut counter = Counter { count: 0 };

    counter.increment();
    counter.increment();
    counter.increment();
    println!("カウント: {}", counter.value());

    let final_value = counter.into_inner();
    println!("最終値: {}", final_value);
    // println!("{}", counter.value());  // エラー！counter は消費済み
}
```

```
カウント: 3
最終値: 3
```

### 関連関数（associated function）

`self` を引数に取らない関数は「関連関数」と呼ばれます。`::` で呼び出します。コンストラクタとしてよく使われます。

```rust
struct Rectangle {
    width: f64,
    height: f64,
}

impl Rectangle {
    // 関連関数（コンストラクタ）
    fn new(width: f64, height: f64) -> Self {
        Self { width, height }
    }

    // 正方形を作る関連関数
    fn square(size: f64) -> Self {
        Self {
            width: size,
            height: size,
        }
    }

    fn area(&self) -> f64 {
        self.width * self.height
    }
}

fn main() {
    let rect = Rectangle::new(10.0, 5.0);  // :: で呼び出し
    let sq = Rectangle::square(7.0);

    println!("長方形の面積: {}", rect.area());
    println!("正方形の面積: {}", sq.area());
}
```

```
長方形の面積: 50
正方形の面積: 49
```

> **ポイント：** `Self`（大文字S）は impl 対象の型そのものを指すエイリアスです。`Rectangle` と書いても同じですが、`Self` の方がリファクタリングに強いです。

### 複数の impl ブロック

1つの構造体に対して、複数の `impl` ブロックを書くことも可能です。

```rust
struct Circle {
    radius: f64,
}

// 基本的なメソッド
impl Circle {
    fn new(radius: f64) -> Self {
        Self { radius }
    }

    fn area(&self) -> f64 {
        std::f64::consts::PI * self.radius * self.radius
    }
}

// 追加のメソッド
impl Circle {
    fn circumference(&self) -> f64 {
        2.0 * std::f64::consts::PI * self.radius
    }

    fn contains_point(&self, x: f64, y: f64) -> bool {
        x * x + y * y <= self.radius * self.radius
    }
}

fn main() {
    let c = Circle::new(5.0);
    println!("面積: {:.2}", c.area());
    println!("円周: {:.2}", c.circumference());
    println!("(3, 4) は内部: {}", c.contains_point(3.0, 4.0));
    println!("(4, 4) は内部: {}", c.contains_point(4.0, 4.0));
}
```

```
面積: 78.54
円周: 31.42
(3, 4) は内部: true
(4, 4) は内部: false
```

### メソッドチェーン（Builder パターン）

`&mut self` を返すメソッドを作ると、メソッドチェーンが実現できます。これは現場でよく使われる「Builder パターン」です。

```rust
struct QueryBuilder {
    table: String,
    conditions: Vec<String>,
    limit: Option<usize>,
}

impl QueryBuilder {
    fn new(table: &str) -> Self {
        Self {
            table: table.to_string(),
            conditions: Vec::new(),
            limit: None,
        }
    }

    fn where_clause(&mut self, condition: &str) -> &mut Self {
        self.conditions.push(condition.to_string());
        self
    }

    fn limit(&mut self, n: usize) -> &mut Self {
        self.limit = Some(n);
        self
    }

    fn build(&self) -> String {
        let mut query = format!("SELECT * FROM {}", self.table);
        if !self.conditions.is_empty() {
            query.push_str(" WHERE ");
            query.push_str(&self.conditions.join(" AND "));
        }
        if let Some(limit) = self.limit {
            query.push_str(&format!(" LIMIT {}", limit));
        }
        query
    }
}

fn main() {
    let query = QueryBuilder::new("users")
        .where_clause("age >= 18")
        .where_clause("active = true")
        .limit(10)
        .build();

    println!("{}", query);
}
```

```
SELECT * FROM users WHERE age >= 18 AND active = true LIMIT 10
```

### ポイントまとめ

| 項目 | 内容 |
|------|------|
| impl ブロック | `impl StructName { ... }` でメソッドを定義 |
| メソッド | 第1引数が `self` / `&self` / `&mut self` |
| 関連関数 | `self` なし。`Type::function()` で呼び出す |
| Self | impl 対象の型のエイリアス |
| Builder パターン | `&mut self` を返してメソッドチェーン |

## 6.4 列挙型（enum）の定義

### 列挙型とは

列挙型（enum, enumeration）は、取りうる値のバリアント（variant）を列挙して定義する型です。

```rust
enum Direction {
    North,
    South,
    East,
    West,
}

fn describe_direction(dir: &Direction) -> &str {
    match dir {
        Direction::North => "北",
        Direction::South => "南",
        Direction::East => "東",
        Direction::West => "西",
    }
}

fn main() {
    let dir = Direction::North;
    println!("方角: {}", describe_direction(&dir));

    let dirs = vec![
        Direction::North,
        Direction::East,
        Direction::South,
        Direction::West,
    ];

    for d in &dirs {
        println!("→ {}", describe_direction(d));
    }
}
```

```
方角: 北
→ 北
→ 東
→ 南
→ 西
```

### enum にデータを持たせる

Rustの enum は各バリアントに異なる型のデータを持たせることができます。これはC言語やJavaの enum にはない強力な機能です。

```
┌──────────────────────────────────────────────────┐
│  C/Java の enum:                                  │
│    enum Color { Red, Green, Blue }               │
│    → 値は定数だけ                                  │
│                                                  │
│  Rust の enum:                                    │
│    enum Message {                                │
│        Quit,                     ← データなし      │
│        Echo(String),             ← String を持つ   │
│        Move { x: i32, y: i32 }, ← 構造体風       │
│        Color(u8, u8, u8),       ← タプル風        │
│    }                                             │
│    → 各バリアントが異なるデータを持てる！              │
└──────────────────────────────────────────────────┘
```

```rust
enum Message {
    Quit,
    Echo(String),
    Move { x: i32, y: i32 },
    ChangeColor(u8, u8, u8),
}

fn process_message(msg: &Message) {
    match msg {
        Message::Quit => {
            println!("終了します");
        }
        Message::Echo(text) => {
            println!("エコー: {}", text);
        }
        Message::Move { x, y } => {
            println!("移動: ({}, {})", x, y);
        }
        Message::ChangeColor(r, g, b) => {
            println!("色変更: RGB({}, {}, {})", r, g, b);
        }
    }
}

fn main() {
    let messages = vec![
        Message::Echo(String::from("Hello!")),
        Message::Move { x: 10, y: 20 },
        Message::ChangeColor(255, 128, 0),
        Message::Quit,
    ];

    for msg in &messages {
        process_message(msg);
    }
}
```

```
エコー: Hello!
移動: (10, 20)
色変更: RGB(255, 128, 0)
終了します
```

### enum にメソッドを定義

構造体と同様に、enum にも `impl` でメソッドを定義できます。

```rust
enum Coin {
    One,
    Five,
    Ten,
    Fifty,
    Hundred,
    FiveHundred,
}

impl Coin {
    fn value(&self) -> i32 {
        match self {
            Coin::One => 1,
            Coin::Five => 5,
            Coin::Ten => 10,
            Coin::Fifty => 50,
            Coin::Hundred => 100,
            Coin::FiveHundred => 500,
        }
    }

    fn name(&self) -> &str {
        match self {
            Coin::One => "1円",
            Coin::Five => "5円",
            Coin::Ten => "10円",
            Coin::Fifty => "50円",
            Coin::Hundred => "100円",
            Coin::FiveHundred => "500円",
        }
    }
}

fn main() {
    let wallet = vec![
        Coin::Hundred,
        Coin::FiveHundred,
        Coin::Fifty,
        Coin::Ten,
        Coin::Ten,
        Coin::Five,
        Coin::One,
        Coin::One,
    ];

    let total: i32 = wallet.iter().map(|c| c.value()).sum();
    println!("=== お財布の中身 ===");
    for coin in &wallet {
        println!("  {} ({}円)", coin.name(), coin.value());
    }
    println!("合計: {}円", total);
}
```

```
=== お財布の中身 ===
  100円 (100円)
  500円 (500円)
  50円 (50円)
  10円 (10円)
  10円 (10円)
  5円 (5円)
  1円 (1円)
  1円 (1円)
合計: 677円
```

### ポイントまとめ

- `enum` で取りうる値のバリアントを定義する
- 各バリアントは異なるデータを保持できる
- `match` で全バリアントを網羅的に処理する
- `impl` でメソッドを追加できる

## 6.5 Option\<T\> 型の活用

### Option\<T\> とは

Rustには `null` がありません。代わりに「値があるかもしれないし、ないかもしれない」状態を `Option<T>` 型で表現します。

```
┌────────────────────────────────────────────────┐
│  Option<T> の定義（標準ライブラリ）:               │
│                                                │
│  enum Option<T> {                              │
│      Some(T),   // 値がある                      │
│      None,      // 値がない                      │
│  }                                             │
│                                                │
│  他言語の null/nil に相当するが、                  │
│  コンパイル時に安全性が保証される！                 │
└────────────────────────────────────────────────┘
```

```rust
fn main() {
    // Option<i32> の例
    let some_number: Option<i32> = Some(42);
    let no_number: Option<i32> = None;

    println!("some_number: {:?}", some_number);
    println!("no_number: {:?}", no_number);

    // Option は直接演算できない（安全！）
    // let sum = some_number + 1;  // エラー！
}
```

```
some_number: Some(42)
no_number: None
```

### match で Option を処理する

```rust
fn find_user(id: i32) -> Option<String> {
    match id {
        1 => Some(String::from("Alice")),
        2 => Some(String::from("Bob")),
        3 => Some(String::from("Charlie")),
        _ => None,
    }
}

fn main() {
    for id in 1..=4 {
        match find_user(id) {
            Some(name) => println!("ID {}: ユーザー「{}」が見つかりました", id, name),
            None => println!("ID {}: ユーザーが見つかりません", id),
        }
    }
}
```

```
ID 1: ユーザー「Alice」が見つかりました
ID 2: ユーザー「Bob」が見つかりました
ID 3: ユーザー「Charlie」が見つかりました
ID 4: ユーザーが見つかりません
```

### if let による簡潔な処理

`match` でひとつのバリアントだけ処理したい場合は `if let` が便利です。

```rust
fn main() {
    let config_value: Option<i32> = Some(100);

    // match で書く場合
    match config_value {
        Some(v) => println!("設定値: {}", v),
        None => {}  // 何もしない
    }

    // if let で書く場合（より簡潔）
    if let Some(v) = config_value {
        println!("設定値（if let）: {}", v);
    }

    // else も使える
    let missing: Option<i32> = None;
    if let Some(v) = missing {
        println!("値: {}", v);
    } else {
        println!("値が設定されていません");
    }
}
```

```
設定値: 100
設定値（if let）: 100
値が設定されていません
```

### Option の便利メソッド

```rust
fn main() {
    let some_val: Option<i32> = Some(42);
    let none_val: Option<i32> = None;

    // unwrap_or: None の場合のデフォルト値
    println!("some: {}", some_val.unwrap_or(0));
    println!("none: {}", none_val.unwrap_or(0));

    // unwrap_or_else: None の場合にクロージャで計算
    let default = none_val.unwrap_or_else(|| {
        println!("デフォルト値を計算中...");
        99
    });
    println!("default: {}", default);

    // map: Some の中身を変換
    let doubled = some_val.map(|x| x * 2);
    println!("doubled: {:?}", doubled);

    let none_doubled = none_val.map(|x| x * 2);
    println!("none_doubled: {:?}", none_doubled);

    // and_then: Some なら別の Option を返す処理（チェーン）
    let result = some_val
        .and_then(|x| if x > 0 { Some(x * 10) } else { None })
        .and_then(|x| if x < 1000 { Some(x) } else { None });
    println!("chained: {:?}", result);

    // is_some / is_none
    println!("some_val は Some: {}", some_val.is_some());
    println!("none_val は None: {}", none_val.is_none());
}
```

```
some: 42
none: 0
デフォルト値を計算中...
default: 99
doubled: Some(84)
none_doubled: None
chained: Some(420)
some_val は Some: true
none_val は None: true
```

### 実践例：設定の取得

```rust
struct Config {
    database_url: Option<String>,
    port: Option<u16>,
    debug_mode: Option<bool>,
}

impl Config {
    fn new() -> Self {
        Self {
            database_url: None,
            port: None,
            debug_mode: None,
        }
    }

    fn database_url(&self) -> &str {
        self.database_url
            .as_deref()
            .unwrap_or("localhost:5432")
    }

    fn port(&self) -> u16 {
        self.port.unwrap_or(8080)
    }

    fn is_debug(&self) -> bool {
        self.debug_mode.unwrap_or(false)
    }
}

fn main() {
    // デフォルト設定
    let default_config = Config::new();
    println!("=== デフォルト設定 ===");
    println!("DB URL: {}", default_config.database_url());
    println!("Port: {}", default_config.port());
    println!("Debug: {}", default_config.is_debug());

    // カスタム設定
    let custom_config = Config {
        database_url: Some(String::from("db.example.com:5432")),
        port: Some(3000),
        debug_mode: Some(true),
    };
    println!("\n=== カスタム設定 ===");
    println!("DB URL: {}", custom_config.database_url());
    println!("Port: {}", custom_config.port());
    println!("Debug: {}", custom_config.is_debug());
}
```

```
=== デフォルト設定 ===

Port: 8080
Debug: false

=== カスタム設定 ===

Port: 3000
Debug: true
```

### ポイントまとめ

| メソッド | 動作 |
|----------|------|
| `unwrap()` | Some なら中身を返す。None ならパニック |
| `unwrap_or(default)` | None ならデフォルト値 |
| `unwrap_or_else(\|\| expr)` | None ならクロージャで計算 |
| `map(f)` | Some の中身を変換 |
| `and_then(f)` | Some なら別の Option を返す |
| `is_some()` / `is_none()` | Some/None の判定 |

> **現場のベストプラクティス：** `unwrap()` は本番コードでは避けましょう。代わりに `unwrap_or`, `unwrap_or_else`, `?` 演算子（第7章で学びます）を使います。

## 6.6 match と enum の組み合わせ

### 網羅的パターンマッチング

Rustの `match` は全バリアントを網羅する必要があります。漏れがあるとコンパイルエラーになります。

```rust
enum Season {
    Spring,
    Summer,
    Autumn,
    Winter,
}

fn season_greeting(season: &Season) -> &str {
    match season {
        Season::Spring => "桜が咲く季節ですね",
        Season::Summer => "暑い日が続きますね",
        Season::Autumn => "食欲の秋ですね",
        Season::Winter => "温かくしてお過ごしください",
        // すべてのバリアントをカバーしないとエラー
    }
}

fn main() {
    let seasons = [Season::Spring, Season::Summer, Season::Autumn, Season::Winter];
    for season in &seasons {
        println!("{}", season_greeting(season));
    }
}
```

```
桜が咲く季節ですね
暑い日が続きますね
食欲の秋ですね
温かくしてお過ごしください
```

### データを持つ enum のパターンマッチ

```rust
enum Shape {
    Circle(f64),                     // 半径
    Rectangle(f64, f64),             // 幅, 高さ
    Triangle { base: f64, height: f64 }, // 底辺, 高さ
}

impl Shape {
    fn area(&self) -> f64 {
        match self {
            Shape::Circle(radius) => {
                std::f64::consts::PI * radius * radius
            }
            Shape::Rectangle(width, height) => {
                width * height
            }
            Shape::Triangle { base, height } => {
                base * height / 2.0
            }
        }
    }

    fn name(&self) -> &str {
        match self {
            Shape::Circle(_) => "円",
            Shape::Rectangle(_, _) => "長方形",
            Shape::Triangle { .. } => "三角形",
        }
    }
}

fn main() {
    let shapes = vec![
        Shape::Circle(5.0),
        Shape::Rectangle(10.0, 3.0),
        Shape::Triangle { base: 8.0, height: 6.0 },
    ];

    for shape in &shapes {
        println!("{}: 面積 = {:.2}", shape.name(), shape.area());
    }
}
```

```
円: 面積 = 78.54
長方形: 面積 = 30.00
三角形: 面積 = 24.00
```

### ワイルドカードとキャッチオール

```rust
fn main() {
    let number = 13;

    // _ で残り全部をキャッチ
    let description = match number {
        1 => "一",
        2 => "二",
        3 => "三",
        _ => "その他",  // ワイルドカード
    };
    println!("{} は「{}」", number, description);

    // 変数束縛で値を使う
    let message = match number {
        0 => String::from("ゼロ"),
        1..=9 => String::from("一桁"),
        10..=99 => format!("二桁の数: {}", number),
        _ => format!("大きな数: {}", number),
    };
    println!("{}", message);
}
```

```
13 は「その他」
二桁の数: 13
```

### ガード条件（match guard）

```rust
enum Temperature {
    Celsius(f64),
    Fahrenheit(f64),
}

fn describe_temp(temp: &Temperature) -> String {
    match temp {
        Temperature::Celsius(c) if *c < 0.0 => {
            format!("{}°C - 氷点下です！", c)
        }
        Temperature::Celsius(c) if *c < 15.0 => {
            format!("{}°C - 寒いです", c)
        }
        Temperature::Celsius(c) if *c < 25.0 => {
            format!("{}°C - 快適です", c)
        }
        Temperature::Celsius(c) => {
            format!("{}°C - 暑いです", c)
        }
        Temperature::Fahrenheit(f) => {
            let c = (f - 32.0) * 5.0 / 9.0;
            format!("{}°F (= {:.1}°C)", f, c)
        }
    }
}

fn main() {
    let temps = vec![
        Temperature::Celsius(-5.0),
        Temperature::Celsius(10.0),
        Temperature::Celsius(22.0),
        Temperature::Celsius(35.0),
        Temperature::Fahrenheit(72.0),
    ];

    for temp in &temps {
        println!("{}", describe_temp(temp));
    }
}
```

```
-5°C - 氷点下です！
10°C - 寒いです
22°C - 快適です
35°C - 暑いです
72°F (= 22.2°C)
```

### ポイントまとめ

- `match` は全バリアントを網羅しないとコンパイルエラー
- `_` でワイルドカード（残り全部をキャッチ）
- `..` で構造体の残りのフィールドを無視
- ガード条件（`if` 節）で追加の条件を指定できる
- `1..=9` で範囲パターンが使える

## 6.7 トレイト（trait）の基礎：Debug と Display

### Debug トレイト

`Debug` トレイトを実装すると、`{:?}` でデバッグ出力できます。`#[derive(Debug)]` で自動導出できます。

```rust
#[derive(Debug)]
struct Point {
    x: f64,
    y: f64,
}

#[derive(Debug)]
enum Color {
    Red,
    Green,
    Blue,
    Custom(u8, u8, u8),
}

fn main() {
    let point = Point { x: 3.0, y: 4.0 };
    let color = Color::Custom(128, 64, 255);

    // {:?} でデバッグ出力
    println!("point: {:?}", point);
    println!("color: {:?}", color);

    // {:#?} で整形出力（pretty print）
    println!("point (pretty):\n{:#?}", point);

    // Vec の中身も表示できる
    let points = vec![
        Point { x: 0.0, y: 0.0 },
        Point { x: 1.0, y: 2.0 },
        Point { x: 3.0, y: 4.0 },
    ];
    println!("points: {:#?}", points);
}
```

```
point: Point { x: 3.0, y: 4.0 }
color: Custom(128, 64, 255)
point (pretty):
Point {
    x: 3.0,
    y: 4.0,
}
points: [
    Point {
        x: 0.0,
        y: 0.0,
    },
    Point {
        x: 1.0,
        y: 2.0,
    },
    Point {
        x: 3.0,
        y: 4.0,
    },
]
```

### Display トレイト

`Display` トレイトを実装すると、`{}` でユーザー向けの表示ができます。こちらは自動導出できないので、手動で実装します。

```rust
use std::fmt;

struct Point {
    x: f64,
    y: f64,
}

impl fmt::Display for Point {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "({}, {})", self.x, self.y)
    }
}

enum Color {
    Red,
    Green,
    Blue,
    Custom(u8, u8, u8),
}

impl fmt::Display for Color {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Color::Red => write!(f, "赤"),
            Color::Green => write!(f, "緑"),
            Color::Blue => write!(f, "青"),
            Color::Custom(r, g, b) => write!(f, "RGB({}, {}, {})", r, g, b),
        }
    }
}

fn main() {
    let p = Point { x: 3.0, y: 4.0 };
    let c = Color::Custom(255, 128, 0);

    // {} で Display を使用
    println!("座標: {}", p);
    println!("色: {}", c);

    // format! マクロでも使える
    let text = format!("点 {} の色は {}", p, c);
    println!("{}", text);
}
```

```
座標: (3, 4)
色: RGB(255, 128, 0)
点 (3, 4) の色は RGB(255, 128, 0)
```

### よく使う derive マクロ

```rust
#[derive(Debug, Clone, PartialEq)]
struct Product {
    name: String,
    price: i32,
}

fn main() {
    let product1 = Product {
        name: String::from("Rustの本"),
        price: 3000,
    };

    // Debug: {:?} で表示
    println!("{:?}", product1);

    // Clone: 明示的なコピー
    let product2 = product1.clone();
    println!("{:?}", product2);

    // PartialEq: == で比較
    println!("同じ商品: {}", product1 == product2);

    let product3 = Product {
        name: String::from("別の本"),
        price: 2000,
    };
    println!("同じ商品: {}", product1 == product3);
}
```

```
Product { name: "Rustの本", price: 3000 }
Product { name: "Rustの本", price: 3000 }
同じ商品: true
同じ商品: false
```

### ポイントまとめ

| トレイト | 用途 | 自動導出 | フォーマット |
|----------|------|----------|------------|
| `Debug` | デバッグ出力 | `#[derive(Debug)]` | `{:?}`, `{:#?}` |
| `Display` | ユーザー向け表示 | 不可（手動実装） | `{}` |
| `Clone` | 明示的なコピー | `#[derive(Clone)]` | `.clone()` |
| `PartialEq` | 等値比較 | `#[derive(PartialEq)]` | `==`, `!=` |

## 6.8 実践演習：構造体と列挙型の総合例

### タスク管理システム

構造体と列挙型を組み合わせた実践的な例です。

```rust
use std::fmt;

#[derive(Debug, Clone, PartialEq)]
enum Priority {
    Low,
    Medium,
    High,
    Critical,
}

impl fmt::Display for Priority {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Priority::Low => write!(f, "低"),
            Priority::Medium => write!(f, "中"),
            Priority::High => write!(f, "高"),
            Priority::Critical => write!(f, "緊急"),
        }
    }
}

#[derive(Debug, Clone, PartialEq)]
enum Status {
    Todo,
    InProgress,
    Done,
}

impl fmt::Display for Status {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Status::Todo => write!(f, "未着手"),
            Status::InProgress => write!(f, "進行中"),
            Status::Done => write!(f, "完了"),
        }
    }
}

#[derive(Debug, Clone)]
struct Task {
    id: u32,
    title: String,
    priority: Priority,
    status: Status,
}

impl Task {
    fn new(id: u32, title: &str, priority: Priority) -> Self {
        Self {
            id,
            title: title.to_string(),
            priority,
            status: Status::Todo,
        }
    }

    fn start(&mut self) {
        self.status = Status::InProgress;
    }

    fn complete(&mut self) {
        self.status = Status::Done;
    }

    fn is_done(&self) -> bool {
        self.status == Status::Done
    }
}

impl fmt::Display for Task {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(
            f,
            "[{}] #{}: {} (優先度: {}, 状態: {})",
            if self.is_done() { "x" } else { " " },
            self.id,
            self.title,
            self.priority,
            self.status
        )
    }
}

struct TaskManager {
    tasks: Vec<Task>,
    next_id: u32,
}

impl TaskManager {
    fn new() -> Self {
        Self {
            tasks: Vec::new(),
            next_id: 1,
        }
    }

    fn add(&mut self, title: &str, priority: Priority) -> u32 {
        let id = self.next_id;
        self.tasks.push(Task::new(id, title, priority));
        self.next_id += 1;
        id
    }

    fn start_task(&mut self, id: u32) -> bool {
        if let Some(task) = self.tasks.iter_mut().find(|t| t.id == id) {
            task.start();
            true
        } else {
            false
        }
    }

    fn complete_task(&mut self, id: u32) -> bool {
        if let Some(task) = self.tasks.iter_mut().find(|t| t.id == id) {
            task.complete();
            true
        } else {
            false
        }
    }

    fn pending_tasks(&self) -> Vec<&Task> {
        self.tasks.iter().filter(|t| !t.is_done()).collect()
    }

    fn print_all(&self) {
        if self.tasks.is_empty() {
            println!("  (タスクなし)");
            return;
        }
        for task in &self.tasks {
            println!("  {}", task);
        }
    }
}

fn main() {
    let mut manager = TaskManager::new();

    // タスクを追加
    manager.add("Rust の環境構築", Priority::High);
    manager.add("変数と所有権の学習", Priority::High);
    manager.add("構造体の演習問題を解く", Priority::Medium);
    manager.add("テストコードを書く", Priority::Low);
    manager.add("本番デプロイ", Priority::Critical);

    println!("=== 全タスク ===");
    manager.print_all();

    // タスクを進行
    manager.start_task(1);
    manager.complete_task(1);
    manager.start_task(2);

    println!("\n=== 更新後 ===");
    manager.print_all();

    // 未完了タスク
    let pending = manager.pending_tasks();
    println!("\n=== 未完了タスク（{}件） ===", pending.len());
    for task in pending {
        println!("  {}", task);
    }
}
```

```
=== 全タスク ===
  [ ] #1: Rust の環境構築 (優先度: 高, 状態: 未着手)
  [ ] #2: 変数と所有権の学習 (優先度: 高, 状態: 未着手)
  [ ] #3: 構造体の演習問題を解く (優先度: 中, 状態: 未着手)
  [ ] #4: テストコードを書く (優先度: 低, 状態: 未着手)
  [ ] #5: 本番デプロイ (優先度: 緊急, 状態: 未着手)

=== 更新後 ===
  [x] #1: Rust の環境構築 (優先度: 高, 状態: 完了)
  [ ] #2: 変数と所有権の学習 (優先度: 高, 状態: 進行中)
  [ ] #3: 構造体の演習問題を解く (優先度: 中, 状態: 未着手)
  [ ] #4: テストコードを書く (優先度: 低, 状態: 未着手)
  [ ] #5: 本番デプロイ (優先度: 緊急, 状態: 未着手)

=== 未完了タスク（4件） ===
  [ ] #2: 変数と所有権の学習 (優先度: 高, 状態: 進行中)
  [ ] #3: 構造体の演習問題を解く (優先度: 中, 状態: 未着手)
  [ ] #4: テストコードを書く (優先度: 低, 状態: 未着手)
  [ ] #5: 本番デプロイ (優先度: 緊急, 状態: 未着手)
```
