## 問題1：セキュリティヘッダーの理解（基本）

以下の各セキュリティヘッダーが防ぐ攻撃を答えてください。

1. `X-Frame-Options: SAMEORIGIN`
2. `X-Content-Type-Options: nosniff`
3. `Strict-Transport-Security: max-age=63072000`
4. `Content-Security-Policy: default-src 'self'`
5. `X-XSS-Protection: 1; mode=block`

<details>
<summary>解答例</summary>

```
1. X-Frame-Options: SAMEORIGIN
   → クリックジャッキング攻撃を防ぐ。
   攻撃者のサイトの iframe に自分のサイトを埋め込むことを禁止する。
   SAMEORIGIN は同一オリジンからのみ iframe 埋め込みを許可。

2. X-Content-Type-Options: nosniff
   → MIMEタイプスニッフィング攻撃を防ぐ。
   ブラウザが Content-Type を無視してファイル内容から
   タイプを推測することを禁止。悪意あるスクリプトが
   画像ファイルとして実行されることを防ぐ。

3. Strict-Transport-Security: max-age=63072000
   → HTTPダウングレード攻撃を防ぐ（HSTS）。
   ブラウザに「このサイトには常にHTTPSでアクセスせよ」と指示。
   max-age は有効期間（秒）で、63072000秒 ≒ 2年間。

4. Content-Security-Policy: default-src 'self'
   → クロスサイトスクリプティング（XSS）や
   データインジェクション攻撃を防ぐ。
   'self' は自分のオリジンからのみリソース読み込みを許可。

5. X-XSS-Protection: 1; mode=block
   → ブラウザ組み込みのXSSフィルターを有効化。
   XSS攻撃を検出した場合、ページの描画をブロックする。
   ※ 現代のブラウザではCSPが推奨されており、
     このヘッダーは補助的な役割。
```

</details>

## 問題2：アクセス制御の設定（基本）

以下の要件を満たすnginx設定を書いてください。

1. `/admin/` は社内ネットワーク（192.168.1.0/24）からのみアクセス可能
2. `/api/health` は全IPからアクセス可能（ヘルスチェック用）
3. `/api/` は認証付き（Basic認証、ファイル: `/etc/nginx/.htpasswd`）

<details>
<summary>解答例</summary>

```nginx
server {
    listen 80;
    server_name example.com;

    # 1. /admin/ は社内ネットワークのみ
    location /admin/ {
        allow 192.168.1.0/24;
        deny all;
        root /var/www/admin;
    }

    # 2. ヘルスチェックは全IPから（認証なし）
    #    /api/ より前に書く（より具体的な方を先に）
    location = /api/health {
        auth_basic off;
        proxy_pass http://localhost:3000;
    }

    # 3. /api/ はBasic認証付き
    location /api/ {
        auth_basic "API Access";
        auth_basic_user_file /etc/nginx/.htpasswd;
        proxy_pass http://localhost:3000;
    }
}
```

</details>

## 問題3：allow/deny の順序問題（基本）

以下の2つの設定で、`192.168.1.10` からのアクセスはそれぞれ許可されますか？

**設定A：**
```nginx
location /admin/ {
    allow 192.168.1.0/24;
    deny all;
}
```

**設定B：**
```nginx
location /admin/ {
    deny all;
    allow 192.168.1.0/24;
}
```

<details>
<summary>解答例</summary>

```
設定A: 許可される ✓
  1. allow 192.168.1.0/24 → 192.168.1.10 にマッチ → 許可
  ※ 上から順に評価し、最初にマッチしたルールが適用される

設定B: 拒否される ✗
  1. deny all → 全IPにマッチ → 拒否
  ※ deny all が先に書かれているため、
    後続の allow は評価されない

→ allow/deny は必ず「許可→拒否」の順で書く。
  deny all は常に最後に配置する。
```

</details>

## 問題4：レートリミットの設定（応用）

以下の要件を満たすレートリミット設定を書いてください。

- `/api/login` : 1IPあたり1秒1リクエスト、バースト5まで
- `/api/` 全体 : 1IPあたり1秒30リクエスト、バースト50まで（即時処理）
- 制限超過時のレスポンスを 429（Too Many Requests）にする

<details>
<summary>解答例</summary>

```nginx
http {
    # レートリミットゾーンの定義
    limit_req_zone $binary_remote_addr zone=login:10m rate=1r/s;
    limit_req_zone $binary_remote_addr zone=api:10m rate=30r/s;

    # 制限超過時のステータスコード
    limit_req_status 429;

    server {
        listen 80;
        server_name api.example.com;

        # ログイン（厳しく制限）
        location /api/login {
            limit_req zone=login burst=5;
            proxy_pass http://localhost:3000;
        }

        # API全体（通常の制限）
        location /api/ {
            limit_req zone=api burst=50 nodelay;
            proxy_pass http://localhost:3000;
        }
    }
}
```

</details>

## 問題5：隠しファイルの保護（応用）

以下のファイル/ディレクトリへのアクセスを適切に制御する設定を書いてください。

- `.git/`, `.env`, `.htaccess` 等のドットファイル → 拒否
- `.well-known/` (Let's Encrypt認証用) → 許可
- `.bak`, `.sql`, `.log` 等のバックアップ/ログファイル → 拒否
- `wp-admin/`, `wp-login.php` (WordPress管理画面) → IP制限

<details>
<summary>解答例</summary>

```nginx
server {
    listen 80;
    server_name example.com;

    # Let's Encrypt 認証用は許可（.well-known）
    location ^~ /.well-known/ {
        allow all;
    }

    # その他のドットファイル/ディレクトリへのアクセスを拒否
    location ~ /\. {
        deny all;
        access_log off;
        log_not_found off;
    }

    # バックアップ・ログファイルへのアクセスを拒否
    location ~* \.(bak|sql|log|ini|sh|config|dist|swp)$ {
        deny all;
        access_log off;
        log_not_found off;
    }

    # WordPress管理画面はIP制限
    location /wp-admin/ {
        allow 192.168.1.0/24;
        deny all;
    }

    location = /wp-login.php {
        allow 192.168.1.0/24;
        deny all;
        # FastCGI設定（PHPの場合）
        # include snippets/fastcgi-php.conf;
        # fastcgi_pass unix:/run/php/php-fpm.sock;
    }
}
```

</details>

## 問題6：セキュリティ設定の監査（応用）

以下のnginx設定にはセキュリティ上の問題が5つあります。すべて指摘し、修正してください。

```nginx
server {
    listen 80;
    server_name example.com;
    root /var/www/html;

    location / {
        autoindex on;
        try_files $uri $uri/ =404;
    }

    location /api/ {
        proxy_pass http://localhost:3000;
    }
}
```

<details>
<summary>ヒント</summary>

以下の観点でチェックしてください。
1. バージョン情報の露出
2. セキュリティヘッダーの不足
3. ディレクトリリスティング
4. 隠しファイルへのアクセス
5. HTTPS対応

</details>

<details>
<summary>解答例</summary>

```
問題点と修正:

1. server_tokens が設定されていない
   → server_tokens off; を追加

2. セキュリティヘッダーが一切設定されていない
   → X-Content-Type-Options, X-Frame-Options 等を追加

3. autoindex on; でディレクトリ一覧が表示される
   → autoindex off; に変更（またはディレクティブを削除）
   → ファイルの一覧が外部に見えてしまう

4. 隠しファイル（.git, .env等）へのアクセスが制限されていない
   → location ~ /\. { deny all; } を追加

5. HTTPSへのリダイレクトがなく、平文通信
   → 443 で SSL を設定し、80 からリダイレクト
```

```nginx
# 修正後
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;  # HTTPSへリダイレクト
}

server {
    listen 443 ssl;
    server_name example.com;
    root /var/www/html;

    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    server_tokens off;                                    # 修正1

    add_header X-Content-Type-Options "nosniff" always;   # 修正2
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Strict-Transport-Security "max-age=63072000" always;

    # 修正4: 隠しファイルへのアクセス拒否
    location ~ /\. {
        deny all;
        access_log off;
        log_not_found off;
    }

    location / {
        # autoindex on; を削除（修正3）
        try_files $uri $uri/ =404;
    }

    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

</details>

## 問題7：総合セキュリティ設定（チャレンジ）

本番環境のWebアプリケーション用に、以下の要件を満たすセキュリティ設定を作成してください。

**要件：**
- HTTPS強制（HTTPからリダイレクト）
- セキュリティヘッダーをすべて適用
- `/admin/` は社内IP（10.0.0.0/8）のみ + Basic認証
- `/api/login` にレートリミット（1秒1リクエスト）
- 隠しファイルへのアクセス拒否（.well-known は除外）
- ファイルアップロード上限: 50MB
- バージョン情報非表示

<details>
<summary>解答例</summary>

```nginx
# === レートリミットゾーン ===
http {
    limit_req_zone $binary_remote_addr zone=login:10m rate=1r/s;
    limit_req_status 429;
    server_tokens off;

    # === HTTP → HTTPS リダイレクト ===
    server {
        listen 80;
        server_name app.example.com;
        return 301 https://$server_name$request_uri;
    }

    # === メインサーバー（HTTPS）===
    server {
        listen 443 ssl http2;
        server_name app.example.com;

        root /var/www/app;
        index index.html;
        client_max_body_size 50m;

        # --- SSL ---
        ssl_certificate /etc/letsencrypt/live/app.example.com/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/app.example.com/privkey.pem;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_prefer_server_ciphers on;

        # --- セキュリティヘッダー ---
        add_header X-XSS-Protection "1; mode=block" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header Referrer-Policy "strict-origin-when-cross-origin" always;
        add_header Strict-Transport-Security "max-age=63072000; includeSubDomains" always;
        add_header Permissions-Policy "camera=(), microphone=()" always;
        add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline';" always;

        # --- .well-known は許可 ---
        location ^~ /.well-known/ {
            allow all;
        }

        # --- 隠しファイルを拒否 ---
        location ~ /\. {
            deny all;
            access_log off;
            log_not_found off;
        }

        # --- 管理画面（IP制限 + Basic認証）---
        location /admin/ {
            allow 10.0.0.0/8;
            deny all;
            auth_basic "Admin Area";
            auth_basic_user_file /etc/nginx/.htpasswd;
            try_files $uri $uri/ /index.html;
        }

        # --- ログインのレートリミット ---
        location /api/login {
            limit_req zone=login burst=5;
            proxy_pass http://localhost:3000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # --- API ---
        location /api/ {
            proxy_pass http://localhost:3000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # --- 静的ファイル ---
        location / {
            try_files $uri $uri/ /index.html;
        }
    }
}
```

</details>
