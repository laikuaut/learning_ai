# 第5章：キーマッピング

## この章のゴール

- `map` と `noremap` の違いを理解する
- モード別のマッピングコマンドを使い分けられる
- 特殊キー（`<CR>`, `<Esc>`, `<C-x>`, `<A-x>`, `<Leader>` 等）の記法を覚える
- Leader キーを設定し、効率的なキーバインドを構築できる
- 実務で役立つマッピングを自分の `.vimrc` に追加できる
- マッピングの一覧表示やデバッグができる

---

## 5.1 マッピングの基本

**キーマッピング（key mapping）** とは、あるキー入力を別のキー操作に割り当てる機能です。
Vim の操作を自分好みにカスタマイズする最も基本的な手段です。

### map と noremap の違い

マッピングには **再帰的（recursive）** と **非再帰的（non-recursive）** の2種類があります。

```vim
" 再帰的マッピング（map）
" → 右辺がさらにマッピング展開される
map j gj
map k gk

" 非再帰的マッピング（noremap）
" → 右辺はマッピング展開されない（常にデフォルトの動作）
noremap j gj
noremap k gk
```

### 再帰マッピングの危険性

以下の例で、再帰マッピングの問題を図解します。

```
【再帰的マッピング（map）の場合】

  map j gj
  map gj j

  j を押す → gj に展開 → gj が j に展開 → j が gj に展開 → ...
  ┌───────────────────────────────────────┐
  │  j → gj → j → gj → j → ...（無限ループ！）│
  └───────────────────────────────────────┘

【非再帰的マッピング（noremap）の場合】

  noremap j gj
  noremap gj j

  j を押す → gj（デフォルトの gj が実行される。展開はここで終了）
  ┌───────────────────────────────────────┐
  │  j → gj（デフォルト動作）  ── 安全！    │
  └───────────────────────────────────────┘
```

### なぜ noremap を使うべきか

- `map` は右辺がさらにマッピング展開されるため、意図しない動作や無限ループの原因になります
- `noremap` は右辺を常にデフォルトのキー操作として解釈するため、安全です
- **原則：常に `noremap` 系を使いましょう。`map` 系はプラグインのマッピングを呼び出す場合にのみ使います**

```vim
" ◯ 推奨：noremap を使う
nnoremap Y y$

" △ 非推奨：map を使う（他のマッピングの影響を受ける可能性あり）
nmap Y y$
```

> **ポイントまとめ**
> - `map` は再帰的マッピング（右辺がさらに展開される）
> - `noremap` は非再帰的マッピング（右辺はデフォルト動作）
> - 安全のため **常に `noremap` 系を使う** のがベストプラクティス
> - `map` 系はプラグインの `<Plug>` マッピングを呼ぶときだけ使う

## 5.2 モード別マッピング

Vim にはモードごとに異なるマッピングコマンドがあります。
モード別に設定することで、同じキーでもモードに応じた動作を割り当てられます。

### マッピングコマンド一覧表

| 再帰的（map系） | 非再帰的（noremap系） | 対象モード | 説明 |
|---|---|---|---|
| `nmap` | `nnoremap` | ノーマル | ノーマルモード専用 |
| `imap` | `inoremap` | インサート | 挿入モード専用 |
| `vmap` | `vnoremap` | ビジュアル + 選択 | ビジュアルモード＋選択モード |
| `xmap` | `xnoremap` | ビジュアル | ビジュアルモード専用 |
| `smap` | `snoremap` | 選択 | 選択モード専用 |
| `cmap` | `cnoremap` | コマンドライン | コマンドラインモード専用 |
| `omap` | `onoremap` | オペレータ待ち | オペレータ待ちモード専用 |
| `map` | `noremap` | ノーマル + ビジュアル + オペレータ待ち | 複数モード一括 |

```
【Vim のモードとマッピングの対応】

                    ┌─── nmap / nnoremap
                    │
  ┌────────────┐    ├─── imap / inoremap
  │            │    │
  │   Vim の   │────├─── xmap / xnoremap（ビジュアルのみ）
  │   モード   │    │
  │            │────├─── vmap / vnoremap（ビジュアル＋選択）
  │            │    │
  └────────────┘    ├─── cmap / cnoremap
                    │
                    └─── omap / onoremap
```

### 使い分けの例

```vim
" ノーマルモード：Y で行末までヤンク
nnoremap Y y$

" 挿入モード：jj で Esc（ノーマルモードに戻る）
inoremap jj <Esc>

" ビジュアルモード：インデント後に選択を維持
xnoremap < <gv
xnoremap > >gv

" コマンドライン：Ctrl+a で行頭に移動
cnoremap <C-a> <Home>

" オペレータ待ち：il で「行内の内容」をテキストオブジェクトに
onoremap il :<C-u>normal! ^vg_<CR>
```

### vmap と xmap の違い

- `vmap` / `vnoremap` はビジュアルモード **と** 選択モードの両方で有効です
- `xmap` / `xnoremap` はビジュアルモードのみで有効です
- 通常は `xnoremap` を使うのが安全です（選択モードへの影響を避けるため）

> **ポイントまとめ**
> - モードごとに専用のマッピングコマンドがある
> - 最もよく使うのは `nnoremap`（ノーマル）と `inoremap`（挿入）
> - ビジュアルモードには `xnoremap` を推奨（`vnoremap` は選択モードにも影響）
> - `map` / `noremap`（プレフィックスなし）は複数モードに一括で設定される

## 5.3 特殊キーの記法

マッピングで使用する特殊キーは、`< >` で囲んだ記法で表現します。

### 特殊キー一覧表

| 記法 | キー | 説明 |
|---|---|---|
| `<CR>` | Enter | キャリッジリターン（Return） |
| `<Enter>` | Enter | `<CR>` と同じ |
| `<Esc>` | Escape | エスケープキー |
| `<Space>` | スペース | スペースキー |
| `<Tab>` | Tab | タブキー |
| `<BS>` | Backspace | バックスペースキー |
| `<Del>` | Delete | デリートキー |
| `<Up>` | ↑ | 上矢印キー |
| `<Down>` | ↓ | 下矢印キー |
| `<Left>` | ← | 左矢印キー |
| `<Right>` | → | 右矢印キー |
| `<Home>` | Home | ホームキー |
| `<End>` | End | エンドキー |
| `<PageUp>` | PageUp | ページアップキー |
| `<PageDown>` | PageDown | ページダウンキー |
| `<F1>` 〜 `<F12>` | ファンクションキー | F1〜F12キー |
| `<C-x>` | Ctrl+x | Ctrl と x の同時押し |
| `<A-x>` | Alt+x | Alt と x の同時押し |
| `<M-x>` | Meta+x | `<A-x>` と同じ（Alt） |
| `<S-x>` | Shift+x | Shift と x の同時押し |
| `<Leader>` | リーダーキー | 後述の Leader キー |
| `<LocalLeader>` | ローカルリーダー | バッファローカルの Leader |
| `<Bar>` | `\|` | パイプ文字 |
| `<lt>` | `<` | 小なり記号 |
| `<gt>` | `>` | 大なり記号 |
| `<Nop>` | 何もしない | キーを無効化するとき使用 |

### 修飾キーの組み合わせ

```vim
" Ctrl + s でファイル保存
nnoremap <C-s> :write<CR>

" Alt + j で行を下に移動
nnoremap <A-j> :move .+1<CR>==

" Shift + Tab でインデントを戻す（挿入モード）
inoremap <S-Tab> <C-d>

" 複数の修飾キーも組み合わせ可能
" Ctrl + Shift + s（ターミナルが対応している場合）
nnoremap <C-S-s> :wall<CR>
```

### 特殊な記法の使用例

```vim
" パイプ文字を含むマッピング
nnoremap <Leader>p :echo "hello" <Bar> echo "world"<CR>

" <lt> と <gt> でリテラルの < > を入力
" 例：挿入モードで HTML タグを素早く入力
inoremap <Leader>t <lt>div<gt><lt>/div<gt><Esc>F<i

" <Nop> でキーを無効化（誤操作防止）
nnoremap Q <Nop>
nnoremap <F1> <Nop>
```

> **ポイントまとめ**
> - 特殊キーは `<CR>`, `<Esc>`, `<C-x>` のように `< >` で囲んで記述する
> - `<C-x>` は Ctrl+x、`<A-x>` は Alt+x、`<S-x>` は Shift+x
> - `<Bar>` はパイプ文字、`<lt>` `<gt>` は `<` `>` のリテラル表現
> - `<Nop>` を使うとキーを無効化できる

## 5.4 Leader キー

**Leader キー** は、マッピングのプレフィックス（接頭辞）として使う特別なキーです。
Leader キーを活用すると、大量のカスタムマッピングを整理して管理できます。

### Leader キーの設定

```vim
" Leader キーを設定する（デフォルトは \ バックスラッシュ）
let mapleader = ","

" スペースキーを Leader にする（現在最も人気のある設定）
let mapleader = " "
```

### なぜスペースキーが人気なのか

```
【Leader キーとしてのスペースキーの利点】

  ┌─────────────────────────────────────────────┐
  │  キーボード上の位置                            │
  │                                               │
  │  ┌─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─────┐        │
  │  │ │ │ │ │ │ │ │ │ │ │ │ │ │     │        │
  │  ├──┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬────┤        │
  │  │   │ │ │ │ │ │ │ │ │ │ │ │ │    │        │
  │  ├───┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴┬┴────┤        │
  │  │    │ │ │ │ │ │ │ │ │ │ │ │      │        │
  │  ├────┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴──────┤        │
  │  │     ┌─────────────────────┐      │        │
  │  │     │  スペースキー ★     │      │        │
  │  │     │  両手の親指で押せる  │      │        │
  │  │     └─────────────────────┘      │        │
  │  └─────────────────────────────────┘        │
  │                                               │
  │  ・両手の親指で自然に押せる（最もアクセスしやすい） │
  │  ・ノーマルモードではスペースは l と同じ（失っても困らない） │
  │  ・キーボード上で最大のキー                    │
  └─────────────────────────────────────────────┘
```

### Leader キーの使い方

```vim
" まず Leader キーを設定（.vimrc の先頭付近で）
let mapleader = " "

" <Leader> を使ったマッピング
nnoremap <Leader>w :write<CR>
nnoremap <Leader>q :quit<CR>
nnoremap <Leader>e :edit<Space>

" 2文字のプレフィックスで分類する例
nnoremap <Leader>ff :find<Space>
nnoremap <Leader>fs :write<CR>
nnoremap <Leader>bn :bnext<CR>
nnoremap <Leader>bp :bprevious<CR>
```

### LocalLeader

`<LocalLeader>` はバッファローカルなマッピング用のリーダーキーです。
ファイルタイプ固有のマッピングに使います。

```vim
" LocalLeader の設定
let maplocalleader = ","

" ファイルタイプ固有のマッピング例
" Python ファイルで <LocalLeader>r で実行
autocmd FileType python nnoremap <buffer> <LocalLeader>r :!python3 %<CR>
```

### Leader キーのタイムアウト

Leader キーを押してから次のキーを押すまでの制限時間があります。

```vim
" タイムアウトの設定（ミリ秒）
set timeout
set timeoutlen=500    " 500ms（デフォルトは1000ms）
```

> **ポイントまとめ**
> - `let mapleader = " "` でスペースキーを Leader にするのが人気
> - Leader キーの設定は `.vimrc` の先頭付近で行うこと（マッピングより前）
> - `<Leader>` をプレフィックスにすることで、大量のマッピングを整理できる
> - `<LocalLeader>` はファイルタイプ固有のマッピングに使う
> - `timeoutlen` でリーダーキーのタイムアウトを調整できる

## 5.5 実用的なマッピング集

現場で広く使われている、実用的なマッピングを紹介します。
以下をそのまま `.vimrc` に追加して使えます。

### 挿入モードから素早く抜ける

```vim
" jj で Esc（挿入モードからノーマルモードへ）
inoremap jj <Esc>

" jk でも Esc（好みに応じて）
" inoremap jk <Esc>
```

### 検索ハイライトの解除

```vim
" Esc 2回で検索ハイライトを消す
nnoremap <Esc><Esc> :nohlsearch<CR>

" Leader + h でも可
nnoremap <Leader>h :nohlsearch<CR>
```

### ウィンドウ間の移動

```vim
" Ctrl + h/j/k/l でウィンドウ間を移動
nnoremap <C-h> <C-w>h
nnoremap <C-j> <C-w>j
nnoremap <C-k> <C-w>k
nnoremap <C-l> <C-w>l
```

### Y で行末までヤンク

```vim
" Y で行末までヤンク（D や C との一貫性）
" デフォルトの Y は yy と同じ動作で不自然
nnoremap Y y$
```

### ビジュアルモードでインデント維持

```vim
" インデント変更後もビジュアル選択を維持
xnoremap < <gv
xnoremap > >gv
```

### 行の移動（Alt + j/k）

```vim
" Alt + j で行を下に移動
nnoremap <A-j> :move .+1<CR>==
nnoremap <A-k> :move .-2<CR>==

" ビジュアルモードでも行を移動
xnoremap <A-j> :move '>+1<CR>gv=gv
xnoremap <A-k> :move '<-2<CR>gv=gv

" 挿入モードでも行を移動
inoremap <A-j> <Esc>:move .+1<CR>==gi
inoremap <A-k> <Esc>:move .-2<CR>==gi
```

### バッファの移動

```vim
" バッファの切り替え
nnoremap <Leader>bn :bnext<CR>
nnoremap <Leader>bp :bprevious<CR>
nnoremap <Leader>bd :bdelete<CR>

" Tab / Shift+Tab でバッファ切り替え
nnoremap <Tab> :bnext<CR>
nnoremap <S-Tab> :bprevious<CR>
```

### ファイル保存・終了のショートカット

```vim
" Ctrl + s で保存（ターミナルの設定によっては stty -ixon が必要）
nnoremap <C-s> :write<CR>
inoremap <C-s> <Esc>:write<CR>

" Leader + w で保存
nnoremap <Leader>w :write<CR>

" Leader + q で終了
nnoremap <Leader>q :quit<CR>

" Leader + Q で強制終了
nnoremap <Leader>Q :quit!<CR>

" Leader + x で保存して終了
nnoremap <Leader>x :xit<CR>
```

### その他の便利なマッピング

```vim
" 折り返された行でも見た目通りに移動
nnoremap j gj
nnoremap k gk

" 行頭・行末への移動を簡単に
nnoremap H ^
nnoremap L $

" 新しい行を追加（ノーマルモードのまま）
nnoremap <Leader>o o<Esc>
nnoremap <Leader>O O<Esc>

" 選択範囲をクリップボードにコピー
xnoremap <Leader>y "+y

" システムクリップボードから貼り付け
nnoremap <Leader>p "+p

" カーソル下の単語を置換する準備
nnoremap <Leader>s :%s/\<<C-r><C-w>\>//g<Left><Left>
```

> **ポイントまとめ**
> - `inoremap jj <Esc>` は最も人気のあるマッピングのひとつ
> - `<C-h/j/k/l>` によるウィンドウ移動は必須級の設定
> - `Y y$` は D, C との一貫性を保つための定番設定
> - `<A-j/k>` による行移動はコーディングで非常に便利
> - マッピングは少しずつ追加して、自分の手に馴染ませるのがコツ

## 5.6 マッピングの管理

マッピングが増えてきたら、一覧表示やデバッグの方法を知っておくことが重要です。

### マッピングの一覧表示

```vim
" 全マッピングを表示
:map

" ノーマルモードのマッピングだけ表示
:nmap

" 挿入モードのマッピングだけ表示
:imap

" 特定のキーから始まるマッピングを表示
:nmap <Leader>

" 特定のキーのマッピングを確認
:nmap Y
```

### 定義元の確認（verbose）

「このマッピングはどのファイルで定義されたのか？」を調べられます。

```vim
" マッピングの定義元を表示
:verbose nmap Y
"   → 出力例：
"   n  Y             y$
"       Last set from ~/.vimrc line 42

" Leader キーのマッピングの定義元
:verbose nmap <Leader>w
```

### マッピングのオプション

マッピングにはオプションを付与できます。

```vim
" <buffer> ── バッファローカル（現在のバッファでのみ有効）
nnoremap <buffer> <Leader>r :!python3 %<CR>

" <silent> ── コマンドラインへの表示を抑制
nnoremap <silent> <Leader>h :nohlsearch<CR>

" <expr> ── 式の評価結果をマッピングに使用
" 例：ポップアップメニュー表示中は Ctrl+j で次の候補、それ以外は通常の Ctrl+j
inoremap <expr> <C-j> pumvisible() ? "\<C-n>" : "\<C-j>"

" <nowait> ── 他のマッピングを待たずに即座に実行
nnoremap <nowait> <Leader>q :quit<CR>

" 複数のオプションを組み合わせ可能
nnoremap <silent> <buffer> <Leader>t :terminal<CR>
```

### マッピングの解除

```vim
" マッピングを解除する
nunmap Y

" 全マッピングを解除する（モード別）
mapclear
nmapclear
imapclear
```

### よくある間違い

```vim
" ✕ 間違い：コメントがマッピングの一部になってしまう
nnoremap j gj " 折り返し対応
" → j を押すと gj "折り返し対応 が実行されてしまう！

" ◯ 正しい：コメントは別の行に書く
" 折り返された行でも見た目通りに移動
nnoremap j gj
```

> **ポイントまとめ**
> - `:map` / `:nmap` / `:imap` でマッピング一覧を確認できる
> - `:verbose nmap {key}` で定義元のファイルと行番号がわかる
> - `<buffer>` でバッファローカル、`<silent>` で出力抑制、`<expr>` で式評価
> - マッピングの行末にコメントを書くと、コメントもマッピングの一部になるので注意
> - 不要なマッピングは `nunmap` 等で解除できる

| 概念 | ポイント |
|------|----------|
| `map` vs `noremap` | `noremap`（非再帰的）を常に使う |
| モード別マッピング | `nnoremap`（ノーマル）、`inoremap`（挿入）等 |
| 特殊キー記法 | `<CR>`, `<Esc>`, `<C-x>`, `<A-x>` 等 |
| Leader キー | `let mapleader = " "` でスペースがおすすめ |
| 実用マッピング | `jj` で Esc、`Y y$`、ウィンドウ移動等 |
| マッピング管理 | `:verbose nmap` で定義元確認 |
| オプション | `<silent>`, `<buffer>`, `<expr>` 等 |
