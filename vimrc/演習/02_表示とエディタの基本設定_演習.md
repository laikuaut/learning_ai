# 第2章 演習：表示とエディタの基本設定

## 演習の目標

- 行番号表示（`number` / `relativenumber`）の設定と使い分けができる
- カーソル表示やステータスラインをカスタマイズできる
- カラースキームの変更方法を理解する
- 不可視文字の表示設定ができる

---

## 基本問題

### 問題1：行番号の設定

以下の3種類の行番号表示を実現する `.vimrc` の記述をそれぞれ答えてください。

1. 絶対行番号のみ表示（1, 2, 3, ...）
2. 相対行番号のみ表示（カーソル行からの距離）
3. ハイブリッド表示（カーソル行は絶対番号、他は相対番号）

**期待される .vimrc の記述：**
```
1. set number
2. set relativenumber
3. set number relativenumber
```

**ハイブリッド表示の見え方の例（カーソルが10行目にある場合）：**
```
  3 | ...
  2 | ...
  1 | ...
 10 | ← カーソル行（絶対行番号で表示）
  1 | ...
  2 | ...
  3 | ...
```

<details>
<summary>ヒント</summary>

- `number` と `relativenumber` を同時に有効にすると、ハイブリッド表示になります
- ハイブリッド表示は、`5j`（5行下に移動）のような相対移動と、`:15` のような絶対移動の両方に便利です

</details>

<details>
<summary>解答例</summary>

```vim
" 1. 絶対行番号のみ
set number
set norelativenumber

" 2. 相対行番号のみ
set norelativenumber
set relativenumber

" 3. ハイブリッド表示（実務で最も推奨される設定）
" number と relativenumber を両方有効にします
set number
set relativenumber

" 補足：モード切替で行番号を自動変更するテクニック
" ノーマルモードでは相対番号、挿入モードでは絶対番号にする
augroup NumberToggle
  autocmd!
  autocmd InsertEnter * set norelativenumber
  autocmd InsertLeave * set relativenumber
augroup END
```

</details>

### 問題2：カーソル表示の設定

以下の設定を `.vimrc` に記述してください。

1. カーソル行をハイライトする
2. カーソル列をハイライトする
3. カーソル位置の行番号・列番号をステータスバーに表示する

**期待される .vimrc の記述：**
```vim
set cursorline
set cursorcolumn
set ruler
```

<details>
<summary>ヒント</summary>

- `cursorline` はカーソルがある行全体をハイライトします
- `cursorcolumn` はカーソルがある列全体を縦にハイライトします
- `ruler` は右下に行番号と列番号を表示します

</details>

<details>
<summary>解答例</summary>

```vim
" カーソル行をハイライトする
" 現在の編集位置が視覚的にわかりやすくなります
set cursorline

" カーソル列をハイライトする
" インデントの確認やCSVの列揃えに便利です
" ※常時有効だと見づらい場合は必要な時だけトグルしてもOKです
set cursorcolumn

" 右下にカーソル位置（行,列）を表示する
set ruler

" 補足：cursorcolumn は処理が重くなることがあるため、
" 必要な時だけトグルする設定も実用的です
nnoremap <Leader>cc :set cursorcolumn!<CR>
```

</details>

### 問題3：シンタックスハイライトとカラースキーム

以下の操作を行うコマンドまたは `.vimrc` の記述を答えてください。

1. シンタックスハイライトを有効にする
2. 利用可能なカラースキームの一覧を確認する
3. カラースキームを `desert` に変更する
4. 背景色の設定をダークモードにする

**期待される操作：**
```
1. syntax on
2. :colorscheme （その後Tabキーで補完）
3. colorscheme desert
4. set background=dark
```

<details>
<summary>ヒント</summary>

- `syntax on` はシンタックスハイライト全体の有効化です
- `:colorscheme` の後にスペースを入れて Tab キーを押すと、利用可能なスキームが補完表示されます
- `background` は `dark` または `light` を指定します

</details>

<details>
<summary>解答例</summary>

```vim
" 1. シンタックスハイライトを有効にする
syntax on

" 2. コマンドラインで利用可能なカラースキームを確認
" :colorscheme [Tab] で補完候補が表示されます
" または以下で一覧表示も可能です
" :echo globpath(&runtimepath, 'colors/*.vim')

" 3. カラースキームを desert に変更する
" Vim に標準で含まれるスキームの一例です
colorscheme desert

" 4. 背景色をダークモードに設定
" colorscheme の前に書くと、ダーク用の配色が適用されます
set background=dark

" 補足：colorscheme を設定する順番の推奨
" background → colorscheme の順に書くのがベストです
set background=dark
colorscheme desert
```

</details>

## 応用問題

### 問題4：ステータスラインのカスタマイズ

以下の情報を表示するステータスラインを `.vimrc` に設定してください。

**表示する情報：**
- ファイル名
- ファイルタイプ
- 文字エンコーディング
- 改行コード
- カーソル位置（行:列）
- ファイル全体に対する現在位置のパーセンテージ

**期待される表示例：**
```
[main.py] [python] [utf-8] [unix]          行:42 列:15 (35%)
```

**期待される .vimrc の記述：**
```vim
set laststatus=2
set statusline=%F\ [%Y]\ [%{&fileencoding}]\ [%{&fileformat}]%=%l:%c\ (%p%%)
```

<details>
<summary>ヒント</summary>

- `laststatus=2` で常にステータスラインを表示します
- `%F` はフルパスのファイル名、`%f` は相対パスです
- `%Y` はファイルタイプ、`%l` は行番号、`%c` は列番号です
- `%=` は左寄せと右寄せの区切りです
- `%{式}` で Vim の式を評価して表示できます

</details>

<details>
<summary>解答例</summary>

```vim
" ステータスラインを常に表示する
" 0=非表示、1=ウィンドウが2つ以上で表示、2=常に表示
set laststatus=2

" ステータスラインの内容をカスタマイズ
set statusline=
set statusline+=[%f]            " ファイル名（相対パス）
set statusline+=\ [%Y]          " ファイルタイプ
set statusline+=\ [%{&fenc}]    " 文字エンコーディング（fenc は fileencoding の短縮形）
set statusline+=\ [%{&ff}]      " 改行コード（ff は fileformat の短縮形）
set statusline+=\ %m            " 変更フラグ（[+] が表示される）
set statusline+=\ %r            " 読み取り専用フラグ
set statusline+=%=              " ここから右寄せ
set statusline+=行:%l           " 行番号
set statusline+=\ 列:%c         " 列番号
set statusline+=\ (%p%%)        " ファイル全体のパーセンテージ

" 補足：statusline を複数行に分けて += で連結すると可読性が高くなります
" 一行で書く場合は以下のようになります：
" set statusline=[%f]\ [%Y]\ [%{&fenc}]\ [%{&ff}]\ %m%r%=%l:%c\ (%p%%)
```

</details>

### 問題5：不可視文字の表示設定

以下の要件を満たす `.vimrc` の記述を作成してください。

**要件：**
1. 不可視文字を表示する
2. タブ文字を `▸-` で表示する
3. 行末のスペースを `·` で表示する
4. 行末の改行を `↲` で表示する
5. `F3` キーで不可視文字の表示/非表示をトグルできるようにする

**期待される表示例（不可視文字表示ON時）：**
```
function hello()▸-------{↲
▸-------console.log("world");↲
}↲
```

<details>
<summary>ヒント</summary>

- `set list` で不可視文字の表示を有効にします
- `listchars` で各文字の表示形式を設定します
- `tab:XX` は2文字で指定します（1文字目がタブの先頭、2文字目が埋め文字）
- `trail:X` は行末の余分なスペースの表示文字です
- `eol:X` は行末の改行文字の表示です

</details>

<details>
<summary>解答例</summary>

```vim
" 不可視文字を表示する
set list

" 不可視文字の表示形式を設定
set listchars=tab:▸-      " タブ文字：▸ で始まり - で埋める
set listchars+=trail:·     " 行末スペース：· で表示
set listchars+=eol:↲       " 改行文字：↲ で表示
set listchars+=extends:»   " 右に続きがある場合の表示
set listchars+=precedes:«  " 左に続きがある場合の表示
set listchars+=nbsp:%      " ノーブレークスペースの表示

" F3 キーで不可視文字表示をトグル
nnoremap <F3> :set list!<CR>

" 補足：listchars は一行で書くこともできます
" set listchars=tab:▸-,trail:·,eol:↲,extends:»,precedes:«,nbsp:%
"
" UTF-8 の記号文字を使うには、encoding=utf-8 が必要です
" set encoding=utf-8 を事前に設定しておいてください
```

</details>

### 問題6：ウィンドウとバッファの表示設定

以下の要件を満たす設定を `.vimrc` に記述してください。

**要件：**
1. ウィンドウを水平分割した時、新しいウィンドウを下に開く
2. ウィンドウを垂直分割した時、新しいウィンドウを右に開く
3. タブページ（タブバー）を常に表示する
4. ウィンドウの最小高さを 0 行に設定する（ステータスラインのみの折りたたみ表示を許可）

**期待される .vimrc の記述：**
```vim
set splitbelow
set splitright
set showtabline=2
set winminheight=0
```

<details>
<summary>ヒント</summary>

- `splitbelow` と `splitright` は分割の方向を制御するブール値オプションです
- `showtabline` は `0`=非表示、`1`=タブが2つ以上で表示、`2`=常に表示です
- `winminheight` はウィンドウの最小行数を指定します

</details>

<details>
<summary>解答例</summary>

```vim
" ウィンドウ分割の方向設定
" デフォルトでは上/左に開くが、下/右に開く方が直感的です
set splitbelow    " 水平分割で新しいウィンドウを下に開く
set splitright    " 垂直分割で新しいウィンドウを右に開く

" タブバーを常に表示する
" 0: 非表示 / 1: タブが2つ以上で表示 / 2: 常に表示
set showtabline=2

" ウィンドウの最小高さを0行にする
" Ctrl-W _ でウィンドウを最小化した時にステータスラインだけになります
set winminheight=0

" 補足：ウィンドウ間の移動を楽にするキーマッピング
nnoremap <C-h> <C-w>h
nnoremap <C-j> <C-w>j
nnoremap <C-k> <C-w>k
nnoremap <C-l> <C-w>l
```

</details>

### 問題7：総合的な表示設定の作成

以下の要件をすべて満たす `.vimrc` の表示設定セクションを作成してください。

**要件：**
1. ハイブリッド行番号表示
2. カーソル行ハイライト
3. 80文字目にカラーカラムを表示
4. 常にステータスラインを表示し、ファイル名・ファイルタイプ・行:列・パーセンテージを含む
5. 不可視文字を表示（タブ・行末スペース・改行）
6. ダークモードで `desert` カラースキーム
7. 挿入モード進入時にカーソル行ハイライトを無効化、離脱時に再有効化
8. コマンドラインの高さを 2 行にする

**期待される .vimrc の構成：**
```vim
" === 表示設定 ===
" 行番号
...

" カーソル
...

" カラーカラム
...

" ステータスライン
...

" 不可視文字
...

" カラースキーム
...

" autocmd
...
```

<details>
<summary>ヒント</summary>

- `colorcolumn` は特定の列にガイドラインを表示するオプションです
- `cmdheight` でコマンドラインの高さを変更できます
- `autocmd InsertEnter` と `InsertLeave` イベントを使って挿入モードの切替を検知できます
- `augroup` で autocmd をまとめることを忘れないでください

</details>

<details>
<summary>解答例</summary>

```vim
" =============================================================================
" 表示設定
" =============================================================================

" --- 行番号 ---
" ハイブリッド表示：カーソル行は絶対番号、他は相対番号
set number
set relativenumber

" --- カーソル ---
" カーソル行をハイライトする
set cursorline

" --- カラーカラム ---
" 80文字目にガイドラインを表示する
" コードの横幅を意識する習慣付けに有効です
set colorcolumn=80

" --- コマンドライン ---
" コマンドラインの高さを2行にする
" メッセージが見切れにくくなります
set cmdheight=2

" --- ステータスライン ---
set laststatus=2
set statusline=
set statusline+=[%f]            " ファイル名
set statusline+=\ [%Y]          " ファイルタイプ
set statusline+=\ %m%r          " 変更・読み取り専用フラグ
set statusline+=%=              " 右寄せ区切り
set statusline+=行:%l\ 列:%c    " 行と列
set statusline+=\ (%p%%)        " パーセンテージ

" --- 不可視文字 ---
set list
set listchars=tab:▸-,trail:·,eol:↲,extends:»,precedes:«

" --- カラースキーム ---
set background=dark
syntax on
colorscheme desert

" --- autocmd ---
augroup MyDisplay
  autocmd!
  " 挿入モード進入時：カーソル行ハイライトを無効化
  " → 挿入中はハイライトが邪魔になることがあるため
  autocmd InsertEnter * set nocursorline
  " 挿入モード離脱時：カーソル行ハイライトを再有効化
  autocmd InsertLeave * set cursorline
augroup END
```

</details>

### 問題8：ステータスラインの高度なカスタマイズ

以下の機能を持つステータスラインを `.vimrc` に構築してください。

**要件：**
- 左側：モード表示（関数を使用）、ファイル名、変更フラグ
- 右側：Git ブランチ名（取得できない場合は空文字）、ファイルタイプ、エンコーディング、行:列

**期待される表示例：**
```
[NORMAL] main.py [+]                    [main] python utf-8 42:15
```

<details>
<summary>ヒント</summary>

- `mode()` 関数で現在のモードを取得できます（`n`=ノーマル、`i`=挿入、`v`=ビジュアル等）
- Git ブランチ名は `system()` 関数と `git` コマンドで取得できます
- 関数を定義して `%{関数名()}` でステータスラインから呼び出せます
- `trim()` で文字列末尾の改行を除去できます

</details>

<details>
<summary>解答例</summary>

```vim
" モード名を表示用の文字列に変換する関数
function! GetModeName() abort
  let l:mode_map = {
    \ 'n':  'NORMAL',
    \ 'i':  'INSERT',
    \ 'v':  'VISUAL',
    \ 'V':  'V-LINE',
    \ "\<C-v>": 'V-BLOCK',
    \ 'R':  'REPLACE',
    \ 'c':  'COMMAND',
    \ 't':  'TERMINAL',
    \ }
  " 現在のモードを取得し、対応する表示名を返す
  " 辞書にないモードの場合はモード文字をそのまま返す
  return get(l:mode_map, mode(), mode())
endfunction

" Git のブランチ名を取得する関数
function! GetGitBranch() abort
  " git コマンドでブランチ名を取得
  let l:branch = trim(system('git rev-parse --abbrev-ref HEAD 2>/dev/null'))
  " Git リポジトリでない場合やエラー時は空文字を返す
  if v:shell_error
    return ''
  endif
  return '[' . l:branch . ']'
endfunction

" ステータスラインを構築
set laststatus=2
set statusline=
set statusline+=[%{GetModeName()}]  " モード表示
set statusline+=\ %f                " ファイル名
set statusline+=\ %m                " 変更フラグ [+]
set statusline+=%=                  " 右寄せ区切り
set statusline+=%{GetGitBranch()}   " Git ブランチ名
set statusline+=\ %Y                " ファイルタイプ
set statusline+=\ %{&fenc}          " エンコーディング
set statusline+=\ %l:%c             " 行:列

" 補足：
" - abort を付けると、関数内でエラーが起きた場合に即座に中断します
" - system() は毎回外部コマンドを実行するため、
"   高頻度の更新が気になる場合はキャッシュの仕組みを入れることも検討してください
" - 実務ではプラグイン（lightline.vim, airline.vim など）を使う方が一般的です
```

</details>
