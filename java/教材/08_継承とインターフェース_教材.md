# 第8章：継承とインターフェース

## この章のゴール

## 8.1 継承の基礎

### 継承とは

継承（Inheritance）は、既存のクラス（親クラス/スーパークラス）の機能を引き継いで、新しいクラス（子クラス/サブクラス）を作る仕組みです。

```
┌───────────────────┐
│  Animal (親クラス)  │
│  ──────────────── │
│  - name           │
│  - age            │
│  ──────────────── │
│  + eat()          │
│  + sleep()        │
└────────┬──────────┘
         │ extends
    ┌────┴────┐
    │         │
┌───┴───┐ ┌──┴────┐
│ Dog   │ │ Cat   │
│ ───── │ │ ───── │
│+bark()│ │+meow()│
└───────┘ └───────┘
  子クラス   子クラス
```

### 基本的な継承

```java
// 親クラス（スーパークラス）
class Animal {
    String name;
    int age;

    Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }

    void eat() {
        System.out.println(name + "が食事中です。");
    }

    void sleep() {
        System.out.println(name + "が眠っています。");
    }

    void showInfo() {
        System.out.println("名前: " + name + ", 年齢: " + age + "歳");
    }
}

// 子クラス（サブクラス）: extends で継承
class Dog extends Animal {
    String breed;

    Dog(String name, int age, String breed) {
        super(name, age);  // 親クラスのコンストラクタを呼ぶ
        this.breed = breed;
    }

    // 子クラス独自のメソッド
    void bark() {
        System.out.println(name + "がワンワン吠えた！");
    }

    // 親クラスのメソッドをオーバーライド
    @Override
    void showInfo() {
        super.showInfo();  // 親のメソッドを呼んでから追加情報を表示
        System.out.println("犬種: " + breed);
    }
}

class Cat extends Animal {
    boolean isIndoor;

    Cat(String name, int age, boolean isIndoor) {
        super(name, age);
        this.isIndoor = isIndoor;
    }

    void meow() {
        System.out.println(name + "がニャーと鳴いた！");
    }

    @Override
    void showInfo() {
        super.showInfo();
        System.out.println("室内飼い: " + (isIndoor ? "はい" : "いいえ"));
    }
}

// テスト用クラス
public class InheritanceDemo {
    public static void main(String[] args) {
        Dog dog = new Dog("ポチ", 3, "柴犬");
        Cat cat = new Cat("タマ", 5, true);

        // 親クラスのメソッドが使える
        dog.eat();       // ポチが食事中です。
        dog.sleep();     // ポチが眠っています。
        dog.bark();      // ポチがワンワン吠えた！
        dog.showInfo();  // 名前: ポチ, 年齢: 3歳 / 犬種: 柴犬

        System.out.println("---");

        cat.eat();       // タマが食事中です。
        cat.meow();      // タマがニャーと鳴いた！
        cat.showInfo();  // 名前: タマ, 年齢: 5歳 / 室内飼い: はい
    }
}
```

### super キーワード

`super` は親クラスを参照するキーワードです。

```java
class Parent {
    String message;

    Parent(String message) {
        this.message = message;
    }

    void greet() {
        System.out.println("Parent: " + message);
    }
}

class Child extends Parent {
    String childMessage;

    Child(String message, String childMessage) {
        super(message);  // 親のコンストラクタを呼ぶ（最初の行に書く）
        this.childMessage = childMessage;
    }

    @Override
    void greet() {
        super.greet();  // 親のメソッドを呼ぶ
        System.out.println("Child: " + childMessage);
    }
}

public class SuperDemo {
    public static void main(String[] args) {
        Child child = new Child("こんにちは", "よろしくお願いします");
        child.greet();
        // Parent: こんにちは
        // Child: よろしくお願いします
    }
}
```

> **よくある間違い：** `super()` はコンストラクタの**最初の行**に書く必要があります。書かない場合は、自動的に引数なしの `super()` が呼ばれますが、親クラスにデフォルトコンストラクタがないとコンパイルエラーになります。

### ポイントまとめ

- `extends` で親クラスを継承し、フィールドやメソッドを引き継ぐ
- `@Override` でメソッドをオーバーライドして動作を変更できる
- `super` で親クラスのコンストラクタやメソッドを呼び出せる
- Javaの継承は単一継承（1つのクラスしか継承できない）

## 8.2 抽象クラス

抽象クラス（abstract class）は、直接インスタンス化できないクラスです。継承して使うことを前提としています。

```java
// 抽象クラス: new できない
abstract class Shape {
    String color;

    Shape(String color) {
        this.color = color;
    }

    // 抽象メソッド: 実装を持たない（子クラスで必ず実装する）
    abstract double getArea();
    abstract double getPerimeter();

    // 通常のメソッド: 実装を持つ
    void showInfo() {
        System.out.printf("色: %s, 面積: %.2f, 周長: %.2f%n",
            color, getArea(), getPerimeter());
    }
}

class Circle extends Shape {
    double radius;

    Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    double getArea() {
        return Math.PI * radius * radius;
    }

    @Override
    double getPerimeter() {
        return 2 * Math.PI * radius;
    }

    @Override
    public String toString() {
        return String.format("Circle(radius=%.1f, color=%s)", radius, color);
    }
}

class Rectangle extends Shape {
    double width;
    double height;

    Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }

    @Override
    double getArea() {
        return width * height;
    }

    @Override
    double getPerimeter() {
        return 2 * (width + height);
    }

    @Override
    public String toString() {
        return String.format("Rectangle(%.1f x %.1f, color=%s)", width, height, color);
    }
}

class Triangle extends Shape {
    double base;
    double height;
    double side1, side2, side3;

    Triangle(String color, double base, double height, double side1, double side2, double side3) {
        super(color);
        this.base = base;
        this.height = height;
        this.side1 = side1;
        this.side2 = side2;
        this.side3 = side3;
    }

    @Override
    double getArea() {
        return 0.5 * base * height;
    }

    @Override
    double getPerimeter() {
        return side1 + side2 + side3;
    }

    @Override
    public String toString() {
        return String.format("Triangle(base=%.1f, height=%.1f, color=%s)", base, height, color);
    }
}

public class AbstractClassDemo {
    public static void main(String[] args) {
        // Shape shape = new Shape("赤");  // コンパイルエラー！抽象クラスはnewできない

        Shape[] shapes = {
            new Circle("赤", 5.0),
            new Rectangle("青", 4.0, 6.0),
            new Triangle("緑", 3.0, 4.0, 3.0, 4.0, 5.0)
        };

        for (Shape shape : shapes) {
            System.out.println(shape);  // toString()
            shape.showInfo();           // 共通メソッド + 抽象メソッドの実装が呼ばれる
            System.out.println();
        }
    }
}
```

### ポイントまとめ

- `abstract` クラスは `new` でインスタンス化できない
- `abstract` メソッドは実装を持たず、子クラスで必ずオーバーライドする
- 抽象クラスには通常のメソッドも定義できる
- 共通の処理を親クラスに、個別の処理を子クラスに書く

## 8.3 インターフェース

### インターフェースとは

インターフェース（Interface）は、クラスが実装すべきメソッドの「契約」を定義するものです。

```java
// インターフェースの定義
interface Printable {
    void print();  // 抽象メソッド（abstract は省略可能）
}

interface Saveable {
    void save(String filename);
    default void autoSave() {  // デフォルトメソッド（Java 8以降）
        save("autosave.dat");
        System.out.println("自動保存しました。");
    }
}

// インターフェースの実装
class Document implements Printable, Saveable {
    private String title;
    private String content;

    Document(String title, String content) {
        this.title = title;
        this.content = content;
    }

    @Override
    public void print() {
        System.out.println("=== " + title + " ===");
        System.out.println(content);
    }

    @Override
    public void save(String filename) {
        System.out.println("ドキュメント「" + title + "」を " + filename + " に保存しました。");
    }
}

class Spreadsheet implements Printable, Saveable {
    private String name;
    private int rows;
    private int cols;

    Spreadsheet(String name, int rows, int cols) {
        this.name = name;
        this.rows = rows;
        this.cols = cols;
    }

    @Override
    public void print() {
        System.out.println("スプレッドシート: " + name + " (" + rows + "行×" + cols + "列)");
    }

    @Override
    public void save(String filename) {
        System.out.println("スプレッドシート「" + name + "」を " + filename + " に保存しました。");
    }
}

public class InterfaceDemo {
    public static void main(String[] args) {
        Document doc = new Document("議事録", "本日の会議内容...");
        Spreadsheet sheet = new Spreadsheet("売上表", 100, 10);

        // インターフェース型の変数に代入できる
        Printable[] printables = {doc, sheet};
        for (Printable p : printables) {
            p.print();
            System.out.println();
        }

        // デフォルトメソッドの利用
        doc.autoSave();    // 自動保存しました。
        sheet.autoSave();  // 自動保存しました。
    }
}
```

### ポイントまとめ

- インターフェースは `interface` キーワードで定義する
- クラスは `implements` でインターフェースを実装する
- 1つのクラスが複数のインターフェースを実装できる（多重実装）
- `default` メソッドでデフォルト実装を持てる（Java 8以降）

## 8.4 ポリモーフィズム

ポリモーフィズム（Polymorphism、多態性）は、同じメソッド呼び出しで異なる動作をする仕組みです。

```java
abstract class Employee {
    String name;
    int baseSalary;

    Employee(String name, int baseSalary) {
        this.name = name;
        this.baseSalary = baseSalary;
    }

    // 各従業員タイプで異なる給与計算
    abstract int calculateSalary();

    void showPaySlip() {
        System.out.printf("%-10s 給与: ¥%,d%n", name, calculateSalary());
    }
}

class FullTimeEmployee extends Employee {
    int bonus;

    FullTimeEmployee(String name, int baseSalary, int bonus) {
        super(name, baseSalary);
        this.bonus = bonus;
    }

    @Override
    int calculateSalary() {
        return baseSalary + bonus;
    }
}

class PartTimeEmployee extends Employee {
    int hoursWorked;
    int hourlyRate;

    PartTimeEmployee(String name, int hourlyRate, int hoursWorked) {
        super(name, 0);
        this.hourlyRate = hourlyRate;
        this.hoursWorked = hoursWorked;
    }

    @Override
    int calculateSalary() {
        return hourlyRate * hoursWorked;
    }
}

class Intern extends Employee {
    Intern(String name) {
        super(name, 0);
    }

    @Override
    int calculateSalary() {
        return 0;  // インターンは無給
    }
}

public class PolymorphismDemo {
    public static void main(String[] args) {
        // 親クラス型の配列に異なる子クラスのオブジェクトを格納
        Employee[] employees = {
            new FullTimeEmployee("田中", 300000, 50000),
            new FullTimeEmployee("鈴木", 350000, 80000),
            new PartTimeEmployee("佐藤", 1200, 80),
            new PartTimeEmployee("高橋", 1500, 60),
            new Intern("山田")
        };

        System.out.println("===== 給与明細 =====");
        int totalPayroll = 0;
        for (Employee emp : employees) {
            emp.showPaySlip();  // 同じメソッド呼び出しで異なる動作
            totalPayroll += emp.calculateSalary();
        }
        System.out.printf("合計支出: ¥%,d%n", totalPayroll);
    }
}
```

### instanceof 演算子

`instanceof` で実際の型を判定できます。

```java
public class InstanceofDemo {
    public static void main(String[] args) {
        Object[] objects = {
            "Hello",
            42,
            3.14,
            true,
            new int[]{1, 2, 3}
        };

        for (Object obj : objects) {
            if (obj instanceof String s) {
                // Java 16以降: パターンマッチング
                System.out.println("String: " + s.toUpperCase());
            } else if (obj instanceof Integer) {
                System.out.println("Integer: " + obj);
            } else if (obj instanceof Double) {
                System.out.println("Double: " + obj);
            } else if (obj instanceof Boolean) {
                System.out.println("Boolean: " + obj);
            } else {
                System.out.println("その他: " + obj.getClass().getSimpleName());
            }
        }
    }
}
```

> **よくある間違い：** `instanceof` を多用するコードは、ポリモーフィズムで置き換えられることが多いです。型チェックが頻繁に必要な場合は、設計を見直しましょう。

### ポイントまとめ

- ポリモーフィズムにより、親クラス型で子クラスのオブジェクトを扱える
- 実行時に実際の型のメソッドが呼ばれる（動的束縛）
- `instanceof` で型判定ができるが、使いすぎは設計の見直しサイン

## 8.5 Objectクラス

Javaのすべてのクラスは暗黙的に `Object` クラスを継承しています。

### 主要なObjectメソッド

```java
public class ObjectMethodsDemo {
    private String name;
    private int value;

    public ObjectMethodsDemo(String name, int value) {
        this.name = name;
        this.value = value;
    }

    // toString(): オブジェクトの文字列表現
    @Override
    public String toString() {
        return "ObjectMethodsDemo{name='" + name + "', value=" + value + "}";
    }

    // equals(): オブジェクトの等価性判定
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;                          // 同一参照
        if (obj == null || getClass() != obj.getClass()) return false;  // null or 型違い
        ObjectMethodsDemo other = (ObjectMethodsDemo) obj;
        return value == other.value && name.equals(other.name);
    }

    // hashCode(): equals()と一貫性を持たせる
    @Override
    public int hashCode() {
        return name.hashCode() * 31 + value;
    }

    public static void main(String[] args) {
        ObjectMethodsDemo a = new ObjectMethodsDemo("test", 42);
        ObjectMethodsDemo b = new ObjectMethodsDemo("test", 42);
        ObjectMethodsDemo c = new ObjectMethodsDemo("other", 99);

        System.out.println(a);                   // toString()
        System.out.println("a == b: " + (a == b));           // false（別オブジェクト）
        System.out.println("a.equals(b): " + a.equals(b));   // true（内容が同じ）
        System.out.println("a.equals(c): " + a.equals(c));   // false

        System.out.println("a.hashCode(): " + a.hashCode());
        System.out.println("b.hashCode(): " + b.hashCode());  // aと同じ値
    }
}
```

> **よくある間違い：** `equals()` をオーバーライドしたら、必ず `hashCode()` もオーバーライドしてください。`HashMap` や `HashSet` が正しく動作しなくなります。

### ポイントまとめ

- すべてのクラスは `Object` を暗黙的に継承している
- `toString()`, `equals()`, `hashCode()` は必要に応じてオーバーライドする
- `equals()` をオーバーライドしたら `hashCode()` も合わせて実装する

## 8.6 抽象クラス vs インターフェース

### 比較表

| 特徴 | 抽象クラス | インターフェース |
|---|---|---|
| キーワード | `abstract class` | `interface` |
| 継承/実装 | `extends`（単一継承） | `implements`（複数実装可） |
| コンストラクタ | 持てる | 持てない |
| フィールド | 通常のフィールドを持てる | 定数（`public static final`）のみ |
| メソッド | 抽象・通常の両方 | 抽象・default・static |
| アクセス修飾子 | 自由に設定可能 | メソッドは暗黙的に `public` |
| 使い分け | 「〜である（is-a）」関係 | 「〜できる（can-do）」関係 |

### 使い分けの指針

```
使い分けの判断基準:

Q: 共通の状態（フィールド）を持たせたい？
  → YES → 抽象クラス
  → NO  ↓

Q: 無関係なクラス間で共通の振る舞いを持たせたい？
  → YES → インターフェース
  → NO  ↓

Q: 既に別のクラスを継承している？
  → YES → インターフェース（多重継承の代替）
  → NO  → 抽象クラスでもインターフェースでも可
```

```java
// 「is-a」関係: 抽象クラスが適切
// 犬は動物である → Dog extends Animal

// 「can-do」関係: インターフェースが適切
// 犬は泳げる → Dog implements Swimmable
// 鳥は飛べる → Bird implements Flyable
// アヒルは泳げて飛べる → Duck implements Swimmable, Flyable

interface Swimmable {
    void swim();
}

interface Flyable {
    void fly();
}

abstract class AnimalBase {
    String name;
    AnimalBase(String name) {
        this.name = name;
    }
    abstract void makeSound();
}

class DuckExample extends AnimalBase implements Swimmable, Flyable {
    DuckExample(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + ": ガーガー");
    }

    @Override
    public void swim() {
        System.out.println(name + "が泳いでいます。");
    }

    @Override
    public void fly() {
        System.out.println(name + "が飛んでいます。");
    }
}

public class AbstractVsInterface {
    public static void main(String[] args) {
        DuckExample duck = new DuckExample("ドナルド");
        duck.makeSound();  // ドナルド: ガーガー
        duck.swim();       // ドナルドが泳いでいます。
        duck.fly();        // ドナルドが飛んでいます。
    }
}
```

### ポイントまとめ

- 抽象クラスは「〜である」関係で使い、共通のフィールドや実装を持てる
- インターフェースは「〜できる」関係で使い、複数実装できる
- 両方を組み合わせて柔軟なクラス設計が可能

## 8.7 実践例：図形階層の設計

```java
interface Drawable {
    void draw();
}

interface Resizable {
    void resize(double factor);
}

abstract class ShapeBase implements Drawable {
    protected String name;
    protected String color;

    ShapeBase(String name, String color) {
        this.name = name;
        this.color = color;
    }

    abstract double area();
    abstract double perimeter();

    @Override
    public String toString() {
        return String.format("%s(色=%s, 面積=%.2f, 周長=%.2f)",
            name, color, area(), perimeter());
    }
}

class CircleShape extends ShapeBase implements Resizable {
    private double radius;

    CircleShape(String color, double radius) {
        super("円", color);
        this.radius = radius;
    }

    @Override
    double area() {
        return Math.PI * radius * radius;
    }

    @Override
    double perimeter() {
        return 2 * Math.PI * radius;
    }

    @Override
    public void draw() {
        System.out.println("○ " + this);
    }

    @Override
    public void resize(double factor) {
        radius *= factor;
        System.out.println(name + "のサイズを" + factor + "倍にしました。半径=" + radius);
    }
}

class RectangleShape extends ShapeBase implements Resizable {
    private double width;
    private double height;

    RectangleShape(String color, double width, double height) {
        super("四角形", color);
        this.width = width;
        this.height = height;
    }

    @Override
    double area() {
        return width * height;
    }

    @Override
    double perimeter() {
        return 2 * (width + height);
    }

    @Override
    public void draw() {
        System.out.println("□ " + this);
    }

    @Override
    public void resize(double factor) {
        width *= factor;
        height *= factor;
        System.out.println(name + "のサイズを" + factor + "倍にしました。"
            + width + "x" + height);
    }
}

public class ShapeHierarchy {
    public static void main(String[] args) {
        ShapeBase[] shapes = {
            new CircleShape("赤", 5.0),
            new RectangleShape("青", 4.0, 6.0),
            new CircleShape("緑", 3.0),
            new RectangleShape("黄", 8.0, 2.0)
        };

        System.out.println("===== 図形の描画 =====");
        for (ShapeBase shape : shapes) {
            shape.draw();
        }

        System.out.println("\n===== サイズ変更 =====");
        for (ShapeBase shape : shapes) {
            if (shape instanceof Resizable r) {
                r.resize(1.5);
            }
        }

        System.out.println("\n===== 変更後の図形 =====");
        for (ShapeBase shape : shapes) {
            shape.draw();
        }
    }
}
```

### ポイントまとめ

- 実践的なクラス設計では抽象クラスとインターフェースを組み合わせる
- `protected` フィールドで子クラスからのアクセスを許可する
- ポリモーフィズムにより、統一的な操作で多様な動作を実現できる

## まとめ

この章では、Javaの継承とインターフェースについて学びました。

1. **継承** - `extends` で親クラスの機能を引き継ぐ
2. **オーバーライド** - `@Override` で親のメソッドを子で上書きする
3. **super** - 親クラスのコンストラクタやメソッドを呼ぶ
4. **抽象クラス** - 直接インスタンス化できない、継承前提のクラス
5. **インターフェース** - メソッドの「契約」を定義し、複数実装できる
6. **ポリモーフィズム** - 同じ型で異なる動作を実現する
7. **Objectクラス** - `toString()`, `equals()`, `hashCode()` のオーバーライド
