## 基本問題

### 問題1：画面一覧の作成

タスク管理Webアプリケーションの画面一覧を作成してください。以下の機能を含むこと。

- ユーザー認証（ログイン、新規登録）
- ダッシュボード
- プロジェクト管理（一覧、詳細）
- タスク管理（一覧、詳細、作成、編集）
- 設定（プロフィール）

**期待される出力例：**

```
| 画面ID  | 画面名    | URL パス    | 認証 | 概要          |
|--------|---------|------------|------|--------------|
| SCR-001| ログイン  | /login     | 不要 | ログインフォーム|
| ...    | ...     | ...        | ...  | ...          |
```

<details>
<summary>ヒント</summary>

URLパスはRESTfulな設計を意識してください。プロジェクトとタスクは親子関係にあります。

</details>

<summary>解答例</summary>

## 応用問題

### 問題4：画面項目定義書の作成

「タスク作成画面（SCR-008）」の画面項目定義書を作成してください。

以下の入力項目を含めること：
- タスク名（必須、最大200文字）
- 説明（任意、最大5000文字）
- 担当者（プルダウン選択）
- 優先度（ラジオボタン：低/中/高/緊急）
- 期限（日付ピッカー）
- 作成ボタン、キャンセルボタン

<details>
<summary>ヒント</summary>

各項目について以下を定義してください。
- 項目種別（テキスト入力、プルダウン、ラジオボタン等）
- バリデーションルール
- エラーメッセージ
- 初期値・デフォルト値

</details>

<details>
<summary>解答例</summary>

```
画面ID  ：SCR-008
画面名  ：タスク作成

■ 入力項目

| No | 項目名  | 項目ID      | 種別           | 必須 | 初期値  | バリデーション         | エラーメッセージ                 |
|----|--------|------------|---------------|------|--------|---------------------|-------------------------------|
| 1  | タスク名| title      | テキスト入力    | ○    | 空      | 1〜200文字          | 「タスク名を入力してください」     |
|    |        |            |               |      |        |                     | 「200文字以内で入力してください」  |
| 2  | 説明   | description| テキストエリア  | -    | 空      | 最大5000文字         | 「5000文字以内で入力してください」 |
| 3  | 担当者  | assignee_id| プルダウン     | -    | 未選択  | プロジェクトメンバーから選択| -                          |
| 4  | 優先度  | priority   | ラジオボタン   | ○    | 中(medium)| low/medium/high/urgent| -（デフォルトがあるため不要） |
| 5  | 期限   | due_date   | 日付ピッカー   | -    | 空      | 今日以降の日付        | 「今日以降の日付を選択してください」|

■ ボタン

| No | 項目名     | 項目ID     | 種別           | アクション                          |
|----|-----------|-----------|---------------|-------------------------------------|
| 6  | 作成       | btn_create| プライマリボタン | バリデーション後、POST APIを呼び出す   |
| 7  | キャンセル  | btn_cancel| セカンダリボタン | 確認ダイアログを表示後、前画面に遷移    |

■ アクション定義

【作成ボタン押下時】
1. フロントエンドバリデーション実行
   ・タスク名が空 → 項目下にエラーメッセージ表示
   ・タスク名が200文字超 → 項目下にエラーメッセージ表示
   ・期限が過去の日付 → 項目下にエラーメッセージ表示

2. バリデーションOKの場合
   ・作成ボタンをローディング状態にする（二重送信防止）
   ・POST /projects/:id/tasks を呼び出す

3. API成功時（201）
   ・「タスクを作成しました」のトーストを表示
   ・作成したタスクの詳細画面（SCR-007）に遷移する

4. API失敗時
   ・400: バリデーションエラーを各項目に表示
   ・403: 「タスクを作成する権限がありません」を表示
   ・500: 「エラーが発生しました。再度お試しください」を表示
   ・作成ボタンを元に戻す

【キャンセルボタン押下時】
1. 入力内容がある場合
   ・「入力内容が破棄されます。よろしいですか？」ダイアログ表示
   ・「はい」→ プロジェクト詳細画面（SCR-006）に遷移
   ・「いいえ」→ ダイアログを閉じる
2. 入力内容がない場合
   ・そのままプロジェクト詳細画面に遷移
```

</details>

### 問題5：画面設計書の作成（総合演習）

社内の問い合わせ管理システムの「チケット一覧画面」について、以下を作成してください。

1. ワイヤーフレーム
2. 画面項目定義書（フィルタ・検索条件を含む）
3. ステートバリエーション（0件時、ローディング時、エラー時）

- チケットの一覧を表形式で表示する（ID、タイトル、ステータス、優先度、担当者、更新日）

- キーワードで検索できる
- ページネーションがある（20件/ページ）

- 検索結果0件
- 初回ローディング中
- フィルタ変更時のローディング中
- API通信エラー時

</details>
