# 実践課題10：リファクタリング ─ レガシーECサイト型定義 ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第2章（インターフェースと型エイリアス）、第4章（ユニオン型と交差型）、第5章（ジェネリクス）、第8章（ユーティリティ型）
> **課題の種類**: リファクタリング
> **学習目標**: `any` だらけの「動くが型安全でない」コードを、ユーティリティ型やジェネリクスを駆使して型安全に改善する力を養う

---

## 課題の説明

以下は「ECサイトの注文処理システム」のコードです。**正しく動作します**が、以下の問題を抱えています。

- `any` が多用されていて型安全でない
- 同じような型定義が重複している
- ユーティリティ型が使われていない
- エラー処理が場当たり的
- 拡張しにくい構造

**このコードを段階的にリファクタリング（型の改善）してください。**

### ゴール

1. **ステップ1**: `any` を具体的な型に置き換える
2. **ステップ2**: ユーティリティ型（`Partial`, `Pick`, `Omit`, `Readonly`, `Record`）を活用する
3. **ステップ3**: ジェネリクスで共通パターンを抽出する

## リファクタリング対象のコード

以下を `ec_legacy.ts` として保存してください。

```typescript
// ECサイト注文処理（レガシー版）
// 動作するが型安全でないコード。リファクタリングしてください。
// 実行方法：npx ts-node ec_legacy.ts

// --- 型定義（問題あり） ---
interface Product {
  id: number;
  name: string;
  price: number;
  stock: number;
  category: string;    // any的な string — 制限がない
  description: string;
}

// 注文作成時のデータ（Product と似ているが微妙に違う）
interface OrderProduct {
  id: number;
  name: string;
  price: number;
}

// 注文のステータス（文字列で管理 — typoの危険）
interface Order {
  id: number;
  products: any[];     // any配列
  customer: any;       // any
  status: string;      // 制限なし
  total: number;
  createdAt: string;
}

// 顧客情報（注文と同じ情報を別途定義）
interface Customer {
  id: number;
  name: string;
  email: string;
  address: string;
}

// --- データ ---
const products: any[] = [
  { id: 1, name: "TypeScript本", price: 2800, stock: 10, category: "書籍", description: "初心者向け" },
  { id: 2, name: "マウス", price: 3500, stock: 5, category: "電子機器", description: "ワイヤレス" },
  { id: 3, name: "Tシャツ", price: 2500, stock: 0, category: "衣類", description: "プログラマー向け" },
];

// --- 関数群（型安全でない） ---
function findProduct(id: any): any {
  return products.find((p: any) => p.id === id);
}

function calculateTotal(items: any[]): any {
  let total = 0;
  for (const item of items) {
    total += item.price * item.quantity;
  }
  return total;
}

function createOrder(customerId: any, items: any[]): any {
  const customer = getCustomer(customerId);
  if (!customer) {
    return { error: "顧客が見つかりません" };
  }

  const orderProducts: any[] = [];
  for (const item of items) {
    const product = findProduct(item.productId);
    if (!product) {
      return { error: `商品ID ${item.productId} が見つかりません` };
    }
    if (product.stock < item.quantity) {
      return { error: `${product.name} の在庫が不足しています` };
    }
    orderProducts.push({
      id: product.id,
      name: product.name,
      price: product.price,
      quantity: item.quantity,
    });
  }

  const total = calculateTotal(orderProducts);
  const order: any = {
    id: Date.now(),
    products: orderProducts,
    customer: customer,
    status: "pending",
    total: total,
    createdAt: new Date().toISOString(),
  };

  return order;
}

function updateOrderStatus(order: any, newStatus: string): any {
  // ステータスの遷移チェックなし
  order.status = newStatus;
  return order;
}

function getCustomer(id: any): any {
  const customers: any[] = [
    { id: 1, name: "田中太郎", email: "tanaka@example.com", address: "東京都" },
    { id: 2, name: "佐藤花子", email: "sato@example.com", address: "大阪府" },
  ];
  return customers.find((c: any) => c.id === id);
}

function formatOrder(order: any): string {
  let result = `注文 #${order.id}\n`;
  result += `顧客: ${order.customer.name}\n`;
  result += `ステータス: ${order.status}\n`;
  result += `商品:\n`;
  for (const p of order.products) {
    result += `  - ${p.name} x${p.quantity} = ¥${(p.price * p.quantity).toLocaleString()}\n`;
  }
  result += `合計: ¥${order.total.toLocaleString()}\n`;
  return result;
}

// --- 実行 ---
console.log("===== ECサイト注文処理 =====\n");

const order = createOrder(1, [
  { productId: 1, quantity: 2 },
  { productId: 2, quantity: 1 },
]);

if (order.error) {
  console.log(`エラー: ${order.error}`);
} else {
  console.log(formatOrder(order));
  const updated = updateOrderStatus(order, "shipped");
  console.log(`ステータス更新: ${updated.status}`);
}
```

## ステップガイド

<details>
<summary>ステップ1：any を具体的な型に置き換える</summary>

まず `any` をすべて洗い出し、適切な型に置き換えます。

```typescript
// ❌ Before
const products: any[] = [...];
function findProduct(id: any): any { ... }

// ✅ After
const products: Product[] = [...];
function findProduct(id: number): Product | undefined { ... }
```

**ポイント:**
- `any` を使っている箇所をリストアップする
- 実際に何の型が入るかをコードの使われ方から推測する
- 見つからない可能性がある場合は `T | undefined` を使う
- エラーの可能性がある場合はユニオン型で表現する

</details>

<details>
<summary>ステップ2：ユーティリティ型を活用する</summary>

```typescript
// ❌ Before: Product と似た OrderProduct を別定義
interface OrderProduct {
  id: number;
  name: string;
  price: number;
}

// ✅ After: Pick で Product から必要なプロパティだけ抽出
type OrderProduct = Pick<Product, "id" | "name" | "price">;

// ❌ Before: status が自由な string
interface Order {
  status: string;
}

// ✅ After: リテラル型のユニオンで制限
type OrderStatus = "pending" | "confirmed" | "shipped" | "delivered" | "cancelled";
interface Order {
  status: OrderStatus;
}
```

</details>

<details>
<summary>ステップ3：ジェネリクスで共通パターンを抽出する</summary>

```typescript
// エラーを含む可能性のある結果型
type Result<T> = 
  | { success: true; data: T }
  | { success: false; error: string };

function createOrder(customerId: number, items: CartItem[]): Result<Order> {
  // ...
  if (!customer) {
    return { success: false, error: "顧客が見つかりません" };
  }
  // ...
  return { success: true, data: order };
}
```

<summary>解答例（初心者向け ─ any の除去とユーティリティ型の導入）</summary>

```typescript
// ECサイト注文処理（リファクタリング版 ─ 初心者向け）
// 学べる内容：any の除去、Pick、Readonly、リテラル型ユニオン
// 実行方法：npx ts-node ec_refactored.ts

// --- カテゴリをリテラル型で制限 ---
type ProductCategory = "書籍" | "電子機器" | "衣類" | "食品";

// --- 注文ステータスをリテラル型で制限 ---
type OrderStatus = "pending" | "confirmed" | "shipped" | "delivered" | "cancelled";

// --- 型定義 ---
interface Product {
  readonly id: number;
  name: string;
  price: number;
  stock: number;
  category: ProductCategory;
  description: string;
}

// Product から必要なプロパティだけ抽出
type OrderProduct = Pick<Product, "id" | "name" | "price"> & {
  quantity: number;
};

interface Customer {
  readonly id: number;
  name: string;
  email: string;
  address: string;
}

interface Order {
  readonly id: number;
  products: OrderProduct[];
  customer: Customer;
  status: OrderStatus;
  total: number;
  createdAt: string;
}

interface CartItem {
  productId: number;
  quantity: number;
}

// --- エラー型 ---
type OrderResult =
  | { success: true; order: Order }
  | { success: false; error: string };

// --- データ ---
const products: Product[] = [
  { id: 1, name: "TypeScript本", price: 2800, stock: 10, category: "書籍", description: "初心者向け" },
  { id: 2, name: "マウス", price: 3500, stock: 5, category: "電子機器", description: "ワイヤレス" },
  { id: 3, name: "Tシャツ", price: 2500, stock: 0, category: "衣類", description: "プログラマー向け" },
];

const customers: Customer[] = [
  { id: 1, name: "田中太郎", email: "tanaka@example.com", address: "東京都" },
  { id: 2, name: "佐藤花子", email: "sato@example.com", address: "大阪府" },
];

// --- 関数（型安全版） ---
function findProduct(id: number): Product | undefined {
  return products.find((p: Product): boolean => p.id === id);
}

function getCustomer(id: number): Customer | undefined {
  return customers.find((c: Customer): boolean => c.id === id);
}

function calculateTotal(items: OrderProduct[]): number {
  let total: number = 0;
  for (const item of items) {
    total += item.price * item.quantity;
  }
  return total;
}

function createOrder(customerId: number, items: CartItem[]): OrderResult {
  const customer: Customer | undefined = getCustomer(customerId);
  if (customer === undefined) {
    return { success: false, error: "顧客が見つかりません" };
  }

  const orderProducts: OrderProduct[] = [];
  for (const item of items) {
    const product: Product | undefined = findProduct(item.productId);
    if (product === undefined) {
      return { success: false, error: `商品ID ${item.productId} が見つかりません` };
    }
    if (product.stock < item.quantity) {
      return { success: false, error: `${product.name} の在庫が不足しています` };
    }
    orderProducts.push({
      id: product.id,
      name: product.name,
      price: product.price,
      quantity: item.quantity,
    });
  }

  const total: number = calculateTotal(orderProducts);
  const order: Order = {
    id: Date.now(),
    products: orderProducts,
    customer: customer,
    status: "pending",
    total: total,
    createdAt: new Date().toISOString(),
  };

  return { success: true, order };
}

// ステータス遷移の定義
const validTransitions: Record<OrderStatus, OrderStatus[]> = {
  pending: ["confirmed", "cancelled"],
  confirmed: ["shipped", "cancelled"],
  shipped: ["delivered"],
  delivered: [],
  cancelled: [],
};

function updateOrderStatus(order: Order, newStatus: OrderStatus): OrderResult {
  const allowed: OrderStatus[] = validTransitions[order.status];
  if (!allowed.includes(newStatus)) {
    return {
      success: false,
      error: `${order.status} から ${newStatus} への遷移はできません`,
    };
  }
  const updated: Order = { ...order, status: newStatus };
  return { success: true, order: updated };
}

function formatOrder(order: Order): string {
  let result: string = `注文 #${order.id}\n`;
  result += `顧客: ${order.customer.name}\n`;
  result += `ステータス: ${order.status}\n`;
  result += `商品:\n`;
  for (const p of order.products) {
    result += `  - ${p.name} x${p.quantity} = ¥${(p.price * p.quantity).toLocaleString()}\n`;
  }
  result += `合計: ¥${order.total.toLocaleString()}\n`;
  return result;
}

// --- 実行 ---
console.log("===== ECサイト注文処理 =====\n");

const result: OrderResult = createOrder(1, [
  { productId: 1, quantity: 2 },
  { productId: 2, quantity: 1 },
]);

if (!result.success) {
  console.log(`エラー: ${result.error}`);
} else {
  console.log(formatOrder(result.order));

  const statusResult: OrderResult = updateOrderStatus(result.order, "confirmed");
  if (statusResult.success) {
    console.log(`ステータス更新: pending → ${statusResult.order.status}`);
  }

  // 無効な遷移のテスト
  const invalidResult: OrderResult = updateOrderStatus(result.order, "delivered");
  if (!invalidResult.success) {
    console.log(`遷移エラー: ${invalidResult.error}`);
  }
}

// 在庫不足のテスト
console.log("\n--- 在庫不足テスト ---");
const errorResult: OrderResult = createOrder(1, [
  { productId: 3, quantity: 1 },
]);
if (!errorResult.success) {
  console.log(`エラー: ${errorResult.error}`);
}
```

<summary>解答例（改良版 ─ ジェネリクスとMapped Typeの活用）</summary>

```typescript
// ECサイト注文処理（改良版）
// 学べる内容：Result型、Mapped Type、条件付き型、ユーティリティ型の組み合わせ
// 実行方法：npx ts-node ec_refactored_v2.ts

// --- 汎用Result型 ---
type Result<T, E = string> =
  | { readonly success: true; readonly data: T }
  | { readonly success: false; readonly error: E };

function ok<T>(data: T): Result<T, never> {
  return { success: true, data };
}

function err<E = string>(error: E): Result<never, E> {
  return { success: false, error };
}

// --- カテゴリとステータス ---
type ProductCategory = "書籍" | "電子機器" | "衣類" | "食品";
type OrderStatus = "pending" | "confirmed" | "shipped" | "delivered" | "cancelled";

// --- 基本型定義 ---
interface Product {
  readonly id: number;
  readonly name: string;
  readonly price: number;
  stock: number;
  readonly category: ProductCategory;
  readonly description: string;
}

interface Customer {
  readonly id: number;
  readonly name: string;
  readonly email: string;
  readonly address: string;
}

// --- ユーティリティ型の活用 ---

// Product からカタログ表示用の情報だけ取り出す
type CatalogItem = Readonly<Pick<Product, "id" | "name" | "price" | "category">>;

// 注文内の商品（Product の一部 + 数量）
type OrderLineItem = Readonly<Pick<Product, "id" | "name" | "price">> & {
  readonly quantity: number;
};

// 注文作成時の入力
type CartItem = { readonly productId: number; readonly quantity: number };

// 注文
interface Order {
  readonly id: number;
  readonly lineItems: readonly OrderLineItem[];
  readonly customer: Readonly<Customer>;
  readonly status: OrderStatus;
  readonly total: number;
  readonly createdAt: string;
}

// 注文の部分更新用
type OrderUpdate = Partial<Pick<Order, "status">>;

// --- ステータス遷移マップ ---
const STATUS_TRANSITIONS: Readonly<Record<OrderStatus, readonly OrderStatus[]>> = {
  pending: ["confirmed", "cancelled"],
  confirmed: ["shipped", "cancelled"],
  shipped: ["delivered"],
  delivered: [],
  cancelled: [],
} as const;

// --- 汎用リポジトリ ---
class Repository<T extends { readonly id: number }> {
  private items: Map<number, T>;

  constructor(items: T[]) {
    this.items = new Map(items.map((item: T): [number, T] => [item.id, item]));
  }

  findById(id: number): T | undefined {
    return this.items.get(id);
  }

  getAll(): T[] {
    return Array.from(this.items.values());
  }

  findBy<K extends keyof T>(key: K, value: T[K]): T[] {
    return this.getAll().filter((item: T): boolean => item[key] === value);
  }
}

// --- データ ---
const productRepo = new Repository<Product>([
  { id: 1, name: "TypeScript本", price: 2800, stock: 10, category: "書籍", description: "初心者向け" },
  { id: 2, name: "マウス", price: 3500, stock: 5, category: "電子機器", description: "ワイヤレス" },
  { id: 3, name: "Tシャツ", price: 2500, stock: 0, category: "衣類", description: "プログラマー向け" },
]);

const customerRepo = new Repository<Customer>([
  { id: 1, name: "田中太郎", email: "tanaka@example.com", address: "東京都" },
  { id: 2, name: "佐藤花子", email: "sato@example.com", address: "大阪府" },
]);

// --- ビジネスロジック ---
function calculateTotal(items: readonly OrderLineItem[]): number {
  return items.reduce(
    (sum: number, item: OrderLineItem): number => sum + item.price * item.quantity,
    0
  );
}

function createOrder(customerId: number, cartItems: readonly CartItem[]): Result<Order> {
  const customer: Customer | undefined = customerRepo.findById(customerId);
  if (!customer) return err("顧客が見つかりません");

  const lineItems: OrderLineItem[] = [];
  for (const cartItem of cartItems) {
    const product: Product | undefined = productRepo.findById(cartItem.productId);
    if (!product) return err(`商品ID ${cartItem.productId} が見つかりません`);
    if (product.stock < cartItem.quantity) return err(`${product.name} の在庫が不足しています`);

    lineItems.push({
      id: product.id,
      name: product.name,
      price: product.price,
      quantity: cartItem.quantity,
    });
  }

  const order: Order = {
    id: Date.now(),
    lineItems,
    customer,
    status: "pending",
    total: calculateTotal(lineItems),
    createdAt: new Date().toISOString(),
  };

  return ok(order);
}

function updateOrderStatus(order: Order, newStatus: OrderStatus): Result<Order> {
  const allowed: readonly OrderStatus[] = STATUS_TRANSITIONS[order.status];
  if (!allowed.includes(newStatus)) {
    return err(`${order.status} から ${newStatus} への遷移はできません`);
  }
  return ok({ ...order, status: newStatus });
}

function formatOrder(order: Order): string {
  const lines: string[] = [
    `注文 #${order.id}`,
    `顧客: ${order.customer.name}`,
    `ステータス: ${order.status}`,
    `商品:`,
    ...order.lineItems.map(
      (item: OrderLineItem): string =>
        `  - ${item.name} x${item.quantity} = ¥${(item.price * item.quantity).toLocaleString()}`
    ),
    `合計: ¥${order.total.toLocaleString()}`,
  ];
  return lines.join("\n");
}

// --- 実行 ---
console.log("===== ECサイト注文処理 =====\n");

const result: Result<Order> = createOrder(1, [
  { productId: 1, quantity: 2 },
  { productId: 2, quantity: 1 },
]);

if (result.success) {
  console.log(formatOrder(result.data));

  const statusResult: Result<Order> = updateOrderStatus(result.data, "confirmed");
  if (statusResult.success) {
    console.log(`\nステータス更新: pending → ${statusResult.data.status}`);
  }

  const invalidResult: Result<Order> = updateOrderStatus(result.data, "delivered");
  if (!invalidResult.success) {
    console.log(`遷移エラー: ${invalidResult.error}`);
  }
} else {
  console.log(`エラー: ${result.error}`);
}

console.log("\n--- 在庫不足テスト ---");
const errorResult: Result<Order> = createOrder(1, [{ productId: 3, quantity: 1 }]);
if (!errorResult.success) {
  console.log(`エラー: ${errorResult.error}`);
}

// カタログ表示のデモ
console.log("\n--- カタログ一覧 ---");
const catalog: CatalogItem[] = productRepo.getAll().map(
  (p: Product): CatalogItem => ({ id: p.id, name: p.name, price: p.price, category: p.category })
);
for (const item of catalog) {
  console.log(`[${item.id}] ${item.name} ¥${item.price.toLocaleString()} (${item.category})`);
}
```

**初心者向けとの違い:**
- `Result<T, E>` ジェネリック型で成功/失敗を統一的に表現
- `ok()` / `err()` ヘルパー関数で Result の生成を簡潔に
- `Repository<T>` ジェネリッククラスでデータ検索を共通化
- `Readonly`, `Pick`, `readonly` を徹底してイミュータビリティを確保
- `as const` でステータス遷移マップをリテラル型として固定
- `CatalogItem` のように用途別の型をユーティリティ型で派生

</details>
