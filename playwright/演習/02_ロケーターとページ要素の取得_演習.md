## 基本問題

### 問題1：getByRoleでボタンを取得する

以下のHTMLに対して、`getByRole` を使って「送信」ボタンと「キャンセル」ボタンをそれぞれ取得し、表示されていることを検証するテストを作成してください。

```html
<form>
  <button type="submit">送信</button>
  <button type="button">キャンセル</button>
</form>
```

**期待される結果:**
```
2つのボタンがどちらも表示されていることが検証される
```

<details>
<summary>ヒント</summary>

`page.getByRole('button', { name: 'ボタン名' })` でボタンを取得できます。`name` オプションにボタンのテキストを指定します。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('getByRoleでボタンを取得する', async ({ page }) => {
  // HTMLをセット（実際のテストではpage.gotoを使用）
  await page.setContent(`
    <form>
      <button type="submit">送信</button>
      <button type="button">キャンセル</button>
    </form>
  `);

  // getByRole でボタンを取得して検証
  const submitButton = page.getByRole('button', { name: '送信' });
  await expect(submitButton).toBeVisible();

  const cancelButton = page.getByRole('button', { name: 'キャンセル' });
  await expect(cancelButton).toBeVisible();
});
```

</details>

### 問題2：getByTextとgetByLabelを使う

以下のHTMLに対して、`getByLabel` でフォームの各入力欄を取得し、`getByText` で説明テキストを取得するテストを作成してください。

```html
<form>
  <label for="username">ユーザー名</label>
  <input id="username" type="text" />
  <label for="email">メールアドレス</label>
  <input id="email" type="email" />
  <p>必須項目はすべて入力してください。</p>
</form>
```

**期待される結果:**
```
ユーザー名とメールアドレスの入力欄が取得できる
説明テキストが表示されていることが検証される
```

<details>
<summary>ヒント</summary>

`page.getByLabel('ラベルテキスト')` は `<label>` と関連付けられた `<input>` を取得します。`page.getByText('テキスト')` はテキストを含む要素を取得します。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('getByLabelとgetByTextで要素を取得する', async ({ page }) => {
  await page.setContent(`
    <form>
      <label for="username">ユーザー名</label>
      <input id="username" type="text" />
      <label for="email">メールアドレス</label>
      <input id="email" type="email" />
      <p>必須項目はすべて入力してください。</p>
    </form>
  `);

  // getByLabel でラベルに関連付けられた入力欄を取得
  const usernameInput = page.getByLabel('ユーザー名');
  await expect(usernameInput).toBeVisible();

  const emailInput = page.getByLabel('メールアドレス');
  await expect(emailInput).toBeVisible();

  // getByText で説明テキストを取得
  const description = page.getByText('必須項目はすべて入力してください。');
  await expect(description).toBeVisible();
});
```

</details>

### 問題3：getByTestIdを使う

以下のHTMLに対して、`getByTestId` を使って各要素を取得し、テキスト内容を検証するテストを作成してください。

```html
<div>
  <span data-testid="user-name">田中太郎</span>
  <span data-testid="user-email">tanaka@example.com</span>
  <span data-testid="user-role">管理者</span>
</div>
```

**期待される結果:**
```
各要素のテキストが正しいことが検証される
```

<details>
<summary>ヒント</summary>

`page.getByTestId('テストID')` で `data-testid` 属性を持つ要素を取得できます。テキストの検証には `toHaveText()` を使います。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('getByTestIdで要素を取得しテキストを検証する', async ({ page }) => {
  await page.setContent(`
    <div>
      <span data-testid="user-name">田中太郎</span>
      <span data-testid="user-email">tanaka@example.com</span>
      <span data-testid="user-role">管理者</span>
    </div>
  `);

  // getByTestId で data-testid 属性を使って要素を取得
  await expect(page.getByTestId('user-name')).toHaveText('田中太郎');
  await expect(page.getByTestId('user-email')).toHaveText('tanaka@example.com');
  await expect(page.getByTestId('user-role')).toHaveText('管理者');
});
```

</details>

### 問題4：filterを使った要素の絞り込み

以下のHTMLに対して、`filter` を使って特定の条件に合う要素を絞り込むテストを作成してください。

```html
<ul>
  <li>
    <span>りんご</span>
    <span class="price">150円</span>
  </li>
  <li>
    <span>バナナ</span>
    <span class="price">100円</span>
  </li>
  <li>
    <span>みかん</span>
    <span class="price">200円</span>
  </li>
</ul>
```

1. `li` 要素の中から「バナナ」を含むものを `filter` で絞り込む
2. 絞り込んだ要素内の価格が「100円」であることを検証する

**期待される結果:**
```
バナナの価格が100円であることが検証される
```

<details>
<summary>ヒント</summary>

`page.locator('li').filter({ hasText: 'バナナ' })` でテキストを含む要素に絞り込めます。絞り込んだロケーターに対して `.locator('.price')` でチェインして子要素を取得できます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('filterで要素を絞り込む', async ({ page }) => {
  await page.setContent(`
    <ul>
      <li>
        <span>りんご</span>
        <span class="price">150円</span>
      </li>
      <li>
        <span>バナナ</span>
        <span class="price">100円</span>
      </li>
      <li>
        <span>みかん</span>
        <span class="price">200円</span>
      </li>
    </ul>
  `);

  // filter でバナナを含む li 要素を絞り込む
  const bananaItem = page.locator('li').filter({ hasText: 'バナナ' });

  // チェインして価格要素を取得し、テキストを検証
  const bananaPrice = bananaItem.locator('.price');
  await expect(bananaPrice).toHaveText('100円');
});
```

</details>

### 問題5：ロケーターのチェインと複合条件

以下のHTMLに対して、ロケーターのチェインを使ってテストを作成してください。

```html
<div class="sidebar">
  <nav>
    <a href="/home">ホーム</a>
    <a href="/about">概要</a>
    <a href="/contact">お問い合わせ</a>
  </nav>
</div>
<div class="main-content">
  <nav>
    <a href="/products">製品一覧</a>
    <a href="/blog">ブログ</a>
  </nav>
</div>
```

1. サイドバー内のナビゲーションリンクが3つであることを検証する
2. メインコンテンツ内のナビゲーションリンクが2つであることを検証する
3. サイドバー内に「ホーム」リンクが存在し、href属性が `/home` であることを検証する

**期待される結果:**
```
サイドバーのリンク数: 3
メインコンテンツのリンク数: 2
ホームリンクのhref属性が "/home" である
```

<details>
<summary>ヒント</summary>

`page.locator('.sidebar nav a')` のようにCSSセレクタでチェインするか、`page.locator('.sidebar').locator('nav').locator('a')` のようにロケーターをチェインできます。要素数の検証には `toHaveCount()` を使います。属性の検証には `toHaveAttribute()` を使います。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ロケーターのチェインで特定領域の要素を取得する', async ({ page }) => {
  await page.setContent(`
    <div class="sidebar">
      <nav>
        <a href="/home">ホーム</a>
        <a href="/about">概要</a>
        <a href="/contact">お問い合わせ</a>
      </nav>
    </div>
    <div class="main-content">
      <nav>
        <a href="/products">製品一覧</a>
        <a href="/blog">ブログ</a>
      </nav>
    </div>
  `);

  // 1. サイドバー内のリンク数を検証
  const sidebarLinks = page.locator('.sidebar').locator('nav a');
  await expect(sidebarLinks).toHaveCount(3);

  // 2. メインコンテンツ内のリンク数を検証
  const mainLinks = page.locator('.main-content').locator('nav a');
  await expect(mainLinks).toHaveCount(2);

  // 3. サイドバー内のホームリンクのhref属性を検証
  const homeLink = page.locator('.sidebar').getByRole('link', { name: 'ホーム' });
  await expect(homeLink).toHaveAttribute('href', '/home');
});
```

</details>

### 問題6：nthとfirstとlastを使う

以下のHTMLに対して、`nth()`、`first()`、`last()` を使ってリスト内の特定要素を取得し、テキストを検証するテストを作成してください。

```html
<ol class="ranking">
  <li>金メダル - 佐藤選手</li>
  <li>銀メダル - 鈴木選手</li>
  <li>銅メダル - 高橋選手</li>
  <li>4位 - 田中選手</li>
  <li>5位 - 伊藤選手</li>
</ol>
```

1. 最初の要素（1位）のテキストを検証する
2. 3番目の要素（3位）のテキストを検証する
3. 最後の要素（5位）のテキストを検証する

**期待される結果:**
```
1位: "金メダル - 佐藤選手"
3位: "銅メダル - 高橋選手"
5位: "5位 - 伊藤選手"
```

<details>
<summary>ヒント</summary>

`locator.first()` は最初の要素、`locator.last()` は最後の要素、`locator.nth(n)` はn番目（0始まり）の要素を返します。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('nth, first, lastで特定の要素を取得する', async ({ page }) => {
  await page.setContent(`
    <ol class="ranking">
      <li>金メダル - 佐藤選手</li>
      <li>銀メダル - 鈴木選手</li>
      <li>銅メダル - 高橋選手</li>
      <li>4位 - 田中選手</li>
      <li>5位 - 伊藤選手</li>
    </ol>
  `);

  const items = page.locator('.ranking li');

  // first() で最初の要素を取得
  await expect(items.first()).toHaveText('金メダル - 佐藤選手');

  // nth(2) で3番目の要素を取得（0始まりなのでインデックスは2）
  await expect(items.nth(2)).toHaveText('銅メダル - 高橋選手');

  // last() で最後の要素を取得
  await expect(items.last()).toHaveText('5位 - 伊藤選手');
});
```

</details>

### 問題7：CSSセレクタをgetByRoleに書き換える

以下のテストコードはCSSセレクタを使って要素を取得しています。これを可能な限り `getByRole`、`getByLabel`、`getByText` などのセマンティックなロケーターに書き換えてください。

```html
<header>
  <h1>ショッピングサイト</h1>
  <nav>
    <a href="/cart">カート</a>
    <a href="/mypage">マイページ</a>
  </nav>
</header>
<main>
  <form>
    <label for="search">商品検索</label>
    <input id="search" type="text" placeholder="キーワードを入力" />
    <button type="submit">検索</button>
  </form>
  <section>
    <h2>おすすめ商品</h2>
    <ul>
      <li>商品A - 1,000円</li>
      <li>商品B - 2,000円</li>
    </ul>
  </section>
</main>
```

**書き換え前のコード:**
```typescript
const heading = page.locator('h1');
const cartLink = page.locator('nav a[href="/cart"]');
const searchInput = page.locator('#search');
const searchButton = page.locator('button[type="submit"]');
const sectionHeading = page.locator('section h2');
```

**期待される結果:**
```
CSSセレクタがセマンティックなロケーターに書き換えられている
テストの動作結果は変わらない
```

<details>
<summary>ヒント</summary>

- `h1` → `getByRole('heading', { level: 1 })`
- `a[href="/cart"]` → `getByRole('link', { name: 'カート' })`
- `#search` → `getByLabel('商品検索')` または `getByPlaceholder('キーワードを入力')`
- `button[type="submit"]` → `getByRole('button', { name: '検索' })`

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('セマンティックなロケーターでショッピングサイトの要素を取得する', async ({ page }) => {
  await page.setContent(`
    <header>
      <h1>ショッピングサイト</h1>
      <nav>
        <a href="/cart">カート</a>
        <a href="/mypage">マイページ</a>
      </nav>
    </header>
    <main>
      <form>
        <label for="search">商品検索</label>
        <input id="search" type="text" placeholder="キーワードを入力" />
        <button type="submit">検索</button>
      </form>
      <section>
        <h2>おすすめ商品</h2>
        <ul>
          <li>商品A - 1,000円</li>
          <li>商品B - 2,000円</li>
        </ul>
      </section>
    </main>
  `);

  // Before: page.locator('h1')
  // After: getByRole で見出しレベルを指定
  const heading = page.getByRole('heading', { level: 1 });
  await expect(heading).toHaveText('ショッピングサイト');

  // Before: page.locator('nav a[href="/cart"]')
  // After: getByRole でリンク名を指定
  const cartLink = page.getByRole('link', { name: 'カート' });
  await expect(cartLink).toBeVisible();

  // Before: page.locator('#search')
  // After: getByLabel でラベルテキストを指定
  const searchInput = page.getByLabel('商品検索');
  await expect(searchInput).toBeVisible();

  // Before: page.locator('button[type="submit"]')
  // After: getByRole でボタン名を指定
  const searchButton = page.getByRole('button', { name: '検索' });
  await expect(searchButton).toBeVisible();

  // Before: page.locator('section h2')
  // After: getByRole で見出しレベル2を指定
  const sectionHeading = page.getByRole('heading', { level: 2 });
  await expect(sectionHeading).toHaveText('おすすめ商品');
});
```

**書き換えのポイント:**

| 書き換え前 | 書き換え後 | 理由 |
|---|---|---|
| `locator('h1')` | `getByRole('heading', { level: 1 })` | アクセシビリティロールで指定 |
| `locator('nav a[href="/cart"]')` | `getByRole('link', { name: 'カート' })` | ユーザーに見えるテキストで指定 |
| `locator('#search')` | `getByLabel('商品検索')` | ラベルとの関連で指定 |
| `locator('button[type="submit"]')` | `getByRole('button', { name: '検索' })` | ボタンロールとテキストで指定 |
| `locator('section h2')` | `getByRole('heading', { level: 2 })` | 見出しレベルで指定 |

</details>

### 問題8：重複要素の特定と処理

以下のHTMLには同じテキストを持つ要素が複数あります。各「詳細」ボタンを正しく区別して操作するテストを作成してください。

```html
<div class="product-list">
  <div class="product" data-testid="product-1">
    <h3>ノートPC</h3>
    <p class="price">120,000円</p>
    <button>詳細</button>
  </div>
  <div class="product" data-testid="product-2">
    <h3>タブレット</h3>
    <p class="price">50,000円</p>
    <button>詳細</button>
  </div>
  <div class="product" data-testid="product-3">
    <h3>スマートフォン</h3>
    <p class="price">80,000円</p>
    <button>詳細</button>
  </div>
</div>
```

1. 「タブレット」の「詳細」ボタンだけをクリックする（filterを使用）
2. 各商品の価格をすべて検証する
3. 「詳細」ボタンの総数が3つであることを検証する

**期待される結果:**
```
タブレットの詳細ボタンだけがクリックされる
各商品の価格が正しいことが検証される
詳細ボタンの総数が3であることが検証される
```

<details>
<summary>ヒント</summary>

`filter({ hasText: 'タブレット' })` で商品カードを絞り込み、その中の `button` をクリックします。`has` オプションを使うと、子要素を条件に絞り込むこともできます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('重複する要素を正しく区別して操作する', async ({ page }) => {
  await page.setContent(`
    <div class="product-list">
      <div class="product" data-testid="product-1">
        <h3>ノートPC</h3>
        <p class="price">120,000円</p>
        <button>詳細</button>
      </div>
      <div class="product" data-testid="product-2">
        <h3>タブレット</h3>
        <p class="price">50,000円</p>
        <button>詳細</button>
      </div>
      <div class="product" data-testid="product-3">
        <h3>スマートフォン</h3>
        <p class="price">80,000円</p>
        <button>詳細</button>
      </div>
    </div>
  `);

  // 1. filter でタブレットの商品カードを絞り込み、詳細ボタンをクリック
  const tabletCard = page.locator('.product').filter({ hasText: 'タブレット' });
  await tabletCard.getByRole('button', { name: '詳細' }).click();

  // 2. 各商品の価格を検証
  // data-testid を使って個別に取得する方法
  await expect(page.getByTestId('product-1').locator('.price')).toHaveText('120,000円');
  await expect(page.getByTestId('product-2').locator('.price')).toHaveText('50,000円');
  await expect(page.getByTestId('product-3').locator('.price')).toHaveText('80,000円');

  // 3. 詳細ボタンの総数を検証
  const detailButtons = page.getByRole('button', { name: '詳細' });
  await expect(detailButtons).toHaveCount(3);
});
```

</details>

### 問題9：getByPlaceholderとgetByAltTextを使う

以下のHTMLに対して、`getByPlaceholder` と `getByAltText` を使って要素を取得し、検証するテストを作成してください。

```html
<div class="profile-editor">
  <img src="/avatar.png" alt="プロフィール画像" />
  <input type="text" placeholder="名前を入力してください" />
  <input type="email" placeholder="メールアドレスを入力してください" />
  <textarea placeholder="自己紹介を書いてください"></textarea>
  <img src="/badge.png" alt="認証バッジ" />
</div>
```

1. プロフィール画像が表示されていることを検証する
2. 各入力欄にテキストを入力する
3. 認証バッジ画像が表示されていることを検証する

**期待される結果:**
```
プロフィール画像が表示されている
各入力欄にテキストが入力される
認証バッジが表示されている
```

<details>
<summary>ヒント</summary>

`page.getByAltText('alt属性のテキスト')` で画像を取得できます。`page.getByPlaceholder('プレースホルダーテキスト')` で入力欄を取得できます。テキスト入力には `fill()` メソッドを使います。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('getByPlaceholderとgetByAltTextで要素を取得する', async ({ page }) => {
  await page.setContent(`
    <div class="profile-editor">
      <img src="/avatar.png" alt="プロフィール画像" />
      <input type="text" placeholder="名前を入力してください" />
      <input type="email" placeholder="メールアドレスを入力してください" />
      <textarea placeholder="自己紹介を書いてください"></textarea>
      <img src="/badge.png" alt="認証バッジ" />
    </div>
  `);

  // 1. getByAltText でプロフィール画像を取得して検証
  const profileImage = page.getByAltText('プロフィール画像');
  await expect(profileImage).toBeVisible();

  // 2. getByPlaceholder で各入力欄を取得してテキストを入力
  await page.getByPlaceholder('名前を入力してください').fill('田中太郎');
  await page.getByPlaceholder('メールアドレスを入力してください').fill('tanaka@example.com');
  await page.getByPlaceholder('自己紹介を書いてください').fill('よろしくお願いします。');
```
