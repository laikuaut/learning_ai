# 第3章：繰り返し

## この章のゴール

- for文を使って回数指定の繰り返しができる
- while文・do-while文で条件による繰り返しができる
- 拡張for文（for-each）で配列やコレクションを反復できる
- break / continue で繰り返しの制御ができる
- ネストされたループを理解し活用できる
- FizzBuzz・九九表などの定番パターンが書ける

---

## 3.1 for文

**for文（for statement）** は、繰り返し回数が決まっている場合に使うループです。

### 基本構文

```
for (初期化; 条件式; 更新) {
    繰り返す処理;
}
```

### 実行の流れ

```
  +---------+
  |  初期化  | ← 最初に1回だけ実行
  +----+----+
       |
       v
  +---------+     false
  | 条件式?  |──────────> ループ終了
  +----+----+
       | true
       v
  +---------+
  |  処理   |
  +----+----+
       |
       v
  +---------+
  |  更新   |
  +----+----+
       |
       +──────> 条件式に戻る
```

```java
public class ForDemo {
    public static void main(String[] args) {
        // 基本のfor文：1〜5を表示
        System.out.println("--- 1〜5 ---");
        for (int i = 1; i <= 5; i++) {
            System.out.println("i = " + i);
        }

        // カウントダウン
        System.out.println("\n--- カウントダウン ---");
        for (int i = 5; i >= 1; i--) {
            System.out.print(i + " ");
        }
        System.out.println("発射！");

        // 2ずつ増やす（偶数の列挙）
        System.out.println("\n--- 偶数（2〜10） ---");
        for (int i = 2; i <= 10; i += 2) {
            System.out.print(i + " ");
        }
        System.out.println();

        // 合計の計算
        System.out.println("\n--- 1〜100の合計 ---");
        int sum = 0;
        for (int i = 1; i <= 100; i++) {
            sum += i;
        }
        System.out.println("合計: " + sum);
    }
}
```

実行方法：
```bash
javac ForDemo.java && java ForDemo
```

実行結果：
```
--- 1〜5 ---
i = 1
i = 2
i = 3
i = 4
i = 5

--- カウントダウン ---
5 4 3 2 1 発射！

--- 偶数（2〜10） ---
2 4 6 8 10

--- 1〜100の合計 ---
合計: 5050
```

> **よくある間違い：** `for (int i = 1; i <= 5; i++)` と `for (int i = 0; i < 5; i++)` は同じ5回のループですが、`i` の値が異なります。配列のインデックスは0から始まるため、配列を扱うときは `i = 0; i < length` のパターンが一般的です。

### ポイントまとめ

- for文は「初期化; 条件; 更新」の3つの要素で構成
- ループ変数のスコープはforブロック内のみ
- 増分は `i++`、`i--`、`i += 2` など自由に設定可能
- 回数が決まっている繰り返しに最適

## 3.2 while文

**while文（while statement）** は、条件が `true` の間、処理を繰り返します。繰り返し回数が事前に分からない場合に使います。

```java
public class WhileDemo {
    public static void main(String[] args) {
        // 基本のwhile文
        System.out.println("--- 1〜5 ---");
        int i = 1;
        while (i <= 5) {
            System.out.println("i = " + i);
            i++;
        }

        // 実用例：桁数を数える
        System.out.println("\n--- 桁数カウント ---");
        int number = 12345;
        int original = number;
        int digits = 0;

        while (number > 0) {
            number /= 10;  // 10で割って1桁ずつ減らす
            digits++;
        }
        System.out.println(original + "は" + digits + "桁です。");

        // 実用例：2の累乗を1000まで
        System.out.println("\n--- 2の累乗（1000以下） ---");
        int power = 1;
        while (power <= 1000) {
            System.out.print(power + " ");
            power *= 2;
        }
        System.out.println();
    }
}
```

実行方法：
```bash
javac WhileDemo.java && java WhileDemo
```

実行結果：
```
--- 1〜5 ---
i = 1
i = 2
i = 3
i = 4
i = 5

--- 桁数カウント ---
12345は5桁です。

--- 2の累乗（1000以下） ---
1 2 4 8 16 32 64 128 256 512
```

### ポイントまとめ

- while文は条件が true の間、処理を繰り返す
- ループ変数の初期化はwhileの前、更新はwhileの中で行う
- 繰り返し回数が不明な場合に適している
- 条件が最初から false なら1回も実行されない

## 3.3 do-while文

**do-while文（do-while statement）** は、必ず1回は処理を実行してから、条件をチェックするループです。

```java
import java.util.Scanner;

public class DoWhileDemo {
    public static void main(String[] args) {
        // 基本のdo-while文
        System.out.println("--- 1〜5 ---");
        int i = 1;
        do {
            System.out.println("i = " + i);
            i++;
        } while (i <= 5);

        // 実用例：入力値の検証（メニュー表示）
        Scanner scanner = new Scanner(System.in);
        int choice;

        do {
            System.out.println("\n=== メニュー ===");
            System.out.println("1. 挨拶する");
            System.out.println("2. 時刻を表示");
            System.out.println("3. 計算する");
            System.out.println("0. 終了");
            System.out.print("選択してください: ");
            choice = scanner.nextInt();

            switch (choice) {
                case 1 -> System.out.println("こんにちは！");
                case 2 -> System.out.println("現在時刻: " + java.time.LocalTime.now());
                case 3 -> System.out.println("1 + 1 = 2");
                case 0 -> System.out.println("プログラムを終了します。");
                default -> System.out.println("無効な選択です。");
            }
        } while (choice != 0);

        scanner.close();
    }
}
```

実行方法：
```bash
javac DoWhileDemo.java && java DoWhileDemo
```

実行例：
```
--- 1〜5 ---
i = 1
i = 2
i = 3
i = 4
i = 5

=== メニュー ===
1. 挨拶する
2. 時刻を表示
3. 計算する
0. 終了
選択してください: 1
こんにちは！

=== メニュー ===
1. 挨拶する
2. 時刻を表示
3. 計算する
0. 終了
選択してください: 0
プログラムを終了します。
```

### while vs do-while の比較

| 特徴 | while | do-while |
|---|---|---|
| 条件チェック | 処理の**前** | 処理の**後** |
| 最低実行回数 | 0回 | **1回** |
| 適した用途 | 一般的なループ | メニュー表示、入力検証 |

> **よくある間違い：** do-while文の末尾の `while(条件);` にはセミコロンが必要です。忘れるとコンパイルエラーになります。

### ポイントまとめ

- do-while文は必ず1回は処理を実行する
- メニュー表示や入力検証に適している
- while文との違いは「条件チェックのタイミング」
- 末尾のセミコロンを忘れないこと

## 3.4 拡張for文（for-each）

**拡張for文（enhanced for statement）** は、配列やコレクションの全要素を順番に処理するためのループです。

```java
public class ForEachDemo {
    public static void main(String[] args) {
        // 配列の全要素を表示
        String[] fruits = {"りんご", "バナナ", "みかん", "ぶどう", "いちご"};

        System.out.println("--- 果物一覧 ---");
        for (String fruit : fruits) {
            System.out.println(fruit);
        }

        // 数値配列の合計
        int[] scores = {85, 92, 78, 95, 88};
        int total = 0;

        for (int score : scores) {
            total += score;
        }
        double average = (double) total / scores.length;

        System.out.println("\n--- 成績 ---");
        System.out.println("合計: " + total + "点");
        System.out.printf("平均: %.1f点%n", average);

        // 最大値・最小値の探索
        int max = scores[0];
        int min = scores[0];

        for (int score : scores) {
            if (score > max) max = score;
            if (score < min) min = score;
        }

        System.out.println("最高点: " + max + "点");
        System.out.println("最低点: " + min + "点");
    }
}
```

実行方法：
```bash
javac ForEachDemo.java && java ForEachDemo
```

実行結果：
```
--- 果物一覧 ---
りんご
バナナ
みかん
ぶどう
いちご

--- 成績 ---
合計: 438点
平均: 87.6点
最高点: 95点
最低点: 78点
```

### for文 vs 拡張for文

| 特徴 | 通常のfor文 | 拡張for文（for-each） |
|---|---|---|
| インデックス | 使える | 使えない |
| 要素の変更 | 可能 | 不可 |
| コードの簡潔さ | やや長い | 簡潔 |
| 適した場面 | インデックスが必要な時 | 全要素を順番に処理する時 |

### ポイントまとめ

- `for (型 変数 : 配列)` で全要素を順番に処理できる
- インデックスが不要な場合は拡張for文を優先して使う
- 要素の追加・削除・インデックスが必要な場合は通常のfor文を使う

## 3.5 break と continue

### break：ループを途中で抜ける

```java
public class BreakDemo {
    public static void main(String[] args) {
        // 探索：特定の値を見つけたらループを抜ける
        int[] numbers = {3, 7, 1, 9, 4, 6, 8, 2, 5};
        int target = 4;

        System.out.println("--- " + target + "を探索 ---");
        for (int i = 0; i < numbers.length; i++) {
            System.out.println("チェック: " + numbers[i]);
            if (numbers[i] == target) {
                System.out.println(target + "を見つけました！（インデックス: " + i + "）");
                break;  // ループを抜ける
            }
        }
    }
}
```

実行方法：
```bash
javac BreakDemo.java && java BreakDemo
```

実行結果：
```
--- 4を探索 ---
チェック: 3
チェック: 7
チェック: 1
チェック: 9
チェック: 4
4を見つけました！（インデックス: 4）
```

### continue：現在の回をスキップする

```java
public class ContinueDemo {
    public static void main(String[] args) {
        // 偶数だけ表示（奇数をスキップ）
        System.out.println("--- 1〜10の偶数 ---");
        for (int i = 1; i <= 10; i++) {
            if (i % 2 != 0) {
                continue;  // 奇数はスキップ
            }
            System.out.print(i + " ");
        }
        System.out.println();

        // 特定の値を除外して合計
        System.out.println("\n--- 負の数を除外して合計 ---");
        int[] values = {10, -5, 20, -3, 15, -8, 25};
        int sum = 0;

        for (int value : values) {
            if (value < 0) {
                continue;  // 負の数はスキップ
            }
            sum += value;
        }
        System.out.println("正の数の合計: " + sum);
    }
}
```

実行方法：
```bash
javac ContinueDemo.java && java ContinueDemo
```

実行結果：
```
--- 1〜10の偶数 ---
2 4 6 8 10

--- 負の数を除外して合計 ---
正の数の合計: 70
```

### break と continue のフローの違い

```
break:                          continue:
for (...) {                     for (...) {
    処理A                           処理A
    if (条件) break; ──┐            if (条件) continue; ──┐
    処理B              │            処理B                  │
}                      │        }  <───────────────────────┘
次の処理 <─────────────┘        次の処理
```

> **よくある間違い：** `break` は **最も内側のループだけ** を抜けます。ネストされたループで外側も抜けたい場合は、ラベル付きbreakを使うか、フラグ変数を使いましょう。

### ポイントまとめ

- `break` はループを完全に終了する
- `continue` は現在の回をスキップして次の回に進む
- 使いすぎるとコードが読みにくくなるので注意

```java
public class ShapeDrawing {
    public static void main(String[] args) {
        int size = 5;

        // 直角三角形
        System.out.println("--- 直角三角形 ---");
        for (int i = 1; i <= size; i++) {
            for (int j = 0; j < i; j++) {
                System.out.print("* ");
            }
            System.out.println();
        }

        // 逆三角形
        System.out.println("\n--- 逆三角形 ---");
        for (int i = size; i >= 1; i--) {
            for (int j = 0; j < i; j++) {
                System.out.print("* ");
            }
            System.out.println();
        }

        // ピラミッド
        System.out.println("\n--- ピラミッド ---");
        for (int i = 1; i <= size; i++) {
            // 空白
            for (int j = 0; j < size - i; j++) {
                System.out.print(" ");
            }
            // 星
            for (int j = 0; j < 2 * i - 1; j++) {
                System.out.print("*");
            }
            System.out.println();
        }
    }
}
```

### 素数判定

```java
public class PrimeChecker {
    public static void main(String[] args) {
        System.out.println("=== 2〜50の素数 ===");

        for (int num = 2; num <= 50; num++) {
            boolean isPrime = true;

            for (int i = 2; i <= Math.sqrt(num); i++) {
                if (num % i == 0) {
                    isPrime = false;
                    break;
                }
            }

            if (isPrime) {
                System.out.print(num + " ");
            }
        }
        System.out.println();
    }
}
```

### 数字当てゲーム

```java
import java.util.Scanner;
import java.util.Random;

public class NumberGuessing {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        Random random = new Random();

        int answer = random.nextInt(100) + 1;  // 1〜100のランダムな数
        int attempts = 0;

        System.out.println("=== 数字当てゲーム ===");
        System.out.println("1〜100の数字を当ててください。");

        while (true) {
            System.out.print("予想を入力: ");
            int guess = scanner.nextInt();
            attempts++;

            if (guess == answer) {
                System.out.println("正解！ " + attempts + "回目で当てました！");
                break;
            } else if (guess < answer) {
                System.out.println("もっと大きい数です。");
            } else {
                System.out.println("もっと小さい数です。");
            }
        }

        scanner.close();
    }
}
```

実行方法：
```bash
javac NumberGuessing.java && java NumberGuessing
```

実行例：
```
=== 数字当てゲーム ===
1〜100の数字を当ててください。
予想を入力: 50
もっと大きい数です。
予想を入力: 75
もっと小さい数です。
予想を入力: 62
正解！ 3回目で当てました！
```

### ポイントまとめ

- FizzBuzzは条件分岐とループの組み合わせの定番問題
- 素数判定は `Math.sqrt()` を使うと効率的
- `while (true)` + `break` のパターンで「条件を満たすまで繰り返す」を表現

## 3.8 無限ループとよくあるバグ

### 無限ループの例

```java
// 【注意】これらは実行しないでください！

// パターン1：更新忘れ
// int i = 0;
// while (i < 5) {
//     System.out.println(i);
//     // i++; を忘れると永遠にi=0
// }

// パターン2：条件の間違い
// for (int i = 0; i >= 0; i++) {
//     System.out.println(i);
//     // iは増え続けるので条件が常にtrue
// }

// パターン3：セミコロンの誤り
// for (int i = 0; i < 5; i++); {  // セミコロンが余分！
//     System.out.println(i);       // 1回だけ実行される
// }
```

> **よくある間違い：** `for` 文の `()` の直後にセミコロンを付けると、空のループが実行されてしまいます。`for (int i = 0; i < 5; i++);` は「何もしないループを5回回す」という意味になり、`{}` 内の処理は1回だけ実行されます。

### 無限ループ発生時の対処

プログラムが止まらなくなったら、ターミナルで `Ctrl + C` を押してプログラムを強制終了してください。

### ポイントまとめ

- ループ変数の更新忘れは無限ループの最大の原因
- for文のセミコロンの位置に注意
- 条件式が常に true にならないか確認する
- 無限ループになったら `Ctrl + C` で強制終了

## まとめ

この章では、プログラムの「繰り返し」を学びました。

1. **for文**：回数が決まっている繰り返し
2. **while文**：条件が true の間の繰り返し
3. **do-while文**：必ず1回は実行する繰り返し
4. **拡張for文**：配列の全要素を処理
5. **break**：ループを途中で終了
6. **continue**：現在の回をスキップ
7. **ネストループ**：ループの中のループ
8. **定番パターン**：FizzBuzz、素数判定、数字当てゲーム

### ループの使い分け

| 場面 | 推奨するループ |
|---|---|
| 回数が決まっている | for文 |
| 条件が満たされるまで | while文 |
| 必ず1回は実行したい | do-while文 |
| 配列・コレクションの全要素 | 拡張for文 |

## 次章への橋渡し

条件分岐とループをマスターしたことで、基本的な処理フローはすべて書けるようになりました。しかし、データが増えてくると変数を1つずつ宣言するのは非効率です。

次の第4章では、**配列とコレクション（ArrayList・HashMap）** を学び、大量のデータを効率的に管理する方法を習得します。
