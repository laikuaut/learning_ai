# 第6章 演習：バッファ・ウィンドウ・タブ

この演習では、Vimのバッファ・ウィンドウ・タブ操作を実践的に学びます。複数ファイルを効率よく扱う技術はVimの生産性を大きく向上させます。

---

## 基本問題

### 問題1：バッファ操作

以下のシナリオに沿って、バッファの基本操作を行ってください。

1. Vimで `index.html` を開いている状態で、新たに `style.css` を開いてください
2. 現在開いているバッファの一覧を表示してください
3. 前のバッファ（`index.html`）に切り替えてください
4. 次のバッファ（`style.css`）に切り替えてください

**期待される操作結果：**

```
1. style.css がバッファに読み込まれ、画面に表示されます
2. バッファ一覧が表示されます（例）：
     1      "index.html"     line 1
     2 %a   "style.css"      line 1
3. index.html が画面に表示されます
4. style.css が画面に表示されます
```

<details>
<summary>ヒント</summary>

- 新しいファイルを開くには `:e ファイル名` を使います
- バッファ一覧の表示にはExコマンドを使います
- バッファの切り替えには「前」と「次」に対応するコマンドがあります

</details>

<details>
<summary>解答例</summary>

```vim
" 1. 新しいファイルをバッファに開く
:e style.css
" :e（edit）は指定したファイルを新しいバッファとして開きます
" 現在のバッファは裏に隠れますが、メモリ上には残っています

" 2. バッファ一覧を表示する
:ls
" :ls または :buffers でバッファ一覧を確認できます
" %a は「アクティブ（現在表示中）」を意味します
" # は「直前にいたバッファ」を意味します

" 3. 前のバッファに切り替える
:bp
" :bp（buffer previous）で前のバッファに移動します
" :bprevious の短縮形です

" 4. 次のバッファに切り替える
:bn
" :bn（buffer next）で次のバッファに移動します
" :bnext の短縮形です

" 補足：バッファ番号を指定して直接切り替えることもできます
:b1
" :b1 でバッファ番号1（index.html）に切り替わります
" :b# で直前のバッファに切り替える方法も便利です
```

</details>

### 問題2：ウィンドウ分割

以下の操作を順番に行ってください。

1. 現在のウィンドウを水平に2分割してください
2. 下のウィンドウに移動してください
3. 下のウィンドウで `app.js` を開いてください
4. 上のウィンドウに戻ってください
5. 今度は垂直に分割してください

**期待される操作結果：**

```
操作1後：同じファイルが上下2つのウィンドウに表示されます
操作3後：下のウィンドウに app.js が表示されます
操作5後：画面が上段（左右2つ）＋下段（1つ）の3つのウィンドウになります
```

<details>
<summary>ヒント</summary>

- 水平分割と垂直分割にはそれぞれ対応するExコマンドがあります
- ウィンドウ間の移動には `Ctrl+w` を起点としたキー操作を使います
- 分割後に別のファイルを開くには `:e` が使えます

</details>

<details>
<summary>解答例</summary>

```vim
" 1. 水平分割（上下に分ける）
:sp
" :sp（split）で水平分割します
" :split の短縮形です
" 分割直後は両方のウィンドウに同じファイルが表示されます

" 2. 下のウィンドウに移動
Ctrl+w j
" Ctrl+w のあとに j（下）を押します
" h/j/k/l でそれぞれ左/下/上/右のウィンドウに移動できます

" 3. 下のウィンドウで app.js を開く
:e app.js
" 現在のウィンドウ（下）で app.js が表示されます

" 4. 上のウィンドウに戻る
Ctrl+w k
" k は上方向への移動です

" 5. 垂直分割（左右に分ける）
:vs
" :vs（vsplit）で垂直分割します
" :vertical split の短縮形です

" 補足：ウィンドウを閉じるには
:q
" 現在のウィンドウだけを閉じます（最後のウィンドウの場合はVimが終了します）
" Ctrl+w c でも現在のウィンドウを閉じられます
" Ctrl+w o で現在のウィンドウ以外をすべて閉じます
```

</details>

### 問題3：タブ操作

以下の操作を順番に行ってください。

1. 新しいタブを開いてください
2. そのタブで `README.md` を開いてください
3. 前のタブに移動してください
4. 次のタブに移動してください
5. 現在のタブを閉じてください

**期待される操作結果：**

```
1. 新しい空のタブが開きます（画面上部にタブバーが表示されます）
2. 新しいタブに README.md が表示されます
3. 最初のタブに戻ります
4. README.md のタブに移動します
5. README.md のタブが閉じ、最初のタブだけが残ります
```

<details>
<summary>ヒント</summary>

- タブの作成はExコマンドで行います
- タブの移動にはノーマルモードのキーコマンドが便利です
- タブを閉じるにはウィンドウを閉じるのと同様のコマンドが使えます

</details>

<details>
<summary>解答例</summary>

```vim
" 1. 新しいタブを開く
:tabnew
" 新しい空のタブが作成され、そこに移動します
" 画面上部にタブバーが表示されます

" 2. タブで README.md を開く
:e README.md
" もしくは最初から :tabnew README.md で一度に開くこともできます

" 3. 前のタブに移動する
gT
" ノーマルモードで gT を押すと前のタブに移動します
" :tabprevious でも同じです

" 4. 次のタブに移動する
gt
" ノーマルモードで gt を押すと次のタブに移動します
" :tabnext でも同じです

" 5. 現在のタブを閉じる
:tabclose
" 現在のタブを閉じます
" :tabc と短縮できます
" 最後のタブの場合は Vim が終了します

" 補足：タブ番号を指定して移動することもできます
" 2gt → 2番目のタブに直接移動
" :tabs → 開いているタブの一覧を確認
```

</details>

## 応用問題

### 問題4：分割ウィンドウでの並行編集

以下のシナリオを実践してください。HTMLファイルとCSSファイルを左右に並べて同時に編集する場面を想定しています。

1. `index.html` をVimで開く
2. 垂直分割して右のウィンドウに `style.css` を開く
3. 右のウィンドウの幅を10カラム広くする
4. 左のウィンドウに移動し、両方のウィンドウを同じ幅に揃える

**期待される操作結果：**

```
操作2後：左に index.html、右に style.css が並んで表示されます
操作3後：右のウィンドウが10カラム分広くなります
操作4後：両方のウィンドウが同じ幅に戻ります
```

<details>
<summary>ヒント</summary>

- `:vs ファイル名` で分割と同時にファイルを開けます
- ウィンドウのリサイズには `Ctrl+w` に続けてサイズ変更のキーを使います
- `>` や `<` でウィンドウの幅を変更できます
- 均等配分にするコマンドもあります

</details>

<details>
<summary>解答例</summary>

```vim
" 1. index.html を開く（ターミナルから）
" vim index.html

" 2. 垂直分割して右に style.css を開く
:vs style.css
" :vs ファイル名 で分割と同時にファイルを開けます
" 新しいウィンドウ（右）にカーソルが移動します

" 3. 右のウィンドウの幅を10カラム広くする
10 Ctrl+w >
" 数字を前置することで繰り返し回数を指定できます
" Ctrl+w > は現在のウィンドウを1カラム広くします
" Ctrl+w < は1カラム狭くします

" 4. 左のウィンドウに移動し、均等幅にする
Ctrl+w h
" 左のウィンドウに移動します

Ctrl+w =
" すべてのウィンドウを均等な大きさに揃えます
" レイアウトが崩れたときにリセットするのに便利です

" 補足：ウィンドウサイズの調整コマンド一覧
" Ctrl+w + → 現在のウィンドウの高さを1行増やす
" Ctrl+w - → 現在のウィンドウの高さを1行減らす
" Ctrl+w > → 現在のウィンドウの幅を1カラム増やす
" Ctrl+w < → 現在のウィンドウの幅を1カラム減らす
" Ctrl+w = → すべてのウィンドウを均等にする
" Ctrl+w _ → 現在のウィンドウの高さを最大化する
" Ctrl+w | → 現在のウィンドウの幅を最大化する
```

</details>

### 問題5：netrwによるファイルブラウジング

以下の操作でVim内蔵のファイルエクスプローラ（netrw）を使ってみましょう。

1. 現在のディレクトリをnetrwで開いてください
2. netrwで目的のファイルを選択して開いてください
3. 垂直分割でnetrwを開いてサイドバーのように使ってください

**期待される操作結果：**

```
1. ディレクトリの中身がファイル一覧として表示されます
2. 選択したファイルが現在のウィンドウで開かれます
3. 左側にファイルツリー、右側にファイル内容が表示されます
```

<details>
<summary>ヒント</summary>

- netrwを開くにはディレクトリ名を `:e` で開きます
- `.` は現在のディレクトリを表します
- Vimにはnetrw専用のExコマンドもあります
- netrw内では `Enter` でファイル/ディレクトリを開きます

</details>

<details>
<summary>解答例</summary>

```vim
" 1. 現在のディレクトリをnetrwで開く
:e .
" ピリオドは現在のディレクトリを表します
" :Explore（短縮形 :Ex）でも同じ結果になります

" 2. ファイルを選択して開く
" netrw画面でカーソルを目的のファイルに合わせて Enter を押します
" j/k でカーソルを上下に移動し、Enter で決定します
" ディレクトリ名の上で Enter を押すとそのディレクトリに移動します
" 「..」を選ぶと1つ上のディレクトリに移動します

" 3. 垂直分割でnetrwを開く（サイドバー風）
:Vex
" :Vexplore の短縮形です
" 左側にファイルツリー、右側にファイル内容が表示されます
" netrwで選択したファイルが右側のウィンドウで開きます

" 補足：netrwの便利な操作
" :Sex → 水平分割でnetrwを開く（:Sexplore の短縮形）
" :Lex → 左側に固定幅のnetrwを開く（:Lexplore の短縮形）
" netrw内のキー操作：
"   Enter → ファイルを開く / ディレクトリに入る
"   - → 親ディレクトリへ移動
"   d → 新しいディレクトリを作成
"   % → 新しいファイルを作成
"   i → 表示形式を切り替え（リスト/詳細/ツリー）
```

</details>

## チャレンジ問題

### 問題6：複数ファイルプロジェクトの効率的な管理

以下のWebプロジェクトのファイル構成を想定して、効率的なファイル管理のワークフローを構築してください。

```
project/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── app.js
│   └── utils.js
└── README.md
```

**課題：** 以下の一連の操作を行い、使用したコマンドをすべて記録してください。

1. `project/` ディレクトリで Vim を起動し、`index.html` を開く
2. タブを使って `css/style.css` も開く
3. さらにタブを使って `js/app.js` も開く
4. `js/app.js` のタブ内でウィンドウを垂直分割し、`js/utils.js` も開く
5. 最初のタブ（`index.html`）に戻る
6. 開いているすべてのバッファの一覧を確認する
7. `README.md` をバッファに追加して、すぐに `index.html` のバッファに戻る
8. すべてのファイルを保存する

**期待される最終状態：**

```
タブ1: index.html
タブ2: css/style.css
タブ3: [js/app.js | js/utils.js]（垂直分割）
バッファ: index.html, style.css, app.js, utils.js, README.md（5ファイル）
```

<details>
<summary>ヒント</summary>

- `:tabnew ファイル名` でタブを開きつつファイルを読み込めます
- タブの移動は `gt` / `gT` または番号指定 `1gt` が使えます
- `:badd` でバッファにファイルを追加できます（表示はしない）
- `:wall`（write all）ですべてのバッファを保存できます
- `:b#` で直前のバッファに戻れます

</details>

<details>
<summary>解答例</summary>

```vim
" 1. ターミナルから Vim を起動して index.html を開く
" vim index.html （ターミナルで実行）

" 2. 新しいタブで style.css を開く
:tabnew css/style.css
" タブ2が作成され、style.css が表示されます

" 3. さらにタブで app.js を開く
:tabnew js/app.js
" タブ3が作成され、app.js が表示されます

" 4. 現在のタブ内で垂直分割して utils.js を開く
:vs js/utils.js
" タブ3の中で左右に app.js と utils.js が並びます
" タブの中にウィンドウ分割を含められるのがVimの柔軟な点です

" 5. 最初のタブ（index.html）に戻る
1gt
" 数字 + gt でタブ番号を指定して直接移動できます
" 1gt → タブ1、2gt → タブ2 ...

" 6. バッファ一覧を確認する
:ls
" 出力例：
"   1 %a   "index.html"       line 1
"   2      "css/style.css"    line 1
"   3      "js/app.js"        line 1
"   4      "js/utils.js"      line 1
" %a は現在アクティブなバッファを示します

" 7. README.md をバッファに追加（表示はしない）して、index.html に留まる
:badd README.md
" :badd はバッファリストにファイルを追加するだけで、表示を切り替えません
" これにより、現在の作業を中断せずにファイルを準備できます

" 8. すべてのファイルを保存する
:wall
" :wall（write all）はすべての変更済みバッファを保存します
" :wa と短縮できます
" 個別に保存する手間が省け、プロジェクト全体を一括保存できます

" 補足：実務で役立つ追加テクニック
" :qall → すべてのウィンドウ・タブを閉じてVimを終了
" :wqall → すべて保存して終了（:xa でも同じ）
" :tabdo %s/old/new/g → すべてのタブで置換を実行
" :bufdo %s/old/new/g → すべてのバッファで置換を実行
" :args *.js | argdo %s/old/new/g | w → 引数リストを使った一括操作
```

</details>
