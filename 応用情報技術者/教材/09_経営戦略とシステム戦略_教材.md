# 第9章 経営戦略とシステム戦略

> BSC、SWOT分析、PPM、BPR、SoE/SoR、EA、SOAなど、経営戦略とIT戦略の融合に関する知識を体系的に学ぶ。午前試験のストラテジ分野で頻出のテーマ。

## 9.1 経営戦略の策定手法

### 9.1.1 SWOT分析

**SWOT分析** は、組織の内部環境（強み・弱み）と外部環境（機会・脅威）を分析するフレームワークです。

```
           プラス要因        マイナス要因
         ┌──────────┬──────────┐
内部環境  │ Strengths │ Weaknesses│
         │ (強み)    │ (弱み)    │
         ├──────────┼──────────┤
外部環境  │Opportunities│ Threats  │
         │ (機会)    │ (脅威)    │
         └──────────┴──────────┘
```

**クロスSWOT分析（戦略の導出）:**

| | 機会(O) | 脅威(T) |
|---|---------|---------|
| 強み(S) | SO戦略: 強みで機会を活かす（積極戦略） | ST戦略: 強みで脅威を避ける（差別化戦略） |
| 弱み(W) | WO戦略: 弱みを補い機会を活かす（改善戦略） | WT戦略: 弱みと脅威の影響を最小化（防衛・撤退） |

### 9.1.2 PPM（Product Portfolio Management）

**PPM（プロダクトポートフォリオマネジメント）** は、BCGマトリクスとも呼ばれ、事業や製品を市場成長率と市場シェアで分類します。

```
市場成長率
  高 │ 花形(Star)      │ 問題児(Question Mark)│
     │ シェア高・成長高  │ シェア低・成長高      │
     │ → 積極投資       │ → 選択と集中          │
     ├────────────────┼────────────────────┤
  低 │ 金のなる木       │ 負け犬(Dog)          │
     │ (Cash Cow)      │ シェア低・成長低      │
     │ シェア高・成長低  │ → 撤退検討           │
     │ → 利益の源泉     │                      │
     └────────────────┴────────────────────┘
          高                  低
               市場シェア
```

### 9.1.3 BSC（Balanced Scorecard）

**BSC（バランススコアカード）** は、4つの視点から経営戦略を評価・管理するフレームワークです。

```
┌──────────────────────────────┐
│         財務の視点            │ 売上、利益率、ROI
│   「株主にどう映るか」        │
└──────────┬───────────────────┘
           │
┌──────────┴───────────────────┐
│         顧客の視点            │ 顧客満足度、市場シェア
│   「顧客にどう映るか」        │
└──────────┬───────────────────┘
           │
┌──────────┴───────────────────┐
│       業務プロセスの視点       │ 品質、コスト、納期
│   「何を改善すべきか」        │
└──────────┬───────────────────┘
           │
┌──────────┴───────────────────┐
│     学習と成長の視点          │ 社員スキル、組織能力
│  「変化・改善する能力」       │
└──────────────────────────────┘

4つの視点は因果関係で結ばれる（戦略マップ）:
学習と成長 → 業務プロセス改善 → 顧客満足度向上 → 財務成果向上
```

**BSCの構成要素:**
| 要素 | 説明 |
|------|------|
| 戦略目標 | 各視点での目標（例: 顧客満足度90%以上） |
| KPI | 目標の達成度を測る指標 |
| ターゲット | KPIの目標値 |
| 施策 | 目標達成のための具体的な取り組み |

### 9.1.4 その他の経営戦略手法

| 手法 | 説明 |
|------|------|
| ファイブフォース分析 | 業界の競争環境を5つの力で分析（ポーター） |
| バリューチェーン分析 | 価値創造の連鎖を分析（ポーター） |
| コアコンピタンス | 他社が模倣できない核となる能力 |
| ブルーオーシャン戦略 | 競争のない市場を創造する戦略 |

**ファイブフォース（5つの力）:**
```
            新規参入の脅威
                ↓
供給者の   ← 業界内の → 買い手の
交渉力       競争      交渉力
                ↑
            代替品の脅威
```

**ポイントまとめ:**
- SWOT分析のクロス分析から4つの戦略を導出できるようにする
- PPMの4象限（花形/金のなる木/問題児/負け犬）を覚える
- BSCの4つの視点とその因果関係を理解する

## 9.2 BPR（Business Process Reengineering）

### 9.2.1 BPRとは

**BPR（ビジネスプロセスリエンジニアリング）** は、業務プロセスを根本的に見直し、劇的な改善を図る手法です。

```
BPR: 業務プロセスの「抜本的な再設計」
BPM: 業務プロセスの「継続的な改善」

BPR = ゼロベースからの再構築（革新的）
BPM = 既存プロセスの段階的改善（漸進的）
```

### 9.2.2 関連する業務改善手法

| 手法 | 説明 |
|------|------|
| BPO（Business Process Outsourcing） | 業務プロセスを外部に委託 |
| RPA（Robotic Process Automation） | 定型業務をソフトウェアロボットで自動化 |
| ナレッジマネジメント | 組織内の知識を共有・活用 |
| ベンチマーキング | 他社の優れた手法を参考に改善 |

### 9.2.3 業務モデリング

| 技法 | 用途 |
|------|------|
| BPMN（Business Process Model and Notation） | 業務プロセスの標準記法 |
| E-R図 | データ構造のモデリング |
| DFD | データの流れのモデリング |
| UML アクティビティ図 | 処理の流れのモデリング |

**ポイントまとめ:**
- BPRは「抜本的再設計」、BPMは「継続的改善」
- RPAは定型業務の自動化（午前で出題増加傾向）

## 9.3 システム戦略

### 9.3.1 SoE / SoR

**SoR（System of Record）** と **SoE（System of Engagement）** は、システムの役割による分類です。

| 種類 | 目的 | 特徴 | 例 |
|------|------|------|-----|
| SoR | 記録のシステム | 正確性・安定性重視 | 基幹系（会計、人事、在庫） |
| SoE | 関わり合いのシステム | 顧客体験・スピード重視 | CRM、SNS連携、モバイルアプリ |
| SoI（System of Insight） | 洞察のシステム | 分析・予測 | BI、AI分析、データレイク |

```
┌──────────────┐
│  SoE          │ ← 顧客接点（変化が速い）
│ モバイル/Web  │
├──────────────┤
│  SoI          │ ← データ分析・洞察
│ BI/AI        │
├──────────────┤
│  SoR          │ ← 基幹業務（安定性重視）
│ ERP/会計     │
└──────────────┘
```

### 9.3.2 DX（デジタルトランスフォーメーション）

**DX** は、デジタル技術を活用してビジネスモデルや業務プロセスを根本的に変革することです。

| 段階 | 内容 |
|------|------|
| デジタイゼーション | アナログ情報のデジタル化（紙→電子） |
| デジタライゼーション | デジタル技術で個別の業務を改善 |
| デジタルトランスフォーメーション | ビジネスモデル自体を変革 |

### 9.3.3 情報システム戦略の策定

```
経営戦略
  │
  ▼
IT戦略（情報システム戦略）
  │
  ├── 全体最適化計画
  ├── 個別システム化計画
  └── IT投資計画
  │
  ▼
個別システムの企画・開発
```

**ポイントまとめ:**
- SoR（記録）とSoE（顧客接点）の違いを区別する
- DXの3段階（デジタイゼーション→デジタライゼーション→DX）を理解する

## 9.4 EA（Enterprise Architecture）

### 9.4.1 EAとは

**EA（エンタープライズアーキテクチャ）** は、組織全体のIT構造を4つのアーキテクチャで体系的に整理するフレームワークです。

### 9.4.2 EAの4階層

```
┌──────────────────────────────┐
│ ビジネスアーキテクチャ（BA）   │ 業務の構造（ビジネスプロセス）
├──────────────────────────────┤
│ データアーキテクチャ（DA）     │ データの構造（情報資産）
├──────────────────────────────┤
│ アプリケーションアーキテクチャ（AA）│ アプリの構造（システム構成）
├──────────────────────────────┤
│ テクノロジアーキテクチャ（TA） │ 技術基盤（インフラ）
└──────────────────────────────┘
```

| 階層 | 対象 | 成果物の例 |
|------|------|-----------|
| BA | 業務プロセス | 業務フロー図、機能構成図 |
| DA | 情報・データ | ER図、データ辞書 |
| AA | アプリケーション | システム構成図、機能一覧 |
| TA | 技術インフラ | ネットワーク図、ハード構成図 |

### 9.4.3 EAの活用

EAは「現状（As-Is）」と「あるべき姿（To-Be）」をそれぞれ4階層で整理し、ギャップを分析して移行計画を策定します。

```
As-Is（現状）        To-Be（あるべき姿）
  BA ──────────→ BA
  DA ──────────→ DA    ← ギャップ分析と移行計画
  AA ──────────→ AA
  TA ──────────→ TA
```

**ポイントまとめ:**
- EAの4階層（BA/DA/AA/TA）を覚える
- As-Is / To-Beのギャップ分析の考え方を理解する
- 午前試験では各階層の成果物を問われることが多い

## 9.5 SOA（Service-Oriented Architecture）

### 9.5.1 SOAとは

**SOA（サービス指向アーキテクチャ）** は、業務機能を「サービス」として部品化し、それらを組み合わせてシステムを構築するアーキテクチャです。

```
┌───────────────────────────────────┐
│          ビジネスプロセス            │
│  注文処理 = 在庫確認 + 与信確認     │
│            + 出荷指示 + 請求処理    │
└──────────┬────────────────────────┘
           │ サービスの呼び出し
           │
    ┌──────┼──────┬──────┐
┌───┴──┐┌──┴──┐┌──┴──┐┌──┴──┐
│在庫   ││与信  ││出荷  ││請求  │  ← 各サービスは独立
│サービス││サービス││サービス││サービス│
└──────┘└─────┘└─────┘└─────┘
    │       │       │       │
    DB1     DB2     DB3     DB4
```

### 9.5.2 SOAの実現技術

| 技術 | 説明 |
|------|------|
| Webサービス | SOAP/WSDL/UDDIによるサービス連携 |
| REST API | HTTP/JSONベースの軽量なサービス連携 |
| ESB（Enterprise Service Bus） | サービス間の仲介・変換・ルーティング |
| マイクロサービス | SOAをさらに細粒度にしたアーキテクチャ |

### 9.5.3 API エコノミー

APIを公開して外部サービスと連携し、新たな価値を創出するビジネスモデルです。

| 種類 | 説明 |
|------|------|
| オープンAPI | 誰でも利用可能（天気、地図等） |
| パートナAPI | 契約パートナーのみ利用可能 |
| プライベートAPI | 組織内部のみ利用可能 |

**ポイントまとめ:**
- SOAは「サービスの部品化と組合せ」がキーワード
- ESBはサービス間の仲介役
- API連携は現在のシステム統合の主流

## 9.6 IT投資の評価

### 9.6.1 投資評価手法

| 手法 | 説明 |
|------|------|
| ROI（投資利益率） | 利益 / 投資額 × 100(%) |
| NPV（正味現在価値） | 将来キャッシュフローを現在価値に割引いた合計 |
| IRR（内部収益率） | NPV=0となる割引率 |
| 回収期間法 | 投資額を回収するまでの期間 |
| TCO（総所有コスト） | 取得から廃棄までの全コスト |

**NPVの計算:**
```
NPV = Σ{CFt / (1+r)^t} - 初期投資

  CFt: t年目のキャッシュフロー
  r: 割引率
  t: 年数

例: 初期投資1,000万円、割引率5%、3年間のCF: 400万円/年

NPV = 400/(1.05) + 400/(1.05)² + 400/(1.05)³ - 1,000
    = 381 + 363 + 345 - 1,000
    = 89万円 > 0 → 投資すべき
```

**ポイントまとめ:**
- NPV > 0 なら投資すべき、NPV < 0 なら見送り
- TCOは「取得コスト＋運用コスト＋廃棄コスト」の合計
