> **難易度**: ★☆☆☆☆（入門）
> **前提知識**: 第1章（環境構築）、第2章（基本操作とプロンプト入力）

## 完成イメージ

以下のシナリオをClaude Codeで実行し、すべてのプロンプトパターンを体験します。

```
===== シナリオ1: 基本的な質問 =====
> Pythonでリストの重複を削除する方法を3つ教えてください

===== シナリオ2: コード生成 =====
> FizzBuzzプログラムをPythonで書いてください。1から100まで出力してください。

===== シナリオ3: ワンショットモード =====
$ claude -p "現在のディレクトリにある全ファイルの行数を集計して"

===== シナリオ4: パイプ連携 =====
$ cat app.py | claude -p "このコードのバグを見つけて"

===== シナリオ5: スラッシュコマンド活用 =====
> /compact
> /model
> /cost
```

---

## 課題の要件

1. テスト用プロジェクトにサンプルファイルを3つ以上配置する
2. 以下の5種類のプロンプトパターンを実際に試す
   - **質問型**: 知識やベストプラクティスを尋ねる
   - **生成型**: コードや設定ファイルの生成を依頼する
   - **分析型**: 既存コードの分析・説明を依頼する
   - **修正型**: 問題のあるコードの修正を依頼する
   - **リファクタリング型**: 動作するコードの改善を依頼する
3. ワンショットモード（`-p`フラグ）を3回以上使用する
4. パイプ（`|`）やリダイレクト（`<`、`>`）を使った入出力を試す
5. `/compact`、`/cost`、`/model`、`/clear`を実行する

## 解答例

<details>
<summary>解答例（初心者向け）── 全操作の実行例</summary>

```bash
# === 準備 ===
mkdir -p ~/claude-code-practice/task02
cd ~/claude-code-practice/task02

# サンプルファイル作成（上記ステップ1参照）

# === 対話モード ===
claude

# 以下、対話モード内で実行
# > Pythonでゼロ除算を安全に処理する方法を教えてください
# → try-except、事前チェック、条件演算子などの方法が説明される

# > calculator.pyを読んで、潜在的な問題点を指摘してください
# → divide関数のゼロ除算問題が指摘される

# > calculator.pyのdivide関数にゼロ除算のエラーハンドリングを追加してください
# → ファイルが編集される（確認プロンプトが表示される）

# → トークン使用量とコストが表示される
```

</details>

<details>
<summary>解答例（改良版）── 効果的なプロンプトのコツ</summary>

同じ目的でも、プロンプトの書き方次第で回答の質が変わります。

```
# ===== 悪い例 =====
> コードを直して

# → 何をどう直すか曖昧で、意図しない修正が行われる可能性がある

# ===== 良い例 =====
> calculator.pyのdivide関数について：
> 1. ゼロ除算が発生した場合にZeroDivisionErrorではなく
>    Noneを返すようにしてください
> 2. 引数の型チェックを追加し、数値以外が渡されたら
>    TypeErrorを発生させてください
> 3. docstringを追加してください

# → 具体的な指示により、期待通りの修正が行われる
```

**効果的なプロンプトの3原則:**

| 原則 | 説明 | 例 |
|---|---|---|
| **具体性** | 何をどうしたいか明確にする | 「バグを直して」→「divide関数のゼロ除算を処理して」 |
| **コンテキスト** | 背景情報を添える | 「Python 3.10以上を対象としています」 |
| **期待値** | 望む出力形式を示す | 「修正前後のdiffを見せてください」 |

**初心者向けとの違い:**
- 初心者向けは「とにかく動かしてみる」ことに重点を置いています
- 改良版では「プロンプトの質を上げてAIの応答精度を高める」ことに焦点を当てています
- 実務では改良版のような具体的なプロンプトを書くことで、手戻りを減らせます

</details>
