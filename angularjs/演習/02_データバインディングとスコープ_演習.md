## 演習2-1（基本）: 双方向データバインディングの体験

### 問題

AngularJSの双方向データバインディング（Two-way Data Binding）を体験するアプリを作成してください。

#### 要件

- テキスト入力フィールドに名前を入力すると、3か所に同時に反映される
  - `<h2>` タグ内の挨拶文
  - `<p>` タグ内の文字数表示
  - 別の `<input>` フィールド（読み取り専用ではなく、こちらからも編集可能）
- 2つの入力フィールドは同じ `ng-model` を共有し、どちらを編集しても同期される

### 期待される出力（「AngularJS」と入力した場合）

```
双方向データバインディングのデモ

入力1: [AngularJS]
入力2: [AngularJS]

こんにちは、AngularJSさん！
入力文字数: 9文字
```

<details>
<summary>ヒント</summary>

- 2つの `<input>` 要素に同じ `ng-model` を指定すると、双方向にバインドされます。
- 文字列の長さは `.length` プロパティで取得できます。

</details>

<details>
<summary>解答例</summary>

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <title>演習2-1: 双方向データバインディング</title>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
  <style>
    body { font-family: sans-serif; margin: 20px; }
    input { padding: 8px; width: 250px; margin: 5px 0; border: 1px solid #ccc; border-radius: 4px; }
    .highlight { color: #e74c3c; font-weight: bold; }
  </style>
</head>
<body ng-controller="BindingController">

  <h1>双方向データバインディングのデモ</h1>

  <!-- 2つのinputが同じng-modelを共有する -->
  <p>入力1: <input type="text" ng-model="userName"></p>
  <p>入力2: <input type="text" ng-model="userName"></p>

  <!-- ng-modelの値がリアルタイムに反映される -->
  <h2>こんにちは、<span class="highlight">{{ userName }}</span>さん！</h2>
  <p>入力文字数: {{ userName.length }}文字</p>

  <script>
    var app = angular.module('myApp', []);

    app.controller('BindingController', function($scope) {
      // 初期値を設定する
      $scope.userName = 'AngularJS';
    });
  </script>
</body>
</html>
```

**解説:**

- 2つの `<input>` に同じ `ng-model="userName"` を指定することで、どちらの入力フィールドを編集しても、もう一方にも即座に反映されます。
- これがAngularJSの**双方向データバインディング**です。Model（$scope.userName）が変更されるとView（HTML）が更新され、View（input）が変更されるとModelが更新されます。
- `{{ userName.length }}` は式の中でJavaScriptのプロパティにアクセスしています。

</details>

## 演習2-2（基本）: スコープの継承

### 問題

親コントローラーと子コントローラーを使って、スコープの継承がどのように動作するかを確認するアプリを作成してください。

#### 要件

- `ParentController`: `familyName`（姓）と `message` を `$scope` に設定する
- `ChildController`: `firstName`（名）を `$scope` に設定する（`familyName` は設定しない）
- 子コントローラーの範囲内で、親のスコープから `familyName` が継承されることを確認する
- 親と子それぞれの範囲で利用可能なプロパティを表示する

### 期待される出力

```
--- 親コントローラーの範囲 ---
姓: 田中
メッセージ: 家族のページです

--- 子コントローラーの範囲 ---
姓: 田中（親から継承）
名: 太郎
フルネーム: 田中 太郎
メッセージ: 家族のページです（親から継承）
```

<details>
<summary>ヒント</summary>

- `ng-controller` をネスト（入れ子）にすることで、子コントローラーは親コントローラーのスコープを継承します。
- 子スコープでは、親スコープのプロパティを直接参照できます。
- JavaScriptのプロトタイプチェーンと同じ仕組みで動作します。

</details>

<details>
<summary>解答例</summary>

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <title>演習2-2: スコープの継承</title>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
  <style>
    body { font-family: sans-serif; margin: 20px; }
    .parent-scope {
      border: 2px solid #3498db; padding: 15px; border-radius: 8px; margin-bottom: 10px;
    }
    .child-scope {
      border: 2px solid #e74c3c; padding: 15px; border-radius: 8px;
      margin-top: 10px; background: #fff5f5;
    }
    .label { font-weight: bold; }
    .inherited { color: #888; font-style: italic; }
  </style>
</head>
<body>

  <!-- 親コントローラーの範囲 -->
  <div class="parent-scope" ng-controller="ParentController">
    <h2>親コントローラーの範囲</h2>
    <p><span class="label">姓:</span> {{ familyName }}</p>
    <p><span class="label">メッセージ:</span> {{ message }}</p>

    <!-- 子コントローラーの範囲（親の中にネスト） -->
    <div class="child-scope" ng-controller="ChildController">
      <h2>子コントローラーの範囲</h2>
      <!-- familyNameは親スコープから継承される -->
      <p><span class="label">姓:</span> {{ familyName }}
        <span class="inherited">（親から継承）</span></p>
      <p><span class="label">名:</span> {{ firstName }}</p>
      <p><span class="label">フルネーム:</span> {{ familyName }} {{ firstName }}</p>
      <p><span class="label">メッセージ:</span> {{ message }}
        <span class="inherited">（親から継承）</span></p>
    </div>
  </div>

  <script>
    var app = angular.module('myApp', []);

    // 親コントローラー: familyNameとmessageを定義する
    app.controller('ParentController', function($scope) {
      $scope.familyName = '田中';
      $scope.message = '家族のページです';
    });

    // 子コントローラー: firstNameのみを定義する
    // familyNameとmessageは親スコープから自動的に継承される
    app.controller('ChildController', function($scope) {
      $scope.firstName = '太郎';
      // familyName や message は定義していないが、親から継承されるため参照できる
    });
  </script>
</body>
</html>
```

**解説:**

- AngularJSのスコープはJavaScriptのプロトタイプ継承を利用しています。子スコープは親スコープのプロパティを参照できます。
- `ChildController` で `familyName` を定義していなくても、親の `ParentController` で定義された値が参照されます。
- この仕組みは便利ですが、次の演習で学ぶ「プリミティブ値の問題」に注意が必要です。

</details>

## 演習2-3（応用）: ドット記法の問題を修正する

### 問題

以下のコードには「プリミティブ値のスコープ継承問題」があります。問題を特定し、ドット記法（Dot Notation）を使って修正してください。

#### バグのあるコード

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <title>バグのあるコード</title>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
</head>
<body>
  <div ng-controller="ParentController">
    <h2>親: {{ name }}</h2>
    <input type="text" ng-model="name">

    <div ng-controller="ChildController">
      <h2>子: {{ name }}</h2>
      <input type="text" ng-model="name">
    </div>
  </div>

  <script>
    var app = angular.module('myApp', []);
    app.controller('ParentController', function($scope) {
      $scope.name = '初期値';
    });
    app.controller('ChildController', function($scope) {
    });
  </script>
</body>
</html>
```

#### 問題の挙動

1. 最初は親・子ともに「初期値」と表示される
2. **親の入力欄**を変更すると、子の表示も更新される（正常）
3. **子の入力欄**を一度でも変更すると、以降は親を変更しても子に反映されなくなる（バグ）

#### 修正要件

- ドット記法を使って、親の入力を変更したら子にも常に反映されるように修正する
- 子の入力欄を変更した場合も、親に反映されるようにする

### 期待される出力（修正後）

```
親を変更 → 子も更新される
子を変更 → 親も更新される
（常に同期が保たれる）
```

<details>
<summary>ヒント</summary>

- プリミティブ値（文字列、数値、真偽値）を `ng-model` にバインドすると、子スコープで値を変更した瞬間に、子スコープ自身にプロパティが作成されてしまい、親スコープとの接続が切れます。
- オブジェクトのプロパティ（例：`data.name`）をバインドすると、プロトタイプチェーンを通じて常に親のオブジェクトを参照するため、同期が保たれます。
- これが「常にドット（`.`）を含む `ng-model` を使う」というAngularJSのベストプラクティスです。

</details>

<details>
<summary>解答例</summary>

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <title>演習2-3: ドット記法の修正</title>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
  <style>
    body { font-family: sans-serif; margin: 20px; }
    .parent { border: 2px solid #3498db; padding: 15px; border-radius: 8px; }
    .child { border: 2px solid #e74c3c; padding: 15px; border-radius: 8px; margin-top: 10px; }
    input { padding: 8px; width: 200px; border: 1px solid #ccc; border-radius: 4px; }
    .note { background: #ffffcc; padding: 10px; border-radius: 5px; margin-top: 15px; font-size: 14px; }
  </style>
</head>
<body>

  <h1>ドット記法の修正</h1>

  <div class="parent" ng-controller="ParentController">
    <h2>親: {{ data.name }}</h2>
    <!-- ドット記法: data.name をバインドする -->
    <input type="text" ng-model="data.name">

    <div class="child" ng-controller="ChildController">
      <h2>子: {{ data.name }}</h2>
      <!-- 子でも同じ data.name をバインドする -->
      <input type="text" ng-model="data.name">
    </div>
  </div>

  <div class="note">
    <strong>ポイント:</strong>
    <code>ng-model="name"</code>（プリミティブ）ではなく
    <code>ng-model="data.name"</code>（オブジェクトのプロパティ）を使うことで、
    親子間の同期が常に保たれます。
  </div>

  <script>
    var app = angular.module('myApp', []);

    app.controller('ParentController', function($scope) {
      // プリミティブ値ではなくオブジェクトのプロパティとして定義する
      // これにより、子スコープはプロトタイプチェーンで親のdataオブジェクトを参照する
      $scope.data = {
        name: '初期値'
      };
    });

    app.controller('ChildController', function($scope) {
      // 何も定義しない
      // data.name は親スコープの data オブジェクトを通じて参照される
    });
  </script>
</body>
</html>
```

**解説:**

- **問題の原因**: プリミティブ値（`$scope.name = '初期値'`）の場合、子スコープで `ng-model="name"` に書き込むと、子スコープ自身に `name` プロパティが新規作成されます（JavaScriptのプロトタイプ継承の仕様）。以降、子は自分自身の `name` を参照するため、親との接続が切れます。
- **修正方法**: オブジェクト（`$scope.data = { name: '初期値' }`）を使い、`ng-model="data.name"` とします。子スコープは親スコープの `data` オブジェクトへの参照を持ち、`data.name` への書き込みはそのオブジェクトのプロパティを変更するため、親子間の同期が維持されます。
- **ベストプラクティス**: 「`ng-model` には必ずドットを含めよ」というのはAngularJSコミュニティの有名な格言です。

</details>

## 演習2-4（応用）: Controller As 構文へのリファクタリング

### 問題

以下の `$scope` を使ったコードを、**Controller As** 構文に書き換えてください。

#### 元のコード（$scopeを使用）

```html
<div ng-controller="TodoController">
  <h2>{{ title }}</h2>
  <input type="text" ng-model="newItem">
  <button ng-click="addItem()">追加</button>
  <ul>
    <li ng-repeat="item in items">{{ item }}</li>
  </ul>
  <p>アイテム数: {{ items.length }}</p>
</div>

<script>
  app.controller('TodoController', function($scope) {
    $scope.title = 'ToDoリスト';
    $scope.items = ['牛乳を買う', '部屋を掃除する'];
    $scope.newItem = '';
    $scope.addItem = function() {
      if ($scope.newItem) {
        $scope.items.push($scope.newItem);
        $scope.newItem = '';
      }
    };
  });
</script>
```

#### 修正要件

- `$scope` を使わず、`this` を使う Controller As 構文に書き換える
- HTML側も `as` 構文に対応させる
- 機能は元のコードと完全に同じにする

### 期待される出力

```
ToDoリスト
入力: [洗濯をする]  [追加]
- 牛乳を買う
- 部屋を掃除する
- 洗濯をする
アイテム数: 3
```

<details>
<summary>ヒント</summary>

- `ng-controller="TodoController as vm"` のように `as エイリアス` を付加します。
- コントローラー内で `$scope.プロパティ` を `this.プロパティ` に置き換えます。
- HTML側では `{{ プロパティ }}` を `{{ vm.プロパティ }}` に変更します。
- コントローラー内の関数で `this` を安全に参照するため、`var vm = this;` とエイリアスを作成します。

</details>

<details>
<summary>解答例</summary>

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <title>演習2-4: Controller As 構文</title>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
  <style>
    body { font-family: sans-serif; margin: 20px; }
    input { padding: 8px; width: 200px; border: 1px solid #ccc; border-radius: 4px; }
    button { padding: 8px 16px; background: #3498db; color: white; border: none;
      border-radius: 4px; cursor: pointer; }
    button:hover { background: #2980b9; }
    li { padding: 4px 0; }
  </style>
</head>
<body>

  <!-- Controller As 構文: "as vm" でエイリアスを指定する -->
  <div ng-controller="TodoController as vm">
    <!-- vm.プロパティ でアクセスする -->
    <h2>{{ vm.title }}</h2>

    <input type="text" ng-model="vm.newItem" placeholder="新しいアイテム">
    <button ng-click="vm.addItem()">追加</button>

    <ul>
      <!-- ng-repeat内でも vm.items を参照する -->
      <li ng-repeat="item in vm.items">{{ item }}</li>
    </ul>

    <p>アイテム数: {{ vm.items.length }}</p>
  </div>

  <script>
    var app = angular.module('myApp', []);

    // Controller As 構文: $scope を注入せず、this を使う
    app.controller('TodoController', function() {
      // thisのエイリアスを作成する（コールバック関数内でもthisを安全に参照するため）
      var vm = this;

      vm.title = 'ToDoリスト';
      vm.items = ['牛乳を買う', '部屋を掃除する'];
      vm.newItem = '';

      vm.addItem = function() {
        if (vm.newItem) {
          vm.items.push(vm.newItem);
          vm.newItem = '';  // 入力フィールドをクリアする
        }
      };
    });
  </script>
</body>
</html>
```

**解説:**

- **Controller As 構文のメリット:**
  1. `$scope` への依存がなくなり、コントローラーが純粋なJavaScriptのコンストラクタ関数になります。
  2. ネストしたコントローラーで、どのコントローラーのプロパティかが明確になります（`vm.name` vs `parent.name`）。
  3. ドット記法が自然に使われるため、プリミティブ値の問題が起きにくくなります。
- `var vm = this;` はコントローラー内で慣例的に使用されるパターンです。`vm` は ViewModel の略です。
- Controller As 構文はAngularJS 1.2以降で使用可能で、Angular（2以降）への移行にも有利です。

</details>

## 演習2-5（チャレンジ）: リアルタイム文字カウンターと$watch

### 問題

`$watch` を使って、リアルタイムに入力を監視し、さまざまな統計情報を表示するテキスト分析アプリを作成してください。

#### 要件

1. テキストエリアにテキストを入力できる
2. 以下の統計情報をリアルタイムに表示する:
   - 文字数（空白含む / 空白除く）
   - 単語数（スペースで区切られた単語）
   - 行数
   - 最も長い単語
3. `$watch` を使って入力テキストを監視し、変更時に統計情報を再計算する
4. `$watchCollection` を使って、入力履歴（最新5件）を管理する
5. テキストが100文字を超えたら警告メッセージを表示する

### 期待される出力

```
テキスト分析ツール

テキスト入力:
[AngularJSは Google が開発した
JavaScriptフレームワークです。
双方向データバインディングが特徴です。]

--- 統計情報 ---
文字数（空白含む）: 55
文字数（空白除く）: 48
単語数: 5
行数: 3
最も長い単語: JavaScriptフレームワークです。

--- 入力履歴（最新5件） ---
1. "AngularJSは"
2. "AngularJSは Google"
3. "AngularJSは Google が"
...

ステータス: 正常（残り45文字）
```

<details>
<summary>ヒント</summary>

- `$scope.$watch('変数名', function(newVal, oldVal) { ... })` で変数の変更を監視できます。
- `$scope.$watchCollection('配列名', function(newVal, oldVal) { ... })` で配列や浅いオブジェクトの変更を監視できます。
- 文字列の分割には `split()` を使います。
- 行数は改行文字 `\n` で分割した配列の長さで計算できます。
- 入力履歴は配列に `push` して、5件を超えたら `shift()` で先頭を削除します。

</details>

<details>
<summary>解答例</summary>

```html
<!DOCTYPE html>
<html ng-app="myApp">
<head>
  <meta charset="UTF-8">
  <title>演習2-5: リアルタイム文字カウンター</title>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.8.3/angular.min.js"></script>
  <style>
    body { font-family: sans-serif; margin: 20px; }
    textarea { width: 100%; max-width: 500px; padding: 10px; border: 1px solid #ccc;
      border-radius: 4px; font-size: 14px; resize: vertical; }
    .stats { background: #f8f9fa; padding: 15px; border-radius: 8px; margin-top: 15px;
      max-width: 500px; }
    .stats p { margin: 5px 0; }
    .warning { color: #e74c3c; font-weight: bold; }
    .ok { color: #27ae60; font-weight: bold; }
    .history { background: #fff3cd; padding: 15px; border-radius: 8px; margin-top: 15px;
      max-width: 500px; }
    .history li { font-size: 13px; color: #555; margin: 3px 0; word-break: break-all; }
    .longest-word { background: #e8f4fd; padding: 3px 8px; border-radius: 3px;
      font-weight: bold; }
  </style>
</head>
<body ng-controller="TextAnalyzerController">

  <h1>テキスト分析ツール</h1>

  <div>
    <label><strong>テキスト入力:</strong></label><br>
    <textarea ng-model="inputText" rows="5"
              placeholder="ここにテキストを入力してください..."></textarea>
  </div>

  <!-- 統計情報 -->
  <div class="stats">
    <h3>統計情報</h3>
    <p>文字数（空白含む）: <strong>{{ stats.charCountWithSpaces }}</strong></p>
    <p>文字数（空白除く）: <strong>{{ stats.charCountWithoutSpaces }}</strong></p>
    <p>単語数: <strong>{{ stats.wordCount }}</strong></p>
    <p>行数: <strong>{{ stats.lineCount }}</strong></p>
    <p>最も長い単語: <span class="longest-word">{{ stats.longestWord || 'なし' }}</span></p>
  </div>

  <!-- ステータス表示 -->
  <div style="margin-top: 10px; max-width: 500px;">
    <p ng-if="stats.charCountWithSpaces <= 100" class="ok">
      ステータス: 正常（残り{{ 100 - stats.charCountWithSpaces }}文字）
    </p>
    <p ng-if="stats.charCountWithSpaces > 100" class="warning">
      警告: 100文字を超えています！（{{ stats.charCountWithSpaces - 100 }}文字オーバー）
    </p>
  </div>

  <!-- 入力履歴 -->
  <div class="history" ng-if="history.length > 0">
    <h3>入力履歴（最新5件）</h3>
    <ol>
      <li ng-repeat="entry in history track by $index">
        "{{ entry }}"
      </li>

  <script>
    var app = angular.module('myApp', []);

    app.controller('TextAnalyzerController', function($scope) {
      // 初期値の設定
      $scope.inputText = '';
      $scope.history = [];

      // 統計情報を保持するオブジェクト
      $scope.stats = {
        charCountWithSpaces: 0,
        charCountWithoutSpaces: 0,
        wordCount: 0,
        lineCount: 0,
        longestWord: ''
      };

      // テキストを分析する関数
      function analyzeText(text) {
        if (!text) {
          // テキストが空の場合はすべて0にリセットする
          $scope.stats = {
            charCountWithSpaces: 0,
            charCountWithoutSpaces: 0,
            wordCount: 0,
            lineCount: 0,
            longestWord: ''
          };
          return;
        }

        // 文字数（空白含む）
        $scope.stats.charCountWithSpaces = text.length;

        // 文字数（空白除く）: 半角・全角スペース、改行を除去する
        $scope.stats.charCountWithoutSpaces = text.replace(/[\s\u3000]/g, '').length;

        // 単語数: 空白文字で分割し、空文字を除外する
        var words = text.trim().split(/[\s\u3000]+/).filter(function(w) {
          return w.length > 0;
        });
        $scope.stats.wordCount = words.length;

        // 行数: 改行で分割する
        $scope.stats.lineCount = text.split('\n').length;

        // 最も長い単語を探す
        $scope.stats.longestWord = words.reduce(function(longest, current) {
          return current.length > longest.length ? current : longest;
        }, '');
      }

      // $watch: inputTextの変更を監視する
      // 第1引数: 監視する式（文字列）
      // 第2引数: 変更時に呼ばれるコールバック関数
      $scope.$watch('inputText', function(newValue, oldValue) {
        // newValueとoldValueが異なる場合のみ処理する
        if (newValue !== oldValue) {
          // テキスト分析を実行する
          analyzeText(newValue);

          // 入力履歴に追加する（空でない場合のみ）
          if (newValue && newValue.trim().length > 0) {
            // 20文字を超える場合は省略して表示する
            var entry = newValue.length > 30
              ? newValue.substring(0, 30) + '...'
              : newValue;
            $scope.history.push(entry);

            // 最新5件のみ保持する（古いものを削除）
            if ($scope.history.length > 5) {
              $scope.history.shift();
            }
          }
        }
      });

      // $watchCollection: history配列の変更を監視する
      // 配列の要素が追加・削除された場合にコールバックが呼ばれる
      $scope.$watchCollection('history', function(newCollection, oldCollection) {
        if (newCollection !== oldCollection) {
          console.log('履歴が更新されました。現在の件数:', newCollection.length);
        }
      });
    });
  </script>
</body>
</html>
```

**解説:**

- **`$watch`**: スコープ上の式の変更を監視します。AngularJSのダイジェストサイクルごとに新旧の値を比較し、変更があればコールバックを実行します。
  - 第1引数に監視する式（文字列または関数）を指定します。
  - 第2引数に変更時のコールバック関数を指定します（`newValue`, `oldValue` を受け取ります）。
  - 初回のダイジェストサイクルでも呼ばれるため、`newValue !== oldValue` のチェックが重要です。
- **`$watchCollection`**: 配列やオブジェクトの浅い（shallow）変更を監視します。要素の追加・削除・入れ替えを検出しますが、要素内のプロパティ変更は検出しません。
- **パフォーマンスの注意**: `$watch` はダイジェストサイクルごとに評価されるため、重い処理を行うとパフォーマンスに影響します。第3引数に `true` を渡すと深い比較（deep watch）になりますが、さらにコストが高くなります。
