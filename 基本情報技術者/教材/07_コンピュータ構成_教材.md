# 第7章：コンピュータ構成

## この章のゴール
- コンピュータの5大装置と各装置の役割を説明できる
- CPUの命令実行サイクル（フェッチ→デコード→実行→格納）を理解する
- メモリの階層構造とキャッシュメモリの仕組みを理解する
- アドレッシングモード（指標、基底、間接など）を使い分けられる
- バス、入出力制御（DMA、割込み）の仕組みを説明できる
- 性能指標（MIPS、クロック周波数等）を計算できる

## 試験での出題傾向
- 午前試験：キャッシュメモリのヒット率計算、MIPS値の計算、アドレッシングモードの判定、パイプラインの計算が定番です。
- 午後試験：アドレス計算やCPUの動作理解が前提となる問題が出題されます。

## 7.1 コンピュータの5大装置

### ノイマン型コンピュータ

現在のほとんどのコンピュータは**ノイマン型（プログラム内蔵方式）**です。プログラムとデータをメモリに格納し、CPUが命令を順次読み出して実行します。

### 5大装置

```
             ┌──────────────────────┐
  入力装置 ──→│  制御装置   演算装置    │──→ 出力装置
             │    （CPU）           │
             └──────┬───────────────┘
                    ↕
               記憶装置
           （主記憶・補助記憶）
```

| 装置 | 役割 | 例 |
|------|------|-----|
| 制御装置（Control Unit） | 命令の解読と各装置の制御 | CPU内部 |
| 演算装置（ALU） | 四則演算、論理演算を行う | CPU内部 |
| 記憶装置（Memory） | データとプログラムの格納 | RAM, HDD, SSD |
| 入力装置（Input） | 外部からデータを取り込む | キーボード, マウス |
| 出力装置（Output） | 処理結果を外部に出す | ディスプレイ, プリンタ |

## 7.2 CPU（Central Processing Unit / 中央処理装置）

### CPUの構成要素

| 構成要素 | 役割 |
|---------|------|
| 制御装置 | 命令の解読、各装置への信号送信 |
| ALU（Arithmetic Logic Unit） | 算術演算・論理演算の実行 |
| レジスタ | CPU内部の高速な記憶素子 |

### 主要なレジスタ

| レジスタ | 英語名 | 役割 |
|---------|--------|------|
| プログラムカウンタ（PC） | Program Counter | 次に実行する命令のアドレスを保持 |
| 命令レジスタ（IR） | Instruction Register | 実行中の命令を保持 |
| アキュムレータ（ACC） | Accumulator | 演算結果を保持 |
| 汎用レジスタ（GR） | General Register | 様々な用途に使用 |
| インデックスレジスタ（IX） | Index Register | 指標アドレッシングで使用 |
| ベースレジスタ（BR） | Base Register | 基底アドレッシングで使用 |
| スタックポインタ（SP） | Stack Pointer | スタック領域の先頭アドレスを保持 |
| フラグレジスタ（FR） | Flag Register | 演算結果の状態（ゼロ、負、桁あふれ等） |

> **試験頻出ポイント：** プログラムカウンタは「次の命令のアドレス」、命令レジスタは「今の命令そのもの」を保持します。この違いを明確に区別しましょう。

### 命令実行サイクル

CPUが1つの命令を実行する過程は、以下の4段階です。

```
1. 命令フェッチ（Fetch）    : PCが指すアドレスの命令を主記憶から取得→IRに格納
2. 命令デコード（Decode）    : IRの命令を解読し、実行内容を決定
3. オペランドフェッチ（Execute）: 必要なデータ（オペランド）を取得
4. 実行・格納（Store）       : 演算を実行し、結果を格納。PCを更新
```

**具体例：「ADD 100」（アドレス100のデータを加算する命令）の実行**

```
1. フェッチ: PC=200 → 主記憶のアドレス200から「ADD 100」を取得 → IRに格納
2. デコード: 「ADD」を解読 → 「加算命令」と判断
3. オペランドフェッチ: アドレス100のデータ（例：50）を取得
4. 実行・格納: ACC = ACC + 50。PC = 201 に更新
```

## 7.3 アドレッシングモード（アドレス指定方式）

命令中のアドレス部から、実際のメモリアドレス（実効アドレス）を求める方法です。

### 各アドレッシングモード

| 方式 | 実効アドレスの求め方 | 特徴 |
|------|-------------------|------|
| 即値アドレッシング | 命令中のオペランドがそのままデータ | メモリアクセス不要 |
| 直接アドレッシング | 命令中のアドレスがそのまま実効アドレス | 最も基本的 |
| 間接アドレッシング | 命令中のアドレスが指す場所に実効アドレスが入っている | 2回メモリ参照 |
| 指標アドレッシング | 命令中のアドレス + インデックスレジスタの値 | 配列アクセスに使用 |
| 基底アドレッシング | 命令中のアドレス + ベースレジスタの値 | プログラムの再配置に使用 |
| 相対アドレッシング | 命令中のアドレス + プログラムカウンタの値 | 分岐命令に使用 |

### 計算例

**指標アドレッシング：**
```
命令のアドレス部 = 200
インデックスレジスタ = 10
実効アドレス = 200 + 10 = 210
```

**間接アドレッシング：**
```
命令のアドレス部 = 300
主記憶のアドレス300の内容 = 500
実効アドレス = 500（アドレス500のデータを使用）
```

**基底アドレッシング：**
```
命令のアドレス部 = 100
ベースレジスタ = 5000
実効アドレス = 100 + 5000 = 5100
```

> **よくある間違い：** 間接アドレッシングでは「2回」メモリにアクセスする必要があります。1回目で実効アドレスを取得し、2回目で実データを取得します。

## 7.4 メモリの階層構造

### メモリ階層

上位ほど高速・高価・小容量、下位ほど低速・安価・大容量です。

```
        レジスタ        ← 最速（1クロック以内）
          ↕
      キャッシュメモリ   ← 高速（数クロック）
          ↕
        主記憶（RAM）    ← 中速（数十〜数百クロック）
          ↕
      補助記憶（SSD/HDD）← 低速（数千〜数百万クロック）
```

| メモリ | 速度 | 容量 | 揮発性 |
|--------|------|------|--------|
| レジスタ | 最速 | 数十〜数百バイト | 揮発性 |
| キャッシュメモリ（SRAM） | 高速 | 数KB〜数十MB | 揮発性 |
| 主記憶（DRAM） | 中速 | 数GB〜数十GB | 揮発性 |
| SSD（フラッシュメモリ） | やや低速 | 数百GB〜数TB | 不揮発性 |
| HDD | 低速 | 数TB〜 | 不揮発性 |

### SRAM と DRAM の比較

| 特性 | SRAM | DRAM |
|------|------|------|
| 速度 | 高速 | SRAMより低速 |
| 容量あたりの価格 | 高い | 安い |
| リフレッシュ | 不要 | 必要 |
| 構造 | フリップフロップ | コンデンサ |
| 用途 | キャッシュメモリ | 主記憶 |

### キャッシュメモリ

主記憶とCPUの間に配置され、主記憶の一部のコピーを保持することで、CPUの待ち時間を削減します。

**実効アクセス時間の計算：**

```
実効アクセス時間 = ヒット率 × キャッシュアクセス時間 + (1 - ヒット率) × 主記憶アクセス時間
```

**計算例：**
```
キャッシュアクセス時間 = 10ns
主記憶アクセス時間 = 100ns
ヒット率 = 0.9（90%）

実効アクセス時間 = 0.9 × 10 + 0.1 × 100 = 9 + 10 = 19ns
```

> **計算のコツ：** ヒット率が高いほど実効アクセス時間はキャッシュの速度に近づきます。ヒット率 0.9 では 19ns、ヒット率 0.95 では 14.5ns です。

**キャッシュの書き込み方式：**

| 方式 | 説明 |
|------|------|
| ライトスルー（Write Through） | キャッシュと主記憶の両方に即座に書き込む。整合性は保たれるが遅い |
| ライトバック（Write Back） | キャッシュのみに書き込み、追い出し時に主記憶に書き戻す。高速だが複雑 |

**キャッシュの配置方式：**

| 方式 | 説明 |
|------|------|
| ダイレクトマッピング | メモリの各ブロックが対応するキャッシュの位置は1つだけ |
| フルアソシアティブ | メモリの各ブロックはキャッシュのどこにでも配置可能 |
| セットアソシアティブ | 上記2つの折衷。複数の候補位置から選ぶ |

## 7.5 パイプライン処理

### パイプラインとは

命令の実行を複数のステージに分割し、各ステージを重ね合わせて並行実行することで、スループット（単位時間あたりの処理量）を向上させる技術です。

```
時間 →  T1   T2   T3   T4   T5   T6   T7
命令1: [F1] [D1] [E1] [S1]
命令2:      [F2] [D2] [E2] [S2]
命令3:           [F3] [D3] [E3] [S3]
命令4:                [F4] [D4] [E4] [S4]

F=フェッチ D=デコード E=実行 S=格納
```

パイプラインなしでは命令1つに4クロック×4命令=16クロックかかるところ、パイプラインにより7クロックで完了します。

**パイプライン処理時間の計算：**

```
ステージ数: k
1ステージの時間: t
命令数: n

パイプラインでの処理時間 = (k + n - 1) × t
パイプラインなしでの処理時間 = k × n × t
```

**計算例：**
```
ステージ数 k=5、1ステージ t=20ns、命令数 n=100

パイプライン: (5 + 100 - 1) × 20 = 104 × 20 = 2080ns
パイプラインなし: 5 × 100 × 20 = 10000ns
速度向上比: 10000 / 2080 ≒ 4.8倍
```

### パイプラインハザード

パイプラインの流れが乱れる原因です。

| ハザード | 原因 | 対策 |
|---------|------|------|
| 構造ハザード（資源ハザード） | ハードウェア資源の競合 | 資源の増設 |
| データハザード | 前の命令の結果が必要 | フォワーディング |
| 制御ハザード（分岐ハザード） | 分岐命令による行き先の不確定 | 分岐予測 |

## 7.6 バスと入出力制御

### バス（Bus）

CPU、メモリ、入出力装置間のデータ伝送路です。

| バスの種類 | 伝送内容 |
|-----------|---------|
| データバス | データそのもの |
| アドレスバス | メモリアドレス |
| 制御バス | 読み書きなどの制御信号 |

**バス幅と性能：**
- データバスが32ビットなら、一度に32ビット（4バイト）のデータを転送可能
- アドレスバスが32ビットなら、2³² = 4GBのメモリ空間をアドレッシング可能

### 入出力制御方式

| 方式 | 説明 | CPU負荷 |
|------|------|---------|
| プログラム制御方式 | CPUが直接入出力を制御 | 高い |
| 割込み方式 | 入出力完了時に割込みで通知 | 中程度 |
| DMA方式（Direct Memory Access） | DMAコントローラが主記憶と直接データ転送 | 低い |
| チャネル方式 | チャネルプロセッサが入出力を専門に処理 | 最も低い |

### 割込み（Interrupt）

現在の処理を一時中断して、別の処理（割込み処理）を実行する仕組みです。

| 種類 | 説明 | 例 |
|------|------|-----|
| 外部割込み（ハードウェア割込み） | ハードウェアが発生 | 入出力完了、タイマー |
| 内部割込み | プログラム実行中に発生 | ゼロ除算、オーバーフロー |
| ソフトウェア割込み | プログラムが意図的に発生 | システムコール |

## 7.7 性能指標

### MIPS（Million Instructions Per Second）

1秒間に実行できる命令数（百万命令単位）。

```
MIPS = クロック周波数(Hz) / CPI / 10⁶
```

**CPI（Clocks Per Instruction）：** 1命令あたりの平均クロック数

**計算例：**
```
クロック周波数: 2GHz = 2 × 10⁹ Hz
CPI: 4

MIPS = (2 × 10⁹) / 4 / 10⁶ = 500 MIPS

つまり、1秒間に5億命令を実行できる。
```

### クロック周波数とクロックサイクル時間

```
クロックサイクル時間 = 1 / クロック周波数

例: 2GHz → 1 / (2×10⁹) = 0.5ns（0.5ナノ秒）
```

### FLOPS（Floating-point Operations Per Second）

1秒間に実行できる浮動小数点演算の回数。科学計算やスーパーコンピュータの性能指標に使います。

## 確認問題

1. プログラムカウンタと命令レジスタの違いを述べよ。
2. キャッシュヒット率が95%、キャッシュアクセス時間5ns、主記憶アクセス時間80nsのとき、実効アクセス時間を求めよ。
3. 5ステージのパイプラインで1ステージ10nsの場合、200命令の処理時間を求めよ。
4. 指標アドレッシングの実効アドレスの求め方を述べよ。
5. DMA方式の利点を1つ述べよ。

<details>
<summary>解答</summary>

1. プログラムカウンタ（PC）は「次に実行する命令が格納されているメモリアドレス」を保持する。命令レジスタ（IR）は「現在実行中の命令そのもの」を保持する。
2. 実効アクセス時間 = 0.95 × 5 + 0.05 × 80 = 4.75 + 4.0 = 8.75ns
3. (5 + 200 - 1) × 10 = 204 × 10 = 2040ns
4. 実効アドレス = 命令のアドレス部 + インデックスレジスタの値。配列の各要素にアクセスするのに適している。
5. CPUを介さずにDMAコントローラが主記憶と入出力装置間で直接データ転送を行うため、CPUの負荷が軽減される（CPUはその間に別の処理を実行できる）。

</details>
