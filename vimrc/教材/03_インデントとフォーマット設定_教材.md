# 第3章：インデントとフォーマット設定

## この章のゴール

- タブとスペースに関する設定（`tabstop`、`shiftwidth`、`softtabstop`、`expandtab`）の違いを理解する
- 自動インデント（`autoindent`、`smartindent`、`cindent`）を使い分けられる
- `retab` コマンドで既存ファイルのタブ⇔スペース変換ができる
- `formatoptions` でテキストの自動整形を制御できる
- `textwidth` と `colorcolumn` で行幅のガイドラインを設定できる
- 言語ごとにインデント設定を切り替えられる

---

## 1. タブとスペースの基本

### 4つの設定オプション

Vim のインデントに関わる主要な設定は4つあります。それぞれの役割を正確に理解しましょう。

```vim
set tabstop=4       " タブ文字（\t）の表示幅
set shiftwidth=4    " >> や << で増減するインデント幅
set softtabstop=4   " Tab キーを押した時に挿入される幅
set expandtab       " タブ文字の代わりにスペースを挿入
```

### 各設定の違い

```
  tabstop（ts）: タブ文字 \t の「見た目の幅」を決めます
  ──────────────────────────────────────────────────
  set tabstop=4 の場合:
  →···if (true) {     ← タブ1個 = スペース4個分の幅
  →···→···return 1;   ← タブ2個 = スペース8個分の幅

  set tabstop=8 の場合:
  →·······if (true) {     ← タブ1個 = スペース8個分の幅


  shiftwidth（sw）: >> や << でのインデント増減幅
  ──────────────────────────────────────────────────
  set shiftwidth=4 の場合:
  hello        →  >>  →      hello    （4スペース分右へ）
      hello    →  <<  →  hello        （4スペース分左へ）


  softtabstop（sts）: Tab キー押下時の挿入幅
  ──────────────────────────────────────────────────
  set softtabstop=4 の場合:
  挿入モードで Tab → 4スペース分のインデントが入る
  （expandtab が ON ならスペース、OFF ならタブ+スペースの組み合わせ）


  expandtab（et）: タブをスペースに変換するかどうか
  ──────────────────────────────────────────────────
  set expandtab   → Tab キーでスペースが挿入される
  set noexpandtab → Tab キーでタブ文字（\t）が挿入される
```

### 推奨設定パターン

```vim
" パターン1: スペース4個（Python, Java, C++ など）
set tabstop=4
set shiftwidth=4
set softtabstop=4
set expandtab

" パターン2: スペース2個（JavaScript, TypeScript, Ruby, YAML など）
set tabstop=2
set shiftwidth=2
set softtabstop=2
set expandtab

" パターン3: タブ文字（Go, Makefile など）
set tabstop=4
set shiftwidth=4
set softtabstop=0
set noexpandtab
```

> **よくある間違い**: `tabstop` だけ変更して `shiftwidth` や `softtabstop` を変更し忘れると、`>>` 操作と `Tab` キーで異なる幅のインデントが入ってしまいます。4つの値は基本的に揃えましょう。

### ポイントまとめ

- `tabstop` はタブ文字の表示幅、`shiftwidth` は `>>` `<<` の幅、`softtabstop` は `Tab` キーの幅です
- `expandtab` を設定するとタブの代わりにスペースが挿入されます
- 4つの値は基本的に同じ値に揃えるのが推奨です

## 2. 自動インデント

### autoindent

`autoindent` は最もシンプルな自動インデントです。新しい行を作成した時に、前の行と同じインデントレベルを維持します。

```vim
set autoindent      " 前の行のインデントを引き継ぐ
```

```
  autoindent の動作:

  def hello():          ← インデントなし
      print("hello")   ← 手動で4スペース入力
      print("world")   ← Enter を押すと自動で4スペースが入る
```

### smartindent

`smartindent` は `autoindent` に加えて、C言語風の構文（`{` や `}` など）を認識してインデントを調整します。

```vim
set smartindent     " 構文に応じてインデントを自動調整
```

```
  smartindent の動作:

  if (condition) {    ← { の後に Enter を押すと...
      do_something(); ← 自動でインデントが1段深くなる
  }                   ← } を入力すると自動でインデントが戻る
```

### cindent

`cindent` はC言語に特化した高度な自動インデントです。C/C++ のコードを書く場合に使用します。

```vim
set cindent         " C言語用の高度な自動インデント
```

### 3つの違い

```
  ┌──────────────┬────────────────────────────────────────┐
  │ 設定         │ 特徴                                   │
  ├──────────────┼────────────────────────────────────────┤
  │ autoindent   │ 前の行のインデントをそのまま引き継ぐ   │
  │ smartindent  │ { } を認識してインデントを増減する     │
  │ cindent      │ C言語の文法を理解した高度なインデント   │
  └──────────────┴────────────────────────────────────────┘

  推奨: filetype indent on を使えば、ファイルタイプごとに
        最適なインデント方式が自動的に選択されます。
```

### filetype indent on との関係

```vim
filetype plugin indent on   " ファイルタイプに応じたインデントを有効化
```

この設定を有効にすると、Vim が言語ごとのインデントルールファイルを自動で読み込みます。多くの場合、`smartindent` や `cindent` を手動で設定する必要はありません。

```vim
" 推奨する基本設定
set autoindent              " ベースとして autoindent を有効化
filetype plugin indent on   " ファイルタイプ別インデントを有効化
" smartindent や cindent は filetype indent に任せる
```

### ポイントまとめ

- `autoindent` は前の行のインデントを引き継ぐ基本機能です
- `smartindent` は `{` `}` を認識する中級機能です
- `filetype plugin indent on` を使えば言語ごとに最適なインデントが適用されます

## 3. retab コマンド

### 既存ファイルのタブ⇔スペース変換

プロジェクトのコーディング規約に合わせて、既存ファイルのタブとスペースを変換する必要がある場合があります。

```vim
" タブをスペースに変換する
set expandtab       " まず expandtab を設定
:retab              " ファイル全体のタブをスペースに変換

" スペースをタブに変換する
set noexpandtab     " まず noexpandtab を設定
:retab!             " ! を付けるとスペースもタブに変換
```

### 注意点

```vim
" retab の前に tabstop を正しく設定しておく
set tabstop=4       " タブ幅を4に設定
set expandtab       " スペースに変換モード
:retab              " 実行

" 特定の範囲だけ変換する場合
:10,20retab         " 10行目から20行目だけ変換
```

> **よくある間違い**: `retab` は文字列リテラル内のタブ文字も変換してしまいます。コード内にタブ文字を含む文字列がある場合は注意が必要です。

### ポイントまとめ

- `:retab` でファイル全体のタブをスペースに変換できます
- `:retab!` でスペースをタブに変換できます
- 実行前に `tabstop` と `expandtab` を正しく設定しておきましょう

## 4. フォーマットオプション

### formatoptions の基本

`formatoptions`（略: `fo`）は、テキストの自動整形ルールを制御するオプションです。各フラグ（1文字）の組み合わせで動作を設定します。

```vim
" 現在の設定を確認
:set formatoptions?

" フラグを追加する
set formatoptions+=r    " r フラグを追加

" フラグを削除する
set formatoptions-=o    " o フラグを削除
```

### 主要なフラグ一覧

```
  ┌──────┬──────────────────────────────────────────────┐
  │ フラグ│ 説明                                        │
  ├──────┼──────────────────────────────────────────────┤
  │  t   │ textwidth でテキストを自動折り返し            │
  │  c   │ textwidth でコメントを自動折り返し            │
  │  r   │ Enter 押下時にコメントリーダーを自動挿入      │
  │  o   │ o/O でコメントリーダーを自動挿入              │
  │  q   │ gq でコメントを整形可能にする                 │
  │  a   │ テキストを自動的に整形（段落の自動リフロー）  │
  │  n   │ 番号付きリストを認識して整形する              │
  │  j   │ コメント行を結合する時にリーダーを削除        │
  │  l   │ 挿入モードで既に長い行を自動整形しない        │
  └──────┴──────────────────────────────────────────────┘
```

### 実用的な設定例

```vim
" プログラミング向けの推奨設定
set formatoptions=tcqrnjl

" 解説:
" t - テキストを textwidth で折り返す
" c - コメントを textwidth で折り返す
" q - gq でコメントを整形できる
" r - Enter でコメントリーダーを継続
" n - 番号付きリストを認識
" j - コメント結合時にリーダーを削除
" l - 既に長い行は整形しない
```

> **よくある間違い**: `formatoptions` は `filetype plugin` によって上書きされることがあります。確実に設定するには `autocmd` を使います。

```vim
" filetype plugin による上書きを防ぐ
augroup MyFormatOptions
  autocmd!
  autocmd FileType * setlocal formatoptions=tcqrnjl
augroup END
```

### ポイントまとめ

- `formatoptions` は1文字のフラグの組み合わせで制御します
- `+=` で追加、`-=` で削除できます
- `r` フラグ（Enter でコメント継続）と `j` フラグ（コメント結合）は便利です
- `filetype plugin` による上書きに注意が必要です

## 5. textwidth と colorcolumn

### textwidth の設定

`textwidth` は、テキストの自動折り返し幅を設定します。`formatoptions` に `t` や `c` が含まれている場合に有効です。

```vim
set textwidth=80    " 80文字で自動折り返し
set textwidth=0     " 自動折り返しを無効化（デフォルト）
```

### colorcolumn でガイドラインを表示

`colorcolumn` は、指定した列にカラーガイドライン（縦線）を表示します。行が長くなりすぎていないか視覚的に確認できます。

```vim
set colorcolumn=80          " 80列目にラインを表示
set colorcolumn=80,120      " 80列目と120列目にラインを表示
set colorcolumn=+1          " textwidth + 1 の位置にラインを表示
```

```
  colorcolumn の表示イメージ:

  1  def process_data(input_data):           │
  2      result = []                          │
  3      for item in input_data:              │
  4          if item.is_valid():              │
  5              processed = transform(item)  │
  6              result.append(processed)     │
  7      return result                        │
                                              ↑
                                          80列目のガイドライン
```

### colorcolumn の色を変更

```vim
" ガイドラインの色をカスタマイズ
highlight ColorColumn ctermbg=235 guibg=#2c2d27
```

### textwidth と colorcolumn の組み合わせ

```vim
" 推奨設定: 80文字ルール
set textwidth=80            " 80文字で自動折り返し
set colorcolumn=+1          " textwidth の次の列にガイドライン表示

" コードは折り返さず、ガイドラインだけ表示したい場合
set textwidth=0             " 自動折り返しは無効
set colorcolumn=80          " 80列目にガイドラインだけ表示
```

### ポイントまとめ

- `textwidth` はテキストの自動折り返し幅を設定します
- `colorcolumn` は視覚的なガイドラインを表示します
- `colorcolumn=+1` で `textwidth` に連動させるのが便利です

## 6. 言語別インデント設定

### autocmd を使った言語別設定

プログラミング言語ごとにインデントのルールは異なります。`autocmd FileType` を使って、言語ごとに設定を切り替えましょう。

```vim
" 言語別インデント設定
augroup FileTypeIndent
  autocmd!
  " Python: スペース4個（PEP 8 準拠）
  autocmd FileType python setlocal tabstop=4 shiftwidth=4 softtabstop=4 expandtab

  " JavaScript / TypeScript: スペース2個
  autocmd FileType javascript,typescript setlocal tabstop=2 shiftwidth=2 softtabstop=2 expandtab

  " HTML / CSS: スペース2個
  autocmd FileType html,css setlocal tabstop=2 shiftwidth=2 softtabstop=2 expandtab

  " Go: タブ文字（gofmt の規約）
  autocmd FileType go setlocal tabstop=4 shiftwidth=4 softtabstop=0 noexpandtab

  " Makefile: タブ文字（必須）
  autocmd FileType make setlocal tabstop=4 shiftwidth=4 softtabstop=0 noexpandtab

  " Ruby: スペース2個
  autocmd FileType ruby setlocal tabstop=2 shiftwidth=2 softtabstop=2 expandtab

  " YAML: スペース2個（インデントが重要な言語）
  autocmd FileType yaml setlocal tabstop=2 shiftwidth=2 softtabstop=2 expandtab

  " C / C++: スペース4個
  autocmd FileType c,cpp setlocal tabstop=4 shiftwidth=4 softtabstop=4 expandtab

  " Java: スペース4個
  autocmd FileType java setlocal tabstop=4 shiftwidth=4 softtabstop=4 expandtab

  " Vim script: スペース2個
  autocmd FileType vim setlocal tabstop=2 shiftwidth=2 softtabstop=2 expandtab
augroup END
```

### 主要言語のインデント規約

```
  ┌─────────────────┬──────────┬──────────────────────┐
  │ 言語            │ インデント│ 根拠                 │
  ├─────────────────┼──────────┼──────────────────────┤
  │ Python          │ 4スペース│ PEP 8                │
  │ JavaScript      │ 2スペース│ 多くのスタイルガイド  │
  │ TypeScript      │ 2スペース│ Angular / React 慣習 │
  │ Go              │ タブ     │ gofmt 公式           │
  │ Ruby            │ 2スペース│ Ruby コミュニティ    │
  │ HTML / CSS      │ 2スペース│ Google スタイルガイド │
  │ Java            │ 4スペース│ Google / Oracle      │
  │ C / C++         │ 4スペース│ Linux カーネル等     │
  │ Makefile        │ タブ     │ make の仕様（必須）  │
  │ YAML            │ 2スペース│ 仕様上の推奨         │
  └─────────────────┴──────────┴──────────────────────┘
```

> **重要**: `autocmd FileType` 内では必ず `setlocal` を使ってください。`set` を使うとグローバルに影響してしまいます。

### setlocal と set の違い

```vim
" set: グローバル設定（全バッファに影響）
set tabstop=4

" setlocal: バッファローカル設定（現在のバッファのみ）
setlocal tabstop=2

" autocmd では setlocal を使うのが正しい
autocmd FileType python setlocal expandtab     " ○ 正しい
autocmd FileType python set expandtab          " × 他のバッファにも影響する
```

### EditorConfig との連携

プロジェクトで `.editorconfig` を使っている場合、Vim 8.2 以降では標準でサポートされています。

```vim
" Vim 8.2 以降では追加設定不要
" .editorconfig ファイルが自動的に読み込まれます

" それ以前のバージョンではプラグインが必要:
" Plug 'editorconfig/editorconfig-vim'
```

### ポイントまとめ

- `autocmd FileType` で言語ごとにインデント設定を切り替えられます
- `augroup` で囲み、`autocmd!` で重複を防ぎましょう
- `setlocal` を使ってバッファローカルに設定するのが重要です
- Go と Makefile はタブ文字が必須です
