# 第9章: クラスの型

## この章のゴール

- TypeScriptのクラスにおけるアクセス修飾子（`public`, `private`, `protected`）を理解し、適切に使い分けられるようになる
- 抽象クラス（`abstract`）の概念と用途を理解する
- インターフェースの実装（`implements`）を使って、クラスの契約を定義できるようになる
- クラスとインターフェースの使い分けを判断できるようになる
- 実務でよく使われるクラス設計パターンを書けるようになる

---

## 9.1 TypeScriptのクラスの基礎

### JavaScriptのクラスとの違い

JavaScriptにもES2015からクラス構文がありますが、TypeScriptでは**型注釈**、**アクセス修飾子**、**抽象クラス**などの機能が追加されています。

```javascript
// JavaScript のクラス（型情報なし）
class User {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    return `こんにちは、${this.name}です`;
  }
}

// 何でも代入できてしまう
const user = new User(42, "太郎"); // 引数が逆でもエラーにならない
user.unknownProp = "何でも追加できる"; // プロパティを自由に追加できてしまう
```

```typescript
// TypeScript のクラス（型情報あり）
class User {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet(): string {
    return `こんにちは、${this.name}です`;
  }
}

// const user = new User(42, "太郎"); // エラー！型が違う
const user = new User("太郎", 25);
// user.unknownProp = "追加不可"; // エラー！宣言されていないプロパティ
```

### コンストラクタの省略記法（Parameter Properties）

TypeScriptでは、コンストラクタのパラメータにアクセス修飾子を付けると、プロパティの宣言と代入を同時に行えます。

```typescript
// 通常の書き方（冗長）
class User {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}

// 省略記法（Parameter Properties）
class User2 {
  constructor(
    public name: string,
    public age: number
  ) {
    // this.name = name; と this.age = age; が自動的に行われる
  }
}

// どちらも同じ動作
const user1 = new User("太郎", 25);
const user2 = new User2("太郎", 25);
console.log(user1.name === user2.name); // true
```

> **ポイント:** Parameter Propertiesは、コードを簡潔に書けるため広く使われています。ただし、コンストラクタに複雑なロジックがある場合は、通常の書き方の方が読みやすいこともあります。

## 9.2 アクセス修飾子

### public, private, protected

TypeScriptには3つのアクセス修飾子があります。

```
┌──────────────────────────────────────────────────┐
│           アクセス修飾子の可視性                    │
│                                                    │
│              クラス内部   サブクラス   外部         │
│  public        ○           ○          ○          │
│  protected     ○           ○          ×          │
│  private       ○           ×          ×          │
│                                                    │
│  ○ = アクセス可能  × = アクセス不可               │
└──────────────────────────────────────────────────┘
```

```typescript
class BankAccount {
  // public: どこからでもアクセス可能（デフォルト）
  public ownerName: string;

  // protected: このクラスとサブクラスからアクセス可能
  protected balance: number;

  // private: このクラスの中からのみアクセス可能
  private pin: string;

  constructor(ownerName: string, initialBalance: number, pin: string) {
    this.ownerName = ownerName;
    this.balance = initialBalance;
    this.pin = pin;
  }

  // publicメソッド
  public getBalance(): number {
    return this.balance;
  }

  // privateメソッド: 内部でのみ使用
  private validatePin(inputPin: string): boolean {
    return this.pin === inputPin;
  }

  public withdraw(amount: number, pin: string): boolean {
    if (!this.validatePin(pin)) {
      console.log("PINが不正です");
      return false;
    }
    if (amount > this.balance) {
      console.log("残高不足です");
      return false;
    }
    this.balance -= amount;
    return true;
  }
}

class SavingsAccount extends BankAccount {
  private interestRate: number;

  constructor(ownerName: string, initialBalance: number, pin: string, rate: number) {
    super(ownerName, initialBalance, pin);
    this.interestRate = rate;
  }

  public addInterest(): void {
    // protected なので、サブクラスからアクセス可能
    this.balance += this.balance * this.interestRate;

    // private なので、サブクラスからはアクセス不可
    // this.pin; // エラー！
  }
}

const account = new BankAccount("太郎", 10000, "1234");
console.log(account.ownerName);    // OK: public
// console.log(account.balance);   // エラー！protected
// console.log(account.pin);       // エラー！private
console.log(account.getBalance()); // OK: publicメソッド経由
```

### ECMAScript private (#)

TypeScript 3.8以降では、JavaScriptネイティブのprivateフィールド（`#`プレフィックス）も使えます。

```typescript
class Counter {
  // #はランタイムでも本当にprivate（TypeScriptのprivateはコンパイル時のみ）
  #count: number = 0;

  increment(): void {
    this.#count++;
  }

  getCount(): number {
    return this.#count;
  }
}

const counter = new Counter();
counter.increment();
console.log(counter.getCount()); // 1
// console.log(counter.#count);  // SyntaxError（ランタイムでもエラー）
```

**TypeScript private vs # private:**

| 特徴 | `private` | `#` |
|------|-----------|-----|
| チェックのタイミング | コンパイル時のみ | ランタイムでも |
| JSコンパイル後 | 普通のプロパティに | WeakMapで真のprivate |
| サブクラスでの名前衝突 | あり得る | 完全に隔離 |
| パフォーマンス | 高い | やや低い（WeakMap） |

### readonly修飾子

`readonly` はプロパティを読み取り専用にします。コンストラクタでのみ初期化できます。

```typescript
class Config {
  readonly apiUrl: string;
  readonly maxRetries: number;

  constructor(apiUrl: string, maxRetries: number = 3) {
    this.apiUrl = apiUrl;
    this.maxRetries = maxRetries;
  }

  // updateUrl(newUrl: string): void {
  //   this.apiUrl = newUrl; // エラー！readonlyプロパティに代入できない
  // }
}

const config = new Config("https://api.example.com");
// config.apiUrl = "new-url"; // エラー！
```

### ポイントまとめ

- `public`: どこからでもアクセス可能（デフォルト）
- `protected`: クラス自身とサブクラスからのみアクセス可能
- `private`: クラス自身からのみアクセス可能
- `#`: ランタイムでも真にプライベート
- `readonly`: コンストラクタでのみ初期化可能

## 9.3 抽象クラス（Abstract Classes）

### 抽象クラスとは

抽象クラスは、**直接インスタンス化できない**クラスです。共通のロジックを提供しつつ、一部のメソッドの実装をサブクラスに強制します。「設計図の設計図」のようなものです。

```typescript
abstract class Shape {
  // 抽象メソッド: サブクラスで必ず実装しなければならない
  abstract getArea(): number;
  abstract getPerimeter(): number;

  // 通常のメソッド: 共通のロジックを提供
  describe(): string {
    return `面積: ${this.getArea().toFixed(2)}, 周長: ${this.getPerimeter().toFixed(2)}`;
  }
}

// const shape = new Shape(); // エラー！抽象クラスはインスタンス化できない

class Circle extends Shape {
  constructor(private radius: number) {
    super();
  }

  // 抽象メソッドの実装（省略するとエラー）
  getArea(): number {
    return Math.PI * this.radius ** 2;
  }

  getPerimeter(): number {
    return 2 * Math.PI * this.radius;
  }
}

class Rectangle extends Shape {
  constructor(
    private width: number,
    private height: number
  ) {
    super();
  }

  getArea(): number {
    return this.width * this.height;
  }

  getPerimeter(): number {
    return 2 * (this.width + this.height);
  }
}

const circle = new Circle(5);
console.log(circle.describe()); // "面積: 78.54, 周長: 31.42"

const rect = new Rectangle(10, 5);
console.log(rect.describe()); // "面積: 50.00, 周長: 30.00"
```

### 抽象プロパティ

メソッドだけでなく、プロパティも抽象化できます。

```typescript
abstract class Vehicle {
  abstract readonly type: string;
  abstract maxSpeed: number;

  describe(): string {
    return `${this.type}: 最高速度 ${this.maxSpeed}km/h`;
  }
}

class Car extends Vehicle {
  readonly type = "乗用車";
  maxSpeed = 180;
}

class Bicycle extends Vehicle {
  readonly type = "自転車";
  maxSpeed = 30;
}
```

### 実践例: テンプレートメソッドパターン

抽象クラスの古典的な使い方である「テンプレートメソッドパターン」です。処理の流れ（骨格）は基底クラスが定義し、各ステップの詳細をサブクラスが実装します。

```typescript
abstract class DataProcessor<T, R> {
  // テンプレートメソッド: 処理の流れを定義
  process(input: T): R {
    const validated = this.validate(input);
    const transformed = this.transform(validated);
    const result = this.format(transformed);
    this.log(result);
    return result;
  }

  // サブクラスで実装する抽象メソッド
  protected abstract validate(input: T): T;
  protected abstract transform(input: T): R;
  protected abstract format(result: R): R;

  // 共通のログ処理（オーバーライド可能）
  protected log(result: R): void {
    console.log("処理完了:", result);
  }
}

class CsvProcessor extends DataProcessor<string, string[][]> {
  protected validate(input: string): string {
    if (!input.trim()) throw new Error("空のCSVです");
    return input.trim();
  }

  protected transform(input: string): string[][] {
    return input.split("\n").map((row) => row.split(","));
  }

  protected format(result: string[][]): string[][] {
    return result.map((row) => row.map((cell) => cell.trim()));
  }
}

const processor = new CsvProcessor();
const data = processor.process("name,age\n太郎,25\n花子,30");
// [["name", "age"], ["太郎", "25"], ["花子", "30"]]
```

### ポイントまとめ

- `abstract class` は直接インスタンス化できない
- `abstract` メソッド/プロパティはサブクラスで必ず実装しなければならない
- 共通ロジックと強制実装を同時に提供する「テンプレート」として機能する

## 9.4 インターフェースの実装（implements）

### implements とは

`implements` キーワードは、クラスが特定のインターフェースに定義された構造を満たすことを宣言します。インターフェースは「契約」であり、クラスはその契約を守る必要があります。

```typescript
interface Serializable {
  serialize(): string;
  deserialize(data: string): void;
}

interface Printable {
  print(): void;
}

// 複数のインターフェースを実装可能
class Document implements Serializable, Printable {
  constructor(
    public title: string,
    public content: string
  ) {}

  serialize(): string {
    return JSON.stringify({ title: this.title, content: this.content });
  }

  deserialize(data: string): void {
    const parsed = JSON.parse(data);
    this.title = parsed.title;
    this.content = parsed.content;
  }

  print(): void {
    console.log(`=== ${this.title} ===`);
    console.log(this.content);
  }
}

// インターフェースが守られていないとコンパイルエラー
// class BadDoc implements Serializable {
//   // serialize()とdeserialize()がない → エラー！
// }
```

### interface vs abstract class

| 特徴 | `interface` | `abstract class` |
|------|-------------|-------------------|
| 実装コード | 持てない | 持てる |
| 複数継承 | `implements` で複数可 | `extends` は1つのみ |
| プロパティ | 型のみ定義 | 初期値も設定可能 |
| コンストラクタ | なし | あり |
| ランタイム存在 | なし（型のみ） | あり |
| 用途 | 構造の契約 | 共通ロジック+構造の契約 |

```typescript
// interfaceが適切: 構造の「形」だけを定義したい場合
interface Logger {
  info(message: string): void;
  error(message: string): void;
  warn(message: string): void;
}

// abstract classが適切: 共通のロジックも提供したい場合
abstract class BaseLogger implements Logger {
  abstract info(message: string): void;
  abstract error(message: string): void;
  abstract warn(message: string): void;

  // 共通のフォーマット処理
  protected formatMessage(level: string, message: string): string {
    const timestamp = new Date().toISOString();
    return `[${timestamp}] [${level}] ${message}`;
  }
}

class ConsoleLogger extends BaseLogger {
  info(message: string): void {
    console.log(this.formatMessage("INFO", message));
  }

  error(message: string): void {
    console.error(this.formatMessage("ERROR", message));
  }

  warn(message: string): void {
    console.warn(this.formatMessage("WARN", message));
  }
}
```

### 実践例: プラグインシステム

```typescript
interface Plugin {
  name: string;
  version: string;
  initialize(): Promise<void>;
  destroy(): Promise<void>;
}

interface HasConfig<T> {
  getConfig(): T;
  updateConfig(config: Partial<T>): void;
}

interface DatabaseConfig {
  host: string;
  port: number;
  database: string;
}

class DatabasePlugin implements Plugin, HasConfig<DatabaseConfig> {
  name = "database";
  version = "1.0.0";

  private config: DatabaseConfig;

  constructor(config: DatabaseConfig) {
    this.config = config;
  }

  async initialize(): Promise<void> {
    console.log(`Connecting to ${this.config.host}:${this.config.port}`);
  }

  async destroy(): Promise<void> {
    console.log("Disconnecting from database");
  }

  getConfig(): DatabaseConfig {
    return { ...this.config };
  }

  updateConfig(config: Partial<DatabaseConfig>): void {
    this.config = { ...this.config, ...config };
  }
}

// Plugin型として扱える
function registerPlugin(plugin: Plugin): void {
  console.log(`Registering plugin: ${plugin.name} v${plugin.version}`);
  plugin.initialize();
}
```

## 9.5 ジェネリッククラス

### ジェネリクスとクラスの組み合わせ

第5章で学んだジェネリクスをクラスに適用する実践例です。

```typescript
// ジェネリックなキュー（FIFO）
class Queue<T> {
  private items: T[] = [];

  enqueue(item: T): void {
    this.items.push(item);
  }

  dequeue(): T | undefined {
    return this.items.shift();
  }

  peek(): T | undefined {
    return this.items[0];
  }

  get size(): number {
    return this.items.length;
  }

  isEmpty(): boolean {
    return this.items.length === 0;
  }

  toArray(): T[] {
    return [...this.items];
  }
}

// 型安全なキュー
const numberQueue = new Queue<number>();
numberQueue.enqueue(1);
numberQueue.enqueue(2);
const num = numberQueue.dequeue(); // number | undefined

const taskQueue = new Queue<{ id: string; priority: number }>();
taskQueue.enqueue({ id: "task-1", priority: 1 });
```

### 実践例: 型安全なEventEmitter

```typescript
// イベントマップの型を受け取るジェネリッククラス
class EventEmitter<TEvents extends Record<string, any>> {
  private handlers: Partial<{
    [K in keyof TEvents]: Array<(data: TEvents[K]) => void>;
  }> = {};

  on<K extends keyof TEvents>(
    event: K,
    handler: (data: TEvents[K]) => void
  ): () => void {
    if (!this.handlers[event]) {
      this.handlers[event] = [];
    }
    this.handlers[event]!.push(handler);

    // unsubscribe関数を返す
    return () => {
      const handlers = this.handlers[event];
      if (handlers) {
        const index = handlers.indexOf(handler);
        if (index > -1) handlers.splice(index, 1);
      }
    };
  }

  emit<K extends keyof TEvents>(event: K, data: TEvents[K]): void {
    const handlers = this.handlers[event];
    if (handlers) {
      handlers.forEach((handler) => handler(data));
    }
  }
}

// 使用例
interface AppEvents {
  userLogin: { userId: string; timestamp: Date };
  userLogout: { userId: string };
  error: { message: string; code: number };
}

const emitter = new EventEmitter<AppEvents>();

const unsubscribe = emitter.on("userLogin", (data) => {
  // data は { userId: string; timestamp: Date } と推論される
  console.log(`ログイン: ${data.userId}`);
});

emitter.emit("userLogin", {
  userId: "user-123",
  timestamp: new Date(),
});

// 不要になったら解除
unsubscribe();
```

## 9.6 静的メンバーの型

### 静的プロパティと静的メソッド

```typescript
class MathUtils {
  // 静的プロパティ
  static readonly PI: number = 3.14159265358979;
  static readonly E: number = 2.71828182845905;

  // 静的メソッド
  static clamp(value: number, min: number, max: number): number {
    return Math.min(Math.max(value, min), max);
  }

  static lerp(a: number, b: number, t: number): number {
    return a + (b - a) * MathUtils.clamp(t, 0, 1);
  }
}

// インスタンス化不要で使える
console.log(MathUtils.PI);           // 3.14159265358979
console.log(MathUtils.clamp(15, 0, 10)); // 10
console.log(MathUtils.lerp(0, 100, 0.5)); // 50
```

### シングルトンパターン

```typescript
class Database {
  private static instance: Database | null = null;

  private constructor(
    private connectionString: string
  ) {}

  static getInstance(connectionString?: string): Database {
    if (!Database.instance) {
      if (!connectionString) {
        throw new Error("初回は接続文字列が必要です");
      }
      Database.instance = new Database(connectionString);
    }
    return Database.instance;
  }

  query(sql: string): void {
    console.log(`Executing: ${sql} on ${this.connectionString}`);
  }
}

// const db = new Database("..."); // エラー！コンストラクタはprivate
const db1 = Database.getInstance("postgres://localhost/mydb");
const db2 = Database.getInstance(); // 同じインスタンスが返る
console.log(db1 === db2); // true
```

## 9.7 this 型

### メソッドチェーンの型安全な実装

```typescript
class QueryBuilder {
  private conditions: string[] = [];
  private orderByClause: string = "";
  private limitValue: number | null = null;

  where(condition: string): this {
    this.conditions.push(condition);
    return this; // this を返すことでメソッドチェーンが可能
  }

  orderBy(column: string, direction: "ASC" | "DESC" = "ASC"): this {
    this.orderByClause = `ORDER BY ${column} ${direction}`;
    return this;
  }

  limit(count: number): this {
    this.limitValue = count;
    return this;
  }

  build(): string {
    let query = "SELECT * FROM table";
    if (this.conditions.length > 0) {
      query += " WHERE " + this.conditions.join(" AND ");
    }
    if (this.orderByClause) {
      query += " " + this.orderByClause;
    }
    if (this.limitValue !== null) {
      query += ` LIMIT ${this.limitValue}`;
    }
    return query;
  }
}

const query = new QueryBuilder()
  .where("age > 20")
  .where("status = 'active'")
  .orderBy("name", "ASC")
  .limit(10)
  .build();
// "SELECT * FROM table WHERE age > 20 AND status = 'active' ORDER BY name ASC LIMIT 10"
```

> **ポイント:** 戻り値の型を `this` にすることで、サブクラスでもメソッドチェーンが正しく機能します。具体的なクラス名を返すと、サブクラスで型が狭くなってしまいます。

## 9.8 よくある間違い

### 1. implements を extends と間違える

```typescript
interface Flyable {
  fly(): void;
}

// ❌ interfaceをextendsすることはできない（クラスではないため）
// class Bird extends Flyable {} // エラー！

// ✅ interfaceはimplementsする
class Bird implements Flyable {
  fly(): void {
    console.log("飛んでいます");
  }
}
```

### 2. protected を使いすぎる

```typescript
// ❌ 全部 protected にしてしまう
class BadClass {
  protected name: string;
  protected age: number;
  protected email: string;

  constructor(name: string, age: number, email: string) {
    this.name = name;
    this.age = age;
    this.email = email;
  }
}

// ✅ 必要最小限のアクセスレベルを設定
class GoodClass {
  public readonly name: string;  // 読み取りは外部から可能
  private age: number;           // 内部でのみ使用
  private email: string;         // 内部でのみ使用

  constructor(name: string, age: number, email: string) {
    this.name = name;
    this.age = age;
    this.email = email;
  }

  // 必要ならgetterで公開
  getAge(): number {
    return this.age;
  }
}
```

### 3. 抽象クラスをインスタンス化しようとする

```typescript
abstract class Animal {
  abstract speak(): string;
}

// ❌ 抽象クラスのインスタンス化はできない
// const animal = new Animal(); // エラー！

// ✅ 具象クラスをインスタンス化する
class Dog extends Animal {
  speak(): string {
    return "ワン！";
  }
}

const dog = new Dog();
```

## 9.9 現場でのベストプラクティス

### 1. クラスよりインターフェース+関数を優先する

モダンなTypeScript/JavaScriptでは、クラスよりも関数とインターフェースの組み合わせが好まれる傾向があります。

```typescript
// ✅ インターフェース+関数（推奨される場面が多い）
interface UserService {
  getUser(id: number): Promise<User>;
  createUser(data: CreateUserInput): Promise<User>;
}

function createUserService(db: Database): UserService {
  return {
    async getUser(id) {
      return db.query(`SELECT * FROM users WHERE id = ${id}`);
    },
    async createUser(data) {
      return db.query(`INSERT INTO users ...`);
    },
  };
}

// ✅ クラスが適切な場面: 状態+メソッドが密接に結びついている場合
class ConnectionPool {
  private connections: Connection[] = [];
  private maxSize: number;

  constructor(maxSize: number) {
    this.maxSize = maxSize;
  }

  acquire(): Connection { /* ... */ }
  release(conn: Connection): void { /* ... */ }
}
```

### 2. 型のために class を使わない

```typescript
// ❌ 型定義のためだけにクラスを使わない
class UserDTO {
  name: string = "";
  email: string = "";
  age: number = 0;
}

// ✅ 型定義にはinterfaceかtype aliasを使う
interface UserDTO {
  name: string;
  email: string;
  age: number;
}
```

### 3. 継承より合成（Composition over Inheritance）

```typescript
// ❌ 深い継承階層は避ける
class Animal { /* ... */ }
class Mammal extends Animal { /* ... */ }
class Dog extends Mammal { /* ... */ }
class GuideDog extends Dog { /* ... */ }

// ✅ 合成（Composition）を使う
interface Walkable {
  walk(): void;
}

interface Trainable {
  train(command: string): void;
}

interface GuideDog extends Walkable, Trainable {
  guideTo(destination: string): void;
}

function createGuideDog(): GuideDog {
  return {
    walk() { console.log("歩いています"); },
    train(command) { console.log(`${command}を訓練中`); },
    guideTo(destination) { console.log(`${destination}へ誘導中`); },
  };
}
```
