# 第9章: イベント処理

## この章のゴール

- イベントの概念とイベント駆動プログラミングを理解する
- addEventListener でイベントを登録・解除できるようになる
- イベントオブジェクトの情報を活用できる
- イベント伝搬（バブリング・キャプチャリング）を理解する
- イベント委譲パターンを使いこなせる

---

## 9.1 イベントとは

**イベント**は、ブラウザで何かが起きたことを通知する仕組みです。

```
ユーザーの操作やブラウザの動作がイベントを発生させる:

  ユーザー操作:
    click      → マウスクリック
    dblclick   → ダブルクリック
    mouseover  → マウスが乗った
    mouseout   → マウスが離れた
    keydown    → キーを押した
    keyup      → キーを離した
    input      → 入力が変化した
    submit     → フォーム送信

  ブラウザ:
    load       → ページ読み込み完了
    DOMContentLoaded → DOM構築完了
    resize     → ウィンドウサイズ変更
    scroll     → スクロール
```

## 9.2 イベントリスナーの登録

```javascript
// addEventListener(イベント名, コールバック関数)
const button = document.querySelector("#myButton");

button.addEventListener("click", function() {
  console.log("ボタンがクリックされました！");
});

// アロー関数で書く場合
button.addEventListener("click", () => {
  console.log("クリック！");
});

// 名前付き関数を使う場合（解除する時に便利）
function handleClick() {
  console.log("クリック！");
}
button.addEventListener("click", handleClick);
```

### イベントリスナーの解除

```javascript
function handleClick() {
  console.log("一度だけクリック");
}

const button = document.querySelector("#myButton");
button.addEventListener("click", handleClick);

// 解除（同じ関数参照が必要）
button.removeEventListener("click", handleClick);

// ★注意: 無名関数は解除できない
button.addEventListener("click", () => { /* ... */ });
// これは removeEventListener で解除できない！

// once オプション — 1回だけ実行して自動解除
button.addEventListener("click", () => {
  console.log("この処理は1回だけ");
}, { once: true });
```

## 9.3 イベントオブジェクト

コールバック関数には自動的に**イベントオブジェクト**が渡されます。

```javascript
const button = document.querySelector("#myButton");

button.addEventListener("click", (event) => {
  // event（慣習的に e と書くことも多い）
  console.log(event.type);       // "click"
  console.log(event.target);     // クリックされた要素
  console.log(event.currentTarget); // リスナーが登録された要素
  console.log(event.clientX);    // マウスのX座標（ビューポート基準）
  console.log(event.clientY);    // マウスのY座標
  console.log(event.timeStamp);  // イベント発生時刻
});
```

### マウスイベント

```javascript
const area = document.querySelector("#area");

area.addEventListener("mousemove", (e) => {
  console.log(`座標: (${e.clientX}, ${e.clientY})`);
});

area.addEventListener("mousedown", (e) => {
  console.log(`ボタン: ${e.button}`); // 0=左, 1=中, 2=右
});
```

### キーボードイベント

```javascript
document.addEventListener("keydown", (e) => {
  console.log(`キー: ${e.key}`);     // "Enter", "a", "ArrowUp" など
  console.log(`コード: ${e.code}`);  // "Enter", "KeyA", "ArrowUp" など
  console.log(`Ctrl: ${e.ctrlKey}`); // true/false
  console.log(`Shift: ${e.shiftKey}`);
  console.log(`Alt: ${e.altKey}`);

  // Ctrl + S でカスタム保存
  if (e.ctrlKey && e.key === "s") {
    e.preventDefault(); // ブラウザのデフォルト動作を阻止
    console.log("保存処理を実行");
  }
});
```

### フォームイベント

```javascript
const input = document.querySelector("#nameInput");
const form = document.querySelector("#myForm");

// input — 入力するたびに発火
input.addEventListener("input", (e) => {
  console.log(`入力中: ${e.target.value}`);
});

// change — 値が確定した時（フォーカスが外れた時）
input.addEventListener("change", (e) => {
  console.log(`確定: ${e.target.value}`);
});

// focus / blur — フォーカスの取得/喪失
input.addEventListener("focus", () => console.log("フォーカスイン"));
input.addEventListener("blur", () => console.log("フォーカスアウト"));

// submit — フォーム送信
form.addEventListener("submit", (e) => {
  e.preventDefault(); // ページ遷移を防ぐ
  const formData = new FormData(form);
  console.log("名前:", formData.get("name"));
  console.log("メール:", formData.get("email"));
});
```

## 9.4 preventDefault と stopPropagation

```javascript
// preventDefault — ブラウザのデフォルト動作を阻止
const link = document.querySelector("a");
link.addEventListener("click", (e) => {
  e.preventDefault(); // リンク先へ遷移しない
  console.log("リンクがクリックされましたが、遷移しません");
});

// フォーム送信の阻止
const form = document.querySelector("form");
form.addEventListener("submit", (e) => {
  e.preventDefault(); // フォームの送信（ページリロード）を阻止
  // JavaScriptで処理を行う
});

// stopPropagation — イベントの伝搬を止める（後述）
```

## 9.5 イベント伝搬（バブリングとキャプチャリング）

```
イベント伝搬の流れ:

  1. キャプチャリングフェーズ（上→下）
     document → html → body → div → button
                                        ↓ [クリック発生]
  2. ターゲットフェーズ
     button（実際にクリックされた要素）
                                        ↓
  3. バブリングフェーズ（下→上）
     button → div → body → html → document
```

```html
<div id="outer" style="padding: 30px; background: lightblue;">
  外側
  <div id="inner" style="padding: 30px; background: lightgreen;">
    内側
    <button id="btn">ボタン</button>
  </div>
</div>

<script>
  // バブリング: 内側から外側へイベントが伝搬する
  document.getElementById("outer").addEventListener("click", () => {
    console.log("outer クリック");
  });

  document.getElementById("inner").addEventListener("click", () => {
    console.log("inner クリック");
  });

  document.getElementById("btn").addEventListener("click", () => {
    console.log("button クリック");
  });

  // ボタンをクリックすると:
  // "button クリック"
  // "inner クリック"
  // "outer クリック"
  // → バブリングにより全ての祖先要素にもイベントが伝わる
</script>
```

### stopPropagation

```javascript
document.getElementById("btn").addEventListener("click", (e) => {
  e.stopPropagation(); // バブリングを止める
  console.log("button クリック");
});
// ボタンをクリックしても "button クリック" だけが出力される
```

## 9.6 イベント委譲（Event Delegation）

多くの子要素にそれぞれリスナーを登録する代わりに、**親要素に1つだけ登録**するパターンです。

```
従来の方法（非効率）:
  <ul>
    <li>項目1</li>  ← addEventListener
    <li>項目2</li>  ← addEventListener
    <li>項目3</li>  ← addEventListener
    <li>項目4</li>  ← addEventListener
    ...100個
  </ul>
  → 100個のリスナーが必要

イベント委譲（効率的）:
  <ul>            ← addEventListener（1つだけ！）
    <li>項目1</li>
    <li>項目2</li>
    <li>項目3</li>
    <li>項目4</li>
    ...100個
  </ul>
  → 1つのリスナーで全てを処理
```

```javascript
const list = document.querySelector("#todoList");

// 親要素（ul）にリスナーを1つ登録
list.addEventListener("click", (e) => {
  // クリックされた要素が li かチェック
  if (e.target.tagName === "LI") {
    e.target.classList.toggle("completed");
    console.log(`${e.target.textContent} をトグル`);
  }
});

// 動的に追加された要素にも自動で効く！
const newItem = document.createElement("li");
newItem.textContent = "新しい項目";
list.append(newItem);
// → この新しい項目もクリックで反応する

// closest() を使うパターン（より柔軟）
list.addEventListener("click", (e) => {
  const item = e.target.closest("li");
  if (item && list.contains(item)) {
    console.log(`クリック: ${item.textContent}`);
  }
});
```

## 9.7 カスタムイベント

```javascript
// カスタムイベントの作成と発火
const event = new CustomEvent("userLogin", {
  detail: { username: "太郎", role: "admin" }
});

// リスナーの登録
document.addEventListener("userLogin", (e) => {
  console.log(`${e.detail.username}がログインしました（${e.detail.role}）`);
});

// イベントの発火
document.dispatchEvent(event);
// "太郎がログインしました（admin）"
```

## 9.8 スロットリングとデバウンス

高頻度で発生するイベント（scroll, resize, input）のパフォーマンス対策です。

```
スロットリング: 一定間隔で1回だけ実行
  ┃━━━━━┃━━━━━┃━━━━━┃
  ↑     ↑     ↑     ↑
  実行  無視   実行  無視  ...

デバウンス: 最後の呼び出しから一定時間後に1回だけ実行
  ┃→┃→┃→┃→┃........┃
              ↑最後の呼び出し  ↑実行
```

```javascript
// デバウンス関数
function debounce(func, delay) {
  let timeoutId;
  return function(...args) {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => func.apply(this, args), delay);
  };
}

// 使用例: 検索入力（タイピングが止まってから300ms後に検索）
const searchInput = document.querySelector("#search");
const handleSearch = debounce((e) => {
  console.log(`検索: ${e.target.value}`);
  // API呼び出しなど
}, 300);
searchInput.addEventListener("input", handleSearch);

// スロットリング関数
function throttle(func, interval) {
  let lastTime = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      func.apply(this, args);
    }
  };
}

// 使用例: スクロールイベント（100msに1回だけ処理）
window.addEventListener("scroll", throttle(() => {
  console.log(`スクロール位置: ${window.scrollY}`);
}, 100));
```

## 9.9 現場で使われるパターン

### モーダル（ダイアログ）の開閉

```javascript
const modal = document.querySelector("#modal");
const overlay = document.querySelector("#overlay");

// 開く
document.querySelector("#openBtn").addEventListener("click", () => {
  modal.classList.add("active");
  overlay.classList.add("active");
});

// 閉じる（複数の方法）
function closeModal() {
  modal.classList.remove("active");
  overlay.classList.remove("active");
}

document.querySelector("#closeBtn").addEventListener("click", closeModal);
overlay.addEventListener("click", closeModal);

// Escキーで閉じる
document.addEventListener("keydown", (e) => {
  if (e.key === "Escape") closeModal();
});
```

### 無限スクロール

```javascript
window.addEventListener("scroll", throttle(() => {
  // ページ下部に近づいたら追加データを読み込む
  const scrollHeight = document.documentElement.scrollHeight;
  const scrollTop = document.documentElement.scrollTop;
  const clientHeight = document.documentElement.clientHeight;

  if (scrollTop + clientHeight >= scrollHeight - 100) {
    console.log("追加データを読み込み...");
    // loadMoreData();
  }
}, 200));
```

### フォームバリデーション

```javascript
const form = document.querySelector("#registrationForm");

form.addEventListener("submit", (e) => {
  e.preventDefault();

  const name = form.querySelector('[name="name"]').value.trim();
  const email = form.querySelector('[name="email"]').value.trim();
  const errors = [];

  if (!name) errors.push("名前を入力してください");
  if (name.length < 2) errors.push("名前は2文字以上で入力してください");
  if (!email) errors.push("メールアドレスを入力してください");
  if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
    errors.push("正しいメールアドレスを入力してください");
  }

  const errorDiv = document.querySelector("#errors");
  if (errors.length > 0) {
    errorDiv.innerHTML = errors.map(err => `<p class="error">${err}</p>`).join("");
  } else {
    errorDiv.innerHTML = '<p class="success">送信しました！</p>';
    // フォームデータの送信処理
  }
});
```

## よくある間違い

### 1. thisがアロー関数で使えない

```javascript
const button = document.querySelector("button");

// NG: アロー関数内の this は外側のスコープを参照する
button.addEventListener("click", () => {
  console.log(this); // Window オブジェクト（ボタンではない！）
});

// OK: 通常の関数なら this はイベントターゲットを指す
button.addEventListener("click", function() {
  console.log(this); // button 要素
});

// OK: event.target / event.currentTarget を使う（推奨）
button.addEventListener("click", (e) => {
  console.log(e.currentTarget); // button 要素
});
```

### 2. ループ内でのイベント登録（クロージャの罠）

```javascript
// NG: var を使うと最後の値しか参照できない
const buttons = document.querySelectorAll("button");
for (var i = 0; i < buttons.length; i++) {
  buttons[i].addEventListener("click", () => {
    console.log(i); // 常に buttons.length が出力される
  });
}

// OK: let を使う（ブロックスコープ）
for (let i = 0; i < buttons.length; i++) {
  buttons[i].addEventListener("click", () => {
    console.log(i); // 正しい値が出力される
  });
}

// OK: forEach を使う
buttons.forEach((button, i) => {
  button.addEventListener("click", () => console.log(i));
});
```

### 3. イベントリスナーの重複登録

```javascript
// NG: ボタンをクリックするたびにリスナーが追加される
function setup() {
  button.addEventListener("click", () => {
    console.log("クリック");
  });
}
// setup() を複数回呼ぶとリスナーが重複する

// OK: 名前付き関数なら同じ関数は重複登録されない
function handleClick() {
  console.log("クリック");
}
button.addEventListener("click", handleClick);
button.addEventListener("click", handleClick); // 重複は無視される
```

## ポイントまとめ

- `addEventListener` でイベントリスナーを登録します。HTML属性（onclick等）は避けましょう
- イベントオブジェクト `e` から、対象要素（`e.target`）や座標、キー情報が取得できます
- `e.preventDefault()` でブラウザのデフォルト動作を、`e.stopPropagation()` で伝搬を止められます
- イベントは子→親へと**バブリング**で伝搬します
- **イベント委譲**は、親要素に1つのリスナーで多数の子要素を処理する効率的なパターンです
- 高頻度イベントには**デバウンス**や**スロットリング**でパフォーマンスを最適化しましょう
- アロー関数内の `this` は要素を指さないため、`e.target` / `e.currentTarget` を使いましょう
