# 第10章 監査と内部統制

> システム監査基準、IT統制、J-SOX法（内部統制報告制度）、監査手続、証拠の評価など、応用情報技術者試験の午後選択問題「システム監査」に直結する知識を体系的に学ぶ。

---

## 10.1 システム監査の基礎

### 10.1.1 システム監査とは

**システム監査** は、情報システムのリスクに対するコントロール（統制）が適切に整備・運用されているかを、独立かつ客観的な立場から検証・評価する活動です。

```
監査の3要素:
┌─────────┐
│ 独立性   │ 被監査部門から独立した立場
├─────────┤
│ 客観性   │ 事実に基づく判断
├─────────┤
│ 専門性   │ 監査に必要な知識と能力
└─────────┘
```

### 10.1.2 システム監査基準（経済産業省）

| 項目 | 内容 |
|------|------|
| 一般基準 | 監査人の資質・独立性・職業倫理 |
| 実施基準 | 監査の計画・実施方法 |
| 報告基準 | 監査報告書の作成・報告方法 |

**監査人の独立性:**
- **外観上の独立性:** 被監査部門との利害関係がないこと
- **精神上の独立性:** 公正不偏の態度で判断すること

### 10.1.3 監査の種類

| 分類基準 | 種類 | 説明 |
|---------|------|------|
| 実施主体 | 内部監査 | 組織内部の監査部門が実施 |
| | 外部監査 | 外部の監査法人等が実施 |
| 目的 | 保証型監査 | コントロールの有効性を保証 |
| | 助言型監査 | 改善のための助言を行う |

**ポイントまとめ:**
- 監査の3要素（独立性・客観性・専門性）を覚える
- 保証型と助言型の違いは午前試験で頻出
- システム監査基準の3つの基準（一般・実施・報告）を区別する

## 10.2 監査手続

### 10.2.1 監査プロセス

```
監査計画 → 予備調査 → 本調査 → 評価・結論 → 報告 → フォローアップ
```

### 10.2.2 監査計画

| 計画の種類 | 内容 |
|-----------|------|
| 中長期計画 | 複数年にわたる監査の基本方針 |
| 年度計画 | 当該年度の監査対象・スケジュール |
| 個別計画 | 個々の監査テーマの詳細計画 |

**リスクアプローチ:**
リスクの高い領域に監査資源を重点的に配分する考え方です。

```
監査リスク = 固有リスク × 統制リスク × 発見リスク

固有リスク: 統制がない場合に誤りや不正が発生するリスク
統制リスク: 内部統制が誤りを防止・発見できないリスク
発見リスク: 監査手続が誤りを発見できないリスク
```

### 10.2.3 監査技法

| 技法 | 説明 |
|------|------|
| チェックリスト法 | 事前に用意した質問項目で確認 |
| ドキュメントレビュー | 文書の閲覧・確認 |
| インタビュー | 関係者へのヒアリング |
| ウォークスルー | 業務プロセスを追跡して確認 |
| 突合・照合 | 複数の記録を突き合わせて整合性を確認 |
| 再実施 | 処理を再度実行して結果を比較 |
| 現地調査 | 実際の現場を視察 |
| コンピュータ支援監査技法（CAAT） | ソフトウェアを使ったデータ分析 |

### 10.2.4 コンピュータ支援監査技法（CAAT）

| 手法 | 説明 |
|------|------|
| テストデータ法 | 監査用テストデータをシステムに投入し、結果を検証 |
| 汎用監査ソフトウェア法 | 汎用ソフトでデータを直接分析 |
| 組込み監査モジュール法 | 監査用モジュールをシステムに組み込み、継続的に監視 |
| スナップショット法 | 処理の途中経過を記録して検証 |
| ITF法（統合テスト法） | 架空のデータをシステム内に作成し、実データと並行処理 |

**ポイントまとめ:**
- 監査リスク = 固有リスク × 統制リスク × 発見リスク を覚える
- 監査技法の種類を覚え、場面に応じた適切な技法を選択できるようにする
- CAATの各手法の違いは午前・午後共に頻出

## 10.3 監査証拠

### 10.3.1 監査証拠の種類

| 種類 | 説明 | 証拠力 |
|------|------|--------|
| 物理的証拠 | 実地調査で確認した事実 | 高い |
| 文書的証拠 | 外部から入手した文書（外部証拠） | 高い |
| | 組織内部で作成した文書（内部証拠） | 中程度 |
| 口頭的証拠 | インタビューでの回答 | 低い |
| 分析的証拠 | データ分析の結果 | 中程度 |
| 電子的証拠 | ログ、電子データ | 中〜高（改ざん防止対策次第） |

### 10.3.2 証拠の評価基準

| 基準 | 説明 |
|------|------|
| 適切性 | 監査目的に関連する証拠であるか |
| 十分性 | 結論を裏付けるのに十分な量があるか |
| 信頼性 | 証拠の情報源は信頼できるか |

```
証拠力の序列:
  外部証拠 > 内部証拠
  物理的証拠 > 文書的証拠 > 口頭的証拠
  直接入手 > 間接入手
  統制が有効な環境の証拠 > 統制が弱い環境の証拠
```

### 10.3.3 監査調書

監査調書は、監査の実施過程と結論を記録する文書です。

**監査調書に含める内容:**
- 監査の目的と範囲
- 実施した監査手続
- 収集した監査証拠
- 発見事項
- 結論と根拠

**ポイントまとめ:**
- 証拠力の序列（外部>内部、物理的>文書的>口頭的）を覚える
- 証拠は「適切性」「十分性」「信頼性」で評価する
- 監査調書は法的な証拠資料としても重要

## 10.4 監査報告とフォローアップ

### 10.4.1 監査報告書

```
監査報告書の構成:
┌───────────────────────┐
│ 監査の目的と範囲        │
├───────────────────────┤
│ 監査の実施概要          │
├───────────────────────┤
│ 保証意見 or 助言内容    │
├───────────────────────┤
│ 指摘事項               │
│  ├── 重要度の分類       │
│  └── 改善勧告          │
├───────────────────────┤
│ 全体の評価結論          │
└───────────────────────┘
```

### 10.4.2 フォローアップ

監査報告後、改善勧告が適切に実施されているかを確認する活動です。

```
監査報告 → 改善勧告 → 被監査部門が改善計画策定
                                │
                                ▼
              フォローアップ監査（改善状況の確認）
```

**ポイントまとめ:**
- 監査報告書には必ず保証意見/助言と改善勧告を含める
- フォローアップは監査プロセスの最終段階として重要

## 10.5 内部統制

### 10.5.1 内部統制の定義

内部統制とは、以下の4つの目的を達成するために組織内に整備・運用される仕組みです。

| 目的 | 説明 |
|------|------|
| 業務の有効性と効率性 | 事業目標の効果的・効率的な達成 |
| 財務報告の信頼性 | 財務諸表の正確性・適正性 |
| 法令の遵守（コンプライアンス） | 法律・規制の遵守 |
| 資産の保全 | 組織の資産の不正な取得・使用・処分の防止 |

### 10.5.2 内部統制の6つの基本的要素

```
┌─────────────────────────────────────────┐
│ ① 統制環境                               │
│    経営者の姿勢、組織文化、倫理観          │
├─────────────────────────────────────────┤
│ ② リスクの評価と対応                      │
│    リスクの識別、分析、対応策の決定         │
├─────────────────────────────────────────┤
│ ③ 統制活動                               │
│    方針・手続の策定と実施（承認、照合等）    │
├─────────────────────────────────────────┤
│ ④ 情報と伝達                             │
│    必要な情報の識別、収集、伝達             │
├─────────────────────────────────────────┤
│ ⑤ モニタリング                            │
│    内部統制の有効性を評価（日常的/独立的）   │
├─────────────────────────────────────────┤
│ ⑥ ITへの対応                              │
│    IT環境への対応、IT利用、IT統制           │
└─────────────────────────────────────────┘
```

### 10.5.3 IT統制の分類

```
IT統制
├── IT全般統制（ITGC: IT General Controls）
│   ├── システム開発・保守に関する統制
│   ├── システムの運用に関する統制
│   ├── アクセス管理に関する統制
│   └── 外部委託に関する統制
│
└── IT業務処理統制（ITAC: IT Application Controls）
    ├── 入力統制（Input Controls）
    ├── 処理統制（Processing Controls）
    └── 出力統制（Output Controls）
```

| 分類 | 説明 | 例 |
|------|------|-----|
| IT全般統制 | システム全体の管理統制 | アクセス管理、変更管理、運用管理 |
| IT業務処理統制 | 個別業務の処理の正確性 | 入力チェック、計算の正確性、帳票の網羅性 |

**IT全般統制とIT業務処理統制の関係:**
```
IT全般統制が有効 → IT業務処理統制も信頼できる
IT全般統制が不十分 → IT業務処理統制の信頼性も低下
```

**ポイントまとめ:**
- 内部統制の4つの目的と6つの基本的要素を覚える
- IT全般統制とIT業務処理統制の違いと関係を理解する
- IT全般統制は「IT業務処理統制の基盤」であることが重要

## 10.6 J-SOX（内部統制報告制度）

### 10.6.1 J-SOXとは

**J-SOX（金融商品取引法に基づく内部統制報告制度）** は、上場企業に対して財務報告に係る内部統制の評価と監査を義務付ける制度です。

```
米国: SOX法（サーベンス・オクスリー法、2002年）
日本: J-SOX（金融商品取引法、2008年施行）
```

### 10.6.2 J-SOXのプロセス

```
経営者による内部統制の評価
  │
  ├── 全社的な内部統制の評価
  │
  ├── 業務プロセスに係る内部統制の評価
  │   ├── 重要な勘定科目の特定
  │   ├── 対象プロセスの識別
  │   ├── リスクと統制の文書化
  │   │   ├── フローチャート（業務の流れ）
  │   │   ├── 業務記述書（業務の詳細）
  │   │   └── リスクコントロールマトリクス（RCM）
  │   └── 統制の有効性のテスト
  │       ├── 整備状況の評価（ウォークスルー）
  │       └── 運用状況の評価（テスト）
  │
  ▼
内部統制報告書の提出
  │
  ▼
外部監査人による内部統制監査
```

### 10.6.3 リスクコントロールマトリクス（RCM）

| リスク | 統制活動 | 統制の種類 | 頻度 | 責任者 |
|--------|---------|-----------|------|--------|
| 注文データの入力誤り | 入力データのダブルチェック | 予防的統制 | 都度 | 入力担当者 |
| 未承認の発注 | 上長承認フロー | 予防的統制 | 都度 | 管理者 |
| 在庫データの不整合 | 月次棚卸と照合 | 発見的統制 | 月次 | 経理担当 |

**統制の分類:**
| 分類 | 説明 |
|------|------|
| 予防的統制 | リスクの発生を事前に防止する統制 |
| 発見的統制 | 発生した問題を早期に検出する統制 |
| 是正的統制 | 検出した問題を修正する統制 |

### 10.6.4 3つの文書化ツール

| ツール | 内容 |
|--------|------|
| フローチャート | 業務プロセスの流れを図示 |
| 業務記述書 | 業務プロセスの詳細を文章で記述 |
| RCM（リスクコントロールマトリクス） | リスクと統制の対応を一覧表で整理 |

**ポイントまとめ:**
- J-SOXは上場企業の財務報告に係る内部統制の評価・監査制度
- 3つの文書化ツール（フローチャート、業務記述書、RCM）を覚える
- 予防的統制と発見的統制の違いは午前・午後共に頻出

## 10.7 個人情報保護と関連法規

### 10.7.1 個人情報保護法

| 概念 | 説明 |
|------|------|
| 個人情報 | 生存する個人を識別できる情報 |
| 個人データ | 個人情報データベースを構成する個人情報 |
| 保有個人データ | 事業者が開示等の権限を持つ個人データ |
| 要配慮個人情報 | 人種、信条、病歴等の機微情報 |

### 10.7.2 関連する法規・ガイドライン

| 法規等 | 概要 |
|--------|------|
| 不正アクセス禁止法 | 不正アクセス行為の禁止と処罰 |
| 不正競争防止法 | 営業秘密の保護、技術的制限手段の不正回避の禁止 |
| 著作権法 | プログラムの著作権保護 |
| 特定電子メール法 | 迷惑メールの規制 |
| マイナンバー法 | 特定個人情報の適正な取扱い |
| GDPR | EU一般データ保護規則（域外適用あり） |
