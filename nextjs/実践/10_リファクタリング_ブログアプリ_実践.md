## リファクタリング前のコード

### app/blog/page.tsx

```tsx
// 問題1: すべてがクライアントコンポーネントになっている
'use client'

import { useState, useEffect } from 'react'
import Link from 'next/link'

type Post = {
  id: number
  title: string
  body: string
  userId: number
}

export default function BlogPage() {
  const [posts, setPosts] = useState<Post[]>([])
  const [loading, setLoading] = useState(true)

  // 問題2: クライアントサイドでデータを取得している
  useEffect(() => {
    fetch('https://jsonplaceholder.typicode.com/posts')
      .then((res) => res.json())
      .then((data) => {
        setPosts(data)
        setLoading(false)
      })
  }, [])

  // 問題3: ページタイトルをクライアント側で設定している
  useEffect(() => {
    document.title = 'ブログ | MyApp'
  }, [])

  if (loading) return <p>読み込み中...</p>

  return (
    <div style={{ maxWidth: '800px', margin: '0 auto', padding: '24px' }}>
      <h1>ブログ</h1>
      <SearchablePostList posts={posts} />
    </div>
  )
}

// 問題4: 検索機能がページコンポーネントと密結合
function SearchablePostList({ posts }: { posts: Post[] }) {
  const [query, setQuery] = useState('')

  const filtered = posts.filter(
    (post) =>
      post.title.toLowerCase().includes(query.toLowerCase()) ||
      post.body.toLowerCase().includes(query.toLowerCase())
  )

  return (
    <div>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="検索..."
        style={{ padding: '8px 16px', border: '1px solid #ddd', borderRadius: '8px', width: '100%', marginBottom: '16px' }}
      />
      {filtered.map((post) => (
        <div key={post.id} style={{ borderBottom: '1px solid #eee', padding: '16px 0' }}>
          <Link href={`/blog/${post.id}`} style={{ textDecoration: 'none' }}>
            <h2 style={{ margin: '0 0 8px', color: '#1e293b' }}>{post.title}</h2>
          </Link>
          <p style={{ color: '#64748b', margin: 0 }}>
            {post.body.substring(0, 100)}...
          </p>
        </div>
      ))}
    </div>
  )
}
```

### 問題5〜8の修正：app/blog/[id]/page.tsx

```tsx
// リファクタリング後 — サーバーコンポーネント
import { Suspense } from 'react'
import type { Metadata } from 'next'
import Link from 'next/link'
import { notFound } from 'next/navigation'

type Post = {
  id: number
  title: string
  body: string
  userId: number
}

type Comment = {
  id: number
  name: string
  email: string
  body: string
}

async function getPost(id: string): Promise<Post | null> {
  const res = await fetch(
    `https://jsonplaceholder.typicode.com/posts/${id}`,
    { next: { revalidate: 300 } }
  )
  if (res.status === 404) return null
  if (!res.ok) throw new Error('記事の取得に失敗しました')
  return res.json()
}

async function getComments(postId: string): Promise<Comment[]> {
  const res = await fetch(
    `https://jsonplaceholder.typicode.com/posts/${postId}/comments`,
    { next: { revalidate: 60 } }
  )
  if (!res.ok) throw new Error('コメントの取得に失敗しました')
  return res.json()
}

// 問題8の修正: generateMetadata で動的メタデータを設定
export async function generateMetadata({
  params,
}: {
  params: Promise<{ id: string }>
}): Promise<Metadata> {
  const { id } = await params
  const post = await getPost(id)

  if (!post) {
    return { title: '記事が見つかりません' }
  }

  return {
    title: post.title,
    description: post.body.substring(0, 160),
    openGraph: {
      title: post.title,
      description: post.body.substring(0, 160),
      type: 'article',
    },
  }
}

// 問題7の修正: コメントを別コンポーネントに分離し、Suspense でストリーミング
async function CommentsSection({ postId }: { postId: string }) {
  const comments = await getComments(postId)

  return (
    <section>
      <h2>コメント ({comments.length})</h2>
      {comments.map((comment) => (
        <div
          key={comment.id}
          style={{
            padding: '12px',
            backgroundColor: '#f8fafc',
            borderRadius: '8px',
            marginBottom: '8px',
          }}
        >
          <strong>{comment.name}</strong>
          <p style={{ margin: '4px 0 0', color: '#475569' }}>{comment.body}</p>
        </div>
      ))}
    </section>
  )
}

function CommentsSkeleton() {
  return (
    <div>
      <div style={{ height: '24px', width: '150px', backgroundColor: '#e2e8f0', borderRadius: '4px', marginBottom: '16px' }} />
      {[1, 2, 3].map((i) => (
        <div key={i} style={{ padding: '12px', backgroundColor: '#f8fafc', borderRadius: '8px', marginBottom: '8px' }}>
          <div style={{ height: '16px', width: '120px', backgroundColor: '#e2e8f0', borderRadius: '4px', marginBottom: '8px' }} />
          <div style={{ height: '14px', width: '80%', backgroundColor: '#e2e8f0', borderRadius: '4px' }} />
        </div>
      ))}
    </div>
  )
}

// 問題5, 6の修正: サーバーコンポーネント + params を Promise で受け取る
export default async function PostPage({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const post = await getPost(id)

  if (!post) {
    notFound()
  }

  return (
    <div style={{ maxWidth: '800px', margin: '0 auto', padding: '24px' }}>
      <Link href="/blog" style={{ color: '#2563eb', textDecoration: 'none' }}>
        ← 一覧に戻る
      </Link>
      <article style={{ marginTop: '24px' }}>
        <h1>{post.title}</h1>
        <p style={{ lineHeight: '1.8', color: '#334155' }}>{post.body}</p>
      </article>

      {/* 問題7の修正: コメントを独立したSuspenseでラップ */}
      <Suspense fallback={<CommentsSkeleton />}>
        <CommentsSection postId={id} />
      </Suspense>
    </div>
  )
}
```

<summary>各問題の詳細解説</summary>

### 問題1: 不必要なクライアントコンポーネント

**問題**: ページ全体に `'use client'` を付けると、記事データの取得、一覧表示などすべてがクライアント側で実行されます。
**修正**: データ取得はサーバーコンポーネントで行い、インタラクティブな部分（検索）のみクライアントコンポーネントに分離します。これにより JavaScript バンドルサイズが削減され、初回表示が高速化します。

### 問題2: クライアントサイドのデータ取得

**問題**: `useEffect` でデータを取得すると、HTML→JS→データ取得の順にウォーターフォールが発生し、初回表示が遅くなります。また SEO にも不利です。
**修正**: サーバーコンポーネントの `async function` で直接 `fetch` します。サーバーでHTMLが生成されるため、初回表示が速く、SEO にも有利です。

### 問題3: document.title による手動タイトル設定

**問題**: `document.title` はクライアント側で実行されるため、SSR 時やクローラーにはタイトルが反映されません。
**修正**: Next.js の Metadata API（`export const metadata` または `generateMetadata`）を使います。サーバーサイドで `<title>` タグが生成されます。

### 問題4: コンポーネントの密結合

**問題**: `SearchablePostList` が同じファイル内の `'use client'` コンポーネントに含まれており、ページ全体がクライアントコンポーネントの影響を受けています。
**修正**: 別ファイル（`SearchablePostList.tsx`）に分離し、`'use client'` をそのファイルのみに適用します。

### 問題5: 記事詳細のクライアントコンポーネント化

**問題**: 記事の本文表示にはインタラクティブ性が不要なのに、`'use client'` でクライアントコンポーネントにしています。
**修正**: サーバーコンポーネントにしてサーバー側でデータを取得・レンダリングします。

### 問題6: params の型が古い

**問題**: Next.js 15 では `params` は `Promise<{ id: string }>` です。直接 `params.id` とアクセスするのは非推奨です。
**修正**: `await params` で値を取り出します。

### 問題7: データ取得のウォーターフォール

**問題**: 記事の取得が完了してからコメントを取得する、直列処理になっています。
**修正**: コメント取得を別の `async` サーバーコンポーネントに分離し、`Suspense` でラップすることで、記事が先に表示され、コメントは後から追加表示（ストリーミング）されます。

### 問題8: メタデータが静的

**問題**: 記事ごとのタイトルやdescriptionがHTMLの `<head>` に設定されず、SEOに不利です。
**修正**: `generateMetadata` で記事データに基づく動的メタデータを生成します。
