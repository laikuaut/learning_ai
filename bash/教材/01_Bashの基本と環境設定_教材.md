## 1.1 Bashとは

### シェルとは何か

シェル（Shell）とは、ユーザーとOS（オペレーティングシステム）のカーネル（kernel）の間に位置する**コマンドラインインタープリタ**（command-line interpreter）です。ユーザーが入力したコマンドを解釈し、カーネルに伝える役割を果たします。

```
┌─────────────────────────────────────────────┐
│                ユーザー                      │
│          （キーボード入力）                  │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│              シェル（Shell）                 │
│     コマンドを解釈し、カーネルに伝える      │
│     例: Bash, zsh, fish, sh                 │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│           カーネル（Kernel）                 │
│     ハードウェアの管理・プロセス実行        │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│          ハードウェア（Hardware）            │
│     CPU, メモリ, ディスク, ネットワーク     │
└─────────────────────────────────────────────┘
```

### Bashの歴史と特徴

Bash（**B**ourne **A**gain **Sh**ell）は、1989年にBrian Foxによって開発されたシェルです。名前は、初期のUnixシェルである「Bourne Shell（sh）」の後継（"born again" = 再生）であることに由来します。

Bashの主な特徴は以下のとおりです。

- **POSIX互換**: POSIX標準に準拠しており、移植性が高いです
- **対話的利用**: コマンド履歴、タブ補完など対話操作に便利な機能が充実しています
- **スクリプト言語**: 変数、条件分岐、ループなど、プログラミング言語としての機能を備えています
- **広い普及率**: 多くのLinuxディストリビューションでデフォルトシェルとして採用されています

### 他のシェルとの違い

| シェル | 特徴 | デフォルト採用 |
|--------|------|----------------|
| **sh (Bourne Shell)** | 最も古い標準シェル。機能は最小限 | 一部のUnix |
| **Bash** | shの上位互換。最も広く使われている | 多くのLinux |
| **zsh** | Bashと互換性が高く、補完機能が強力 | macOS (Catalina以降) |
| **fish** | ユーザーフレンドリー。POSIX非互換 | - |
| **dash** | 軽量・高速。スクリプト実行向け | Ubuntu (/bin/sh) |

```bash
# 現在使用しているシェルを確認する
echo $SHELL

# 利用可能なシェルの一覧を表示する
cat /etc/shells
```

> **よくある間違い**: `$SHELL` は「ログインシェル」を表示します。現在実行中のシェルを確認するには `echo $0` や `ps -p $$` を使います。

## 1.2 Bashのバージョン確認と環境

### バージョンの確認方法

```bash
# 方法1: --version オプション
bash --version

# 出力例:
# GNU bash, version 5.2.15(1)-release (x86_64-pc-linux-gnu)

# 方法2: 組み込み変数を使う
echo $BASH_VERSION

# 出力例:
# 5.2.15(1)-release
```

バージョンによって使える機能が異なります。特に **Bash 4.0 以降**で追加された連想配列（associative array）や、**Bash 4.4 以降**の `${var@Q}` などは、古い環境では動作しません。

### Bashのパス

```bash
# Bashの場所を確認する
which bash
# 出力例: /usr/bin/bash  または  /bin/bash

# type コマンドでも確認できる
type bash
# 出力例: bash is /usr/bin/bash
```

### シバン行（Shebang）

スクリプトの1行目に書く `#!` で始まる行を**シバン行**（shebang line）と呼びます。スクリプトをどのインタープリタで実行するかを指定します。

```bash
#!/bin/bash
# /bin/bash を直接指定する方法

#!/usr/bin/env bash
# env コマンド経由で bash を検索する方法（推奨）
# 環境によって bash の場所が異なる場合に対応できます
```

> **現場のベストプラクティス**: `#!/usr/bin/env bash` を使うのが推奨されています。`/bin/bash` のパスは環境によって異なることがありますが、`env` コマンドは `PATH` からbashを検索するため、移植性が高くなります。

### スクリプトの実行方法

```bash
# 方法1: bash コマンドで実行
bash script.sh

# 方法2: 実行権限を付与して直接実行
chmod +x script.sh
./script.sh

# 方法3: source コマンドで現在のシェルで実行
source script.sh
# または
. script.sh
```

> **注意**: `source` で実行した場合、スクリプト内で定義した変数や関数が現在のシェルに残ります。`bash script.sh` や `./script.sh` はサブシェルで実行されるため、現在のシェルには影響しません。

## 1.3 設定ファイル

### 主な設定ファイル

Bashの設定ファイルは複数あり、**シェルの起動方法**によって読み込まれるファイルが異なります。

| ファイル | 説明 |
|----------|------|
| `/etc/profile` | 全ユーザー共通のログインシェル設定 |
| `~/.bash_profile` | ユーザー個別のログインシェル設定 |
| `~/.bash_login` | `~/.bash_profile` がない場合に読まれる |
| `~/.profile` | `~/.bash_profile` も `~/.bash_login` もない場合に読まれる |
| `~/.bashrc` | 対話的な非ログインシェルの設定 |
| `~/.bash_logout` | ログインシェル終了時に実行される |

### ログインシェルと非ログインシェルの読み込み順序

```
┌──────────────────────────────────────────────────────────────┐
│               ログインシェル (Login Shell)                    │
│         例: SSH接続、コンソールログイン、su -                 │
│                                                              │
│  読み込み順序:                                               │
│  1. /etc/profile                                             │
│  2. 以下のうち最初に見つかったもの1つだけ:                    │
│     ~/.bash_profile → ~/.bash_login → ~/.profile             │
│                                                              │
│  ※ 終了時: ~/.bash_logout                                   │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│           非ログインシェル (Non-login Shell)                  │
│         例: ターミナルエミュレータ、bash コマンド             │
│                                                              │
│  読み込み順序:                                               │
│  1. ~/.bashrc のみ                                           │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│           非対話的シェル (Non-interactive Shell)              │
│         例: スクリプト実行                                    │
│                                                              │
│  読み込み:                                                   │
│  - $BASH_ENV で指定されたファイル（設定されている場合のみ）   │
└──────────────────────────────────────────────────────────────┘
```

### 判定のフローチャート

```
シェル起動
  │
  ├── ログインシェルか？
  │     │
  │     ├── YES → /etc/profile を読む
  │     │          → ~/.bash_profile or ~/.bash_login or ~/.profile を読む
  │     │
  │     └── NO → 対話的シェルか？
  │               │
  │               ├── YES → ~/.bashrc を読む
  │               │
  │               └── NO → $BASH_ENV があれば読む
  │
  └── 終了時（ログインシェルの場合のみ）
        → ~/.bash_logout を読む
```

### 現場でよく使われるパターン

多くの環境では、`~/.bash_profile` から `~/.bashrc` を読み込む構成にします。こうすることで、ログインシェルでも非ログインシェルでも同じ設定が適用されます。

```bash
# ~/.bash_profile の内容（よくあるパターン）
# ログインシェル固有の設定（環境変数など）
export PATH="$HOME/bin:$PATH"
export EDITOR=vim
export LANG=ja_JP.UTF-8

# .bashrc を読み込む
if [ -f ~/.bashrc ]; then
    source ~/.bashrc
fi
```

```bash
# ~/.bashrc の内容（よくあるパターン）
# 対話的シェルでない場合は何もしない
case $- in
    *i*) ;;
      *) return;;
esac

# エイリアスの設定
alias ll='ls -alF'
alias la='ls -A'

# プロンプトの設定
PS1='\u@\h:\w\$ '

# 履歴の設定
HISTSIZE=10000
HISTFILESIZE=20000
HISTCONTROL=ignoredups:erasedups
```

> **よくある間違い**: 環境変数（`export`）は `~/.bash_profile` に書き、エイリアスや関数は `~/.bashrc` に書くのが一般的です。`~/.bashrc` に `export` を書いても動作しますが、設定の意図が不明確になります。

## 1.4 エイリアスとプロンプトのカスタマイズ

### エイリアス（alias）

エイリアス（alias）は、長いコマンドに短い別名を付ける機能です。

```bash
# エイリアスの定義
alias ll='ls -alF'
alias la='ls -A'
alias l='ls -CF'
alias ..='cd ..'
alias ...='cd ../..'
alias grep='grep --color=auto'

# エイリアスの一覧を表示
alias

# 特定のエイリアスの内容を確認
alias ll
# 出力例: alias ll='ls -alF'

# エイリアスの削除
unalias ll

# 全エイリアスの削除
unalias -a
```

```bash
# 実務でよく使うエイリアス例

# Gitコマンドの短縮
alias gs='git status'
alias ga='git add'
alias gc='git commit'
alias gp='git push'
alias gd='git diff'
alias gl='git log --oneline --graph'

# 安全のための確認付きコマンド
alias rm='rm -i'
alias cp='cp -i'
alias mv='mv -i'

# ディスク容量を見やすく表示
alias df='df -h'
alias du='du -h'

# エイリアスを無視してオリジナルのコマンドを実行する場合
\rm file.txt       # バックスラッシュでエイリアスを無効化
command rm file.txt # command コマンドでも同様
```

> **注意**: エイリアスは現在のシェルセッションでのみ有効です。永続化するには `~/.bashrc` に記述します。

### プロンプトのカスタマイズ（PS1変数）

PS1（Primary Prompt String）変数でプロンプトの表示をカスタマイズできます。

```
┌──────────────┬──────────────────────────────────────┐
│ エスケープ   │ 説明                                  │
├──────────────┼──────────────────────────────────────┤
│  \u          │ ユーザー名                            │
│  \h          │ ホスト名（短縮形）                    │
│  \H          │ ホスト名（完全修飾）                  │
│  \w          │ カレントディレクトリ（フルパス）      │
│  \W          │ カレントディレクトリ（末尾のみ）      │
│  \d          │ 日付（"曜日 月 日" 形式）             │
│  \t          │ 時刻（24時間制 HH:MM:SS）             │
│  \T          │ 時刻（12時間制 HH:MM:SS）             │
│  \n          │ 改行                                  │
│  \$          │ rootなら#、一般ユーザーなら$          │
│  \[...\]     │ 非表示文字の囲み（色指定に使用）      │
└──────────────┴──────────────────────────────────────┘
```

```bash
# デフォルトのプロンプト
PS1='\u@\h:\w\$ '
# 表示例: user@hostname:/home/user$

# カラー付きプロンプト
PS1='\[\033[01;32m\]\u@\h\[\033[00m\]:\[\033[01;34m\]\w\[\033[00m\]\$ '
# 表示例: user@hostname:/home/user$  （ユーザー名が緑、ディレクトリが青）

# シンプルなプロンプト
PS1='\W \$ '
# 表示例: user $

# 日時入りプロンプト
PS1='[\d \t] \u@\h:\w\n\$ '
# 表示例:
# [Mon Jan 15 14:30:00] user@hostname:/home/user
# $
```

### PROMPT_COMMAND

`PROMPT_COMMAND` は、プロンプトが表示される直前に毎回実行されるコマンドを設定する変数です。

```bash
# コマンド実行後に自動で空行を入れる
PROMPT_COMMAND='echo'

# ターミナルのタイトルを動的に変更する
PROMPT_COMMAND='echo -ne "\033]0;${USER}@${HOSTNAME}: ${PWD}\007"'

# 履歴をリアルタイムで保存する（複数ターミナル間で共有）
PROMPT_COMMAND='history -a; history -c; history -r'

# 直前のコマンドの終了ステータスを表示する
PROMPT_COMMAND='echo "Exit status: $?"'
```

## 1.5 Bashの便利な機能

### タブ補完（Tab Completion）

タブキーを押すと、コマンド名やファイル名を自動補完できます。

```bash
# 例: "ls /etc/hos" まで入力してTabキーを押す
ls /etc/hos[Tab]
# → ls /etc/hostname  のように補完される

# 候補が複数ある場合はTabを2回押すと一覧が表示される
ls /etc/h[Tab][Tab]
# → host.conf  hostname  hosts  hosts.allow  hosts.deny

# コマンド名も補完できる
sys[Tab][Tab]
# → sysctl  syslog  systemctl ...
```

> **現場のベストプラクティス**: `bash-completion` パッケージをインストールすると、Git のブランチ名や systemctl のユニット名なども補完できるようになります。

### コマンド履歴（Command History）

```bash
# 履歴を表示する
history

# 直近の10件だけ表示する
history 10

# 履歴から特定のコマンドを再実行する
!42        # 履歴番号42のコマンドを実行
!!         # 直前のコマンドを再実行
!ls        # 最後に実行した ls で始まるコマンドを再実行
!?keyword  # 最後に "keyword" を含むコマンドを再実行

# 引数の再利用
!$         # 直前のコマンドの最後の引数を参照
!^         # 直前のコマンドの最初の引数を参照
!*         # 直前のコマンドの全引数を参照

# 使用例
cat /etc/hostname
vim !$
# → vim /etc/hostname が実行される

# sudo を付け忘れた場合の便利技
apt update
# → Permission denied
sudo !!
# → sudo apt update が実行される

# 履歴の検索
history | grep "git"
# git を含むコマンドの履歴を検索

# 履歴の消去
history -c    # メモリ上の履歴をクリア
history -w    # メモリの履歴をファイルに書き込み
```

### Ctrl キーショートカット

```
┌──────────────┬─────────────────────────────────────┐
│ ショートカット │ 動作                                │
├──────────────┼─────────────────────────────────────┤
│  Ctrl + R    │ 履歴をインクリメンタル検索           │
│  Ctrl + A    │ 行頭に移動                           │
│  Ctrl + E    │ 行末に移動                           │
│  Ctrl + W    │ カーソル前の単語を削除               │
│  Ctrl + U    │ カーソルから行頭まで削除             │
│  Ctrl + K    │ カーソルから行末まで削除             │
│  Ctrl + Y    │ 最後に削除したテキストを貼り付け     │
│  Ctrl + L    │ 画面クリア（clear と同じ）           │
│  Ctrl + C    │ 実行中のコマンドを中断               │
│  Ctrl + D    │ シェルを終了（入力がない場合）       │
│  Ctrl + Z    │ 実行中のコマンドを一時停止           │
│  Alt + B     │ 1単語分左に移動                      │
│  Alt + F     │ 1単語分右に移動                      │
└──────────────┴─────────────────────────────────────┘
```

### Ctrl+R による履歴検索

```bash
# Ctrl+R を押すと以下のようなプロンプトが表示されます
(reverse-i-search)`':

# 検索したい文字列を入力すると、履歴からマッチするコマンドが表示されます
(reverse-i-search)`git': git push origin main

# Enter で実行
# Ctrl+R で次の候補（さらに古い履歴）
# Ctrl+G または Ctrl+C でキャンセル
# → や ← でコマンドラインに取り込んで編集
```

### 履歴に関する設定

```bash
# ~/.bashrc に追加する設定例

# 履歴のサイズを増やす
HISTSIZE=10000          # メモリ上の履歴数
HISTFILESIZE=20000      # ファイルに保存する履歴数

# 重複するコマンドを履歴に残さない
HISTCONTROL=ignoredups:erasedups

# スペースで始まるコマンドを履歴に残さない（秘密のコマンドに便利）
HISTCONTROL=ignoreboth  # ignoredups + ignorespace

# 特定のコマンドを履歴に残さない
HISTIGNORE="ls:cd:pwd:exit:history"

# タイムスタンプを記録する
HISTTIMEFORMAT="%Y-%m-%d %H:%M:%S  "

# 履歴ファイルに即座に追記する
shopt -s histappend
```
