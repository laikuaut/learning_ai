# 第8章：継承とポリモーフィズム

> **この章のゴール**: クラスの継承（inheritance）とポリモーフィズム（polymorphism）の仕組みを理解し、基底クラス・派生クラスの設計、仮想関数によるポリモーフィックな振る舞い、多重継承の注意点を実践的に使えるようになることです。

---

## 8.1 継承の基本

### 継承とは何か

継承（inheritance）とは、既存のクラスの機能を引き継いで新しいクラスを定義する仕組みです。コードの再利用性を高め、「is-a」関係を表現します。

```
┌─────────────────────┐
│     Animal（動物）    │  ← 基底クラス（base class）
│─────────────────────│
│  - name             │
│  + speak()          │
└─────────┬───────────┘
          │ 継承
    ┌─────┴─────┐
    │           │
┌───▼───┐  ┌───▼───┐
│  Dog  │  │  Cat  │  ← 派生クラス（derived class）
│───────│  │───────│
│+speak()│ │+speak()│
└───────┘  └───────┘
```

### 基底クラスと派生クラス

基底クラス（base class）は共通の機能を定義し、派生クラス（derived class）はそれを継承して固有の機能を追加・変更します。

```cpp
#include <iostream>
#include <string>

// 基底クラス
class Animal {
protected:
    std::string name_;
public:
    Animal(const std::string& name) : name_(name) {
        std::cout << "Animal コンストラクタ: " << name_ << std::endl;
    }

    void eat() const {
        std::cout << name_ << " は食事をしています。" << std::endl;
    }

    std::string getName() const { return name_; }

    ~Animal() {
        std::cout << "Animal デストラクタ: " << name_ << std::endl;
    }
};

// 派生クラス（public 継承）
class Dog : public Animal {
public:
    Dog(const std::string& name) : Animal(name) {
        std::cout << "Dog コンストラクタ: " << name_ << std::endl;
    }

    void bark() const {
        std::cout << name_ << " : ワンワン！" << std::endl;
    }

    ~Dog() {
        std::cout << "Dog デストラクタ: " << name_ << std::endl;
    }
};

int main() {
    Dog d("ポチ");
    d.eat();   // 基底クラスのメソッドを呼べる
    d.bark();  // 派生クラス固有のメソッド
    return 0;
}
```

**実行結果：**

```
Animal コンストラクタ: ポチ
Dog コンストラクタ: ポチ
ポチ は食事をしています。
ポチ : ワンワン！
Dog デストラクタ: ポチ
Animal デストラクタ: ポチ
```

### アクセス指定子と継承

継承の際のアクセス指定子（access specifier）によって、基底クラスのメンバーの可視性が変わります。

```
┌──────────────┬────────────────┬────────────────┬────────────────┐
│ 基底クラスの │ public 継承    │ protected 継承 │ private 継承   │
│ アクセス     │                │                │                │
├──────────────┼────────────────┼────────────────┼────────────────┤
│ public       │ public         │ protected      │ private        │
│ protected    │ protected      │ protected      │ private        │
│ private      │ アクセス不可   │ アクセス不可   │ アクセス不可   │
└──────────────┴────────────────┴────────────────┴────────────────┘
```

実務では `public` 継承がほとんどです。「Dog は Animal である」という is-a 関係を自然に表現できるためです。

## 8.2 コンストラクタとデストラクタの呼び出し順序

継承関係にあるクラスでは、オブジェクトの生成・破棄時に呼ばれる順序が決まっています。

```
【構築時（コンストラクタ）】        【破棄時（デストラクタ）】
  基底クラス → 派生クラス            派生クラス → 基底クラス

  Animal() ──→ Dog()                Dog::~Dog() ──→ Animal::~Animal()
  (親が先)                          (子が先)
```

### メンバー初期化子リストによる基底クラスの初期化

派生クラスのコンストラクタでは、メンバー初期化子リスト（member initializer list）を使って基底クラスのコンストラクタを明示的に呼び出します。

```cpp
#include <iostream>
#include <string>

class Shape {
protected:
    std::string color_;
public:
    Shape(const std::string& color) : color_(color) {
        std::cout << "[Shape] 色: " << color_ << " で構築" << std::endl;
    }
    ~Shape() {
        std::cout << "[Shape] デストラクタ" << std::endl;
    }
};

class Circle : public Shape {
    double radius_;
public:
    // メンバー初期化子リストで基底クラスを初期化
    Circle(const std::string& color, double radius)
        : Shape(color), radius_(radius) {
        std::cout << "[Circle] 半径: " << radius_ << " で構築" << std::endl;
    }
    ~Circle() {
        std::cout << "[Circle] デストラクタ" << std::endl;
    }

    double area() const { return 3.14159265 * radius_ * radius_; }
};

int main() {
    std::cout << "--- 構築 ---" << std::endl;
    Circle c("赤", 5.0);
    std::cout << "面積: " << c.area() << std::endl;
    std::cout << "--- 破棄 ---" << std::endl;
    return 0;
}
```

**実行結果：**

```
--- 構築 ---
[Shape] 色: 赤 で構築
[Circle] 半径: 5 で構築
面積: 78.5398
--- 破棄 ---
[Circle] デストラクタ
[Shape] デストラクタ
```

> **よくある間違い**: 派生クラスのコンストラクタ本体の中で基底クラスのコンストラクタを呼ぼうとするのは誤りです。基底クラスの初期化は必ずメンバー初期化子リストで行います。

## 8.3 virtual 関数とオーバーライド

### 仮想関数（virtual function）

基底クラスのポインタや参照を通じて派生クラスの関数を呼び出すには、`virtual` キーワードが必要です。

```cpp
#include <iostream>
#include <string>
#include <memory>
#include <vector>

class Animal {
protected:
    std::string name_;
public:
    Animal(const std::string& name) : name_(name) {}

    // virtual を付けないと基底クラスの関数が呼ばれてしまう
    virtual void speak() const {
        std::cout << name_ << " は何か言っています。" << std::endl;
    }

    virtual ~Animal() = default;  // 仮想デストラクタ（重要！）

    std::string getName() const { return name_; }
};

class Dog : public Animal {
public:
    Dog(const std::string& name) : Animal(name) {}

    // override キーワードで明示的にオーバーライドを宣言
    void speak() const override {
        std::cout << name_ << " : ワンワン！" << std::endl;
    }
};

class Cat : public Animal {
public:
    Cat(const std::string& name) : Animal(name) {}

    void speak() const override {
        std::cout << name_ << " : ニャー！" << std::endl;
    }
};

int main() {
    // 基底クラスのポインタで派生クラスを扱う
    std::vector<std::unique_ptr<Animal>> animals;
    animals.push_back(std::make_unique<Dog>("ポチ"));
    animals.push_back(std::make_unique<Cat>("タマ"));
    animals.push_back(std::make_unique<Dog>("ハチ"));

    for (const auto& a : animals) {
        a->speak();  // 各派生クラスの speak() が呼ばれる
    }

    return 0;
}
```

**実行結果：**

```
ポチ : ワンワン！
タマ : ニャー！
ハチ : ワンワン！
```

### override キーワードの重要性

C++11 以降で導入された `override` キーワードは、基底クラスの仮想関数を正しくオーバーライドしていることをコンパイラに検証させるものです。

```cpp
class Base {
public:
    virtual void process(int value) const {}
    virtual ~Base() = default;
};

class Derived : public Base {
public:
    // override を付けると、シグネチャの間違いがコンパイルエラーになる
    // void process(double value) const override {}  // エラー！引数の型が違う
    // void process(int value) override {}            // エラー！const が抜けている

    void process(int value) const override {}  // 正しい
};
```

> **よくある間違い**: `override` を付けずにオーバーライドしようとして、引数の型や const の有無を間違えると、新しい関数が定義されてしまい、意図したポリモーフィズムが機能しません。**常に `override` を付ける** 習慣をつけましょう。

### 仮想デストラクタ

基底クラスのポインタで `delete` する場合、仮想デストラクタ（virtual destructor）がないと派生クラスのデストラクタが呼ばれず、リソースリークが発生します。

```
【仮想デストラクタなしの場合】          【仮想デストラクタありの場合】

Animal* p = new Dog("ポチ");           Animal* p = new Dog("ポチ");
delete p;                              delete p;

→ ~Animal() のみ実行（危険！）         → ~Dog() → ~Animal() 正しく実行
  Dog のリソースが解放されない
```

> **鉄則**: 継承される可能性のあるクラスには `virtual ~ClassName() = default;` を宣言してください。

## 8.4 純粋仮想関数と抽象クラス

### 純粋仮想関数（pure virtual function）

関数の宣言に `= 0` を付けると、純粋仮想関数（pure virtual function）になります。純粋仮想関数を1つ以上持つクラスは抽象クラス（abstract class）となり、直接インスタンス化できません。

```cpp
#include <iostream>
#include <string>
#include <memory>
#include <vector>
#include <cmath>

// 抽象クラス（インターフェースとして機能）
class Shape {
public:
    virtual double area() const = 0;           // 純粋仮想関数
    virtual double perimeter() const = 0;      // 純粋仮想関数
    virtual std::string typeName() const = 0;  // 純粋仮想関数

    void describe() const {
        std::cout << typeName()
                  << " - 面積: " << area()
                  << ", 周長: " << perimeter() << std::endl;
    }

    virtual ~Shape() = default;
};

class Rectangle : public Shape {
    double width_, height_;
public:
    Rectangle(double w, double h) : width_(w), height_(h) {}

    double area() const override { return width_ * height_; }
    double perimeter() const override { return 2 * (width_ + height_); }
    std::string typeName() const override { return "長方形"; }
};

class Circle : public Shape {
    double radius_;
    static constexpr double PI = 3.14159265358979;
public:
    Circle(double r) : radius_(r) {}

    double area() const override { return PI * radius_ * radius_; }
    double perimeter() const override { return 2 * PI * radius_; }
    std::string typeName() const override { return "円"; }
};

class Triangle : public Shape {
    double a_, b_, c_;  // 三辺の長さ
public:
    Triangle(double a, double b, double c) : a_(a), b_(b), c_(c) {}

    double area() const override {
        // ヘロンの公式
        double s = (a_ + b_ + c_) / 2.0;
        return std::sqrt(s * (s - a_) * (s - b_) * (s - c_));
    }
    double perimeter() const override { return a_ + b_ + c_; }
    std::string typeName() const override { return "三角形"; }
};

int main() {
    // Shape s;  // コンパイルエラー！抽象クラスはインスタンス化できない

    std::vector<std::unique_ptr<Shape>> shapes;
    shapes.push_back(std::make_unique<Rectangle>(4.0, 5.0));
    shapes.push_back(std::make_unique<Circle>(3.0));
    shapes.push_back(std::make_unique<Triangle>(3.0, 4.0, 5.0));

    for (const auto& s : shapes) {
        s->describe();
    }

    return 0;
}
```

**実行結果：**

```
長方形 - 面積: 20, 周長: 18
円 - 面積: 28.2743, 周長: 18.8496
三角形 - 面積: 6, 周長: 12
```

### 抽象クラスの設計パターン

```
┌──────────────────────────┐
│   Shape（抽象クラス）      │
│──────────────────────────│
│ + area() = 0       [純粋] │
│ + perimeter() = 0  [純粋] │
│ + typeName() = 0   [純粋] │
│ + describe()     [具象]   │  ← 共通処理はここに実装
└──────────┬───────────────┘
           │
   ┌───────┼───────┐
   │       │       │
┌──▼──┐ ┌─▼──┐ ┌──▼────┐
│Rect │ │ ○  │ │ △    │
│angle│ │Circ│ │Triang│
│     │ │le  │ │le    │
└─────┘ └────┘ └──────┘
  すべて area(), perimeter(), typeName() を実装
```

## 8.5 ポリモーフィズムの実践

ポリモーフィズム（polymorphism）とは、「同じインターフェースで異なる振る舞いをする」仕組みです。基底クラスのポインタや参照を通じて、実際のオブジェクトの型に応じた関数が呼ばれます。

### 実践例：ログ出力システム

```cpp
#include <iostream>
#include <string>
#include <memory>
#include <vector>
#include <fstream>
#include <ctime>

// ログ出力の抽象基底クラス
class Logger {
protected:
    std::string formatMessage(const std::string& level,
                              const std::string& msg) const {
        std::time_t now = std::time(nullptr);
        char buf[20];
        std::strftime(buf, sizeof(buf), "%Y-%m-%d %H:%M:%S",
                      std::localtime(&now));
        return std::string(buf) + " [" + level + "] " + msg;
    }

public:
    virtual void log(const std::string& level,
                     const std::string& message) = 0;
    virtual ~Logger() = default;
};

// コンソールに出力するロガー
class ConsoleLogger : public Logger {
public:
    void log(const std::string& level,
             const std::string& message) override {
        std::cout << formatMessage(level, message) << std::endl;
    }
};

// ファイルに出力するロガー
class FileLogger : public Logger {
    std::string filename_;
public:
    FileLogger(const std::string& filename) : filename_(filename) {}

    void log(const std::string& level,
             const std::string& message) override {
        std::ofstream ofs(filename_, std::ios::app);
        if (ofs) {
            ofs << formatMessage(level, message) << std::endl;
        }
    }
};

// 複数のロガーをまとめるロガー（Compositeパターン）
class MultiLogger : public Logger {
    std::vector<std::unique_ptr<Logger>> loggers_;
public:
    void addLogger(std::unique_ptr<Logger> logger) {
        loggers_.push_back(std::move(logger));
    }

    void log(const std::string& level,
             const std::string& message) override {
        for (auto& logger : loggers_) {
            logger->log(level, message);
        }
    }
};

int main() {
    MultiLogger multi;
    multi.addLogger(std::make_unique<ConsoleLogger>());
    multi.addLogger(std::make_unique<FileLogger>("app.log"));

    // どちらのロガーにも同じインターフェースで書き込める
    multi.log("INFO", "アプリケーションが起動しました");
    multi.log("WARN", "メモリ使用量が80%を超えています");
    multi.log("ERROR", "データベース接続に失敗しました");

    std::cout << "\n--- app.log の内容 ---" << std::endl;
    std::ifstream ifs("app.log");
    std::string line;
    while (std::getline(ifs, line)) {
        std::cout << line << std::endl;
    }

    return 0;
}
```

このように、`Logger` という共通インターフェースを通じて、実際の出力先（コンソール・ファイル）が異なる実装を透過的に扱えるのがポリモーフィズムの力です。

## 8.6 多重継承

C++ では複数のクラスから同時に継承する多重継承（multiple inheritance）が可能です。ただし、設計上の注意が必要です。

### 多重継承の基本

```cpp
#include <iostream>
#include <string>

class Printable {
public:
    virtual void print() const = 0;
    virtual ~Printable() = default;
};

class Serializable {
public:
    virtual std::string serialize() const = 0;
    virtual ~Serializable() = default;
};

// 2つのインターフェースを多重継承
class UserProfile : public Printable, public Serializable {
    std::string name_;
    int age_;
public:
    UserProfile(const std::string& name, int age)
        : name_(name), age_(age) {}

    void print() const override {
        std::cout << "名前: " << name_ << ", 年齢: " << age_ << std::endl;
    }

    std::string serialize() const override {
        return "{\"name\":\"" + name_ + "\",\"age\":"
               + std::to_string(age_) + "}";
    }
};

int main() {
    UserProfile user("田中太郎", 30);

    // Printable として扱う
    Printable& p = user;
    p.print();

    // Serializable として扱う
    Serializable& s = user;
    std::cout << s.serialize() << std::endl;

    return 0;
}
```

**実行結果：**

```
名前: 田中太郎, 年齢: 30
{"name":"田中太郎","age":30}
```

### ダイヤモンド継承問題と仮想継承

多重継承では、同じ基底クラスが2つの経路から継承される「ダイヤモンド継承」問題が発生することがあります。

```
      ┌─────────┐
      │  Base   │
      └────┬────┘
       ┌───┴───┐
       │       │
   ┌───▼──┐ ┌─▼────┐
   │Left  │ │Right │
   └───┬──┘ └──┬───┘
       │       │
       └───┬───┘
       ┌───▼───┐
       │Bottom │  ← Base のメンバーが2つ存在してしまう！
       └───────┘
```

仮想継承（virtual inheritance）でこの問題を解決できます。

```cpp
#include <iostream>

class Base {
public:
    int value = 0;
    Base() { std::cout << "Base コンストラクタ" << std::endl; }
};

// virtual を付けて仮想継承する
class Left : virtual public Base {
public:
    Left() { std::cout << "Left コンストラクタ" << std::endl; }
};

class Right : virtual public Base {
public:
    Right() { std::cout << "Right コンストラクタ" << std::endl; }
};

class Bottom : public Left, public Right {
public:
    Bottom() { std::cout << "Bottom コンストラクタ" << std::endl; }
};

int main() {
    Bottom b;
    b.value = 42;  // 仮想継承なら曖昧さなし（Base は1つだけ）
    std::cout << "value = " << b.value << std::endl;
    return 0;
}
```

**実行結果：**

```
Base コンストラクタ
Left コンストラクタ
Right コンストラクタ
Bottom コンストラクタ
value = 42
```

> **よくある間違い**: 多重継承を多用すると設計が複雑になりがちです。実務では **インターフェース（純粋仮想関数のみのクラス）の多重継承** に留め、実装を持つクラスの多重継承は避けるのがベストプラクティスです。

## 8.7 dynamic_cast による安全なダウンキャスト

基底クラスのポインタから派生クラスのポインタへの変換（ダウンキャスト）には `dynamic_cast` を使います。実行時に型チェックが行われ、安全に変換できます。

```cpp
#include <iostream>
#include <string>
#include <memory>
#include <vector>

class Animal {
public:
    virtual void speak() const = 0;
    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    void speak() const override {
        std::cout << "ワンワン！" << std::endl;
    }
    void fetch() const {
        std::cout << "ボールを取ってきました！" << std::endl;
    }
};

class Cat : public Animal {
public:
    void speak() const override {
        std::cout << "ニャー！" << std::endl;
    }
    void purr() const {
        std::cout << "ゴロゴロ..." << std::endl;
    }
};

int main() {
    std::vector<std::unique_ptr<Animal>> animals;
    animals.push_back(std::make_unique<Dog>());
    animals.push_back(std::make_unique<Cat>());
    animals.push_back(std::make_unique<Dog>());

    for (const auto& a : animals) {
        a->speak();

        // ポインタに対する dynamic_cast（失敗時は nullptr）
        if (Dog* dog = dynamic_cast<Dog*>(a.get())) {
            dog->fetch();
        }
        if (Cat* cat = dynamic_cast<Cat*>(a.get())) {
            cat->purr();
        }
    }

    // 参照に対する dynamic_cast（失敗時は std::bad_cast 例外）
    try {
        Animal& ref = *animals[1];  // Cat
        Dog& dogRef = dynamic_cast<Dog&>(ref);  // 失敗！
        dogRef.fetch();
    } catch (const std::bad_cast& e) {
        std::cout << "キャスト失敗: " << e.what() << std::endl;
    }

    return 0;
}
```

**実行結果：**

```
ワンワン！
ボールを取ってきました！
ニャー！
ゴロゴロ...
ワンワン！
ボールを取ってきました！
キャスト失敗: std::bad_cast
```

### dynamic_cast の使い分け

```
┌──────────────────────────────────────────────────────────┐
│                dynamic_cast の使い分け                     │
├───────────────┬──────────────────────────────────────────┤
│ ポインタ版     │ 失敗すると nullptr を返す                  │
│               │ → if 文でチェック可能                      │
├───────────────┼──────────────────────────────────────────┤
│ 参照版        │ 失敗すると std::bad_cast 例外を投げる      │
│               │ → try-catch でハンドリング                 │
├───────────────┼──────────────────────────────────────────┤
│ 前提条件       │ 基底クラスに1つ以上の virtual 関数が必要   │
└───────────────┴──────────────────────────────────────────┘
```

> **よくある間違い**: `dynamic_cast` を頻繁に使う設計は、ポリモーフィズムが適切に活用できていないサインです。まずは仮想関数で解決できないか検討し、`dynamic_cast` は最後の手段として使いましょう。

## 8.8 ポイントまとめ

### 継承の基本

| 概念 | 説明 |
|------|------|
| 基底クラス（base class） | 共通機能を定義する親クラス |
| 派生クラス（derived class） | 基底クラスを継承した子クラス |
| public 継承 | is-a 関係を表す最も一般的な継承 |
| protected メンバー | 派生クラスからアクセス可能、外部からは不可 |

### 仮想関数とポリモーフィズム

| 概念 | 説明 |
|------|------|
| `virtual` | 派生クラスでオーバーライド可能にするキーワード |
| `override` | オーバーライドを明示し、コンパイル時に検証するキーワード |
| 純粋仮想関数（`= 0`） | 派生クラスで必ず実装すべき関数 |
| 抽象クラス | 純粋仮想関数を持ち、インスタンス化できないクラス |
| 仮想デストラクタ | 基底クラスポインタで正しく破棄するために必須 |

### 多重継承と型変換

| 概念 | 説明 |
|------|------|
| 多重継承 | 複数のクラスを同時に継承する機能 |
| ダイヤモンド継承 | 同じ基底クラスが重複する問題 |
| 仮想継承（`virtual`） | ダイヤモンド継承の重複を解決する手法 |
| `dynamic_cast` | 実行時型チェック付きの安全なダウンキャスト |

### 設計の指針

1. **継承よりもコンポジション（composition）を優先する** — 継承は強い結合を生むため、委譲で済む場合はそちらを選ぶ
2. **インターフェース（純粋仮想関数のみ）の継承を基本にする** — 実装の多重継承は複雑さの元
3. **基底クラスには仮想デストラクタを必ず宣言する** — リソースリーク防止
4. **`override` を常に付ける** — シグネチャの不一致を早期に検出
5. **`dynamic_cast` の多用は設計を見直すサイン** — ポリモーフィズムで解決できないか再考する
