# 実践課題11：リファクタリング ─ 天気予報アプリ ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第4章（サービス）、第5章（カスタムディレクティブ）、第8章（実践パターンとベストプラクティス）
> **課題の種類**: リファクタリング
> **学習目標**: 巨大な単一コントローラに詰め込まれたコードを、サービス分離・カスタムディレクティブ（または `.component()`）への分割・Controller As 構文への書き換えで段階的に改善する

---

## 課題の説明

以下の「天気予報アプリ」は**正しく動作しますが**、すべてのロジックとUIが1つのコントローラに詰まっています。段階的にリファクタリングしてください。

### 問題点

1. `WeatherCtrl` が200行以上で肥大化している
2. データ取得ロジックとUI表示ロジックが分離されていない
3. `$scope` を直接使っており、スコープ継承の問題が起きやすい
4. 再利用可能なUI部品（天気カード、お気に入りリスト）がコンポーネント化されていない
5. 温度の単位変換ロジックが至る所に散在している

### ゴール

1. **ステップ1**: `Controller As` 構文に書き換える
2. **ステップ2**: データ取得ロジックを `WeatherService` に切り出す
3. **ステップ3**: 単位変換を `TemperatureFilter` に切り出す
4. **ステップ4**: 天気カードを `.component()` または `directive` に分割する

## ステップ別リファクタリングガイド

<details>
<summary>ステップ1：Controller As 構文への書き換え</summary>

**作業内容:**
- コントローラ内の `$scope` を `var vm = this` に変更
- テンプレートで `ng-controller="WeatherCtrl as vm"` を指定
- テンプレートの変数参照に `vm.` プレフィックスを追加

**注意点:**
- `$timeout` は引き続き注入が必要です（`$scope` を使わないだけ）
- `ng-keypress` 内の式も `vm.searchCity()` に変更します

</details>

<details>
<summary>ステップ2：WeatherService への分離</summary>

**作業内容:**
- 天気データの定義・検索ロジックを `WeatherService` factory に移す
- 非同期処理は `$q` を使ってPromiseを返す

```javascript
app.factory('WeatherService', ['$q', '$timeout', function($q, $timeout) {
  var weatherDB = { /* ... */ };
  return {
    fetch: function(city) {
      var deferred = $q.defer();
      $timeout(function() {
        if (weatherDB[city]) {
          deferred.resolve({ current: ..., forecast: ... });
        } else {
          deferred.reject('都市が見つかりません');
        }
      }, 800);
      return deferred.promise;
    }
  };
}]);
```

</details>

<details>
<summary>ステップ3：温度フィルターの作成</summary>

**作業内容:**
- `convertToF` をカスタムフィルター `temperature` に置き換え

```javascript
app.filter('temperature', function() {
  return function(celsius, unit) {
    if (unit === 'fahrenheit') {
      return Math.round(celsius * 9 / 5 + 32) + '℉';
    }
    return celsius + '℃';
  };
});
```

テンプレートで:
```html
{{ weather.temp | temperature:vm.unit }}
```

</details>

<details>
<summary>ステップ4：コンポーネントへの分割</summary>

**作業内容:**
- 天気カードを `<weather-card>` コンポーネントに

- お気に入りリストを `<favorite-list>` コンポーネントに

        '大阪': { temp: 24, condition: '曇り', humidity: 65, wind: 2 },

            vm.forecast = data.forecast;

        if (i >= 0) vm.favorites.splice(i, 1);
