# 第1章：基数変換と数値表現

## 学習目標
- 2進数・8進数・10進数・16進数の相互変換ができる
- 補数表現による負の数の表現を理解する
- 浮動小数点数の仕組みを理解する
- 各種誤差の種類と発生原因を説明できる

---

## 1.1 基数（n進数）の基本

### 基数とは
基数とは、その数値体系で使用する数字の種類の数である。

| 進数 | 基数 | 使用する数字 | 用途 |
|------|------|-------------|------|
| 2進数 | 2 | 0, 1 | コンピュータ内部処理 |
| 8進数 | 8 | 0〜7 | ファイルパーミッション等 |
| 10進数 | 10 | 0〜9 | 日常生活 |
| 16進数 | 16 | 0〜9, A〜F | メモリアドレス、色コード |

### 表記法
- 2進数：(1010)₂ または 0b1010
- 8進数：(12)₈ または 012
- 16進数：(A)₁₆ または 0x0A

---

## 1.2 基数変換

### n進数 → 10進数（各桁の重みを足す）

**公式：** 各桁 × 基数^(桁の位置)

**例：(1011)₂ → 10進数**
```
1×2³ + 0×2² + 1×2¹ + 1×2⁰
= 8 + 0 + 2 + 1
= 11
```

**例：(2F)₁₆ → 10進数**
```
2×16¹ + F(=15)×16⁰
= 32 + 15
= 47
```

### 10進数 → n進数（繰り返し割り算）

**例：47 → 2進数**
```
47 ÷ 2 = 23 余り 1  ↑
23 ÷ 2 = 11 余り 1  ↑
11 ÷ 2 = 5  余り 1  ↑
 5 ÷ 2 = 2  余り 1  ↑
 2 ÷ 2 = 1  余り 0  ↑
 1 ÷ 2 = 0  余り 1  ↑（読む方向）
→ (101111)₂
```

### 小数部の変換（繰り返し掛け算）

**例：0.625 → 2進数**
```
0.625 × 2 = 1.25  → 整数部 1 ↓
0.25  × 2 = 0.5   → 整数部 0 ↓
0.5   × 2 = 1.0   → 整数部 1 ↓（読む方向）
→ (0.101)₂
```

### 2進数 ↔ 8進数（3桁ずつ変換）

```
2進数:  101 | 011 | 110
8進数:   5  |  3  |  6
→ (101011110)₂ = (536)₈
```

### 2進数 ↔ 16進数（4桁ずつ変換）

```
2進数:  1010 | 1111
16進数:   A  |   F
→ (10101111)₂ = (AF)₁₆
```

> **試験頻出ポイント：** 2進数↔16進数の変換は特に頻出。4桁ずつ区切る方法を確実にマスターすること。

## 1.3 補数表現

### 補数とは
補数は、減算を加算で行うために使われる概念である。

### 1の補数と2の補数

| 種類 | 求め方 | 例（8ビットで-5を表現） |
|------|--------|----------------------|
| 1の補数 | 全ビットを反転 | 5=(00000101)₂ → (11111010)₂ |
| 2の補数 | 1の補数+1 | (11111010)₂ + 1 = (11111011)₂ |

### 2の補数による負の数の表現（8ビットの場合）

| 10進数 | 2進数（8ビット） |
|--------|-----------------|
| +127 | 01111111 |
| +1 | 00000001 |
| 0 | 00000000 |
| -1 | 11111111 |
| -128 | 10000000 |

**重要な性質：**
- 最上位ビット(MSB)が0なら正、1なら負
- nビットで表現できる範囲：**-2^(n-1) 〜 2^(n-1)-1**
- 8ビットなら：-128 〜 +127

### 2の補数での計算例

**7 + (-3) を8ビットで計算：**
```
  00000111  （+7）
+ 11111101  （-3 の2の補数）
----------
  00000100  （+4）  ※最上位からの桁上がりは無視
```

## 1.4 固定小数点数と浮動小数点数

### 固定小数点数
小数点の位置が固定されている表現方法。

### 浮動小数点数（IEEE 754）

**形式：** (-1)^s × 1.M × 2^(E-バイアス)

| 精度 | 全体 | 符号(s) | 指数部(E) | 仮数部(M) | バイアス |
|------|------|---------|----------|----------|---------|
| 単精度 | 32ビット | 1ビット | 8ビット | 23ビット | 127 |
| 倍精度 | 64ビット | 1ビット | 11ビット | 52ビット | 1023 |

**例：-6.5 を単精度浮動小数点で表現**
```
1. 符号：負なので s = 1
2. 絶対値を2進数に：6.5 = 110.1
3. 正規化：1.101 × 2²
4. 指数部：2 + 127(バイアス) = 129 = 10000001
5. 仮数部：101000...0（「1.」は省略＝ケチ表現）

結果：1 10000001 10100000000000000000000
```

## 1.5 誤差の種類

### 丸め誤差（まるめごさ）
- **原因：** 有限桁数で表現するために切り捨て・切り上げ・四捨五入が発生
- **例：** 1/3 = 0.333... → 0.333 で打ち切り

### 桁落ち（けたおち）
- **原因：** ほぼ等しい2つの数の減算で、有効数字が大幅に減少
- **例：** 1.23456 - 1.23451 = 0.00005（有効数字が6桁→1桁に）

### 情報落ち（じょうほうおち）
- **原因：** 絶対値が大きく異なる2つの数の加減算で、小さい方の情報が失われる
- **例：** 1.0 × 10^10 + 1.0 → 小さい方の1.0が反映されない

### 桁あふれ（オーバーフロー）
- **原因：** 計算結果が表現可能な最大値を超える
- **例：** 8ビット符号なし整数で 255 + 1 → オーバーフロー

### アンダーフロー
- **原因：** 計算結果が表現可能な最小値（0に最も近い値）より小さくなる
- **例：** 浮動小数点数で非常に小さい数同士の乗算

### 打切り誤差
- **原因：** 無限級数の計算を有限回で打ち切ることで生じる
- **例：** e^x = 1 + x + x²/2! + ... の計算を途中で止める

## 1.6 シフト演算

### 論理シフト
- 空いたビットに0を補充
- 符号を考慮しない

### 算術シフト
- 符号ビットを保持
- 左シフト：空いたビットに0を補充
- 右シフト：空いたビットに符号ビットと同じ値を補充

**重要：**
- 左に1ビットシフト = ×2
- 右に1ビットシフト = ÷2

**例：(00001100)₂ = 12 を左に2ビット論理シフト**
```
00001100 → 00110000 = 48（= 12 × 2² = 12 × 4）
```

## 1.7 BCD（2進化10進数）

10進数の各桁を4ビットの2進数で表す方式。

**例：** 259 → 0010 0101 1001（BCD）

| 10進 | BCD |
|------|-----|
| 0 | 0000 |
| 1 | 0001 |
| ... | ... |
| 9 | 1001 |

## まとめ

| 項目 | 重要ポイント |
|------|-------------|
| 基数変換 | 10進→n進は割り算、n進→10進は重みの足し算 |
| 2進↔16進 | 4桁ずつ区切って変換 |
| 2の補数 | ビット反転+1、nビットで -2^(n-1)〜2^(n-1)-1 |
| 浮動小数点 | IEEE754、バイアス、ケチ表現を理解 |
| 誤差 | 桁落ち＝近い値の引き算、情報落ち＝大小差の足し算 |
| シフト演算 | 左1ビット＝×2、右1ビット＝÷2 |
