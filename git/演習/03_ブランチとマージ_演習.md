# 第3章 演習：ブランチとマージ

---

## 基本問題

### 問題1: ブランチの作成・切り替え・削除

練習用リポジトリで以下を実行してください。

1. `feature/login` ブランチを作成して切り替える
2. `git branch` で現在のブランチを確認する
3. `main` に戻り、`feature/login` を削除する

期待される出力例（手順2）：
```
* feature/login
  main
```

<details>
<summary>ヒント</summary>

`git switch -c <名前>` で作成と切り替えを同時に行えます。削除は `git branch -d` です。
</details>

<details>
<summary>解答</summary>

```bash
$ mkdir practice && cd practice && git init -b main
$ echo "# Project" > README.md && git add README.md && git commit -m "Initial commit"

$ git switch -c feature/login    # 作成＋切り替え
$ git branch                     # * が現在のブランチ
* feature/login
  main

$ git switch main
$ git branch -d feature/login
Deleted branch feature/login (was abc1234).
```

- `-d` は安全な削除（マージ済みのみ）、`-D` は強制削除です
- ブランチ名に `/` を使うと階層的に整理できます（`feature/`, `fix/` など）
</details>

### 問題2: Fast-forward マージ

`main` で初期コミット後、`feature/header` ブランチで `header.html` を追加し、`main` にマージしてください。

期待される出力例（`git log --oneline`）：
```
def5678 (HEAD -> main, feature/header) feat: ヘッダーを追加
abc1234 Initial commit
```

<details>
<summary>ヒント</summary>

main に分岐後の新コミットがなければ、ポインタを進めるだけの Fast-forward マージになります。
</details>

<details>
<summary>解答</summary>

```bash
$ git switch -c feature/header
$ echo "<header>My Site</header>" > header.html
$ git add header.html && git commit -m "feat: ヘッダーを追加"
$ git switch main
$ git merge feature/header     # Fast-forward（マージコミットなし）
```

- `git merge --no-ff` でマージコミットを強制的に作ることもできます
</details>

### 問題3: コンフリクトの発生と解決

同じファイルを2つのブランチで変更し、コンフリクトを発生・解決してください。

1. `main` で `message.txt`（内容:「おはよう」）をコミット
2. `feature/greeting` で「こんにちは」に、`main` で「こんばんは」にそれぞれ変更
3. マージしてコンフリクトを解決する

期待される出力例：
```
CONFLICT (content): Merge conflict in message.txt
```

<details>
<summary>ヒント</summary>

マーカー `<<<<<<<`, `=======`, `>>>>>>>` を削除し、`git add` → `git commit` します。
</details>

<details>
<summary>解答</summary>

```bash
$ echo "おはよう" > message.txt
$ git add message.txt && git commit -m "挨拶を追加"
$ git switch -c feature/greeting
$ echo "こんにちは" > message.txt && git add message.txt && git commit -m "昼に変更"
$ git switch main
$ echo "こんばんは" > message.txt && git add message.txt && git commit -m "夜に変更"

$ git merge feature/greeting   # → CONFLICT 発生
$ cat message.txt              # マーカーを確認
<<<<<<< HEAD
こんばんは
=======
こんにちは
>>>>>>> feature/greeting

$ echo "こんにちは / こんばんは" > message.txt   # 手動で解決
$ git add message.txt && git commit -m "merge: 挨拶を統合"
```

- `git merge --abort` でマージを中止して元に戻すこともできます
</details>

## 応用問題

### 問題4: rebase で履歴を直線に整理する

1. `feature/about` で `about.txt` を追加、`main` で `index.html` を追加（分岐状態）
2. `feature/about` で `git rebase main` → `main` でマージ（Fast-forward になる）

期待される出力例（`git log --oneline --graph`）：
```
* ddd (HEAD -> main, feature/about) feat: aboutを追加
* bbb feat: indexを追加
* aaa Initial commit
```

<details>
<summary>ヒント</summary>

`git rebase main` は現在のブランチのコミットを main の先端に載せ替えます。
</details>

<details>
<summary>解答</summary>

```bash
$ git switch -c feature/about
$ echo "About" > about.txt && git add about.txt && git commit -m "feat: aboutを追加"
$ git switch main
$ echo "<html>Welcome</html>" > index.html && git add index.html && git commit -m "feat: indexを追加"

$ git switch feature/about && git rebase main   # 載せ替え
$ git switch main && git merge feature/about     # Fast-forward マージ
```

- rebase するとコミットハッシュが変わります
- **push 済みブランチの rebase は厳禁**（他開発者の履歴を壊します）
</details>

### 問題5: 複数ブランチの並行開発

`feature/login` と `feature/dashboard`（両方 main から分岐）を作成し、`main` に `--no-ff` でマージしてください。

期待される出力例（`git log --oneline --graph`）：
```
*   fff Merge branch 'feature/dashboard'
|\
| * ddd feat: ダッシュボードを追加
* |   eee Merge branch 'feature/login'
|\ \
| |/
| * ccc feat: ログイン機能を追加
|/
* aaa Initial commit
```

<details>
<summary>ヒント</summary>

`--no-ff` でマージコミットを必ず作成すると、ブランチの履歴が残ります。
</details>

<details>
<summary>解答</summary>

```bash
$ git switch -c feature/login
$ echo 'def login(): pass' > login.py
$ git add login.py && git commit -m "feat: ログイン機能を追加"

$ git switch main && git switch -c feature/dashboard
$ echo 'def dashboard(): pass' > dashboard.py
$ git add dashboard.py && git commit -m "feat: ダッシュボードを追加"

$ git switch main
$ git merge --no-ff feature/login -m "Merge branch 'feature/login'"
$ git merge --no-ff feature/dashboard -m "Merge branch 'feature/dashboard'"
```

- `--no-ff` により「どのブランチで開発されたか」が履歴に残ります
</details>

## チャレンジ問題

### 問題6: Git Flow 風ワークフロー

`main`(v1.0) → `develop` → 2つの feature ブランチ → `develop` → `main`(v1.1) の流れを再現してください。

期待される出力例（`git tag`）：
```
v1.0
v1.1
```

<details>
<summary>ヒント</summary>

feature は必ず develop から分岐し develop にマージします。すべて `--no-ff` を使いましょう。
</details>

<details>
<summary>解答</summary>

```bash
$ mkdir gitflow && cd gitflow && git init -b main
$ echo "# App" > README.md && git add README.md && git commit -m "v1.0" && git tag v1.0
$ git switch -c develop
$ echo "dev" > CHANGELOG.md && git add CHANGELOG.md && git commit -m "develop開始"

$ git switch -c feature/search
$ echo 'def search(): pass' > search.py
$ git add search.py && git commit -m "feat: 検索機能を追加"
$ git switch develop && git merge --no-ff feature/search -m "Merge feature/search"

$ git switch -c feature/filter
$ echo 'def filter(): pass' > filter.py
$ git add filter.py && git commit -m "feat: フィルター機能を追加"
$ git switch develop && git merge --no-ff feature/filter -m "Merge feature/filter"

$ git switch main && git merge --no-ff develop -m "v1.1リリース" && git tag v1.1
```

- Git Flow: `main`（本番）→ `develop`（統合）→ `feature/*`（機能）の3層構造です
</details>
