# 第3章：データ構造

## 学習目標
- 主要なデータ構造の特徴と用途を理解する
- 各データ構造の操作（追加・削除・探索）の計算量を把握する
- 木構造とグラフの基本概念を理解する
- ハッシュの仕組みと衝突解決法を説明できる

---

## 3.1 配列（Array）

### 特徴
- 同じ型のデータを連続したメモリ領域に格納
- インデックス（添字）でO(1)のランダムアクセスが可能
- サイズは固定（静的配列の場合）

### 1次元配列のアドレス計算

**先頭アドレスが a、要素サイズが s のとき：**
```
要素 A[i] のアドレス = a + i × s
```
※ 添字が0始まりの場合

**例：** 先頭アドレス 1000、要素サイズ 4バイト
```
A[0]: 1000
A[1]: 1004
A[5]: 1000 + 5 × 4 = 1020
```

### 2次元配列のアドレス計算

**行優先（row-major）：** A[i][j] のアドレス = a + (i × 列数 + j) × s

**列優先（column-major）：** A[i][j] のアドレス = a + (j × 行数 + i) × s

**例：** A[3][4]（3行4列）、先頭1000、要素サイズ2バイト、行優先
```
A[2][1] = 1000 + (2×4 + 1) × 2 = 1000 + 18 = 1018
```

## 3.2 リスト（Linked List）

### 単方向リスト（Singly Linked List）
```
[データ|次→] → [データ|次→] → [データ|NULL]
```
- 各ノードはデータと次のノードへのポインタを持つ
- 先頭への挿入・削除：O(1)
- 探索：O(n)

### 双方向リスト（Doubly Linked List）
```
NULL ← [←前|データ|次→] ⇄ [←前|データ|次→] ⇄ [←前|データ|次→] → NULL
```
- 前後両方向にたどれる
- 任意位置の挿入・削除が容易

### 循環リスト（Circular List）
- 最後のノードが先頭のノードを指す
- 終端がなく、リングバッファなどに利用

### 配列 vs リスト

| 操作 | 配列 | リスト |
|------|------|--------|
| ランダムアクセス | O(1) | O(n) |
| 先頭への挿入 | O(n) | O(1) |
| 末尾への挿入 | O(1)※ | O(n)※ |
| 途中への挿入 | O(n) | O(1)※ |
| メモリ効率 | 良い | ポインタ分のオーバーヘッド |

※条件による

## 3.3 スタック（Stack）

### 特徴
- **LIFO（Last In, First Out）**：後入れ先出し
- 主な操作：push（積む）、pop（取り出す）

```
push(A) → [A]
push(B) → [A, B]
push(C) → [A, B, C]
pop()   → C を返す → [A, B]
pop()   → B を返す → [A]
```

### 用途
- 関数の呼び出し管理（コールスタック）
- 再帰処理
- 括弧の対応チェック
- 逆ポーランド記法（後置記法）の計算
- Undo機能

### 逆ポーランド記法（後置記法）

通常の式（中置記法）をスタックで計算する方法。

**中置 → 後置の変換例：**
```
中置記法：(A + B) × C
後置記法：A B + C ×
```

**後置記法の計算手順（3 4 + 2 ×）：**
```
3 → push(3)     スタック：[3]
4 → push(4)     スタック：[3, 4]
+ → pop2つ計算   3+4=7 → push(7)  スタック：[7]
2 → push(2)     スタック：[7, 2]
× → pop2つ計算   7×2=14 → push(14) スタック：[14]
結果：14
```

## 3.4 キュー（Queue）

### 特徴
- **FIFO（First In, First Out）**：先入れ先出し
- 主な操作：enqueue（追加）、dequeue（取り出し）

```
enqueue(A) → [A]
enqueue(B) → [A, B]
enqueue(C) → [A, B, C]
dequeue()  → A を返す → [B, C]
```

### 用途
- 印刷ジョブの管理（スプーリング）
- プロセスの実行待ち行列
- 幅優先探索（BFS）
- バッファ

### 優先度付きキュー（Priority Queue）
- 優先度の高い要素が先に取り出される
- ヒープで効率的に実装可能

## 3.5 木構造（Tree）

### 基本用語

```
        A          ← 根（ルート）
       / \
      B   C        ← B,C は A の子、A は B,C の親
     / \   \
    D   E   F      ← 葉（リーフ）：子を持たないノード
```

| 用語 | 意味 |
|------|------|
| 根（ルート） | 最上位のノード |
| 葉（リーフ） | 子を持たないノード |
| 深さ（レベル） | 根からの距離 |
| 高さ | 最も深い葉までの距離 |
| 次数 | 子の数 |

### 二分木（Binary Tree）
- 各ノードの子は最大2つ（左の子、右の子）

### 完全二分木
- 全ての葉が同じ深さにある二分木
- 深さdの完全二分木のノード数：**2^(d+1) - 1**
- 深さdの完全二分木の葉の数：**2^d**

### 二分探索木（Binary Search Tree）
- 左の子 < 親 < 右の子 という順序を保つ
- 探索：O(log n)（平衡の場合）

```
        5
       / \
      3   8
     / \ / \
    1  4 7  9
```

### 木の走査（トラバーサル）

上の二分探索木を例にすると：

| 走査方法 | 順序 | 結果 |
|---------|------|------|
| 前順（先行順・preorder） | 根→左→右 | 5, 3, 1, 4, 8, 7, 9 |
| 中順（中間順・inorder） | 左→根→右 | 1, 3, 4, 5, 7, 8, 9 |
| 後順（後行順・postorder） | 左→右→根 | 1, 4, 3, 7, 9, 8, 5 |

> **重要：** 二分探索木の中順走査は昇順にデータを取得できる。

### ヒープ（Heap）

**最大ヒープ：** 親 ≧ 子
**最小ヒープ：** 親 ≦ 子

```
最大ヒープの例：
        9
       / \
      7   8
     / \ /
    3  5 6
```

- 配列で実装可能：A[i]の子は A[2i+1] と A[2i+2]
- 挿入・削除：O(log n)
- 最大値（最小値）の取得：O(1)

### B木（B-tree）
- 多分岐の平衡木
- データベースのインデックスに使用
- ディスクアクセスを最小化する設計
- 各ノードは複数のキーを持てる

## 3.6 ハッシュ（Hash）

### ハッシュ法の仕組み
1. キーにハッシュ関数を適用してハッシュ値を計算
2. ハッシュ値をインデックスとしてデータを格納

```
キー "apple" → ハッシュ関数 → ハッシュ値 3 → 配列[3]に格納
```

### 計算量
- 平均：探索・挿入・削除 O(1)
- 最悪（全衝突）：O(n)

### 衝突（コリジョン）解決法

#### 1. オープンアドレス法（開番地法）
衝突したら別の空き領域を探す。

- **線形探索法：** 次の空きスロットを順番に探す
  - h(key), h(key)+1, h(key)+2, ...

#### 2. チェイン法（連鎖法）
同じハッシュ値のデータをリストでつなぐ。

```
[0] → NULL
[1] → [data1] → [data4] → NULL
[2] → [data2] → NULL
[3] → [data3] → NULL
```

### シノニム
同じハッシュ値になるキーの集まりをシノニムという。

## 3.7 グラフ（Graph）

### 基本用語
- **頂点（ノード）**：グラフの点
- **辺（エッジ）**：頂点間のつながり
- **有向グラフ**：辺に方向がある
- **無向グラフ**：辺に方向がない
- **重み付きグラフ**：辺にコスト（重み）がある
- **隣接**：辺でつながっている頂点同士

### グラフの表現方法

#### 隣接行列
```
    A  B  C  D
A [ 0  1  1  0 ]
B [ 1  0  1  1 ]
C [ 1  1  0  0 ]
D [ 0  1  0  0 ]
```

#### 隣接リスト
```
A → B → C
B → A → C → D
C → A → B
D → B
```

### グラフの探索

| 方法 | データ構造 | 特徴 |
|------|-----------|------|
| 深さ優先探索（DFS） | スタック | 行けるところまで進んで戻る |
| 幅優先探索（BFS） | キュー | 近いノードから順に探索 |

## まとめ

| データ構造 | 特徴 | 主な用途 |
|-----------|------|---------|
| 配列 | ランダムアクセスO(1) | インデックスでのアクセス |
| リスト | 挿入・削除が容易 | 動的なデータ管理 |
| スタック | LIFO | 関数呼び出し、後置記法 |
| キュー | FIFO | 待ち行列、BFS |
| 二分探索木 | 探索O(log n) | 効率的な探索 |
| ヒープ | 最大/最小値O(1) | 優先度付きキュー |
| ハッシュ | 平均O(1) | 高速な検索 |

## 確認問題

1. 後置記法 "5 3 2 × + 4 -" の計算結果を求めよ。
2. 深さ4の完全二分木のノード数と葉の数を求めよ。
3. ハッシュ法における衝突解決法を2つ挙げ、それぞれ説明せよ。
4. スタックとキューの違いを説明せよ。

<details>
<summary>解答</summary>

1. 5, 3, 2 → 3×2=6 → 5+6=11 → 11-4=7。答え：7
2. ノード数：2^5 - 1 = 31、葉の数：2^4 = 16
3. オープンアドレス法：衝突時に別の空きスロットを探す。チェイン法：同じハッシュ値のデータをリストでつなぐ。
4. スタックはLIFO（後入れ先出し）、キューはFIFO（先入れ先出し）。
</details>
