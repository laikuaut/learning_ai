# 第2章 演習：変数と展開

## 演習の目的
Bashにおける変数の代入・参照、各種展開（パラメータ展開・算術展開・コマンド置換）を正しく使えるようになることを目指します。

---

## 基本問題

### 問題1：変数の代入と参照
以下の要件を満たすスクリプトを作成してください。

1. 変数 `name` に自分の名前を代入する
2. 変数 `age` に年齢（数値）を代入する
3. 変数 `hobby` に趣味を代入する
4. これらの変数を使って自己紹介文を出力する

**期待される出力例：**
```
私の名前は田中太郎です。
年齢は25歳です。
趣味はプログラミングです。
```

<details>
<summary>ヒント</summary>

- 変数の代入は `変数名=値`（`=` の前後にスペースを入れない）
- 変数の参照は `$変数名` または `${変数名}`
- ダブルクォーテーション内では変数が展開されます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash
# 変数の代入と参照の基本
# 注意: = の前後にスペースを入れるとエラーになる

# 変数に値を代入（スペースを含む場合はクォーテーションで囲む）
name="田中太郎"
age=25
hobby="プログラミング"

# ダブルクォーテーション内で変数を展開して出力
echo "私の名前は${name}です。"
echo "年齢は${age}歳です。"
echo "趣味は${hobby}です。"
```

</details>

### 問題2：環境変数の確認と設定
以下の操作を行ってください。

1. 主要な環境変数（`HOME`, `USER`, `LANG`, `PWD`）の値を表示する
2. 新しい環境変数 `MY_APP_ENV` に `development` を設定する
3. `export` した場合としなかった場合の違いを確認する

**期待される出力例：**
```
HOME=/home/user
USER=user
LANG=ja_JP.UTF-8
PWD=/home/user

# exportなし：子プロセスでは参照できない
サブシェルでの MY_VAR: (空)

# exportあり：子プロセスでも参照できる
サブシェルでの MY_APP_ENV: development
```

<details>
<summary>ヒント</summary>

- 環境変数は大文字で定義するのが慣習です
- `export` すると子プロセスに変数が引き継がれます
- `bash -c 'コマンド'` でサブシェルを起動できます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash
# 環境変数の確認と export の動作

# 1. 主要な環境変数を表示
echo "HOME=$HOME"
echo "USER=$USER"
echo "LANG=$LANG"
echo "PWD=$PWD"
echo ""

# 2. exportなしの変数（シェル変数）
MY_VAR="ローカル値"
# サブシェル（子プロセス）では参照できない
echo "# exportなし：子プロセスでは参照できない"
bash -c 'echo "サブシェルでの MY_VAR: $MY_VAR"'
echo ""

# 3. exportありの変数（環境変数）
export MY_APP_ENV="development"
# サブシェル（子プロセス）でも参照できる
echo "# exportあり：子プロセスでも参照できる"
bash -c 'echo "サブシェルでの MY_APP_ENV: $MY_APP_ENV"'
```

</details>

### 問題3：特殊変数
以下のスクリプトを作成し、引数を付けて実行してください。特殊変数の値を表示するスクリプトです。

**実行方法：** `bash script.sh arg1 arg2 arg3`

**期待される出力例：**
```
スクリプト名   : script.sh
引数の個数     : 3
第1引数        : arg1
第2引数        : arg2
第3引数        : arg3
全引数（$@）   : arg1 arg2 arg3
全引数（$*）   : arg1 arg2 arg3
プロセスID     : 12345
直前の終了ステータス: 0
```

<details>
<summary>ヒント</summary>

- `$0` → スクリプト名
- `$#` → 引数の個数
- `$1`, `$2`, `$3` → 位置パラメータ
- `$@` → 全引数（個別の文字列として）
- `$*` → 全引数（一つの文字列として）
- `$$` → 現在のプロセスID
- `$?` → 直前のコマンドの終了ステータス

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash
# 特殊変数の表示スクリプト
# 使い方: bash script.sh arg1 arg2 arg3

# $0: 実行されたスクリプト名（パスを含む場合もある）
echo "スクリプト名   : $0"

# $#: 渡された引数の個数
echo "引数の個数     : $#"

# $1, $2, $3: 位置パラメータ（各引数）
echo "第1引数        : $1"
echo "第2引数        : $2"
echo "第3引数        : $3"

# $@: 全引数を個別の文字列として保持
# "$@" → "arg1" "arg2" "arg3"（それぞれ独立した文字列）
echo "全引数（\$@）   : $@"

# $*: 全引数をIFSの最初の文字で結合した一つの文字列
# "$*" → "arg1 arg2 arg3"（一つの文字列）
echo "全引数（\$*）   : $*"

# $$: 現在のシェルのプロセスID
echo "プロセスID     : $$"

# $?: 直前に実行したコマンドの終了ステータス（0=成功）
echo "直前の終了ステータス: $?"
```

</details>

### 問題4：クォーティングの違い
以下の各コマンドの出力結果を予測し、実際に実行して確認してください。

```bash
word="World"

echo "Hello $word"
echo 'Hello $word'
echo Hello $word
echo "Hello \"$word\""
echo "現在のディレクトリ: $(pwd)"
echo '現在のディレクトリ: $(pwd)'
echo "バッククォートも動く: `date +%Y`"
```

**期待される出力例：**
```
Hello World
Hello $word
Hello World
Hello "World"
現在のディレクトリ: /home/user
現在のディレクトリ: $(pwd)
バッククォートも動く: 2026
```

<details>
<summary>ヒント</summary>

- ダブルクォーテーション（`"`）：変数展開・コマンド置換が行われる
- シングルクォーテーション（`'`）：一切の展開が行われない（リテラル文字列）
- クォーテーションなし：変数展開に加えてワード分割・グロブ展開も行われる

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash
# クォーティングの違いを確認するスクリプト

word="World"

# ダブルクォート: 変数が展開される → Hello World
echo "Hello $word"

# シングルクォート: 変数が展開されない → Hello $word
echo 'Hello $word'

# クォートなし: 変数は展開される → Hello World
# （スペースを含む値だとワード分割される点に注意）
echo Hello $word

# ダブルクォート内のエスケープ: \" でリテラルの " を表示
echo "Hello \"$word\""

# ダブルクォート内のコマンド置換: $() が実行される
echo "現在のディレクトリ: $(pwd)"

# シングルクォート内のコマンド置換: 実行されずそのまま表示
echo '現在のディレクトリ: $(pwd)'

# バッククォートによるコマンド置換（ダブルクォート内）
# $() と同じ動作だが、$() の方がネストしやすく推奨
echo "バッククォートも動く: $(date +%Y)"
```

</details>

### 問題5：パラメータ展開
以下のパラメータ展開を実際に試してください。

1. デフォルト値の設定（`:-`）
2. 変数の文字数取得（`#`）
3. 文字列の部分取得（`:offset:length`）
4. パターン削除（`#`, `##`, `%`, `%%`）
5. 文字列置換（`/`, `//`）

**期待される出力例：**
```
# デフォルト値
greeting: こんにちは

# 文字数
文字数: 11

# 部分文字列
部分: World

# パターン削除
ファイル名: document.backup.tar.gz
拡張子除去（最短）: document.backup.tar
拡張子除去（最長）: document
ディレクトリ除去（最短）: user/docs/file.txt
ディレクトリ除去（最長）: file.txt

# 文字列置換
最初だけ: Hello Bash World
全て置換: Hello_Bash_World
```

<details>
<summary>ヒント</summary>

- `${var:-default}` → 変数が未定義または空なら `default` を返す
- `${#var}` → 変数の文字数
- `${var:offset:length}` → 部分文字列の抽出
- `${var#pattern}` → 先頭からの最短マッチを削除
- `${var##pattern}` → 先頭からの最長マッチを削除
- `${var%pattern}` → 末尾からの最短マッチを削除
- `${var%%pattern}` → 末尾からの最長マッチを削除
- `${var/old/new}` → 最初のマッチを置換
- `${var//old/new}` → 全てのマッチを置換

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash
# パラメータ展開の各種機能を確認するスクリプト

# 1. デフォルト値（:- 演算子）
# unset_var が未定義なので「こんにちは」が使われる
unset unset_var
echo "# デフォルト値"
echo "greeting: ${unset_var:-こんにちは}"
echo ""

# 2. 文字数（# 演算子）
text="Hello World"
echo "# 文字数"
echo "文字数: ${#text}"
echo ""

# 3. 部分文字列（:offset:length）
# 0始まりのインデックスで位置6から5文字を取得
echo "# 部分文字列"
echo "部分: ${text:6:5}"
echo ""

# 4. パターン削除
filepath="/home/user/docs/file.txt"
filename="document.backup.tar.gz"

echo "# パターン削除"
echo "ファイル名: $filename"

# % は末尾からの最短マッチを削除（最後の.以降を除去）
echo "拡張子除去（最短）: ${filename%.*}"

# %% は末尾からの最長マッチを削除（最初の.以降を全て除去）
echo "拡張子除去（最長）: ${filename%%.*}"

# # は先頭からの最短マッチを削除（最初の/まで除去）
echo "ディレクトリ除去（最短）: ${filepath#*/}"

# ## は先頭からの最長マッチを削除（最後の/まで除去）
echo "ディレクトリ除去（最長）: ${filepath##*/}"
echo ""

# 5. 文字列置換
words="Hello Bash World"
echo "# 文字列置換"
# / は最初のマッチだけ置換
echo "最初だけ: ${words/ /_}"      # 本例では最初のスペースだけ置換
# // は全てのマッチを置換
echo "全て置換: ${words// /_}"
```

</details>

### 問題6：算術展開と算術演算
以下の計算をBashの算術展開 `$(( ))` を使って行ってください。

1. 2つの数値の四則演算（加算・減算・乗算・除算・剰余）
2. インクリメント・デクリメント
3. 比較演算（結果は1=真、0=偽）
4. 複合的な計算式

**期待される出力例：**
```
# 四則演算
10 + 3 = 13
10 - 3 = 7
10 * 3 = 30
10 / 3 = 3
10 % 3 = 1

# インクリメント・デクリメント
count初期値: 5
インクリメント後: 6
デクリメント後: 5

# 比較演算
10 > 3 → 1 (真)
10 < 3 → 0 (偽)
10 == 10 → 1 (真)

# 複合計算
(10 + 5) * 2 - 3 = 27
```

<details>
<summary>ヒント</summary>

- `$(( 式 ))` で算術展開ができます
- `$(( ))` 内では変数名の前の `$` は省略できます
- Bashの除算は整数除算（小数点以下切り捨て）です
- 比較演算子は `>`, `<`, `==`, `!=`, `>=`, `<=` が使えます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash
# 算術展開の各種操作を確認するスクリプト

a=10
b=3

# 1. 四則演算
echo "# 四則演算"
echo "$a + $b = $(( a + b ))"     # 加算
echo "$a - $b = $(( a - b ))"     # 減算
echo "$a * $b = $(( a * b ))"     # 乗算
echo "$a / $b = $(( a / b ))"     # 除算（整数除算・小数は切り捨て）
echo "$a % $b = $(( a % b ))"     # 剰余（余り）
echo ""

# 2. インクリメント・デクリメント
count=5
echo "# インクリメント・デクリメント"
echo "count初期値: $count"
(( count++ ))                      # インクリメント（1加算）
echo "インクリメント後: $count"
(( count-- ))                      # デクリメント（1減算）
echo "デクリメント後: $count"
echo ""

# 3. 比較演算（真=1, 偽=0）
echo "# 比較演算"
echo "$a > $b → $(( a > b )) (真)"
echo "$a < $b → $(( a < b )) (偽)"
echo "$a == $a → $(( a == a )) (真)"
echo ""

# 4. 複合計算
echo "# 複合計算"
echo "($a + 5) * 2 - 3 = $(( (a + 5) * 2 - 3 ))"
```

</details>

### 問題7：設定ファイルパーサー
`KEY=VALUE` 形式の設定ファイルを読み込み、各設定値を表示するスクリプトを作成してください。以下の機能を実装してください。

- `#` で始まる行はコメントとしてスキップ
- 空行はスキップ
- デフォルト値の設定（設定ファイルに無い項目にはデフォルト値を使用）

**入力ファイル例（config.txt）：**
```
# アプリケーション設定
APP_NAME=MyApp
APP_VERSION=1.0.0

# データベース設定
DB_HOST=localhost
DB_PORT=5432
# DB_PASSWORD=secret
```

**期待される出力例：**
```
===== 設定ファイル読み込み結果 =====
APP_NAME    : MyApp
APP_VERSION : 1.0.0
DB_HOST     : localhost
DB_PORT     : 5432
DB_PASSWORD : (未設定 → デフォルト: password)
DB_NAME     : (未設定 → デフォルト: mydb)
================================
```

<details>
<summary>ヒント</summary>

- `while IFS='=' read -r key value` でKEYとVALUEを分割読み取りできます
- `[[ "$line" =~ ^# ]]` でコメント行を判定できます
- `[[ -z "$line" ]]` で空行を判定できます
- `${var:-default}` でデフォルト値を設定できます

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash
# 設定ファイルパーサー
# KEY=VALUE 形式の設定ファイルを読み込み、変数に格納する

CONFIG_FILE="config.txt"

# 設定ファイルが存在しない場合はサンプルを作成
if [ ! -f "$CONFIG_FILE" ]; then
    cat > "$CONFIG_FILE" << 'HEREDOC'
# アプリケーション設定
APP_NAME=MyApp
APP_VERSION=1.0.0

# データベース設定
DB_HOST=localhost
DB_PORT=5432
# DB_PASSWORD=secret
HEREDOC
    echo "サンプル設定ファイルを作成しました: $CONFIG_FILE"
fi

# 設定ファイルを1行ずつ読み込む
while IFS= read -r line || [ -n "$line" ]; do
    # 空行をスキップ
    [[ -z "$line" ]] && continue

    # コメント行（#で始まる行）をスキップ
    [[ "$line" =~ ^[[:space:]]*# ]] && continue

    # KEY=VALUE の形式で分割
    # IFS='=' を使い、最初の = で分割する
    key="${line%%=*}"
    value="${line#*=}"

    # 変数として設定（declare を使用）
    declare "$key=$value"
done < "$CONFIG_FILE"

# 結果を表示（デフォルト値付き）
echo "===== 設定ファイル読み込み結果 ====="
echo "APP_NAME    : ${APP_NAME}"
echo "APP_VERSION : ${APP_VERSION}"
echo "DB_HOST     : ${DB_HOST}"
echo "DB_PORT     : ${DB_PORT}"

# デフォルト値の適用（:- 演算子）
# 設定ファイルに無い項目にはデフォルト値を表示
if [ -z "$DB_PASSWORD" ]; then
    echo "DB_PASSWORD : (未設定 → デフォルト: password)"
    DB_PASSWORD="${DB_PASSWORD:-password}"
else
    echo "DB_PASSWORD : $DB_PASSWORD"
fi

if [ -z "$DB_NAME" ]; then
    echo "DB_NAME     : (未設定 → デフォルト: mydb)"
    DB_NAME="${DB_NAME:-mydb}"
else
    echo "DB_NAME     : $DB_NAME"
fi

echo "================================"
```

</details>

### 問題8：文字列操作ユーティリティ
パラメータ展開を活用して、以下の文字列操作を行うスクリプトを作成してください。

入力文字列: `/home/user/documents/report_2026_final.tar.gz`

以下の情報を抽出してください。
1. ファイル名のみ（ディレクトリパスを除去）
2. ディレクトリパスのみ
3. 拡張子のみ（`.gz`）
4. 全ての拡張子を除去した名前（`report_2026_final`）
5. ファイル名の `2026` を `2027` に置換
6. ファイル名の文字数

**期待される出力例：**
```
===== 文字列操作ユーティリティ =====
元の文字列     : /home/user/documents/report_2026_final.tar.gz
ファイル名     : report_2026_final.tar.gz
ディレクトリ   : /home/user/documents
拡張子         : .gz
名前（拡張子なし）: report_2026_final
年号置換       : /home/user/documents/report_2027_final.tar.gz
文字数         : 49
====================================
```

<details>
<summary>ヒント</summary>

- `${var##*/}` でディレクトリ部分を除去（`basename` 相当）
- `${var%/*}` でファイル名部分を除去（`dirname` 相当）
- `${var##*.}` で拡張子を取得
- `${var%%.*}` を使うとき、パスに `.` が含まれる場合に注意
- `${var/old/new}` で文字列置換

</details>

<details>
<summary>解答例</summary>

```bash
#!/bin/bash
# 文字列操作ユーティリティ
# パラメータ展開を使ってファイルパスから各種情報を抽出する

path="/home/user/documents/report_2026_final.tar.gz"

echo "===== 文字列操作ユーティリティ ====="
echo "元の文字列     : $path"

# 1. ファイル名のみ（## で先頭からの最長マッチを削除）
# */ パターンで最後の / までを全て削除
filename="${path##*/}"
echo "ファイル名     : $filename"

# 2. ディレクトリパスのみ（% で末尾からの最短マッチを削除）
# /* パターンで最後の / 以降を削除
dirname="${path%/*}"
echo "ディレクトリ   : $dirname"

# 3. 拡張子のみ（## で先頭からの最長マッチを削除）
# *. パターンで最後の . までを全て削除し、残った部分が拡張子
ext=".${filename##*.}"
echo "拡張子         : $ext"

# 4. 全拡張子を除去（%% で末尾からの最長マッチを削除）
# ファイル名部分から操作するため、先にディレクトリを除去
basename_only="${filename%%.*}"
echo "名前（拡張子なし）: $basename_only"

# 5. 文字列置換（/ で最初のマッチを置換）
replaced="${path/2026/2027}"
echo "年号置換       : $replaced"

# 6. 文字数（# 演算子）
echo "文字数         : ${#path}"

echo "===================================="
```

</details>
