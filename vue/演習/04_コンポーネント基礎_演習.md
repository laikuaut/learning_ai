## 演習4-1：props の定義と受け渡し（基本）

### 問題

ユーザーカードコンポーネント `UserCard.vue` を作成し、親コンポーネントから props でデータを渡してください。

**UserCard.vue の要件:**
1. 以下の props を受け取る:
   - `name`（文字列、必須）
   - `email`（文字列、必須）
   - `role`（文字列、デフォルト値: `'viewer'`）
   - `isActive`（真偽値、デフォルト値: `true`）
2. props の値をカード形式で表示する

**親コンポーネントの要件:**
- `UserCard` を3つ配置し、異なるデータを渡す

<details>
<summary>ヒント</summary>

- `defineProps` マクロで props を定義します（インポート不要）
- デフォルト値は `default` プロパティで指定します
- 必須の props は `required: true` で指定します
- 親から渡すときは `:propName="値"` の構文です

</details>

<details>
<summary>解答例</summary>

```vue
<!-- components/UserCard.vue -->
<script setup>
// defineProps はインポート不要のコンパイラマクロ
// オブジェクト構文でバリデーション付きの props を定義
const props = defineProps({
  name: {
    type: String,
    required: true    // 必須プロパティ — 親が渡さないと警告が出る
  },
  email: {
    type: String,
    required: true
  },
  role: {
    type: String,
    default: 'viewer'  // デフォルト値 — 親が渡さなかった場合この値になる
  },
  isActive: {
    type: Boolean,
    default: true
  }
})
</script>

<template>
  <div :class="['user-card', { inactive: !isActive }]">
    <h3>{{ name }}</h3>
    <p>{{ email }}</p>
    <span class="role-badge">{{ role }}</span>
    <span :class="['status', isActive ? 'online' : 'offline']">
      {{ isActive ? 'アクティブ' : '非アクティブ' }}
    </span>
  </div>
</template>

<style scoped>
.user-card {
  border: 1px solid #e2e8f0;
  border-radius: 8px;
  padding: 16px;
  margin-bottom: 12px;
}

.inactive {
  opacity: 0.5;
}

.role-badge {
  background-color: #e2e8f0;
  padding: 2px 8px;
  border-radius: 12px;
  font-size: 0.85em;
}

.status.online { color: #38a169; }
.status.offline { color: #e53e3e; }
</style>
```

```vue
<!-- App.vue（親コンポーネント） -->
<script setup>
import UserCard from './components/UserCard.vue'
</script>

<template>
  <div>
    <h1>ユーザー一覧</h1>

    <!-- 全ての props を明示的に渡す -->
    <UserCard
      name="山田太郎"
      email="taro@example.com"
      role="admin"
      :is-active="true"
    />

    <!-- role と isActive はデフォルト値が使われる -->
    <UserCard
      name="佐藤花子"
      email="hanako@example.com"
      role="editor"
    />

    <!-- isActive に false を渡す（: が必要！） -->
    <UserCard
      name="田中一郎"
      email="ichiro@example.com"
      :is-active="false"
    />
  </div>
</template>
```

**ポイント:**
- 真偽値を渡すときは `:is-active="false"` のように `:`（v-bind）が必要です。`is-active="false"` だと文字列 `"false"` が渡されます。
- HTML属性はケバブケース（`is-active`）、JavaScript はキャメルケース（`isActive`）で書きます。Vue が自動変換してくれます。

</details>

## 演習4-2：emit でイベントを親に通知（基本）

### 問題

評価（星1～5）を入力できる `StarRating.vue` コンポーネントを作成してください。

**StarRating.vue の要件:**
1. `modelValue` props で現在の評価値を受け取る
2. 星をクリックしたら `update:modelValue` イベントで新しい値を親に通知する（`v-model` 対応）
3. 星は5つ表示し、現在の評価以下の星を塗りつぶし表示する

**親コンポーネントの要件:**
- `StarRating` を `v-model` で使い、選択された評価値を表示する

<details>
<summary>ヒント</summary>

- Vue 3 の `v-model` は内部的に `:modelValue` と `@update:modelValue` に展開されます
- `defineEmits` マクロでイベントを宣言します
- 星の表示は `v-for="n in 5"` でループし、`n <= modelValue` で塗りつぶし判定ができます

</details>

<details>
<summary>解答例</summary>

```vue
<!-- components/StarRating.vue -->
<script setup>
// props: v-model と連動させるため modelValue を受け取る
const props = defineProps({
  modelValue: {
    type: Number,
    default: 0
  }
})

// emit: 親コンポーネントにイベントを通知するための宣言
// 'update:modelValue' は v-model の規約
const emit = defineEmits(['update:modelValue'])

// 星がクリックされたとき、新しい値を親に通知する
function setRating(value) {
  emit('update:modelValue', value)
}
</script>

<template>
  <div class="star-rating">
    <!--
      v-for="n in 5" で 1〜5 の星を生成
      n <= modelValue なら塗りつぶし、そうでなければ空の星
    -->
    <span
      v-for="n in 5"
      :key="n"
      class="star"
      :class="{ filled: n <= modelValue }"
      @click="setRating(n)"
    >
      {{ n <= modelValue ? '★' : '☆' }}
    </span>
  </div>
</template>

<style scoped>
.star {
  font-size: 24px;
  cursor: pointer;
  transition: transform 0.1s;
}

.star:hover {
  transform: scale(1.2);
}

.star.filled {
  color: #f6ad55;
}
</style>
```

```vue
<!-- App.vue（親コンポーネント） -->
<script setup>
import { ref } from 'vue'
import StarRating from './components/StarRating.vue'

const rating = ref(3)
</script>

<template>
  <div>
    <h2>商品の評価</h2>

    <!--
      v-model は以下の省略形:
      :modelValue="rating" @update:modelValue="rating = $event"
    -->
    <StarRating v-model="rating" />

    <p>現在の評価: {{ rating }} / 5</p>
  </div>
</template>
```

**v-model のカスタムコンポーネント対応の仕組み:**
```
親: <StarRating v-model="rating" />
    ↓ 展開すると同じ意味
親: <StarRating :modelValue="rating" @update:modelValue="val => rating = val" />
    ↓
子: props.modelValue で受け取り、emit('update:modelValue', newVal) で返す
```

</details>

## 演習4-3：バグ修正 — props の誤用（基本）

### 問題

以下のコードにはコンポーネント間のデータの流れに関するバグが2つあります。修正してください。

```vue
<!-- 子コンポーネント: TodoItem.vue -->
<script setup>
const props = defineProps({
  title: String,
  done: Boolean
})

// タスクの完了状態をトグル
function toggle() {
  props.done = !props.done  // ← バグ1
}
</script>

<template>
  <li>
    <input type="checkbox" :checked="done" @change="toggle" />
    {{ title }}
  </li>
</template>
```

```vue
<!-- 親コンポーネント -->
<script setup>
import { ref } from 'vue'
import TodoItem from './TodoItem.vue'

const tasks = ref([
  { id: 1, title: '買い物', done: false },
  { id: 2, title: '掃除', done: true }
])
</script>

<template>
  <ul>
    <TodoItem
      v-for="task in tasks"
      :key="task.id"
      :title="task.title"
      :done="task.done"
    />  <!-- ← バグ2: 子からの通知を受け取っていない -->
  </ul>
</template>
```

<details>
<summary>ヒント</summary>

- **バグ1:** Vue では props は読み取り専用です。子コンポーネントで直接変更することはできません（単方向データフロー）。
- **バグ2:** 子コンポーネントからの変更を親が受け取る仕組み（emit）がありません。

</details>

<details>
<summary>解答例</summary>

**バグ1: props を直接変更している**

Vue の props は**単方向データフロー**（親→子）です。子コンポーネントで props を変更しようとすると、コンソールに警告が出ます。代わりに `emit` で親に通知します。

**バグ2: 親が子のイベントをリッスンしていない**

`@toggle-done` でイベントを受け取り、親側でデータを更新する必要があります。

```vue
<!-- TodoItem.vue（修正後） -->
<script setup>
const props = defineProps({
  title: String,
  done: Boolean
})

// ✅ emit を宣言して、親にイベントを通知する
const emit = defineEmits(['toggle-done'])

function toggle() {
  // ✅ props を直接変更せず、emit で親に通知する
  emit('toggle-done')
}
</script>

<template>
  <li>
    <input type="checkbox" :checked="done" @change="toggle" />
    {{ title }}
  </li>
</template>
```

```vue
<!-- 親コンポーネント（修正後） -->
<script setup>
import { ref } from 'vue'
import TodoItem from './TodoItem.vue'

const tasks = ref([
  { id: 1, title: '買い物', done: false },
  { id: 2, title: '掃除', done: true }
])

// 親側でデータを更新する関数
function handleToggle(taskId) {
  const task = tasks.value.find(t => t.id === taskId)
  if (task) {
    task.done = !task.done
  }
}
</script>

<template>
  <ul>
    <TodoItem
      v-for="task in tasks"
      :key="task.id"
      :title="task.title"
      :done="task.done"
      @toggle-done="handleToggle(task.id)"
    />
  </ul>
</template>
```

**単方向データフローの原則:**
```
親 ──(props)──→ 子     データは親から子へ流れる
親 ←──(emit)── 子     イベントは子から親へ通知される
```

</details>

## 演習4-4：スロットの活用（応用）

### 問題

汎用的なモーダルダイアログ `ModalDialog.vue` を、スロットを使って作成してください。

**要件:**
1. **デフォルトスロット** — モーダルの本文
2. **名前付きスロット `header`** — モーダルのヘッダー（指定がなければ「お知らせ」を表示）
3. **名前付きスロット `footer`** — モーダルのフッター（指定がなければ「閉じる」ボタンを表示）
4. `isOpen` props でモーダルの表示/非表示を制御する
5. `close` イベントを emit して親に閉じる操作を通知する

<details>
<summary>ヒント</summary>

- 名前付きスロットは `<slot name="header">` で定義し、親側では `<template #header>` で使います
- スロットのフォールバック（デフォルト内容）は `<slot>フォールバック内容</slot>` で定義します
- オーバーレイ（背景の半透明黒）をクリックで閉じる実装も考慮しましょう

</details>

<details>
<summary>解答例</summary>

```vue
<!-- components/ModalDialog.vue -->
<script setup>
defineProps({
  isOpen: {
    type: Boolean,
    required: true
  }
})

const emit = defineEmits(['close'])

function handleClose() {
  emit('close')
}
</script>

<template>
  <!-- Teleport を使うと、DOM ツリー上の別の場所にレンダリングできる -->
  <Teleport to="body">
    <div v-if="isOpen" class="modal-overlay" @click.self="handleClose">
      <div class="modal-content">
        <!-- ヘッダースロット（フォールバック付き） -->
        <header class="modal-header">
          <slot name="header">
            <!-- 親がヘッダーを指定しなかった場合に表示されるデフォルト -->
            <h2>お知らせ</h2>
          </slot>
        </header>

        <!-- デフォルトスロット（本文） -->
        <main class="modal-body">
          <slot>
            内容がありません
          </slot>
        </main>

        <!-- フッタースロット（フォールバック付き） -->
        <footer class="modal-footer">
          <slot name="footer">
            <!-- 親がフッターを指定しなかった場合のデフォルトボタン -->
            <button @click="handleClose">閉じる</button>
          </slot>
        </footer>
      </div>
    </div>
  </Teleport>
</template>

<style scoped>
.modal-overlay {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background-color: rgba(0, 0, 0, 0.5);
  display: flex;
  justify-content: center;
  align-items: center;
  z-index: 1000;
}

.modal-content {
  background: white;
  border-radius: 12px;
  min-width: 400px;
  max-width: 600px;
  box-shadow: 0 20px 60px rgba(0, 0, 0, 0.3);
}

.modal-header {
  padding: 16px 20px;
  border-bottom: 1px solid #e2e8f0;
}

.modal-body {
  padding: 20px;
}

.modal-footer {
  padding: 12px 20px;
  border-top: 1px solid #e2e8f0;
  text-align: right;
}
</style>
```

```vue
<!-- App.vue — ModalDialog の利用例 -->
<script setup>
import { ref } from 'vue'
import ModalDialog from './components/ModalDialog.vue'

const showModal = ref(false)
const showConfirm = ref(false)

function handleConfirm() {
  console.log('確認されました')
  showConfirm.value = false
}
</script>

<template>
  <!-- 使用例1: デフォルトのヘッダーとフッター -->
  <button @click="showModal = true">お知らせを見る</button>
  <ModalDialog :is-open="showModal" @close="showModal = false">
    <p>Vue.js 3 の学習を始めましょう！</p>
  </ModalDialog>

  <!-- 使用例2: カスタムヘッダーとフッター -->
  <button @click="showConfirm = true">削除の確認</button>
  <ModalDialog :is-open="showConfirm" @close="showConfirm = false">
    <!-- #header で名前付きスロットにコンテンツを差し込む -->
    <template #header>
      <h2 style="color: red;">削除の確認</h2>
    </template>

    <p>本当にこのデータを削除しますか？この操作は取り消せません。</p>

    <!-- #footer でフッターをカスタマイズ -->
    <template #footer>
      <button @click="showConfirm = false">キャンセル</button>
      <button @click="handleConfirm" class="danger">削除する</button>
    </template>
  </ModalDialog>
</template>
```

**スロットを使う利点:**
- コンポーネントの「枠組み」は共通化しつつ、中身は呼び出し側が自由にカスタマイズできます。
- ボタンのテキスト、アイコン、レイアウトなど、使う場面に応じた柔軟性が得られます。

</details>

## 演習4-5：スコープ付きスロット（応用）

### 問題

データリストを汎用的に表示する `DataList.vue` を、スコープ付きスロットを使って作成してください。

**DataList.vue の要件:**
1. `items` 配列を props で受け取る
2. 各アイテムの表示方法はスコープ付きスロットで呼び出し側に委ねる
3. スロットに `item`（データ）と `index`（インデックス）を公開する
4. リストが空のときのフォールバック表示を用意する

**親コンポーネントで、同じ `DataList` を2つの異なる見た目で使う:**
- ユーザーリスト（アバター + 名前）
- 商品リスト（名前 + 価格 + カートボタン）

<details>
<summary>ヒント</summary>

- スコープ付きスロットは `<slot :item="item" :index="index">` で定義します
- 親側では `<template #default="{ item, index }">` で受け取ります
- 同じコンポーネントでも、スロットの内容を変えることで異なる表示ができます

</details>

<details>
<summary>解答例</summary>

```vue
<!-- components/DataList.vue -->
<script setup>
defineProps({
  items: {
    type: Array,
    required: true
  },
  title: {
    type: String,
    default: 'リスト'
  }
})
</script>

<template>
  <div class="data-list">
    <h3>{{ title }}（{{ items.length }}件）</h3>

    <ul v-if="items.length > 0">
      <li v-for="(item, index) in items" :key="index">
        <!--
          スコープ付きスロット:
          slot に渡したデータ（item, index）を
          親コンポーネントのテンプレートで使える
        -->
        <slot :item="item" :index="index">
          <!-- フォールバック: スロットが提供されない場合の表示 -->
          {{ item }}
        </slot>
      </li>
    </ul>

    <!-- リストが空のときの表示 -->
    <div v-else class="empty">
      <slot name="empty">
        <p>データがありません</p>
      </slot>
    </div>
  </div>
</template>

<style scoped>
.data-list {
  border: 1px solid #e2e8f0;
  border-radius: 8px;
  padding: 16px;
  margin-bottom: 16px;
}
ul {
  list-style: none;
  padding: 0;
}
li {
  padding: 8px 0;
  border-bottom: 1px solid #f0f0f0;
}
.empty {
  text-align: center;
  color: #a0aec0;
  padding: 20px;
}
</style>
```

```vue
<!-- App.vue — 同じ DataList を2つの異なる見た目で使う -->
<script setup>
import { ref } from 'vue'
import DataList from './components/DataList.vue'

const users = ref([
  { name: '山田太郎', avatar: '👤', role: 'admin' },
  { name: '佐藤花子', avatar: '👩', role: 'editor' },
  { name: '田中一郎', avatar: '👨', role: 'viewer' }
])

const products = ref([
  { name: 'Vue.js入門書', price: 2980 },
  { name: 'TypeScriptガイド', price: 3480 },
  { name: 'Web開発辞典', price: 4200 }
])

function addToCart(product) {
  alert(`${product.name}をカートに追加しました`)
}
</script>

<template>
  <!-- 使い方1: ユーザーリスト -->
  <DataList :items="users" title="ユーザー一覧">
    <!-- スコープ付きスロットで item と index を受け取る -->
    <template #default="{ item, index }">
      <div class="user-row">
        <span>{{ item.avatar }}</span>
        <strong>{{ index + 1 }}. {{ item.name }}</strong>
        <span class="badge">{{ item.role }}</span>
      </div>
    </template>
  </DataList>

  <!-- 使い方2: 商品リスト（全く異なる表示） -->
  <DataList :items="products" title="商品一覧">
    <template #default="{ item }">
      <div class="product-row">
        <span>{{ item.name }}</span>
        <span class="price">¥{{ item.price.toLocaleString() }}</span>
        <button @click="addToCart(item)">カートに入れる</button>
      </div>
    </template>

    <!-- 空のときの表示もカスタマイズ -->
    <template #empty>
      <p>商品がありません。入荷をお待ちください。</p>
    </template>
  </DataList>
</template>
```

**スコープ付きスロットの使いどころ:**
- テーブルやリストの「行の表示方法」を呼び出し側に任せるUIライブラリ
- ドラッグ&ドロップリストの各要素の見た目をカスタマイズする場合
- レンダレスコンポーネント（ロジックだけ提供し、見た目は委ねる）

</details>

## 演習4-6：コンポーネント設計 — 通知システム（チャレンジ）

### 問題

以下の3つのコンポーネントで構成される通知システムを設計・実装してください。

**コンポーネント構成:**
1. `NotificationContainer.vue` — 通知一覧を管理する親コンポーネント
2. `NotificationItem.vue` — 個々の通知を表示する子コンポーネント
3. `NotificationForm.vue` — 新しい通知を追加するフォーム

**機能要件:**
- 通知にはタイプ（`success`, `warning`, `error`, `info`）がある
- 通知は自動で5秒後に消える（手動で閉じることも可能）
- 新しい通知はリストの先頭に追加される
- タイプに応じて色を変える

<details>
<summary>ヒント</summary>

- データは `NotificationContainer` で一元管理し、子コンポーネントに props で渡します
- `NotificationForm` から `NotificationContainer` へは emit で通知します
- 自動削除には `setTimeout` を使い、`onUnmounted` でクリーンアップします
- Transition コンポーネントでアニメーションを付けるとUXが向上します

</details>

<details>
<summary>解答例</summary>

```vue
<!-- components/NotificationItem.vue -->
<script setup>
import { onMounted, onUnmounted } from 'vue'

const props = defineProps({
  id: { type: Number, required: true },
  message: { type: String, required: true },
  type: {
    type: String,
    default: 'info',
    validator: (val) => ['success', 'warning', 'error', 'info'].includes(val)
  },
  duration: { type: Number, default: 5000 }  // 自動削除までのミリ秒
})

const emit = defineEmits(['dismiss'])

let timer = null

// マウント時にタイマー開始
onMounted(() => {
  if (props.duration > 0) {
    timer = setTimeout(() => {
      emit('dismiss', props.id)
    }, props.duration)
  }
})
```

```vue
<!-- components/NotificationContainer.vue -->
<script setup>
import { ref } from 'vue'
import NotificationItem from './NotificationItem.vue'
import NotificationForm from './NotificationForm.vue'

// 通知リストの状態を一元管理（単一責任: この親が全データを管理）
const notifications = ref([])
let nextId = 1

// NotificationForm からのイベントを受け取り、通知を追加
function addNotification({ message, type }) {
  // 先頭に追加（unshift）
  notifications.value.unshift({
    id: nextId++,
    message,
    type
  })
}

// NotificationItem からの閉じるイベントを受け取り、通知を削除
function dismissNotification(id) {
  notifications.value = notifications.value.filter(n => n.id !== id)
}
</script>

<template>
  <div class="notification-container">
    <h2>通知システム</h2>

    <!-- フォーム: 新規通知の追加 -->
    <NotificationForm @add-notification="addNotification" />

    <!-- 通知一覧: TransitionGroup でアニメーション -->
    <TransitionGroup name="notification" tag="div">
      <NotificationItem
        v-for="notification in notifications"
        :key="notification.id"
        :id="notification.id"
        :message="notification.message"
        :type="notification.type"
        @dismiss="dismissNotification"
      />
    </TransitionGroup>

    <p v-if="notifications.length === 0" class="empty">通知はありません</p>
  </div>
</template>

<style scoped>
.notification-container {
  max-width: 500px;
  padding: 20px;
}

/* TransitionGroup のアニメーション */
.notification-enter-from {
  opacity: 0;
  transform: translateX(30px);
}
.notification-enter-active {
  transition: all 0.3s ease;
}
.notification-leave-active {
  transition: all 0.3s ease;
}
.notification-leave-to {
  opacity: 0;
  transform: translateX(-30px);
}

.empty { color: #a0aec0; text-align: center; }
</style>
```

**設計のポイント:**
- **データの一元管理:** 通知データは `NotificationContainer` が管理し、子に props で渡す
- **イベントの伝搬:** Form → Container → Item の流れで emit を使う
- **クリーンアップ:** `onUnmounted` でタイマーを確実にクリアし、メモリリークを防ぐ
- **TransitionGroup:** リストの追加・削除時にアニメーションを適用
