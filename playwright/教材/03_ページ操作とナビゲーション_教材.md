# 第3章：ページ操作とナビゲーション

## この章のゴール

- ページ遷移（`goto`、リンククリック）を使いこなす
- クリック、テキスト入力、選択などの基本操作ができる
- キーボード操作とマウス操作を理解する
- スクリーンショットの撮影方法を把握する
- 複数タブ・ポップアップの操作方法を知る

---

## 1. ページ遷移

### `page.goto()` — URLに直接移動

```typescript
// 基本
await page.goto('https://example.com');

// オプション付き
await page.goto('https://example.com', {
  waitUntil: 'domcontentloaded',  // 待機条件
  timeout: 10000,                  // タイムアウト（ミリ秒）
});
```

### waitUntil の選択肢

| 値 | 待機条件 | 用途 |
|----|---------|------|
| `'domcontentloaded'` | DOMの構築が完了 | 一般的な用途に最適 |
| `'load'` | 画像等も含めて完全に読み込み完了 | 画像が重要なページ |
| `'networkidle'` | ネットワーク通信が落ち着くまで | SPA・API通信が多いページ |
| `'commit'` | レスポンスを受信した時点 | 最も速い |

### ナビゲーション関連メソッド

```typescript
// 戻る・進む
await page.goBack();
await page.goForward();

// リロード
await page.reload();

// 現在のURL取得
const url = page.url();
```

### baseURL を活用する

`playwright.config.ts` で `baseURL` を設定すると、テスト内で相対パスが使えます。

```typescript
// playwright.config.ts
export default defineConfig({
  use: {
    baseURL: 'http://localhost:3000',
  },
});
```

```typescript
// テスト内
await page.goto('/login');      // http://localhost:3000/login
await page.goto('/dashboard');  // http://localhost:3000/dashboard
```

## 2. クリック操作

### 基本のクリック

```typescript
// ロケーターで要素を見つけてクリック
await page.getByRole('button', { name: '送信' }).click();

// リンクをクリック
await page.getByRole('link', { name: 'ホーム' }).click();
```

### クリックのオプション

```typescript
// ダブルクリック
await page.getByRole('button').dblclick();

// 右クリック
await page.getByRole('button').click({ button: 'right' });

// Shift + クリック
await page.getByRole('button').click({ modifiers: ['Shift'] });

// 要素内の特定の位置をクリック
await page.getByRole('button').click({ position: { x: 10, y: 10 } });

// クリックを強制する（要素が隠れていても実行）
await page.getByRole('button').click({ force: true });
```

### Playwrightの自動待機

Playwrightは `click()` を実行する前に、自動で以下を確認します。

```
┌─────────────────────────────────────────────┐
│  click() 実行前の自動チェック                │
├─────────────────────────────────────────────┤
│ 1. 要素がDOMに存在するか       ✓            │
│ 2. 要素が表示されているか       ✓            │
│ 3. 要素が安定しているか         ✓            │
│    （アニメーション中でないか）               │
│ 4. 要素が他の要素に覆われていないか ✓        │
│ 5. 要素が有効（enabled）か      ✓            │
│                                              │
│ すべてOKになるまで自動的にリトライ           │
│ → 明示的な wait が不要！                     │
└─────────────────────────────────────────────┘
```

## 3. テキスト入力

### `fill()` — テキストを入力する

```typescript
// 入力欄にテキストを入力（既存の内容はクリアされる）
await page.getByLabel('名前').fill('田中太郎');

// メールアドレスを入力
await page.getByLabel('メールアドレス').fill('taro@example.com');
```

### `fill()` と `type()` の違い

```typescript
// fill() — 値を一括で設定する（高速、イベントは最小限）
await page.getByLabel('検索').fill('Playwright');

// type() — 1文字ずつ入力する（キーイベントが発生）
// → 廃止予定。代わりに pressSequentially() を使用
await page.getByLabel('検索').pressSequentially('Playwright', { delay: 100 });
```

| メソッド | 動作 | 用途 |
|---------|------|------|
| `fill()` | 値を一括設定 | 一般的なフォーム入力 |
| `pressSequentially()` | 1文字ずつ入力 | オートコンプリート、入力中のバリデーション |
| `clear()` | 入力内容をクリア | 入力欄をリセット |

### テキストエリアの操作

```typescript
// テキストエリアに複数行入力
await page.getByLabel('メッセージ').fill('1行目\n2行目\n3行目');

// 入力内容をクリア
await page.getByLabel('メッセージ').clear();
```

## 4. フォーム操作

### チェックボックス

```typescript
// チェックを入れる
await page.getByRole('checkbox', { name: '利用規約に同意' }).check();

// チェックを外す
await page.getByRole('checkbox', { name: '利用規約に同意' }).uncheck();

// チェック状態を切り替える
await page.getByRole('checkbox', { name: '利用規約に同意' }).setChecked(true);
```

### ラジオボタン

```typescript
// 選択する
await page.getByRole('radio', { name: '男性' }).check();
```

### セレクトボックス（ドロップダウン）

```typescript
// 値で選択
await page.getByLabel('都道府県').selectOption('tokyo');

// 表示テキストで選択
await page.getByLabel('都道府県').selectOption({ label: '東京都' });

// 複数選択
await page.getByLabel('趣味').selectOption(['reading', 'music', 'sports']);
```

### ファイルアップロード

```typescript
// ファイルを選択
await page.getByLabel('プロフィール画像').setInputFiles('photo.jpg');

// 複数ファイル
await page.getByLabel('添付ファイル').setInputFiles(['file1.pdf', 'file2.pdf']);

// ファイル選択をリセット
await page.getByLabel('プロフィール画像').setInputFiles([]);
```

## 5. キーボード操作

### 特殊キーの入力

```typescript
// Enterキー
await page.getByLabel('検索').press('Enter');

// Tabキー
await page.getByLabel('名前').press('Tab');

// ショートカットキー
await page.keyboard.press('Control+a');   // 全選択
await page.keyboard.press('Control+c');   // コピー
await page.keyboard.press('Control+v');   // ペースト
await page.keyboard.press('Escape');      // Escapeキー
```

### キーの組み合わせ

```typescript
// Shift+Enterで改行（チャットアプリなど）
await page.getByLabel('メッセージ').press('Shift+Enter');

// Alt+F4でウィンドウを閉じる
await page.keyboard.press('Alt+F4');
```

## 6. スクリーンショット

### ページ全体のスクリーンショット

```typescript
// 表示されている範囲を撮影
await page.screenshot({ path: 'screenshot.png' });

// ページ全体（スクロール範囲を含む）を撮影
await page.screenshot({ path: 'full-page.png', fullPage: true });
```

### 特定の要素のスクリーンショット

```typescript
// 特定の要素だけを撮影
const card = page.locator('.product-card').first();
await card.screenshot({ path: 'product-card.png' });
```

### ビジュアルリグレッションテスト

```typescript
// スクリーンショットを比較（初回は基準画像を保存）
await expect(page).toHaveScreenshot('homepage.png');

// 特定要素のスクリーンショット比較
await expect(page.locator('.header')).toHaveScreenshot('header.png');
```

初回実行時に基準画像（スナップショット）が保存され、以降の実行で差分が検出されます。

## 7. 複数タブとポップアップ

### 新しいタブを扱う

```typescript
test('新しいタブで開くリンク', async ({ page }) => {
  // 新しいページ（タブ）が開くのを待ちながらクリック
  const [newPage] = await Promise.all([
    page.waitForEvent('popup'),
    page.getByRole('link', { name: '外部サイト' }).click(),
  ]);

  // 新しいタブで操作
  await expect(newPage).toHaveTitle('外部サイトのタイトル');
  await newPage.close();
});
```

### 別のブラウザコンテキスト（シークレットモード相当）

```typescript
test('別のユーザーとしてログイン', async ({ browser }) => {
  // 2つの独立したコンテキスト（Cookieやセッションが別）
  const context1 = await browser.newContext();
  const context2 = await browser.newContext();

  const page1 = await context1.newPage();
  const page2 = await context2.newPage();

  // page1 でユーザーAとしてログイン
  await page1.goto('/login');
  await page1.getByLabel('メール').fill('userA@example.com');

  // page2 でユーザーBとしてログイン
  await page2.goto('/login');
  await page2.getByLabel('メール').fill('userB@example.com');

  await context1.close();
  await context2.close();
});
```

## 8. 実践：ページ操作の総合例

```typescript
import { test, expect } from '@playwright/test';

test.describe('検索機能のテスト', () => {

  test('キーワードで検索して結果が表示される', async ({ page }) => {
    // 1. トップページに移動
    await page.goto('https://example.com');

    // 2. 検索欄にキーワードを入力
    await page.getByRole('searchbox').fill('Playwright');

    // 3. Enterキーで検索実行
    await page.getByRole('searchbox').press('Enter');

    // 4. 検索結果ページに遷移したことを確認
    await expect(page).toHaveURL(/search\?q=Playwright/);

    // 5. 検索結果が表示されていることを確認
    const results = page.getByRole('listitem');
    await expect(results.first()).toBeVisible();

    // 6. スクリーンショットを撮る
    await page.screenshot({ path: 'search-results.png' });
  });

  test('検索欄を空で送信するとエラーメッセージが表示される', async ({ page }) => {
    await page.goto('https://example.com');

    // 空のまま検索
    await page.getByRole('searchbox').press('Enter');

    // エラーメッセージの確認
    await expect(page.getByText('検索キーワードを入力してください')).toBeVisible();
  });
});
```

## まとめ

| 項目 | ポイント |
|------|---------|
| ページ遷移 | `page.goto(url)` で移動、`baseURL` で相対パス |
| クリック | `.click()` で自動待機付きクリック |
| テキスト入力 | `.fill()` で一括入力、`.pressSequentially()` で1文字ずつ |
| チェックボックス | `.check()` / `.uncheck()` / `.setChecked()` |
| セレクトボックス | `.selectOption()` で値またはラベルで選択 |
| キーボード | `.press('Enter')`, `keyboard.press('Control+a')` |
| スクリーンショット | `page.screenshot()` / `expect(page).toHaveScreenshot()` |
| 複数タブ | `page.waitForEvent('popup')` で新タブを取得 |
| 自動待機 | クリック・入力前に要素の状態を自動チェック |
