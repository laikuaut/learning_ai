# 第4章 演習：検索と置換の設定

## 演習の目標

- `hlsearch` / `incsearch` / `smartcase` の設定を使いこなせる
- 検索ハイライトの解除をキーマッピングで素早く行える
- `grepprg` を外部ツール（ripgrep 等）に切り替えられる
- `magic` / `very magic` の正規表現モードの違いを理解できる
- Quickfix ウィンドウを活用した検索ワークフローを構築できる

---

## 基本問題

### 問題1：検索の基本設定

以下の要件を満たす `.vimrc` の記述を作成してください。

**要件：**
1. 検索結果をハイライト表示する
2. 検索文字の入力中にリアルタイムでマッチ箇所を表示する
3. 検索が末尾に到達したら先頭に戻る
4. 大文字小文字を区別しない検索にする（ただし、大文字を含むパターンでは区別する）

**期待される .vimrc の記述：**
```vim
set hlsearch
set incsearch
set wrapscan
set ignorecase
set smartcase
```

<details>
<summary>ヒント</summary>

- `hlsearch` は検索結果のハイライト表示です
- `incsearch` は入力中のリアルタイム検索（インクリメンタルサーチ）です
- `wrapscan` は検索のラップアラウンド（末尾→先頭への折り返し）を制御します
- `smartcase` は `ignorecase` と組み合わせて使います（`ignorecase` が前提条件）

</details>

<details>
<summary>解答例</summary>

```vim
" 検索結果をハイライト表示する
" マッチした全箇所が色付きで表示されます
set hlsearch

" インクリメンタルサーチを有効にする
" /で検索文字を入力中に、リアルタイムでマッチ箇所にジャンプします
set incsearch

" 検索の折り返しを有効にする
" ファイル末尾まで検索したら先頭に戻って検索を続けます
" （デフォルトで有効ですが、明示的に記述しておくと安心です）
set wrapscan

" 大文字小文字を区別しない
set ignorecase

" 大文字を含む場合は区別する（スマートケース）
" ignorecase と組み合わせて使います
" 例: /hello → Hello, HELLO, hello すべてにマッチ
"     /Hello → Hello にだけマッチ（大文字を含むため）
set smartcase
```

</details>

### 問題2：検索ハイライトの解除マッピング

検索後に残るハイライトを素早く解除するためのキーマッピングを2種類作成してください。

1. `Esc` キー2回押しでハイライトを解除する
2. リーダーキー + `h` でハイライトを解除する

**期待される .vimrc の記述：**
```vim
nnoremap <Esc><Esc> :nohlsearch<CR>
nnoremap <Leader>h :nohlsearch<CR>
```

**動作の確認手順：**
```
1. /search_word でハイライトされた状態にする
2. Esc を2回押す → ハイライトが消える
3. n で次のマッチに移動 → ハイライトが再表示される
```

<details>
<summary>ヒント</summary>

- `:nohlsearch`（省略形 `:noh`）は一時的にハイライトを消すコマンドです
- `set nohlsearch` とは異なり、次の検索でハイライトが復活します
- `<CR>` はエンターキーの意味で、コマンドの実行を確定します
- `<Leader>` はデフォルトでは `\` キー、`let mapleader = " "` ならスペースです

</details>

<details>
<summary>解答例</summary>

```vim
" Esc 2回押しで検索ハイライトを解除
" :nohlsearch は一時的にハイライトを消します
" 次に / や * で検索すると再度ハイライトされます
nnoremap <Esc><Esc> :nohlsearch<CR>

" リーダーキー + h でも解除できるようにする
" スペースキーがリーダーなら、スペース→h で解除
nnoremap <Leader>h :nohlsearch<CR>

" 補足：:nohlsearch と :set nohlsearch の違い
"   :nohlsearch    → 一時的に消す（次の検索で復活）
"   :set nohlsearch → hlsearch オプション自体を無効化（検索しても光らなくなる）
"
" 実務では :nohlsearch をマッピングするのが正解です
```

</details>

### 問題3：検索パターンの magic モード

以下の検索パターンを、通常の `magic` モードと `very magic` モードそれぞれで記述してください。

**検索したい文字列：**
1. `function` の後に `(` が続くパターン
2. 行頭の数字1桁以上
3. `http` または `https`

**期待される回答：**
```
magic モード:
1. /function(
2. /^\d\+
3. /https\=

very magic モード:
1. /\vfunction\(
2. /\v^\d+
3. /\vhttps=
```

<details>
<summary>ヒント</summary>

- `magic` モード（デフォルト）では `+`, `(`, `)` 等はリテラル文字です。メタ文字にするには `\` が必要です
- `very magic` モード（`\v`）では、ほぼすべての特殊文字がメタ文字として扱われます（PCRE に近い）
- `\=` / `=`（very magic）は「直前の文字が0回または1回」（他の正規表現の `?` に相当）

</details>

<details>
<summary>解答例</summary>

```vim
" magic モード（デフォルト）
" 特殊文字をメタ文字として使うには \ が必要
"
" 1. function の後に ( が続くパターン
"    ( はリテラルなのでそのまま書けます
/function(

" 2. 行頭の数字1桁以上
"    \d は数字、\+ は「1回以上」のメタ文字
/^\d\+

" 3. http または https
"    \= は「直前の文字が0回または1回」
/https\=

" ----------------------------------------------------------------

" very magic モード（\v を先頭に付ける）
" ほぼすべての記号がメタ文字になるため、\ が不要になります
" ただし ( ) もメタ文字になるため、リテラルの ( は \( と書きます
"
" 1. function の後にリテラルの ( が続くパターン
"    ( はメタ文字（グループ化）なので \( でエスケープ
/\vfunction\(

" 2. 行頭の数字1桁以上
"    + がそのままメタ文字として使えます
/\v^\d+

" 3. http または https
"    = は「0回または1回」（magic の \= と同じ）
/\vhttps=

" 補足：.vimrc で very magic をデフォルトにするマッピング
" 検索コマンドの / を入力した時点で \v を自動挿入します
nnoremap / /\v
vnoremap / /\v
```

</details>

### 問題4：grepprg の設定と外部検索ツール連携

以下の要件を満たす `.vimrc` の記述を作成してください。

**要件：**
1. `:grep` コマンドの内部で使う検索プログラムを ripgrep（`rg`）に変更する
2. ripgrep の出力フォーマットに合わせて `grepformat` を設定する
3. `:grep` 実行後に自動で Quickfix ウィンドウを開く
4. ripgrep がインストールされていない場合は、デフォルトの grep を使う

**期待される .vimrc の記述：**
```vim
if executable('rg')
  set grepprg=rg\ --vimgrep\ --smart-case
  set grepformat=%f:%l:%c:%m
endif

autocmd QuickFixCmdPost *grep* cwindow
```

**動作確認コマンド：**
```
:grep pattern       → カレントディレクトリ以下を再帰検索
:grep pattern %     → 現在のファイルだけ検索
:grep pattern src/  → src/ ディレクトリ以下を検索
```

<details>
<summary>ヒント</summary>

- `executable()` は指定したコマンドが実行可能かどうかを判定します
- `grepprg` のスペースは `\` でエスケープする必要があります
- `--vimgrep` は ripgrep に Vim 互換の出力形式を指定するオプションです
- `grepformat` は `%f`=ファイル名、`%l`=行番号、`%c`=列番号、`%m`=メッセージです
- `QuickFixCmdPost` イベントで grep 実行後のアクションを設定できます

</details>

<details>
<summary>解答例</summary>

```vim
" ripgrep（rg）がインストールされている場合のみ設定を変更
if executable('rg')
  " grepprg を ripgrep に変更
  " --vimgrep: Vim が解析できる形式で出力（ファイル名:行:列:内容）
  " --smart-case: 小文字のみなら大文字小文字を無視、大文字を含めば区別
  set grepprg=rg\ --vimgrep\ --smart-case

  " ripgrep の出力フォーマットに合わせる
  " %f=ファイル名, %l=行番号, %c=列番号, %m=マッチ内容
  set grepformat=%f:%l:%c:%m
endif

" grep 実行後に自動で Quickfix ウィンドウを開く
augroup GrepAutoOpen
  autocmd!
  " *grep* は :grep, :lgrep, :vimgrep 等すべてにマッチします
  autocmd QuickFixCmdPost *grep* cwindow
augroup END

" 補足：便利なキーマッピング
" カーソル下の単語を grep 検索する
nnoremap <Leader>g :grep <C-r><C-w><CR>

" 補足：ripgrep のインストール
" Ubuntu/Debian: sudo apt install ripgrep
" macOS: brew install ripgrep
" Windows: choco install ripgrep
```

</details>

### 問題5：Quickfix ウィンドウの活用

以下の Quickfix 操作を行うコマンドと、それを便利に使うためのキーマッピングを答えてください。

**Quickfix 操作：**
1. Quickfix ウィンドウを開く
2. Quickfix ウィンドウを閉じる
3. 次のエントリに移動する
4. 前のエントリに移動する
5. 最初のエントリに移動する

**期待されるコマンドとキーマッピング：**
```vim
" コマンド
:copen     " Quickfix を開く
:cclose    " Quickfix を閉じる
:cnext     " 次のエントリ
:cprevious " 前のエントリ
:cfirst    " 最初のエントリ

" キーマッピング
nnoremap <Leader>co :copen<CR>
nnoremap <Leader>cc :cclose<CR>
nnoremap ]q :cnext<CR>
nnoremap [q :cprevious<CR>
```

<details>
<summary>ヒント</summary>

- Quickfix はコンパイルエラーや grep 結果などのリストを管理する機能です
- `]q` / `[q` は「次/前の Quickfix エントリ」を意味する慣習的なマッピングです
- `[` と `]` は Vim のマッピングで「前へ/次へ」を表すのが慣例です

</details>

<details>
<summary>解答例</summary>

```vim
" === Quickfix 操作コマンド ===
" :copen     Quickfix ウィンドウを開く
" :cclose    Quickfix ウィンドウを閉じる
" :cnext     次のエントリに移動（ファイルをまたいで移動）
" :cprevious 前のエントリに移動
" :cfirst    最初のエントリに移動
" :clast     最後のエントリに移動

" === キーマッピング ===
" Quickfix ウィンドウの開閉
nnoremap <Leader>co :copen<CR>
nnoremap <Leader>cc :cclose<CR>

" Quickfix エントリ間の移動
" ]q で次、[q で前に移動します
" この [/] の慣例は多くのプラグインでも採用されています
nnoremap ]q :cnext<CR>
nnoremap [q :cprevious<CR>

" 最初と最後のエントリに移動
nnoremap [Q :cfirst<CR>
nnoremap ]Q :clast<CR>

" 補足：Location List（バッファローカルな Quickfix）の操作
" Quickfix がグローバルなのに対し、Location List はバッファごとに持てます
" nnoremap ]l :lnext<CR>
" nnoremap [l :lprevious<CR>
"
" 補足：Quickfix の典型的なワークフロー
" 1. :grep pattern    → 検索を実行
" 2. :copen           → 結果一覧を表示
" 3. Enter            → 選択したエントリのファイルを開く
" 4. ]q / [q          → 次/前のエントリに移動
" 5. :cclose          → Quickfix を閉じる
```

</details>

### 問題6：置換の設定とテクニック

以下の要件を満たす設定とコマンドを答えてください。

**問1：** 置換コマンド `:substitute` のデフォルト動作を「行内の全マッチを置換」にする `.vimrc` の設定を書いてください。

**問2：** 以下の置換操作を実行するコマンドを答えてください。
1. 現在の行の `foo` を `bar` に置換（全マッチ）
2. ファイル全体の `foo` を `bar` に置換（全マッチ・確認付き）
3. 選択範囲内の `foo` を `bar` に置換

**期待される回答：**
```vim
" 問1
set gdefault

" 問2
:s/foo/bar/g
:%s/foo/bar/gc
:'<,'>s/foo/bar/g
```

<details>
<summary>ヒント</summary>

- `gdefault` を有効にすると、`:s/foo/bar/` だけで行内の全マッチが置換されます（`g` フラグが不要になります）
- `%` はファイル全体を意味する範囲指定です
- `c` フラグは確認（confirm）モードで、1つずつ置換するか選べます
- `'<,'>` はビジュアルモードの選択範囲です

</details>

<details>
<summary>解答例</summary>

```vim
" 問1：gdefault の設定
" デフォルトで行内全マッチを置換する
" これにより :s/foo/bar/ で g フラグと同じ動作になります
set gdefault

" 注意：gdefault 有効時に g フラグを付けると逆に「最初の1つだけ置換」になります
" 混乱を避けるため、gdefault を使わずに毎回 g を付ける流儀もあります

" ---

" 問2-1：現在の行の foo を bar に置換（全マッチ）
:s/foo/bar/g

" 問2-2：ファイル全体の foo を bar に置換（確認付き）
" c フラグで1つずつ確認できます
" y=置換, n=スキップ, a=残り全部置換, q=中止, l=置換して終了
:%s/foo/bar/gc

" 問2-3：選択範囲内の foo を bar に置換
" ビジュアルモードで範囲選択 → : を押すと '<,'> が自動入力されます
:'<,'>s/foo/bar/g

" 補足：よく使う置換テクニック
" カーソル下の単語を置換（入力を省略）
" :%s/<C-r><C-w>/新しい文字列/gc
"
" very magic を使った置換（正規表現が書きやすい）
" :%s/\v(\w+)@(\w+)/\2@\1/g
```

</details>

## チャレンジ問題

### 問題7：総合的な検索・置換環境の構築

以下の要件をすべて満たす `.vimrc` の検索・置換設定を作成してください。

**要件：**
1. 検索の基本設定（ハイライト、インクリメンタルサーチ、スマートケース）
2. `Esc` 2回でハイライト解除
3. 検索コマンドで `very magic` をデフォルトにする
4. ripgrep を `grepprg` に設定（未インストール時はフォールバック）
5. grep 実行後に自動で Quickfix を開く
6. Quickfix のナビゲーション用キーマッピング（`]q`, `[q`）
7. カーソル下の単語をプロジェクト全体から grep 検索するキーマッピング
8. 選択した文字列をプロジェクト全体から grep 検索するキーマッピング（ビジュアルモード）

**期待される .vimrc の構成：**
```vim
" === 検索基本設定 ===
...

" === ハイライト解除 ===
...

" === very magic デフォルト化 ===
...

" === grepprg 設定 ===
...

" === Quickfix 自動オープン ===
...

" === キーマッピング ===
...
```

<details>
<summary>ヒント</summary>

- ビジュアルモードで選択中のテキストを取得するには、選択範囲をヤンクしてレジスタから読む方法があります
- `<C-r>"` でデフォルトレジスタの内容をコマンドラインに挿入できます
- `<C-r><C-w>` でカーソル下の単語をコマンドラインに挿入できます
- ビジュアルモード用のマッピングは `vnoremap` で定義します

</details>

<details>
<summary>解答例</summary>

```vim
" =============================================================================
" 検索・置換設定
" =============================================================================

" === 検索基本設定 ===
set hlsearch       " 検索結果をハイライト
set incsearch      " インクリメンタルサーチ
set wrapscan       " 末尾から先頭へ折り返す
set ignorecase     " 大文字小文字を無視
set smartcase      " 大文字を含む場合は区別する

" === ハイライト解除 ===
" Esc 2回で検索ハイライトを消す
nnoremap <Esc><Esc> :nohlsearch<CR>

" === very magic デフォルト化 ===
" 検索パターンに \v を自動付与し、正規表現を書きやすくする
nnoremap / /\v
vnoremap / /\v

" === grepprg 設定 ===
" ripgrep がインストールされていれば使用、なければデフォルト grep
if executable('rg')
  set grepprg=rg\ --vimgrep\ --smart-case
  set grepformat=%f:%l:%c:%m
endif

" === Quickfix 自動オープン ===
augroup GrepQuickfix
  autocmd!
  autocmd QuickFixCmdPost *grep* cwindow
augroup END

" === キーマッピング ===

" --- Quickfix ナビゲーション ---
nnoremap ]q :cnext<CR>      " 次のエントリ
nnoremap [q :cprevious<CR>  " 前のエントリ
nnoremap [Q :cfirst<CR>     " 最初のエントリ
nnoremap ]Q :clast<CR>      " 最後のエントリ
nnoremap <Leader>co :copen<CR>   " Quickfix を開く
nnoremap <Leader>cc :cclose<CR>  " Quickfix を閉じる

" --- プロジェクト全体の検索 ---
" ノーマルモード: カーソル下の単語を grep 検索
" <C-r><C-w> でカーソル下の単語をコマンドラインに挿入します
nnoremap <Leader>g :grep <C-r><C-w><CR>

" ビジュアルモード: 選択した文字列を grep 検索
" y で選択範囲をヤンクし、<C-r>" でレジスタの内容を挿入します
vnoremap <Leader>g y:grep <C-r>"<CR>

" 補足：
" - vnoremap のマッピングでは、まず y でヤンクしてから
"   :grep コマンドにレジスタの内容を貼り付けています
" - 検索文字列に特殊文字が含まれる場合は
"   fnameescape() や shellescape() で処理するとより安全です
" - 実務ではこれらの設定をベースに、
"   fzf.vim や telescope.nvim などの検索プラグインを
"   追加するのが一般的です
```

</details>
