# 第1章：Claude Code概論と環境構築

> **この章のゴール**: Claude Codeの概要を理解し、インストールから初回起動までの環境構築を完了できるようになること

## 1.1 Claude Codeとは

Claude Codeは、Anthropic社が提供する**公式AIコーディングツール**です。ターミナル（terminal）上で動作するCLI（Command Line Interface：コマンドラインインターフェース）ツールであり、自然言語でプログラミングに関する指示を出すと、AIがコードの読み書き・実行・Git操作などを自動的に行ってくれます。

### Claude Codeの特徴

```
┌─────────────────────────────────────────────────────┐
│                   Claude Code                        │
│                                                      │
│  ┌───────────┐  ┌───────────┐  ┌───────────────┐   │
│  │ ファイル   │  │ Git操作   │  │ コマンド実行  │   │
│  │ 読み書き   │  │ コミット  │  │ テスト・ビルド│   │
│  └─────┬─────┘  └─────┬─────┘  └──────┬────────┘   │
│        │              │               │              │
│        └──────────────┼───────────────┘              │
│                       │                              │
│              ┌────────▼────────┐                     │
│              │  Claude AI モデル │                    │
│              │  (Opus/Sonnet)   │                    │
│              └─────────────────┘                     │
└─────────────────────────────────────────────────────┘
```

主な特徴は以下の通りです：

| 特徴 | 説明 |
|---|---|
| **エージェント型** | 単なるコード補完ではなく、自律的にタスクを遂行します |
| **ターミナルネイティブ** | CLI上で動作し、既存の開発ワークフローに統合できます |
| **マルチツール** | ファイル操作、Git、コマンド実行など多数のツールを使えます |
| **コンテキスト理解** | プロジェクト全体を把握してコードを書きます |
| **安全設計** | 危険な操作は確認を求め、ユーザーの承認なしに実行しません |

```
└──────────────────┴──────────────┴───────────────┴──────────────┘
```

Claude Codeの最大の特徴は、**エディタに依存しない**点です。ターミナルさえあれば、どのエディタやIDEを使っていても利用できます。また、VS CodeやJetBrains向けのIDE拡張（extension）も提供されており、エディタ内から直接利用することも可能です。

### ポイントまとめ
- Claude Codeはターミナルで動作するAIコーディングエージェント
- ファイル操作・Git・コマンド実行など幅広いタスクを自律的に遂行
- エディタに依存せず、既存のワークフローに統合可能

## 1.2 動作の仕組み

Claude Codeは以下のような流れで動作します：

```
ユーザー                    Claude Code                  Claude API
  │                           │                            │
  │  「バグを修正して」        │                            │
  │ ─────────────────────────>│                            │
  │                           │  プロンプト送信             │
  │                           │ ──────────────────────────>│
  │                           │                            │
  │                           │  ツール呼び出し指示         │
  │                           │ <──────────────────────────│
  │                           │                            │
  │  [Read] ファイル読み込み？│                            │
  │ <─────────────────────────│                            │
  │  承認                     │                            │
  │ ─────────────────────────>│                            │
  │                           │  ファイル内容 + 続き        │
  │                           │ ──────────────────────────>│
  │                           │                            │
  │                           │  編集指示                  │
  │                           │ <──────────────────────────│
  │                           │                            │
  │  [Edit] コード修正？      │                            │
  │ <─────────────────────────│                            │
  │  承認                     │                            │
  │ ─────────────────────────>│                            │
  │                           │  完了報告                  │
  │                           │ ──────────────────────────>│
  │                           │                            │
  │  修正完了しました         │                            │
  │ <─────────────────────────│                            │
```

1. ユーザーが自然言語で指示を入力します
2. Claude Codeが指示をClaude APIに送信します
3. AIモデルが必要なツール（ファイル読み込み、編集など）を判断します
4. ユーザーに承認を求めた後、ツールを実行します
5. 結果をAIモデルに返し、次のアクションを決定します
6. タスクが完了するまでこのループを繰り返します

### 権限モデル（Permission Model）

Claude Codeは安全性を重視した設計になっています。ツールの実行には以下の3つのモードがあります：

| モード | 説明 |
|---|---|
| **確認モード** | すべてのツール使用時にユーザーに確認を求めます（デフォルト） |
| **許可リスト** | 指定したツールのみ自動実行を許可します |
| **自動承認** | すべてのツールを自動実行します（上級者向け） |

### ポイントまとめ
- Claude Codeはユーザーの指示→AIの判断→ツール実行のループで動作
- 安全のため、ツール実行前にユーザーの承認を求める仕組み
- 権限モードを調整して、作業効率と安全性のバランスを取れる

## 1.3 環境構築

### 前提条件

Claude Codeを使用するには、以下が必要です：

| 要件 | 詳細 |
|---|---|
| **Node.js** | バージョン18以上 |
| **npm** | Node.jsに同梱 |
| **OS** | macOS、Linux、またはWindows（WSL経由） |
| **APIキー** | Anthropic APIキー、またはClaude Pro/Teamプラン |

### Node.jsの確認

まず、Node.jsがインストールされているか確認しましょう：

```bash
# Node.jsのバージョン確認
node --version
# 出力例: v20.11.0

# npmのバージョン確認
npm --version
# 出力例: 10.2.4
```

Node.jsがインストールされていない場合は、公式サイト（https://nodejs.org/）からインストールしてください。

### よくある間違い
> **Node.jsのバージョンが古い**: Claude Codeはv18以上が必要です。`node --version`で確認し、古い場合はアップデートしてください。nvm（Node Version Manager）を使うと、複数のバージョンを管理しやすくなります。

### インストール

```bash
# Claude Codeをグローバルにインストール
npm install -g @anthropic-ai/claude-code
```

インストールが完了したら、バージョンを確認します：

```bash
# バージョン確認
claude --version
```

### 認証の設定

Claude Codeを使用するには認証が必要です。主に2つの方法があります：

#### 方法1：Anthropic APIキーを使用

```bash
# 環境変数でAPIキーを設定
export ANTHROPIC_API_KEY="sk-ant-xxxxxxxxxxxxx"

# または、初回起動時に対話的に設定
claude
# → APIキーの入力を求められます
```

#### 方法2：Claude Pro/Teamプランを使用

Claude ProまたはTeamプランのサブスクリプション（subscription：定額課金）をお持ちの場合、Webブラウザ経由のOAuth認証（OAuth authentication）で利用できます。

```bash
# 初回起動
claude
# → ブラウザが開き、ログイン画面が表示されます
# → Claudeアカウントでログインすると認証が完了します
```

### よくある間違い
> **APIキーの取り扱い**: APIキーは秘密情報です。コードにハードコーディングしたり、Gitリポジトリにコミットしたりしないでください。環境変数や`.env`ファイル（.gitignoreに追加）で管理しましょう。

### ポイントまとめ
- Node.js v18以上が必要
- `npm install -g @anthropic-ai/claude-code`でインストール
- APIキーまたはClaude Pro/Teamプランで認証

## 1.4 初回起動と基本動作確認

### Claude Codeの起動

ターミナルで任意のプロジェクトディレクトリに移動し、`claude`コマンドを実行します：

```bash
# プロジェクトディレクトリに移動
cd ~/my-project

# Claude Codeを起動
claude
```

起動すると、以下のような対話画面が表示されます：

```
╭──────────────────────────────────────────╮
│ ✻ Welcome to Claude Code!                │
│                                          │
│   /help for help, /status for status     │
│                                          │
╰──────────────────────────────────────────╯

>
```

### 最初のプロンプト

簡単な指示を入力してみましょう：

```
> このプロジェクトの構成を教えてください
```

Claude Codeは自動的にプロジェクトのファイル構造を調べ、説明してくれます。この際、以下のようなツール呼び出しが行われます：

```
Claude Code が Bash(ls -la) を実行しようとしています。許可しますか？
[Y]es / [N]o / [A]lways allow
```

`Y`を入力すると、コマンドが実行されます。`A`を選ぶと、同じ種類のツールは以降自動的に許可されます。

### ワンショットモード（One-shot Mode）

対話モードではなく、1回だけ指示を出して結果を得ることもできます：

```bash
# -p フラグでワンショットモード
claude -p "このプロジェクトのREADMEを要約して"

# パイプで入力を渡すことも可能
cat error.log | claude -p "このエラーログの原因を分析して"
```

### 利用可能な環境

Claude Codeは以下の環境で利用できます：

```
┌─────────────────────────────────────────────────┐
│              Claude Code の利用環境               │
├─────────────────┬───────────────────────────────┤
│ ターミナルCLI   │ claude コマンドで直接利用      │
│ VS Code拡張     │ エディタ内のパネルで利用       │
│ JetBrains拡張   │ IntelliJ等のIDE内で利用        │
│ Web版           │ claude.ai/code で利用          │
│ デスクトップアプリ│ Mac/Windows向けアプリで利用   │
└─────────────────┴───────────────────────────────┘
```

### よくある間違い
> **プロジェクトディレクトリ外での起動**: Claude Codeは起動したディレクトリをプロジェクトのルートとして認識します。必ず作業したいプロジェクトのディレクトリで起動してください。

## 1.5 APIの料金と使用量の管理

Claude CodeはAPIベースで動作するため、使用量に応じた料金が発生します（Claude Pro/Teamプランの場合は月額料金に含まれます）。

### コストの確認

```bash
# セッション中にコストを確認
> /cost
```

`/cost`スラッシュコマンド（slash command）を使うと、現在のセッションでの使用トークン数と概算コストを確認できます。

### コスト管理のコツ

| 方法 | 説明 |
|---|---|
| **具体的な指示** | 曖昧な指示は試行錯誤が増えてコストが上がります |
| **コンテキスト制限** | 必要なファイルだけを参照させましょう |
| **モデル選択** | Sonnetモデルの方がOpusより安価です |
| **/compact** | 会話が長くなったら/compactで圧縮できます |

### ポイントまとめ
- Claude CodeはAPI使用量に応じて課金される
- `/cost`でセッション中のコストを確認できる
- 具体的な指示とコンテキスト制限でコストを最適化

## 1.6 トラブルシューティング

### よくある問題と解決策

#### インストールに失敗する

```bash
# 権限エラーの場合
sudo npm install -g @anthropic-ai/claude-code

# npmキャッシュのクリア
npm cache clean --force
```

#### APIキーが認証されない

```bash
# APIキーが正しく設定されているか確認
echo $ANTHROPIC_API_KEY

# シェルの設定ファイルに追加（永続化）
echo 'export ANTHROPIC_API_KEY="sk-ant-xxxxx"' >> ~/.bashrc
source ~/.bashrc
```

#### コマンドが見つからない

```bash
# npmのグローバルパスを確認
npm config get prefix

# PATHに追加されているか確認
echo $PATH
```

### よくある間違い
> **WSLでの利用時**: Windows上で直接実行するのではなく、WSL（Windows Subsystem for Linux）環境内で実行してください。PowerShellやコマンドプロンプトでは正常に動作しない場合があります。
