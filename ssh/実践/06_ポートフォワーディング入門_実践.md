# 実践課題06：ポートフォワーディング入門 ★3

> **難易度**: ★★★☆☆（中級）
> **前提知識**: 第2章（SSHクライアントの基本操作）、第4章（SSH設定ファイル）、第6章（ポートフォワーディング）
> **課題の種類**: ミニプロジェクト
> **学習目標**: ローカル・リモート・ダイナミックの3種類のポートフォワーディングを設定し、SSHトンネルの仕組みと実務での活用パターンを身につける

---

## 完成イメージ

```
# ローカルポートフォワーディング：リモートのDB(3306)にローカルからアクセス
$ ssh -L 13306:localhost:3306 user@db-server -N
# 別ターミナルで
$ mysql -h 127.0.0.1 -P 13306 -u dbuser -p

# リモートポートフォワーディング：ローカルのWebアプリをリモートに公開
$ ssh -R 8080:localhost:3000 user@public-server -N
# public-server:8080 でローカルの3000番ポートにアクセス可能

# ダイナミックフォワーディング：SOCKSプロキシとして使用
$ ssh -D 1080 user@proxy-server -N
# ブラウザのプロキシ設定で SOCKS5 localhost:1080 を指定
```

---

## 課題の要件

1. ローカルポートフォワーディング（`-L`）を設定し、リモートサービスにローカルからアクセスする
2. リモートポートフォワーディング（`-R`）を設定し、ローカルサービスをリモートに公開する
3. ダイナミックフォワーディング（`-D`）を設定し、SOCKSプロキシ（SOCKS proxy）として使用する
4. `~/.ssh/config` にポートフォワーディングの設定を記述する
5. トンネルの稼働状況を確認する方法を理解する

---

## ステップガイド

<details>
<summary>ステップ1：ローカルポートフォワーディング（-L）を試す</summary>

ローカルのポートへのアクセスを、SSHサーバー経由でリモートのサービスに転送します。

```
+----------+          +----------+          +----------+
| ローカル  |   SSH    | SSHサーバー|          | DBサーバー |
| PC       |=========>|          |--------->|          |
| :13306   |  暗号化   |          |          | :3306    |
+----------+          +----------+          +----------+
```

```bash
# 構文: ssh -L ローカルポート:転送先ホスト:転送先ポート SSHサーバー
# -N: コマンドを実行しない（トンネルのみ）
# -f: バックグラウンドで実行

# 例：リモートのMySQL(3306)にローカルの13306でアクセス
$ ssh -L 13306:localhost:3306 user@db-server -N

# 別のターミナルで接続テスト
$ mysql -h 127.0.0.1 -P 13306 -u dbuser -p
```

**よくある使い方：**
- リモートのデータベースにローカルのGUIツールで接続
- ファイアウォールの内側にあるWebアプリにアクセス
- リモートのRedis・Elasticsearch等にローカルから接続

</details>

<details>
<summary>ステップ2：リモートポートフォワーディング（-R）を試す</summary>

リモートのポートへのアクセスを、ローカルのサービスに転送します。

```
+----------+          +----------+
| ローカル  |   SSH    | リモート   |
| :3000    |<=======  | :8080    |
| (Webアプリ)| 暗号化   | (外部公開) |
+----------+          +----------+
```

```bash
# 構文: ssh -R リモートポート:転送先ホスト:転送先ポート SSHサーバー

# 例：ローカルのWebアプリ(3000)をリモートの8080で公開
$ ssh -R 8080:localhost:3000 user@public-server -N
```

**よくある使い方：**
- ローカルで開発中のWebアプリをチームメンバーに見せる
- NATの内側にいるPCにリモートからアクセスする

</details>

<details>
<summary>ステップ3：ダイナミックフォワーディング（-D）を試す</summary>

SOCKSプロキシとして動作し、任意の宛先への通信を転送します。

```bash
# 構文: ssh -D ローカルポート SSHサーバー

# 例：SOCKS5プロキシをローカルの1080番で起動
$ ssh -D 1080 user@proxy-server -N
```

ブラウザの設定でSOCKS5プロキシ `localhost:1080` を指定すると、すべてのWeb通信がSSHサーバー経由になります。

**よくある使い方：**
- 社内ネットワークからのみアクセスできるサイトに外部からアクセス
- 通信の暗号化（公共Wi-Fi使用時）

</details>

<details>
<summary>ステップ4：~/.ssh/config に設定を書く</summary>

毎回長いオプションを入力する代わりに、設定ファイルに記述できます。

```
# ローカルポートフォワーディング
Host db-tunnel
    HostName db-server.example.com
    User myuser
    LocalForward 13306 localhost:3306
    RequestTTY no

# リモートポートフォワーディング
Host expose-local
    HostName public-server.example.com
    User myuser
    RemoteForward 8080 localhost:3000
    RequestTTY no

# ダイナミックフォワーディング
Host socks-proxy
    HostName proxy-server.example.com
    User myuser
    DynamicForward 1080
    RequestTTY no
```

使い方：
```bash
$ ssh -N db-tunnel      # DBトンネル起動
$ ssh -N expose-local   # ローカルサービス公開
$ ssh -N socks-proxy    # SOCKSプロキシ起動
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け）─ 各種トンネルの起動と確認</summary>

```bash
#!/bin/bash
# SSHポートフォワーディング練習スクリプト
# 学べる内容：ローカル/リモート/ダイナミックフォワーディング
# 実行方法：bash port_forwarding_practice.sh <ユーザー名> <サーバー>

USER="$1"
SERVER="$2"
REMOTE="${USER}@${SERVER}"

if [ -z "$USER" ] || [ -z "$SERVER" ]; then
    echo "使い方: $0 <ユーザー名> <サーバー>"
    exit 1
fi

echo "===== ポートフォワーディング練習 ====="

# --- ローカルポートフォワーディング ---
echo ""
echo "--- 1. ローカルポートフォワーディング ---"
echo "リモートの80番ポートをローカルの8080で公開します。"
echo "コマンド: ssh -L 8080:localhost:80 $REMOTE -N -f"
ssh -L 8080:localhost:80 "$REMOTE" -N -f
echo "起動しました。http://localhost:8080 でアクセスできます。"

# 接続テスト
echo "接続テスト..."
if curl -s -o /dev/null -w "%{http_code}" http://localhost:8080 2>/dev/null | grep -q "200\|301\|302"; then
    echo "  [OK] ローカルポートフォワーディング成功"
else
    echo "  [情報] リモートの80番ポートにWebサーバーが動いていない可能性があります"
fi

# --- ダイナミックフォワーディング ---
echo ""
echo "--- 2. ダイナミックフォワーディング ---"
echo "SOCKSプロキシをローカルの1080番で起動します。"
echo "コマンド: ssh -D 1080 $REMOTE -N -f"
ssh -D 1080 "$REMOTE" -N -f
echo "起動しました。SOCKSプロキシ: localhost:1080"

# プロキシ経由テスト
echo "プロキシ経由テスト..."
if curl -s --socks5 localhost:1080 http://example.com > /dev/null 2>&1; then
    echo "  [OK] ダイナミックフォワーディング成功"
else
    echo "  [情報] プロキシ経由の接続テストに失敗しました"
fi

# --- 稼働状況の確認 ---
echo ""
echo "--- 3. トンネルの稼働状況 ---"
echo "SSHトンネルのプロセス一覧:"
ps aux | grep "ssh -[LRD]" | grep -v grep || echo "  稼働中のトンネルはありません"

echo ""
echo "リスニング中のポート:"
ss -tlnp 2>/dev/null | grep -E "8080|1080" || \
    netstat -tlnp 2>/dev/null | grep -E "8080|1080" || \
    echo "  （ss/netstat コマンドが利用できません）"

# --- クリーンアップ ---
echo ""
echo "--- クリーンアップ ---"
echo "トンネルを停止するには:"
echo "  kill \$(pgrep -f 'ssh -L 8080')"
echo "  kill \$(pgrep -f 'ssh -D 1080')"

echo ""
echo "===== 完了 ====="
```

<details>
<summary>解答例（改良版）─ トンネル管理ツール</summary>

```bash
#!/bin/bash
# SSHトンネル管理ツール（改良版）
# 学べる内容：トンネルのライフサイクル管理、PIDファイル、ヘルスチェック
# 実行方法：bash ssh_tunnel_manager.sh [start|stop|status|config]

set -euo pipefail

TUNNEL_DIR="$HOME/.ssh/tunnels"
CONFIG_FILE="$TUNNEL_DIR/tunnels.conf"

# === 初期化 ===

mkdir -p "$TUNNEL_DIR"

# 設定ファイルがなければサンプルを作成
if [ ! -f "$CONFIG_FILE" ]; then
    cat > "$CONFIG_FILE" << 'EOF'
# SSHトンネル設定ファイル
# 形式: 名前 種類(L/R/D) ローカルポート リモートホスト:リモートポート SSHサーバー
# 例:
db-tunnel L 13306 localhost:3306 user@db-server
web-tunnel L 8080 localhost:80 user@web-server
socks-proxy D 1080 - user@proxy-server
EOF
fi

# === ユーティリティ関数 ===

get_pid_file() {
    echo "$TUNNEL_DIR/$1.pid"
}

is_running() {
    local name="$1"
    local pid_file
    pid_file=$(get_pid_file "$name")

    if [ -f "$pid_file" ]; then
        local pid
        pid=$(cat "$pid_file")
        if kill -0 "$pid" 2>/dev/null; then
            return 0
        else
            rm -f "$pid_file"
        fi
    fi
    return 1
}

start_tunnel() {
    local name="$1"
    local type="$2"
    local local_port="$3"
    local remote="$4"
    local ssh_server="$5"
    local pid_file
    pid_file=$(get_pid_file "$name")

    if is_running "$name"; then
        echo "  [スキップ] $name は既に稼働中です (PID: $(cat "$pid_file"))"
        return
    fi

    local ssh_opts="-N -f -o ServerAliveInterval=30 -o ServerAliveCountMax=3 -o ExitOnForwardFailure=yes"

    case "$type" in
        L)
            ssh $ssh_opts -L "${local_port}:${remote}" "$ssh_server"
            ;;
        R)
            ssh $ssh_opts -R "${local_port}:${remote}" "$ssh_server"
            ;;
        D)
            ssh $ssh_opts -D "$local_port" "$ssh_server"
            ;;
        *)
            echo "  [エラー] 不明なトンネル種類: $type"
            return 1
            ;;
    esac

    # PIDを取得して保存
    local pid
    pid=$(pgrep -f "ssh.*-${type}.*${local_port}.*${ssh_server}" | tail -1)
    if [ -n "$pid" ]; then
        echo "$pid" > "$pid_file"
        echo "  [起動] $name (PID: $pid, ポート: $local_port)"
    else
        echo "  [エラー] $name の起動に失敗しました"
    fi
}

stop_tunnel() {
    local name="$1"
    local pid_file
    pid_file=$(get_pid_file "$name")

    if ! is_running "$name"; then
        echo "  [スキップ] $name は稼働していません"
        return
    fi

    local pid
    pid=$(cat "$pid_file")
    kill "$pid" 2>/dev/null
    rm -f "$pid_file"
    echo "  [停止] $name (PID: $pid)"
}

show_status() {
    echo "--- SSHトンネル稼働状況 ---"
    echo ""

    local found=false
    for pid_file in "$TUNNEL_DIR"/*.pid; do
        [ -f "$pid_file" ] || continue
        found=true
        local name
        name=$(basename "$pid_file" .pid)
        local pid
        pid=$(cat "$pid_file")

        if kill -0 "$pid" 2>/dev/null; then
            # プロセスの詳細を取得
            local cmd
            cmd=$(ps -p "$pid" -o args= 2>/dev/null || echo "情報取得不可")
            echo "  [$name]"
            echo "    状態: 稼働中"
            echo "    PID: $pid"
            echo "    コマンド: $cmd"
        else
            echo "  [$name]"
            echo "    状態: 停止（PIDファイルが残っています）"
            rm -f "$pid_file"
        fi
        echo ""
    done

    if [ "$found" = false ]; then
        echo "  稼働中のトンネルはありません。"
    fi

    echo ""
    echo "--- リスニングポート ---"
    ss -tlnp 2>/dev/null | head -20 || \
        netstat -tlnp 2>/dev/null | head -20 || \
        echo "  （ポート情報の取得にはss/netstatが必要です）"
}

# === メイン処理 ===

ACTION="${1:-help}"

case "$ACTION" in
    start)
        echo "===== SSHトンネル起動 ====="
        # 設定ファイルからトンネルを読み込んで起動
        while IFS=' ' read -r name type local_port remote ssh_server; do
            # コメント行と空行をスキップ
            [[ "$name" =~ ^#.*$ ]] && continue
            [ -z "$name" ] && continue
            start_tunnel "$name" "$type" "$local_port" "$remote" "$ssh_server"
        done < "$CONFIG_FILE"
        ;;

    stop)
        echo "===== SSHトンネル停止 ====="
        for pid_file in "$TUNNEL_DIR"/*.pid; do
            [ -f "$pid_file" ] || continue
            local name
            name=$(basename "$pid_file" .pid)
            stop_tunnel "$name"
        done
        ;;

    status)
        echo "===== SSHトンネル状況確認 ====="
        show_status
        ;;

    config)
        echo "===== トンネル設定ファイル ====="
        echo "ファイル: $CONFIG_FILE"
        echo ""
        cat "$CONFIG_FILE"
        ;;

    help|*)
        echo "===== SSHトンネル管理ツール ====="
        echo ""
        echo "使い方: $0 [コマンド]"
        echo ""
        echo "コマンド:"
        echo "  start   設定ファイルのトンネルを全て起動"
        echo "  stop    稼働中のトンネルを全て停止"
        echo "  status  トンネルの稼働状況を表示"
        echo "  config  設定ファイルを表示"
        echo ""
        echo "設定ファイル: $CONFIG_FILE"
        ;;
esac

echo ""
echo "===== 完了 ====="
```

**初心者向けとの違い:**
- PIDファイルでトンネルのライフサイクル管理 → 停止忘れや二重起動を防止
- 設定ファイルからトンネルを一括管理 → 定義の追加・削除が容易
- `ExitOnForwardFailure=yes` でポート競合時に即座にエラー → 問題を見逃さない
- start/stop/status のサブコマンド方式 → 実務のツールと同じ操作感

</details>
