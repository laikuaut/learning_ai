# 第2章: インターフェースと型エイリアス

## 2.1 インターフェース（interface）とは

インターフェースは、オブジェクトの「形状（shape）」を定義する仕組みです。どんなプロパティがあり、それぞれどんな型を持つかを宣言します。

```typescript
// インターフェースの基本
interface User {
  name: string;
  age: number;
  email: string;
}

// インターフェースに従ったオブジェクト
const user: User = {
  name: "田中太郎",
  age: 30,
  email: "tanaka@example.com",
};

// プロパティが足りない → エラー
// const badUser: User = { name: "太郎" }; // エラー！age, emailがない

// 余分なプロパティ → エラー（オブジェクトリテラルの場合）
// const badUser2: User = { name: "太郎", age: 30, email: "...", phone: "..." }; // エラー！
```

## 2.2 オプショナルプロパティ（?）

`?` を付けると、そのプロパティは省略可能になります。

```typescript
interface Config {
  host: string;
  port: number;
  protocol?: string;  // 省略可能
  timeout?: number;    // 省略可能
}

// protocolとtimeoutは省略可能
const config1: Config = {
  host: "localhost",
  port: 3000,
};

const config2: Config = {
  host: "localhost",
  port: 3000,
  protocol: "https",
  timeout: 5000,
};

// オプショナルプロパティにアクセスする際の注意
function getTimeout(config: Config): number {
  // config.timeout は number | undefined になる
  return config.timeout ?? 3000; // Nullish Coalescingで安全にデフォルト値を設定
}
```

---

## 2.3 readonlyプロパティ

`readonly` を付けると、プロパティの変更を禁止できます。

```typescript
interface Point {
  readonly x: number;
  readonly y: number;
}

const origin: Point = { x: 0, y: 0 };
// origin.x = 10; // エラー！readonlyプロパティは変更できない

// ReadonlyArray
interface ImmutableList {
  readonly items: readonly string[];
}

const list: ImmutableList = { items: ["a", "b", "c"] };
// list.items = ["d"]; // エラー！
// list.items.push("d"); // エラー！readonly配列はpushできない
```

---

## 2.4 インターフェースの拡張（extends）

`extends` を使って、既存のインターフェースを拡張できます。

```typescript
// 基本のインターフェース
interface Animal {
  name: string;
  age: number;
}

// Animalを拡張
interface Dog extends Animal {
  breed: string;       // 犬種
  isVaccinated: boolean;
}

const myDog: Dog = {
  name: "ポチ",
  age: 3,
  breed: "柴犬",
  isVaccinated: true,
};

// 複数のインターフェースを同時に拡張
interface Loggable {
  log(): void;
}

interface Serializable {
  toJSON(): string;
}

interface Entity extends Loggable, Serializable {
  id: number;
  name: string;
}

// Entityは id, name, log(), toJSON() を全て持つ必要がある
const entity: Entity = {
  id: 1,
  name: "テスト",
  log() {
    console.log(`Entity: ${this.id} - ${this.name}`);
  },
  toJSON() {
    return JSON.stringify({ id: this.id, name: this.name });
  },
};
```

## 2.5 インデックスシグネチャ

キーの名前は不明だが型が分かっている場合に使います。

```typescript
// 文字列キーのインデックスシグネチャ
interface Dictionary {
  [key: string]: string;
}

const dict: Dictionary = {
  hello: "こんにちは",
  goodbye: "さようなら",
  thanks: "ありがとう",
};

// 数値キーのインデックスシグネチャ
interface NumberArray {
  [index: number]: string;
}

const arr: NumberArray = ["a", "b", "c"];
console.log(arr[0]); // "a"

// 固定プロパティとインデックスシグネチャの併用
interface UserMap {
  // 固定プロパティ
  count: number;
  // インデックスシグネチャ（固定プロパティの型を含む必要あり）
  [userId: string]: number | string; // countがnumberなので含める
}

// Record型を使ったより安全なパターン（第7章で詳しく解説）
type SafeUserMap = Record<string, string>;
```

## 2.6 型エイリアス（type）

`type` キーワードを使って、型に名前を付けることができます。

```typescript
// 基本的な型エイリアス
type ID = string | number;
type Name = string;
type Age = number;

// オブジェクト型の型エイリアス
type User = {
  id: ID;
  name: Name;
  age: Age;
};

const user: User = {
  id: "user-001",
  name: "太郎",
  age: 25,
};

// ユニオン型の型エイリアス
type Status = "active" | "inactive" | "pending";
type Result = "success" | "error";

// 関数型の型エイリアス
type Callback = (data: string) => void;
type AsyncCallback = (data: string) => Promise<void>;

// タプル型の型エイリアス
type Coordinate = [number, number];
type NameAge = [name: string, age: number];

const point: Coordinate = [10, 20];
```

## 2.7 型エイリアスの交差（&）による拡張

型エイリアスは `extends` の代わりに交差型（`&`）で拡張できます。

```typescript
type Animal = {
  name: string;
  age: number;
};

type Pet = Animal & {
  owner: string;
  isVaccinated: boolean;
};

const myPet: Pet = {
  name: "タマ",
  age: 5,
  owner: "山田花子",
  isVaccinated: true,
};

// 複数の型を交差
type Timestamped = {
  createdAt: Date;
  updatedAt: Date;
};

type SoftDeletable = {
  deletedAt: Date | null;
};

type BaseEntity = {
  id: number;
};

// 全部を合体
type FullEntity = BaseEntity & Timestamped & SoftDeletable;
```

## 2.8 interface と type の違い

### 宣言マージ（Declaration Merging）

interfaceは同じ名前で複数回宣言すると自動的にマージされます。typeではできません。

```typescript
// interfaceの宣言マージ
interface Window {
  title: string;
}

interface Window {
  appVersion: string;
}

// WindowはtitleとappVersionの両方を持つ
const win: Window = {
  title: "MyApp",
  appVersion: "1.0.0",
};

// typeでは同じ名前で再宣言するとエラー
// type Point = { x: number };
// type Point = { y: number }; // エラー！識別子 'Point' が重複しています
```

### 使い分けの指針

| 特徴 | `interface` | `type` |
|------|------------|--------|
| オブジェクトの形状定義 | 得意 | 可能 |
| 拡張方法 | `extends` | `&`（交差型） |
| 宣言マージ | できる | できない |
| ユニオン型の定義 | できない | できる |
| プリミティブ型に名前を付ける | できない | できる |
| タプル型の定義 | できない | できる |
| `implements` で使う | できる | できる |
| 計算プロパティ | できない | できる |

### 推奨される使い分け

```typescript
// ★ オブジェクトの形状（APIレスポンス、モデルなど） → interface
interface UserResponse {
  id: number;
  name: string;
  email: string;
}

// ★ ユニオン型、タプル型、プリミティブのエイリアス → type
type Status = "success" | "error" | "pending";
type ID = string | number;
type Pair = [string, number];

// ★ 関数型 → type
type Handler = (event: Event) => void;

// ★ 既存の型を変換（ユーティリティ型の適用） → type
type ReadonlyUser = Readonly<UserResponse>;
type PartialUser = Partial<UserResponse>;
```

## 2.9 インターフェースでの関数定義

```typescript
// 方法1: メソッド構文
interface Calculator1 {
  add(a: number, b: number): number;
  subtract(a: number, b: number): number;
}

// 方法2: プロパティ構文（関数型）
interface Calculator2 {
  add: (a: number, b: number) => number;
  subtract: (a: number, b: number) => number;
}

// 実装
const calc: Calculator1 = {
  add(a, b) {
    return a + b;
  },
  subtract(a, b) {
    return a - b;
  },
};

// コーラブル（呼び出し可能な）インターフェース
interface Formatter {
  (input: string): string;    // 関数として呼び出せる
  locale: string;              // プロパティも持てる
}

const upperFormatter: Formatter = Object.assign(
  (input: string) => input.toUpperCase(),
  { locale: "ja-JP" }
);

console.log(upperFormatter("hello")); // "HELLO"
console.log(upperFormatter.locale);    // "ja-JP"
```

## 2.10 実践的なインターフェース設計

```typescript
// APIレスポンスの型定義
interface ApiResponse<T> {
  status: number;
  message: string;
  data: T;
  timestamp: string;
}

interface PaginationInfo {
  currentPage: number;
  totalPages: number;
  totalItems: number;
  itemsPerPage: number;
}

interface PaginatedResponse<T> extends ApiResponse<T[]> {
  pagination: PaginationInfo;
}

// 使用例
interface Product {
  id: number;
  name: string;
  price: number;
  category: string;
}

// 単一のProductを返すレスポンス
type SingleProductResponse = ApiResponse<Product>;

// Product一覧をページネーション付きで返すレスポンス
type ProductListResponse = PaginatedResponse<Product>;

// 具体的な値の例
const response: ProductListResponse = {
  status: 200,
  message: "OK",
  data: [
    { id: 1, name: "TypeScript入門書", price: 3000, category: "書籍" },
    { id: 2, name: "プログラミングキーボード", price: 15000, category: "周辺機器" },
  ],
  timestamp: "2025-01-01T00:00:00Z",
  pagination: {
    currentPage: 1,
    totalPages: 5,
    totalItems: 50,
    itemsPerPage: 10,
  },
};
```

## 2.11 まとめ

| トピック | 内容 |
|---------|------|
| interface | オブジェクトの形状を定義する |
| オプショナル（?） | プロパティを省略可能にする |
| readonly | プロパティを変更不可にする |
| extends | インターフェースの拡張 |
| インデックスシグネチャ | 動的なキーの型を定義 |
| type | 型に名前を付ける |
| interface vs type | 用途に応じて使い分ける |
