# 第3章：ファイル操作とコード編集

> **この章のゴール**: Claude Codeのツール体系を理解し、ファイルの検索・読み込み・作成・編集を効率的に行えるようになること

---

## 3.1 Claude Codeのツール体系

Claude Codeは内部的に複数の**ツール**（tools）を持っており、ユーザーの指示に応じて適切なツールを自動選択して実行します。

### 主要ツール一覧

```
┌────────────────────────────────────────────────────────────┐
│                Claude Code ツール体系                       │
├──────────┬─────────────────────────────────────────────────┤
│          │                                                 │
│  読み取り│  Read   ─── ファイルの内容を読み込む            │
│  系      │  Glob   ─── パターンでファイルを検索            │
│          │  Grep   ─── ファイル内容をテキスト検索          │
│          │                                                 │
├──────────┼─────────────────────────────────────────────────┤
│          │                                                 │
│  書き込み│  Write  ─── 新規ファイルを作成・上書き          │
│  系      │  Edit   ─── 既存ファイルの部分編集              │
│          │                                                 │
├──────────┼─────────────────────────────────────────────────┤
│          │                                                 │
│  実行系  │  Bash   ─── シェルコマンドを実行                │
│          │                                                 │
├──────────┼─────────────────────────────────────────────────┤
│          │                                                 │
│  その他  │  Agent  ─── サブエージェントを起動              │
│          │  TodoWrite ─ タスクリストを管理                  │
│          │                                                 │
└──────────┴─────────────────────────────────────────────────┘
```

各ツールは明確な役割を持っており、Claude Codeは状況に応じて最適なツールを選びます。

### ポイントまとめ
- Claude Codeは複数のツールを自動的に使い分ける
- 読み取り系（Read, Glob, Grep）、書き込み系（Write, Edit）、実行系（Bash）の3カテゴリ
- ユーザーはツール名を意識する必要はなく、自然言語で指示すればOK

---

## 3.2 ファイルの検索

### Glob - ファイル名パターン検索

Glob（グロブ）ツールは、ファイル名のパターン（pattern）を使ってファイルを検索します。

```
> src ディレクトリ内のすべての TypeScript ファイルを探して
```

Claude Codeは内部的に以下のようなGlob検索を実行します：

```
Glob: src/**/*.ts
→ 結果:
  src/app.ts
  src/index.ts
  src/utils/helper.ts
  src/services/userService.ts
  ...
```

#### よく使うGlobパターン

```
┌───────────────────────┬──────────────────────────────┐
│ パターン               │ 説明                         │
├───────────────────────┼──────────────────────────────┤
│ **/*.ts               │ 全階層のTypeScriptファイル    │
│ src/**/*.test.js      │ src配下のテストファイル       │
│ *.md                  │ カレントディレクトリのMD      │
│ **/*.{ts,tsx}         │ tsとtsxファイルの両方         │
│ src/components/*.tsx   │ components直下のTSXファイル  │
└───────────────────────┴──────────────────────────────┘
```

### Grep - コンテンツ検索

Grep（グレップ）ツールは、ファイルの**中身**を正規表現（regular expression）で検索します。

```
> "handleSubmit" という関数がどこで定義されているか探して
```

Claude Codeは内部的に以下のような検索を実行します：

```
Grep: pattern="function handleSubmit|const handleSubmit|handleSubmit ="
→ 結果:
  src/components/LoginForm.tsx:15: const handleSubmit = async (e: FormEvent) => {
  src/components/RegisterForm.tsx:22: const handleSubmit = (data: FormData) => {
```

### 検索の使い分け

```
ファイルを探したい場合：
  「○○というファイルはどこにある？」 → Glob が使われる

コードの中身を探したい場合：
  「○○という関数はどこで定義されている？」 → Grep が使われる

両方を組み合わせる場合：
  「src/配下のTSファイルで、fetchAPIを使っている箇所を探して」
  → Glob + Grep が組み合わされる
```

### ポイントまとめ
- ファイル名で探す場合はGlob、中身で探す場合はGrep
- 自然言語で検索意図を伝えれば適切なツールが選ばれる
- パターンを知っておくと、より精密な検索指示が出せる

## 3.3 ファイルの読み込み

### Read - ファイルの読み込み

Read（リード）ツールは、指定されたファイルの内容を読み込みます。

```
> src/app.ts の内容を見せて
```

Claude Codeは自動的にファイルを読み込んで、内容を把握した上で回答します。

#### 部分読み込み

大きなファイルの場合、必要な部分だけを読み込むこともできます：

```
> src/app.ts の100行目から150行目を見せて
```

#### 画像・PDFの読み込み

Claude Codeは画像ファイルやPDFファイルも読み込めます：

```
> docs/architecture.png を見て、このアーキテクチャを説明して

> docs/spec.pdf の1-5ページを読んで要約して
```

### よくある間違い
> **ディレクトリの読み込み**: Readツールはファイル専用です。ディレクトリの中身を見たい場合は「このディレクトリのファイル一覧を見せて」のように指示してください。Claude Codeは`ls`コマンドやGlobを使って対応します。

### ポイントまとめ
- Readツールでファイルの内容を読み込める
- 大きなファイルは行範囲を指定して部分読み込みが可能
- 画像やPDFも読み込んで内容を理解できる

---

## 3.4 ファイルの作成

### Write - 新規ファイル作成

Write（ライト）ツールは、新しいファイルを作成したり、既存ファイルを上書きしたりします。

```
> src/utils/formatDate.ts に日付フォーマット用の関数を作成して
```

Claude Codeは以下のように動作します：

```
Write: src/utils/formatDate.ts

[ファイルの内容が表示される]

このファイルを作成してよいですか？
[Y]es / [N]o
```

#### 実践例：設定ファイルの作成

```
> .prettierrc ファイルを作成して。
> インデントはスペース2つ、セミコロンなし、
> シングルクォートを使う設定にして
```

結果：

```json
{
  "semi": false,
  "singleQuote": true,
  "tabWidth": 2,
  "trailingComma": "es5"
}
```

### ポイントまとめ
- Writeツールで新規ファイルを作成できる
- 既存ファイルの完全な書き直しにも使われる
- ファイル作成前にユーザーの確認が求められる

---

## 3.5 コードの編集

### Edit - 部分編集

Edit（エディット）ツールは、既存ファイルの**一部分だけ**を変更します。Writeと異なり、ファイル全体を書き直すのではなく、差分（diff）だけを適用します。

```
> src/app.ts の handleLogin 関数にエラーハンドリングを追加して
```

Claude Codeは内部的に以下のような編集を行います：

```
Edit: src/app.ts
  old_string: "const result = await login(email, password);"
  new_string: "try {\n  const result = await login(email, password);\n} catch (error) {\n  console.error('Login failed:', error);\n  throw error;\n}"
```

### WriteとEditの使い分け

```
┌──────────────┬────────────────────────────────────────┐
│ ツール        │ 使われる場面                           │
├──────────────┼────────────────────────────────────────┤
│ Write        │ 新規ファイル作成                       │
│              │ ファイル全体の書き直し                  │
├──────────────┼────────────────────────────────────────┤
│ Edit         │ 既存ファイルの一部修正                  │
│              │ 関数の追加・変更                        │
│              │ バグの修正                              │
│              │ 変数名の一括変更                        │
└──────────────┴────────────────────────────────────────┘
```

### 実践例：リファクタリング

```
> src/utils/validate.ts の validateEmail 関数を
> 正規表現を使った実装に変更してください
```

```
> UserService クラスの全メソッドに適切なエラーハンドリングを
> 追加してください
```

```
> このファイルの "userName" という変数名をすべて
> "username" に変更してください
```

### よくある間違い
> **編集範囲が曖昧な指示**: 「このファイルを改善して」のような曖昧な指示では、意図しない変更が行われる可能性があります。「○○関数の△△の部分を□□に変更して」のように、変更箇所と内容を具体的に指示しましょう。

### ポイントまとめ
- Editツールは既存ファイルの部分的な変更に使われる
- 差分のみを適用するため、ファイルの他の部分に影響しない
- 具体的な変更箇所と内容を指示するのが重要

## 3.6 複数ファイルの操作

Claude Codeは1回の指示で複数のファイルを同時に操作できます。

### プロジェクト横断的な変更

```
> すべてのTypeScriptファイルで、console.log を logger.debug に
> 置き換えてください
```

Claude Codeは以下の手順で処理します：

```
1. Grep で console.log を含むファイルを検索
2. 各ファイルを Read で読み込み
3. Edit で console.log を logger.debug に変更
4. 必要に応じて logger の import 文を追加
```

### 関連ファイルの同時作成

```
> ユーザー管理機能を作成してください：
> - src/models/User.ts（モデル定義）
> - src/services/UserService.ts（ビジネスロジック）
> - src/controllers/UserController.ts（APIエンドポイント）
> - src/tests/UserService.test.ts（テスト）
```

### 依存関係を考慮した編集

Claude Codeはファイル間の依存関係（dependency）を理解して編集します：

```
> User モデルに phoneNumber フィールドを追加して、
> 関連するサービス、コントローラー、テストも更新してください
```

この場合、Claude Codeは：
1. モデルにフィールドを追加
2. サービスのバリデーションロジックを更新
3. コントローラーのリクエスト処理を更新
4. テストケースを追加

と、関連する全ファイルを整合性を保って更新します。

### ポイントまとめ
- 1回の指示で複数ファイルを同時に操作できる
- プロジェクト横断的な一括置換が可能
- ファイル間の依存関係を考慮した編集ができる

---

## 3.7 Bashツールの活用

### Bash - コマンド実行

Bash（バッシュ）ツールは、シェルコマンドを実行します。ファイル操作以外にも、テストの実行やパッケージのインストールなど、幅広い用途で使われます。

```
> テストを実行して
```

```
Bash: npm test
→ テスト結果が表示される
```

```
> このプロジェクトの依存パッケージをインストールして
```

```
Bash: npm install
→ インストール結果が表示される
```

### よくある間違い
> **Bashで何でもやらせようとする**: Claude Codeはファイルの読み書きには専用ツール（Read, Write, Edit）を使います。「cat でファイルを表示して」ではなく、「ファイルの内容を見せて」と指示しましょう。Claude Codeが最適なツールを選択します。
