## 演習3-1：条件付きレンダリング（基本）

### 問題

ユーザーのログイン状態に応じて表示を切り替える `LoginStatus.vue` を作成してください。

**要件:**
1. `isLoggedIn`（真偽値）と `userRole`（文字列: `'admin'`, `'editor'`, `'viewer'`）を定義する
2. ログインしていないとき: ログインボタンを表示
3. ログイン済みのとき: ロールに応じた表示を切り替える
   - `admin` → 「管理者ダッシュボード」と「ユーザー管理」ボタン
   - `editor` → 「エディターダッシュボード」と「記事管理」ボタン
   - `viewer` → 「閲覧者ページ」
4. ログイン/ログアウトのトグルボタンを実装する

<details>
<summary>ヒント</summary>

- `v-if` / `v-else-if` / `v-else` のチェーンを使います
- これらのディレクティブは**隣接する兄弟要素**に書く必要があります
- 複数要素をまとめて条件分岐させるには `<template>` タグを使います

</details>

<details>
<summary>解答例</summary>

```vue
<!-- LoginStatus.vue -->
<script setup>
import { ref } from 'vue'

const isLoggedIn = ref(false)
const userRole = ref('viewer')  // 'admin' | 'editor' | 'viewer'

// ログイン・ログアウトの切り替え
function toggleLogin() {
  isLoggedIn.value = !isLoggedIn.value
}

// ロール変更（デモ用）
function changeRole(role) {
  userRole.value = role
}
</script>

<template>
  <div class="login-status">
    <!-- 未ログイン時 -->
    <div v-if="!isLoggedIn">
      <p>ログインしていません</p>
      <button @click="toggleLogin">ログイン</button>
    </div>

    <!-- ログイン済み：ロール別の表示 -->
    <template v-else>
      <!-- admin ロール -->
      <div v-if="userRole === 'admin'" class="dashboard admin">
        <h2>管理者ダッシュボード</h2>
        <button>ユーザー管理</button>
        <button>システム設定</button>
      </div>

      <!-- editor ロール -->
      <div v-else-if="userRole === 'editor'" class="dashboard editor">
        <h2>エディターダッシュボード</h2>
        <button>記事管理</button>
        <button>メディア管理</button>
      </div>

      <!-- viewer ロール（それ以外） -->
      <div v-else class="dashboard viewer">
        <h2>閲覧者ページ</h2>
        <p>コンテンツを閲覧できます</p>
      </div>

      <button @click="toggleLogin">ログアウト</button>
    </template>

    <!-- ロール切り替え（デモ用） -->
    <div class="role-switcher">
      <span>ロール切替: </span>
      <button @click="changeRole('admin')">Admin</button>
      <button @click="changeRole('editor')">Editor</button>
      <button @click="changeRole('viewer')">Viewer</button>
    </div>
  </div>
</template>
```

**ポイント:**
- `<template v-else>` を使うと、余計な DOM ノードを追加せずに複数要素をグループ化できます。
- `v-if` はDOMから要素を完全に削除/追加します。初期表示のコストが高いが、切り替え頻度が低い場合に向いています。

</details>

## 演習3-2：v-if と v-show の使い分け（基本）

### 問題

FAQページを作成する `FaqPage.vue` を作ってください。質問をクリックすると回答が表示/非表示を切り替わります。

**要件:**
1. 3つのFAQ項目を用意する（質問文と回答文）
2. **回答の表示/非表示** には `v-show` を使う（理由を説明できること）
3. 「全て展開」「全て閉じる」ボタンを実装する

<details>
<summary>ヒント</summary>

- `v-show` は CSS の `display: none` で要素を隠すだけで、DOM上には残ります
- 頻繁に切り替える要素には `v-show` が適しています（DOM の追加/削除コストがない）
- 各FAQの開閉状態を配列やオブジェクトで管理しましょう

</details>

<details>
<summary>解答例</summary>

```vue
<!-- FaqPage.vue -->
<script setup>
import { ref } from 'vue'

// FAQ データ（質問文と回答文、開閉状態を管理）
const faqs = ref([
  {
    question: 'Vue.jsとは何ですか？',
    answer: 'Vue.jsはユーザーインターフェースを構築するためのプログレッシブJavaScriptフレームワークです。',
    isOpen: false
  },
  {
    question: 'Composition APIとOptions APIの違いは？',
    answer: 'Composition APIは関数ベースでロジックを記述し、Options APIはオプションオブジェクト形式で記述します。Vue 3ではComposition APIが推奨されています。',
    isOpen: false
  },
  {
    question: 'Viteとは何ですか？',
    answer: 'ViteはVue.jsの公式推奨ビルドツールで、高速なHMR（Hot Module Replacement）と最適化されたビルドを提供します。',
    isOpen: false
  }
])

// 個別のFAQの開閉を切り替え
function toggleFaq(index) {
  faqs.value[index].isOpen = !faqs.value[index].isOpen
}

// 全て展開
function expandAll() {
  faqs.value.forEach(faq => { faq.isOpen = true })
}

// 全て閉じる
function collapseAll() {
  faqs.value.forEach(faq => { faq.isOpen = false })
}
</script>

<template>
  <div class="faq-page">
    <h1>よくある質問</h1>

    <div class="controls">
      <button @click="expandAll">全て展開</button>
      <button @click="collapseAll">全て閉じる</button>
    </div>

    <div class="faq-list">
      <div v-for="(faq, index) in faqs" :key="index" class="faq-item">
        <!-- 質問（クリックで開閉） -->
        <h3 class="question" @click="toggleFaq(index)">
          <span>{{ faq.isOpen ? '▼' : '▶' }}</span>
          {{ faq.question }}
        </h3>

        <!--
          v-show を使う理由:
          FAQの回答は頻繁に表示/非表示が切り替わるため、
          DOMの追加・削除を行う v-if よりも、
          CSSで表示を制御する v-show の方がパフォーマンスが良い
        -->
        <p v-show="faq.isOpen" class="answer">
          {{ faq.answer }}
        </p>
      </div>
    </div>
  </div>
</template>

<style scoped>
.question {
  cursor: pointer;
  padding: 12px;
  background-color: #f5f5f5;
  border-radius: 4px;
  user-select: none;
}

.question:hover {
  background-color: #e8e8e8;
}

.answer {
  padding: 12px 12px 12px 28px;
  color: #555;
  line-height: 1.6;
}

.faq-item {
  margin-bottom: 8px;
  border: 1px solid #ddd;
  border-radius: 4px;
}
</style>
```

**v-if vs v-show の判断基準:**

| 条件 | 推奨 |
|------|------|
| 頻繁に切り替わる | `v-show` |
| 一度決まったら変わらない | `v-if` |
| 初期表示で非表示が多い | `v-if`（DOMを軽くできる） |
| 切り替え時のコストを下げたい | `v-show` |

</details>

## 演習3-3：v-for によるリスト表示（基本）

### 問題

社員名簿を表示する `EmployeeList.vue` を作成してください。

**要件:**
1. 社員データ配列（名前、部署、入社年）を5名分定義する
2. `v-for` でテーブル形式で表示する（`:key` を適切に設定）
3. 「部署」でフィルタリングするセレクトボックスを追加する
4. フィルタリングされた結果の件数を表示する

<details>
<summary>ヒント</summary>

- `v-for="(item, index) in array"` で配列をループできます
- `:key` にはユニークな値（id など）を指定します
- フィルタリングには `computed` を使って、元の配列を加工した結果を返しましょう
- セレクトボックスは `<select v-model="...">` でバインドします

</details>

<details>
<summary>解答例</summary>

```vue
<!-- EmployeeList.vue -->
<script setup>
import { ref, computed } from 'vue'

// 社員データ
const employees = ref([
  { id: 1, name: '山田太郎', department: '開発部', joinYear: 2020 },
  { id: 2, name: '佐藤花子', department: '営業部', joinYear: 2019 },
  { id: 3, name: '田中一郎', department: '開発部', joinYear: 2021 },
  { id: 4, name: '鈴木美咲', department: '人事部', joinYear: 2022 },
  { id: 5, name: '高橋健太', department: '営業部', joinYear: 2020 }
])

// フィルター用の選択値
const selectedDepartment = ref('all')

// 部署一覧を動的に取得（computed で自動更新）
const departments = computed(() => {
  // Set で重複を排除し、配列に戻す
  const deptSet = new Set(employees.value.map(e => e.department))
  return ['all', ...deptSet]
})

// フィルタリングされた社員リスト
const filteredEmployees = computed(() => {
  if (selectedDepartment.value === 'all') {
    return employees.value
  }
  return employees.value.filter(
    e => e.department === selectedDepartment.value
  )
})
</script>

<template>
  <div class="employee-list">
    <h2>社員名簿</h2>

    <!-- フィルター -->
    <div class="filter">
      <label>
        部署で絞り込み:
        <select v-model="selectedDepartment">
          <option v-for="dept in departments" :key="dept" :value="dept">
            {{ dept === 'all' ? '全部署' : dept }}
          </option>
        </select>
      </label>
      <span>（{{ filteredEmployees.length }}名）</span>
    </div>

    <!-- 社員テーブル -->
    <table>
      <thead>
        <tr>
          <th>ID</th>
          <th>名前</th>
          <th>部署</th>
          <th>入社年</th>
        </tr>
      </thead>
      <tbody>
        <!--
          v-for で配列をループ
          :key に一意の id を指定することで、Vue が効率的に DOM を更新できる
        -->
        <tr v-for="employee in filteredEmployees" :key="employee.id">
          <td>{{ employee.id }}</td>
          <td>{{ employee.name }}</td>
          <td>{{ employee.department }}</td>
          <td>{{ employee.joinYear }}年</td>
        </tr>
      </tbody>
    </table>

    <!-- データが空の場合 -->
    <p v-if="filteredEmployees.length === 0" class="no-data">
      該当する社員がいません
    </p>
  </div>
</template>

<style scoped>
table {
  width: 100%;
  border-collapse: collapse;
  margin-top: 12px;
}
th, td {
  border: 1px solid #ddd;
  padding: 10px;
  text-align: left;
}
th {
  background-color: #f0f0f0;
}
.filter {
  margin-bottom: 12px;
}
.no-data {
  color: #888;
  text-align: center;
  padding: 20px;
}
</style>
```

**`:key` の重要性:**
- `:key` を正しく設定しないと、Vue がリスト更新時にどの要素を使い回すかを正しく判断できず、バグやパフォーマンス低下の原因になります。
- `index` を `:key` に使うのは、並び替えや削除がない場合のみ許容されます。基本的にはユニークな `id` を使いましょう。

</details>

## 演習3-4：バグ修正 — v-for と v-if の共存（応用）

### 問題

以下のコードには Vue 3 のベストプラクティスに反する問題があります。修正してください。

```vue
<script setup>
import { ref } from 'vue'

const tasks = ref([
  { id: 1, title: '買い物', done: false },
  { id: 2, title: '掃除', done: true },
  { id: 3, title: '料理', done: false },
  { id: 4, title: '洗濯', done: true }
])
</script>

<template>
  <ul>
    <!-- 未完了のタスクだけ表示したい -->
    <li v-for="task in tasks" v-if="!task.done" :key="task.id">
      {{ task.title }}
    </li>
  </ul>
</template>
```

<details>
<summary>ヒント</summary>

- Vue 3 では、同じ要素に `v-for` と `v-if` を同時に指定するのは**非推奨**です
- `v-if` は `v-for` より優先度が高いため、`v-if` 内で `v-for` の変数（`task`）にアクセスできません（エラーになる場合がある）
- 解決策: `computed` でフィルタリングするか、`<template>` タグで分離する

</details>

<details>
<summary>解答例</summary>

**問題点:** Vue 3 では `v-if` が `v-for` より優先されるため、`v-if="!task.done"` の時点で `task` が未定義になり、正しく動作しません。

**修正方法1: computed でフィルタリング（推奨）**

```vue
<script setup>
import { ref, computed } from 'vue'

const tasks = ref([
  { id: 1, title: '買い物', done: false },
  { id: 2, title: '掃除', done: true },
  { id: 3, title: '料理', done: false },
  { id: 4, title: '洗濯', done: true }
])

// ✅ computed でフィルタリングしてからテンプレートに渡す
const pendingTasks = computed(() => {
  return tasks.value.filter(task => !task.done)
})
</script>

<template>
  <ul>
    <!-- フィルタリング済みの配列をループ -->
    <li v-for="task in pendingTasks" :key="task.id">
      {{ task.title }}
    </li>
  </ul>
</template>
```

**修正方法2: `<template>` で分離**

```vue
<template>
  <ul>
    <!-- template で v-for を行い、li で v-if を行う -->
    <template v-for="task in tasks" :key="task.id">
      <li v-if="!task.done">
        {{ task.title }}
      </li>
    </template>
  </ul>
</template>
```

**ベストプラクティス:** 方法1（computed でフィルタリング）が推奨されます。テンプレートはシンプルに保ち、ロジックは script に書くのが Vue の設計思想です。

</details>

## 演習3-5：クラスとスタイルのバインディング（応用）

### 問題

テーマ切り替えとカード選択機能を持つ `ThemeCards.vue` を作成してください。

**要件:**
1. `theme`（`'light'` or `'dark'`）でページ全体のテーマを切り替える
2. 3枚のカードを表示し、クリックで選択状態をトグルする
3. 選択されたカードは `:class` でハイライトする
4. カードの枠線色は `:style` で動的に設定する（選択時: 青、非選択時: グレー）

<details>
<summary>ヒント</summary>

- `:class` にはオブジェクト構文 `:class="{ active: isActive }"` や配列構文 `:class="[baseClass, extraClass]"` が使えます
- `:style` にはオブジェクト構文 `:style="{ color: textColor }"` が使えます
- カードの選択状態は `Set` や配列で管理できます

</details>

<details>
<summary>解答例</summary>

```vue
<!-- ThemeCards.vue -->
<script setup>
import { ref, computed } from 'vue'

// テーマ状態
const theme = ref('light')

// カードデータ
const cards = ref([
  { id: 1, title: 'Vue.js', description: 'プログレッシブフレームワーク' },
  { id: 2, title: 'React', description: 'UIライブラリ' },
  { id: 3, title: 'Angular', description: 'フルスタックフレームワーク' }
])

// 選択されたカードのIDを管理する Set
const selectedIds = ref(new Set())

// カードの選択をトグル
function toggleCard(id) {
  const newSet = new Set(selectedIds.value)
  if (newSet.has(id)) {
    newSet.delete(id)
  } else {
    newSet.add(id)
  }
  // Set は ref で包んでいるため、新しい Set を代入して更新を通知
  selectedIds.value = newSet
}

// カードが選択されているか判定
function isSelected(id) {
  return selectedIds.value.has(id)
}

// カードのスタイルを動的に生成
function cardStyle(id) {
  return {
    borderColor: isSelected(id) ? '#3182ce' : '#ccc',
    borderWidth: isSelected(id) ? '3px' : '1px',
    transform: isSelected(id) ? 'scale(1.03)' : 'scale(1)',
    transition: 'all 0.2s ease'
  }
}

// テーマ切り替え
function toggleTheme() {
  theme.value = theme.value === 'light' ? 'dark' : 'light'
}

// 選択数
const selectedCount = computed(() => selectedIds.value.size)
</script>

<template>
  <!--
    :class のオブジェクト構文: { クラス名: 条件 }
    条件が true のとき、そのクラスが付与される
  -->
  <div :class="['container', `theme-${theme}`]">
    <h2>フレームワーク選択</h2>

    <button @click="toggleTheme">
      {{ theme === 'light' ? '🌙 ダークモード' : '☀ ライトモード' }}
    </button>

    <p>選択中: {{ selectedCount }}件</p>

    <div class="card-grid">
      <div
        v-for="card in cards"
        :key="card.id"
        :class="{
          card: true,
          selected: isSelected(card.id)
        }"
        :style="cardStyle(card.id)"
        @click="toggleCard(card.id)"
      >
        <h3>{{ card.title }}</h3>
        <p>{{ card.description }}</p>
        <span v-show="isSelected(card.id)" class="check">選択済み</span>
      </div>
    </div>
  </div>
</template>

<style scoped>
.container {
  padding: 20px;
  min-height: 100vh;
}

/* テーマごとのスタイル */
.theme-light {
  background-color: #ffffff;
  color: #333;
}

.theme-dark {
  background-color: #1a202c;
  color: #e2e8f0;
}

.card-grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
  margin-top: 16px;
}

.card {
  border-style: solid;
  border-radius: 8px;
  padding: 20px;
  cursor: pointer;
  user-select: none;
}

.card.selected {
  background-color: #ebf8ff;
}

.theme-dark .card.selected {
  background-color: #2a4365;
}

.check {
  color: #3182ce;
  font-weight: bold;
}
</style>
```

**クラスバインディングの3つの書き方:**
```html
<!-- 1. オブジェクト構文（条件付きクラス） -->
<div :class="{ active: isActive, 'text-bold': isBold }">

<!-- 2. 配列構文（複数クラスの組み合わせ） -->
<div :class="[baseClass, sizeClass]">

<!-- 3. 配列 + オブジェクトの混合 -->
<div :class="['card', { selected: isSelected }]">
```

</details>

## 演習3-6：v-for の高度な使い方（応用）

### 問題

入れ子のデータ（カテゴリーとその中の商品）を表示する `NestedList.vue` を作成してください。

**要件:**
1. カテゴリー配列（各カテゴリーに `name` と `products` 配列を持つ）を定義する
2. `v-for` のネスト（二重ループ）で全商品を表示する
3. 各カテゴリーの商品数を表示する
4. `v-for` で `(item, index)` のインデックスも使う

```js
// データ構造のイメージ
const categories = [
  { name: '果物', products: ['りんご', 'バナナ', 'みかん'] },
  { name: '野菜', products: ['にんじん', 'キャベツ'] },
  { name: '飲料', products: ['お茶', 'コーヒー', 'ジュース', '水'] }
]
```

<details>
<summary>ヒント</summary>

- `v-for` はネスト（入れ子）にできます
- 外側のループで `categories` を、内側のループで `category.products` を回します
- `:key` はそれぞれのループで一意になるようにします

</details>

<details>
<summary>解答例</summary>

```vue
<!-- NestedList.vue -->
<script setup>
import { ref } from 'vue'

const categories = ref([
  {
    id: 'fruit',
    name: '果物',
    products: [
      { id: 'f1', name: 'りんご', price: 200 },
      { id: 'f2', name: 'バナナ', price: 150 },
      { id: 'f3', name: 'みかん', price: 100 }
    ]
  },
  {
    id: 'vegetable',
    name: '野菜',
    products: [
      { id: 'v1', name: 'にんじん', price: 120 },
      { id: 'v2', name: 'キャベツ', price: 180 }
    ]
  },
  {
    id: 'drink',
    name: '飲料',
    products: [
      { id: 'd1', name: 'お茶', price: 150 },
      { id: 'd2', name: 'コーヒー', price: 250 },
      { id: 'd3', name: 'ジュース', price: 130 },
      { id: 'd4', name: '水', price: 100 }
    ]
  }
])
</script>

<template>
  <div class="nested-list">
    <h2>商品カタログ</h2>

    <!-- 外側のループ: カテゴリー -->
    <div
      v-for="(category, catIndex) in categories"
      :key="category.id"
      class="category"
    >
      <!-- catIndex を使ってカテゴリー番号を表示 -->
      <h3>{{ catIndex + 1 }}. {{ category.name }}（{{ category.products.length }}品）</h3>

      <ul>
        <!-- 内側のループ: カテゴリー内の商品 -->
        <li
          v-for="(product, prodIndex) in category.products"
          :key="product.id"
        >
          <!-- catIndex と prodIndex の両方を使う例 -->
          {{ catIndex + 1 }}-{{ prodIndex + 1 }}.
          {{ product.name }} — ¥{{ product.price }}
        </li>
      </ul>
    </div>

    <!-- v-for でオブジェクトもループ可能 -->
    <h3>おまけ: オブジェクトの v-for</h3>
    <ul>
      <!--
        オブジェクトをループすると (value, key, index) の3つが取得できる
      -->
      <li v-for="(value, key, index) in { name: 'Vue', version: 3, type: 'framework' }" :key="key">
        {{ index }}: {{ key }} = {{ value }}
      </li>
    </ul>
  </div>
</template>

<style scoped>
.category {
  margin-bottom: 20px;
  padding: 12px;
  border: 1px solid #e2e8f0;
  border-radius: 8px;
}

h3 {
  color: #2d3748;
  border-bottom: 2px solid #42b883;
  padding-bottom: 4px;
}
</style>
```

**v-for の構文バリエーション:**
```html
<!-- 配列: (要素, インデックス) -->
<li v-for="(item, index) in array" :key="item.id">

<!-- オブジェクト: (値, キー名, インデックス) -->
<li v-for="(value, key, index) in object" :key="key">

<!-- 数値の範囲: 1 から n まで -->
<span v-for="n in 10" :key="n">{{ n }}</span>
```

## 演習3-7：総合問題 — タスクボード（チャレンジ）

### 問題

3つのカラム（未着手・進行中・完了）を持つタスクボードを作成してください。

**要件:**
1. タスクには `id`, `title`, `status`（`'todo'` | `'doing'` | `'done'`）がある
2. ステータスごとにカラムに分けて表示する（`computed` でフィルタリング）
3. 各タスクに「次のステータスへ」ボタンを付ける（todo → doing → done）
4. 新しいタスクを追加する入力欄とボタンを実装する
5. 各カラムのタスク数を表示する
6. `:class` で完了タスクに取り消し線スタイルを適用する

<details>
<summary>ヒント</summary>

- 3つの `computed` で各カラムのタスクをフィルタリングします
- ステータス遷移は `Map` や `Object` で次のステータスを定義すると管理しやすいです
- 新規タスクの `id` には `Date.now()` が簡便です

</details>

<details>
<summary>解答例</summary>

```vue
<!-- TaskBoard.vue -->
<script setup>
import { ref, computed } from 'vue'

// タスクデータ
const tasks = ref([
  { id: 1, title: 'Vue教材を読む', status: 'todo' },
  { id: 2, title: 'カウンターアプリを作る', status: 'doing' },
  { id: 3, title: '環境構築', status: 'done' },
  { id: 4, title: 'Piniaを学ぶ', status: 'todo' }
])

// 新規タスクの入力値
const newTaskTitle = ref('')

// ステータスの次の状態を定義
const nextStatus = {
  todo: 'doing',
  doing: 'done',
  done: 'done'  // 完了は最終状態
}

// ステータスの日本語ラベル
const statusLabels = {
  todo: '未着手',
  doing: '進行中',
  done: '完了'
}

// 各ステータスのタスクを computed でフィルタリング
const todoTasks = computed(() =>
  tasks.value.filter(t => t.status === 'todo')
)
const doingTasks = computed(() =>
  tasks.value.filter(t => t.status === 'doing')
)
const doneTasks = computed(() =>
  tasks.value.filter(t => t.status === 'done')
)

// タスクを次のステータスに進める
function advanceTask(taskId) {
  const task = tasks.value.find(t => t.id === taskId)
  if (task && task.status !== 'done') {
    task.status = nextStatus[task.status]
  }
}

// 新しいタスクを追加
function addTask() {
  const title = newTaskTitle.value.trim()
  if (!title) return

  tasks.value.push({
    id: Date.now(),  // 簡易的なユニークID
    title,
    status: 'todo'
  })
  newTaskTitle.value = ''  // 入力欄をクリア
}
</script>

<template>
  <div class="task-board">
    <h2>タスクボード</h2>

    <!-- タスク追加フォーム -->
    <div class="add-task">
      <input
        v-model="newTaskTitle"
        type="text"
        placeholder="新しいタスク..."
        @keyup.enter="addTask"
      />
      <button @click="addTask" :disabled="!newTaskTitle.trim()">追加</button>
    </div>

    <!-- 3カラムレイアウト -->
    <div class="columns">
      <!-- 未着手カラム -->
      <div class="column">
        <h3>未着手（{{ todoTasks.length }}）</h3>
        <div
          v-for="task in todoTasks"
          :key="task.id"
          class="task-card"
        >
          <span>{{ task.title }}</span>
          <button @click="advanceTask(task.id)" class="btn-advance">
            着手 →
          </button>
        </div>
        <p v-if="todoTasks.length === 0" class="empty">タスクなし</p>
      </div>

      <!-- 進行中カラム -->
      <div class="column">
        <h3>進行中（{{ doingTasks.length }}）</h3>
        <div
          v-for="task in doingTasks"
          :key="task.id"
          class="task-card doing"
        >
          <span>{{ task.title }}</span>
          <button @click="advanceTask(task.id)" class="btn-advance">
            完了 →
          </button>
        </div>
        <p v-if="doingTasks.length === 0" class="empty">タスクなし</p>
      </div>

      <!-- 完了カラム -->
      <div class="column">
        <h3>完了（{{ doneTasks.length }}）</h3>
        <div
          v-for="task in doneTasks"
          :key="task.id"
          :class="['task-card', 'done']"
        >
          <!-- :class で 'done' クラスを付与し、取り消し線を適用 -->
          <span class="done-text">{{ task.title }}</span>
        </div>
        <p v-if="doneTasks.length === 0" class="empty">タスクなし</p>
      </div>
    </div>
  </div>
</template>

<style scoped>
.columns {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
  margin-top: 16px;
}

.column {
  background-color: #f7fafc;
  border-radius: 8px;
  padding: 12px;
  min-height: 200px;
}

.task-card {
  background: white;
  border: 1px solid #e2e8f0;
  border-radius: 6px;
  padding: 10px;
  margin-bottom: 8px;
  display: flex;
  justify-content: space-between;
  align-items: center;
}

.task-card.doing {
  border-left: 4px solid #ecc94b;
}

.task-card.done {
  border-left: 4px solid #48bb78;
  opacity: 0.7;
}

.done-text {
  text-decoration: line-through;
  color: #a0aec0;
}

.btn-advance {
  font-size: 0.8em;
  padding: 2px 8px;
  cursor: pointer;
}

.add-task {
  display: flex;
  gap: 8px;
  margin-bottom: 8px;
}

.add-task input {
  flex: 1;
  padding: 8px;
  border: 1px solid #ccc;
  border-radius: 4px;
}

.empty {
  color: #a0aec0;
  text-align: center;
  font-size: 0.9em;
}
</style>
```

**この演習で使ったテンプレート構文のまとめ:**
- `v-for` — リストのループ表示
- `v-if` — 条件付き表示（空リストの判定）
- `:class` — 動的クラスバインディング
- `v-model` — フォーム入力の双方向バインド
- `@click` — クリックイベント
- `@keyup.enter` — Enterキーのイベント修飾子
- `:disabled` — 動的属性バインディング
