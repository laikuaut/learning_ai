> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第2章（変数と展開）、第4章（ループと繰り返し）、第7章（入出力とリダイレクト）
> **課題の種類**: ミニプロジェクト
> **学習目標**: ファイル読み込み、パイプ、リダイレクトを活用してテキスト解析ツールを作成する

## 完成イメージ


---

## 課題の要件

1. コマンドライン引数でテキストファイルを受け取る
2. ファイルが存在しない場合はエラーメッセージを表示する
3. 行数・単語数・文字数を表示する
4. 単語の出現頻度を集計し、上位10件を表示する
5. パイプ入力（`cat file | ./wordcount.sh`）にも対応する

---

## ステップガイド

<details>
<summary>ステップ1：引数チェックとファイル読み込み</summary>

引数があればファイルから、なければ標準入力から読み込みます。

```bash
if [ -n "${1:-}" ]; then
    if [ ! -f "$1" ]; then
        echo "エラー: '$1' が見つかりません"
        exit 1
    fi
    input_file="$1"
    filename=$(basename "$1")
else
    # 標準入力を一時ファイルに保存
    input_file=$(mktemp)
    cat > "$input_file"
    filename="(標準入力)"
fi
```

</details>

<details>
<summary>ステップ2：行数・単語数・文字数を取得する</summary>

`wc` コマンドで基本統計を取得します。

```bash
lines=$(wc -l < "$input_file")
words=$(wc -w < "$input_file")
chars=$(wc -c < "$input_file")
```

</details>

<details>
<summary>ステップ3：単語頻度を集計する</summary>

`tr` で単語を1行1語にし、`sort | uniq -c | sort -rn` で集計します。

```bash
tr -s '[:space:]' '\n' < "$input_file" | sort | uniq -c | sort -rn | head -10
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

```bash
#!/usr/bin/env bash
# 単語カウンター
# 学べる内容：ファイル読み込み、wc、パイプ、リダイレクト、一時ファイル
# 実行方法：chmod +x wordcount.sh && ./wordcount.sh <テキストファイル>

# --- 入力の決定 ---
if [ -n "${1:-}" ]; then
    if [ ! -f "$1" ]; then
        echo "エラー: '$1' が見つかりません" >&2
        exit 1
    fi
    input_file="$1"
    filename=$(basename "$1")
else
    # 標準入力を一時ファイルに保存
    input_file=$(mktemp)
    cat > "$input_file"
    filename="(標準入力)"
    trap 'rm -f "$input_file"' EXIT
fi

# --- 基本統計 ---
lines=$(wc -l < "$input_file")
words=$(wc -w < "$input_file")
chars=$(wc -c < "$input_file")

# --- 出力 ---
echo ""
echo "===== テキスト分析結果 ====="
printf "ファイル名: %s\n" "$filename"
printf "行数:       %d\n" "$lines"
printf "単語数:     %d\n" "$words"
printf "文字数:     %d\n" "$chars"

echo ""
echo "--- 単語出現頻度 TOP10 ---"
tr -s '[:space:]' '\n' < "$input_file" | grep -v '^$' | sort | uniq -c | sort -rn | head -10
```

</details>

<details>
<summary>解答例（改良版 ─ 関数化とオプション対応）</summary>

```bash
#!/usr/bin/env bash
# 単語カウンター（改良版）
# オプション解析・大文字小文字無視・除外パターン対応
# 実行方法：chmod +x wordcount.sh && ./wordcount.sh [-i] [-n NUM] [-e PATTERN] <ファイル>
#   -i : 大文字小文字を区別しない
#   -n NUM : 上位NUM件を表示（デフォルト: 10）
#   -e PATTERN : 除外する単語パターン（正規表現）

set -euo pipefail

# --- デフォルト設定 ---
ignore_case=false
top_n=10
exclude_pattern=""

# --- オプション解析 ---
while getopts ":in:e:" opt; do
    case "$opt" in
        i) ignore_case=true ;;
        n) top_n="$OPTARG" ;;
        e) exclude_pattern="$OPTARG" ;;
        *) echo "使い方: $0 [-i] [-n NUM] [-e PATTERN] <ファイル>" >&2; exit 1 ;;
    esac
done
shift $((OPTIND - 1))

# --- 入力の決定 ---
cleanup() {
    [ -n "${tmp_file:-}" ] && rm -f "$tmp_file"
}
trap cleanup EXIT

if [ -n "${1:-}" ]; then
    if [ ! -f "$1" ]; then
        echo "エラー: '$1' が見つかりません" >&2
        exit 1
    fi
    input_file="$1"
    filename=$(basename "$1")
else
    tmp_file=$(mktemp)
    cat > "$tmp_file"
    input_file="$tmp_file"
    filename="(標準入力)"
fi

# --- 基本統計 ---
lines=$(wc -l < "$input_file" | tr -d '[:space:]')
words=$(wc -w < "$input_file" | tr -d '[:space:]')
chars=$(wc -c < "$input_file" | tr -d '[:space:]')

# --- 出力 ---
echo ""
echo "===== テキスト分析結果 ====="
printf "%-12s %s\n" "ファイル名:" "$filename"
printf "%-12s %d\n" "行数:" "$lines"
printf "%-12s %d\n" "単語数:" "$words"
printf "%-12s %d\n" "文字数:" "$chars"

echo ""
echo "--- 単語出現頻度 TOP${top_n} ---"

# パイプラインを構築
pipeline="tr -s '[:space:][:punct:]' '\n' < \"$input_file\" | grep -v '^$'"

if $ignore_case; then
    pipeline="$pipeline | tr '[:upper:]' '[:lower:]'"
fi

if [ -n "$exclude_pattern" ]; then
    pipeline="$pipeline | grep -vE '$exclude_pattern'"
fi

pipeline="$pipeline | sort | uniq -c | sort -rn | head -${top_n}"

eval "$pipeline"

# --- ユニーク単語数 ---
unique_count=$(tr -s '[:space:][:punct:]' '\n' < "$input_file" | grep -v '^$' | sort -u | wc -l | tr -d '[:space:]')
echo ""
echo "ユニーク単語数: ${unique_count}"
```

**初心者向けとの違い:**
- `getopts` でオプション解析 → `-i`, `-n`, `-e` の組み合わせに対応
- `set -euo pipefail` で堅牢化 → エラー時に即座に停止
- `trap cleanup EXIT` で一時ファイルを確実に削除 → リソースリーク防止
- 句読点除去 `[:punct:]` → より正確な単語カウント
- ユニーク単語数を追加表示 → テキストの語彙豊富さがわかる
- `wc` の出力から空白除去 → macOS互換（macOS の wc は先頭に空白が入る）

</details>
