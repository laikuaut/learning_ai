## 1.1 HTMLとは何か

**HTML**（HyperText Markup Language）は、Webページの「構造」を記述するためのマークアップ言語です。プログラミング言語とは異なり、HTMLはブラウザに「この部分は見出し」「この部分は段落」「この部分はリンク」といった意味（セマンティクス）を伝える役割を持ちます。

### HTMLの歴史
| バージョン | 年 | 主な特徴 |
|---|---|---|
| HTML 1.0 | 1993年 | 最初の仕様 |
| HTML 2.0 | 1995年 | フォーム機能追加 |
| HTML 3.2 | 1997年 | テーブル、アプレット |
| HTML 4.01 | 1999年 | CSS分離、アクセシビリティ |
| XHTML 1.0 | 2000年 | XMLベースの厳格な構文 |
| HTML5 | 2014年 | マルチメディア、セマンティック要素、API |
| HTML Living Standard | 現在 | WHATWGによる継続的な更新 |

> **ポイント**: 現在のHTMLは「HTML Living Standard」として、WHATWGが管理する「生きた仕様」です。バージョン番号はなく、継続的に更新されています。

## 1.2 DOCTYPE宣言

HTMLファイルの先頭には必ず **DOCTYPE宣言**（文書型宣言）を記述します。

```html
<!DOCTYPE html>
```

### DOCTYPE宣言の役割
1. **標準モード（Standards Mode）** でレンダリングすることをブラウザに指示する
2. DOCTYPEがないと「**互換モード（Quirks Mode）**」でレンダリングされ、古いブラウザの挙動を模倣してしまう
3. 大文字・小文字は区別されない（`<!doctype html>` でもOK）

### 標準モードと互換モードの違い
| 項目 | 標準モード | 互換モード |
|---|---|---|
| CSSボックスモデル | W3C標準 | IE独自（border-box風） |
| インラインの縦方向配置 | baseline | bottom |
| テーブル内のフォントサイズ | 継承する | 継承しない場合がある |
| 結果 | 予測可能 | ブラウザごとに異なる |

> **重要**: 必ずHTMLファイルの**1行目**にDOCTYPE宣言を記述してください。空行やコメントも前に入れてはいけません。

## 1.3 html要素：ルート要素

```html
<!DOCTYPE html>
<html lang="ja">
  <!-- ここにhead要素とbody要素が入る -->
</html>
```

### lang属性の重要性
- **スクリーンリーダー**が正しい言語で読み上げるために使われる
- **検索エンジン**がページの言語を判定するヒントにする
- **翻訳ツール**が翻訳の要否を判定する
- **ブラウザ**がフォント選択やハイフネーション処理の参考にする

### よく使うlang属性の値
| 値 | 言語 |
|---|---|
| `ja` | 日本語 |
| `en` | 英語 |
| `zh-Hans` | 中国語（簡体字） |
| `ko` | 韓国語 |
| `fr` | フランス語 |

## 1.4 head要素：メタ情報の格納

`<head>` 要素には、ページに関する**メタ情報**（ページ自体の設定や説明）を記述します。ブラウザの画面には直接表示されません。

```html
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>ページのタイトル</title>
</head>
```

### 1.4.1 meta charset — 文字エンコーディング

```html
<meta charset="UTF-8">
```

- **UTF-8** は世界中の文字をカバーするエンコーディング
- 日本語を含むページでは必ず指定する
- `<head>` の**最初の子要素**として記述するのが推奨
- これがないと日本語が**文字化け**する可能性がある

#### 文字化けが起こる仕組み
1. ファイルがUTF-8で保存されている
2. ブラウザがShift_JISとして解釈しようとする
3. バイト列の解釈が食い違い、意味不明な文字が表示される

> **対策**: HTMLファイル自体もUTF-8で保存し、`<meta charset="UTF-8">` を必ず記述しましょう。

### 1.4.2 meta viewport — レスポンシブ対応

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

- モバイルデバイスでの表示を最適化する
- `width=device-width`: 画面幅をデバイスの幅に合わせる
- `initial-scale=1.0`: 初期ズーム倍率を100%にする
- これがないとスマートフォンでページが縮小表示される

### 1.4.3 headに含められる要素一覧
| 要素 | 用途 | 例 |
|---|---|---|
| `<meta>` | メタ情報 | charset, viewport, description |
| `<title>` | ページタイトル | ブラウザタブに表示 |
| `<link>` | 外部リソース参照 | CSS、ファビコン |
| `<style>` | 内部CSS | ページ固有のスタイル |
| `<script>` | JavaScript | 動的な処理 |
| `<base>` | 基準URL | 相対パスの基準 |

## 1.5 title要素

```html
<title>私のホームページ | HTML入門</title>
```

### titleが使われる場所
1. **ブラウザのタブ**に表示される
2. **ブックマーク**のデフォルト名になる
3. **検索結果のタイトル**（検索エンジンが参考にする）
4. **SNSでシェア**されたときのタイトル

### 良いtitleの書き方
- **ページ固有の内容** を反映する（全ページ同じtitleはNG）
- **簡潔**で**わかりやすい**文言にする（30〜60文字程度）
- **キーワード**を含めるとSEOに有利
- サイト名は後ろに付ける：`ページ名 | サイト名`

---

## 1.6 body要素：ページの本体

`<body>` 要素には、ブラウザに**実際に表示される内容**をすべて記述します。

```html
<body>
  <h1>こんにちは、世界！</h1>
  <p>これは私の最初のWebページです。</p>
</body>
```

## 1.7 HTMLの基本テンプレート（まとめ）

以下が最小限のHTMLテンプレートです。**すべてのHTMLファイルはこの構造をベースに作成**してください。

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>ページタイトル</title>
</head>
<body>
  <!-- ここにコンテンツを記述 -->
</body>
</html>
```

### 各行の意味
| 行 | 意味 |
|---|---|
| `<!DOCTYPE html>` | HTML5文書であることを宣言 |
| `<html lang="ja">` | ルート要素、言語は日本語 |
| `<meta charset="UTF-8">` | 文字コードはUTF-8 |
| `<meta name="viewport" ...>` | モバイル対応 |
| `<title>` | ブラウザタブに表示されるタイトル |
| `<body>` | 画面に表示される内容 |

## 1.8 HTMLの基本ルール

### タグの種類
HTMLには2種類のタグがあります。

#### 1. 通常のタグ（開始タグ + 内容 + 終了タグ）
```html
<p>これは段落です。</p>
<h1>見出しです</h1>
<div>コンテナです</div>
```

#### 2. 空要素（自己完結型タグ）
内容を持たないタグです。終了タグは不要です。
```html
<br>        <!-- 改行 -->
<hr>        <!-- 水平線 -->
<img>       <!-- 画像 -->
<input>     <!-- 入力フィールド -->
<meta>      <!-- メタ情報 -->
<link>      <!-- 外部リソース -->
```

> **注意**: HTML5では `<br />` のようなスラッシュは不要ですが、書いてもエラーにはなりません。

### 属性（Attribute）
タグには追加の情報を**属性**として付けることができます。

```html
<a href="https://example.com" target="_blank">リンクテキスト</a>
```

- `href` と `target` が属性
- 属性値は**ダブルクォート**（`""`）で囲む
- 複数の属性はスペースで区切る
- 属性名は小文字で書くのが慣例

### コメント
```html
<!-- これはコメントです。ブラウザには表示されません。 -->

<!--
  複数行の
  コメントも
  書けます。
-->
```

### インデント（字下げ）
入れ子構造を見やすくするために、インデント（通常はスペース2個か4個）を使います。

```html
<!-- 良い例 -->
<div>
  <p>インデントされた段落</p>
</div>

<!-- 悪い例（読みにくい） -->
<div>
<p>インデントがない</p>
</div>
```

## 1.9 HTMLファイルの作成と表示

### テキストエディタの選択

HTMLを書くためには**テキストエディタ**が必要です。

| エディタ | 特徴 | 価格 |
|---|---|---|
| **Visual Studio Code** | 最も人気。拡張機能が豊富 | 無料 |
| Sublime Text | 軽量で高速 | 有料（試用可） |
| Atom | GitHubが開発（開発終了） | 無料 |
| メモ帳（Windows） | 標準搭載 | 無料 |

> **おすすめ**: Visual Studio Code（VSCode）をインストールしましょう。

### ファイルの作成手順
1. テキストエディタを開く
2. テンプレートを入力する（VSCodeでは `!` と入力してTabキーで自動生成）
3. `index.html` というファイル名で保存する（文字コードはUTF-8）
4. ファイルをブラウザにドラッグ＆ドロップ、またはダブルクリックで開く

### ファイル名のルール
- 拡張子は `.html`（または `.htm`）
- **半角英数字**を使う（日本語ファイル名は避ける）
- スペースの代わりに**ハイフン**（`-`）か**アンダースコア**（`_`）を使う
- 小文字を使う
- 例：`index.html`, `about-me.html`, `contact_form.html`

## 1.10 ブラウザの開発者ツール（DevTools）

### 開発者ツールの開き方
| ブラウザ | ショートカット |
|---|---|
| Chrome | `F12` または `Ctrl + Shift + I` |
| Firefox | `F12` または `Ctrl + Shift + I` |
| Edge | `F12` または `Ctrl + Shift + I` |
| Safari | `Cmd + Option + I`（要設定） |

### 主要なタブ

#### Elements（要素）タブ
- HTMLの**DOM構造**をツリー形式で確認できる
- 要素をクリックすると、適用されているCSSが表示される
- 要素を右クリックして「Edit as HTML」でリアルタイム編集ができる
- **活用場面**: HTMLの構造を確認したい、なぜ表示が崩れているか調べたい

#### Console（コンソール）タブ
- **JavaScriptのエラー**が表示される
- JavaScriptのコードを直接実行できる
- `console.log()` の出力を確認できる
- **活用場面**: エラーの原因調査、簡単なJavaScript実行

#### Network（ネットワーク）タブ
- ページが読み込む**すべてのファイル**（HTML, CSS, JS, 画像等）を一覧表示
- 各ファイルの**読み込み時間**やサイズを確認できる
- **活用場面**: ページが重い原因の調査、リソースの読み込み確認

#### Sources（ソース）タブ
- ページのソースコードを閲覧できる
- **ブレークポイント**を設定してJavaScriptのデバッグができる

### DevToolsでよく使う操作
1. **要素の検証**: ページ上で右クリック →「検証」（Inspect）
2. **デバイスエミュレーション**: DevToolsの左上にあるデバイスアイコンをクリック → スマートフォン表示をシミュレート
3. **リアルタイム編集**: Elements タブでHTMLやCSSを一時的に変更して表示を確認

> **注意**: DevTools上での変更は一時的なものです。ページをリロードすると元に戻ります。ファイルを修正するにはエディタで編集する必要があります。

## 1.11 よくあるエラーと対処法

### 文字化け
- **原因**: `<meta charset="UTF-8">` がない、またはファイルがUTF-8以外で保存されている
- **対処**: meta charsetを追加し、エディタの文字コード設定をUTF-8にして保存し直す

### ページが真っ白
- **原因**: `<body>` タグ内に何も記述していない、またはタグの閉じ忘れ
- **対処**: bodyの中にテキストを入れる、タグの対応を確認する

### タグが画面に文字として表示される
- **原因**: ファイルの拡張子が `.html` ではない（例: `.txt`）
- **対処**: 拡張子を `.html` に変更する

### 閉じタグの忘れ
- **原因**: `<p>テキスト` のように終了タグ `</p>` がない
- **対処**: 開始タグを書いたら、すぐに終了タグも書く習慣をつける
