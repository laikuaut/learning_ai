## 1.1 AIエージェントとは

### 定義

AIエージェント（AI Agent）とは、**自律的にタスクを遂行するAIシステム**のことです。人間が逐一指示を出さなくても、与えられた目標に対して自ら考え、判断し、行動を実行します。

従来のチャットAI（対話型AI）との最大の違いは、**行動を起こせるかどうか**です。

```
┌──────────────────────────────────────────────────────┐
│           従来のチャットAI                             │
│                                                      │
│   ユーザー ──(質問)──> AI ──(回答)──> ユーザー         │
│                                                      │
│   ※ 回答を返すだけ。実際の作業は人間が行う             │
└──────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────┐
│           AIエージェント                               │
│                                                      │
│   ユーザー ──(目標)──> AI ──(計画)──> ツール実行       │
│                          ↑              │            │
│                          └──(結果確認)──┘            │
│                                                      │
│   ※ 自分で考え、ツールを使い、結果を確認しながら      │
│     タスクを完了するまで繰り返す                       │
└──────────────────────────────────────────────────────┘
```

### 「知覚→思考→行動」のループ

AIエージェントの動作は、以下の3ステップのループで表現できます。

```
    ┌─────────┐
    │  知覚    │  環境やユーザーからの情報を受け取る
    │ Perceive│
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │  思考    │  情報を分析し、次の行動を決定する
    │  Think  │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │  行動    │  ツールを使って実際にアクションを起こす
    │   Act   │
    └────┬────┘
         │
         └──────→ 結果を知覚してループを繰り返す
```

たとえば「このプロジェクトのバグを修正して」と依頼された場合、AIエージェントは次のように動作します。

1. **知覚**：エラーログやソースコードを読み取る
2. **思考**：原因を分析し、修正方針を決める
3. **行動**：コードを編集し、テストを実行する
4. **知覚**：テスト結果を確認する
5. **思考**：テストが通ったか判断する
6. **行動**：通っていなければ再修正、通っていれば完了を報告する

### 従来のAIとの比較

| 特徴 | 従来のチャットAI | AIエージェント |
|---|---|---|
| 入出力 | テキスト→テキスト | 目標→成果物 |
| 自律性 | 受動的（質問に答える） | 能動的（自ら行動する） |
| ツール使用 | なし | ファイル操作、API呼び出し等 |
| 複数ステップ | 1回の応答で完結 | 複数ステップを自律的に実行 |
| 環境との相互作用 | なし | あり（結果を観察してフィードバック） |
| エラー対応 | ユーザーが再指示 | 自己修正を試みる |

> **ポイントまとめ**
> - AIエージェントは「自律的にタスクを遂行するAIシステム」
> - 従来のチャットAIとの違いは「行動を起こせること」と「自律的なループ」
> - 知覚→思考→行動のサイクルを繰り返してタスクを完了する

## 1.2 AIエージェントの歴史と進化

AIエージェントの概念は新しいものではありませんが、近年のLLM（大規模言語モデル）の発展により急速に実用化が進みました。

### 進化の流れ

```
時代          手法                          特徴
─────────────────────────────────────────────────────────
1980年代〜    ルールベース                   if-then ルールで動作
              (Rule-based)                  人間が全ルールを定義

2000年代〜    機械学習ベース                 データから学習
              (ML-based)                    特定タスクに特化

2020年〜      LLMベース                     自然言語で推論
              (LLM-based)                   汎用的なタスク対応

2024年〜      LLMエージェント               ツール使用 + 自律行動
              (LLM Agent)                   複雑なタスクを自律遂行
```

### 2024-2026年の急速な発展

| 時期 | 出来事 |
|---|---|
| 2024年前半 | GPT-4、Claude 3 などがFunction Calling（関数呼び出し）を本格サポート |
| 2024年後半 | Anthropic が Computer Use（コンピュータ操作）を発表、Devin が登場 |
| 2025年前半 | Claude Code、OpenAI Codex などコーディングエージェントが普及 |
| 2025年後半 | マルチエージェントシステムが実用レベルに到達 |
| 2026年 | エージェント同士の連携、企業での本格導入が進む |

この進化を支えた技術的要因は主に3つです。

1. **LLMの推論能力向上**：複雑なタスクの計画・分解が可能に
2. **ツール使用の標準化**：Function Calling の仕様が整備された
3. **コンテキストウィンドウの拡大**：より多くの情報を保持して推論可能に

> **ポイントまとめ**
> - ルールベース→ML→LLM→LLMエージェントと進化してきた
> - 2024年以降、ツール使用とコンテキスト拡大により実用化が加速
> - 現在はマルチエージェントやエージェント間連携の段階に入っている

## 1.3 エージェントの分類

AIエージェントはさまざまな軸で分類できます。

### 構成による分類

| 分類 | 説明 | 例 |
|---|---|---|
| シングルエージェント（Single Agent） | 1つのAIが全てを担当 | Claude Code、GitHub Copilot |
| マルチエージェント（Multi-Agent） | 複数のAIが役割分担して協調 | CrewAI、AutoGen |

```
シングルエージェント:
┌──────────────────────┐
│   エージェント        │
│  ┌────┐ ┌────┐      │
│  │計画│→│実行│→ 完了 │
│  └────┘ └────┘      │
└──────────────────────┘

マルチエージェント:
┌──────────┐   ┌──────────┐   ┌──────────┐
│ 計画担当  │──>│ 実行担当  │──>│ 検証担当  │
│エージェント│   │エージェント│   │エージェント│
└──────────┘   └──────────┘   └──────────┘
```

### 動作方式による分類

| 分類 | 説明 | 特徴 |
|---|---|---|
| リアクティブ（Reactive） | 入力に対して即座に反応 | 高速だが計画性なし |
| プロアクティブ（Proactive） | 目標に向けて能動的に行動 | 計画的だが処理に時間がかかる |

### 用途による分類

| 分類 | 説明 | 例 |
|---|---|---|
| タスク特化型（Specialized） | 特定のタスクに最適化 | コーディング、データ分析 |
| 汎用型（General-purpose） | 幅広いタスクに対応 | AutoGPT、汎用アシスタント |

実用的には、**タスク特化型のシングルエージェント**が現時点で最も信頼性が高く、多くの場面で活用されています。

> **ポイントまとめ**
> - シングル/マルチ、リアクティブ/プロアクティブ、特化型/汎用型で分類できる
> - 現時点ではタスク特化型シングルエージェントが最も実用的
> - マルチエージェントは複雑なタスクに有効だが、制御が難しい

## 1.4 エージェントの構成要素

AIエージェントは、大きく分けて **4つの要素** で構成されます。

```
┌───────────────────────────────────────────────┐
│              AIエージェント                     │
│                                               │
│   ┌─────────────────────────────────────┐     │
│   │        LLM（頭脳 / Brain）           │     │
│   │   自然言語の理解・推論・判断          │     │
│   └──────────────┬──────────────────────┘     │
│                  │                             │
│   ┌──────────────┼──────────────────────┐     │
│   │              │                      │     │
│   ▼              ▼                      ▼     │
│ ┌──────┐   ┌──────────┐   ┌──────────┐       │
│ │ツール │   │ メモリ    │   │プランニング│      │
│ │(手足) │   │ (記憶)   │   │ (計画)    │      │
│ │      │   │          │   │          │       │
│ │API   │   │短期記憶  │   │タスク分解 │       │
│ │ファイル│  │長期記憶  │   │優先順位   │       │
│ │DB    │   │作業記憶  │   │ステップ管理│      │
│ └──────┘   └──────────┘   └──────────┘       │
└───────────────────────────────────────────────┘
```

### 各要素の役割

**1. LLM（大規模言語モデル / Large Language Model）- 頭脳**

エージェントの中核です。自然言語を理解し、状況を分析し、次に何をすべきか判断します。Claude、GPT-4、Gemini などが使われます。

**2. ツール（Tools）- 手足**

エージェントが実際に外部と相互作用するための手段です。

- ファイルの読み書き
- Web検索・API呼び出し
- コードの実行
- データベースへのクエリ
- コンピュータの操作（マウス・キーボード）

**3. メモリ（Memory）- 記憶**

エージェントが情報を保持する仕組みです。

- **短期メモリ**：現在の会話やタスクの文脈（コンテキストウィンドウ）
- **長期メモリ**：過去の経験や知識（外部データベース、ファイル）
- **作業メモリ**：現在のタスクの中間結果や計画

**4. プランニング（Planning）- 計画**

複雑なタスクをステップに分解し、実行順序を決める仕組みです。

- タスク分解（大きな目標を小さな手順に分ける）
- 優先順位付け（どの手順から実行するか）
- 進捗管理（どこまで完了したか追跡する）

> **ポイントまとめ**
> - エージェントはLLM・ツール・メモリ・プランニングの4要素で構成される
> - LLMが頭脳、ツールが手足、メモリが記憶、プランニングが計画を担う
> - これら4要素の組み合わせにより、自律的なタスク遂行が可能になる

## 1.5 主要なAIエージェントの実例

2025-2026年現在、さまざまなAIエージェントが実用化されています。

### コーディングエージェント

| エージェント | 開発元 | 特徴 |
|---|---|---|
| Claude Code | Anthropic | ターミナルで動作するコーディングエージェント。ファイル編集・コマンド実行・Git操作を自律的に実行 |
| Devin | Cognition | 「AIソフトウェアエンジニア」として開発タスク全体を遂行。独自の開発環境を持つ |
| OpenAI Codex | OpenAI | クラウドサンドボックスでコードを書き、テストまで自動実行 |
| GitHub Copilot（Agent mode） | GitHub/Microsoft | IDE内でコード生成からファイル編集、ターミナル操作まで対応 |

### 汎用エージェント

| エージェント | 特徴 |
|---|---|
| AutoGPT | 初期のLLMエージェント。目標を与えると自律的にタスクを分解・実行 |
| Computer Use（Claude） | GUIを操作するエージェント。マウスやキーボードでアプリケーションを操作 |

### 各エージェントの動作例

Claude Code を使ったバグ修正の流れを見てみましょう。

```
ユーザー: 「このテストが失敗しているので修正して」

Claude Code の動作:
1. [知覚] テストファイルを読み込む
2. [思考] テストの失敗原因を分析
3. [行動] 関連するソースコードを検索
4. [知覚] ソースコードの内容を確認
5. [思考] バグの原因を特定し、修正方針を決定
6. [行動] コードを修正
7. [行動] テストを再実行
8. [知覚] テスト結果を確認 → 全テスト成功
9. [行動] 修正内容をコミット
```

> **ポイントまとめ**
> - コーディングエージェント（Claude Code、Devin等）が最も実用化が進んでいる
> - 汎用エージェントはまだ発展途上だが急速に改善している
> - 各エージェントには得意分野と制約がある

## 1.6 AIエージェントの可能性と限界

### できること（可能性）

- **反復的な作業の自動化**：定型的なコード修正、データ処理、レポート生成
- **複雑なタスクの分解と遂行**：大きなプロジェクトを小さなステップに分けて実行
- **24時間稼働**：人間が休んでいる間も作業を継続
- **一貫した品質**：疲労によるミスがない（ただし別種のミスはある）
- **高速な情報検索と分析**：大量のコードやドキュメントを瞬時に調査

### できないこと（限界）

- **真の創造性**：完全に新しい概念の創出は困難
- **物理的な作業**：現実世界での作業はできない（ロボティクスとの連携は進行中）
- **完璧な判断**：曖昧な要件や高度なビジネス判断は人間が必要
- **最新情報のリアルタイム把握**：学習データのカットオフ以降の情報には限りがある

### よくある間違い

| 間違い | 正しい理解 |
|---|---|
| 「AIエージェントは完璧に動作する」 | ハルシネーション（幻覚）や判断ミスは起こりうる。人間による確認が必要 |
| 「放っておけば何でもやってくれる」 | 適切な指示（プロンプト）と監督が必要。暴走を防ぐためのガードレールが重要 |
| 「従来のAIと大差ない」 | ツール使用と自律的ループにより、質的に異なるレベルのタスクが遂行可能 |
| 「人間の仕事がなくなる」 | エージェントは人間の能力を拡張するもの。指示・判断・監督は人間の役割 |

### 安全性の課題

AIエージェントには特有のリスクがあります。

1. **ハルシネーション（Hallucination）**：事実と異なる内容を自信を持って出力する
2. **意図しない行動**：指示を誤解して予期しない操作を実行する
3. **過剰な権限**：必要以上の権限を与えるとセキュリティリスクになる
4. **連鎖的なエラー**：1つのミスが後続の全ステップに波及する

これらのリスクに対処するため、以下の対策が重要です。

- **最小権限の原則**：必要最小限の権限だけを与える
- **人間による承認ステップ**：重要な操作の前に確認を求める
- **サンドボックス環境**：安全な隔離環境で実行する
- **ログと監査**：全ての行動を記録し、後から検証可能にする

> **ポイントまとめ**
> - AIエージェントは反復作業や複雑なタスクの自動化に強い
> - ハルシネーション・意図しない行動・権限管理が主要な課題
> - 人間による監督と適切なガードレールの設計が不可欠
> - エージェントは「人間を置き換える」ものではなく「人間を拡張する」もの
