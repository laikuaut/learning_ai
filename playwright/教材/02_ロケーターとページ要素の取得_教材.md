# 第2章：ロケーターとページ要素の取得

## この章のゴール

- ロケーターの概念と役割を理解する
- 推奨されるロケーターの使い分けができる
- `getByRole`、`getByText`、`getByLabel` などを使いこなす
- CSSセレクター・XPathとの違いを理解する
- ロケーターの絞り込み（フィルター・チェーン）ができる

---

## 1. ロケーターとは

**ロケーター（Locator）** は、ページ上の要素を見つけるための「住所」のようなものです。テストでボタンをクリックしたり、テキストを入力したりするには、まず対象の要素をロケーターで特定する必要があります。

```
┌─────────────────────────────────────────────────────┐
│                  ブラウザのページ                     │
│                                                      │
│  ┌──────────────┐  ┌──────────────────────┐         │
│  │ ログイン      │  │ メールアドレス [    ] │         │
│  │               │  │ パスワード    [    ] │         │
│  │               │  │        [ログイン]    │         │
│  └──────────────┘  └──────────────────────┘         │
│                                                      │
│  ロケーター = 要素への「住所」                        │
│                                                      │
│  page.getByRole('button', { name: 'ログイン' })      │
│  page.getByLabel('メールアドレス')                    │
│  page.getByLabel('パスワード')                        │
└─────────────────────────────────────────────────────┘
```

### ロケーターの重要性

| 良いロケーター | 悪いロケーター |
|---------------|---------------|
| UIの意味に基づいている | HTMLの内部構造に依存している |
| レイアウト変更に強い | CSS変更で壊れる |
| 読んで意図がわかる | 暗号的で意味不明 |
| `getByRole('button', { name: '送信' })` | `page.locator('#root > div:nth-child(3) > button')` |

## 2. 推奨ロケーター（優先度順）

Playwrightでは、以下の優先順位でロケーターを使うことが推奨されています。

```
┌──────────────────────────────────────────────────────────┐
│  優先度   ロケーター        用途                          │
├──────────────────────────────────────────────────────────┤
│  ★★★    getByRole        ロール（役割）で探す            │
│  ★★★    getByText        表示テキストで探す              │
│  ★★★    getByLabel       ラベルで探す（フォーム向け）    │
│  ★★☆    getByPlaceholder プレースホルダーで探す          │
│  ★★☆    getByAltText     alt属性で探す（画像向け）       │
│  ★★☆    getByTitle       title属性で探す                 │
│  ★★☆    getByTestId      data-testid属性で探す           │
│  ★☆☆    locator          CSSセレクター/XPathで探す       │
└──────────────────────────────────────────────────────────┘
```

---

## 3. `getByRole` — 最も推奨されるロケーター

`getByRole` は、HTML要素の**アクセシビリティロール**（役割）で要素を見つけます。ユーザーがページをどう認識するかに近い方法です。

### 基本的な使い方

```typescript
// ボタンを探す
page.getByRole('button', { name: 'ログイン' });

// リンクを探す
page.getByRole('link', { name: 'ホーム' });

// 見出しを探す
page.getByRole('heading', { name: 'ようこそ' });

// テキストボックスを探す
page.getByRole('textbox', { name: 'メールアドレス' });

// チェックボックスを探す
page.getByRole('checkbox', { name: '利用規約に同意する' });
```

### 主要なロール一覧

| ロール | 対応するHTML要素 | 例 |
|--------|-----------------|-----|
| `button` | `<button>`, `<input type="submit">` | ボタン |
| `link` | `<a href="...">` | リンク |
| `heading` | `<h1>`〜`<h6>` | 見出し |
| `textbox` | `<input type="text">`, `<textarea>` | テキスト入力 |
| `checkbox` | `<input type="checkbox">` | チェックボックス |
| `radio` | `<input type="radio">` | ラジオボタン |
| `combobox` | `<select>` | ドロップダウン |
| `list` | `<ul>`, `<ol>` | リスト |
| `listitem` | `<li>` | リスト項目 |
| `navigation` | `<nav>` | ナビゲーション |
| `img` | `<img>` | 画像 |
| `dialog` | `<dialog>` | ダイアログ |
| `table` | `<table>` | テーブル |
| `row` | `<tr>` | テーブルの行 |
| `cell` | `<td>` | テーブルのセル |

### `getByRole` のオプション

```typescript
// name: アクセシブルネーム（テキスト内容やaria-label）で絞り込み
page.getByRole('button', { name: 'ログイン' });

// exact: 完全一致にする（デフォルトは部分一致）
page.getByRole('button', { name: 'ログイン', exact: true });

// level: 見出しのレベルを指定
page.getByRole('heading', { level: 1 });  // h1のみ

// checked: チェック状態で絞り込み
page.getByRole('checkbox', { checked: true });

// pressed: 押下状態で絞り込み
page.getByRole('button', { pressed: true });
```

## 4. テキスト系ロケーター

### `getByText` — 表示テキストで探す

画面に表示されているテキストで要素を見つけます。

```typescript
// テキストを含む要素を探す（部分一致）
page.getByText('ようこそ');

// 完全一致
page.getByText('ようこそ、太郎さん', { exact: true });

// 正規表現
page.getByText(/ようこそ.*さん/);
```

### `getByLabel` — フォームのラベルで探す

`<label>` 要素と紐づいた入力フィールドを見つけます。

```html
<!-- HTML -->
<label for="email">メールアドレス</label>
<input id="email" type="email" />
```

```typescript
// ラベルテキストで入力欄を取得
page.getByLabel('メールアドレス');
```

### `getByPlaceholder` — プレースホルダーで探す

```html
<input placeholder="名前を入力" />
```

```typescript
page.getByPlaceholder('名前を入力');
```

### `getByAltText` — 画像のalt属性で探す

```html
<img src="logo.png" alt="会社のロゴ" />
```

```typescript
page.getByAltText('会社のロゴ');
```

### `getByTitle` — title属性で探す

```html
<span title="設定を開く">⚙</span>
```

```typescript
page.getByTitle('設定を開く');
```

## 5. `getByTestId` — テスト専用属性

UIの見た目や文言が頻繁に変わる場合、`data-testid` 属性を使います。

```html
<button data-testid="submit-button">今すぐ購入</button>
```

```typescript
page.getByTestId('submit-button');
```

### いつ使うべきか

```
┌────────────────────────────────────────────────────┐
│ getByTestId を使うべき場面                          │
├────────────────────────────────────────────────────┤
│ ・テキストがよく変わる（「購入」→「今すぐ購入」）   │
│ ・同じロールの要素が多く区別が難しい                │
│ ・多言語対応でテキストが言語ごとに違う              │
│ ・アイコンボタンなどテキストがない要素              │
├────────────────────────────────────────────────────┤
│ getByTestId を避けるべき場面                        │
├────────────────────────────────────────────────────┤
│ ・getByRole や getByLabel で十分に特定できる場合    │
│ ・テスト専用属性を追加する権限がない場合            │
└────────────────────────────────────────────────────��
```

### カスタム属性名の設定

`data-testid` 以外の属性名を使いたい場合、`playwright.config.ts` で変更できます。

```typescript
// playwright.config.ts
export default defineConfig({
  use: {
    testIdAttribute: 'data-test',  // data-test を使う
  },
});
```

## 6. CSSセレクターと XPath

`getByRole` 等で見つからない場合、`page.locator()` でCSSセレクターやXPathが使えます。

### CSSセレクター

```typescript
// IDで探す
page.locator('#submit-btn');

// クラスで探す
page.locator('.error-message');

// 属性で探す
page.locator('[data-status="active"]');

// 複合セレクター
page.locator('form .input-field');

// 擬似クラス
page.locator('li:first-child');
```

### XPath

```typescript
// XPathで探す
page.locator('xpath=//button[contains(text(), "送信")]');
```

> **注意**: CSSセレクターやXPathは、HTMLの構造変更で壊れやすいです。`getByRole` 等を優先し、最後の手段として使いましょう。

## 7. ロケーターの絞り込み

### `filter` — 条件で絞り込む

```typescript
// テキストで絞り込む
page.getByRole('listitem').filter({ hasText: 'りんご' });

// 正規表現で絞り込む
page.getByRole('listitem').filter({ hasText: /\d+円/ });

// 特定の子要素を持つものに絞り込む
page.getByRole('listitem').filter({
  has: page.getByRole('button', { name: '削除' }),
});

// 条件を否定する（持たないものを選ぶ）
page.getByRole('listitem').filter({
  hasNot: page.getByRole('button', { name: '完了' }),
});
```

### チェーン — 親から子へ辿る

```typescript
// ナビゲーション内のリンクを探す
page.getByRole('navigation').getByRole('link', { name: 'ホーム' });

// フォーム内のテキストボックスを探す
page.locator('form#login').getByLabel('メールアドレス');

// リスト内の特定アイテム内のボタン
page.getByRole('listitem')
  .filter({ hasText: 'りんご' })
  .getByRole('button', { name: '削除' });
```

### `nth` — n番目の要素

```typescript
// 最初の要素
page.getByRole('listitem').first();

// 最後の要素
page.getByRole('listitem').last();

// n番目の要素（0始まり）
page.getByRole('listitem').nth(2);  // 3番目
```

## 8. 実践：ロケーターの選び方

以下のHTMLに対して、良いロケーターと悪いロケーターを比較します。

```html
<form id="login-form" class="auth-form">
  <h2>ログイン</h2>
  <div class="form-group">
    <label for="email">メールアドレス</label>
    <input id="email" type="email" placeholder="example@mail.com" />
  </div>
  <div class="form-group">
    <label for="password">パスワード</label>
    <input id="password" type="password" placeholder="8文字以上" />
  </div>
  <div class="form-group">
    <input type="checkbox" id="remember" />
    <label for="remember">ログイン状態を保持する</label>
  </div>
  <button type="submit" class="btn btn-primary">ログイン</button>
  <a href="/forgot-password">パスワードを忘れた方はこちら</a>
</form>
```

```typescript
// ✅ 良いロケーター — ユーザーの視点に近い
const emailInput = page.getByLabel('メールアドレス');
const passwordInput = page.getByLabel('パスワード');
const rememberCheckbox = page.getByRole('checkbox', { name: 'ログイン状態を保持する' });
const loginButton = page.getByRole('button', { name: 'ログイン' });
const forgotLink = page.getByRole('link', { name: 'パスワードを忘れた方はこちら' });

// ❌ 悪いロケーター — HTMLの構造に依存
const emailInput2 = page.locator('#login-form > div:nth-child(2) > input');
const loginButton2 = page.locator('.btn.btn-primary');
const forgotLink2 = page.locator('form a[href="/forgot-password"]');
```

## まとめ

| 項目 | ポイント |
|------|---------|
| ロケーター | ページ上の要素を見つけるための「住所」 |
| 優先順位 | `getByRole` > `getByText`/`getByLabel` > `getByTestId` > `locator` |
| `getByRole` | アクセシビリティロールで探す。最も推奨 |
| `getByLabel` | フォーム要素に最適 |
| `getByText` | 表示テキストで探す |
| `getByTestId` | テスト専用属性。UIが不安定な場合に使う |
| フィルター | `.filter({ hasText: '...' })` で絞り込み |
| チェーン | `.getByRole(...).getByRole(...)` で親→子へ辿る |
| nth | `.first()`, `.last()`, `.nth(n)` でn番目を選択 |

### ポイントまとめ

- **ロケーターはユーザーの視点で書く** — 「IDが `#email` の要素」ではなく「メールアドレスの入力欄」
- **HTML構造に依存するロケーターは避ける** — リファクタリングで壊れる
- **`getByRole` を最初に検討する** — アクセシビリティも同時にテストできる
