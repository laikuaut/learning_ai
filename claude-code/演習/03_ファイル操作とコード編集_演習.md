## 基本問題

### 問題1：ツールの役割

Claude Codeの各ツールと、その主な役割を正しく組み合わせてください。

| ツール | 役割 |
|---|---|
| 1. Read | A. ファイル名パターンで検索 |
| 2. Write | B. 既存ファイルの部分編集 |
| 3. Edit | C. ファイルの内容を読み込む |
| 4. Glob | D. シェルコマンドを実行 |
| 5. Grep | E. 新規ファイルの作成・上書き |
| 6. Bash | F. ファイル内容のテキスト検索 |

**期待される解答：**
```
1-?, 2-?, 3-?, 4-?, 5-?, 6-?
```

<details>
<summary>ヒント</summary>

- Read（読む）、Write（書く）、Edit（編集する）はそのまま英語の意味通りです
- Globはファイル名のパターンマッチング、Grepは中身の検索です

</details>

<details>
<summary>解答例</summary>

```
1-C, 2-E, 3-B, 4-A, 5-F, 6-D
```

- **Read**: ファイルの内容を読み込んで表示します
- **Write**: 新規ファイルを作成するか、既存ファイルを完全に上書きします
- **Edit**: 既存ファイルの特定の部分だけを変更します（差分適用）
- **Glob**: `**/*.ts`のようなパターンでファイルを検索します
- **Grep**: 正規表現でファイルの中身を検索します
- **Bash**: ターミナルコマンド（npm test、ls等）を実行します

</details>

### 問題2：Globパターンの理解

以下の検索目的に合ったGlobパターンを書いてください。

1. プロジェクト全体のJavaScriptファイルを検索
2. `src/components/`直下のTSXファイルのみを検索
3. すべてのテストファイル（`.test.ts`または`.spec.ts`）を検索
4. `docs/`ディレクトリ内のMarkdownファイルを検索（サブディレクトリ含む）

**期待される解答：**
```
1. ___
2. ___
3. ___
4. ___
```

<details>
<summary>ヒント</summary>

- `**`は任意の階層のディレクトリにマッチします
- `*`は任意のファイル名にマッチします
- `{a,b}`はaまたはbにマッチします

</details>

<details>
<summary>解答例</summary>

```
1. **/*.js
   （**で全階層を対象とし、*.jsでJSファイルにマッチ）

2. src/components/*.tsx
   （直下のみなので**は不要。*.tsxでTSXファイルにマッチ）

3. **/*.{test,spec}.ts
   （{test,spec}でどちらのパターンにもマッチ）

4. docs/**/*.md
   （docs/配下の全階層で*.mdファイルを検索）
```

</details>

### 問題3：WriteとEditの使い分け

以下の操作で使われるツールは Write と Edit のどちらですか？

1. 新しい設定ファイル `.eslintrc.json` を作成する
2. 既存の `app.ts` の import 文を1行追加する
3. 既存のファイルを完全に書き直す
4. 関数内のバグを1行修正する
5. 変数名 `userName` を `username` にすべて変更する

**期待される解答：**
```
1. ___
2. ___
3. ___
4. ___
5. ___
```

<details>
<summary>ヒント</summary>

- Writeは新規作成や全体の書き直し
- Editは既存ファイルの一部分の変更

</details>

<details>
<summary>解答例</summary>

```
1. Write （新規ファイルの作成はWriteを使います）
2. Edit  （既存ファイルへの部分的な追加はEditです）
3. Write （ファイル全体の書き直しはWriteを使います）
4. Edit  （1行の修正は部分編集なのでEditです）
5. Edit  （変数名の一括変更はEditのreplace_all機能です）
```

</details>

### 問題5：複数ファイル操作のシナリオ

あなたのプロジェクトでは、APIのベースURL定数が複数のファイルに直接ハードコーディングされています。これを1つの設定ファイルに集約したいとします。

Claude Codeに出すプロンプトを、段階的に3ステップで設計してください。

**現状：**
```
// src/services/userService.ts
const API_URL = "https://api.example.com/v1";

// src/services/productService.ts
const API_URL = "https://api.example.com/v1";

// src/services/orderService.ts
const API_URL = "https://api.example.com/v1";
```

<details>
<summary>ヒント</summary>

1. まず現状を把握する（どのファイルにハードコーディングされているか）
2. 設定ファイルを作成する
3. 各ファイルを更新する

</details>

<details>
<summary>解答例</summary>

```
Step 1: 現状の把握
> プロジェクト内で "https://api.example.com" がハードコーディング
> されている箇所をすべてリストアップしてください。

Step 2: 設定ファイルの作成
> src/config/api.ts を作成して、API_BASE_URL 定数を
> エクスポートしてください。値は "https://api.example.com/v1" です。

Step 3: 各ファイルの更新
> ハードコーディングされていた全ファイルを更新して、
> src/config/api.ts から API_BASE_URL をインポートするように
> 変更してください。ローカルの API_URL 定数は削除してください。
```

- 段階的に進めることで、各ステップの結果を確認できます
- Claude Codeはファイル間の依存関係を考慮して更新します
- 更新後はテストを実行して動作確認するのが良い習慣です

</details>

## チャレンジ問題

### 問題6：リファクタリング指示の設計

以下のコードには複数の問題があります。Claude Codeに対して、具体的なリファクタリング指示を3つ作成してください。

```typescript
// src/services/dataProcessor.ts
export function processData(data: any) {
  let result = [];
  for (let i = 0; i < data.length; i++) {
    if (data[i].status == "active") {
      if (data[i].age > 18) {
        if (data[i].name != null && data[i].name != undefined && data[i].name != "") {
          result.push({
            name: data[i].name.charAt(0).toUpperCase() + data[i].name.slice(1),
            age: data[i].age,
            email: data[i].email ? data[i].email.toLowerCase() : "no-email"
          });
        }
      }
    }
  }
  console.log("Processed " + result.length + " items");
  return result;
}
```

<details>
<summary>ヒント</summary>

以下の問題点に注目してください：
- `any`型の使用
- ネストが深いif文
- `==`と`!=`の使用（`===`と`!==`であるべき）
- `console.log`の使用

</details>

<details>
<summary>解答例</summary>

**指示1: 型安全性の改善**
```
> src/services/dataProcessor.ts の processData 関数の引数 data の型を
> any から適切なインターフェースに変更してください。
> data の各要素は status(string), age(number), name(string|null),
> email(string|null) を持ちます。戻り値の型も定義してください。
```

**指示2: ネストの解消とモダンな書き方**
```
> src/services/dataProcessor.ts の processData 関数で、
> ネストしたif文を早期returnまたはfilter/mapを使った
> 関数型スタイルにリファクタリングしてください。
> また、== と != を === と !== に修正してください。
```

**指示3: console.logの除去**
```
> src/services/dataProcessor.ts から console.log を削除してください。
> デバッグ用のログが必要な場合は、代わりに関数の戻り値に
> 処理件数の情報を含めるか、別途ロガーを使う設計を提案してください。
```

リファクタリング後のコード例：
```typescript
interface UserData {
  status: string;
  age: number;
  name: string | null;
  email: string | null;
}

interface ProcessedUser {
  name: string;
  age: number;
  email: string;
}

export function processData(data: UserData[]): ProcessedUser[] {
  return data
    .filter(item => item.status === "active")
    .filter(item => item.age > 18)
    .filter(item => item.name !== null && item.name !== undefined && item.name !== "")
    .map(item => ({
      name: item.name!.charAt(0).toUpperCase() + item.name!.slice(1),
      age: item.age,
      email: item.email ? item.email.toLowerCase() : "no-email"
    }));
}
```

</details>

### 問題7：実践シナリオ - プロジェクト構成の作成

新しいExpressプロジェクトのベース構成をClaude Codeで作成します。以下のファイルを作成するためのプロンプトを1つ設計してください。

**作成したいファイル構成：**
```
src/
├── index.ts          # エントリポイント（Express起動）
├── routes/
│   └── index.ts      # ルーティング定義
├── controllers/
│   └── healthController.ts  # ヘルスチェック
├── middleware/
│   └── errorHandler.ts      # エラーハンドリング
└── config/
    └── app.ts        # アプリ設定
```

<details>
<summary>ヒント</summary>

- 1つのプロンプトで複数ファイルの作成を依頼できます
- 各ファイルの役割と技術的な要件を明記すると精度が上がります
- TypeScript + Express という技術スタックを明示しましょう

</details>

<details>
<summary>解答例</summary>

```
> TypeScript + Express のプロジェクトベースを以下の構成で
> 作成してください：
>
> src/index.ts - Expressサーバーのエントリポイント（ポート3000）
> src/routes/index.ts - ルーティング定義（/api/health のルート）
> src/controllers/healthController.ts - GET /api/health で
>   { status: "ok", timestamp: <現在時刻> } を返す
> src/middleware/errorHandler.ts - グローバルエラーハンドリング
>   ミドルウェア（500エラーのJSON応答）
> src/config/app.ts - ポート番号等のアプリ設定を定数で管理
>
> 各ファイルは相互に正しくインポートし、
> src/index.ts から起動できるようにしてください。
```

ポイント：
- ファイル構成と各ファイルの役割を具体的に記述
- 技術スタック（TypeScript + Express）を明記
- ファイル間の連携（インポート関係）も指定
- エントリポイントから動作することを要件に含める

</details>
