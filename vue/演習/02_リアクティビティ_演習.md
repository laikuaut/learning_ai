## 演習2-1：ref と reactive の使い分け（基本）

### 問題

以下の要件を満たす `UserProfile.vue` を作成してください。

1. `ref` を使って管理する変数:
   - `age`（数値: `25`）
   - `isOnline`（真偽値: `true`）
2. `reactive` を使って管理する変数:
   - `user` オブジェクト: `{ name: '太郎', email: 'taro@example.com', hobbies: ['読書', 'プログラミング'] }`
3. テンプレートに全ての情報を表示する
4. 「1歳年を取る」ボタンで `age` を増やす
5. `user.hobbies` に新しい趣味を追加するボタンを作る

<details>
<summary>ヒント</summary>

- `ref` はプリミティブ値（数値、文字列、真偽値）に適しています
- `reactive` はオブジェクトや配列に適しています
- `reactive` で定義した変数は `.value` が不要です
- 配列に要素を追加するには `.push()` を使います

</details>

<details>
<summary>解答例</summary>

```vue
<!-- UserProfile.vue -->
<script setup>
import { ref, reactive } from 'vue'

// プリミティブ値には ref を使う
const age = ref(25)
const isOnline = ref(true)

// オブジェクトには reactive を使う
// reactive で作った変数は .value が不要で、直接プロパティにアクセスできる
const user = reactive({
  name: '太郎',
  email: 'taro@example.com',
  hobbies: ['読書', 'プログラミング']
})

// 年齢を1つ増やす
function incrementAge() {
  age.value++  // ref なので .value が必要
}

// 趣味を追加する
function addHobby() {
  // reactive オブジェクトのプロパティは直接変更できる
  // 配列の push もリアクティブに検知される
  const newHobby = prompt('新しい趣味を入力してください')
  if (newHobby) {
    user.hobbies.push(newHobby)
  }
}
</script>

<template>
  <div class="profile">
    <h2>ユーザープロフィール</h2>

    <!-- reactive オブジェクトのプロパティにはドットでアクセス -->
    <p>名前: {{ user.name }}</p>
    <p>メール: {{ user.email }}</p>

    <!-- ref はテンプレートで自動アンラップ -->
    <p>年齢: {{ age }}歳</p>
    <p>状態: {{ isOnline ? 'オンライン' : 'オフライン' }}</p>

    <!-- 配列の表示 -->
    <p>趣味:</p>
    <ul>
      <li v-for="hobby in user.hobbies" :key="hobby">{{ hobby }}</li>
    </ul>

    <button @click="incrementAge">1歳年を取る</button>
    <button @click="addHobby">趣味を追加</button>
  </div>
</template>
```

**ref と reactive の使い分けガイド:**

| 場面 | 推奨 | 理由 |
|------|------|------|
| 数値、文字列、真偽値 | `ref` | プリミティブ値は reactive で包めない |
| オブジェクト | `ref` または `reactive` | どちらでもOK（チーム規約に従う） |
| 配列 | `ref` または `reactive` | どちらでもOK |
| 再代入が必要な場合 | `ref` | `reactive` は再代入するとリアクティビティが切れる |

</details>

## 演習2-2：computed で算出プロパティ（基本）

### 問題

ショッピングカートの合計金額を計算する `CartSummary.vue` を作成してください。

**要件:**
1. `items` を `ref` で定義（配列）:
   ```js
   [
     { name: 'りんご', price: 200, quantity: 3 },
     { name: 'バナナ', price: 150, quantity: 5 },
     { name: 'みかん', price: 100, quantity: 10 }
   ]
   ```
2. 以下の `computed` を定義する:
   - `totalItems` — 全商品の合計個数
   - `subtotal` — 小計（各商品の price * quantity の合計）
   - `tax` — 消費税（小計の10%、小数点以下切り捨て）
   - `totalPrice` — 合計金額（小計 + 消費税）
3. テンプレートに一覧表と合計を表示する

<details>
<summary>ヒント</summary>

- `computed(() => ...)` で算出プロパティを定義します
- 配列の合計は `reduce` メソッドが便利です: `array.reduce((sum, item) => sum + item.price, 0)`
- `computed` は依存するリアクティブデータが変わると自動的に再計算されます
- `Math.floor()` で切り捨て

</details>

<details>
<summary>解答例</summary>

```vue
<!-- CartSummary.vue -->
<script setup>
import { ref, computed } from 'vue'

// カートの商品データ
const items = ref([
  { name: 'りんご', price: 200, quantity: 3 },
  { name: 'バナナ', price: 150, quantity: 5 },
  { name: 'みかん', price: 100, quantity: 10 }
])

// 合計個数 — items の各要素の quantity を合計する
const totalItems = computed(() => {
  return items.value.reduce((sum, item) => sum + item.quantity, 0)
})

// 小計 — 各商品の (price × quantity) を合計する
const subtotal = computed(() => {
  return items.value.reduce((sum, item) => sum + item.price * item.quantity, 0)
})

// 消費税 — 小計の10%（小数点以下切り捨て）
// computed の中で別の computed を参照できる（依存チェーンが自動追跡される）
const tax = computed(() => {
  return Math.floor(subtotal.value * 0.1)
})

// 合計金額 — 小計 + 消費税
const totalPrice = computed(() => {
  return subtotal.value + tax.value
})
</script>

<template>
  <div class="cart-summary">
    <h2>カート内容</h2>

    <!-- 商品一覧テーブル -->
    <table>
      <thead>
        <tr>
          <th>商品名</th>
          <th>単価</th>
          <th>数量</th>
          <th>小計</th>
        </tr>
      </thead>
      <tbody>
        <tr v-for="item in items" :key="item.name">
          <td>{{ item.name }}</td>
          <td>¥{{ item.price.toLocaleString() }}</td>
          <td>{{ item.quantity }}</td>
          <td>¥{{ (item.price * item.quantity).toLocaleString() }}</td>
        </tr>
      </tbody>
    </table>

    <!-- 合計表示 -->
    <div class="totals">
      <p>合計個数: {{ totalItems }}個</p>
      <p>小計: ¥{{ subtotal.toLocaleString() }}</p>
      <p>消費税（10%）: ¥{{ tax.toLocaleString() }}</p>
      <p class="grand-total">合計: ¥{{ totalPrice.toLocaleString() }}</p>
    </div>
  </div>
</template>

<style scoped>
table {
  width: 100%;
  border-collapse: collapse;
  margin-bottom: 16px;
}

th, td {
  border: 1px solid #ddd;
  padding: 8px;
  text-align: center;
}

.grand-total {
  font-size: 1.3em;
  font-weight: bold;
  color: #e53e3e;
}
</style>
```

**ポイント解説:**
- `computed` はキャッシュされます。依存するデータが変わらない限り、何度参照しても再計算されません。
- 一方、メソッドはテンプレートが再描画されるたびに毎回実行されます。
- `computed` の中から別の `computed` を参照すると、Vue は自動的に依存関係を追跡します。

</details>

## 演習2-3：バグ修正問題 — reactiveの罠（基本）

### 問題

以下のコンポーネントにはリアクティビティに関するバグが2つあります。見つけて修正してください。

```vue
<script setup>
import { reactive, computed } from 'vue'

const state = reactive({
  firstName: '太郎',
  lastName: '山田'
})

const fullName = computed(() => {
  return state.lastName + ' ' + state.firstName
})

function resetUser() {
  state = reactive({
    firstName: 'ゲスト',
    lastName: ''
  })
}

let { firstName, lastName } = state

function updateFirstName(newName) {
  firstName = newName
}
</script>

<template>
  <p>氏名: {{ fullName }}</p>
  <p>名前だけ: {{ firstName }}</p>
  <button @click="resetUser">リセット</button>
  <button @click="updateFirstName('花子')">名前を花子に変更</button>
</template>
```

<details>
<summary>ヒント</summary>

- `reactive` で作ったオブジェクトを**再代入**するとリアクティビティが切れます
- `reactive` オブジェクトを**分割代入**するとリアクティビティが切れます

</details>

<details>
<summary>解答例</summary>

**バグ1:** `resetUser` で `state` に新しい `reactive` を再代入している

```js
// ❌ reactive オブジェクトを再代入するとリアクティビティが切れる
// state 変数が別のオブジェクトを指すようになり、テンプレートとの接続が切れる
state = reactive({ firstName: 'ゲスト', lastName: '' })

// ✅ プロパティを個別に変更する
function resetUser() {
  state.firstName = 'ゲスト'
  state.lastName = ''
}

// ✅ または Object.assign で上書きする
function resetUser() {
  Object.assign(state, { firstName: 'ゲスト', lastName: '' })
}
```

**バグ2:** 分割代入で `firstName`, `lastName` を取り出している

```js
// ❌ 分割代入するとプリミティブ値のコピーが作られ、リアクティビティが切れる
let { firstName, lastName } = state

// ✅ toRefs を使って ref に変換する
import { toRefs } from 'vue'
const { firstName, lastName } = toRefs(state)

// ✅ または常に state.firstName のようにドットアクセスする
```

**修正後の完全なコード:**

```vue
<script setup>
import { reactive, computed, toRefs } from 'vue'

const state = reactive({
  firstName: '太郎',
  lastName: '山田'
})

const fullName = computed(() => {
  return state.lastName + ' ' + state.firstName
})

// ✅ プロパティを個別に変更してリアクティビティを維持
function resetUser() {
  state.firstName = 'ゲスト'
  state.lastName = ''
}

// ✅ toRefs でリアクティブな参照を保つ
const { firstName, lastName } = toRefs(state)

function updateFirstName(newName) {
  // toRefs で得た ref は .value でアクセスし、元の reactive にも反映される
  firstName.value = newName
}
</script>

<template>
  <p>氏名: {{ fullName }}</p>
  <p>名前だけ: {{ firstName }}</p>
  <button @click="resetUser">リセット</button>
  <button @click="updateFirstName('花子')">名前を花子に変更</button>
</template>
```

**reactive の2大注意点:**
1. **再代入禁止** — `state = { ... }` で新しいオブジェクトを代入するとリアクティビティが失われる
2. **分割代入注意** — `let { a } = state` ではリアクティビティが切れる。`toRefs()` を使う

</details>

## 演習2-4：watch と watchEffect（応用）

### 問題

検索キーワードの入力に応じてフィルタリングを行い、検索履歴を記録する `SearchWithHistory.vue` を作成してください。

**要件:**
1. `searchQuery`（検索キーワード）と `searchHistory`（検索履歴の配列）を定義する
2. `watch` を使い、`searchQuery` が変更されたとき、2文字以上なら `searchHistory` に追加する
3. `watchEffect` を使い、現在の検索状態をコンソールに出力する
4. 「履歴クリア」ボタンで `searchHistory` を空にする

<details>
<summary>ヒント</summary>

- `watch(対象, (newVal, oldVal) => { ... })` で特定の値の変化を監視します
- `watchEffect(() => { ... })` は関数内で参照しているリアクティブ変数が変わると自動で再実行されます
- 重複防止には `includes()` で既に履歴にあるかチェックします

</details>

<details>
<summary>解答例</summary>

```vue
<!-- SearchWithHistory.vue -->
<script setup>
import { ref, watch, watchEffect } from 'vue'

const searchQuery = ref('')
const searchHistory = ref([])
const resultCount = ref(0)

// watch: searchQuery の変化を監視
// 第1引数 = 監視対象、第2引数 = コールバック（新しい値と古い値を受け取れる）
watch(searchQuery, (newQuery, oldQuery) => {
  console.log(`検索語が変更: "${oldQuery}" → "${newQuery}"`)

  // 2文字以上で、まだ履歴にない場合のみ追加
  if (newQuery.length >= 2 && !searchHistory.value.includes(newQuery)) {
    searchHistory.value.push(newQuery)
  }
})

// watchEffect: 関数内で参照しているリアクティブ変数を自動で追跡
// searchQuery が変わるたびに自動で再実行される
watchEffect(() => {
  // ダミーの検索結果数をシミュレート
  resultCount.value = searchQuery.value.length > 0
    ? Math.floor(Math.random() * 100)
    : 0

  console.log(`現在の検索状態 — キーワード: "${searchQuery.value}", 結果: ${resultCount.value}件`)
})

// 履歴をクリアする
function clearHistory() {
  searchHistory.value = []
}
</script>

<template>
  <div class="search-container">
    <h2>検索</h2>

    <!-- 検索入力 -->
    <input
      v-model="searchQuery"
      type="text"
      placeholder="キーワードを入力..."
    />

    <p>検索結果: {{ resultCount }}件</p>

    <!-- 検索履歴 -->
    <div class="history">
      <h3>検索履歴（{{ searchHistory.length }}件）</h3>
      <ul v-if="searchHistory.length > 0">
        <li v-for="(query, index) in searchHistory" :key="index">
          {{ query }}
        </li>
      </ul>
      <p v-else>履歴はありません</p>
      <button @click="clearHistory" :disabled="searchHistory.length === 0">
        履歴クリア
      </button>
    </div>
  </div>
</template>
```

**watch と watchEffect の違い:**

| | `watch` | `watchEffect` |
|------|---------|---------------|
| 監視対象 | 明示的に指定 | 自動で追跡 |
| 旧値の取得 | `(newVal, oldVal)` で取得可 | 取得不可 |
| 初回実行 | デフォルトでは実行しない | 即座に実行する |
| 用途 | 特定の値の変化に対するリアクション | 副作用の自動追跡 |

</details>

## 演習2-5：深いオブジェクトの監視（応用）

### 問題

以下のコードで「設定を変更しても watch が発火しない」というバグを修正してください。

```vue
<script setup>
import { ref, watch } from 'vue'

const settings = ref({
  theme: 'light',
  notifications: {
    email: true,
    push: false
  },
  language: 'ja'
})

// この watch が設定変更時に発火しない！
watch(settings, (newSettings) => {
  console.log('設定が変更されました:', newSettings)
  // サーバーに設定を保存する処理を想定
})

function toggleTheme() {
  settings.value.theme = settings.value.theme === 'light' ? 'dark' : 'light'
}

function toggleEmailNotification() {
  settings.value.notifications.email = !settings.value.notifications.email
}
</script>

<template>
  <div>
    <p>テーマ: {{ settings.theme }}</p>
    <p>メール通知: {{ settings.notifications.email ? 'ON' : 'OFF' }}</p>
    <button @click="toggleTheme">テーマ切替</button>
    <button @click="toggleEmailNotification">メール通知切替</button>
  </div>
</template>
```

<details>
<summary>ヒント</summary>

- `ref` でオブジェクトを監視する場合、オブジェクト内部のプロパティの変更はデフォルトでは検知されません
- `watch` の第3引数にオプションを渡せます
- `{ deep: true }` オプションを使うか、ゲッター関数で特定のプロパティを監視します

</details>

<details>
<summary>解答例</summary>

**原因:** `watch(settings, ...)` はデフォルトでは `settings.value` の**参照**が変わったときだけ発火します。オブジェクトのプロパティを変更しても参照自体は変わらないため、コールバックは実行されません。

**修正方法1: `deep: true` を使う（全プロパティを監視）**

```vue
<script setup>
import { ref, watch } from 'vue'

const settings = ref({
  theme: 'light',
  notifications: { email: true, push: false },
  language: 'ja'
})

// ✅ deep: true で内部プロパティの変更も検知する
watch(
  settings,
  (newSettings) => {
    console.log('設定が変更されました:', JSON.stringify(newSettings))
  },
  { deep: true }  // ← これを追加
)

function toggleTheme() {
  settings.value.theme = settings.value.theme === 'light' ? 'dark' : 'light'
}

function toggleEmailNotification() {
  settings.value.notifications.email = !settings.value.notifications.email
}
</script>
```

**修正方法2: ゲッター関数で特定のプロパティだけ監視（パフォーマンスに優れる）**

```js
// テーマの変更だけを監視したい場合
watch(
  () => settings.value.theme,    // ゲッター関数
  (newTheme, oldTheme) => {
    console.log(`テーマ変更: ${oldTheme} → ${newTheme}`)
  }
)

// 複数プロパティを同時に監視したい場合
watch(
  [() => settings.value.theme, () => settings.value.notifications.email],
  ([newTheme, newEmail], [oldTheme, oldEmail]) => {
    console.log('設定変更検知')
  }
)
```

**`deep: true` の注意点:**
- オブジェクトの全プロパティを再帰的に監視するため、大きなオブジェクトではパフォーマンスに影響する可能性があります
- 必要なプロパティだけをゲッター関数で監視する方が効率的です

</details>

## 演習2-6：リアクティビティ総合問題 — 単位変換ツール（チャレンジ）

### 問題

温度の単位変換ツール `TemperatureConverter.vue` を作成してください。

**要件:**
1. `celsius`（摂氏）を `ref` で管理する
2. `fahrenheit`（華氏）を `computed` の **書き込み可能（getter/setter）** 版で定義する
   - 計算式: `F = C × 9/5 + 32` / `C = (F - 32) × 5/9`
3. `kelvin`（絶対温度）を通常の `computed`（読み取り専用）で定義する
   - 計算式: `K = C + 273.15`
4. 摂氏の入力欄と華氏の入力欄を用意し、どちらを変更しても連動すること
5. `watch` で温度が0度以下になったら「凍結注意！」と警告を表示する

<details>
<summary>ヒント</summary>

- 書き込み可能な `computed` は `get` と `set` を持つオブジェクトを渡して定義します:
  ```js
  const myComputed = computed({
    get() { return ... },
    set(newValue) { ... }
  })
  ```
- 華氏の `set` 内で `celsius.value` を逆算して更新すれば、自動的に全ての computed が連動します

</details>

<details>
<summary>解答例</summary>

```vue
<!-- TemperatureConverter.vue -->
<script setup>
import { ref, computed, watch } from 'vue'

// 摂氏を基準値として管理
const celsius = ref(20)

// 凍結警告フラグ
const freezeWarning = ref(false)

// 書き込み可能な computed（get + set）
// 華氏の入力欄に v-model を使えるようになる
const fahrenheit = computed({
  // 摂氏 → 華氏 の変換（読み取り時）
  get() {
    return parseFloat((celsius.value * 9 / 5 + 32).toFixed(2))
  },
  // 華氏 → 摂氏 の逆変換（書き込み時）
  set(newFahrenheit) {
    celsius.value = parseFloat(((newFahrenheit - 32) * 5 / 9).toFixed(2))
  }
})

// 読み取り専用の computed（ケルビン）
const kelvin = computed(() => {
  return parseFloat((celsius.value + 273.15).toFixed(2))
})

// 摂氏が0度以下になったら警告
watch(celsius, (newCelsius) => {
  freezeWarning.value = newCelsius <= 0
})

// 代表的な温度のプリセット
function setPreset(temp) {
  celsius.value = temp
}
</script>

<template>
  <div class="converter">
    <h2>温度変換ツール</h2>

    <!-- 凍結警告 -->
    <div v-if="freezeWarning" class="warning">
      凍結注意！ 水が凍る温度です。
    </div>

    <div class="inputs">
      <!-- 摂氏入力 -->
      <label>
        摂氏 (°C):
        <input v-model.number="celsius" type="number" step="0.1" />
      </label>

      <!-- 華氏入力 — computed の setter が呼ばれ、celsius が逆算される -->
      <label>
        華氏 (°F):
        <input v-model.number="fahrenheit" type="number" step="0.1" />
      </label>

      <!-- ケルビン表示（読み取り専用） -->
      <label>
        ケルビン (K):
        <input :value="kelvin" type="number" readonly />
      </label>
    </div>

    <!-- プリセットボタン -->
    <div class="presets">
      <button @click="setPreset(0)">氷点 (0°C)</button>
      <button @click="setPreset(36.5)">体温 (36.5°C)</button>
      <button @click="setPreset(100)">沸点 (100°C)</button>
    </div>
  </div>
</template>

<style scoped>
.converter {
```

## 演習2-7：shallowRef とパフォーマンス（チャレンジ）

### 問題

大量データ（1000件の商品リスト）を扱うコンポーネントで、`ref` と `shallowRef` の違いを体験してください。

**要件:**
1. `shallowRef` で1000件の商品データを管理する
2. 「全商品の価格を10%上げる」ボタンを実装する
3. `shallowRef` の値を更新する正しい方法と、更新されない間違った方法を両方示す

<details>
<summary>ヒント</summary>

- `shallowRef` は `.value` の参照が変わったときだけリアクティブに反応します
- 配列の中身を変更しても `.value` の参照は変わらないため、更新が検知されません
- `triggerRef()` で手動で更新を発火させるか、新しい配列を代入する必要があります

</details>

<details>
<summary>解答例</summary>

```vue
<!-- LargeList.vue -->
<script setup>
import { shallowRef, triggerRef } from 'vue'

// shallowRef: .value の参照変更のみを監視（内部プロパティの変更は無視）
// 大量データでのパフォーマンス改善に有効
const products = shallowRef(
  Array.from({ length: 1000 }, (_, i) => ({
    id: i + 1,
    name: `商品${i + 1}`,
    price: Math.floor(Math.random() * 10000) + 100
  }))
)

// ❌ 間違い: 配列の中身を直接変更しても画面は更新されない
function raisePrice_WRONG() {
  products.value.forEach(p => {
    p.price = Math.floor(p.price * 1.1)
  })
  // shallowRef は .value の参照が変わらないと反応しない
  // → 画面は更新されない！
}

// ✅ 方法1: 新しい配列を作って .value に代入する（参照が変わる）
function raisePrice_Method1() {
  products.value = products.value.map(p => ({
    ...p,
    price: Math.floor(p.price * 1.1)
  }))
}

// ✅ 方法2: 中身を変更した後、triggerRef で手動更新を発火
function raisePrice_Method2() {
  products.value.forEach(p => {
    p.price = Math.floor(p.price * 1.1)
  })
  triggerRef(products)  // 明示的に「値が変わった」と Vue に伝える
}

// 表示用: 先頭10件だけ表示
const displayedProducts = () => products.value.slice(0, 10)
</script>

<template>
  <div>
    <h2>商品リスト（{{ products.length }}件中 先頭10件）</h2>

    <div>
      <button @click="raisePrice_Method1">全商品10%値上げ（新配列作成）</button>
      <button @click="raisePrice_Method2">全商品10%値上げ（triggerRef）</button>
    </div>

    <table>
      <thead>
        <tr><th>ID</th><th>商品名</th><th>価格</th></tr>
      </thead>
      <tbody>
        <tr v-for="product in displayedProducts()" :key="product.id">
          <td>{{ product.id }}</td>
          <td>{{ product.name }}</td>
          <td>¥{{ product.price.toLocaleString() }}</td>
        </tr>
      </tbody>
    </table>
  </div>
</template>
```

**ref vs shallowRef の違い:**

| | `ref` | `shallowRef` |
|------|-------|-------------|
| 反応する変更 | `.value` とその内部プロパティ全て | `.value` の参照変更のみ |
| パフォーマンス | 深いオブジェクトでは重くなる可能性 | 軽い（浅い監視のみ） |
| 使いどころ | 一般的な用途 | 大量データ、頻繁な更新 |

</details>
