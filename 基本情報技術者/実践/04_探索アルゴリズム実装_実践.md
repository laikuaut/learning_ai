## 課題パート1：手計算問題

### 基本問題

- mid の要素と探索値の比較結果
- 次の操作（左半分 / 右半分 / 発見）

**Q2.** 要素数 n のデータに対して、以下の平均比較回数を答えてください。
- 線形探索の平均比較回数: ?
- 二分探索の最大比較回数: ?

**Q3.** 要素数が 1,000,000 のとき、二分探索の最大比較回数は何回ですか？（log₂(1000000) を計算）

### 応用問題

**Q4.** 二分探索が使えるための前提条件は何ですか？その理由を説明してください。

**Q5.** 番兵（Sentinel）を使った線形探索とは何か、通常の線形探索と比べた利点を説明してください。

## ステップガイド

<details>
<summary>ステップ1：線形探索を実装する</summary>

先頭から順番に見ていくシンプルなアルゴリズムです。

```python
def linear_search(data, target):
    """線形探索。(インデックス, 比較回数) を返す"""
    count = 0
    for i in range(len(data)):
        count += 1
        if data[i] == target:
            return i, count
    return -1, count  # 見つからなかった場合
```

</details>

<details>
<summary>ステップ2：二分探索を実装する</summary>

整列済みデータの中央値と比較して、探索範囲を半分に絞ります。

```python
def binary_search(data, target):
    """二分探索。(インデックス, 比較回数) を返す"""
    low = 0
    high = len(data) - 1
    count = 0

    while low <= high:
        mid = (low + high) // 2
        count += 1
        if data[mid] == target:
            return mid, count
        elif data[mid] < target:
            low = mid + 1
        else:
            high = mid - 1

    return -1, count
```

## 解答例

<details>
<summary>手計算問題の解答</summary>

**Q1.** 配列: `[2, 5, 8, 12, 16, 23, 38, 56]`（インデックス0〜7）、探索値: 23

| ステップ | low | high | mid | A[mid] | 比較結果 | 操作 |
|---------|-----|------|-----|--------|---------|------|
| 1 | 0 | 7 | 3 | 12 | 23 > 12 | 右半分へ（low = 4） |
| 2 | 4 | 7 | 5 | 23 | 23 == 23 | **発見！** |

→ **2回の比較**で発見

**Q2.**
- 線形探索の平均比較回数: **n/2**（先頭から末尾まで均等に分布する場合）
- 二分探索の最大比較回数: **⌈log₂(n)⌉ + 1** ≒ **log₂(n)**

**Q3.** `log₂(1,000,000) = log₂(10⁶) ≒ 6 × 3.32 ≒ **20回**`
（100万件でもたった20回の比較で探索完了）

**Q4.** 前提条件：**データが整列（ソート）されていること。** 中央値との大小比較で探索範囲を絞るため、データが整列されていないと正しく動作しません。

**Q5.** 番兵（Sentinel）法：配列の末尾に探索値を追加してから探索する方法。ループ内で「配列の終端チェック」が不要になるため、比較が1回分減り、微小ながら高速化できます。

```python
# 探索アルゴリズム比較
# 学べる内容：線形探索、二分探索、計算量の違い
# 実行方法：python 04_search_algorithms.py

import math

def linear_search(data, target):
    """線形探索：先頭から順番に探す"""
    count = 0
    for i in range(len(data)):
        count += 1
        if data[i] == target:
            return i, count
    return -1, count


def binary_search(data, target):
    """二分探索：中央と比較して範囲を半分にする"""
    low = 0
    high = len(data) - 1
    count = 0

    while low <= high:
        mid = (low + high) // 2
        count += 1

        if data[mid] == target:
            return mid, count
        elif data[mid] < target:
            low = mid + 1
        else:
            high = mid - 1

    return -1, count


# --- メイン ---
print("===== 探索アルゴリズム比較 =====\n")

n = int(input("データ件数を入力 (例: 1000): "))
data = list(range(1, n + 1))  # 1 から n までの整列済みデータ

target = int(input(f"探索する値 (1〜{n}): "))

# 線形探索
lin_idx, lin_count = linear_search(data, target)
print(f"\n--- 線形探索 (Linear Search) ---")
if lin_idx != -1:
    print(f"  結果: 見つかりました（インデックス: {lin_idx}）")
else:
    print(f"  結果: 見つかりませんでした")
print(f"  比較回数: {lin_count}")

# 二分探索
bin_idx, bin_count = binary_search(data, target)
print(f"\n--- 二分探索 (Binary Search) ---")
if bin_idx != -1:
    print(f"  結果: 見つかりました（インデックス: {bin_idx}）")
else:
    print(f"  結果: 見つかりませんでした")
print(f"  比較回数: {bin_count}")

# 比較
print(f"\n--- 比較 ---")
print(f"  線形探索: {lin_count}回")
print(f"  二分探索: {bin_count}回")
if bin_count > 0:
    ratio = lin_count / bin_count
    print(f"  二分探索は線形探索の {ratio:.1f} 倍高速でした！")
print(f"  理論値: log₂({n}) ≒ {math.log2(n):.1f}")
```

<summary>解答例（改良版 ─ データ件数別のベンチマーク）</summary>

```python
# 探索アルゴリズム比較（改良版）
# データ件数を段階的に変えて、計算量の増加傾向を可視化する

import math
import random


def linear_search(data, target):
    """線形探索"""
    count = 0
    for i in range(len(data)):
        count += 1
        if data[i] == target:
            return i, count
    return -1, count


def binary_search(data, target):
    """二分探索"""
    low, high, count = 0, len(data) - 1, 0
    while low <= high:
        mid = (low + high) // 2
        count += 1
        if data[mid] == target:
            return mid, count
        elif data[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1, count


def sentinel_search(data, target):
    """番兵法による線形探索"""
    n = len(data)
    work = data + [target]  # 番兵を追加
    count = 0
    i = 0
    while work[i] != target:
        count += 1
        i += 1
    count += 1  # 最後の比較もカウント
    if i < n:
        return i, count
    return -1, count


def benchmark(sizes):
    """データ件数別のベンチマーク"""
    print("\n===== ベンチマーク結果 =====")
    print(f"{'件数':>10} | {'線形探索':>10} | {'番兵法':>10} | {'二分探索':>10} | {'log₂(n)':>10}")
    print("-" * 62)

    for n in sizes:
        data = list(range(1, n + 1))
        # 平均的なケース（中央付近をランダムに探索）
        target = random.randint(1, n)

        _, lin_count = linear_search(data, target)
        _, sen_count = sentinel_search(data, target)
        _, bin_count = binary_search(data, target)
        theory = math.ceil(math.log2(n))

        print(f"{n:>10,} | {lin_count:>10,} | {sen_count:>10,} | {bin_count:>10,} | {theory:>10}")


def interactive_demo():
    """インタラクティブなデモ"""
    print("===== 探索アルゴリズム比較 =====\n")

    n = int(input("データ件数 (例: 100): "))
    data = list(range(1, n + 1))
    target = int(input(f"探索する値 (1〜{n}): "))

    algorithms = [
        ("線形探索", linear_search),
        ("番兵法", sentinel_search),
        ("二分探索", binary_search),
    ]

    for name, func in algorithms:
        idx, count = func(data, target)
        status = f"インデックス {idx}" if idx != -1 else "見つからず"
        print(f"\n  {name}: {status}（比較 {count}回）")

    print(f"\n  理論値 log₂({n}) ≒ {math.log2(n):.1f}")


def main():
    print("===== 探索アルゴリズム実験 =====\n")
    print("  1. インタラクティブデモ")
    print("  2. ベンチマーク（件数別比較）")

    choice = input("\n選択 (1-2): ").strip()

    if choice == "1":
        interactive_demo()
    elif choice == "2":
        sizes = [10, 100, 1_000, 10_000, 100_000, 1_000_000]
        benchmark(sizes)
    else:
        print("無効な選択です。")


if __name__ == "__main__":
    main()
```

**初心者向けとの違い:**
- 番兵法（Sentinel Search）も実装 → 試験に出題される手法をカバー
- ベンチマークモードで件数別の比較回数を一覧表示 → O(n) と O(log n) の差を実感
- 関数を引数として渡す設計 → アルゴリズム追加が容易
- `random` で毎回異なるターゲットを探索 → 統計的な理解を促進
