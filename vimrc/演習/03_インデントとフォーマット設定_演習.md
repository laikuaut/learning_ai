# 第3章 演習：インデントとフォーマット設定

## 演習の目標

- `tabstop` / `shiftwidth` / `softtabstop` / `expandtab` の役割と関係を理解できる
- `autoindent` / `smartindent` の違いを説明できる
- 言語ごとにインデント設定を切り替える方法を習得する
- `formatoptions` と `textwidth` を使ったテキスト整形ができる

---

## 基本問題

### 問題1：タブとスペースの基本設定

以下の要件を満たす `.vimrc` の記述を作成してください。

**要件：**
- タブ文字の表示幅を 4 に設定する
- インデント操作（`>>` や `<<`）の幅を 4 に設定する
- Tab キーを押した時にスペースが挿入されるようにする
- Tab キーで挿入されるスペースの数を 4 にする

**期待される .vimrc の記述：**
```vim
set tabstop=4
set shiftwidth=4
set expandtab
set softtabstop=4
```

<details>
<summary>ヒント</summary>

- `tabstop` はファイル内に既にあるタブ文字（`\t`）の表示幅です
- `shiftwidth` は `>>` / `<<` / `cindent` 等で使われるインデント幅です
- `expandtab` はタブ文字の代わりにスペースを挿入します
- `softtabstop` は Tab キーを押した時に挿入される空白の幅です

</details>

<details>
<summary>解答例</summary>

```vim
" タブ文字の表示幅を4に設定
" ファイル内のタブ文字（\t）が4文字分の幅で表示されます
set tabstop=4

" インデント操作の幅を4に設定
" >> や << でのインデント増減が4文字分になります
set shiftwidth=4

" Tab キーでスペースを挿入する
" タブ文字（\t）の代わりにスペースが入力されます
set expandtab

" Tab キーで挿入されるスペース数を4に設定
" expandtab と組み合わせると、Tab キーで4つのスペースが入力されます
" BS キーで4つのスペースをまとめて削除することもできます
set softtabstop=4

" 補足：4つのオプションの関係
"   tabstop     → 既存のタブ文字の「見た目」の幅
"   shiftwidth  → >> や << 操作での「インデント」幅
"   softtabstop → Tab キーを押した時の「入力」幅
"   expandtab   → タブ文字の代わりにスペースを入力するか
```

</details>

### 問題2：自動インデントの設定

以下の3つのインデントオプションの違いを説明し、それぞれを有効にする `.vimrc` の記述を答えてください。

1. `autoindent`
2. `smartindent`
3. `cindent`

**期待される説明：**
```
autoindent  - 新しい行で前の行のインデントをそのまま引き継ぐ
smartindent - autoindent + { の後で自動インデント増加、} で減少
cindent     - C言語系の構文を認識した高精度なインデント
```

<details>
<summary>ヒント</summary>

- `autoindent` は最もシンプルで、前の行のインデントレベルを維持するだけです
- `smartindent` は `{` と `}` を認識して自動的にインデントを増減します
- `cindent` は C 言語の構文を解析し、`if`, `for`, `switch` 等に対応した精密なインデントを行います

</details>

<details>
<summary>解答例</summary>

```vim
" 1. autoindent：前の行のインデントを引き継ぐ
" 最もシンプルな自動インデントです
" どの言語でも無難に動作します
set autoindent

" 2. smartindent：中括弧を認識したインデント
" { の後の改行でインデントが1段増え、
" } のある行でインデントが1段減ります
" C系言語やJavaScript等に適しています
set smartindent

" 3. cindent：C言語系の高精度インデント
" if, for, switch, case 等の構文を認識します
" C/C++ を書く場合に最適です
set cindent

" 実務での推奨設定：
" - 一般的なプログラミング：autoindent + smartindent
" - C/C++ 開発：autoindent + cindent
" - それ以外：filetype ごとの設定に任せる
"
" 最も汎用的な設定：
set autoindent
set smartindent
"
" 補足：filetype indent on を有効にすると、
" 各言語専用のインデントプラグインが使われ、
" smartindent や cindent より正確に動作します
filetype indent on
```

</details>

### 問題3：retab コマンドの使い方

以下の操作を行うコマンドを答えてください。

1. 現在のファイル内のタブ文字をすべてスペースに変換する
2. 現在のファイル内のスペースによるインデントをタブ文字に変換する
3. 選択範囲だけタブをスペースに変換する

**期待される操作：**
```
1. :set expandtab → :retab
2. :set noexpandtab → :retab!
3. ビジュアルモードで範囲選択 → :'<,'>retab
```

<details>
<summary>ヒント</summary>

- `retab` は `expandtab` の設定に基づいてタブとスペースを変換します
- `expandtab` が有効な場合、`retab` でタブ→スペースに変換されます
- `retab!` は空白の連続もタブに変換します（`noexpandtab` 時）
- ビジュアルモードで範囲選択後、`:` を押すと `'<,'>` が自動入力されます

</details>

<details>
<summary>解答例</summary>

```vim
" 1. タブ文字をスペースに変換
" まず expandtab を有効にしてから retab を実行します
:set expandtab
:retab

" 2. スペースのインデントをタブ文字に変換
" noexpandtab にしてから retab! を実行します
" 「!」を付けると空白の連続もタブに変換されます
:set noexpandtab
:retab!

" 3. 選択範囲だけ変換
" ビジュアルモード（V）で範囲選択した後に実行します
:'<,'>retab

" 注意：
" - retab は tabstop の値に基づいて変換します
" - 変換前に :set tabstop=4 等で正しい値を設定してください
" - 文字列リテラル内のタブも変換されてしまうため、
"   コードの意味が変わらないか注意が必要です
```

</details>

## 応用問題

### 問題4：言語別インデント設定

以下の言語ごとのインデントルールを `.vimrc` に設定してください。

| 言語 | タブ幅 | インデント | タブ/スペース |
|------|--------|-----------|-------------|
| Python | 4 | 4 | スペース |
| JavaScript | 2 | 2 | スペース |
| Go | 4 | 4 | タブ |
| Makefile | 8 | 8 | タブ |

**期待される .vimrc の記述：**
```vim
augroup FileTypeIndent
  autocmd!
  autocmd FileType python setlocal tabstop=4 shiftwidth=4 softtabstop=4 expandtab
  autocmd FileType javascript setlocal tabstop=2 shiftwidth=2 softtabstop=2 expandtab
  autocmd FileType go setlocal tabstop=4 shiftwidth=4 noexpandtab
  autocmd FileType make setlocal tabstop=8 shiftwidth=8 noexpandtab
augroup END
```

<details>
<summary>ヒント</summary>

- `autocmd FileType` でファイルタイプごとに設定を切り替えます
- `setlocal` を使って、該当バッファだけに設定を適用します（他のバッファには影響しません）
- Go や Makefile ではタブ文字を使うのがルールです（`noexpandtab`）
- `augroup` で囲むことで、`source` 時の重複を防ぎます

</details>

<details>
<summary>解答例</summary>

```vim
" 言語別インデント設定
" augroup で囲み、autocmd! でクリアして重複を防ぐ
augroup FileTypeIndent
  autocmd!

  " Python: PEP 8 準拠（スペース4つ）
  autocmd FileType python setlocal
    \ tabstop=4
    \ shiftwidth=4
    \ softtabstop=4
    \ expandtab

  " JavaScript: 一般的な規約（スペース2つ）
  autocmd FileType javascript setlocal
    \ tabstop=2
    \ shiftwidth=2
    \ softtabstop=2
    \ expandtab

  " Go: 公式スタイル（タブ文字）
  " gofmt がタブを使うため、必ず noexpandtab にします
  autocmd FileType go setlocal
    \ tabstop=4
    \ shiftwidth=4
    \ noexpandtab

  " Makefile: タブ文字必須（スペースだとエラーになる）
  autocmd FileType make setlocal
    \ tabstop=8
    \ shiftwidth=8
    \ noexpandtab
augroup END

" 補足：行末の \ は行の継続を意味します
" 長い設定を見やすく複数行に分けて書けます
"
" filetype plugin indent on を忘れずに設定してください
" これがないとファイルタイプの自動検出が動きません
filetype plugin indent on
```

</details>

### 問題5：formatoptions と textwidth の設定

以下の要件を満たす `.vimrc` の記述を作成してください。

**要件：**
1. テキストの折り返し幅を 80 文字に設定する
2. 80 文字目にカラーカラム（ガイドライン）を表示する
3. コメントの自動折り返しを有効にする
4. コメント行で Enter を押した時、次の行にもコメント記号を自動挿入する
5. ノーマルモードでの `o` / `O` ではコメント記号を自動挿入しない

**期待される .vimrc の記述：**
```vim
set textwidth=80
set colorcolumn=80
set formatoptions+=c
set formatoptions+=r
set formatoptions-=o
```

<details>
<summary>ヒント</summary>

- `textwidth` は自動改行が発生する文字数です
- `formatoptions` の各フラグの意味：
  - `c`：コメントを `textwidth` で自動折り返し
  - `r`：挿入モードで Enter 時にコメントリーダーを自動挿入
  - `o`：ノーマルモードで `o`/`O` 時にコメントリーダーを自動挿入
- `+=` で追加、`-=` で除去ができます

</details>

<details>
<summary>解答例</summary>

```vim
" テキストの折り返し幅を 80 文字に設定
" この値を超えると自動的に改行が挿入されます
set textwidth=80

" 80 文字目にカラーカラム（縦線ガイド）を表示
" 視覚的に行の長さを確認できます
set colorcolumn=80

" formatoptions の設定
" c: コメントを textwidth で自動折り返しする
set formatoptions+=c

" r: 挿入モードで Enter を押した時、コメントリーダー（// や # など）を自動挿入
set formatoptions+=r

" o: ノーマルモードの o / O でコメントリーダーを自動挿入しない
" 意図せずコメントが続くのを防ぐため、無効にするのが一般的です
set formatoptions-=o

" 補足：formatoptions のよく使うフラグ一覧
"   t - テキストを textwidth で自動折り返し
"   c - コメントを textwidth で自動折り返し
"   r - Insert で Enter 時にコメントリーダーを継続
"   o - Normal で o/O 時にコメントリーダーを継続
"   q - gq でコメントの整形を許可
"   j - コメント行を J で結合した時にリーダーを除去
"   n - 番号付きリストを認識して整形
"
" 推奨設定（モダンな構成）:
" set formatoptions=tcqrj
"
" 注意：filetype プラグインが formatoptions を上書きすることがあります
" その場合は autocmd で再設定してください：
" autocmd FileType * setlocal formatoptions=tcqrj
```

</details>

### 問題6：shiftround と インデント操作

以下の設問に答えてください。

**問1：** `shiftround` オプションの効果を説明してください。

**問2：** 以下の設定の場合、3文字分のインデントがある行で `>>` を実行すると何文字のインデントになるか、`shiftround` が有効/無効の場合それぞれ答えてください。

```vim
set shiftwidth=4
```

**期待される回答：**
```
問1: >> や << でインデントする時、shiftwidth の倍数に揃える

問2:
  shiftround 無効: 3 + 4 = 7文字のインデント
  shiftround 有効: 4文字のインデント（shiftwidth=4 の倍数に切り上げ）
```

<details>
<summary>ヒント</summary>

- `shiftround` がない場合、`>>` は現在のインデントに `shiftwidth` を加算します
- `shiftround` がある場合、`>>` は次の `shiftwidth` の倍数に丸めます
- 例：`shiftwidth=4` で 3 文字インデントの行を `>>` すると、次の倍数 = 4 に揃います

</details>

<details>
<summary>解答例</summary>

```vim
" shiftround を有効にする
" >> や << の操作で、shiftwidth の倍数にインデントを揃えます
set shiftround

" 解説：
"
" shiftwidth=4 で 3文字インデントの行に >> を実行した場合
"
" shiftround 無効:
"   3（現在のインデント）+ 4（shiftwidth）= 7文字
"   ___func()  →  _______func()  （_はスペース）
"
" shiftround 有効:
"   次の shiftwidth の倍数 = 4 に切り上げ
"   ___func()  →  ____func()
"   インデントが4の倍数（4文字）に揃います
"
" shiftround を有効にすることで、
" インデントが常に shiftwidth の倍数に統一されます
" チーム開発でインデントの乱れを防ぐのに効果的です

" 推奨設定：
set shiftwidth=4
set shiftround
```

</details>

## チャレンジ問題

### 問題7：総合的なインデント・フォーマット設定

以下の要件をすべて満たす `.vimrc` の設定を作成してください。

**要件：**
1. デフォルトのインデント設定：スペース4つ
2. 言語別の設定：
   - HTML/CSS/JavaScript/TypeScript/YAML：スペース2つ
   - Python：スペース4つ、textwidth=79（PEP 8 準拠）
   - Go：タブ、幅4
   - Makefile：タブ、幅8
3. `shiftround` 有効
4. `autoindent` と `smartindent` 有効
5. 80 文字目にカラーカラム表示（Python は 79 文字目）
6. formatoptions：コメント自動折り返し、Enter でコメント継続、`o` でコメント継続しない、`J` でコメントリーダー除去
7. 新しいファイルを開いた時、そのファイルの既存のインデント（タブ/スペース）を自動検出する簡易関数を作成

**期待される .vimrc の構成：**
```vim
" === デフォルト設定 ===
...

" === 言語別設定 ===
augroup FileTypeIndent
  ...
augroup END

" === フォーマット ===
...

" === インデント自動検出関数 ===
function! DetectIndent()
  ...
endfunction
```

<details>
<summary>ヒント</summary>

- 簡易的なインデント検出は、ファイルの先頭数十行を走査してタブかスペースかを判定する方法が一般的です
- `getline()` でバッファの行を取得できます
- `match()` で正規表現にマッチするか判定できます
- `BufReadPost` イベントでファイルを開いた後に関数を呼び出せます

</details>

<details>
<summary>解答例</summary>

```vim
" =============================================================================
" インデント・フォーマット設定
" =============================================================================

" === デフォルト設定 ===
" スペース4つをデフォルトとする
set tabstop=4
set shiftwidth=4
set softtabstop=4
set expandtab

" 自動インデント
set autoindent
set smartindent

" インデントを shiftwidth の倍数に揃える
set shiftround

" ファイルタイプに応じたインデントプラグインを有効にする
filetype plugin indent on

" === 言語別設定 ===
augroup FileTypeIndent
  autocmd!

  " HTML/CSS/JS/TS/YAML: スペース2つ
  autocmd FileType html,css,javascript,typescript,yaml setlocal
    \ tabstop=2 shiftwidth=2 softtabstop=2 expandtab

  " Python: PEP 8 準拠（スペース4つ、79文字制限）
  autocmd FileType python setlocal
    \ tabstop=4 shiftwidth=4 softtabstop=4 expandtab
    \ textwidth=79 colorcolumn=79

  " Go: 公式スタイル（タブ文字）
  autocmd FileType go setlocal
    \ tabstop=4 shiftwidth=4 noexpandtab

  " Makefile: タブ文字必須
  autocmd FileType make setlocal
    \ tabstop=8 shiftwidth=8 noexpandtab
augroup END

" === フォーマット設定 ===
" 80文字目にカラーカラム（Python は autocmd で79に上書き）
set colorcolumn=80

" formatoptions の設定
" c: コメントを textwidth で折り返し
" r: Enter でコメントリーダーを継続
" q: gq でコメントの整形を許可
" j: J でコメント行を結合した時にリーダーを除去
set formatoptions=crqj

" o フラグを除去（o/O でコメント継続しない）
" ※ filetype プラグインが上書きする場合があるため autocmd でも設定
augroup FormatOptions
  autocmd!
  autocmd FileType * setlocal formatoptions-=o
augroup END

" === インデント自動検出関数 ===
" ファイルの先頭30行を調べ、タブかスペースかを自動判定する
function! DetectIndent() abort
  let l:tab_count = 0
  let l:space_count = 0
  " 先頭30行（またはファイル全体が30行未満ならその行数）を走査
  let l:max_lines = min([line('$'), 30])

  for l:i in range(1, l:max_lines)
    let l:line = getline(l:i)
    " 行頭がタブで始まる場合
    if match(l:line, '^\t') >= 0
      let l:tab_count += 1
    " 行頭がスペース2つ以上で始まる場合
    elseif match(l:line, '^  \+\S') >= 0
      let l:space_count += 1
    endif
  endfor

  " タブの方が多い場合は noexpandtab に切り替え
  if l:tab_count > l:space_count && l:tab_count > 2
    setlocal noexpandtab
  endif
endfunction

" ファイルを開いた後にインデント検出を実行
augroup DetectIndent
  autocmd!
  autocmd BufReadPost * call DetectIndent()
augroup END
```

</details>
