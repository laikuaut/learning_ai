# 実践課題03：ファイル操作とコード編集 ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第1章（環境構築）、第2章（基本操作）、第3章（ファイル操作とコード編集）
> **課題の種類**: ミニプロジェクト
> **学習目標**: Claude Codeのツール体系（Read, Glob, Grep, Write, Edit）を理解し、ファイルの検索・読み込み・作成・編集を自然言語で操作できるようになる

---

## 完成イメージ

以下のプロジェクト構造をClaude Codeの指示だけで作成し、各種ファイル操作を体験します。

```
task03-webapp/
├── README.md
├── src/
│   ├── index.html
│   ├── style.css
│   └── app.js
├── tests/
│   └── app.test.js
└── .gitignore
```

操作の流れ:
```
> このディレクトリに簡単なWebアプリのプロジェクト構造を作成してください
（Claude Codeがフォルダとファイルを作成）

> src/app.jsに「TODOリスト」機能を実装してください
（Claude Codeがコードを生成）

> app.jsの中でaddTodo関数を探して表示してください
（Claude CodeがGrep/Readで検索して表示）

> addTodo関数にバリデーション処理を追加してください
（Claude CodeがEditで部分編集）
```

---

## 課題の要件

1. Claude Codeにプロジェクト構造の作成を依頼する（Write/Bash）
2. 作成されたファイルの内容確認を依頼する（Read）
3. 特定のパターンでファイルを検索する（Glob）
4. コード内の関数や変数をテキスト検索する（Grep）
5. 既存ファイルの部分的な編集を依頼する（Edit）
6. 新規ファイルの作成を依頼する（Write）
7. 各操作でClaude Codeがどのツールを使っているか意識して観察する

Claude Codeがどのツールを使っているかは、実行時のログに表示されます。以下のパターンを観察しましょう。

```
# 予想されるツール使用パターン
Read   → ファイルの内容を確認するとき
Glob   → ファイル名でファイルを探すとき
Grep   → ファイルの中身をテキスト検索するとき

Edit   → 既存ファイルの一部を変更するとき
```

## 解答例

<details>
<summary>解答例（初心者向け）── 1つずつ確認しながら進める</summary>

```bash
# 準備
mkdir -p ~/claude-code-practice/task03
cd ~/claude-code-practice/task03
claude
```

対話モードで以下を順番に入力します:

```
# 1. プロジェクト作成
> task03-webappというディレクトリに、TODOリストアプリのプロジェクトを作ってください。
> src/index.html、src/style.css、src/app.js、tests/app.test.js、README.md、.gitignoreを含めてください。

# 2. 確認
> 作成されたファイルの一覧を見せてください

# 3. 内容確認
> src/app.jsの内容を見せてください

# 4. 検索
> プロジェクト内でfunctionという文字を含むファイルを探してください

# 5. 編集
> src/app.jsのaddTodo関数に、入力が空の場合にアラートを出す処理を追加してください

# 6. 結果確認
> 修正後のsrc/app.jsを見せてください
```

</details>

<details>
<summary>解答例（改良版）── 効率的な一括指示</summary>

改良版では、複数の要件をまとめて伝えることで、Claude Codeの作業回数を減らします。

```
# まとめて指示する例
> task03-webappディレクトリにTODOリストアプリを作成してください。
> 要件：
> - src/index.html: HTML5テンプレート、TODOリストのUI
> - src/style.css: モダンなデザイン、完了済みに取り消し線
> - src/app.js: 追加・削除・完了トグル機能、空入力バリデーション付き
> - tests/app.test.js: addTodoとdeleteTodoのテストケース各3つ
> - README.md: プロジェクト概要と使い方
> - .gitignore: node_modules, .DS_Store, *.log を除外
>
> app.jsの各関数にはJSDocコメントを付けてください。
```

**初心者向けとの違い:**
- 初心者向けは1ステップずつ確認しながら進めるため安心感があります
- 改良版は要件を最初にまとめて伝えるため、Claude Codeとの往復回数が減り、トークン消費を抑えられます
- 実務では、要件が明確な場合は改良版のようにまとめて伝える方が効率的です

</details>
