## 基本問題

### 問題1: シバンとスクリプトの実行
以下の質問に答えてください。

1. シバン（shebang）とは何ですか？どのように書きますか？
2. スクリプトファイルを実行可能にするコマンドは何ですか？
3. 実行権限なしでスクリプトを実行する方法は何ですか？

<details>
<summary>解答</summary>

```bash
# 1. シバンは、スクリプトをどのインタプリタで実行するかを指定する1行目の記述
#!/bin/bash          # bash で実行
#!/usr/bin/env bash  # 環境に依存しない書き方（推奨）

# 2. 実行権限を付与
$ chmod +x script.sh

# 3. 実行権限なしで実行（bash コマンドで直接実行）
$ bash script.sh
```

</details>

### 問題2: 変数の基本
以下のスクリプトの出力を予測してください。

```bash
#!/bin/bash
name="Linux"
version=22
echo "OS: $name"
echo "Version: $version"
echo 'name is $name'
echo "${name}_server"
```

<details>
<summary>解答</summary>

```
OS: Linux
Version: 22
name is $name
Linux_server
```

解説：
- ダブルクォート `"..."` 内では変数が展開される
- シングルクォート `'...'` 内では変数は展開されない（リテラル）
- `${name}` で変数名の境界を明確にできる

</details>

### 問題4: 条件分岐
ファイルが存在するかチェックし、存在すればファイルサイズを表示、存在しなければエラーメッセージを表示するスクリプトを書いてください。ファイルパスは第1引数で受け取ります。

期待される出力例（ファイルが存在する場合）：
```
/etc/hosts は存在します
サイズ: 221 バイト
```

期待される出力例（ファイルが存在しない場合）：
```
/tmp/nofile は存在しません
```

<details>
<summary>ヒント</summary>

`-f` でファイルの存在チェック、`wc -c < ファイル` でバイト数を取得できます。

</details>

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# check_file.sh - ファイルの存在確認

# 引数チェック
if [ $# -eq 0 ]; then
    echo "使い方: $0 <ファイルパス>"
    exit 1
fi

FILE="$1"

if [ -f "$FILE" ]; then
    SIZE=$(wc -c < "$FILE")
    echo "$FILE は存在します"
    echo "サイズ: $SIZE バイト"
else
    echo "$FILE は存在しません"
fi
```

</details>

### 問題5: for ループ
1から10までの数字を表示し、偶数には「(偶数)」と付けるスクリプトを書いてください。

期待される出力：
```
1
2 (偶数)
3
4 (偶数)
5
6 (偶数)
7
8 (偶数)
9
10 (偶数)
```

<details>
<summary>ヒント</summary>

`$(( 数値 % 2 ))` で余りを計算できます。

</details>

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# even_odd.sh - 偶数判定

for i in {1..10}; do
    if [ $((i % 2)) -eq 0 ]; then
        echo "$i (偶数)"
    else
        echo "$i"
    fi
done
```

</details>

### 問題6: while ループとファイル読み込み
`/etc/passwd` の各行からユーザー名（1列目）だけを取り出して表示するスクリプトを `while` ループで書いてください。

期待される出力例：
```
ユーザー: root
ユーザー: daemon
ユーザー: bin
...
```

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# list_users.sh - ユーザー一覧の表示

while IFS=: read -r username _; do
    echo "ユーザー: $username"
done < /etc/passwd
```

解説：
- `IFS=:` で区切り文字をコロンに設定
- `read -r username _` で最初のフィールドを username に、残りを `_`（使わない）に格納
- `< /etc/passwd` でファイルから入力を受け取る

</details>

### 問題7: case 文
引数で受け取った拡張子に応じて、ファイルの種類を表示するスクリプトを書いてください。

対応する拡張子：
- `.sh` → シェルスクリプト
- `.py` → Pythonスクリプト
- `.txt` / `.md` → テキストファイル
- `.tar.gz` / `.zip` → アーカイブファイル
- その他 → 不明なファイル形式

期待される出力例：
```
$ ./filetype.sh script.sh
script.sh はシェルスクリプトです

$ ./filetype.sh data.csv
data.csv は不明なファイル形式です
```

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# filetype.sh - ファイル種類の判定

if [ $# -eq 0 ]; then
    echo "使い方: $0 <ファイル名>"
    exit 1
fi

FILE="$1"

case "$FILE" in
    *.sh)
        echo "$FILE はシェルスクリプトです"
        ;;
    *.py)
        echo "$FILE はPythonスクリプトです"
        ;;
    *.txt|*.md)
        echo "$FILE はテキストファイルです"
        ;;
    *.tar.gz|*.zip)
        echo "$FILE はアーカイブファイルです"
        ;;
    *)
        echo "$FILE は不明なファイル形式です"
        ;;
esac
```

</details>

### 問題8: 関数の作成
以下の要件を満たすスクリプトを書いてください。

- `log_info` 関数: タイムスタンプ付きの情報ログを表示
- `log_error` 関数: タイムスタンプ付きのエラーログを標準エラー出力に表示
- メイン処理で両方の関数を使用

期待される出力例：
```
[2024-03-20 15:30:00] [INFO] 処理を開始します
[2024-03-20 15:30:00] [INFO] ファイルを処理中...
[2024-03-20 15:30:00] [ERROR] ファイルが見つかりません
[2024-03-20 15:30:00] [INFO] 処理を終了します
```

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# logging.sh - ログ関数の例

# ログ関数の定義
log_info() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [INFO] $1"
}

log_error() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [ERROR] $1" >&2
}

# メイン処理
log_info "処理を開始します"
log_info "ファイルを処理中..."
log_error "ファイルが見つかりません"
log_info "処理を終了します"
```

解説：
- `>&2` で標準エラー出力に出力（エラーログとして分離可能）
- `$(date '+%Y-%m-%d %H:%M:%S')` でタイムスタンプを生成
- `local` を使えば関数内の局所変数を定義できる

</details>

### 問題9: パイプとリダイレクト
以下の各操作を行うコマンドを書いてください。

1. `/etc/passwd` のユーザー数を数える
2. `ls -la` の結果を `filelist.txt` に保存しつつ画面にも表示する
3. コマンドの標準出力と標準エラー出力を別々のファイルに保存する
4. ヒアドキュメントを使って設定ファイルを作成する

<details>
<summary>解答</summary>

```bash
# 1. ユーザー数を数える

42
# または
$ wc -l < /etc/passwd
42

# 2. tee コマンドで画面とファイルに同時出力
$ ls -la | tee filelist.txt

# 3. 標準出力とエラー出力を別ファイルに保存
$ ./script.sh > output.log 2> error.log

# 4. ヒアドキュメントで設定ファイルを作成
$ cat << EOF > config.txt
server_name=web01
port=8080
environment=production
EOF
```

### 問題10: 引数バリデーション付きスクリプト
以下の要件を満たす計算スクリプト `calc.sh` を作成してください。

- 第1引数: 数値1
- 第2引数: 演算子（+, -, x, /）
- 第3引数: 数値2
- 引数が3つでなければ使い方を表示して終了
- 0での割り算はエラーメッセージを表示

期待される出力例：
```
$ ./calc.sh 10 + 3
10 + 3 = 13

$ ./calc.sh 10 / 0
エラー: 0で割ることはできません

$ ./calc.sh 10
使い方: ./calc.sh <数値1> <演算子> <数値2>
演算子: +, -, x, /
```

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# calc.sh - 簡易計算機

# 引数の数をチェック
if [ $# -ne 3 ]; then
    echo "使い方: $0 <数値1> <演算子> <数値2>"
    echo "演算子: +, -, x, /"
    exit 1
fi

NUM1="$1"
OP="$2"
NUM2="$3"

case "$OP" in
    +)
        RESULT=$((NUM1 + NUM2))
        ;;
    -)
        RESULT=$((NUM1 - NUM2))
        ;;
    x)
        RESULT=$((NUM1 * NUM2))
        ;;
    /)
        # 0除算チェック
        if [ "$NUM2" -eq 0 ]; then
            echo "エラー: 0で割ることはできません"
            exit 1
        fi
        RESULT=$((NUM1 / NUM2))
        ;;
    *)
        echo "エラー: 不明な演算子 '$OP'"
        echo "使用可能な演算子: +, -, x, /"
        exit 1
        ;;
esac

echo "$NUM1 $OP $NUM2 = $RESULT"
```

</details>

### 問題11: 配列を使ったスクリプト
サーバーリストを配列で定義し、各サーバーの死活を確認するスクリプトを書いてください。

- `ping -c 1 -W 2` で確認する
- 結果を `[OK]` または `[NG]` で表示する
- 最後に OK / NG の数を集計して表示する

期待される出力例：
```
=== サーバー死活監視 ===
[OK] web01 は応答しています
[NG] web02 が応答しません！
[OK] db01 は応答しています

=== 結果サマリー ===
OK: 2 台
NG: 1 台
```

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# server_check.sh - サーバー死活監視

# サーバーリスト（環境に合わせて変更）
servers=("web01" "web02" "db01")

ok_count=0
ng_count=0

echo "=== サーバー死活監視 ==="

for server in "${servers[@]}"; do
    if ping -c 1 -W 2 "$server" > /dev/null 2>&1; then
        echo "[OK] $server は応答しています"
        ok_count=$((ok_count + 1))
    else
        echo "[NG] $server が応答しません！"
        ng_count=$((ng_count + 1))
    fi
done

echo ""
echo "=== 結果サマリー ==="
echo "OK: $ok_count 台"
echo "NG: $ng_count 台"
```

</details>
