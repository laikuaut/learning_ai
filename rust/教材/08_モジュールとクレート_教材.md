## 8.1 モジュール（mod）の基本

### 8.1.1 モジュールとは

モジュール（module）は、コードを名前空間（namespace）で整理する仕組みです。関連する関数・構造体・定数などをグループ化して管理します。

```
┌──────────────────────────────────────────────┐
│  モジュールの階層構造                           │
│                                              │
│  crate (ルート)                               │
│  ├── main.rs / lib.rs                        │
│  ├── mod math                                │
│  │   ├── fn add()                            │
│  │   ├── fn subtract()                       │
│  │   └── mod geometry                        │
│  │       ├── fn area()                       │
│  │       └── fn perimeter()                  │
│  └── mod utils                               │
│      ├── fn format_number()                  │
│      └── fn validate()                       │
└──────────────────────────────────────────────┘
```

### 8.1.2 インラインモジュールの定義

`mod` キーワードで同じファイル内にモジュールを定義できます。

```rust
// モジュールの定義
mod math {
    pub fn add(a: i32, b: i32) -> i32 {
        a + b
    }

    pub fn subtract(a: i32, b: i32) -> i32 {
        a - b
    }

    pub fn multiply(a: i32, b: i32) -> i32 {
        a * b
    }

    // pubを付けない → モジュール外からはアクセスできない
    fn _internal_helper() -> i32 {
        42
    }
}

mod greeting {
    pub fn hello(name: &str) {
        println!("こんにちは、{}さん！", name);
    }

    pub fn goodbye(name: &str) {
        println!("さようなら、{}さん！", name);
    }
}

fn main() {
    // モジュール内の関数をフルパスで呼び出す
    let result = math::add(10, 20);
    println!("10 + 20 = {}", result);

    let result = math::multiply(3, 7);
    println!("3 * 7 = {}", result);

    greeting::hello("田中");
    greeting::goodbye("田中");

    // math::_internal_helper();  // コンパイルエラー！private関数
}
```

**実行結果：**
```
10 + 20 = 30
3 * 7 = 21
こんにちは、田中さん！
さようなら、田中さん！
```

### 8.1.3 ネストしたモジュール

モジュールはネスト（入れ子）にできます。

```rust
mod company {
    pub mod engineering {
        pub fn develop() {
            println!("開発中...");
            // 親モジュールの関数を呼ぶには super:: を使う
            super::log("engineering::develop が呼ばれました");
        }

        pub mod backend {
            pub fn setup_api() {
                println!("APIをセットアップ");
                // 2階層上の親モジュールの関数
                super::super::log("backend::setup_api が呼ばれました");
            }
        }
    }

    pub mod sales {
        pub fn make_deal() {
            println!("商談成立！");
        }
    }

    // company モジュール内のプライベート関数
    fn log(message: &str) {
        println!("[LOG] {}", message);
    }
}

fn main() {
    company::engineering::develop();
    company::engineering::backend::setup_api();
    company::sales::make_deal();
}
```

**実行結果：**
```
開発中...
[LOG] engineering::develop が呼ばれました
APIをセットアップ
[LOG] backend::setup_api が呼ばれました
商談成立！
```

### ポイントまとめ

- `mod` でモジュールを定義し、コードを名前空間で整理
- `::` でモジュール内のアイテムにアクセス
- `super::` で親モジュールにアクセス
- モジュールはネストできる
- デフォルトはプライベート（外部からアクセス不可）

## 8.2 pub によるアクセス制御

### 8.2.1 pub の基本

Rustではすべてのアイテム（関数、構造体、フィールド、モジュールなど）はデフォルトでプライベートです。外部に公開するには `pub` キーワードを付けます。

```
┌──────────────────────────────────────────────────┐
│  アクセス制御の種類                                  │
│                                                  │
│  pub         → 完全に公開                          │
│  pub(crate)  → 同じクレート内でのみ公開              │
│  pub(super)  → 親モジュールにのみ公開                │
│  (なし)      → 同じモジュール内でのみアクセス可能     │
└──────────────────────────────────────────────────┘
```

```rust
mod database {
    // 公開構造体
    pub struct Connection {
        pub host: String,        // 公開フィールド
        pub port: u16,           // 公開フィールド
        password: String,        // 非公開フィールド（外部から直接アクセス不可）
        connected: bool,         // 非公開フィールド
    }

    impl Connection {
        // 公開コンストラクタ
        pub fn new(host: &str, port: u16, password: &str) -> Connection {
            Connection {
                host: String::from(host),
                port,
                password: String::from(password),
                connected: false,
            }
        }

        // 公開メソッド
        pub fn connect(&mut self) {
            // 内部でプライベート関数を使う
            if validate_password(&self.password) {
                self.connected = true;
                println!("{}:{} に接続しました", self.host, self.port);
            } else {
                println!("パスワードが無効です");
            }
        }

        pub fn is_connected(&self) -> bool {
            self.connected
        }

        pub fn query(&self, sql: &str) {
            if self.connected {
                println!("SQL実行: {}", sql);
            } else {
                println!("エラー: 接続されていません");
            }
        }
    }

    // プライベート関数（モジュール内部でのみ使用）
    fn validate_password(password: &str) -> bool {
        password.len() >= 8
    }
}

fn main() {
    let mut conn = database::Connection::new("localhost", 5432, "mypassword123");
    println!("ホスト: {}", conn.host);   // pub フィールド → OK
    println!("ポート: {}", conn.port);   // pub フィールド → OK
    // println!("{}", conn.password);     // 非公開 → コンパイルエラー！

    conn.connect();
    println!("接続状態: {}", conn.is_connected());
    conn.query("SELECT * FROM users");
}
```

**実行結果：**
```
ホスト: localhost
ポート: 5432
localhost:5432 に接続しました
接続状態: true
SQL実行: SELECT * FROM users
```

### 8.2.2 pub(crate) と pub(super)

```rust
mod outer {
    pub mod inner {
        // クレート全体に公開
        pub(crate) fn crate_visible() {
            println!("クレート内から見える");
        }

        // 親モジュール(outer)にのみ公開
        pub(super) fn parent_visible() {
            println!("親モジュールから見える");
        }

        // このモジュール内でのみ
        fn private_fn() {
            println!("innerモジュール内でのみ");
        }

        pub fn public_fn() {
            // 同じモジュール内からはすべてアクセス可能
            private_fn();
            parent_visible();
            crate_visible();
        }
    }

    pub fn use_inner() {
        inner::parent_visible();  // OK: 親モジュールから
        inner::crate_visible();   // OK: 同じクレート内
        // inner::private_fn();   // NG: プライベート
    }
}

fn main() {
    outer::inner::public_fn();
    outer::inner::crate_visible();  // OK: 同じクレート内
    // outer::inner::parent_visible();  // NG: mainはouterの親ではない
    outer::use_inner();
}
```

**実行結果：**
```
innerモジュール内でのみ
親モジュールから見える
クレート内から見える
親モジュールから見える
クレート内から見える
```

### 8.2.3 構造体のフィールドとenumのアクセス制御

```rust
mod shapes {
    // 構造体: フィールドごとにpubを制御
    pub struct Circle {
        pub radius: f64,        // 外部に公開
        center_x: f64,          // 非公開（内部実装の詳細）
        center_y: f64,          // 非公開
    }

    impl Circle {
        pub fn new(radius: f64, x: f64, y: f64) -> Circle {
            Circle { radius, center_x: x, center_y: y }
        }

        pub fn area(&self) -> f64 {
            std::f64::consts::PI * self.radius * self.radius
        }

        pub fn center(&self) -> (f64, f64) {
            (self.center_x, self.center_y)
        }
    }

    // enum: pubを付けるとバリアントも自動的に公開
    pub enum Color {
        Red,           // 自動的にpub
        Green,         // 自動的にpub
        Blue,          // 自動的にpub
        Custom(u8, u8, u8),  // 自動的にpub
    }
}

fn main() {
    // Circleはコンストラクタ経由でのみ作成可能
    let c = shapes::Circle::new(5.0, 10.0, 20.0);
    println!("半径: {}", c.radius);        // OK: pubフィールド
    // println!("{}", c.center_x);         // NG: 非公開フィールド
    println!("中心: {:?}", c.center());    // OK: pubメソッド経由
    println!("面積: {:.2}", c.area());

    // enumのバリアントは自動的にpub
    let color = shapes::Color::Custom(128, 64, 255);
    match color {
        shapes::Color::Red => println!("赤"),
        shapes::Color::Green => println!("緑"),
        shapes::Color::Blue => println!("青"),
        shapes::Color::Custom(r, g, b) => println!("カスタム({},{},{})", r, g, b),
    }
}
```

**実行結果：**
```
半径: 5
中心: (10.0, 20.0)
面積: 78.54
カスタム(128,64,255)
```

> **現場のプロの知見：** 非公開フィールドを持つ構造体は、コンストラクタ（`new` 関数）経由でしか作成できません。これにより、不正な状態のインスタンスが作られることを防げます。これは「情報隠蔽」（information hiding）と呼ばれる重要な設計原則です。

### ポイントまとめ

| 可視性 | 構文 | アクセス範囲 |
|---|---|---|
| プライベート | (なし) | 同じモジュール内のみ |
| 親モジュールまで | `pub(super)` | 親モジュールまで |
| クレート内公開 | `pub(crate)` | 同じクレート内 |
| 完全公開 | `pub` | どこからでも |

- 構造体: フィールドごとに `pub` を制御
- enum: `pub` を付けるとバリアントも自動的に公開
- デフォルトはプライベート → 必要最小限だけ公開する

## 8.3 use キーワードとパス

### 8.3.1 use の基本

`use` キーワードでモジュールパスをスコープに取り込み、長いパスの繰り返しを避けられます。

```rust
mod geometry {
    pub mod shapes {
        pub struct Rectangle {
            pub width: f64,
            pub height: f64,
        }

        impl Rectangle {
            pub fn new(width: f64, height: f64) -> Rectangle {
                Rectangle { width, height }
            }

            pub fn area(&self) -> f64 {
                self.width * self.height
            }
        }

        pub fn circle_area(radius: f64) -> f64 {
            std::f64::consts::PI * radius * radius
        }
    }

    pub mod utils {
        pub fn format_area(area: f64) -> String {
            format!("{:.2} 平方単位", area)
        }
    }
}

// useでパスを短くする
use geometry::shapes::Rectangle;
use geometry::shapes::circle_area;
use geometry::utils::format_area;

fn main() {
    // useのおかげで短い名前で使える
    let rect = Rectangle::new(10.0, 5.0);
    println!("長方形の面積: {}", format_area(rect.area()));
    println!("円の面積: {}", format_area(circle_area(3.0)));
}
```

**実行結果：**
```
長方形の面積: 50.00 平方単位
円の面積: 28.27 平方単位
```

### 8.3.2 ネストしたuse とグロブ

```rust
mod app {
    pub mod models {
        pub struct User {
            pub name: String,
        }

        pub struct Post {
            pub title: String,
        }

        pub struct Comment {
            pub text: String,
        }
    }

    pub mod helpers {
        pub fn greet(name: &str) -> String {
            format!("Hello, {}!", name)
        }

        pub fn format_date(year: u32, month: u32, day: u32) -> String {
            format!("{:04}-{:02}-{:02}", year, month, day)
        }
    }
}

// ネストしたuse（複数アイテムをまとめてインポート）
use app::models::{User, Post, Comment};
use app::helpers::{greet, format_date};

// selfを使うパターン
// use app::models::{self, User};  // models自体とUserの両方をインポート

fn main() {
    let user = User { name: String::from("田中") };
    let post = Post { title: String::from("Rustの学習") };
    let comment = Comment { text: String::from("良い記事ですね") };

    println!("{}", greet(&user.name));
    println!("投稿: {}", post.title);
    println!("コメント: {}", comment.text);
    println!("日付: {}", format_date(2026, 4, 5));
}
```

**実行結果：**
```
Hello, 田中!
投稿: Rustの学習
コメント: 良い記事ですね
日付: 2026-04-05
```

### 8.3.3 as による別名

### 8.3.4 useの慣例（ベストプラクティス）

```rust
mod data {
    pub mod processing {
        pub fn transform(s: &str) -> String {
            s.to_uppercase()
        }
        pub fn validate(s: &str) -> bool {
            !s.is_empty()
        }
    }

    pub struct Record {
        pub name: String,
        pub value: i32,
    }
}

// 慣例1: 関数は親モジュールまでuseする（どこから来たか分かるように）
use data::processing;

// 慣例2: 構造体・enumはフルパスでuseする
use data::Record;

fn main() {
    // 関数: processing::transform() のように呼ぶ → 出所が明確
    let result = processing::transform("hello");
    println!("{}", result);

    if processing::validate("test") {
        println!("有効な入力です");
    }

    // 構造体: Record として直接使う
    let r = Record {
        name: String::from("test"),
        value: 42,
    };
    println!("{}: {}", r.name, r.value);
}
```

**実行結果：**
```
HELLO
有効な入力です
test: 42
```

> **現場のプロの知見：** `use` の慣例として、関数は親モジュールまでインポートし、構造体・enumは直接インポートします。これにより「この関数はどのモジュールの関数か」が一目で分かります。

### ポイントまとめ

- `use` でパスをスコープに取り込み、長いパスの繰り返しを避ける
- `{}` でネストしたインポート：`use mod::{A, B, C};`
- `as` で別名を付ける（名前衝突の回避に有用）
- 関数は親モジュールまで、構造体・enum は直接 `use` するのが慣例

## 8.4 ファイル分割とモジュール

### 8.4.1 ファイル構成とモジュールの対応

実際のプロジェクトでは、モジュールをファイルに分割して管理します。

```
┌──────────────────────────────────────────────────┐
│  ファイル構成 → モジュール構成                       │
│                                                  │
│  src/                                            │
│  ├── main.rs           → crate ルート             │
│  ├── math.rs           → mod math                │
│  ├── utils.rs          → mod utils               │
│  └── network/                                    │
│      ├── mod.rs        → mod network             │
│      ├── http.rs       → mod network::http       │
│      └── tcp.rs        → mod network::tcp        │
│                                                  │
│  ※ mod.rs はディレクトリのモジュール定義ファイル      │
│                                                  │
│  新しいスタイル（Rust 2018以降推奨）:                │
│  src/                                            │
│  ├── main.rs                                     │
│  ├── math.rs                                     │
│  ├── utils.rs                                    │
│  ├── network.rs        → mod network の定義       │
│  └── network/                                    │
│      ├── http.rs       → mod network::http       │
│      └── tcp.rs        → mod network::tcp        │
└──────────────────────────────────────────────────┘
```

### 8.4.2 ファイル分割の例

以下は単一ファイルの例ですが、ファイル分割した場合の構成をコメントで示します。

```rust
// ===== 実際のプロジェクトでは以下のようにファイル分割します =====
//
// src/main.rs:
//   mod math;
//   mod utils;
//   fn main() { ... }
//
// src/math.rs:
//   pub fn add(a: i32, b: i32) -> i32 { a + b }
//   pub fn multiply(a: i32, b: i32) -> i32 { a * b }
//
// src/utils.rs:
//   pub fn repeat(s: &str, n: usize) -> String { s.repeat(n) }
//
// ここでは動作確認のため、1つのファイルにまとめています。

mod math {
    pub fn add(a: i32, b: i32) -> i32 {
        a + b
    }

    pub fn multiply(a: i32, b: i32) -> i32 {
        a * b
    }

    pub fn factorial(n: u64) -> u64 {
        if n <= 1 { 1 } else { n * factorial(n - 1) }
    }
}

mod utils {
    pub fn repeat(s: &str, n: usize) -> String {
        s.repeat(n)
    }

    pub fn truncate(s: &str, max_len: usize) -> String {
        if s.len() > max_len {
            format!("{}...", &s[..max_len])
        } else {
            s.to_string()
        }
    }
}

use math::{add, multiply, factorial};
use utils::{repeat, truncate};

fn main() {
    println!("{} + {} = {}", 3, 5, add(3, 5));
    println!("{} * {} = {}", 4, 7, multiply(4, 7));
    println!("{}! = {}", 10, factorial(10));
    println!("{}", repeat("Ha", 3));
    println!("{}", truncate("Hello, World! This is Rust.", 15));
}
```

**実行結果：**
```
3 + 5 = 8
4 * 7 = 28
10! = 3628800
HaHaHa
Hello, World! ...
```

### 8.4.3 サブモジュールのファイル分割例

```rust
// ===== ファイル分割した場合の構成 =====
//
// src/network.rs（またはsrc/network/mod.rs）:
//   pub mod http;
//   pub mod dns;
//
// src/network/http.rs:
//   pub struct Request { ... }
//   pub struct Response { ... }
//
// src/network/dns.rs:
//   pub fn resolve(domain: &str) -> String { ... }
//
// ここでは1ファイルにまとめています。

mod network {
    pub mod http {
        #[derive(Debug)]
        pub struct Request {
            pub method: String,
            pub url: String,
        }

        #[derive(Debug)]
        pub struct Response {
            pub status: u16,
            pub body: String,
        }

        impl Request {
            pub fn get(url: &str) -> Request {
                Request {
                    method: String::from("GET"),
                    url: String::from(url),
                }
            }

            pub fn send(&self) -> Response {
                println!("{} {} を送信中...", self.method, self.url);
                Response {
                    status: 200,
                    body: String::from("{\"message\": \"success\"}"),
                }
            }
        }

        impl std::fmt::Display for Response {
            fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
                write!(f, "HTTP {} - {}", self.status, self.body)
            }
        }
    }

    pub mod dns {
        pub fn resolve(domain: &str) -> String {
            // 擬似的なDNS解決
            match domain {
                "localhost" => String::from("127.0.0.1"),
                "example.com" => String::from("93.184.216.34"),
                _ => String::from("0.0.0.0"),
            }
        }
    }
}

use network::http::Request;
use network::dns;

fn main() {
    // DNS解決
    let ip = dns::resolve("example.com");
    println!("example.com → {}", ip);

    // HTTPリクエスト
    let req = Request::get("https://example.com/api/data");
    let res = req.send();
    println!("レスポンス: {}", res);
}
```

**実行結果：**
```
example.com → 93.184.216.34
GET https://example.com/api/data を送信中...
レスポンス: HTTP 200 - {"message": "success"}
```

### ポイントまとめ

- 1ファイル = 1モジュールの対応でコードを整理
- `mod モジュール名;` でファイルからモジュールを読み込む
- サブモジュールは `mod.rs` またはディレクトリ名と同名の `.rs` ファイルで定義
- Rust 2018以降は `network.rs` + `network/` ディレクトリのスタイルが推奨

## 8.5 Cargo.toml の基本

### 8.5.1 プロジェクトの作成

### 8.6.2 外部クレートの使用例（標準ライブラリのみで模擬）

```rust
// 外部クレートを使う場合、use で取り込む
// use rand::Rng;          // 実際のプロジェクトではこう書く
// use serde::{Serialize, Deserialize};

// 以下は外部クレートの「使い方の概念」を標準ライブラリで再現

// ランダム風の処理（本来はrand crateを使う）
mod pseudo_random {
    use std::collections::hash_map::DefaultHasher;
    use std::hash::{Hash, Hasher};
    use std::time::SystemTime;

    pub fn random_u32(max: u32) -> u32 {
        let seed = SystemTime::now()
            .duration_since(SystemTime::UNIX_EPOCH)
            .unwrap()
            .subsec_nanos();
        let mut hasher = DefaultHasher::new();
        seed.hash(&mut hasher);
        (hasher.finish() % max as u64) as u32
    }
}

// JSON風のシリアライズ（本来はserde_json crateを使う）
mod pseudo_json {
    pub fn to_json_string(pairs: &[(&str, &str)]) -> String {
        let fields: Vec<String> = pairs.iter()
            .map(|(k, v)| format!("  \"{}\": \"{}\"", k, v))
            .collect();
        format!("{{\n{}\n}}", fields.join(",\n"))
    }
}

fn main() {
    // ランダム値の生成
    let number = pseudo_random::random_u32(100);
    println!("ランダム値（0-99）: {}", number);

    // JSONシリアライズ
    let data = vec![
        ("name", "田中太郎"),
        ("email", "tanaka@example.com"),
        ("role", "developer"),
    ];
    let json = pseudo_json::to_json_string(&data);
    println!("JSON:\n{}", json);

    println!("\n--- 実際のプロジェクトでの書き方 ---");
    println!("// Cargo.toml:");
    println!("// [dependencies]");
    println!("// rand = \"0.8\"");
    println!("// serde = {{ version = \"1.0\", features = [\"derive\"] }}");
    println!("// serde_json = \"1.0\"");
    println!("//");
    println!("// use rand::Rng;");
    println!("// let n = rand::thread_rng().gen_range(0..100);");
}

## 8.7 テストの書き方

### 8.7.1 ユニットテストの基本

`#[test]` 属性を付けた関数がテスト関数になります。`cargo test` で実行します。

```rust
// テスト対象の関数
fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn is_even(n: i32) -> bool {
    n % 2 == 0
}

fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err(String::from("ゼロ除算"))
    } else {
        Ok(a / b)
    }
}

fn first_word(s: &str) -> &str {
    match s.find(' ') {
        Some(pos) => &s[..pos],
        None => s,
    }
}

// テストモジュール
#[cfg(test)]  // テスト時のみコンパイル
mod tests {
    use super::*;  // 親モジュールのアイテムをインポート

    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
        assert_eq!(add(-1, 1), 0);
        assert_eq!(add(0, 0), 0);
    }

    #[test]
    fn test_is_even() {
        assert!(is_even(4));
        assert!(is_even(0));
        assert!(!is_even(3));
        assert!(!is_even(-1));
    }

    #[test]
    fn test_divide_success() {
        let result = divide(10.0, 3.0);
        assert!(result.is_ok());
        let value = result.unwrap();
        assert!((value - 3.3333).abs() < 0.001);
    }

    #[test]
    fn test_divide_by_zero() {
        let result = divide(10.0, 0.0);
        assert!(result.is_err());
        assert_eq!(result.unwrap_err(), "ゼロ除算");
    }

    #[test]
    fn test_first_word() {
        assert_eq!(first_word("hello world"), "hello");
        assert_eq!(first_word("single"), "single");
        assert_eq!(first_word(""), "");
    }
}

fn main() {
    println!("テストは cargo test で実行します");
    println!("add(2, 3) = {}", add(2, 3));
    println!("is_even(4) = {}", is_even(4));
    println!("divide(10, 3) = {:?}", divide(10.0, 3.0));
    println!("first_word(\"hello world\") = {}", first_word("hello world"));
}

### 8.7.2 assert! 系マクロ

```rust
#[cfg(test)]
mod tests {
    #[test]
    fn demo_assert_macros() {
        // assert!: 条件がtrueであることを確認
        assert!(1 + 1 == 2);

        // assert_eq!: 2つの値が等しいことを確認
        assert_eq!(2 + 2, 4);

        // assert_ne!: 2つの値が異なることを確認
        assert_ne!(2 + 2, 5);

        // カスタムメッセージ
        let value = 42;
        assert!(value > 0, "値は正の数であるべき: got {}", value);
        assert_eq!(value, 42, "値は42であるべき: got {}", value);

    #[should_panic]
    fn test_should_panic() {

    #[test]
    #[should_panic(expected = "ゼロ除算")]
    fn test_panic_message() {
        panic!("ゼロ除算エラー");
    }

    // Resultを返すテスト（?演算子が使える）
    #[test]
    fn test_with_result() -> Result<(), String> {
        let result: i32 = "42".parse().map_err(|e| format!("{}", e))?;
        assert_eq!(result, 42);
        Ok(())
    }

    // テストを無視する
    #[test]
    #[ignore]
    fn expensive_test() {
        // 時間のかかるテスト（通常は実行しない）
        // cargo test -- --ignored で実行
        assert!(true);
    }
}

}

### 8.7.3 テスト駆動開発（TDD）の例

```rust
// ステップ1: まずテストを書く
// ステップ2: テストが通るように実装する
// ステップ3: リファクタリング

struct Calculator {
    history: Vec<String>,
}

impl Calculator {
    fn new() -> Calculator {
        Calculator {
            history: Vec::new(),
        }
    }

    fn add(&mut self, a: f64, b: f64) -> f64 {
        let result = a + b;
        self.history.push(format!("{} + {} = {}", a, b, result));
        result
    }

    fn subtract(&mut self, a: f64, b: f64) -> f64 {
        let result = a - b;
        self.history.push(format!("{} - {} = {}", a, b, result));
        result
    }

    fn multiply(&mut self, a: f64, b: f64) -> f64 {
        let result = a * b;
        self.history.push(format!("{} * {} = {}", a, b, result));
        result
    }

    fn divide(&mut self, a: f64, b: f64) -> Result<f64, String> {
        if b == 0.0 {
            return Err(String::from("ゼロ除算"));
        }
        let result = a / b;
        self.history.push(format!("{} / {} = {}", a, b, result));
        Ok(result)
    }

    fn history(&self) -> &[String] {
        &self.history
    }

    fn clear_history(&mut self) {
        self.history.clear();
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_add() {
        let mut calc = Calculator::new();
        assert_eq!(calc.add(2.0, 3.0), 5.0);
        assert_eq!(calc.add(-1.0, 1.0), 0.0);
    }

    #[test]
    fn test_subtract() {
        let mut calc = Calculator::new();
        assert_eq!(calc.subtract(5.0, 3.0), 2.0);
        assert_eq!(calc.subtract(1.0, 5.0), -4.0);
    }

    #[test]
    fn test_multiply() {
        let mut calc = Calculator::new();
        assert_eq!(calc.multiply(3.0, 4.0), 12.0);
        assert_eq!(calc.multiply(-2.0, 3.0), -6.0);
    }

    #[test]
    fn test_divide_success() {
        let mut calc = Calculator::new();
        assert_eq!(calc.divide(10.0, 2.0), Ok(5.0));
    }

    #[test]
    fn test_divide_by_zero() {
        let mut calc = Calculator::new();
        assert_eq!(calc.divide(10.0, 0.0), Err(String::from("ゼロ除算")));
    }

    #[test]
    fn test_history() {
        let mut calc = Calculator::new();
        calc.add(1.0, 2.0);
        calc.multiply(3.0, 4.0);
        assert_eq!(calc.history().len(), 2);
    }

    #[test]
    fn test_clear_history() {
        let mut calc = Calculator::new();
        calc.add(1.0, 2.0);
        calc.clear_history();
        assert!(calc.history().is_empty());
    }
}

fn main() {
    let mut calc = Calculator::new();

    println!("電卓デモ:");
    println!("  2 + 3 = {}", calc.add(2.0, 3.0));
    println!("  10 - 4 = {}", calc.subtract(10.0, 4.0));
    println!("  3 * 7 = {}", calc.multiply(3.0, 7.0));

    match calc.divide(15.0, 4.0) {
        Ok(result) => println!("  15 / 4 = {}", result),
        Err(e) => println!("  エラー: {}", e),
    }

    println!("\n計算履歴:");
    for entry in calc.history() {
        println!("  {}", entry);
    }
}
```

**実行結果：**
```
電卓デモ:
  2 + 3 = 5
  10 - 4 = 6
  3 * 7 = 21
  15 / 4 = 3.75

計算履歴:
  2 + 3 = 5
  10 - 4 = 6
  3 * 7 = 21
  15 / 4 = 3.75
```

## 8.8 ドキュメンテーション

### 8.8.1 ドキュメンテーションコメント

`///` で書くコメントはドキュメンテーションコメントです。`cargo doc` でHTMLドキュメントが生成されます。Markdownが使えます。

```rust
/// 2つの数値を加算します。
///
/// # 引数
///
/// * `a` - 1つ目の数値
/// * `b` - 2つ目の数値
///
/// # 戻り値
///
/// 2つの数値の和を返します。
///
/// # 例
///
/// ```
/// let result = add(2, 3);
/// assert_eq!(result, 5);
/// ```
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

/// 文字列を指定回数繰り返します。
///
/// # 引数
///
/// * `s` - 繰り返す文字列
/// * `n` - 繰り返し回数
///
/// # 例
///
/// ```
/// let result = repeat_string("Ha", 3);
/// assert_eq!(result, "HaHaHa");
/// ```
///
/// # パニック
///
/// この関数はパニックしません。
pub fn repeat_string(s: &str, n: usize) -> String {
    s.repeat(n)
}

/// 安全な除算を行います。
///
/// ゼロ除算の場合は `Err` を返します。
///
/// # 引数
///
/// * `a` - 被除数
/// * `b` - 除数（0以外）
///
/// # 戻り値
///
/// * `Ok(f64)` - 除算結果
/// * `Err(String)` - エラーメッセージ
///
/// # 例
///
/// ```
/// assert_eq!(safe_divide(10.0, 2.0), Ok(5.0));
/// assert!(safe_divide(10.0, 0.0).is_err());
/// ```
pub fn safe_divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err(String::from("ゼロ除算エラー"))
    } else {
        Ok(a / b)
    }
}

/// ユーザー情報を表す構造体。
///
/// # フィールド
///
/// * `name` - ユーザー名
/// * `age` - 年齢
///
/// # 例
///
/// ```
/// let user = User::new("田中", 30);
/// assert_eq!(user.name(), "田中");
/// ```
pub struct User {
    name: String,
    age: u32,
}

impl User {
    /// 新しい `User` インスタンスを作成します。
    ///
    /// # 引数
    ///
    /// * `name` - ユーザー名
    /// * `age` - 年齢
    pub fn new(name: &str, age: u32) -> User {
        User {
            name: String::from(name),
            age,
        }
    }

    /// ユーザー名を返します。
    pub fn name(&self) -> &str {
        &self.name
    }

    /// 年齢を返します。
    pub fn age(&self) -> u32 {
        self.age
    }

    /// ユーザー情報を文字列で返します。
    pub fn info(&self) -> String {
        format!("{} ({}歳)", self.name, self.age)
    }
}

fn main() {
    // 関数のデモ
    println!("2 + 3 = {}", add(2, 3));
    println!("{}", repeat_string("Rust! ", 3));

    match safe_divide(10.0, 3.0) {
        Ok(result) => println!("10 / 3 = {:.4}", result),
        Err(e) => println!("エラー: {}", e),
    }

    // 構造体のデモ
    let user = User::new("田中太郎", 30);
    println!("ユーザー: {}", user.info());

    println!("\nドキュメント生成: cargo doc --open");
}
```

**実行結果：**
```

Rust! Rust! Rust! 
10 / 3 = 3.3333
ユーザー: 田中太郎 (30歳)

ドキュメント生成: cargo doc --open
```

ドキュメント内のコード例（````rust` ブロック）は `cargo test` で自動的にテストされます。

```
┌──────────────────────────────────────────────────┐
│  ドキュメントテストの利点                            │
│                                                  │
│  1. ドキュメントが常に最新の状態に保たれる            │
│  2. サンプルコードが確実に動作することが保証される     │
│  3. テストとドキュメントを1箇所で管理できる           │
│                                                  │
│  /// ```                                         │
│  /// let result = add(2, 3);                     │
│  /// assert_eq!(result, 5);  // テストされる！     │
│  /// ```                                         │
└──────────────────────────────────────────────────┘
```

> **現場のプロの知見：** Rustのドキュメントテストは非常に強力な機能です。ドキュメントに書いたコード例が自動テストされるため、「ドキュメントは古いが実装は新しい」という状況を防げます。

### ポイントまとめ

| コメント種類 | 構文 | 用途 |
|---|---|---|
| ドキュメンテーション | `///` | 関数・構造体・enumなどの説明 |
| モジュールドキュメント | `//!` | モジュール全体の説明 |
| 通常のコメント | `//` | 内部実装のメモ |

- ドキュメントコメントはMarkdown形式で書ける
- `# 例`、`# 引数`、`# 戻り値`、`# パニック` などのセクションが慣例
- ドキュメント内のコード例は `cargo test` で自動テストされる
- `cargo doc --open` でHTMLドキュメントを生成・閲覧

## 8.9 実践例：ミニライブラリの構成

学んだ知識を統合した実践例です。

```rust
//! # テキスト処理ライブラリ
//!
//! 文字列の統計情報やフォーマットを行うユーティリティです。

/// テキストの統計情報を保持する構造体。
#[derive(Debug, Clone)]
pub struct TextStats {
    /// 文字数
    pub chars: usize,
    /// 単語数
    pub words: usize,
    /// 行数
    pub lines: usize,
    /// 文の数
    pub sentences: usize,
}

impl std::fmt::Display for TextStats {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        write!(f, "文字:{} 単語:{} 行:{} 文:{}",
            self.chars, self.words, self.lines, self.sentences)
    }
}

/// テキスト分析モジュール
mod analyzer {
    use super::TextStats;

    /// テキストの統計情報を計算します。
    ///
    /// # 引数
    ///
    /// * `text` - 分析対象のテキスト
    ///
    /// # 戻り値
    ///
    /// テキストの統計情報を含む `TextStats`
    pub fn analyze(text: &str) -> TextStats {
        let chars = text.chars().count();
        let words = count_words(text);
        let lines = if text.is_empty() { 0 } else { text.lines().count() };
        let sentences = count_sentences(text);

        TextStats { chars, words, lines, sentences }
    }

    fn count_words(text: &str) -> usize {
        text.split_whitespace().count()
    }

    fn count_sentences(text: &str) -> usize {
        text.chars().filter(|c| matches!(c, '.' | '!' | '?')).count()
    }

    /// 最も頻出する単語を返します。
    pub fn most_common_word(text: &str) -> Option<(String, usize)> {
        use std::collections::HashMap;

        let mut counts: HashMap<String, usize> = HashMap::new();
        for word in text.split_whitespace() {
            let cleaned: String = word.chars()
                .filter(|c| c.is_alphanumeric())
                .collect::<String>()
                .to_lowercase();
            if !cleaned.is_empty() {
                *counts.entry(cleaned).or_insert(0) += 1;
            }
        }

        counts.into_iter().max_by_key(|&(_, count)| count)
    }
}

/// テキストフォーマットモジュール
mod formatter {
    /// テキストを指定幅で折り返します。
    pub fn word_wrap(text: &str, max_width: usize) -> String {
        let mut result = String::new();
        let mut current_line_len = 0;

        for word in text.split_whitespace() {
            if current_line_len + word.len() > max_width && current_line_len > 0 {
                result.push('\n');
                current_line_len = 0;
            } else if current_line_len > 0 {
                result.push(' ');
                current_line_len += 1;
            }
            result.push_str(word);
            current_line_len += word.len();
        }

        result
    }

    /// テキストを中央揃えにします。
    pub fn center(text: &str, width: usize) -> String {
        text.lines()
            .map(|line| {
                let trimmed = line.trim();
                if trimmed.len() >= width {
                    trimmed.to_string()
                } else {
                    let padding = (width - trimmed.len()) / 2;
                    format!("{}{}", " ".repeat(padding), trimmed)
                }
            })
            .collect::<Vec<_>>()
            .join("\n")
    }

    /// テキストをボックスで囲みます。
    pub fn boxed(text: &str) -> String {
        let lines: Vec<&str> = text.lines().collect();
        let max_len = lines.iter().map(|l| l.len()).max().unwrap_or(0);

        let border = format!("+{}+", "-".repeat(max_len + 2));
        let mut result = vec![border.clone()];

        for line in &lines {
            result.push(format!("| {:<width$} |", line, width = max_len));
        }

        result.push(border);
        result.join("\n")
    }
}

// モジュールの公開API
use analyzer::{analyze, most_common_word};
use formatter::{word_wrap, center, boxed};

fn main() {
    let text = "Rust is a systems programming language. \
                Rust focuses on safety and performance. \
                Rust is great for building reliable software. \
                Many developers love Rust!";

    // テキスト分析
    println!("=== テキスト分析 ===");
    let stats = analyze(text);
    println!("{}", stats);

    if let Some((word, count)) = most_common_word(text) {
        println!("最頻出単語: \"{}\" ({}回)", word, count);
    }

    // テキストフォーマット
    println!("\n=== 折り返し（幅40） ===");
    println!("{}", word_wrap(text, 40));

    println!("\n=== 中央揃え（幅50） ===");
    let centered = center("Rust Programming\nSafe and Fast\n2026", 50);
    println!("{}", centered);

    println!("\n=== ボックス表示 ===");
    let info = "Project: Rust Learning\nVersion: 1.0.0\nAuthor: Student";
    println!("{}", boxed(info));

    // テストの存在を確認
    println!("\n※ テストは cargo test で実行できます");
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_analyze_empty() {
        let stats = analyze("");
        assert_eq!(stats.chars, 0);
        assert_eq!(stats.words, 0);
        assert_eq!(stats.lines, 0);
    }

    #[test]
    fn test_analyze_simple() {
        let stats = analyze("Hello World.");
        assert_eq!(stats.words, 2);
        assert_eq!(stats.sentences, 1);
        assert_eq!(stats.lines, 1);
    }

    #[test]
    fn test_most_common_word() {
        let result = most_common_word("the cat and the dog");
        assert_eq!(result, Some((String::from("the"), 2)));
    }

    #[test]
    fn test_most_common_word_empty() {
        let result = most_common_word("");
        assert_eq!(result, None);
    }

    #[test]
    fn test_word_wrap() {
        let wrapped = word_wrap("hello world foo bar", 12);
        assert!(wrapped.contains('\n'));
    }

    #[test]

        assert!(result.starts_with('+'));
        assert!(result.contains("Hello"));
    }

    #[test]
    fn test_text_stats_display() {
        let stats = analyze("Hello World.");
        let display = format!("{}", stats);
        assert!(display.contains("単語:2"));
    }
}
