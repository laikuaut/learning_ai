## 設問1（分析問題）

(1) C社員が受信したメールは、どのような攻撃手法に該当しますか。攻撃の名称と、通常のスパムメールとの違いを説明してください。

(2) ウイルス対策ソフトがマルウェアを検知できなかった理由として考えられるものを2つ挙げてください。

(3) マルウェアがC&Cサーバと通信を行う目的を2つ挙げてください。

<details>
<summary>解答</summary>

**(1) 攻撃手法**

**標的型攻撃（Targeted Attack）** に分類される **スピアフィッシング（Spear Phishing）** です。

通常のスパムメールとの違い:

| 項目 | スパムメール | スピアフィッシング |
|------|------------|-----------------|
| 対象 | 不特定多数 | 特定の組織・個人 |
| 内容 | 汎用的（広告、詐欺等） | ターゲットに合わせてカスタマイズ（取引先を装う等） |
| 成功率 | 低い | 高い（事前調査に基づく巧妙な偽装） |
| 目的 | 金銭詐取、マルウェア配布 | 機密情報の窃取、長期的な侵入の維持 |

**(2) ウイルス対策ソフトが検知できなかった理由**

1. **ゼロデイマルウェア（未知のマルウェア）** であったため、パターンファイル（シグネチャベース）による検知ができなかった。標的型攻撃では、攻撃者がターゲット固有のマルウェアを作成することが多い。

2. **ファイルレスマルウェア** または正規のツール（PowerShell等）を悪用する手法（Living off the Land）を使用していた可能性がある。この場合、マルウェア本体がファイルとして保存されないため、ファイルスキャンでは検知できない。

**(3) C&Cサーバとの通信目的**

1. **遠隔操作の指令受信**: 攻撃者がC&Cサーバを通じて、マルウェアに対して次の行動指示（ファイル探索、データ収集、横展開など）を送信する。

2. **窃取データの送信（データエクスフィルトレーション）**: 収集した機密情報（ソースコード、顧客情報等）をC&Cサーバ経由で外部に送信する。今回の事例では500MBのデータが送信されている。

E主任がインシデント対応として実施すべき手順を、以下の選択肢から適切な順序に並べ替えてください。また、各手順で具体的に何を行うべきか説明してください。

**選択肢:**
- (ア) C社員のPCをネットワークから隔離する
- (イ) 経営層・関係者への報告を行う

- (エ) 被害範囲の特定（他のPCへの感染有無の調査）
- (オ) C社員のPCのフォレンジック調査（証拠保全）
- (カ) 再発防止策の策定と実施
- (キ) 全社員のパスワード強制変更

<details>
<summary>解答</summary>

**正しい順序と各手順の説明:**

**1. (ア) C社員のPCをネットワークから隔離する【封じ込め - 初動】**
- LANケーブルの抜去またはネットワークポートの無効化
- **最優先**: 被害の拡大とデータの追加流出を防ぐ
- ただし電源は切らない（メモリ上の証拠が消失するため）

**2. (ウ) C&Cサーバの通信先を特定し遮断する【封じ込め - ネットワーク】**
- UTMでC&CサーバのIPアドレスをブロックリストに追加
- 他のPCからの同一IPへの通信有無も確認
- 該当ドメインのDNSブロックも実施

**3. (イ) 経営層・関係者への報告【報告】**
- インシデントの概要、現在の状況、想定される被害を報告
- 必要に応じて、取引先D社への確認、警察・IPA等への届出を検討
- 個人情報漏洩の可能性がある場合は個人情報保護委員会への報告義務

**4. (オ) フォレンジック調査（証拠保全）【分析】**
- C社員PCのディスクイメージの取得（証拠の完全性を保つ）
- メモリダンプの取得（揮発性データの保全）
- マルウェアの検体確保と動作分析
- タイムラインの作成（いつ、何が起きたか）

**5. (エ) 被害範囲の特定【分析の拡大】**
- ADサーバのログからC社員アカウントの認証履歴を確認
- 社内Gitサーバのアクセスログを確認（ソースコードの閲覧・ダウンロード履歴）
- 全PCに対するIOC（Indicator of Compromise: 侵害指標）スキャン

**6. (キ) 全社員のパスワード強制変更【根絶】**
- C社員のアカウントが侵害されているため、関連する認証情報を無効化
- 横展開により他のアカウントも侵害されている可能性があるため、全社員のパスワードリセット
- ADのKerberosチケットの無効化（Pass-the-Ticket攻撃への対策）

**7. (カ) 再発防止策の策定と実施【事後対応】**
- インシデントの根本原因分析（Root Cause Analysis）
- セキュリティ対策の見直しと強化
- 教訓の文書化と全社共有

## 設問3（対策提案）

今回のインシデントを踏まえ、B社のセキュリティ対策の問題点を3つ指摘し、それぞれの改善策を提案してください。

<details>
<summary>解答</summary>

**問題点1: ログの定期分析が未実施**

金曜日に発生した不審な通信が月曜日まで検知されなかった。定期的なログ分析が行われていれば、C&Cサーバへの通信をより早期に発見できた可能性がある。

**改善策:**
- SIEM（Security Information and Event Management）の導入
- 異常な通信パターン（深夜帯の外部通信、大量データ送信等）のアラート設定
- SOC（Security Operation Center）による24時間監視（外部委託も含め検討）

**問題点2: 標的型メール訓練が未実施**

C社員がスピアフィッシングメールの添付ファイルを開封してしまった。定期的な訓練が行われていれば、不審メールへの対応力が向上していた可能性がある。

**改善策:**
- 四半期ごとの標的型メール訓練の実施
- 不審メール報告の窓口設置とフローの整備
- 開封してしまった場合の初動対応手順の周知

**問題点3: エンドポイント対策がシグネチャベースのみ**

パターンファイルに依存するウイルス対策ソフトでは、未知のマルウェアを検知できない。

**改善策:**
- EDR（Endpoint Detection and Response）の導入
  - 振る舞い検知によりゼロデイマルウェアにも対応
  - プロセスの不審な動作（不正なPowerShell実行、メモリインジェクション等）を検知
- サンドボックス型のメール検査（添付ファイルを仮想環境で実行して判定）

</details>

```python
def generate_sample_logs(count=200):

    for i in range(count):
        time = base_time + datetime.timedelta(minutes=random.randint(0, 3000))

        if i < count - 30:
            # 通常の通信
            src = random.choice(normal_ips)
            dst = random.choice(normal_dests)
            port = random.choice([80, 443, 53])
            size = random.randint(100, 10000)
        else:
            # 不審な通信（C社員PC → C&Cサーバ）
            src = suspicious_ip
            dst = cnc_server
            port = 443
            size = random.randint(50000, 500000)  # 大量データ
            time = base_time + datetime.timedelta(
                minutes=random.randint(20, 3000))

        logs.append({
            "timestamp": time.strftime("%Y-%m-%d %H:%M:%S"),
            "src_ip": src,
            "dst_ip": dst,
            "dst_port": port,
            "bytes": size,
            "action": "ALLOW",
        })

    logs.sort(key=lambda x: x["timestamp"])
    return logs

def analyze_logs(logs):
    """ログを分析して不審な通信を検出"""
    # IPごとの通信量を集計
    ip_stats = {}
    for log in logs:
        src = log["src_ip"]
        if src not in ip_stats:
            ip_stats[src] = {"count": 0, "total_bytes": 0, "destinations": set()}
        ip_stats[src]["count"] += 1
        ip_stats[src]["total_bytes"] += log["bytes"]
        ip_stats[src]["destinations"].add(log["dst_ip"])

    # 外部宛先ごとの通信量を集計
    dest_stats = {}
    for log in logs:
        dst = log["dst_ip"]
        if not dst.startswith("192.168."):
            if dst not in dest_stats:
                dest_stats[dst] = {"count": 0, "total_bytes": 0, "sources": set()}
            dest_stats[dst]["count"] += 1
            dest_stats[dst]["total_bytes"] += log["bytes"]
            dest_stats[dst]["sources"].add(log["src_ip"])

    return ip_stats, dest_stats
```

```python
# UTMログ分析ツール（改良版）
# 学べる内容：時系列分析、統計的異常検知、インシデント調査
# 実行方法：python log_analyzer_advanced.py

import datetime
import random
import math

def generate_sample_logs(count=300):
    """リアルなサンプルログデータを生成"""
    logs = []
    internal_pcs = {
        "192.168.10.50": "C社員PC（開発部）",
        "192.168.10.11": "D社員PC（開発部）",
        "192.168.10.22": "E社員PC（開発部）",
        "192.168.20.5": "F社員PC（営業部）",
        "192.168.20.15": "G社員PC（営業部）",
    }
    normal_dests = ["93.184.216.34", "151.101.1.69", "172.217.14.206",
                    "13.107.42.14", "52.96.108.18"]
    cnc_server = "45.33.32.156"

    base_time = datetime.datetime(2025, 4, 11, 8, 0, 0)

    for i in range(count):
        time_offset = random.randint(0, 4320)  # 3日分（金～日）
        time = base_time + datetime.timedelta(minutes=time_offset)

        if random.random() < 0.85:
            src = random.choice(list(internal_pcs.keys()))
            dst = random.choice(normal_dests)
            port = random.choice([80, 443])
            size = random.randint(500, 15000)
        else:
            src = "192.168.10.50"
            dst = cnc_server
            port = 443
            size = random.randint(100000, 800000)

        logs.append({
            "timestamp": time,
            "src_ip": src,
            "dst_ip": dst,
            "dst_port": port,
            "bytes": size,
        })

    logs.sort(key=lambda x: x["timestamp"])
    return logs, internal_pcs

def detect_anomalies(logs, internal_pcs):
    """統計的手法で異常を検出"""
    # IPペアごとの通信量を集計
    pair_stats = {}
    for log in logs:
        pair = (log["src_ip"], log["dst_ip"])
        if pair not in pair_stats:
            pair_stats[pair] = {"count": 0, "bytes": 0, "times": []}
        pair_stats[pair]["count"] += 1
        pair_stats[pair]["bytes"] += log["bytes"]
        pair_stats[pair]["times"].append(log["timestamp"])

    # 全ペアの平均通信量を計算
    all_bytes = [s["bytes"] for s in pair_stats.values()]
    mean_bytes = sum(all_bytes) / len(all_bytes) if all_bytes else 0
    variance = sum((b - mean_bytes) ** 2 for b in all_bytes) / len(all_bytes) if all_bytes else 0
    std_bytes = math.sqrt(variance)

    # 平均 + 2σ を超える通信ペアを異常とみなす
    threshold = mean_bytes + 2 * std_bytes
    anomalies = []
    for pair, stats in pair_stats.items():
        if stats["bytes"] > threshold:
            anomalies.append({
                "src": pair[0],
                "dst": pair[1],
                "src_name": internal_pcs.get(pair[0], "不明"),
                "count": stats["count"],
                "bytes": stats["bytes"],
                "first_seen": min(stats["times"]),
                "last_seen": max(stats["times"]),
                "deviation": (stats["bytes"] - mean_bytes) / std_bytes if std_bytes > 0 else 0,
            })

    return anomalies, mean_bytes, std_bytes, threshold

# --- メイン ---
print("===== UTMログ分析ツール（改良版） =====\n")

logs, internal_pcs = generate_sample_logs(300)
print(f"分析期間: {logs[0]['timestamp'].strftime('%Y-%m-%d %H:%M')} ～ "
      f"{logs[-1]['timestamp'].strftime('%Y-%m-%d %H:%M')}")
print(f"総ログ件数: {len(logs)}件\n")

# 異常検知
anomalies, mean_bytes, std_bytes, threshold = detect_anomalies(logs, internal_pcs)

print(f"--- 統計情報 ---")
print(f"通信ペアあたり平均通信量: {mean_bytes:,.0f} バイト")
print(f"標準偏差: {std_bytes:,.0f} バイト")
print(f"異常検知閾値（μ+2σ）: {threshold:,.0f} バイト\n")

# 異常通信の報告
print(f"--- 異常通信検出結果 ({len(anomalies)}件) ---")
for i, a in enumerate(sorted(anomalies, key=lambda x: x["bytes"], reverse=True), 1):
    print(f"\n[異常 #{i}]")
    print(f"  送信元: {a['src']} ({a['src_name']})")
    print(f"  宛先:   {a['dst']}")
    print(f"  通信回数: {a['count']}回")
    print(f"  総通信量: {a['bytes']:,} バイト ({a['bytes']/1024/1024:.1f} MB)")
    print(f"  偏差:     平均の{a['deviation']:.1f}σ上")
    print(f"  期間:     {a['first_seen'].strftime('%m/%d %H:%M')} ～ {a['last_seen'].strftime('%m/%d %H:%M')}")
```
