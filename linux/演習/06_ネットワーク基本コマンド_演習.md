## 基本問題

### 問題1: ネットワークの基礎知識
以下の質問に答えてください。

1. プライベートIPアドレスの範囲を3つ挙げてください
2. ポート番号 22、80、443 はそれぞれ何のサービスで使われますか？
3. TCP と UDP の主な違いは何ですか？

<details>
<summary>解答</summary>

```
1. プライベートIPアドレスの範囲:
   - 10.0.0.0/8       (10.0.0.0 ～ 10.255.255.255)
   - 172.16.0.0/12     (172.16.0.0 ～ 172.31.255.255)
   - 192.168.0.0/16    (192.168.0.0 ～ 192.168.255.255)

2. ポート番号とサービス:
   - 22:  SSH（リモートログイン）
   - 80:  HTTP（Webサイト、暗号化なし）
   - 443: HTTPS（Webサイト、暗号化あり）

3. TCP と UDP の違い:
   - TCP: 信頼性重視。データの到達を保証する（3ウェイハンドシェイク）。
         用途: Web、メール、SSH等
   - UDP: 速度重視。データの到達を保証しない。
         用途: DNS、動画ストリーミング、オンラインゲーム等
```

</details>

### 問題2: ip コマンド
以下の情報を取得する `ip` コマンドを書いてください。

1. 自分のIPアドレスを確認する
2. デフォルトゲートウェイを確認する
3. ネットワークインターフェースの通信統計を確認する

<details>
<summary>解答</summary>

```bash
# 1. IPアドレスの確認
$ ip addr show
# または短縮形
$ ip a

# 2. デフォルトゲートウェイの確認
$ ip route show
# または短縮形
$ ip r
# 出力例: default via 192.168.1.1 dev eth0

# 3. 通信統計の確認
$ ip -s link show eth0
```

</details>

### 問題3: ping コマンド
以下の要件を満たす `ping` コマンドを書いてください。

1. google.com に4回だけ ping を送信する
2. 192.168.1.1 に3秒のタイムアウトで1回 ping を送信する
3. ping の結果から成功/失敗を判定する（スクリプトで使える形式）

<details>
<summary>解答</summary>

```bash
# 1. 4回だけ送信
$ ping -c 4 google.com

# 2. タイムアウト付きで1回送信
$ ping -c 1 -W 3 192.168.1.1

# 3. スクリプトで使える形式（出力を破棄し、終了ステータスで判定）
$ ping -c 1 -W 2 192.168.1.1 > /dev/null 2>&1 && echo "OK" || echo "NG"
```

</details>

### 問題4: curl コマンド
以下の操作を行う `curl` コマンドを書いてください。

1. https://example.com のHTTPステータスコードだけを取得する
2. https://example.com のHTTPヘッダーのみを表示する
3. Web APIにJSON形式のPOSTリクエストを送信する

<details>
<summary>解答</summary>

```bash
# 1. ステータスコードのみ取得
$ curl -s -o /dev/null -w "%{http_code}" https://example.com
200

# 2. HTTPヘッダーのみ表示
$ curl -I https://example.com

# 3. JSON形式のPOSTリクエスト
$ curl -X POST \
    -H "Content-Type: application/json" \
    -d '{"name":"tanaka","email":"tanaka@example.com"}' \
    https://api.example.com/users
```

### 問題6: DNS の確認
以下の操作を行うコマンドを書いてください。

1. `google.com` のIPアドレスを調べる
2. `8.8.8.8` のドメイン名を逆引きする
3. `google.com` のメールサーバー（MXレコード）を調べる
4. 現在設定されているDNSサーバーを確認する

<details>
<summary>解答</summary>

```bash
# 1. ドメイン名からIPアドレスを調べる
$ host google.com
# または
$ dig +short google.com

# 2. 逆引き（IPからドメイン名）
$ host 8.8.8.8
# 出力例: 8.8.8.8.in-addr.arpa domain name pointer dns.google.

# 3. MXレコードの確認
$ dig google.com MX
# または
$ host -t MX google.com

# 4. DNSサーバーの確認
$ cat /etc/resolv.conf
```

</details>

### 問題7: ネットワーク疎通のトラブルシューティング
Webサーバーにアクセスできない場合のトラブルシューティング手順を、使用するコマンドとともに書いてください。

状況: `http://192.168.1.100` にブラウザからアクセスできない

<details>
<summary>ヒント</summary>

下から上に（物理層→アプリケーション層）確認していくのが基本です。

</details>

<details>
<summary>解答</summary>

```bash
# Step 1: ネットワーク疎通の確認（ICMP レベル）
$ ping -c 3 192.168.1.100
# → 通れば：ネットワーク接続は OK
# → 通らない：LAN/ルーティングの問題

# Step 2: ポートの確認（TCP レベル）
$ nc -zv 192.168.1.100 80
# → 接続できれば：ポートは開いている
# → 接続できない：ファイアウォールか、サービスが起動していない

# Step 3: HTTP レスポンスの確認（アプリケーションレベル）
$ curl -v http://192.168.1.100
# → レスポンスの内容やエラーメッセージを確認

# Step 4: （サーバー側で確認できる場合）サービスの状態確認
$ sudo systemctl status nginx
$ sudo ss -tlnp | grep :80

# Step 5: ファイアウォールの確認
$ sudo iptables -L -n | grep 80
```

</details>

### 問題8: wget と curl の使い分け
以下の各シナリオで、`wget` と `curl` のどちらを使うべきかを答え、コマンドを書いてください。

1. 大きなファイルをダウンロードし、途中で中断しても再開したい
2. REST API のレスポンスを確認したい
3. ファイルをバックグラウンドでダウンロードしたい

<details>
<summary>解答</summary>

```bash
# 1. 大きなファイルのダウンロード（再開可能）→ wget
$ wget -c https://example.com/large_file.iso
# -c オプションで中断されたダウンロードを再開できる

# 2. REST API のレスポンス確認 → curl
$ curl -s https://api.example.com/users | jq .
# curl は出力制御やヘッダー操作が柔軟

# 3. バックグラウンドでダウンロード → wget
$ wget -b https://example.com/large_file.iso
# -b オプションでバックグラウンド実行し、wget-log にログを出力
```

</details>

### 問題9: iptables の設定
以下の要件を満たすファイアウォールルールを `iptables` コマンドで書いてください。

- デフォルトポリシーは INPUT を DROP
- ループバック通信を許可
- 確立済みの接続を許可
- SSH（ポート22）を許可
- HTTP（ポート80）と HTTPS（ポート443）を許可
- ping（ICMP）を許可

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# firewall_setup.sh

# 既存のルールをクリア
sudo iptables -F

# デフォルトポリシー（すべて拒否）
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# ループバック通信を許可（必須）
sudo iptables -A INPUT -i lo -j ACCEPT

# 確立済みの接続を許可（必須）
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# SSH を許可
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# HTTP と HTTPS を許可
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# ping を許可
sudo iptables -A INPUT -p icmp --icmp-type echo-request -j ACCEPT

# 設定の確認
sudo iptables -L -n -v
```

重要ポイント：
- ループバックと確立済み接続の許可は必ず先に設定する
- SSH の許可は DROP ポリシーを設定する前に追加しないとリモート接続が切れる

</details>

### 問題10: ネットワーク監視スクリプト
複数のサーバーとポートの疎通確認を行い、結果をレポートするスクリプトを書いてください。

確認対象：
- web01:80
- web01:443
- db01:3306
- cache01:6379

期待される出力例：
```
=== ネットワーク疎通確認レポート ===
日時: 2024-03-20 15:30:00

[OK]  web01:80    - HTTP
[OK]  web01:443   - HTTPS
[NG]  db01:3306   - MySQL
[OK]  cache01:6379 - Redis

結果: OK=3, NG=1
```

<details>
<summary>解答</summary>

```bash
#!/bin/bash
# network_check.sh - ネットワーク疎通確認

# 確認対象の定義（ホスト:ポート:サービス名）
targets=(
    "web01:80:HTTP"
    "web01:443:HTTPS"
    "db01:3306:MySQL"
    "cache01:6379:Redis"
)

ok_count=0
ng_count=0

echo "=== ネットワーク疎通確認レポート ==="
echo "日時: $(date '+%Y-%m-%d %H:%M:%S')"
echo ""

for target in "${targets[@]}"; do
    # コロン区切りで分割
    host=$(echo "$target" | cut -d: -f1)
    port=$(echo "$target" | cut -d: -f2)
    service=$(echo "$target" | cut -d: -f3)

    # nc でポートの疎通確認（タイムアウト3秒）
    if nc -z -w 3 "$host" "$port" 2>/dev/null; then
        printf "[OK]  %-12s - %s\n" "${host}:${port}" "$service"
        ok_count=$((ok_count + 1))
    else
        printf "[NG]  %-12s - %s\n" "${host}:${port}" "$service"
        ng_count=$((ng_count + 1))
    fi
done

echo ""
echo "結果: OK=$ok_count, NG=$ng_count"
```

</details>
