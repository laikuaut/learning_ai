# 第6章：ポインタと参照

> **この章のゴール**: C++のポインタ（pointer）と参照（reference）の仕組みを理解し、メモリの扱い方・動的メモリ確保・スマートポインタを使った安全なメモリ管理ができるようになる。

---

## 6.1 メモリの仕組み

プログラムが実行されるとき、変数はコンピュータのメモリ（memory）上に格納されます。メモリには1バイトごとに「アドレス（address）」という番号が振られています。

```
メモリのイメージ（各マスが1バイト）

アドレス:  0x1000  0x1001  0x1002  0x1003  0x1004  0x1005  0x1006  0x1007
         ┌───────┬───────┬───────┬───────┬───────┬───────┬───────┬───────┐
         │       │       │       │       │       │       │       │       │
         └───────┴───────┴───────┴───────┴───────┴───────┴───────┴───────┘

例: int num = 42; の場合（intは通常4バイト）

アドレス:  0x1000  0x1001  0x1002  0x1003
         ┌───────┬───────┬───────┬───────┐
         │  42   │ (int型は4バイトを使用)  │   ← 変数 num
         └───────┴───────┴───────┴───────┘
         ↑
         numのアドレス = 0x1000
```

### アドレスを確認する

```cpp
#include <iostream>
using namespace std;

int main() {
    int num = 42;
    double pi = 3.14;
    char ch = 'A';

    cout << "num の値: " << num << ",  アドレス: " << &num << endl;
    cout << "pi  の値: " << pi  << ", アドレス: " << &pi << endl;
    cout << "ch  の値: " << ch  << ",  アドレス: " << (void*)&ch << endl;

    cout << endl;
    cout << "int のサイズ:    " << sizeof(int) << " バイト" << endl;
    cout << "double のサイズ: " << sizeof(double) << " バイト" << endl;
    cout << "char のサイズ:   " << sizeof(char) << " バイト" << endl;

    return 0;
}
```

`&`（アドレス演算子、address-of operator）を変数の前に付けると、その変数のメモリアドレスを取得できます。

## 6.2 ポインタの基本

ポインタ（pointer）とは、メモリアドレスを格納するための変数です。

### 宣言と基本操作

```
┌──────────────────────────────────────────┐
│  ポインタの2大演算子                      │
├──────────────────────────────────────────┤
│  &  : アドレス演算子（変数のアドレスを取得）│
│  *  : 間接参照演算子（アドレスの先の値）    │
└──────────────────────────────────────────┘
```

```cpp
#include <iostream>
using namespace std;

int main() {
    int num = 42;

    // ポインタの宣言: 型* ポインタ変数名
    int* ptr = &num;  // ptrにnumのアドレスを格納

    cout << "num の値:       " << num << endl;     // 42
    cout << "num のアドレス: " << &num << endl;     // 0x7fff...
    cout << "ptr の値:       " << ptr << endl;      // 0x7fff...（numのアドレス）
    cout << "*ptr の値:      " << *ptr << endl;     // 42（ptrが指す先の値）

    // ポインタ経由で値を変更
    *ptr = 100;
    cout << "変更後の num: " << num << endl;  // 100

    return 0;
}
```

### ポインタの図解

```
変数 num              ポインタ ptr
┌─────────┐          ┌─────────────┐
│   42    │  ←────── │  0x1000     │
└─────────┘          └─────────────┘
アドレス: 0x1000      ptr は num のアドレスを保持

  num = 42
  ptr = &num    →  ptr には 0x1000 が入る
  *ptr          →  0x1000 の先の値 = 42
```

### ポインタの宣言スタイル

```cpp
// 以下はすべて同じ意味（int型へのポインタ）
int* ptr1;   // 型に*を付けるスタイル（推奨）
int *ptr2;   // 変数名に*を付けるスタイル
int * ptr3;  // 間にスペースを入れるスタイル

// 注意: 複数宣言する場合
int* a, b;   // aはポインタ、bは普通のint！（よくある罠）
int *a, *b;  // 両方ポインタにするには各変数に*が必要
```

### ポイントまとめ
- ポインタは「メモリアドレスを格納する変数」
- `&` でアドレスを取得、`*` でアドレスの先の値にアクセス
- ポインタの型は「指す先のデータの型」に合わせる

## 6.3 ポインタと配列

C++では配列とポインタは深い関係があります。配列名は先頭要素のアドレスを表します。

```cpp
#include <iostream>
using namespace std;

int main() {
    int arr[] = {10, 20, 30, 40, 50};

    // 配列名は先頭要素のアドレス
    int* ptr = arr;  // &arr[0] と同じ

    cout << "配列の先頭アドレス: " << arr << endl;
    cout << "ptr の値:           " << ptr << endl;

    // ポインタ演算（pointer arithmetic）
    for (int i = 0; i < 5; i++) {
        cout << "*(ptr + " << i << ") = " << *(ptr + i)
             << "  arr[" << i << "] = " << arr[i] << endl;
    }

    return 0;
}
```

出力:
```
*(ptr + 0) = 10  arr[0] = 10
*(ptr + 1) = 20  arr[1] = 20
*(ptr + 2) = 30  arr[2] = 30
*(ptr + 3) = 40  arr[3] = 40
*(ptr + 4) = 50  arr[4] = 50
```

### ポインタ演算の図解

```
arr:  [10] [20] [30] [40] [50]
ptr:   ↑
       ptr    ptr+1  ptr+2  ptr+3  ptr+4

int が 4バイトの場合:
  ptr     = 0x1000  → 値: 10
  ptr + 1 = 0x1004  → 値: 20  （4バイト進む）
  ptr + 2 = 0x1008  → 値: 30  （8バイト進む）

ポインタ演算は「型のサイズ分」だけアドレスが移動する
```

### 関数に配列を渡す

```cpp
#include <iostream>
using namespace std;

// 配列を関数に渡す（実際にはポインタが渡される）
void printArray(const int* arr, int size) {
    for (int i = 0; i < size; i++) {
        cout << arr[i] << " ";
    }
    cout << endl;
}

// 配列の合計（別の書き方）
int sumArray(const int arr[], int size) {
    int total = 0;
    for (int i = 0; i < size; i++) {
        total += arr[i];
    }
    return total;
}

int main() {
    int data[] = {1, 2, 3, 4, 5};
    int size = sizeof(data) / sizeof(data[0]);

    printArray(data, size);     // 1 2 3 4 5
    cout << "合計: " << sumArray(data, size) << endl;  // 合計: 15

    return 0;
}
```

### よくある間違い

```cpp
// NG: 関数内で配列のサイズを sizeof で取得しようとする
void badFunc(int arr[]) {
    int size = sizeof(arr) / sizeof(arr[0]);  // ポインタのサイズ÷intのサイズになってしまう！
}

// OK: サイズは別の引数で渡す、またはモダンC++ではvectorを使う
void goodFunc(const vector<int>& arr) {
    cout << "サイズ: " << arr.size() << endl;
}
```

## 6.4 nullポインタ（nullptr）

ポインタがどこも指していない状態を表すために `nullptr` を使います。

```cpp
#include <iostream>
using namespace std;

int main() {
    int* ptr = nullptr;  // どこも指していない

    // nullptrチェック
    if (ptr == nullptr) {
        cout << "ポインタはnullです" << endl;
    }

    // C++11以降は nullptr を使う（NULLや0は使わない）
    // int* p = NULL;  // 古いスタイル（非推奨）
    // int* p = 0;     // 古いスタイル（非推奨）
    int* p = nullptr;  // モダンC++（推奨）

    // nullptrを間接参照すると未定義動作（クラッシュ）
    // *ptr = 10;  // 危険！絶対にやらない！

    int value = 42;
    ptr = &value;  // 有効なアドレスを設定

    if (ptr != nullptr) {
        cout << "値: " << *ptr << endl;  // 値: 42
    }

    return 0;
}
```

### ポイントまとめ
- 初期化されていないポインタは危険。必ず `nullptr` か有効なアドレスで初期化する
- ポインタを使う前に `nullptr` チェックを行う習慣をつける
- C++11以降は `NULL` や `0` ではなく `nullptr` を使う

## 6.5 参照（reference）

参照（reference）は、既存の変数に対する「別名（エイリアス）」です。ポインタよりも安全で使いやすい機能です。

```cpp
#include <iostream>
using namespace std;

int main() {
    int original = 42;

    // 参照の宣言: 型& 参照名 = 元の変数
    int& ref = original;  // ref は original の別名

    cout << "original: " << original << endl;  // 42
    cout << "ref:      " << ref << endl;       // 42

    // 参照を通じて値を変更
    ref = 100;
    cout << "original: " << original << endl;  // 100（変わった！）

    // 同じメモリを指している
    cout << "original のアドレス: " << &original << endl;
    cout << "ref のアドレス:      " << &ref << endl;
    // → 同じアドレスが表示される

    return 0;
}
```

### 参照の図解

```
ポインタの場合:                    参照の場合:
┌──────────┐    ┌──────────┐      ┌──────────┐
│ original │    │   ptr    │      │ original │ ← ref（別名）
│   42     │←───│ 0x1000   │      │   42     │
└──────────┘    └──────────┘      └──────────┘
 アドレス:0x1000                    アドレス:0x1000

ポインタ: 別の変数としてアドレスを保持
参照:     同じ変数の別の名前（追加メモリなし）
```

### 参照の制約

```cpp
int a = 10;
int b = 20;

int& ref = a;  // OK: 初期化時に結びつける
// int& ref2;  // NG: 初期化なしで宣言できない
// ref = b;    // これは ref を b に再束縛するのではなく、a に b の値（20）を代入する

// 参照の特徴:
// 1. 宣言時に必ず初期化が必要
// 2. 一度結びつけたら別の変数に変更できない
// 3. nullにできない（nullptrのような概念がない）
```

## 6.6 ポインタ vs 参照

```
┌──────────────────┬────────────────────┬────────────────────┐
│      特性        │   ポインタ          │   参照              │
├──────────────────┼────────────────────┼────────────────────┤
│ null可能         │ ○（nullptr）        │ ×                  │
│ 再代入           │ ○（別の先を指せる） │ ×（初期化時のみ）  │
│ 初期化必須       │ ×（推奨はされる）   │ ○                  │
│ 構文の簡潔さ     │ △（* や & が必要）  │ ○（通常の変数同様）│
│ 演算             │ ○（ポインタ演算）   │ ×                  │
│ 安全性           │ △                  │ ○                  │
└──────────────────┴────────────────────┴────────────────────┘

使い分けの指針:
  - 基本は参照を使う
  - 「何も指さない」状態が必要ならポインタ（nullptr）
  - 動的メモリ管理が必要ならスマートポインタ
  - レガシーコードやCライブラリとの連携ではポインタ
```

### 関数での使い分け

```cpp
#include <iostream>
#include <string>
using namespace std;

// 値を変更する場合: 参照渡し
void increment(int& value) {
    value++;
}

// 大きなオブジェクトを読むだけ: const参照
void display(const string& text) {
    cout << text << endl;
}

// オプショナル（あるかもしれない）: ポインタ
void processIfExists(const int* data) {
    if (data != nullptr) {
        cout << "データ: " << *data << endl;
    } else {
        cout << "データなし" << endl;
    }
}

int main() {
    int count = 10;
    increment(count);
    cout << "count: " << count << endl;  // 11

    display("こんにちは");

    int value = 42;
    processIfExists(&value);     // データ: 42
    processIfExists(nullptr);    // データなし

    return 0;
}
```

## 6.7 動的メモリ確保（dynamic memory allocation）

プログラム実行中にメモリを確保する仕組みです。`new` で確保し、`delete` で解放します。

### メモリの種類

```
┌───────────────────────────────────────────┐
│           プログラムのメモリ領域           │
├───────────────────────────────────────────┤
│  スタック（stack）                         │
│  ├── ローカル変数                          │
│  ├── 関数の引数                            │
│  └── 自動で確保・解放される                │
├───────────────────────────────────────────┤
│  ヒープ（heap）                            │
│  ├── new で確保する領域                    │
│  ├── プログラマが明示的に解放する          │
│  └── 解放し忘れるとメモリリーク            │
└───────────────────────────────────────────┘
```

### new と delete

```cpp
#include <iostream>
using namespace std;

int main() {
    // 単一の変数を動的に確保
    int* p = new int(42);
    cout << "*p = " << *p << endl;  // 42
    delete p;  // メモリ解放

    // 配列を動的に確保
    int size = 5;
    int* arr = new int[size]{10, 20, 30, 40, 50};

    for (int i = 0; i < size; i++) {
        cout << arr[i] << " ";
    }
    cout << endl;

    delete[] arr;  // 配列の解放は delete[] を使う

    return 0;
}
```

### よくある間違い

```cpp
// NG: delete忘れ → メモリリーク
void leakExample() {
    int* p = new int(42);
    // deleteを忘れた！この関数を呼ぶたびにメモリが漏れる
}

// NG: 二重delete → 未定義動作（クラッシュの可能性）
int* p = new int(42);
delete p;
delete p;  // 危険！二重解放

// NG: 配列に対して delete（delete[] でない）を使う
int* arr = new int[10];
delete arr;    // NG! → delete[] arr; を使うこと

// NG: 解放後にアクセス（ダングリングポインタ）
int* p2 = new int(42);
delete p2;
cout << *p2;  // 危険！解放済みメモリへのアクセス
```

## 6.8 スマートポインタ（smart pointer）

手動の `new`/`delete` はバグの温床です。モダンC++では**スマートポインタ**を使ってメモリを自動管理します。

```
┌──────────────────────────────────────────────────────┐
│            スマートポインタの種類                      │
├──────────────┬───────────────────────────────────────┤
│ unique_ptr   │ 所有者が1つだけ。最も基本的。           │
│ shared_ptr   │ 複数から共有。参照カウント方式。        │
│ weak_ptr     │ shared_ptrの循環参照を防ぐ。           │
└──────────────┴───────────────────────────────────────┘
```

### unique_ptr（ユニークポインタ）

一つのポインタだけがオブジェクトを「所有」します。所有者がスコープを抜けると自動で `delete` されます。

```cpp
#include <iostream>
#include <memory>  // スマートポインタを使うのに必要
using namespace std;

int main() {
    // unique_ptr の作成（C++14以降は make_unique を使う）
    unique_ptr<int> ptr = make_unique<int>(42);

    cout << "*ptr = " << *ptr << endl;  // 42

    // unique_ptr はコピーできない（所有者は1人だけ）
    // unique_ptr<int> ptr2 = ptr;  // コンパイルエラー！

    // 所有権の移動（move）は可能
    unique_ptr<int> ptr2 = move(ptr);
    cout << "*ptr2 = " << *ptr2 << endl;  // 42
    // ptr はもう何も指していない（nullptr）

    if (ptr == nullptr) {
        cout << "ptrはnullです（所有権が移動した）" << endl;
    }

    // スコープを抜けると自動でメモリが解放される
    // delete 不要！

    return 0;
}
```

### unique_ptr と配列

```cpp
#include <iostream>
#include <memory>
using namespace std;

int main() {
    // 配列の動的確保もスマートポインタで
    auto arr = make_unique<int[]>(5);

    for (int i = 0; i < 5; i++) {
        arr[i] = (i + 1) * 10;
    }

    for (int i = 0; i < 5; i++) {
        cout << arr[i] << " ";  // 10 20 30 40 50
    }
    cout << endl;

    // delete[] 不要！自動で解放される

    return 0;
}
```

### shared_ptr（シェアードポインタ）

複数のポインタが同じオブジェクトを共有できます。最後の `shared_ptr` が消えたときに自動で解放されます。

```cpp
#include <iostream>
#include <memory>
using namespace std;

int main() {
    // shared_ptr の作成
    shared_ptr<int> ptr1 = make_shared<int>(42);
    cout << "参照カウント: " << ptr1.use_count() << endl;  // 1

    {
        // コピーして共有
        shared_ptr<int> ptr2 = ptr1;
        cout << "参照カウント: " << ptr1.use_count() << endl;  // 2

        shared_ptr<int> ptr3 = ptr1;
        cout << "参照カウント: " << ptr1.use_count() << endl;  // 3

        cout << "*ptr1 = " << *ptr1 << endl;  // 42
        cout << "*ptr2 = " << *ptr2 << endl;  // 42
        cout << "*ptr3 = " << *ptr3 << endl;  // 42

    }  // ptr2, ptr3 がスコープを抜ける → 参照カウントが減る

    cout << "参照カウント: " << ptr1.use_count() << endl;  // 1

    // ptr1 がスコープを抜けると参照カウントが0になり、メモリ解放

    return 0;
}
```

### shared_ptr の参照カウントの図解

```
最初: ptr1 → [42]  参照カウント: 1

ptr2 = ptr1 後:
  ptr1 ──→ [42]  参照カウント: 2
  ptr2 ──↗

ptr3 = ptr1 後:
  ptr1 ──→ [42]  参照カウント: 3
  ptr2 ──↗
  ptr3 ──↗

ptr2, ptr3 が消えた後:
  ptr1 ──→ [42]  参照カウント: 1

ptr1 が消えた後:
  参照カウント: 0 → メモリ解放！
```

## 6.9 実践例：学生管理システム

スマートポインタを活用した実践的な例です。

```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <string>
using namespace std;

struct Student {
    string name;
    int age;
    double gpa;

    Student(const string& n, int a, double g)
        : name(n), age(a), gpa(g) {
        cout << name << " を作成しました" << endl;
    }

    ~Student() {
        cout << name << " を解放しました" << endl;
    }

    void display() const {
        cout << "  名前: " << name
             << ", 年齢: " << age
             << ", GPA: " << gpa << endl;
    }
};

int main() {
    // unique_ptr を使った学生管理
    vector<unique_ptr<Student>> students;

    students.push_back(make_unique<Student>("田中太郎", 20, 3.5));
    students.push_back(make_unique<Student>("佐藤花子", 21, 3.8));
    students.push_back(make_unique<Student>("鈴木一郎", 19, 3.2));

    cout << "\n=== 学生一覧 ===" << endl;
    for (const auto& s : students) {
        s->display();
    }

    // GPA が最も高い学生を探す
    const auto& best = *max_element(
        students.begin(), students.end(),
        [](const unique_ptr<Student>& a, const unique_ptr<Student>& b) {
            return a->gpa < b->gpa;
        }
    );

    cout << "\n最優秀学生: " << best->name
         << "（GPA: " << best->gpa << "）" << endl;

    cout << "\n--- プログラム終了 ---" << endl;
    // students がスコープを抜けると、全Studentが自動で解放される

    return 0;
}
```

## 6.10 メモリリークの防止

### メモリリーク（memory leak）とは

確保したメモリを解放し忘れることです。プログラムが長時間実行されると、使えるメモリがどんどん減っていきます。

```
メモリリークの流れ:

  1回目の呼び出し:  [確保] → ポインタ紛失 → 解放できない！
  2回目の呼び出し:  [確保] → ポインタ紛失 → 解放できない！
  3回目の呼び出し:  [確保] → ポインタ紛失 → 解放できない！
  ...
  使用メモリがどんどん増えていく → 最終的にメモリ不足
```

### 防止策のまとめ

```cpp
#include <iostream>
#include <memory>
#include <vector>
using namespace std;

int main() {
    // 方法1: スマートポインタを使う（最も推奨）
    {
        auto ptr = make_unique<int>(42);
        // 自動で解放される
    }

    // 方法2: コンテナを使う（vectorなど）
    {
        vector<int> data(1000);  // 動的メモリを内部で管理
        // 自動で解放される
    }

    // 方法3: RAII（Resource Acquisition Is Initialization）
    // リソースの確保をオブジェクトの初期化に結びつけ、
    // デストラクタで自動解放する

    cout << "全てのメモリが正しく管理されています" << endl;

    return 0;
}
```

### モダンC++でのメモリ管理ルール

```
┌─────────────────────────────────────────────────────────┐
│  メモリ管理のベストプラクティス                          │
├─────────────────────────────────────────────────────────┤
│  1. 生の new/delete は使わない                          │
│  2. 単独所有には unique_ptr を使う                      │
│  3. 共有所有には shared_ptr を使う                      │
│  4. 可能な限り vector などのコンテナを使う              │
│  5. make_unique / make_shared で生成する                │
│  6. 関数の引数にはスマートポインタではなく参照を渡す    │
│  7. 所有権の移動には std::move を使う                   │
└─────────────────────────────────────────────────────────┘
```

## 6.11 章全体のポイントまとめ

```
┌────────────────────────────────────────────────────────────┐
│  第6章のまとめ                                              │
├────────────────────────────────────────────────────────────┤
│  1. ポインタはメモリアドレスを格納する変数                  │
│  2. & でアドレス取得、* で間接参照                         │
│  3. 参照はポインタより安全な「変数の別名」                  │
│  4. nullptr で「何も指さない」状態を表す                   │

│  6. new/delete による手動管理は避ける                      │
│  7. unique_ptr: 唯一の所有者。基本はこれを使う             │
│  8. shared_ptr: 共有所有。参照カウントで管理               │
│  9. RAII の考え方でメモリリークを防止する                  │
│ 10. 実務では生ポインタより参照・スマートポインタを使う     │
└────────────────────────────────────────────────────────────┘
```
