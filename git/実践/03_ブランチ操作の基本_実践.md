# 実践課題03：ブランチ操作の基本 ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第2章（基本操作）、第3章（ブランチとマージ）
> **課題の種類**: ミニプロジェクト
> **学習目標**: ブランチの作成・切り替え・マージ・削除を実践し、並行開発の基本を体験する

---

## 完成イメージ

```
$ git log --oneline --graph --all
*   e5f6a7b (HEAD -> main) Merge branch 'feature/dessert-menu'
|\
| * c3d4e5f (feature/dessert-menu) デザートメニューを追加
| * a1b2c3d デザートセクションの枠を作成
|/
* f4e5d6c ドリンクメニューの作成
* b2c3d4e 初回コミット：README作成
```

---

## 課題の要件

1. メインブランチでプロジェクトの基本ファイル（README.md, menu.txt）を作成する
2. `feature/dessert-menu` ブランチを作成して切り替える
3. そのブランチでデザートメニューを追加する（2回コミット）
4. メインブランチに戻ってマージする
5. マージ後、不要になったブランチを削除する
6. `git log --graph` でブランチの履歴を確認する

---

## ステップガイド

<details>
<summary>ステップ1：メインブランチで初期セットアップ</summary>

```bash
mkdir branch-practice
cd branch-practice
git init

# README を作成
echo "# カフェプロジェクト" > README.md
git add README.md
git commit -m "初回コミット：README作成"

# ドリンクメニューを作成
cat << 'EOF' > menu.txt
===== カフェメニュー =====

【ドリンク】
コーヒー    450円
紅茶        400円
カフェラテ  500円
EOF

git add menu.txt
git commit -m "ドリンクメニューの作成"
```

</details>

<details>
<summary>ステップ2：ブランチを作成して切り替える</summary>

```bash
# 現在のブランチを確認
git branch

# 新しいブランチを作成して切り替え
git switch -c feature/dessert-menu
# または: git checkout -b feature/dessert-menu

# ブランチが切り替わったことを確認
git branch
# 出力例: * feature/dessert-menu
#          main
```

</details>

<details>
<summary>ステップ3：ブランチ上で変更をコミットする</summary>

```bash
# menu.txt にデザートセクションの枠を追加
cat << 'EOF' >> menu.txt

【デザート】
（準備中）
EOF

git add menu.txt
git commit -m "デザートセクションの枠を作成"

# デザートの中身を追加
# 「（準備中）」の行を実際のメニューに書き換える
cat << 'EOF' > menu.txt
===== カフェメニュー =====

【ドリンク】
コーヒー    450円
紅茶        400円
カフェラテ  500円

【デザート】
チーズケーキ  600円
ティラミス    650円
ガトーショコラ 700円
EOF

git add menu.txt
git commit -m "デザートメニューを追加"
```

</details>

<details>
<summary>ステップ4：メインブランチに戻ってマージする</summary>

```bash
# main ブランチに戻る
git switch main
# または: git checkout main

# menu.txt の中身を確認（デザートがないことを確認）
cat menu.txt

# マージ実行
git merge feature/dessert-menu

# menu.txt の中身を再確認（デザートが追加されていることを確認）
cat menu.txt
```

</details>

<details>
<summary>ステップ5：ブランチの削除と履歴確認</summary>

```bash
# マージ済みブランチを削除
git branch -d feature/dessert-menu

# ブランチ一覧を確認（feature/dessert-menu が消えていること）
git branch

# グラフ付き履歴を確認
git log --oneline --graph --all
```

</details>

```bash
# ===== 準備 =====
mkdir branch-practice

git init

# ===== main ブランチでの作業 =====
echo "# カフェプロジェクト" > README.md

紅茶        400円

git add menu.txt

# 確認

# 出力例:
```

<details>
<summary>解答例（改良版 ─ 複数ブランチの並行開発）</summary>

実務では複数の機能を並行して開発することがよくあります。

```bash
# ===== 準備（初心者向けと同じ）=====
mkdir branch-practice
cd branch-practice
git init

echo "# カフェプロジェクト" > README.md
git add README.md
git commit -m "初回コミット：README作成"

cat << 'EOF' > menu.txt
===== カフェメニュー =====

【ドリンク】
コーヒー    450円
紅茶        400円
カフェラテ  500円
EOF

git add menu.txt
git commit -m "ドリンクメニューの作成"

# ===== ブランチ1：デザートメニュー =====
git switch -c feature/dessert-menu

cat << 'EOF' >> menu.txt

【デザート】
チーズケーキ  600円
ティラミス    650円
EOF

git add menu.txt
git commit -m "デザートメニューを追加"

# ===== ブランチ2に切り替え（main から分岐）=====
git switch main
git switch -c feature/lunch-menu

cat << 'EOF' >> menu.txt

【ランチセット】
パスタセット   1200円
サンドイッチセット 980円
EOF

git add menu.txt
git commit -m "ランチメニューを追加"

# ===== main にそれぞれマージ =====
git switch main

# 1つ目のマージ（競合なし）
git merge feature/dessert-menu
echo "--- デザートメニューをマージ完了 ---"

# 2つ目のマージ（同じファイルを編集しているので競合する可能性あり）
git merge feature/lunch-menu
# ※ 競合が発生したら、手動で解決してから git add → git commit

# ===== 後片付け =====
git branch -d feature/dessert-menu
git branch -d feature/lunch-menu

# ===== 確認 =====
git log --oneline --graph --all
git branch  # main だけ残っていること
```

**初心者向けとの違い:**
- 2つのブランチを並行して運用し、実務に近い並行開発を体験
- 同じファイルを別ブランチで変更した場合のコンフリクト（conflict）発生を体験できる
- コンフリクト解消の練習にもなる

</details>
