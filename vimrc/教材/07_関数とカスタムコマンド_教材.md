# 第7章：関数とカスタムコマンド

## この章のゴール

- Vim script の変数とデータ型を理解する
- 条件分岐を使って設定を動的に切り替えられるようになる
- 自分で関数を定義して vimrc を効率化できるようになる
- カスタムコマンドを作成して日常のエディタ操作を自動化できるようになる
- script-local スコープを理解し、安全な設定ファイルを書けるようになる

## 7.1 Vim script の変数

Vim script では `let` コマンドを使って変数に値を代入します。

### 基本的な変数代入

```vim
" 文字列の代入
let g:my_name = "太郎"

" 数値の代入
let g:tab_size = 4

" 変数の参照
echo g:my_name       " => 太郎
echo g:tab_size      " => 4

" 変数の削除
unlet g:my_name
```

### データ型

Vim script には以下のデータ型があります。

```
+----------+----------------------------+------------------+
| 型       | 説明                       | 例               |
+----------+----------------------------+------------------+
| Number   | 整数                       | 42, -3, 0xFF     |
| Float    | 浮動小数点数               | 3.14, 1.0e10     |
| String   | 文字列                     | "hello", 'world' |
| List     | 順序付きリスト             | [1, 2, 3]        |
| Dict     | キーと値のペア             | {'a': 1, 'b': 2} |
| Funcref  | 関数への参照               | function('strlen')|
+----------+----------------------------+------------------+
```

```vim
" Number（整数）
let g:count = 42
let g:hex = 0xFF        " 16進数（255）
let g:oct = 0o77        " 8進数（63）

" Float（浮動小数点数）
let g:pi = 3.14159
let g:big = 1.0e6       " 1000000.0

" String（文字列）
let g:msg1 = "ダブルクォート：特殊文字が使える \n \t"
let g:msg2 = 'シングルクォート：そのまま表示される'

" List（リスト）
let g:fruits = ['apple', 'banana', 'cherry']
echo g:fruits[0]        " => apple
echo len(g:fruits)      " => 3
call add(g:fruits, 'date')

" Dictionary（辞書）
let g:user = {'name': '太郎', 'age': 25}
echo g:user['name']     " => 太郎
echo g:user.age         " => 25（ドット記法でもアクセス可能）
```

### 変数のスコープ

Vim script の変数にはスコープ（有効範囲）があり、プレフィックスで区別します。

```
+-----------+-------------------+--------------------------------------+
| プレフィ  | スコープ          | 説明                                 |
| ックス    |                   |                                      |
+-----------+-------------------+--------------------------------------+
| g:        | グローバル        | どこからでもアクセス可能             |
| s:        | スクリプトローカル| 定義されたファイル内のみ有効         |
| l:        | 関数ローカル      | 関数内でのみ有効                     |
| a:        | 引数              | 関数の引数（読み取り専用）           |
| b:        | バッファローカル  | 現在のバッファでのみ有効             |
| w:        | ウィンドウローカル| 現在のウィンドウでのみ有効           |
| t:        | タブローカル      | 現在のタブでのみ有効                 |
| v:        | Vim定義変数       | Vimが提供する読み取り専用変数        |
+-----------+-------------------+--------------------------------------+
```

```vim
" グローバル変数：vimrc 全体で使える
let g:mapleader = ','

" バッファローカル変数：バッファごとに異なる値を持てる
let b:is_modified = 1

" Vim定義変数の例
echo v:version       " Vimのバージョン（例：900）
echo v:true          " 真（1）
echo v:false         " 偽（0）
```

> **よくある間違い**: 関数内でスコープ指定なしに `let x = 1` と書くと、`l:x`（ローカル変数）として扱われます。グローバルに使いたい場合は必ず `g:` を付けましょう。

### ポイントまとめ

- `let` で変数を代入し、`unlet` で削除する
- データ型は Number, Float, String, List, Dict, Funcref の6種類
- スコープのプレフィックス（`g:`, `s:`, `l:` など）を正しく使い分けることが重要
- `v:` 変数は Vim が提供する特別な変数（読み取り専用が多い）

## 7.2 条件分岐

Vim script でも `if` / `elseif` / `else` / `endif` で条件分岐ができます。

### 基本構文

```vim
if 条件式
  " 条件が真のとき実行
elseif 別の条件式
  " 別の条件が真のとき実行
else
  " どの条件も満たさないとき実行
endif
```

### 実用例：OS に応じた設定

```vim
if has('mac')
  set clipboard=unnamed
elseif has('unix')
  set clipboard=unnamedplus
elseif has('win32')
  set clipboard=unnamed
endif
```

### 比較演算子

Vim script の比較演算子には、大文字・小文字の扱いが異なる3種類があります。

```
+--------+----------------------------------------------+
| 演算子 | 意味                                         |
+--------+----------------------------------------------+
| ==#    | 等しい（大文字・小文字を区別する）            |
| ==?    | 等しい（大文字・小文字を区別しない）          |
| ==     | ignorecase 設定に依存（非推奨）               |
+--------+----------------------------------------------+
| !=#    | 等しくない（大文字・小文字を区別する）        |
| !=?    | 等しくない（大文字・小文字を区別しない）      |
+--------+----------------------------------------------+
| =~#    | 正規表現マッチ（大文字・小文字を区別する）    |
| =~?    | 正規表現マッチ（大文字・小文字を区別しない）  |
| !~#    | 正規表現非マッチ（大文字・小文字を区別する）  |
+--------+----------------------------------------------+
```

```vim
" 大文字・小文字を区別して比較（推奨）
if l:filetype ==# 'python'
  setlocal expandtab shiftwidth=4
endif

" 大文字・小文字を無視して比較
if l:answer ==? 'yes'
  echo '了解しました'
endif

" 正規表現マッチ
if l:filename =~# '\.py$'
  echo 'Pythonファイルです'
endif
```

> **よくある間違い**: `==` は `&ignorecase` の設定によって動作が変わります。常に `==#` または `==?` を使うことで、予期しない動作を防げます。これは Vim script の重要なベストプラクティスです。

### 機能チェック関数

```vim
" has() - Vimの機能が利用可能かチェック
if has('clipboard')
  set clipboard=unnamedplus
endif

if has('terminal')
  " ターミナル機能が使える
  nnoremap <Leader>t :terminal<CR>
endif

if has('python3')
  echo 'Python3が利用可能です'
endif

" exists() - 変数・関数・コマンドの存在チェック
if exists('g:loaded_my_plugin')
  " プラグインが既に読み込まれている
  finish
endif
let g:loaded_my_plugin = 1

" exists() で関数の存在をチェック
if exists('*MyFunction')
  call MyFunction()
endif

" exists() でコマンドの存在をチェック
if exists(':NERDTree')
  nnoremap <Leader>e :NERDTreeToggle<CR>
endif
```

### ポイントまとめ

- 条件分岐は `if` / `elseif` / `else` / `endif` で記述する
- 比較演算子は `==#`（大文字小文字区別）を基本的に使うのが安全
- `has()` で Vim の機能やプラットフォームを判定できる
- `exists()` で変数・関数・コマンドの存在を確認できる

## 7.3 関数の定義

Vim script では `function` / `endfunction` で独自の関数を定義できます。

### 基本構文

```vim
" function! の ! は同名関数の上書きを許可する
function! MyGreeting(name)
  echo 'こんにちは、' . a:name . 'さん！'
endfunction

" 呼び出し
call MyGreeting('太郎')
" => こんにちは、太郎さん！
```

> **注意**: ユーザー定義関数の名前は**大文字で始める**必要があります。小文字で始めるとエラーになります（script-local 関数の `s:` 付きは例外）。

### 引数の扱い

関数内で引数にアクセスするには `a:` プレフィックスを使います。

```vim
function! AddNumbers(x, y)
  " 引数は a: でアクセスする（読み取り専用）
  let l:result = a:x + a:y
  return l:result
endfunction

echo AddNumbers(3, 5)
" => 8
```

### 可変長引数

```vim
function! PrintAll(first, ...)
  " a:first - 最初の引数
  " a:0     - 可変長引数の数
  " a:1     - 可変長引数の1番目
  " a:000   - 可変長引数のリスト
  echo '最初の引数: ' . a:first
  echo '残りの引数の数: ' . a:0

  for l:item in a:000
    echo '  - ' . l:item
  endfor
endfunction

call PrintAll('りんご', 'バナナ', 'みかん')
" => 最初の引数: りんご
" => 残りの引数の数: 2
" =>   - バナナ
" =>   - みかん
```

### range 指定

`range` を付けると、ビジュアル選択範囲で関数が呼ばれたとき、行ごとではなく一度だけ実行されます。

```vim
function! CountSelectedLines() range
  let l:count = a:lastline - a:firstline + 1
  echo l:count . '行が選択されています'
endfunction

" ビジュアル選択後に呼び出す
" :'<,'>call CountSelectedLines()
```

### abort 指定

`abort` を付けると、関数内でエラーが発生した時点で関数の実行を停止します。

```vim
function! SafeFunction() abort
  " エラーが発生したらここで関数が停止する
  let l:data = readfile('/存在しないファイル')
  " ↑ エラー発生 → 以降の行は実行されない
  echo 'この行は実行されません'
endfunction
```

### 戻り値

```vim
function! GetFileSize(path) abort
  if !filereadable(a:path)
    return -1
  endif
  let l:lines = readfile(a:path)
  return len(l:lines)
endfunction

let g:size = GetFileSize(expand('~/.vimrc'))
if g:size >= 0
  echo 'vimrc は ' . g:size . ' 行です'
else
  echo 'ファイルが見つかりません'
endif
```

### ポイントまとめ

- 関数は `function! 関数名(引数) ... endfunction` で定義する
- `!` を付けると同名の関数を上書きできる（vimrc の再読み込み時に必須）
- 引数は `a:` プレフィックスでアクセスする（読み取り専用）
- `a:000` で可変長引数をリストとして受け取れる
- `range` で選択範囲対応、`abort` でエラー時の即座停止ができる

## 7.4 カスタムコマンドの定義

`command` で独自の Ex コマンドを定義できます。`:MyCommand` のように呼び出せるようになります。

### 基本構文

```vim
" コマンド名は大文字で始める
command! MyHello echo 'Hello, Vim!'

" 使い方
" :MyHello
" => Hello, Vim!
```

### -nargs オプション（引数の数）

```
+------------+-------------------------------+
| オプション | 意味                          |
+------------+-------------------------------+
| -nargs=0   | 引数なし（デフォルト）        |
| -nargs=1   | 引数1つ（必須）               |
| -nargs=*   | 引数0個以上                   |
| -nargs=+   | 引数1個以上                   |
| -nargs=?   | 引数0個または1個              |
+------------+-------------------------------+
```

```vim
" 引数を受け取るコマンド
command! -nargs=1 SayHello echo 'Hello, ' . <args> . '!'

" 使い方
" :SayHello "太郎"
" => Hello, 太郎!

" 複数引数に対応
command! -nargs=+ MyEcho echo <args>

" 関数と組み合わせる（<q-args> で引数を文字列として渡す）
function! s:Greet(name) abort
  echo 'こんにちは、' . a:name . 'さん！'
endfunction

command! -nargs=1 Greet call s:Greet(<q-args>)

" 使い方
" :Greet 太郎
" => こんにちは、太郎さん！
```

### -range オプション（範囲指定）

```vim
function! s:CountLines(line1, line2) abort
  echo (a:line2 - a:line1 + 1) . '行が選択されています'
endfunction

" <line1> と <line2> で範囲を受け取る
command! -range CountLines call s:CountLines(<line1>, <line2>)

" 使い方
" :'<,'>CountLines   （ビジュアル選択してから実行）
" :10,20CountLines   （10〜20行目を指定）
```

### -bang オプション（! 付き呼び出し）

```vim
function! s:CloseBuffer(bang) abort
  if a:bang ==# '!'
    " 保存せずに閉じる
    bdelete!
  else
    " 変更があれば確認
    bdelete
  endif
endfunction

command! -bang CloseBuffer call s:CloseBuffer('<bang>')

" 使い方
" :CloseBuffer    （変更があれば警告）
" :CloseBuffer!   （強制的に閉じる）
```

### -complete オプション（補完）

```vim
" ファイル名補完
command! -nargs=1 -complete=file EditFile edit <args>

" カスタム補完
function! s:MyComplete(ArgLead, CmdLine, CursorPos) abort
  return filter(['debug', 'info', 'warning', 'error'],
        \ 'v:val =~# "^" . a:ArgLead')
endfunction

command! -nargs=1 -complete=customlist,s:MyComplete
      \ SetLogLevel echo 'ログレベル: ' . <q-args>

" 使い方
" :SetLogLevel d<Tab>   → debug に補完される
" :SetLogLevel <Tab>    → debug, info, warning, error が候補に出る
```

### ポイントまとめ

- `command!` でカスタム Ex コマンドを定義する（`!` で上書き許可）
- `-nargs` で引数の数を指定し、`<args>` や `<q-args>` で参照する
- `-range` で範囲指定に対応、`-bang` で `!` 付き呼び出しに対応
- `-complete` で Tab 補完を追加できる
- 関数と組み合わせることで高機能なコマンドが作れる

## 7.5 実用的な関数例

ここでは、実際の vimrc でよく使われる関数を紹介します。

### 末尾空白（trailing whitespace）の削除

```vim
function! s:StripTrailingWhitespace() abort
  " カーソル位置と検索パターンを保存
  let l:save_cursor = getpos('.')
  let l:save_search = @/

  " 末尾の空白を削除（e フラグでマッチしなくてもエラーにしない）
  %s/\s\+$//e

  " カーソル位置と検索パターンを復元
  call setpos('.', l:save_cursor)
  let @/ = l:save_search
endfunction

" コマンドとして定義
command! StripTrailingWhitespace call s:StripTrailingWhitespace()

" 保存時に自動実行（任意）
" autocmd BufWritePre * call s:StripTrailingWhitespace()
```

### トグル関数（行番号表示の切替）

```vim
function! s:ToggleNumber() abort
  if &number
    set nonumber norelativenumber
    echo '行番号: OFF'
  else
    set number relativenumber
    echo '行番号: ON (相対)'
  endif
endfunction

command! ToggleNumber call s:ToggleNumber()
nnoremap <Leader>n :ToggleNumber<CR>
```

### トグル関数（折り返しの切替）

```vim
function! s:ToggleWrap() abort
  if &wrap
    set nowrap
    echo '折り返し: OFF'
  else
    set wrap
    echo '折り返し: ON'
  endif
endfunction

command! ToggleWrap call s:ToggleWrap()
nnoremap <Leader>w :ToggleWrap<CR>
```

### ステータスライン用関数

```vim
" 現在のGitブランチ名を取得する関数
function! g:GitBranch() abort
  let l:branch = system('git rev-parse --abbrev-ref HEAD 2>/dev/null')
  if v:shell_error
    return ''
  endif
  return ' ' . substitute(l:branch, '\n', '', 'g') . ' '
endfunction

" 文字コードとファイル形式を表示する関数
function! g:FileInfo() abort
  let l:enc = &fileencoding !=# '' ? &fileencoding : &encoding
  let l:ff = &fileformat
  return l:enc . ' | ' . l:ff
endfunction

" ステータスラインに関数の戻り値を表示
set statusline=%f                          " ファイル名
set statusline+=\ %m%r                     " 変更フラグ・読み取り専用フラグ
set statusline+=%=                         " 左右の区切り
set statusline+=%{g:GitBranch()}           " Gitブランチ
set statusline+=\ %{g:FileInfo()}          " 文字コード・ファイル形式
set statusline+=\ %l:%c                    " 行:列
```

### ファイルタイプに応じた設定関数

```vim
function! s:SetupPython() abort
  setlocal expandtab
  setlocal shiftwidth=4
  setlocal softtabstop=4
  setlocal textwidth=79
  setlocal colorcolumn=80
endfunction

function! s:SetupJavaScript() abort
  setlocal expandtab
  setlocal shiftwidth=2
  setlocal softtabstop=2
endfunction

augroup MyFileTypeSettings
  autocmd!
  autocmd FileType python call s:SetupPython()
  autocmd FileType javascript,typescript call s:SetupJavaScript()
augroup END
```

### ポイントまとめ

- 末尾空白削除はカーソル位置と検索パターンの保存・復元がポイント
- トグル関数では `&option` で現在のオプション値を取得できる
- ステータスライン関数は `%{関数名()}` で呼び出す
- `autocmd` と関数を組み合わせるとファイルタイプ別の設定が整理しやすい

## 7.6 script-local スコープ

`s:` プレフィックスを付けた関数や変数は、定義されたスクリプトファイル内でのみ有効です。他のプラグインや設定と名前が衝突するのを防ぎます。

### s: プレフィックスの基本

```vim
" このファイル内でのみ有効な変数
let s:counter = 0

" このファイル内でのみ有効な関数
function! s:Increment() abort
  let s:counter += 1
  echo 'カウント: ' . s:counter
endfunction

" 同じファイル内から呼び出せる
call s:Increment()   " => カウント: 1
call s:Increment()   " => カウント: 2
```

### `<SID>` の使い方

マッピングから `s:` 関数を呼び出すには `<SID>` を使います。

```vim
function! s:ToggleSpell() abort
  if &spell
    set nospell
    echo 'スペルチェック: OFF'
  else
    set spell spelllang=en_us,cjk
    echo 'スペルチェック: ON'
  endif
endfunction

" マッピングから s: 関数を呼び出す場合は <SID> を使う
nnoremap <Leader>s :call <SID>ToggleSpell()<CR>
```

### なぜ `<SID>` が必要なのか？

```
┌──────────────────────────────────────────────┐
│  vimrc ファイル内                             │
│                                              │
│  function! s:MyFunc()    ← s: で定義         │
│    ...                                       │
│  endfunction                                 │
│                                              │
│  " 直接呼び出し → OK                         │
│  call s:MyFunc()                             │
│                                              │
│  " マッピング経由 → <SID> が必要             │
│  nnoremap <key> :call <SID>MyFunc()<CR>      │
│                                              │
│  ※ マッピングはグローバルに登録されるため、   │
│    s: のままでは関数を見つけられない。         │
│    <SID> がスクリプト固有のIDに変換される。    │
└──────────────────────────────────────────────┘
```

`<SID>` は Vim が内部的にスクリプト固有の番号（例: `<SNR>42_`）に変換します。これにより、マッピングが正しいスクリプトの関数を呼び出せるようになります。

### グローバル関数 vs script-local 関数

```vim
" グローバル関数（他のスクリプトからも呼べる）
function! g:MyGlobalFunc() abort
  echo 'グローバル関数です'
endfunction

" script-local 関数（このファイル内でのみ呼べる）
function! s:MyLocalFunc() abort
  echo 'ローカル関数です'
endfunction

" コマンド経由なら s: 関数も外部から利用可能にできる
command! MyCommand call s:MyLocalFunc()
```

> **ベストプラクティス**: vimrc の関数は基本的に `s:` で定義しましょう。グローバル関数はステータスラインの `%{g:Func()}` のように、外部から参照が必要な場合のみ使います。

### ポイントまとめ

- `s:` プレフィックスでスクリプトローカルな関数・変数を定義できる
- マッピングから `s:` 関数を呼ぶには `<SID>` を使う
- `<SID>` は内部的に `<SNR>番号_` に変換される
- 名前の衝突を防ぐため、vimrc の関数は基本的に `s:` で定義するのが推奨

| 引数 | `a:` でアクセス、`a:000` で可変長引数リスト |
| カスタムコマンド | `command!` で定義、`-nargs` `-range` `-bang` `-complete` で拡張 |
| script-local | `s:` で名前衝突を防止、マッピングには `<SID>` を使う |
