## 基本問題

### 問題1：要素の表示を待機する

「データを読み込む」ボタンをクリックすると、2秒後にデータテーブルが表示されるページがあります。テーブルが表示されるまで待機し、内容を検証するテストを書いてください。

**要件：**
- ボタンをクリック後、テーブルが表示されるまで待機する
- テーブルのヘッダーに「商品名」「価格」「在庫」が含まれることを確認する
- テーブルの行が3行以上あることを確認する

**期待される結果：**
```
テーブルが表示され、ヘッダーと行数が期待通りである
```

<details>
<summary>ヒント</summary>

Playwrightの `expect()` アサーションは自動待機（auto-waiting）が組み込まれているため、明示的な `waitFor()` を呼ばなくても要素が表示されるまで待ちます。タイムアウトはデフォルトで5秒です。

```typescript
// 自動待機が効くので、要素が表示されるまで待ってくれる
await expect(page.getByRole('table')).toBeVisible();
```

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('データテーブルが表示されるまで待機する', async ({ page }) => {
  await page.goto('https://example.com/lazy-table');

  // ========================================
  // ボタンをクリックしてデータ読み込みを開始
  // ========================================
  await page.getByRole('button', { name: 'データを読み込む' }).click();

  // ========================================
  // Playwrightの自動待機：toBeVisible() は要素が
  // 表示されるまで自動的にリトライする（デフォルト5秒）
  // ========================================
  const table = page.getByRole('table');
  await expect(table).toBeVisible();

  // ========================================
  // テーブルヘッダーの検証
  // ========================================
  await expect(page.getByRole('columnheader', { name: '商品名' })).toBeVisible();
  await expect(page.getByRole('columnheader', { name: '価格' })).toBeVisible();
  await expect(page.getByRole('columnheader', { name: '在庫' })).toBeVisible();

  // ========================================
  // テーブル行数の検証
  // tbody内のrowを数える（ヘッダー行を除く）
  // ========================================
  const rows = page.locator('tbody tr');
  await expect(rows).toHaveCount(3);  // ちょうど3行の場合
  // 3行以上の場合はこちら：
  // const count = await rows.count();
  // expect(count).toBeGreaterThanOrEqual(3);
});
```

**ポイント：**
- Playwrightのアサーション（`expect`）は自動でリトライするため、`sleep` や `setTimeout` は不要です
- `toHaveCount()` も自動待機が効きます
- 要素が表示されるまでの時間が長い場合は `{ timeout: 10000 }` のようにタイムアウトを調整できます

</details>

### 問題2：API レスポンスの待機

ページを開くと `/api/users` にリクエストが飛び、レスポンスを受け取ってからユーザー一覧が表示されるSPAがあります。APIレスポンスを待機してからテストするコードを書いてください。

**要件：**
- `/api/users` へのレスポンスを待機する
- レスポンスのステータスコードが200であることを確認する
- レスポンスのJSONにユーザーデータが含まれていることを確認する
- 画面にユーザーが表示されていることを確認する

**期待される結果：**
```
APIレスポンスのステータス: 200
レスポンスJSON: ユーザーデータの配列
画面: ユーザー名が表示されている
```

<details>
<summary>ヒント</summary>

`page.waitForResponse()` でURLパターンに一致するレスポンスを待つことができます。

```typescript
const response = await page.waitForResponse('**/api/users');
```

URLパターンにはグロブ（`**`）や正規表現が使えます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('APIレスポンスを待機してユーザー一覧を検証する', async ({ page }) => {
  // ========================================
  // waitForResponse でAPIレスポンスを待ちつつページを開く
  // Promise.all で並行して待機する
  // ========================================
  const [response] = await Promise.all([
    page.waitForResponse('**/api/users'),
    page.goto('https://example.com/users'),
  ]);

  // ========================================
  // レスポンスのステータスコードを検証
  // ========================================
  expect(response.status()).toBe(200);

  // ========================================
  // レスポンスのJSONデータを検証
  // ========================================
  const data = await response.json();
  expect(data).toBeInstanceOf(Array);
  expect(data.length).toBeGreaterThan(0);
  expect(data[0]).toHaveProperty('name');

  // ========================================
  // 画面上の表示を検証
  // ========================================
  await expect(page.getByRole('listitem').first()).toBeVisible();
});
```

**ポイント：**
- `waitForResponse()` に渡すURLパターンは部分一致でOKです（`**/api/users` でパスの末尾に一致）
- `Promise.all` を使って「レスポンスの待機」と「ページ遷移」を同時に行うのが定石です
- レスポンスの `json()` メソッドでJSONパース済みのデータを取得できます

</details>

### 問題3：明示的な待機（waitFor）

ローディングスピナーが表示され、データ読み込み完了後にスピナーが消えてコンテンツが表示されるページをテストしてください。

**要件：**
- ページを開いた直後にローディングスピナーが表示されていることを確認する
- スピナーが消えるまで待機する
- コンテンツが表示されたことを確認する

**期待される結果：**
```
1. ローディングスピナーが表示される
2. スピナーが消える
3. メインコンテンツが表示される
```

<details>
<summary>ヒント</summary>

`waitFor({ state: 'hidden' })` で要素が非表示になるまで待機できます。

```typescript
await page.locator('.spinner').waitFor({ state: 'hidden' });
```

`state` には `'visible'` / `'hidden'` / `'attached'` / `'detached'` が指定できます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ローディング完了後にコンテンツが表示される', async ({ page }) => {
  await page.goto('https://example.com/dashboard');

  // ========================================
  // ローディングスピナーが表示されていることを確認
  // ========================================
  const spinner = page.locator('.loading-spinner');
  await expect(spinner).toBeVisible();

  // ========================================
  // waitFor でスピナーが非表示になるまで待機
  // state: 'hidden' → 要素がDOMにあっても非表示になれば通過
  // state: 'detached' → 要素がDOMから完全に削除されれば通過
  // ========================================
  await spinner.waitFor({ state: 'hidden', timeout: 10000 });

  // ========================================
  // メインコンテンツの表示を確認
  // ========================================
  await expect(page.getByRole('main')).toBeVisible();
  await expect(page.getByRole('heading', { name: 'ダッシュボード' })).toBeVisible();
});
```

**ポイント：**
- `waitFor()` は要素が指定した状態になるまで待機するメソッドです
- `state: 'hidden'` はCSS非表示（`display: none` や `visibility: hidden`）も対象です
- `state: 'detached'` はDOMから完全に削除された場合のみ通過します
- 自動待機が効く `expect().not.toBeVisible()` でも同様の待機ができます

</details>

### 問題4：page.route() でAPIレスポンスをモックする

ユーザー一覧を表示するページで、実際のAPIを呼ばずにモックデータを返すテストを書いてください。

**要件：**
- `/api/users` へのリクエストをインターセプトしてモックデータを返す
- モックデータに3人のユーザー情報を含める
- 画面に3人のユーザー名が表示されることを確認する

**期待される結果：**
```
画面にモックで指定した3人のユーザー名が正しく表示される
```

<details>
<summary>ヒント</summary>

`page.route()` でURLパターンに一致するリクエストをインターセプトし、`route.fulfill()` でモックレスポンスを返せます。

```typescript
await page.route('**/api/users', async (route) => {
  await route.fulfill({
    status: 200,
    contentType: 'application/json',
    body: JSON.stringify([...]),
  });
});
```

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('APIをモックしてユーザー一覧を表示する', async ({ page }) => {
  // ========================================
  // page.route() でAPIをインターセプト
  // ページ遷移の「前に」設定する必要がある
  // ========================================
  const mockUsers = [
    { id: 1, name: '田中太郎', email: 'tanaka@example.com', role: 'admin' },
    { id: 2, name: '鈴木花子', email: 'suzuki@example.com', role: 'user' },
    { id: 3, name: '佐藤次郎', email: 'sato@example.com', role: 'user' },
  ];

  await page.route('**/api/users', async (route) => {
    // ========================================
    // route.fulfill() でモックレスポンスを返す
    // 実際のサーバーにはリクエストが飛ばない
    // ========================================
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify(mockUsers),
    });
  });

  // ========================================
  // ページ遷移（この時点でAPIモックが有効）
  // ========================================
  await page.goto('https://example.com/users');

  // ========================================
  // モックデータが画面に反映されていることを検証
  // ========================================
  await expect(page.getByText('田中太郎')).toBeVisible();
  await expect(page.getByText('鈴木花子')).toBeVisible();
  await expect(page.getByText('佐藤次郎')).toBeVisible();

  // ユーザーが3人であることも検証
  const userItems = page.getByTestId('user-item');
  await expect(userItems).toHaveCount(3);
});
```

**ポイント：**
- `page.route()` はページ遷移の前に設定する必要があります
- モックを使うと外部サービスに依存しないテストが書けます
- テスト終了時に `await page.unroute('**/api/users')` でモックを解除できます

</details>

### 問題5：ローディング状態のテスト

APIからデータを取得する間にローディング表示が出て、取得完了後にデータが表示されるUIをテストしてください。`page.route()` を使ってAPIレスポンスを遅延させ、ローディング状態を検証します。

**要件：**
- APIレスポンスを意図的に2秒遅延させる
- ローディング中に「読み込み中...」と表示されることを確認する
- データ取得後に「読み込み中...」が消えてデータが表示されることを確認する

**期待される結果：**
```
1. 「読み込み中...」が表示される
2. 2秒後にデータが表示され、「読み込み中...」が消える
```

<details>
<summary>ヒント</summary>

`page.route()` 内で `setTimeout` を使ってレスポンスを遅延させます。あるいは `route.fetch()` で実際のリクエストを行ってからレスポンスを返す方法もあります。

```typescript
await page.route('**/api/data', async (route) => {
  await new Promise(resolve => setTimeout(resolve, 2000));
  await route.fulfill({ ... });
});
```

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ローディング状態が正しく表示される', async ({ page }) => {
  // ========================================
  // APIを遅延させるモックを設定
  // setTimeout でレスポンスを2秒遅らせる
  // ========================================
  await page.route('**/api/products', async (route) => {
    // 2秒間遅延させる
    await new Promise(resolve => setTimeout(resolve, 2000));

    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify([
        { id: 1, name: 'ノートPC', price: 120000 },
        { id: 2, name: 'マウス', price: 3000 },
      ]),
    });
  });

  await page.goto('https://example.com/products');

  // ========================================
  // ローディング中の表示を検証
  // API応答前なので「読み込み中...」が表示されているはず
  // ========================================
  await expect(page.getByText('読み込み中...')).toBeVisible();

  // ========================================
  // データ表示後の検証
  // 自動待機でデータが表示されるまで待つ
  // ========================================
  await expect(page.getByText('ノートPC')).toBeVisible({ timeout: 5000 });
  await expect(page.getByText('読み込み中...')).not.toBeVisible();
});
```

**ポイント：**
- APIレスポンスを遅延させることで、ローディング状態を確実にテストできます
- 実際のE2Eテストでは、ローディング表示のテストは重要なUX検証ポイントです
- `timeout` を指定して待機時間を明示することで、テストの意図が分かりやすくなります

</details>

### 問題6：リクエストの検証（waitForRequest）

フォーム送信時にサーバーに送られるリクエストの内容（HTTPメソッド、ボディ）を検証するテストを書いてください。

**要件：**
- お問い合わせフォームに値を入力して送信する
- `/api/contact` へのPOSTリクエストが送られたことを確認する
- リクエストボディに入力した値（名前、メール、メッセージ）が含まれていることを確認する

**期待される結果：**
```
POSTリクエストが/api/contactに送信され、
リクエストボディに入力したフォームデータが含まれている
```

<details>
<summary>ヒント</summary>

`page.waitForRequest()` でリクエストを待ち受け、`request.method()` と `request.postDataJSON()` で内容を検証します。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('フォーム送信のリクエスト内容を検証する', async ({ page }) => {
  await page.goto('https://example.com/contact');

  // ========================================
  // フォームに値を入力
  // ========================================
  await page.getByLabel('名前').fill('テスト太郎');
  await page.getByLabel('メールアドレス').fill('test@example.com');
  await page.getByLabel('メッセージ').fill('テストメッセージです。');

  // ========================================
  // APIモックを設定（リクエストは受け取るが、レスポンスは返す）
  // ========================================
  await page.route('**/api/contact', async (route) => {
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify({ success: true }),
    });
  });

  // ========================================
  // リクエストの待機とフォーム送信を同時に実行
  // ========================================
  const [request] = await Promise.all([
    page.waitForRequest('**/api/contact'),
    page.getByRole('button', { name: '送信' }).click(),
  ]);

  // ========================================
  // リクエストの内容を検証
  // ========================================
  // HTTPメソッドがPOSTであること
  expect(request.method()).toBe('POST');

  // リクエストボディの内容を検証
  const postData = request.postDataJSON();
  expect(postData).toEqual({
    name: 'テスト太郎',
    email: 'test@example.com',
    message: 'テストメッセージです。',
  });
});
```

**ポイント：**
- `waitForRequest()` と `waitForResponse()` はどちらもURLパターン、正規表現、または関数を引数に取れます
- `request.postDataJSON()` はリクエストボディをJSONとしてパースして返します
- `request.postData()` は文字列として返すため、フォームエンコードのデータも扱えます

</details>

### 問題7：エラーレスポンスのモックとエラー画面のテスト

APIがエラーを返した場合のUIの振る舞いをテストしてください。さまざまなHTTPステータスコードに対するエラーハンドリングを検証します。

**要件：**
- 500エラーをモックし、「サーバーエラーが発生しました」と表示されることを確認する
- 404エラーをモックし、「データが見つかりませんでした」と表示されることを確認する
- ネットワークエラーをモックし、「ネットワークエラーが発生しました」と表示されることを確認する

**期待される結果：**
```
500エラー: 「サーバーエラーが発生しました」が表示される
404エラー: 「データが見つかりませんでした」が表示される
ネットワークエラー: 「ネットワークエラーが発生しました」が表示される
```

<details>
<summary>ヒント</summary>

ステータスコードの変更は `route.fulfill({ status: 500 })` で行います。ネットワークエラーのシミュレーションは `route.abort()` を使います。

```typescript
// ネットワークエラーをシミュレート
await page.route('**/api/data', (route) => route.abort());
```

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test.describe('APIエラー時のUI表示', () => {

  test('500エラー時にサーバーエラーメッセージが表示される', async ({ page }) => {
    // ========================================
    // 500 Internal Server Error をモック
    // ========================================
    await page.route('**/api/products', async (route) => {
      await route.fulfill({
        status: 500,
        contentType: 'application/json',
        body: JSON.stringify({ error: 'Internal Server Error' }),
      });
    });

    await page.goto('https://example.com/products');
    await expect(page.getByText('サーバーエラーが発生しました')).toBeVisible();
  });

  test('404エラー時にデータ不在メッセージが表示される', async ({ page }) => {
    // ========================================
    // 404 Not Found をモック
    // ========================================
    await page.route('**/api/products', async (route) => {
      await route.fulfill({
        status: 404,
        contentType: 'application/json',
        body: JSON.stringify({ error: 'Not Found' }),
      });
    });

    await page.goto('https://example.com/products');
    await expect(page.getByText('データが見つかりませんでした')).toBeVisible();
  });

  test('ネットワークエラー時にエラーメッセージが表示される', async ({ page }) => {
    // ========================================
    // route.abort() でネットワーク切断をシミュレート
    // 'failed' はネットワーク障害を表す
    // ========================================
    await page.route('**/api/products', async (route) => {
      await route.abort('failed');
    });

    await page.goto('https://example.com/products');
    await expect(page.getByText('ネットワークエラーが発生しました')).toBeVisible();
  });
});
```

**ポイント：**
- `route.abort()` の引数には `'failed'`、`'timedout'`、`'aborted'` などのエラータイプを指定できます
- エラーハンドリングのテストはE2Eテストで非常に重要です（ハッピーパスだけでは不十分）
- 各エラーケースを `test.describe()` でグループ化すると読みやすくなります

</details>

### 問題8：ネットワーク遅延のシミュレーションとタイムアウト処理

APIレスポンスが極端に遅い場合に、アプリケーション側のタイムアウト処理が正しく動作するかテストしてください。

**要件：**
- APIレスポンスを10秒遅延させるモックを設定する
- アプリケーション側の5秒タイムアウトが発動し、「タイムアウトしました。再試行してください。」と表示されることを確認する
- 「再試行」ボタンが表示されることを確認する

**期待される結果：**
```
5秒後にタイムアウトメッセージが表示され、
再試行ボタンが使用可能な状態で表示される
```

<details>
<summary>ヒント</summary>

長い遅延をモックで設定し、アプリケーション側のタイムアウトが先に発動するようにします。テスト自体のタイムアウトも考慮して `test.setTimeout()` で延長しましょう。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('タイムアウト時にエラー表示と再試行ボタンが表示される', async ({ page }) => {
  // ========================================
  // テスト自体のタイムアウトを延長（デフォルト30秒→60秒）
  // アプリの5秒タイムアウト + 余裕を見て設定
  // ========================================
  test.setTimeout(60000);

  // ========================================
  // APIレスポンスを10秒遅延させるモック
  // アプリ側のタイムアウト（5秒）より長くする
  // ========================================
  await page.route('**/api/data', async (route) => {
    await new Promise(resolve => setTimeout(resolve, 10000));
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify({ data: 'test' }),
    });
  });

  await page.goto('https://example.com/slow-page');

  // ========================================
  // タイムアウトメッセージが表示されるまで待機
  // アプリ側の5秒タイムアウト + バッファで8秒待つ
  // ========================================
  await expect(
    page.getByText('タイムアウトしました。再試行してください。')
  ).toBeVisible({ timeout: 8000 });

  // ========================================
  // 再試行ボタンが表示されていることを確認
  // ========================================
  const retryButton = page.getByRole('button', { name: '再試行' });
  await expect(retryButton).toBeVisible();
  await expect(retryButton).toBeEnabled();
});
```

**ポイント：**
- `test.setTimeout()` でテスト自体のタイムアウトを延長できます
- アプリケーション側のタイムアウト処理が正しく動くことを検証するには、APIの遅延 > アプリのタイムアウト値にします
- 本番のテストスイートでは、長時間待機するテストは別のグループに分けるとCI時間を短縮できます

</details>

### 問題9：レスポンスを部分的に変更する（modify）

実際のAPIレスポンスを取得しつつ、一部のデータだけ書き換えてテストしたい場合があります。`route.fetch()` を使って、レスポンスの一部を変更するテストを書いてください。

**要件：**
- `/api/user/profile` の実際のレスポンスを取得する
- レスポンスの `name` フィールドだけを「テストユーザー」に変更して返す
- 画面に「テストユーザー」と表示されることを確認する

**期待される結果：**
```
実際のAPIレスポンスのうち、名前だけが「テストユーザー」に変更され、
画面上に「テストユーザー」と表示される
```

<details>
<summary>ヒント</summary>

`route.fetch()` で実際のリクエストを送信し、レスポンスを取得してから加工できます。

```typescript
await page.route('**/api/user/profile', async (route) => {
  const response = await route.fetch();  // 実際にリクエストを送信
  const json = await response.json();    // レスポンスをJSON化
  json.name = '変更後の値';               // 一部を変更
  await route.fulfill({ response, json }); // 変更したデータで応答
});
```

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('APIレスポンスの一部を変更する', async ({ page }) => {
  // ========================================
  // route.fetch() で実際のAPIを呼び、レスポンスを加工する
  // これにより他のフィールドは実データのまま、特定のフィールドだけ変更できる
  // ========================================
  await page.route('**/api/user/profile', async (route) => {
    // 実際にサーバーへリクエストを送信
    const response = await route.fetch();
    // レスポンスをJSONとして取得
    const json = await response.json();

    // ========================================
    // 名前フィールドだけを書き換え
    // 他のフィールド（email, avatarなど）は元のまま
    // ========================================
    json.name = 'テストユーザー';

    // ========================================
    // 変更したJSONで応答を返す
    // response オプションで元のヘッダー等を引き継ぐ
    // ========================================
    await route.fulfill({ response, json });
  });

  await page.goto('https://example.com/profile');

  // ========================================
  // 変更した名前が画面に反映されていることを検証
  // ========================================
  await expect(page.getByTestId('user-name')).toHaveText('テストユーザー');
});
```

**ポイント：**
- `route.fetch()` を使うと実際のサーバーにリクエストを送りつつ、レスポンスを加工できます
- `route.fulfill({ response, json })` の `response` オプションでステータスコードやヘッダーを元のまま引き継げます
- この手法は「実データの大部分はそのまま使いたいが、一部だけテスト用に変えたい」場合に便利です

</details>

### 問題10：複数のAPIリクエストを同時にモックする

ダッシュボードページが複数のAPIエンドポイントからデータを取得するケースをテストしてください。

**要件：**
- `/api/stats` （統計データ）をモックする
- `/api/notifications` （通知データ）をモックする
- `/api/recent-activities` （最近のアクティビティ）をモックする
- 3つのセクションすべてが正しく表示されることを確認する

**期待される結果：**
```
ダッシュボードに統計、通知、最近のアクティビティが
すべてモックデータで正しく表示される
```

<details>
<summary>ヒント</summary>

`page.route()` は複数回呼び出すことで、複数のエンドポイントをそれぞれモックできます。すべてのモックをページ遷移の前に設定してください。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('複数のAPIをモックしてダッシュボードを表示する', async ({ page }) => {
  // ========================================
  // 統計データのモック
  // ========================================
  await page.route('**/api/stats', async (route) => {
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify({
        totalUsers: 1500,
        activeUsers: 320,
        revenue: 5000000,
      }),
    });
  });

  // ========================================
  // 通知データのモック
  // ========================================
  await page.route('**/api/notifications', async (route) => {
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify([
        { id: 1, message: '新しいユーザーが登録しました', read: false },
        { id: 2, message: 'レポートが生成されました', read: true },
      ]),
    });
  });

  // ========================================
  // アクティビティデータのモック
  // ========================================
  await page.route('**/api/recent-activities', async (route) => {
    await route.fulfill({
      status: 200,
      contentType: 'application/json',
      body: JSON.stringify([
        { id: 1, action: 'ログイン', user: '田中太郎', time: '10分前' },
        { id: 2, action: 'ファイルアップロード', user: '鈴木花子', time: '30分前' },
      ]),
    });
  });

  // ========================================
  // すべてのモックを設定した後でページを開く
  // ========================================
  await page.goto('https://example.com/dashboard');

  // ========================================
  // 統計セクションの検証
  // ========================================
  await expect(page.getByText('1,500')).toBeVisible();   // totalUsers
  await expect(page.getByText('320')).toBeVisible();     // activeUsers

  // ========================================
  // 通知セクションの検証
  // ========================================
  await expect(page.getByText('新しいユーザーが登録しました')).toBeVisible();

  // ========================================
  // アクティビティセクションの検証
  // ========================================
  await expect(page.getByText('田中太郎')).toBeVisible();
  await expect(page.getByText('ファイルアップロード')).toBeVisible();
});
```

**ポイント：**
- 複数の `page.route()` を設定する場合、すべてページ遷移前に呼び出します
- モックを使うことで外部サービスの状態に左右されない安定したテストが実現できます
- 実務では、モックデータをテストファイル外の fixture に切り出すとメンテナンスしやすくなります

</details>
