# 第2章: 演算子と型変換

## はじめに

プログラミングでは、データに対してさまざまな「演算」を行います。この章では、JavaScriptの演算子と、JavaScript特有の型変換の仕組みについて詳しく学びます。

---

## 2.1 算術演算子

基本的な四則演算と、その他の数学的演算子です。

```javascript
// 基本の四則演算
console.log(10 + 3);  // 13  加算
console.log(10 - 3);  // 7   減算
console.log(10 * 3);  // 30  乗算
console.log(10 / 3);  // 3.3333...  除算
console.log(10 % 3);  // 1   剰余（余り）
console.log(2 ** 10); // 1024 べき乗（累乗）

// 剰余の活用
console.log(7 % 2);   // 1（奇数判定に使える）
console.log(8 % 2);   // 0（偶数判定に使える）

// 文字列の連結（+演算子のオーバーロード）
console.log("Hello" + " " + "World"); // "Hello World"
console.log("値は" + 42);             // "値は42"（数値が文字列に変換される）

// 注意: 文字列 + 数値の罠
console.log("3" + 4);   // "34"（文字列連結になる！）
console.log(3 + 4 + "5"); // "75"（左から評価: 3+4=7, 7+"5"="75"）
console.log("3" + 4 + 5); // "345"（"3"+4="34", "34"+5="345"）
```

---

## 2.2 代入演算子

値を変数に代入する演算子と、複合代入演算子です。

```javascript
let x = 10;    // 基本の代入

x += 5;  // x = x + 5   → 15
x -= 3;  // x = x - 3   → 12
x *= 2;  // x = x * 2   → 24
x /= 4;  // x = x / 4   → 6
x %= 4;  // x = x % 4   → 2
x **= 3; // x = x ** 3  → 8

// インクリメント・デクリメント
let count = 0;
count++;  // count = count + 1（後置: 式の値は加算前）
++count;  // count = count + 1（前置: 式の値は加算後）
count--;  // count = count - 1
--count;  // count = count - 1

// 前置と後置の違い
let a = 5;
let b = a++;  // bは5（代入してからインクリメント）
console.log(a, b); // 6, 5

let c = 5;
let d = ++c;  // dは6（インクリメントしてから代入）
console.log(c, d); // 6, 6

// 文字列の複合代入
let greeting = "Hello";
greeting += ", World!"; // "Hello, World!"

// 論理代入演算子（ES2021）
let val1 = null;
val1 ??= "デフォルト";  // nullまたはundefinedならデフォルト値を代入
console.log(val1); // "デフォルト"

let val2 = "";
val2 ||= "デフォルト";  // falsy値ならデフォルト値を代入
console.log(val2); // "デフォルト"

let val3 = 1;
val3 &&= 2;  // truthy値なら代入
console.log(val3); // 2
```

## 2.3 比較演算子

### 厳密等価演算子（=== と !==）— 推奨

**型変換なし**で比較します。常にこちらを使うべきです。

```javascript
console.log(5 === 5);       // true
console.log(5 === "5");     // false（型が異なる）
console.log(null === undefined); // false

console.log(5 !== "5");     // true
console.log(5 !== 5);       // false
```

### 抽象等価演算子（== と !=）— 非推奨

**型変換を行ってから**比較するため、予期しない結果が生まれます。

```javascript
console.log(5 == "5");      // true（文字列"5"が数値5に変換される）
console.log(0 == false);    // true
console.log("" == false);   // true
console.log(null == undefined); // true
console.log(0 == "");       // true
console.log(0 == "0");      // true
console.log("" == "0");     // false（！ 一貫性がない）

// これらの混乱を避けるため、常に === を使いましょう
```

### 大小比較演算子

```javascript
console.log(5 > 3);    // true
console.log(5 >= 5);   // true
console.log(3 < 5);    // true
console.log(3 <= 3);   // true

// 文字列の比較（辞書順＝Unicode順）
console.log("apple" < "banana");  // true
console.log("abc" < "abd");       // true
console.log("a" < "A");           // false（小文字のコードが大きい）

// 数値と文字列の比較（暗黙の型変換が起きる）
console.log("10" > 9);   // true（"10"が数値10に変換される）
console.log("10" > "9"); // false（文字列比較！"1" < "9"）
```

## 2.4 論理演算子

### 基本の3つ

```javascript
// AND（論理積）: 両方trueならtrue
console.log(true && true);   // true
console.log(true && false);  // false

// OR（論理和）: どちらかがtrueならtrue
console.log(true || false);  // true
console.log(false || false); // false

// NOT（論理否定）: trueとfalseを反転
console.log(!true);   // false
console.log(!false);  // true
console.log(!!0);     // false（二重否定で真偽値に変換）
console.log(!!"hello"); // true
```

### 短絡評価（ショートサーキット）

論理演算子は、結果が確定した時点で評価を止め、**最後に評価した値**を返します。

```javascript
// && は左がfalsyなら左を返し、truthyなら右を返す
console.log("hello" && "world"); // "world"
console.log(0 && "world");       // 0
console.log(null && "world");    // null

// || は左がtruthyなら左を返し、falsyなら右を返す
console.log("hello" || "world"); // "hello"
console.log(0 || "world");       // "world"
console.log("" || "デフォルト"); // "デフォルト"

// 実用例: デフォルト値の設定
const userName = "" || "ゲスト";     // "ゲスト"
const port = 0 || 3000;              // 3000（0もfalsyなので注意！）

// 実用例: 条件付き実行
const isAdmin = true;
isAdmin && console.log("管理者メニューを表示");
```

## 2.5 Nullish Coalescing演算子（??）

`??` は `null` または `undefined` の場合**のみ**右側の値を返します。`||` と違い、`0` や `""` をそのまま使えます。

```javascript
// ?? と || の違い
console.log(0 || "デフォルト");    // "デフォルト"（0はfalsyなので）
console.log(0 ?? "デフォルト");    // 0（0はnullでもundefinedでもない）

console.log("" || "デフォルト");   // "デフォルト"
console.log("" ?? "デフォルト");   // ""

console.log(false || "デフォルト"); // "デフォルト"
console.log(false ?? "デフォルト"); // false

console.log(null ?? "デフォルト");      // "デフォルト"
console.log(undefined ?? "デフォルト"); // "デフォルト"

// 実用例
function getConfig(options) {
  const timeout = options.timeout ?? 3000;   // 0も有効な値として受け入れたい
  const retries = options.retries ?? 3;
  const verbose = options.verbose ?? false;
  return { timeout, retries, verbose };
}

console.log(getConfig({ timeout: 0, retries: 1 }));
// { timeout: 0, retries: 1, verbose: false }
```

## 2.6 Optional Chaining（?.）

深くネストされたオブジェクトのプロパティに安全にアクセスできます。途中で `null` や `undefined` があっても**エラーにならず** `undefined` を返します。

```javascript
const user = {
  name: "太郎",
  address: {
    city: "東京",
    zip: "100-0001"
  },
  hobbies: ["読書", "プログラミング"],
  greet() {
    return `こんにちは、${this.name}です`;
  }
};

// 従来の書き方（冗長）
const city1 = user && user.address && user.address.city;

// Optional Chaining（簡潔）
const city2 = user?.address?.city; // "東京"

// 存在しないプロパティ
const country = user?.address?.country; // undefined（エラーにならない）

// 配列のアクセス
const firstHobby = user?.hobbies?.[0]; // "読書"

// メソッドの呼び出し
const greeting = user?.greet?.(); // "こんにちは、太郎です"
const unknown = user?.unknownMethod?.(); // undefined

// nullまたはundefinedの場合
const guest = null;
console.log(guest?.name); // undefined（エラーにならない）

// ?? との組み合わせ（よく使われるパターン）
const displayName = user?.profile?.displayName ?? "名無し";
console.log(displayName); // "名無し"
```

## 2.7 型変換（Type Coercion）

JavaScriptは**動的型付け言語**であり、必要に応じて自動的に型変換を行います。

### 暗黙の型変換（自動）

```javascript
// 文字列への変換（+演算子に文字列が含まれる場合）
console.log("5" + 3);     // "53"（数値が文字列に変換される）
console.log("hello" + true); // "hellotrue"
console.log(`値: ${42}`);    // "値: 42"

// 数値への変換（算術演算子 -, *, /, %）
console.log("10" - 3);    // 7
console.log("5" * "3");   // 15
console.log("10" / 2);    // 5
console.log(true + 1);    // 2（true → 1）
console.log(false + 1);   // 1（false → 0）
console.log(null + 1);    // 1（null → 0）
console.log(undefined + 1); // NaN

// 真偽値への変換（条件式の中で自動的に行われる）
// falsy値（falseと見なされる値）:
// false, 0, -0, 0n, "", null, undefined, NaN
// truthy値: 上記以外のすべて（"0", "false", [], {} なども含む！）
if ("hello") console.log("truthy!");     // 表示される
if (0)       console.log("表示されない"); // 表示されない
if ([])      console.log("空配列はtruthy!"); // 表示される！
if ({})      console.log("空オブジェクトもtruthy!"); // 表示される！
```

### 明示的な型変換（推奨）

```javascript
// 文字列への変換
String(42);         // "42"
String(true);       // "true"
String(null);       // "null"
String(undefined);  // "undefined"
(42).toString();    // "42"
(255).toString(16); // "ff"（16進数）
(8).toString(2);    // "1000"（2進数）

// 数値への変換
Number("42");       // 42
Number("3.14");     // 3.14
Number("");         // 0
Number("abc");      // NaN
Number(true);       // 1
Number(false);      // 0
Number(null);       // 0
Number(undefined);  // NaN
parseInt("42px");   // 42（整数部分を解析）
parseFloat("3.14"); // 3.14
+"42";              // 42（単項プラス演算子、簡潔だが可読性は低い）

// 真偽値への変換
Boolean(1);         // true
Boolean(0);         // false
Boolean("hello");   // true
Boolean("");        // false
Boolean(null);      // false
Boolean(undefined); // false
Boolean([]);        // true（注意！）
Boolean({});        // true（注意！）
!!42;               // true（二重否定、よく使われる）
!!"";               // false
```

## 2.8 ビット演算子（参考）

低レベルのデータ操作で使われることがあります。

```javascript
console.log(5 & 3);   // 1  AND（ビット単位）
console.log(5 | 3);   // 7  OR
console.log(5 ^ 3);   // 6  XOR
console.log(~5);       // -6 NOT
console.log(5 << 1);   // 10 左シフト
console.log(5 >> 1);   // 2  右シフト
console.log(-1 >>> 0); // 4294967295 符号なし右シフト

// 実用例: 小数点以下の切り捨て（Math.floorの代替）
console.log(~~3.7);    // 3
console.log(3.7 | 0);  // 3

// 実用例: フラグ管理
const READ    = 0b001;
const WRITE   = 0b010;
const EXECUTE = 0b100;

let permission = READ | WRITE; // 0b011
console.log((permission & READ) !== 0);    // true（読み取り権限あり）
console.log((permission & EXECUTE) !== 0); // false（実行権限なし）
```

## 2.9 三項（条件）演算子

`if...else` を1行で書ける便利な演算子です。

```javascript
// 構文: 条件 ? trueの場合 : falseの場合
const age = 20;
const status = age >= 18 ? "成人" : "未成年";
console.log(status); // "成人"

// 関数の引数に直接使える
console.log(`あなたは${age >= 18 ? "成人" : "未成年"}です`);

// ネスト（可読性が下がるので注意）
const score = 85;
const grade = score >= 90 ? "A"
            : score >= 80 ? "B"
            : score >= 70 ? "C"
            : score >= 60 ? "D"
            : "F";
console.log(grade); // "B"
```

## 2.10 演算子の優先順位

優先順位が高い方が先に評価されます（抜粋）。

| 優先順位 | 演算子 | 説明 |
|---------|--------|------|
| 1（最高） | `()` | グループ化 |
| 2 | `!`, `~`, `typeof`, `++`, `--` | 単項演算子 |
| 3 | `**` | べき乗 |
| 4 | `*`, `/`, `%` | 乗除 |
| 5 | `+`, `-` | 加減 |
| 6 | `<`, `<=`, `>`, `>=` | 比較 |
| 7 | `===`, `!==`, `==`, `!=` | 等価 |
| 8 | `&&` | 論理AND |
| 9 | `\|\|` | 論理OR |
| 10 | `??` | Nullish Coalescing |
| 11 | `? :` | 三項演算子 |
| 12（最低） | `=`, `+=`, `-=` など | 代入 |

```javascript
// 例: 優先順位の影響
console.log(2 + 3 * 4);       // 14（*が先に評価される）
console.log((2 + 3) * 4);     // 20（()で優先順位を変更）
console.log(true || false && false); // true（&&が先に評価される）
```

## まとめ

| 概念 | 説明 |
|------|------|
| `===` / `!==` | 厳密等価比較（常にこちらを使う） |
| `==` / `!=` | 抽象等価比較（型変換あり、非推奨） |
| `&&`, `\|\|` | 論理演算子（短絡評価を行う） |
| `??` | Nullish Coalescing（null/undefinedのみデフォルト値） |
| `?.` | Optional Chaining（安全なプロパティアクセス） |
| 暗黙の型変換 | JavaScriptが自動で型を変換する仕組み |
| falsy値 | `false`, `0`, `""`, `null`, `undefined`, `NaN` |
| `Number()`, `String()`, `Boolean()` | 明示的な型変換 |
