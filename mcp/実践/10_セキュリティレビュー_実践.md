#### [重大] 脆弱性1: SQLインジェクション（add_note, search_notes, delete_notes_by_category）

**場所**: 脆弱性C, D, G

**説明**: f-string でSQLクエリを組み立てており、ユーザー入力がそのままSQL文に挿入されます。

**攻撃シナリオ**:
```
add_note(title="'); DROP TABLE notes; --", content="", category="")
→ SQL: INSERT INTO notes ... VALUES (''); DROP TABLE notes; --', '', '')
→ テーブル全削除
```

**修正案**: パラメータ化クエリを使用
```python
db.execute(
    "INSERT INTO notes (title, content, category) VALUES (?, ?, ?)",
    (title, content, category),
)
```

---

#### [重大] 脆弱性2: 任意SQLクエリ実行（execute_query）

**場所**: 脆弱性J

**説明**: 任意のSQL文を実行できるツールが公開されています。

**攻撃シナリオ**:
```
execute_query("DROP TABLE notes")
execute_query("SELECT * FROM sqlite_master")  → スキーマ情報漏洩
```

**修正案**: このツールを削除する。必要な操作は個別のツールとして安全に提���する。

---

#### [重大] 脆弱性3: コマンドインジェクション（run_export）

**場所**: 脆弱性F

**���明**: `subprocess.run` に `shell=True` でユーザー入力を含むコマンドを渡しています。

**攻撃��ナリオ**:
```
run_export("json; rm -rf /")
→ コ��ンド: python export.py --format json; rm -rf /
→ 任意のOSコマンド実行
```

**修正案**: `shell=True` を使わず、引数をリストで渡す。入力をホワイトリストで検証する。
```python
ALLOWED_FORMATS = {"json", "csv", "text"}
if format not in ALLOWED_FORMATS:
    raise ValueError(f"不正なフォーマット: {format}")
result = subprocess.run(
    ["python", "export.py", "--format", format],
    capture_output=True, text=True,
)
```

#### [重大] 脆弱性4: パストラバーサル（read_note_file）

**場所**: 脆弱���E

**説明**: ファイル名の検証なしに `os.path.join` を使っており、ディレクトリ外のファイルが読めます。

**攻撃シナリオ**:
```
read_note_file("../../etc/passwd")
→ /etc/passwd の内容が返される
```

**修正案**: パスを正規化し、許可ディレクトリ内かチェックする。
```python
from pathlib import Path
NOTES_DIR = Path("./notes").resolve()

def read_note_file(filename: str) -> str:
    resolved = (NOTES_DIR / filename).resolve()
    if not str(resolved).startswith(str(NOTES_DIR)):
        raise ValueError("許可ディレクトリ外へのアクセスです")
    return resolved.read_text(encoding="utf-8")
```

**修正案**: ユーザー向けの一般的なエラーメッセージを返す。

**修正案**: `output_format` に変更する。

```python
"""
ノート管理MCPサーバー（修正版）
全脆弱性を修正した安全なバージョンです。
"""
import os
import sqlite3
from contextlib import contextmanager
from pathlib import Path
from typing import Generator

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("note-server")

# ── 設定（環境変数から読み込み） ──
DB_PATH = os.environ.get("NOTE_DB_PATH", "notes.db")
NOTES_DIR = Path(os.environ.get("NOTES_DIR", "./notes")).resolve()
NOTES_DIR.mkdir(parents=True, exist_ok=True)

ALLOWED_EXTENSIONS = {".txt", ".md", ".json"}
MAX_FILE_SIZE = 1_000_000  # 1MB


@contextmanager
def get_db() -> Generator[sqlite3.Connection, None, None]:
    """安全なDB接続を提供します。"""
    db = sqlite3.connect(DB_PATH, check_same_thread=False)
    db.row_factory = sqlite3.Row
    db.execute("PRAGMA foreign_keys=ON")
    try:
        yield db
    except Exception:
        db.rollback()
        raise
    finally:
        db.close()


def init_db():
    with get_db() as db:
        db.execute("""
            CREATE TABLE IF NOT EXISTS notes (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                title TEXT NOT NULL CHECK(length(title) > 0),
                content TEXT NOT NULL,
                category TEXT DEFAULT 'general',
                created_at TEXT DEFAULT (datetime('now', 'localtime'))
            )
        """)
        db.commit()


init_db()


def validate_path(filename: str) -> Path:
    """パスを検証します。"""
    if ".." in filename or "/" in filename or "\\" in filename:
        raise ValueError("不正なファイル名です")
    resolved = (NOTES_DIR / filename).resolve()
    try:
        resolved.relative_to(NOTES_DIR)
    except ValueError:
        raise ValueError("���可ディレクトリ外へのアクセスです")
    if resolved.suffix not in ALLOWED_EXTENSIONS:
        raise ValueError(f"許可されていない拡張子です: {resolved.suffix}")
    return resolved


# ── Tools（すべてパラメータ化クエリを使用） ──

@mcp.tool()
def add_note(title: str, content: str, category: str = "general") -> str:
    """ノートを追加します。

    Args:
        title: ノートのタイトル
        content: ノートの��容
        category: カテゴリ（デフォルト: "general"）
    """
    title = title.strip()
    content = content.strip()
    if not title:
        raise ValueError("タイトルは必須です")

    with get_db() as db:
        cursor = db.execute(
            "INSERT INTO notes (title, content, category) VALUES (?, ?, ?)",
            (title, content, category),
        )
        db.commit()
        return f"ノー���「{title}」を追加しました（ID: {cursor.lastrowid}）"


@mcp.tool()
def search_notes(keyword: str) -> str:
    """ノートを検索します。

    Args:
        keyword: 検索キーワード
    """
    with get_db() as db:
        rows = db.execute(
            "SELECT id, title, category, created_at FROM notes "
            "WHERE title LIKE ? OR content LIKE ? "
            "ORDER BY created_at DESC LIMIT 50",
            (f"%{keyword}%", f"%{keyword}%"),
        ).fetchall()

        if not rows:
            return "見つかりません"

        lines = [f"検索結果（{len(rows)}件）:"]
        for row in rows:
            lines.append(
                f"  [{row['id']}] {row['title']} "
                f"（{row['category']}、{row['created_at']}）"
            )
        return "\n".join(lines)


@mcp.tool()
def read_note_file(filename: str) -> str:
    """ノートファイルを読み取ります。

    Args:
        filename: ファイル名（notesディレクトリ内のファイル）
    """
    resolved = validate_path(filename)

    if not resolved.is_file():
        raise ValueError(f"ファイルが見つかりません: {filename}")

    size = resolved.stat().st_size
    if size > MAX_FILE_SIZE:
        raise ValueError("ファイルサイズが上限を超えています")

    return resolved.read_text(encoding="utf-8")


@mcp.tool()
def delete_notes_by_category(category: str) -> str:
    """カテゴリ指定でノートを一括削除します。

    Args:
        category: 削除するカテゴリ名
    """
    with get_db() as db:
        count = db.execute(
            "SELECT COUNT(*) as cnt FROM notes WHERE category = ?",
            (category,),
        ).fetchone()["cnt"]

        if count == 0:
            return f"カテゴリ「{category}」のノートはありません"

        db.execute(
            "DELETE FROM notes WHERE category = ?",
            (category,),
        )
        db.commit()
        return f"カテゴリ「{category}」のノート{count}件を削除しました"


# ── Resources ──

@mcp.resource("notes://all")
def get_all_notes() -> str:
    """全ノートの一覧です。"""
    with get_db() as db:
        rows = db.execute(
            "SELECT id, title, category, created_at FROM notes "
            "ORDER BY created_at DESC LIMIT 100"
        ).fetchall()

        if not rows:
            return "ノートはまだありません"

        lines = [f"ノート一覧（{len(rows)}件）:"]
        for row in rows:
            lines.append(
                f"  [{row['id']}] {row['title']} "
                f"（{row['category']}、{row['created_at']}）"
            )
        return "\n".join(lines)


if __name__ == "__main__":
    mcp.run()
```

### 初心者向けとの違い

| ポイント | 初心者向け | 改良版 |
|---------|-----------|--------|
| 成果物 | 脆弱性一覧と修正案 | 修正済みコード全体 + チェックリスト |
| 修正の深さ | 個別の脆弱性修正 | 設計レベルの改善（不要ツール削除等） |
| 再利用性 | 個別の報告書 | 汎用チェックリスト |
| 実務応用 | 指摘と修正 | コードレビュープロセスの確立 |

**実務のポイント**: セキュリティレビューは「脆弱性を見つける」だけでなく、「安全な設計パターンを適用する」ことが重要です。MCPサーバーでは特に「信頼境界を明確にする」「LLM経由で渡される入力を常に検証する」「最小権限の原則を適用する」の3点を意識しましょう。

## よくある間違い

| 間違い | 正しい理解 |
|--------|-----------|
| 「LLM経由なので安全」と思い込む | プロンプトインジェクション（prompt injection）により、LLM経由で悪意のある入力が渡されることがあります。MCPツールの入力は常に信頼できないものとして扱いましょう |
| 「ローカルで動くから問題ない」と思い込む | MCPサーバーは将来的にリモート化される可能性があります。最初からセキュアに設計しましょう |
| 脆弱性の修正が表面的 | 「入力をエスケープする」より「パラメータ化クエリを使う」のように、根本的な対策を採用しましょう |
| エラーメッセージに詳細な情報を含める | SQLエラーやスタックトレースを返すと、攻撃者にシステムの内部構造を教えることになります |
