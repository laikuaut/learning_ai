## 基本問題

### 問題1：ページ遷移とURL検証

以下の操作を行うテストを作成してください。

1. `https://playwright.dev/` にアクセスする
2. 現在のURLが正しいことを検証する
3. `https://playwright.dev/docs/intro` に移動する
4. 遷移後のURLを検証する

**期待される結果:**
```
2つのページそれぞれでURLが正しいことが検証される
```

<details>
<summary>ヒント</summary>

`page.goto(url)` でページに移動します。`expect(page).toHaveURL()` でURLを検証できます。文字列でも正規表現でも指定可能です。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ページ遷移とURL検証', async ({ page }) => {
  // 1. トップページにアクセス
  await page.goto('https://playwright.dev/');

  // 2. 現在のURLを検証
  await expect(page).toHaveURL('https://playwright.dev/');

  // 3. ドキュメントページに移動
  await page.goto('https://playwright.dev/docs/intro');

  // 4. 遷移後のURLを検証
  await expect(page).toHaveURL(/.*\/docs\/intro/);
});
```

</details>

### 問題2：クリック操作

以下のHTMLに対して、各ボタンをクリックしてカウンターの値が変化することを検証するテストを作成してください。

```html
<div id="app">
  <p id="count">カウント: 0</p>
  <button id="increment">+1</button>
  <button id="decrement">-1</button>
  <button id="reset">リセット</button>
</div>
<script>
  let count = 0;
  const display = document.getElementById('count');
  document.getElementById('increment').onclick = () => {
    count++;
    display.textContent = `カウント: ${count}`;
  };
  document.getElementById('decrement').onclick = () => {
    count--;
    display.textContent = `カウント: ${count}`;
  };
  document.getElementById('reset').onclick = () => {
    count = 0;
    display.textContent = `カウント: ${count}`;
  };
</script>
```

1. +1ボタンを3回クリックして「カウント: 3」になることを確認
2. -1ボタンを1回クリックして「カウント: 2」になることを確認
3. リセットボタンをクリックして「カウント: 0」になることを確認

**期待される結果:**
```
各操作後のカウント値が正しいことが検証される
```

<details>
<summary>ヒント</summary>

`click()` メソッドでボタンをクリックします。同じボタンを複数回クリックする場合は `click()` を複数回呼ぶか、`for` ループを使います。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('カウンターのクリック操作を検証する', async ({ page }) => {
  await page.setContent(`
    <div id="app">
      <p id="count">カウント: 0</p>
      <button id="increment">+1</button>
      <button id="decrement">-1</button>
      <button id="reset">リセット</button>
    </div>
    <script>
      let count = 0;
      const display = document.getElementById('count');
      document.getElementById('increment').onclick = () => {
        count++;
        display.textContent = 'カウント: ' + count;
      };
      document.getElementById('decrement').onclick = () => {
        count--;
        display.textContent = 'カウント: ' + count;
      };
      document.getElementById('reset').onclick = () => {
        count = 0;
        display.textContent = 'カウント: ' + count;
      };
    </script>
  `);

  const countDisplay = page.locator('#count');
  const incrementBtn = page.getByRole('button', { name: '+1' });
  const decrementBtn = page.getByRole('button', { name: '-1' });
  const resetBtn = page.getByRole('button', { name: 'リセット' });

  // 1. +1ボタンを3回クリック
  await incrementBtn.click();
  await incrementBtn.click();
  await incrementBtn.click();
  await expect(countDisplay).toHaveText('カウント: 3');

  // 2. -1ボタンを1回クリック
  await decrementBtn.click();
  await expect(countDisplay).toHaveText('カウント: 2');

  // 3. リセットボタンをクリック
  await resetBtn.click();
  await expect(countDisplay).toHaveText('カウント: 0');
});
```

</details>

### 問題3：フォーム入力

以下のHTMLのフォームに値を入力し、入力値が正しいことを検証するテストを作成してください。

```html
<form id="contact-form">
  <label for="name">お名前</label>
  <input id="name" type="text" />
  <label for="email">メールアドレス</label>
  <input id="email" type="email" />
  <label for="message">メッセージ</label>
  <textarea id="message"></textarea>
  <label for="category">カテゴリ</label>
  <select id="category">
    <option value="">選択してください</option>
    <option value="inquiry">お問い合わせ</option>
    <option value="feedback">フィードバック</option>
    <option value="bug">バグ報告</option>
  </select>
</form>
```

1. 名前に「山田花子」を入力する
2. メールアドレスに「hanako@example.com」を入力する
3. メッセージに「テストメッセージです。」を入力する
4. カテゴリで「フィードバック」を選択する
5. 各フィールドの値を検証する

**期待される結果:**
```
各入力欄に正しい値が設定されていることが検証される
セレクトボックスで "feedback" が選択されている
```

<details>
<summary>ヒント</summary>

テキスト入力には `fill()` メソッド、セレクトボックスの選択には `selectOption()` メソッドを使います。値の検証には `toHaveValue()` を使います。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('フォームに値を入力して検証する', async ({ page }) => {
  await page.setContent(`
    <form id="contact-form">
      <label for="name">お名前</label>
      <input id="name" type="text" />
      <label for="email">メールアドレス</label>
      <input id="email" type="email" />
      <label for="message">メッセージ</label>
      <textarea id="message"></textarea>
      <label for="category">カテゴリ</label>
      <select id="category">
        <option value="">選択してください</option>
        <option value="inquiry">お問い合わせ</option>
        <option value="feedback">フィードバック</option>
        <option value="bug">バグ報告</option>
      </select>
    </form>
  `);

  // 1. 名前を入力
  await page.getByLabel('お名前').fill('山田花子');

  // 2. メールアドレスを入力
  await page.getByLabel('メールアドレス').fill('hanako@example.com');

  // 3. メッセージを入力
  await page.getByLabel('メッセージ').fill('テストメッセージです。');

  // 4. カテゴリを選択（valueで指定）
  await page.getByLabel('カテゴリ').selectOption('feedback');

  // 5. 各フィールドの値を検証
  await expect(page.getByLabel('お名前')).toHaveValue('山田花子');
  await expect(page.getByLabel('メールアドレス')).toHaveValue('hanako@example.com');
  await expect(page.getByLabel('メッセージ')).toHaveValue('テストメッセージです。');
  await expect(page.getByLabel('カテゴリ')).toHaveValue('feedback');
});
```

</details>

### 問題4：スクリーンショットの撮影

以下の3種類のスクリーンショットを撮影するテストを作成してください。

1. ページ全体のスクリーンショット
2. ページの表示領域（ビューポート）のみのスクリーンショット
3. 特定の要素だけのスクリーンショット

対象サイトは `https://playwright.dev/` を使用してください。

**期待される結果:**
```
3つのスクリーンショットファイルが生成される
- full-page.png（ページ全体）
- viewport.png（ビューポート）
- element.png（特定要素のみ）
```

<details>
<summary>ヒント</summary>

`page.screenshot()` でスクリーンショットを撮影します。`fullPage: true` オプションでページ全体を撮影できます。特定要素のスクリーンショットは `locator.screenshot()` を使います。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('スクリーンショットを撮影する', async ({ page }) => {
  await page.goto('https://playwright.dev/');

  // 1. ページ全体のスクリーンショット（スクロール領域を含む）
  await page.screenshot({
    path: 'screenshots/full-page.png',
    fullPage: true,
  });

  // 2. ビューポート（表示領域）のスクリーンショット
  await page.screenshot({
    path: 'screenshots/viewport.png',
    // fullPage を指定しない場合、ビューポートのみ撮影される
  });

  // 3. 特定要素のスクリーンショット（ヘッダー部分）
  const heading = page.getByRole('heading', { level: 1 });
  await heading.screenshot({
    path: 'screenshots/element.png',
  });
});
```

</details>

### 問題5：キーボード操作

以下のHTMLに対して、キーボード操作を使ったテストを作成してください。

```html
<div id="app">
  <input id="search" type="text" placeholder="検索..." />
  <div id="result" style="display:none;">検索結果が表示されます</div>
  <textarea id="editor" rows="5" cols="40"></textarea>
</div>
<script>
  const searchInput = document.getElementById('search');
  const result = document.getElementById('result');
  searchInput.addEventListener('keydown', (e) => {
    if (e.key === 'Enter') {
      result.style.display = 'block';
      result.textContent = '検索結果: ' + searchInput.value;
    }
  });
</script>
```

1. 検索欄にテキストを入力し、Enterキーを押して検索結果が表示されることを検証する
2. テキストエリアに文字列を入力し、Ctrl+A で全選択してから Delete で削除する
3. テキストエリアが空になったことを検証する

**期待される結果:**
```
Enter押下後に検索結果が表示される
Ctrl+A → Delete でテキストエリアが空になる
```

<details>
<summary>ヒント</summary>

`press('Enter')` でEnterキーを押せます。`press('Control+a')` でCtrl+Aの操作ができます。`fill('')` でもテキストをクリアできますが、ここではキーボード操作の練習として `press()` を使いましょう。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('キーボード操作のテスト', async ({ page }) => {
  await page.setContent(`
    <div id="app">
      <input id="search" type="text" placeholder="検索..." />
      <div id="result" style="display:none;">検索結果が表示されます</div>
      <textarea id="editor" rows="5" cols="40"></textarea>
    </div>
    <script>
      const searchInput = document.getElementById('search');
      const result = document.getElementById('result');
      searchInput.addEventListener('keydown', (e) => {
        if (e.key === 'Enter') {
          result.style.display = 'block';
          result.textContent = '検索結果: ' + searchInput.value;
        }
      });
    </script>
  `);

  // 1. 検索欄にテキスト入力してEnterキー押下
  const searchInput = page.getByPlaceholder('検索...');
  await searchInput.fill('Playwright');
  await searchInput.press('Enter');

  // 検索結果が表示されることを検証
  const result = page.locator('#result');
  await expect(result).toBeVisible();
  await expect(result).toHaveText('検索結果: Playwright');

  // 2. テキストエリアに入力
  const editor = page.locator('#editor');
  await editor.fill('これはテスト文章です。');
  await expect(editor).toHaveValue('これはテスト文章です。');

  // Ctrl+A で全選択してから Delete で削除
  await editor.press('Control+a');
  await editor.press('Delete');

  // 3. テキストエリアが空になったことを検証
  await expect(editor).toHaveValue('');
});
```

</details>

### 問題6：フォーム送信のテスト

以下のHTMLのログインフォームに対して、入力から送信までの一連の操作をテストしてください。

```html
<div id="app">
  <form id="login-form">
    <h2>ログイン</h2>
    <label for="username">ユーザー名</label>
    <input id="username" type="text" required />
    <label for="password">パスワード</label>
    <input id="password" type="password" required />
    <label>
      <input type="checkbox" id="remember" />
      ログイン状態を保持する
    </label>
    <button type="submit">ログイン</button>
  </form>
  <div id="message" style="display:none;"></div>
</div>
<script>
  document.getElementById('login-form').addEventListener('submit', (e) => {
    e.preventDefault();
    const username = document.getElementById('username').value;
    const msg = document.getElementById('message');
    msg.style.display = 'block';
    msg.textContent = username + 'さん、ようこそ！';
  });
</script>
```

1. ユーザー名とパスワードを入力する
2. 「ログイン状態を保持する」チェックボックスをチェックする
3. ログインボタンをクリックする
4. ウェルカムメッセージが表示されることを検証する

**期待される結果:**
```
フォーム送信後に「(ユーザー名)さん、ようこそ！」が表示される
```

<details>
<summary>ヒント</summary>

チェックボックスの操作には `check()` メソッドを使います。チェック状態の検証には `toBeChecked()` を使えます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ログインフォームの操作テスト', async ({ page }) => {
  await page.setContent(`
    <div id="app">
      <form id="login-form">
        <h2>ログイン</h2>
        <label for="username">ユーザー名</label>
        <input id="username" type="text" required />
        <label for="password">パスワード</label>
        <input id="password" type="password" required />
        <label>
          <input type="checkbox" id="remember" />
          ログイン状態を保持する
        </label>
        <button type="submit">ログイン</button>
      </form>
      <div id="message" style="display:none;"></div>
    </div>
    <script>
      document.getElementById('login-form').addEventListener('submit', (e) => {
        e.preventDefault();
        const username = document.getElementById('username').value;
        const msg = document.getElementById('message');
        msg.style.display = 'block';
        msg.textContent = username + 'さん、ようこそ！';
      });
    </script>
  `);

  // 1. ユーザー名とパスワードを入力
  await page.getByLabel('ユーザー名').fill('tanaka');
  await page.getByLabel('パスワード').fill('password123');

  // 2. チェックボックスをチェック
  await page.getByLabel('ログイン状態を保持する').check();
  // チェック状態を検証
  await expect(page.getByLabel('ログイン状態を保持する')).toBeChecked();

  // 3. ログインボタンをクリック
  await page.getByRole('button', { name: 'ログイン' }).click();

  // 4. ウェルカムメッセージの表示を検証
  const message = page.locator('#message');
  await expect(message).toBeVisible();
  await expect(message).toHaveText('tanakaさん、ようこそ！');
});
```

### 問題7：複数ステップのナビゲーションテスト

Playwrightの公式サイトで以下の操作を順番に行い、各ステップでURLと表示内容を検証するテストを作成してください。

1. `https://playwright.dev/` にアクセスする
2. 「Get started」リンクをクリックしてドキュメントページに移動する
3. ドキュメントページでサイドバーの「Writing tests」リンクをクリックする
4. ブラウザの「戻る」ボタンで前のページに戻る
5. 再び前のページに戻り、トップページに到着する

**期待される結果:**
```
各ステップでURLが正しいことが検証される
ブラウザの履歴を使った前後移動が正しく動作する
```

<details>
<summary>ヒント</summary>

ブラウザの「戻る」操作には `page.goBack()` を使います。「進む」操作には `page.goForward()` を使います。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('複数ステップのナビゲーションと履歴操作', async ({ page }) => {
  // Step 1: トップページにアクセス
  await page.goto('https://playwright.dev/');
  await expect(page).toHaveURL('https://playwright.dev/');
  await expect(page).toHaveTitle(/Playwright/);

  // Step 2: "Get started" をクリック
  await page.getByRole('link', { name: 'Get started' }).click();
  await expect(page).toHaveURL(/.*\/docs\/intro/);

  // Step 3: サイドバーの "Writing tests" をクリック
  await page.getByRole('link', { name: 'Writing tests' }).click();
  await expect(page).toHaveURL(/.*\/docs\/writing-tests/);

  // Step 4: ブラウザの「戻る」で前のページに戻る
  await page.goBack();
  await expect(page).toHaveURL(/.*\/docs\/intro/);

  // Step 5: もう一度「戻る」でトップページに戻る
  await page.goBack();
  await expect(page).toHaveURL('https://playwright.dev/');
});
```

</details>

### 問題8：スクリーンショットによるビジュアルリグレッションテスト

以下のHTMLのコンポーネントに対して、ビジュアルリグレッションテスト（Visual Regression Test）を行うテストを作成してください。

```html
<div class="card" style="width:300px; padding:20px; border:1px solid #ccc; border-radius:8px; font-family:sans-serif;">
  <h3 style="margin:0 0 10px 0;">商品カード</h3>
  <p style="color:#666;">これはサンプル商品の説明文です。</p>
  <span style="font-size:24px; font-weight:bold; color:#e74c3c;">¥3,980</span>
  <button style="display:block; margin-top:15px; padding:10px 20px; background:#3498db; color:#fff; border:none; border-radius:4px; cursor:pointer;">
    カートに追加
  </button>
</div>
```

1. コンポーネントのスクリーンショットを撮影し、スナップショットと比較する
2. ページ全体のスクリーンショットも撮影して比較する

**期待される結果:**
```
初回実行時: スナップショットファイルが生成される
2回目以降: 前回のスナップショットと比較して差異がないことが検証される
```

<details>
<summary>ヒント</summary>

`expect(page).toHaveScreenshot()` を使うと、自動的にスナップショットファイルが生成・比較されます。要素単位では `expect(locator).toHaveScreenshot()` を使います。初回実行時は `--update-snapshots` フラグが必要です。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('商品カードのビジュアルリグレッションテスト', async ({ page }) => {
  await page.setContent(`
    <div class="card" style="width:300px; padding:20px; border:1px solid #ccc; border-radius:8px; font-family:sans-serif;">
      <h3 style="margin:0 0 10px 0;">商品カード</h3>
      <p style="color:#666;">これはサンプル商品の説明文です。</p>
      <span style="font-size:24px; font-weight:bold; color:#e74c3c;">¥3,980</span>
      <button style="display:block; margin-top:15px; padding:10px 20px; background:#3498db; color:#fff; border:none; border-radius:4px; cursor:pointer;">
        カートに追加
      </button>
    </div>
  `);

  // 1. コンポーネント単位のスクリーンショット比較
  // 商品カード要素だけを撮影してスナップショットと比較
  const card = page.locator('.card');
  await expect(card).toHaveScreenshot('product-card.png');

  // 2. ページ全体のスクリーンショット比較
  await expect(page).toHaveScreenshot('product-card-page.png');
});

// ※ 初回実行時はスナップショットが存在しないためテストが失敗します
// 以下のコマンドでスナップショットを生成してください：
// npx playwright test --update-snapshots
//
// 生成されたスナップショットは以下のフォルダに保存されます：
// tests/ファイル名.spec.ts-snapshots/
//
// maxDiffPixels や threshold でピクセル差異の許容範囲を設定できます：
// await expect(card).toHaveScreenshot('product-card.png', {
//   maxDiffPixels: 100,  // 最大100ピクセルの差異を許容
// });
```

</details>

### 問題9：複数タブの操作

以下のHTMLで、リンクが新しいタブで開かれる動作をテストしてください。

```html
<div>
  <h1>メインページ</h1>
  <a href="https://playwright.dev/" target="_blank">Playwright公式サイト（新しいタブで開く）</a>
</div>
```

1. リンクをクリックして新しいタブが開かれることを検出する
2. 新しいタブのURLとタイトルを検証する
3. 新しいタブを閉じてから元のタブに戻ったことを確認する

**期待される結果:**
```
新しいタブが開かれ、URLとタイトルが正しい
タブを閉じた後、元のタブのコンテンツが表示されている
```

<details>
<summary>ヒント</summary>

新しいタブ（ポップアップ）の検出には `page.waitForEvent('popup')` を使います。`Promise.all` でクリックとポップアップの待機を同時に行うと確実です。

</details>

<details>
<summary>解答例</summary>

```typescript

  await expect(page.getByRole('heading', { name: 'メインページ' })).toBeVisible();
```

### 問題10：ドラッグ&ドロップとホバー操作

以下のHTMLに対して、ホバー操作でツールチップが表示されることをテストしてください。

```html
<div id="app">
  <button id="hover-btn" style="padding:10px 20px; position:relative;">
    ホバーしてください
    <span id="tooltip" style="display:none; position:absolute; top:-30px; left:0; background:#333; color:#fff; padding:5px 10px; border-radius:4px; font-size:12px;">
      ツールチップです
    </span>
  </button>
</div>
<script>
  const btn = document.getElementById('hover-btn');
  const tooltip = document.getElementById('tooltip');
  btn.addEventListener('mouseenter', () => { tooltip.style.display = 'block'; });
  btn.addEventListener('mouseleave', () => { tooltip.style.display = 'none'; });
</script>
```

1. ボタンにホバーする前はツールチップが非表示であることを確認する
2. ボタンにホバーしてツールチップが表示されることを検証する
3. ツールチップのテキストが正しいことを検証する

**期待される結果:**
```
ホバー前: ツールチップが非表示
ホバー中: ツールチップが表示され、テキストが "ツールチップです" である
```

<details>
<summary>ヒント</summary>

ホバー操作には `locator.hover()` メソッドを使います。非表示の検証には `toBeHidden()` または `not.toBeVisible()` を使います。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ホバーでツールチップが表示されるテスト', async ({ page }) => {
  await page.setContent(`
    <div id="app">
      <button id="hover-btn" style="padding:10px 20px; position:relative;">
        ホバーしてください
        <span id="tooltip" style="display:none; position:absolute; top:-30px; left:0; background:#333; color:#fff; padding:5px 10px; border-radius:4px; font-size:12px;">
          ツールチップです
        </span>
      </button>
    </div>
    <script>
      const btn = document.getElementById('hover-btn');
      const tooltip = document.getElementById('tooltip');
      btn.addEventListener('mouseenter', () => { tooltip.style.display = 'block'; });
      btn.addEventListener('mouseleave', () => { tooltip.style.display = 'none'; });
    </script>
  `);

  const button = page.locator('#hover-btn');
  const tooltip = page.locator('#tooltip');

  // 1. ホバー前はツールチップが非表示
  await expect(tooltip).toBeHidden();

  // 2. ボタンにホバーする
  await button.hover();

  // ツールチップが表示されることを検証
  await expect(tooltip).toBeVisible();

  // 3. ツールチップのテキストを検証
  await expect(tooltip).toHaveText('ツールチップです');
});
```
