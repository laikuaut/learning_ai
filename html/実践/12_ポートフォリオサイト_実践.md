# 実践課題12：ポートフォリオサイト ★5

> **難易度**: ★★★★★（総合）
> **前提知識**: 第1章〜第8章（すべての章）
> **課題の種類**: ミニプロジェクト（総合）
> **学習目標**: HTML全8章の知識を総動員して、実務レベルの個人ポートフォリオサイトを1から設計・実装する

---

## 課題の説明

あなたはWeb開発を学んでいるエンジニア志望の学生です。就職活動に向けて、自分のスキルや作品を紹介するポートフォリオサイトを作成してください。

**HTMLだけで構造が正しく伝わるページ** を目指します。CSSでのデザインは最低限で構いませんが、セマンティクス、アクセシビリティ、メタ情報のすべてを高いレベルで実装してください。

## 課題の要件

### 必須要件

1. **head セクション（第7章）**
   - `charset`、`viewport`、`description`
   - OGP メタタグ（`og:title`、`og:description`、`og:image`、`og:type`）
   - Twitter Card メタタグ
   - ファビコン指定
   - ページタイトル（SEOを意識した形式：「名前 | 肩書き」）

2. **セマンティック構造（第6章）**
   - `<header>`、`<nav>`、`<main>`、`<section>`、`<article>`、`<aside>`、`<footer>` を適切に使い分ける
   - 見出し階層（h1〜h4）が正しいこと

3. **ナビゲーション（第2章・第6章）**
   - ページ内リンク（`#` アンカー）で各セクションにジャンプできること
   - 複数の `<nav>` にそれぞれ `aria-label` を設定すること

4. **テキストとリンク（第2章）**
   - 外部リンクに `target="_blank"` と `rel="noopener noreferrer"` を設定すること
   - `<strong>`、`<em>` を意味に応じて使い分けること

5. **画像（第3章）**
   - すべての `<img>` に適切な `alt` 属性を設定すること
   - `width` と `height` を設定すること（レイアウトシフト防止）
   - 装飾的な画像は `alt=""` にすること
   - `loading="lazy"` を適切に使うこと

6. **リストとテーブル（第4章）**
   - スキルリストは `<ul>` を使うこと
   - 経歴は `<table>` を使い、`<thead>` / `<tbody>` / `<th>` を適切に使うこと

7. **フォーム（第5章）**
   - お問い合わせフォームに `<label>` と `<input>` の関連付けを行うこと
   - `required`、`type="email"`、`placeholder`、`autocomplete` を適切に使うこと
   - `<fieldset>` と `<legend>` でグループ化すること

8. **HTML5 API の活用（第8章）**
   - `<details>` / `<summary>` を1箇所以上使うこと
   - `<dialog>` または `<details>` でインタラクティブな要素を作ること

### 評価ポイント

| 観点 | 内容 |
|---|---|
| セマンティクス | 各要素の選択が意味的に正しいか |
| 見出し階層 | h1〜h4 が論理的に正しいか |
| アクセシビリティ | alt、label、aria属性が適切か |
| メタ情報 | OGP、description、viewportが正しいか |
| フォーム | バリデーション、label関連付け、適切なtype |
| 実用性 | 実際のポートフォリオとして機能するか |

**初心者向けとの違い:**

- **スキップリンク**を追加 → キーボードユーザーがナビゲーションを飛ばせる
- **JSON-LD構造化データ**を追加 → Google検索でリッチリザルト表示の可能性
- **`aria-hidden="true"`** を必須マーク（*）に追加 → スクリーンリーダーが「アスタリスク」と読み上げるのを防止
- **`aria-required="true"`** をフォーム要素に追加 → スクリーンリーダーが「必須」と伝える
- **`aria-label`** を技術タグリストに追加 → リストの用途が明確に
- **`position: sticky`** でヘッダーを固定 → スクロールしてもナビが見える
- **CSSグリッド/Flexbox** で作品をカード形式に配置 → 実務レベルのレイアウト
- **レスポンシブ対応** → スマートフォンでも崩れない
- **`<address>`** で連絡先を構造化 → セマンティクスが正確
- **OGPに `og:url`、`og:site_name`** を追加 → SNSシェア時の表示が最適化
