# 第4章 演習：リモートリポジトリ

## 基本問題

### 問題1: リモートの追加と確認

ローカルリポジトリにリモートを追加・確認・名前変更してください。

1. リポジトリを作成し、`origin` で `https://github.com/user/app.git` を追加
2. `git remote -v` で確認
3. 名前を `github` に変更して再確認

期待される出力例（手順2）：
```
origin  https://github.com/user/app.git (fetch)
origin  https://github.com/user/app.git (push)
```

<details>
<summary>ヒント</summary>

`git remote add <名前> <URL>` で追加、`git remote rename <旧> <新>` で変更します。
</details>

<details>
<summary>解答</summary>

```bash
$ mkdir my-app && cd my-app && git init -b main
$ echo "# App" > README.md && git add README.md && git commit -m "Initial commit"

$ git remote add origin https://github.com/user/app.git
$ git remote -v
origin  https://github.com/user/app.git (fetch)
origin  https://github.com/user/app.git (push)

$ git remote rename origin github
$ git remote -v   # → github に変更されている
```

- `origin` はリモートのデフォルト名（慣習）です
- `git remote remove <名前>` で削除できます
</details>

### 問題2: push と pull の基本操作

ベアリポジトリでリモートを再現し、2つのローカル間で push/pull してください。

1. ベアリポジトリ作成 → `dev-a` から push → `dev-b` でクローン・変更・push → `dev-a` で pull

期待される出力例（push 時）：
```
To /home/user/shared.git
 * [new branch]      main -> main
```

<details>
<summary>ヒント</summary>

`git init --bare` でベアリポジトリを作成します。`-u` で追跡ブランチを設定します。
</details>

<details>
<summary>解答</summary>

```bash
$ git init --bare ~/shared.git

$ mkdir ~/dev-a && cd ~/dev-a && git init -b main
$ git remote add origin ~/shared.git
$ echo "# Project" > README.md && git add README.md && git commit -m "Initial commit"
$ git push -u origin main

$ git clone ~/shared.git ~/dev-b && cd ~/dev-b
$ echo "新機能" >> README.md && git add README.md
$ git commit -m "docs: 機能説明を追加" && git push origin main

$ cd ~/dev-a && git pull origin main   # 開発者Aが最新を取得
```

- `-u` を設定すれば以降は `git push` / `git pull` だけで通信できます
</details>

### 問題3: fetch と pull の使い分け

`git fetch` でダウンロードだけ行い、差分を確認してからマージしてください。

期待される出力例（fetch 後の `git log --oneline --all`）：
```
ghi9012 (origin/main) feat: 設定ファイルを追加
def5678 (HEAD -> main) docs: 機能説明を追加
```

<details>
<summary>ヒント</summary>

`fetch` はダウンロードのみ。`pull` は `fetch` + `merge` です。
</details>

<details>
<summary>解答</summary>

```bash
$ cd ~/dev-b
$ echo "API=1" > config.txt && git add config.txt
$ git commit -m "feat: 設定ファイルを追加" && git push origin main

$ cd ~/dev-a
$ git fetch origin                    # ダウンロードのみ
$ git log --oneline --all             # 差を確認
$ git diff main origin/main           # 変更内容を確認
$ git merge origin/main               # 問題なければ統合
```

- 現場では fetch → 差分確認 → merge の流れが推奨されます
</details>

## 応用問題

### 問題4: 追跡ブランチの設定と確認

`feature/api` ブランチを `-u` 付きで push し、`git branch -vv` で追跡情報を確認してください。追跡の解除と再設定も試してください。

期待される出力例（`git branch -vv`）：
```
* feature/api abc1234 [origin/feature/api] feat: API機能を追加
  main        def5678 [origin/main] Initial commit
```

<details>
<summary>ヒント</summary>

`git branch -vv` で追跡先を確認。`--unset-upstream` で解除、`--set-upstream-to` で設定します。
</details>

<details>
<summary>解答</summary>

```bash
$ git switch -c feature/api
$ echo '{"api": "v1"}' > api.json
$ git add api.json && git commit -m "feat: API機能を追加"
$ git push -u origin feature/api     # push と同時に追跡設定

$ git branch -vv                     # 追跡情報を確認
$ git branch -r                      # リモートブランチ一覧

# 追跡を解除して再設定
$ git branch --unset-upstream feature/api
$ git branch --set-upstream-to=origin/feature/api feature/api
```

- `[ahead 2]`: リモートより2コミット先 / `[behind 3]`: 3コミット遅れ
</details>

### 問題5: リモートブランチの管理と整理

3ブランチを push した後、1つをリモートから削除し、古い追跡参照を整理してください。

期待される出力例：
```
To /home/user/shared.git
 - [deleted]         feature/old
```

<details>
<summary>ヒント</summary>

`git push origin --delete <ブランチ>` で削除。`git fetch --prune` で不要な参照を整理します。
</details>

<details>
<summary>解答</summary>

```bash
# 3ブランチを push（alpha, beta, old）
$ for name in alpha beta old; do
    git switch main && git switch -c feature/$name
    echo "$name" > $name.txt && git add $name.txt
    git commit -m "feat: $name" && git push -u origin feature/$name
  done

$ git push origin --delete feature/old   # リモートから削除
$ git fetch --prune                      # 古い追跡参照を整理
$ git switch main && git branch -d feature/old   # ローカルも削除
```

- `git config --global fetch.prune true` で fetch 時に常に自動整理できます
</details>

## チャレンジ問題

### 問題6: フォークワークフローの実践

OSS貢献のフォークワークフローをシミュレーションしてください。upstream と fork の2つのリモートを設定し、feature ブランチで修正後、upstream の変更をローカルに同期します。

期待されるリモート設定：
```
origin    /home/user/fork.git (fetch)
origin    /home/user/fork.git (push)
upstream  /home/user/upstream.git (fetch)
upstream  /home/user/upstream.git (push)
```

<details>
<summary>ヒント</summary>

`origin`（自分のfork）と `upstream`（元リポジトリ）の2つのリモートを登録します。
</details>

<details>
<summary>解答</summary>

```bash
# upstream を作成
$ git init --bare ~/upstream.git
$ mkdir ~/tmp-up && cd ~/tmp-up && git init -b main
$ echo "print('Hello')" > main.py
$ git add main.py && git commit -m "Initial commit"
$ git remote add origin ~/upstream.git && git push -u origin main

# fork を作成してクローン
$ git clone --bare ~/upstream.git ~/fork.git
$ git clone ~/fork.git ~/my-work && cd ~/my-work
$ git remote add upstream ~/upstream.git

# feature ブランチで修正 → origin に push
$ git switch -c feature/fix-typo
$ echo "print('Hello, World!')" > main.py
$ git add main.py && git commit -m "fix: typoを修正"
$ git push -u origin feature/fix-typo

# upstream の変更をローカルに同期
$ cd ~/tmp-up && echo "# Guide" > CONTRIBUTING.md
$ git add CONTRIBUTING.md && git commit -m "docs: ガイド追加" && git push origin main
$ cd ~/my-work && git switch main
$ git fetch upstream && git merge upstream/main
$ git push origin main   # fork の main も更新
```

- **upstream**: 元リポジトリ（読み取り） / **origin**: 自分のfork（読み書き）
- upstream の変更は `fetch` → `merge` で定期的に取り込みましょう
</details>
