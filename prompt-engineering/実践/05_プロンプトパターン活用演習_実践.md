## 課題の要件

### パート1：パターン識別クイズ（8問）

以下の8つのタスク説明を読み、最も適切なプロンプトパターンを選んでください。

**選択肢となるパターン**:

| パターン名 | 概要 |
|-----------|------|
| A. ペルソナパターン（Persona Pattern） | 特定の専門家や立場を設定する |
| B. テンプレートパターン（Template Pattern） | 出力のテンプレートを提供する |
| C. メタ言語パターン（Meta Language Pattern） | 独自の記法やルールを定義する |
| D. レシピパターン（Recipe Pattern） | 手順・ステップを順序立てて出力させる |
| E. リフレクションパターン（Reflection Pattern） | 出力後に自己評価・振り返りを行わせる |
| F. フリップパターン（Flipped Interaction Pattern） | LLMから質問させて情報を引き出す |
| G. ゲームプレイパターン（Gameplay Pattern） | ゲーム形式で学習やタスクを進める |
| H. リファイナリーパターン（Refinery Pattern） | 繰り返し改善を重ねて品質を向上させる |

**クイズ問題**:

| 問題 | タスク説明 |
|:----:|-----------|
| Q1 | 「マーケティング部門の視点で、この製品の販売戦略を考えてほしい」 |
| Q2 | 「以下のフォーマットに従ってプレスリリースを作成してほしい」 |
| Q3 | 「初心者向けにPythonの環境構築手順を1ステップずつ説明してほしい」 |
| Q4 | 「作成した提案書の弱点を自分で分析してから、改善版を出してほしい」 |
| Q5 | 「私の状況を理解するために、まずいくつか質問をしてほしい」 |
| Q6 | 「クイズ形式でプロンプトエンジニアリングの知識を確認したい」 |
| Q7 | 「この文章を3回ブラッシュアップして、最終版を出してほしい」 |
| Q8 | 「[製品名]、[価格]、[ターゲット]を入れると広告コピーが生成される仕組みを作りたい」 |

### パート2：パターン適用演習（5テーマ）

以下の5つのテーマに対して、指定された3つのパターンをそれぞれ適用したプロンプトを作成し、出力を比較してください。

| テーマ | 適用パターン |
|--------|-------------|
| 1. ビジネスメール作成 | ペルソナ / テンプレート / リフレクション |
| 2. 技術文書の執筆 | レシピ / リファイナリー / ペルソナ |
| 3. 学習計画の策定 | フリップ / レシピ / ゲームプレイ |
| 4. プレゼン資料の構成 | テンプレート / リフレクション / リファイナリー |
| 5. トラブルシューティング | フリップ / ペルソナ / レシピ |

### パート3：パターン選択フローチャート

パート1・2の経験を踏まえ、「どのタスクにどのパターンを使うべきか」を判断するフローチャートを作成してください。

<summary>ステップ2：パート2のプロンプトを作成する</summary>

テーマ1「ビジネスメール作成」を例に、3パターンの適用方法を示します。

**ペルソナパターン適用**:
```
あなたは外資系企業で10年の経験を持つビジネスコミュニケーションの専門家です。
日本のビジネスマナーと英語圏のビジネス慣習の両方に精通しています。

以下の状況でクライアントに送るフォローアップメールを作成してください。
- 状況：先週の商談で提案した新サービスについて、検討状況を確認する
- トーン：丁寧だが、適度にカジュアル
```

**テンプレートパターン適用**:
```
以下のテンプレートに従って、ビジネスメールを作成してください。
[括弧]内の指示に従って内容を埋めてください。

---
件名：[15文字以内で要件が分かる件名]

[相手の名前] 様

[季節の挨拶 1文]

[用件の説明 2〜3文。先週の商談のフォローアップとして、
新サービスの検討状況を確認する内容]

[次のアクションの提案 1文]

[結びの挨拶 1文]

[自分の署名]
---
```

**リフレクションパターン適用**:
```
以下の状況でクライアントへのフォローアップメールを作成してください。
- 状況：先週の商談で提案した新サービスについて、検討状況を確認する

メールを作成した後、以下の観点で自己評価を行い、改善版を出力してください。
評価観点：
1. 相手に失礼な表現がないか
2. 用件が明確に伝わるか
3. 次のアクションが具体的か
4. 全体の長さは適切か

出力形式：
- 初版
- 自己評価（4観点それぞれ）
- 改善版
```

各テーマで、どのパターンが最も良い結果をもたらしたか記録してください。

</details>

```
│

│

│

│   └─ 複数回の改善が必要 → リファイナリーパターン
│
└─ 学習・練習目的か？
    └─ Yes → ゲームプレイパターン
```

### パート2：テーマ3「学習計画の策定」の比較

#### フリップパターン

```
私はプログラミング未経験の社会人です。Pythonを学びたいと思っています。
最適な学習計画を作成するために、まず私にいくつか質問してください。
私の回答を踏まえて、カスタマイズされた学習計画を提案してください。
```

**出力**: LLMが「現在の職種は？」「学習に使える時間は？」「目標は？」などを質問。回答後に個別最適化された計画が提案される。

**評価**: 情報不足の状態から始められるのが利点。ただし会話が長くなる。

#### レシピパターン

```
プログラミング未経験の社会人がPythonを3ヶ月で基礎習得するための
学習計画を、以下のステップで作成してください。

ステップ1：学習のゴール設定（何ができるようになるか）
ステップ2：必要なツールの準備
ステップ3：月ごとの学習テーマと目標
ステップ4：週ごとの具体的な学習スケジュール
ステップ5：つまずいたときの対処法
```

**出力**: 構造化された学習計画が手順通りに出力される。

**評価**: 計画の構造が明確だが、個人の状況に合わせたカスタマイズが弱い。

#### ゲームプレイパターン

```
Pythonの学習をRPG形式で進めたいです。
以下のルールでゲームを開始してください。

- レベル1：変数と型を理解する（経験値10で次のレベルへ）
- 各レベルで小クイズ（3問）を出題
- 正解すると経験値+5、不正解だと+1（解説あり）
- レベルアップ時にスキルツリーを表示

「ゲームスタート」と言ったら開始してください。
```

**出力**: ゲーム形式で学習が進行。モチベーション維持に効果的。

**評価**: 楽しみながら学べるが、体系的な計画としては不十分。

**最適パターンの結論**: 初回はフリップパターンで状況をヒアリングし、その情報をもとにレシピパターンで具体的な計画を作成するのが最も効果的でした。

### パート3：パターン選択フローチャート

```
何をしたいか？
│
├─ 文書を作成したい
│   ├─ 形式が決まっている → テンプレートパターン
│   ├─ 専門的な内容     → ペルソナパターン
│   └─ 品質を高めたい   → リフレクション or リファイナリーパターン
│
├─ 情報を整理・分析したい
│   ├─ 手順が必要       → レシピパターン
│   └─ 自分の状況に合わせたい → フリップパターン
│
├─ 学習・練習したい
│   └─ ゲームプレイパターン
│
└─ 再利用できる仕組みを作りたい
    └─ メタ言語パターン + テンプレートパターン
```

### パート2：テーマ5「トラブルシューティング」の比較（改良版）

#### フリップパターン（改良版）

```
あなたはITサポートの上級エンジニアです。
私のPCで発生している問題を診断してください。

以下の手順で進めてください：
1. まず、症状を正確に把握するための質問を5つしてください
2. 私の回答に基づいて、追加の質問（最大3つ）をしてください
3. 収集した情報から、考えられる原因を可能性の高い順に3つ挙げてください
4. 各原因に対する解決手順を提示してください

最初の5つの質問を始めてください。
```

**改良ポイント**:
- フリップパターンとペルソナパターンを組み合わせ
- 質問の回数と診断のステップを明確に定義
- 「可能性の高い順」で出力を制御

#### ペルソナ × レシピ 複合パターン（改良版）

```
あなたは以下の3つの専門家の視点を持つトラブルシューティングのプロです。

1. ネットワークエンジニア（通信・接続の問題に強い）
2. システム管理者（OS・ソフトウェアの問題に強い）
3. セキュリティアナリスト（セキュリティ関連の問題に強い）

以下の問題を、各専門家の視点で順番に分析してください。

【問題】
社内のWebアプリケーションが突然遅くなりました。
特定のページだけでなく、全体的に遅い状態です。
昨日までは正常に動作していました。

【分析手順】
各専門家の視点で：
1. この問題で最初に確認すべきことを3つ挙げる
2. 考えられる原因を2つ挙げる
3. 各原因の切り分け方法を説明する

最後に、3人の分析結果を統合して、推奨する対応手順を優先度順に提示してください。
```

**改良ポイント**:
- 単一ペルソナではなく、複数の専門家視点を統合
- レシピパターンで分析の手順を構造化
- 各視点の分析結果を最後に統合する流れを指定

---

### パターンの組み合わせマトリクス（改良版独自）

| 組み合わせ | 効果 | 適用場面 |
|-----------|------|---------|
| ペルソナ + テンプレート | 専門家の知見を定型フォーマットで出力 | レポート、診断書 |
| フリップ + レシピ | ヒアリング後にステップ形式で提案 | コンサルティング、計画策定 |
| リフレクション + リファイナリー | 自己評価→改善を複数回反復 | 文書の品質向上 |
| ペルソナ + リフレクション | 専門家の視点で作成→別の視点で評価 | レビュー、品質保証 |
| メタ言語 + テンプレート | 変数を含む再利用可能なテンプレート | 自動化、定型業務 |
| ゲームプレイ + レシピ | ゲーム形式で段階的に進行 | 教育、トレーニング |

### パターン選択フローチャート（改良版）

```
タスクの性質を判断
│
├─ Q1: 入力情報は十分か？
│   ├─ No → フリップパターンで情報収集
│   │        → 収集後、Q2へ進む
│   └─ Yes → Q2へ
│
├─ Q2: 専門知識が必要か？
│   ├─ Yes → ペルソナパターンを基盤に設定
│   └─ No  → 次へ
│
├─ Q3: 出力に構造が必要か？
│   ├─ 固定フォーマットがある → テンプレートパターン
│   ├─ 手順・プロセスを示す  → レシピパターン
│   └─ 自由形式でOK         → 次へ
│
├─ Q4: 品質向上が必要か？
│   ├─ 1回の自己チェックで十分 → リフレクションパターン
│   ├─ 複数回の改善が必要     → リファイナリーパターン
│   └─ 初版で十分             → 次へ
│
├─ Q5: 再利用性が必要か？
│   ├─ Yes → メタ言語パターン
│   └─ No  → 次へ
│
└─ Q6: モチベーション維持が重要か？
    ├─ Yes → ゲームプレイパターン
    └─ No  → 基本テクニック（役割・指示・文脈・形式）で対応
```

**初心者向けとの違い**:
- パターンの組み合わせ（複合パターン）を積極的に活用
- 複数の専門家視点を統合する高度なペルソナ設定
- パターン組み合わせマトリクスで実務での判断を支援
- フローチャートを質問形式にし、段階的に最適パターンを絞り込む設計に変更
