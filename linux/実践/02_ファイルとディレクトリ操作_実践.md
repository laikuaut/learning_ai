# 実践課題02：ファイルとディレクトリ操作 ★1

> **難易度**: ★☆☆☆☆（入門）
> **前提知識**: 第1章（Linuxの基本）、第2章（ファイルとディレクトリ）
> **課題の種類**: ミニプロジェクト
> **学習目標**: ファイルとディレクトリの作成・移動・コピー・削除を自在に操作できるようになる

---

## 完成イメージ

以下のディレクトリ構造を、コマンドだけで作成します。

```
project/
├── docs/
│   ├── README.md
│   └── CHANGELOG.md
├── src/
│   ├── main.sh
│   └── utils/
│       ├── helper.sh
│       └── config.sh
├── tests/
│   └── test_main.sh
├── backup/
│   └── src_backup/
│       ├── main.sh        ← src/main.sh のコピー
│       └── utils/         ← src/utils/ のコピー
│           ├── helper.sh
│           └── config.sh
└── .gitignore
```

---

## 課題の要件

1. `project` ディレクトリとすべてのサブディレクトリを作成する
2. 各ファイルを作成し、簡単な内容を書き込む
3. `src/` ディレクトリの内容を `backup/src_backup/` にコピーする
4. ファイルの一覧を `ls -la` で確認する
5. `tree` コマンド（またはそれに代わる方法）でディレクトリ構造を表示する

---

## ステップガイド

<details>
<summary>ステップ1：ディレクトリを作成する</summary>

`mkdir` コマンドに `-p` オプションを付けると、親ディレクトリも一括で作成できます。

```bash
# 複数のディレクトリを一度に作成
$ mkdir -p project/docs
$ mkdir -p project/src/utils
$ mkdir -p project/tests
$ mkdir -p project/backup/src_backup
```

あるいは一行で書くこともできます：

```bash
$ mkdir -p project/{docs,src/utils,tests,backup/src_backup}
```

`{...}` はブレース展開（brace expansion）と呼ばれるBashの機能です。

</details>

<details>
<summary>ステップ2：ファイルを作成する</summary>

`echo` とリダイレクト、または `touch` を使ってファイルを作成します。

```bash
# echo でファイルに内容を書き込む
$ echo "# プロジェクトREADME" > project/docs/README.md
$ echo "# 変更履歴" > project/docs/CHANGELOG.md

# メインスクリプト
$ echo '#!/bin/bash' > project/src/main.sh
$ echo 'echo "Hello, Linux!"' >> project/src/main.sh

# touch で空のファイルを作成
$ touch project/.gitignore
```

</details>

<details>
<summary>ステップ3：ファイルをコピーする</summary>

`cp` コマンドの `-r` オプションでディレクトリを丸ごとコピーできます。

```bash
# ファイルのコピー
$ cp project/src/main.sh project/backup/src_backup/

# ディレクトリを丸ごとコピー（-r: 再帰的）
$ cp -r project/src/utils project/backup/src_backup/
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

```bash
#!/bin/bash
# ファイルとディレクトリ操作の実践
# 学べる内容：mkdir, touch, echo, cp, ls, find
# 実行方法：bash create_project.sh

# --- ディレクトリ作成 ---
mkdir -p project/docs
mkdir -p project/src/utils
mkdir -p project/tests
mkdir -p project/backup/src_backup

# --- ファイル作成 ---
# docs/
echo "# プロジェクトREADME" > project/docs/README.md
echo "" >> project/docs/README.md
echo "このプロジェクトはLinux学習用です。" >> project/docs/README.md

echo "# 変更履歴" > project/docs/CHANGELOG.md
echo "" >> project/docs/CHANGELOG.md
echo "## v1.0.0 ($(date '+%Y-%m-%d'))" >> project/docs/CHANGELOG.md
echo "- 初版作成" >> project/docs/CHANGELOG.md

# src/
echo '#!/bin/bash' > project/src/main.sh
echo '# メインスクリプト' >> project/src/main.sh
echo 'echo "Hello, Linux!"' >> project/src/main.sh

echo '#!/bin/bash' > project/src/utils/helper.sh
echo '# ヘルパー関数' >> project/src/utils/helper.sh
echo 'greet() { echo "Welcome, $1!"; }' >> project/src/utils/helper.sh

echo '#!/bin/bash' > project/src/utils/config.sh
echo '# 設定ファイル' >> project/src/utils/config.sh
echo 'APP_NAME="MyProject"' >> project/src/utils/config.sh
echo 'VERSION="1.0.0"' >> project/src/utils/config.sh

# tests/
echo '#!/bin/bash' > project/tests/test_main.sh
echo '# テストスクリプト' >> project/tests/test_main.sh
echo 'echo "Running tests..."' >> project/tests/test_main.sh

# .gitignore
echo "*.log" > project/.gitignore
echo "*.tmp" >> project/.gitignore
echo "backup/" >> project/.gitignore

# --- バックアップ（コピー） ---
cp project/src/main.sh project/backup/src_backup/
cp -r project/src/utils project/backup/src_backup/

# --- 確認 ---
echo "=== 作成されたディレクトリ構造 ==="
find project -type f | sort

echo ""
echo "=== 各ファイルのサイズ ==="
find project -type f -exec ls -lh {} \;

echo ""
echo "=== ディレクトリ構造 ==="
# tree がインストールされていなければ find で代替
if command -v tree > /dev/null 2>&1; then
    tree project
else
    find project | sort | sed 's|[^/]*/|  |g'
fi
```

</details>

<details>
<summary>解答例（改良版 ─ ブレース展開とヒアドキュメント活用）</summary>

```bash
#!/bin/bash
# ファイルとディレクトリ操作の実践（改良版）
# 学べる内容：ブレース展開、ヒアドキュメント、chmod
# 実行方法：bash create_project_v2.sh

set -e  # エラーが発生したら即座に停止

PROJECT="project"

# --- ディレクトリ作成（ブレース展開で一括） ---
mkdir -p "$PROJECT"/{docs,src/utils,tests,backup/src_backup}

# --- ファイル作成（ヒアドキュメントで読みやすく） ---
cat > "$PROJECT/docs/README.md" << 'EOF'
# プロジェクトREADME

このプロジェクトはLinux学習用です。

## 使い方

```bash
bash src/main.sh
```
EOF

cat > "$PROJECT/docs/CHANGELOG.md" << EOF
# 変更履歴

## v1.0.0 ($(date '+%Y-%m-%d'))
- 初版作成
EOF

cat > "$PROJECT/src/main.sh" << 'EOF'
#!/bin/bash
# メインスクリプト
source "$(dirname "$0")/utils/config.sh"
source "$(dirname "$0")/utils/helper.sh"
echo "=== ${APP_NAME} v${VERSION} ==="
greet "User"
EOF

cat > "$PROJECT/src/utils/helper.sh" << 'EOF'
#!/bin/bash
# ヘルパー関数
greet() {
    echo "Welcome, $1!"
}
EOF

cat > "$PROJECT/src/utils/config.sh" << 'EOF'
#!/bin/bash
# 設定ファイル
APP_NAME="MyProject"
VERSION="1.0.0"
EOF

cat > "$PROJECT/tests/test_main.sh" << 'EOF'
#!/bin/bash
# テストスクリプト
echo "Running tests..."
output=$(bash src/main.sh)
if echo "$output" | grep -q "MyProject"; then
    echo "PASS: アプリ名が表示された"
else
    echo "FAIL: アプリ名が表示されない"
fi
EOF

printf "*.log\n*.tmp\nbackup/\n" > "$PROJECT/.gitignore"

# --- 実行権限を付与 ---
chmod +x "$PROJECT/src/main.sh"
chmod +x "$PROJECT/src/utils/helper.sh"
chmod +x "$PROJECT/tests/test_main.sh"

# --- バックアップ ---
cp "$PROJECT/src/main.sh" "$PROJECT/backup/src_backup/"
cp -r "$PROJECT/src/utils" "$PROJECT/backup/src_backup/"

# --- 確認 ---
echo "ディレクトリ構造:"
find "$PROJECT" | sort | while read -r path; do
    depth=$(echo "$path" | tr -cd '/' | wc -c)
    indent=$(printf '%*s' "$((depth * 2))" '')
    basename_path=$(basename "$path")
    if [ -d "$path" ]; then
        echo "${indent}${basename_path}/"
    else
        size=$(wc -c < "$path")
        echo "${indent}${basename_path} (${size} bytes)"
    fi
done
```

**初心者向けとの違い:**
- `set -e` でエラー発生時に即停止 → 途中で失敗してもゴミファイルが残りにくい
- ブレース展開（`{docs,src/utils,...}`）でディレクトリ作成を1行に圧縮
- ヒアドキュメント（`<< 'EOF'`）で複数行のファイル内容を見やすく記述
- `chmod +x` で実行権限を付与（実務では忘れがち）
- `source` で他のスクリプトを読み込む実践的な構成

</details>
