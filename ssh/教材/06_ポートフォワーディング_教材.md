# 第6章：ポートフォワーディング

## この章のゴール
- ポートフォワーディング（SSHトンネル）の概念を理解する
- ローカル・リモート・ダイナミックの3種類を使い分けられるようになる
- 実務での活用パターンを身につける

---

## 6.1 ポートフォワーディングとは

**ポートフォワーディング（SSHトンネル）** は、SSH接続を通じて、別のポートへの通信を暗号化して転送する技術です。

```
【ポートフォワーディングのイメージ】

通常のアクセス（暗号化なし）:
  ブラウザ ---[平文]--→ リモートサービス

SSHトンネル経由（暗号化あり）:
  ブラウザ → localhost:8080 =[SSHトンネル（暗号化）]=→ リモートサービス:80
```

### 3つの種類

| 種類 | オプション | 方向 | 用途 |
|------|----------|------|------|
| ローカルフォワーディング | `-L` | ローカル → リモート | リモートサービスにローカルからアクセス |
| リモートフォワーディング | `-R` | リモート → ローカル | ローカルサービスをリモートに公開 |
| ダイナミックフォワーディング | `-D` | SOCKS プロキシ | 任意の宛先への通信を転送 |

---

## 6.2 ローカルポートフォワーディング（-L）

ローカルのポートへのアクセスを、SSHサーバー経由でリモートのサービスに転送します。

### 構文

```bash
ssh -L [ローカルアドレス:]ローカルポート:リモートホスト:リモートポート ユーザー@SSHサーバー
```

### 基本例：リモートのWebサーバーにアクセス

```
+----------+          +----------+          +----------+
| ローカル  |   SSH    | SSHサーバー|          | Webサーバー|
| PC       |=========>|          |--------->|          |
| :8080    |  暗号化   |          |  :80     | :80      |
+----------+          +----------+          +----------+
  ↑
ブラウザで
http://localhost:8080
にアクセス
```

```bash
# リモートの80番ポート（Web）をローカルの8080番にフォワード
$ ssh -L 8080:localhost:80 user@example.com

# ブラウザで http://localhost:8080 にアクセスすると
# example.com の 80番ポートに接続される
```

### 実践例1：リモートのデータベースに接続

```bash
# リモートの PostgreSQL（5432番）をローカルの15432番にフォワード
$ ssh -L 15432:localhost:5432 user@db-server.example.com

# 別のターミナルから接続
$ psql -h localhost -p 15432 -U dbuser mydb
```

```
+----------+          +----------+
| ローカル  |   SSH    | DBサーバー |
| PC       |=========>|          |
| :15432   |  暗号化   | :5432    |
+----------+          | (PostgreSQL)|
  ↑                   +----------+
psql -h localhost
     -p 15432
```

### 実践例2：踏み台経由で内部サービスにアクセス

```bash
# 踏み台サーバー経由で内部Webサーバーの3000番にアクセス
$ ssh -L 3000:internal-web:3000 user@bastion.example.com

# internal-web は踏み台サーバーから見えるホスト名
```

```
+----------+          +----------+          +----------+
| ローカル  |   SSH    |  踏み台    |          | 内部Web   |
| PC       |=========>| (bastion) |--------->| サーバー   |
| :3000    |  暗号化   |          | :3000   | :3000    |
+----------+          +----------+          +----------+
  ↑                   （インターネット経由）   （内部ネットワーク）
ブラウザで
http://localhost:3000
```

### バックグラウンドで実行

```bash
# -N: リモートコマンドを実行しない
# -f: バックグラウンドに移行
$ ssh -f -N -L 8080:localhost:80 user@example.com

# 確認（プロセスIDを調べる）
$ ps aux | grep "ssh -f"
user  12345  ... ssh -f -N -L 8080:localhost:80 user@example.com

# 停止
$ kill 12345
```

### 複数のフォワーディングを同時に設定

```bash
# Web（80番）とDB（5432番）を同時にフォワード
$ ssh -L 8080:localhost:80 -L 15432:localhost:5432 user@example.com

# ~/.ssh/config での設定
# Host dev
#     HostName dev.example.com
#     LocalForward 8080 localhost:80
#     LocalForward 15432 localhost:5432
```

## 6.3 リモートポートフォワーディング（-R）

リモートサーバーのポートへのアクセスを、SSHクライアント（ローカル）側のサービスに転送します。ローカルフォワーディングの逆方向です。

### 構文

```bash
ssh -R [リモートアドレス:]リモートポート:ローカルホスト:ローカルポート ユーザー@SSHサーバー
```

### 基本例：ローカルのWebアプリを公開

```
+----------+          +----------+
| ローカル  |   SSH    | リモート   |
| PC       |=========>| サーバー   |
| :3000    |  暗号化   | :8080    |
| (開発中   |          |          |
|  のアプリ) |          |          |
+----------+          +----------+
                        ↑
               外部からリモートの
               :8080にアクセスすると
               ローカルの:3000に転送
```

```bash
# ローカルの3000番（開発中のアプリ）をリモートの8080番に公開
$ ssh -R 8080:localhost:3000 user@example.com

# リモートサーバーの8080番にアクセスすると、
# ローカルの3000番（開発中のアプリ）につながる
```

### 実践例：NAT内のサーバーに外部からアクセス

```
+----------+          +----------+          +----------+
| 外部の    |  HTTP    | VPS      |   SSH    | NAT内の   |
| ユーザー   |--------->| (公開IP) |<=========| サーバー   |
|          | :8080    |          | 暗号化   | :80      |
+----------+          +----------+          +----------+

# NAT内のサーバーから実行:
$ ssh -R 8080:localhost:80 user@vps.example.com

# 外部ユーザーは vps.example.com:8080 にアクセスして
# NAT内のWebサーバーに到達できる
```

> **注意**: リモートフォワーディングで外部に公開する場合、sshdの `GatewayPorts yes` 設定が必要です。デフォルトでは localhost からのみアクセス可能です。

## 6.4 ダイナミックポートフォワーディング（-D）

SOCKSプロキシとして動作し、任意の宛先への通信をSSH経由で転送します。

### 構文

```bash
ssh -D [ローカルアドレス:]ポート ユーザー@SSHサーバー
```

### 基本例

```
+----------+          +----------+          +----------+
| ローカル  |   SSH    | SSHサーバー|   HTTP   | 任意の     |
| PC       |=========>|  (海外)   |--------->| Webサイト  |
| :1080    |  暗号化   |          |          |          |
| (SOCKS   |          |          |          |          |
|  プロキシ) |          |          |          |          |
+----------+          +----------+          +----------+
  ↑
ブラウザの
プロキシ設定:
SOCKS5 localhost:1080
```

```bash
# SOCKSプロキシをローカルの1080番で起動
$ ssh -D 1080 user@remote-server.example.com

# バックグラウンドで実行
$ ssh -f -N -D 1080 user@remote-server.example.com
```

### ブラウザでの設定

```
Firefox の場合:
  設定 → ネットワーク設定 → 接続設定
  → 手動でプロキシを設定する
  → SOCKSホスト: localhost  ポート: 1080
  → SOCKS v5 を選択
  → 「SOCKS v5 を使用するときは DNS もプロキシを使用する」にチェック

curl での使用:
$ curl --socks5-hostname localhost:1080 https://example.com
```

### 用途

```
- 社内ネットワーク内のWebサービスに外部からアクセス
- 地域制限のあるコンテンツへのアクセス（正当な理由がある場合）
- 公共Wi-Fi使用時の通信の暗号化
```

## 6.5 ~/.ssh/config でのフォワーディング設定

```
# ローカルフォワーディング
Host dev
    HostName dev.example.com
    User developer
    LocalForward 8080 localhost:80
    LocalForward 15432 localhost:5432

# リモートフォワーディング
Host tunnel
    HostName vps.example.com
    User user
    RemoteForward 8080 localhost:3000

# ダイナミックフォワーディング
Host proxy
    HostName proxy-server.example.com
    User user
    DynamicForward 1080
```

## 6.6 動的にフォワーディングを追加

既存のSSH接続に、後からフォワーディングを追加できます。

```bash
# 1. SSHセッション中に Enter を押す
# 2. ~C と入力（エスケープシーケンス）
ssh> -L 8080:localhost:80
Forwarding port.

# 追加したフォワーディングを確認
ssh> -KL 8080
Canceled forwarding.
```

## 6.7 実務でよく使うパターン

### パターン1：Webアプリ開発時のDB接続

```bash
# 開発環境のDBにローカルから接続
$ ssh -L 5432:db.internal:5432 user@bastion

# ローカルのアプリケーションから接続
# DATABASE_URL=postgresql://user:pass@localhost:5432/mydb
```

### パターン2：管理画面へのアクセス

```bash
# 管理画面（外部に公開されていない）にアクセス
$ ssh -L 9090:admin.internal:9090 user@bastion

# ブラウザで http://localhost:9090 にアクセス
```

### パターン3：Redisへの接続

```bash
# リモートのRedisにローカルから接続
$ ssh -L 6379:redis.internal:6379 user@bastion

$ redis-cli -h localhost -p 6379
```

## 6.8 注意事項とセキュリティ

### ローカルフォワーディングのバインドアドレス

```bash
# デフォルト: localhost（127.0.0.1）のみリッスン
$ ssh -L 8080:remote:80 user@server
# localhost:8080 からのみアクセス可能

# すべてのインターフェースでリッスン（注意が必要）
$ ssh -L 0.0.0.0:8080:remote:80 user@server
# 同じネットワーク上の他のPCからもアクセス可能になる
```

### トンネルが切れた時の対処

```bash
# autossh で自動再接続（別途インストールが必要）
$ autossh -M 0 -f -N -L 8080:localhost:80 user@example.com
# -M 0: モニタリングポートを使用しない（ServerAliveInterval で代用）

# ~/.ssh/config でキープアライブを設定
# Host *
#     ServerAliveInterval 60
#     ServerAliveCountMax 3
```

## ポイントまとめ

```
1. ローカルフォワーディング(-L): リモートサービスにローカルからアクセス
2. リモートフォワーディング(-R): ローカルサービスをリモートに公開
3. ダイナミックフォワーディング(-D): SOCKSプロキシとして任意の通信を転送
4. -f -N オプションでバックグラウンド実行（シェル不要の場合）
5. ~/.ssh/config の LocalForward/RemoteForward で設定を永続化
6. 踏み台経由で内部サービスにアクセスする構成が実務で頻出
7. バインドアドレスに注意（デフォルトは localhost のみ）
```
