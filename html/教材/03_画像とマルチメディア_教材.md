## 3.1 img要素：画像の表示

### 基本構文
```html
<img src="画像のパス" alt="画像の説明テキスト">
```

- `<img>` は**空要素**（終了タグなし）
- `src` 属性：画像ファイルのパス（必須）
- `alt` 属性：代替テキスト（必須）

### 使用例
```html
<!-- ローカル画像 -->
<img src="images/photo.jpg" alt="東京タワーの夜景">

<!-- 外部画像 -->
<img src="https://example.com/images/logo.png" alt="Example社のロゴ">

<!-- 同じフォルダの画像 -->
<img src="cat.png" alt="眠っている猫の写真">
```

---

## 3.2 alt属性の重要性

`alt` 属性は以下の場面で使われます。

### alt属性が使われる場面
1. **画像が読み込めないとき** — alt テキストが代わりに表示される
2. **スクリーンリーダー** — 視覚障害者にalt テキストが読み上げられる
3. **検索エンジン** — 画像の内容をalt テキストで判断する
4. **テキストブラウザ** — 画像の代わりにalt テキストを表示する

### alt属性の書き方ガイド

```html
<!-- 良い例：画像の内容を具体的に説明 -->
<img src="chart.png" alt="2024年の月別売上グラフ。3月が最高で500万円">

<!-- 良い例：装飾的な画像は空のaltにする -->
<img src="decoration-line.png" alt="">

<!-- 悪い例：意味のないalt -->
<img src="photo.jpg" alt="画像">
<img src="photo.jpg" alt="写真">
<img src="photo.jpg" alt="image">

<!-- 悪い例：ファイル名をそのまま書く -->
<img src="IMG_0123.jpg" alt="IMG_0123.jpg">
```

### alt属性の書き方ルール
| 画像の種類 | altの書き方 | 例 |
|---|---|---|
| 情報を伝える画像 | 内容を具体的に説明 | `alt="赤いリンゴ3個"` |
| リンク画像 | リンク先の情報 | `alt="ホームページへ戻る"` |
| 装飾画像 | 空文字にする | `alt=""` |
| グラフ・チャート | データの要約 | `alt="売上は前年比120%"` |
| ロゴ | 企業名やサービス名 | `alt="Google"` |
| ボタン画像 | ボタンの機能 | `alt="検索"` |

> **重要**: `alt` 属性を省略すると、スクリーンリーダーがファイル名を読み上げてしまいます。空でも必ず `alt=""` を付けましょう。

## 3.3 画像のサイズ指定

### width / height 属性
```html
<img src="photo.jpg" alt="風景写真" width="800" height="600">
```

- 値はピクセル単位（`px` は書かない）
- **CLS（Cumulative Layout Shift）防止**のため、width と height を指定するのが推奨
- ブラウザが画像読み込み前にスペースを確保できる

### CLS（レイアウトシフト）とは
画像のサイズが未指定の場合：
1. ページが読み込まれる → テキストが表示される
2. 画像が遅れて読み込まれる → テキストが下にずれる
3. ユーザーが読んでいた場所を見失う（UXが悪い）

widthとheightを指定すれば、ブラウザは事前にスペースを確保するため、レイアウトシフトが起きません。

### CSSでのサイズ制御
```html
<!-- レスポンシブ画像（画面幅に合わせる） -->
<img src="photo.jpg" alt="風景写真" width="800" height="600"
     style="max-width: 100%; height: auto;">
```

## 3.4 主な画像フォーマット

| フォーマット | 拡張子 | 特徴 | 用途 |
|---|---|---|---|
| **JPEG** | .jpg, .jpeg | 写真向き、非可逆圧縮、透過不可 | 写真、複雑な画像 |
| **PNG** | .png | 透過対応、可逆圧縮 | ロゴ、イラスト、スクリーンショット |
| **GIF** | .gif | アニメーション対応、256色 | 簡単なアニメーション |
| **SVG** | .svg | ベクター形式、拡大しても劣化なし | アイコン、ロゴ、図形 |
| **WebP** | .webp | JPEGより高圧縮、透過対応 | 写真全般（モダンブラウザ向け） |
| **AVIF** | .avif | WebPよりさらに高圧縮 | 最新ブラウザ向け |

### 選び方の基準
- **写真** → JPEG or WebP
- **ロゴ・アイコン** → SVG or PNG
- **透過が必要** → PNG or WebP
- **アニメーション** → GIF（短い） or MP4（長い）
- **最適化重視** → WebP or AVIF

## 3.5 figure / figcaption：画像のグループ化

`<figure>` は画像とキャプションをセットにするための要素です。

```html
<figure>
  <img src="tokyo-tower.jpg" alt="ライトアップされた東京タワー">
  <figcaption>東京タワーの夜景（2024年1月撮影）</figcaption>
</figure>
```

### figureの特徴
- `<figure>` は自己完結型のコンテンツを表す
- 画像だけでなく、コードブロック、引用、図表にも使える
- `<figcaption>` は `<figure>` の最初か最後の子要素として配置

```html
<!-- コードブロックにfigureを使う例 -->
<figure>
  <pre><code>
    function hello() {
      console.log("Hello, World!");
    }
  </code></pre>
  <figcaption>JavaScriptの Hello World プログラム</figcaption>
</figure>

<!-- 引用にfigureを使う例 -->
<figure>
  <blockquote>
    <p>知は力なり。</p>
  </blockquote>
  <figcaption>— フランシス・ベーコン</figcaption>
</figure>
```

### 複数の画像をグループ化
```html
<figure>
  <img src="before.jpg" alt="リフォーム前のリビング">
  <img src="after.jpg" alt="リフォーム後のリビング">
  <figcaption>リビングのリフォーム ビフォー&アフター</figcaption>
</figure>
```

## 3.6 video要素：動画の埋め込み

### 基本構文
```html
<video src="movie.mp4" controls>
  お使いのブラウザは動画再生に対応していません。
</video>
```

### 属性一覧
| 属性 | 説明 | 例 |
|---|---|---|
| `src` | 動画ファイルのパス | `src="video.mp4"` |
| `controls` | 再生コントロールを表示 | `controls` |
| `autoplay` | 自動再生（muted必須） | `autoplay` |
| `muted` | ミュート状態で開始 | `muted` |
| `loop` | ループ再生 | `loop` |
| `poster` | サムネイル画像 | `poster="thumb.jpg"` |
| `width` / `height` | サイズ指定 | `width="640"` |
| `preload` | 読み込み方法 | `preload="metadata"` |
| `playsinline` | インライン再生（iOS向け） | `playsinline` |

### preload属性の値
| 値 | 動作 |
|---|---|
| `auto` | ページ読み込み時に動画全体を読み込む |
| `metadata` | メタデータ（長さ等）のみ読み込む |
| `none` | 再生ボタンが押されるまで読み込まない |

### 複数フォーマットの指定
```html
<video controls width="640" height="360" poster="thumbnail.jpg">
  <source src="movie.webm" type="video/webm">
  <source src="movie.mp4" type="video/mp4">
  <p>お使いのブラウザは動画再生に対応していません。
     <a href="movie.mp4">動画をダウンロード</a>してください。</p>
</video>
```

- ブラウザは上から順に再生できるフォーマットを選択する
- `type` 属性でMIMEタイプを指定するとブラウザの判定が早い
- どのフォーマットも対応していない場合、フォールバックテキストが表示される

### 自動再生に関する注意
```html
<!-- 音声付き自動再生はブラウザにブロックされる -->
<video src="movie.mp4" autoplay>  <!-- 多くのブラウザで再生されない -->

<!-- ミュートなら自動再生可能 -->
<video src="movie.mp4" autoplay muted loop playsinline>
```

> **重要**: ほとんどのブラウザは、音声付きの自動再生をブロックします。自動再生したい場合は `muted` を必ず付けてください。

### 字幕の追加（track要素）
```html
<video controls>
  <source src="movie.mp4" type="video/mp4">
  <track src="subtitles_ja.vtt" kind="subtitles" srclang="ja" label="日本語" default>
  <track src="subtitles_en.vtt" kind="subtitles" srclang="en" label="English">
</video>
```

## 3.7 audio要素：音声の埋め込み

### 基本構文
```html
<audio src="music.mp3" controls>
  お使いのブラウザは音声再生に対応していません。
</audio>
```

### 属性
video要素と同じ属性が多く使えます（`controls`, `autoplay`, `muted`, `loop`, `preload`）。

### 複数フォーマット
```html
<audio controls>
  <source src="music.ogg" type="audio/ogg">
  <source src="music.mp3" type="audio/mpeg">
  <p>お使いのブラウザは音声再生に対応していません。</p>
</audio>
```

### 主な音声フォーマット
| フォーマット | MIMEタイプ | 特徴 |
|---|---|---|
| MP3 | `audio/mpeg` | 最も広くサポート |
| AAC | `audio/aac` | Apple系で一般的 |
| OGG Vorbis | `audio/ogg` | オープンフォーマット |
| WAV | `audio/wav` | 非圧縮、高品質 |
| WebM | `audio/webm` | Googleが推進 |

## 3.8 picture要素とsource要素：レスポンシブ画像

`<picture>` 要素を使うと、画面サイズやデバイスに応じて最適な画像を提供できます。

### 基本構文
```html
<picture>
  <source media="(min-width: 1024px)" srcset="large.jpg">
  <source media="(min-width: 768px)" srcset="medium.jpg">
  <img src="small.jpg" alt="レスポンシブ画像の例">
</picture>
```

- ブラウザは `<source>` を上から順に評価し、条件に合う最初のものを使用
- どの条件にも合わない場合は `<img>` がフォールバック
- `<img>` は**必須**

### フォーマットの切り替え
```html
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="フォーマット別の画像提供">
</picture>
```

対応ブラウザにはAVIF、非対応ならWebP、それも非対応ならJPEGを表示。

### アートディレクション
画面サイズによって異なる構図の画像を表示する場合：

```html
<picture>
  <!-- デスクトップ：横長の全体写真 -->
  <source media="(min-width: 1024px)" srcset="hero-wide.jpg">
  <!-- タブレット：中間サイズ -->
  <source media="(min-width: 768px)" srcset="hero-medium.jpg">
  <!-- スマートフォン：縦長のクローズアップ -->
  <img src="hero-mobile.jpg" alt="商品のヒーロー画像">
</picture>
```

### srcset属性とsizes属性（img要素のレスポンシブ機能）
```html
<!-- ピクセル密度による切り替え -->
<img src="logo.png"
     srcset="logo.png 1x, logo@2x.png 2x, logo@3x.png 3x"
     alt="サイトロゴ">

<!-- 幅による切り替え -->
<img src="photo-400.jpg"
     srcset="photo-400.jpg 400w,
             photo-800.jpg 800w,
             photo-1200.jpg 1200w"
     sizes="(max-width: 600px) 100vw,
            (max-width: 1200px) 50vw,
            33vw"
     alt="レスポンシブ写真">
```

#### srcset + sizes の仕組み
1. `srcset`: 利用可能な画像とそのサイズ（`400w` = 幅400px）
2. `sizes`: 画面幅に応じた画像の表示サイズ
3. ブラウザが最適な画像を自動選択

## 3.9 遅延読み込み（Lazy Loading）

### loading属性
```html
<img src="photo.jpg" alt="写真" loading="lazy">
```

| 値 | 動作 |
|---|---|
| `eager` | ページ読み込み時にすぐ読み込む（デフォルト） |
| `lazy` | 画像が画面近くに来るまで読み込みを遅延 |

### メリット
- 初期表示速度が向上する
- 帯域幅を節約できる（見えない画像を読み込まない）
- Core Web Vitals（LCP）の改善に効果的

### 使い方の指針
```html
<!-- ファーストビュー（画面上部）の画像：遅延読み込みしない -->
<img src="hero.jpg" alt="ヒーロー画像" loading="eager">

<!-- スクロールしないと見えない画像：遅延読み込みする -->
<img src="photo1.jpg" alt="ギャラリー画像1" loading="lazy">
<img src="photo2.jpg" alt="ギャラリー画像2" loading="lazy">
<img src="photo3.jpg" alt="ギャラリー画像3" loading="lazy">
```

> **注意**: ファーストビュー（ページ読み込み時に表示される領域）の画像には `loading="lazy"` を使わないでください。LCP（Largest Contentful Paint）が遅くなります。

### iframeにも使える
```html
<iframe src="https://www.youtube.com/embed/xxx" loading="lazy"
        width="560" height="315" title="YouTube動画"></iframe>
```

## 3.10 画像の最適化ベストプラクティス

### チェックリスト
1. **適切なフォーマット**を選ぶ（写真→JPEG/WebP、イラスト→PNG/SVG）
2. **圧縮**する（ファイルサイズを小さくする）
3. **適切な解像度**にする（表示サイズより大きすぎる画像を使わない）
4. **width / height**を指定する（CLSを防ぐ）
5. **alt属性**を適切に書く（アクセシビリティ）
6. **loading="lazy"**を適切に使う（パフォーマンス）
7. **picture要素**でフォーマットを切り替える（対応ブラウザに最適化）

### 画像圧縮の目安
| 用途 | 推奨サイズ |
|---|---|
| サムネイル | 〜50KB |
| 一般的な写真 | 50〜200KB |
| ヒーロー画像 | 100〜500KB |
| 背景画像 | 100〜300KB |
