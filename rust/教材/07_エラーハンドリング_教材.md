# 第7章：エラーハンドリング

## この章のゴール

この章を学び終えると、以下のことができるようになります。

- panic と通常のエラーハンドリングの違いを理解できる
- Result\<T, E\> 型を使ってエラーを安全に処理できる
- unwrap, expect の役割と使いどころを判断できる
- ? 演算子を使ってエラーを簡潔に伝播できる
- カスタムエラー型を定義し、独自のエラー体系を構築できる
- From トレイトを使ったエラー変換を実装できる
- 実践的なエラーハンドリングパターンを適用できる
- panic と Result の使い分けの判断基準を持てる

## 7.1 Rust のエラーハンドリング哲学

### 2種類のエラー

Rustは、エラーを「回復可能なエラー」と「回復不可能なエラー」の2種類に分けています。

```
┌──────────────────────────────────────────────────────┐
│                Rust のエラーモデル                      │
│                                                      │
│  回復不可能なエラー（panic!）                           │
│  ├─ プログラムの即時停止                               │
│  ├─ バグの発見・アサーション失敗                        │
│  └─ 例: 配列の範囲外アクセス、論理的に起こり得ない状態    │
│                                                      │
│  回復可能なエラー（Result<T, E>）                       │
│  ├─ 呼び出し元に判断を委ねる                           │
│  ├─ ファイルが見つからない、ネットワーク障害など           │
│  └─ 例: ファイルIO、パース、ネットワーク通信             │
│                                                      │
│  他言語（Java, Python）の例外と違い:                    │
│  → Rust は例外機構を持たない                           │
│  → 代わりに型システムでエラーを表現                     │
│  → コンパイル時にエラー処理漏れを検出できる              │
└──────────────────────────────────────────────────────┘
```

### ポイントまとめ

- Rust に例外（exception）はない
- `panic!` はプログラムを停止させる
- `Result<T, E>` は型としてエラーを表現する
- コンパイラがエラー処理の漏れを検出してくれる

## 7.2 panic と unwind

### panic! マクロ

`panic!` はプログラムを即座に停止させるマクロです。バグが見つかった場合や、回復不可能な状態で使います。

```rust
fn main() {
    println!("panic の前");
    // panic!("致命的なエラーが発生しました！");
    // ↑ コメントを外すとプログラムが停止する

    // 代わりに安全な例を示す
    let v = vec![1, 2, 3];
    println!("v[0] = {}", v[0]);
    // println!("v[10] = {}", v[10]);  // ← 範囲外アクセスで panic

    println!("この行は実行されます");
}
```

```
panic の前
v[0] = 1
この行は実行されます
```

### panic が発生する典型的なケース

```rust
fn main() {
    // ケース1: 明示的な panic!
    // panic!("何かがおかしい");

    // ケース2: 配列の範囲外アクセス
    let v = vec![1, 2, 3];
    // let _ = v[10];  // パニック！

    // ケース3: unwrap() で None に遭遇
    let x: Option<i32> = None;
    // let _ = x.unwrap();  // パニック！

    // ケース4: unwrap() で Err に遭遇
    let result: Result<i32, &str> = Err("失敗");
    // let _ = result.unwrap();  // パニック！

    println!("すべてのパニックはコメントアウトされています");
    println!("コメントを外すと動作を確認できます");
}
```

```
すべてのパニックはコメントアウトされています
コメントを外すと動作を確認できます
```

### unwind と abort

panic が発生すると、デフォルトでは「アンワインド（unwind）」が起きます。

```
┌─────────────────────────────────────────────────┐
│  unwind（デフォルト）:                             │
│  ├─ スタックを巻き戻す                            │
│  ├─ 各変数の Drop を実行（リソース解放）            │
│  └─ バックトレースを表示                          │
│                                                 │
│  abort（Cargo.toml で設定可能）:                   │
│  ├─ 即座にプロセスを終了                          │
│  ├─ Drop は実行されない                           │
│  └─ バイナリサイズが小さくなる                     │
│                                                 │
│  [profile.release]                              │
│  panic = "abort"  ← リリースビルドで abort にする   │
└─────────────────────────────────────────────────┘
```

### ポイントまとめ

- `panic!` はプログラムを停止させる（バグの発見用）
- 配列範囲外アクセス、`unwrap()` の失敗などで発生する
- 通常のエラー処理には `Result` を使うべき
- `panic!` は「プログラマのミス」を示す場面で使う

## 7.3 Result\<T, E\> 型

### Result とは

`Result<T, E>` は成功（`Ok(T)`）または失敗（`Err(E)`）を表す列挙型です。

```
┌────────────────────────────────────────────┐
│  Result<T, E> の定義（標準ライブラリ）:      │
│                                            │
│  enum Result<T, E> {                       │
│      Ok(T),    // 成功時の値               │
│      Err(E),   // 失敗時のエラー情報         │
│  }                                         │
│                                            │
│  T: 成功した場合の型                         │
│  E: エラーの場合の型                         │
└────────────────────────────────────────────┘
```

### 基本的な使い方

```rust
fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err(String::from("ゼロで割ることはできません"))
    } else {
        Ok(a / b)
    }
}

fn main() {
    // 成功するケース
    match divide(10.0, 3.0) {
        Ok(result) => println!("10 / 3 = {:.4}", result),
        Err(e) => println!("エラー: {}", e),
    }

    // 失敗するケース
    match divide(10.0, 0.0) {
        Ok(result) => println!("10 / 0 = {}", result),
        Err(e) => println!("エラー: {}", e),
    }
}
```

```
10 / 3 = 3.3333
エラー: ゼロで割ることはできません
```

### 文字列のパース

標準ライブラリの多くの関数が `Result` を返します。

```rust
fn main() {
    // 文字列から数値へのパース
    let num1: Result<i32, _> = "42".parse();
    let num2: Result<i32, _> = "abc".parse();

    println!("\"42\" のパース: {:?}", num1);
    println!("\"abc\" のパース: {:?}", num2);

    // match で処理
    let input = "123";
    match input.parse::<i32>() {
        Ok(n) => println!("{} は数値 {} です", input, n),
        Err(e) => println!("{} はパースできません: {}", input, e),
    }

    // 複数の値をパース
    let values = vec!["10", "20", "abc", "30", "xyz"];
    let mut sum = 0;
    let mut errors = 0;

    for v in &values {
        match v.parse::<i32>() {
            Ok(n) => {
                sum += n;
                println!("  {} → {}", v, n);
            }
            Err(_) => {
                errors += 1;
                println!("  {} → パース失敗", v);
            }
        }
    }
    println!("合計: {}, エラー数: {}", sum, errors);
}
```

```
"42" のパース: Ok(42)
"abc" のパース: Err(invalid digit found in string)
123 は数値 123 です
  10 → 10
  20 → 20
  abc → パース失敗
  30 → 30
  xyz → パース失敗
合計: 60, エラー数: 2
```

### Result の便利メソッド

```rust
fn main() {
    let ok_val: Result<i32, String> = Ok(42);
    let err_val: Result<i32, String> = Err(String::from("エラー"));

    // is_ok / is_err
    println!("ok_val.is_ok(): {}", ok_val.is_ok());
    println!("err_val.is_err(): {}", err_val.is_err());

    // unwrap_or: Err の場合のデフォルト値
    println!("ok: {}", ok_val.unwrap_or(0));
    println!("err: {}", err_val.unwrap_or(0));

    // map: Ok の中身を変換
    let doubled = ok_val.map(|x| x * 2);
    println!("doubled: {:?}", doubled);

    let err_doubled = err_val.map(|x| x * 2);
    println!("err_doubled: {:?}", err_doubled);

    // map_err: Err の中身を変換
    let new_err = Err::<i32, String>(String::from("original"))
        .map_err(|e| format!("変換後: {}", e));
    println!("map_err: {:?}", new_err);

    // and_then: Ok ならさらに Result を返す処理へ
    let result = ok_val.and_then(|x| {
        if x > 0 {
            Ok(x * 10)
        } else {
            Err(String::from("正の数が必要です"))
        }
    });
    println!("and_then: {:?}", result);

    // unwrap_or_else: Err ならクロージャで計算
    let value = err_val.unwrap_or_else(|e| {
        println!("エラーが発生: {}", e);
        -1
    });
    println!("unwrap_or_else: {}", value);
}
```

```
ok_val.is_ok(): true
err_val.is_err(): true
ok: 42
err: 0
doubled: Ok(84)
err_doubled: Err("エラー")
map_err: Err("変換後: original")
and_then: Ok(420)
エラーが発生: エラー
unwrap_or_else: -1
```

### ポイントまとめ

| メソッド | Ok の場合 | Err の場合 |
|----------|----------|-----------|
| `unwrap()` | 中身を返す | パニック |
| `unwrap_or(default)` | 中身を返す | default を返す |
| `unwrap_or_else(f)` | 中身を返す | f(err) を返す |
| `map(f)` | Ok(f(val)) | そのまま Err |
| `map_err(f)` | そのまま Ok | Err(f(err)) |
| `and_then(f)` | f(val) | そのまま Err |
| `is_ok()` | true | false |

## 7.4 unwrap と expect

### unwrap

`unwrap()` は `Ok` なら中身を取り出し、`Err` ならパニックします。

```rust
fn main() {
    // OK のケース
    let result: Result<i32, &str> = Ok(42);
    let value = result.unwrap();
    println!("value: {}", value);

    // プロトタイプや例示コードでは便利
    let num: i32 = "42".parse().unwrap();
    println!("num: {}", num);

    // Err のケース → パニック！
    // let bad: Result<i32, &str> = Err("失敗");
    // bad.unwrap();  // thread 'main' panicked at '失敗'
}
```

```
value: 42
num: 42
```

### expect

`expect()` は `unwrap()` と同じですが、パニック時のメッセージを指定できます。

```rust
fn main() {
    // expect はエラーメッセージを指定できる
    let config_port: i32 = "8080".parse()
        .expect("PORT の値が不正です");
    println!("port: {}", config_port);

    // エラーの場合のメッセージが分かりやすい
    // let bad: i32 = "abc".parse()
    //     .expect("PORT の値が不正です");
    // → thread 'main' panicked at 'PORT の値が不正です: ...'
}
```

```
port: 8080
```

### unwrap / expect の使い分けガイド

```
┌──────────────────────────────────────────────────────┐
│  unwrap / expect を使ってよい場面:                     │
│                                                      │
│  1. プロトタイプ・学習中のコード                        │
│     → 素早く書くため                                  │
│                                                      │
│  2. テストコード                                      │
│     → テスト失敗は panic で OK                         │
│                                                      │
│  3. 論理的に Err にならないことが確実な場合              │
│     → let num: i32 = "42".parse().unwrap();           │
│     → コメントで理由を書くこと                         │
│                                                      │
│  本番コードでは:                                       │
│  → unwrap / expect は避ける                           │
│  → ? 演算子、unwrap_or、match を使う                   │
└──────────────────────────────────────────────────────┘
```

### ポイントまとめ

- `unwrap()`: 簡潔だがパニックのリスクがある
- `expect("メッセージ")`: パニック時にメッセージが出る（デバッグしやすい）
- 本番コードでは `?` 演算子や `unwrap_or` を優先する
- テストやプロトタイプでは `unwrap` / `expect` を積極的に使ってよい

## 7.5 ? 演算子（エラー伝播）

### ? 演算子の基本

`?` 演算子は、`Result` が `Err` の場合にその関数から即座にエラーを返します。`match` で書くと冗長なコードを、1文字で簡潔にできます。

```rust
use std::num::ParseIntError;

// ? 演算子なしの場合（冗長）
fn parse_and_double_verbose(s: &str) -> Result<i32, ParseIntError> {
    let n = match s.parse::<i32>() {
        Ok(val) => val,
        Err(e) => return Err(e),  // エラーなら即リターン
    };
    Ok(n * 2)
}

// ? 演算子ありの場合（簡潔）
fn parse_and_double(s: &str) -> Result<i32, ParseIntError> {
    let n = s.parse::<i32>()?;  // Err なら即リターン
    Ok(n * 2)
}

fn main() {
    println!("verbose: {:?}", parse_and_double_verbose("21"));
    println!("concise: {:?}", parse_and_double("21"));
    println!("error:   {:?}", parse_and_double("abc"));
}
```

```
verbose: Ok(42)
concise: Ok(42)
error: Err(invalid digit found in string)
```

### ? 演算子のチェーン

```rust
use std::num::ParseIntError;

fn calculate(a: &str, b: &str) -> Result<i32, ParseIntError> {
    let x = a.parse::<i32>()?;
    let y = b.parse::<i32>()?;
    Ok(x + y)
}

fn main() {
    match calculate("10", "20") {
        Ok(result) => println!("10 + 20 = {}", result),
        Err(e) => println!("エラー: {}", e),
    }

    match calculate("10", "abc") {
        Ok(result) => println!("結果: {}", result),
        Err(e) => println!("エラー: {}", e),
    }
}
```

```
10 + 20 = 30
エラー: invalid digit found in string
```

### ? 演算子の動作を図で理解

```
┌─────────────────────────────────────────────────────┐
│  let value = expression?;                           │
│                                                     │
│  ↓ これは以下と同じ意味:                              │
│                                                     │
│  let value = match expression {                     │
│      Ok(v) => v,              // 成功: 値を取り出す   │
│      Err(e) => return Err(e.into()),  // 失敗: 返す  │
│  };                                                 │
│                                                     │
│  ポイント:                                           │
│  1. Ok なら中身を取り出して処理を続行                   │
│  2. Err なら関数から即座にリターン                      │
│  3. .into() でエラー型の自動変換が行われる              │
└─────────────────────────────────────────────────────┘
```

### Option に対する ? 演算子

`?` は `Option` に対しても使えます。`None` なら即座に `None` を返します。

```rust
fn first_even(numbers: &[i32]) -> Option<i32> {
    let first = numbers.first()?;  // 空なら None を返す
    if first % 2 == 0 {
        Some(*first)
    } else {
        None
    }
}

fn get_middle_char(s: &str) -> Option<char> {
    let len = s.len();
    if len == 0 {
        return None;
    }
    s.chars().nth(len / 2)
}

fn main() {
    let nums1 = vec![4, 1, 7, 2];
    let nums2: Vec<i32> = vec![];
    let nums3 = vec![3, 6, 9];

    println!("first_even({:?}): {:?}", nums1, first_even(&nums1));
    println!("first_even({:?}): {:?}", nums2, first_even(&nums2));
    println!("first_even({:?}): {:?}", nums3, first_even(&nums3));

    println!("middle of 'hello': {:?}", get_middle_char("hello"));
    println!("middle of '': {:?}", get_middle_char(""));
}
```

```
first_even([4, 1, 7, 2]): Some(4)
first_even([]): None
first_even([3, 6, 9]): None
middle of 'hello': Some('l')
middle of '': None
```

### 実践例：複数のステップを ? でつなぐ

```rust
#[derive(Debug)]
struct Config {
    host: String,
    port: u16,
}

fn parse_config(input: &str) -> Result<Config, String> {
    // "host:port" の形式をパース
    let parts: Vec<&str> = input.split(':').collect();

    if parts.len() != 2 {
        return Err(format!("不正なフォーマット: '{}'（host:port が必要）", input));
    }

    let host = parts[0].to_string();
    if host.is_empty() {
        return Err("ホスト名が空です".to_string());
    }

    let port: u16 = parts[1].parse()
        .map_err(|e| format!("ポート番号が不正です: {}", e))?;

    if port == 0 {
        return Err("ポート番号は1以上が必要です".to_string());
    }

    Ok(Config { host, port })
}

fn main() {
    let test_cases = vec![
        "localhost:8080",
        "example.com:443",
        "invalid",
        "host:abc",
        ":8080",
        "host:0",
    ];

    for input in test_cases {
        match parse_config(input) {
            Ok(config) => println!("OK: '{}' → {:?}", input, config),
            Err(e) => println!("NG: '{}' → {}", input, e),
        }
    }
}
```

```
OK: 'localhost:8080' → Config { host: "localhost", port: 8080 }
OK: 'example.com:443' → Config { host: "example.com", port: 443 }
NG: 'invalid' → 不正なフォーマット: 'invalid'（host:port が必要）
NG: 'host:abc' → ポート番号が不正です: invalid digit found in string
NG: ':8080' → ホスト名が空です
NG: 'host:0' → ポート番号は1以上が必要です
```

### ポイントまとめ

- `?` は `Err` を即座にリターンする糖衣構文
- `match` で書く冗長なエラー処理を1文字に圧縮できる
- `Result` と `Option` の両方で使える
- 戻り値の型が `Result` または `Option` の関数内でのみ使用可能
- `map_err` と組み合わせてエラーメッセージを改善できる

## 7.6 カスタムエラー型の定義

### なぜカスタムエラー型が必要か

実際のアプリケーションでは、複数の種類のエラーを統一的に扱う必要があります。

```
┌──────────────────────────────────────────────────┐
│  問題: 異なるエラー型を返す関数を組み合わせたい      │
│                                                  │
│  parse() → ParseIntError                         │
│  ファイルIO → io::Error                           │
│  独自のバリデーション → ???                         │
│                                                  │
│  解決: カスタムエラー型で統一する                    │
│                                                  │
│  enum AppError {                                 │
│      Parse(ParseIntError),                       │
│      Io(io::Error),                              │
│      Validation(String),                         │
│  }                                               │
└──────────────────────────────────────────────────┘
```

### 基本的なカスタムエラー型

```rust
use std::fmt;
use std::num::ParseIntError;

#[derive(Debug)]
enum AppError {
    ParseError(ParseIntError),
    ValidationError(String),
    NotFound(String),
}

impl fmt::Display for AppError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            AppError::ParseError(e) => write!(f, "パースエラー: {}", e),
            AppError::ValidationError(msg) => write!(f, "バリデーションエラー: {}", msg),
            AppError::NotFound(item) => write!(f, "見つかりません: {}", item),
        }
    }
}

fn parse_age(input: &str) -> Result<u32, AppError> {
    let age: i32 = input.parse()
        .map_err(AppError::ParseError)?;

    if age < 0 {
        return Err(AppError::ValidationError(
            "年齢は0以上である必要があります".to_string(),
        ));
    }
    if age > 150 {
        return Err(AppError::ValidationError(
            "年齢が大きすぎます".to_string(),
        ));
    }

    Ok(age as u32)
}

fn main() {
    let test_inputs = vec!["25", "abc", "-5", "200", "0"];

    for input in test_inputs {
        match parse_age(input) {
            Ok(age) => println!("'{}' → 年齢: {}", input, age),
            Err(e) => println!("'{}' → エラー: {}", input, e),
        }
    }
}
```

```
'25' → 年齢: 25
'abc' → エラー: パースエラー: invalid digit found in string
'-5' → エラー: バリデーションエラー: 年齢は0以上である必要があります
'200' → エラー: バリデーションエラー: 年齢が大きすぎます
'0' → 年齢: 0
```

### ポイントまとめ

- カスタムエラー型は `enum` で定義する
- `Display` トレイトを実装してユーザー向けメッセージを提供する
- `Debug` は `#[derive(Debug)]` で自動導出する
- 各バリアントに元のエラーやメッセージを格納する

## 7.7 エラーの変換（From トレイト）

### From トレイトとは

`From` トレイトを実装すると、`?` 演算子が自動的にエラー型を変換してくれます。

```rust
use std::fmt;
use std::num::ParseIntError;

#[derive(Debug)]
enum CalcError {
    Parse(ParseIntError),
    DivisionByZero,
    Overflow,
}

impl fmt::Display for CalcError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            CalcError::Parse(e) => write!(f, "パースエラー: {}", e),
            CalcError::DivisionByZero => write!(f, "ゼロ除算エラー"),
            CalcError::Overflow => write!(f, "オーバーフローエラー"),
        }
    }
}

// From トレイトの実装: ParseIntError → CalcError の自動変換
impl From<ParseIntError> for CalcError {
    fn from(e: ParseIntError) -> Self {
        CalcError::Parse(e)
    }
}

fn safe_divide(a: &str, b: &str) -> Result<i32, CalcError> {
    let x: i32 = a.parse()?;  // ParseIntError → CalcError に自動変換
    let y: i32 = b.parse()?;  // 同上

    if y == 0 {
        return Err(CalcError::DivisionByZero);
    }

    x.checked_div(y).ok_or(CalcError::Overflow)
}

fn main() {
    let test_cases = vec![
        ("10", "3"),
        ("abc", "2"),
        ("10", "0"),
        ("100", "7"),
    ];

    for (a, b) in test_cases {
        match safe_divide(a, b) {
            Ok(result) => println!("{} / {} = {}", a, b, result),
            Err(e) => println!("{} / {} → エラー: {}", a, b, e),
        }
    }
}
```

```
10 / 3 = 3
abc / 2 → エラー: パースエラー: invalid digit found in string
10 / 0 → エラー: ゼロ除算エラー
100 / 7 = 14
```

### 複数のエラー型からの変換

```rust
use std::fmt;
use std::num::{ParseIntError, ParseFloatError};

#[derive(Debug)]
enum DataError {
    IntParse(ParseIntError),
    FloatParse(ParseFloatError),
    InvalidFormat(String),
}

impl fmt::Display for DataError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            DataError::IntParse(e) => write!(f, "整数パースエラー: {}", e),
            DataError::FloatParse(e) => write!(f, "小数パースエラー: {}", e),
            DataError::InvalidFormat(msg) => write!(f, "フォーマットエラー: {}", msg),
        }
    }
}

impl From<ParseIntError> for DataError {
    fn from(e: ParseIntError) -> Self {
        DataError::IntParse(e)
    }
}

impl From<ParseFloatError> for DataError {
    fn from(e: ParseFloatError) -> Self {
        DataError::FloatParse(e)
    }
}

// "名前,年齢,身長" の形式をパース
fn parse_record(line: &str) -> Result<(String, i32, f64), DataError> {
    let parts: Vec<&str> = line.split(',').collect();
    if parts.len() != 3 {
        return Err(DataError::InvalidFormat(
            format!("3つのフィールドが必要です（現在: {}）", parts.len()),
        ));
    }

    let name = parts[0].to_string();
    let age: i32 = parts[1].trim().parse()?;      // ParseIntError → DataError
    let height: f64 = parts[2].trim().parse()?;    // ParseFloatError → DataError

    Ok((name, age, height))
}

fn main() {
    let lines = vec![
        "Alice, 30, 165.5",
        "Bob, abc, 170.0",
        "Charlie, 25, xyz",
        "Diana",
        "Eve, 28, 158.0",
    ];

    println!("=== データ読み込み ===");
    for line in &lines {
        match parse_record(line) {
            Ok((name, age, height)) => {
                println!("  OK: {} ({}歳, {:.1}cm)", name, age, height);
            }
            Err(e) => {
                println!("  NG: '{}' → {}", line, e);
            }
        }
    }
}
```

```
=== データ読み込み ===
  OK: Alice (30歳, 165.5cm)
  NG: 'Bob, abc, 170.0' → 整数パースエラー: invalid digit found in string
  NG: 'Charlie, 25, xyz' → 小数パースエラー: invalid float literal
  NG: 'Diana' → フォーマットエラー: 3つのフィールドが必要です（現在: 1）
  OK: Eve (28歳, 158.0cm)
```

### ポイントまとめ

- `From` トレイトを実装すると `?` 演算子が自動変換してくれる
- `impl From<SourceError> for MyError` の形で書く
- 複数のエラー型を1つのカスタムエラー型に統合できる
- これにより `?` だけで異なるエラー型を透過的に扱える

        age,
        email,
    })

fn validate_all(fields: &[(&str, &str)]) -> Result<Vec<(String, String)>, Vec<String>> {
    let mut results = Vec::new();
    let mut errors = Vec::new();

    for (key, value) in fields {
        if value.trim().is_empty() {
            errors.push(format!("'{}' は必須項目です", key));
        } else {
            results.push((key.to_string(), value.trim().to_string()));
        }
    }

    if errors.is_empty() {
        Ok(results)
    } else {
        Err(errors)
    }
}

fn main() {
    let good_data = vec![
        ("名前", "Alice"),
        ("メール", "alice@example.com"),
        ("電話", "090-1234-5678"),
    ];

    let bad_data = vec![
        ("名前", ""),
        ("メール", "bob@example.com"),
        ("電話", ""),
    ];

    match validate_all(&good_data) {
        Ok(fields) => {
            println!("=== バリデーション成功 ===");
            for (k, v) in &fields {
                println!("  {}: {}", k, v);
            }
        }
        Err(errors) => {
            println!("=== バリデーション失敗 ===");
            for e in &errors {
                println!("  - {}", e);
            }
        }
    }

    println!();

    match validate_all(&bad_data) {
        Ok(fields) => {
            println!("=== バリデーション成功 ===");
            for (k, v) in &fields {
                println!("  {}: {}", k, v);
            }
        }
        Err(errors) => {
            println!("=== バリデーション失敗 ===");
            for e in &errors {
                println!("  - {}", e);
            }
        }
    }
}

use std::num::ParseIntError;

#[derive(Debug)]
struct ContextError {

    source: Box<dyn fmt::Display>,

impl fmt::Display for ContextError {

fn parse_with_context(input: &str, field_name: &str) -> Result<i32, ContextError> {

    match parse_with_context("xyz", "ポート番号") {

## 7.9 panic vs Result の使い分け

### 判断基準

```
┌──────────────────────────────────────────────────────┐
│              panic! を使うべき場面                      │
├──────────────────────────────────────────────────────┤
│                                                      │
│  1. プログラムのバグ（論理エラー）                      │
│     → 配列の不正アクセス、不変条件の違反               │
│                                                      │
│  2. 回復不可能な初期化エラー                            │
│     → 設定ファイルの読み込み失敗（起動時）              │
│                                                      │
│  3. テストコード                                      │
│     → assert! 系マクロの失敗                          │
│                                                      │
│  4. プロトタイプ段階                                   │
│     → 後で Result に置き換える前提の unwrap            │
│                                                      │
├──────────────────────────────────────────────────────┤
│              Result を使うべき場面                      │
├──────────────────────────────────────────────────────┤
│                                                      │
│  1. ユーザー入力のエラー                               │
│     → パース失敗、バリデーション違反                    │
│                                                      │
│  2. 外部リソースのエラー                               │
│     → ファイルIO、ネットワーク、データベース             │
│                                                      │
│  3. ビジネスロジックのエラー                            │
│     → 在庫不足、認証失敗、権限エラー                    │
│                                                      │
│  4. ライブラリの API                                   │
│     → 呼び出し元に判断を委ねる                         │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 実践例

```rust
use std::fmt;

// === Result を使うべき例 ===

#[derive(Debug)]
enum TransferError {
    InsufficientFunds { balance: i64, amount: i64 },
    InvalidAmount,
    AccountNotFound(String),
}

impl fmt::Display for TransferError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            TransferError::InsufficientFunds { balance, amount } => {
                write!(f, "残高不足（残高: {}円, 送金額: {}円）", balance, amount)
            }
            TransferError::InvalidAmount => write!(f, "送金額が不正です"),
            TransferError::AccountNotFound(id) => {
                write!(f, "口座 '{}' が見つかりません", id)
            }
        }
    }
}

struct Account {
    id: String,
    balance: i64,
}

fn transfer(from: &mut Account, to: &mut Account, amount: i64) -> Result<(), TransferError> {
    if amount <= 0 {
        return Err(TransferError::InvalidAmount);
    }
    if from.balance < amount {
        return Err(TransferError::InsufficientFunds {
            balance: from.balance,
            amount,
        });
    }

    from.balance -= amount;
    to.balance += amount;
    Ok(())
}

fn main() {
    let mut alice = Account { id: "alice".to_string(), balance: 10000 };
    let mut bob = Account { id: "bob".to_string(), balance: 5000 };

    println!("=== 送金処理 ===");
    println!("Alice: {}円, Bob: {}円", alice.balance, bob.balance);

    // 正常な送金
    match transfer(&mut alice, &mut bob, 3000) {
        Ok(()) => println!("3000円の送金成功"),
        Err(e) => println!("送金失敗: {}", e),
    }
    println!("Alice: {}円, Bob: {}円", alice.balance, bob.balance);

    // 残高不足
    match transfer(&mut alice, &mut bob, 50000) {
        Ok(()) => println!("50000円の送金成功"),
        Err(e) => println!("送金失敗: {}", e),
    }

    // 不正な金額
    match transfer(&mut alice, &mut bob, -100) {
        Ok(()) => println!("送金成功"),
        Err(e) => println!("送金失敗: {}", e),
    }

    // === panic! を使ってよい例 ===

    // テストや不変条件の検証
    let values = vec![1, 2, 3];
    assert!(!values.is_empty(), "values は空であってはならない");
    // ↑ これが失敗するならプログラムのバグ

    println!("\n全ての処理が完了しました");
}
```

```
=== 送金処理 ===
Alice: 10000円, Bob: 5000円
3000円の送金成功
Alice: 7000円, Bob: 8000円
送金失敗: 残高不足（残高: 7000円, 送金額: 50000円）
送金失敗: 送金額が不正です

全ての処理が完了しました
```

### ポイントまとめ

| 状況 | 使うべきもの | 理由 |
|------|-------------|------|
| ユーザー入力のミス | `Result` | 回復可能 |
| ファイルが見つからない | `Result` | 呼び出し元が対処できる |
| 配列インデックスが不正 | `panic!` | プログラムのバグ |
| 設定の必須値が欠落 | 起動時: `panic!`, 実行時: `Result` | 状況による |
| テストの検証 | `assert!` (panic) | 失敗 = テスト不合格 |
