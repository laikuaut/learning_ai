## 設問

以下の設問に回答してください。

<details>
<summary>設問1：このアプリのアーキテクチャを説明してください</summary>

**模範回答:**

このアプリは以下の3つの層で構成されています。

1. **サービス層** (`EmployeeService` factory): データの保持と検索ロジックを担当。コントローラから独立しており、他のコントローラやディレクティブからも再利用可能。
2. **コントローラ層** (`MainCtrl`): ビューとサービスをつなぐ役割。ユーザー入力（検索クエリ、部署、ソート、表示切替）の状態を管理し、サービスの検索結果をビューに渡す。
3. **ディレクティブ層** (`employeeDetail`): 社員詳細のモーダルUIをカプセル化。スコープ分離（`=` と `&`）で親コントローラとの通信を行う。

Controller As 構文を使い、`$scope` への依存を排除しています。

</details>

<details>
<summary>設問2��`ng-model-options="{ debounce: 300 }"` は何をしていますか？</summary>

**模範回答:**

ユーザーが検索ボックスにキーを入力するたびにモデルを更新するのではなく、入力が止まってから300ミリ秒待ってからモデルを更新します。これにより、高速タイピング中に何度もフィルタリング処理が走ることを防ぎ、パフォーマンスが向上します（デバウンス処理）。

</details>

<details>
<summary>設問3：`EmployeeService` の `search` メソッドはどのような検索を行いますか？</summary>

**模範回答:**

2つの条件をAND検索しています。

1. **部署フィルタ**: `dept` が指定されていれば、その部署に完全一致する社員のみ残す
2. **キーワード検索**: `query` が指定されていれば、社員名またはスキルのいずれかに部分一致（大文字小文字を無視）する社員を残す

名前とスキルはOR条件（どちらかに一致すればOK）で、部署とキーワードはAND条件（両方満たす必要がある）です。

</details>

<details>
<summary>設問4：`employeeDetail` ディレクティブの `scope` 定義を説明してください</summary>

**模範回答:**

```javascript
scope: {
  emp: '=',       // 双方向バインディング: 親の selectedEmp と同期
  onClose: '&'    // コールバック: 親の closeDetail() を呼び出す式
}
```

- `=`（双方向）: 親コントローラの `main.selectedEmp` と子ディレクティブの `emp` が同期される。親でデータが変わると子にも反映される。
- `&`（式バインディング）: 親のスコープで `main.closeDetail()` を評価する。子ディレクティブからの「閉じる」アクションを親に伝達する。

</details>

<details>
<summary>設問5：モーダルのオーバーレイクリックで閉じる実装を説明してください</summary>

**模範回答:**

```javascript
scope.close = function($event) {
  if ($event.target === $event.currentTarget) {
    scope.onClose();
  }
};
```

`$event.target` はクリックされた実際の要素、`$event.currentTarget` は `ng-click` が設定された要素（`modal-overlay`）です。オーバーレイ自体がクリックされた場合のみ `onClose()` を呼びます。モーダル内部をクリックした場合はイベントがバブリングしてオーバーレイまで到達しますが、`target` と `currentTarget` が異なるため閉じません。

</details>

<details>
<summary>設問6：`| limitTo:3` は何のために使われていますか？</summary>

**模範回答:**

リスト表示ではスキルを最大3つまでに制限し、グリッド表示では2つまでに制限しています（`limitTo:2`）。これにより、UIがスキルの数に関わらず一定のサイズに収まり、レイアウトが崩れることを防いでいます。`limitTo` はAngularJSの組み込みフィルターで、配列の先頭からN件を取り出します。

</details>

<details>
<summary>設問7：`Controller As` 構文の利点をこのコードから説明してください</summary>

**模範回答:**

`MainCtrl as main` により、テンプレートでは `main.query`、`main.dept` のようにプレフィックス付きでプロパティにアクセスします。

利点:
1. **所有者の明確化**: テンプレート内でどのコントローラのプロパティかが一目でわかる
2. **スコープ継承の問題回避**: `ng-if` や `ng-repeat` が作る子スコープ内でも `main.query` は正しく親の値を参照する（ドット記法ルール）
3. **$scope への依存排除**: コントローラ内で `this` にプロパティをバインドするため、`$scope` を注入する必要がない

</details>

<details>
<summary>設問8：`factory` と `service` の違いを、このコードの `EmployeeService` を例に説明してください</summary>

**模範回答:**

このコードでは `factory` を使っています。`factory` は戻り値のオブジェクトがそのままサービスのインスタンスになります。

```javascript
app.factory('EmployeeService', function() {
  var employees = [...]; // プライベート変数
  return {
    getAll: function() { ... },    // 公開メソッド
    search: function() { ... }     // 公開メソッド
  };
});
```

`service` で書き直す場合は `this` にメソッドを設定します:

```javascript
app.service('EmployeeService', function() {
  var employees = [...];
  this.getAll = function() { ... };
  this.search = function() { ... };
});
```

`factory` の利点は、クロージャでプライベート変数を隠蔽できる点です（`employees` 配列は外部から直接アクセスできない）。

</details>

<details>
<summary>設問9：検索パフォーマンスを改善するにはどうすればよいですか？</summary>

**模範回答:**

現在の実装では `main.getFiltered()` がテンプレートから直接呼ばれているため、ダイジェストサイクルのたびに再計算されます。改善策:

1. **ワンタイムバインディング**: 変更されないデータには `::` を使う（例: `{{ ::emp.name }}`）
2. **ng-repeat の track by**: `track by emp.id` を追加してDOM再利用を促進する
3. **フィルタ結果のキャッシュ**: `$watch` で検索条件の変更を監視し、結果を変数に保存する:

```javascript
main.$onInit = function() {
  $scope.$watchGroup(['main.query', 'main.dept'], function() {
    main.filteredList = EmployeeService.search(main.query, main.dept);
  });
};
```

4. **仮想スクロール**: 大量データの場合はすべてをDOMに追加せず、画面に見える分だけレンダリングする

</details>

<details>
<summary>設問10：このアプリを改善するとしたら、何をしますか？（3つ以上）</summary>

**模範回答（例）:**

1. **`ng-repeat` に `track by` を追加**: `track by emp.id` でDOMの再利用効率を上げる
2. **ワンタイムバインディング**: 変更されない項目（名前、部署など）に `::` を適用
3. **`.component()` への書き換え**: ディレクティブを `.component()` で書き直し、第8章のベストプラクティスに従う
4. **ルーティングの導入**: 詳細画面をモーダルではなく別ルートにして、URLでの直接アクセスを可能にする
5. **$http でのデータ取得**: ハードコードされたデータを API からの取得に変更する
6. **ページネーション**: 社員数が多い場合に対応
7. **エラーハンドリング**: データ取得失敗時の表示を追加
8. **アクセシビリティ**: モーダルにフォーカストラップとESCキーでの閉じる機能を追加

</details>
