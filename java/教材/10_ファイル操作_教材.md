# 第10章：ファイル操作

## この章のゴール

- Javaでファイルの読み書きができるようになる
- BufferedReader / BufferedWriter の使い方を理解する
- try-with-resources を使ったリソース管理を身につける

## 10.1 ファイル操作の基本概念

プログラムのデータは通常メモリ上にあり、プログラムが終了すると消えてしまいます。データを永続的に保存するには**ファイル**に書き出す必要があります。

```
┌──────────────┐     書き込み     ┌──────────────┐
│  プログラム    │  ──────────→  │  ファイル      │
│  （メモリ上）  │               │  （ディスク上） │
│              │  ←──────────  │              │
└──────────────┘     読み込み     └──────────────┘
```

### Java のファイル操作 API の歴史

| API | パッケージ | 特徴 |
|---|---|---|
| java.io（従来） | `java.io.*` | Java 1.0から存在。File, FileReader 等 |
| java.nio（新） | `java.nio.file.*` | Java 7以降。Path, Files 等。推奨 |

この章では両方を紹介しますが、新しいコードでは **java.nio.file** を使うことを推奨します。

## 10.2 File クラスの基本

`java.io.File` クラスでファイルやディレクトリの情報を取得できます。

```java
import java.io.File;

public class FileBasic {
    public static void main(String[] args) {
        File file = new File("test.txt");

        // ファイルの情報を取得
        System.out.println("ファイル名: " + file.getName());
        System.out.println("絶対パス: " + file.getAbsolutePath());
        System.out.println("存在する？: " + file.exists());
        System.out.println("ファイル？: " + file.isFile());
        System.out.println("ディレクトリ？: " + file.isDirectory());

        if (file.exists()) {
            System.out.println("サイズ: " + file.length() + " バイト");
            System.out.println("読み取り可？: " + file.canRead());
            System.out.println("書き込み可？: " + file.canWrite());
        }
    }
}
```

### ポイントまとめ
- `File` オブジェクトの作成はファイルの作成ではない（パスの表現にすぎない）
- `exists()` で存在確認、`isFile()` / `isDirectory()` で種別判定
- `length()` でファイルサイズ（バイト数）を取得

## 10.3 テキストファイルの読み込み

### BufferedReader を使った読み込み

```java
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class ReadFile {
    public static void main(String[] args) {
        // try-with-resources で自動クローズ
        try (BufferedReader reader = new BufferedReader(new FileReader("sample.txt"))) {
            String line;
            int lineNumber = 1;
            while ((line = reader.readLine()) != null) {
                System.out.printf("%3d: %s%n", lineNumber, line);
                lineNumber++;
            }
        } catch (IOException e) {
            System.err.println("ファイルの読み込みに失敗しました: " + e.getMessage());
        }
    }
}
```

### try-with-resources の仕組み

第9章で学んだ `try-with-resources` は、ファイル操作で特に重要です。

```
┌─ try-with-resources ──────────────────────┐
│                                           │
│  try (リソースの宣言) {                     │
│      // リソースを使った処理                 │
│  } catch (例外) {                          │
│      // エラー処理                          │
│  }                                        │
│  // ← ここで自動的に close() が呼ばれる     │
│                                           │
└───────────────────────────────────────────┘
```

> **よくある間違い：** `try-with-resources` を使わずに手動で `close()` する場合、例外が発生するとリソースが閉じられないことがあります。必ず `try-with-resources` を使いましょう。

### Files クラスを使った簡易読み込み（Java 7+）

```java
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.List;
import java.io.IOException;

public class ReadFileNio {
    public static void main(String[] args) {
        try {
            // 全行をリストとして読み込み
            List<String> lines = Files.readAllLines(Path.of("sample.txt"));
            for (int i = 0; i < lines.size(); i++) {
                System.out.printf("%3d: %s%n", i + 1, lines.get(i));
            }

            // 全内容を1つの文字列として読み込み（Java 11+）
            String content = Files.readString(Path.of("sample.txt"));
            System.out.println("\n--- 全内容 ---");
            System.out.println(content);

        } catch (IOException e) {
            System.err.println("読み込みエラー: " + e.getMessage());
        }
    }
}
```

### ポイントまとめ
- `BufferedReader` の `readLine()` で1行ずつ読む（大きなファイル向き）
- `Files.readAllLines()` で全行を一括読み込み（小さなファイル向き）
- `Files.readString()` で全内容を文字列として取得（Java 11+）

## 10.4 テキストファイルへの書き込み

### BufferedWriter を使った書き込み

```java
import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;

public class WriteFile {
    public static void main(String[] args) {
        // 新規作成（既存ファイルは上書き）
        try (BufferedWriter writer = new BufferedWriter(new FileWriter("output.txt"))) {
            writer.write("1行目：Javaのファイル操作");
            writer.newLine();  // 改行
            writer.write("2行目：BufferedWriterの使い方");
            writer.newLine();
            writer.write("3行目：try-with-resourcesで安全に書き込み");
            writer.newLine();

            System.out.println("ファイルに書き込みました。");
        } catch (IOException e) {
            System.err.println("書き込みエラー: " + e.getMessage());
        }
    }
}
```

### 追記モード

```java
// 第2引数に true を指定すると追記モード
try (BufferedWriter writer = new BufferedWriter(new FileWriter("output.txt", true))) {
    writer.write("追記した行");
    writer.newLine();
}
```

### Files クラスを使った簡易書き込み

```java
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.StandardOpenOption;
import java.util.List;
import java.io.IOException;

public class WriteFileNio {
    public static void main(String[] args) {
        try {
            // 文字列を書き込み（Java 11+）
            Files.writeString(Path.of("output1.txt"), "Hello, Java!\n新しいファイルです。\n");

            // リストを行として書き込み
            List<String> lines = List.of("1行目", "2行目", "3行目");
            Files.write(Path.of("output2.txt"), lines);

            // 追記モード
            Files.writeString(
                Path.of("output1.txt"),
                "追記行\n",
                StandardOpenOption.APPEND
            );

            System.out.println("書き込み完了！");
        } catch (IOException e) {
            System.err.println("書き込みエラー: " + e.getMessage());
        }
    }
}
```

### ポイントまとめ
- `BufferedWriter` で効率的にファイルに書き込む
- `FileWriter` の第2引数 `true` で追記モード
- `Files.writeString()` / `Files.write()` で簡易書き込み

## 10.5 Path と Paths クラス

Java 7 以降の `java.nio.file.Path` はファイルパスを表現するモダンな方法です。

```java
import java.nio.file.Path;

public class PathExample {
    public static void main(String[] args) {
        // Path の作成
        Path path1 = Path.of("data", "input", "sample.txt");
        System.out.println("パス: " + path1);
        // → data/input/sample.txt（OSに応じた区切り文字）

        // パスの情報
        System.out.println("ファイル名: " + path1.getFileName());
        System.out.println("親ディレクトリ: " + path1.getParent());
        System.out.println("絶対パス: " + path1.toAbsolutePath());

        // パスの結合
        Path base = Path.of("/home/user");
        Path full = base.resolve("documents/report.txt");
        System.out.println("結合: " + full);
        // → /home/user/documents/report.txt
    }
}
```

### ポイントまとめ
- `Path.of()` でパスオブジェクトを作成（Java 11+）
- `resolve()` でパスを結合
- OS に応じた区切り文字を自動で使用

## 10.6 ファイル・ディレクトリの操作

```java
import java.nio.file.Files;
import java.nio.file.Path;
import java.io.IOException;

public class FileOperations {
    public static void main(String[] args) {
        try {
            // ディレクトリの作成
            Path dir = Path.of("testdir");
            if (!Files.exists(dir)) {
                Files.createDirectory(dir);
                System.out.println("ディレクトリ作成: " + dir);
            }

            // ファイルの作成
            Path file = dir.resolve("test.txt");
            Files.writeString(file, "テストファイルです。");
            System.out.println("ファイル作成: " + file);

            // ファイル情報
            System.out.println("サイズ: " + Files.size(file) + " バイト");
            System.out.println("最終更新: " + Files.getLastModifiedTime(file));

            // ファイルのコピー
            Path copy = dir.resolve("test_copy.txt");
            Files.copy(file, copy);
            System.out.println("コピー完了: " + copy);

            // ファイルの移動（リネーム）
            Path moved = dir.resolve("test_renamed.txt");
            Files.move(copy, moved);
            System.out.println("移動完了: " + moved);

            // ディレクトリ内のファイル一覧
            System.out.println("\nディレクトリ内容:");
            try (var stream = Files.list(dir)) {
                stream.forEach(p -> System.out.println("  " + p.getFileName()));
            }

            // ファイルの削除
            Files.delete(moved);
            Files.delete(file);
            Files.delete(dir);
            System.out.println("\nクリーンアップ完了");

        } catch (IOException e) {
            System.err.println("エラー: " + e.getMessage());
        }
    }
}
```

### よく使うファイル操作メソッド

| メソッド | 機能 |
|---|---|
| `Files.exists(path)` | 存在確認 |
| `Files.createDirectory(path)` | ディレクトリ作成 |
| `Files.createDirectories(path)` | 中間ディレクトリも含めて作成 |
| `Files.copy(src, dst)` | コピー |
| `Files.move(src, dst)` | 移動・リネーム |
| `Files.delete(path)` | 削除 |
| `Files.size(path)` | サイズ取得 |
| `Files.list(dir)` | ディレクトリ一覧 |

## 10.7 CSV ファイルの処理

CSV（Comma-Separated Values）はデータ交換でよく使われるフォーマットです。

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.FileReader;
import java.io.FileWriter;
import java.io.IOException;
import java.util.ArrayList;
import java.util.List;

public class CsvProcessor {
    public static void main(String[] args) {
        // まず CSV ファイルを作成
        createSampleCsv("students.csv");

        // CSV ファイルの読み込みと処理
        System.out.println("=== 成績データ ===\n");
        List<String[]> records = readCsv("students.csv");

        // ヘッダー表示
        if (!records.isEmpty()) {
            String[] header = records.get(0);
            System.out.printf("%-8s %-6s %-6s %-6s %-8s%n",
                header[0], header[1], header[2], header[3], "平均");
            System.out.println("-".repeat(38));
        }

        // データ表示と平均計算
        double totalAvg = 0;
        int dataCount = 0;
        for (int i = 1; i < records.size(); i++) {
            String[] row = records.get(i);
            String name = row[0];
            int math = Integer.parseInt(row[1]);
            int english = Integer.parseInt(row[2]);
            int science = Integer.parseInt(row[3]);
            double avg = (math + english + science) / 3.0;
            totalAvg += avg;
            dataCount++;

            System.out.printf("%-8s %5d  %5d  %5d  %7.1f%n",
                name, math, english, science, avg);
        }

        System.out.println("-".repeat(38));
        System.out.printf("全体平均: %.1f%n", totalAvg / dataCount);
    }

    static void createSampleCsv(String filename) {
        try (BufferedWriter writer = new BufferedWriter(new FileWriter(filename))) {
            writer.write("名前,数学,英語,理科\n");
            writer.write("田中,85,78,92\n");
            writer.write("鈴木,92,88,76\n");
            writer.write("佐藤,76,95,88\n");
            writer.write("山田,88,82,90\n");
        } catch (IOException e) {
            System.err.println("CSV作成エラー: " + e.getMessage());
        }
    }

    static List<String[]> readCsv(String filename) {
        List<String[]> records = new ArrayList<>();
        try (BufferedReader reader = new BufferedReader(new FileReader(filename))) {
            String line;
            while ((line = reader.readLine()) != null) {
                String[] fields = line.split(",");
                records.add(fields);
            }
        } catch (IOException e) {
            System.err.println("CSV読み込みエラー: " + e.getMessage());
        }
        return records;
    }
}
```

実行結果：
```
=== 成績データ ===

名前     数学   英語   理科   平均
--------------------------------------
田中        85     78     92     85.0
鈴木        92     88     76     85.3
佐藤        76     95     88     86.3
山田        88     82     90     86.7
--------------------------------------
全体平均: 85.8
```

### ポイントまとめ
- CSV は `split(",")` で簡単にパースできる
- 実務では Apache Commons CSV や OpenCSV などのライブラリを使うことが多い
- クォートやカンマを含むデータの場合はライブラリの利用を推奨

## 10.8 文字エンコーディング

ファイル操作では**文字エンコーディング（character encoding）**を意識する必要があります。

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.IOException;
import java.nio.charset.Charset;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;

public class EncodingExample {
    public static void main(String[] args) {
        Path file = Path.of("encoding_test.txt");

        try {
            // UTF-8 で書き込み（明示的に指定）
            try (BufferedWriter writer = Files.newBufferedWriter(file, StandardCharsets.UTF_8)) {
                writer.write("こんにちは、Java！");
                writer.newLine();
                writer.write("文字エンコーディングのテストです。");
            }

            // UTF-8 で読み込み
            try (BufferedReader reader = Files.newBufferedReader(file, StandardCharsets.UTF_8)) {
                String line;
                while ((line = reader.readLine()) != null) {
                    System.out.println(line);
                }
            }

            // デフォルトエンコーディングの確認
            System.out.println("\nデフォルト: " + Charset.defaultCharset());

            // クリーンアップ
            Files.delete(file);

        } catch (IOException e) {
            System.err.println("エラー: " + e.getMessage());
        }
    }
}
```

### 主なエンコーディング

| エンコーディング | 定数 | 用途 |
|---|---|---|
| UTF-8 | `StandardCharsets.UTF_8` | 国際標準。最も推奨 |
| Shift_JIS | `Charset.forName("Shift_JIS")` | 日本語Windows環境のレガシーシステム |
| ISO-8859-1 | `StandardCharsets.ISO_8859_1` | 西欧言語 |
| US-ASCII | `StandardCharsets.US_ASCII` | 英数字のみ |

> **よくある間違い：** エンコーディングを指定しないと、実行環境のデフォルトが使われます。開発環境と本番環境で文字化けが起こる原因になるため、**必ず UTF-8 を明示指定**しましょう。

### ポイントまとめ
- ファイルの読み書きには文字エンコーディングを明示指定する
- 新規プロジェクトでは **UTF-8** を標準にする
- `StandardCharsets.UTF_8` で指定するのが安全

## 10.9 実践例：ログファイル解析

複数の概念を組み合わせた実践的な例です。

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Path;
import java.util.HashMap;
import java.util.Map;

public class LogAnalyzer {
    public static void main(String[] args) {
        Path logFile = Path.of("app.log");
        Path reportFile = Path.of("report.txt");

        // サンプルログを作成
        createSampleLog(logFile);

        // ログを解析
        Map<String, Integer> levelCount = new HashMap<>();
        int totalLines = 0;

        try (BufferedReader reader = Files.newBufferedReader(logFile)) {
            String line;
            while ((line = reader.readLine()) != null) {
                totalLines++;
                // ログレベルを抽出（例: [INFO], [ERROR]）
                if (line.contains("[INFO]")) {
                    levelCount.merge("INFO", 1, Integer::sum);
                } else if (line.contains("[WARN]")) {
                    levelCount.merge("WARN", 1, Integer::sum);
                } else if (line.contains("[ERROR]")) {
                    levelCount.merge("ERROR", 1, Integer::sum);
                }
            }
        } catch (IOException e) {
            System.err.println("ログ読み込みエラー: " + e.getMessage());
            return;
        }

        // レポートを生成して書き出し
        StringBuilder report = new StringBuilder();
        report.append("===== ログ解析レポート =====\n\n");
        report.append("総行数: ").append(totalLines).append("\n\n");
        report.append("ログレベル別集計:\n");
        for (Map.Entry<String, Integer> entry : levelCount.entrySet()) {
            report.append(String.format("  %-8s: %d件%n", entry.getKey(), entry.getValue()));
        }

        // コンソールに表示
        System.out.println(report);

        // ファイルに保存
        try {
            Files.writeString(reportFile, report.toString());
            System.out.println("レポートを " + reportFile + " に保存しました。");
        } catch (IOException e) {
            System.err.println("レポート書き込みエラー: " + e.getMessage());
        }

        // クリーンアップ
        try {
            Files.deleteIfExists(logFile);
            Files.deleteIfExists(reportFile);
        } catch (IOException e) {
            // 無視
        }
    }

    static void createSampleLog(Path path) {
        try (BufferedWriter writer = Files.newBufferedWriter(path)) {
            writer.write("[INFO] 2026-03-29 09:00:00 アプリケーション起動\n");
            writer.write("[INFO] 2026-03-29 09:00:01 データベース接続成功\n");
            writer.write("[INFO] 2026-03-29 09:01:00 ユーザーログイン: user001\n");
            writer.write("[WARN] 2026-03-29 09:05:30 メモリ使用率80%超過\n");
            writer.write("[INFO] 2026-03-29 09:10:00 データ処理開始\n");
            writer.write("[ERROR] 2026-03-29 09:10:05 ファイルが見つかりません: data.csv\n");
            writer.write("[INFO] 2026-03-29 09:10:10 リトライ処理実行\n");
            writer.write("[INFO] 2026-03-29 09:10:15 データ処理完了\n");
            writer.write("[WARN] 2026-03-29 09:30:00 セッションタイムアウト: user001\n");
            writer.write("[INFO] 2026-03-29 10:00:00 アプリケーション停止\n");
        } catch (IOException e) {
            System.err.println("ログ作成エラー: " + e.getMessage());
        }
    }
}
```

## 10.10 よくある間違いとトラブルシューティング

| ミス | 症状 | 対処法 |
|---|---|---|
| リソースを閉じ忘れる | メモリリーク、ファイルロック | try-with-resources を使う |
| ファイルが存在しない | FileNotFoundException | 事前に `Files.exists()` で確認 |
| パスの区切り文字をハードコード | OS依存の不具合 | `Path.of()` や `File.separator` を使う |
| エンコーディング未指定 | 文字化け | `StandardCharsets.UTF_8` を明示 |
| 大きなファイルを一括読み込み | OutOfMemoryError | `BufferedReader` で1行ずつ処理 |
| 書き込み後に flush しない | データが書き込まれない | close() で自動 flush。try-with-resources 推奨 |
