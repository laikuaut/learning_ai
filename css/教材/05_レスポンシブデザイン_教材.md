## 1. レスポンシブデザインとは？

**レスポンシブデザイン** とは、1つのHTMLファイルで、スマホ・タブレット・PCなど、どの画面サイズでも最適に表示されるデザイン手法です。

```
スマホ（375px）     タブレット（768px）      PC（1200px）
┌───────┐          ┌──────────────┐        ┌──────────────────────┐
│ ロゴ   │          │ ロゴ  ナビ    │        │ ロゴ   ナビゲーション   │
├───────┤          ├────┬─────────┤        ├────┬────────┬────────┤
│       │          │    │         │        │    │        │        │
│ メイン │          │サイド│ メイン   │        │サイド│ メイン  │ サイド  │
│       │          │    │         │        │    │        │        │
├───────┤          └────┴─────────┘        └────┴────────┴────────┘
│サイド  │
└───────┘
（1カラム）         （2カラム）              （3カラム）
```

---

## 2. viewport の設定

レスポンシブデザインに**必須**のmetaタグです。HTMLの `<head>` に必ず書きます。

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

| 属性 | 意味 |
|------|------|
| `width=device-width` | ページの幅をデバイスの画面幅に合わせる |
| `initial-scale=1.0` | 初期のズーム倍率を1倍にする |

> **これがないと：** スマホでもPC用の幅（980px程度）で表示され、すべてが小さく見えてしまいます。

## 3. メディアクエリ（Media Queries）

画面サイズに応じてCSSを切り替える仕組みです。

### 基本的な書き方

```css
/* 通常のスタイル（すべての画面に適用） */
.container {
  padding: 16px;
}

/* 画面幅が768px以上の場合に適用 */
@media (min-width: 768px) {
  .container {
    padding: 32px;
  }
}

/* 画面幅が1200px以上の場合に適用 */
@media (min-width: 1200px) {
  .container {
    padding: 48px;
    max-width: 1200px;
    margin: 0 auto;
  }
}
```

### min-width と max-width

| 書き方 | 意味 | 使う場面 |
|--------|------|---------|
| `@media (min-width: 768px)` | 768px**以上**で適用 | モバイルファースト（推奨） |
| `@media (max-width: 767px)` | 767px**以下**で適用 | デスクトップファースト |

### 複数の条件を組み合わせる

```css
/* 768px以上 かつ 1199px以下（タブレットのみ） */
@media (min-width: 768px) and (max-width: 1199px) {
  .sidebar {
    width: 200px;
  }
}
```

## 4. モバイルファースト設計（推奨）

**モバイルファースト** とは、スマホ向けのスタイルを基本として書き、画面が大きくなるにつれてスタイルを追加していく考え方です。

### モバイルファーストの書き方

```css
/* ① ベース：モバイル向け（すべての画面に適用） */
.card-grid {
  display: grid;
  grid-template-columns: 1fr;    /* 1列 */
  gap: 16px;
}

/* ② タブレット以上（768px〜） */
@media (min-width: 768px) {
  .card-grid {
    grid-template-columns: repeat(2, 1fr);  /* 2列 */
    gap: 24px;
  }
}

/* ③ PC以上（1200px〜） */
@media (min-width: 1200px) {
  .card-grid {
    grid-template-columns: repeat(3, 1fr);  /* 3列 */
    gap: 32px;
  }
}
```

### なぜモバイルファーストが推奨されるのか？

1. **シンプルから始められる** ─ スマホのレイアウトは1カラムが多く、簡単
2. **追加のスタイルが少ない** ─ 基本を小さい画面にすると、上書きが減る
3. **パフォーマンス** ─ スマホは回線が遅いため、最小限のCSSで表示できる
4. **世界的なトレンド** ─ スマホでのアクセスが過半数を超えている

### よく使うブレイクポイント

| デバイス | ブレイクポイント | 名称例 |
|---------|---------------|-------|
| スマホ | 〜 767px | mobile |
| タブレット | 768px 〜 1023px | tablet |
| 小さめのPC | 1024px 〜 1199px | laptop |
| PC | 1200px 〜 | desktop |

> ブレイクポイントは「一般的な目安」であり、デザインに合わせて調整してOKです。

## 5. レスポンシブなナビゲーション

### PC：横並び / スマホ：縦並び

```css
.nav {
  display: flex;
  flex-direction: column;       /* モバイル：縦並び */
  gap: 8px;
}

.nav-link {
  padding: 12px 16px;
  text-decoration: none;
  color: #333;
}

@media (min-width: 768px) {
  .nav {
    flex-direction: row;        /* タブレット以上：横並び */
    gap: 0;
  }
}
```

### ハンバーガーメニュー（概念）

スマホではナビゲーションを隠し、ハンバーガーアイコンで開閉するパターンです。

```css
.hamburger {
  display: block;              /* モバイル：表示 */
}

.nav-links {
  display: none;               /* モバイル：非表示 */
}

.nav-links.open {
  display: flex;               /* メニューが開いた時に表示 */
  flex-direction: column;
}

@media (min-width: 768px) {
  .hamburger {
    display: none;             /* PC：ハンバーガーを非表示 */
  }

  .nav-links {
    display: flex;             /* PC：ナビを常に表示 */
    flex-direction: row;
  }
}
```

> **注意：** ハンバーガーメニューの開閉には通常JavaScriptが必要です。

## 6. レスポンシブなレイアウトパターン

### パターン1：1カラム → 2カラム → 3カラム

```css
.grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 16px;
}

@media (min-width: 768px) {
  .grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (min-width: 1200px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
  }
}
```

### パターン2：auto-fit による自動調整（メディアクエリ不要！）

```css
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 16px;
}
```

> これだけで画面幅に応じて列数が自動的に変わります。非常に便利です。

### パターン3：サイドバーの切り替え

```css
.layout {
  display: grid;
  grid-template-columns: 1fr;       /* モバイル：1カラム */
  gap: 24px;
}

@media (min-width: 768px) {
  .layout {
    grid-template-columns: 250px 1fr;  /* PC：サイドバー + メイン */
  }
}
```

### パターン4：grid-template-areas の切り替え

```css
.page {
  display: grid;
  grid-template-areas:
    "header"
    "main"
    "sidebar"
    "footer";
  gap: 16px;
}

@media (min-width: 768px) {
  .page {
    grid-template-columns: 250px 1fr;
    grid-template-areas:
      "header  header"
      "sidebar main"
      "footer  footer";
  }
}

@media (min-width: 1200px) {
  .page {
    grid-template-columns: 200px 1fr 200px;
    grid-template-areas:
      "header  header  header"
      "left    main    right"
      "footer  footer  footer";
  }
}
```

## 7. レスポンシブな文字サイズ

### メディアクエリで切り替え

```css
h1 {
  font-size: 24px;         /* モバイル */
}

@media (min-width: 768px) {
  h1 {
    font-size: 32px;       /* タブレット */
  }
}

@media (min-width: 1200px) {
  h1 {
    font-size: 40px;       /* PC */
  }
}
```

### clamp() で滑らかに変化（モダンCSS）

メディアクエリなしで、画面幅に応じて文字サイズが滑らかに変化します。

```css
h1 {
  font-size: clamp(24px, 5vw, 48px);
  /* 最小24px、基準は画面幅の5%、最大48px */
}

p {
  font-size: clamp(14px, 1.5vw, 18px);
}
```

> **clamp(最小値, 推奨値, 最大値)** ─ 推奨値が最小値と最大値の間に収まるように調整されます。

## 8. レスポンシブな画像

### 基本：画像がコンテナからはみ出さないようにする

```css
img {
  max-width: 100%;    /* 親要素の幅を超えない */
  height: auto;       /* 比率を維持 */
  display: block;     /* 下部の余白を消す */
}
```

> **ベストプラクティス：** この3行はリセットCSSとして常に書いておきましょう。

### object-fit で画像のサイズを制御

```css
.thumbnail {
  width: 300px;
  height: 200px;
  object-fit: cover;     /* 枠に合わせてトリミング（最もよく使う） */
}
```

| 値 | 説明 |
|----|------|
| `cover` | 枠を埋めるようにリサイズ（はみ出す部分はトリミング） |
| `contain` | 枠に収まるようにリサイズ（余白ができることがある） |
| `fill` | 枠にぴったり合わせる（比率が崩れる） |
| `none` | リサイズしない |

### アスペクト比を固定する

```css
.card-image {
  width: 100%;
  aspect-ratio: 16 / 9;    /* 16:9の比率を維持 */
  object-fit: cover;
}
```

## 9. レスポンシブに役立つ便利なCSS

### CSS変数でブレイクポイントを管理

```css
:root {
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --max-width: 1200px;
}

.container {
  width: 100%;
  max-width: var(--max-width);
  margin: 0 auto;
  padding: 0 var(--spacing-md);
}

@media (min-width: 768px) {
  .container {
    padding: 0 var(--spacing-lg);
  }
}
```

### min() / max() / clamp()

```css
.container {
  /* 画面幅の90%だが、最大は1200px */
  width: min(90%, 1200px);

  /* paddingは最低16px、できれば5vw */
  padding: max(16px, 5vw);
}
```

### 表示・非表示の切り替え

```css
/* モバイルでは非表示、PCでは表示 */
.desktop-only {
  display: none;
}

@media (min-width: 768px) {
  .desktop-only {
    display: block;
  }
}

/* PCでは非表示、モバイルでは表示 */
.mobile-only {
  display: block;
}

@media (min-width: 768px) {
  .mobile-only {
    display: none;
  }
}
```

## 11. コンテナクエリ（Container Queries）

メディアクエリは「画面全体の幅」を基準にしますが、**コンテナクエリ** は「親要素（コンテナ）の幅」を基準にスタイルを切り替えます。

### なぜコンテナクエリが必要？

同じカードコンポーネントでも、サイドバーに置く場合とメインエリアに置く場合で幅が違います。メディアクエリでは対応が難しいですが、コンテナクエリなら親の幅に応じて自動的にレイアウトを変えられます。

### 基本的な書き方

```css
/* ① 親要素をコンテナとして定義 */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

/* ② コンテナの幅に応じてスタイルを切り替え */
@container card (min-width: 400px) {
  .card {
    display: flex;
    gap: 16px;
  }
  .card-image {
    width: 40%;
  }
}

@container card (min-width: 600px) {
  .card {
    font-size: 18px;
  }
}
```

### container-type の値

| 値 | 意味 |
|-----|------|
| `inline-size` | 横幅のみを基準にする（最もよく使う） |
| `size` | 横幅と高さの両方を基準にする |
| `normal` | コンテナクエリを使わない（デフォルト） |

### 省略記法

```css
/* container-type と container-name を一行で */
.wrapper {
  container: card / inline-size;
}
```

> **ブラウザ対応：** 2023年以降のモダンブラウザはすべて対応しています。
