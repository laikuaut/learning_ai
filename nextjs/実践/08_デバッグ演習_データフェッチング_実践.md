### バグ1：サーバーコンポーネントで useState を使っている

```tsx
// 修正前 — サーバーコンポーネントで useState は使えない
import { useState } from 'react'

export default async function NewsPage() {
  const articles = await getArticles()
  const [filter, setFilter] = useState('')  // エラー！
  // ...
}

// 修正後 — 検索機能をクライアントコンポーネントに分離
// app/news/SearchFilter.tsx
'use client'
import { useState } from 'react'

export default function SearchFilter({ children }: { children: React.ReactNode }) {
  const [filter, setFilter] = useState('')
  // ...
}

// app/news/page.tsx — サーバーコンポーネント
export default async function NewsPage() {
  const articles = await getArticles()
  // useState を使わず、サーバーコンポーネントとしてデータ取得のみ行う
}
```

**解説**: サーバーコンポーネント（`async function` で定義）では `useState` などの React Hooks は使用できません。インタラクティブな機能は別のクライアントコンポーネントに分離します。

### バグ2：キャッシュ設定が不適切

```tsx
// 修正前 — ニュースなのに永続キャッシュ
const res = await fetch('https://...', {
  cache: 'force-cache',  // ビルド時の結果がずっと使われる
})

// 修正後 — 適切なキャッシュ戦略を選択
const res = await fetch('https://...', {
  next: { revalidate: 60 },  // 60秒ごとに再検証（ISR）
})
// または
const res = await fetch('https://...', {
  cache: 'no-store',  // 常に最新データを取得（SSR）
})
```

**解説**: ニュースのように頻繁に更新されるデータに `force-cache` を使うと、古いデータが表示され続けます。`next.revalidate` で適切な再検証間隔を設定するか、`no-store` でリアルタイムデータを取得します。

---

### バグ3：key の位置が間違い

```tsx
// 修正前 — key がコンポーネント内部にある
{articles.map((article) => (
  <ArticleCard article={article} />  // key がない
))}

function ArticleCard({ article }) {
  return <div key={article.id}>...</div>  // ここに key があるが無効
}

// 修正後 — key は map の直下の要素に指定する
{articles.map((article) => (
  <ArticleCard key={article.id} article={article} />
))}

function ArticleCard({ article }) {
  return <div>...</div>  // key は不要
}
```

**解説**: React の `key` は `map` のコールバック内で返される最上位の要素に指定する必要があります。子コンポーネント内部の要素に `key` を付けても、React はそれを認識しません。

---

### バグ4：params の型が間違い

```tsx
// 修正前 — Next.js 15 では params は Promise
export default async function ArticlePage({
  params,
}: {
  params: { id: string }  // 古い型
}) {
  const article = await fetch(`.../${params.id}`)
}

// 修正後 — Promise を await する
export default async function ArticlePage({
  params,
}: {
  params: Promise<{ id: string }>  // Promise 型
}) {
  const { id } = await params
  const article = await fetch(`.../${id}`)
}
```

**解説**: Next.js 15 の App Router では、`params` と `searchParams` は `Promise` として渡されます。使用する前に `await` が必要です。

---

### バグ5：エラーハンドリングがない

```tsx
// 修正前 — fetch 失敗時にクラッシュする
const res = await fetch(`.../${id}`)
const article = await res.json()

// 修正後 — エラーチェックを追加
import { notFound } from 'next/navigation'

const res = await fetch(`.../${id}`)

if (!res.ok) {
  if (res.status === 404) {
    notFound()  // 404 ページを表示
  }
  throw new Error('記事の取得に失敗しました')  // error.tsx で処理
}

const article: Article = await res.json()
```

**解説**: `fetch` はネットワークエラー以外で例外を投げません。ステータスコードを確認し、404の場合は `notFound()` を、それ以外のエラーは `throw` して `error.tsx` に処理を委ねます。

---

### バグ6：クライアント機能をサーバーコンポーネント内で定義

```tsx
// 修正前 — サーバーコンポーネントのファイル内で useState と onClick を使用
function LikeButton({ articleId }: { articleId: number }) {
  const [likes, setLikes] = React.useState(0)
  return <button onClick={() => setLikes(likes + 1)}>♥ {likes}</button>
}

// 修正後 — 別ファイルのクライアントコンポーネントに分離
// app/news/[id]/LikeButton.tsx
'use client'

import { useState } from 'react'

export default function LikeButton({ articleId }: { articleId: number }) {
  const [likes, setLikes] = useState(0)

  return (
    <button onClick={() => setLikes(likes + 1)}>
      ♥ {likes}
    </button>
  )
}
```

**解説**: `useState` や `onClick` を使うコンポーネントは `'use client'` ディレクティブが必要です。サーバーコンポーネントのファイル内に直接書くことはできません。

---

### バグ7：'use server' ディレクティブがない

```tsx
// 修正前 — Server Actions として認識されない
import { revalidatePath } from 'next/cache'

export async function addComment(formData: FormData) { ... }

// 修正後 — ファイル先頭に 'use server' を追加
'use server'

import { revalidatePath } from 'next/cache'

export async function addComment(formData: FormData) { ... }
```

**解説**: Server Actions は `'use server'` ディレクティブで宣言する必要があります。ファイル先頭に書くとファイル内のすべてのエクスポート関数がServer Actionsになります。

---

### バグ8：revalidatePath の引数が間違い

```tsx
// 修正前 — テンプレート構文を渡している
revalidatePath('/news/[id]')  // 実際のパスではない

// 修正後 — 実際のパスを渡す
revalidatePath(`/news/${articleId}`)

// または全ニュースページを再検証する場合
revalidatePath('/news', 'layout')
```

**解説**: `revalidatePath` には動的ルートのテンプレート（`[id]`）ではなく、実際のパス（`/news/1` など）を渡す必要があります。`'layout'` を第2引数に指定すると、そのレイアウト以下の全ページが再検証されます。
