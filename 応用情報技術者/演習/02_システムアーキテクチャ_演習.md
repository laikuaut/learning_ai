## 基本問題

### 問1（四択・アムダールの法則）
あるプログラムの処理全体のうち60%を占める部分を3倍に高速化した場合、プログラム全体の高速化率として最も近いものはどれか。

ア 1.5倍　　イ 1.8倍　　ウ 2.0倍　　エ 2.5倍

<details><summary>ヒント</summary>

アムダールの法則: 全体の高速化率 = 1 / {(1-P) + P/S}

</details>

<details><summary>解答・解説</summary>

**正解: ア 1.5倍**

```
P = 0.6（高速化可能な部分の割合）
S = 3（高速化倍率）

全体の高速化率 = 1 / {(1 - 0.6) + 0.6/3}
              = 1 / {0.4 + 0.2}
              = 1 / 0.6
              ≈ 1.67

最も近いのは ア 1.5倍
```

実際は約1.67倍ですが、選択肢の中では1.5倍が最も近いです。

</details>

### 問2（四択・MIPS）
50MIPSのCPUで、1トランザクションあたり25万命令が必要な処理がある。このCPUが1秒間に処理できる最大トランザクション数はどれか。

ア 100　　イ 200　　ウ 500　　エ 1000

<details><summary>解答・解説</summary>

**正解: イ 200**

```
50 MIPS = 50 × 10⁶ 命令/秒 = 50,000,000 命令/秒
1トランザクション = 250,000 命令

最大TPS = 50,000,000 / 250,000 = 200 TPS
```

</details>

### 問3（四択・稼働率）
稼働率0.95のサーバ2台を並列構成にした場合の稼働率として、正しいものはどれか。

ア 0.9025　　イ 0.9500　　ウ 0.9750　　エ 0.9975

<details><summary>ヒント</summary>

並列システムの稼働率 = 1 - (1-R₁)(1-R₂)

</details>

<details><summary>解答・解説</summary>

**正解: エ 0.9975**

```
並列システムの稼働率 = 1 - (1-0.95)(1-0.95)
                    = 1 - 0.05 × 0.05
                    = 1 - 0.0025
                    = 0.9975
```

参考: 直列の場合は 0.95 × 0.95 = 0.9025（選択肢ア）

</details>

### 問4（四択・可用性）
MTBFが5,000時間、MTTRが200時間のシステムの稼働率として、正しいものはどれか。

ア 94.0%　　イ 95.0%　　ウ 96.2%　　エ 97.5%

<details><summary>解答・解説</summary>

**正解: ウ 96.2%**

```
稼働率 = MTBF / (MTBF + MTTR)
       = 5,000 / (5,000 + 200)
       = 5,000 / 5,200
       ≈ 0.9615 ≈ 96.2%
```

</details>

### 問5（四択・RAID）
RAID5に関する記述として、正しいものはどれか。

ア ミラーリングによりデータを二重化する方式である
イ パリティ情報を分散して各ディスクに配置する方式である
ウ ストライピングのみを行い、冗長性はない方式である
エ 二重のパリティ情報を持つことで2台の同時故障に対応できる方式である

<details><summary>解答・解説</summary>

**正解: イ**

- ア: RAID1の説明
- イ: 正しい。RAID5はパリティ分散方式
- ウ: RAID0の説明
- エ: RAID6の説明

RAID5は最低3台のディスクが必要で、利用可能容量は(N-1)台分、1台の故障に耐えられます。

</details>

## 応用問題

### 問6（四択・複合システムの稼働率）
次のシステム構成の全体稼働率を求めよ。各サーバの稼働率はすべて0.9とする。

```
      ┌→ [A] →┐
→ ──┤         ├── → [C] →
      └→ [B] →┘
```

ア 0.891　　イ 0.899　　ウ 0.900　　エ 0.981

<details><summary>ヒント</summary>

A, Bは並列、その後にCが直列です。

</details>

<details><summary>解答・解説</summary>

**正解: ア 0.891**

```
手順1: A, Bの並列部分の稼働率
  = 1 - (1-0.9)(1-0.9)
  = 1 - 0.1 × 0.1
  = 1 - 0.01
  = 0.99

手順2: 並列部分とCの直列の稼働率
  = 0.99 × 0.9
  = 0.891
```

</details>

### 問7（四択・クラウド）
IaaS、PaaS、SaaSのうち、利用者がOSやミドルウェアの管理を行う必要があるサービスモデルはどれか。

ア IaaSのみ
イ IaaSとPaaS
ウ PaaSとSaaS
エ SaaSのみ

<details><summary>解答・解説</summary>

**正解: ア IaaSのみ**

| モデル | 利用者の管理範囲 |
|--------|---------------|
| IaaS | OS、ミドルウェア、アプリケーション |
| PaaS | アプリケーションのみ |
| SaaS | 設定・データのみ |

IaaSではOSからの管理が利用者責任です。PaaSではOSやミドルウェアはプロバイダが管理します。

</details>

### 問8（記述・性能計算）
あるシステムで、平均100TPS（トランザクション/秒）の処理が発生し、ピーク時は平均の4倍になる。CPU利用率を80%以下に抑えたい場合、必要なCPU処理能力は何TPS以上か。計算過程も示せ。

<details><summary>解答・解説</summary>

```
ピーク時TPS = 100 × 4 = 400 TPS

CPU利用率80%以下とするため:
必要処理能力 = ピーク時TPS / 目標CPU利用率
            = 400 / 0.8
            = 500 TPS

よって、500TPS以上の処理能力が必要です。
```

考え方: CPU利用率80%で400TPSを処理するには、100%時に500TPS処理できる能力が必要。残り20%はバッファとして確保します。

</details>

## チャレンジ問題

### 問9（記述・午後試験形式）
次のシステム構成に関する設問に答えなさい。

あるECサイトでは、以下の3層構成を採用している。

```
[ロードバランサ]
    ├── Webサーバ1 (稼働率: 0.95)
    ├── Webサーバ2 (稼働率: 0.95)
    └── Webサーバ3 (稼働率: 0.95)
         │
    [APサーバ] (稼働率: 0.98) × 1台
         │
    [DBサーバ]
    ├── マスタDB (稼働率: 0.99)
    └── スレーブDB (稼働率: 0.99)
```

- Webサーバは3台のうち少なくとも1台が稼働すればサービス継続可能
- APサーバは1台構成（冗長性なし）
- DBサーバはマスタとスレーブのうち少なくとも1台が稼働すればサービス継続可能

**設問1:** このシステム全体の稼働率を計算せよ（小数第4位まで）。

**設問2:** 全体の稼働率を0.99以上にするために、APサーバをもう1台追加して並列構成にした場合の全体稼働率を求めよ。0.99以上になるか判定せよ。

<details><summary>解答・解説</summary>

**設問1の解答:**

```
Webサーバ（3台並列）:
  = 1 - (1-0.95)³
  = 1 - 0.05³
  = 1 - 0.000125
  = 0.999875

APサーバ（1台）:
  = 0.98

DBサーバ（2台並列）:
  = 1 - (1-0.99)²
  = 1 - 0.01²
  = 1 - 0.0001
  = 0.9999

全体（直列）:
  = 0.999875 × 0.98 × 0.9999
  = 0.999875 × 0.98 × 0.9999
  ≈ 0.9799（小数第4位まで）
```

全体稼働率は約0.9799（97.99%）です。

**設問2の解答:**

```
APサーバ（2台並列）:
  = 1 - (1-0.98)²
  = 1 - 0.02²
  = 1 - 0.0004
  = 0.9996

全体:
  = 0.999875 × 0.9996 × 0.9999
  ≈ 0.999875 × 0.9996 × 0.9999
  ≈ 0.9994

0.9994 > 0.99 なので、目標の0.99以上を達成できます。
```

APサーバがボトルネックだったことがわかります。1台構成（0.98）が全体の稼働率を下げていました。並列化により0.9996となり、全体が大幅に改善されました。

</details>

### 問10（記述・午後試験形式）
仮想化技術を導入する際の設計判断について、次の設問に答えなさい。

ある企業が、物理サーバ10台で運用しているシステムを仮想化によるサーバ統合する計画を立てている。各物理サーバのCPU利用率は平均20%であり、2台の物理サーバ上にVMとして統合する予定である。

**設問1:** VMとコンテナの違いについて、起動速度、リソース消費、分離レベルの3つの観点で比較して説明せよ。

**設問2:** サーバ統合後の各物理サーバのCPU利用率の想定値を計算し、この統合計画の妥当性について考察せよ。余裕率を20%以上確保すべきとする。

<details><summary>解答・解説</summary>

**設問1の解答:**

| 観点 | 仮想マシン(VM) | コンテナ |
|------|-------------|---------|
| 起動速度 | 分単位（ゲストOS起動が必要） | 秒単位（ホストOSのカーネルを共有） |
| リソース消費 | 大きい（VM毎にOS分のメモリ・ディスクを消費） | 小さい（カーネル共有でオーバーヘッドが少ない） |
| 分離レベル | 高い（ハイパーバイザにより完全分離） | 中程度（カーネルを共有するため、カーネルの脆弱性の影響あり） |

**設問2の解答:**

```
現状の総CPU処理量 = 10台 × 20% = 200%分の処理量

2台に統合した場合:
各サーバのCPU利用率 = 200% / 2 = 100%

しかし、仮想化のオーバーヘッド（約10%と仮定）を考慮すると:
実際のCPU利用率 ≈ 100% + 10% = 110%（既にオーバー）
```

この統合計画は妥当ではありません。2台への統合では100%を超えてしまい、余裕率20%の確保は不可能です。

適切な統合先台数:
```
必要CPU容量 = 200%（総処理量）/ 0.8（余裕率20%確保）= 250%
仮想化オーバーヘッド10%を考慮: 250% / 0.9 ≈ 278%
必要台数 = 278% / 100% = 2.78 → 3台以上

3台構成の場合: 200% / 3 ≈ 67%（オーバーヘッド含めて約74%、余裕率26%）
```

最低3台の物理サーバに統合することが妥当です。

</details>
