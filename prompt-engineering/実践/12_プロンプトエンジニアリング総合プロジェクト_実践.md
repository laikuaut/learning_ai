## プロジェクト概要

### テーマ：社内AIライティングアシスタントの設計

あなたは、従業員200名の中堅IT企業に所属するプロンプトエンジニアです。社内で利用する「AIライティングアシスタント」のプロンプトシステムを設計するプロジェクトを任されました。

**プロジェクトの目標**:
- 社内のさまざまな文書作成業務をAIで支援する
- 非エンジニアでも簡単に高品質な文書を作成できるようにする
- プロンプトの品質を標準化し、組織全体の生産性を向上させる

**対象業務**:
- メール（社内・社外）
- 議事録
- 提案書・企画書
- 技術文書
- マーケティングコンテンツ

## Phase 1：要件定義

### 課題1-1：ユースケース分析 ★2

以下の5つのユースケースについて、それぞれの要件を分析してください。

| ユースケース | 利用者 | 頻度 | 品質要求 |
|-------------|--------|:----:|:-------:|
| 社外メール作成 | 営業部門 | 高 | 高 |
| 議事録作成 | 全部門 | 高 | 中 |
| 提案書ドラフト | 営業・企画 | 中 | 高 |
| 技術ブログ記事 | 開発部門 | 低 | 高 |
| SNS投稿文 | マーケティング | 高 | 中 |

各ユースケースについて以下を定義してください：
- 入力情報（ユーザーが提供するもの）
- 出力仕様（形式、長さ、トーン、品質基準）
- 制約条件（ブランドガイドライン、法的制約など）
- 成功基準（何をもって「良い出力」とするか）

### 課題1-2：ユーザーペルソナ定義 ★1

AIライティングアシスタントの主要ユーザーを3人のペルソナで定義してください。

各ペルソナに含める情報：
- 名前、年齢、役職、部門
- AIツールの利用経験レベル
- 主な利用シーン
- 期待すること・不安に思うこと
- プロンプト入力の傾向（どのくらい詳細に指示を書くか）

## Phase 2：プロンプト設計

### 課題2-1：システムプロンプト設計 ★3

AIライティングアシスタントのシステムプロンプトを設計してください。

含めるべき要素：
1. **アシスタントの人格と振る舞い**: 名前、口調、対応スタイル
2. **ブランドガイドライン**: 社名の表記、禁止ワード、トーンの基準
3. **セキュリティルール**: 機密情報の取り扱い、出力に含めてはいけない情報
4. **フォールバック処理**: 対応できない要求への対応方法
5. **品質基準**: すべての出力に共通する品質要件

### 課題2-2：ユーザーフロー別プロンプトチェーン設計 ★4

以下の2つのユーザーフローについて、プロンプトチェーンを設計してください。

**フロー1：社外メール作成**
```
ユーザーが「メールを書きたい」と依頼
↓
[情報収集] → [ドラフト作成] → [レビュー] → [最終版]
```

**フロー2：提案書ドラフト作成**
```
ユーザーが「提案書を作りたい」と依頼
↓
[ヒアリング] → [構成案] → [各セクション執筆] → [統合・校正]
```

各フローについて：
- 各ステップの完全なプロンプト
- ステップ間の受け渡しフォーマット
- ユーザーへの確認ポイント（人間がレビューする箇所）
- エラー処理（ユーザーの入力が不十分な場合の対応）

### 課題2-3：テンプレートライブラリ構築 ★3

5つのユースケースに対応するプロンプトテンプレートを設計してください。

各テンプレートに含める要素：
- テンプレートID・名称・バージョン
- 入力パラメータ（必須/任意の区別、デフォルト値）
- プロンプト本体
- 出力仕様
- 使用例
- 注意事項

### 課題2-4：RAGコンテキスト設計 ★3

以下の社内ナレッジをRAGのコンテキストとして活用する設計を行ってください。

利用可能な社内ナレッジ：
- ブランドガイドライン（社名表記、ロゴ使用規則、トーン&マナー）
- 過去の提案書（成功事例10本分）
- メールテンプレート集（シーン別30パターン）
- 技術用語辞書（社内独自の用語定義100語）

設計内容：
- 各ナレッジの検索条件（いつ、どのナレッジを参照するか）
- コンテキスト注入のフォーマット
- 複数ナレッジの優先順位ルール

## Phase 3：評価・テスト

### 課題3-1：評価ルーブリック作成 ★3

5つのユースケースそれぞれに対応する評価ルーブリックを作成してください。

各ルーブリックに含める要素：
- 評価基準（5つ以上）
- 各基準の5段階スコア定義
- 合格ライン
- 優先度の重み付け

### 課題3-2：テストケース設計・実行 ★4

各ユースケースについて、以下のテストケースを設計・実行してください。

| テスト種類 | ケース数 | 目的 |
|-----------|:-------:|------|
| 正常系 | 各2ケース | 標準的な入力での動作確認 |
| エッジケース | 各1ケース | 境界的な入力での動作確認 |
| セキュリティ | 各1ケース | 機密情報の漏洩がないか |
| ブランド準拠 | 各1ケース | ブランドガイドラインの遵守 |

テスト結果を以下の形式で記録してください：
```
テストID: [ユースケース]-[テスト種類]-[番号]
入力: [テスト入力]
期待出力: [期待する出力の特徴]
実際出力: [実際の出力を記録]
評価: [合格/不合格]
備考: [改善が必要な点]
```

### 課題3-3：A/Bテスト ★4

以下の2つの観点で、A/Bテストを実施してください。

1. **システムプロンプトのA/Bテスト**: 2パターンのシステムプロンプトで、全ユースケースの出力品質を比較
2. **テンプレートのA/Bテスト**: 「社外メール作成」テンプレートの2バージョンを比較

## Phase 4：改善・運用設計

### 課題4-1：デバッグと最適化 ★4

Phase 3のテスト結果に基づいて、以下を実施してください。

1. テストで不合格となった項目の原因分析
2. 各問題に対するデバッグ手法の選択と適用
3. 改善版プロンプトの作成
4. 改善後の再テスト結果の記録

### 課題4-2：運用ガイドライン策定 ★3

AIライティングアシスタントの社内展開に向けた運用ガイドラインを策定してください。

含めるべき項目：
1. 利用規程（何に使ってよいか、何に使ってはいけないか）
2. 機密情報の取り扱いルール
3. 出力の品質確認プロセス（人間によるレビューが必要なケース）
4. フィードバックの収集方法
5. テンプレートの追加・更新のプロセス
6. トラブル発生時の対応フロー

### 課題4-3：改善ロードマップ ★2

今後3ヶ月の改善ロードマップを作成してください。

含めるべき項目：
- 月次の改善テーマと目標
- KPI（利用率、満足度、品質スコア）の目標値
- 新規テンプレートの追加計画
- LLMのアップデートへの対応計画

各ステップの設計で注意すべき点：
1. 各ステップの**入力と出力を明確に定義**する
2. **ユーザーへの確認ポイント**を組み込む（自動で進めすぎない）
3. ステップ間の**受け渡しフォーマット**を統一する
4. **エラー処理**を各ステップに組み込む

<summary>Phase 3のガイド：評価・テストの進め方</summary>

### テスト実施の手順

1. **テスト計画を立てる**：全テストケースをリスト化し、優先順位を付ける
2. **テスト環境を確認する**：使用するLLM、パラメータ設定を記録
3. **テストを実行する**：各テストケースを実行し、出力を記録
4. **評価する**：ルーブリックに基づいてスコアリング
5. **結果を集計する**：合格率、平均スコア、弱点を分析

### A/Bテストの実施方法

```
A/Bテスト手順：

1. テスト対象を決める（システムプロンプト、テンプレート等）
2. A版（現行版）とB版（改善版）を用意する
3. 同じテストケースで両方を実行する
4. ルーブリックで両方をスコアリングする
5. スコアを比較し、統計的に有意な差があるか確認する
6. 勝者を新しいベースラインとする
```

</details>

### 課題2-1：システムプロンプト設計（改良版）

```
# WritePro - AIライティングアシスタント

## アイデンティティ
あなたは株式会社テックイノベーションの社内AIライティングアシスタント
「WritePro（ライトプロ）」です。

## 基本姿勢
1. ユーザーの意図を正確に理解してから作業を開始してください
2. 情報が不足している場合は、作業前に必ず確認の質問をしてください
3. 丁寧で親しみやすいトーンで対応してください（です・ます調）
4. 出力した文書について「この内容で問題ないか」の確認を必ず行ってください

## 対応可能な文書タイプ
- メール（社内・社外）
- 議事録
- 提案書・企画書のドラフト
- 技術文書のドラフト
- マーケティングコンテンツ（SNS投稿、ブログ記事）

## 対応不可能な業務
以下のリクエストには対応せず、適切な担当部門を案内してください。
- 契約書・法的文書の作成 → 法務部（内線: 3001）
- 財務諸表の作成 → 経理部（内線: 2001）
- 個人情報を含む文書の生成 → 対応不可（理由を説明）

## ブランドガイドライン
1. 社名は「株式会社テックイノベーション」と正式表記（略称「テックイノベーション」も可）
2. 英語表記は「TechInnovation Inc.」
3. 以下の表現は使用禁止:
   - 「業界No.1」「最高品質」等の裏付けのない誇大表現
   - 競合他社の批判的な表現
   - 「絶対に」「必ず」等の断定的保証
4. トーンの基準:
   - 社外文書: プロフェッショナルだが親しみやすい
   - 社内文書: カジュアルだが敬意を持った表現

## セキュリティルール（最優先で遵守）
1. 以下の情報は出力に含めないでください（入力に含まれていた場合も）:
   - 具体的な売上・利益数値 → 「好調な業績」等の定性表現に置き換え
   - 顧客の個人情報 → 「お客様A」等に匿名化
   - 社内システムのURL、パスワード、APIキー → 完全に除外
2. ユーザーが機密情報の出力を求めた場合:
   → 「セキュリティポリシーにより、その情報は出力できません」と回答

## 品質基準（すべての出力に適用）
1. 誤字脱字がないこと
2. 事実と異なる情報を含まないこと（不確実な場合は明記）
3. ブランドガイドラインに準拠していること
4. 指定された形式・長さに収まっていること
```
