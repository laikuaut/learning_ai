## 8.1 セキュリティの原則

### MCPにおけるセキュリティの考え方

MCPサーバーはAIアシスタントにシステムリソースへのアクセスを提供するため、セキュリティは最も重要な設計要素です。

```
┌──────────┐     ┌──────────────┐     ┌──────────────────┐
│   ユーザー  │────►│ AIアシスタント  │────►│   MCPサーバー      │
│  (人間)    │     │ (LLM)        │     │  (システムアクセス)  │
└──────────┘     └──────────────┘     └────────┬─────────┘
                                               │
                   信頼できない入力が              ▼
                   ここを通過する          ┌──────────────┐
                                         │ ファイル/DB/API │
                                         │ (保護対象)      │
                                         └──────────────┘
```

### 4つの基本原則

#### 1. 最小権限の原則（Principle of Least Privilege）

MCPサーバーには必要最小限の権限だけを付与します。

```python
# 悪い例：全ファイルシステムへのアクセスを許可
ALLOWED_DIR = Path("/")  # ルートディレクトリ → 危険！

# 良い例：特定ディレクトリのみに制限
ALLOWED_DIR = Path(os.environ.get("ALLOWED_DIR", "./workspace")).resolve()
```

| 項目 | 悪い設計 | 良い設計 |
|-----|---------|---------|
| ファイルアクセス | `/` 全体を許可 | 特定ディレクトリのみ許可 |
| DB操作 | 全SQL文を許可 | SELECT + 限定的なINSERT/UPDATE |
| Git操作 | 任意のgitコマンドを実行 | `status`, `log`, `diff` のみ |
| API呼び出し | 全エンドポイントを許可 | 必要なエンドポイントのみ |

#### 2. 入力バリデーション（Input Validation）

AIアシスタントからの入力は信頼できません。すべてバリデーションが必要です。

```python
@mcp.tool()
def read_file(path: str) -> str:
    """ファイルを読み取ります。"""
    # バリデーション1: 空文字チェック
    if not path.strip():
        return "エラー: パスが空です。"

    # バリデーション2: パストラバーサル防止
    safe_path = (ALLOWED_DIR / path).resolve()
    if not str(safe_path).startswith(str(ALLOWED_DIR)):
        return "エラー: 許可ディレクトリ外へのアクセスです。"

    # バリデーション3: ファイル存在チェック
    if not safe_path.exists():
        return f"エラー: ファイルが見つかりません: {path}"

    # バリデーション4: サイズ制限
    if safe_path.stat().st_size > 10 * 1024 * 1024:
        return "エラー: ファイルが大きすぎます（上限10MB）。"

    return safe_path.read_text(encoding="utf-8")
```

#### 3. サンドボックス（Sandbox）

MCPサーバーの操作範囲を隔離された環境に限定します。

```python
# サンドボックスの実装例：許可リストによるコマンド制限
ALLOWED_COMMANDS = {
    "status": ["git", "status", "--short"],
    "log": ["git", "log", "--oneline", "-20"],
    "diff": ["git", "diff"],
}

def run_sandboxed_command(command_name: str) -> str:
    """許可リストに基づいてコマンドを安全に実行します。"""
    if command_name not in ALLOWED_COMMANDS:
        return f"エラー: '{command_name}' は許可されていないコマンドです。"

    cmd = ALLOWED_COMMANDS[command_name]
    result = subprocess.run(cmd, capture_output=True, text=True, timeout=30)
    return result.stdout
```

#### 4. ユーザー同意モデル（User Consent Model）

破壊的な操作（ファイル削除、データ更新等）の前に、ユーザーの明示的な同意を求める設計にします。

```python
@mcp.tool()
def delete_file(path: str, confirm: bool = False) -> str:
    """ファイルを削除します。confirm=Trueを指定する必要があります。

    Args:
        path: 削除するファイルのパス
        confirm: 削除の確認フラグ（Trueでないと実行されない）
    """
    if not confirm:
        return (
            f"確認: '{path}' を本当に削除しますか？\n"
            f"削除を実行するには confirm=True を指定して再度呼び出してください。"
        )
    # 実際の削除処理...
    safe_path = validate_path(path)
    safe_path.unlink()
    return f"削除しました: {path}"
```

### ポイントまとめ

- 最小権限の原則に従い、MCPサーバーに不必要な権限を与えないようにします
- AIからの入力はすべて信頼できないものとして、厳格にバリデーションします
- サンドボックスにより操作範囲を制限し、意図しない操作を防止します
- 破壊的操作には確認フラグを設け、ユーザーの同意を得る設計にします

## 8.2 認証と認可

### 環境変数による機密情報管理

APIキーやデータベース接続情報などの機密情報（Secret）は、環境変数で管理します。

```python
import os

# 悪い例：コードに直接記載（ハードコーディング）
API_KEY = "sk-abc123..."  # 危険！GitHubに公開される可能性

# 良い例：環境変数から取得
API_KEY = os.environ.get("API_KEY")
if not API_KEY:
    raise RuntimeError("環境変数 API_KEY が設定されていません。")
```

```json

      "env": {
        "API_KEY": "sk-abc123...",

```

### OAuth 2.0連携

リモートMCPサーバーでは、OAuth 2.0（オーオース 2.0）による認証連携が可能です。

```
┌──────────┐    1. 認証リクエスト     ┌──────────────┐
│  MCPクライ │ ─────────────────────► │  認可サーバー    │
│  アント    │                        │ (OAuth 2.0)   │
│           │ ◄───────────────────── │               │
│           │    2. アクセストークン    └──────────────┘
│           │
│           │    3. トークン付きリクエスト
│           │ ─────────────────────► ┌──────────────┐
│           │                        │  MCPサーバー    │
│           │ ◄───────────────────── │  (リモート)     │
└──────────┘    4. レスポンス         └──────────────┘
```

```python
# リモートMCPサーバーでのOAuth対応例（Streamable HTTP）
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("secure-server")

def verify_token(token: str) -> dict | None:
    """アクセストークンを検証します。"""
    # 実際にはOAuthプロバイダーのトークン検証エンドポイントを呼び出す
    # ここでは簡略化した例を示します
    import urllib.request
    import json

    req = urllib.request.Request(
        "https://oauth-provider.example.com/verify",
        headers={"Authorization": f"Bearer {token}"}
    )
    try:
        with urllib.request.urlopen(req, timeout=5) as resp:
            return json.loads(resp.read())
    except Exception:
        return None
```

### よくある間違い

| 間違い | リスク | 対策 |
|-------|-------|------|
| APIキーをコードに直接記載 | GitHubに公開されて漏洩 | 環境変数で管理 |
| `.env` ファイルをGitにコミット | 機密情報が履歴に残る | `.gitignore` に `.env` を追加 |
| 全ユーザーに同じ権限を付与 | 不正操作のリスク増大 | ロールベースのアクセス制御 |
| トークンの有効期限を設けない | 漏洩時の被害が永続化 | 短い有効期限 + リフレッシュトークン |

### ポイントまとめ

- 機密情報は絶対にソースコードに直接記載せず、環境変数で管理します
- `.env` ファイルは `.gitignore` に追加し、Git管理対象から除外します
- リモートMCPサーバーではOAuth 2.0による認証連携を検討します

## 8.3 入力バリデーションとサニタイズ

### パストラバーサル防止

パストラバーサル（Path Traversal）とは、`../` を使って意図しないディレクトリにアクセスする攻撃です。

```python
from pathlib import Path

ALLOWED_DIR = Path("/app/workspace").resolve()

def validate_path(user_input: str) -> Path:
    """パストラバーサル攻撃を防止するバリデーションです。"""
    # 1. 入力パスをベースディレクトリからの相対パスとして結合
    requested = (ALLOWED_DIR / user_input).resolve()

    # 2. 正規化後のパスがベースディレクトリ内かチェック
    if not str(requested).startswith(str(ALLOWED_DIR) + os.sep) and requested != ALLOWED_DIR:
        raise ValueError("アクセス拒否: ディレクトリ外へのアクセスです。")

    return requested

# 攻撃例と防御結果
# validate_path("../../../etc/passwd")
#   → /app/workspace/../../../etc/passwd
#   → resolve() → /etc/passwd
#   → startswith("/app/workspace") → False → 拒否！
```

### SQLインジェクション防止

SQLインジェクション（SQL Injection）とは、ユーザー入力にSQL文を挿入して不正な操作を行う攻撃です。

```python
import sqlite3

# ===== 悪い例：文字列結合でSQLを組み立てる =====
def bad_search(name: str) -> list:
    conn = sqlite3.connect("app.db")
    # 攻撃例: name = "'; DROP TABLE users; --"
    query = f"SELECT * FROM users WHERE name = '{name}'"  # 危険！
    return conn.execute(query).fetchall()

# ===== 良い例：プレースホルダー（パラメータバインディング）を使う =====
def safe_search(name: str) -> list:
    conn = sqlite3.connect("app.db")
    query = "SELECT * FROM users WHERE name = ?"  # プレースホルダー
    return conn.execute(query, (name,)).fetchall()  # パラメータとして渡す


# MCPサーバーでの実装例
@mcp.tool()
def search_users(name: str) -> str:
    """ユーザーを名前で検索します。

    Args:
        name: 検索する名前
    """
    # バリデーション
    if not name.strip():
        return "エラー: 名前を入力してください。"
    if len(name) > 100:
        return "エラー: 名前が長すぎます。"

    with get_connection() as conn:
        cursor = conn.cursor()
        # パラメータバインディングでSQLインジェクションを防止
        cursor.execute(
            "SELECT id, name, email FROM users WHERE name LIKE ?",
            (f"%{name}%",)
        )
        rows = cursor.fetchall()
        if not rows:
            return "該当するユーザーが見つかりません。"
        return "\n".join(f"ID:{r[0]} 名前:{r[1]} メール:{r[2]}" for r in rows)
```

### コマンドインジェクション防止

コマンドインジェクション（Command Injection）とは、ユーザー入力にOSコマンドを挿入する攻撃です。

```python
import subprocess

# ===== 悪い例：シェル経由でコマンドを実行 =====
def bad_ping(host: str) -> str:
    # 攻撃例: host = "127.0.0.1; rm -rf /"
    result = os.system(f"ping -c 1 {host}")  # 危険！シェルインジェクション
    return str(result)

# ===== 良い例：リスト形式で引数を渡す =====
def safe_ping(host: str) -> str:
    # バリデーション：許可される文字のみ
    import re
    if not re.match(r'^[a-zA-Z0-9.\-]+$', host):
        return "エラー: 無効なホスト名です。"

    result = subprocess.run(
        ["ping", "-c", "1", host],  # リスト形式 → シェル解釈されない
        capture_output=True,
        text=True,
        timeout=10,
        # shell=False がデフォルト。shell=True は絶対に使わない！
    )
    return result.stdout
```

### バリデーションのチェックリスト

MCPツールの引数に対して行うべきバリデーション一覧です。

| チェック項目 | 対象 | 方法 |
|------------|------|------|
| 空文字チェック | 全入力 | `if not value.strip()` |
| 長さ制限 | 文字列 | `if len(value) > MAX_LENGTH` |
| 型チェック | 数値等 | 型ヒント + 範囲チェック |
| パストラバーサル | ファイルパス | `resolve()` + `startswith()` |
| SQLインジェクション | SQL関連 | プレースホルダー使用 |
| コマンドインジェクション | OS操作 | リスト形式の `subprocess.run()` |
| 許可リスト照合 | 制限的入力 | `if value not in ALLOWED_VALUES` |

### ポイントまとめ

- パストラバーサルは `resolve()` + `startswith()` で防止します
- SQLインジェクションは必ずプレースホルダー（`?`）を使って防止します
- コマンドインジェクションは `subprocess.run()` のリスト形式引数で防止します
- `shell=True` や `os.system()` は絶対に使わないでください

## 8.4 ログと監査

### 構造化ログ（Structured Logging）

MCPサーバーの運用には、構造化されたログの記録が不可欠です。

```python
# logging_config.py - MCPサーバー向けログ設定
import logging
import json
from datetime import datetime, timezone

class JsonFormatter(logging.Formatter):
    """JSON形式でログを出力するフォーマッターです。"""

    def format(self, record):
        log_entry = {
            "timestamp": datetime.now(timezone.utc).isoformat(),
            "level": record.levelname,
            "message": record.getMessage(),
            "module": record.module,
            "function": record.funcName,
        }
        # 追加情報がある場合
        if hasattr(record, "tool_name"):
            log_entry["tool_name"] = record.tool_name
        if hasattr(record, "duration_ms"):
            log_entry["duration_ms"] = record.duration_ms
        if hasattr(record, "error"):
            log_entry["error"] = record.error
        return json.dumps(log_entry, ensure_ascii=False)


def setup_logger(name: str, log_file: str = "mcp_server.log") -> logging.Logger:
    """MCPサーバー用のロガーをセットアップします。"""
    logger = logging.getLogger(name)
    logger.setLevel(logging.INFO)

    # ファイルハンドラー（JSON形式）
    file_handler = logging.FileHandler(log_file, encoding="utf-8")
    file_handler.setFormatter(JsonFormatter())
    logger.addHandler(file_handler)

    # コンソールハンドラー（デバッグ用）
    console_handler = logging.StreamHandler()
    console_handler.setFormatter(
        logging.Formatter("%(asctime)s [%(levelname)s] %(message)s")
    )
    logger.addHandler(console_handler)

    return logger
```

### 操作ログの記録

ツール呼び出しごとにログを記録するデコレーター（Decorator）パターンです。

```python
import time
import functools

logger = setup_logger("mcp-server")

def log_tool_call(func):
    """ツール呼び出しをログに記録するデコレーターです。"""
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        tool_name = func.__name__
        start_time = time.time()

        logger.info(
            f"ツール呼び出し開始: {tool_name}",
            extra={"tool_name": tool_name}
        )

        try:
            result = func(*args, **kwargs)
            duration = (time.time() - start_time) * 1000

            logger.info(
                f"ツール呼び出し成功: {tool_name} ({duration:.1f}ms)",
                extra={"tool_name": tool_name, "duration_ms": round(duration, 1)}
            )
            return result
        except Exception as e:
            duration = (time.time() - start_time) * 1000
            logger.error(
                f"ツール呼び出し失敗: {tool_name} - {e}",
                extra={
                    "tool_name": tool_name,
                    "duration_ms": round(duration, 1),
                    "error": str(e),
                }
            )
            raise

    return wrapper


# デコレーターの使用例
@mcp.tool()
@log_tool_call
def query(sql: str) -> str:
    """SELECT文を実行して結果を返します。"""
    # ...実装...
    pass
```

### ログ出力例

```json
{"timestamp": "2026-03-26T10:30:00+00:00", "level": "INFO", "message": "ツール呼び出し開始: query", "module": "server", "function": "wrapper", "tool_name": "query"}
{"timestamp": "2026-03-26T10:30:00+00:00", "level": "INFO", "message": "ツール呼び出し成功: query (15.2ms)", "module": "server", "function": "wrapper", "tool_name": "query", "duration_ms": 15.2}
{"timestamp": "2026-03-26T10:30:05+00:00", "level": "ERROR", "message": "ツール呼び出し失敗: execute - SQLエラー", "module": "server", "function": "wrapper", "tool_name": "execute", "duration_ms": 3.1, "error": "no such table: orders"}
```

### ポイントまとめ

- 構造化ログ（JSON形式）は解析ツールとの連携が容易で、運用に適しています
- デコレーターパターンで既存のツール関数に影響を与えずログ機能を追加できます
- エラー発生時のログには十分な情報（ツール名、所要時間、エラー内容）を含めます
- ログファイルには機密情報（パスワード、APIキー等）を記録しないよう注意してください

## 8.5 パフォーマンスと信頼性

### レート制限（Rate Limiting）

MCPサーバーが過剰なリクエストで過負荷にならないよう、レート制限を実装します。

```python
import time
from collections import defaultdict

class TokenBucketRateLimiter:
    """トークンバケットアルゴリズムによるレート制限です。"""

    def __init__(self, max_tokens: int, refill_rate: float):
        """
        Args:
            max_tokens: バケットの最大トークン数
            refill_rate: 1秒あたりのトークン補充数
        """
        self.max_tokens = max_tokens
        self.refill_rate = refill_rate
        self.tokens = max_tokens
        self.last_refill = time.time()

    def consume(self, tokens: int = 1) -> bool:
        """トークンを消費します。利用可能ならTrue、制限超過ならFalseを返します。"""
        self._refill()
        if self.tokens >= tokens:
            self.tokens -= tokens
            return True
        return False

    def _refill(self):
        """時間経過に応じてトークンを補充します。"""
        now = time.time()
        elapsed = now - self.last_refill
        self.tokens = min(
            self.max_tokens,
            self.tokens + elapsed * self.refill_rate
        )
        self.last_refill = now


# 使用例：1分間に30リクエストまで許可（毎秒0.5トークン補充）
limiter = TokenBucketRateLimiter(max_tokens=30, refill_rate=0.5)
```

### キャッシュ戦略（Caching Strategy）

```python
import time
import hashlib

class LRUCache:
    """LRU（Least Recently Used）キャッシュの実装です。"""

    def __init__(self, max_size: int = 100, ttl_seconds: int = 300):
        self.max_size = max_size
        self.ttl = ttl_seconds
        self.cache: dict[str, tuple[float, any]] = {}
        self.access_order: list[str] = []

    def _make_key(self, *args, **kwargs) -> str:
        """引数からキャッシュキーを生成します。"""
        raw = str(args) + str(sorted(kwargs.items()))
        return hashlib.md5(raw.encode()).hexdigest()

    def get(self, key: str):
        """キャッシュから値を取得します。"""
        if key in self.cache:
            timestamp, value = self.cache[key]
            if time.time() - timestamp < self.ttl:
                # アクセス順序を更新
                if key in self.access_order:
                    self.access_order.remove(key)
                self.access_order.append(key)
                return value
            else:
                del self.cache[key]
        return None

    def set(self, key: str, value) -> None:
        """キャッシュに値を保存します。"""
        # サイズ上限チェック
        if len(self.cache) >= self.max_size and key not in self.cache:
            # 最も古いエントリを削除
            if self.access_order:
                oldest = self.access_order.pop(0)
                self.cache.pop(oldest, None)

        self.cache[key] = (time.time(), value)
        if key in self.access_order:
            self.access_order.remove(key)
        self.access_order.append(key)


# MCPツールでキャッシュを使用する例
api_cache = LRUCache(max_size=100, ttl_seconds=300)

@mcp.tool()
def get_data(query: str) -> str:
    """データを取得します（キャッシュ対応）。"""
    cache_key = api_cache._make_key(query)
    cached = api_cache.get(cache_key)
    if cached is not None:
        return f"{cached}\n（キャッシュから取得）"

    # 実際のデータ取得処理
    result = expensive_operation(query)
    api_cache.set(cache_key, result)
    return result
```

### タイムアウト設定

外部リソースへのアクセスにはタイムアウトを必ず設定します。

```python
import urllib.request
import subprocess

# HTTP リクエストのタイムアウト
def fetch_with_timeout(url: str, timeout: int = 10) -> str:
    """タイムアウト付きのHTTPリクエストです。"""
    try:
        req = urllib.request.Request(url)
        with urllib.request.urlopen(req, timeout=timeout) as response:
            return response.read().decode("utf-8")
    except urllib.error.URLError as e:
        return f"ネットワークエラー: {e}"
    except TimeoutError:
        return f"タイムアウト: {timeout}秒以内に応答がありませんでした。"

# サブプロセスのタイムアウト
def run_command_with_timeout(args: list[str], timeout: int = 30) -> str:
    """タイムアウト付きのコマンド実行です。"""
    try:
        result = subprocess.run(
            args,
            capture_output=True,
            text=True,
            timeout=timeout,
        )
        return result.stdout
    except subprocess.TimeoutExpired:
        return f"タイムアウト: コマンドが{timeout}秒以内に完了しませんでした。"
```

### ヘルスチェック（Health Check）

リモートMCPサーバーでは、ヘルスチェックエンドポイントを提供することが推奨されます。

```python
import time

SERVER_START_TIME = time.time()
REQUEST_COUNT = 0
ERROR_COUNT = 0

@mcp.tool()
def health_check() -> str:
    """サーバーのヘルス情報を返します。"""
    uptime = time.time() - SERVER_START_TIME
    hours = int(uptime // 3600)
    minutes = int((uptime % 3600) // 60)
    return (
        f"ステータス: 正常\n"
        f"稼働時間: {hours}時間{minutes}分\n"
        f"リクエスト数: {REQUEST_COUNT}\n"
        f"エラー数: {ERROR_COUNT}\n"
        f"エラー率: {ERROR_COUNT / max(REQUEST_COUNT, 1) * 100:.1f}%"
    )
```

### ポイントまとめ

- レート制限はトークンバケットアルゴリズムが実装しやすくおすすめです
- キャッシュにはTTL（有効期限）を設定し、古いデータが返されないようにします
- 外部リソースへのアクセスには必ずタイムアウトを設定してください
- ヘルスチェックにより、サーバーの稼働状態を監視できるようにします

## 8.6 デプロイメント

### ローカルデプロイ（stdioトランスポート）

最もシンプルなデプロイ方法です。Claude DesktopなどのMCPクライアントが直接サーバーを起動します。

```
┌──────────────────┐         stdin/stdout          ┌──────────────┐
│  Claude Desktop   │ ◄──────────────────────────► │  MCPサーバー    │
│  (MCPクライアント)  │        (stdioトランスポート)    │  (ローカル)    │
└──────────────────┘                               └──────────────┘
```

Claude Desktop設定（`claude_desktop_config.json`）：

```json
{
  "mcpServers": {
    "my-server": {
      "command": "python",
      "args": ["/path/to/server.py"],
      "env": {
        "DB_PATH": "/path/to/database.db"
      }
    }
  }
}
```

### リモートデプロイ（Streamable HTTPトランスポート）

ネットワーク経由でMCPサーバーにアクセスする方式です。

```
┌──────────────┐     HTTPS      ┌─────────────────────┐
│  MCPクライアント │ ──────────► │  リモートMCPサーバー     │
│  (どこからでも)  │             │  (Streamable HTTP)   │
└──────────────┘              └─────────────────────┘
```

```python
# remote_server.py - Streamable HTTPで公開するMCPサーバー
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("remote-server")

@mcp.tool()
def hello(name: str) -> str:
    """挨拶を返します。"""
    return f"こんにちは、{name}さん！"

# Streamable HTTPで起動（ポート8000）
if __name__ == "__main__":
    mcp.run(transport="streamable-http", host="0.0.0.0", port=8000)
```

### Docker化

```dockerfile

    CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')" || exit 1

EXPOSE 8000

CMD ["python", "server.py"]
```

### デプロイ方式の比較

| 項目 | stdio（ローカル） | Streamable HTTP（リモート） | Docker |
|------|-----------------|--------------------------|--------|
| セットアップ | 簡単 | 中程度 | 中程度 |
| ネットワーク | 不要 | 必要 | 必要 |
| スケーラビリティ | 低い | 高い | 高い |
| セキュリティ | OS依存 | HTTPS + 認証 | 隔離環境 |
| 運用監視 | 困難 | 可能 | 可能 |
| 主な用途 | 個人開発 | チーム/サービス | 本番運用 |

### ポイントまとめ

- ローカル開発にはstdio、チームやサービス向けにはStreamable HTTPを使います
- Docker化により、環境依存の問題を解消し、再現可能なデプロイを実現します
- CI/CDパイプラインで自動テスト・デプロイを構成し、品質を保ちます
- 非rootユーザーでの実行やヘルスチェックはDocker運用の基本です

## 8.7 MCPサーバー設計のベストプラクティスまとめ

### 命名規約（Naming Convention）

ツールとリソースの命名は、利用するAIアシスタントが正しく理解できるかを左右します。

```python
# ===== 悪い例 =====
@mcp.tool()
def proc(d: str) -> str:      # 名前が不明瞭
    """処理します。"""          # 説明も曖昧
    pass

@mcp.tool()
def doStuff(x: int) -> str:   # Python規約に違反（キャメルケース）
    pass

# ===== 良い例 =====
@mcp.tool()
def search_users(query: str) -> str:  # 動詞_名詞 の形式
    """ユーザーを名前またはメールアドレスで検索します。

    Args:
        query: 検索キーワード（名前またはメールの一部）
    """
    pass

@mcp.tool()
def get_file_content(file_path: str) -> str:  # get_ で取得系と分かる
    """指定されたファイルの内容を読み取ります。

    Args:
        file_path: 読み取るファイルの相対パス（ベースディレクトリからの相対）
    """
    pass
```

命名パターンの一覧：

| パターン | 例 | 用途 |
|---------|-----|------|
| `get_*` | `get_user`, `get_weather` | データ取得 |
| `list_*` | `list_tables`, `list_files` | 一覧取得 |
| `search_*` | `search_users`, `search_files` | 検索 |
| `create_*` | `create_user`, `create_file` | 新規作成 |
| `update_*` | `update_user`, `update_config` | 更新 |
| `delete_*` | `delete_user`, `delete_file` | 削除 |

### ドキュメント（Description）

ツールの説明文はAIアシスタントがツールを選択する際の判断材料になります。

```python
# 悪い例：説明が不十分
@mcp.tool()
def query(sql: str) -> str:
    """クエリを実行します。"""  # どんなクエリ？制約は？
    pass

# 良い例：詳細で正確な説明
@mcp.tool()
def query(sql: str) -> str:
    """SQLiteデータベースに対してSELECT文を実行し、結果をテーブル形式で返します。
    SELECT文のみ許可されます。データの変更にはexecuteツールを使用してください。
    結果が100行を超える場合はLIMIT句の使用を推奨します。

    Args:
        sql: 実行するSELECT文（例: "SELECT * FROM users WHERE age > 20"）
    """
    pass
```

### バージョニング（Versioning）

MCPサーバーのバージョン管理により、互換性の問題を防ぎます。

```python
from mcp.server.fastmcp import FastMCP

# サーバー名にバージョンを含める
mcp = FastMCP(
    "database-server",
    version="1.2.0",  # セマンティックバージョニング
)

# リソースでバージョン情報を公開
@mcp.resource("server://info")
def server_info() -> str:
    """サーバーのバージョンと機能情報を返します。"""
    return (
        "サーバー: database-server\n"
        "バージョン: 1.2.0\n"
        "対応ツール: query, list_tables, describe_table, execute\n"
        "データベース: SQLite 3\n"
    )
```

### エラーメッセージ

ユーザーフレンドリーなエラーメッセージを返すことが重要です。

```python
# 悪い例：技術的すぎるエラーメッセージ
return f"sqlite3.OperationalError: no such table: {table}"

# 良い例：原因と対処法を含むエラーメッセージ
return (
    f"エラー: テーブル '{table}' が見つかりません。\n"
    f"利用可能なテーブルはlist_tablesツールで確認できます。"
)
```

エラーメッセージの設計指針：

| 要素 | 説明 | 例 |
|-----|------|-----|
| 何が起きたか | エラーの概要 | 「テーブル 'users' が見つかりません」 |
| なぜ起きたか | 原因の説明 | 「指定されたテーブルはデータベースに存在しません」 |
| どうすればよいか | 解決策の提示 | 「list_tablesで利用可能なテーブルを確認してください」 |

### 設計チェックリスト

MCPサーバーをリリースする前に確認すべき項目です。

```
セキュリティ
  □ 最小権限の原則に従っているか
  □ 入力バリデーションは全引数に実装されているか
  □ パストラバーサル対策は実装されているか
  □ SQLインジェクション対策は実装されているか
  □ 機密情報は環境変数で管理されているか

品質
  □ 全ツールに型ヒントとdocstringがあるか
  □ エラーメッセージは原因と対処法を含むか
  □ ユニットテストが書かれているか
  □ タイムアウトが設定されているか

運用
  □ ログが記録されているか
  □ ヘルスチェックが実装されているか（リモートの場合）
  □ レート制限が実装されているか
  □ バージョン情報が公開されているか
```

### よくある間違い

| 間違い | 正しい設計 |
|-------|-----------|
| ツール名が曖昧（`process`, `run`等） | `search_users`, `get_weather` 等、動詞_名詞 形式にする |
| docstringが1行しかない | 制約条件、使用例、引数の説明を含む詳細な説明を書く |
| エラー時にPythonの例外メッセージをそのまま返す | ユーザーが理解できるメッセージに変換して返す |
| ツールの数が多すぎる | 1サーバーあたり5-10個程度に抑え、責務を分離する |
| 全操作を1つのツールにまとめる | 操作ごとにツールを分ける（read / write / delete） |

### ポイントまとめ

- ツール名は `動詞_名詞` 形式で統一し、AIが正しく選択できるようにします
- docstringは「何ができるか」「制約は何か」「引数の例」を含む詳細なものにします
- セマンティックバージョニングでバージョン管理し、破壊的変更を追跡します
- エラーメッセージには「何が起きたか」「なぜか」「どうすればよいか」の3要素を含めます
- リリース前に設計チェックリストを活用し、品質を担保します
