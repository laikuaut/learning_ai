## 基本問題

### 問題1: 3種類のフォワーディング
ポートフォワーディングの3つの種類（ローカル、リモート、ダイナミック）の特徴を、それぞれ1行で説明してください。

<details>
<summary>解答</summary>

```
ローカルフォワーディング（-L）:
  ローカルのポートへのアクセスを、SSHサーバー経由でリモートのサービスに転送する

リモートフォワーディング（-R）:
  リモートのポートへのアクセスを、SSHクライアント（ローカル）側のサービスに転送する

ダイナミックフォワーディング（-D）:
  SOCKSプロキシとして動作し、任意の宛先への通信をSSH経由で転送する
```

</details>

### 問題2: ローカルフォワーディングの基本
以下の操作をコマンドで書いてください。

1. リモートサーバー `server` の80番ポートをローカルの8080番にフォワード
2. リモートサーバー `server` のPostgreSQL（5432番）をローカルの15432番にフォワード
3. 上記2つを同時にフォワードする

<details>
<summary>解答</summary>

```bash
# 1.
$ ssh -L 8080:localhost:80 user@server

# 2.
$ ssh -L 15432:localhost:5432 user@server

# 3. 同時にフォワード
$ ssh -L 8080:localhost:80 -L 15432:localhost:5432 user@server
```

</details>

### 問題3: オプションの組み合わせ
以下のコマンドの各オプションの意味を説明してください。

```bash
$ ssh -f -N -L 8080:localhost:80 user@server
```

<details>
<summary>解答</summary>

```
-f  バックグラウンドで実行する（コマンド入力後にシェルに戻る）
-N  リモートコマンドを実行しない（シェルを開かない）
-L 8080:localhost:80  ローカルの8080番ポートを、
                      SSHサーバーから見た localhost:80 に転送する

つまり、バックグラ��ンドでポートフォワーディングだけを行い、
シェルセッションは開かない設定です。
```

</details>

### 問題4: フォワーディングの構文
`-L` オプションの構文 `-L ローカルポート:リモートホスト:リモートポート` において、「リモートホスト」は何から見たホスト名ですか？

```bash
$ ssh -L 3000:internal-web:3000 user@bastion
```

<details>
<summary>解答</summary>

「リモートホスト」は **SSHサーバー（bastion）から見た**ホスト名です。

```
この例では:
- bastion（SSHサーバー）から見た internal-web の3000番ポートに転送
- internal-web はクライアント（ローカルPC）からは直接アクセスできないが、
  bastion からはアクセスできるホスト

+----------+          +----------+          +--------------+
| ローカル  |   SSH    | bastion  |          | internal-web |
| :3000    |=========>|          |--------->| :3000        |
+----------+          +----------+          +--------------+
                       ↑ここから見た
                         ホスト名
```

</details>

### 問題5: リモートフォワーディング
以下のシナリオをコマンドで実現してください。

「ローカルPCで開発中のWebアプリ（ポート3000）を、リモートサーバー `vps` のポート8080で公開して、チームメンバーがアクセスできるよ���にしたい」

<details>
<summary>ヒント</summary>

ローカルのサービスをリモー��に公開するのは「リモートフォワーディング（-R）」です。

</details>

<details>
<summary>解答</summary>

```bash
# リモートフォワーディング
$ ssh -R 8080:localhost:3000 user@vps

# バックグラウンドで実行する場合
$ ssh -f -N -R 8080:localhost:3000 user@vps

# チームメンバーは以下でアクセス:
# http://vps:8080
```

注意: リモートサーバーの `sshd_config` で `GatewayPorts yes` が設定されていないと、リモートの localhost からしかアクセスできません。

</details>

### 問題6: 踏み台経由のフォワーディング
以下の構成で、ローカルPCから内部のRedis（10.0.1.30:6379）に接続するためのコマンドを書いてください。

```
ローカルPC → bastion.example.com（踏み台） → 10.0.1.30（Redis）
```

<details>
<summary>解答</summary>

```bash
# 方法1: コマンドライン
$ ssh -L 6379:10.0.1.30:6379 user@bastion.example.com

# 方法2: バックグラウンドで実行
$ ssh -f -N -L 6379:10.0.1.30:6379 user@bastion.example.com

# Redisに接続
$ redis-cli -h localhost -p 6379

# 方法3: ~/.ssh/config で設定
# Host bastion
#     HostName bastion.example.com
#     User user
#     LocalForward 6379 10.0.1.30:6379
```

</details>

### 問題7: ダイナミックフォワーディング
ダイナミックフォワーディング（SOCKSプロキシ）をローカルのポート1080で起動するコマンドを書いてください。また、curl を使ってそのプロキシ経由でWebページにアクセスするコマンドも書いてください。

<details>
<summary>解答</summary>

```bash
# SOCKSプロキシの起動
$ ssh -D 1080 user@remote-server
# または
$ ssh -f -N -D 1080 user@remote-server

# curl でプロキシ経由アクセス
$ curl --socks5-hostname localhost:1080 https://example.com
```

ブラウザで使用する場合は、プロキシ設定で「SOCKSホスト: localhost、ポート: 1080」を指定します。

</details>

### 問題8: ~/.ssh/config でのフォワーディング
以下のフォワーディング設定を `~/.ssh/config` で書いてください。

1. ホスト `dev` で、ローカル8080をリモートlocalhost:80に転送
2. ホスト `dev` で、ローカル15432をリモートlocalhost:5432に転送
3. ホスト `proxy` で、SOCKSプロキシをポート1080で起動

<details>
<summary>解答</summary>

```
Host dev
    HostName dev.example.com
    User developer
    LocalForward 8080 localhost:80
    LocalForward 15432 localhost:5432

Host proxy
    HostName proxy-server.example.com
    User user
    DynamicForward 1080
```

使い方:
```bash
$ ssh dev     # 接続と同時にポートフォワーデ���ングが有効になる
$ ssh proxy   # 接続と同時にSOCKSプロキシが起動する
```

</details>

## チャレンジ問題

### 問題9: 実務シナリオ
以下の状況で適切なポートフォワーディングの種類とコマンドを答えてください。

**シナリオ1**: 開発環境のDB（PostgreSQL）にローカルのpgAdminから接続したい。DBは踏み台サーバー経由でしかアクセスできない。

**シナリオ2**: 自宅のNAS（192.168.1.100:8080）を、外出先からVPS経由でアクセスしたい。

**シナリオ3**: カフェの公共Wi-Fiで作業する際、すべてのWeb通信をSSH経由で暗号化したい。

<details>
<summary>解答</summary>

**シナリオ1: ローカルフォワーディング（-L）**
```bash
$ ssh -L 5432:db-server:5432 user@bastion
# pgAdmin から localhost:5432 に接続
```

**シナリオ2: リモートフォワーディング（-R）**
```bash
# 自宅のPCから実行（NASにアクセスできるPC）
$ ssh -R 8080:192.168.1.100:8080 user@vps
# 外出先から vps:8080 にアクセスしてNASに到達
```

**シナリオ3: ダイナミックフォワーディング（-D）**
```bash
$ ssh -D 1080 user@trusted-server
# ブラウザのプロキシ設定: SOCKS5 localhost:1080
# すべてのWeb通信がSSH経由で暗号化される
```

</details>

### 問題10: フォワーディン��の停止
バックグラウンドで実行中のポートフォワーディングを停止する方法を2つ答えてください。

```bash
$ ssh -f -N -L 8080:localhost:80 user@server
```

<details>
<summary>解答</summary>

```bash
# 方法1: プロセスを検索してkill
$ ps aux | grep "ssh -f"
user  12345  ... ssh -f -N -L 8080:localhost:80 user@server
$ kill 12345

# 方法2: ポートを使用しているプロセスを特定してkill
$ lsof -i :8080
COMMAND   PID USER   FD   TYPE DEVICE SIZE/OFF NODE NAME
ssh     12345 user    4u  IPv4  12345      0t0  TCP localhost:8080 (LISTEN)
$ kill 12345

# 方法3（ControlMaster使用時）:
$ ssh -O exit server
```

</details>
