# 実践課題04：コンフリクト解消トレーニング ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第2章（基本操作）、第3章（ブランチとマージ）
> **課題の種類**: ミニプロジェクト
> **学習目標**: マージ時のコンフリクト（conflict / 競合）を意図的に発生させ、解消の手順を確実に身につける

---

## 完成イメージ

コンフリクト発生時のファイル内容：

```
<<<<<<< HEAD
定価: 500円（税込550円）
=======
定価: 480円（キャンペーン価格）
>>>>>>> feature/campaign
```

解消後のファイル内容：

```
定価: 480円（キャンペーン価格、通常500円 / 税込550円）
```

```
$ git log --oneline --graph
*   a1b2c3d (HEAD -> main) コンフリクト解消：通常価格とキャンペーン価格を併記
|\
| * d4e5f6a (feature/campaign) キャンペーン価格に変更
|/
* b2c3d4e 商品情報の作成
```

---

## 課題の要件

1. `main` ブランチで商品情報ファイルを作成してコミットする
2. `feature/campaign` ブランチを作成し、価格をキャンペーン価格に変更する
3. `main` ブランチに戻り、同じ行を別の内容に変更する（税込表示の追加）
4. `main` に `feature/campaign` をマージし、コンフリクトを発生させる
5. コンフリクトマーカー（`<<<<<<<`、`=======`、`>>>>>>>`）を理解する
6. 手動でコンフリクトを解消し、マージコミットを完了させる

<details>
<summary>ステップ5：コンフリクトを解消する</summary>

ファイルを開くと、以下のようなコンフリクトマーカーが入っています。

```
<<<<<<< HEAD
定価: 500円（税込550円）
=======
定価: 480円（キャンペーン価格）
>>>>>>> feature/campaign
```

これを手動で、両方の意図を活かした内容に書き換えます。

```bash
# エディタで product.txt を編集し、コンフリクト部分を以下に書き換える：
# 定価: 480円（キャンペーン価格、通常500円 / 税込550円）

# 解消したらステージングしてコミット
git add product.txt
git commit -m "コンフリクト解消：通常価格とキャンペーン価格を併記"
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け：丁寧に1ステップずつ）</summary>

```bash
# ===== 準備 =====
mkdir conflict-practice
cd conflict-practice
git init

# ===== ベースファイルの作成 =====
cat << 'EOF' > product.txt
===== 商品情報 =====

商品名: プレミアムブレンドコーヒー
定価: 500円
内容量: 200g
原産国: ブラジル・コロンビア
EOF

git add product.txt
git commit -m "商品情報の作成"

# ===== ブランチでキャンペーン価格に変更 =====
git switch -c feature/campaign

sed -i 's/定価: 500円/定価: 480円（キャンペーン価格）/' product.txt

# 変更を確認
git diff
# 出力例:
# -定価: 500円
# +定価: 480円（キャンペーン価格）

git add product.txt
git commit -m "キャンペーン価格に変更"

# ===== main で税込表示を追加 =====
git switch main

# main ではまだ元のまま（500円）であることを確認
cat product.txt

sed -i 's/定価: 500円/定価: 500円（税込550円）/' product.txt

git add product.txt
git commit -m "税込価格を追加"

# ===== マージ → コンフリクト発生 =====
git merge feature/campaign
# 出力:
# Auto-merging product.txt
# CONFLICT (content): Merge conflict in product.txt
# Automatic merge failed; fix conflicts and then commit the result.

# 状態確認
git status

# コンフリクトマーカーを確認
cat product.txt
# 出力例:
# ===== 商品情報 =====
#
# 商品名: プレミアムブレンドコーヒー
# <<<<<<< HEAD
# 定価: 500円（税込550円）
# =======
# 定価: 480円（キャンペーン価格）
# >>>>>>> feature/campaign
# 内容量: 200g
# 原産国: ブラジル・コロンビア

# ===== コンフリクトマーカーの見方 =====
# <<<<<<< HEAD        ← 現在のブランチ（main）の内容の開始
# 定価: 500円（税込550円）
# =======             ← 区切り線
# 定価: 480円（キャンペーン価格）
# >>>>>>> feature/campaign ← マージしようとしたブランチの内容の終了

# ===== コンフリクト解消 =====
# ファイルを手動で編集して、両方の変更を統合する
cat << 'EOF' > product.txt
===== 商品情報 =====

商品名: プレミアムブレンドコーヒー
定価: 480円（キャンペーン価格、通常500円 / 税込550円）
内容量: 200g
原産国: ブラジル・コロンビア
EOF

# 解消をステージングしてコミット
git add product.txt
git commit -m "コンフリクト解消：通常価格とキャンペーン価格を併記"

# ===== 確認 =====
git log --oneline --graph --all
cat product.txt

# ===== 後片付け =====
git branch -d feature/campaign
```

</details>

<details>
<summary>解答例（改良版 ─ コンフリクト解消の3パターン）</summary>

コンフリクトの解消方法は状況によって使い分けます。以下の3パターンを練習しましょう。

```bash
# ===== 準備（初心者向けと同じ手順でコンフリクトを発生させる）=====
mkdir conflict-practice
cd conflict-practice
git init

cat << 'EOF' > product.txt
===== 商品情報 =====

商品名: プレミアムブレンドコーヒー
定価: 500円
内容量: 200g
原産国: ブラジル・コロンビア
EOF

git add product.txt
git commit -m "商品情報の作成"

git switch -c feature/campaign
sed -i 's/定価: 500円/定価: 480円（キャンペーン価格）/' product.txt
git add product.txt
git commit -m "キャンペーン価格に変更"

git switch main
sed -i 's/定価: 500円/定価: 500円（税込550円）/' product.txt
git add product.txt
git commit -m "税込価格を追加"

# ===== パターン1：両方を統合する（推奨） =====
git merge feature/campaign
# CONFLICT 発生

# 手動で両方の意図を活かす
cat << 'EOF' > product.txt
===== 商品情報 =====

商品名: プレミアムブレンドコーヒー
定価: 480円（キャンペーン価格、通常500円 / 税込550円）
内容量: 200g
原産国: ブラジル・コロンビア
EOF

git add product.txt
git commit -m "コンフリクト解消：両方の変更を統合"

# ===== マージを取り消して別パターンを試す場合 =====
# git reset --hard HEAD~1  # 直前のマージコミットを取り消す

# ===== パターン2：自分（HEAD）の変更を優先する =====
# git merge feature/campaign
# git checkout --ours product.txt   # HEAD 側を採用
# git add product.txt
# git commit -m "コンフリクト解消：main 側の変更を採用"

# ===== パターン3：相手の変更を優先する =====
# git merge feature/campaign
# git checkout --theirs product.txt  # マージ元を採用
# git add product.txt
# git commit -m "コンフリクト解消：campaign 側の変更を採用"

# ===== マージを中止する場合 =====
# コンフリクト発生後にマージ自体をやめたいとき：
# git merge --abort
```

**初心者向けとの違い:**
- `--ours`（自分側を採用）と `--theirs`（相手側を採用）の使い分けを紹介
- `git merge --abort` でマージを中止する方法も紹介
- 実務では「両方を統合」が最も多いパターンですが、状況によって使い分けが必要

</details>
