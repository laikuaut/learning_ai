# 実践課題01：基本コマンド操作 ★1

> **難易度**: ★☆☆☆☆（入門）
> **前提知識**: 第1章（Linuxの基本）
> **課題の種類**: ミニプロジェクト
> **学習目標**: ターミナルでの基本的なコマンド操作に慣れ、システム情報を確認できるようになる

---

## 完成イメージ

以下のような情報を順番に確認し、結果を1つのテキストファイルにまとめます。

```
===== システム情報レポート =====
作成日時: 2024-04-01 15:30:00

【OS情報】
NAME="Ubuntu"
VERSION="22.04.3 LTS (Jammy Jellyfish)"

【カーネルバージョン】
Linux myhost 5.15.0-91-generic #101-Ubuntu SMP x86_64

【ホスト名】
myhost

【現在のユーザー】
tanaka

【現在のディレクトリ】
/home/tanaka

【ディスク使用量】
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        50G   12G   36G  25% /

【メモリ使用量】
              total        used        free
Mem:           7.7G        2.1G        4.2G

【稼働時間】
 15:30:00 up 30 days, 5:23, 1 user
```

---

## 課題の要件

1. 以下のコマンドを実行して、それぞれの出力を確認する
   - `cat /etc/os-release`（OS情報）
   - `uname -a`（カーネル情報）
   - `hostname`（ホスト名）
   - `whoami`（現在のユーザー）
   - `pwd`（現在のディレクトリ）
   - `df -h`（ディスク使用量）
   - `free -h`（メモリ使用量）
   - `uptime`（稼働時間）
2. 上記の結果をすべて `system_report.txt` にまとめて保存する
3. 保存したファイルの内容を `cat` で確認する

---

## ステップガイド

<details>
<summary>ステップ1：各コマンドを1つずつ実行して確認する</summary>

まずはそれぞれのコマンドを手動で実行し、出力の意味を理解しましょう。

```bash
# OS情報を確認
$ cat /etc/os-release

# カーネル情報を確認
$ uname -a

# ホスト名を確認
$ hostname

# 現在のユーザーを確認
$ whoami

# 現在のディレクトリを確認
$ pwd

# ディスク使用量を確認
$ df -h

# メモリ使用量を確認
$ free -h

# システム稼働時間を確認
$ uptime
```

各コマンドの意味がわからない場合は `man コマンド名` でマニュアルを確認できます。
例: `man uname`

</details>

<details>
<summary>ステップ2：結果をファイルに書き出す</summary>

リダイレクト（redirect）を使って出力をファイルに書き込みます。

- `>` : ファイルに上書き（最初のコマンドで使う）
- `>>` : ファイルに追記（2つ目以降のコマンドで使う）

```bash
# 最初にヘッダーを書き込む（> で新規作成）
echo "===== システム情報レポート =====" > system_report.txt
echo "作成日時: $(date '+%Y-%m-%d %H:%M:%S')" >> system_report.txt
echo "" >> system_report.txt

# OS情報を追記（>> で追記）
echo "【OS情報】" >> system_report.txt
cat /etc/os-release >> system_report.txt
echo "" >> system_report.txt
```

同様に残りのコマンドも `>>` で追記していきます。

</details>

<details>
<summary>ステップ3：ファイルの内容を確認する</summary>

```bash
# ファイル全体を表示
$ cat system_report.txt

# ファイルのサイズも確認してみましょう
$ ls -l system_report.txt

# 行数を数える
$ wc -l system_report.txt
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

```bash
#!/bin/bash
# システム情報レポート作成スクリプト
# 学べる内容：基本コマンド、リダイレクト
# 実行方法：bash system_report.sh

REPORT="system_report.txt"

# ヘッダー
echo "===== システム情報レポート =====" > "$REPORT"
echo "作成日時: $(date '+%Y-%m-%d %H:%M:%S')" >> "$REPORT"
echo "" >> "$REPORT"

# OS情報
echo "【OS情報】" >> "$REPORT"
cat /etc/os-release >> "$REPORT"
echo "" >> "$REPORT"

# カーネルバージョン
echo "【カーネルバージョン】" >> "$REPORT"
uname -a >> "$REPORT"
echo "" >> "$REPORT"

# ホスト名
echo "【ホスト名】" >> "$REPORT"
hostname >> "$REPORT"
echo "" >> "$REPORT"

# 現在のユーザー
echo "【現在のユーザー】" >> "$REPORT"
whoami >> "$REPORT"
echo "" >> "$REPORT"

# 現在のディレクトリ
echo "【現在のディレクトリ】" >> "$REPORT"
pwd >> "$REPORT"
echo "" >> "$REPORT"

# ディスク使用量
echo "【ディスク使用量】" >> "$REPORT"
df -h >> "$REPORT"
echo "" >> "$REPORT"

# メモリ使用量
echo "【メモリ使用量】" >> "$REPORT"
free -h >> "$REPORT"
echo "" >> "$REPORT"

# 稼働時間
echo "【稼働時間】" >> "$REPORT"
uptime >> "$REPORT"
echo "" >> "$REPORT"

echo "レポートを ${REPORT} に保存しました。"

```

</details>

<details>
<summary>解答例（改良版 ─ 関数を使って整理）</summary>

```bash
#!/bin/bash
# システム情報レポート作成スクリプト（改良版）
# 学べる内容：関数、ヒアドキュメント、変数展開
# 実行方法：bash system_report_v2.sh

REPORT="system_report.txt"

# セクションを書き込む関数
add_section() {
    local title="$1"
    shift
    echo "【${title}】" >> "$REPORT"
    "$@" >> "$REPORT"
    echo "" >> "$REPORT"
}

# ヘッダー
cat > "$REPORT" << EOF
===== システム情報レポート =====
作成日時: $(date '+%Y-%m-%d %H:%M:%S')
ホスト名: $(hostname)
ユーザー: $(whoami)

EOF

# 各セクションを関数で追加
add_section "OS情報" cat /etc/os-release
add_section "カーネルバージョン" uname -a
add_section "現在のディレクトリ" pwd
add_section "ディスク使用量" df -h
add_section "メモリ使用量" free -h
add_section "稼働時間" uptime

echo "レポートを ${REPORT} に保存しました（$(wc -l < "$REPORT") 行）。"
```

**初心者向けとの違い:**
- `add_section` 関数でセクション追加を共通化 → コードの重複を排除
- ヒアドキュメント（`<< EOF`）でヘッダー部分を見やすく記述
- `wc -l` で行数を表示し、レポートの規模を確認できる
- 関数の引数にコマンドそのものを渡す柔軟な設計

</details>
