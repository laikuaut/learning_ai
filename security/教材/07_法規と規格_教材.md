# 第7章：法規と規格

## この章のゴール

- 個人情報保護法の基本概念と事業者の義務を理解すること
- 不正アクセス禁止法の対象行為を説明できること
- サイバーセキュリティ基本法の目的と体制を把握すること
- ISO/IEC 27001（ISMS）等の国際規格を理解すること
- その他の関連法規（電子署名法、マイナンバー法等）の概要を知ること

---

## 7.1 個人情報保護法

### 7.1.1 個人情報保護法の概要

**個人情報の保護に関する法律**（個人情報保護法）は、個人情報の適正な取扱いに関するルールを定め、個人の権利利益を保護することを目的とした法律です。

### 7.1.2 基本用語

| 用語 | 定義 |
|------|------|
| **個人情報** | 生存する個人に関する情報で、特定の個人を識別できるもの。氏名、生年月日、顔写真等。個人識別符号（マイナンバー、DNA、指紋等）も含む |
| **要配慮個人情報** | 人種、信条、病歴、犯罪歴、障害等の、不当な差別・偏見が生じ得る情報。取得に原則本人同意が必要 |
| **個人情報データベース等** | 個人情報を含む情報の集合物で、検索できるように体系的に構成したもの |
| **個人データ** | 個人情報データベース等を構成する個人情報 |
| **保有個人データ** | 個人情報取扱事業者が開示・訂正・削除等の権限を有する個人データ |
| **個人情報取扱事業者** | 個人情報データベース等を事業に利用している者（規模の要件は撤廃） |

### 7.1.3 事業者の主な義務

```
【個人情報取扱事業者の義務（概要）】

  ① 利用目的の特定と通知・公表
     ├── 利用目的をできる限り特定する
     ├── 取得時に利用目的を本人に通知又は公表
     └── 目的外利用の禁止（本人同意がない限り）

  ② 適正な取得
     ├── 偽りその他不正な手段で取得してはならない
     └── 要配慮個人情報は原則本人同意が必要

  ③ 安全管理措置
     ├── 組織的安全管理措置（体制整備）
     ├── 人的安全管理措置（従業員の教育・監督）
     ├── 物理的安全管理措置（入退室管理）
     └── 技術的安全管理措置（アクセス制御、暗号化）

  ④ 第三者提供の制限
     ├── 原則として本人同意が必要
     └── オプトアウト手続きによる例外あり

  ⑤ 本人の権利への対応
     ├── 開示請求への対応
     ├── 訂正・削除請求への対応
     └── 利用停止・消去請求への対応

  ⑥ 漏洩等の報告義務
     ├── 個人情報保護委員会への報告
     └── 本人への通知
```

### 7.1.4 2022年改正のポイント

| 改正内容 | 説明 |
|---------|------|
| 漏洩等の報告義務化 | 一定の漏洩発生時に個人情報保護委員会への報告と本人通知が義務化 |
| 個人の権利の拡充 | 利用停止・消去の請求権の拡大 |
| 罰則の強化 | 法人に対する罰金の上限引き上げ（最大1億円） |
| 仮名加工情報の新設 | 他の情報と照合しない限り特定の個人を識別できないよう加工した情報 |

## 7.2 不正アクセス禁止法

### 7.2.1 不正アクセス禁止法の概要

**不正アクセス行為の禁止等に関する法律**は、ネットワークを通じた不正アクセス行為とその助長行為を禁止する法律です。

### 7.2.2 禁止される行為

```
【不正アクセス禁止法の対象行為】

  ① 不正アクセス行為
     ├── なりすまし行為: 他人のID・パスワードを使って不正ログイン
     ├── セキュリティホール攻撃: 脆弱性を突いてアクセス制御を突破
     └── アクセス制御機能を回避して不正にアクセス

  ② 不正アクセスを助長する行為
     └── 他人のID・パスワードを無断で第三者に提供

  ③ 不正に取得する行為
     └── フィッシング等で他人のID・パスワードを不正取得

  ④ 不正に保管する行為
     └── 不正に取得したID・パスワードを保管

  ⑤ 入力を求める行為
     └── フィッシングサイトを設置して認証情報の入力を求める
```

### 7.2.3 罰則

| 行為 | 罰則 |
|------|------|
| 不正アクセス行為 | 3年以下の懲役又は100万円以下の罰金 |
| 不正アクセスを助長する行為 | 1年以下の懲役又は50万円以下の罰金 |
| ID・パスワードの不正取得・保管 | 1年以下の懲役又は50万円以下の罰金 |

> **注意**: 不正アクセス禁止法は「ネットワーク経由」のアクセスが対象です。物理的にPCを操作して不正にログインする行為は、本法の直接的な対象ではありません（他の法律で対処）。

## 7.3 サイバーセキュリティ基本法

### 7.3.1 概要

**サイバーセキュリティ基本法**（2014年制定）は、サイバーセキュリティに関する施策を総合的かつ効果的に推進するための基本理念と国の責務を定めた法律です。

### 7.3.2 主な内容

| 項目 | 内容 |
|------|------|
| 目的 | サイバーセキュリティに関する施策の推進 |
| NISC | 内閣サイバーセキュリティセンターを設置 |
| サイバーセキュリティ戦略本部 | 内閣に設置、国家戦略を策定 |
| 国の責務 | サイバーセキュリティに関する総合的な施策の策定・実施 |
| 重要インフラ | 電力、ガス、水道、通信、金融等の保護 |

## 7.4 その他の関連法規

### 7.4.1 主要な法規一覧

| 法規 | 概要 |
|------|------|
| **電子署名及び認証業務に関する法律（電子署名法）** | 電子署名に手書き署名と同等の法的効力を付与。特定認証業務（認証局）の認定制度を規定 |
| **マイナンバー法（番号法）** | 個人番号（マイナンバー）の利用範囲を社会保障・税・災害対策に限定。特定個人情報の厳格な保護を規定 |
| **刑法（電磁的記録に関する罪）** | 電子計算機損壊等業務妨害罪、不正指令電磁的記録（マルウェア）に関する罪 |
| **特定電子メール法（迷惑メール防止法）** | 広告宣伝メールの送信にオプトイン（事前同意）を義務化 |
| **電気通信事業法** | 電気通信事業者の通信の秘密の保護義務 |
| **著作権法** | プログラムも著作物として保護される |
| **不正競争防止法** | 営業秘密の不正取得・使用・開示を禁止 |

### 7.4.2 営業秘密の3要件（不正競争防止法）

不正競争防止法で保護される「営業秘密」として認められるには、以下の3要件を全て満たす必要があります。

| 要件 | 説明 |
|------|------|
| **秘密管理性** | 秘密として管理されていること（アクセス制限、マル秘表示等） |
| **有用性** | 事業活動に有用な情報であること |
| **非公知性** | 公然と知られていないこと |

## 7.5 国際規格・ガイドライン

### 7.5.1 ISO/IEC 27000ファミリー

| 規格番号 | 名称 | 内容 |
|---------|------|------|
| **ISO/IEC 27000** | 概要と用語 | ISMS関連用語の定義 |
| **ISO/IEC 27001** | ISMS要求事項 | ISMS認証の基準（認証規格） |
| **ISO/IEC 27002** | 実践規範 | 管理策の実施ガイドライン |
| **ISO/IEC 27005** | リスクマネジメント | 情報セキュリティリスク管理のガイドライン |
| **ISO/IEC 27017** | クラウドセキュリティ | クラウドサービスの管理策 |
| **ISO/IEC 27701** | プライバシー | プライバシー情報管理のための拡張 |

### 7.5.2 その他の重要な規格・ガイドライン

| 規格/ガイドライン | 説明 |
|-----------------|------|
| **ISO/IEC 15408（Common Criteria / CC）** | IT製品・システムのセキュリティ評価基準。EAL1〜7のレベルで評価 |
| **PCI DSS** | クレジットカード業界のセキュリティ基準 |
| **NIST SP 800シリーズ** | 米国NISTが発行するセキュリティガイドライン |
| **NIST CSF（Cybersecurity Framework）** | サイバーセキュリティのフレームワーク（識別、防御、検知、対応、復旧） |
| **CIS Controls** | サイバーセキュリティの優先的対策リスト |
| **GDPR** | EU一般データ保護規則。EU域内の個人データ保護 |

### 7.5.3 ISMS認証取得の流れ

```
【ISMS認証（ISO 27001）取得の流れ】

  ① 準備
     ├── 適用範囲の決定
     ├── リスクアセスメントの実施
     └── 管理策の選定と実施

  ② 文書化
     ├── 情報セキュリティポリシーの策定
     ├── 適用宣言書（SoA）の作成
     └── 各種手順書の整備

  ③ 運用
     ├── 管理策の実施
     ├── 教育・訓練
     └── 内部監査の実施

  ④ 審査
     ├── 第1段階審査（文書審査）
     └── 第2段階審査（実地審査）

  ⑤ 認証取得
     └── 3年ごとの更新審査（毎年の維持審査）
```

## 7.6 まとめ

### ポイントまとめ

1. **個人情報保護法** は個人情報の適正な取扱いを義務付けます。安全管理措置と漏洩時の報告義務が重要です
2. **不正アクセス禁止法** はネットワーク経由の不正アクセスとその助長・準備行為を禁止します
3. **サイバーセキュリティ基本法** は国のサイバーセキュリティ戦略の基盤となる法律です
4. **ISO/IEC 27001** はISMS認証の国際規格で、体系的なセキュリティ管理を実現します
5. **営業秘密** の保護には秘密管理性・有用性・非公知性の3要件が必要です
6. **GDPR** はEU域内の個人データ保護に関する規則で、グローバルビジネスでは遵守が必要です

### 試験対策キーワード

- 個人情報保護法、個人情報、要配慮個人情報、安全管理措置
- 不正アクセス禁止法、なりすまし、セキュリティホール攻撃
- サイバーセキュリティ基本法、NISC
- ISO 27001、ISO 27002、ISMS認証
- 電子署名法、マイナンバー法、不正競争防止法
- 営業秘密（秘密管理性、有用性、非公知性）
- PCI DSS、NIST CSF、GDPR
