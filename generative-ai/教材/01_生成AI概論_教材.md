## 1.1 生成AI（Generative AI）とは

### 生成AIの定義

生成AI（Generative AI）とは、学習したデータのパターンをもとに、新しいコンテンツ（テキスト、画像、音声、動画、コードなど）を生成できる人工知能の総称です。

従来のAIが「分類する」「予測する」「検出する」といった分析的なタスクを得意としていたのに対し、生成AIは「つくる」ことができる点が大きな特徴です。

```
┌─────────────────────────────────────────────────┐
│              AIの進化と生成AIの位置づけ            │
├─────────────────────────────────────────────────┤
│                                                  │
│  従来のAI（識別AI / Discriminative AI）           │
│  ┌──────────────────────────────────┐            │
│  │ 入力 → [モデル] → 分類・予測     │            │
│  │ 例：画像 → 「猫」「犬」          │            │
│  │ 例：メール → 「スパム」「正常」   │            │
│  └──────────────────────────────────┘            │
│                     ↓ 進化                       │
│  生成AI（Generative AI）                          │
│  ┌──────────────────────────────────┐            │
│  │ 入力（指示）→ [モデル] → 新しい  │            │
│  │                       コンテンツ │            │
│  │ 例：「猫の絵を描いて」→ 猫の画像 │            │
│  │ 例：「要約して」→ 要約文         │            │
│  └──────────────────────────────────┘            │
└─────────────────────────────────────────────────┘
```

### 従来のAIとの違い

| 項目 | 従来のAI（識別AI） | 生成AI |
|------|-------------------|--------|
| 主な目的 | 分類・予測・検出 | 新しいコンテンツの生成 |
| 入力と出力 | データ → ラベル・数値 | 指示（プロンプト） → コンテンツ |
| 代表例 | スパムフィルタ、画像認識 | ChatGPT、Claude、Stable Diffusion |
| 学習方法 | 教師あり学習が中心 | 大量のデータからパターンを自己学習 |
| 出力の性質 | 決定的（同じ入力に同じ出力） | 確率的（同じ入力でも異なる出力） |
| 必要データ量 | タスク固有のデータセット | 大規模な汎用データ |

### 基盤モデル（Foundation Model）の概念

基盤モデル（Foundation Model）とは、大量のデータで事前学習（Pre-training）された大規模なAIモデルのことです。特定のタスクに特化せず、さまざまなタスクに対応できる汎用性を持っています。

```
┌─────────────────────────────────────────────────┐
│            基盤モデルの概念図                      │
├─────────────────────────────────────────────────┤
│                                                  │
│   大量のデータ（テキスト、画像、コード等）          │
│        ↓ 事前学習（Pre-training）                │
│   ┌──────────────────────────┐                   │
│   │      基盤モデル          │                   │
│   │  （Foundation Model）    │                   │
│   └──────────┬───────────────┘                   │
│       ┌──────┼──────┬──────┐                     │
│       ↓      ↓      ↓      ↓                    │
│     チャット 翻訳  要約  コード生成               │
│     ボット          ツール  支援                  │
│                                                  │
│   ※ 1つのモデルを多用途に活用できる               │
└─────────────────────────────────────────────────┘
```

基盤モデルの代表的な特徴は以下のとおりです。

- **大規模性**：数十億〜数兆のパラメータを持つ
- **汎用性**：ファインチューニング（Fine-tuning）により多様なタスクに適応可能
- **創発能力**（Emergent Ability）：学習していない新しいタスクにも対応できる

> **ポイントまとめ**
> - 生成AIは「新しいコンテンツをつくる」AIである
> - 従来のAIが「分類・予測」なら、生成AIは「創造」が得意
> - 基盤モデルは1つのモデルで多様なタスクに対応できる汎用的なAI

## 1.2 生成AIの種類と分類

生成AIは、生成するコンテンツの種類によって大きく分類できます。

### 生成AIの分類一覧

| 種類 | 説明 | 代表的なモデル・サービス |
|------|------|------------------------|
| テキスト生成 | 文章・コードの生成 | Claude、GPT-4o、Gemini |
| 画像生成 | イラスト・写真風画像の生成 | Stable Diffusion、Midjourney、DALL-E 3 |
| 音声生成 | 音声合成・音楽生成 | ElevenLabs、Suno、Udio |
| 動画生成 | 動画コンテンツの生成 | Sora、Runway、Kling |
| コード生成 | プログラムコードの生成・補完 | GitHub Copilot、Claude Code、Cursor |
| 3D生成 | 3Dモデルの生成 | Meshy、Tripo |
| マルチモーダル | 複数種類の入出力に対応 | GPT-4o、Gemini、Claude |

### テキスト生成AI

テキスト生成AIは、最も広く利用されている生成AIです。大規模言語モデル（LLM: Large Language Model）をベースとしており、自然言語による指示に対して文章を生成します。

主な活用場面は以下のとおりです。

- 文章作成（メール、レポート、ブログ記事）
- 要約・翻訳
- 質問応答・調査
- コード生成・レビュー
- アイデア出し・ブレインストーミング

### 画像生成AI

画像生成AIは、テキストによる指示（テキストプロンプト）や参考画像から新しい画像を生成します。

```
テキストから画像への変換（Text-to-Image）の流れ：

  「夕焼けの海辺で遊ぶ猫」    ┌──────────────┐    生成された
  ──────────────────────────→ │ 画像生成モデル │ ──→ 画像
  （テキストプロンプト）       └──────────────┘
```

### 音声・動画生成AI

音声生成AIはテキストからリアルな音声を合成したり、音楽を生成したりします。動画生成AIはテキストや画像から短い動画を生成できます。2025年以降急速に品質が向上している分野です。

### マルチモーダルAI

マルチモーダル（Multimodal）AIは、テキスト・画像・音声・動画など複数の種類のデータを入力・出力として扱えるAIです。2025年現在の主要なLLMの多くがマルチモーダル対応となっています。

```
┌─────────────────────────────────────────┐
│       マルチモーダルAIのイメージ          │
├─────────────────────────────────────────┤
│                                          │
│   入力                    出力           │
│  ┌──────┐              ┌──────┐         │
│  │テキスト│              │テキスト│         │
│  │画像  │  → [モデル] → │画像  │         │
│  │音声  │              │音声  │         │
│  │動画  │              │動画  │         │
│  └──────┘              └──────┘         │
│                                          │
│  例：画像を見せて「何が写っていますか？」   │
│      → テキストで回答                     │
└─────────────────────────────────────────┘
```

> **ポイントまとめ**
> - 生成AIにはテキスト・画像・音声・動画・コード・3Dなど多くの種類がある
> - テキスト生成AI（LLM）が最も幅広く活用されている
> - マルチモーダルAIにより複数種類のデータを統合的に扱える時代になった

### サービス選択の考え方

用途に合わせて適切なサービスを選択することが重要です。

```
用途別おすすめサービスの目安：

  文章作成・分析    → Claude、ChatGPT
  Web検索統合      → Perplexity、Gemini
  画像生成         → Midjourney、Stable Diffusion
  コード開発       → GitHub Copilot、Claude Code、Cursor
  業務文書作成     → Copilot（Microsoft 365連携）
  動画生成         → Sora、Runway
```

### よくある間違い

- **「1つのサービスだけ使えばよい」と思い込む**：サービスにはそれぞれ得意・不得意があります。複数のサービスを使い分けることで効率が上がります
- **「無料版で十分」と判断する**：無料版では回数制限や機能制限があり、業務利用には有料プランの検討が必要です
- **「最新モデルが常に最良」と考える**：タスクによっては軽量なモデルのほうが速く、コストも低く、十分な品質を得られます

> **ポイントまとめ**
> - テキスト生成AIはClaude、ChatGPT、Geminiが三大サービス
> - 画像生成ではMidjourneyとStable Diffusionが広く使われている
> - 用途に応じた使い分けが効率的

## 1.4 生成AIの仕組み（概要）

ここでは、生成AIの技術的な仕組みを概念レベルで理解します。数式は使わず、考え方の本質を押さえましょう。

### Transformer（トランスフォーマー）

Transformer（トランスフォーマー）は、2017年にGoogleの研究チームが発表した、現在の大規模言語モデル（LLM）の基盤となるアーキテクチャ（設計構造）です。

Transformerの核となる仕組みは「注意機構」（Attention Mechanism）です。

```
┌──────────────────────────────────────────────┐
│       注意機構（Attention）のイメージ          │
├──────────────────────────────────────────────┤
│                                               │
│  入力：「日本の首都は＿＿＿です」              │
│                                               │
│  「首都」という単語に注目したとき：             │
│                                               │
│   日本  の  首都  は  ___  です                │
│   ████  ░   ███   ░  ████  ░                 │
│   高い  低  高い  低  高い  低   ← 関連度      │
│                                               │
│  → 「日本」「首都」の関連が高いので            │
│    「東京」が生成される                        │
│                                               │
│  ※ 文中の全単語の関係を同時に計算する          │
└──────────────────────────────────────────────┘
```

LLMがテキストを生成する基本的な仕組みは「次の単語（トークン）の予測」です。

```
テキスト生成の仕組み（概念図）：

  「今日の天気は」
       ↓
  [LLM] → 次に来る確率が最も高い単語を予測
       ↓
  「晴れ」（確率40%）「曇り」（確率25%）「雨」（確率20%）...
       ↓
  「晴れ」を選択 → 「今日の天気は晴れ」
       ↓
  次の単語を予測 → 「です」を選択
       ↓
  「今日の天気は晴れです」
```

### 拡散モデル（Diffusion Model）

拡散モデル（Diffusion Model）は、画像生成AIで広く使われている手法です。ノイズ（ランダムな乱れ）から段階的にクリアな画像を復元する仕組みです。

```
┌──────────────────────────────────────────────┐
│        拡散モデルのイメージ                    │
├──────────────────────────────────────────────┤
│                                               │
│  【学習時：画像にノイズを加えていく】           │
│  きれいな画像 → 少しノイズ → もっとノイズ      │
│  → 完全なノイズ                               │
│                                               │
│  █████   ▓▓▓▓▓   ░░░░░   ?????              │
│  猫の絵   ↓       ↓       完全なノイズ        │
│                                               │
│  【生成時：ノイズから画像を復元】               │
│  完全なノイズ → ノイズ除去 → さらに除去        │
│  → きれいな画像                               │
│                                               │
│  ?????   ░░░░░   ▓▓▓▓▓   █████              │
│  ノイズ   ↓       ↓      猫の絵               │
│                                               │
│  ※ テキスト指示が復元の方向をガイドする        │
└──────────────────────────────────────────────┘
```

### トークン（Token）とコンテキストウィンドウ（Context Window）

LLMは文章を「トークン」（Token）という単位に分割して処理します。

```
トークン分割の例：

  英語："Hello, world!" → ["Hello", ",", " world", "!"] = 4トークン
  日本語："東京は日本の首都です" → ["東京", "は", "日本", "の", "首都", "です"]
                                  ≈ 6〜10トークン（モデルにより異なる）
```

コンテキストウィンドウ（Context Window）とは、モデルが一度に扱える入力＋出力のトークン数の上限です。

| モデル | コンテキストウィンドウ（目安） |
|--------|-------------------------------|
| Claude Opus 4 / Sonnet 4 | 約200,000トークン |
| GPT-4o | 約128,000トークン |
| Gemini 2.5 Pro | 約1,000,000トークン |

### よくある間違い

- **「AIが文章を理解している」と思う**：LLMは統計的なパターンに基づいて次のトークンを予測しているだけで、人間のように「意味を理解」しているわけではありません
- **「学習データにないことは回答できない」と思う**：基盤モデルは学習データのパターンを組み合わせることで、直接学習していない内容にも対応できます（ただし正確性は保証されません）
- **「ハルシネーション（Hallucination）は誤動作」と思う**：もっともらしい嘘（ハルシネーション）は仕組み上の特性であり、バグではありません。常に出力の検証が必要です

> **ポイントまとめ**
> - LLMはTransformerアーキテクチャと「次の単語予測」が基本原理
> - 画像生成AIは拡散モデルによる「ノイズからの復元」が主流
> - トークンとコンテキストウィンドウはLLM活用の基本概念

## 1.5 生成AIの現在地と今後のトレンド

### 2025-2026年の主要トレンド

#### 1. AIエージェント（AI Agent）の台頭

AIエージェントとは、単なる質問応答にとどまらず、複数のステップを自律的に実行してタスクを完遂するAIシステムです。

```
┌──────────────────────────────────────────────┐
│     チャットAI vs AIエージェントの違い          │
├──────────────────────────────────────────────┤
│                                               │
│  【従来のチャットAI】                          │
│  ユーザー: 「旅行プランを作って」              │
│  AI: 「おすすめプランはこちらです...」（回答）  │
│  → 1回のやり取りで完結                        │
│                                               │
│  【AIエージェント】                            │
│  ユーザー: 「来月の沖縄旅行を予約して」        │
│  AI: 1. 航空券の空き状況を検索                 │
│      2. ホテルの比較・選定                     │
│      3. レンタカーの手配                       │
│      4. 予約の実行                             │
│      5. 旅程表の作成・送付                     │
│  → 複数ステップを自律的に実行                  │
└──────────────────────────────────────────────┘
```

#### 2. マルチモーダルの深化

テキスト・画像・音声・動画をシームレスに扱えるモデルが標準になりつつあります。リアルタイムの音声会話、画像を見ながらの対話、動画の理解と生成が実用レベルに達しています。

#### 3. ローカルAI・オンデバイスAI

スマートフォンやPCのローカル環境で動作する小型モデルが普及し、プライバシーを保護しながら生成AIを利用できる環境が整いつつあります。

#### 4. RAG（Retrieval-Augmented Generation）の普及

RAG（検索拡張生成）は、外部の知識ベースから情報を検索してLLMの回答に組み込む技術です。企業内の社内文書やナレッジベースを活用したAIアシスタント構築の標準手法となっています。

```
┌──────────────────────────────────────────────┐
│           RAGの基本的な仕組み                  │
├──────────────────────────────────────────────┤
│                                               │
│  ユーザーの質問                                │
│       ↓                                      │
│  ① 質問に関連する文書を検索（Retrieval）       │
│       ↓                                      │
│  ② 検索結果＋質問をLLMに渡す（Augmented）     │
│       ↓                                      │
│  ③ LLMが回答を生成（Generation）              │
│       ↓                                      │
│  根拠のある正確な回答                          │
│                                               │
│  ※ ハルシネーション対策としても有効             │
└──────────────────────────────────────────────┘
```

#### 5. 生成AIのビジネス実装

2025-2026年は「実験段階」から「本格導入段階」への移行期です。

| フェーズ | 時期 | 特徴 |
|---------|------|------|
| 実験期 | 2023年 | 個人利用、試験的な導入 |
| 拡大期 | 2024年 | 部門レベルでの導入 |
| 本格導入期 | 2025-2026年 | 業務プロセスへの組み込み |
| 定着期 | 2027年〜 | AIがインフラとして当たり前に |

### 生成AI活用における注意点

生成AIを活用するうえで、以下の点を常に意識する必要があります。

- **正確性の検証**：生成AIの出力は必ず人間が確認する（ハルシネーションのリスク）
- **著作権・知的財産**：生成物の著作権や、学習データに含まれる著作物への配慮
- **個人情報・機密情報**：機密データを外部サービスに入力しない
- **バイアス**：学習データに含まれる偏り（Bias）が出力に反映される可能性
- **依存リスク**：AIに頼りすぎず、人間の判断力・思考力を維持する

### よくある間違い

- **「生成AIは万能ツール」と思う**：得意・不得意があり、向かないタスクも多くあります
- **「出力をそのまま使ってよい」と思う**：業務利用では必ず人間のレビューを経てから使用しましょう
- **「社内情報をそのまま入力してよい」と思う**：利用規約やセキュリティポリシーを確認し、機密情報の取り扱いに注意が必要です

> **ポイントまとめ**
> - AIエージェントが「対話」から「実行」へとAIの役割を広げている
> - RAGにより企業独自の知識を活用した生成AIが実現している
> - 生成AI活用では正確性の検証・情報セキュリティ・著作権への配慮が不可欠
