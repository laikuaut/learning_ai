# 第8章：ファイル操作

## この章のゴール

- ファイル操作の基本概念（ストリーム）を理解する
- `fopen` / `fclose` によるファイルのオープン・クローズができる
- テキストファイルとバイナリファイルの読み書きができる
- ファイル位置の制御（`fseek`, `ftell`, `rewind`）を使える
- エラー処理を適切に行える
- コマンドライン引数（`argc`, `argv`）を扱える
- CSVファイルの読み書きやログ出力など実用的なプログラムが書ける

## 8.1 ファイル操作の概念 ― ストリーム

C言語では、ファイルへのデータの流れを**ストリーム（stream）**という抽象概念で扱います。プログラムとファイルの間にストリームという「パイプ」をつなぎ、そこにデータを流し込んだり読み出したりするイメージです。

```
  ┌──────────────┐        ストリーム         ┌──────────────┐
  │              │  ─────────────────────▶   │              │
  │  プログラム   │   (データの流れ)           │   ファイル    │
  │              │  ◀─────────────────────   │              │
  └──────────────┘                           └──────────────┘
       fopen() でストリームを開く
       fclose() でストリームを閉じる
```

プログラム起動時には、以下の3つの標準ストリームが自動的に開かれています。

| ストリーム | 名前 | 説明 |
|---|---|---|
| `stdin`  | 標準入力（standard input）   | キーボードからの入力 |
| `stdout` | 標準出力（standard output）  | 画面への出力 |
| `stderr` | 標準エラー出力（standard error） | エラーメッセージの出力 |

`printf` は内部で `stdout` に書き込み、`scanf` は `stdin` から読み取っています。ファイル操作では、これと同じ仕組みを自分で開いたファイルに対して行います。

### ポイントまとめ
- ストリームはプログラムとファイルをつなぐデータの流れ
- `stdin`, `stdout`, `stderr` は自動的に開かれる標準ストリーム
- ファイル操作は「開く → 読み書き → 閉じる」が基本の流れ

---

## 8.2 FILE型ポインタと fopen / fclose

ファイルを操作するには、まず `fopen` 関数でファイルを開き、**FILE型ポインタ（file pointer）**を取得します。操作が終わったら `fclose` で閉じます。

```c
#include <stdio.h>

int main(void) {
    FILE *fp;  /* FILE型ポインタの宣言 */

    /* ファイルを書き込みモードで開く */
    fp = fopen("hello.txt", "w");
    if (fp == NULL) {
        perror("ファイルを開けませんでした");
        return 1;
    }

    /* ファイルに書き込む */
    fprintf(fp, "Hello, File!\n");
    fprintf(fp, "C言語のファイル操作です。\n");

    /* ファイルを閉じる */
    fclose(fp);

    printf("ファイルに書き込みました。\n");
    return 0;
}
```

`fopen` の戻り値が `NULL` の場合、ファイルのオープンに失敗しています。**必ず NULL チェックを行いましょう。**

### オープンモード一覧

| モード | 意味 | ファイルが存在しない場合 | ファイルが存在する場合 |
|---|---|---|---|
| `"r"`  | 読み込み（read）           | エラー（NULL） | 先頭から読み込み |
| `"w"`  | 書き込み（write）          | 新規作成       | 内容を消して上書き |
| `"a"`  | 追記（append）             | 新規作成       | 末尾に追記 |
| `"r+"` | 読み書き                   | エラー（NULL） | 先頭から読み書き |
| `"w+"` | 読み書き（新規作成）        | 新規作成       | 内容を消して読み書き |
| `"a+"` | 読み込み＋追記              | 新規作成       | 読み込みは任意の位置、書き込みは末尾 |
| `"rb"` | バイナリ読み込み            | エラー（NULL） | 先頭から読み込み |
| `"wb"` | バイナリ書き込み            | 新規作成       | 内容を消して上書き |

```
  ファイル操作の基本フロー
  ========================

  fopen()  ──▶  読み書き処理  ──▶  fclose()
    │              │                  │
    │  FILE *fp    │  fprintf 等      │  リソース解放
    │  を取得      │  で操作          │  バッファ書き出し
    ▼              ▼                  ▼
  [開く]        [処理]            [閉じる]
```

### ポイントまとめ
- `fopen` でファイルを開き、`FILE *` を取得する
- 必ず `NULL` チェックでオープン失敗を検出する
- 用途に応じて適切なオープンモードを選ぶ
- 使い終わったら必ず `fclose` で閉じる

## 8.3 テキストファイルの読み書き

### 書き込み：fprintf, fputs

```c
#include <stdio.h>

int main(void) {
    FILE *fp = fopen("output.txt", "w");
    if (fp == NULL) {
        perror("fopen");
        return 1;
    }

    /* fprintf: printf と同じ書式でファイルに書き込む */
    fprintf(fp, "名前: %s\n", "田中太郎");
    fprintf(fp, "年齢: %d\n", 25);
    fprintf(fp, "身長: %.1f cm\n", 170.5);

    /* fputs: 文字列をそのまま書き込む（改行は自分で付ける） */
    fputs("--- ここまで ---\n", fp);

    fclose(fp);
    printf("output.txt に書き込みました。\n");
    return 0;
}
```

### 読み込み：fgets（推奨）

`fgets` は1行ずつ安全に読み込めるため、最もよく使われます。

```c
#include <stdio.h>

int main(void) {
    FILE *fp = fopen("output.txt", "r");
    if (fp == NULL) {
        perror("fopen");
        return 1;
    }

    char buf[256];

    /* fgets: 1行ずつ読み込む（バッファサイズを超えない） */
    while (fgets(buf, sizeof(buf), fp) != NULL) {
        printf("読み込み: %s", buf);  /* buf には改行が含まれる */
    }

    fclose(fp);
    return 0;
}
```

### 読み込み：fscanf

`fscanf` は書式を指定して値を読み取れますが、空白や改行の扱いに注意が必要です。

```c
#include <stdio.h>

int main(void) {
    FILE *fp = fopen("data.txt", "w");
    if (fp == NULL) { perror("fopen"); return 1; }
    fprintf(fp, "100 200 300\n");
    fprintf(fp, "400 500 600\n");
    fclose(fp);

    /* 書き込んだファイルを読み込む */
    fp = fopen("data.txt", "r");
    if (fp == NULL) { perror("fopen"); return 1; }

    int a, b, c;
    while (fscanf(fp, "%d %d %d", &a, &b, &c) == 3) {
        printf("合計: %d\n", a + b + c);
    }

    fclose(fp);
    return 0;
}
```

### よくある間違い

1. **`fgets` の改行文字を忘れる** ― `fgets` は改行文字 `\n` も読み込みます。必要に応じて除去しましょう。
2. **`fclose` を忘れる** ― ファイルを閉じないとバッファに残ったデータが書き込まれない場合があります。
3. **`"w"` モードで既存ファイルを開いて中身を消してしまう** ― 追記なら `"a"` を使いましょう。

### ポイントまとめ
- `fprintf` は `printf` のファイル版、書式指定で書き込める
- `fgets` は安全な1行読み込み、バッファオーバーフローを防げる
- `fscanf` は書式付き読み込み、戻り値で読み取り成功数を確認する

## 8.4 バイナリファイルの読み書き

テキストファイルは人間が読める文字列で保存しますが、**バイナリファイル（binary file）**はデータをメモリ上の表現そのままで保存します。構造体の保存・復元などに便利です。

```
  テキストファイル               バイナリファイル
  ┌─────────────────┐          ┌─────────────────┐
  │ "100"           │          │ 0x64 0x00 0x00   │
  │ → 3バイト       │          │ 0x00             │
  │ (文字 '1','0','0')│        │ → 4バイト (int)   │
  └─────────────────┘          └─────────────────┘
```

### fwrite と fread

```c
#include <stdio.h>
#include <string.h>

typedef struct {
    char name[32];
    int age;
    double height;
} Person;

int main(void) {
    /* 書き込み */
    Person people[3] = {
        {"田中太郎", 25, 170.5},
        {"鈴木花子", 30, 158.0},
        {"佐藤次郎", 22, 175.3}
    };

    FILE *fp = fopen("people.dat", "wb");
    if (fp == NULL) { perror("fopen"); return 1; }

    /* fwrite(データのアドレス, 1要素のサイズ, 要素数, ファイルポインタ) */
    size_t written = fwrite(people, sizeof(Person), 3, fp);
    printf("%zu 件書き込みました。\n", written);
    fclose(fp);

    /* 読み込み */
    Person loaded[3];
    fp = fopen("people.dat", "rb");
    if (fp == NULL) { perror("fopen"); return 1; }

    /* fread(格納先のアドレス, 1要素のサイズ, 要素数, ファイルポインタ) */
    size_t read_count = fread(loaded, sizeof(Person), 3, fp);
    printf("%zu 件読み込みました。\n", read_count);
    fclose(fp);

    for (int i = 0; i < 3; i++) {
        printf("名前: %s, 年齢: %d, 身長: %.1f\n",
               loaded[i].name, loaded[i].age, loaded[i].height);
    }
    return 0;
}
```

### よくある間違い

- **テキストモード `"w"` でバイナリデータを書き込む** ― Windows では改行コードの変換が起きてデータが壊れます。必ず `"wb"`, `"rb"` を使いましょう。
- **異なる環境間でバイナリファイルを共有する** ― 構造体のパディングやエンディアンが異なるため、移植性が必要な場合はテキスト形式（JSONやCSVなど）を使いましょう。

### ポイントまとめ
- `fwrite` / `fread` はメモリの内容をそのままファイルに読み書きする
- バイナリモード（`"wb"`, `"rb"`）を忘れないこと
- 構造体の配列をまとめて保存・復元できるため効率的

## 8.5 ファイル位置の制御

ファイル内の読み書き位置は**ファイル位置指示子（file position indicator）**で管理されています。`fseek`, `ftell`, `rewind` で自由に制御できます。

```
  ファイル内容: [A][B][C][D][E][F][G][H]
  位置:          0  1  2  3  4  5  6  7

  fseek(fp, 0, SEEK_SET)  → 先頭（位置 0）へ移動
  fseek(fp, 3, SEEK_SET)  → 位置 3 へ移動
  fseek(fp, -2, SEEK_END) → 末尾から2つ前（位置 6）へ移動
  ftell(fp)               → 現在の位置を返す
  rewind(fp)              → 先頭に戻す（fseek(fp, 0, SEEK_SET) と同等）
```

### fseek の基準位置

| 定数 | 意味 |
|---|---|
| `SEEK_SET` | ファイル先頭からのオフセット |
| `SEEK_CUR` | 現在位置からのオフセット |
| `SEEK_END` | ファイル末尾からのオフセット |

### ファイルサイズを取得する例

```c
#include <stdio.h>

int main(void) {
    FILE *fp = fopen("people.dat", "rb");
    if (fp == NULL) { perror("fopen"); return 1; }

    /* 末尾に移動 */
    fseek(fp, 0, SEEK_END);

    /* 現在位置（＝ファイルサイズ）を取得 */
    long size = ftell(fp);
    printf("ファイルサイズ: %ld バイト\n", size);

    /* 先頭に戻す */
    rewind(fp);

    fclose(fp);
    return 0;
}
```

### ポイントまとめ
- `fseek` でファイル内の任意の位置に移動できる
- `ftell` で現在の位置（先頭からのバイト数）を取得できる
- `rewind` で先頭に戻せる
- バイナリファイルのランダムアクセスに特に有効

## 8.6 エラー処理

ファイル操作では様々なエラーが起こり得ます。堅牢なプログラムを書くためにはエラー処理が不可欠です。

### perror と errno

`perror` 関数は、直前のエラー原因を人間が読める形で表示してくれます。

```c
#include <stdio.h>
#include <errno.h>

int main(void) {
    FILE *fp = fopen("存在しないファイル.txt", "r");
    if (fp == NULL) {
        /* perror: エラーメッセージを表示 */
        perror("エラー");
        /* 出力例: エラー: No such file or directory */

        printf("errno の値: %d\n", errno);
        return 1;
    }
    fclose(fp);
    return 0;
}
```

### feof と ferror

ファイル読み込みループの終了原因を判別するために使います。

```c
#include <stdio.h>

int main(void) {
    FILE *fp = fopen("output.txt", "r");
    if (fp == NULL) { perror("fopen"); return 1; }

    char buf[256];
    while (fgets(buf, sizeof(buf), fp) != NULL) {
        printf("%s", buf);
    }

    /* ループ終了の原因を判別 */
    if (feof(fp)) {
        printf("\n--- ファイルの末尾に到達しました ---\n");
    } else if (ferror(fp)) {
        printf("\n--- 読み込みエラーが発生しました ---\n");
    }

    fclose(fp);
    return 0;
}
```

### ポイントまとめ
- `fopen` の戻り値は必ず `NULL` チェックする
- `perror` でエラーの原因を表示できる
- `feof` と `ferror` でループ終了の原因を判別する

## 8.7 コマンドライン引数

`main` 関数は引数を受け取ることができます。**コマンドライン引数（command-line arguments）**と呼ばれ、プログラム実行時にファイル名やオプションを指定できます。

```
  実行例:  ./program input.txt output.txt
           ─────── ───────── ──────────
           argv[0]  argv[1]   argv[2]
           argc = 3
```

```c
#include <stdio.h>

int main(int argc, char *argv[]) {
    printf("引数の数: %d\n", argc);

    for (int i = 0; i < argc; i++) {
        printf("argv[%d] = \"%s\"\n", i, argv[i]);
    }

    /* 引数チェック */
    if (argc < 2) {
        fprintf(stderr, "使い方: %s <ファイル名>\n", argv[0]);
        return 1;
    }

    /* 引数で指定されたファイルを読む */
    FILE *fp = fopen(argv[1], "r");
    if (fp == NULL) {
        perror(argv[1]);
        return 1;
    }

    char buf[256];
    while (fgets(buf, sizeof(buf), fp) != NULL) {
        printf("%s", buf);
    }

    fclose(fp);
    return 0;
}
```

コンパイルと実行方法：

```bash
gcc -o fileread fileread.c
./fileread output.txt
```

### ポイントまとめ
- `argc` は引数の数、`argv` は引数の文字列配列
- `argv[0]` はプログラム自身の名前
- 引数の数を必ずチェックしてから使う
- ファイル名を引数で受け取ると汎用的なツールが作れる

## 8.8 実用例：CSVファイルの読み書き

CSVファイルの処理は実務で非常によく使われます。ここでは構造体と組み合わせた実用的な例を紹介します。

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define MAX_RECORDS 100
#define MAX_LINE 256

typedef struct {
    int id;
    char name[64];
    int score;
} Student;

/* CSVファイルに書き込む */
int write_csv(const char *filename, Student students[], int count) {
    FILE *fp = fopen(filename, "w");
    if (fp == NULL) {
        perror(filename);
        return -1;
    }

    /* ヘッダー行 */
    fprintf(fp, "ID,名前,点数\n");

    /* データ行 */
    for (int i = 0; i < count; i++) {
        fprintf(fp, "%d,%s,%d\n",
                students[i].id, students[i].name, students[i].score);
    }

    fclose(fp);
    return 0;
}

/* CSVファイルから読み込む */
int read_csv(const char *filename, Student students[], int max_count) {
    FILE *fp = fopen(filename, "r");
    if (fp == NULL) {
        perror(filename);
        return -1;
    }

    char line[MAX_LINE];
    int count = 0;

    /* ヘッダー行を読み飛ばす */
    if (fgets(line, sizeof(line), fp) == NULL) {
        fclose(fp);
        return 0;
    }

    /* データ行を読み込む */
    while (fgets(line, sizeof(line), fp) != NULL && count < max_count) {
        /* 改行文字を除去 */
        line[strcspn(line, "\n")] = '\0';

        /* カンマ区切りで解析 */
        if (sscanf(line, "%d,%63[^,],%d",
                   &students[count].id,
                   students[count].name,
                   &students[count].score) == 3) {
            count++;
        }
    }

    fclose(fp);
    return count;
}

int main(void) {
    /* テストデータ */
    Student students[] = {
        {1, "田中太郎", 85},
        {2, "鈴木花子", 92},
        {3, "佐藤次郎", 78},
        {4, "高橋美咲", 95},
        {5, "伊藤健一", 63}
    };
    int count = sizeof(students) / sizeof(students[0]);

    /* CSVに書き込む */
    if (write_csv("students.csv", students, count) != 0) {
        return 1;
    }
    printf("students.csv に %d 件書き込みました。\n", count);

    /* CSVから読み込む */
    Student loaded[MAX_RECORDS];
    int loaded_count = read_csv("students.csv", loaded, MAX_RECORDS);
    if (loaded_count < 0) {
        return 1;
    }

    printf("\n--- 読み込み結果 ---\n");
    int total = 0;
    for (int i = 0; i < loaded_count; i++) {
        printf("ID:%d  %-12s  %d点\n",
               loaded[i].id, loaded[i].name, loaded[i].score);
        total += loaded[i].score;
    }
    if (loaded_count > 0) {
        printf("平均点: %.1f\n", (double)total / loaded_count);
    }

    return 0;
}
```

## 8.9 実用例：ログファイル出力

プログラムの動作記録をファイルに出力する「ログ（log）」は実務で欠かせません。追記モード `"a"` を使います。

```c
#include <stdio.h>
#include <time.h>

/* ログメッセージを書き込む関数 */
int write_log(const char *filename, const char *level, const char *message) {
    FILE *fp = fopen(filename, "a");  /* 追記モード */
    if (fp == NULL) {
        perror(filename);
        return -1;
    }

    /* 現在時刻を取得 */
    time_t now = time(NULL);
    struct tm *t = localtime(&now);
    char timestamp[32];
    strftime(timestamp, sizeof(timestamp), "%Y-%m-%d %H:%M:%S", t);

    /* ログ形式: [日時] [レベル] メッセージ */
    fprintf(fp, "[%s] [%s] %s\n", timestamp, level, message);

    fclose(fp);
    return 0;
}

int main(void) {
    const char *logfile = "app.log";

    write_log(logfile, "INFO",  "アプリケーションを起動しました");
    write_log(logfile, "INFO",  "データの読み込みを開始します");
    write_log(logfile, "WARN",  "設定ファイルが見つかりません。デフォルト値を使用します");
    write_log(logfile, "INFO",  "処理が完了しました");
    write_log(logfile, "ERROR", "ファイルの保存に失敗しました");

    printf("ログを %s に出力しました。\n", logfile);

    /* ログファイルの内容を表示 */
    FILE *fp = fopen(logfile, "r");
    if (fp != NULL) {
        char buf[256];
        printf("\n--- ログ内容 ---\n");
        while (fgets(buf, sizeof(buf), fp) != NULL) {
            printf("%s", buf);
        }
        fclose(fp);
    }

    return 0;
}
```

実行結果の例：

```
ログを app.log に出力しました。

[2026-03-28 10:30:15] [INFO]  アプリケーションを起動しました
[2026-03-28 10:30:15] [INFO]  データの読み込みを開始します
[2026-03-28 10:30:15] [WARN]  設定ファイルが見つかりません。デフォルト値を使用します
[2026-03-28 10:30:15] [INFO]  処理が完了しました
[2026-03-28 10:30:15] [ERROR] ファイルの保存に失敗しました
```

## 8.10 よくある間違いまとめ

| 間違い | 正しい対処 |
|---|---|
| `fopen` の戻り値を確認しない | 必ず `NULL` チェックする |
| `fclose` を忘れる | 開いたファイルは必ず閉じる |
| `"w"` で開いて既存データを消してしまう | 追記なら `"a"` を使う |
| バイナリファイルをテキストモードで開く | `"rb"`, `"wb"` を使う |
| `fgets` の改行文字を考慮しない | `strcspn` 等で改行を除去する |
| バッファサイズを超える読み込み | `fgets` の第2引数でサイズを制限する |
| `fscanf` の戻り値を無視する | 読み取り成功数を確認する |
| コマンドライン引数の数を確認しない | `argc` をチェックしてからアクセスする |

## 8.11 章末まとめ

```
  ファイル操作の全体像
  ====================

  ┌─ テキスト ─┐    ┌─ バイナリ ──┐
  │ fprintf    │    │ fwrite     │
  │ fputs      │    │ fread      │
  │ fgets      │    │            │
  │ fscanf     │    │            │
  └─────┬──────┘    └─────┬──────┘
        │                 │
        ▼                 ▼
  ┌──────────────────────────────┐
  │  fopen / fclose              │
  │  FILE *fp                    │
  ├──────────────────────────────┤
  │  位置制御: fseek/ftell/rewind│
  │  エラー : perror/feof/ferror │
  └──────────────────────────────┘
```

この章で学んだ主要な関数をまとめます。

| 関数 | 用途 |
|---|---|
| `fopen` / `fclose` | ファイルのオープン・クローズ |
| `fprintf` / `fscanf` | 書式付き読み書き |
| `fgets` / `fputs` | 行・文字列単位の読み書き |
| `fread` / `fwrite` | バイナリ読み書き |
| `fseek` / `ftell` / `rewind` | ファイル位置の制御 |
| `perror` | エラーメッセージの表示 |
| `feof` / `ferror` | 終了原因の判別 |

ファイル操作はC言語プログラミングの中でも特に実用的な技術です。「開く → 処理 → 閉じる」の基本パターンを確実に身につけ、エラー処理を怠らないようにしましょう。次の章以降では、ここで学んだ知識を活かしてより高度なプログラムを作成していきます。
