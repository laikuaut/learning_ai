# 第4章：検索と置換の設定

## この章のゴール

- 検索に関する基本設定（`hlsearch`、`incsearch`、`ignorecase`、`smartcase`）を理解する
- 検索ハイライトの制御方法とマッピングを設定できる
- 正規表現のモード（`magic`、`very magic`）を使い分けられる
- 外部 grep ツール（ripgrep、ag）と Vim を連携できる
- 置換コマンドの便利な設定を活用できる
- quickfix リストを使った検索結果のナビゲーションができる

---

## 1. 検索の基本設定

### 4つの必須設定

```vim
set hlsearch        " 検索結果をハイライト表示
set incsearch       " インクリメンタルサーチ（入力中にリアルタイム検索）
set ignorecase      " 検索で大文字・小文字を区別しない
set smartcase       " 検索パターンに大文字が含まれる場合は区別する
```

### 各設定の効果

```
  hlsearch（ハイライトサーチ）:
  ──────────────────────────────
  /hello と検索すると...

  set hlsearch の場合:
  │ This is a [hello] world program.     │
  │ Say [hello] to everyone.             │  ← マッチ箇所がハイライト
  │ [Hello] there!                       │

  set nohlsearch の場合:
  │ This is a hello world program.       │
  │ Say hello to everyone.               │  ← ハイライトなし
  │ Hello there!                         │


  incsearch（インクリメンタルサーチ）:
  ──────────────────────────────
  /hel と入力中に...

  set incsearch の場合:
  → 入力するたびにリアルタイムで最初のマッチにジャンプ
  → /h → /he → /hel と1文字ずつマッチが更新される

  set noincsearch の場合:
  → Enter を押すまで検索が実行されない


  ignorecase + smartcase の組み合わせ:
  ──────────────────────────────
  set ignorecase + set smartcase の場合:

  /hello   → hello, Hello, HELLO すべてにマッチ（全小文字なので）
  /Hello   → Hello のみにマッチ（大文字を含むので区別する）
  /HELLO   → HELLO のみにマッチ（大文字を含むので区別する）

  ※ smartcase は ignorecase が有効な場合にのみ機能します
```

### wrapscan の設定

```vim
set wrapscan        " ファイル末尾まで検索したら先頭に戻る（デフォルト）
set nowrapscan      " ファイル末尾で検索を停止する
```

### ポイントまとめ

- `hlsearch` + `incsearch` は検索の必須設定です
- `ignorecase` + `smartcase` のペアで柔軟な検索ができます
- `smartcase` は `ignorecase` が ON の時だけ機能します

## 2. 検索ハイライトの制御

### nohlsearch コマンド

検索後にハイライトが残り続けるのは煩わしいことがあります。`:nohlsearch`（略: `:noh`）でハイライトを一時的に消すことができます。

```vim
" コマンドラインから実行
:nohlsearch         " 検索ハイライトを一時消去
:noh                " 省略形
```

### 定番のマッピング

毎回 `:noh` と入力するのは面倒なので、キーマッピングを設定するのが定番です。

```vim
" ESC 2回で検索ハイライトを消去（最も人気のマッピング）
nnoremap <Esc><Esc> :nohlsearch<CR>

" 別の選択肢: Leader + h で消去
nnoremap <Leader>h :nohlsearch<CR>

" 別の選択肢: Ctrl + l でハイライト消去 + 画面再描画
nnoremap <C-l> :nohlsearch<CR><C-l>
```

### 検索時に画面中央に表示

検索でジャンプした時に、マッチ位置を画面中央に表示するマッピングも便利です。

```vim
" n/N で次/前の検索結果に移動した時に画面中央に表示
nnoremap n nzzzv
nnoremap N Nzzzv

" * / # で検索した時も画面中央に表示
nnoremap * *zzzv
nnoremap # #zzzv
```

```
  zz の効果:

  zz なしの場合:              zz ありの場合:
  ┌─────────────────────┐    ┌─────────────────────┐
  │ line 1              │    │ line 8              │
  │ line 2              │    │ line 9              │
  │ line 3              │    │ [match] ← 画面中央  │
  │ line 4              │    │ line 11             │
  │ [match] ← 画面下部  │    │ line 12             │
  └─────────────────────┘    └─────────────────────┘
```

### ポイントまとめ

- `:nohlsearch` で検索ハイライトを一時的に消せます
- `Esc Esc` にマッピングするのが定番です
- `nzzzv` で検索結果を画面中央に表示するテクニックも便利です

## 3. 正規表現の設定

### Vim の正規表現モード

Vim の正規表現には4つのモードがあります。デフォルトは `magic` モードです。

```
  ┌────────────┬───────┬────────────────────────────────┐
  │ モード     │ 指定  │ 特徴                           │
  ├────────────┼───────┼────────────────────────────────┤
  │ nomagic    │ \M    │ ほぼ全てのメタ文字にエスケープ │
  │ magic      │ \m    │ デフォルト。一部要エスケープ   │
  │ very magic │ \v    │ ほぼ全てがメタ文字（推奨）     │
  │ very       │ \V    │ リテラル検索（. も文字通り）   │
  │ nomagic    │       │                                │
  └────────────┴───────┴────────────────────────────────┘
```

### magic モード（デフォルト）と very magic モード

```vim
" magic モード（デフォルト）: () {} + をエスケープする必要がある
/\(hello\|world\)\+

" very magic モード: エスケープ不要でほぼ普通の正規表現
/\v(hello|world)+

" 同じ意味ですが、very magic の方が読みやすい
```

### very magic を常に使う設定

```vim
" 検索時に自動で \v を付けるマッピング
nnoremap / /\v
vnoremap / /\v

" 置換時にも very magic を使う
cnoremap %s/ %s/\v
```

> **注意**: この設定は便利ですが、Vim の標準的な検索パターンと異なる動作になるため、他人の環境で混乱する可能性があります。チームで使う場合は注意しましょう。

### ポイントまとめ

- Vim のデフォルトは `magic` モードで、一部のメタ文字にエスケープが必要です
- `\v`（very magic）を使うと一般的な正規表現に近い書き方ができます
- `/\v` をマッピングすると常に very magic で検索できます

## 4. grep の統合

### grepprg の設定

Vim の `:grep` コマンドはデフォルトでシステムの `grep` を使いますが、より高速なツールに変更できます。

```vim
" ripgrep（rg）を使用する場合（推奨）
if executable('rg')
  set grepprg=rg\ --vimgrep\ --smart-case
  set grepformat=%f:%l:%c:%m
endif

" The Silver Searcher（ag）を使用する場合
if executable('ag')
  set grepprg=ag\ --vimgrep
  set grepformat=%f:%l:%c:%m
endif
```

### :grep コマンドの使い方

```vim
" 基本的な使い方
:grep パターン                " カレントディレクトリを再帰検索
:grep パターン src/           " src/ ディレクトリ内を検索
:grep パターン %              " 現在のファイル内を検索

" 実行例
:grep "function" src/         " src/ 内の "function" を検索
:grep "TODO" **/*.py          " 全 .py ファイルの "TODO" を検索
```

### grepprg の各ツール比較

```
  ┌──────────┬──────────┬────────────────────────────┐
  │ ツール   │ 速度     │ 特徴                       │
  ├──────────┼──────────┼────────────────────────────┤
  │ grep     │ 遅い     │ どの環境にもある           │
  │ ag       │ 速い     │ .gitignore を自動認識      │
  │ rg       │ とても速い│ .gitignore 認識、最も人気  │
  └──────────┴──────────┴────────────────────────────┘
```

### ポイントまとめ

- `grepprg` で `:grep` が使う外部ツールを変更できます
- ripgrep（`rg`）が最速で、最も推奨されるツールです
- `executable()` で存在確認してから設定するのが安全です

## 5. 置換の便利設定

### 置換の基本

```vim
" 置換の基本構文
:%s/検索パターン/置換文字列/フラグ

" よく使うフラグ
" g - 行内の全てのマッチを置換（これがないと最初の1つだけ）
" c - 置換前に確認する
" i - 大文字・小文字を区別しない
" e - エラーを無視する
```

### gdefault の設定

```vim
" g フラグをデフォルトにする
set gdefault
```

```
  gdefault の効果:

  set nogdefault（デフォルト）の場合:
  :%s/foo/bar/    → 各行の最初の foo だけ置換
  :%s/foo/bar/g   → 各行の全ての foo を置換

  set gdefault の場合:
  :%s/foo/bar/    → 各行の全ての foo を置換（g がデフォルト）
  :%s/foo/bar/g   → 各行の最初の foo だけ置換（g で反転）
```

> **注意**: `gdefault` は便利ですが、Vim の標準動作と異なるため混乱する場合もあります。使うかどうかは好みで判断してください。

### 確認付き置換（c フラグ）

```vim
" 1つずつ確認しながら置換
:%s/foo/bar/gc

" 確認時の操作:
" y - 置換する
" n - スキップ
" a - 残り全て置換
" q - 中止
" l - 置換して中止（last）
```

### inccommand（Neovim 限定）

Neovim では `inccommand` を設定すると、置換のプレビューがリアルタイムで表示されます。

```vim
" Neovim のみ: 置換のリアルタイムプレビュー
if has('nvim')
  set inccommand=split     " 分割ウィンドウでプレビュー
  " set inccommand=nosplit  " 現在のウィンドウ内でプレビュー
endif
```

### ポイントまとめ

- `c` フラグで1つずつ確認しながら安全に置換できます
- `gdefault` は便利ですが標準動作と異なる点に注意です
- Neovim の `inccommand` は置換作業を大幅に効率化します

## 6. quickfix と locationlist

### quickfix リストとは

quickfix リストは、`:grep` や `:make` の結果を一覧表示し、順番にジャンプできる機能です。

```vim
" quickfix の基本操作
:copen          " quickfix ウィンドウを開く
:cclose         " quickfix ウィンドウを閉じる
:cnext          " 次の結果にジャンプ
:cprev          " 前の結果にジャンプ
:cfirst         " 最初の結果にジャンプ
:clast          " 最後の結果にジャンプ
```

```
  quickfix ウィンドウの表示例:

  :grep "TODO" src/ を実行した後:

  ┌─ src/main.py ─────────────────────────────┐
  │  15  def process():                        │
  │  16      # TODO: エラー処理を追加          │  ← カーソル位置
  │  17      pass                              │
  ├─ quickfix ─────────────────────────────────┤
  │  src/main.py|16| # TODO: エラー処理を追加  │
  │  src/utils.py|8| # TODO: リファクタリング  │
  │  src/test.py|42| # TODO: テスト追加        │
  └────────────────────────────────────────────┘
```

### quickfix のキーマッピング

```vim
" quickfix ナビゲーションのマッピング
nnoremap <Leader>co :copen<CR>      " quickfix を開く
nnoremap <Leader>cc :cclose<CR>     " quickfix を閉じる
nnoremap ]q :cnext<CR>              " 次の結果
nnoremap [q :cprev<CR>              " 前の結果
```

### locationlist

locationlist はウィンドウごとの quickfix です。`:lgrep`、`:lmake` などで使います。

```vim
" locationlist の基本操作
:lopen          " locationlist ウィンドウを開く
:lclose         " locationlist ウィンドウを閉じる
:lnext          " 次の結果にジャンプ
:lprev          " 前の結果にジャンプ

" マッピング
nnoremap ]l :lnext<CR>             " 次の locationlist 結果
nnoremap [l :lprev<CR>             " 前の locationlist 結果
```

### grep 実行後に自動で quickfix を開く

```vim
" :grep の後に自動で quickfix ウィンドウを開く
augroup QuickfixAutoOpen
  autocmd!
  autocmd QuickFixCmdPost *grep* cwindow
augroup END
```

### ポイントまとめ

- quickfix リストで `:grep` の結果を一覧表示・ナビゲーションできます
- `]q` `[q` のマッピングで素早く結果を移動できます
- `QuickFixCmdPost` イベントで grep 後に自動で quickfix を開けます
- locationlist はウィンドウローカルな quickfix です
