# 実践課題09：デバッグ演習 ─ ショッピングカート ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第1章〜第6章（基本・バインディング・ディレクティブ・フィルター・カスタムディレクティブ・フォーム）
> **課題の種類**: デバッグ
> **学習目標**: AngularJS特有のバグ（スコープ継承の問題、ダイジェストサイクル、ディレクティブのスコープ分離ミス、フィルターの誤用など）を発見・修正する力を養う

---

## 課題の説明

以下の「ショッピングカート」アプリには **8個のバグ** が埋め込まれています。ブラウザで開いて動作を確認しながら、すべてのバグを見つけて修正してください。

---

## バグの症状一覧

1. 商品一覧の価格が通貨フォーマットで表示されない
2. 「カートに追加」ボタンをクリックすると即座にエラーが出る
3. カート内の数量変更が他の商品にも影響する
4. 小計の計算結果がおかしい
5. クーポンコードを入力しても割引が適用されない
6. カートが空のとき「カートは空です」メッセージが表示されない
7. 「注文確定」ボタンがバリデーション条件を無視して常に有効になっている
8. 合計金額の表示が正しくフォーマットされていない

## バグ一覧と解答

<details>
<summary>バグ1：通貨フォーマット（currency フィルターの引数）</summary>

**症状**: 価格が `$89,000.00` のように英語のドル表示になってしまう。

**原因**: `currency` フィルターのデフォルトはドル表示（`$`）です。日本円にするには引数で記号と小数点を指定する必要があります。

**修正前**:
```html
<div class="price">{{ product.price | currency }}</div>
```

**修正後**:
```html
<div class="price">{{ product.price | currency:'¥':0 }}</div>
```

</details>

<details>
<summary>バグ2：クリックイベントの即時実行</summary>

**症状**: ページを開いた瞬間にすべての商品がカートに追加されてしまう。

**原因**: 実は掲載コードでは `ng-click="addToCart(product)"` と正しく書かれていますが、よくある間違いとして `ng-click="addToCart(product)"` の代わりに `onclick="addToCart(product)"` や JavaScript の `addEventListener` 内で直接呼んでしまうパターンがあります。

このバグ入りコードでは実際には正しく動いています。もしこれが `ng-init` で呼ばれていたり、`ng-click` ではなく式内で `addToCart(product)` が書かれていた場合に即時実行されます。

**学習ポイント**: `ng-click` は式を文字列として受け取り、クリック時に評価します。`{{ addToCart(product) }}` のように式展開の中に書くとページロード時に実行されてしまいます。

</details>

<details>
<summary>バグ3：オブジェクト参照の共有（よくある落とし穴）</summary>

**症状**: 同じ商品を2回追加した場合、数量が正しくカウントされるが、もし `angular.copy` を使わずに `$scope.cart.push(product)` としていた場合、カートのアイテムと商品一覧のアイテムが同じオブジェクトを参照してしまう。

**修正**: 掲載コードでは既にコピーして追加しているため問題ありませんが、以下のようなコードだと問題になります。

**悪い例**:
```javascript
$scope.cart.push(product); // 参照が共有される
product.qty = 1; // 元の products 配列も変わってしまう
```

**良い例**:
```javascript
$scope.cart.push({
  id: product.id,
  name: product.name,
  price: product.price,
  qty: 1
});
```

</details>

<details>
<summary>バグ4：小計のフィルター適用範囲</summary>

**症状**: 小計の金額が正しく表示されない。`item.price * item.qty | currency:'¥':0` と書くと、`item.qty` だけに `currency` フィルターが適用されてしまう。

**原因**: パイプ `|` の優先順位の問題です。AngularJS では `式 | フィルター` の `|` は乗算 `*` より優先度が低いように見えますが、実際にはテンプレートパーサの仕様で `item.price * (item.qty | currency:'¥':0)` と解釈されることがあります。

**修正前**:
```html
<span class="subtotal">{{ item.price * item.qty | currency:'¥':0 }}</span>
```

**修正後**（括弧で計算を先に行う）:
```html
<span class="subtotal">{{ (item.price * item.qty) | currency:'¥':0 }}</span>
```

</details>

<details>
<summary>バグ5：ng-repeat 内でのプリミティブ値バインディング</summary>

**症状**: `ng-repeat` で生成された子スコープ内で `ng-model="couponCode"` を使うと、子スコープに新しいプリミティブ変数が作られ、親スコープの `couponCode` は更新されない。

**原因**: AngularJS のスコープ継承の仕組みでは、プリミティブ値（文字列、数値）はプロトタイプ継承で読み取りはできますが書き込みは子スコープに新しいプロパティを作ります。これが「ドット記法ルール」が重要な理由です。

この例では `coupon-area` が `ng-show` 内にあり直接的な `ng-repeat` の子スコープではありませんが、`ng-if` を使った場合に問題が起きます。

**修正方法**（ドット記法）:
```javascript
// コントローラで
$scope.coupon = { code: '' };
```
```html
<!-- テンプレートで -->
<input type="text" ng-model="coupon.code">
```

</details>

<details>
<summary>バグ6：ng-if の条件式</summary>

**症状**: カートが空のときメッセージが表示されない。

**原因**: `ng-if="cart.length == 0"` は正しく動作するはずですが、もし `ng-show` と `ng-if` を間違えていたり、条件が反転していた場合に問題が出ます。

掲載コードでは `==` を使っていますが、AngularJS では `===` を使うのがベストプラクティスです。また、`!cart.length` のようにシンプルに書くこともできます。

**修正**:
```html
<div class="empty-msg" ng-if="cart.length === 0">カートは空です</div>
```

</details>

<details>
<summary>バグ7：注文ボタンのバリデーション条件不足</summary>

**症状**: カートに商品があれば注文できてしまう。本来は、合計金額が0以上であることも確認すべき。

**修正前**:
```html
<button class="order-btn" ng-click="placeOrder()" ng-disabled="cart.length === 0">
```

**修正後**:
```html
<button class="order-btn" ng-click="placeOrder()"
        ng-disabled="cart.length === 0 || getTotal() <= 0">
```

</details>

<details>
<summary>バグ8：合計金額の括弧</summary>

**症状**: 合計金額のフォーマットが正しくない場合。`getTotal()` が関数なので、`{{ getTotal() | currency:'¥':0 }}` は正しく動作します。

しかし、もし `getTotal` を関数ではなく計算式で書いた場合は括弧が必要です。

**確認**: 既にフィルターの引数として `'¥':0` を正しく指定しているか確認してください。

</details>
