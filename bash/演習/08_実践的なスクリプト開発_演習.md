## 基本問題

### 問題1: set オプション
以下のスクリプトの各行の動作を説明してください。

```bash
#!/usr/bin/env bash
set -euo pipefail
```

1. `set -e` は何をしますか？
2. `set -u` は何をしますか？
3. `set -o pipefail` は何をしますか？
4. `set -e` の状態で、失敗してもスクリプトを続行させたい場合どうしますか？

<details>
<summary>解答</summary>

```bash
# 1. set -e (errexit)
# コマンドが0以外の終了ステータスで終了した場合、スクリプトを即座に中断する

# 2. set -u (nounset)
# 未定義の変数を参照した場合、エラーとしてスクリプトを中断する

# 3. set -o pipefail
# パイプラインの途中のコマンドがエラーになった場合、
# パイプライン全体の終了ステータスをそのエラーにする
# （通常は最後のコマンドの終了ステータスのみが使われる）

# 4. 失敗を許容する方法:
# 方法A: || true を付ける
grep "pattern" file.txt || true

# 方法B: if文で囲む
if ! grep "pattern" file.txt; then
    echo "見つかりませんでした"
fi
```

</details>

### 問題2: trap の基本
一時ファイルを作成し、スクリプトが正常終了しても異常終了しても確実に削除されるスクリプトを書いてください。一時ファイルに現在の日時を書き込み、内容を表示してください。

<details>
<summary>解答</summary>

```bash
#!/usr/bin/env bash
set -euo pipefail

# 一時ファイルを作成
tmpfile=$(mktemp)

# trap で EXIT 時にクリーンアップ
trap 'echo "クリーンアップ: $tmpfile を削除"; rm -f "$tmpfile"' EXIT

# 一時ファイルに書き込み
date '+%Y-%m-%d %H:%M:%S' > "$tmpfile"
echo "一時ファイルの内容:"
cat "$tmpfile"
echo "スクリプト正常終了"
# EXIT trap が自動実行される
```

</details>

### 問題3: getopts
以下のオプションを受け付けるスクリプトを `getopts` で書いてください。

- `-n NAME` : 名前を指定
- `-a AGE` : 年齢を指定
- `-v` : 詳細モード
- `-h` : ヘルプを表示

実行例：
```
$ ./script.sh -n Tanaka -a 30 -v
名前: Tanaka
年齢: 30
詳細モード: ON
```

<details>
<summary>解答</summary>

```bash
#!/usr/bin/env bash
set -euo pipefail

usage() {
    echo "使い方: $(basename "$0") -n NAME -a AGE [-v] [-h]"
    echo "  -n NAME   名前を指定"
    echo "  -a AGE    年齢を指定"
    echo "  -v        詳細モード"
    echo "  -h        ヘルプを表示"
}

name=""
age=""
verbose=false

while getopts "n:a:vh" opt; do
    case "$opt" in
        n) name="$OPTARG" ;;
        a) age="$OPTARG" ;;
        v) verbose=true ;;
        h) usage; exit 0 ;;
        ?) usage >&2; exit 1 ;;
    esac
done

# 必須引数のチェック
if [[ -z "$name" || -z "$age" ]]; then
    echo "エラー: -n と -a は必須です" >&2
    usage >&2
    exit 1
fi

echo "名前: $name"
echo "年齢: $age"
echo "詳細モード: $([[ "$verbose" = true ]] && echo "ON" || echo "OFF")"
```

</details>

### 問題4: エラーハンドリング
以下の要件を満たすスクリプトを書いてください。

- `set -euo pipefail` を使用
- ERR trap でエラー発生行を表示
- EXIT trap でクリーンアップ
- 引数で指定されたファイルの行数を表示
- ファイルが存在しない場合はエラーメッセージを表示

<details>
<summary>解答</summary>

```bash
#!/usr/bin/env bash
set -euo pipefail

# ERR trap でエラー行を表示
on_error() {
    echo "エラー発生: 行 $1, コマンド: $2" >&2
}
trap 'on_error $LINENO "$BASH_COMMAND"' ERR

# EXIT trap でクリーンアップ
cleanup() {
    echo "スクリプト終了"
}
trap cleanup EXIT

# メイン処理
main() {
    local file="${1:?ファイルを指定してください}"

    if [[ ! -f "$file" ]]; then
        echo "エラー: ファイルが見つかりません: $file" >&2
        exit 1
    fi

    local count
    count=$(wc -l < "$file")
    echo "$file: $count 行"
}

main "$@"
```

</details>

### 問題5: main関数パターン
以下の要件を満たす、構造化されたスクリプトテンプレートを書いてください。

- シバン行と `set -euo pipefail`
- SCRIPT_DIR と SCRIPT_NAME を定数で定義
- usage関数
- log_info と log_error 関数
- cleanup関数（EXIT trap）
- `-h`/`--help` と `-v`/`--verbose` をサポートするmain関数
- main関数の最後に `"$@"` で呼び出し

<details>
<summary>解答</summary>

```bash
#!/usr/bin/env bash
set -euo pipefail

# --- 定数 ---
readonly SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
readonly SCRIPT_NAME="$(basename "$0")"

# --- ログ関数 ---
log_info() {
    echo "[INFO] $(date '+%H:%M:%S') $*"
}

log_error() {
    echo "[ERROR] $(date '+%H:%M:%S') $*" >&2
}

# --- 使い方 ---
usage() {
    cat <<EOF
使い方: $SCRIPT_NAME [オプション]

オプション:
  -h, --help     ヘルプを表示
  -v, --verbose  詳細モード
EOF
}

# --- クリーンアップ ---
cleanup() {
    log_info "クリーンアップ完了"
}

# --- メイン処理 ---
main() {
    trap cleanup EXIT

    local verbose=false

    while [[ $# -gt 0 ]]; do
        case "$1" in
            -h|--help) usage; exit 0 ;;
            -v|--verbose) verbose=true; shift ;;
            *) break ;;
        esac
    done

    log_info "スクリプト開始 ($SCRIPT_NAME)"
    log_info "ディレクトリ: $SCRIPT_DIR"

    if [[ "$verbose" = true ]]; then
        log_info "詳細モードが有効です"
    fi

    # ここにメイン処理を書く
    log_info "処理が完了しました"
}

main "$@"
```

### 問題6: デバッグ機能付きスクリプト
以下の要件を満たすスクリプトを書いてください。

- 環境変数 `DEBUG=true` でデバッグモードが有効になる
- デバッグモードでは `set -x` が有効になり、PS4がカスタマイズされる
- `debug_log` 関数はデバッグモード時のみメッセージを表示
- 引数で渡された数値の合計を計算して表示

実行例：
```
$ ./script.sh 1 2 3 4 5
合計: 15

$ DEBUG=true ./script.sh 1 2 3
+[script.sh:30] main 1 2 3
...
合計: 6
```

<details>
<summary>解答</summary>

```bash
#!/usr/bin/env bash
set -euo pipefail

# デバッグモードの設定
DEBUG="${DEBUG:-false}"
if [[ "$DEBUG" = "true" ]]; then
    export PS4='+[${BASH_SOURCE##*/}:${LINENO}] '
    set -x
fi

debug_log() {
    if [[ "$DEBUG" = "true" ]]; then
        echo "[DEBUG] $*" >&2
    fi
}

main() {
    if [[ $# -eq 0 ]]; then
        echo "使い方: $(basename "$0") 数値1 数値2 ..." >&2
        exit 1
    fi

    debug_log "引数: $*"

    local sum=0
    for num in "$@"; do
        # 数値チェック
        if [[ ! "$num" =~ ^-?[0-9]+$ ]]; then
            echo "エラー: '$num' は数値ではありません" >&2
            exit 1
        fi
        debug_log "加算: $sum + $num"
        ((sum += num))
    done

    echo "合計: $sum"
}

main "$@"
```

### 問題7: 総合問題 - ファイルバックアップスクリプト
以下の要件を満たすバックアップスクリプトを書いてください。

**要件:**
- `set -euo pipefail` とtrapによるエラーハンドリング
- `getopts` でオプション処理: `-s SOURCE`（バックアップ元）、`-d DEST`（バックアップ先）、`-v`（詳細モード）、`-h`（ヘルプ）
- バックアップ元が存在しない場合はエラー
- バックアップ先ディレクトリがなければ作成
- バックアップファイル名にタイムスタンプを含める
- 詳細モードではファイル数とサイズを表示

<details>
<summary>解答</summary>

```bash
#!/usr/bin/env bash
set -euo pipefail

readonly SCRIPT_NAME="$(basename "$0")"

usage() {
    cat <<EOF
使い方: $SCRIPT_NAME -s SOURCE -d DEST [-v] [-h]

オプション:
  -s SOURCE   バックアップ元のディレクトリ
  -d DEST     バックアップ先のディレクトリ
  -v          詳細モード
  -h          ヘルプを表示
EOF
}

log_info() { echo "[INFO] $(date '+%H:%M:%S') $*"; }
log_error() { echo "[ERROR] $(date '+%H:%M:%S') $*" >&2; }

on_error() { log_error "エラー発生: 行 $1"; }
trap 'on_error $LINENO' ERR

main() {
    local source="" dest="" verbose=false

    while getopts "s:d:vh" opt; do
        case "$opt" in
            s) source="$OPTARG" ;;
            d) dest="$OPTARG" ;;
            v) verbose=true ;;
            h) usage; exit 0 ;;
            ?) usage >&2; exit 1 ;;
        esac
    done

    # 必須引数チェック
    if [[ -z "$source" || -z "$dest" ]]; then
        log_error "-s と -d は必須です"
        usage >&2
        exit 1
    fi

    # バックアップ元の存在チェック
    if [[ ! -d "$source" ]]; then
        log_error "バックアップ元が見つかりません: $source"
        exit 1
    fi

    # バックアップ先の作成
    mkdir -p "$dest"

    # タイムスタンプ付きファイル名
    local timestamp
    timestamp=$(date '+%Y%m%d_%H%M%S')
    local backup_name="backup_${timestamp}.tar.gz"
    local backup_path="$dest/$backup_name"

    log_info "バックアップ開始: $source → $backup_path"

    # バックアップ実行
    tar -czf "$backup_path" -C "$(dirname "$source")" "$(basename "$source")"

    log_info "バックアップ完了: $backup_path"

    # 詳細モード
    if [[ "$verbose" = true ]]; then
        local file_count size
        file_count=$(find "$source" -type f | wc -l)
        size=$(du -sh "$backup_path" | cut -f1)
        log_info "ファイル数: $file_count"
        log_info "バックアップサイズ: $size"
    fi
}

main "$@"
```

</details>
