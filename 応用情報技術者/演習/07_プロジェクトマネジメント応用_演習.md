## 基本問題

### 問1（四択・EVM）
EVMにおいて、EV=800万円、PV=1,000万円、AC=900万円のとき、スケジュール差異（SV）とコスト差異（CV）の組合せとして正しいものはどれか。

ア SV=-200万円、CV=-100万円
イ SV=-200万円、CV=+100万円
ウ SV=+200万円、CV=-100万円
エ SV=+200万円、CV=+100万円

<details><summary>ヒント</summary>

SV = EV - PV, CV = EV - AC

</details>

<details><summary>解答・解説</summary>

**正解: ア SV=-200万円、CV=-100万円**

```
SV = EV - PV = 800 - 1,000 = -200万円（スケジュール遅延）
CV = EV - AC = 800 - 900 = -100万円（コスト超過）
```

SVが負: 計画より遅れている
CVが負: 予算を超過している

</details>

### 問2（四択・EVM）
プロジェクト全体予算（BAC）が2,000万円、CPI（コスト効率指数）が0.8のとき、完了時総コスト見積り（EAC）はどれか。

ア 1,600万円　　イ 2,000万円　　ウ 2,400万円　　エ 2,500万円

<details><summary>解答・解説</summary>

**正解: エ 2,500万円**

```
EAC = BAC / CPI = 2,000 / 0.8 = 2,500万円
```

CPI=0.8は「1円の成果を出すのに1.25円かかっている」状態です。現在のペースが続くと、2,000万円の予算に対して2,500万円かかる見込みです。

</details>

### 問3（四択・PERT）
PERT法における3点見積りで、楽観値=4日、最可能値=6日、悲観値=14日のとき、期待値はどれか。

ア 6日　　イ 7日　　ウ 8日　　エ 9日

<details><summary>ヒント</summary>

期待値 = (O + 4M + P) / 6

</details>

<details><summary>解答・解説</summary>

**正解: イ 7日**

```
期待値 = (O + 4M + P) / 6
       = (4 + 4×6 + 14) / 6
       = (4 + 24 + 14) / 6
       = 42 / 6
       = 7日
```

</details>

### 問4（四択・品質管理）
QC七つ道具のうち、問題の重要な少数を特定するために使用され、棒グラフと累積曲線を組み合わせたものはどれか。

ア ヒストグラム　　イ パレート図　　ウ 管理図　　エ 散布図

<details><summary>解答・解説</summary>

**正解: イ パレート図**

パレート図は不良項目などを件数の多い順に棒グラフで並べ、累積比率を折れ線グラフで重ねたものです。80:20の法則（問題の80%は20%の原因から発生する）を視覚化し、重点的に取り組むべき項目を特定します。

</details>

### 問5（四択・調達管理）
RFP（提案依頼書）の目的として、最も適切なものはどれか。

ア ベンダーの技術力や実績に関する一般的な情報を収集する
イ システム要件を提示し、ベンダーから具体的な提案を受ける
ウ ベンダーとの契約条件の最終交渉を行う
エ ベンダーの財務状況を調査する

<details><summary>解答・解説</summary>

**正解: イ**

- ア: RFI（情報提供依頼）の目的
- イ: 正しい。RFPは要件を提示して具体的な提案書を求める文書
- ウ: 契約交渉はRFP後のプロセス
- エ: 財務調査はRFPの主目的ではない

RFI → RFP → 提案評価 → 契約 の順序を覚えましょう。

</details>

## 応用問題

### 問6（四択・クリティカルパス）
次のプロジェクトのクリティカルパスと最短完了日数を求めよ。

| 作業 | 所要日数 | 先行作業 |
|------|---------|---------|
| A | 3 | なし |
| B | 5 | なし |
| C | 4 | A |
| D | 2 | B |
| E | 6 | C, D |

ア クリティカルパス: A→C→E、最短13日
イ クリティカルパス: B→D→E、最短13日
ウ クリティカルパス: A→C→E、最短11日
エ クリティカルパス: B→D→E、最短11日

<details><summary>ヒント</summary>

各パスの所要日数を計算し、最長のパスがクリティカルパスです。

</details>

<details><summary>解答・解説</summary>

**正解: ア クリティカルパス: A→C→E、最短13日**

```
パス1: A→C→E = 3+4+6 = 13日
パス2: B→D→E = 5+2+6 = 13日

両方とも13日で、2つのクリティカルパスが存在します。
```

実はパス1とパス2の両方が13日で等しいため、両方ともクリティカルパスです。選択肢の中ではアが正解です。

なお、Eの最早開始はmax(A+C, B+D) = max(7, 7) = 7日目で、完了は7+6=13日目です。

</details>

### 問7（計算・EVM総合）
あるプロジェクトの状況が以下の通りである。各指標を計算せよ。

- BAC（プロジェクト総予算）: 5,000万円
- PV（計画出来高）: 3,000万円
- EV（達成出来高）: 2,500万円
- AC（実コスト）: 3,200万円

求めるもの: SV, CV, SPI, CPI, EAC, VAC

<details><summary>解答・解説</summary>

```
SV = EV - PV = 2,500 - 3,000 = -500万円（スケジュール遅延）
CV = EV - AC = 2,500 - 3,200 = -700万円（コスト超過）
SPI = EV / PV = 2,500 / 3,000 ≈ 0.833（約17%遅延）
CPI = EV / AC = 2,500 / 3,200 ≈ 0.781（約22%超過）
EAC = BAC / CPI = 5,000 / 0.781 ≈ 6,402万円
VAC = BAC - EAC = 5,000 - 6,402 = -1,402万円

診断:
- スケジュールが約17%遅延し、コストが約22%超過している
- このペースが続くと、最終的に約6,400万円かかり、約1,400万円の予算超過となる
- 早急にコスト管理とスケジュールの是正措置が必要
```

</details>

### 問8（記述・品質管理）
プロジェクトのバグ管理において、信頼度成長曲線（バグ曲線）のパターンから品質状況をどのように判断するか、以下の3パターンについて説明せよ。

1. S字カーブを描いて収束している
2. 直線的に増加し続けている
3. 急激に増加した後すぐに収束している

<details><summary>解答・解説</summary>

1. **S字カーブで収束:**
   - 判断: 理想的な状態。テストが順調に進み、バグが出尽くしたと考えられる
   - 初期は発見率が高く、徐々に新規バグの発見が減り、最終的に収束する
   - テスト完了の判断基準として使える

2. **直線的に増加し続けている:**
   - 判断: 品質が悪い、またはテストが不十分。以下の原因が考えられる
   - a) ソフトウェアの品質そのものが低く、テストを続けるほどバグが出る
   - b) テスト範囲を拡大し続けており、新しい領域のバグが出ている
   - 対策: テストを一旦中断し、設計・コードの品質を見直す必要がある

3. **急激に増加した後すぐに収束:**
   - 判断: テストが不十分な可能性がある
   - 初期に表面的なバグだけを発見し、深いテストができていない疑い
   - テストケースの網羅性を確認し、カバレッジが十分かレビューする必要がある
   - 見かけ上は収束しているが、潜在的なバグが残っている可能性が高い

</details>

## チャレンジ問題

### 問9（記述・午後試験形式）
次のプロジェクトスケジュールについて、設問に答えなさい。

| 作業 | 楽観値(O) | 最可能値(M) | 悲観値(P) | 先行作業 |
|------|----------|------------|----------|---------|
| A | 2 | 4 | 12 | なし |
| B | 3 | 5 | 7 | なし |
| C | 1 | 3 | 5 | A |
| D | 4 | 6 | 8 | B |
| E | 2 | 5 | 8 | C, D |

**設問1:** 各作業の期待値と分散を求めよ。

**設問2:** クリティカルパスを特定し、プロジェクトの期待完了日数を求めよ。

**設問3:** プロジェクトが13日以内に完了する確率を求めよ（正規分布表: Z=0.5→69.1%, Z=1.0→84.1%, Z=1.5→93.3%, Z=2.0→97.7%）。

<details><summary>解答・解説</summary>

**設問1の解答:**

```
作業A: Te = (2+4×4+12)/6 = (2+16+12)/6 = 30/6 = 5日
       σ² = {(12-2)/6}² = (10/6)² ≈ 2.78

作業B: Te = (3+4×5+7)/6 = (3+20+7)/6 = 30/6 = 5日
       σ² = {(7-3)/6}² = (4/6)² ≈ 0.44

作業C: Te = (1+4×3+5)/6 = (1+12+5)/6 = 18/6 = 3日
       σ² = {(5-1)/6}² = (4/6)² ≈ 0.44

作業D: Te = (4+4×6+8)/6 = (4+24+8)/6 = 36/6 = 6日
       σ² = {(8-4)/6}² = (4/6)² ≈ 0.44

作業E: Te = (2+4×5+8)/6 = (2+20+8)/6 = 30/6 = 5日
       σ² = {(8-2)/6}² = (6/6)² = 1.00
```

**設問2の解答:**

```
パス1: A→C→E = 5+3+5 = 13日
パス2: B→D→E = 5+6+5 = 16日

クリティカルパス: B→D→E（16日）
プロジェクト期待完了日数: 16日
```

**設問3の解答:**

```
クリティカルパス(B→D→E)の分散の合計:
σ² = 0.44 + 0.44 + 1.00 = 1.88
標準偏差σ = √1.88 ≈ 1.37日

13日以内に完了する確率:
Z = (13 - 16) / 1.37 = -3 / 1.37 ≈ -2.19

Z = -2.19 は非常に低い確率（約1.4%）

つまり、13日以内に完了する確率は約1.4%であり、ほぼ不可能です。
```

プロジェクトの期待完了日数は16日であり、13日での完了は標準偏差の2倍以上短いため、非常に困難です。

</details>

### 問10（記述・午後試験形式）
G社のシステム開発プロジェクトの調達管理について、以下の設問に答えなさい。

G社は新しい顧客管理システムの開発を外部ベンダーに委託する予定である。

**設問1:** 定額契約（FP）と実費償還契約（CR）を比較し、それぞれのリスク配分と適用に適した場面を説明せよ。

**設問2:** RFP作成時に含めるべき主要項目を5つ挙げ、各項目の目的を簡潔に説明せよ。

<details><summary>解答・解説</summary>

**設問1の解答:**

| 項目 | 定額契約（FP） | 実費償還契約（CR） |
|------|-------------|-----------------|
| 概要 | 成果物に対して固定金額を支払う | 実際にかかった費用を精算して支払う |
| リスク配分 | 受注者（ベンダー）がリスクを負う。見積りより工数がかかってもベンダーの負担 | 発注者（G社）がリスクを負う。コスト増加は発注者の負担 |
| 適用場面 | 要件が明確で、スコープの変更が少ないプロジェクト | 要件が不確定で、スコープ変更の可能性が高いプロジェクト |

G社の場合、顧客管理システムの要件が明確であれば定額契約が適切です。要件が流動的であれば実費償還契約を検討しますが、コスト管理のためにインセンティブ条項（目標コスト以下で完了した場合の報奨金など）を設けることが望ましいです。

**設問2の解答:**

1. **システム概要と目的:** ベンダーがプロジェクトの背景と目標を理解し、適切な提案を行うための前提情報を提供する

2. **機能要件と非機能要件:** システムが実現すべき機能と、性能・可用性・セキュリティなどの品質要件を明示し、ベンダーの提案範囲を明確にする

3. **予算とスケジュール:** 予算の上限とマイルストーンを提示し、ベンダーが実現可能性を判断して現実的な提案を行えるようにする

4. **評価基準と選定方法:** 提案書のどの項目をどのような基準で評価するかを事前に明示し、公平かつ透明な選定を行う

5. **契約条件と遵守事項:** 知的財産権の帰属、秘密保持義務、SLA要件、瑕疵担保責任などの契約条件を明示し、後のトラブルを予防する

</details>
