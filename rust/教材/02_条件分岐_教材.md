## 2.1 if / else if / else

### 基本構文

Rustの `if` 文は他の言語と似ていますが、**条件式にカッコ `()` は不要**です（付けても動きますが、コンパイラが警告を出します）。

```rust
fn main() {
    let temperature = 28;

    if temperature >= 30 {
        println!("暑い！熱中症に注意しましょう");
    } else if temperature >= 20 {
        println!("快適な気温です");
    } else if temperature >= 10 {
        println!("少し肌寒いです");
    } else {
        println!("寒い！暖かくしましょう");
    }
    // 出力: 快適な気温です
}
```

> **よくある間違い（他の言語経験者向け）：**
> ```rust
> // NG: カッコは不要（警告が出る）
> if (temperature >= 30) {
>     println!("暑い！");
> }
>
> // OK: カッコなし
> if temperature >= 30 {
>     println!("暑い！");
> }
> ```

### 条件式は必ず bool 型

Rustでは条件式は**必ず `bool` 型**でなければなりません。C/C++ のように整数を条件として使うことはできません。

```rust
fn main() {
    let number = 3;

    // OK: bool 型の比較式
    if number != 0 {
        println!("number はゼロではありません");
    }
    // 出力: number はゼロではありません

    // NG: 整数を直接条件にはできない
    // if number {  // エラー！ expected `bool`, found `i32`
    //     println!("これはエラー");
    // }
}
```

### 複数条件の組み合わせ

```rust
fn main() {
    let age = 20;
    let has_id = true;

    if age >= 20 && has_id {
        println!("入場できます");
    } else if age >= 20 && !has_id {
        println!("身分証明書が必要です");
    } else {
        println!("年齢制限により入場できません");
    }
    // 出力: 入場できます
}
```

### ポイントまとめ

- `if` の条件式にカッコ `()` は不要です
- 条件式は必ず `bool` 型でなければなりません（暗黙の型変換なし）
- `else if` で複数条件を連鎖させ、`else` でどれにも当てはまらない場合を処理します

## 2.2 比較演算子と論理演算子

### 比較演算子（comparison operators）

```
┌──────────┬──────────────────┬─────────────────┐
│ 演算子    │ 意味              │ 例               │
├──────────┼──────────────────┼─────────────────┤
│ ==       │ 等しい            │ x == 5          │
│ !=       │ 等しくない         │ x != 0          │
│ <        │ より小さい         │ x < 10          │
│ >        │ より大きい         │ x > 0           │
│ <=       │ 以下              │ x <= 100        │
│ >=       │ 以上              │ x >= 1          │
└──────────┴──────────────────┴─────────────────┘
```

### 論理演算子（logical operators）

```
┌──────────┬──────────────────┬────────────────────────┐
│ 演算子    │ 意味              │ 例                     │
├──────────┼──────────────────┼────────────────────────┤
│ &&       │ 論理AND（かつ）    │ x > 0 && x < 100     │
│ ||       │ 論理OR（または）   │ x == 0 || x == 1     │
│ !        │ 論理NOT（否定）    │ !is_empty            │
└──────────┴──────────────────┴────────────────────────┘
```

```rust
fn main() {
    let x = 15;

    // 比較演算子
    println!("x == 15: {}", x == 15);   // true
    println!("x != 10: {}", x != 10);   // true
    println!("x > 10: {}", x > 10);     // true
    println!("x <= 15: {}", x <= 15);   // true

    // 論理演算子
    let in_range = x >= 10 && x <= 20;
    println!("10〜20の範囲内: {}", in_range);  // true

    let is_special = x == 0 || x == 100;
    println!("特別な値: {}", is_special);       // false

    let is_not_zero = x != 0;
    println!("ゼロでない: {}", is_not_zero);    // true

    // 短絡評価（short-circuit evaluation）
    // && は左がfalseなら右を評価しない
    // || は左がtrueなら右を評価しない
    let a = false;
    let b = true;
    println!("false && true: {}", a && b);  // false（b は評価されない）
    println!("true || false: {}", b || a);  // true（a は評価されない）
}
```

### 文字列の比較

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = String::from("hello");
    let s3 = "hello";

    // String 同士の比較
    println!("s1 == s2: {}", s1 == s2);  // true

    // String と &str の比較も可能
    println!("s1 == s3: {}", s1 == s3);  // true

    // 辞書順の比較
    println!("\"apple\" < \"banana\": {}", "apple" < "banana");  // true
}
```

### ポイントまとめ

- 比較演算子は `==`, `!=`, `<`, `>`, `<=`, `>=` の6種類です
- 論理演算子は `&&`（AND）、`||`（OR）、`!`（NOT）です
- `&&` と `||` は短絡評価（左の結果で右を評価するか決まる）します
- 異なる型同士は比較できません（型安全）

## 2.3 if は式（expression）

### 式と文の違い

Rustでは、**式（expression）**は値を返し、**文（statement）**は値を返しません。

```
┌──────────────────────────────────────────────────────────┐
│  式（expression）と 文（statement）                        │
│                                                           │
│  式: 値を返す                                             │
│    例: 5 + 3        → 8 を返す                           │
│    例: if x > 0 { "正" } else { "負" }  → &str を返す   │
│    例: { let x = 5; x + 1 }  → 6 を返す                 │
│                                                           │
│  文: 値を返さない                                         │
│    例: let x = 5;   → 代入文                             │
│    例: fn foo() {}  → 関数定義                            │
│                                                           │
│  ※ Rustでは式の末尾にセミコロンを付けると文になる           │
└──────────────────────────────────────────────────────────┘
```

### if 式で値を返す

Rustの `if` は式なので、**値を変数に代入**できます。三項演算子（`? :`）がない代わりにこの書き方を使います。

```rust
fn main() {
    let condition = true;

    // if 式で値を代入
    let number = if condition { 5 } else { 10 };
    println!("number = {}", number);
    // 出力: number = 5

    // 他の言語の三項演算子に相当
    // JavaScript: const number = condition ? 5 : 10;
    // Python:     number = 5 if condition else 10
    // Rust:       let number = if condition { 5 } else { 10 };
}
```

### 実践的な if 式の使い方

```rust
fn main() {
    let score = 85;

    // 成績判定
    let grade = if score >= 90 {
        "A"
    } else if score >= 80 {
        "B"
    } else if score >= 70 {
        "C"
    } else if score >= 60 {
        "D"
    } else {
        "F"
    };

    println!("スコア: {} → 成績: {}", score, grade);
    // 出力: スコア: 85 → 成績: B
}
```

> **よくある間違い：** if 式の各ブロックが返す型は**全て同じ**でなければなりません。
> ```rust
> // エラー！ if と else で型が違う
> let value = if true { 5 } else { "hello" };
> // → expected integer, found `&str`
> ```

### ブロック式

`{}` で囲まれたブロックも式であり、**最後の式の値**がブロック全体の値になります。

```rust
fn main() {
    let y = {
        let x = 3;
        x + 1       // セミコロンなし → この値がブロックの戻り値
    };

    println!("y = {}", y);
    // 出力: y = 4
}
```

> **重要：** ブロックの最後の行に**セミコロンを付けない**と、それが戻り値になります。セミコロンを付けると文になり、ブロックはユニット型 `()` を返します。
> ```rust
> let y = {
>     let x = 3;
>     x + 1;  // セミコロンあり → ブロックは () を返す
> };
> // y の型は () になる（おそらく意図と違う）
> ```

### ポイントまとめ

- Rustの `if` は式であり、値を返すことができます
- 三項演算子の代わりに `let x = if cond { a } else { b };` を使います
- if 式の全ブロックは同じ型を返す必要があります
- ブロック `{}` の最後の式（セミコロンなし）がブロックの戻り値です

## 2.4 match 式の基本

### match とは

`match`（マッチ）式は Rust の最も強力な制御フロー構文です。値をパターンと比較し、マッチしたパターンに対応するコードを実行します。C言語の `switch` に似ていますが、はるかに強力です。

```rust
fn main() {
    let number = 3;

    match number {
        1 => println!("一"),
        2 => println!("二"),
        3 => println!("三"),
        4 => println!("四"),
        5 => println!("五"),
        _ => println!("その他"),  // _ はワイルドカード（どの値にもマッチ）
    }
    // 出力: 三
}
```

### match の基本構文

```
match 対象の値 {
    パターン1 => 式1,
    パターン2 => 式2,
    パターン3 => {
        // 複数行の処理
        式3
    },
    _ => デフォルトの式,
}
```

### match は網羅的（exhaustive）でなければならない

Rustの `match` は**全ての可能性を網羅**しなければなりません。これにより、処理の漏れを防ぎます。

```rust
fn main() {
    let number: i32 = 42;

    // NG: 全ての i32 の値を網羅していない
    // match number {
    //     1 => println!("一"),
    //     2 => println!("二"),
    //     // コンパイルエラー！ non-exhaustive patterns
    // }

    // OK: _ で残り全てをカバー
    match number {
        1 => println!("一"),
        2 => println!("二"),
        _ => println!("その他: {}", number),
    }
    // 出力: その他: 42
}
```

### match も式である

`if` と同様に、`match` も式なので値を返せます。

```rust
fn main() {
    let number = 2;

    let text = match number {
        1 => "one",
        2 => "two",
        3 => "three",
        _ => "other",
    };

    println!("{} → {}", number, text);
    // 出力: 2 → two
}
```

### 複数パターンの | 演算子

`|`（パイプ）で複数のパターンをまとめられます。

```rust
fn main() {
    let day = "土";

    let day_type = match day {
        "月" | "火" | "水" | "木" | "金" => "平日",
        "土" | "日" => "休日",
        _ => "不明",
    };

    println!("{}曜日は{}", day, day_type);
    // 出力: 土曜日は休日
}
```

### ポイントまとめ

- `match` は値をパターンと照合する強力な式です
- `_` はワイルドカードで、どの値にもマッチします
- 全ての可能性を網羅する必要があります（コンパイラが漏れを検出）
- `match` も式なので値を変数に代入できます
- `|` で複数パターンをまとめられます

## 2.5 パターンマッチング

### リテラルパターン

具体的な値と比較します。

```rust
fn main() {
    let status_code = 404;

    let message = match status_code {
        200 => "OK",
        301 => "Moved Permanently",
        400 => "Bad Request",
        401 => "Unauthorized",
        403 => "Forbidden",
        404 => "Not Found",
        500 => "Internal Server Error",
        _ => "Unknown Status",
    };

    println!("HTTP {}: {}", status_code, message);
    // 出力: HTTP 404: Not Found
}
```

### 範囲パターン（range pattern）

`..=` で範囲を指定できます。

```rust
fn main() {
    let score = 85;

    let grade = match score {
        90..=100 => "A（優秀）",
        80..=89  => "B（良好）",
        70..=79  => "C（普通）",
        60..=69  => "D（可）",
        0..=59   => "F（不可）",
        _        => "範囲外",
    };

    println!("点数: {} → 評価: {}", score, grade);
    // 出力: 点数: 85 → 評価: B（良好）
}
```

```rust
fn main() {
    let ch = 'k';

    let category = match ch {
        'a'..='z' => "小文字アルファベット",
        'A'..='Z' => "大文字アルファベット",
        '0'..='9' => "数字",
        _ => "その他",
    };

    println!("'{}' は {} です", ch, category);
    // 出力: 'k' は 小文字アルファベット です
}
```

### 変数束縛（variable binding）

マッチした値を変数に束縛して使えます。

```rust
fn main() {
    let age = 25;

    let message = match age {
        0..=5 => String::from("幼児です"),
        6..=12 => String::from("小学生です"),
        13..=15 => String::from("中学生です"),
        16..=18 => String::from("高校生です"),
        19..=22 => String::from("大学生かもしれません"),
        n if n >= 65 => format!("{}歳、シニアです", n),
        n => format!("{}歳、社会人です", n),  // n に値が束縛される
    };

    println!("{}", message);
    // 出力: 25歳、社会人です
}
```

### ガード条件（match guard）

パターンに追加の `if` 条件を付けられます。

```rust
fn main() {
    let number = 4;

    let description = match number {
        n if n < 0 => "負の数",
        0 => "ゼロ",
        n if n % 2 == 0 => "正の偶数",
        _ => "正の奇数",
    };

    println!("{} は {} です", number, description);
    // 出力: 4 は 正の偶数 です
}
```

### タプルのパターンマッチ

```rust
fn main() {
    let point = (0, -2);

    let location = match point {
        (0, 0) => "原点",
        (x, 0) => {
            println!("x軸上: x={}", x);
            "x軸上"
        }
        (0, y) => {
            println!("y軸上: y={}", y);
            "y軸上"
        }
        (x, y) => {
            println!("座標: ({}, {})", x, y);
            "平面上"
        }
    };

    println!("位置: {}", location);
    // 出力:
    // y軸上: y=-2
    // 位置: y軸上
}
```

### 実践例：簡易計算機

```rust
fn main() {
    let operation = '+';
    let a = 10.0_f64;
    let b = 3.0_f64;

    let result = match operation {
        '+' => a + b,
        '-' => a - b,
        '*' => a * b,
        '/' => {
            if b == 0.0 {
                println!("エラー: ゼロ除算");
                f64::NAN
            } else {
                a / b
            }
        }
        _ => {
            println!("エラー: 未知の演算子 '{}'", operation);
            f64::NAN
        }
    };

    println!("{} {} {} = {:.2}", a, operation, b, result);
    // 出力: 10 + 3 = 13.00
}
```

### ポイントまとめ

- リテラルパターンで具体的な値にマッチします
- `..=` で範囲パターンが使えます（整数、文字）
- 変数束縛でマッチした値を変数に格納できます
- `if` ガード条件で追加の条件を付けられます
- タプルの構造を分解してマッチできます

## 2.6 Option 型と if let

### Option 型とは

Rust には `null` や `nil` がありません。代わりに「値があるかもしれないし、ないかもしれない」を表す `Option<T>` 型を使います。

```
┌──────────────────────────────────────────────────┐
│              Option<T> 型                         │
│                                                   │
│  Some(T)  → 値がある（T は任意の型）              │
│  None     → 値がない                              │
│                                                   │
│  例: Option<i32> は Some(42) または None           │
│                                                   │
│  ┌──────────┐    ┌──────────┐                    │
│  │ Some(42) │    │   None   │                    │
│  │ 値がある  │    │ 値がない  │                    │
│  └──────────┘    └──────────┘                    │
│                                                   │
│  ※ null参照（null pointer）によるバグを              │
│    コンパイル時に防ぐ Rust の重要な仕組み           │
└──────────────────────────────────────────────────┘
```

```rust
fn main() {
    // Option 型の例
    let some_number: Option<i32> = Some(42);
    let no_number: Option<i32> = None;

    println!("some_number: {:?}", some_number);
    // 出力: some_number: Some(42)

    println!("no_number: {:?}", no_number);
    // 出力: no_number: None

    // 配列の安全なアクセス → Option を返す
    let numbers = [10, 20, 30];
    let first = numbers.get(0);   // Some(&10)
    let tenth = numbers.get(9);   // None

    println!("first: {:?}", first);  // Some(10)
    println!("tenth: {:?}", tenth);  // None
}
```

### match で Option を処理する

```rust
fn main() {
    let values = [10, 20, 30];

    // 安全なアクセス
    let result = values.get(1);  // Option<&i32>

    match result {
        Some(value) => println!("値: {}", value),
        None => println!("値が見つかりません"),
    }
    // 出力: 値: 20

    // 範囲外アクセス
    let result = values.get(5);

    match result {
        Some(value) => println!("値: {}", value),
        None => println!("値が見つかりません"),
    }
    // 出力: 値が見つかりません
}
```

### if let 構文

`match` で特定のパターン1つだけを処理したい場合、`if let` を使うとコードが簡潔になります。

```rust
fn main() {
    let some_value: Option<i32> = Some(42);

    // match で書く場合（少し冗長）
    match some_value {
        Some(v) => println!("match: 値は {}", v),
        None => {}  // 何もしない
    }
    // 出力: match: 値は 42

    // if let で書く場合（簡潔）
    if let Some(v) = some_value {
        println!("if let: 値は {}", v);
    }
    // 出力: if let: 値は 42

    // else と組み合わせ
    let no_value: Option<i32> = None;

    if let Some(v) = no_value {
        println!("値は {}", v);
    } else {
        println!("値がありません");
    }
    // 出力: 値がありません
}
```

### Option の便利メソッド

```rust
fn main() {
    let some_val: Option<i32> = Some(42);
    let none_val: Option<i32> = None;

    // unwrap_or: None の場合にデフォルト値を使う
    println!("{}", some_val.unwrap_or(0));   // 42
    println!("{}", none_val.unwrap_or(0));   // 0

    // is_some / is_none: 値の有無を確認
    println!("some_val.is_some(): {}", some_val.is_some());  // true
    println!("none_val.is_none(): {}", none_val.is_none());  // true

    // map: Some の中の値を変換
    let doubled = some_val.map(|x| x * 2);
    println!("doubled: {:?}", doubled);  // Some(84)

    let doubled_none = none_val.map(|x| x * 2);
    println!("doubled_none: {:?}", doubled_none);  // None

    // unwrap_or_else: None の場合にクロージャでデフォルト値を計算
    let result = none_val.unwrap_or_else(|| {
        println!("デフォルト値を計算中...");
        99
    });
    println!("result: {}", result);
    // 出力:
    // デフォルト値を計算中...
    // result: 99
}
```

> **よくある間違い：** `unwrap()` は None の場合にパニック（プログラムが異常終了）するため、**プロダクションコードでは基本的に使いません**。
> ```rust
> let x: Option<i32> = None;
> // let v = x.unwrap();  // パニック！ thread 'main' panicked at 'called `Option::unwrap()` on a `None` value'
> ```
> 代わりに `unwrap_or`、`if let`、`match` を使いましょう。

### ポイントまとめ

- Rustには `null` がなく、代わりに `Option<T>`（`Some(T)` / `None`）を使います
- `match` で `Some` と `None` の両方を処理できます
- `if let` は1つのパターンだけ処理したい場合に便利です
- `unwrap_or` や `map` で Option を簡潔に扱えます
- `unwrap()` はパニックの原因になるため、プロダクションコードでは避けましょう

## 2.7 実践的な条件分岐パターン

### パターン1: 入力バリデーション

```rust
fn validate_age(age: i32) -> Result<String, String> {
    match age {
        n if n < 0 => Err(format!("年齢が負の値です: {}", n)),
        0 => Err("年齢が0です".to_string()),
        1..=17 => Ok("未成年".to_string()),
        18..=64 => Ok("成人".to_string()),
        65..=120 => Ok("シニア".to_string()),
        n => Err(format!("年齢が範囲外です: {}", n)),
    }
}

fn main() {
    let test_ages = [25, -5, 0, 70, 150, 15];

    for &age in &test_ages {
        match validate_age(age) {
            Ok(category) => println!("年齢 {}: {} ✓", age, category),
            Err(error) => println!("年齢 {}: エラー - {}", age, error),
        }
    }
    // 出力:
    // 年齢 25: 成人 ✓
    // 年齢 -5: エラー - 年齢が負の値です: -5
    // 年齢 0: エラー - 年齢が0です
    // 年齢 70: シニア ✓
    // 年齢 150: エラー - 年齢が範囲外です: 150
    // 年齢 15: 未成年 ✓
}
```

```rust
            println!("  3. バナナ");
```

### パターン4: 状態遷移

```rust
fn main() {
    let mut state = "idle";
    let events = ["start", "process", "complete", "reset"];

    println!("状態遷移シミュレーション:");
    println!("初期状態: {}", state);

    for event in &events {
        state = match (state, *event) {
            ("idle", "start") => "running",
            ("running", "process") => "processing",
            ("processing", "complete") => "completed",
            ("completed", "reset") | ("idle", "reset") => "idle",
            (s, e) => {
                println!("  無効な遷移: {} -> {} (イベント: {})", s, e, e);
                s
            }
        };
        println!("  イベント '{}' → 状態: {}", event, state);
    }
    // 出力:
    // 状態遷移シミュレーション:
    // 初期状態: idle
    //   イベント 'start' → 状態: running
    //   イベント 'process' → 状態: processing
    //   イベント 'complete' → 状態: completed
    //   イベント 'reset' → 状態: idle
}
```

## 2.8 while let

`if let` と同様に、`while let` はパターンがマッチする間ループを続けます。

```rust
fn main() {
    let mut stack = vec![1, 2, 3, 4, 5];

    println!("スタックの中身を順に取り出す:");

    // pop() は Option<T> を返す
    // Some(value) にマッチする間ループを続ける
    while let Some(top) = stack.pop() {
        println!("  取り出し: {}", top);
    }
    // 出力:
    // スタックの中身を順に取り出す:
    //   取り出し: 5
    //   取り出し: 4
    //   取り出し: 3
    //   取り出し: 2
    //   取り出し: 1

    println!("スタックは空になりました: {:?}", stack);
    // 出力: スタックは空になりました: []
}
```

### ポイントまとめ

- `while let` はパターンがマッチする間繰り返します
- `Option` を返すメソッド（`pop()` など）と組み合わせると便利です

```rust
fn main() {
    println!("=== BMI 計算・判定プログラム ===\n");

    let people = vec![
        ("田中", 170.0, 65.0),
        ("鈴木", 160.0, 80.0),
        ("佐藤", 180.0, 70.0),
        ("高橋", 155.0, 45.0),
    ];

    for (name, height_cm, weight) in &people {
        let height_m = height_cm / 100.0;
        let bmi = calculate_bmi(*weight, height_m);

        if let Some(bmi_value) = bmi {
            let category = categorize_bmi(bmi_value);
            let advice = get_advice(bmi_value);

            println!("{}さん:", name);
            println!("  身長: {}cm, 体重: {}kg", height_cm, weight);
            println!("  BMI: {:.1} ({})", bmi_value, category);
            println!("  アドバイス: {}", advice);
            println!();
        } else {
            println!("{}さん: BMI計算エラー（入力値を確認してください）\n", name);
        }
    }
}

fn calculate_bmi(weight: f64, height_m: f64) -> Option<f64> {
    if weight <= 0.0 || height_m <= 0.0 {
        None
    } else {
        Some(weight / (height_m * height_m))
    }
}

fn categorize_bmi(bmi: f64) -> &'static str {
    match bmi as u32 {
        0..=17 => "低体重（痩せ型）",
        18..=24 => "普通体重",
        25..=29 => "肥満（1度）",
        30..=34 => "肥満（2度）",
        35..=39 => "肥満（3度）",
        _ => "肥満（4度）",
    }
}

fn get_advice(bmi: f64) -> &'static str {
    if bmi < 18.5 {
        "栄養バランスの良い食事を心がけましょう"
    } else if bmi < 25.0 {
        "現在の生活習慣を維持しましょう"
    } else if bmi < 30.0 {
        "適度な運動と食事管理を始めましょう"
    } else {
        "専門家に相談することをおすすめします"
    }
}
```
