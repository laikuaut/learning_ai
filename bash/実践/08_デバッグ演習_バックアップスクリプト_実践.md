> **難易度**: ★★★★☆（上級）
> **前提知識**: 第1章〜第8章（全章）
> **課題の種類**: デバッグ
> **学習目標**: バグだらけのBashスクリプトを読み解き、シェルスクリプト特有のバグ（クォーティング、変数展開、リダイレクト等）を特定・修正する力を養う

## 課題の説明

以下の「ディレクトリバックアップスクリプト」には **10個のバグ** が埋め込まれています。
スクリプトを実行し、エラーメッセージや意図しない動作を手がかりにすべてのバグを見つけて修正してください。

### 進め方

1. 下のバグ入りコードを `backup_buggy.sh` として保存する
2. テスト用ディレクトリを作成する（`mkdir -p /tmp/test_src && echo "hello" > /tmp/test_src/test.txt`）
3. 実行してエラーを確認する（`bash backup_buggy.sh /tmp/test_src /tmp/backup_dest`）
4. 1つずつバグを修正し、再実行する
5. すべてのバグを修正して正常に動作すれば完了

---

## バグ入りコード

以下のコードをそのまま `backup_buggy.sh` として保存してください。

```bash
#!/usr/bin/env bash
# ディレクトリバックアップスクリプト（バグ入り版）
# このコードには10個のバグが含まれています。すべて修正してください。
# 使い方: ./backup_buggy.sh <バックアップ元> <バックアップ先>

set -euo pipefail

# バグ1: ログファイルのパス
LOG_FILE=~/backup.log

# --- ログ出力関数 ---
log_message() {
    local level=$1
    local message=$2
    local timestamp
    timestamp=$(date "+%Y-%m-%d %H:%M:%S")
    echo "[$timestamp] [$level] $message" | tee -a $LOG_FILE  # バグ2
}

# --- バックアップサイズ計算 ---
get_dir_size() {
    local dir=$1
    du -sh $dir | awk '{print $1}'  # バグ3
}

# --- 古いバックアップの削除 ---
cleanup_old_backups() {
    local backup_dir=$1
    local keep_count=$2

    local backups=($(ls -d ${backup_dir}/backup_* 2>/dev/null | sort -r))  # バグ4
    local total=${#backups[@]}

    if [ $total -gt $keep_count ]; then
        log_message "INFO" "${total}件中、古い$((total - keep_count))件を削除します"
        for ((i = keep_count; i < total; i++)); do
            rm -rf ${backups[$i]}
            log_message "INFO" "削除: ${backups[$i]}"
        done
    fi
}

# --- 引数チェック ---
if [ $# -ne 2 ]; then  # バグ5
    echo "使い方: $0 <バックアップ元> <バックアップ先>"
    exit 1
fi

src_dir=$1
dest_dir=$2

# ソースディレクトリの存在確認
if [ ! -d $src_dir ]; then  # バグ6
    log_message "ERROR" "ソースディレクトリが見つかりません: $src_dir"
    exit 1
fi

# バックアップ先ディレクトリの作成
mkdir -p "$dest_dir"

# タイムスタンプ付きバックアップディレクトリ
timestamp=$(date "+%Y%m%d_%H%M%S")
backup_name="backup_${timestamp}"
backup_path="${dest_dir}/${backup_name}"

log_message "INFO" "バックアップ開始: $src_dir → $backup_path"

# バックアップ前のサイズ
src_size=$(get_dir_size "$src_dir")
log_message "INFO" "ソースサイズ: $src_size"

# rsyncでバックアップ（rsyncがない場合はcpにフォールバック）
if command -v rsync > /dev/null 2>&1; then
    rsync -a "$src_dir" "$backup_path"  # バグ7
else
    cp -r "$src_dir" "$backup_path"
fi

# バックアップ後の検証
if [ -d "$backup_path" ]; then
    backup_size=$(get_dir_size "$backup_path")
    log_message "INFO" "バックアップ成功: $backup_size"
else
    log_message "ERROR" "バックアップに失敗しました"
    exit 1
fi

# バックアップ一覧の表示
echo ""
echo "--- バックアップ一覧 ---"
for dir in "$dest_dir"/backup_*; do
    if [ -d "$dir" ]; then
        size=$(get_dir_size "$dir")
        name=$(basename "$dir")
        date_part=${name#backup_}
        echo "  ${date_part}  ${size}"
    fi
done

# 古いバックアップの削除（5世代保持）
cleanup_old_backups "$dest_dir" 5

# 完了メッセージ
file_count=$(find "$backup_path" -type f | wc -l)  # バグ8
log_message "INFO" "バックアップ完了: ${file_count}ファイル"

# バグ9: 終了コードの扱い
echo "バックアップが正常に完了しました"
exit $?

# バグ10: この関数は呼ばれることがない
verify_backup() {
    local src=$1
    local dest=$2
    local src_count=$(find "$src" -type f | wc -l)
    local dest_count=$(find "$dest" -type f | wc -l)
    if [ "$src_count" -eq "$dest_count" ]; then
        log_message "INFO" "検証OK: ファイル数一致 (${src_count}件)"
        return 0
    else
        log_message "WARN" "検証NG: ソース${src_count}件 / バックアップ${dest_count}件"
        return 1
    fi
}
```

## 解答例

<details>
<summary>全10個のバグと修正方法</summary>

### バグ1：パス展開 ─ `~` がクォート内で展開されない

```bash
# 修正前
LOG_FILE=~/backup.log
# これ自体は動作するが、ダブルクォートで囲むと展開されない
# 一貫性と安全性のため $HOME を使う

# 修正後
LOG_FILE="${HOME}/backup.log"
```

実は `LOG_FILE=~/backup.log` は代入文では展開されます。しかし、他の場所で `"$LOG_FILE"` と使う場合の一貫性と、スクリプトの可読性を考えて `$HOME` を推奨します。ここでは実際の問題はバグ2と複合しています。

### バグ2：クォーティング不足 ─ `tee` のファイルパス

```bash
# 修正前
echo "..." | tee -a $LOG_FILE
# 修正後
echo "..." | tee -a "$LOG_FILE"
```

`$LOG_FILE` のパスにスペースが含まれる場合、ワードスプリッティングが起きます。

### バグ3：クォーティング不足 ─ `du` のパス

```bash
# 修正前
du -sh $dir | awk '{print $1}'
# 修正後
du -sh "$dir" | awk '{print $1}'
```

### バグ4：ワードスプリッティング ─ 配列代入

```bash
# 修正前（スペースを含むパスで壊れる）
local backups=($(ls -d ${backup_dir}/backup_* 2>/dev/null | sort -r))

# 修正後（mapfileを使う）
local -a backups=()
mapfile -t backups < <(ls -d "${backup_dir}"/backup_* 2>/dev/null | sort -r)
```

`$(...)` の結果を配列に代入すると、パスにスペースが含まれる場合に分割されます。`mapfile`（`readarray`）を使うのが安全です。

### バグ5：`set -u` との衝突 ─ 引数チェック

```bash
# 修正前（$# は問題ないが、set -u 環境で $1, $2 が未定義だとエラー）
if [ $# -ne 2 ]; then

# 修正後（$# 自体は常に定義されているので問題ないが、クォーティングを追加）
if [ "$#" -ne 2 ]; then
```

実は `$#` は常に定義されるので `set -u` では問題になりませんが、クォーティングの一貫性のため修正します。本当の問題は、この後の `$1`, `$2` の参照前にチェックが必要なことです。

### バグ6：クォーティング不足 ─ ディレクトリ確認

```bash
# 修正前
if [ ! -d $src_dir ]; then
# 修正後
if [ ! -d "$src_dir" ]; then
```

### バグ7：rsyncの末尾スラッシュ ─ ディレクトリ構造の違い

```bash
# 修正前（src_dir の中身がさらにサブディレクトリとしてコピーされる）
rsync -a "$src_dir" "$backup_path"
# 修正後（末尾に / を付けて中身だけをコピー）
rsync -a "$src_dir/" "$backup_path/"
```

`rsync` では末尾の `/` の有無で動作が変わります。
- `rsync -a /tmp/test_src /tmp/backup` → `/tmp/backup/test_src/...` になる
- `rsync -a /tmp/test_src/ /tmp/backup/` → `/tmp/backup/...` になる

### バグ8：`wc -l` の空白 ─ macOS互換性

```bash
# 修正前（macOSのwcは先頭に空白が入る）
file_count=$(find "$backup_path" -type f | wc -l)
# 修正後
file_count=$(find "$backup_path" -type f | wc -l | tr -d '[:space:]')
```

### バグ9：終了コードの誤用

```bash
# 修正前（echo は常に成功するので $? は 0。意味がない）
echo "バックアップが正常に完了しました"
exit $?

# 修正後
echo "バックアップが正常に完了しました"
exit 0
```

`exit $?` は直前のコマンドの終了コードを返します。`echo` は通常成功するので常に `exit 0` と同じですが、意図が不明確です。明示的に `exit 0` とすべきです。

### バグ10：未使用関数の位置 ─ `verify_backup` が呼ばれない

```bash
# 修正前：exit の後に定義されており、呼び出しもない

# 修正後：exit の前に関数を呼び出す
verify_backup "$src_dir" "$backup_path"

log_message "INFO" "バックアップ完了: ${file_count}ファイル"
echo "バックアップが正常に完了しました"
exit 0
```

バックアップ後の検証は重要な工程です。関数が定義されているのに呼び出されていないのはバグです。

#!/usr/bin/env bash
# ディレクトリバックアップスクリプト（修正済み版）
# 使い方: ./backup.sh <バックアップ元> <バックアップ先>

    local message="$2"
    local timestamp
    timestamp=$(date "+%Y-%m-%d %H:%M:%S")

# --- バックアップ検証（修正10: exit前に移動） ---

    local dest="$2"
    local src_count
    local dest_count
    src_count=$(find "$src" -type f | wc -l | tr -d '[:space:]')

    # 修正4: mapfile を使用

    mapfile -t backups < <(ls -d "${backup_dir}"/backup_* 2>/dev/null | sort -r)
    local total=${#backups[@]}

if [ "$#" -ne 2 ]; then  # 修正5: クォーティング

backup_path="${dest_dir}/${backup_name}"

# バックアップ前のサイズ
src_size=$(get_dir_size "$src_dir")
log_message "INFO" "ソースサイズ: $src_size"

# rsyncでバックアップ
if command -v rsync > /dev/null 2>&1; then
