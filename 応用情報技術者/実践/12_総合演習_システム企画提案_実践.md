## シナリオ

### 株式会社フレッシュマート ─ 中堅スーパーマーケットチェーンのDX推進

**【会社概要】**
- 企業名: 株式会社フレッシュマート
- 業種: 食品スーパーマーケットチェーン
- 店舗数: 関東圏に30店舗
- 従業員数: 2,000名（正社員500名、パート・アルバイト1,500名）
- 年商: 300億円
- IT部門: 正社員8名

**【現状のシステム構成】**

```
┌──────────────────────────────────────────────┐
│                  本社データセンター             │
│                                               │
│  [POSサーバ]  [在庫管理]  [会計システム]       │
│  (10年前導入)  (7年前導入)  (5年前導入)        │
│       │           │           │               │
│       └─────┬─────┘           │               │
│             │                 │               │
│  [ファイルサーバ]  [メールサーバ(オンプレミス)]  │
│                                               │
└──────────┬────────────────────────────────────┘
           │ VPN（各店舗と接続）
     ┌─────┼─────┬─────┐
   店舗1  店舗2  ...  店舗30
   [POS端末5台] [POS端末5台]
```

**【経営課題】**

社長から情報システム部門に以下の要望が出されています。

1. **食品ロスの削減**: 現在の廃棄率は売上の3%（年間9億円）。これを1.5%以下に半減したい。
2. **人手不足への対応**: パート・アルバイトの採用が困難になっており、業務効率化で必要人員を10%削減したい。
3. **顧客体験の向上**: 競合のネットスーパーに顧客が流出している。オンライン注文・配達サービスを開始したい。
4. **データ活用**: 各店舗のPOSデータ・在庫データが分散しており、全社横断での分析ができていない。
5. **IT基盤の刷新**: POSサーバが老朽化しており、メーカーサポートが2年後に終了する。

**【予算制約】**
- 初期投資上限: 3億円
- 年間IT運用予算: 1.5億円（現在: 1.2億円）

**設計方針:**
- **クラウド移行**: POSサーバ、在庫管理、データ分析基盤をクラウドに移行。老朽化サーバからの脱却と拡張性確保
- **SaaS活用**: 会計システムはSaaS（クラウド会計）に移行。IT部門の運用負荷を軽減
- **エッジ処理**: POS端末はクラウドPOS化するが、オフライン対応のためローカルキャッシュを保持
- **API連携**: 各システム間はAPIで疎結合に連携。マイクロサービス的なアプローチ

**課題2-2: ネットワーク構成**

```
インターネット
     │
  [CDN/WAF]
     │
  [クラウドVPC]
     │
  ┌──┴────────────────────────────────┐
  │  [ALB] → [EC/Webサーバ群]        │
  │  [内部ALB] → [API/バッチサーバ群]  │
  │  [RDS(Multi-AZ)] [S3] [Redis]    │
  └──┬───────────────────────────────┘
     │
  [SD-WAN ハブ]
     │
  ┌──┼──────────┬──────────┐
  │  │          │          │
 店舗1(SD-WAN) 店舗2      店舗30
  ├─[L2SW]      ...
  ├─[POS端末×5]
  ├─[バックヤードPC]
  └─[IoTセンサ(温度管理)]
```

接続方式: SD-WAN（Software-Defined WAN）
- 各店舗にSD-WANルータを設置
- 主回線: 光回線、副回線: LTE回線（冗長化）
- トラフィックの優先制御（POS通信 > バッチ通信 > その他）

**課題2-3: 可用性要件**

| システム | 稼働率目標 | 年間許容DT | 達成手段 |
|---------|-----------|-----------|---------|
| POS系 | 99.99% | 52分 | クラウドPOS＋ローカルフォールバック |
| EC/注文 | 99.9% | 8.7時間 | マルチAZ＋オートスケーリング |
| 在庫管理 | 99.9% | 8.7時間 | Multi-AZ RDS＋リードレプリカ |
| データ分析 | 99.5% | 43.8時間 | 日次バッチ処理、障害時は翌日再実行 |

POS系は店舗営業に直結するため最高の可用性が必要。ローカルキャッシュにより、クラウド障害時でも会計処理を継続できる設計とします。

```sql
-- 顧客テーブル
CREATE TABLE customers (
    customer_id   SERIAL PRIMARY KEY,
    name          VARCHAR(100) NOT NULL,
    email         VARCHAR(255) NOT NULL UNIQUE,
    address       VARCHAR(500),
    phone         VARCHAR(20),
    member_rank   VARCHAR(10) DEFAULT 'regular'
                  CHECK(member_rank IN ('regular','silver','gold','platinum')),
    created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 店舗テーブル
CREATE TABLE stores (
    store_id      SERIAL PRIMARY KEY,
    store_name    VARCHAR(100) NOT NULL,
    address       VARCHAR(500) NOT NULL,
    phone         VARCHAR(20),
    is_active     BOOLEAN DEFAULT TRUE
);

-- 商品テーブル
CREATE TABLE products (
    product_id    SERIAL PRIMARY KEY,
    product_name  VARCHAR(200) NOT NULL,
    category      VARCHAR(50) NOT NULL,
    unit_price    INTEGER NOT NULL CHECK(unit_price >= 0),
    expiry_days   INTEGER,  -- 消費期限（製造から何日）
    is_active     BOOLEAN DEFAULT TRUE
);

-- 在庫テーブル
CREATE TABLE inventory (
    store_id      INTEGER REFERENCES stores(store_id),
    product_id    INTEGER REFERENCES products(product_id),
    quantity      INTEGER NOT NULL DEFAULT 0 CHECK(quantity >= 0),
    updated_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (store_id, product_id)
);

-- 注文テーブル
CREATE TABLE orders (
    order_id      SERIAL PRIMARY KEY,
    customer_id   INTEGER NOT NULL REFERENCES customers(customer_id),
    store_id      INTEGER NOT NULL REFERENCES stores(store_id),
    order_date    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    delivery_type VARCHAR(20) CHECK(delivery_type IN ('delivery','pickup')),
    total_amount  INTEGER NOT NULL DEFAULT 0,
    status        VARCHAR(20) DEFAULT 'pending'
                  CHECK(status IN ('pending','confirmed','preparing',
                                   'delivering','completed','cancelled'))
);

-- 注文明細テーブル
CREATE TABLE order_details (
    order_id      INTEGER REFERENCES orders(order_id),
    product_id    INTEGER REFERENCES products(product_id),
    quantity      INTEGER NOT NULL CHECK(quantity > 0),
    unit_price    INTEGER NOT NULL CHECK(unit_price >= 0),
    PRIMARY KEY (order_id, product_id)
);

-- インデックス
CREATE INDEX idx_orders_customer ON orders(customer_id);
CREATE INDEX idx_orders_store ON orders(store_id);
CREATE INDEX idx_orders_date ON orders(order_date);
CREATE INDEX idx_inventory_product ON inventory(product_id);
```

**課題4-1: セキュリティリスクと対策**

| # | リスク | 発生可能性 | 影響度 | スコア | 対策 |
|---|--------|-----------|--------|--------|------|
| 1 | EC顧客情報の漏洩 | 3 | 5 | 15 | WAF導入、脆弱性診断の定期実施、暗号化（AES-256） |
| 2 | ランサムウェアによる業務停止 | 3 | 5 | 15 | EDR導入、バックアップの3-2-1ルール、セグメンテーション |

| 4 | クラウド設定ミスによる情報公開 | 3 | 4 | 12 | CSPM（Cloud Security Posture Management）導入、IaC化 |
| 5 | SD-WAN経由の不正アクセス | 2 | 4 | 8 | ゼロトラスト原則の適用、mTLS認証、トラフィック監視 |

**課題4-2: 個人情報管理方針**

1. **収集の最小化**: 注文処理に必要な最低限の情報のみ収集
2. **暗号化**: DB保存時はカラムレベル暗号化、通信はTLS1.3
3. **アクセス制御**: 役割ベースアクセス制御（RBAC）、最小権限原則
4. **保存期間**: 退会後3年で自動削除、配送先情報は注文完了後1年
5. **監査ログ**: 個人情報へのアクセスを全記録、月次レビュー
6. **インシデント対応**: 72時間以内の個人情報保護委員会への報告体制
7. **プライバシーポリシー**: 利用目的の明示、オプトアウトの仕組み

**課題5-2: クリティカルパス**

主要な依存関係:

```
要件定義(4ヶ月) → クラウド基盤(3.5ヶ月) → EC開発(7.5ヶ月) → 統合テスト(2ヶ月) → 移行(5ヶ月)
```

クリティカルパス: 要件定義 → EC開発 → 統合テスト → 移行
合計工期: 4 + 7.5 + 2 + 5 = **18.5ヶ月**（バッファ含めて20ヶ月）

**課題5-3: EVM（6か月目時点のシミュレーション）**

BAC = 3億円、6か月目のPV = 1.0億円と仮定

```
PV = 1.0億円（計画: 33%完了予定）
EV = 0.85億円（実績: 28%完了）
AC = 0.95億円（実際に使った金額）

SV = 0.85 - 1.0 = -0.15億円（遅延）
CV = 0.85 - 0.95 = -0.10億円（超過）
SPI = 0.85 / 1.0 = 0.85
CPI = 0.85 / 0.95 = 0.89

EAC = 3.0 / 0.89 = 3.37億円（予算超過見込み）
```

```python
import random

# --- サンプルデータ生成 ---
def generate_sales_data(days=90):
    """90日分の売上データを生成"""
    data = []
    for day in range(1, days + 1):
        # 曜日による変動（0=月 ... 6=日）
        weekday = day % 7
        base = 100
        if weekday in [5, 6]:  # 土日は売上増
            base = 150
        # ランダム変動
        sales = max(0, int(base + random.gauss(0, 20)))
        # 発注量（現在は固定120個/日）
        ordered = 120
        # 在庫計算
        waste = max(0, ordered - sales)  # 売れ残り＝廃棄
        stockout = max(0, sales - ordered)  # 欠品

        data.append({
            "day": day,
            "weekday": weekday,
            "sales": sales,
            "ordered": ordered,
            "waste": waste,
            "stockout": stockout,
        })
    return data

def moving_average(data, window=7):
    """移動平均で需要を予測"""
    predictions = []
    for i in range(len(data)):
        if i < window:
            # データが不足している場合は全体平均
            avg = sum(d["sales"] for d in data[:i+1]) / (i + 1)
        else:
            avg = sum(d["sales"] for d in data[i-window:i]) / window
        predictions.append(int(avg))
    return predictions

def weekday_name(wd):
    """曜日番号を日本語に変換"""
    names = ["月", "火", "水", "木", "金", "土", "日"]
    return names[wd]
```

```python
# フレッシュマート食品ロス分析＆需要予測シミュレータ（改良版）
# 学べる内容：曜日別予測、複数カテゴリ分析、最適化効果の可視化
# 実行方法：python freshmart_advanced.py

import random
import math

class DemandPredictor:
    """曜日別移動平均による需要予測"""

    def __init__(self, window_weeks=4):
        self.window_weeks = window_weeks
        self.history = {wd: [] for wd in range(7)}

    def add_observation(self, weekday, sales):
        """実績データを追加"""
        self.history[weekday].append(sales)

    def predict(self, weekday):
        """曜日別の予測値を返す"""
        history = self.history[weekday]
        if not history:
            return 100  # デフォルト値
        window = min(self.window_weeks, len(history))
        recent = history[-window:]
        return int(sum(recent) / len(recent))

    def predict_with_confidence(self, weekday, z=1.645):
        """予測値と信頼区間を返す（90%信頼区間）"""
        history = self.history[weekday]
        if len(history) < 2:
            pred = self.predict(weekday)
            return pred, pred, pred
        window = min(self.window_weeks, len(history))
        recent = history[-window:]
        mean = sum(recent) / len(recent)
        variance = sum((x - mean) ** 2 for x in recent) / (len(recent) - 1)
        std = math.sqrt(variance)
        lower = max(0, int(mean - z * std))
        upper = int(mean + z * std)
        return int(mean), lower, upper

def simulate_category(name, base_demand, variability, days=90):
    """1カテゴリのシミュレーション"""
    predictor = DemandPredictor(window_weeks=4)
    results = {"fixed": {"waste": 0, "stockout": 0, "ordered": 0},
               "optimized": {"waste": 0, "stockout": 0, "ordered": 0}}
    daily_data = []

    for day in range(1, days + 1):
        weekday = day % 7
        # 実際の需要
        wd_factor = 1.4 if weekday in [5, 6] else 1.0
        actual = max(0, int(base_demand * wd_factor + random.gauss(0, variability)))

        # 固定発注
        fixed_order = int(base_demand * 1.1)
        results["fixed"]["ordered"] += fixed_order
        results["fixed"]["waste"] += max(0, fixed_order - actual)
        results["fixed"]["stockout"] += max(0, actual - fixed_order)

        # 最適化発注（曜日別予測 + 安全在庫）
        pred, lower, upper = predictor.predict_with_confidence(weekday)
        # 安全在庫: 予測値と上限の中間
        optimal_order = int((pred + upper) / 2)
        results["optimized"]["ordered"] += optimal_order
        results["optimized"]["waste"] += max(0, optimal_order - actual)
        results["optimized"]["stockout"] += max(0, actual - optimal_order)

        predictor.add_observation(weekday, actual)
        daily_data.append({"day": day, "actual": actual, "predicted": pred})

    return results, daily_data

# --- メイン ---
print("=" * 65)
print("  フレッシュマート 食品ロス削減シミュレータ（改良版）")
print("=" * 65)

random.seed(42)

categories = [
    {"name": "弁当・惣菜", "base": 80, "var": 15, "cost": 300, "price": 500},
    {"name": "パン類", "base": 120, "var": 25, "cost": 100, "price": 200},
    {"name": "乳製品", "base": 60, "var": 10, "cost": 150, "price": 250},
    {"name": "カット野菜", "base": 50, "var": 12, "cost": 120, "price": 200},
]

total_saving = 0

print(f"\n{'カテゴリ':<12} | {'現行廃棄率':>8} | {'最適化後':>8} | {'削減額(90日)':>12} | {'年間削減見込':>12}")
print("-" * 70)

for cat in categories:
    results, daily = simulate_category(cat["name"], cat["base"], cat["var"])

    fix = results["fixed"]
    opt = results["optimized"]

    fix_rate = fix["waste"] / fix["ordered"] * 100 if fix["ordered"] > 0 else 0
    opt_rate = opt["waste"] / opt["ordered"] * 100 if opt["ordered"] > 0 else 0

    # コスト計算
    fix_loss = fix["waste"] * cat["cost"] + fix["stockout"] * cat["price"]
    opt_loss = opt["waste"] * cat["cost"] + opt["stockout"] * cat["price"]
    saving = fix_loss - opt_loss
    total_saving += saving

    print(f"{cat['name']:<12} | {fix_rate:>7.1f}% | {opt_rate:>7.1f}% | {saving:>11,}円 | {saving*4:>11,}円")

print("-" * 70)
print(f"{'合計':<12} | {'':<8} | {'':<8} | {total_saving:>11,}円 | {total_saving*4:>11,}円")

# 全30店舗への展開効果
annual_all_stores = total_saving * 4 * 30
print(f"\n全30店舗展開時の年間削減見込: {annual_all_stores:,}円（{annual_all_stores/100000000:.1f}億円）")

# 投資対効果

annual_benefit = annual_all_stores / 10000  # 万円に変換
roi = (annual_benefit * 5 - investment) / investment * 100
payback = investment / annual_benefit

print(f"\n--- 投資対効果 ---")
print(f"AI需要予測システム投資額: {investment:,}万円")
print(f"年間削減効果: {annual_benefit:,.0f}万円")
print(f"単純回収期間: {payback:.1f}年")
print(f"5年間ROI: {roi:.0f}%")
```
