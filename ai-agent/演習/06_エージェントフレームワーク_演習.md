### 問題1：主要フレームワークの特徴比較

以下の3つのエージェントフレームワークについて、それぞれの特徴・強み・弱みを整理してください。

1. LangGraph
2. Claude Agent SDK（Anthropic）
3. CrewAI

**期待される出力例：**

```
1. LangGraph
   特徴：...
   強み：...
   弱み：...
```

<details>
<summary>解答例</summary>

```
1. LangGraph
   特徴：LangChainエコシステム上に構築されたステートフルなグラフベースの
         エージェントフレームワーク。ノードとエッジでワークフローを定義し、
         状態遷移を明示的に管理できる。
   強み：
   - ステートグラフによる明示的なフロー制御（条件分岐・ループが直感的）
   - 永続的な状態管理（チェックポイント機能で中断・再開が可能）
   - Human-in-the-Loopの組み込みが容易
   - LangChainの豊富なツール・インテグレーションをそのまま活用可能
   - ストリーミング対応が充実
   弱み：
   - LangChainへの依存度が高く、学習コストがやや大きい
   - グラフ定義が複雑になると可読性が下がる
   - LangChainのバージョンアップに追従する必要がある

2. Claude Agent SDK（Anthropic）
   特徴：Anthropicが提供するエージェント構築SDK。Claudeモデルとの
         統合に最適化されており、ツール使用やマルチターンの対話を
         シンプルなAPIで実現できる。
   強み：
   - Claudeモデルの能力（長文理解、指示追従）を最大限に活用できる
   - APIがシンプルで学習コストが低い
   - ツール定義が直感的（関数定義をそのままツールとして登録）
   - 安全性・ガードレール機能が組み込まれている
   - 拡張されたthinking機能による推論の透明性
   弱み：
   - Claudeモデル専用であり、他のLLMとの組み合わせが制限される
   - エコシステムがLangChainほど成熟していない
   - 複雑なマルチエージェント構成は自前で設計する必要がある

3. CrewAI
   特徴：「AI Crew（チーム）」のメタファーで複数エージェントの協調を
         定義するフレームワーク。エージェントに役割・目標・背景を設定し、
         タスクを割り当てる宣言的なアプローチが特徴。
   強み：
   - 役割ベースの直感的な設計（人間のチーム編成のように定義できる）
   - 宣言的な記述でマルチエージェントの構成が容易
   - デリゲーション（委任）機能によるエージェント間の動的なタスク委譲
   - LLMに依存しない設計（OpenAI、Anthropic、ローカルLLM等を選択可能）
   弱み：
   - 細かいフロー制御がLangGraphほど柔軟ではない
   - 内部の動作が抽象化されすぎて、デバッグが難しい場合がある
   - 大規模なエージェント構成ではパフォーマンスの課題がある
```

</details>

### 問題2：フレームワーク選定の判断基準

以下のプロジェクト要件に対して、最適なフレームワーク（LangGraph / Claude Agent SDK / CrewAI / フレームワーク不要）を選び、理由を説明してください。

| プロジェクト | 要件 |
|-------------|------|
| P1 | Claudeを使った社内文書の質問応答チャットボット。ツールとしてドキュメント検索と社内DB参照を使用 |
| P2 | 5人の専門家エージェント（法務・財務・技術・マーケ・人事）がそれぞれ分析レポートを作成し、統合するM&Aデューデリジェンスシステム |
| P3 | ユーザーの質問に対して、Web検索→情報整理→回答生成の3ステップで処理するRAGシステム。途中でユーザー確認を挟む |
| P4 | 簡単なAPIリクエストを送って結果をフォーマットして返すだけのスクリプト |
| P5 | 複数のLLM（GPT-4、Claude、Gemini）を状況に応じて切り替えながら、コード生成→レビュー→修正のサイクルを回す開発支援ツール |

**期待される出力例：**

```
P1: Claude Agent SDK
理由：Claudeモデル専用であり、ツール使用のシンプルな構成に最適。
```

<details>
<summary>解答例</summary>

```
P1: Claude Agent SDK
理由：Claudeモデルのみを使用し、ツール（検索・DB参照）が明確に定義
されている。Claude Agent SDKのシンプルなツール定義機能で十分対応可能。
複雑なワークフローは不要なため、軽量なSDKが最適。

P2: CrewAI
理由：5つの専門家エージェントに役割・目標を設定してチームとして
協調させるユースケースは、CrewAIの「Crew」メタファーに完全に合致する。
各エージェントの役割定義が宣言的に記述でき、開発効率が高い。
タスクの委任（デリゲーション）機能も活用できる。

P3: LangGraph
理由：3ステップの処理フローに加え、途中でHuman-in-the-Loopを挟む
要件がある。LangGraphのステートグラフで明示的にフローを定義し、
チェックポイント機能でユーザー確認待ちの状態を管理できる。
条件分岐（ユーザーがOKした場合/修正を求めた場合）も自然に表現可能。

P4: フレームワーク不要
理由：単純なAPI呼び出しとデータ整形のみであり、エージェントの自律的な
判断やツール使用が不要。通常のPythonスクリプトとrequestsライブラリで
十分対応でき、フレームワークの導入はオーバーエンジニアリングとなる。

P5: LangGraph（またはCrewAI）
理由：複数LLMの動的切り替えが必要なため、特定モデルに依存しない
フレームワークが必須。LangGraphならLLMの切り替えロジックをノードとして
定義し、コード生成→レビュー→修正のループをグラフで明示的に表現できる。
CrewAIも対応可能だが、修正ループの細かい制御はLangGraphが優位。
```

</details>

### 問題3：フレームワークの構成要素

LangGraphの主要な構成要素（State、Node、Edge、Conditional Edge）について、それぞれの役割を説明し、以下のシナリオをステートグラフとして表現してください。

**シナリオ：**
```
お問い合わせ自動応答システム
1. ユーザーのメッセージを受け取る
2. メッセージを分類する（質問/クレーム/その他）
3a. 質問の場合 → FAQデータベースを検索して回答
3b. クレームの場合 → 謝罪文を生成し、担当者にエスカレーション
3c. その他の場合 → 一般的な応答を生成
4. 応答をユーザーに返す
```

**期待される出力例：**

```
State: メッセージ内容、分類結果、応答テキスト、エスカレーションフラグ
Node: receive_message, classify, ...
Edge: receive_message → classify
Conditional Edge: classify → (質問: search_faq, クレーム: handle_complaint, ...)
```

<details>
<summary>ヒント</summary>

- Stateはワークフロー全体で共有されるデータ構造です
- Nodeは処理の各ステップ（関数）です
- Edgeはノード間の遷移を表します
- Conditional Edgeは条件によって遷移先が変わるエッジです

</details>

<details>
<summary>解答例</summary>

```
■ 各構成要素の役割：

State（状態）：
  ワークフロー全体で共有されるデータ構造。各ノードが読み書きする。
  グラフの実行中に状態が更新されていくことで処理が進行する。

Node（ノード）：
  処理の単位。Stateを受け取り、処理を行い、Stateを更新して返す関数。
  LLM呼び出し、ツール実行、データ変換などを担当する。

Edge（エッジ）：
  ノード間の遷移を定義する。あるノードの処理完了後、次にどのノードを
  実行するかを決定する。無条件の遷移を表す。

Conditional Edge（条件付きエッジ）：
  Stateの内容に基づいて、遷移先を動的に決定するエッジ。
  分岐ロジック（if/switch相当）をグラフ上で表現する。

■ ステートグラフ設計：

【State定義】
  class InquiryState(TypedDict):
      message: str           # ユーザーのメッセージ
      category: str          # 分類結果（question/complaint/other）
      response: str          # 応答テキスト
      escalation: bool       # エスカレーションフラグ
      faq_results: list      # FAQ検索結果

【Node定義】
  1. receive_message  : ユーザーメッセージを受け取りStateに格納
  2. classify         : メッセージを分類（question/complaint/other）
  3. search_faq       : FAQデータベースを検索し回答を生成
  4. handle_complaint : 謝罪文を生成し、エスカレーションフラグをTrueに設定
  5. general_response : 一般的な応答を生成
  6. send_response    : 応答をユーザーに返す

【Edge定義】
  receive_message → classify              （無条件遷移）
  search_faq      → send_response         （無条件遷移）
  handle_complaint → send_response        （無条件遷移）
  general_response → send_response        （無条件遷移）

【Conditional Edge定義】
  classify → route_by_category:
    if category == "question"  → search_faq
    if category == "complaint" → handle_complaint
    if category == "other"     → general_response

【グラフ図】

  [START]
     │
     ▼
  receive_message
     │
     ▼
  classify
     │
     ├─ question ──→ search_faq ────────┐
     │                                   │
     ├─ complaint ─→ handle_complaint ──┤
     │                                   │
     └─ other ─────→ general_response ──┤
                                         │
                                         ▼
                                   send_response
                                         │
                                         ▼
                                       [END]
```

</details>

### 問題4：Claude Agent SDKを使ったエージェント実装

Claude Agent SDKを使用して、「天気情報エージェント」を実装してください。以下の要件を満たすコードを記述してください。

**要件：**
- ユーザーが都市名を指定すると、天気情報を返すエージェント
- ツールとして `get_weather` 関数を定義（実際のAPIは呼ばず、ダミーデータを返す）
- エージェントが自然言語で天気の概要を説明する

**期待される出力例：**

```python
# 実行例
# ユーザー: 「東京の天気を教えて」
# エージェント: 「東京の現在の天気は晴れで、気温は22℃です。
#               湿度は45%で、過ごしやすい天気ですね。」
```

<details>
<summary>ヒント</summary>

- Claude Agent SDKではツールを関数として定義し、エージェントに登録します
- ツール関数にはdocstringで説明を記述すると、エージェントが適切に使用できます
- ダミーデータは辞書で都市名→天気情報のマッピングを作成しましょう

</details>

<details>
<summary>解答例</summary>

```python
import anthropic
import json

# Anthropicクライアントの初期化
client = anthropic.Anthropic()
MODEL_NAME = "claude-sonnet-4-20250514"

# --- ツール定義 ---

# ダミーの天気データ
WEATHER_DATA = {
    "東京": {"weather": "晴れ", "temp": 22, "humidity": 45, "wind": "北東 3m/s"},
    "大阪": {"weather": "曇り", "temp": 20, "humidity": 60, "wind": "南西 2m/s"},
    "札幌": {"weather": "雪", "temp": -2, "humidity": 70, "wind": "北 5m/s"},
    "福岡": {"weather": "雨", "temp": 18, "humidity": 80, "wind": "南 4m/s"},
    "名古屋": {"weather": "晴れ", "temp": 21, "humidity": 50, "wind": "東 2m/s"},
}

# ツールのスキーマ定義
tools = [
    {
        "name": "get_weather",
        "description": "指定された都市の現在の天気情報を取得します。"
                       "都市名は日本語で指定してください。",
        "input_schema": {
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "天気を取得したい都市名（例：東京、大阪）"
                }
            },
            "required": ["city"]
        }
    }
]

def get_weather(city: str) -> str:
    """指定された都市の天気情報を返す（ダミーデータ）"""
    if city in WEATHER_DATA:
        data = WEATHER_DATA[city]
        return json.dumps({
            "city": city,
            "weather": data["weather"],
            "temperature": f"{data['temp']}℃",
            "humidity": f"{data['humidity']}%",
            "wind": data["wind"],
            "status": "success"
        }, ensure_ascii=False)
    else:
        return json.dumps({
            "city": city,
            "status": "error",
            "message": f"{city}の天気データは見つかりませんでした。"
                       f"対応都市：{', '.join(WEATHER_DATA.keys())}"
        }, ensure_ascii=False)

# --- ツール実行処理 ---

def process_tool_call(tool_name: str, tool_input: dict) -> str:
    """ツール呼び出しを処理して結果を返す"""
    if tool_name == "get_weather":
        return get_weather(tool_input["city"])
    else:
        return json.dumps({"error": f"未知のツール: {tool_name}"})

# --- エージェントループ ---

def run_weather_agent(user_message: str) -> str:
    """
    天気情報エージェントを実行する。
    ツール使用が不要になるまでループし、最終的な応答を返す。
    """
    messages = [
        {"role": "user", "content": user_message}
    ]

    system_prompt = (
        "あなたは親切な天気情報アシスタントです。"
        "ユーザーが都市名を指定したら、get_weatherツールを使って"
        "天気情報を取得し、自然な日本語で分かりやすく伝えてください。"
        "気温に応じた服装のアドバイスも一言添えてください。"
    )

    # エージェントループ（ツール使用がなくなるまで繰り返す）
    while True:
        response = client.messages.create(
            model=MODEL_NAME,
            max_tokens=1024,
            system=system_prompt,
            tools=tools,
            messages=messages,
        )

        # stop_reasonがend_turnなら最終応答
        if response.stop_reason == "end_turn":
            # テキストブロックを抽出して返す
            for block in response.content:
                if hasattr(block, "text"):
                    return block.text
            return ""

        # tool_useがあればツールを実行
        if response.stop_reason == "tool_use":
            # アシスタントの応答をメッセージ履歴に追加
            messages.append({
                "role": "assistant",
                "content": response.content
            })

            # 各ツール呼び出しを処理
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    result = process_tool_call(block.name, block.input)
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": result,
                    })

            # ツール結果をメッセージ履歴に追加
            messages.append({
                "role": "user",
                "content": tool_results
            })

# --- 実行 ---
if __name__ == "__main__":
    # テスト実行
    queries = [
        "東京の天気を教えて",
        "大阪と札幌の天気を比較して",
        "沖縄の天気はどう？",
    ]

    for query in queries:
        print(f"ユーザー: {query}")
        answer = run_weather_agent(query)
        print(f"エージェント: {answer}")
        print("-" * 50)
```

<!-- 解説：
     - tools リストでツールのスキーマを定義し、APIに渡す
     - エージェントループ（agentic loop）でツール使用→結果返却を繰り返す
     - stop_reason が "end_turn" になったら最終応答として返す
     - stop_reason が "tool_use" の間はツールを実行してループを継続
     - 存在しない都市を指定された場合のエラーハンドリングも実装 -->

</details>

### 問題5：LangGraphのステートグラフ設計

LangGraphを使用して、以下の「レポート作成ワークフロー」をステートグラフとして設計・実装してください。

**ワークフロー：**
1. ユーザーがテーマを入力
2. リサーチエージェントが情報を収集
3. ライティングエージェントがレポートを作成
4. レビューエージェントが品質をチェック
   - スコアが80点以上 → 完成
   - スコアが80点未満 → フィードバック付きでライティングに差し戻し（最大3回）
5. 完成したレポートを出力

以下の項目を記述してください。
1. Stateクラスの定義
2. 各ノード関数の実装
3. グラフの構築（エッジとConditional Edgeの定義）
4. 実行コード

**期待される出力例：**

```python
# State定義
class ReportState(TypedDict):
    theme: str
    research_data: str
    ...

# ノード関数
def research(state: ReportState) -> ReportState:
    ...

# グラフ構築
graph = StateGraph(ReportState)
graph.add_node("research", research)
...
```

<details>
<summary>ヒント</summary>

- StateはTypedDictで定義し、全ノードで共有するデータを含めます
- 各ノード関数はStateを受け取り、更新されたStateを返します
- Conditional Edgeではレビュースコアとリトライ回数で分岐します
- LLMの実際の呼び出しの代わりに、ダミーの処理でも構いません

</details>

<details>
<summary>解答例</summary>

```python
from typing import TypedDict
from langgraph.graph import StateGraph, END

# ============================================================
# 1. State クラスの定義
# ============================================================
class ReportState(TypedDict):
    theme: str              # レポートのテーマ
    research_data: str      # リサーチ結果
    draft: str              # レポート草稿
    review_score: int       # レビュースコア（0-100）
    review_feedback: str    # レビューフィードバック
    retry_count: int        # リトライ回数
    final_report: str       # 最終レポート
    status: str             # ステータス（processing/completed/max_retries）

# ============================================================
# 2. 各ノード関数の実装
# ============================================================

def research(state: ReportState) -> dict:
    """リサーチノード：テーマに基づいて情報を収集する"""
    theme = state["theme"]
    # 実際にはLLM + 検索ツールで情報収集を行う
    # ここではダミーデータを返す
    research_data = (
        f"【{theme}に関するリサーチ結果】\n"
        f"1. {theme}の市場規模は年間約1兆円で、年率15%で成長中\n"
        f"2. 主要プレイヤーはA社、B社、C社の3社\n"
        f"3. 最新トレンドとしてAI活用が加速している\n"
        f"4. 課題として人材不足とコスト高が挙げられる\n"
        f"5. 今後5年間でさらなる成長が見込まれる"
    )
    print(f"[リサーチ] {theme} の情報収集完了")
    return {"research_data": research_data}


def write_report(state: ReportState) -> dict:
    """ライティングノード：リサーチ結果を基にレポートを作成する"""
    theme = state["theme"]
    research_data = state["research_data"]
    retry_count = state.get("retry_count", 0)
    feedback = state.get("review_feedback", "")

    # フィードバックがある場合はそれを反映
    if feedback:
        print(f"[ライティング] フィードバックを反映して修正中（{retry_count}回目）")
        draft = (
            f"# {theme} レポート（修正版 v{retry_count + 1}）\n\n"
            f"## 概要\n{research_data}\n\n"
            f"## 分析\n前回のフィードバック「{feedback}」を反映し、"
            f"より詳細な分析を追加しました。\n\n"
            f"## 結論\n{theme}は今後も成長が見込まれる分野です。"
        )
    else:
        print(f"[ライティング] 初稿を作成中")
        draft = (
            f"# {theme} レポート\n\n"
            f"## 概要\n{research_data}\n\n"
            f"## 分析\n上記データに基づく分析です。\n\n"
            f"## 結論\n{theme}は注目すべき分野です。"
        )

    return {"draft": draft}


def review(state: ReportState) -> dict:
    """レビューノード：レポートの品質をチェックする"""
    draft = state["draft"]
    retry_count = state.get("retry_count", 0)

    # ダミーのスコアリング（リトライ回数が増えるとスコアが上がる想定）
    # 実際にはLLMがレポートの品質を評価する
    base_score = 65
    score = min(base_score + (retry_count * 10), 95)

    if score >= 80:
        feedback = "品質基準を満たしています。"
        print(f"[レビュー] スコア: {score}/100 → 合格")
    else:
        feedback = "分析セクションの深掘りが不足しています。具体的な数値や事例を追加してください。"
        print(f"[レビュー] スコア: {score}/100 → 差し戻し（フィードバック: {feedback}）")

    return {
        "review_score": score,
        "review_feedback": feedback,
        "retry_count": retry_count + 1,
    }


def finalize(state: ReportState) -> dict:
    """最終化ノード：レポートを完成させる"""
    print(f"[最終化] レポート完成（スコア: {state['review_score']}/100）")
    return {
        "final_report": state["draft"],
        "status": "completed",
    }


def handle_max_retries(state: ReportState) -> dict:
    """最大リトライ到達ノード：最善の版を採用して完了する"""
    print(f"[最大リトライ] {state['retry_count']}回修正しましたが基準未達。現在の版を採用します。")
    return {
        "final_report": state["draft"] + "\n\n※注意：品質基準を完全には満たしていません。",
        "status": "max_retries",
    }

# 3. ルーティング関数
# ============================================================

def route_after_review(state: ReportState) -> str:
    """レビュー後のルーティング：スコアとリトライ回数で分岐"""
    if state["review_score"] >= 80:
        return "finalize"           # 合格 → 最終化へ
    elif state["retry_count"] >= 3:
        return "handle_max_retries" # 最大リトライ到達
    else:
        return "write_report"       # 差し戻し → 再ライティング

# ============================================================
# 4. グラフの構築
# ============================================================

def build_report_graph():
    """レポート作成ワークフローのグラフを構築する"""
    graph = StateGraph(ReportState)

    # ノードの追加
    graph.add_node("research", research)
    graph.add_node("write_report", write_report)
    graph.add_node("review", review)
    graph.add_node("finalize", finalize)
    graph.add_node("handle_max_retries", handle_max_retries)

    # エッジの定義（無条件遷移）
    graph.set_entry_point("research")           # 開始ノード
    graph.add_edge("research", "write_report")  # リサーチ → ライティング
    graph.add_edge("write_report", "review")    # ライティング → レビュー
    graph.add_edge("finalize", END)             # 最終化 → 終了
    graph.add_edge("handle_max_retries", END)   # 最大リトライ → 終了

    # 条件付きエッジ（レビュー後の分岐）
    graph.add_conditional_edges(
        "review",                    # 分岐元ノード
        route_after_review,          # ルーティング関数
        {                            # 遷移先マッピング
            "finalize": "finalize",
            "handle_max_retries": "handle_max_retries",
            "write_report": "write_report",
        }
    )

    return graph.compile()

# ============================================================
# 5. 実行
# ============================================================

if __name__ == "__main__":
    # グラフをコンパイル
    app = build_report_graph()

    # 初期状態の設定
    initial_state = {
        "theme": "生成AIの企業活用",
        "research_data": "",
        "draft": "",
        "review_score": 0,
        "review_feedback": "",
        "retry_count": 0,
        "final_report": "",
        "status": "processing",
    }

    # 実行
    print("=" * 60)
    print("レポート作成ワークフロー開始")
    print("=" * 60)

    result = app.invoke(initial_state)

    print("=" * 60)
    print(f"ステータス: {result['status']}")
    print(f"リトライ回数: {result['retry_count']}")
    print(f"最終スコア: {result['review_score']}/100")
    print("=" * 60)
    print("【最終レポート】")
    print(result["final_report"])
```
