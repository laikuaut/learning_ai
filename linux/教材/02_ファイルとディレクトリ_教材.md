# 第2章：ファイルとディレクトリ

## 2.1 絶対パスと相対パス

Linuxではファイルやディレクトリの場所を**パス（path）**で指定する。

### 絶対パス

ルート（`/`）から始まる完全なパス。どこにいても同じ場所を指す。

```bash
/home/tanaka/documents/report.txt
/etc/nginx/nginx.conf
/var/log/syslog
```

### 相対パス

現在のディレクトリからの相対的なパス。

```bash
# 現在 /home/tanaka にいるとき
documents/report.txt        # → /home/tanaka/documents/report.txt
./documents/report.txt      # 同上（.は現在のディレクトリ）
../suzuki/memo.txt          # → /home/suzuki/memo.txt（..は一つ上）
../../etc/hosts             # → /etc/hosts
```

### 特殊なパス記号

| 記号 | 意味 | 例 |
|------|------|-----|
| `/` | ルートディレクトリ | `cd /` |
| `~` | ホームディレクトリ | `cd ~` = `cd /home/ユーザー名` |
| `.` | 現在のディレクトリ | `./script.sh`（現在地のスクリプト実行） |
| `..` | 一つ上のディレクトリ | `cd ..` |
| `-` | 直前のディレクトリ | `cd -` |

```bash
$ pwd
/home/tanaka/projects

$ cd ~
$ pwd
/home/tanaka

$ cd -
$ pwd
/home/tanaka/projects    # 直前のディレクトリに戻った
```

## 2.3 ls - ファイル一覧を表示

### 基本的な使い方

```bash
$ ls
Desktop  Documents  Downloads  Music  Pictures  Videos

$ ls /etc
adduser.conf  alternatives  apt  bash.bashrc  cron.d  ...
```

### 主要なオプション

```bash
# -l: 詳細表示（ロングフォーマット）
$ ls -l
total 32
drwxr-xr-x 2 tanaka tanaka 4096 Mar 15 10:00 Desktop
drwxr-xr-x 3 tanaka tanaka 4096 Mar 20 14:30 Documents
-rw-r--r-- 1 tanaka tanaka  256 Mar 18 09:15 memo.txt

# 各フィールドの意味：
# drwxr-xr-x  2  tanaka  tanaka  4096  Mar 15 10:00  Desktop
# │            │  │       │       │     │              │
# パーミッション リンク数 所有者 グループ サイズ  更新日時  ファイル名

# -a: 隠しファイルも表示
$ ls -a
.  ..  .bashrc  .config  .ssh  Desktop  Documents

# -la: 詳細 + 隠しファイル（最もよく使う組み合わせ）
$ ls -la
total 52
drwxr-xr-x 8 tanaka tanaka 4096 Mar 20 15:00 .
drwxr-xr-x 3 root   root   4096 Mar  1 00:00 ..
-rw------- 1 tanaka tanaka 1245 Mar 20 14:00 .bash_history
-rw-r--r-- 1 tanaka tanaka  220 Mar  1 00:00 .bash_logout
-rw-r--r-- 1 tanaka tanaka 3771 Mar  1 00:00 .bashrc
drwxr-xr-x 2 tanaka tanaka 4096 Mar 15 10:00 Desktop

# -h: サイズを人間が読みやすい形式で表示
$ ls -lh
-rw-r--r-- 1 tanaka tanaka 1.2K Mar 18 09:15 memo.txt
-rw-r--r-- 1 tanaka tanaka 2.5M Mar 19 11:00 photo.jpg
-rw-r--r-- 1 tanaka tanaka 150M Mar 20 08:00 backup.tar.gz

# -t: 更新日時順にソート
$ ls -lt

# -r: 逆順ソート
$ ls -ltr    # 古い順に表示（最新が最後）

# -S: ファイルサイズ順にソート
$ ls -lS

# -R: 再帰的に表示（サブディレクトリの中も表示）
$ ls -R
```

### ワイルドカードとの組み合わせ

```bash
# すべてのtxtファイル
$ ls *.txt
memo.txt  notes.txt  readme.txt

# 名前が「log」で始まるファイル
$ ls log*

# 1文字のファイル名
$ ls ?

# 特定の拡張子
$ ls *.{txt,md,log}
```

## 2.4 cd - ディレクトリの移動

```bash
# 絶対パスで移動
$ cd /var/log

# 相対パスで移動
$ cd Documents

# ホームディレクトリに戻る
$ cd
$ cd ~

# 一つ上に移動
$ cd ..

# 二つ上に移動
$ cd ../..

# 直前のディレクトリに戻る
$ cd -
/home/tanaka/Documents    # 直前の場所が表示される
```

## 2.5 mkdir - ディレクトリの作成

```bash
# ディレクトリを作成
$ mkdir myproject
$ ls
myproject

# 複数のディレクトリを一度に作成
$ mkdir dir1 dir2 dir3

# 深い階層を一度に作成（-p オプション）
$ mkdir -p projects/web/frontend/src
# -pがないと、途中のディレクトリが存在しない場合エラーになる

# 作成と同時にパーミッション指定
$ mkdir -m 755 shared_dir
```

---

## 2.6 rmdir - 空ディレクトリの削除

```bash
# 空のディレクトリを削除
$ rmdir empty_dir

# 中にファイルがあるとエラー
$ rmdir mydir
rmdir: failed to remove 'mydir': Directory not empty

# 空のディレクトリなら -p で階層ごと削除
$ rmdir -p a/b/c    # c, b, a を順に削除（すべて空の場合）
```

> **ヒント**: 中身があるディレクトリを削除したい場合は `rm -r` を使う（後述）。

## 2.7 touch - ファイル作成 / タイムスタンプ更新

```bash
# 空のファイルを作成
$ touch newfile.txt
$ ls -l newfile.txt
-rw-r--r-- 1 tanaka tanaka 0 Mar 20 15:30 newfile.txt

# 複数ファイルを一度に作成
$ touch file1.txt file2.txt file3.txt

# 既存ファイルのタイムスタンプを更新
$ touch existingfile.txt    # 中身は変わらない、日時だけ更新

# 特定の日時を指定
$ touch -t 202401011200 file.txt    # 2024年1月1日12:00に設定
```

## 2.8 cp - ファイル / ディレクトリのコピー

```bash
# ファイルをコピー
$ cp original.txt copy.txt

# 別のディレクトリにコピー
$ cp memo.txt ~/backup/

# 別のディレクトリにリネームしてコピー
$ cp memo.txt ~/backup/memo_backup.txt

# ディレクトリごとコピー（-r: recursive）
$ cp -r myproject myproject_backup

# 上書き前に確認（-i: interactive）
$ cp -i file.txt /tmp/
cp: overwrite '/tmp/file.txt'? y

# パーミッションやタイムスタンプを保持してコピー（-p）
$ cp -p important.txt backup/

# -a: アーカイブモード（-rと-pの組み合わせ + 他）
$ cp -a /var/www/ /backup/www/

# 進捗表示付き（大きなファイルの場合）
$ cp -v largefile.iso /backup/
'largefile.iso' -> '/backup/largefile.iso'
```

## 2.9 mv - ファイルの移動 / リネーム

```bash
# ファイルの移動
$ mv file.txt ~/Documents/

# ファイル名の変更（リネーム）
$ mv old_name.txt new_name.txt

# ディレクトリの移動
$ mv myproject ~/workspace/

# ディレクトリ名の変更
$ mv old_dir new_dir

# 上書き前に確認
$ mv -i file.txt /tmp/

# 複数ファイルを一括移動
$ mv *.txt ~/Documents/
```

## 2.10 rm - ファイル / ディレクトリの削除

### 基本的な使い方

```bash
# ファイルを削除
$ rm file.txt

# 確認付き削除（-i）
$ rm -i file.txt
rm: remove regular file 'file.txt'? y

# 複数ファイルを削除
$ rm file1.txt file2.txt file3.txt

# ワイルドカードで削除
$ rm *.log
```

### ディレクトリの削除

```bash
# ディレクトリとその中身を再帰的に削除（-r）
$ rm -r myproject

# 確認なしで強制削除（-f）
$ rm -rf old_backup
```

> **警告**: `rm -rf /` は絶対に実行してはならない。システム全体が削除される。現代のLinuxでは `--no-preserve-root` がないと実行されないが、`rm -rf ~` でホームディレクトリが消える事故は実際に起きている。

### 安全なrm の使い方

```bash
# 削除前に必ず確認する習慣をつける
$ rm -i important_file.txt

# 削除対象を事前に確認
$ ls target_dir/
file1.txt  file2.txt  file3.txt
$ rm -r target_dir/

# ゴミ箱を使う（trash-cliパッケージ）
$ trash-put file.txt
$ trash-list
$ trash-restore
```

## 2.11 tree - ディレクトリ構造をツリー表示

```bash
# インストール（入っていない場合）
$ sudo apt install tree    # Debian/Ubuntu
$ sudo yum install tree    # CentOS/RHEL

# 基本表示
$ tree
.
├── Documents
│   ├── report.txt
│   └── notes.md
├── Downloads
│   └── package.tar.gz
└── scripts
    ├── backup.sh
    └── deploy.sh

3 directories, 5 files

# 深さを制限（-L）
$ tree -L 1
.
├── Documents
├── Downloads
└── scripts

3 directories, 0 files

# ディレクトリのみ表示（-d）
$ tree -d
.
├── Documents
├── Downloads
└── scripts

# ファイルサイズを表示（-h）
$ tree -h
.
├── [4.0K]  Documents
│   ├── [1.2K]  report.txt
│   └── [ 856]  notes.md
└── [4.0K]  scripts
    ├── [ 512]  backup.sh
    └── [ 384]  deploy.sh

# 隠しファイルも表示（-a）
$ tree -a
```

## 2.12 ホームディレクトリと隠し（ドット）ファイル

### ホームディレクトリ

各ユーザーに割り当てられた個人用ディレクトリ。通常は `/home/ユーザー名/` にある。

```bash
$ echo $HOME
/home/tanaka

$ cd ~
$ pwd
/home/tanaka

# 他のユーザーのホームディレクトリ
$ ls /home/
tanaka  suzuki  yamada

# rootのホームディレクトリ
$ echo ~root
/root
```

### 隠し（ドット）ファイル

ファイル名が `.`（ドット）で始まるファイルは隠しファイルとなり、通常の `ls` では表示されない。設定ファイルに多く使われる。

```bash
$ ls -la ~
drwxr-xr-x 20 tanaka tanaka 4096 Mar 20 15:00 .
drwxr-xr-x  3 root   root   4096 Mar  1 00:00 ..
-rw-------  1 tanaka tanaka 5432 Mar 20 14:00 .bash_history
-rw-r--r--  1 tanaka tanaka  220 Mar  1 00:00 .bash_logout
-rw-r--r--  1 tanaka tanaka 3771 Mar  1 00:00 .bashrc
drwx------  3 tanaka tanaka 4096 Mar  5 10:00 .config
-rw-r--r--  1 tanaka tanaka  807 Mar  1 00:00 .profile
drwx------  2 tanaka tanaka 4096 Mar  3 09:00 .ssh
-rw-------  1 tanaka tanaka 8192 Mar 20 12:00 .viminfo
```

### 重要な隠しファイル/ディレクトリ

| ファイル | 用途 |
|----------|------|
| `~/.bashrc` | bashの設定（エイリアス、プロンプト等） |
| `~/.bash_profile` | ログイン時の設定 |
| `~/.bash_history` | コマンド履歴 |
| `~/.ssh/` | SSH鍵、設定ファイル |
| `~/.config/` | アプリケーションの設定 |
| `~/.vimrc` | Vimの設定 |
| `~/.gitconfig` | Gitの設定 |

## 2.13 実践テクニック

### コマンドの組み合わせ例

```bash
# ディレクトリを作成して即座に移動
$ mkdir newproject && cd newproject

# プロジェクトのひな型を一度に作成
$ mkdir -p myapp/{src,tests,docs,config}
$ tree myapp
myapp
├── config
├── docs
├── src
└── tests

# バックアップを取ってからファイルを編集
$ cp config.txt config.txt.bak
$ nano config.txt

# 日付入りのバックアップ
$ cp important.txt "important_$(date +%Y%m%d).txt"
$ ls
important.txt  important_20240320.txt
```

## 2.14 まとめ

| コマンド | 機能 | よく使うオプション |
|----------|------|--------------------|
| `pwd` | 現在のディレクトリ表示 | - |
| `ls` | ファイル一覧 | `-la`, `-lh`, `-lt` |
| `cd` | ディレクトリ移動 | `-`（直前に戻る） |
| `mkdir` | ディレクトリ作成 | `-p`（深い階層） |
| `rmdir` | 空ディレクトリ削除 | - |
| `touch` | ファイル作成 / タイムスタンプ更新 | - |
| `cp` | コピー | `-r`（ディレクトリ）, `-i`（確認） |
| `mv` | 移動 / リネーム | `-i`（確認） |
| `rm` | 削除 | `-r`（再帰）, `-i`（確認） |
| `tree` | ツリー表示 | `-L 深さ`, `-d`（ディレクトリのみ） |
