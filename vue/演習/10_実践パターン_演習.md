## 演習10-1：provide / inject でテーマを共有（基本）

### 問題

`provide` / `inject` を使って、アプリ全体にテーマ（ライト/ダーク）を共有してください。

**要件:**
1. 祖先コンポーネントでテーマ状態と切替関数を `provide`
2. 深い階層の子孫コンポーネントで `inject` して使用
3. テーマに応じて背景色・文字色が切り替わる
4. `readonly` でデータを保護し、変更関数のみ公開

<details>
<summary>ヒント</summary>

- `provide('key', value)` で値を提供し、`inject('key')` で取得します
- `readonly` でラップすると、子孫コンポーネントから直接変更できなくなります
- 変更関数も一緒に `provide` するのがベストプラクティスです

</details>

<details>
<summary>解答例</summary>

```vue
<!-- App.vue（祖先）-->
<script setup>
import { ref, provide, readonly } from 'vue'

const theme = ref('light')

function toggleTheme() {
  theme.value = theme.value === 'light' ? 'dark' : 'light'
}

// readonly でデータを保護し、変更関数は別途 provide
provide('theme', readonly(theme))
provide('toggleTheme', toggleTheme)
</script>

<template>
  <div :style="{
    backgroundColor: theme === 'dark' ? '#1a1a2e' : '#ffffff',
    color: theme === 'dark' ? '#e0e0e0' : '#333',
    minHeight: '100vh', padding: '20px'
  }">
    <h1>テーマ共有デモ</h1>
    <button @click="toggleTheme">テーマ切替</button>
    <!-- 深い階層のコンポーネント -->
    <MiddleComponent />
  </div>
</template>
```

```vue
<!-- MiddleComponent.vue（中間層 — provide/inject に関知しない）-->
<template>
  <div style="padding: 20px;">
    <p>中間コンポーネント（provide/injectの処理は不要）</p>
    <DeepChild />
  </div>
</template>
```

```vue
<!-- DeepChild.vue（深い子孫）-->
<script setup>
import { inject } from 'vue'

// inject で祖先から提供された値を取得
const theme = inject('theme', 'light')  // 第2引数はデフォルト値
const toggleTheme = inject('toggleTheme')
</script>

<template>
  <div :style="{
    padding: '16px', borderRadius: '8px',
    backgroundColor: theme === 'dark' ? '#16213e' : '#f0f0f0'
  }">
    <p>深い子孫コンポーネント</p>
    <p>現在のテーマ: {{ theme }}</p>
    <button @click="toggleTheme">ここからもテーマを切替できる</button>
  </div>
</template>
```

**ポイント:**
- props のバケツリレーなしに、深い階層にデータを渡せます
- `readonly` を使うことで、子孫からの不正な変更を防ぎます
- 中間コンポーネントは provide/inject に関知する必要がありません

</details>

## 演習10-2：Teleport でモーダルを作ろう（基本）

### 問題

`<Teleport>` を使って、DOMツリーの外にモーダルをレンダリングしてください。

**要件:**
1. モーダルを `<body>` 直下にテレポート
2. オーバーレイ（半透明背景）クリックで閉じる
3. Escape キーで閉じる
4. 開閉時のトランジションアニメーション

<details>
<summary>ヒント</summary>

- `<Teleport to="body">` で `<body>` 直下にレンダリングします
- キーボードイベントは `onMounted` で `addEventListener` を登録します
- `<Transition>` コンポーネントで開閉アニメーションを付けられます

</details>

<details>
<summary>解答例</summary>

```vue
<script setup>
import { ref, onMounted, onUnmounted } from 'vue'

const isOpen = ref(false)

function open() { isOpen.value = true }
function close() { isOpen.value = false }

// Escapeキーでモーダルを閉じる
function handleKeydown(e) {
  if (e.key === 'Escape' && isOpen.value) close()
}

onMounted(() => {
  document.addEventListener('keydown', handleKeydown)
})

onUnmounted(() => {
  document.removeEventListener('keydown', handleKeydown)
})
</script>

<template>
  <div style="padding: 20px;">
    <h2>Teleport モーダル</h2>
    <button @click="open" style="padding: 10px 20px;">モーダルを開く</button>

    <!--
      Teleport: DOMツリーの外（body直下）にレンダリング
      z-index の問題を回避し、レイアウトに影響しない
    -->
    <Teleport to="body">
      <Transition name="modal">
        <div v-if="isOpen" class="overlay" @click.self="close">
          <div class="modal-content">
            <header style="display: flex; justify-content: space-between; align-items: center; margin-bottom: 16px;">
              <h3>モーダルタイトル</h3>
              <button @click="close" style="border: none; background: none; font-size: 24px; cursor: pointer;">×</button>
            </header>
            <p>このモーダルは body 直下にテレポートされています。</p>
            <p style="color: #888; font-size: 13px; margin-top: 8px;">
              Escapeキーまたはオーバーレイクリックで閉じます。
            </p>
            <footer style="margin-top: 20px; text-align: right;">
              <button @click="close" style="padding: 8px 20px;">閉じる</button>
            </footer>
          </div>
        </div>
      </Transition>
    </Teleport>
  </div>
</template>

<style>
.overlay {
  position: fixed; top: 0; left: 0; width: 100%; height: 100%;
  background: rgba(0,0,0,0.5); display: flex; justify-content: center; align-items: center; z-index: 1000;
}
.modal-content {
  background: white; border-radius: 12px; padding: 24px; min-width: 400px; max-width: 600px;
  box-shadow: 0 20px 60px rgba(0,0,0,0.3);
}
.modal-enter-from, .modal-leave-to { opacity: 0; }
.modal-enter-from .modal-content, .modal-leave-to .modal-content { transform: scale(0.9); }
.modal-enter-active, .modal-leave-active { transition: opacity 0.3s ease; }
.modal-enter-active .modal-content, .modal-leave-active .modal-content { transition: transform 0.3s ease; }
</style>
```

**Teleport の利点:**
- CSS の `overflow: hidden` や `z-index` の問題を回避できます
- コンポーネントの論理的な階層は維持しつつ、DOM上は別の場所にレンダリングします
- モーダル、トースト通知、ツールチップなどに最適です

</details>

## 演習10-3：動的コンポーネント（応用）

### 問題

`<component :is="">` を使ったタブ切り替えUIを作成してください。

**要件:**
1. 3つのタブ（プロフィール / 設定 / 通知）
2. タブクリックで対応するコンポーネントに切り替え
3. `<KeepAlive>` で非表示タブの状態を保持
4. 各タブ内にフォームを配置し、切り替えても入力内容が保持されることを確認

<details>
<summary>ヒント</summary>

- `<component :is="currentComponent" />` で動的にコンポーネントを切り替えます
- `<KeepAlive>` でラップすると、非表示になったコンポーネントの state が保持されます
- `shallowRef` でコンポーネント自体をリアクティブに管理できます

</details>

<details>
<summary>解答例</summary>

```vue
<script setup>
import { ref, shallowRef, markRaw } from 'vue'

// 各タブのコンポーネント（インラインで定義）
const ProfileTab = {
  setup() {
    const name = ref('')
    return { name }
  },
  template: `
    <div style="padding: 16px;">
      <h3>プロフィール</h3>
      <input v-model="name" placeholder="名前を入力..." style="padding: 8px; width: 100%;" />
      <p style="margin-top: 8px; color: #888;">入力中: {{ name }}</p>
      <p style="color: #666; font-size: 13px;">タブを切り替えても入力内容が保持されます（KeepAlive）</p>
    </div>
  `
}

const SettingsTab = {
  setup() {
    const darkMode = ref(false)
    const fontSize = ref(16)
    return { darkMode, fontSize }
  },
  template: `
    <div style="padding: 16px;">
      <h3>設定</h3>
      <label><input type="checkbox" v-model="darkMode" /> ダークモード</label>
      <div style="margin-top: 8px;">
        <label>フォントサイズ: {{ fontSize }}px</label>
        <input type="range" v-model.number="fontSize" min="12" max="24" />
      </div>
    </div>
  `
}

const NotificationsTab = {
  setup() {
    const enabled = ref(true)
    return { enabled }
  },
  template: `
    <div style="padding: 16px;">
      <h3>通知設定</h3>
      <label><input type="checkbox" v-model="enabled" /> 通知を有効にする</label>
      <p style="margin-top: 8px;">通知: {{ enabled ? 'ON' : 'OFF' }}</p>
    </div>
  `
}

const tabs = [
  { label: 'プロフィール', component: markRaw(ProfileTab) },
  { label: '設定', component: markRaw(SettingsTab) },
  { label: '通知', component: markRaw(NotificationsTab) },
]

const activeTab = shallowRef(tabs[0])
</script>

<template>
  <div style="max-width: 500px; margin: 0 auto; padding: 20px;">
    <h2>動的コンポーネント</h2>

    <!-- タブバー -->
    <div style="display: flex; border-bottom: 2px solid #e0e0e0; margin-bottom: 16px;">
      <button
        v-for="tab in tabs" :key="tab.label"
        @click="activeTab = tab"
        :style="{
          padding: '10px 20px', border: 'none', cursor: 'pointer',
          background: 'transparent', fontWeight: activeTab === tab ? 'bold' : 'normal',
          borderBottom: activeTab === tab ? '2px solid #5c6bc0' : '2px solid transparent',
          color: activeTab === tab ? '#5c6bc0' : '#666'
        }"
      >
        {{ tab.label }}
      </button>
    </div>

    <!-- 動的コンポーネント + KeepAlive -->
    <KeepAlive>
      <component :is="activeTab.component" />
    </KeepAlive>
  </div>
</template>
```

**ポイント:**
- `<component :is="">` でコンポーネントを動的に切り替えられます
- `<KeepAlive>` を使わないと、タブ切替時にコンポーネントが破棄され、入力内容が失われます
- `markRaw` はリアクティブ化を防ぐ最適化です（コンポーネント定義はリアクティブにする必要がない）

</details>

## 演習10-4：実践アプリ設計（チャレンジ）

### 問題

以下の要件を満たす「ミニダッシュボード」を設計・実装してください。これまで学んだ知識を総合的に活用します。

**要件:**
1. **provide/inject** でテーマ管理
2. **Teleport** でトースト通知
3. **動的コンポーネント** でタブ切り替え
4. **composable** でデータ取得ロジック再利用
5. タブ：「ユーザー一覧」「統計」「設定」

<details>
<summary>ヒント</summary>

これまでの演習で学んだパターンを組み合わせましょう。
- テーマ: provide/inject（演習10-1）
- 通知: Teleport（演習10-2）
- タブ: 動的コンポーネント（演習10-3）
- データ取得: useFetch composable（演習9-2）

</details>

<details>
<summary>解答例</summary>

```vue
<script setup>
import { ref, provide, readonly, shallowRef, markRaw, onMounted, onUnmounted } from 'vue'

// ========================================
// 1. テーマ管理 (provide/inject)
// ========================================
const theme = ref('light')
function toggleTheme() {
  theme.value = theme.value === 'light' ? 'dark' : 'light'
}
provide('theme', readonly(theme))
provide('toggleTheme', toggleTheme)

// ========================================
// 2. トースト通知システム
// ========================================
const toasts = ref([])
let toastId = 0

function addToast(message, type = 'info') {
  const id = ++toastId
  toasts.value.push({ id, message, type })
  setTimeout(() => {
    toasts.value = toasts.value.filter(t => t.id !== id)
  }, 3000)
}
provide('addToast', addToast)

// ========================================
// 3. タブコンポーネント定義
// ========================================
const UsersTab = markRaw({
  setup() {
    const users = ref([])
    const isLoading = ref(true)
    onMounted(async () => {
      try {
        const res = await fetch('https://jsonplaceholder.typicode.com/users')
        users.value = await res.json()
      } finally {
        isLoading.value = false
      }
    })
    return { users, isLoading }
  },
  template: `
    <div>
      <h3>ユーザー一覧</h3>
      <p v-if="isLoading">読み込み中...</p>
      <div v-else v-for="user in users" :key="user.id"
        style="padding: 12px; margin: 4px 0; background: rgba(128,128,128,0.1); border-radius: 8px;">
        <strong>{{ user.name }}</strong>
        <span style="color: #888; margin-left: 8px;">{{ user.email }}</span>
      </div>
    </div>
  `
})

const StatsTab = markRaw({
  template: `
    <div>
      <h3>統計</h3>
      <div style="display: grid; grid-template-columns: repeat(2, 1fr); gap: 12px;">
        <div style="padding: 20px; background: rgba(92,107,192,0.1); border-radius: 8px; text-align: center;">
          <div style="font-size: 32px; font-weight: bold; color: #5c6bc0;">128</div>
          <div style="color: #888;">アクティブユーザー</div>
        </div>
        <div style="padding: 20px; background: rgba(76,175,80,0.1); border-radius: 8px; text-align: center;">
          <div style="font-size: 32px; font-weight: bold; color: #4caf50;">95%</div>
          <div style="color: #888;">稼働率</div>
        </div>
        <div style="padding: 20px; background: rgba(255,152,0,0.1); border-radius: 8px; text-align: center;">
          <div style="font-size: 32px; font-weight: bold; color: #ff9800;">42</div>
          <div style="color: #888;">本日のリクエスト</div>
        </div>
        <div style="padding: 20px; background: rgba(233,69,96,0.1); border-radius: 8px; text-align: center;">
          <div style="font-size: 32px; font-weight: bold; color: #e94560;">3</div>
          <div style="color: #888;">未解決エラー</div>
        </div>
      </div>
    </div>
  `
})

const SettingsTab = markRaw({
  inject: ['theme', 'toggleTheme', 'addToast'],
  setup() {
    const name = ref('管理者')
    return { name }
  },
  template: `
    <div>
      <h3>設定</h3>
      <div style="margin-bottom: 16px;">
        <label style="display: block; margin-bottom: 4px;">表示名</label>
        <input v-model="name" style="padding: 8px; width: 100%;" />
      </div>
      <div style="margin-bottom: 16px;">
        <label>テーマ: {{ theme }}</label>
        <button @click="toggleTheme" style="margin-left: 8px; padding: 4px 12px;">切替</button>
      </div>
      <button @click="addToast('設定を保存しました', 'success')" style="padding: 8px 16px; background: #5c6bc0; color: white; border: none; border-radius: 6px; cursor: pointer;">
        保存
      </button>
    </div>
  `
})

const tabs = [
  { label: 'ユーザー', icon: '👥', component: UsersTab },
  { label: '統計', icon: '📊', component: StatsTab },
  { label: '設定', icon: '⚙️', component: SettingsTab },
]

const activeTab = shallowRef(tabs[0])

// ========================================
// テーマに応じたスタイル
// ========================================
const themeStyles = {
  light: { bg: '#ffffff', text: '#333333', card: '#f8f9fa' },
  dark: { bg: '#1a1a2e', text: '#e0e0e0', card: '#16213e' },
}
</script>

<template>
  <div :style="{
    backgroundColor: themeStyles[theme].bg,
    color: themeStyles[theme].text,
    minHeight: '100vh', padding: '20px'
  }">
    <div style="max-width: 700px; margin: 0 auto;">
      <h1>ミニダッシュボード</h1>

      <!-- タブバー -->
      <div style="display: flex; gap: 4px; margin-bottom: 20px;">
        <button
          v-for="tab in tabs" :key="tab.label"
          @click="activeTab = tab"
          :style="{
            padding: '10px 16px', border: 'none', borderRadius: '8px', cursor: 'pointer',
            background: activeTab === tab ? '#5c6bc0' : themeStyles[theme].card,
            color: activeTab === tab ? 'white' : themeStyles[theme].text
          }"
        >
          {{ tab.icon }} {{ tab.label }}
        </button>
      </div>

      <!-- 動的コンポーネント -->
      <div :style="{ padding: '20px', background: themeStyles[theme].card, borderRadius: '12px' }">
        <KeepAlive>
          <component :is="activeTab.component" />
        </KeepAlive>
      </div>
    </div>

    <!-- Teleport: トースト通知 -->
    <Teleport to="body">
      <div style="position: fixed; top: 20px; right: 20px; z-index: 1000;">
        <TransitionGroup name="toast">
          <div v-for="toast in toasts" :key="toast.id" :style="{
            padding: '12px 20px', marginBottom: '8px', borderRadius: '8px',
            boxShadow: '0 4px 12px rgba(0,0,0,0.15)', minWidth: '250px',
            background: toast.type === 'success' ? '#e8f5e9' : toast.type === 'error' ? '#ffebee' : '#e3f2fd',
            color: toast.type === 'success' ? '#2e7d32' : toast.type === 'error' ? '#c62828' : '#1565c0',
          }">
            {{ toast.message }}
          </div>
        </TransitionGroup>
      </div>
    </Teleport>
  </div>
</template>

<style>
.toast-enter-from { opacity: 0; transform: translateX(100%); }
.toast-enter-active { transition: all 0.3s ease; }
.toast-leave-active { transition: all 0.3s ease; }
.toast-leave-to { opacity: 0; transform: translateX(100%); }
</style>
```

**設計のポイント:**
- **provide/inject**: テーマとトースト関数をグローバルに共有
- **Teleport**: トースト通知を body 直下にレンダリングしてレイアウトに影響させない
- **動的コンポーネント + KeepAlive**: タブ切り替えで状態を保持
- 各パターンを組み合わせることで、実用的なアプリケーションを構築できます

</details>
