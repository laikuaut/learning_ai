# 実践課題08：デバッグ演習 ─ ECサイト商品ページ ★4

> **難易度**: ★★★★☆（上級）
> **前提知識**: 第1章〜第7章（HTMLの基本構造、テキストとリンク、画像、リストとテーブル、フォーム、セマンティックHTML、メタ情報とhead）
> **課題の種類**: デバッグ
> **学習目標**: バグだらけのHTMLコードを読み解き、構文エラー・セマンティクスの誤り・アクセシビリティの問題を特定して修正する力を養う

---

## 課題の説明

以下の「ECサイト商品ページ」には **12個のバグ** が埋め込まれています。
ファイルをブラウザで開き、表示崩れや意図しない動作を手がかりにすべてのバグを見つけて修正してください。

### バグの種類

- 構文エラー（タグの閉じ忘れ、属性の誤り）
- セマンティクスの誤り（不適切なタグの使い方）
- アクセシビリティの問題（alt属性の欠落など）
- フォームの不備（バリデーション関連）
- メタ情報の問題

### 進め方

1. 下のバグ入りコードを `shop_buggy.html` として保存する
2. ブラウザで開いて表示を確認する
3. 開発者ツール（F12）でエラーや警告を確認する
4. 1つずつバグを修正し、保存してブラウザを再読み込みする
5. すべて正常に表示・動作すれば完了

## 解答例

<details>
<summary>全12個のバグと修正方法</summary>

### バグ1：`<html>` に `lang` 属性がない

```html
<!-- 修正前 -->
<html>
<!-- 修正後 -->
<html lang="ja">
```

`lang` 属性はスクリーンリーダーが正しい言語で読み上げるために必要です。

### バグ2：`<meta charset>` がない

```html
<!-- 修正前（head内にcharset指定なし） -->
<head>
  <title>...</title>
  <meta name="viewport" content="width=device-width">
</head>
<!-- 修正後 -->
<head>
  <meta charset="utf-8">
  <title>...</title>
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
```

`<meta charset="utf-8">` は `<head>` の最初に書きます。日本語の文字化けを防ぎます。

### バグ3：`<div id="header">` → `<header>` に変更

```html
<!-- 修正前 -->
<div id="header">
  ...
</div>
<!-- 修正後 -->
<header>
  ...
</header>
```

### バグ4：`<div id="nav">` → `<nav>` に変更

```html
<!-- 修正前 -->
<div id="nav">
<!-- 修正後 -->
<nav>
```

### バグ5：`<img>` に `alt` 属性がない

```html
<!-- 修正前 -->
<img src="..." width="400" height="300">
<!-- 修正後 -->
<img src="..." alt="ワイヤレスヘッドフォン WH-500 の商品画像" width="400" height="300">
```

`alt` 属性は画像が読み込めないとき、またスクリーンリーダーのために必須です。

### バグ6：廃止された `<font>` タグの使用

```html
<!-- 修正前 -->
<p><font color="red">セール中！20%OFF</font></p>
<!-- 修正後 -->
<p><span style="color: red; font-weight: bold;">セール中！20%OFF</span></p>
<!-- または -->
<p><strong style="color: red;">セール中！20%OFF</strong></p>
```

`<font>` タグはHTML5で廃止されました。スタイルはCSSで指定します。

### バグ7：開始タグと終了タグの不一致

```html
<!-- 修正前 -->
<h3>商品説明</h2>
<!-- 修正後 -->
<h3>商品説明</h3>
```

`<h3>` で開始して `</h2>` で閉じています。開始タグと終了タグは一致させます。

### バグ8：テーブルの見出しセルがない

```html
<!-- 修正前 -->
<tr>
  <td>ドライバーサイズ</td>
  <td>40mm</td>
</tr>
<!-- 修正後 -->
<tr>
  <th>ドライバーサイズ</th>
  <td>40mm</td>
</tr>
```

項目名は `<th>`（見出しセル）にします。スクリーンリーダーがデータの意味を正しく伝えられます。

### バグ9：label と input が関連付けられていない

```html
<!-- 修正前 -->
<label>数量：</label>
<input type="number" value="0" min="0">
<!-- 修正後 -->
<label for="quantity">数量：</label>
<input type="number" id="quantity" name="quantity" value="1" min="1">
```

`<label>` の `for` と `<input>` の `id` を一致させます。また `name` 属性がないとフォーム送信時にデータが含まれません。

### バグ10：数量の初期値と最小値が不適切

```html
<!-- 修正前 -->
<input type="number" value="0" min="0">
<!-- 修正後 -->
<input type="number" id="quantity" name="quantity" value="1" min="1" max="10">
```

商品を0個カートに入れることは意味がありません。初期値1、最小値1が適切です。

### バグ11：見出し階層のスキップ

```html
<!-- 修正前 -->
<h5>とても満足！ ★★★★★</h5>
<!-- 修正後 -->
<h4>とても満足！ ★★★★★</h4>
```

見出し階層は h1 → h2 → h3 → h4 の順に使います。h3 の次にh5を使うと、h4をスキップしてしまい、文書のアウトライン構造が壊れます。

### バグ12：`<div id="footer">` → `<footer>` に変更

```html
<!-- 修正前 -->
<div id="footer">
  ...
</div>
<!-- 修正後 -->
<footer>
  ...
</footer>
```
