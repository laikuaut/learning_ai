# 第6章：セキュリティマネジメント 演習問題

## 基本問題

### 問題6-1：ISMSの基本
以下の問いに答えてください。

(1) ISMSとは何ですか。正式名称と目的を答えてください。
(2) ISMSの認証規格（ISO規格番号）を答えてください。
(3) ISMSのPDCAサイクルの各フェーズの内容を簡潔に説明してください。

<details>
<summary>解答</summary>

(1) ISMS（Information Security Management System / 情報セキュリティマネジメントシステム）は、組織における情報セキュリティを体系的に管理するための枠組みです。技術的対策だけでなく、人的・組織的・物理的対策を含めた包括的なセキュリティ管理を実現することが目的です。

(2) **ISO/IEC 27001** がISMSの認証規格（要求事項）です。ISO/IEC 27002 は実践規範（ガイドライン）です。

(3) PDCAサイクル：
- **Plan（計画）**：リスクアセスメントの実施、セキュリティ対策の計画
- **Do（実行）**：対策の実施、教育・訓練の実施
- **Check（点検）**：内部監査、有効性の評価、マネジメントレビュー
- **Act（改善）**：是正処置、予防処置、継続的改善

</details>

### 問題6-2：リスクアセスメント
リスクアセスメントの3つのステップを順番に挙げ、それぞれの内容を説明してください。

<details>
<summary>解答</summary>

1. **リスク特定**：保護すべき情報資産を洗い出し、それぞれに対する脅威と脆弱性を特定する。例：顧客データベースに対する不正アクセスの脅威、パッチ未適用という脆弱性

2. **リスク分析**：特定したリスクの大きさを評価する。定量的分析（年間予想損失額の計算）または定性的分析（高・中・低のランク付け）で行う。リスク値 = 脅威の発生可能性 × 影響度

3. **リスク評価**：分析結果に基づき、組織のリスク受容基準と比較して対応の優先順位を決定する。受容基準を超えるリスクに対して対応策を講じる

</details>

### 問題6-3：リスク対応
リスク対応の4つの選択肢を挙げ、それぞれ具体例を示してください。

<details>
<summary>解答</summary>

| リスク対応 | 説明 | 具体例 |
|-----------|------|--------|
| **リスク低減（軽減）** | 対策を実施してリスクを減らす | ファイアウォールの導入、暗号化の実施、パッチ適用 |
| **リスク回避** | リスクの原因となる活動自体を停止する | 個人情報を扱うサービスの中止、危険な地域への出張禁止 |
| **リスク移転（共有）** | リスクを第三者に移す | サイバー保険への加入、セキュリティ運用のアウトソーシング |
| **リスク受容（保有）** | リスクがある状態を受け入れる | 対策コストが被害額を上回る場合に経営判断として受容 |

</details>

### 問題6-4：インシデント対応
セキュリティインシデント対応の6つのフェーズを順番に答えてください。

<details>
<summary>解答</summary>

1. **準備（Preparation）**：インシデント対応計画の策定、CSIRTの設置、ツール・環境の整備、訓練の実施
2. **検知と分析（Detection & Analysis）**：異常の検知、インシデントの判定、影響範囲の特定、トリアージ
3. **封じ込め（Containment）**：被害拡大の防止（例：感染端末のネットワーク隔離）
4. **根絶（Eradication）**：マルウェアの駆除、侵入経路の遮断、脆弱性の修正
5. **復旧（Recovery）**：システムの復元、サービスの再開、監視の強化
6. **事後対応（Lessons Learned）**：インシデント報告書の作成、原因分析と再発防止策、手順の改善

</details>

## 応用問題

### 問題6-5：リスク分析の実践
以下の情報資産に対するリスク分析を行ってください。

情報資産：顧客データベース（10万件の個人情報）
脅威：SQLインジェクションによる情報漏洩
脆弱性：入力値のサニタイズ不備

(1) 定性的分析（高/中/低）でリスクレベルを評価してください。
(2) 適切なリスク対応を選択し、具体的な対策を提案してください。

<details>
<summary>解答</summary>

**(1) 定性的分析**

| 評価項目 | 評価 | 理由 |
|---------|------|------|
| 情報資産の価値 | **高** | 10万件の個人情報は漏洩時の損害が甚大（賠償、信用失墜） |
| 脅威の発生可能性 | **高** | SQLインジェクションは最も一般的なWeb攻撃の一つ |
| 脆弱性の程度 | **高** | 入力値のサニタイズが不備で、攻撃の成功率が高い |
| **リスクレベル** | **高（緊急対応が必要）** | 全ての要素が「高」のため |

**(2) リスク対応**
**リスク低減（軽減）** を選択します。

具体的な対策：
1. **即時対応**：全SQL文をパラメータ化クエリ（プレースホルダ）に修正
2. **追加防御**：WAF（Web Application Firewall）の導入
3. **検証**：脆弱性診断を実施して修正の確認
4. **予防**：セキュアコーディングガイドラインの策定と開発者教育

</details>

### 問題6-6：BCP/BCM
以下の用語を説明し、具体的な数値例を示してください。

1. RTO（Recovery Time Objective）
2. RPO（Recovery Point Objective）

<details>
<summary>解答</summary>

**1. RTO（目標復旧時間）**
インシデント発生からシステムを復旧させるまでの最大許容時間です。

例：ECサイトのRTO = 4時間
→ システム停止から4時間以内にサービスを復旧させる必要がある。それ以上停止すると事業に重大な影響が出ると判断。

**2. RPO（目標復旧時点）**
インシデント発生時に、どの時点のデータまで復旧させるかの目標値です。

例：ECサイトのRPO = 1時間
→ 最大1時間分のデータ損失を許容する。1時間ごとにバックアップを取得する必要がある。

```
  ←── RPO(1時間) ──→│ 障害発生 │←── RTO(4時間) ──→│
                      │         │                   │
  最後のバックアップ    │  最大1時間│    復旧完了       │
                      │  のデータ │                   │
  ────────────────────┼──損失───┼───────────────────→ 時間
```

RTOとRPOが厳しい（短い）ほど、バックアップの頻度向上や冗長構成が必要となり、コストが増加します。事業への影響度と対策コストのバランスで決定します。

</details>

## チャレンジ問題

### 問題6-7：インシデント対応シナリオ
以下のシナリオに対するインシデント対応を、各フェーズに沿って具体的に述べてください。

> 月曜朝、社員からIT部門に「PCの動作が極端に遅く、見慣れないポップアップが表示される」との報告がありました。調査の結果、ランサムウェアに感染していることが判明しました。

<details>
<summary>解答例</summary>

**① 検知と分析**
- 感染PCを特定し、ランサムウェアの種類を調査
- 感染経路の推定（メール添付ファイル、Web閲覧、VPN脆弱性等）
- 影響範囲の特定：他のPCやサーバに感染が拡大していないか確認
- トリアージ：重要度を判定（個人情報や業務データへの影響）

**② 封じ込め**
- 感染PCを即座にネットワークから隔離（LANケーブル抜線、Wi-Fi無効化）
- 感染拡大の防止：共有フォルダへのアクセスを一時遮断
- 未感染PCのネットワークセグメント分離を検討
- 全社員に注意喚起（不審なファイルを開かない等）

**③ 根絶**
- 感染PCのマルウェアを駆除（または初期化）
- 侵入経路の特定と遮断（例：脆弱なVPN機器のパッチ適用）
- ウイルス対策ソフトのパターンファイルを全PCで最新に更新

**④ 復旧**
- バックアップからデータを復元
- PCの再セットアップとサービスの復旧
- 復旧後の監視強化（再感染の兆候がないか）

**⑤ 事後対応**
- インシデント報告書の作成（時系列、原因、対応内容、影響範囲）
- 個人情報の漏洩可能性がある場合は個人情報保護委員会への報告
- 再発防止策の策定（メールフィルタリング強化、パッチ管理の改善、セキュリティ教育の実施）
- 対応手順の見直しと改善

</details>

### 問題6-8：CSIRT
CSIRTの役割を5つ挙げ、組織内にCSIRTを設置するメリットを述べてください。

<details>
<summary>解答</summary>

**CSIRTの役割：**
1. **インシデント対応**：セキュリティインシデントの検知・分析・封じ込め・復旧を実施
2. **脆弱性情報の収集・分析**：新たな脅威や脆弱性の情報を収集し、組織への影響を評価
3. **社内への注意喚起**：セキュリティに関する情報を社内に周知・啓発
4. **外部組織との連携**：JPCERT/CC、IPA、他組織のCSIRTとの情報共有と協力
5. **教育・訓練**：セキュリティ意識向上のための研修やインシデント対応訓練の実施

**CSIRTを設置するメリット：**
- インシデント発生時の初動対応が迅速化し、被害を最小限に抑えられる
- 専門的な知見に基づいた判断ができ、対応品質が向上する
- 外部組織との連携窓口が明確になり、最新の脅威情報を入手できる
- 平時からの予防的活動（脆弱性管理、教育）により、インシデントの発生自体を減らせる

</details>
