## 課題の要件

1. 以下の前提条件で、複数の生成AIを統合した業務改善プロジェクトの企画・設計書を作成する
   - **企業**: 従業員300名のITソリューション企業
   - **対象業務**: 営業部門の提案書作成プロセス（月間30件の提案書を作成）
   - **課題**: 1件あたり平均12時間かかり、営業担当者の本来業務（顧客訪問）の時間を圧迫
   - **目標**: 提案書作成の工数を75%削減し、品質を維持または向上させる
2. 現状業務フロー（As-Is）と改善後の業務フロー（To-Be）を設計する
3. 使用する生成AIの組み合わせとそれぞれの役割を定義する
4. システム構成図を作成する（各コンポーネントの連携を明示）
5. 導入計画（スケジュール、体制、予算）を策定する
6. 効果測定のKPIと測定方法を設計する
7. リスク管理計画を策定する
8. 倫理・コンプライアンスへの対応策を記載する

<summary>ステップ1：現状業務を分析する（As-Is分析）</summary>

まず現状の提案書作成プロセスを詳細に把握しましょう。

**分析の観点：**
- 各工程にかかる時間（工数）
- 各工程で使用するツール・情報源
- ボトルネック（最も時間がかかる工程）
- 品質のばらつきが発生する工程
- 手作業で非効率な工程

```
工程1：顧客情報収集（2時間）
  └─ 顧客のWebサイト、過去の商談記録、業界情報を手動で収集

工程2：競合調査（3時間）
  └─ 競合製品の情報を手動で調査・比較表作成

工程3：提案書本文作成（4時間）★ボトルネック
  └─ テンプレートをベースに、顧客ごとにカスタマイズ

工程4：資料デザイン（2時間）
  └─ 図表、イメージ画像の作成・配置

工程5：上長レビュー・修正（1時間）
  └─ 内容の正確性、表現の適切さを確認
```

**AIの活用ポイント：**
- 情報収集・整理 → テキスト生成AI + RAG
- 競合調査 → テキスト生成AI（要約・分析）
- 提案書本文 → テキスト生成AI（ドラフト生成）

- レビュー → コード生成AI（チェックリスト自動実行）

**よくある間違い**: 全工程をAIに任せようとすること。人間が介在すべきポイント（顧客固有の関係性、戦略的判断）を明確にしておくことが重要です。

### 改良版の特徴

#### プロンプトチェーンの詳細設計

提案書作成を1つの巨大なプロンプトではなく、段階的なプロンプトチェーンで実行します。

```
プロンプト1：顧客分析
  入力：顧客名、業界、過去の商談記録
  出力：顧客の課題・ニーズの整理（構造化JSON）
     │
     ▼
プロンプト2：ソリューション選定
  入力：顧客課題（プロンプト1出力）+ 自社製品DB（RAG）
  出力：推奨ソリューションと根拠
     │
     ▼
プロンプト3：提案書構成案
  入力：顧客課題 + ソリューション + 過去の成功提案書（RAG）
  出力：提案書の目次と各セクションの概要
     │
     ▼
プロンプト4：各セクション本文生成（並列実行）
  ├─ 4a：課題認識セクション
  ├─ 4b：ソリューション提案セクション
  ├─ 4c：導入計画・スケジュールセクション
  ├─ 4d：費用見積もりセクション
  └─ 4e：会社紹介・実績セクション
     │
     ▼
プロンプト5：統合・整合性チェック
  入力：全セクションの本文
  出力：統合された提案書 + 整合性チェック結果
     │
     ▼
プロンプト6：品質レビュー
  入力：統合された提案書
  出力：修正提案リスト（深刻度付き）
```

**メリット:**
- 各ステップの品質を個別に管理できる
- プロンプト4a〜4eは並列実行でき、処理時間を短縮
- 問題があるステップだけを再実行できる（全体をやり直す必要がない）

     ▼
