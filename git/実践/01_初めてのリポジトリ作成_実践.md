# 実践課題01：初めてのリポジトリ作成 ★1

> **難易度**: ★☆☆☆☆（入門）
> **前提知識**: 第1章（Gitの基本概念とセットアップ）
> **課題の種類**: ミニプロジェクト
> **学習目標**: リポジトリの作成から初回コミットまでの一連の流れを体験し、Gitの基本的なワークフローを身につける

---

## 完成イメージ

```
$ git log --oneline
a1b2c3d (HEAD -> main) 初回コミット：プロジェクトの基本構造を作成
```

```
$ git status
On branch main
nothing to commit, working tree clean
```

```
my-first-project/
├── .gitignore
├── README.md
└── src/
    └── hello.txt
```

---

## 課題の要件

1. 新しいディレクトリ `my-first-project` を作成し、Gitリポジトリとして初期化する
2. `README.md` ファイルを作成し、プロジェクトの簡単な説明を書く
3. `.gitignore` ファイルを作成し、一般的な不要ファイルを除外する設定を書く
4. `src/` ディレクトリを作成し、その中に `hello.txt` を作成する
5. すべてのファイルをステージング（staging）してコミットする
6. `git log` と `git status` で状態を確認する

---

## ステップガイド

<details>
<summary>ステップ1：プロジェクトディレクトリを作成して初期化する</summary>

まずディレクトリを作成し、その中で `git init` を実行します。

```bash
# ディレクトリを作成して移動
mkdir my-first-project
cd my-first-project

# Gitリポジトリとして初期化
git init

# 確認：.git ディレクトリが作成されていること
ls -la
```

`git init` を実行すると `.git/` という隠しディレクトリが作成されます。これがGitの管理情報を保存する場所です。

</details>

<details>
<summary>ステップ2：README.md を作成する</summary>

プロジェクトの説明ファイルを作成します。README（リードミー）は「まず読んでください」という意味で、プロジェクトの概要を記載する慣習があります。

```bash
# README.md を作成
cat << 'EOF' > README.md
# My First Project

Git の学習用プロジェクトです。

## 概要

このプロジェクトは、Git の基本操作を練習するために作成しました。
EOF
```

</details>

<details>
<summary>ステップ3：.gitignore を作成する</summary>

バージョン管理に含めたくないファイルのパターンを指定します。

```bash
cat << 'EOF' > .gitignore
# OS が自動生成するファイル
.DS_Store
Thumbs.db

# エディタの設定ファイル
*.swp
*.swo
*~
.vscode/
.idea/

# ログファイル
*.log

# 一時ファイル
tmp/
temp/
EOF
```

</details>

<details>
<summary>ステップ4：src ディレクトリとファイルを作成する</summary>

```bash
# ディレクトリを作成
mkdir src

# ファイルを作成
echo "Hello, Git!" > src/hello.txt
```

</details>

<details>
<summary>ステップ5：ステージングしてコミットする</summary>

```bash
# 現在の状態を確認
git status

# すべてのファイルをステージング
git add .

# ステージングされたファイルを確認
git status

# コミット（変更を記録）
git commit -m "初回コミット：プロジェクトの基本構造を作成"
```

</details>

## 解答例

<details>
<summary>解答例（初心者向け：1つずつ確認しながら進める）</summary>

```bash
# ===== 1. リポジトリの作成 =====
mkdir my-first-project
cd my-first-project
git init
# 出力例: Initialized empty Git repository in /home/user/my-first-project/.git/

# 現在の状態を確認
git status
# 出力例: On branch main  No commits yet  nothing to commit

# ===== 2. README.md の作成 =====
cat << 'EOF' > README.md
# My First Project

Git の学習用プロジェクトです。

## 概要

このプロジェクトは、Git の基本操作を練習するために作成しました。
EOF

# ファイルが作成されたか確認
git status
# 出力例: Untracked files: README.md

# ===== 3. .gitignore の作成 =====
cat << 'EOF' > .gitignore
# OS が自動生成するファイル
.DS_Store
Thumbs.db

# エディタの設定ファイル
*.swp
*.swo
*~
.vscode/
.idea/

# ログファイル
*.log

# 一時ファイル
tmp/
temp/
EOF

# ===== 4. src ディレクトリとファイルの作成 =====
mkdir src
echo "Hello, Git!" > src/hello.txt

# すべてのファイルを確認
git status
# 出力例:
# Untracked files:
#   .gitignore
#   README.md
#   src/

# ===== 5. ステージングとコミット =====
# 全ファイルをステージング
git add .

# ステージされた内容を確認
git status
# 出力例:
# Changes to be committed:
#   new file:   .gitignore
#   new file:   README.md
#   new file:   src/hello.txt

# コミット
git commit -m "初回コミット：プロジェクトの基本構造を作成"

# ===== 6. 最終確認 =====
git log --oneline
# 出力例: a1b2c3d (HEAD -> main) 初回コミット：プロジェクトの基本構造を作成

git status
# 出力例: On branch main  nothing to commit, working tree clean
```

</details>

<details>
<summary>解答例（改良版 ─ 実務的なプロジェクト構造）</summary>

実務では、より構造化されたプロジェクトを作ることが一般的です。

```bash
# ===== プロジェクトの作成と初期化 =====
mkdir my-first-project
cd my-first-project
git init

# ===== 実務的なディレクトリ構造を一括作成 =====
mkdir -p src tests docs

# ===== README.md（より詳細なテンプレート）=====
cat << 'EOF' > README.md
# My First Project

Git の学習用プロジェクトです。

## ディレクトリ構成

```
my-first-project/
├── src/     # ソースコード
├── tests/   # テストコード
├── docs/    # ドキュメント
└── README.md
```

## セットアップ

```bash
git clone <リポジトリURL>
cd my-first-project
```

## ライセンス

MIT License
EOF

# ===== .gitignore（言語やフレームワークに合わせて充実させる）=====
cat << 'EOF' > .gitignore
# OS
.DS_Store
Thumbs.db

# エディタ
*.swp
*.swo
*~
.vscode/
.idea/

# 依存パッケージ
node_modules/
__pycache__/
*.pyc
venv/

# ビルド成果物
dist/
build/
*.o
*.out

# 環境設定（秘密情報を含む可能性）
.env
.env.local

# ログ
*.log
EOF

# ===== ソースファイル =====
echo "Hello, Git!" > src/hello.txt

# ===== 空ディレクトリを Git に含めるための .gitkeep =====
# Git は空ディレクトリを追跡しないため、慣習的に .gitkeep ファイルを置きます
touch tests/.gitkeep
touch docs/.gitkeep

# ===== コミット =====
git add .
git commit -m "初回コミット：プロジェクトの基本構造を作成"

# ===== 確認 =====
git log --oneline
git status
```

**初心者向けとの違い:**
- `tests/` や `docs/` など実務で使うディレクトリ構造を最初から作成
- `.gitignore` に依存パッケージやビルド成果物のパターンも追加
- `.env` など秘密情報を含むファイルも除外対象に設定
- `.gitkeep` で空ディレクトリもGit管理下に含める工夫

</details>
