## 問題1：Partial と Required（基本）

`User` インターフェースに対して `Partial` と `Required` を使った型を活用してください。

1. `updateUser(id, updates)` — `Partial<User>` で部分更新
2. `createConfig(userConfig)` — `Required<Config>` でデフォルト値を適用

**期待される出力：**
```
更新前: {name: "太郎", email: "taro@old.com", age: 25}
更新後: {name: "太郎", email: "taro@new.com", age: 25}
設定: {host: "0.0.0.0", port: 8080, debug: false, logLevel: "info"}
```

<details>
<summary>解答例</summary>

```typescript
interface User {
  name: string;
  email: string;
  age: number;
}

// Partialで部分更新
function updateUser(user: User, updates: Partial<User>): User {
  return { ...user, ...updates };
}

const user: User = { name: "太郎", email: "taro@old.com", age: 25 };
console.log(`更新前: ${JSON.stringify(user)}`);
const updated = updateUser(user, { email: "taro@new.com" });
console.log(`更新後: ${JSON.stringify(updated)}`);

// Requiredでデフォルト値の適用
interface Config {
  host?: string;
  port?: number;
  debug?: boolean;
  logLevel?: "debug" | "info" | "warn" | "error";
}

const DEFAULT_CONFIG: Required<Config> = {
  host: "localhost",
  port: 3000,
  debug: false,
  logLevel: "info",
};

function createConfig(userConfig: Config): Required<Config> {
  return { ...DEFAULT_CONFIG, ...userConfig };
}

const config = createConfig({ host: "0.0.0.0", port: 8080 });
console.log(`設定: ${JSON.stringify(config)}`);
```

</details>

## 問題2：Pick と Omit（基本）

`Product` インターフェースから用途に応じた派生型を作成してください。

1. `ProductSummary` — `Pick` で `id`, `name`, `price` のみ
2. `CreateProductInput` — `Omit` で `id`, `createdAt`, `updatedAt` を除外

**期待される出力：**
```
商品概要: {id: 1, name: "TypeScript本", price: 3000}
新規作成: {name: "新商品", price: 1500, category: "書籍", inStock: true}
```

<details>
<summary>解答例</summary>

```typescript
interface Product {
  id: number;
  name: string;
  price: number;
  category: string;
  inStock: boolean;
  createdAt: Date;
  updatedAt: Date;
}

// Pickで必要なプロパティのみ抽出
type ProductSummary = Pick<Product, "id" | "name" | "price">;

// Omitで不要なプロパティを除外
type CreateProductInput = Omit<Product, "id" | "createdAt" | "updatedAt">;

// ProductSummaryの例
const summary: ProductSummary = {
  id: 1,
  name: "TypeScript本",
  price: 3000,
};
console.log(`商品概要: ${JSON.stringify(summary)}`);

// CreateProductInputの例
const input: CreateProductInput = {
  name: "新商品",
  price: 1500,
  category: "書籍",
  inStock: true,
};
console.log(`新規作成: ${JSON.stringify(input)}`);
```

</details>

## 問題3：Record の活用（基本）

`Record` を使って、以下のルックアップテーブルを型安全に作成してください。

1. 曜日 → 日本語名のマッピング
2. ユーザーロール → 権限リストのマッピング

**期待される出力：**
```
Monday → 月曜日
admin の権限: read, write, delete, manage
viewer の権限: read
```

<details>
<summary>解答例</summary>

```typescript
// 曜日のマッピング
type DayOfWeek = "Monday" | "Tuesday" | "Wednesday" | "Thursday" | "Friday";

const dayNames: Record<DayOfWeek, string> = {
  Monday: "月曜日",
  Tuesday: "火曜日",
  Wednesday: "水曜日",
  Thursday: "木曜日",
  Friday: "金曜日",
};

console.log(`Monday → ${dayNames.Monday}`);

// ロールと権限のマッピング
type Role = "admin" | "editor" | "viewer";
type Permission = "read" | "write" | "delete" | "manage";

const rolePermissions: Record<Role, Permission[]> = {
  admin: ["read", "write", "delete", "manage"],
  editor: ["read", "write"],
  viewer: ["read"],
};

console.log(`admin の権限: ${rolePermissions.admin.join(", ")}`);
console.log(`viewer の権限: ${rolePermissions.viewer.join(", ")}`);
```

</details>

## 問題4：Exclude と Extract（応用）

ユニオン型に対して `Exclude` と `Extract` を使ったフィルタリングを行ってください。

1. 全イベント型からマウス関連イベントだけを抽出
2. 全イベント型からキーボード関連イベントを除外

**期待される出力：**
```
マウスイベント: click, mousedown, mouseup
非キーボードイベント: click, mousedown, mouseup, scroll, resize
```

<details>
<summary>解答例</summary>

```typescript
type AllEvents =
  | "click"
  | "mousedown"
  | "mouseup"
  | "keydown"
  | "keyup"
  | "scroll"
  | "resize";

type MouseEvent = "click" | "mousedown" | "mouseup";
type KeyboardEvent = "keydown" | "keyup";

// Extractでマウスイベントを抽出
type MouseEvents = Extract<AllEvents, MouseEvent>;
// "click" | "mousedown" | "mouseup"

// Excludeでキーボードイベントを除外
type NonKeyboardEvents = Exclude<AllEvents, KeyboardEvent>;
// "click" | "mousedown" | "mouseup" | "scroll" | "resize"

// 動作確認用の関数
function printEvents<T extends string>(label: string, ...events: T[]): void {
  console.log(`${label}: ${events.join(", ")}`);
}

const mouseEvents: MouseEvents[] = ["click", "mousedown", "mouseup"];
printEvents("マウスイベント", ...mouseEvents);

const nonKbEvents: NonKeyboardEvents[] = [
  "click", "mousedown", "mouseup", "scroll", "resize",
];
printEvents("非キーボードイベント", ...nonKbEvents);
```

</details>

## 問題5：ReturnType と Parameters（応用）

既存の関数から型情報を取得して、ラッパー関数を作成してください。

1. `createUser` 関数の戻り値型を `ReturnType` で取得
2. `sendEmail` 関数のパラメータ型を `Parameters` で取得
3. ログ付きラッパー関数を作成

**期待される出力：**
```
ユーザー作成: {id: 0.123..., name: "太郎", age: 25, createdAt: "2026-..."}
メール送信ログ: to=user@example.com, subject=テスト
```

<details>
<summary>解答例</summary>

```typescript
// 元の関数
function createUser(name: string, age: number) {
  return {
    id: Math.random(),
    name,
    age,
    createdAt: new Date(),
  };
}

function sendEmail(to: string, subject: string, body: string): boolean {
  // メール送信処理（省略）
  return true;
}

// ReturnTypeで戻り値の型を取得
type User = ReturnType<typeof createUser>;

// Parametersでパラメータの型を取得
type EmailParams = Parameters<typeof sendEmail>;

// ラッパー関数
function createUserWithLog(
  ...args: Parameters<typeof createUser>
): ReturnType<typeof createUser> {
  const user = createUser(...args);
  console.log(`ユーザー作成: ${JSON.stringify(user)}`);
  return user;
}

function sendEmailWithLog(...args: EmailParams): ReturnType<typeof sendEmail> {
  const [to, subject] = args;
  console.log(`メール送信ログ: to=${to}, subject=${subject}`);
  return sendEmail(...args);
}

// テスト
createUserWithLog("太郎", 25);
sendEmailWithLog("user@example.com", "テスト", "本文です");
```

## 問題6：ユーティリティ型の組み合わせ（応用）

以下のカスタムユーティリティ型を作成してください。

1. `PartialBy<T, K>` — 指定したプロパティだけオプショナルにする
2. `RequiredBy<T, K>` — 指定したプロパティだけ必須にする

**期待される出力：**
```
PartialBy: {id: 1, name: "太郎"} (email, ageは省略OK)
RequiredBy: {host: "localhost", apiKey: "abc123"} (port, debugは省略OK)
```

<details>
<summary>解答例</summary>

```typescript
// 指定プロパティだけオプショナル
type PartialBy<T, K extends keyof T> = Omit<T, K> & Partial<Pick<T, K>>;

// 指定プロパティだけ必須
type RequiredBy<T, K extends keyof T> = T & Required<Pick<T, K>>;

// テスト: PartialBy
interface User {
  id: number;
  name: string;
  email: string;
  age: number;
}

type UserBasic = PartialBy<User, "email" | "age">;
// { id: number; name: string; email?: string; age?: number }

const user: UserBasic = { id: 1, name: "太郎" };
console.log(`PartialBy: ${JSON.stringify(user)} (email, ageは省略OK)`);

// テスト: RequiredBy
interface Config {
  host?: string;
  port?: number;
  debug?: boolean;
  apiKey?: string;
}

type ProductionConfig = RequiredBy<Config, "apiKey">;
// apiKeyだけ必須、他はオプショナルのまま

const config: ProductionConfig = {
  host: "localhost",
  apiKey: "abc123",
};
console.log(`RequiredBy: ${JSON.stringify(config)} (port, debugは省略OK)`);
```

</details>

## 問題7：CRUD型セットの自動生成（チャレンジ）

エンティティの基底型から、CRUD操作に必要な型セットを自動生成するユーティリティ型を作成してください。

```typescript
interface Entity {
  id: number;
  createdAt: Date;
  updatedAt: Date;
}
```

1. `CreateInput<T>` — Entity系プロパティを除外
2. `UpdateInput<T>` — CreateInputのPartial
3. `EntitySummary<T>` — idと指定プロパティだけ抽出

**期待される出力：**
```
=== Product CRUD ===
Create: {name: "新商品", price: 1500, category: "書籍"}
Update: {price: 2000}
Summary: {id: 1, name: "TypeScript本"}
```

<details>
<summary>解答例</summary>

```typescript
interface Entity {
  id: number;
  createdAt: Date;
  updatedAt: Date;
}

// CRUD用ユーティリティ型
type CreateInput<T extends Entity> = Omit<T, keyof Entity>;
type UpdateInput<T extends Entity> = Partial<CreateInput<T>>;

// 使用例
interface Product extends Entity {
  name: string;
  price: number;
  category: string;
}

// 作成用入力
const createInput: CreateInput<Product> = {
  name: "新商品",
  price: 1500,
  category: "書籍",
};

// 更新用入力（部分更新）
const updateInput: UpdateInput<Product> = {
  price: 2000,
};

// 概要型（Pickで必要なフィールドだけ）
type ProductSummary = Pick<Product, "id" | "name">;

const summary: ProductSummary = {
  id: 1,
  name: "TypeScript本",
};

console.log("=== Product CRUD ===");
console.log(`Create: ${JSON.stringify(createInput)}`);
console.log(`Update: ${JSON.stringify(updateInput)}`);
console.log(`Summary: ${JSON.stringify(summary)}`);
```

</details>

## 問題8：Readonlyの深い変換（チャレンジ）

`Readonly<T>` が浅い変換であることを確認し、再帰的な `DeepReadonly<T>` を自作してください。

1. `Readonly<T>` の浅さを示すコード例を書く
2. `DeepReadonly<T>` を実装
3. ネストしたオブジェクトに適用して動作を確認

**期待される出力：**
```
Readonly<T>は浅い: ネストしたオブジェクトは変更可能
DeepReadonly<T>は深い: 全てのプロパティが読み取り専用
設定値: localhost:5432/mydb
```

<details>
<summary>解答例</summary>

```typescript
// ネストした設定型
interface AppConfig {
  server: {
    host: string;
    port: number;
  };
  database: {
    connection: {
      url: string;
      pool: number;
    };
  };
}

// Readonlyの浅さを確認
const shallowConfig: Readonly<AppConfig> = {
  server: { host: "localhost", port: 3000 },
  database: { connection: { url: "postgres://localhost", pool: 5 } },
};

// shallowConfig.server = { ... }; // エラー！直接プロパティは変更不可
shallowConfig.server.port = 8080; // しかしネストした値は変更できてしまう！
console.log("Readonly<T>は浅い: ネストしたオブジェクトは変更可能");

// DeepReadonly の実装
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object
    ? T[P] extends Function
      ? T[P] // 関数はそのまま
      : DeepReadonly<T[P]> // オブジェクトは再帰的にreadonly化
    : T[P]; // プリミティブはそのまま
};

const deepConfig: DeepReadonly<AppConfig> = {
  server: { host: "localhost", port: 5432 },
  database: { connection: { url: "postgres://localhost/mydb", pool: 10 } },
};

// deepConfig.server.port = 8080; // エラー！深い階層もreadonly
// deepConfig.database.connection.url = "..."; // エラー！
console.log("DeepReadonly<T>は深い: 全てのプロパティが読み取り専用");

const { host, port } = deepConfig.server;
const { url } = deepConfig.database.connection;
console.log(`設定値: ${host}:${port}${url.split("localhost")[1]}`);
```

</details>
