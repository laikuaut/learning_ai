## 課題の要件

1. `getByRole` でh1見出し「ユーザー管理画面」を取得して検証する
2. `getByLabel` で「ユーザー名」入力欄を取得し、テキストを入力して検証する
3. `getByText` で「このページはロケーターの練習用です。」を取得して検証する
4. `getByTestId` で `edit-user-1` ボタンを取得して検証する
5. `getByPlaceholder` で `"例：taro@example.com"` 入力欄を取得して検証する
6. `getByRole('button')` に `filter` を使い、「登録する」ボタンだけを絞り込んで検証する

---

## ステップガイド

<details>
<summary>ステップ1：HTMLファイルをテストで開く</summary>

ローカルのHTMLファイルをPlaywrightで開くには、`file://` プロトコルを使うか、`path` モジュールで絶対パスを指定します。

```typescript
import { test, expect } from '@playwright/test';
import path from 'path';

test.beforeEach(async ({ page }) => {
  const filePath = path.resolve(__dirname, '../test-page.html');
  await page.goto(`file://${filePath}`);
});
```

</details>

<details>
<summary>ステップ2：各ロケーターを試す</summary>

Playwrightの推奨ロケーターは以下の優先順位です。

```
1. getByRole     — アクセシビリティロール（最推奨）
2. getByLabel    — フォームのラベル
3. getByPlaceholder — プレースホルダ
4. getByText     — テキスト内容
5. getByTestId   — data-testid属性
```

まずは `getByRole` から試してみましょう。

</details>

## 解答例

<details>
<summary>解答例（初心者向け）</summary>

```typescript
// tests/locator-drill.spec.ts
// 学べる内容：各種ロケーターの使い分け
// 実行方法：npx playwright test tests/locator-drill.spec.ts --project=chromium

import { test, expect } from '@playwright/test';
import path from 'path';

test.describe('ロケーター練習', () => {

  test.beforeEach(async ({ page }) => {
    // ローカルHTMLファイルを開く
    const filePath = path.resolve(__dirname, '../test-page.html');
    await page.goto(`file://${filePath}`);
  });

  // 1. getByRole：見出しの取得
  test('getByRoleでh1見出しを取得できる', async ({ page }) => {
    const heading = page.getByRole('heading', { name: 'ユーザー管理画面', level: 1 });
    await expect(heading).toBeVisible();
    await expect(heading).toHaveText('ユーザー管理画面');
  });

  // 2. getByLabel：フォーム要素の取得
  test('getByLabelでフォーム要素を取得して入力できる', async ({ page }) => {
    const usernameInput = page.getByLabel('ユーザー名');
    await usernameInput.fill('tanaka_taro');
    await expect(usernameInput).toHaveValue('tanaka_taro');
  });

  // 3. getByText：テキスト要素の取得
  test('getByTextでテキスト要素を取得できる', async ({ page }) => {
    const description = page.getByText('このページはロケーターの練習用です。');
    await expect(description).toBeVisible();
  });

  // 4. getByTestId：data-testid属性での取得
  test('getByTestIdで要素を取得できる', async ({ page }) => {
    const editButton = page.getByTestId('edit-user-1');
    await expect(editButton).toBeVisible();
    await expect(editButton).toHaveText('編集');
  });

  // 5. getByPlaceholder：プレースホルダでの取得
  test('getByPlaceholderで入力欄を取得できる', async ({ page }) => {
    const emailInput = page.getByPlaceholder('例：taro@example.com');
    await emailInput.fill('hanako@example.com');
    await expect(emailInput).toHaveValue('hanako@example.com');
  });

  // 6. filter：要素の絞り込み
  test('フィルターでボタンを絞り込める', async ({ page }) => {
    const submitButton = page.getByRole('button').filter({ hasText: '登録する' });
    await expect(submitButton).toBeVisible();
    await expect(submitButton).toHaveClass(/primary/);
  });
});
```

<details>
<summary>解答例（改良版 ─ より多くのロケーターを活用）</summary>

```typescript
// tests/locator-drill-advanced.spec.ts
// 改良版：実務で使うパターンを網羅
// 実行方法：npx playwright test tests/locator-drill-advanced.spec.ts --project=chromium

import { test, expect } from '@playwright/test';
import path from 'path';

test.describe('ロケーター練習（応用）', () => {

  test.beforeEach(async ({ page }) => {
    const filePath = path.resolve(__dirname, '../test-page.html');
    await page.goto(`file://${filePath}`);
  });

  test('getByRoleで複数の見出しを区別する', async ({ page }) => {
    // level指定でh1とh2を区別
    const h1 = page.getByRole('heading', { level: 1 });
    await expect(h1).toHaveText('ユーザー管理画面');

    // name指定で同レベルのh2を区別
    const h2Registration = page.getByRole('heading', { name: 'ユーザー登録' });
    const h2List = page.getByRole('heading', { name: 'ユーザー一覧' });
    await expect(h2Registration).toBeVisible();
    await expect(h2List).toBeVisible();
  });

  test('getByLabelでセレクトボックスを操作する', async ({ page }) => {
    const roleSelect = page.getByLabel('権限');
    await roleSelect.selectOption('admin');
    await expect(roleSelect).toHaveValue('admin');
  });

  test('getByTextで部分一致と完全一致を使い分ける', async ({ page }) => {
    // 部分一致（デフォルト）
    const partial = page.getByText('ロケーターの練習用');
    await expect(partial).toBeVisible();

    // 完全一致
    const exact = page.getByText('このページはロケーターの練習用です。', { exact: true });
    await expect(exact).toBeVisible();
  });

  test('locator().filter()で行の中の特定ボタンを取得する', async ({ page }) => {
    // テーブルの行をfilterで絞り込み、その中のボタンを操作
    const row = page.getByRole('row').filter({ hasText: '佐藤花子' });
    const deleteButton = row.getByRole('button', { name: '削除' });
    await expect(deleteButton).toBeVisible();
    await expect(deleteButton).toHaveAttribute('data-testid', 'delete-user-2');
  });

  test('disabledな要素を検証する', async ({ page }) => {
    const prevButton = page.getByRole('button', { name: '前へ' });
    await expect(prevButton).toBeDisabled();

    const nextButton = page.getByRole('button', { name: '次へ' });
    await expect(nextButton).toBeEnabled();
  });

  test('aria-labelでナビゲーション要素を取得する', async ({ page }) => {
    const nav = page.getByRole('navigation', { name: 'ページナビゲーション' });
    await expect(nav).toBeVisible();
  });
});
```

**初心者向けとの違い:**
- `getByRole` の `level` と `name` を組み合わせて複数の見出しを区別している
- `filter` と `hasText` を使って、テーブルの特定行内のボタンを取得する実務的なパターンを使用
- `toBeDisabled()` や `toBeEnabled()` で要素の状態を検証
- `aria-label` を利用したアクセシビリティベースのロケーター取得
