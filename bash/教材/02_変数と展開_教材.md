## 2.1 変数の基本

### 変数の代入と参照

Bashの変数は、**型宣言なしで**使えます。代入時に `=` の前後にスペースを入れてはいけません。

```bash
# 変数の代入（= の前後にスペースを入れない！）
name="太郎"
age=25
message="Hello, World!"

# 変数の参照（$ を付ける）
echo $name          # 太郎
echo ${name}        # 太郎（中括弧で囲む書き方、推奨）

# 変数名の直後に文字が続く場合は {} が必須
file="report"
echo ${file}_2024.txt   # report_2024.txt
echo $file_2024.txt     # NG: $file_2024 という変数を探してしまう
```

> **よくある間違い**: `name = "太郎"` のように `=` の前後にスペースを入れると、`name` をコマンド、`=` と `"太郎"` を引数として解釈してしまい、エラーになります。

### 変数の削除と読み取り専用

```bash
# 変数の削除
name="太郎"
echo $name    # 太郎
unset name
echo $name    # （空文字が出力される）

# 読み取り専用変数
readonly PI=3.14159
echo $PI      # 3.14159
PI=3.0        # エラー: readonly variable

# declare -r でも同じ
declare -r MAX_SIZE=100
```

### 変数のスコープ

```bash
# 通常の変数はシェル変数（そのシェルでのみ有効）
my_var="hello"
bash -c 'echo $my_var'   # （何も表示されない）

# export するとサブシェルにも引き継がれる
export my_var="hello"
bash -c 'echo $my_var'   # hello
```

### declare コマンドによる型指定

```bash
# 整数型として宣言
declare -i num=10
num=num+5
echo $num       # 15（算術式として評価される）

# 配列として宣言
declare -a fruits=("apple" "banana" "cherry")
echo ${fruits[0]}   # apple

# 連想配列として宣言（Bash 4.0以降）
declare -A colors
colors[red]="#FF0000"
colors[green]="#00FF00"
echo ${colors[red]}  # #FF0000

# 小文字に変換（Bash 4.0以降）
declare -l lower="HELLO"
echo $lower     # hello

# 大文字に変換（Bash 4.0以降）
declare -u upper="hello"
echo $upper     # HELLO
```

## 2.2 環境変数

### 環境変数とシェル変数の違い

```
┌──────────────────────────────────────────────────────────┐
│                     現在のシェル                          │
│                                                          │
│  シェル変数: そのシェルプロセスのみで有効                │
│    my_var="hello"                                        │
│                                                          │
│  環境変数: 子プロセスにも引き継がれる                    │
│    export MY_ENV="world"                                 │
│                                                          │
│  ┌──────────────────────────────────────────────────┐   │
│  │            サブシェル（子プロセス）               │   │
│  │                                                  │   │
│  │  echo $my_var   → （空）   ← 引き継がれない     │   │
│  │  echo $MY_ENV   → world   ← 引き継がれる       │   │
│  │                                                  │   │
│  └──────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────┘
```

### export コマンド

```bash
# 変数を環境変数にする
MY_VAR="hello"
export MY_VAR

# 定義と同時にexport
export MY_VAR="hello"

# 環境変数の一覧を表示
env
# または
printenv

# 特定の環境変数を表示
printenv HOME
echo $HOME
```

### よく使われる環境変数

```bash
# 主要な環境変数一覧
echo $HOME       # ホームディレクトリ (/home/user)
echo $USER       # ユーザー名 (user)
echo $PATH       # コマンド検索パス (/usr/local/bin:/usr/bin:...)
echo $PWD        # 現在のディレクトリ
echo $OLDPWD     # 1つ前のディレクトリ
echo $SHELL      # ログインシェル (/bin/bash)
echo $HOSTNAME   # ホスト名
echo $LANG       # 言語設定 (ja_JP.UTF-8)
echo $TERM       # ターミナルタイプ (xterm-256color)
echo $EDITOR     # デフォルトエディタ (vim)
echo $RANDOM     # 0〜32767のランダムな整数
echo $SECONDS    # シェル起動からの秒数
echo $LINENO     # 現在の行番号（スクリプト内で使用）
```

### PATHの操作

```bash
# PATHの確認
echo $PATH

# PATHの先頭にディレクトリを追加（優先度が高くなる）
export PATH="$HOME/bin:$PATH"

# PATHの末尾にディレクトリを追加
export PATH="$PATH:/opt/myapp/bin"

# PATHの内容を見やすく表示
echo $PATH | tr ':' '\n'
```

> **現場のベストプラクティス**: PATHに追加する際は、必ず既存の `$PATH` を含めてください。`export PATH="/new/path"` と書くと、元のPATHが上書きされ、基本的なコマンドすら実行できなくなります。

## 2.3 特殊変数

Bashには、スクリプトの実行状態に関する情報を保持する特殊変数（special variable）が用意されています。

### 一覧と説明

```
┌──────────┬──────────────────────────────────────────┐
│  変数    │  説明                                      │
├──────────┼──────────────────────────────────────────┤
│  $0      │  スクリプト名（または実行中のシェル名）   │
│  $1〜$9  │  位置パラメータ（引数）                   │
│  ${10}   │  10番目以降の引数（{} が必要）            │
│  $#      │  引数の数                                 │
│  $@      │  全引数（個別にクォートされる）           │
│  $*      │  全引数（1つの文字列として扱われる）     │
│  $$      │  現在のシェルのプロセスID                 │
│  $!      │  最後にバックグラウンド実行したPID        │
│  $?      │  直前のコマンドの終了ステータス           │
│  $_      │  直前のコマンドの最後の引数               │
│  $-      │  現在のシェルオプションフラグ             │
└──────────┴──────────────────────────────────────────┘
```

### 使用例

以下のスクリプトを `test_args.sh` として保存してください。

```bash
#!/usr/bin/env bash
# 特殊変数のデモ

echo "スクリプト名: $0"
echo "引数の数: $#"
echo "第1引数: $1"
echo "第2引数: $2"
echo "全引数(\$@): $@"
echo "全引数(\$*): $*"
echo "プロセスID: $$"
```

```bash
# 実行例
bash test_args.sh apple banana cherry

# 出力:
# スクリプト名: test_args.sh
# 引数の数: 3
# 第1引数: apple
# 第2引数: banana
# 全引数($@): apple banana cherry
# 全引数($*): apple banana cherry
```

### $@ と $* の違い

この2つは**クォートした場合**に動作が異なります。

```bash
#!/usr/bin/env bash
# $@ と $* の違いを確認するスクリプト

echo "--- \"\$@\" の場合 ---"
for arg in "$@"; do
    echo "  [$arg]"
done

echo "--- \"\$*\" の場合 ---"
for arg in "$*"; do
    echo "  [$arg]"
done
```

```bash
# 実行
bash diff_test.sh "hello world" "foo bar"

# 出力:
# --- "$@" の場合 ---
#   [hello world]     ← 各引数が個別に保持される
#   [foo bar]
# --- "$*" の場合 ---
#   [hello world foo bar]  ← 全引数が1つの文字列になる
```

> **現場のベストプラクティス**: 引数を他のコマンドに渡す場合は、必ず `"$@"` を使ってください。`$*` や クォートなしの `$@` では、スペースを含む引数が分割されてしまいます。

### 終了ステータス（$?）

```bash
# コマンドが成功すると 0
ls /etc/hostname
echo $?    # 0

# コマンドが失敗すると 0 以外
ls /nonexistent
echo $?    # 2（ファイルが見つからない）

# スクリプトでの終了ステータス制御
# exit 0  → 成功
# exit 1  → 一般的なエラー
# exit 2  → 使い方の間違い
```

## 2.4 クォーティング

Bashにおけるクォーティング（quoting）は、特殊文字の解釈を制御するために非常に重要です。

### クォートの種類と違い

```
┌──────────────────────────────────────────────────────────────┐
│                   クォーティングの比較                        │
├──────────────┬───────────────────────────────────────────────┤
│ 種類         │ 特徴                                          │
├──────────────┼───────────────────────────────────────────────┤
│ シングル     │ 全ての特殊文字が無効化される                  │
│ 'text'       │ 変数展開もコマンド置換もされない              │
├──────────────┼───────────────────────────────────────────────┤
│ ダブル       │ $, `, \, !, " は特殊文字として機能する       │
│ "text"       │ 変数展開とコマンド置換が行われる              │
├──────────────┼───────────────────────────────────────────────┤
│ バッククォート│ コマンド置換（非推奨、$() を使う）           │
│ `command`    │ ネストできない                                │
├──────────────┼───────────────────────────────────────────────┤
│ $()          │ コマンド置換（推奨）                          │
│ $(command)   │ ネスト可能で読みやすい                        │
└──────────────┴───────────────────────────────────────────────┘
```

### 具体例

```bash
name="太郎"

# シングルクォート: 変数展開されない
echo 'Hello, $name'
# 出力: Hello, $name

# ダブルクォート: 変数展開される
echo "Hello, $name"
# 出力: Hello, 太郎

# クォートなし: 変数展開 + ワード分割 + グロブ展開が行われる
files="*.txt"
echo $files       # ワイルドカードが展開される（ファイル名一覧）
echo "$files"     # *.txt がそのまま表示される

# バッククォート vs $()
echo "Today is `date`"           # 古い書き方（非推奨）
echo "Today is $(date)"          # 新しい書き方（推奨）

# $() はネストできる
echo "Kernel: $(uname -r | cut -d. -f1-2)"
echo "Files: $(ls $(dirname /etc/hostname))"
```

### エスケープ

```bash
# バックスラッシュで特殊文字をエスケープ
echo "Price: \$100"    # Price: $100
echo "She said \"Hi\"" # She said "Hi"

# シングルクォート内でシングルクォートを使う場合
echo 'It'\''s a pen'   # It's a pen（一旦閉じて \' を挟む）
echo "It's a pen"      # ダブルクォートで囲む方が簡単

# $'...' 形式（ANSIエスケープ）
echo $'Line1\nLine2'   # 改行が解釈される
echo $'Tab\there'      # タブが解釈される
```

> **よくある間違い**: ダブルクォートの中で `$` を表示したい場合は `\$` とエスケープが必要です。シングルクォートで囲めばエスケープ不要ですが、その場合は他の変数も展開されません。

## 2.5 パラメータ展開

パラメータ展開（parameter expansion）は、Bashの強力な機能の一つです。変数の値を加工・変換する多彩な方法が用意されています。

### デフォルト値の設定

```bash
# ${var:-default} : var が未設定 or 空の場合に default を返す（var は変更しない）
echo ${name:-"名無し"}     # name が未設定なら "名無し"
echo $name                  # name は空のまま

# ${var:=default} : var が未設定 or 空の場合に default を代入して返す
echo ${name:=名無し}       # name が未設定なら "名無し" を代入
echo $name                  # name は "名無し" になっている

# ${var:+value} : var が設定済みの場合に value を返す
name="太郎"
echo ${name:+"名前は ${name}"}   # 名前は 太郎
unset name
echo ${name:+"名前は ${name}"}   # （空文字）

# ${var:?error} : var が未設定 or 空の場合にエラーメッセージを表示して終了
echo ${REQUIRED_VAR:?"この変数は必須です"}
# REQUIRED_VAR が未設定だと: bash: REQUIRED_VAR: この変数は必須です
```

```bash
# 実務での使用例: 設定ファイルのデフォルト値
LOG_DIR=${LOG_DIR:-"/var/log/myapp"}
MAX_RETRIES=${MAX_RETRIES:-3}
PORT=${PORT:=8080}

echo "ログディレクトリ: $LOG_DIR"
echo "最大リトライ: $MAX_RETRIES"
echo "ポート: $PORT"
```

### 文字列の長さ

```bash
text="Hello, World!"
echo ${#text}    # 13（文字数を返す）

# 空文字の判定にも使える
if [ ${#text} -eq 0 ]; then
    echo "空文字です"
fi
```

### パターン削除

```bash
# ${var%pattern}  : 後方から最短一致で削除
# ${var%%pattern} : 後方から最長一致で削除
# ${var#pattern}  : 前方から最短一致で削除
# ${var##pattern} : 前方から最長一致で削除

filepath="/home/user/documents/report.tar.gz"

# 前方からの削除
echo ${filepath#*/}      # home/user/documents/report.tar.gz（最初の / まで削除）
echo ${filepath##*/}     # report.tar.gz（最後の / まで削除 = ファイル名取得）

# 後方からの削除
echo ${filepath%.*}      # /home/user/documents/report.tar（最後の . 以降を削除）
echo ${filepath%%.*}     # /home/user/documents/report（最初の . 以降を全削除）

# ディレクトリ部分だけ取得（dirname と同等）
echo ${filepath%/*}      # /home/user/documents

# 拡張子だけ取得
echo ${filepath##*.}     # gz
```

```
パターン削除の覚え方:

  # は前方削除（キーボードで # は $ の左にある）
  % は後方削除（キーボードで % は $ の右にある）

  1個 = 最短一致（控えめ）
  2個 = 最長一致（貪欲）

  ${var#pattern}   前方・最短
  ${var##pattern}  前方・最長
  ${var%pattern}   後方・最短
  ${var%%pattern}  後方・最長
```

### パターン置換

```bash
# ${var/pattern/replace}  : 最初にマッチしたものを置換
# ${var//pattern/replace} : 全てのマッチを置換
# ${var/#pattern/replace} : 先頭にマッチした場合に置換
# ${var/%pattern/replace} : 末尾にマッチした場合に置換

text="apple banana apple cherry apple"

echo ${text/apple/orange}     # orange banana apple cherry apple（最初の1つだけ）
echo ${text//apple/orange}    # orange banana orange cherry orange（全て置換）

filename="photo.jpeg"
echo ${filename/%jpeg/jpg}    # photo.jpg（末尾のjpegをjpgに置換）
echo ${filename/#photo/image} # image.jpeg（先頭のphotoをimageに置換）

# 置換文字列を省略すると削除になる
echo ${text//apple/}          #  banana  cherry （appleを削除）
```

### 部分文字列の抽出

```bash
# ${var:offset}       : offset 位置から末尾まで
# ${var:offset:length}: offset 位置から length 文字

text="Hello, World!"

echo ${text:0:5}     # Hello（0文字目から5文字）
echo ${text:7}       # World!（7文字目から末尾まで）
echo ${text:7:5}     # World（7文字目から5文字）
echo ${text: -6}     # orld!（末尾から6文字。マイナスの前にスペースが必要）
echo ${text: -6:3}   # orl（末尾から6文字目を起点に3文字）
```

### 大文字・小文字変換（Bash 4.0以降）

```bash
text="Hello World"

echo ${text^^}    # HELLO WORLD（全て大文字に）
echo ${text,,}    # hello world（全て小文字に）
echo ${text^}     # Hello World（先頭のみ大文字に）
echo ${text,}     # hello World（先頭のみ小文字に）
```

## 2.6 コマンド置換と算術展開

### コマンド置換（Command Substitution）

コマンドの実行結果を変数に代入したり、文字列の中に埋め込んだりできます。

```bash
# $() を使う方法（推奨）
today=$(date +%Y-%m-%d)
echo "今日は $today です"

# バッククォートを使う方法（非推奨、互換性のために存在）
today=`date +%Y-%m-%d`

# ネストの例（$() が推奨される理由）
# $() はネスト可能
parent_dir=$(basename $(dirname /home/user/docs/file.txt))
echo $parent_dir    # docs

# バッククォートではネストが困難
parent_dir=`basename \`dirname /home/user/docs/file.txt\``
```

```bash
# 実務での使用例
# ファイル一覧を変数に格納
file_count=$(ls -1 | wc -l)
echo "ファイル数: $file_count"

# ログファイル名に日時を含める
log_file="app_$(date +%Y%m%d_%H%M%S).log"
echo "ログファイル: $log_file"

# コマンドの実行結果で条件分岐
if [ "$(whoami)" = "root" ]; then
    echo "root ユーザーです"
fi
```

### 算術展開（Arithmetic Expansion）

```bash
# $(( )) を使う方法（推奨）
echo $((1 + 2))       # 3
echo $((10 - 3))      # 7
echo $((4 * 5))       # 20
echo $((10 / 3))      # 3（整数除算）
echo $((10 % 3))      # 1（剰余）
echo $((2 ** 10))     # 1024（べき乗）

# 変数を使った計算（$ を省略できる）
x=10
y=3
echo $((x + y))       # 13
echo $((x * y))       # 30

# インクリメント・デクリメント
echo $((x++))         # 10（後置: 先に値を返してからインクリメント）
echo $x               # 11
echo $((++x))         # 12（前置: インクリメントしてから値を返す）

# 代入演算子
x=10
((x += 5))
echo $x               # 15
((x -= 3))
echo $x               # 12
((x *= 2))
echo $x               # 24
```

### let コマンド

```bash
# let を使った算術演算
let result=5+3
echo $result     # 8

let "result = 10 * 2"  # スペースを含む場合はクォートが必要
echo $result     # 20

# 複数の計算を同時に
let "a = 5" "b = 10" "c = a + b"
echo $c          # 15
```

### expr コマンド

```bash
# expr を使った算術演算（古い方法、$(( )) が推奨）
result=$(expr 5 + 3)     # 演算子の前後にスペースが必要
echo $result              # 8

result=$(expr 10 \* 2)    # * はエスケープが必要
echo $result              # 20
```

> **現場のベストプラクティス**: 算術演算には `$(( ))` を使いましょう。`let` や `expr` よりも読みやすく、サブシェルを生成しないため効率的です。ただし、Bashの算術演算は**整数のみ**です。小数の計算には `bc` コマンドを使います。

```bash
# 小数の計算には bc を使う
result=$(echo "scale=2; 10 / 3" | bc)
echo $result    # 3.33
```
