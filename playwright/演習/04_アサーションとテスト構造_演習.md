### 問題1：基本的なWebアサーション

以下のHTMLに対して、各種Webアサーションを使って要素の状態を検証するテストを作成してください。

```html
<div id="app">
  <h1>ユーザープロフィール</h1>
  <p id="status" class="active online">アクティブ</p>
  <input id="username" type="text" value="tanaka" disabled />
  <a href="/settings">設定ページへ</a>
  <button id="save-btn" style="display:none;">保存</button>
</div>
```

以下のアサーションを書いてください。

1. `h1` が表示されていて、テキストが「ユーザープロフィール」であること
2. ステータスが「アクティブ」というテキストを持つこと
3. ユーザー名入力欄の値が「tanaka」であること
4. ユーザー名入力欄が無効化（disabled）されていること
5. リンクのhref属性が「/settings」であること
6. 保存ボタンが非表示であること

**期待される結果:**
```
6つのアサーションがすべて成功する
```

<details>
<summary>ヒント</summary>

使用するアサーション: `toBeVisible()`, `toHaveText()`, `toHaveValue()`, `toBeDisabled()`, `toHaveAttribute()`, `toBeHidden()`

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('基本的なWebアサーション', async ({ page }) => {
  await page.setContent(`
    <div id="app">
      <h1>ユーザープロフィール</h1>
      <p id="status" class="active online">アクティブ</p>
      <input id="username" type="text" value="tanaka" disabled />
      <a href="/settings">設定ページへ</a>
      <button id="save-btn" style="display:none;">保存</button>
    </div>
  `);

  // 1. h1 が表示されていてテキストが正しいことを検証
  await expect(page.getByRole('heading', { level: 1 })).toBeVisible();
  await expect(page.getByRole('heading', { level: 1 })).toHaveText('ユーザープロフィール');

  // 2. ステータスのテキスト検証
  await expect(page.locator('#status')).toHaveText('アクティブ');

  // 3. 入力欄の値を検証
  await expect(page.locator('#username')).toHaveValue('tanaka');

  // 4. 入力欄が無効化されていることを検証
  await expect(page.locator('#username')).toBeDisabled();

  // 5. リンクのhref属性を検証
  await expect(page.getByRole('link', { name: '設定ページへ' })).toHaveAttribute('href', '/settings');

  // 6. 保存ボタンが非表示であることを検証
  await expect(page.locator('#save-btn')).toBeHidden();
});
```

</details>

### 問題2：.notを使った否定のアサーション

以下のHTMLに対して、`.not` を使った否定のアサーションを作成してください。

```html
<div id="app">
  <input id="email" type="email" value="" />
  <button id="submit-btn" disabled>送信</button>
  <p id="error" style="display:none;">エラーがあります</p>
  <p id="success">処理が完了しました</p>
  <div id="content">
    <p>これは本文です。</p>
  </div>
</div>
```

以下の否定アサーションを書いてください。

1. メール入力欄が空であること（値が空文字列でないことの逆ではなく、空であることを `.not` を使わずに検証し、比較として `.not` バージョンも書く）
2. 送信ボタンが有効でない（enabled でない）ことを `.not` で検証する
3. エラーメッセージが表示されていないことを `.not` で検証する
4. コンテンツに「エラー」というテキストが含まれていないことを `.not` で検証する

**期待される結果:**
```
すべての否定アサーションが成功する
```

<details>
<summary>ヒント</summary>

`.not` はアサーションの前に付けて否定にします。例: `await expect(locator).not.toBeVisible()`。`toHaveValue('')` で空を直接検証する方法と、`not.toHaveValue()` で特定の値でないことを検証する方法があります。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('.notを使った否定のアサーション', async ({ page }) => {
  await page.setContent(`
    <div id="app">
      <input id="email" type="email" value="" />
      <button id="submit-btn" disabled>送信</button>
      <p id="error" style="display:none;">エラーがあります</p>
      <p id="success">処理が完了しました</p>
      <div id="content">
        <p>これは本文です。</p>
      </div>
    </div>
  `);

  // 1. メール入力欄が空であることの検証
  // 方法A: .not を使わない（空文字列であることを直接検証）
  await expect(page.locator('#email')).toHaveValue('');
  // 方法B: .not を使う（特定の値でないことを検証）
  await expect(page.locator('#email')).not.toHaveValue('test@example.com');

  // 2. 送信ボタンが有効でないことを .not で検証
  await expect(page.locator('#submit-btn')).not.toBeEnabled();
  // ※ toBeDisabled() でも同じ意味になる
  await expect(page.locator('#submit-btn')).toBeDisabled();

  // 3. エラーメッセージが表示されていないことを .not で検証
  await expect(page.locator('#error')).not.toBeVisible();

  // 4. コンテンツに「エラー」が含まれていないことを .not で検証
  await expect(page.locator('#content')).not.toHaveText(/エラー/);
});
```

</details>

### 問題3：toHaveURLとtoHaveTitleの使い分け

Playwrightの公式サイト（`https://playwright.dev/`）にアクセスし、以下の検証を行うテストを作成してください。

1. URLが完全一致で正しいことを検証する
2. URLが正規表現で特定のパターンに一致することを検証する
3. タイトルが正規表現で部分一致することを検証する
4. 「Get started」をクリック後、URLとタイトルの両方が変わったことを検証する

**期待される結果:**
```
各アサーションが成功する
ページ遷移後のURLとタイトルが正しく変わっていることが確認される
```

<details>
<summary>ヒント</summary>

`toHaveURL()` は文字列（完全一致）と正規表現（パターン一致）の両方を受け付けます。`toHaveTitle()` も同様です。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('toHaveURLとtoHaveTitleの使い分け', async ({ page }) => {
  await page.goto('https://playwright.dev/');

  // 1. URLの完全一致検証
  await expect(page).toHaveURL('https://playwright.dev/');

  // 2. URLの正規表現による検証
  await expect(page).toHaveURL(/https:\/\/playwright\.dev\/?/);

  // 3. タイトルの正規表現による部分一致検証
  await expect(page).toHaveTitle(/Playwright/);

  // 4. ページ遷移後の検証
  await page.getByRole('link', { name: 'Get started' }).click();

  // URLが変わったことを検証
  await expect(page).toHaveURL(/.*\/docs\/intro/);
  // タイトルも変わったことを検証（ドキュメントページのタイトル）
  await expect(page).toHaveTitle(/Installation/);
});
```

</details>

### 問題4：beforeEachとdescribeを使ったテスト構造

以下のHTMLのToDoアプリに対して、`test.describe` と `test.beforeEach` を使って構造化されたテストを作成してください。

```html
<div id="app">
  <h1>ToDoリスト</h1>
  <input id="new-todo" type="text" placeholder="新しいタスクを入力..." />
  <button id="add-btn">追加</button>
  <ul id="todo-list"></ul>
  <p id="count">タスク数: 0</p>
</div>
<script>
  const input = document.getElementById('new-todo');
  const list = document.getElementById('todo-list');
  const countEl = document.getElementById('count');
  let count = 0;
  document.getElementById('add-btn').addEventListener('click', () => {
    if (input.value.trim()) {
      const li = document.createElement('li');
      li.textContent = input.value;
      const delBtn = document.createElement('button');
      delBtn.textContent = '削除';
      delBtn.onclick = () => { li.remove(); count--; countEl.textContent = 'タスク数: ' + count; };
      li.appendChild(delBtn);
      list.appendChild(li);
      count++;
      countEl.textContent = 'タスク数: ' + count;
      input.value = '';
    }
  });
</script>
```

テスト構造:
- `describe('ToDoアプリ')` でグループ化する
- `beforeEach` でページのセットアップを行う
- テスト1: タスクを追加できること
- テスト2: 複数タスクを追加してカウントが正しいこと
- テスト3: タスクを削除できること

**期待される結果:**
```
3つのテストがすべて成功する
各テストの前にbeforeEachが実行される
```

<details>
<summary>ヒント</summary>

`test.beforeEach` の中で `page.setContent()` や `page.goto()` を呼ぶと、各テストの前に共通のセットアップが行われます。`beforeEach` の引数にも `{ page }` を受け取れます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

// HTML文字列を定数として定義（beforeEach で使い回す）
const todoAppHtml = `
  <div id="app">
    <h1>ToDoリスト</h1>
    <input id="new-todo" type="text" placeholder="新しいタスクを入力..." />
    <button id="add-btn">追加</button>
    <ul id="todo-list"></ul>
    <p id="count">タスク数: 0</p>
  </div>
  <script>
    const input = document.getElementById('new-todo');
    const list = document.getElementById('todo-list');
    const countEl = document.getElementById('count');
    let count = 0;
    document.getElementById('add-btn').addEventListener('click', () => {
      if (input.value.trim()) {
        const li = document.createElement('li');
        li.textContent = input.value;
        const delBtn = document.createElement('button');
        delBtn.textContent = '削除';
        delBtn.onclick = () => { li.remove(); count--; countEl.textContent = 'タスク数: ' + count; };
        li.appendChild(delBtn);
        list.appendChild(li);
        count++;
        countEl.textContent = 'タスク数: ' + count;
        input.value = '';
      }
    });
  </script>
`;

test.describe('ToDoアプリ', () => {

  // 各テストの前にページをセットアップ
  test.beforeEach(async ({ page }) => {
    await page.setContent(todoAppHtml);
  });

  test('タスクを追加できる', async ({ page }) => {
    // 入力してボタンをクリック
    await page.getByPlaceholder('新しいタスクを入力...').fill('買い物に行く');
    await page.getByRole('button', { name: '追加' }).click();

    // タスクがリストに表示されることを検証
    await expect(page.locator('#todo-list li')).toHaveCount(1);
    await expect(page.locator('#todo-list li').first()).toContainText('買い物に行く');

    // 入力欄がクリアされていることを検証
    await expect(page.getByPlaceholder('新しいタスクを入力...')).toHaveValue('');
  });

  test('複数タスクを追加してカウントが正しい', async ({ page }) => {
    const tasks = ['買い物に行く', 'レポートを書く', '運動する'];

    for (const task of tasks) {
      await page.getByPlaceholder('新しいタスクを入力...').fill(task);
      await page.getByRole('button', { name: '追加' }).click();
    }

    // タスク数が3であることを検証
    await expect(page.locator('#todo-list li')).toHaveCount(3);
    // カウント表示を検証
    await expect(page.locator('#count')).toHaveText('タスク数: 3');
  });

  test('タスクを削除できる', async ({ page }) => {
    // まずタスクを追加
    await page.getByPlaceholder('新しいタスクを入力...').fill('削除するタスク');
    await page.getByRole('button', { name: '追加' }).click();
    await expect(page.locator('#todo-list li')).toHaveCount(1);

    // 削除ボタンをクリック
    await page.getByRole('button', { name: '削除' }).click();

    // タスクが削除されたことを検証
    await expect(page.locator('#todo-list li')).toHaveCount(0);
    await expect(page.locator('#count')).toHaveText('タスク数: 0');
  });
});
```

</details>

### 問題5：ソフトアサーションの活用

以下のHTMLのダッシュボード画面に対して、ソフトアサーション（`expect.soft`）を使って複数の要素を一度にチェックするテストを作成してください。通常のアサーションとソフトアサーションの違いを理解してください。

```html
<div class="dashboard">
  <header>
    <h1>ダッシュボード</h1>
    <span id="user-name">田中太郎</span>
  </header>
  <div class="stats">
    <div class="stat-card" data-testid="visitors">
      <h3>訪問者数</h3>
      <p class="value">1,234</p>
    </div>
    <div class="stat-card" data-testid="sales">
      <h3>売上</h3>
      <p class="value">¥567,890</p>
    </div>
    <div class="stat-card" data-testid="orders">
      <h3>注文数</h3>
      <p class="value">89</p>
    </div>
    <div class="stat-card" data-testid="conversion">
      <h3>コンバージョン率</h3>
      <p class="value">7.2%</p>
    </div>
  </div>
</div>
```

1. ソフトアサーションを使って4つのカードの値をすべてチェックする
2. 1つが失敗しても残りのチェックが続行されることを確認する
3. 通常のアサーションとの挙動の違いをコメントで説明する

**期待される結果:**
```
ソフトアサーション: 1つ失敗しても残りのアサーションが実行される
通常のアサーション: 最初の失敗で停止する
すべての結果がテストレポートにまとめて表示される
```

<details>
<summary>ヒント</summary>

`expect.soft(locator)` を使うとソフトアサーションになります。ソフトアサーションは失敗してもテストの実行を続行し、すべてのアサーションを実行した後にまとめて結果を報告します。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ソフトアサーションでダッシュボードを検証する', async ({ page }) => {
  await page.setContent(`
    <div class="dashboard">
      <header>
        <h1>ダッシュボード</h1>
        <span id="user-name">田中太郎</span>
      </header>
      <div class="stats">
        <div class="stat-card" data-testid="visitors">
          <h3>訪問者数</h3>
          <p class="value">1,234</p>
        </div>
        <div class="stat-card" data-testid="sales">
          <h3>売上</h3>
          <p class="value">¥567,890</p>
        </div>
        <div class="stat-card" data-testid="orders">
          <h3>注文数</h3>
          <p class="value">89</p>
        </div>
        <div class="stat-card" data-testid="conversion">
          <h3>コンバージョン率</h3>
          <p class="value">7.2%</p>
        </div>
      </div>
    </div>
  `);

  // --- ソフトアサーション ---
  // expect.soft() を使うと、アサーションが失敗しても次のアサーションに進む
  // すべてのアサーション結果がまとめてレポートされる

  // ヘッダーの検証
  await expect.soft(page.locator('#user-name')).toHaveText('田中太郎');

  // 各統計カードの値を検証（1つ失敗しても残りは実行される）
  await expect.soft(
    page.getByTestId('visitors').locator('.value')
  ).toHaveText('1,234');

  await expect.soft(
    page.getByTestId('sales').locator('.value')
  ).toHaveText('¥567,890');

  await expect.soft(
    page.getByTestId('orders').locator('.value')
  ).toHaveText('89');

  await expect.soft(
    page.getByTestId('conversion').locator('.value')
  ).toHaveText('7.2%');

  // --- 通常のアサーションとの違い ---
  // 通常の expect(): 失敗した時点でテストが即座に停止する
  //   → 最初のエラーしか分からない
  //
  // expect.soft(): 失敗してもテストの実行を続行する
  //   → すべてのエラーをまとめて確認できる
  //   → ダッシュボードのような複数項目の一括検証に最適
  //
  // 注意: ソフトアサーションが失敗しても、テスト結果はFAILになる
});
```

</details>

### 問題6：afterEachとテストのクリーンアップ

`test.afterEach` を使って、各テスト実行後にスクリーンショットを撮影するテストスイートを作成してください。また、`test.beforeAll` と `test.afterAll` の使い方も含めてください。

**期待される結果:**
```
各テスト後にスクリーンショットが撮影される
beforeAll/afterAll がスイート全体の前後で1回ずつ実行される
```

<details>
<summary>ヒント</summary>

`test.afterEach` はテスト結果に関わらず（成功しても失敗しても）実行されます。`testInfo` 引数からテスト名やステータスを取得できます。`test.beforeAll` / `test.afterAll` はスイート全体で1回だけ実行されます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test.describe('ライフサイクルフックのデモ', () => {

  // スイート全体の前に1回だけ実行される
  test.beforeAll(async () => {
    console.log('テストスイート開始');
    // 例: テストデータのセットアップ、DBの初期化など
  });

  // 各テストの前に実行される
  test.beforeEach(async ({ page }) => {
    await page.setContent(`
      <div>
        <h1>テストページ</h1>
        <p id="content">初期コンテンツ</p>
        <button id="change-btn">変更</button>
      </div>
      <script>
        document.getElementById('change-btn').onclick = () => {
          document.getElementById('content').textContent = '変更後のコンテンツ';
        };
      </script>
    `);
  });

  // 各テストの後に実行される（成功・失敗に関わらず）
  test.afterEach(async ({ page }, testInfo) => {
    // テスト名を使ったファイル名でスクリーンショットを撮影
    const screenshotName = `screenshots/${testInfo.title.replace(/\s+/g, '-')}.png`;
    await page.screenshot({ path: screenshotName });

    // テストが失敗した場合のログ出力
    if (testInfo.status !== testInfo.expectedStatus) {
      console.log(`テスト失敗: ${testInfo.title}`);
      console.log(`ステータス: ${testInfo.status}`);
    }
  });

  // スイート全体の後に1回だけ実行される
  test.afterAll(async () => {
    console.log('テストスイート終了');
    // 例: テストデータのクリーンアップなど
  });

  test('初期表示の確認', async ({ page }) => {
    await expect(page.locator('#content')).toHaveText('初期コンテンツ');
  });

  test('ボタンクリックで内容が変わる', async ({ page }) => {
    await page.getByRole('button', { name: '変更' }).click();
    await expect(page.locator('#content')).toHaveText('変更後のコンテンツ');
  });
});
```

</details>

### 問題7：test.skipとtest.onlyとタグの活用

以下の条件に基づいて、テストのスキップ、単独実行、タグ付けを活用したテストスイートを作成してください。

1. モバイル環境でのみ実行するテスト（デスクトップではスキップ）
2. `@smoke` タグ付きのスモークテスト
3. `@regression` タグ付きの回帰テスト
4. 未実装機能のテスト（`test.fixme` を使用）

**期待される結果:**
```
条件に応じてテストがスキップまたは実行される
タグを使ってテストをフィルタリングできる
- npx playwright test --grep @smoke  → スモークテストのみ実行
- npx playwright test --grep @regression → 回帰テストのみ実行
```

<details>
<summary>ヒント</summary>

`test.skip(condition, reason)` で条件付きスキップができます。タグはテスト名に `@tagname` を含めるか、`test('name', { tag: '@smoke' }, ...)` の形式で指定します。`test.fixme()` は未実装のテストをマークします。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test.describe('テストのスキップとタグ', () => {

  // 1. 条件付きスキップ: モバイル環境でのみ実行
  test('モバイルメニューの表示テスト', async ({ page, isMobile }) => {
    // デスクトップ環境ではスキップ
    test.skip(!isMobile, 'このテストはモバイル環境専用です');

    await page.goto('https://playwright.dev/');
    // モバイルメニュー（ハンバーガーメニュー）の検証
    const menuButton = page.getByRole('button', { name: 'Toggle navigation bar' });
    await expect(menuButton).toBeVisible();
  });

  // 2. @smoke タグ付きスモークテスト
  test('トップページの表示確認 @smoke', async ({ page }) => {
    await page.goto('https://playwright.dev/');
    await expect(page).toHaveTitle(/Playwright/);
```

### 問題8：タイムアウトの設定と制御

以下のHTMLに対して、タイムアウトの設定をテスト内で使い分けるテストを作成してください。

```html
<div id="app">
  <button id="slow-btn">処理開始</button>
  <p id="status">待機中</p>
</div>
<script>
  document.getElementById('slow-btn').addEventListener('click', () => {
    document.getElementById('status').textContent = '処理中...';
    setTimeout(() => {
      document.getElementById('status').textContent = '完了！';
    }, 3000);
  });
</script>
```

1. テスト全体のタイムアウトを設定する
2. 特定のアサーションにカスタムタイムアウトを設定する
3. `expect` のデフォルトタイムアウトを変更する方法を示す

**期待される結果:**
```
遅い処理（3秒かかる）を適切なタイムアウト設定で待機できる
タイムアウト設定の3つの方法が理解できる
```

<details>
<summary>ヒント</summary>

テスト全体のタイムアウトは `test.setTimeout()` で設定します。個別のアサーションには `toHaveText('テキスト', { timeout: ミリ秒 })` でタイムアウトを指定できます。`playwright.config.ts` の `expect.timeout` でデフォルトタイムアウトを変更できます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

// 方法1: テスト全体のタイムアウトを設定
test('遅い処理のテスト（テスト全体のタイムアウト）', async ({ page }) => {
  // このテストのタイムアウトを10秒に設定
  test.setTimeout(10000);

  await page.setContent(`
    <div id="app">
      <button id="slow-btn">処理開始</button>
      <p id="status">待機中</p>
    </div>
    <script>
      document.getElementById('slow-btn').addEventListener('click', () => {
        document.getElementById('status').textContent = '処理中...';
        setTimeout(() => {
          document.getElementById('status').textContent = '完了！';
        }, 3000);
      });
    </script>
  `);

  await page.getByRole('button', { name: '処理開始' }).click();

  // デフォルトの expect タイムアウト（5秒）で待機
  await expect(page.locator('#status')).toHaveText('完了！');
});

// 方法2: 個別のアサーションにタイムアウトを指定
test('遅い処理のテスト（個別タイムアウト）', async ({ page }) => {
  await page.setContent(`
    <div id="app">
      <button id="slow-btn">処理開始</button>
      <p id="status">待機中</p>
    </div>

      document.getElementById('slow-btn').addEventListener('click', () => {
        document.getElementById('status').textContent = '処理中...';
        setTimeout(() => {
          document.getElementById('status').textContent = '完了！';
        }, 3000);
      });
    </script>
  `);

  await page.getByRole('button', { name: '処理開始' }).click();

  // このアサーションだけタイムアウトを5秒に設定
  await expect(page.locator('#status')).toHaveText('完了！', {
    timeout: 5000,
  });

  // 中間状態の検証（すぐに確認したい場合はタイムアウトを短くする）
  // ※ この例では処理完了後なので意味はないが、パターンとして示す
  await expect(page.locator('#status')).not.toHaveText('処理中...', {
    timeout: 1000,
  });
});

// 方法3: playwright.config.ts でデフォルトを変更
// ※ これは設定ファイルの例（テストファイル内ではなく playwright.config.ts に記述）
/*
import { defineConfig } from '@playwright/test';

export default defineConfig({
  // expect のデフォルトタイムアウトを10秒に変更

    timeout: 10000,
  },
  // テスト全体のデフォルトタイムアウトを30秒に変更
  timeout: 30000,
  // アクション（click, fillなど）のデフォルトタイムアウト
  use: {
    actionTimeout: 5000,
    navigationTimeout: 10000,
  },
});
*/
```

### 問題9：パラメタライズドテスト

以下の要件を満たすパラメタライズドテスト（データ駆動テスト）を作成してください。

複数のURL・タイトルのペアに対して同じテストロジックを繰り返し実行する。

テストデータ:
| URL | 期待されるタイトル（部分一致） |
|---|---|
| `https://playwright.dev/` | Playwright |
| `https://playwright.dev/docs/intro` | Installation |
| `https://playwright.dev/docs/api/class-page` | Page |

**期待される結果:**
```
3つのURLに対してそれぞれテストが生成される
各URLでタイトルが正しいことが検証される
テスト名にURLが含まれ、どのテストか識別できる
```

<details>
<summary>ヒント</summary>

配列の `forEach` を使って `test()` を動的に生成できます。テストデータの配列を定義し、各エントリに対して `test()` を呼び出します。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

// テストデータの定義
const testCases = [
  {
    url: 'https://playwright.dev/',
    expectedTitle: /Playwright/,
    description: 'トップページ',
  },
  {
    url: 'https://playwright.dev/docs/intro',
    expectedTitle: /Installation/,
    description: 'インストールガイド',
  },
  {
    url: 'https://playwright.dev/docs/api/class-page',
    expectedTitle: /Page/,
    description: 'Page APIリファレンス',
  },
];

test.describe('各ページのタイトル検証（パラメタライズド）', () => {

  // forEach でテストデータごとにテストを生成
  testCases.forEach(({ url, expectedTitle, description }) => {
    test(`${description}（${url}）のタイトルを検証する`, async ({ page }) => {
      // 各URLにアクセス
      await page.goto(url);

      // タイトルの検証
      await expect(page).toHaveTitle(expectedTitle);

      // ページが正しく読み込まれたことの追加検証
      await expect(page.getByRole('heading', { level: 1 })).toBeVisible();
    });
  });
});

// --- 別のパターン: 配列をシンプルに使う ---
const urls = [
  'https://playwright.dev/',
  'https://playwright.dev/docs/intro',
  'https://playwright.dev/docs/api/class-page',
];

test.describe('各ページのアクセス確認', () => {
  for (const url of urls) {
    test(`${url} にアクセスできる`, async ({ page }) => {
      const response = await page.goto(url);
      // HTTPステータスコードが200であることを検証
      expect(response?.status()).toBe(200);
      // ページのh1が表示されていることを検証
      await expect(page.getByRole('heading', { level: 1 })).toBeVisible();
    });
  }
});
```

</details>

### 問題10：ジェネリックアサーションとカスタム検証

以下のHTMLに対して、Webアサーションだけでなくジェネリックアサーション（`expect(値)` 形式）も組み合わせてテストを作成してください。

```html
<div id="app">
  <table id="user-table">
    <thead>
      <tr><th>名前</th><th>年齢</th><th>メール</th></tr>
    </thead>
    <tbody>
      <tr><td>田中</td><td>25</td><td>tanaka@example.com</td></tr>
      <tr><td>鈴木</td><td>30</td><td>suzuki@example.com</td></tr>
      <tr><td>佐藤</td><td>28</td><td>sato@example.com</td></tr>
    </tbody>
  </table>
</div>
```

1. テーブルの行数が3行であることを検証する（ヘッダー除く）
2. 全員の年齢が20以上であることをジェネリックアサーションで検証する
3. メールアドレスがすべて `@example.com` で終わることを検証する
4. 名前の一覧を配列として取得し、期待値と一致することを検証する

**期待される結果:**
```
テーブルのデータが正しいことが多角的に検証される
ジェネリックアサーション（toBe, toContain, toEqual等）が使われている
```

<details>
<summary>ヒント</summary>

`locator.allTextContents()` で全要素のテキストを配列で取得できます。取得した値に対して `expect(value).toBe()` や `expect(array).toEqual()` などのジェネリックアサーションを使います。`toBeGreaterThanOrEqual()` で数値の範囲チェックができます。

</details>

<details>
<summary>解答例</summary>

```typescript
import { test, expect } from '@playwright/test';

test('ジェネリックアサーションでテーブルデータを検証する', async ({ page }) => {
  await page.setContent(`
    <div id="app">
      <table id="user-table">
        <thead>
          <tr><th>名前</th><th>年齢</th><th>メール</th></tr>
        </thead>
        <tbody>
          <tr><td>田中</td><td>25</td><td>tanaka@example.com</td></tr>
          <tr><td>鈴木</td><td>30</td><td>suzuki@example.com</td></tr>
          <tr><td>佐藤</td><td>28</td><td>sato@example.com</td></tr>
        </tbody>
      </table>
    </div>
  `);

  // 1. テーブルの行数を検証（Webアサーション）
  const rows = page.locator('#user-table tbody tr');
  await expect(rows).toHaveCount(3);

  // 2. 全員の年齢が20以上であることを検証（ジェネリックアサーション）
  const ageTexts = await page.locator('#user-table tbody tr td:nth-child(2)').allTextContents();
  for (const ageText of ageTexts) {
    const age = parseInt(ageText, 10);
    // ジェネリックアサーション: 数値が20以上であること
    expect(age).toBeGreaterThanOrEqual(20);
  }

  // 3. メールアドレスがすべて @example.com で終わること（ジェネリックアサーション）
  const emails = await page.locator('#user-table tbody tr td:nth-child(3)').allTextContents();
  for (const email of emails) {
    // ジェネリックアサーション: 文字列が特定のパターンに一致すること
    expect(email).toMatch(/@example\.com$/);
  }

  // 4. 名前の一覧を配列として検証（ジェネリックアサーション）
  const names = await page.locator('#user-table tbody tr td:nth-child(1)').allTextContents();
  // toEqual で配列の完全一致を検証
  expect(names).toEqual(['田中', '鈴木', '佐藤']);

  // 追加: 配列に特定の要素が含まれることを検証
  expect(names).toContain('鈴木');

  // 追加: 配列の長さを検証
  expect(names).toHaveLength(3);
});
```

</details>
