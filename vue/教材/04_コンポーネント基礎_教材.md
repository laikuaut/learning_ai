# 第4章：コンポーネント基礎

## 4.1 コンポーネントとは

コンポーネントは、テンプレート（HTML）、ロジック（JavaScript）、スタイル（CSS）をカプセル化した**再利用可能なUI部品**です。

```
App.vue
├── Header.vue
│   ├── Logo.vue
│   └── Navigation.vue
├── MainContent.vue
│   ├── ArticleCard.vue
│   └── Sidebar.vue
└── Footer.vue
```

### なぜコンポーネントを使うのか

1. **再利用性** — 同じUIを複数箇所で使い回せる
2. **保守性** — 小さな単位で管理でき、修正が容易
3. **可読性** — 各部品の責務が明確になる
4. **テスタビリティ** — 単体テストが書きやすい

## 4.2 単一ファイルコンポーネント（SFC）

Vue のコンポーネントは `.vue` ファイルとして記述します。

```vue
<!-- components/GreetingCard.vue -->
<script setup>
// JavaScript ロジック
import { ref } from 'vue'

const isExpanded = ref(false)
</script>

<template>
  <!-- HTML テンプレート -->
  <div class="card">
    <h2>ようこそ！</h2>
    <button @click="isExpanded = !isExpanded">
      {{ isExpanded ? '閉じる' : '詳細を見る' }}
    </button>
    <p v-show="isExpanded">
      Vue.js 3 のコンポーネントシステムへようこそ。
    </p>
  </div>
</template>

<style scoped>
/* スコープ付きCSS（このコンポーネントだけに適用） */
.card {
  border: 1px solid #ddd;
  border-radius: 8px;
  padding: 1rem;
  margin: 1rem 0;
}

h2 {
  color: #42b883;
}
</style>
```

### SFC の3つのブロック

| ブロック | 役割 | 必須 |
|----------|------|------|
| `<script setup>` | ロジック（Composition API） | いいえ |
| `<template>` | テンプレート（HTML） | はい |
| `<style scoped>` | スコープ付きスタイル | いいえ |

## 4.3 コンポーネントの登録と使用

### ローカル登録（推奨）

`<script setup>` では、インポートするだけで自動的に使えます。

```vue
<!-- App.vue -->
<script setup>
// インポートするだけで登録完了
import GreetingCard from './components/GreetingCard.vue'
import UserProfile from './components/UserProfile.vue'
</script>

<template>
  <div>
    <!-- コンポーネントをHTMLタグのように使う -->
    <GreetingCard />
    <UserProfile />

    <!-- 複数回使える -->
    <GreetingCard />
  </div>
</template>
```

### グローバル登録

アプリケーション全体で使うコンポーネント（アイコン、ボタンなど）に適しています。

```js
// main.js
import { createApp } from 'vue'
import App from './App.vue'
import BaseButton from './components/BaseButton.vue'
import BaseIcon from './components/BaseIcon.vue'

const app = createApp(App)

// グローバル登録（どのコンポーネントからも import なしで使える）
app.component('BaseButton', BaseButton)
app.component('BaseIcon', BaseIcon)

app.mount('#app')
```

### 命名規則

```vue
<template>
  <!-- PascalCase（推奨: SFC ではこちらが標準） -->
  <MyComponent />
  <UserProfile />

  <!-- kebab-case（DOM テンプレートではこちらを使う） -->
  <my-component></my-component>
  <user-profile></user-profile>
</template>
```

## 4.4 Props — 親から子へのデータ受け渡し

### defineProps の基本

```vue
<!-- components/UserCard.vue（子コンポーネント） -->
<script setup>
// defineProps でプロパティを定義（import 不要）
const props = defineProps({
  // 型のみ指定
  name: String,
  age: Number,

  // 詳細な定義
  email: {
    type: String,
    required: true    // 必須
  },
  role: {
    type: String,
    default: '一般'   // デフォルト値
  },
  isActive: {
    type: Boolean,
    default: true
  },
  tags: {
    type: Array,
    default: () => []  // オブジェクト/配列は関数で返す
  }
})

// props は読み取り専用
console.log(props.name)
// props.name = '花子'  // ❌ エラー: props は変更不可
</script>

<template>
  <div class="user-card" :class="{ inactive: !isActive }">
    <h3>{{ name }}</h3>
    <p>{{ age }}歳 / {{ email }}</p>
    <span class="role">{{ role }}</span>
    <div v-if="tags.length > 0">
      <span v-for="tag in tags" :key="tag" class="tag">{{ tag }}</span>
    </div>
  </div>
</template>
```

### 親コンポーネントからの渡し方

```vue
<!-- App.vue（親コンポーネント） -->
<script setup>
import { ref } from 'vue'
import UserCard from './components/UserCard.vue'

const userName = ref('太郎')
const userAge = ref(20)
</script>

<template>
  <!-- 静的な値 -->
  <UserCard
    name="花子"
    :age="25"
    email="hanako@example.com"
    role="管理者"
    :tags="['Vue', 'TypeScript']"
  />

  <!-- 動的な値（v-bind） -->
  <UserCard
    :name="userName"
    :age="userAge"
    email="taro@example.com"
    :is-active="true"
  />

  <!-- Boolean の略記法 -->
  <UserCard name="次郎" :age="18" email="jiro@example.com" is-active />
  <!-- ↑ is-active だけ書くと true になる -->
</template>
```

### Props の型バリデーション

```js
defineProps({
  // 基本的な型チェック
  propA: Number,
  propB: String,
  propC: Boolean,
  propD: Array,
  propE: Object,
  propF: Function,
  propG: Date,
  propH: Symbol,

  // 複数の型を許可
  propI: [String, Number],

  // カスタムバリデーション
  score: {
    type: Number,
    validator(value) {
      return value >= 0 && value <= 100
    }
  }
})
```

### TypeScript での Props 定義

```vue
<script setup lang="ts">
// TypeScript の型で定義（推奨）
interface Props {
  name: string
  age: number
  email: string
  role?: string        // オプショナル
  isActive?: boolean
  tags?: string[]
}

const props = withDefaults(defineProps<Props>(), {
  role: '一般',
  isActive: true,
  tags: () => []
})
</script>
```

## 4.5 Emits — 子から親へのイベント通知

### defineEmits の基本

```vue
<!-- components/CounterButton.vue（子） -->
<script setup>
// defineEmits でイベントを定義
const emit = defineEmits(['increment', 'decrement', 'reset'])

function handleIncrement() {
  emit('increment')  // 親にイベントを通知
}

function handleDecrement() {
  emit('decrement')
}

function handleReset() {
  emit('reset', 0)  // データ（ペイロード）も渡せる
}
</script>

<template>
  <div class="counter-buttons">
    <button @click="handleIncrement">+1</button>
    <button @click="handleDecrement">-1</button>
    <button @click="handleReset">リセット</button>
  </div>
</template>
```

```vue
<!-- App.vue（親） -->
<script setup>
import { ref } from 'vue'
import CounterButton from './components/CounterButton.vue'

const count = ref(0)

// 子コンポーネントからのイベントを受け取る
function onIncrement() {
  count.value++
}

function onDecrement() {
  count.value--
}

function onReset(value) {
  count.value = value
}
</script>

<template>
  <p>カウント: {{ count }}</p>
  <CounterButton
    @increment="onIncrement"
    @decrement="onDecrement"
    @reset="onReset"
  />
</template>
```

### バリデーション付き Emits

```vue
<script setup>
const emit = defineEmits({
  // バリデーション関数
  submit(payload) {
    // true を返すとイベントが発火、false で警告
    if (payload.email && payload.password) {
      return true
    }
    console.warn('email と password は必須です')
    return false
  },
  // バリデーションなし
  cancel: null
})

function handleSubmit() {
  emit('submit', {
    email: 'user@example.com',
    password: 'secret'
  })
}
</script>
```

### TypeScript での Emits 定義

```vue
<script setup lang="ts">
const emit = defineEmits<{
  (e: 'increment'): void
  (e: 'update', value: number): void
  (e: 'submit', payload: { email: string; password: string }): void
}>()
</script>
```

## 4.6 コンポーネントの v-model

親子間で双方向バインディングを実現します。

```vue
<!-- components/CustomInput.vue（子） -->
<script setup>
// modelValue プロパティと update:modelValue イベントで v-model を実現
defineProps({
  modelValue: String
})

const emit = defineEmits(['update:modelValue'])

function onInput(event) {
  emit('update:modelValue', event.target.value)
}
</script>

<template>
  <input :value="modelValue" @input="onInput" class="custom-input" />
</template>
```

```vue
<!-- App.vue（親） -->
<script setup>
import { ref } from 'vue'
import CustomInput from './components/CustomInput.vue'

const username = ref('')
</script>

<template>
  <!-- v-model で双方向バインディング -->
  <CustomInput v-model="username" />
  <p>入力値: {{ username }}</p>
</template>
```

### 名前付き v-model（複数の v-model）

```vue
<!-- components/UserForm.vue -->
<script setup>
defineProps({
  firstName: String,
  lastName: String
})

const emit = defineEmits(['update:firstName', 'update:lastName'])
</script>

<template>
  <input
    :value="firstName"
    @input="emit('update:firstName', $event.target.value)"
    placeholder="名"
  />
  <input
    :value="lastName"
    @input="emit('update:lastName', $event.target.value)"
    placeholder="姓"
  />
</template>
```

```vue
<!-- 親コンポーネント -->
<template>
  <UserForm
    v-model:first-name="first"
    v-model:last-name="last"
  />
</template>
```

### defineModel（Vue 3.4+）

Vue 3.4 以降では `defineModel` でさらに簡潔に書けます。

```vue
<!-- components/CustomInput.vue -->
<script setup>
// defineModel で v-model を簡潔に定義
const model = defineModel()
</script>

<template>
  <input v-model="model" class="custom-input" />
</template>
```

```vue
<!-- 名前付き defineModel -->
<script setup>
const firstName = defineModel('firstName')
const lastName = defineModel('lastName')
</script>

<template>
  <input v-model="firstName" placeholder="名" />
  <input v-model="lastName" placeholder="姓" />
</template>
```

## 4.7 スロット — コンテンツの差し込み

スロットを使うと、親コンポーネントから子コンポーネントにHTMLコンテンツを差し込めます。

### デフォルトスロット

```vue
<!-- components/Card.vue -->
<template>
  <div class="card">
    <slot>
      <!-- フォールバック（何も渡されなかった場合のデフォルト） -->
      <p>コンテンツがありません</p>
    </slot>
  </div>
</template>
```

```vue
<!-- 使用側 -->
<template>
  <Card>
    <h2>タイトル</h2>
    <p>カスタムコンテンツ</p>
  </Card>

  <!-- 何も渡さないとフォールバックが表示される -->
  <Card />
</template>
```

### 名前付きスロット

```vue
<!-- components/PageLayout.vue -->
<template>
  <div class="layout">
    <header>
      <slot name="header">デフォルトヘッダー</slot>
    </header>
    <main>
      <slot>デフォルトコンテンツ</slot>
    </main>
    <footer>
      <slot name="footer">デフォルトフッター</slot>
    </footer>
  </div>
</template>
```

```vue
<!-- 使用側 -->
<template>
  <PageLayout>
    <!-- v-slot:name または省略記法 #name -->
    <template #header>
      <h1>マイサイト</h1>
    </template>

    <!-- デフォルトスロット -->
    <p>メインコンテンツ</p>

    <template #footer>
      <p>&copy; 2026 マイサイト</p>
    </template>
  </PageLayout>
</template>
```

### スコープ付きスロット

子コンポーネントのデータを親のスロットコンテンツで使用できます。

```vue
<!-- components/ItemList.vue -->
<script setup>
defineProps({
  items: Array
})
</script>

<template>
  <ul>
    <li v-for="(item, index) in items" :key="item.id">
      <!-- スロットにデータを渡す -->
      <slot :item="item" :index="index">
        {{ item.name }}
      </slot>
    </li>
  </ul>
</template>
```

```vue
<!-- 使用側: スロットからデータを受け取ってカスタム表示 -->
<template>
  <ItemList :items="products">
    <template #default="{ item, index }">
      <span>{{ index + 1 }}. </span>
      <strong>{{ item.name }}</strong>
      <span> — {{ item.price }}円</span>
    </template>
  </ItemList>
</template>
```

## 4.8 コンポーネント設計のベストプラクティス

### 命名規則

```
components/
├── BaseButton.vue       # 基礎UIコンポーネント（Base/App/V プレフィックス）
├── BaseCard.vue
├── BaseIcon.vue
├── TheHeader.vue        # 1度だけ使うコンポーネント（The プレフィックス）
├── TheSidebar.vue
├── UserProfile.vue      # 機能別コンポーネント
├── UserAvatar.vue
├── TodoList.vue
└── TodoItem.vue
```

### 単一責任の原則

```
✅ 良い設計:
- UserAvatar.vue    → アバター画像の表示のみ
- UserProfile.vue   → ユーザー情報の表示
- UserEditForm.vue  → ユーザー情報の編集

❌ 悪い設計:
- User.vue          → すべてのユーザー関連処理を1つに詰め込む
```

### Props の流れ（単方向データフロー）

```
親 ──props──→ 子
親 ←──emits── 子

❌ 子コンポーネントが props を直接変更してはいけない
✅ emit で親に通知し、親が自身のデータを更新する
```

## 4.9 本章のまとめ

| 概念 | 説明 |
|------|------|
| SFC（.vue） | script + template + style を1ファイルに |
| `defineProps` | 親から子へのデータ受け渡し |
| `defineEmits` | 子から親へのイベント通知 |
| `defineModel` | v-model の簡潔な定義（Vue 3.4+） |
| スロット | 親から子へのコンテンツ差し込み |
| `scoped` | コンポーネントスコープのCSS |
| 単方向データフロー | props は下へ、events は上へ |
