## 基本問題

### 問題1：見積もり手法の選択

以下のシナリオで最も適切な見積もり手法（アナロジー / パラメトリック / ボトムアップ / 三点見積もり）を選び、理由を述べてください。

1. 営業から「来週までに概算見積もりが欲しい」と言われた。過去に似たECサイト構築の実績がある。
2. 要件定義が完了し、WBSも作成済み。正式な予算を策定する段階。
3. 新技術（AI搭載チャットボット）を使うため、工数の不確実性が高い。
4. 自社で50件以上のWebアプリ開発実績があり、画面数と工数の相関データが蓄積されている。

**期待される出力例：**

```
1. アナロジー見積もり - 短期間で概算を出す必要があり、類似実績がある
```

<details>
<summary>解答例</summary>

```
1. アナロジー見積もり（類推見積もり）
   理由：時間が限られており（来週まで）、詳細な分析は困難。
   過去の類似ECサイトの実績データを基に、規模・複雑さの差分を
   調整して概算を算出するのが最適。

2. ボトムアップ見積もり
   理由：WBSが作成済みで、各作業パッケージの詳細が把握できている。
   各タスクを個別に見積もって積み上げることで、最も精度の高い
   予算を策定できる。正式予算の策定に適している。

3. 三点見積もり（PERT見積もり）
   理由：新技術の不確実性が高いため、楽観値・最頻値・悲観値の
   3つの値で見積もることでリスクを定量化できる。
   単一値の見積もりでは不確実性を表現できない。

4. パラメトリック見積もり
   理由：50件以上の実績データから「画面数 × 単位コスト」という
   統計モデルを構築できる。データに基づく客観的な見積もりが可能。
```

</details>

### 問題2：EVM（アーンドバリューマネジメント）の計算

以下のプロジェクトデータからEVMの各指標を算出し、プロジェクトの健全性を評価してください。

```
プロジェクト概要：
  総予算（BAC）: 2,400万円
  期間: 6ヶ月
  現在: 4ヶ月目の末日

EVMデータ：
  PV（計画価値）: 1,600万円
  EV（出来高）  : 1,200万円
  AC（実コスト）: 1,500万円
```

以下を求めてください：
1. CV（コスト差異）
2. SV（スケジュール差異）
3. CPI（コスト効率指数）
4. SPI（スケジュール効率指数）
5. EAC（完了時見積もりコスト）
6. プロジェクトの状態を総合評価

**期待される出力例：**

```
CV = -300万円（予算超過）
SV = -400万円（スケジュール遅延）
```

<details>
<summary>ヒント</summary>

計算式：
- CV = EV - AC
- SV = EV - PV
- CPI = EV / AC
- SPI = EV / PV
- EAC = BAC / CPI

</details>

<details>
<summary>解答例</summary>

```
■ 差異分析

CV（コスト差異）= EV - AC = 1,200 - 1,500 = -300万円
→ 300万円の予算超過

SV（スケジュール差異）= EV - PV = 1,200 - 1,600 = -400万円
→ 400万円分のスケジュール遅延

■ 効率指標

CPI（コスト効率指数）= EV / AC = 1,200 / 1,500 = 0.80
→ 1円あたり0.80円分の成果しか出ていない（コスト非効率）

SPI（スケジュール効率指数）= EV / PV = 1,200 / 1,600 = 0.75
→ 計画の75%しか進んでいない（スケジュール遅延）

■ 完了時見積もり

EAC（完了時見積もりコスト）= BAC / CPI = 2,400 / 0.80 = 3,000万円
→ このままのペースだと3,000万円かかる見込み（600万円超過）

VAC（完了時コスト差異）= BAC - EAC = 2,400 - 3,000 = -600万円
→ 最終的に600万円の予算超過が予想される

■ 総合評価

このプロジェクトは「コスト超過」かつ「スケジュール遅延」の状態です。

CPI = 0.80 → 予算の使い方が非効率。1円あたり80銭分の成果
SPI = 0.75 → 計画の75%しか完了していない

【アクション】
1. コスト超過の原因を調査（手戻り？見積もり不足？）
2. スケジュール遅延の原因を特定（リソース不足？技術的課題？）
3. このままでは600万円超過の見込みをスポンサーに報告
4. 是正措置（スコープ調整、リソース追加等）を検討
```

</details>

### 問題3：ボトムアップ見積もりの実施

以下のWBSに基づいて、ボトムアップ見積もりを行ってください。人月単価は80万円とします。

```
社内チャットアプリ開発：
├── 要件定義
│   ├── ユーザーヒアリング
│   └── 要件書作成
├── 設計
│   ├── DB設計
│   ├── API設計
│   └── UI/UXデザイン
├── 実装
│   ├── ユーザー管理機能
│   ├── チャット機能（1対1）
│   ├── グループチャット機能
│   └── ファイル共有機能
├── テスト
│   ├── 単体テスト
│   ├── 統合テスト
│   └── UAT（ユーザー受入テスト）
└── PM・管理
```

各作業の人月を見積もり、合計コスト（予備費10%含む）を算出してください。

<details>
<summary>ヒント</summary>

各タスクの人月を見積もる際は、1つの作業パッケージが0.5〜3人月程度になるようにしましょう。実装が最もコストがかかります。

</details>

<details>
<summary>解答例</summary>

```
■ ボトムアップ見積もり（人月単価: 80万円）

社内チャットアプリ開発：
├── 要件定義                           小計: 2.5人月
│   ├── ユーザーヒアリング       : 1.0人月 =  80万円
│   └── 要件書作成               : 1.5人月 = 120万円
├── 設計                               小計: 3.5人月
│   ├── DB設計                   : 1.0人月 =  80万円
│   ├── API設計                  : 1.0人月 =  80万円
│   └── UI/UXデザイン            : 1.5人月 = 120万円
├── 実装                               小計: 10.0人月
│   ├── ユーザー管理機能         : 2.0人月 = 160万円
│   ├── チャット機能（1対1）     : 3.0人月 = 240万円
│   ├── グループチャット機能     : 3.0人月 = 240万円
│   └── ファイル共有機能         : 2.0人月 = 160万円
├── テスト                             小計: 3.0人月
│   ├── 単体テスト               : 1.0人月 =  80万円
│   ├── 統合テスト               : 1.0人月 =  80万円
│   └── UAT                     : 1.0人月 =  80万円
└── PM・管理                    : 3.0人月 = 240万円
                                         ──────────
                        合計: 22.0人月 = 1,760万円
                  + 予備費(10%):          176万円
                           ──────────
                        総計:            1,936万円

■ 見積もりの根拠（補足）
・チャット機能はリアルタイム通信（WebSocket）が必要で複雑度が高い
・PM・管理は全期間（約5.5ヶ月）の0.5人月/月で計算
・予備費10%は技術的な不確実性への対応として設定
```

</details>

### 問題4：コスト超過への対処

以下のシナリオを読んで、コスト超過への対処案を3つ提示してください。

**シナリオ：** 6ヶ月のWebアプリ開発プロジェクト（予算3,000万円）で、4ヶ月目にEVM分析を行ったところ、CPI=0.85、SPI=0.90でした。このまま進むと3,529万円（529万円超過）かかる見込みです。

<details>
<summary>解答例</summary>

```
【現状分析】
・CPI = 0.85 → コスト効率が15%悪い
・SPI = 0.90 → スケジュールが10%遅延
・EAC = BAC / CPI = 3,000 / 0.85 ≈ 3,529万円（529万円超過見込み）

【対処案】

■ 対策1：スコープの縮小（推奨度：高）
  内容：優先度の低い機能をフェーズ2に延期する。
  具体的には：
  ・MoSCoWで再評価し、Could/Won'tの機能を洗い出す
  ・POと協議し、500万円分相当の機能を次フェーズに移動
  メリット：
  ・確実にコストを削減できる
  ・品質を維持したまま予算内に収められる
  デメリット：
  ・ステークホルダーの期待値調整が必要
  ・「機能が減った」という印象を与える可能性

■ 対策2：生産性の改善（推奨度：中）
  内容：開発プロセスを見直し、効率を上げる。
  具体的には：
  ・手戻りの原因を分析し、コードレビュープロセスを改善
  ・自動テストを拡充して手動テスト工数を削減
  ・技術的負債の解消で実装速度を向上
  メリット：
  ・スコープを維持したまま改善できる可能性がある
  ・チームの成長にもつながる
  デメリット：
  ・効果が出るまでに時間がかかる
  ・改善幅が読めない

■ 対策3：予備費・管理予備の活用（推奨度：状況次第）
  内容：プロジェクトの予備費（コンティンジェンシーリザーブ）や
  組織の管理予備（マネジメントリザーブ）を活用する。
  具体的には：
  ・予備費の残額を確認する
  ・スポンサーに管理予備の使用を申請する
  メリット：
  ・スコープもスケジュールも維持できる
  ・正当な理由があれば承認されやすい
  デメリット：
  ・予備費が不足している場合は使えない
  ・管理予備の承認が得られない可能性がある

【推奨アクション】
1. まず対策1でスコープの見直しを実施（影響の少ない機能をフェーズ2へ）
2. 並行して対策2で生産性改善を推進
3. それでも不足する分は対策3で予備費を申請
4. スポンサーへの月次報告でEVM分析結果と対策を説明
```

</details>

## チャレンジ問題

### 問題5：EVM総合演習

以下のプロジェクトデータを使って、全指標を計算し、グラフをアスキーアートで描いてください。

```
予算（BAC）: 1,000万円
期間: 5ヶ月

月次データ：
| 月 | PV（累計） | EV（累計） | AC（累計） |
|----|-----------|-----------|-----------|
| 1  | 100       | 80        | 90        |
| 2  | 250       | 220       | 240       |
| 3  | 500       | 430       | 480       |
| 4  | 750       | 620       | 700       |
| 5  | 1,000     | ???       | ???       |
```

4ヶ月目時点で以下を求めてください：
1. CV, SV, CPI, SPI
2. EAC（完了時見積もりコスト）
3. ETC（残作業コスト見積もり）
4. 現在のトレンドが続いた場合、予算内に収まるか判断

<details>
<summary>解答例</summary>

```
■ 4ヶ月目の指標計算

CV = EV - AC = 620 - 700 = -80万円（予算超過）
SV = EV - PV = 620 - 750 = -130万円（スケジュール遅延）
CPI = EV / AC = 620 / 700 = 0.886
SPI = EV / PV = 620 / 750 = 0.827

■ 完了時予測

EAC = BAC / CPI = 1,000 / 0.886 ≈ 1,129万円
ETC = EAC - AC = 1,129 - 700 = 429万円
VAC = BAC - EAC = 1,000 - 1,129 = -129万円

■ 判断
このトレンドが続くと約129万円の予算超過となり、
予算内には収まらない。

■ アスキーアート（累計コスト推移）

万円
1100 |                                    ╱ EAC予測
1000 |                              ★ ╱ BAC
 900 |                           ╱ ╱
 800 |                        ╱ ╱
 700 |                     ● ╱     ← AC(実コスト)
 600 |                  ○╱         ← EV(出来高)
 500 |              ╱ ╱
 400 |           ╱╱
 300 |        ╱╱
 200 |     ╱╱
 100 |  ╱╱
   0 +-------+-------+-------+-------+-------→ 月
     0       1       2       3       4       5

  ╱ = PV（計画値）
  ● = AC（実コスト） → PVの上を推移（予算超過）
  ○ = EV（出来高）   → PVの下を推移（遅延）

■ 月ごとの差異分析
| 月 | CV    | SV    | CPI  | SPI  | 傾向    |
|----|-------|-------|------|------|---------|
| 1  | -10   | -20   | 0.89 | 0.80 | 要注意  |
| 2  | -20   | -30   | 0.92 | 0.88 | やや改善|
| 3  | -50   | -70   | 0.90 | 0.86 | 悪化    |
| 4  | -80   | -130  | 0.89 | 0.83 | 悪化    |

→ トレンドが改善していないため、早急な是正措置が必要
```

</details>
