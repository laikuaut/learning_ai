## 基本問題

### 問題1：生成AIと識別AIの分類

以下のタスクが「生成AI（Generative AI）」と「識別AI（Discriminative AI）」のどちらに該当するか分類してください。

1. 顧客レビューのテキストからポジティブ／ネガティブを判定する
2. テキストの指示から商品紹介用の画像を作成する
3. 受信メールをスパムか正常かに振り分ける
4. 会議の録音データから議事録を自動生成する
5. 製造ラインの画像から不良品を検出する
6. ユーザーの質問に対して自然な回答文を出力する

**期待される出力例：**
```
1. 識別AI（テキスト分類タスク）
2. 生成AI（テキストから画像を生成）
3. 識別AI（二値分類タスク）
4. 生成AI（音声からテキストを生成）
5. 識別AI（画像分類・異常検知タスク）
6. 生成AI（テキスト生成タスク）
```

<details>
<summary>解答例</summary>

```
1. 識別AI（テキスト分類タスク）
   → 既存のデータを分類・判定するタスクです。
     感情分析（Sentiment Analysis）と呼ばれる典型的な識別タスクです。

2. 生成AI（テキストから画像を生成）
   → 入力テキストから新しい画像コンテンツを生成します。
     DALL-E、Midjourney、Stable Diffusion等が代表的なサービスです。

3. 識別AI（二値分類タスク）
   → スパムか否かの2択に分類するタスクです。
     新しいコンテンツを作るのではなく、既存データにラベルを付けます。

4. 生成AI（音声からテキストを生成）
   → 音声データから新たなテキストコンテンツを生成します。
     Whisper等の音声認識モデルが活用されます。

5. 識別AI（画像分類・異常検知タスク）
   → 画像を「良品」「不良品」に分類するタスクです。
     生成ではなく判定が目的です。

6. 生成AI（テキスト生成タスク）
   → 入力に基づいて新しいテキストを生成するタスクです。
     ChatGPT、Claude等の大規模言語モデル（LLM）が該当します。
```
</details>

### 問題3：Transformerアーキテクチャの基本概念

以下の説明文の空欄（A）〜（E）に当てはまる用語を選択肢から選んでください。

> Transformerは2017年にGoogleが発表した論文「（A）」で提案されたアーキテクチャです。
> 従来のRNNと異なり、（B）メカニズムにより入力系列全体を同時に参照できるため、
> 並列処理が可能になりました。入力テキストは（C）という単位に分割されてから処理されます。
> モデルは大量のテキストデータで（D）された後、特定のタスクに合わせて（E）されます。

**選択肢：**
トークン / Attention Is All You Need / Self-Attention（自己注意） / 事前学習（Pre-training） / ファインチューニング（Fine-tuning）

**期待される出力例：**
```
A: Attention Is All You Need
B: Self-Attention（自己注意）
C: トークン
D: 事前学習（Pre-training）
E: ファインチューニング（Fine-tuning）
```

<details>
<summary>ヒント</summary>

- （A）はTransformerを提案した論文のタイトルです。AIの歴史で最も引用される論文の1つです。
- （B）は入力データの各要素が他の全要素との関連度を計算する仕組みです。
- （C）は文章を処理可能な最小単位に分割したものです。単語より細かい場合もあります。
</details>

<details>
<summary>解答例</summary>

```
A: Attention Is All You Need
   → 2017年にGoogle Brainチームが発表した論文です。
     Transformerアーキテクチャを初めて提案しました。

B: Self-Attention（自己注意）
   → 入力系列の各位置が、他のすべての位置との関連性を計算する仕組みです。
     これにより「私は猫が好きです」の「好き」が「私」と「猫」の
     両方に関連することを学習できます。

C: トークン
   → テキストを処理する最小単位です。
     英語では単語単位に近いですが、日本語では文字やサブワード単位に
     分割されることが多いです。GPT-4oでは日本語1文字が約1〜3トークンです。

D: 事前学習（Pre-training）
   → インターネット上の大量テキストを使って、次のトークンの予測を
     繰り返し学習する段階です。汎用的な言語能力を獲得します。

E: ファインチューニング（Fine-tuning）
   → 事前学習済みモデルを特定のタスクや用途に合わせて
     追加で学習させる段階です。RLHF（人間のフィードバックによる強化学習）も
     この段階に含まれます。
```
</details>

### 問題4：生成AIの技術トレンド用語

以下の用語の説明として最も適切なものをA〜Fから選んでください。

**用語：**
1. マルチモーダル（Multimodal）
2. RAG（Retrieval-Augmented Generation）
3. エージェント（AI Agent）
4. ハルシネーション（Hallucination）
5. RLHF（Reinforcement Learning from Human Feedback）

**説明：**
- A: AIが自律的にツールを使い、複数のステップを組み合わせてタスクを遂行する仕組み
- B: 外部のデータベースや文書から情報を検索し、生成時の参考情報として活用する手法
- C: テキスト・画像・音声など複数の種類のデータを統合的に扱える能力
- D: 人間の評価・フィードバックを使ってモデルの出力品質を改善する学習手法
- E: モデルが事実に基づかない情報をもっともらしく生成してしまう現象

**期待される出力例：**
```
1. C（マルチモーダル → 複数のデータ種別を統合的に扱う能力）
2. B（RAG → 外部情報検索を活用した生成）
3. A（エージェント → 自律的なタスク遂行）
4. E（ハルシネーション → 事実に基づかない生成）
5. D（RLHF → 人間のフィードバックによる改善）
```

<details>
<summary>解答例</summary>

```
1. C: マルチモーダル
   → GPT-4o、Gemini等は画像入力にも対応しています。
     テキスト・画像・音声・動画など異なるモダリティ（情報の種類）を
     横断的に処理できるモデルが主流になりつつあります。

2. B: RAG（Retrieval-Augmented Generation）
   → 検索拡張生成と訳されます。社内文書や最新情報など、
     モデルが学習していないデータを外部から取得して
     プロンプトに含めることで、正確な回答を生成させます。

3. A: エージェント
   → ツール呼び出し（Function Calling）やコード実行を組み合わせ、
     複雑なタスクを自律的に遂行するAIシステムです。
     2025年以降、特に注目を集めている分野です。

4. E: ハルシネーション
   → 生成AIの根本的な課題の1つです。実在しない論文の引用、
     架空の統計データの提示など、もっともらしいが虚偽の情報を
     生成してしまいます。RAGやファクトチェックで軽減可能です。

5. D: RLHF
   → ChatGPTの成功に大きく貢献した技術です。
     人間が「良い回答」「悪い回答」を評価し、
     そのフィードバックを元にモデルを改善します。
```
</details>

### 問題5：ユースケースに適した生成AIサービスの選定

以下の業務シナリオそれぞれについて、最適な生成AIサービスの種類と、具体的なサービス名を1つ以上推薦してください。選定理由も述べてください。

**シナリオ一覧：**

| # | 業務内容 | 部門 |
|---|---------|------|
| 1 | 毎週の社内ニュースレターの草稿作成 | 広報部 |
| 2 | ECサイト用の商品画像バリエーション作成 | マーケティング部 |
| 3 | 社内システムのコードレビュー自動化 | 開発部 |
| 4 | 海外拠点とのミーティング音声の文字起こし・翻訳 | 海外事業部 |
| 5 | 営業資料のプレゼンスライド作成支援 | 営業部 |

**期待される出力例：**
```
シナリオ1: 社内ニュースレターの草稿作成
  種類: テキスト生成AI
  推薦: Claude / ChatGPT
  理由: 長文の構成力が求められ、社内情報を基にした文章生成に適している。
        Claudeは長文コンテキストに強く、ChatGPTは汎用性が高い。
...
```

<details>
<summary>ヒント</summary>

- 各シナリオで「何を入力し、何を出力するか」を整理すると、必要なAIの種類が明確になります。
- コスト・セキュリティ・導入の容易さも選定基準に含めましょう。
- 1つのシナリオに対して複数のサービスを組み合わせる場合もあります。
</details>

<details>
<summary>解答例</summary>

```
シナリオ1: 社内ニュースレターの草稿作成
  種類: テキスト生成AI
  推薦: Claude / ChatGPT
  理由: 長文の構成力が高く、トーンの指定も可能です。
        社内機密を含む場合はAPI経由での利用を推奨します。
        Claudeは長い文脈の処理が得意で、過去号を参考にした生成に向いています。

シナリオ2: ECサイト用の商品画像バリエーション作成
  種類: 画像生成AI
  推薦: Midjourney / DALL-E 3 / Stable Diffusion
  理由: 商品写真の背景差し替えやバリエーション生成に適しています。
        Midjourneyは品質が高く、Stable Diffusionはローカル実行で
        データ流出リスクを抑えられます。
        商用利用時はライセンスの確認が必須です。

シナリオ3: 社内システムのコードレビュー自動化
  種類: コード生成・解析AI
  推薦: GitHub Copilot / Claude Code
  理由: コードの品質チェック、セキュリティ脆弱性の検出、
        リファクタリング提案が可能です。
        GitHub CopilotはIDE統合が強力で、CI/CDパイプラインへの
        組み込みも容易です。

シナリオ4: ミーティング音声の文字起こし・翻訳
  種類: 音声認識AI + テキスト生成AI
  推薦: Whisper（文字起こし）+ Claude/ChatGPT（翻訳・要約）
  理由: 多言語対応が必要なため、2つのAIを組み合わせます。
        Whisperで音声をテキスト化した後、
        LLMで翻訳と要約を行うワークフローが効率的です。

シナリオ5: プレゼンスライド作成支援
  種類: テキスト生成AI + 画像生成AI
  推薦: ChatGPT（構成）+ Gamma / Beautiful.ai（スライド生成）
  理由: まずLLMでスライドの構成とテキストを生成し、
        スライド生成AIツールで視覚的なデザインを自動化します。
        図表やグラフはデータの正確性を人間が確認する必要があります。
```
</details>

### 問題6：生成AIサービスのコスト比較

以下の利用条件をもとに、各サービスの月間推定コストを計算してください。

**利用条件：**
- 1回のリクエスト：入力2,000トークン、出力1,000トークン
- 月間リクエスト回数：5,000回
- 1ドル = 150円で計算

**料金表（APIの1Kトークンあたりの価格）：**

| サービス | 入力（$/1Kトークン） | 出力（$/1Kトークン） |
|---------|--------------------:|--------------------:|
| GPT-4o | $0.0025 | $0.010 |
| GPT-4o mini | $0.000150 | $0.000600 |
| Claude 3.5 Sonnet | $0.003 | $0.015 |
| Gemini 1.5 Pro | $0.00125 | $0.005 |

**期待される出力例：**
```
GPT-4o:
  入力コスト: 2 × $0.0025 × 5,000 = $25.00
  出力コスト: 1 × $0.010 × 5,000 = $50.00
  月間合計: $75.00（約11,250円）
...
```

<details>
<summary>ヒント</summary>

- 入力トークン数は2,000 = 2K、出力トークン数は1,000 = 1Kとして計算します。
- 月間コスト = （入力単価 × 入力Kトークン + 出力単価 × 出力Kトークン）× リクエスト回数
</details>

<details>
<summary>解答例</summary>

```
【GPT-4o】
  入力コスト: 2 × $0.0025 × 5,000 = $25.00
  出力コスト: 1 × $0.010 × 5,000 = $50.00
  月間合計: $75.00（約11,250円）

【GPT-4o mini】
  入力コスト: 2 × $0.000150 × 5,000 = $1.50
  出力コスト: 1 × $0.000600 × 5,000 = $3.00
  月間合計: $4.50（約675円）

【Claude 3.5 Sonnet】
  入力コスト: 2 × $0.003 × 5,000 = $30.00
  出力コスト: 1 × $0.015 × 5,000 = $75.00
  月間合計: $105.00（約15,750円）

【Gemini 1.5 Pro】
  入力コスト: 2 × $0.00125 × 5,000 = $12.50
  出力コスト: 1 × $0.005 × 5,000 = $25.00
  月間合計: $37.50（約5,625円）

【コスト順位（安い順）】
  1位: GPT-4o mini   → $4.50（約675円）
  2位: Gemini 1.5 Pro → $37.50（約5,625円）
  3位: GPT-4o        → $75.00（約11,250円）
  4位: Claude 3.5 Sonnet → $105.00（約15,750円）

※ コストだけでなく、品質・速度・コンテキスト長も考慮して選定することが重要です。
  GPT-4o miniは最安ですが、複雑なタスクではGPT-4oやClaudeの方が
  高品質な出力を得られる場合があります。
```
</details>

### 問題7：生成AI導入時のリスク分析

ある企業が社内業務に生成AIを導入する際に考慮すべきリスクを、以下のカテゴリごとに2つずつ挙げてください。また、それぞれのリスクに対する軽減策も提案してください。

**カテゴリ：**
1. セキュリティ・プライバシー
2. 品質・信頼性
3. 法務・コンプライアンス
4. 組織・人材

**期待される出力例：**
```
1. セキュリティ・プライバシー
   リスクA: 機密データのAIサービスへの漏洩
   軽減策: APIの利用、データのオプトアウト設定、プライベートクラウド環境の活用
   ...
```

<details>
<summary>解答例</summary>

```
1. セキュリティ・プライバシー
   リスクA: 機密データの外部AIサービスへの漏洩
   軽減策: API経由での利用（学習データに使われないプランの選択）、
           DLP（Data Loss Prevention）の導入、入力データのマスキング

   リスクB: 個人情報を含むデータの不適切な処理
   軽減策: 個人情報の匿名化処理を事前に実施、
           利用ガイドラインで個人情報の入力を禁止、
           GDPR等のプライバシー規制への準拠確認

2. 品質・信頼性
   リスクA: ハルシネーションによる誤情報の社外発信
   軽減策: 人間によるレビュープロセスの必須化、
           RAGを用いた社内データとの照合、ファクトチェック体制の構築

   リスクB: モデルの出力品質が不安定
   軽減策: temperatureの適切な設定、プロンプトテンプレートの標準化、
           出力の定期的な品質モニタリング

3. 法務・コンプライアンス
   リスクA: 著作権侵害のリスク（生成物が既存作品に類似）
   軽減策: 生成コンテンツの著作権チェック体制、
           商用利用可能なモデル・サービスの選定、法務部門との連携

   リスクB: AI生成物の責任の所在が不明確
   軽減策: AI利用ポリシーの策定と社内周知、
           最終的な責任者を明確に定義、
           AI利用の記録を残す仕組みの導入

4. 組織・人材
   リスクA: 従業員のAIへの過度な依存によるスキル低下
   軽減策: AIはあくまで補助ツールとして位置づけ、
           定期的なスキル研修の実施、批判的思考力の維持を奨励

   リスクB: AI導入に対する組織内の抵抗
   軽減策: 段階的な導入と成功事例の共有、
           従業員への教育プログラムの提供、
           AI活用による業務改善の可視化
```
</details>

**提案書に含めるべき項目：**
