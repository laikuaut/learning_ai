# 第4章：アサーションとテスト構造

## この章のゴール

- Playwrightのアサーション（検証）を網羅的に使いこなす
- 自動リトライの仕組みを理解する
- `test.describe`、`test.beforeEach`、`test.afterEach` でテストを構造化できる
- テストのスキップ、フォーカス、タグ付けができる
- テストのタイムアウト設定を理解する

---

## 1. アサーションの基本

**アサーション（assertion）** は「この条件を満たしているはず」という検証です。条件を満たさない場合、テストが失敗します。

### 2種類のアサーション

```
┌───────────────────────────────────────────────────────────┐
│ 種類                │ 特徴                                 │
├───────────────────────────────────────────────────────────┤
│ Web アサーション    │ await expect(locator).toBeVisible()  │
│ （自動リトライ）    │ → 条件を満たすまで繰り返し確認       │
│                     │ → 非同期（await が必要）              │
├───────────────────────────────────────────────────────────┤
│ 汎用アサーション    │ expect(value).toBe(42)               ���
│ （即時判定）        │ → その場で1回だけ判定                │
│                     │ → 同期（await 不要）                  │
└───────────────────────────────────────────────────────────┘
```

---

## 2. Web アサーション（自動リトライ）

Web アサーションは、条件を満たすまでデフォルト5秒間リトライします。非同期なUIの変化を安定してテストできます。

### ページのアサーション

```typescript
// タイトルの検証
await expect(page).toHaveTitle('ホーム | MyApp');
await expect(page).toHaveTitle(/ホーム/);        // 正規表現

// URLの検証
await expect(page).toHaveURL('https://example.com/dashboard');
await expect(page).toHaveURL(/\/dashboard/);
```

### 要素の表示・非表示

```typescript
const button = page.getByRole('button', { name: '送信' });

// 表示されている
await expect(button).toBeVisible();

// 非表示（hidden、display:none など）
await expect(button).toBeHidden();

// DOMに存在する（表示/非表示は問わない）
await expect(button).toBeAttached();

// DOMから消えた
await expect(button).not.toBeAttached();
```

### テキストの検証

```typescript
const heading = page.getByRole('heading', { level: 1 });

// テキスト内容の検証
await expect(heading).toHaveText('ようこそ');
await expect(heading).toHaveText(/ようこそ/);       // 正規表現

// テキストを含む（部分一致）
await expect(heading).toContainText('ようこそ');

// 複数要素のテキストを一括検証
const items = page.getByRole('listitem');
await expect(items).toHaveText(['りんご', 'みかん', 'バナナ']);
```

### フォーム要素の検証

```typescript
// 入力値の検証
await expect(page.getByLabel('名前')).toHaveValue('田中太郎');
await expect(page.getByLabel('名前')).toHaveValue(/田中/);

// チェック状態
await expect(page.getByRole('checkbox')).toBeChecked();
await expect(page.getByRole('checkbox')).not.toBeChecked();

// 有効/無効
await expect(page.getByRole('button')).toBeEnabled();
await expect(page.getByRole('button')).toBeDisabled();

// フォーカス
await expect(page.getByLabel('名前')).toBeFocused();

// 編集可能
await expect(page.getByLabel('名前')).toBeEditable();
```

### CSS・属性の検証

```typescript
const element = page.locator('.status-badge');

// CSS クラスの検証
await expect(element).toHaveClass(/active/);
await expect(element).toHaveClass('badge active');  // 完全一致

// CSS プロパティの検証
await expect(element).toHaveCSS('color', 'rgb(255, 0, 0)');
await expect(element).toHaveCSS('display', 'flex');

// HTML 属性の検証
await expect(page.getByRole('link')).toHaveAttribute('href', '/about');
await expect(page.getByRole('img')).toHaveAttribute('alt', 'ロゴ');

// 要素数の検証
await expect(page.getByRole('listitem')).toHaveCount(5);
```

## 3. 汎用アサーション（即時判定）

JavaScriptの値を即座に判定するアサーションです。`await` は不要です。

```typescript
// 等値
expect(5).toBe(5);
expect('hello').toBe('hello');

// オブジェクトの比較（深い比較）
expect({ name: '太郎' }).toEqual({ name: '太郎' });

// 真偽値
expect(true).toBeTruthy();
expect(0).toBeFalsy();

// 数値
expect(10).toBeGreaterThan(5);
expect(10).toBeLessThanOrEqual(10);
expect(0.1 + 0.2).toBeCloseTo(0.3);  // 浮動小数点

// 文字列
expect('Playwright is great').toContain('Playwright');
expect('hello@example.com').toMatch(/^[\w.]+@[\w.]+$/);

// 配列
expect([1, 2, 3]).toContain(2);
expect([1, 2, 3]).toHaveLength(3);

// null / undefined
expect(null).toBeNull();
expect(undefined).toBeUndefined();
expect('value').toBeDefined();
```

## 4. 否定のアサーション（`.not`）

すべてのアサーションは `.not` で否定できます。

```typescript
// 表示されていないことを確認
await expect(page.getByText('エラー')).not.toBeVisible();

// 特定のテキストを含まないことを確認
await expect(page.getByRole('heading')).not.toHaveText('エラー');

// 汎用アサーション
expect(result).not.toBe(0);
expect(list).not.toHaveLength(0);
```

## 5. ソフトアサーション

通常のアサーションは失敗するとその場でテストが中断されます。**ソフトアサーション**は失敗してもテストを続行し、最後にまとめて結果を報告します。

```typescript
test('複数の要素を一度にチェック', async ({ page }) => {
  await page.goto('/profile');

  // ソフトアサーション — 失敗しても続行
  await expect.soft(page.getByText('名前: 太郎')).toBeVisible();
  await expect.soft(page.getByText('年齢: 25')).toBeVisible();
  await expect.soft(page.getByText('職業: エンジニア')).toBeVisible();
  await expect.soft(page.getByText('住所: 東京')).toBeVisible();

  // 上のどれかが失敗しても、ここまで実行される
  // テスト終了時に失敗がまとめて報告される
});
```

## 6. テストの構造化

### `test.describe` — テストのグループ化

```typescript
import { test, expect } from '@playwright/test';

test.describe('ユーザー登録', () => {
  test('正常に登録できる', async ({ page }) => {
    // ...
  });

  test('メールアドレスが不正だとエラー', async ({ page }) => {
    // ...
  });

  test('パスワードが短いとエラー', async ({ page }) => {
    // ...
  });
});
```

### `test.describe` のネスト

```typescript
test.describe('ユーザー管理', () => {
  test.describe('登録', () => {
    test('正常系', async ({ page }) => { /* ... */ });
    test('バリデーションエラー', async ({ page }) => { /* ... */ });
  });

  test.describe('ログイン', () => {
    test('正常系', async ({ page }) => { /* ... */ });
    test('パスワード間違い', async ({ page }) => { /* ... */ });
  });
});
```

### `test.beforeEach` / `test.afterEach` — 前後処理

```typescript
test.describe('ダッシュボード', () => {
  // 各テストの前に実行される
  test.beforeEach(async ({ page }) => {
    // ログイン処理
    await page.goto('/login');
    await page.getByLabel('メール').fill('user@example.com');
    await page.getByLabel('パスワード').fill('password123');
    await page.getByRole('button', { name: 'ログイン' }).click();
    await expect(page).toHaveURL('/dashboard');
  });

  // 各テストの後に実行される
  test.afterEach(async ({ page }) => {
    // ログアウト処理（必要に応じて）
  });

  test('ユーザー名が表示される', async ({ page }) => {
    await expect(page.getByText('user@example.com')).toBeVisible();
  });

  test('統計情報が表示される', async ({ page }) => {
    await expect(page.getByRole('heading', { name: '統計' })).toBeVisible();
  });
});
```

### `test.beforeAll` / `test.afterAll` — 全テスト共通の1回だけの処理

```typescript
test.describe('データベースを使うテスト', () => {
  test.beforeAll(async () => {
    // テストデータの準備（全テストで1回だけ）
    console.log('テストデータを作成');
  });

  test.afterAll(async () => {
    // テストデータのクリーンアップ
    console.log('テストデータを削除');
  });

  test('テスト1', async ({ page }) => { /* ... */ });
  test('テスト2', async ({ page }) => { /* ... */ });
});
```

```
実行順序:
  beforeAll     ← 1回だけ
  ├── beforeEach → テスト1 → afterEach
  └── beforeEach → テスト2 → afterEach
  afterAll      ← 1回だけ
```

## 7. テストの制御

### テストのスキップ

```typescript
// 特定のテストをスキップ
test.skip('未実装の機能', async ({ page }) => {
  // このテストは実行されない
});

// 条件付きスキップ
test('Windowsでのみ実行', async ({ page, browserName }) => {
  test.skip(browserName === 'webkit', 'WebKitでは未対応');
  // ...
});
```

### 特定のテストだけ実行（デバッグ時）

```typescript
// このテストだけ実行される（他はスキップ）
test.only('このテストだけ実行', async ({ page }) => {
  // ...
});
```

> **注意**: `test.only` をコミットしないように気をつけましょう。CI/CDで他のテストが実行されなくなります。

### テストのタグ付け

```typescript
// タグを付ける
test('ログインテスト @smoke', async ({ page }) => {
  // ...
});

test('複雑なワークフロー @slow', async ({ page }) => {
  // ...
});
```

```bash
# タグで絞り込んで実行
npx playwright test --grep @smoke
npx playwright test --grep-invert @slow  # @slow以外を実行
```

## 8. タイムアウト設定

### テスト全体のタイムアウト

```typescript
// playwright.config.ts で設定
export default defineConfig({
  timeout: 30000,  // 30秒（デフォルト）
});

// 個別のテストで変更
test('長時間かかるテスト', async ({ page }) => {
  test.setTimeout(60000);  // 60秒に延長
  // ...
});
```

### アサーションのタイムアウト

```typescript
// デフォルト: 5秒
// 設定で変更
export default defineConfig({
  expect: {
    timeout: 10000,  // 10秒
  },
});

// 個別のアサーションで変更
await expect(page.getByText('処理完了')).toBeVisible({ timeout: 15000 });
```

## まとめ

| 項目 | ポイント |
|------|---------|
| Web アサーション | `await expect(locator)` — 自動リトライ付き |
| 汎用アサーション | `expect(value)` — 即時判定 |
| `.not` | すべてのアサーションを否定できる |
| ソフトアサーション | `expect.soft()` — 失敗してもテスト続行 |
| `test.describe` | テストをグループ化 |
| `beforeEach/afterEach` | 各テストの前後に実行 |
| `beforeAll/afterAll` | 全テストで1回だけ実行 |
| `test.skip` | テストをスキップ |
| `test.only` | 特定テストだけ実行（デバッグ用） |
| タグ | `@smoke` のようにタグ付けして絞り込み実行 |
