# 実践課題06：フック設定による自動化 ★3

> **難易度**: ★★★☆☆（中級）
> **前提知識**: 第1章〜第5章（基本操作、ファイル操作、Git連携、MCP）、第6章（フックとカスタマイズ）
> **課題の種類**: ミニプロジェクト
> **学習目標**: Claude Codeのフックシステム（PreToolUse、PostToolUse、Notification、Stop）を理解し、ファイル編集時の自動リントやコミット前チェックなどの自動化を実装する

---

## 完成イメージ

フックを設定することで、Claude Codeがファイルを編集するたびに自動的にリンター（linter）が実行される環境を構築します。

```
設定後の動作フロー:

  1. Claude Codeにコード修正を依頼
  2. Claude CodeがEditツールでファイルを編集
  3. PostToolUseフックが自動発火 ← フック設定！
  4. リンターが自動実行
  5. エラーがあればClaude Codeにフィードバック
  6. Claude Codeが自動で再修正
```

settings.jsonのフック設定:
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "command": "cd $CLAUDE_PROJECT_DIR && npx eslint --fix $CLAUDE_FILE_PATH 2>&1 || true"
      }
    ]
  }
}
```

---

## 課題の要件

1. JavaScript/TypeScriptプロジェクトにESLintを設定する
2. `PostToolUse` フックでファイル編集後にESLintを自動実行する設定を書く
3. `PreToolUse` フックで危険なBashコマンドをブロックする設定を書く
4. フックマッチャー（`matcher`）のパターン指定を理解する
5. フックで利用できる環境変数（`CLAUDE_PROJECT_DIR`、`CLAUDE_FILE_PATH`等）を確認する
6. フックの動作を実際に確認する

<details>
<summary>ステップ2：PostToolUseフックの設定</summary>

ファイル編集後にESLintを自動実行するフックを設定します。

```bash
# .claude/settings.jsonを作成
mkdir -p .claude
cat > .claude/settings.json << 'JSONEOF'
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "command": "if echo \"$CLAUDE_FILE_PATH\" | grep -qE '\\.(js|ts|jsx|tsx)$'; then cd \"$CLAUDE_PROJECT_DIR\" && npx eslint \"$CLAUDE_FILE_PATH\" 2>&1 || true; fi"
      }
    ]
  }
}
JSONEOF
```

この設定のポイント:

| 要素 | 説明 |
|---|---|
| `PostToolUse` | ツール実行後に発火するフックタイプ |
| `matcher: "Edit\|Write"` | EditまたはWriteツール使用時にマッチ |
| `CLAUDE_FILE_PATH` | 操作対象のファイルパスが入る環境変数 |
| `CLAUDE_PROJECT_DIR` | プロジェクトルートのパスが入る環境変数 |
| `\|\| true` | リンターエラーでフック自体が失敗しないようにする |

</details>

<details>
<summary>ステップ3：PreToolUseフックで危険なコマンドをブロック</summary>

`rm -rf` のような危険なコマンドの実行を防ぐフックを追加します。

```bash
# settings.jsonを更新
cat > .claude/settings.json << 'JSONEOF'
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "if echo \"$CLAUDE_TOOL_INPUT\" | grep -qE 'rm\\s+-rf\\s+/|sudo\\s+rm|:(){ :|:& };:'; then echo 'BLOCKED: 危険なコマンドを検出しました' >&2; exit 2; fi"
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "command": "if echo \"$CLAUDE_FILE_PATH\" | grep -qE '\\.(js|ts|jsx|tsx)$'; then cd \"$CLAUDE_PROJECT_DIR\" && npx eslint \"$CLAUDE_FILE_PATH\" 2>&1 || true; fi"
      }
    ]
  }
}
JSONEOF
```

フックの終了コード（exit code）の意味:

| 終了コード | 動作 |
|---|---|
| `0` | 成功 ─ ツール実行を許可（PostToolUseでは結果をフィードバック） |
| `2` | ブロック ─ ツール実行を拒否（PreToolUseのみ） |
| その他 | エラー ─ フック自体のエラーとして報告 |

</details>

<details>
<summary>ステップ5：利用可能な環境変数の確認</summary>

フックコマンド内で使える環境変数の一覧です。

| 環境変数 | 利用可能なフック | 説明 |
|---|---|---|
| `CLAUDE_PROJECT_DIR` | 全フック | プロジェクトのルートディレクトリ |
| `CLAUDE_FILE_PATH` | PostToolUse | 操作対象のファイルパス |
| `CLAUDE_TOOL_NAME` | PreToolUse, PostToolUse | 実行されるツール名 |
| `CLAUDE_TOOL_INPUT` | PreToolUse | ツールへの入力内容（JSON） |

確認方法:
```bash
# 環境変数を出力するフックを一時的に設定
cat > .claude/settings.json << 'JSONEOF'
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "command": "echo \"PROJECT: $CLAUDE_PROJECT_DIR\" && echo \"FILE: $CLAUDE_FILE_PATH\" && echo \"TOOL: $CLAUDE_TOOL_NAME\""
      }
    ]
  }
}
JSONEOF
```

<details>
<summary>解答例（改良版）── 複数フックの実用的な組み合わせ</summary>

実務では複数のフックを組み合わせて使います。

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "if echo \"$CLAUDE_TOOL_INPUT\" | grep -qE 'rm\\s+-rf\\s+/|sudo\\s+rm|DROP\\s+TABLE|TRUNCATE'; then echo 'BLOCKED: 危険なコマンドを検出' >&2; exit 2; fi"
      },
      {
        "matcher": "Write",
        "command": "if echo \"$CLAUDE_FILE_PATH\" | grep -qE '\\.(env|pem|key)$'; then echo 'BLOCKED: 機密ファイルの作成は禁止されています' >&2; exit 2; fi"
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "command": "if echo \"$CLAUDE_FILE_PATH\" | grep -qE '\\.(js|ts|jsx|tsx)$'; then cd \"$CLAUDE_PROJECT_DIR\" && npx eslint --fix \"$CLAUDE_FILE_PATH\" 2>&1 || true; fi"
      },
      {
        "matcher": "Edit|Write",
        "command": "if echo \"$CLAUDE_FILE_PATH\" | grep -qE '\\.(py)$'; then cd \"$CLAUDE_PROJECT_DIR\" && python -m py_compile \"$CLAUDE_FILE_PATH\" 2>&1 || true; fi"
      }
    ],
    "Notification": [
      {
        "matcher": "",
        "command": "echo \"$(date '+%H:%M:%S') - Claude Code通知\" >> /tmp/claude-notifications.log"
      }
    ]
  }
}
```

**初心者向けとの違い:**
- 初心者向けは1つのフックだけで基本を体験します
- 改良版は複数のフックタイプを組み合わせ、セキュリティ・コード品質・通知を網羅しています
- `--fix`オプションを使うと、リンターが自動修正可能なエラーはその場で直してくれます
- 言語ごとに異なるツール（ESLint、py_compile等）を使い分けるパターンは実務でよく使います

</details>
