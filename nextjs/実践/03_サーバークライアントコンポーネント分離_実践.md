# 実践課題03：サーバー/クライアントコンポーネント分離 ★2

> **難易度**: ★★☆☆☆（基礎）
> **前提知識**: 第1章（Next.jsの基本）、第3章（サーバーコンポーネント）
> **課題の種類**: ミニプロジェクト
> **学習目標**: サーバーコンポーネントとクライアントコンポーネントの使い分け、`'use client'` ディレクティブの適切な配置を実践する

---

## 完成イメージ

商品一覧ページを構築します。商品データの取得はサーバーコンポーネントで行い、お気に入りボタンやカート追加ボタンなどのインタラクティブな部分はクライアントコンポーネントで実装します。

```
┌──────────────────────────────────────────┐
│  🛍️ 商品一覧           カート: 2個       │
├──────────────────────────────────────────┤
│                                          │
│  ┌─────────────┐  ┌─────────────┐       │
│  │  商品画像     │  │  商品画像     │       │
│  │  ワイヤレス    │  │  USBハブ     │       │
│  │  イヤホン     │  │              │       │
│  │  ¥4,980      │  │  ¥2,480      │       │
│  │ [♥] [カート] │  │ [♡] [カート] │       │
│  └─────────────┘  └─────────────┘       │
│                                          │
│  サーバー: 商品データ取得（RSC）           │
│  クライアント: ♥ ボタン・カートボタン      │
└──────────────────────────────────────────┘
```

---

## 課題の要件

1. 商品一覧ページ（`app/products/page.tsx`）を**サーバーコンポーネント**として作成する
2. 商品データはサーバーコンポーネント内で直接取得する（`fetch` または配列）
3. お気に入りボタン（`FavoriteButton`）を**クライアントコンポーネント**として作成する
4. カートに追加する機能を**クライアントコンポーネント**として作成する
5. サーバーコンポーネントの中にクライアントコンポーネントをネストする構成にする

---

<details>
<summary>ステップ1：コンポーネントの設計を考える</summary>

どのコンポーネントをサーバー・クライアントにするか設計します。

```
app/products/page.tsx         ← サーバーコンポーネント（データ取得）
  └── ProductCard             ← サーバーコンポーネント（表示のみ）
        ├── FavoriteButton    ← クライアント（useState を使う）
        └── AddToCartButton   ← クライアント（onClick を使う）
```

**判断基準:**
- `useState`、`useEffect`、`onClick` を使う → クライアントコンポーネント
- データ取得のみ、静的な表示のみ → サーバーコンポーネント

import { useState } from 'react'

export default function FavoriteButton({ productName }: { productName: string }) {
  const [isFavorite, setIsFavorite] = useState(false)

  function handleClick() {
    setIsFavorite(!isFavorite)
  }

  return (
    <button
      onClick={handleClick}
      style={{
        border: 'none',
        background: 'none',
        fontSize: '20px',
        cursor: 'pointer',
        padding: '4px 8px',
      }}
      aria-label={
        isFavorite
          ? `${productName}をお気に入りから削除`

      }
    >
      {isFavorite ? '♥' : '♡'}
    </button>
  )
}

### app/products/CartContext.tsx

```tsx
'use client'

import { createContext, useContext, useState, ReactNode } from 'react'

type CartItem = {
  id: number
  name: string
  price: number
  quantity: number
}

type CartContextType = {
  items: CartItem[]
  addItem: (id: number, name: string, price: number) => void
  totalCount: number
  totalPrice: number
}

const CartContext = createContext<CartContextType | null>(null)

export function CartProvider({ children }: { children: ReactNode }) {
  const [items, setItems] = useState<CartItem[]>([])

  function addItem(id: number, name: string, price: number) {
    setItems((prev) => {
      const existing = prev.find((item) => item.id === id)
      if (existing) {
        return prev.map((item) =>
          item.id === id ? { ...item, quantity: item.quantity + 1 } : item
        )
      }
      return [...prev, { id, name, price, quantity: 1 }]
    })
  }

  const totalCount = items.reduce((sum, item) => sum + item.quantity, 0)
  const totalPrice = items.reduce((sum, item) => sum + item.price * item.quantity, 0)

  return (
    <CartContext.Provider value={{ items, addItem, totalCount, totalPrice }}>
      {children}
    </CartContext.Provider>
  )
}

export function useCart() {
  const context = useContext(CartContext)
  if (!context) {
    throw new Error('useCart は CartProvider の中で使用してください')
  }
  return context
}
```

**初心者向けとの違い:**
- `CartContext` でカート状態をアプリ全体で共有

- 重複追加時に数量をインクリメントする仕組み
